# Генератор студијског плана са Chainlit & Microsoft Learn Docs MCP

## Захтеви

- Python 3.8 или новији
- pip (менаџер пакета за Python)
- Интернет конекција за повезивање на Microsoft Learn Docs MCP сервер

## Инсталација

1. Клонирајте овај репозиторијум или преузмите фајлове пројекта.
2. Инсталирајте потребне зависности:

   ```bash
   pip install -r requirements.txt
   ```

## Коришћење

### Сценарио 1: Једноставан упит ка Docs MCP
Командна линија клијент који се повезује на Docs MCP сервер, шаље упит и исписује резултат.

1. Покрените скрипту:
   ```bash
   python scenario1.py
   ```
2. Укуцајте своје питање о документацији на промпт.

### Сценарио 2: Генератор студијског плана (Chainlit веб апликација)
Веб интерфејс (користећи Chainlit) који омогућава корисницима да генеришу персонализовани студијски план по недељама за било коју техничку тему.

1. Покрените Chainlit апликацију:
   ```bash
   chainlit run scenario2.py
   ```
2. Отворите локални URL који вам је приказан у терминалу (нпр. http://localhost:8000) у вашем прегледачу.
3. У прозору за разговор унесите своју тему студија и број недеља које желите да учите (нпр. "AI-900 сертификација, 8 недеља").
4. Апликација ће одговорити планом студирања по недељама, укључујући линкове до релевантне Microsoft Learn документације.

**Потребне променљиве окружења:**

Да бисте користили Сценарио 2 (Chainlit веб апликацију са Azure OpenAI), морате поставити следеће променљиве окружења у `.env` фајл у `python` директоријуму:

```
AZURE_OPENAI_CHAT_DEPLOYMENT_NAME=
AZURE_OPENAI_API_KEY=
AZURE_OPENAI_ENDPOINT=
AZURE_OPENAI_API_VERSION=
```

Унесите своје податке о Azure OpenAI ресурсу пре покретања апликације.

> [!TIP]
> Лако можете деплојовати своје моделе користећи [Microsoft Foundry](https://ai.azure.com/).

### Сценарио 3: Документација у едитору са MCP сервером у VS Code

Уместо да мењате табове прегледача да бисте претраживали документацију, можете довести Microsoft Learn Docs директно у VS Code користећи MCP сервер. Ово вам омогућава да:
- Претражујете и читате документацију унутар VS Code без напуштања окружења за кодирање.
- Референцирате документацију и убацујете линкове директно у README или курсне фајлове.
- Користите GitHub Copilot и MCP заједно за беспрекорну AI-подржану радну средину за документацију.

**Примери употребе:**
- Брзо додајте референце у README док пишете документацију курса или пројекта.
- Користите Copilot за генерисање кода, а MCP за тренутно проналажење и цитирање релевантне документације.
- Останите фокусирани у едитору и повећајте продуктивност.

> [!IMPORTANT]
> Проверите да ли имате важећу [`mcp.json`](../../../../../../09-CaseStudy/docs-mcp/solution/scenario3/mcp.json) конфигурацију у вашем радном простору (локација је `.vscode/mcp.json`).

## Зашто Chainlit за Сценарио 2?

Chainlit је модеран open-source фрејмворк за прављење конверзацијских веб апликација. Олакшава креирање чат базираних корисничких интерфејса који се повезују са бекенд сервисима као што је Microsoft Learn Docs MCP сервер. Овај пројекат користи Chainlit како би обезбедио једноставан, интерактиван начин за генерисање персонализованих студијских планова у реалном времену. Помоћу Chainlit-а можете брзо израдити и имплементирати алате засноване на разговору који побољшавају продуктивност и учење.

## Шта ово ради

Ова апликација омогућава корисницима да креирају персонализовани студијски план тако што једноставно унесу тему и временски период. Апликација анализира унос, шаље упит Microsoft Learn Docs MCP серверу за релевантни садржај и организује резултате у структуриран, недељни план. Препоруке за сваку недељу се приказују у ћаскању, што олакшава праћење напретка. Интеграција гарантује да увек добијате најновије и најрелевантније ресурсе за учење.

## Примери упита

Покушајте ове упите у прозору за разговор да видите како апликација реагује:

- `AI-900 сертификација, 8 недеља`
- `Learn Azure Functions, 4 недеље`
- `Azure DevOps, 6 недеља`
- `Data engineering on Azure, 10 недеља`
- `Microsoft security fundamentals, 5 недеља`
- `Power Platform, 7 недеља`
- `Azure AI services, 12 недеља`
- `Cloud architecture, 9 недеља`

Ови примери илуструју флексибилност апликације за различите циљеве учења и временске оквире.
