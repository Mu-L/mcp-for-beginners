# Сценарио 3: Документација у уређивачу са MCP сервером у VS Code

## Преглед

У овом сценарију ћете научити како да унесете Microsoft Learn документацију директно у своје Visual Studio Code окружење користећи MCP сервер. Уместо да стално мењате картице прегледача да бисте тражили документацију, можете приступити, претраживати и позивати званичну документацију право из свог уређивача. Оваквим приступом поједностављујете свој радни ток, задржавате фокус и омогућавате беспрекорну интеграцију са алатима као што је GitHub Copilot.

- Претражујте и читајте документацију унутар VS Code без напуштања вашег окружења за кодирање.
- Позивајте документацију и убацујте линкове директно у ваш README или датотеке курса.
- Користите GitHub Copilot и MCP заједно за беспрекоран радни ток документације са подршком вештачке интелигенције.

## Циљеви учења

До краја овог поглавља разумете како да конфигуришете и користите MCP сервер унутар VS Code да бисте унапредили свој радни ток документације и развоја. Бићете у могућности да:

- Конфигуришете своје радно окружење да користите MCP сервер за претрагу документације.
- Претражујете и убацујете документацију директно из VS Code.
- Комбинујете моћ GitHub Copilot и MCP-а за продуктивнији, AI-подржан радни ток.

Ове вештине ће вам помоћи да останете фокусирани, побољшате квалитет документације и повећате продуктивност као програмер или технички писац.

## Решење

Да бисте омогућили приступ документацији у уређивачу, пратићете низ корака који интегришу MCP сервер са VS Code и GitHub Copilot. Ово решење је идеално за ауторе курсева, писце документације и програмере који желе да задрже фокус у уређивачу док раде са документацијом и Copilot-ом.

- Брзо додајте референтне линкове у README док пишете курс или документацију пројекта.
- Користите Copilot за генерисање кода и MCP за тренутно проналажење и цитирање релевантне документације.
- Задржите фокус у уређивачу и повећајте продуктивност.

### Водич корак по корак

Да бисте почели, пратите ове кораке. За сваки корак можете додати снимак екрана из фасцикле assets да визуелно представите процес.

1. **Додајте MCP конфигурацију:**
   У корену пројекта направите `.vscode/mcp.json` датотеку и додајте следећу конфигурацију:
   ```json
   {
     "servers": {
       "LearnDocsMCP": {
         "url": "https://learn.microsoft.com/api/mcp"
       }
     }
   }
   ```
   Ова конфигурација говори VS Code-у како да се повеже са [`Microsoft Learn Docs MCP сервером`](https://github.com/MicrosoftDocs/mcp).
   
   ![Step 1: Add mcp.json to .vscode folder](../../../../../../translated_images/sr/step1-mcp-json.c06a007fccc3edfa.webp)
    
2. **Отворите GitHub Copilot Chat панел:**
   Уколико још немате инсталиран GitHub Copilot екстензију, идите у приказ Extensions у VS Code-у и инсталирајте је. Можете је директно преузети са [Visual Studio Code Marketplace](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot-chat). Затим, отворите Copilot Chat панел са бочне траке.

   ![Step 2: Open Copilot Chat panel](../../../../../../translated_images/sr/step2-copilot-panel.f1cc86e9b9b8cd1a.webp)

3. **Омогућите agent режим и проверите алате:**
   У Copilot Chat панелу омогућите agent режим.

   ![Step 3: Enable agent mode and verify tools](../../../../../../translated_images/sr/step3-agent-mode.cdc32520fd7dd1d1.webp)

   Након укључивања agent режима, проверите да ли је MCP сервер наведен као један од расположивих алата. Ово осигурава да Copilot agent може приступити серверу документације ради преузимања релевантних информација.
   
   ![Step 3: Verify MCP server tool](../../../../../../translated_images/sr/step3-verify-mcp-tool.76096a6329cbfecd.webp)
4. **Започните нови чет и упутите агента:**
   Отворите нови чет у Copilot Chat панелу. Сада можете да упућујете упите агенту у вези са документацијом. Агент ће користити MCP сервер да преузме и прикаже релевантну Microsoft Learn документацију директно у вашем уређивачу.

   - *"Покушавам да направим план учења за тему X. Планирам да је учим 8 недеља, за сваку недељу предложи ми садржај који треба да обрадим."*

   ![Step 4: Prompt the agent in chat](../../../../../../translated_images/sr/step4-prompt-chat.12187bb001605efc.webp)

5. **Живи упит:**

   > Узмимо живи упит из [#get-help](https://discord.gg/D6cRhjHWSC) секције у Microsoft Foundry Discord-у ([погледај оригиналну поруку](https://discord.com/channels/1113626258182504448/1385498306720829572)):
   
   *"Тражим одговоре како да имплементирам мулти-агентско решење са AI агентима развијеним у Azure AI Foundry. Видим да нема директног начина за имплементацију као што су Copilot Studio канали. Који су различити начини да се овај имплементација обави за корпоративне кориснике да се лакше сарађује и заврши посао?
Постоје бројни чланци/блогови који кажу да можемо користити Azure Bot сервис за овај посао који би могао деловати као мост између MS Teams и Azure AI Foundry агената, да ли ће ово функционисати ако подесим Azure бота који се повезује са Orchestrator агентом на Azure AI Foundry преко Azure функције за извођење оркестрације или је потребно креирати Azure функцију за сваког AI агента који је део мулти-агентског решења да се изврши оркестрација на Bot framework-у? Све друге предлоге радо прихватам.
"*

   ![Step 5: Live queries](../../../../../../translated_images/sr/step5-live-queries.49db3e4a50bea273.webp)

   Агент ће одговорити са релевантним линковима на документацију и резимеима које затим можете директно уметнути у своје markdown датотеке или користити као референце у свом коду.
   
### Примери упита

Ево неколико примера упита које можете испробати. Ови упити ће показати како MCP сервер и Copilot могу заједно да раде како би пружили тренутну документацију са свешћу о контексту и референцама без напуштања VS Code-а:

- "Покажи ми како се користе Azure Functions тригери."
- "Убаци линк ка званичној документацији за Azure Key Vault."
- "Које су најбоље праксе за обезбеђење Azure ресурса?"
- "Пронађи брзи почетак за Azure AI сервисе."

Ови упити ће показати како MCP сервер и Copilot могу заједно да раде како би пружили тренутну документацију са свешћу о контексту и референцама без напуштања VS Code-а.
