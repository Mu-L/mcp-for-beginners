## 🔧 Разумевање MCP сервера: Водич за почетнике

### Шта су MCP сервери?

Као почетник у Model Context Protocol-у (MCP), можда се питате: "Шта је тачно MCP сервер и зашто је важно?" Почнимо са једним једноставним примером.

Размислите о MCP серверима као о специјализованим помоћницима који омогућавају вашем AI суоператору у кодирању (попут GitHub Copilot-а) да се повеже са спољним алатима и услугама. Баш као што користите различите апликације на вашем телефону за различите задатке — једну за време, једну за навигацију, једну за банкарство — MCP сервери дају вашем AI помоћнику способност да интерагује са различитим развојним алатима и услугама.

### Проблем који MCP сервери решавају

Пре MCP сервера, ако сте хтели да:
- Проверите своје Azure ресурсе
- Креирате GitHub issue
- Упитате базу података
- Претражите документацију

Морате да прекинете са кодирањем, отворите прегледач, одете на одговарајући сајт и ручно извршите те радње. Овај непрекидни прелазак контекста ремети ваш рад и смањује продуктивност.

### Како MCP сервери трансформишу ваш развојни процес

Са MCP серверима можете остати у свом развојном окружењу (VS Code, Visual Studio и друго) и једноставно затражити од AI помоћника да обради те задатке. На пример:

**Уместо овог традиционалног рада:**
1. Прекини са кодирањем
2. Отвори прегледач
3. Иди на Azure портал
4. Претражи детаље налога за складиште
5. Врати се у VS Code
6. Настави са кодирањем

**Сада можете овако:**
1. Питај AI: "Који је статус мојих Azure складишних рачуна?"
2. Настави да кодираш са добијеним информацијама

## Почетак рада са MCP у VS Code и Visual Studio 🚀

Подешавање ових MCP сервера је једноставно ако користите Visual Studio Code или Visual Studio 2022 са GitHub Copilot-ом.

### Подешавање у VS Code-у

Основни процес у VS Code-у је:

1. **Омогући Agent мод**: У VS Code-у пребаци се на Agent режим у прозору Copilot Chat
2. **Конфигуриши MCP сервере**: Додај конфигурације сервера у свој settings.json фајл
3. **Покрени сервере**: Кликни на дугме "Start" за сваки сервер који желиш да користиш
4. **Изабери алате**: Одабери које MCP сервере ћеш омогућити у овој сесији

За детаљна упутства, погледајте [документацију за MCP у VS Code-у](https://code.visualstudio.com/docs/copilot/copilot-mcp).

> **💡 Про савет: Управљај MCP серверима као стручњак!**
> 
> Приказ екстензија у VS Code-у сада укључује [погодан нови интерфејс за управљање инсталираним MCP серверима](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode)! Имате брз приступ покретању, заустављању и управљању било којим MCP сервером уз јасан и једноставан интерфејс. Испробајте!

### Подешавање у Visual Studio 2022

За Visual Studio 2022 (верзија 17.14 или новија):

1. **Омогући Agent мод**: Кликните на падајући мени „Ask“ у прозору GitHub Copilot Chat-а и изаберите „Agent“
2. **Креирајте конфигурациони фајл**: Направите `.mcp.json` фајл у вашем фолдеру решења (рекомењовано место: `<SOLUTIONDIR>\.mcp.json`)
3. **Конфигуришите сервере**: Додајте конфигурације MCP сервера користећи стандардни MCP формат
4. **Одобрење алата**: Када будете упитани, одобријте алате које желите да користите са одговарајућим дозволама опсега

За детаљна упутства за Visual Studio погледајте [Visual Studio MCP документацију](https://learn.microsoft.com/visualstudio/ide/mcp-servers).

Сваки MCP сервер има своје захтеве за конфигурацију (конекцијске низове, аутентификацију итд.), али образац подешавања је доследан у оба IDE-а.

### 1. 📚 Microsoft Learn Docs MCP сервер

[![Инсталирај у VS Code](https://img.shields.io/badge/VS_Code-Install_Microsoft_Docs_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D) [![Инсталирај у VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Microsoft_Docs_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/mcp)

**Шта ради**: Microsoft Learn Docs MCP Server је сервис у облаку који обезбеђује AI помоћницима приступ званичној Microsoft документацији у реалном времену преко Model Context Protocol-а. Повезује се на `https://learn.microsoft.com/api/mcp` и омогућава семантичку претрагу кроз Microsoft Learn, Azure документацију, Microsoft 365 документацију и друге званичне Microsoft изворе.

**Зашто је користан**: Иако изгледа као „само документација“, овај сервер је кључан за сваког програмера који користи Microsoft технологије. Једна од највећих критика програмера .NET-а према AI помоћницима у кодирању је да нису ажурни по питању најновијих .NET и C# верзија. Microsoft Learn Docs MCP Server ово решава тако што обезбеђује приступ најновијој документацији, API референцама и најбољим праксама у реалном времену. Без обзира да ли радите са најновијим Azure SDK-овима, истражујете нове C# 13 функције или имплементирате најновије .NET Aspire паттерне, овај сервер омогућава да ваш AI помоћник има приступ ауторитетним, ажурираним информацијама које су му потребне за генерисање прецизног, модерног кода.

**Стварна употреба**: „Које су az cli команде за креирање Azure container апликације према званичној Microsoft Learn документацији?“ или „Како да конфигуришем Entity Framework са dependency injection у ASP.NET Core?“ Или, како вам звучи „Прегледај овај код да провериш да ли је у складу са препорукама за перформансе из Microsoft Learn документације.“ Сервер пружа свеобухватну покривеност Microsoft Learn-а, Azure документације и Microsoft 365 документације користећи напредну семантичку претрагу која налази најрелевантније информације у контексту. Враћа до 10 висококвалитетних садржајних делова са насловима чланака и URL адресама, увек приступајући најновијој Microsoft документацији одмах по објављивању.

**Илустративни пример**: Сервер излаже алат `microsoft_docs_search` који обавља семантичку претрагу по званичној техничкој документацији Microsoft-а. Када је конфигурисан, можете питати питања као што су „Како имплементирати JWT аутентификацију у ASP.NET Core?“ и добити детаљне, званичне одговоре са референцама на изворе. Квалитет претраге је изузетан јер разуме контекст – питање о „контейнерима“ у Azure контексту враћа документацију за Azure Container Instances, док исти термин у .NET контексту враћа релевантне информације о C# колекцијама.

Ово је посебно корисно за брзо променљиве или недавно ажуриране библиотеке и сценарије употребе. На пример, у неким новијим пројектима кода желео сам да искористим могућности у најновијим издањима .NET Aspire и Microsoft.Extensions.AI. Укључивањем Microsoft Learn Docs MCP сервера успео сам да искористим не само API документацију, већ и туторијале и смернице које су тек објављене.

### 2. ☁️ Azure MCP сервер
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

**Шта ради**: Azure MCP Server је свеобухватан сет од преко 15 специјализованих конектора Azure услуга који уноси целокупни Azure екосистем у ваш AI радни ток. Ово није само један сервер – то је моћна колекција која укључује управљање ресурсима, повезивање са базама података (PostgreSQL, SQL Server), анализу Azure Monitor логова са KQL, интеграцију Cosmos DB и још много тога.

**Зашто је користан**: Поред управљања Azure ресурсима, овај сервер драматично побољшава квалитет кода при раду са Azure SDK-овима. Када користите Azure MCP у Agent моду, он вам не само помаже да пишете код – већ вам помаже да пишете *бољи* Azure код који прати тренутне обрасце аутентикације, најбоље праксе за руковање грешкама и користи најновије функције SDK-а. Уместо да добијете генерички код који можда ради, добијате код који прати препоручене Azure обрасце за продукционе радне токове.

**Кључни модули укључују**:
- **🗄️ Конектори за базе података**: Директан приступ на природном језику Azure Database за PostgreSQL и SQL Server
- **📊 Azure Monitor**: Анализа логова покретана KQL-ом и оперативне информације
- **🌐 Управљање ресурсима**: Комплетан циклус живота ресурса у Azure-у
- **🔐 Аутентикација**: Обрасци DefaultAzureCredential и managed identity
- **📦 Сервис складиштења**: Операције Blob Storage, Queue Storage и Table Storage
- **🚀 Сервис контејнера**: Управљање Azure Container Apps, Container Instances и AKS-ом
- **И још многи специјализовани конектори**

**Примена у стварном свету**: "Листај моје Azure складишне налоге", "Питати мој Log Analytics workspace за грешке у последњем сату" или "Помози ми да направим Azure апликацију користећи Node.js са исправном аутентикацијом"

**Пуни демонстрациони сценарио**: Ево комплетног водича који показује снагу комбиновaња Azure MCP са GitHub Copilot for Azure екстензијом у VS Code. Када имате оба инсталирана и упутите захтев:

> "Креирај Python скрипту која отпрема фајл у Azure Blob Storage користећи DefaultAzureCredential аутентикацију. Скрипта треба да се повеже са мојим Azure складишним налогом под именом 'mycompanystorage', отпреми у контејнер под именом 'documents', направи тест фајл са тренутним временским печатом за отпремање, лепо обради грешке и пружи информативни излаз, прати најбоље Azure праксе за аутентикацију и руковање грешкама, укључи коментаре који објашњавају како DefaultAzureCredential аутентикација функционише и направи скрипту добро структуираном са одговарајућим функцијама и документацијом."

Azure MCP Server ће генерисати комплетну, продукцијски спремну Python скрипту која:
- Користи најновији Azure Blob Storage SDK са исправним async обрасцима
- Имплементира DefaultAzureCredential са свеобухватним објашњењем fallback ланца
- Укључује робусно руковање грешкама са специфичним типовима Azure изузетака
- Прати најбоље праксе Azure SDK-а за управљање ресурсима и руковање конекцијама
- Пружа детаљно логовање и информативни излаз на конзоли
- Креира правилно структуирану скрипту са функцијама, документацијом и типским наговештајима

Оно што је значајно јесте да без Azure MCP можда добијете генерички blob storage код који ради али не прати тренутне Azure обрасце. Са Azure MCP добијате код који користи најновије методе аутентикације, обрађује Azure-специфичне сценарије грешака и прати Microsoft-ове препоруке за продукционе апликације.

**Илустративни пример**: Мучио сам се да запамтим специфичне команде `az` и `azd` CLI-ја за повремени рад. Увек ми је то двостепени процес: прво потражим синтаксу, затим покренем команду. Често једноставно одем на портал и кликнем по потреби јер не желим да признам да не могу да се сетим CLI синтаксе. Могућност да једноставно опишем шта желим је невероватна, а још боље је то што могу да то радим без напуштања IDE-а!

Постоји одлична листа употребних случајева у [Azure MCP репозиторијуму](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) за почетак. За детаљне водиче за подешавање и напредне опције конфигурације, погледајте [званичну Azure MCP документацију](https://learn.microsoft.com/azure/developer/azure-mcp-server/).

### 5. 📝 MarkItDown MCP Server
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_MarkItDown_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_MarkItDown_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/markitdown)

**Шта ради**: MarkItDown је свеобухватан сервер за конверзију докумената који претвара различите формате фајлова у висококвалитетан Markdown, оптимизован за коришћење LLM и токове рада за анализу текста.

**Зашто је користан**: Неопходан за модерне токове рада са документацијом! MarkItDown обрађује импресиван распон формата фајлова уз очување кључне структуре документа као што су наслови, листе, табеле и линкови. За разлику од једноставних алата за извлачење текста, фокусира се на одржавање семантичког значења и форматирања које је корисно и за AI обраду и за људску читљивост.

**Подржани формати фајлова**:
- **Офице документи**: PDF, PowerPoint (PPTX), Word (DOCX), Excel (XLSX/XLS)
- **Медијски фајлови**: Слике (са EXIF метаподацима и OCR), Аудио (са EXIF метаподацима и транскрипцијом говора)
- **Веб садржај**: HTML, RSS фидови, YouTube URL-ови, Википедијини странице
- **Формати података**: CSV, JSON, XML, ZIP фајлови (рекурзивно обрађује садржај)
- **Формати за публиковање**: EPub, Jupyter бележнице (.ipynb)
- **Е-пошта**: Outlook поруке (.msg)
- **Напредно**: Интеграција Azure Document Intelligence за побољшану обраду PDF фајлова

**Напредне могућности**: MarkItDown подржава описе слика које покреће LLM (када се обезбеди OpenAI клијент), Azure Document Intelligence за побољшану обраду PDF-ова, транскрипцију аудио садржаја, и систем плагина за проширење на додатне формате фајлова.

**Коришћење у стварном свету**: "Конвертуј ову PowerPoint презентацију у Markdown за наш сајт са документацијом", "Извуци текст из овог PDF-а са правилном структуром наслова", или "Претвори ову Excel табелу у читљив табеларни формат".

**Илустровани пример**: Да цитирамо [MarkItDown документацију](https://github.com/microsoft/markitdown#why-markdown):

> Markdown је изузетно близу обичном тексту, са минималним маркирањем или форматирањем, али ипак пружа начин да се представи важна структура документа. Главни LLM-ови, као што је OpenAI GPT-4o, нативно „говоре“ Markdown и често укључују Markdown у своје одговоре неспровођено. То указује да су обучавани на огромним количинама текста форматираног у Markdown, и да га добро разумеју. Као додатна предност, Markdown конвенције су такође веома ефикасне у коришћењу токена.

MarkItDown изузетно добро очувава структуру документа, што је важно за AI токове рада. На пример, приликом конверзије PowerPoint презентације, одржава организацију слајдова са исправним насловима, извлачи табеле као Markdown табеле, укључује алтернативни текст за слике, па чак обрађује и белешке говорника. Графикони се претварају у читљиве табеле података, а резултујући Markdown одржава логички ток оригиналне презентације. Ово је идеално за уношење презентацијског садржаја у AI системе или креирање документације од постојећих слајдова.

Оно што ово чини изузетно моћним је што прелази јаз између инструкција природног језика и извршног тест кода. Традиционални приступи захтевају ручно писање тестова или приступ коду за контекст. Али са Playwright MCP, можете тестирати екстерне сајтове, клијентске апликације или радити у сценаријима црне кутије где приступ коду није доступан.

### 9. 🤖 Microsoft Foundry MCP Server
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Microsoft_Foundry_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20Foundry%20MCP%20Server&config=%7B%22type%22%3A%22stdio%22%2C%22command%22%3A%22uvx%22%2C%22args%22%3A%5B%22--prerelease%3Dallow%22%2C%22--from%22%2C%22git%2Bhttps%3A%2F%2Fgithub.com%2Fazure-ai-foundry%2Fmcp-foundry.git%22%2C%22run-azure-ai-foundry-mcp%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Microsoft_Foundry_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20Foundry%20MCP%20Server&config=%7B%22type%22%3A%22stdio%22%2C%22command%22%3A%22uvx%22%2C%22args%22%3A%5B%22--prerelease%3Dallow%22%2C%22--from%22%2C%22git%2Bhttps%3A%2F%2Fgithub.com%2Fazure-ai-foundry%2Fmcp-foundry.git%22%2C%22run-azure-ai-foundry-mcp%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/azure-ai-foundry/mcp-foundry)

**Шта ради**: Microsoft Foundry MCP Server пружа програмерима свеобухватан приступ Azure AI екосистему, укључујући каталоге модела, управљање распоредом, индексирање знања са Azure AI Search и алате за евалуацију. Овај експериментални сервер пребацује јаз између развоја АИ и моћне инфраструктуре Azure-AI, олакшавајући изградњу, распоређивање и евалуацију АИ апликација.

**Зашто је користан**: Овај сервер мења начин на који радите са Azure AI услугама увођењем AI могућности на нивоу предузећа директно у ваш развојни ток рада. Уместо пребацивања између Azure портала, документације и вашег IDE-а, можете откривати моделе, распореживати услуге, управљати базама знања и процењивати АИ перформансе кроз наредбе на природном језику. Посебно је моћан за програмере који граде РАГ (Retrieval-Augmented Generation) апликације, управљају мулти-модел размењивањем или имплементирају свеобухватне евалуационе процесе за АИ.

**Кључне могућности за програмере**:
- **🔍 Откривање и распореда модела**: Истражите Microsoft Foundry каталог модела, добијте детаљне информације о моделу са примерима кода и распореживајте моделе на Azure AI сервисима
- **📚 Управљање знањем**: Креирајте и управљајте индексирањем у Azure AI Search, додајте документе, конфигуришите индексере и градите сложене РАГ системе
- **⚡ Интеграција AI агената**: Повежите се са Azure AI агенатима, упитајте постојеће агенте и процените њихове перформансе у продукцијским сценаријима
- **📊 Евалуациони оквир**: Извршавајте комплетне текстуалне и агентске евалуације, генеришите Markdown извештаје и имплементирајте контролу квалитета за АИ апликације
- **🚀 Алати за прототиповање**: Добијте упутства за подешавање прототиповања базираног на GitHub-у и приступите Microsoft Foundry Labs за најсавременије истраживачке моделе

**Примена у стварном свету**: "Распореди Phi-4 модел на Azure AI Services за моју апликацију", "Направи нови индекс за претрагу за мој RAG систем са документацијом", "Процени одговоре мог агента према метрици квалитета" или "Пронађи најбољи модел размишљања за моје сложене аналитичке задатке".

**Цео демонстрациони сценарио**: Ево моћног тока развоја АИ:

> "Градим агента за корисничку подршку. Помозите ми да пронађем добар модел размишљања из каталога, распоредим га на Azure AI Services, направим базу знања из наше документације, поставим евалуациони оквир за тестирање квалитета одговора и онда помогнете да прототипујем интеграцију са GitHub токеном за тестирање."

Microsoft Foundry MCP Server ће:
- Испитати каталог модела да препоручи оптималне моделе размишљања на основу ваших захтева
- Обезбедити наредбе за распоредање и информације о квотама за жељени Azure регион
- Подесити Azure AI Search индексе са одговарајућом шемом за вашу документацију
- Конфигурисати евалуационе процесе са метриком квалитета и безбедносним проверама
- Генерисати код за прототиповање са GitHub аутентификацијом за непосредно тестирање
- Обезбедити свеобухватне водиче за подешавање прилагођене вашем специфичном технолошком скупу

**Истакнути пример**: Као развојни програмер, тешко сам успевао да пратим различите LLM моделе који су доступни. Познајем неке главне, али сам имао осећај да губим продуктивност и ефикасност. И токени и квоте су напорни и тешки за управљање – никада нисам знао да ли бирањем правог модела за задатак трошим буџет неефикасно. Управо сам чуо за овај MCP сервер од James Montemagno-а док сам проверавао препоруке од тимских сарадника за овај пост, и узбуђен сам да га испробам! Могућности откривања модела нарочито ми изгледају импресивно када желим да истражим ван уобичајених избора и пронађем моделе оптимизоване за специфичне задатке. Евалуациони оквир треба да ми помогне да потврдим да заиста добијам боље резултате, а не само да испробавам нешто ново ради самог испробавања.

## 📋 Закључак

Model Context Protocol (MCP) мења начин на који програмери комуницирају са AI асистентима и спољним алатима. Ових 10 Microsoft MCP сервера демонстрирају снагу стандардизоване интеграције AI, омогућавајући несметан ток рада који задржава програмере у њиховом "флоу" стању док приступају моћним спољним могућностима.

Од свеобухватне интеграције Azure екосистема до специјализованих алата као што су Playwright за аутоматизацију прегледача и MarkItDown за обраду докумената, ови сервери показују како MCP може повећати продуктивност у разним сценаријима развоја. Стандардизовани протокол осигурава да сви ти алати раде заједно беспрекорно, стварајући јединствено искуство развоја.

Како MCP екосистем наставља да се развија, остајање активним у заједници, истраживање нових сервера и израда прилагођених решења биће кључ за максимализацију ваше продуктивности у развоју. Отворени стандард MCP значи да можете комбиновати алате различитих добављача како бисте креирали савршен ток рада за ваше специфичне потребе.
