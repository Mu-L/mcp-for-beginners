# AGENTS.md

## Преглед пројекта

**MCP за почетнике** је отворени образовни програм за учење Протокола контекста модела (MCP) - стандардизованог оквира за интеракције између AI модела и клијентских апликација. Ово складиште пружа свеобухватне наставне материјале са практичним примерима кода на више програмских језика.

### Кључне технологије

- **Програмирање језици**: C#, Java, JavaScript, TypeScript, Python, Rust
- **Фрејмворкови и SDK-ови**: 
  - MCP SDK (`@modelcontextprotocol/sdk`)
  - Spring Boot (Java)
  - FastMCP (Python)
  - LangChain4j (Java)
- **Базе података**: PostgreSQL са pgvector екстензијом
- **Облачно окружење**: Azure (Container Apps, OpenAI, Content Safety, Application Insights)
- **Алати за изградњу**: npm, Maven, pip, Cargo
- **Документација**: Markdown са аутоматизованим преводом на више језика (48+ језика)

### Архитектура

- **11 основних модула (00-11)**: Секвенцијални пут учења од основа до напредних тема
- **Практичне радионице**: Практични задаци са комплетним кодом у више језика
- **Пример пројеката**: Функционални MCP сервер и клијент имплементације
- **Систем превода**: Аутоматизовани GitHub Actions ток рада за подршку више језика
- **Слике и ресурси**: Централизован каталог слика са преведеним верзијама

## Команде за подешавање

Ово је складиште фокусирано на документацију. Већина подешавања се врши унутар појединачних пример пројеката и радионица.

### Подешавање складишта

```bash
# Клонирај репозиторијум
git clone https://github.com/microsoft/mcp-for-beginners.git
cd mcp-for-beginners
```

### Рад са пример пројектима

Пример пројекти се налазе у:
- `03-GettingStarted/samples/` - Језично специфични примери
- `03-GettingStarted/01-first-server/solution/` - Прве сервер имплементације
- `03-GettingStarted/02-client/solution/` - Клијентске имплементације
- `11-MCPServerHandsOnLabs/` - Свеобухватне радионице интеграције база података

Сваки пример пројекта садржи своја упутства за подешавање:

#### TypeScript/JavaScript пројекти
```bash
cd <project-directory>
npm install
npm start
```

#### Python пројекти
```bash
cd <project-directory>
pip install -r requirements.txt
# или
pip install -e .
python main.py
```

#### Java пројекти
```bash
cd <project-directory>
mvn clean install
mvn spring-boot:run
```

## Радни ток развоја

### Структура документације

- **Модули 00-11**: Основни наставни садржај по редоследу
- **translations/**: Језичке верзије (аутоматски генерисане, немојте директно уређивати)
- **translated_images/**: Локализоване верзије слика (аутоматски генерисане)
- **images/**: Изворне слике и дијаграми

### Измјене у документацији

1. Уређујте само енглеске markdown датотеке у главним директоријумима модула (00-11)
2. По потреби ажурирајте слике у директоријуму `images/`
3. co-op-translator GitHub Action ће аутоматски генерисати преводе
4. Преводи се генералишу приликом push-а у главну грану

### Рад са преводима

- **Аутоматизовани превод**: GitHub Actions обрађује све преводе
- **НЕ уређујте ручно** датотеке у директоријуму `translations/`
- Мета подаци о преводу уграђени у сваку преведену датотеку
- Подржани језици: 48+ језика укључујући арапски, кинески, француски, немачки, хинди, јапански, корејски, португалски, руски, шпански и многе друге

## Упутства о стилу кода

### Стил документације

- Користите јасан, приступачан језик за почетнике
- Укључите примере кода на више језика када је могуће
- Следите најбоље праксе за markdown:
  - Користите ATX стил заглавља (`#` синтакса)
  - Користите ограђене блокове кода са идентификаторима језика
  - Укључите описни alt текст за слике
  - Држите дужину линија разумном (нема тврдих ограничења, али будите практични)

### Стил примера кода

#### TypeScript/JavaScript
- Користите ES модуле (`import`/`export`)
- Следите TypeScript строги режим
- Укључите типске анотације
- Циљајте ES2022

#### Python
- Следите PEP 8 стил
- Користите типске наводе где је примењиво
- Укључите docstrings за функције и класе
- Користите савремене Python карактеристике (3.8+)

#### Java
- Следите Spring Boot конвенције
- Користите Java 21 карактеристике
- Следите стандардну Maven структуру пројекта
- Укључите Javadoc коментаре

### Организација датотека

```
<module-number>-<ModuleName>/

│   ├── typescript/
│   ├── python/
│   ├── java/
│   └── ...
└── solution/              # Complete working solutions
    └── <language>/
```

## Упутства за допринос

### Процес Pull Request-а

1. **Форк и клон**: Форкирајте репозиторјум и клоновајте свој форк локално
2. **Креирај грану**: Користите описне називе грана (нпр. `fix/typo-module-3`, `add/python-example`)
3. **Направи измене**: Уређујте само енглеске markdown датотеке (не преводе)
4. **Локално тествај**: Проверите правилно приказивање markdown-а
5. **Пошаљи PR**: Користите јасне наслове и описе у PR-у
6. **CLA**: Потпишите Microsoft Contributor License Agreement када буде затражено

### Формат наслова PR-а

Користите јасне, описне наслове:
- `[Module XX] Кратак опис` за измене специфичне за модул
- `[Samples] Опис` за измене примера кода
- `[Docs] Опис` за опште ажурирање документације

### Шта допринети

- Исправке грешака у документацији или примерима кода
- Нови примери кода на додатним језицима
- Појашњења и побољшања постојећег садржаја
- Нове студије случаја или практични примери
- Пријаве проблема за нејасан или нетачан садржај

### Шта НЕ радити

- Немојте директно уређивати датотеке у `translations/` директоријуму
- Немојте уређивати `translated_images/` директоријум
- Немојте додавати велике бинарне датотеке без претходне дискусије
- Немојте мењати ток рада превођења без координације

### Уобичајени проблеми и решења

**П: Мој PR не пролази проверу превода**  
О: Проверите да сте изменили само енглеске markdown датотеке у главним директоријумима модула, а не преводе.

**П: Како да додам нови језик?**  
О: Подршка језицима се управља кроз co-op-translator ток рада. Отворите issue за дискусију о додавању нових језика.

**П: Примери кода не раде**  
О: Проверите да ли сте следили упутства за подешавање у README-у конкретног примера. Проверите да ли имате инсталиране исправне верзије зависности.

**П: Слике се не приказују**  
О: Потврдите да су путеви до слика релативни и користе косу црту (/). Слике треба да буду у `images/` или у `translated_images/` за локализоване верзије.
