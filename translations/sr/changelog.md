### Ојачање безбедности и исправке рањивости зависности

Извршена је детаљна безбедносна провера свих манифеста зависности и примерског извора, затим су отклоњене све пријављене npm препоруке и један налаз на нивоу кода. Након исправки, `npm audit` извештава **0 рањивости** у свим провереним директоријумима.

#### npm рањивости зависности (транзитивне) — Поправљене

Проверавано свих 15 објављених `package-lock.json` фајлова. Ранивости су биле ограничене на транзитивне зависности које уносе MCP Inspector алат за развој, OpenAI клијент и MCP SDK; све су сада решене без разбијања примера:
- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/inspector** и **lab3/code/weather_mcp/inspector**: Повећан `@modelcontextprotocol/inspector` (`0.16.6` / `0.14.1` → `0.22.0`), што је уклонило проблеме са `ajv`, `brace-expansion`, `diff`, `path-to-regexp` и `ws` препорукама. Додат npm `overrides` запис који принудно користи исправљени `shell-quote@1.8.4` да елиминише преосталу критичну препоруку коју доноси `concurrently`; регенерисани оба lockfile фајла (сада 0 рањивости)
- **03-GettingStarted/samples/typescript**: `npm audit fix` ажурирао транзитивни `qs` (умерена) на исправљену верзију
- **03-GettingStarted/samples/javascript**: `npm audit fix` ажурирао транзитивни `hono` (умерена) на исправљену верзију
- **03-GettingStarted/03-llm-client/solution/typescript**: `npm audit fix` ажурирао транзитивни `form-data` (висока) на исправљену верзију
- **03-GettingStarted/11-simple-auth/solution/typescript**: Генерисан недостајући `package-lock.json` ради репродуковивости и провере (0 рањивости)

#### Безбедносна поправка на нивоу кода (OWASP A03: Injection)

- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/src/server.py**: Уклоњен `shell=True` из `open_in_vscode` алата. Претходни `subprocess.run(["start", "", vscode_path, folder_path], shell=True)` је дозволио да мета-карактере шкољке у путањи фасцикле тумачи `cmd.exe` (вектор командне инјекције). Сада директно покреће решени `Code.exe` са фасциклом као аргументом — без шкољке — што је функционално еквивалентно и безбедно

#### Провера зависности за Python

- Провераване су све Python dependencies са `pip-audit`. `05-AdvancedTopics` и `03-GettingStarted/samples/python` пријављују **нема познатих рањивости** (њихове верзије `mcp` / `httpx` / `pydantic` / `python-dotenv` воде ка актуелним исправљеним издањима)
- **09-CaseStudy/docs-mcp/solution/python/requirements.txt**: `pip-audit` је означио транзитивну зависност **`werkzeug` 3.1.1** са три DoS препоруке везане за `safe_join` Windows имена уређаја — `CVE-2025-66221`, `CVE-2026-21860` и `CVE-2026-27199` (све исправљено у 3.1.6). Додат изричит безбедносни пин `werkzeug>=3.1.6` да се реши исправљено издање; проверено да се ограничење правилно резолује уз `chainlit` / `mcp` / `semantic-kernel` стек
