### Ојачавање безбедности и отклањање рањивости зависности

Покренули смо комплетан безбедносни преглед свих манифеста зависности и примера кода, затим уклонили све пријављене npm савете и једно налаз на нивоу кода. Након исправки, `npm audit` извештава о **0 рањивости** у свим провереним директоријумима.

#### npm рањивости зависности (транзитивне) — Исправљено

Аудитовали смо свих 15 укључених `package-lock.json` фајлова. Ранљивости су биле ограничене на транзитивне зависности које вуче MCP Inspector дев алат, OpenAI клијент и MCP SDK; све су сада решене без ломљења примера:
- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/inspector** и **lab3/code/weather_mcp/inspector**: Подигнут `@modelcontextprotocol/inspector` (`0.16.6` / `0.14.1` → `0.22.0`), чиме су уклоњени савети за уграђене `ajv`, `brace-expansion`, `diff`, `path-to-regexp` и `ws`. Додат npm `overrides` запис који приморава патчовни `shell-quote@1.8.4` да уклони преосталу критичну рањивост коју носи `concurrently`; регенерисани оба lock-фајла (сада 0 рањивости)
- **03-GettingStarted/samples/typescript**: `npm audit fix` ажурирао транзитивни `qs` (умерена рањивост) на патч издање
- **03-GettingStarted/samples/javascript**: `npm audit fix` ажурирао транзитивни `hono` (умерена рањивост) на патч издање
- **03-GettingStarted/03-llm-client/solution/typescript**: `npm audit fix` ажурирао транзитивни `form-data` (висока рањивост) на патч издање
- **03-GettingStarted/11-simple-auth/solution/typescript**: Генерисан недостајући `package-lock.json` како би пројекат био репродуцибилан и проверљив (0 рањивости)

#### Безбедносна исправка на нивоу кода (OWASP A03: Инјекција)

- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/src/server.py**: Уклоњен `shell=True` из алата `open_in_vscode`. Претходни `subprocess.run(["start", "", vscode_path, folder_path], shell=True)` омогућавао је да мета-карактере љуске у путањи фолдера интерпретира `cmd.exe` (вектор командне инјекције). Сада директно покреће резолвирани `Code.exe` са фолдером као аргументом — без љуске — што је функционално једнако и безбедно

#### Питхон аудит зависности

- Аудитовани сви Питхон скупови захтева помоћу `pip-audit`. `05-AdvancedTopics` и `03-GettingStarted/samples/python` нису пријавили познате рањивости (њихове `mcp` / `httpx` / `pydantic` / `python-dotenv` зоне резолуирају на тренутна патч издања)
- **09-CaseStudy/docs-mcp/solution/python/requirements.txt**: `pip-audit` је означио транзитивну зависност **`werkzeug` 3.1.1** са три DoS савета везана за функцију `safe_join` на Windows уређаје — `CVE-2025-66221`, `CVE-2026-21860`, и `CVE-2026-27199` (све исправљено у 3.1.6). Додат експлицитни безбедносни пин `werkzeug>=3.1.6` тако да се патч издање резолуира; потврђено да ограничење чисто резолуира са стогом `chainlit` / `mcp` / `semantic-kernel`

  - Ажурирано са тренутним методским обрасцима за откривање (`*/list`), преузимање (`*/get`), и извршавање (`*/call`)

  - Подаци слој: Основе JSON-RPC 2.0 са управљањем животним циклусом и примитивима  
  - Транспортни слој: STDIO (локално) и Streamable HTTP са SSE (удаљени) транспортним механизмима  
- **Безбедносни оквир**: Свеобухватни безбедносни принципи укључујући јасан кориснички пристанак, заштиту приватности података, безбедност извршавања алата и безбедност транспортног слоја
