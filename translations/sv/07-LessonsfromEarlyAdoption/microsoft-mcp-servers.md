## 🔧 Förstå MCP-servrar: En nybörjarguide

### Vad är MCP-servrar?

Som nybörjare på Model Context Protocol (MCP) kanske du undrar: "Vad är egentligen en MCP-server och varför ska jag bry mig?" Låt oss börja med en enkel liknelse.

Tänk på MCP-servrar som specialiserade assistenter som hjälper din AI-kodningskompanjon (som GitHub Copilot) att koppla upp sig mot externa verktyg och tjänster. Precis som du kan använda olika appar på din telefon för olika uppgifter – en för väder, en för navigation, en för bankärenden – ger MCP-servrar din AI-assistent förmågan att interagera med olika utvecklingsverktyg och tjänster.

### Problemet som MCP-servrar löser

Innan MCP-servrar, om du ville:
- Kontrollera dina Azure-resurser
- Skapa en GitHub-issue
- Fråga din databas
- Söka i dokumentation

Skulle du behöva sluta koda, öppna en webbläsare, navigera till rätt webbplats och manuellt utföra dessa uppgifter. Detta ständiga kontextbyte bryter ditt fokus och minskar produktiviteten.

### Hur MCP-servrar förändrar din utvecklingsupplevelse

Med MCP-servrar kan du stanna i din utvecklingsmiljö (VS Code, Visual Studio osv.) och helt enkelt be din AI-assistent att hantera dessa uppgifter. Till exempel:

**Istället för detta traditionella arbetsflöde:**
1. Sluta koda  
2. Öppna webbläsaren  
3. Gå till Azure-portalen  
4. Leta upp detaljer om lagringskonto  
5. Återgå till VS Code  
6. Fortsätt koda  

**Kan du nu göra så här:**
1. Fråga AI: "Vad är status på mina Azure-lagringskonton?"  
2. Fortsätt koda med den givna informationen

## Kom igång med MCP i VS Code och Visual Studio 🚀

Att konfigurera dessa MCP-servrar är enkelt om du använder Visual Studio Code eller Visual Studio 2022 med GitHub Copilot.

### Setup för VS Code

Här är den grundläggande processen för VS Code:

1. **Aktivera Agent-läge**: I VS Code, växla till Agent-läge i Copilot Chat-fönstret  
2. **Konfigurera MCP-servrar**: Lägg till serverkonfigurationer i din VS Codes settings.json-fil  
3. **Starta servrar**: Klicka på "Start" för varje server du vill använda  
4. **Välj verktyg**: Välj vilka MCP-servrar som ska aktiveras för din nuvarande session  

För detaljerade installationsinstruktioner, se [VS Code MCP-dokumentationen](https://code.visualstudio.com/docs/copilot/copilot-mcp).

> **💡 Proffstips: Hantera MCP-servrar som ett proffs!**  
>  
> VS Codes Extensions-vy har nu ett [praktiskt nytt gränssnitt för att hantera installerade MCP-servrar](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode)! Du har snabb åtkomst för att starta, stoppa och hantera alla installerade MCP-servrar via ett tydligt och enkelt gränssnitt. Testa det!

### Setup för Visual Studio 2022

För Visual Studio 2022 (version 17.14 eller senare):

1. **Aktivera Agent-läge**: Klicka på "Fråga"-rullgardinsmenyn i GitHub Copilot Chat-fönstret och välj "Agent"  
2. **Skapa konfigurationsfil**: Skapa en `.mcp.json`-fil i din lösningsmapp (rekommenderad plats: `<SOLUTIONDIR>\.mcp.json`)  
3. **Konfigurera servrar**: Lägg till dina MCP-serverkonfigurationer med standard MCP-format  
4. **Verktygs-approval**: Godkänn de verktyg du vill använda med lämpliga behörigheter när du uppmanas  

För detaljerade instruktioner för Visual Studio, se [Visual Studio MCP-dokumentationen](https://learn.microsoft.com/visualstudio/ide/mcp-servers).

Varje MCP-server har sina egna konfigurationskrav (anslutningssträngar, autentisering osv.), men installationsmönstret är konsekvent i båda IDE:erna.

### 1. 📚 Microsoft Learn Docs MCP-server

[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Microsoft_Docs_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Microsoft_Docs_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/mcp)

**Vad den gör**: Microsoft Learn Docs MCP-server är en molnbaserad tjänst som ger AI-assistenter realtidsåtkomst till officiell Microsoft-dokumentation via Model Context Protocol. Den kopplar upp sig mot `https://learn.microsoft.com/api/mcp` och möjliggör semantisk sökning i Microsoft Learn, Azure-dokumentation, Microsoft 365-dokumentation och andra officiella Microsoft-källor.

**Varför den är användbar**: Även om det kan verka som "bara dokumentation" är denna server avgörande för varje utvecklare som arbetar med Microsoft-teknologier. En av de största klagomålen från .NET-utvecklare om AI-kodassistenter är att de inte är uppdaterade med de senaste .NET- och C#-versionerna. Microsoft Learn Docs MCP-server löser detta genom att ge realtidsåtkomst till den mest aktuella dokumentationen, API-referenser och bästa praxis. Oavsett om du arbetar med de senaste Azure SDK:erna, utforskar nya C# 13-funktioner eller implementerar banbrytande .NET Aspire-mönster, säkerställer denna server att din AI-assistent har tillgång till auktoritativ, uppdaterad information för att generera korrekt, modern kod.

**Verklig användning**: "Vilka är az cli-kommandona för att skapa en Azure container app enligt officiell Microsoft Learn-dokumentation?" eller "Hur konfigurerar jag Entity Framework med dependency injection i ASP.NET Core?" Eller vad sägs om "Granska denna kod för att säkerställa att den följer prestandarekommendationerna i Microsoft Learn-dokumentationen." Servern erbjuder omfattande täckning över Microsoft Learn, Azure docs och Microsoft 365-dokumentation genom avancerad semantisk sökning för att hitta mest kontextuellt relevanta information. Den returnerar upp till 10 högkvalitativa innehållsbitar med artikeltitlar och URL:er, alltid med åtkomst till den senaste Microsoft-dokumentationen när den publiceras.

**Utvalt exempel**: Servern exponerar verktyget `microsoft_docs_search` som utför semantisk sökning mot Microsofts officiella tekniska dokumentation. När den är konfigurerad kan du ställa frågor som "Hur implementerar jag JWT-autentisering i ASP.NET Core?" och få detaljerade, officiella svar med käll-länkar. Sökkvaliteten är exceptionell eftersom den förstår kontext – att fråga om "containers" i en Azure-kontext returnerar dokumentation för Azure Container Instances, medan samma term i en .NET-kontext ger relevant C#-samlinginformation.

Detta är särskilt användbart för snabbt föränderliga eller nyligen uppdaterade bibliotek och användningsfall. Till exempel, i några nyligen genomförda kodningsprojekt ville jag dra nytta av funktioner i de senaste releaserna av .NET Aspire och Microsoft.Extensions.AI. Genom att inkludera Microsoft Learn Docs MCP-servern kunde jag använda inte bara API-docs, utan även guider och vägledning som just publicerats.

### 2. ☁️ Azure MCP-server
[![Installera i VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![Installera i VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

**Vad den gör**: Azure MCP Server är en omfattande svit av över 15 specialiserade Azure-tjänstekopplingar som för in hela Azure-ekosystemet i ditt AI-arbetsflöde. Det är inte bara en enskild server – det är en kraftfull samling som inkluderar resursförvaltning, databasanslutning (PostgreSQL, SQL Server), Azure Monitor logganalys med KQL, Cosmos DB-integration och mycket mer.

**Varför den är användbar**: Utöver att bara hantera Azure-resurser förbättrar denna server dramatiskt kodkvaliteten när du arbetar med Azure SDK:er. När du använder Azure MCP i Agent-läge hjälper den dig inte bara att skriva kod – den hjälper dig att skriva *bättre* Azure-kod som följer aktuella autentiseringsmönster, bästa praxis för felhantering och utnyttjar de senaste SDK-funktionerna. Istället för att få generell kod som kan fungera får du kod som följer Azures rekommenderade mönster för produktionsarbetsbelastningar.

**Viktiga moduler inkluderar**:
- **🗄️ Databaskopplingar**: Direkt naturligt språkligt tillträde till Azure Database för PostgreSQL och SQL Server
- **📊 Azure Monitor**: KQL-driven logganalys och operationella insikter
- **🌐 Resurshantering**: Fullständig livscykelhantering av Azure-resurser
- **🔐 Autentisering**: DefaultAzureCredential och hanterade identitetsmönster
- **📦 Lagringstjänster**: Blob Storage, Queue Storage och Table Storage-operationer
- **🚀 Containertjänster**: Azure Container Apps, Container Instances och AKS-hantering
- **Och många fler specialiserade kopplingar**

**Användning i verkliga världen**: "Lista mina Azure-lagringskonton", "Fråga min Log Analytics-arbetsyta efter fel under senaste timmen" eller "Hjälp mig bygga en Azure-applikation med Node.js med korrekt autentisering"

**Fullt demoscenario**: Här är en komplett genomgång som visar kraften i att kombinera Azure MCP med GitHub Copilot for Azure-tillägget i VS Code. När du har båda installerade och ger prompten:

> "Skapa ett Python-script som laddar upp en fil till Azure Blob Storage med DefaultAzureCredential-autentisering. Skriptet ska ansluta till mitt Azure-lagringskonto som heter 'mycompanystorage', ladda upp till en container som heter 'documents', skapa en testfil med aktuell tidsstämpel för uppladdning, hantera fel på ett snyggt sätt och tillhandahålla informativ output, följa Azures bästa praxis för autentisering och felhantering, inkludera kommentarer som förklarar hur DefaultAzureCredential-autentiseringen fungerar, och göra skriptet välstrukturerat med korrekta funktioner och dokumentation."

Azure MCP Server genererar ett komplett, produktionsklart Python-script som:
- Använder senaste Azure Blob Storage SDK med korrekta asynkrona mönster
- Implementerar DefaultAzureCredential med utförlig förklaring av fallback-kedjan
- Inkluderar robust felhantering med specifika Azure-undantagstyper
- Följer Azure SDK:s bästa praxis för resursförvaltning och anslutningshantering
- Ger detaljerad loggning och informativ konsolutskrift
- Skapar ett välstrukturerat skript med funktioner, dokumentation och typanvisningar

Det som gör detta anmärkningsvärt är att utan Azure MCP kan du få generell blob storage-kod som fungerar men inte följer aktuella Azure-mönster. Med Azure MCP får du kod som utnyttjar de senaste autentiseringsmetoderna, hanterar Azure-specifika felfall och följer Microsofts rekommenderade praxis för produktionsapplikationer.

**Exempel i fokus**: Jag har kämpat med att komma ihåg de specifika kommandona för `az`- och `azd`-CLI:er för ad hoc-användning. Det är alltid en tvåstegsprocess för mig: först slå upp syntaxen, sedan köra kommandot. Jag går ofta bara in i portalen och klickar runt för att få jobbet gjort eftersom jag inte vill erkänna att jag inte kan minnas CLI-syntax. Att kunna bara beskriva vad jag vill är fantastiskt, och ännu bättre att kunna göra det utan att lämna min IDE!

Det finns en bra lista med användningsfall i [Azure MCP-förvaret](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) för att komma igång. För omfattande installationsguider och avancerade konfigurationsalternativ, se den [officiella Azure MCP-dokumentationen](https://learn.microsoft.com/azure/developer/azure-mcp-server/).

### 3. 🐙 GitHub MCP Server

**Vad den gör**: Den officiella GitHub MCP Server tillhandahåller sömlös integration med hela GitHubs ekosystem, och erbjuder både värdbaserad fjärråtkomst och lokala Docker-distributionsalternativ. Det handlar inte bara om grundläggande repository-operationer – det är ett omfattande verktygspaket som inkluderar hantering av GitHub Actions, pull request-arbetsflöden, ärendehantering, säkerhetsscanning, notifieringar och avancerade automatiseringsmöjligheter.

**Varför den är användbar**: Denna server förändrar hur du interagerar med GitHub genom att föra hela plattformsupplevelsen direkt in i din utvecklingsmiljö. Istället för att ständigt växla mellan VS Code och GitHub.com för projektledning, kodgranskningar och CI/CD-övervakning kan du hantera allt via naturliga språkkonton medan du håller fokus på din kod.

> **ℹ️ Notera: Olika typer av 'Agentar'**
> 
> Förväxla inte denna GitHub MCP Server med GitHubs Coding Agent (den AI-agent som du kan tilldela ärenden för automatiserade kodningsuppgifter). GitHub MCP Server fungerar inom VS Codes Agent-läge för att tillhandahålla GitHub API-integration, medan GitHubs Coding Agent är en separat funktion som skapar pull requests när den tilldelas GitHub-ärenden.

**Viktiga funktioner inkluderar**:
- **⚙️ GitHub Actions**: Komplett CI/CD-pipelinehantering, övervakning av arbetsflöden och hantering av artefakter
- **🔀 Pull Requests**: Skapa, granska, slå ihop och hantera PR:er med omfattande statusspårning
- **🐛 Ärenden**: Full livscykelhantering av ärenden, kommentering, märkning och tilldelning
- **🔒 Säkerhet**: Kodsökningslarm, sekretessdetektion och Dependabot-integration
- **🔔 Notifieringar**: Smart notifieringshantering och kontroll av repository-prenumerationer
- **📁 Repository-hantering**: Filoperationer, grenhantering och repository-administration
- **👥 Samarbete**: Användar- och organisationssökning, teamhantering och åtkomstkontroll

**Användning i verkliga världen**: "Skapa en pull request från min feature-bransch", "Visa alla misslyckade CI-körningar denna vecka", "Lista öppna säkerhetslarm för mina repositories" eller "Hitta alla ärenden som är tilldelade mig över mina organisationer"

**Fullt demoscenario**: Här är ett kraftfullt arbetsflöde som demonstrerar GitHub MCP Servers kapaciteter:

> "Jag behöver förbereda för vår sprintöversyn. Visa alla pull requests jag skapat denna vecka, kontrollera status för våra CI/CD-pipelines, skapa en sammanfattning av eventuella säkerhetslarm vi behöver åtgärda, och hjälp mig att utarbeta release-noteringar baserade på sammanslagna PR:er med etiketten 'feature'."

GitHub MCP Server kommer att:
- Fråga dina senaste pull requests med detaljerad statusinformation
- Analysera arbetsflödeskörningar och lyfta fram eventuella fel eller prestandaproblem
- Sammanställa säkerhetsscanningsresultat och prioritera kritiska larm
- Generera omfattande releasenoteringar genom att extrahera information från sammanslagna PR:er
- Ge handlingsbara nästa steg för sprintplanering och releaseförberedelser

**Exempel i fokus**: Jag älskar att använda detta för kodgranskningsarbetsflöden. Istället för att hoppa mellan VS Code, GitHub-notifieringar och pull request-sidor kan jag säga "Visa alla PR:er som väntar på min granskning" och sedan "Lägg till en kommentar på PR #123 och fråga om felhanteringen i autentiseringsmetoden." Servern hanterar GitHub API-anropen, behåller kontext om diskussionen och hjälper mig till och med att formulera mer konstruktiva granskningskommentarer.

**Autentiseringsalternativ**: Servern stöder både OAuth (sömlöst i VS Code) och Personal Access Tokens, med konfigurerbara verktygssatser för att bara aktivera den GitHub-funktionalitet du behöver. Du kan köra den som en fjärrhostad tjänst för snabb installation eller lokalt via Docker för full kontroll.

**Varför det är användbart**: Väsentligt för moderna dokumentationsarbetsflöden! MarkItDown hanterar ett imponerande utbud av filformat samtidigt som det bevarar kritisk dokumentstruktur som rubriker, listor, tabeller och länkar. Till skillnad från enkla textutvinningsverktyg fokuserar det på att behålla semantisk betydelse och formatering som är värdefull för både AI-bearbetning och mänsklig läsbarhet.

**Stödda filformat**:
- **Office-dokument**: PDF, PowerPoint (PPTX), Word (DOCX), Excel (XLSX/XLS)

- **Webbinnehåll**: HTML, RSS-flöden, YouTube-URL:er, Wikipedia-sidor
- **Dataformat**: CSV, JSON, XML, ZIP-filer (bearbetar innehåll rekursivt)
- **Publiceringsformat**: EPub, Jupyter-notebookar (.ipynb)
- **E-post**: Outlook-meddelanden (.msg)
- **Avancerat**: Azure Document Intelligence-integration för förbättrad PDF-bearbetning

**Avancerade funktioner**: MarkItDown stödjer LLM-drivna bildbeskrivningar (när en OpenAI-klient finns tillgänglig), Azure Document Intelligence för förbättrad PDF-bearbetning, ljudtranskribering för talinnehåll samt ett pluginsystem för utvidgning till fler filformat.

**Praktisk användning**: "Konvertera denna PowerPoint-presentation till Markdown för vår dokumentationssida", "Extrahera text från denna PDF med korrekt rubrikstruktur" eller "Omvandla detta Excel-kalkylblad till ett läsbart tabellformat"

**Framhävt exempel**: För att citera [MarkItDown-dokumentationen](https://github.com/microsoft/markitdown#why-markdown):

> Markdown är extremt nära vanlig text, med minimal märkning eller formatering, men ger ändå ett sätt att representera viktig dokumentstruktur. Vanliga LLM:er, såsom OpenAI:s GPT-4o, "talar" inbyggt Markdown och inkluderar ofta Markdown i sina svar utan att bli uppmanade till det. Detta tyder på att de har tränats på enorma mängder Markdown-formaterad text och förstår det väl. Som en extra fördel är Markdown-konventioner också mycket token-effektiva.

MarkItDown är mycket bra på att bevara dokumentstruktur, vilket är viktigt för AI-arbetsflöden. Till exempel, vid konvertering av en PowerPoint-presentation behålls slide-organisation med rätt rubriker, tabeller extraheras som Markdown-tabeller, alt-text inkluderas för bilder och även talarnotiser bearbetas. Diagram konverteras till läsbara datatabeller, och den resulterande Markdown behåller den logiska flödet från originalpresentationen. Detta gör det perfekt för att mata presentationsinnehåll till AI-system eller skapa dokumentation från befintliga slides.

Det standardiserade protokollet säkerställer att dessa verktyg fungerar perfekt tillsammans och skapar en sammanhängande utvecklingsupplevelse.

När MCP-ekosystemet fortsätter att utvecklas kommer det vara viktigt att vara engagerad i communityn, utforska nya servrar och bygga egna lösningar för att maximera din utvecklingsproduktivitet. Den öppna standarden MCP innebär att du kan kombinera verktyg från olika leverantörer för att skapa det perfekta arbetsflödet för dina specifika behov.
