## 🔧 MCP Sunucularını Anlamak: Yeni Başlayanlar için Kılavuz

### MCP Sunucuları Nedir?

Model Context Protocol (MCP) konusunda yeniyseniz, muhtemelen şöyle düşünebilirsiniz: "MCP sunucusu tam olarak nedir ve neden ilgilenmeliyim?" Basit bir benzetme ile başlayalım.

MCP sunucularını, AI kodlama arkadaşınızın (GitHub Copilot gibi) dış araçlar ve hizmetlerle bağlanmasına yardımcı olan özel asistanlar olarak düşünün. Telefonunuzda farklı görevler için farklı uygulamalar kullandığınız gibi — biri hava durumu, biri navigasyon, biri bankacılık için — MCP sunucuları AI asistanınıza farklı geliştirme araçları ve hizmetleriyle etkileşim kurma yeteneği verir.

### MCP Sunucularının Çözüm Sağladığı Sorun

MCP sunucuları olmadan, eğer şunları yapmak isteseydiniz:
- Azure kaynaklarınızı kontrol etmek
- Bir GitHub sorunu oluşturmak
- Veritabanınızı sorgulamak
- Dokümantasyon içinde arama yapmak

Kod yazmayı bırakarak bir tarayıcı açmanız, uygun web sitesine gitmeniz ve bu işleri manuel yapmanız gerekirdi. Bu sürekli bağlam değiştirme, akışınızı böler ve verimliliği azaltır.

### MCP Sunucuları Geliştirme Deneyiminizi Nasıl Dönüştürür

MCP sunucuları sayesinde geliştirme ortamınızda (VS Code, Visual Studio vb.) kalabilir ve AI asistanınıza bu görevleri halletmesini söyleyebilirsiniz. Örneğin:

**Geleneksel iş akışı yerine:**
1. Kod yazmayı bırak
2. Tarayıcıyı aç
3. Azure portalına git
4. Depolama hesabı detaylarına bak
5. VS Code’a geri dön
6. Kod yazmaya devam et

**Şimdi şu şekilde yapabilirsiniz:**
1. AI’ya "Azure depolama hesaplarımın durumu nedir?" diye sor
2. Sağlanan bilgilerle kod yazmaya devam et

### Yeni Başlayanlar için Temel Faydalar

#### 1. 🔄 **Akış Halinde Kalın**
- Artık birden fazla uygulama arasında geçiş yok
- Yazdığınız koda odaklanmaya devam edin

- Teknik detayları AI’ya bırakın, mantığa odaklanın

#### 3. 🔗 **Birden Çok Aracı Bir Araya Getirin**
- Farklı hizmetleri birleştirerek güçlü iş akışları oluşturun
- Örnek: "Son tüm GitHub sorunlarını al ve karşılık gelen Azure DevOps iş öğelerini oluştur"

#### 4. 🌐 **Genişleyen Bir Ekosisteme Erişin**

### Yeni Başlayanlar için Gerçek Dünya Örneği

Diyelim ki web geliştirmede yenisiniz ve ilk projenizi yapıyorsunuz. MCP sunucuları şöyle yardımcı olabilir:

**Geleneksel yaklaşım:**
```
1. Code a feature
2. Open browser → Navigate to GitHub
3. Create an issue for testing
4. Open another tab → Check Azure docs for deployment
5. Open third tab → Look up database connection examples
6. Return to VS Code
7. Try to remember what you were doing
```

**MCP sunucuları ile:**
```
1. Code a feature
2. Ask AI: "Create a GitHub issue for testing this login feature"
3. Ask AI: "How do I deploy this to Azure according to the docs?"
4. Ask AI: "Show me the best way to connect to my database"
5. Continue coding with all the information you need
```

## VS Code ve Visual Studio’da MCP ile Başlamak 🚀

Bu MCP sunucularını kurmak, Visual Studio Code veya GitHub Copilot destekli Visual Studio 2022 kullanıyorsanız oldukça basittir.

### VS Code Kurulumu

VS Code için temel süreç şudur:

1. **Agent Modunu Etkinleştir**: VS Code’da Copilot Sohbet penceresinde Agent moduna geçin 
2. **MCP Sunucularını Yapılandır**: Sunucu ayarlarını VS Code settings.json dosyanıza ekleyin
3. **Sunucuları Başlat**: Kullanmak istediğiniz her sunucu için "Başlat" düğmesine tıklayın
4. **Araçları Seç**: Mevcut oturumunuz için hangi MCP sunucularının etkin olacağını seçin

Detaylı kurulum talimatları için [VS Code MCP dokümantasyonuna](https://code.visualstudio.com/docs/copilot/copilot-mcp) bakınız.

> **💡 Uzman İpucu: MCP Sunucularını Profesyonelce Yönetin!**
> 
> VS Code Uzantılar görünümünde şimdi [yüklenen MCP Sunucularını yönetmek için kullanışlı yeni bir UI](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode) var! Yüklü MCP Sunucularını başlatmak, durdurmak ve yönetmek için hızlı erişime sahipsiniz ve sade, açıklayıcı bir arayüz üzerinden kontrol sağlayabilirsiniz. Deneyin!

### Visual Studio 2022 Kurulumu

Visual Studio 2022 (sürüm 17.14 veya sonraki) için:

1. **Agent Modunu Etkinleştir**: GitHub Copilot Sohbet penceresindeki "Sor" açılır menüsünden "Agent"ı seçin
2. **Yapılandırma Dosyası Oluştur**: Çözüm dizininize `.mcp.json` dosyası oluşturun (önerilen konum: `<SOLUTIONDIR>\.mcp.json`)
3. **Sunucuları Yapılandır**: MCP sunucusu ayarlarını standart MCP formatında ekleyin
4. **Araç Onayı**: İstendiğinde, kullanmak istediğiniz araçlar için uygun kapsam izinlerini onaylayın

Detaylı Visual Studio kurulum talimatları için [Visual Studio MCP dokümantasyonuna](https://learn.microsoft.com/visualstudio/ide/mcp-servers) bakınız.

Her MCP sunucusu kendi yapılandırma gereksinimlerine sahiptir (bağlantı dizeleri, kimlik doğrulama vb.), ancak her iki IDE’de de kurulum deseni aynıdır.

### 1. 📚 Microsoft Learn Docs MCP Sunucusu

[![VS Code’da Yükle](https://img.shields.io/badge/VS_Code-Install_Microsoft_Docs_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D) [![VS Code Insiders’da Yükle](https://img.shields.io/badge/VS_Code_Insiders-Install_Microsoft_Docs_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/mcp)  

**Ne yapar**: Microsoft Learn Docs MCP Sunucusu, AI asistanlarına Model Context Protocol üzerinden resmi Microsoft dokümantasyonuna gerçek zamanlı erişim sağlayan bulut tabanlı bir hizmettir. `https://learn.microsoft.com/api/mcp` adresine bağlanır ve Microsoft Learn, Azure dokümantasyonu, Microsoft 365 dokümantasyonu ve diğer resmi Microsoft kaynakları arasında anlamsal arama yapılmasını mümkün kılar.

**Neden faydalıdır**: "Sadece dokümantasyon" gibi görünse de, bu sunucu Microsoft teknolojilerini kullanan her geliştirici için kritik önemdedir. .NET geliştiricilerinin AI kodlama asistanlarıyla ilgili en büyük şikayetlerinden biri, en son .NET ve C# sürümleri hakkında güncel olmamalarıdır. Microsoft Learn Docs MCP Sunucusu, en güncel dokümantasyon, API referansları ve en iyi uygulamalara gerçek zamanlı erişim sağlayarak bu sorunu çözer. En son Azure SDK’larıyla çalışıyor, yeni C# 13 özelliklerini keşfediyor veya son teknoloji .NET Aspire kalıplarını uyguluyorsanız, bu sunucu AI asistanınızın doğru ve güncel bilgiye erişmesini sağlar ve modern kod üretmesine olanak tanır.

**Gerçek dünya kullanımı**: "Resmi Microsoft Learn dokümantasyonuna göre Azure konteyner uygulaması oluşturmak için az cli komutları nelerdir?" veya "ASP.NET Core’da Entity Framework’ü bağımlılık enjeksiyonu ile nasıl yapılandırırım?" Ya da "Bu kodu, Microsoft Learn Dokümantasyonundaki performans önerilerine uyup uymadığını kontrol et." Sunucu, gelişmiş anlamsal arama kullanarak Microsoft Learn, Azure dokümanları ve Microsoft 365 dokümantasyonu genelinde kapsamlı içerik sunar. En uygun bağlamsal bilgiyi bulmak için arama yapar ve her zaman yayımlandığı anda en son Microsoft dokümantasyonuna erişir. Maksimum 10 yüksek kaliteli içerik parçası, makale başlıkları ve URL’ler ile birlikte döner.

**Öne çıkan örnek**: Sunucu, Microsoft’un resmi teknik dokümantasyonuna karşı anlamsal arama yapan `microsoft_docs_search` aracını sunar. Yapılandırıldıktan sonra "ASP.NET Core’da JWT kimlik doğrulamasını nasıl uygularım?" gibi sorular sorabilir ve kaynak bağlantıları ile ayrıntılı, resmi yanıtlar alabilirsiniz. Arama kalitesi mükemmeldir çünkü bağlamı anlar – “konteynerler”i Azure bağlamında sorarsanız Azure Container Instances dokümantasyonunu; .NET bağlamında sorarsanız ilgili C# koleksiyon bilgilerini döner.

Bu özellikle hızla değişen veya yeni güncellenmiş kütüphaneler ve kullanım senaryoları için faydalıdır. Örneğin, bazı yakın projelerde .NET Aspire ve Microsoft.Extensions.AI’ın en son sürümlerindeki özelliklerden yararlanmak istedim. Microsoft Learn Docs MCP sunucusunu dahil ederek yalnızca API belgelerini değil, yeni yayımlanmış rehberlik ve adım adım açıklamaları da kullanabildim.

> **💡 Uzman İpucu**

### 2. ☁️ Azure MCP Sunucusu
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

**Ne yapar**: Azure MCP Sunucusu, tüm Azure ekosistemini AI iş akışınıza getiren 15'ten fazla özel Azure servis bağlayıcısının kapsamlı bir paketidir. Bu sadece tek bir sunucu değil – kaynak yönetimi, veritabanı bağlantısı (PostgreSQL, SQL Server), KQL ile Azure Monitor günlük analizi, Cosmos DB entegrasyonu ve çok daha fazlasını içeren güçlü bir koleksiyondur.

**Neden faydalıdır**: Sadece Azure kaynaklarını yönetmenin ötesinde, bu sunucu Azure SDK’ları ile çalışırken kod kalitesini önemli ölçüde artırır. Azure MCP’yi Agent modunda kullandığınızda, yalnızca kod yazmanıza yardımcı olmakla kalmaz – mevcut kimlik doğrulama kalıplarını, hata yönetimi en iyi uygulamalarını takip eden ve en yeni SDK özelliklerinden yararlanan *daha iyi* Azure kodu yazmanıza da yardımcı olur. Çalışabilir basit kodlar almak yerine, Azure’un üretim iş yükleri için önerdiği kalıplara uygun kod alırsınız.

**Ana modüller şunları içerir**:
- **🗄️ Veritabanı Bağlayıcıları**: Azure Database for PostgreSQL ve SQL Server için doğal dil ile doğrudan erişim
- **📊 Azure Monitor**: KQL destekli günlük analizi ve operasyonel içgörüler
- **🌐 Kaynak Yönetimi**: Tam Azure kaynak yaşam döngüsü yönetimi
- **🔐 Kimlik Doğrulama**: DefaultAzureCredential ve yönetilen kimlik kalıpları
- **📦 Depolama Hizmetleri**: Blob Storage, Kuyruk Depolama ve Tablo Depolama işlemleri
- **🚀 Konteyner Hizmetleri**: Azure Container Apps, Container Instances ve AKS yönetimi
- **Ve birçok özel bağlayıcı daha**

**Gerçek dünya kullanımı**: "Azure depolama hesaplarımı listele", "Son saatte Log Analytics çalışma alanımdaki hataları sorgula" veya "Doğru kimlik doğrulama ile Node.js kullanarak bir Azure uygulaması oluşturmama yardım et"

**Tam demo senaryosu**: İşte Azure MCP ile VS Code’da GitHub Copilot Azure eklentisinin gücünü birleştiren eksiksiz bir rehber. İkisi yüklüyken şu istemi verirseniz:

> "DefaultAzureCredential kimlik doğrulamasını kullanarak bir dosyayı Azure Blob Storage’a yükleyen bir Python betiği oluştur. Betik 'mycompanystorage' adlı Azure depolama hesabıma bağlanmalı, 'documents' adlı bir konteynere yükleme yapmalı, yüklemek için geçerli zaman damgasıyla bir test dosyası oluşturmalı, hataları düzgün yönetmeli ve bilgilendirici çıktı sağlamalı, Azure kimlik doğrulama ve hata yönetimi en iyi uygulamalarına uygun olmalı, DefaultAzureCredential kimlik doğrulamasının nasıl çalıştığını açıklayan yorumlar içermeli ve betik iyi yapılandırılmış fonksiyonlar ve dokümantasyon içermeli."

Azure MCP Sunucusu tam olarak şunu yapacak olan üretim kalitesinde eksiksiz bir Python betiği oluşturur:
- En son Azure Blob Storage SDK'sını uygun async kalıplarla kullanır
- DefaultAzureCredential’ı kapsamlı yedekleme zinciri açıklamasıyla uygular
- Azure’a özgü istisna türleriyle sağlam hata yönetimi içerir
- Kaynak yönetimi ve bağlantı işlemleri için Azure SDK en iyi uygulamalarını takip eder
- Ayrıntılı günlükleme ve bilgilendirici konsol çıktısı sağlar
- Fonksiyonlar, dokümantasyon ve tür ipuçları ile düzgün yapılandırılmış betik oluşturur

Bu durumun harika yanı, Azure MCP olmadan genel blob depolama kodu alabileceğiniz ama güncel Azure kalıplarını takip etmeyen kod almanızdır. Azure MCP ile, en son kimlik doğrulama yöntemlerini kullanan, Azure’a özgü hata senaryolarını yöneten ve Microsoft’un üretim uygulamaları için önerdiği uygulamaları takip eden kod elde edersiniz.

**Öne çıkan örnek**: `az` ve `azd` CLI komutlarını anlık kullanım için hatırlamakta zorlanıyorum. Benim için her zaman iki adımlı bir süreçtir: önce sözdizimini bakarım, sonra komutu çalıştırırım. Genellikle portalda dolaşıp işimi yapmayı tercih ederim çünkü CLI sözdizimini hatırlayamadığımı itiraf etmek istemem. Sadece ne istediğimi tarif edebilmek harika, hele bunu IDE’den çıkmadan yapabilmek çok daha iyi!

Başlamak için [Azure MCP deposundaki](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) harika kullanım örnekleri listesine göz atabilirsiniz. Kapsamlı kurulum rehberleri ve ileri düzey yapılandırma seçenekleri için [resmi Azure MCP dokümantasyonuna](https://learn.microsoft.com/azure/developer/azure-mcp-server/) bakabilirsiniz.

**Neden faydalıdır**: Bu sunucu, GitHub ile etkileşim şeklinizi tamamen değiştirir; tam platform deneyimini doğrudan geliştirme ortamınıza getirir. Sürekli olarak proje yönetimi, kod incelemeleri ve CI/CD takibi için VS Code ile GitHub.com arasında geçiş yapmak yerine, her şeyi doğal dil komutları ile kodunuzdan kopmadan halledebilirsiniz.

> **ℹ️ Not: Farklı 'Agent' Türleri**
> 
> Bu GitHub MCP Sunucusunu, GitHub'ın Kodlama Agenti (otomatik kodlama görevleri için atanan AI agent) ile karıştırmayın. GitHub MCP Sunucusu VS Code Agent modunda GitHub API entegrasyonu sunarken, GitHub Kodlama Agenti atandığında GitHub sorunları için çekme istekleri oluşturur.

**Ana yetenekleri şunlardır**:
- **⚙️ GitHub Actions**: Komple CI/CD pipeline yönetimi, iş akışı izleme ve artefakt işlemleri
- **🔀 Çekme İstekleri**: Çekme istekleri oluşturma, inceleme, birleştirme ve kapsamlı durum takibi
- **🐛 Sorunlar**: Tam sorun yaşam döngüsü yönetimi, yorumlama, etiketleme ve atama
- **🔒 Güvenlik**: Kod tarama uyarıları, gizli anahtar tespiti ve Dependabot entegrasyonu
- **🔔 Bildirimler**: Akıllı bildirim yönetimi ve depo abonelik kontrolü
- **📁 Depo Yönetimi**: Dosya işlemleri, dal yönetimi ve depo idaresi
- **👥 İşbirliği**: Kullanıcı ve organizasyon araması, takım yönetimi ve erişim kontrolü

**Gerçek dünya kullanımı**: "Özellik dalımdan çekme isteği oluştur", "Bu haftaki başarısız CI çalıştırmalarını göster", "Depolarım için açık güvenlik uyarılarını listele" veya "Organizasyonlarımda bana atanmış tüm sorunları bul"

**Tam demo senaryosu**: İşte GitHub MCP Sunucusunun yeteneklerini gösteren güçlü bir iş akışı:

> "Sprint incelememize hazırlanmalıyım. Bu hafta oluşturduğum tüm çekme isteklerini göster, CI/CD pipeline’larımızın durumunu kontrol et, ele almamız gereken güvenlik uyarılarının bir özetini oluştur ve 'feature' etiketi olan birleştirilmiş PR’lerden sürüm notları taslağı hazırlamama yardım et."

GitHub MCP Sunucusu şunları yapacak:
- Detaylı durum bilgisiyle yakın zamanda oluşturduğunuz çekme isteklerini sorgular
- İş akışı çalıştırmalarını analiz eder, başarısızlıkları veya performans sorunlarını vurgular
- Güvenlik tarama sonuçlarını derler ve kritik uyarıları önceliklendirir
- Birleştirilmiş PR’lerden bilgi çıkararak kapsamlı sürüm notları oluşturur
- Sprint planlama ve sürüm hazırlığı için uygulanabilir sonraki adımları sunar

**Öne çıkan örnek**: Kod inceleme iş akışları için bunu kullanmayı seviyorum. VS Code, GitHub bildirimleri ve çekme isteği sayfaları arasında sürekli geçiş yapmak yerine “Bana incelemem bekleyen tüm PR’ları göster” diyebiliyorum, sonra “PR #123’e kimlik doğrulama yönteminde hata yönetimi ile ilgili bir yorum ekle” diyorum. Sunucu GitHub API çağrılarını hallediyor, tartışma bağlamını koruyor ve daha yapıcı inceleme yorumları oluşturmama bile yardımcı oluyor.

**Kimlik doğrulama seçenekleri**: Sunucu, hem OAuth (VS Code’da sorunsuz) hem de Kişisel Erişim Tokenlarını destekler; sadece ihtiyacınız olan GitHub işlevselliğini etkinleştirecek şekilde yapılandırılabilir. Anında kurulum için uzak bir barındırılan hizmet olarak ya da tam kontrol için yerel Docker ile çalıştırabilirsiniz.

> **💡 İpucu**
> 
> MCP sunucu ayarlarınızda `--toolsets` parametresini yapılandırarak yalnızca ihtiyacınız olan araç setlerini etkinleştirip bağlam boyutunu küçültür ve AI araç seçimini iyileştirebilirsiniz. Örneğin, temel geliştirme iş akışları için MCP yapılandırma argümanlarınıza `"--toolsets", "repos,issues,pull_requests,actions"` ekleyin veya esas olarak GitHub izleme yetenekleri istiyorsanız `"--toolsets", "notifications, security"` kullanın.

### 5. 📝 MarkItDown MCP Server
[![VS Code'a Yükle](https://img.shields.io/badge/VS_Code-Install_MarkItDown_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D) [![VS Code Insiders'a Yükle](https://img.shields.io/badge/VS_Code_Insiders-Install_MarkItDown_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/markitdown)

**Ne yapar**: MarkItDown, çeşitli dosya formatlarını yüksek kaliteli Markdown'a dönüştüren kapsamlı bir belge dönüştürme sunucusudur; LLM tüketimi ve metin analizi iş akışları için optimize edilmiştir.

**Neden faydalı**: Modern dokümantasyon iş akışları için vazgeçilmez! MarkItDown, başlıklar, listeler, tablolar ve bağlantılar gibi kritik belge yapısını koruyarak etkileyici bir dosya formatı yelpazesini işleyebilir. Basit metin çıkarma araçlarının aksine, hem yapay zeka işlemesi hem de insan okunabilirliği için değerli olan anlamsal anlamı ve biçimlendirmeyi korumaya odaklanır.

**Desteklenen dosya formatları**:
- **Office Belgeleri**: PDF, PowerPoint (PPTX), Word (DOCX), Excel (XLSX/XLS)
- **Medya Dosyaları**: Görseller (EXIF meta verisi ve OCR ile), Ses (EXIF meta verisi ve konuşma yazıya dökme ile)
- **Web İçerikleri**: HTML, RSS beslemeleri, YouTube URL'leri, Wikipedia sayfaları
- **Veri Formatları**: CSV, JSON, XML, ZIP dosyaları (içerikleri yinelemeli olarak işler)
- **Yayın Formatları**: EPub, Jupyter defterleri (.ipynb)
- **E-posta**: Outlook mesajları (.msg)
- **Gelişmiş**: Gelişmiş PDF işleme için Azure Document Intelligence entegrasyonu

**Gelişmiş yetenekler**: MarkItDown, OpenAI istemcisi sağlandığında LLM destekli görsel açıklamalar, gelişmiş PDF işleme için Azure Document Intelligence, konuşma içeriği için ses yazıya dökme ve ek dosya formatlarına uzanmak için eklenti sistemi destekler.

**Gerçek dünya kullanımı**: "Bu PowerPoint sunumunu dokümantasyon sitemiz için Markdown’a dönüştür", "Bu PDF’den uygun başlık yapısı ile metni çıkar" veya "Bu Excel tablosunu okunabilir tablo formatına çevir"

**Öne çıkan örnek**: [MarkItDown dokümanlarından](https://github.com/microsoft/markitdown#why-markdown) alıntı yapmak gerekirse:

> Markdown, minimum işaretleme ve biçimlendirme ile düz metne çok yakındır ancak yine de önemli belge yapısını temsil etmenin bir yolunu sağlar. OpenAI’nin GPT-4o gibi ana akım LLM'ler, varsayılan olarak Markdown "konuşur" ve yanıtlarında sıklıkla Markdown kullanır. Bu, büyük miktarda Markdown formatlı metin üzerinde eğitildiklerini ve onu iyi anladıklarını gösterir. Ek bir fayda olarak, Markdown konvansiyonları aynı zamanda yüksek token verimliliğine sahiptir.

MarkItDown, belge yapısını koruma konusunda gerçekten iyidir; bu yapay zeka iş akışları için önemlidir. Örneğin, bir PowerPoint sunumu dönüştürürken, slayt organizasyonunu doğru başlıklarla tutar, tabloları Markdown tabloları olarak çıkarır, görseller için alt metni içerir ve hatta konuşmacı notlarını işler. Grafikler okunabilir veri tablolarına dönüştürülür ve ortaya çıkan Markdown, orijinal sunumun mantıksal akışını korur. Bu, sunum içeriğinin yapay zeka sistemlerine beslenmesi veya mevcut slaytlardan dokümantasyon oluşturmak için mükemmeldir.

**Ne yapar**: Microsoft Foundry MCP Server, geliştiricilere model katalogları, dağıtım yönetimi, Azure AI Search ile bilgi indekslemesi ve değerlendirme araçları dahil olmak üzere Azure'un AI ekosistemine kapsamlı erişim sağlar. Bu deneysel sunucu, AI geliştirme ile Azure’un güçlü AI altyapısı arasındaki boşluğu kapatarak AI uygulamalarını oluşturmayı, dağıtmayı ve değerlendirmeyi kolaylaştırır.

**Neden faydalı**: Bu sunucu, kurumsal düzeyde AI yeteneklerini doğrudan geliştirme iş akışınıza getirerek Azure AI servisleri ile çalışma şeklinizi dönüştürür. Azure portalı, dokümantasyon ve IDE arasında geçiş yapmak yerine modelleri keşfedebilir, servisleri dağıtabilir, bilgi tabanlarını yönetebilir ve doğal dil komutlarıyla AI performansını değerlendirebilirsiniz. Özellikle RAG (Retrieval-Augmented Generation) uygulamaları geliştiren, çoklu model dağıtımlarını yöneten veya kapsamlı AI değerlendirme hatları uygulayan geliştiriciler için güçlüdür.

**Ana geliştirici yetenekleri**:
- **🔍 Model Keşfi ve Dağıtımı**: Microsoft Foundry'nin model kataloğunu keşfedin, kod örnekleriyle detaylı model bilgisi alın ve modelleri Azure AI Servislerine dağıtın
- **📚 Bilgi Yönetimi**: Azure AI Search indeksleri oluşturun ve yönetin, belgeler ekleyin, indeksleyicileri yapılandırın ve gelişmiş RAG sistemleri kurun
- **⚡ AI Ajan Entegrasyonu**: Azure AI Ajanları ile bağlantı kurun, mevcut ajanları sorgulayın ve üretim ortamlarında ajan performansını değerlendirin
- **📊 Değerlendirme Çerçevesi**: Kapsamlı metin ve ajan değerlendirmeleri yapın, markdown raporları oluşturun ve AI uygulamaları için kalite güvencesi uygulayın
- **🚀 Prototip Araçları**: GitHub tabanlı prototip kurulum talimatlarını alın ve Microsoft Foundry Labs aracılığıyla ileri araştırma modellerine erişin

**Gerçek dünya geliştirici kullanımı**: "Uygulamam için Azure AI Servislerine Phi-4 modelini dağıt", "Dokümantasyon RAG sistemim için yeni bir arama indeksi oluştur", "Ajanımın yanıtlarını kalite metriklerine karşı değerlendir" veya "Karmaşık analiz görevlerim için en iyi mantık modelini bul"

**Tam demo senaryosu**: İşte güçlü bir AI geliştirme iş akışı:

> "Bir müşteri destek ajanı oluşturuyorum. Katalogdan iyi bir mantık modeli bulmama yardım et, Azure AI Servislerine dağıt, dokümantasyonumuzdan bir bilgi tabanı oluştur, yanıt kalitesini test etmek için bir değerlendirme çerçevesi kur ve ardından test için GitHub token ile entegrasyon prototipi oluşturmama yardım et."

Microsoft Foundry MCP Server şunları yapacak:
- Gereksinimlerinize göre optimal mantık modellerini önermek için model kataloğunu sorgular
- tercih ettiğiniz Azure bölgesi için dağıtım komutları ve kota bilgileri sağlar
- Dokümantasyonunuz için uygun şema ile Azure AI Search indeksleri kurar
- Kalite metrikleri ve güvenlik kontrolleri ile değerlendirme boru hatlarını yapılandırır
- Anında test için GitHub kimlik doğrulamalı prototip kodu oluşturur
- Belirli teknoloji yığınına uygun kapsamlı kurulum kılavuzları sunar

**Öne çıkan örnek**: Geliştirici olarak mevcut birçok LLM modeliyle ayak uydurmakta zorluk çekiyorum. Birkaç ana modeli biliyorum ama bazı verimlilik ve performans kazanımlarını kaçırıyormuş gibi hissediyorum. Ayrıca tokenlar ve kotalar yönetmesi stresli ve zor – doğru görev için doğru modeli seçip seçmediğimi veya bütçemi verimsizce harcıyor muyum hiçbir zaman bilemiyorum. Bu MCP Server’ı, bu gönderi için MCP Server önerileri almak üzere ekip arkadaşlarımla konuşurken James Montemagno’dan duydum ve kullanmak için heyecanlıyım! Model keşif yetenekleri, sıradan modellerin ötesine geçmek ve belirli görevlere optimize edilmiş modeller bulmak isteyen biri için özellikle etkileyici görünüyor. Değerlendirme çerçevesi ise gerçekten daha iyi sonuçlar aldığımı doğrulamama yardımcı olacak, sadece yenilik yapmak için bir şeyler denemek değil.

> **ℹ️ Deneysel Durum**
> 
> Bu MCP sunucusu deneyseldir ve aktif olarak geliştirilmekte olup, özellikler ve API'ler değişebilir. Azure AI özelliklerini keşfetmek ve prototip oluşturmak için idealdir, ancak üretim kullanımı için kararlılık gereksinimlerini doğrulayın.

## 📋 Sonuç

Model Context Protocol (MCP), geliştiricilerin AI asistanları ve harici araçlarla etkileşimini dönüştürüyor. Bu 10 Microsoft MCP sunucusu, standartlaştırılmış AI entegrasyonunun gücünü göstererek, geliştiricilerin güçlü harici yeteneklere erişirken iş akışlarında akış durumlarını korumalarını sağlar.

Kapsamlı Azure ekosistem entegrasyonundan, tarayıcı otomasyonu için Playwright ve belge işleme için MarkItDown gibi özel araçlara kadar bu sunucular, MCP'nin çeşitli geliştirme senaryolarında üretkenliği nasıl artırabileceğini örnekliyor. Standart protokol, bu araçların uyum içinde çalışmasını sağlayarak tutarlı bir geliştirme deneyimi oluşturur.

MCP ekosistemi gelişmeye devam ettikçe, toplulukla etkileşimde kalmak, yeni sunucuları keşfetmek ve özel çözümler geliştirmek, geliştirme verimliliğinizi en üst düzeye çıkarmanın anahtarı olacaktır. MCP'nin açık standart doğası, farklı satıcıların araçlarını karıştırıp eşleştirerek ihtiyaçlarınıza özel mükemmel iş akışını oluşturmanızı sağlar.
