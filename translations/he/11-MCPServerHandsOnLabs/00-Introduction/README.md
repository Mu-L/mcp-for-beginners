# מבוא לאינטגרציה עם מסדי נתונים ב-MCP

## 🎯 מה המעבדה הזו מכסה

מעבדה מבוא זו מספקת סקירה מקיפה של בניית שרתי Model Context Protocol (MCP) עם אינטגרציה למסד נתונים. תלמדו את מקרה העסקים, הארכיטקטורה הטכנית והיישומים במציאות דרך מקרה השימוש בניתוח נתוני קמעונאות של Zava Retail בכתובת https://github.com/microsoft/MCP-Server-and-PostgreSQL-Sample-Retail.

## סקירה

**Model Context Protocol (MCP)** מאפשר לעוזרי AI לגשת ולהתفاعل בצורה מאובטחת עם מקורות נתונים חיצוניים בזמן אמת. בשילוב אינטגרציה עם מסדי נתונים, MCP משחרר יכולות עוצמתיות ליישומי AI מבוססי נתונים.

מסלול הלמידה הזה מלמד לבנות שרתי MCP מוכנים לפרודקשן שמחברים עוזרי AI לנתוני מכירות קמעונאיים דרך PostgreSQL, עם יישום תבניות ארגוניות כמו Row Level Security, חיפוש סמנטי וגישה לנתונים משתפי-משנה.

## מטרות למידה

עם סיום המעבדה תוכל/י:

- **להגדיר** את Model Context Protocol ואת היתרונות המרכזיים שלו לאינטגרציה עם מסדי נתונים  
- **לזהות** רכיבים מרכזיים בארכיטקטורת שרת MCP עם מסדי נתונים  
- **להבין** את מקרה השימוש של Zava Retail ודרישות העסק  
- **להכיר** תבניות ארגוניות לגישה מאובטחת ומדרגת למסדי נתונים  
- **לרשום** את הכלים והטכנולוגיות המשמשים לאורך מסלול הלמידה הזה  

## 🧭 האתגר: AI פוגש נתוני אמת

### מגבלות AI המסורתי

עוזרי AI מודרניים הם עוצמתיים אך מתמודדים עם מגבלות משמעותיות בעבודה עם נתוני עסקים מהעולם האמיתי:

| **אתגר** | **תיאור** | **השפעה עסקית** |
|---------------|-----------------|-------------------|
| **ידע סטטי** | מודלים של AI מאומנים על מערכי נתונים קבועים אינם יכולים לגשת לנתוני עסקים עדכניים | תובנות מיושנות, הזדמנויות מבוזבזות |
| **צברי נתונים** | מידע נעול במסדי נתונים, APIs ומערכות ש-AI אינו יכול להגיע אליהן | ניתוח לא מלא, תהליכים מפוזרים |
| **מגבלות אבטחה** | גישה ישירה למסד נתונים מעלה דאגות אבטחה וציות | פריסה מוגבלת, הכנת נתונים ידנית |
| **שאילתות מורכבות** | משתמשי עסקים זקוקים לידע טכני כדי להפיק תובנות מנתונים | אימוץ מופחת, תהליכים לא יעילים |

### הפתרון של MCP

Model Context Protocol מתמודד עם אתגרים אלה על ידי:

- **גישה לנתונים בזמן אמת**: עוזרי AI שואלים מסדי נתונים ו-APIs חיים  
- **אינטגרציה מאובטחת**: גישה מבוקרת עם אימות והרשאות  
- **ממשק שפה טבעית**: משתמשי עסקים שואלים שאלות באנגלית פשוטה  
- **פרוטוקול סטנדרטי**: עובד על פני פלטפורמות AI וכלים שונים  

## 🏪 הכירו את Zava Retail: מקרה הלמידה שלנו https://github.com/microsoft/MCP-Server-and-PostgreSQL-Sample-Retail

במהלך מסלול הלמידה הזה נבנה שרת MCP עבור **Zava Retail**, רשת קמעונאות DIY בדיונית עם מספר סניפים. תרחיש ריאלי זה מדגים יישום MCP ברמת ארגון.

### הקשר עסקי

**Zava Retail** מפעילה:  
- **8 חנויות פיזיות** ברחבי מדינת וושינגטון (סיאטל, בלוויו, טאקומה, ספוקיין, אברט, רדמונד, קירקלאנד)  
- **חנות אונליין אחת** למכירות מסחר אלקטרוני  
- **קטלוג מוצר מגוון** הכולל כלים, חומרה, מוצרי גינה וחומרי בניין  
- **ניהול רב-רמה** עם מנהלי חנויות, מנהלי אזור ומנהלים בכירים  

### דרישות עסקיות

מנהלי החנויות והמנכ"לים זקוקים לניתוח מבוסס AI כדי:

1. **לנתח ביצועי מכירות** בכל החנויות ותקופות זמן  
2. **לעקוב אחר מלאי** ולזהות צרכי תוספת  
3. **להבין התנהגות לקוחות** ודפוסי רכישה  
4. **לגלות תובנות מוצר** דרך חיפוש סמנטי  
5. **ליצור דוחות** עם שאילתות בשפה טבעית  
6. **לשמור על אבטחת נתונים** עם בקרת גישה מבוססת תפקיד  

### דרישות טכניות

שרת ה-MCP חייב לספק:

- **גישה לנתונים משתפי-משנה** שבה מנהלי חנויות רואים רק את נתוני החנות שלהם  
- **שאילתות גמישות** התומכות בפעולות SQL מורכבות  
- **חיפוש סמנטי** לגילוי מוצרים והמלצות  
- **נתונים בזמן אמת** המשקפים את מצב העסק הנוכחי  
- **אימות מאובטח** עם אבטחת רמת שורה (RLS)  
- **ארכיטקטורה מדרגת** התומכת במספר משתמשים בו-זמנית

### רכיבים מרכזיים

#### **1. שכבת שרת MCP**
- **FastMCP Framework**: מימוש שרת MCP מודרני בפייתון  
- **רישום כלי עבודה**: הגדרות כלים דקלרטיביות עם בטיחות טיפוס  
- **הקשר בקשה**: ניהול זהות משתמש ומפגש  
- **טיפול בשגיאות**: ניהול שגיאות ורישום איתן  

#### **2. שכבת אינטגרציה למסד נתונים**
- **שרשור חיבורים**: ניהול יעיל של חיבורים עם asyncpg  
- **ספק סכימות**: גילוי סכימות טבלאיות דינמי  
- **מבצע שאילתות**: ביצוע מאובטח של SQL עם הקשר RLS  
- **ניהול עסקאות**: תאימות ACID וטיפול בהרחקה  

#### **3. שכבת אבטחה**
- **אבטחת רמת שורה**: RLS ב-PostgreSQL לבידוד נתונים משתפי-משנה  
- **זהות משתמש**: אימות והרשאות של מנהל החנות  
- **בקרת גישה**: הרשאות מדויקות ומעקב ביקורת  
- **אימות קלט**: מניעת SQL injection ואימות שאילתות  

#### **4. שכבת שיפור AI**
- **חיפוש סמנטי**: הטמעות וקטור למוצר גילוי  
- **אינטגרציה עם Azure OpenAI**: יצירת הטמעות טקסט  
- **אלגוריתמים לדמיון**: חיפוש דמיון קוסינוס באמצעות pgvector  
- **אופטימיזציית חיפוש**: אינדקס וטיוב ביצועים

## 🔒 אבטחה ופרטים על רב-שכבתיות

היישום שלנו שם דגש על אבטחה ברמת ארגון:

### אבטחת רמת שורה (RLS)

PostgreSQL RLS מבטיח בידוד נתונים:

```sql
-- Store managers see only their store's data
CREATE POLICY store_manager_policy ON retail.orders
  FOR ALL TO store_managers
  USING (store_id = get_current_user_store());

-- Regional managers see multiple stores
CREATE POLICY regional_manager_policy ON retail.orders
  FOR ALL TO regional_managers
  USING (store_id = ANY(get_user_store_list()));
```

### ניהול זהות משתמש

כל חיבור MCP כולל:  
- **מזהה מנהל חנות**: מזהה ייחודי להקשר RLS  
- **הקצאת תפקיד**: הרשאות ורמות גישה  
- **ניהול מפגש**: אסימוני אימות מאובטחים  
- **רישום ביקורת**: היסטוריית גישה מלאה  

### הגנת נתונים

רב־שכבות אבטחה:  
- **הצפנת חיבור**: TLS לכל חיבורי מסד הנתונים
