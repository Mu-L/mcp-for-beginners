#### אימות דוגמאות מול SDKs נוכחיים

- **TypeScript (03-GettingStarted/01-first-server/solution/typescript)**: הפקודה `npm install` פתרה את `@modelcontextprotocol/sdk@1.29.0`; `tsc --noEmit` עבר ללא שגיאות טיפוס — ממשקי `McpServer`/`StdioServerTransport` קיימים עדיין תקפים
- **Python (03-GettingStarted/01-first-server/solution/python)**: אותר ב-.venv מבודד עם `mcp[cli]` (1.27.2); `py_compile` עבר ו-`FastMCP.list_tools()` החזיר כראוי את כלי ה-`add` וה-`subtract`
- אושר שכל טווחי גרסאות הדוגמא של `@modelcontextprotocol/sdk` (`>=1.26.0` / `^1.26.0` / `^1.27.0`) מתעדכנים היטב לגרסה הנוכחית `1.29.0` ללא שינויים מפורקים בממשק API

### חיזוק אבטחה ותיקון חשיפות תלות

בוצע מעבר אבטחה מלא על כל רשומות התלויות וקוד המקור בדוגמאות, ולאחר מכן תוקנו כל אזהרות ה-npm המדווחות וממצא אבטחה אחד ברמת הקוד. לאחר התיקון, `npm audit` מדווח על **0 פרצות** בכל תיקיה שבדוקה.

#### פרצות אבטחה של תלות npm (עקיפות) — תוקנו

נותחו כל 15 הקבצים `package-lock.json` המחויבים. הפרצות היו מוגבלות לתלויות עקיפות שנמשכו על ידי כלי הפיתוח MCP Inspector, לקוח OpenAI, ו-SDK MCP; כולם כעת תוקנו ללא שבירת הדוגמאות:
- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/inspector** ו-**lab3/code/weather_mcp/inspector**: עודכן `@modelcontextprotocol/inspector` (`0.16.6` / `0.14.1` → `0.22.0`), שנתקנו אזהרות ה-`ajv`, `brace-expansion`, `diff`, `path-to-regexp` ו-`ws` המצורפות. נוסף קלט כפייתי ל-npm `overrides` המכריח תיקון ל-`shell-quote@1.8.4` לביטול האזהרה הקריטית שנותרה שנשאה `concurrently`; שני קבצי הנעילה חודשו (כעת ללא פרצות)
- **03-GettingStarted/samples/typescript**: `npm audit fix` פתר את בעיית התלות העקיפה `qs` (בינונית) באמצעות שחרור מתוקן
- **03-GettingStarted/samples/javascript**: `npm audit fix` פתר את בעיית התלות העקיפה `hono` (בינונית) באמצעות שחרור מתוקן
- **03-GettingStarted/03-llm-client/solution/typescript**: `npm audit fix` פתר את בעיית התלות העקיפה `form-data` (גבוהה) באמצעות שחרור מתוקן
- **03-GettingStarted/11-simple-auth/solution/typescript**: נוצר קובץ `package-lock.json` חסר לפרויקט כדי שיהיה ניתן לשחזרו ולבצע בו ביקורת (0 פרצות)

#### תיקון אבטחה ברמת קוד (OWASP A03: הזרקה)

- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/src/server.py**: הוסר `shell=True` מכלי `open_in_vscode`. הפקודה הקודמת `subprocess.run(["start", "", vscode_path, folder_path], shell=True)` אפשרה תווי מעטפת בנתיב התיקייה שיועברו ל-`cmd.exe` (וקטור הזרקת פקודה). כעת מפעיל ישירות את `Code.exe` הפותר עם התיקייה כארגומנט — ללא מעטפת — זה פונקציונלי שווה ובטוח

#### ביקורת תלות Python

- בוצע סריקה לכל קבוצות התלויות ב-Python עם `pip-audit`. ב-`05-AdvancedTopics` וב-`03-GettingStarted/samples/python` לא נמצאו **פרצות ידועות** (טווחי הגרסאות של `mcp` / `httpx` / `pydantic` / `python-dotenv` מתעדכנים לשחרורים מתוקנים נוכחיים)

- **Primitive לקוח חדש - Roots**: נוסף תיעוד מקיף על Primitive Roots של הלקוח, המאפשר לשרתים להבין גבולות מערכת קבצים והרשאות גישה
