# AGENTS.md

## סקירת הפרויקט

**MCP למתחילים** הוא תוכנית לימודים חינוכית בקוד פתוח ללימוד פרוטוקול הקשר למודלים (MCP) - מסגרת סטנדרטית לאינטראקציות בין מודלי AI ליישומי לקוח. מאגר זה מספק חומרים לימודיים מקיפים עם דוגמאות קוד מעשיות בשפות תכנות מרובות.

### טכנולוגיות מפתח

- **שפות תכנות**: C#, Java, JavaScript, TypeScript, Python, Rust
- **מסגרות & SDK**: 
  - MCP SDK (`@modelcontextprotocol/sdk`)
  - Spring Boot (Java)
  - FastMCP (Python)
  - LangChain4j (Java)
- **מסדי נתונים**: PostgreSQL עם תוסף pgvector
- **פלטפורמות ענן**: Azure (Container Apps, OpenAI, Content Safety, Application Insights)
- **כלי בנייה**: npm, Maven, pip, Cargo
- **תיעוד**: Markdown עם תרגום אוטומטי מרובה שפות (48+ שפות)

### ארכיטקטורה

- **11 מודולים מרכזיים (00-11)**: נתיב למידה עוקב מהבסיס לנושאים מתקדמים
- **מעבדות מעשיות**: תרגילים מעשיים עם קוד פתרון מלא בשפות מרובות
- **פרויקטים לדוגמה**: מימושי שרת ולקוח MCP עובדים
- **מערכת תרגום**: תהליך אוטומטי של GitHub Actions לתמיכה בריבוי שפות
- **משאבי תמונות**: ספריית תמונות מרכזית עם גרסאות מתורגמות

## פקודות התקנה

זהו מאגר הממוקד בתיעוד. רוב ההתקנה מתרחשת בתוך פרויקטים לדוגמה ומעבדות בודדות.

### התקנת המאגר

```bash
# שכפל את המאגר
git clone https://github.com/microsoft/mcp-for-beginners.git
cd mcp-for-beginners
```

### עבודה עם פרויקטים לדוגמה

פרויקטים לדוגמה נמצאים ב:
- `03-GettingStarted/samples/` - דוגמאות בשפה ספציפית
- `03-GettingStarted/01-first-server/solution/` - מימושי שרת ראשונים
- `03-GettingStarted/02-client/solution/` - מימושי לקוח
- `11-MCPServerHandsOnLabs/` - מעבדות אינטגרציה מקיפות למסד נתונים

כל פרויקט לדוגמה מכיל הוראות התקנה משלו:

#### פרויקטים ב-TypeScript/JavaScript
```bash
cd <project-directory>
npm install
npm start
```

#### פרויקטים ב-Python
```bash
cd <project-directory>
pip install -r requirements.txt
# או
pip install -e .
python main.py
```

#### פרויקטים ב-Java
```bash
cd <project-directory>
mvn clean install
mvn spring-boot:run
```

## זרימת פיתוח

### מבנה התיעוד

- **מודולים 00-11**: תוכן תוכנית הלימודים העיקרית בסדר עוקב
- **translations/**: גרסאות בשפות שונות (נוצרות אוטומטית, אין לערוך ישירות)
- **translated_images/**: גרסאות תמונות מתורגמות (נוצרות אוטומטית)
- **images/**: תמונות ומדגים מקוריים

### ביצוע שינויים בתיעוד

1. ערוך רק את קבצי ה-markdown באנגלית בספריות המודולים הראשיות (00-11)
2. עדכן תמונות בספריה `images/` במידת הצורך
3. הפעולה co-op-translator ב-GitHub תייצר באופן אוטומטי את התרגומים
4. התרגומים מתעדכנים אוטומטית עם דחיפה לענף הראשי

### עבודה עם תרגומים

- **תרגום אוטומטי**: תהליך GitHub Actions מטפל בכל התרגומים
- **אין לערוך ידנית** קבצים בספריית `translations/`
- מטה-נתוני תרגום מוטמעים בכל קובץ מתורגם
- שפות נתמכות: 48+ שפות כולל ערבית, סינית, צרפתית, גרמנית, הינדית, יפנית, קוריאנית, פורטוגזית, רוסית, ספרדית ועוד רבות

## קווי הנחיה לסגנון קוד

### סגנון תיעוד

- השתמשו בשפה ברורה, נגישה למתחילים
- כלול דוגמאות קוד בשפות שונות בעת הצורך
- עקוב אחר כללי Markdown הטובים:
  - השתמש בכותרות בסגנון ATX (`#` סימון)
  - השתמש בבלוקים מסומנים עם זיהוי שפה
  - כלול טקסט אלטרנטיבי תיאורי לתמונות
  - שמור על אורך שורות סביר (אין הגבלה נוקשה, אבל היגיוני)

### סגנון דוגמאות קוד

#### TypeScript/JavaScript
- השתמש במודולים ES (`import`/`export`)
- עקוב אחרי מצבי TypeScript מחמירים
- כלול צ'ק סוגים (type annotations)
- יעד ES2022

#### Python
- עקוב אחרי קווי הנחיה של PEP 8
- השתמש ברמזי טיפוס היכן שמתאים
- כלול דוקסטרינגס לפונקציות ומחלקות
- השתמש בתכונות מודרניות של Python (3.8+)

#### Java
- עקוב אחרי הקונבנציה של Spring Boot
- השתמש בתכונות Java 21
- הקפד על מבנה פרויקט Maven סטנדרטי
- כלול הערות Javadoc

## קווי הנחיה לתרומה

### תהליך בקשה למשיכה (Pull Request)

1. **שכפל וקלן**: שכפל את המאגר וקלן את השכפול המקומי
2. **צור ענף**: השתמש בשם ענף תיאורי (למשל: `fix/typo-module-3`, `add/python-example`)
3. **בצע שינויים**: ערוך רק את קבצי Markdown באנגלית (לא את התרגומים)
4. **בדוק מקומית**: ודא שה-Markdown מציג נכון
5. **שלח בקשה למשיכה**: השתמש בכותרות ותיאורים ברורים
6. **CLA**: חתום על הסכם רישיון תורמים של מיקרוסופט כשמתבקש

### פורמט כותרת PR

השתמש בכותרות ברורות ותיאוריות:
- `[Module XX] תיאור קצר` לשינויים במודול ספציפי
- `[Samples] תיאור` לשינויים בדוגמאות קוד
- `[Docs] תיאור` לעדכוני תיעוד כלליים

### מה לתרום

- תיקוני באגים בתיעוד או בדוגמאות קוד
- דוגמאות קוד חדשות בשפות נוספות
- הבהרות ושיפורים לתוכן קיים
- מקרי מבחן או דוגמאות מעשיות חדשות
- דיווחים על נושאים לא ברורים או שגויים

### מה לא לעשות

- אל תערוך ישירות קבצים בספריית `translations/`
- אל תערוך את ספריית `translated_images/`
- אל תוסיף קבצים בינאריים גדולים בלי תיאום
- אל תשנה קבצי זרימת עבודה של תרגום ללא תיאום
