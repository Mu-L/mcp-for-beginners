## 🔧 Razumijevanje MCP servera: Vodič za početnike

### Što su MCP serveri?

Kao početnik u Model Context Protocolu (MCP), možda se pitate: "Što je točno MCP server i zašto me treba zanimati?" Počnimo s jednostavnom analogijom.

Zamislite MCP servere kao specijalizirane pomoćnike koji pomažu vašem AI suputnik-u za kodiranje (kao GitHub Copilot) da se poveže s vanjskim alatima i uslugama. Baš kao što na telefonu koristite različite aplikacije za različite zadatke—jednu za vremensku prognozu, jednu za navigaciju, jednu za bankarstvo—MCP serveri omogućuju vašem AI asistentu da komunicira s različitim razvojnim alatima i uslugama.

### Problem koji MCP serveri rješavaju

Prije MCP servera, ako ste htjeli:
- Provjeriti svoje Azure resurse
- Kreirati GitHub issue
- Izvršiti upit u bazi podataka
- Pretražiti dokumentaciju

Morali biste prestati kodirati, otvoriti preglednik, otići na odgovarajuću web stranicu i ručno obaviti te zadatke. Stalno mijenjanje konteksta prekida vaš tok rada i smanjuje produktivnost.

### Kako MCP serveri transformiraju vaše razvojno iskustvo

Uz MCP servere, možete ostati u razvojnom okruženju (VS Code, Visual Studio itd.) i jednostavno zamoliti svog AI asistenta da obavi te zadatke. Na primjer:

**Umjesto ovog tradicionalnog tijeka rada:**
1. Prestanak kodiranja
2. Otvaranje preglednika
3. Navigacija do Azure portala
4. Pregled detalja računa za pohranu
5. Povratak u VS Code
6. Nastavak kodiranja

**Sada možete ovo:**
1. Pitajte AI: "Koji je status mojih Azure računa za pohranu?"
2. Nastavite kodirati koristeći pružene informacije

### Ključne prednosti za početnike

#### 1. 🔄 **Ostanite u svom stanju fokusa**
- Nema više prebacivanja između više aplikacija
- Održavajte fokus na kodu koji pišete
- Smanjite mentalno opterećenje upravljanja različitim alatima

#### 2. 🤖 **Koristite prirodni jezik umjesto složenih naredbi**
- Umjesto da pamtite SQL sintaksu, opišite koje podatke trebate
- Umjesto da pamtite Azure CLI naredbe, objasnite što želite postići
- Neka AI rukuje tehničkim detaljima dok se vi fokusirate na logiku

#### 3. 🔗 **Povežite više alata zajedno**
- Kreirajte moćne tijekove rada kombiniranjem različitih usluga
- Primjer: "Dohvati sve nedavne GitHub issue-e i kreiraj odgovarajuće Azure DevOps radne stavke"
- Izgradite automatizaciju bez pisanja složenih skripti

#### 4. 🌐 **Pristupite rastućem ekosustavu**
- Iskoristite servere koje su izgradili Microsoft, GitHub i druge tvrtke
- Kombinirajte alate različitih dobavljača bez problema
- Pridružite se standardiziranom ekosustavu koji radi s različitim AI asistentima

#### 5. 🛠️ **Učite kroz praktičan rad**
- Počnite s unaprijed izgrađenim serverima kako biste razumjeli koncepte
- Postupno kreirajte vlastite servere kako vam udobnost raste
- Koristite dostupne SDK-ove i dokumentaciju kao vodič

### Primjer iz stvarnog svijeta za početnike

Recimo da ste novi u web razvoju i radite na svom prvom projektu. Evo kako MCP serveri mogu pomoći:

**Tradicionalni pristup:**
```
1. Code a feature
2. Open browser → Navigate to GitHub
3. Create an issue for testing
4. Open another tab → Check Azure docs for deployment
5. Open third tab → Look up database connection examples
6. Return to VS Code
7. Try to remember what you were doing
```

**S MCP serverima:**
```
1. Code a feature
2. Ask AI: "Create a GitHub issue for testing this login feature"
3. Ask AI: "How do I deploy this to Azure according to the docs?"
4. Ask AI: "Show me the best way to connect to my database"
5. Continue coding with all the information you need
```

### Prednost industrijskog standarda

MCP postaje standard u cijeloj industriji, što znači:
- **Dosljednost**: Sličan doživljaj na različitim alatima i kod različitih tvrtki

- **Zaštita za budućnost**: Vještine i postavke se prenose između različitih AI asistenata
- **Zajednica**: Veliki ekosustav zajedničkog znanja i resursa

## Početak s MCP u VS Code i Visual Studio 🚀

Postavljanje ovih MCP servera jednostavno je ako koristite Visual Studio Code ili Visual Studio 2022 s GitHub Copilotom.

### Postavljanje u VS Code

Osnovni postupak u VS Code:

1. **Omogućite Agent Mode**: U VS Code-u prebacite se na Agent mode u prozoru Copilot Chat
2. **Konfigurirajte MCP servere**: Dodajte konfiguracije servera u datoteku settings.json vašeg VS Code-a
3. **Pokrenite servere**: Kliknite "Start" za svaki server koji želite koristiti
4. **Odaberite alate**: Izaberite koje MCP servere želite omogućiti za trenutnu sesiju

Za detaljne upute postavljanja pogledajte [VS Code MCP dokumentaciju](https://code.visualstudio.com/docs/copilot/copilot-mcp).

> **💡 Profesionalni savjet: Upravljajte MCP serverima kao profesionalac!**
> 
> Pregled ekstenzija u VS Code sada uključuje [praktičan novi UI za upravljanje instaliranim MCP serverima](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode)! Imate brz pristup za pokretanje, zaustavljanje i upravljanje svakim instaliranim MCP serverom koristeći jasan i jednostavan sučelje. Isprobajte!

### Postavljanje u Visual Studio 2022

Za Visual Studio 2022 (verzija 17.14 ili novija):

1. **Omogućite Agent Mode**: Kliknite na padajući izbornik "Ask" u prozoru GitHub Copilot Chat i odaberite "Agent"
2. **Kreirajte konfiguracijsku datoteku**: Kreirajte `.mcp.json` datoteku u direktoriju svog rješenja (preporučena lokacija: `<SOLUTIONDIR>\.mcp.json`)
3. **Konfigurirajte servere**: Dodajte konfiguracije MCP servera koristeći standardni MCP format
4. **Odobrenje alata**: Kada se od vas zatraži, odobrite alate koje želite koristiti s odgovarajućim dozvolama za opseg

Za detaljne upute o postavljanju u Visual Studio pogledajte [Visual Studio MCP dokumentaciju](https://learn.microsoft.com/visualstudio/ide/mcp-servers).

Svaki MCP server ima svoje zahtjeve za konfiguraciju (npr. veze, autentikaciju itd.), ali je obrazac postavljanja dosljedan u oba IDE-a.

**Što radi**: Microsoft Learn Docs MCP Server je usluga temeljena na oblaku koja AI asistentima pruža pristup službenoj Microsoftovoj dokumentaciji u stvarnom vremenu putem Model Context Protocola. Povezuje se s `https://learn.microsoft.com/api/mcp` i omogućuje semantičko pretraživanje unutar Microsoft Learn, Azure dokumentacije, Microsoft 365 dokumentacije i drugih službenih izvora tvrtke Microsoft.

**Zašto je koristan**: Iako na prvi pogled izgleda kao "samo dokumentacija", ovaj server je zapravo ključan za svakog programera koji koristi Microsoftove tehnologije. Jedna od najvećih pritužbi .NET programera na AI asistente za kodiranje jest da nisu ažurirani s najnovijim verzijama .NET-a i C#. Microsoft Learn Docs MCP Server to rješava tako što pruža pristup u stvarnom vremenu najnovijoj dokumentaciji, referencama API-ja i najboljim praksama. Bilo da radite s najnovijim Azure SDK-ovima, istražujete nove značajke C# 13 ili implementirate najsuvremenije .NET Aspire obrasce, ovaj server osigurava da vaš AI asistent ima pristup autoritativnim, ažuriranim informacijama potrebnima za generiranje točnog i modernog koda.

**Primjena u stvarnom svijetu**: "Koje su az cli naredbe za kreiranje Azure container aplikacije prema službenoj Microsoft Learn dokumentaciji?" ili "Kako konfigurirati Entity Framework s dependency injection u ASP.NET Core?" Ili kako o "Pregledaj ovaj kod da se uvjeriš da odgovara preporukama za performanse iz Microsoft Learn dokumentacije". Server pruža sveobuhvatno pokrivanje Microsoft Learn, Azure i Microsoft 365 dokumenata koristeći naprednu semantičku pretragu za pronalaženje najrelevantnijih informacija u kontekstu. Vraća do 10 kvalitetnih sadržaja s naslovima članaka i URL-ovima, uvijek pristupajući najnovijoj Microsoftovoj dokumentaciji čim je objavljena.

**Istaknuti primjer**: Server izlaže alat `microsoft_docs_search` koji izvršava semantičko pretraživanje naspram službene Microsoftove tehničke dokumentacije. Nakon konfiguracije, možete postavljati pitanja poput "Kako implementirati JWT autentikaciju u ASP.NET Core?" i dobiti detaljne službene odgovore s poveznicama na izvor. Kvaliteta pretraživanja je izvanredna jer razumije kontekst – upit o "containers" u Azure kontekstu vraća dokumentaciju o Azure Container Instances, dok isti pojam u .NET kontekstu vraća relevantne informacije o C# kolekcijama.

Ovo je posebno korisno za brzo mijenjajuće ili nedavno ažurirane biblioteke i primjere korištenja. Na primjer, u nekim nedavnim projekatima kodiranja želio sam iskoristiti značajke najnovijih izdanja .NET Aspire i Microsoft.Extensions.AI. Uključivanjem Microsoft Learn Docs MCP servera uspio sam iskoristiti ne samo API dokumentaciju, već i vodiče i smjernice koje su upravo objavljene.

### 2. ☁️ Azure MCP Server
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

**Što radi**: Azure MCP Server je sveobuhvatan paket od 15+ specijaliziranih Azure servisnih konektora koji donose cijeli Azure ekosustav u vaš AI radni tijek. Ovo nije samo jedan server – to je snažna kolekcija koja uključuje upravljanje resursima, povezivanje s bazama podataka (PostgreSQL, SQL Server), analizu logova Azure Monitora pomoću KQL-a, integraciju s Cosmos DB i još mnogo toga.

**Zašto je koristan**: Osim upravljanja Azure resursima, ovaj server značajno poboljšava kvalitetu koda pri radu s Azure SDK-ovima. Kada koristite Azure MCP u Agent modu, on vam ne pomaže samo pisati kod – pomaže vam pisati *bolji* Azure kod koji slijedi trenutne obrasce autentifikacije, najbolje prakse za rukovanje pogreškama i koristi najnovije značajke SDK-a. Umjesto da dobijete generički kod koji možda radi, dobijete kod koji prati preporučene Azure obrasce za produkcijske radne opterećenja.

**Ključni moduli uključuju**:
- **🗄️ Konektori za baze podataka**: Direktan pristup prirodnim jezikom Azure Database za PostgreSQL i SQL Server
- **📊 Azure Monitor**: Analiza logova pokretana KQL-om i operativni uvidi
- **🌐 Upravljanje resursima**: Potpuno upravljanje životnim ciklusom Azure resursa
- **🔐 Autentifikacija**: obrasci DefaultAzureCredential i managed identity
- **📦 Storage servisi**: operacije Blob Storagea, Queue Storagea i Table Storagea
- **🚀 Servisi za kontejnere**: Upravljanje Azure Container Apps, Container Instances i AKS
- **I mnogi drugi specijalizirani konektori**

**Primjena u stvarnom svijetu**: "Prikaži moje Azure storage račune", "Upitaj moj Log Analytics workspace za greške u zadnjem satu", ili "Pomozi mi izgraditi Azure aplikaciju koristeći Node.js s pravilnom autentifikacijom"

**Potpuni demo scenarij**: Evo potpunog vodiča koji pokazuje snagu kombiniranja Azure MCP s GitHub Copilot for Azure ekstenzijom u VS Codeu. Kad imate oboje instalirano i upitate:

> "Kreiraj Python skriptu koja upload-a datoteku u Azure Blob Storage koristeći DefaultAzureCredential autentifikaciju. Skripta treba spojiti se na moj Azure storage račun nazvan 'mycompanystorage', uploadati u kontejner nazvan 'documents', kreirati testnu datoteku sa trenutnim vremenskim žigom za upload, rukovati greškama na pristojan način i dati informativni ispis, slijediti Azure najbolje prakse za autentifikaciju i rukovanje greškama, uključiti komentare koji objašnjavaju kako DefaultAzureCredential autentifikacija funkcionira, te napraviti dobro strukturiranu skriptu s odgovarajućim funkcijama i dokumentacijom."

Azure MCP Server će generirati potpunu, za produkciju spremnu Python skriptu koja:
- Koristi najnoviji Azure Blob Storage SDK s pravilnim asinhronim obrascima
- Implementira DefaultAzureCredential s detaljnim objašnjenjem kompletne fallback lanca
- Uključuje robusno upravljanje pogreškama sa specifičnim Azure tipovima iznimaka
- Slijedi najbolje prakse Azure SDK-a za upravljanje resursima i obradom veza
- Pruža detaljno evidentiranje i informativni ispis na konzolu
- Kreira pravilno strukturiranu skriptu s funkcijama, dokumentacijom i tipiziranjem

Izvanredno je to što bez Azure MCP-a možete dobiti generički kod za blob storage koji radi, ali ne prati trenutne Azure obrasce. S Azure MCP-om dobijete kod koji koristi najnovije metode autentifikacije, rukuje Azure-specifičnim pogreškama i slijedi Microsoftove preporuke za produkcijske aplikacije.

**Istaknuti primjer**: Mučio sam se s prisjećanjem točnih naredbi za `az` i `azd` CLI-jeve za ad-hoc korištenje. Uvijek je proces u dva koraka: prvo naći sintaksu, pa pokrenuti naredbu. Često bih samo ušao u portal i kliknuo da obavim posao jer nisam htio priznati da ne mogu zapamtiti sintaksu CLI-ja. Mogućnost da samo opišem što želim je nevjerojatna, a još bolje što to mogu učiniti bez napuštanja IDE-a!

Postoji odličan popis scenarija u [Azure MCP repozitoriju](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) za početak. Za detaljne vodiče za postavljanje i napredne konfiguracije, pogledajte [službenu Azure MCP dokumentaciju](https://learn.microsoft.com/azure/developer/azure-mcp-server/).

Ovo nije samo za osnovne operacije s repozitorijima – to je kompletni alatni set koji uključuje upravljanje GitHub Actions, tijekove povlačnih zahtjeva, praćenje problema, skeniranje sigurnosti, obavijesti i napredne mogućnosti automatizacije.

**Zašto je koristan**: Ovaj server mijenja način na koji komunicirate s GitHubom dovodeći cijelo GitHub iskustvo direktno u vaše razvojno okruženje. Umjesto da stalno prebacujete između VS Codea i GitHub.com stranice za upravljanje projektima, preglede koda i nadzor CI/CD-a, sve to možete obavljati putem naredbi na prirodnom jeziku dok ste usredotočeni na svoj kod.

> **ℹ️ Napomena: Različite vrste 'Agenta'**
> 
> Nemojte brkati ovaj GitHub MCP Server s GitHub Coding Agentom (AI agentom koji se može dodijeliti za automatske zadatke kodiranja). GitHub MCP Server radi unutar VS Code Agent moda kako bi pružio integraciju s GitHub API-jem, dok je GitHub Coding Agent zasebna značajka koja kreira pull requestove kada se dodijeli GitHub problemima.

**Ključne mogućnosti uključuju**:
- **⚙️ GitHub Actions**: Potpuno upravljanje CI/CD cjevovodima, nadzor tijekova i rukovanje artefaktima
- **🔀 Pull Requests**: Kreirajte, pregledajte, spajajte i upravljajte PR-ovima s detaljnim praćenjem statusa

**Zašto je koristan**: Neophodan za moderne radne tokove dokumentacije! MarkItDown obrađuje impresivan spektar formata datoteka uz očuvanje ključne strukture dokumenta poput naslova, popisa, tablica i poveznica. Za razliku od jednostavnih alata za ekstrakciju teksta, fokusira se na održavanje semantičkog značenja i formatiranja koje je vrijedno kako za AI obradu tako i za čitljivost ljudi.

**Podržani formati datoteka**:
- **Uredsko dokumenti**: PDF, PowerPoint (PPTX), Word (DOCX), Excel (XLSX/XLS)

- **Web sadržaj**: HTML, RSS feedovi, YouTube URL-ovi, Wikipedia stranice
- **Podatkovni formati**: CSV, JSON, XML, ZIP datoteke (rekurzivno obrađuje sadržaj)
- **Formati za izdavaštvo**: EPub, Jupyter bilježnice (.ipynb)
- **E-pošta**: Outlook poruke (.msg)
- **Napredno**: Integracija Azure Document Intelligence za poboljšanu obradu PDF-a

**Napredne mogućnosti**: MarkItDown podržava opisivanje slika pokretano LLM-om (ako je dostupan OpenAI klijent), Azure Document Intelligence za poboljšanu obradu PDF-a, transkripciju audio zapisa za govorni sadržaj te sustav dodataka za proširenje podrške za dodatne formate datoteka.

**Istaknuti primjer**: Citirajući [MarkItDown dokumentaciju](https://github.com/microsoft/markitdown#why-markdown):

> Markdown je izuzetno blizak običnom tekstu, s minimalnim markupom ili formatiranjem, ali i dalje pruža način za predstavljanje važne strukture dokumenta. Glavni LLM modeli, poput OpenAI-jeva GPT-4o, izvorno "govore" Markdown i često ga uključuju u svoje odgovore bez posebnog poticanja. To sugerira da su trenirani na ogromnim količinama teksta u Markdown formatu i dobro ga razumiju. Kao dodatna prednost, Markdown konvencije su također vrlo učinkovite u pogledu tokena.

MarkItDown je zaista dobar u očuvanju strukture dokumenta, što je važno za AI radne tokove. Na primjer, kod pretvorbe PowerPoint prezentacije, održava organizaciju slajdova s pravim naslovima, izvlači tablice kao Markdown tablice, uključuje alt tekst za slike i čak obrađuje bilješke predavača. Grafikoni se pretvaraju u čitljive podatkovne tablice, a rezultirajući Markdown čuva logički tok izvornog sadržaja. To ga čini savršenim za unos prezentacijskog sadržaja u AI sustave ili izradu dokumentacije iz postojećih slajdova.

Upravljanje tokenima i kvotama je bilo stresno i teško – nikada ne znam jesam li odabrao pravi model za pravi zadatak ili nepotrebno trošim budžet. Upravo sam čuo za ovaj MCP Server od Jamesa Montemagna dok sam istraživao preporuke MCP Servera sa suradnicima za ovaj članak i uzbuđen sam da ga upotrijebim! Mogućnosti otkrivanja modela izgledaju posebno impresivno za nekoga poput mene tko želi istražiti izvan uobičajenih opcija i pronaći modele optimizirane za određene zadatke. Okvir za evaluaciju trebao bi mi pomoći potvrditi da zapravo dobivam bolje rezultate, a ne samo isprobavam nešto novo radi samog isprobavanja.

## 📋 Zaključak

Model Context Protocol (MCP) mijenja način na koji programeri komuniciraju s AI asistentima i vanjskim alatima. Ova 10 Microsoft MCP poslužitelja demonstriraju moć standardizirane AI integracije, omogućujući besprijekorne tokove rada koji programerima omogućuju da ostanu u stanju toka dok pristupaju moćnim vanjskim mogućnostima.

Od sveobuhvatne integracije Azure ekosustava do specijaliziranih alata poput Playwrighta za automatizaciju preglednika i MarkItDowna za obradu dokumenata, ovi poslužitelji pokazuju kako MCP može poboljšati produktivnost u raznim razvojnim scenarijima. Standardizirani protokol osigurava da ti alati rade skladno, stvarajući koherentan razvojni doživljaj.

Kako ekosustav MCP nastavlja rasti, održavanje veze s zajednicom, istraživanje novih poslužitelja i izgradnja prilagođenih rješenja bit će ključ za maksimalnu razvojnu produktivnost. Otvorena standardna priroda MCP-a znači da možete kombinirati alate različitih dobavljača za stvaranje savršenog tijeka rada za vaše specifične potrebe.
