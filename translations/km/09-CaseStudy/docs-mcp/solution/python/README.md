# ជំនួយផែនការសិក្សា​ជាមួយ Chainlit & Microsoft Learn Docs MCP

## អ្វីដែលត្រូវមានជាមុន

- Python 3.8 ឬខ្ពស់ជាងនេះ
- pip (កម្មវិធីគ្រប់គ្រងផ្នែកបន្ថែម Python)
- ការចូលប្រើអ៊ីនធឺណិត ដើម្បីភ្ជាប់ទៅម៉ាស៊ីនបម្រើ Microsoft Learn Docs MCP

## ការដំឡើង

1. ចម្លងឃ្លាំងនេះ ឬទាញយកឯកសារគំរោង។
2. install រៀបចំអាសយដ្ឋានដែលត្រូវការ:

   ```bash
   pip install -r requirements.txt
   ```

## ការប្រើប្រាស់

### សេណារីយ៉ូ ១៖ សំណួរងាយៗទៅ Docs MCP
កម្មវិធីបន្ទាត់បញ្ជា ដែលភ្ជាប់ទៅម៉ាស៊ីនបម្រើ Docs MCP, បញ្ចូនសំណួរ និងបោះពុម្ពលទ្ធផល។

1. បើកស្គ្រីប:
   ```bash
   python scenario1.py
   ```
2. បញ្ចូលសំនួររបស់អ្នកអំពីឯកសារនៅលើបន្ទាត់បញ្ជា។

### សេណារីយ៉ូ ២៖ ជំនួយផែនការសិក្សា (កម្មវិធីវែប Chainlit)
ចំណុចចូលប្រព័ន្ធវែប (ប្រើ Chainlit) ដែលអនុញ្ញាតឲ្យអ្នកប្រើ បង្កើតផែនការសិក្សារប្រចាំសប្តាហ៍ផ្ទាល់ខ្លួនសម្រាប់ប្រធានបទបច្ចេកទេសណាមួយ។

1. ចាប់ផ្តើមកម្មវិធី Chainlit:
   ```bash
   chainlit run scenario2.py
   ```
2. បើក URL ក្នុងកុំព្យូទ័រផ្ទាល់ខ្លួនដែលបានបង្ហាញនៅក្នុងចុងក្រោម (ឧទាហរណ៍ http://localhost:8000) ក្នុងកម្មវិធីរុករក។
3. នៅក្នុងបង្អួចសម្តី ជ្រាបចូលប្រធានបទសិក្សារបស់អ្នក និងចំនួនសប្តាហ៍ដែលអ្នកចង់សិក្សា (ឧទាហរណ៍ "វិញ្ញាបនបត្រ AI-900, 8 សប្តាហ៍")។
4. កម្មវិធីនឹងឆ្លើយតបជាមួយផែនការសិក្សាប្រចាំសប្តាហ៍ ស្រុងនឹងមានតំណភ្ជាប់ទៅឯកសារ Microsoft Learn ទាក់ទង។

**អថេរបរិស្ថានត្រូវការជា៖**

ដើម្បីប្រើសេណារីយ៉ូ ២ (កម្មវិធីវែប Chainlit ជាមួយ Azure OpenAI) អ្នកត្រូវកំណត់អថេរបរិស្ថានទាំងនេះក្នុងឯកសារ `.env` ក្នុងថត `python`៖

```
AZURE_OPENAI_CHAT_DEPLOYMENT_NAME=
AZURE_OPENAI_API_KEY=
AZURE_OPENAI_ENDPOINT=
AZURE_OPENAI_API_VERSION=
```

បំពេញតម្លៃទាំងនេះជាមួយព័ត៌មានធនធាន Azure OpenAI របស់អ្នក មុនពេលរត់កម្មវិធី។

> [!TIP]
> អ្នកអាចបញ្ចូលម៉ូដែលផ្ទាល់ខ្លួនបានដោយស្រួល ប្រើ [Microsoft Foundry](https://ai.azure.com/) ។

### សេណារីយ៉ូ ៣៖ ឯកសារ docs ក្នុងកម្មវិធីកែសម្រួលជាមួយម៉ាស៊ីនបម្រើ MCP នៅ VS Code

ជំនួសការបើកតាបរុករកដើម្បីស្វែងរកឯកសារ អ្នកអាចយក Microsoft Learn Docs ទៅផ្ទាល់ក្នុង VS Code របស់អ្នកជាមួយម៉ាស៊ីនបម្រើ MCP។ វាអនុញ្ញាតឲ្យអ្នក៖
- ស្វែងរក និងអានឯកសារនៅក្នុង VS Code ដោយមិនចាកចេញពីបរិបទកូដ។
- ធ្វើយោងឯកសារ និងបញ្ចូលតំណភ្ជាប់ទៅ README ឬឯកសារបណ្តុះបណ្តាលរបស់អ្នក។
- ប្រើ GitHub Copilot និង MCP រួមគ្នា សម្រាប់ប្រព័ន្ធឯកសារដឹកនាំដោយ AI ដែលរលូន។

**ឧទាហរណ៍ការប្រើប្រាស់៖**
- បន្ថែមតំណភ្ជាប់យោងទៅ README ជាបន្ទាន់ ខណៈកំពុងសរសេរឯកសារបណ្តុះបណ្តាល ឬគម្រោង។
- ប្រើ Copilot ដើម្បីបង្កើតកូដ ហើយ MCP ដើម្បីស្វែងរក ឯកសារដែលពាក់ព័ន្ធភ្លាមៗ។
- តម្រួតគំនិតនៅក្នុងកម្មវិធីកែសម្រួល និងបង្កើនផលិតផលភាព។

> [!IMPORTANT]
> សូមធ្វើឲ្យប្រាកដថាអ្នកមានការកំណត់តម្លៃ[`mcp.json`](../../../../../../09-CaseStudy/docs-mcp/solution/scenario3/mcp.json) ត្រឹមត្រូវនៅក្នុងកន្លែងធ្វើការងារ របស់អ្នក (ទីតាំងគឺ `.vscode/mcp.json`)។

## ហេតុអ្វីបានជាត្រូវប្រើ Chainlit សម្រាប់សេណារីយ៉ូ ២?

Chainlit គឺជាស៊ែបវើសិនបើកចំហថ្មីសម្រាប់ការបង្កើតកម្មវិធីវែបដែលផ្តោតលើការសន្ទនា។ វាធ្វើឲ្យងាយស្រួលក្នុងការបង្កើតផ្ទាំងប្រើប្រាស់ជាសម្តីដែលភ្ជាប់ទៅសេវាកម្មក្រោយដូចជា ម៉ាស៊ីនបម្រើ Microsoft Learn Docs MCP។ គម្រោងនេះប្រើ Chainlit ដើម្បីផ្តល់នូវវិធីសាស្រ្តងាយស្រួល និងមានអន្តរកម្មសម្រាប់បង្កើតផែនការសិក្សាប្រចាំខ្លួនក្នុងពេលពិត។ ដោយប្រើ Chainlit អ្នកអាចបង្កើត និងផ្សព្វផ្សាយឧបករណ៍chat ដែលបង្កើនផលិតផលភាព និងការសិក្សា។

## អ្វីដែលកម្មវិធីនេះធ្វើ

កម្មវិធីនេះអនុញ្ញាតឲ្យអ្នកប្រើបង្កើតផែនការសិក្សា​ផ្ទាល់ខ្លួន ដោយគ្រាន់តែបញ្ចូលប្រធានបទ និងរយៈពេលសិក្សា។ កម្មវិធីនឹងបង្ហាញចេញពីការបញ្ចូលរបស់អ្នក សួរសំណួរទៅម៉ាស៊ីនបម្រើ Microsoft Learn Docs MCP សម្រាប់មាតិកាដែលពាក់ព័ន្ធ ហើយរៀបចំលទ្ធផលជា ផែនការសិក្សាប្រចាំសប្តាហ៍ដែលមានរចនាសម្ព័ន្ធ។ ការណែនាំប្រចាំសប្តាហ៍នីមួយៗត្រូវបានបង្ហាញនៅក្នុងជជែក making it easy to follow and track your progress. ការរួមបញ្ចូលនេះធ្វើឲ្យអ្នកទទួលបានធនធានសិក្សាថ្មីៗ និងមានប្រសិទ្ធភាពជានិច្ច។
