## សង្ខេប

**Model Context Protocol (MCP)** ធ្វើអោយជំនួយការបញ្ញាសិប្បនិម្មិតអាចចូលប្រើ និងអន្តរកម្មជាមួយប្រភពទិន្នន័យខាងក្រៅដោយសុវត្ថិភាពក្នុងពេលវេលាពិតប្រាកដ។ នៅពេលរួមបញ្ចូលជាមួយការរួមបញ្ចូលមូលដ្ឋានទិន្នន័យ MCP បើកលទ្ធភាពដ៏ខ្លាំងសម្រាប់កម្មវិធី AI អាស្រ័យលើទិន្នន័យ។

ផ្លូវចំណេះដឹងនេះបង្ហាញអ្នកពីវិធីកសាងម៉ាស៊ីនបម្រើ MCP ដែលមានស្រាប់សម្រាប់ផលិតកម្ម ដែលភ្ជាប់ជំនួយការបញ្ញាសិប្បនិម្មិតទៅទិន្នន័យលក់រាយតាមរយៈ PostgreSQL ដោយអនុវត្តលំនាំស្ថាបត្យកម្មអាជីវកម្មដូចជា Row Level Security, ការស្វែងរកអត្ថន័យ និងការចូលប្រើទិន្នន័យពហុម៉ូត៍។

|-------------|-----------------|-------------------------|

### ដំណោះស្រាយ MCP

Model Context Protocol ដោះស្រាយបញ្ហាទាំងនេះដោយផ្តល់:

- **ការចូលប្រើទិន្នន័យពេលវេលាពិតប្រាកដ**: ជំនួយការត្រូវបានសុំទិន្នន័យពេលវេលាពិតប្រាកដពីមូលដ្ឋានទិន្នន័យ និង API
- **ការរួមបញ្ចូលសុវត្ថិភាព**: ការចូលប្រើត្រូវបានគ្រប់គ្រងជាមួយការផ្ទៀងផ្ទាត់ និងអនុសាសន៍
- **ចំណុចប្រទាក់ភាសាតាមធម្មជាតិ**: អ្នកប្រើអាជីវកម្មសួរបញ្ហា ដោយភាសាអង់គ្លេសសាមញ្ញ
- **ប្រព័ន្ធប្រតិបត្តិការមានបែបបទស្ដង់ដារ**: ប្រតិបត្តិការជាមួយវេទិកា និងឧបករណ៍ AI ផ្សេងៗ

## 🔒 សុវត្ថិភាព និងទីផ្សារ Multi-Tenancy

ការអនុវត្តរបស់យើងផ្ដល់សុវត្ថិភាពកម្រិតសាធារណៈ៖

### សុវត្ថិភាពកម្រិតជួរ (RLS)

PostgreSQL RLSធានាការបំបែកទិន្នន័យ៖

```sql
-- Store managers see only their store's data
CREATE POLICY store_manager_policy ON retail.orders
  FOR ALL TO store_managers
  USING (store_id = get_current_user_store());

-- Regional managers see multiple stores
CREATE POLICY regional_manager_policy ON retail.orders
  FOR ALL TO regional_managers
  USING (store_id = ANY(get_user_store_list()));
```

### ការគ្រប់គ្រងអត្តសញ្ញាណអ្នកប្រើ

ចំណុចភ្ជាប់ MCP នីមួយៗរួមបញ្ជាក់៖  
- **អត្តសញ្ញាណអ្នកគ្រប់គ្រងហាង**៖ លេខសម្គាល់សម្រាប់បរិបទ RLS  
- **ការចាត់តួនាទី**: សិទ្ធិ និងកម្រិតចូលប្រើ  
- **គ្រប់គ្រងសម័យ**: តួអក្សរផ្ទៀងផ្ទាត់សុវត្ថិភាព  
- **កំណត់ហេតុ Audit**: ប្រវត្តិការចូលប្រើពេញលេញ

### ការពារទិន្នន័យ

ស្រទាប់សុវត្ថិភាពច្រើន:  
- **ការអ៊ីនគ្រីបតភ្ជាប់**: TLS សម្រាប់ការតភ្ជាប់មូលដ្ឋានទិន្នន័យទាំងអស់
