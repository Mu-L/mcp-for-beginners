## 🔧 Porozumění MCP serverům: Průvodce pro začátečníky

### Co jsou MCP servery?

Jako začátečník s Model Context Protocol (MCP) se možná ptáte: „Co přesně je MCP server a proč by mě to mělo zajímat?“ Začněme jednoduchou analogií.

Představte si MCP servery jako specializované asistenty, kteří pomáhají vašemu AI kódovacímu společníkovi (jako je GitHub Copilot) navázat spojení s externími nástroji a službami. Stejně jako na svém telefonu používáte různé aplikace pro různé úkoly – jednu na počasí, druhou na navigaci, třetí na bankovnictví – MCP servery dávají vašemu AI asistentovi schopnost interagovat s různými vývojářskými nástroji a službami.

### Problém, který MCP servery řeší

Před existencí MCP serverů, pokud jste chtěli:
- Zkontrolovat zdroje Azure
- Vytvořit GitHub issue
- Dotázat se na databázi
- Prohledat dokumentaci

Museli jste přestat kódovat, otevřít prohlížeč, přejít na příslušnou webovou stránku a tyto úkoly vykonávat ručně. Tento neustálý přepínání kontextu přerušuje vaši koncentraci a snižuje produktivitu.

### Jak MCP servery mění váš vývojářský zážitek

S MCP servery můžete zůstat ve svém vývojovém prostředí (VS Code, Visual Studio apod.) a jednoduše požádat svého AI asistenta, aby tyto úkoly vyřešil. Například:

**Místo tohoto tradičního postupu:**
1. Přestat kódovat  
2. Otevřít prohlížeč  
3. Jít na Azure portál  
4. Vyhledat podrobnosti o úložišti  
5. Vrátit se do VS Code  
6. Pokračovat v kódování

**Nyní můžete udělat toto:**
1. Zeptat se AI: „Jaký je stav mých Azure úložišť?“  
2. Pokračovat v kódování s poskytnutými informacemi

## Začínáme s MCP ve VS Code a Visual Studio 🚀

Nastavení těchto MCP serverů je jednoduché, pokud používáte Visual Studio Code nebo Visual Studio 2022 s GitHub Copilot.

### Nastavení ve VS Code

Základní postup ve VS Code:

1. **Aktivujte Agent Mode**: Ve VS Code přepněte na Agent Mode v okně Copilot Chat  
2. **Nakonfigurujte MCP servery**: Přidejte konfigurace serverů do souboru settings.json VS Code  
3. **Spusťte servery**: Klikněte na tlačítko „Start“ u každého serveru, který chcete používat  
4. **Vyberte nástroje**: Zvolte MCP servery, které chcete zapnout pro aktuální relaci

Podrobné instrukce pro nastavení najdete v [dokumentaci MCP pro VS Code](https://code.visualstudio.com/docs/copilot/copilot-mcp).

> **💡 Profesionální tip: Spravujte MCP servery jako profík!**  
> 
> Pohled na rozšíření ve VS Code nyní obsahuje [pohodlné nové uživatelské rozhraní pro správu nainstalovaných MCP serverů](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode)! Máte rychlý přístup k spouštění, zastavování a správě všech nainstalovaných MCP serverů prostřednictvím jasného a jednoduchého rozhraní. Vyzkoušejte to!

### Nastavení ve Visual Studio 2022

Pro Visual Studio 2022 (verze 17.14 a novější):

1. **Aktivujte Agent Mode**: Klikněte na rozbalovací menu „Ask“ v okně GitHub Copilot Chat a vyberte „Agent“  
2. **Vytvořte konfigurační soubor**: Vytvořte soubor `.mcp.json` ve složce řešení (doporučené umístění: `<SOLUTIONDIR>\.mcp.json`)  
3. **Nakonfigurujte servery**: Přidejte konfigurace MCP serverů pomocí standardního MCP formátu  
4. **Schválení nástrojů**: Když budete vyzváni, schvalte nástroje a patřičná oprávnění

Podrobné pokyny najdete v [dokumentaci MCP pro Visual Studio](https://learn.microsoft.com/visualstudio/ide/mcp-servers).

Každý MCP server má své vlastní požadavky na konfiguraci (připojovací řetězce, autentizace apod.), ale vzor nastavení je konzistentní v obou IDE.

### 1. 📚 Microsoft Learn Docs MCP server

[![Instalovat ve VS Code](https://img.shields.io/badge/VS_Code-Install_Microsoft_Docs_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D) [![Instalovat ve VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Microsoft_Docs_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/mcp)

**Co dělá**: Microsoft Learn Docs MCP server je cloudová služba, která poskytuje AI asistentům přístup v reálném čase k oficiální dokumentaci Microsoftu přes Model Context Protocol. Připojuje se k `https://learn.microsoft.com/api/mcp` a umožňuje sémantické vyhledávání v Microsoft Learn, dokumentaci Azure, Microsoft 365 a dalších oficiálních zdrojích Microsoftu.

**Proč je užitečný**: I když to může vypadat jako „pouze dokumentace“, tento server je zásadní pro každého vývojáře používajícího technologie Microsoftu. Jednou z největších výtek vývojářů .NET na AI kódovací asistenty je, že nejsou aktuální s nejnovějšími verzemi .NET a C#. Microsoft Learn Docs MCP server tuto situaci řeší tím, že poskytuje přístup k nejaktuálnější dokumentaci, referencím API a osvědčeným postupům v reálném čase. Ať už pracujete s nejnovějšími SDK Azure, zkoumáte nové funkce C# 13, nebo implementujete moderní .NET Aspire vzory, tento server zajistí, že váš AI asistent má přístup k autoritativním, aktuálním informacím pro generování přesného a moderního kódu.

**Reálné použití**: „Jaké jsou az cli příkazy pro vytvoření Azure container app dle oficiální dokumentace Microsoft Learn?“ nebo „Jak nakonfigurovat Entity Framework s dependency injection v ASP.NET Core?“ Nebo třeba „Zkontroluj tento kód, jestli odpovídá doporučením výkonu v Microsoft Learn dokumentaci.“ Server poskytuje komplexní pokrytí napříč Microsoft Learn, dokumentací Azure a Microsoft 365 pomocí pokročilého sémantického vyhledávání, které najde nejrelevantnější informace v daném kontextu. Vrací až 10 kvalitních obsahových bloků s titulky článků a URL odkazy, vždy přistupuje k nejnovější dokumentaci Microsoftu, jakmile je publikována.

**Příklad v akci**: Server poskytuje nástroj `microsoft_docs_search`, který provádí sémantické vyhledávání v oficiální technické dokumentaci Microsoftu. Po konfiguraci se můžete ptát na otázky typu „Jak implementovat JWT autentizaci v ASP.NET Core?“ a získat podrobné oficiální odpovědi s odkazy na zdroje. Kvalita vyhledávání je vynikající, protože rozumí kontextu – dotaz na „containers“ v kontextu Azure vrátí dokumentaci Azure Container Instances, zatímco stejný termín v kontextu .NET vrátí relevantní informace o kolekcích v C#.

To je obzvláště užitečné u rychle se měnících nebo nedávno aktualizovaných knihoven a scénářů. Například v některých nedávných projektech jsem chtěl využít funkce z posledních verzí .NET Aspire a Microsoft.Extensions.AI. Přidáním Microsoft Learn Docs MCP serveru jsem mohl využít nejen API dokumentaci, ale i návody a pokyny, které právě byly publikovány.

Tento mód přímo využívá Microsoft Learn Docs MCP server k úklidu a modernizaci C# kódu s použitím nejnovějších vzorů a osvědčených postupů.

### 2. ☁️ Azure MCP server
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

**Co to dělá**: Azure MCP Server je komplexní sada více než 15 specializovaných konektorů služeb Azure, která přináší celý ekosystém Azure do vašeho AI pracovního toku. Nejde jen o jeden server – je to výkonná kolekce, která zahrnuje správu prostředků, připojení k databázím (PostgreSQL, SQL Server), analýzu logů Azure Monitoru pomocí KQL, integraci Cosmos DB a mnohem více.

**Proč je užitečný**: Kromě samotné správy prostředků Azure tento server výrazně zlepšuje kvalitu kódu při práci s Azure SDK. Pokud používáte Azure MCP v režimu Agenta, nepomáhá vám jen psát kód – pomáhá vám psát *lepší* Azure kód, který dodržuje aktuální vzory autentizace, osvědčené postupy pro zpracování chyb a využívá nejnovější funkce SDK. Místo generického kódu, který může fungovat, dostanete kód, který následuje doporučené vzory Azure pro produkční nasazení.

**Klíčové moduly zahrnují**:
- **🗄️ Konektory databází**: Přímý přístup v přirozeném jazyce k Azure Database pro PostgreSQL a SQL Server
- **📊 Azure Monitor**: Analýza logů a provozní přehledy poháněné KQL
- **🌐 Správa prostředků**: Kompletní správa životního cyklu Azure prostředků
- **🔐 Autentizace**: Vzory DefaultAzureCredential a spravované identity
- **📦 Služby úložiště**: Operace s Blob Storage, Queue Storage a Table Storage
- **🚀 Kontejnerové služby**: Azure Container Apps, Container Instances a správa AKS
- **A mnoho dalších specializovaných konektorů**

**Praktické použití**: „Vyjmenuj mé Azure účty úložiště“, „Prohledej moje pracovní prostory Log Analytics pro chyby z poslední hodiny“ nebo „Pomoz mi vytvořit Azure aplikaci pomocí Node.js s řádnou autentizací“.

**Úplné demo scénáře**: Zde je kompletní průchod, který ukazuje sílu kombinace Azure MCP s GitHub Copilot for Azure rozšířením ve VS Code. Když máte oba nainstalované a zadáte:

> „Vytvoř Python skript, který nahraje soubor do Azure Blob Storage pomocí autentizace DefaultAzureCredential. Skript by se měl připojit k mému Azure storage účtu jménem 'mycompanystorage', nahrát do kontejneru 'documents', vytvořit testovací soubor s aktuálním časovým razítkem k nahrání, zpracovat chyby srozumitelně a poskytnout informativní výstup, dodržovat osvědčené postupy Azure pro autentizaci a zpracování chyb, obsahovat komentáře vysvětlující fungování autentizace DefaultAzureCredential, a být dobře strukturovaný s vhodnými funkcemi a dokumentací.“

Azure MCP Server vygeneruje kompletní, produkčně připravený Python skript, který:
- Používá nejnovější Azure Blob Storage SDK s řádným asynchronním vzorem
- Implementuje DefaultAzureCredential s podrobným vysvětlením fallback řetězce
- Obsahuje robustní zpracování chyb s konkrétními typy výjimek Azure
- Dodržuje osvědčené postupy Azure SDK pro správu prostředků a připojení
- Poskytuje detailní logování a informativní výstup na konzoli
- Vytváří správně strukturovaný skript s funkcemi, dokumentací a typovými anotacemi

Co je na tom pozoruhodné, je, že bez Azure MCP můžete dostat jen generický kód pro blob storage, který funguje, ale nedodržuje aktuální vzory Azure. S Azure MCP získáte kód, který využívá nejnovější autentizační metody, zpracovává specifické scénáře chyb Azure a následuje doporučení Microsoftu pro produkční aplikace.

**Vybraný příklad**: Měl jsem problém si zapamatovat konkrétní příkazy pro `az` a `azd` CLI pro ad-hoc použití. Vždy to byla dvoukroková akce: nejdřív si vyhledat syntaxi, pak spustit příkaz. Často jsem jen vstoupil do portálu a kliknul pro dokončení práce, protože jsem nechtěl přiznat, že si nepamatuji syntaxi CLI. Možnost jednoduše popsat, co chci, je úžasná, a ještě lepší je to zvládnout přímo v IDE!

Skvělý seznam příkladů použití najdete v [Azure MCP úložišti](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server), který vám pomůže začít. Pro kompletní průvodce nastavením a pokročilé možnosti konfigurace navštivte [oficiální dokumentaci Azure MCP](https://learn.microsoft.com/azure/developer/azure-mcp-server/).

> **ℹ️ Poznámka: Různé typy „Agentů“**

GitHub MCP Server funguje v režimu Agent ve VS Code k integraci GitHub API, zatímco GitHub Coding Agent je samostatná funkce, která vytváří pull requesty po přiřazení k GitHub issues.

- **⚙️ GitHub Actions**: Kompletní správa CI/CD pipeline, sledování workflow a správa artefaktů

- **📁 Správa repozitářů**: Operace se soubory, správa větví a administrace repozitářů
- **👥 Spolupráce**: Vyhledávání uživatelů a organizací, správa týmů a kontrola přístupů

**Praktické použití**: „Vytvoř pull request z mé větev ‚feature‘“, „Ukáž mi všechny neúspěšné CI běhy tento týden“, „Vyjmenuj otevřené bezpečnostní upozornění pro mé repozitáře“ nebo „Najdi všechny úkoly přiřazené mně v mých organizacích“.

**Proč je to užitečné**: Nezbytné pro moderní pracovní postupy dokumentace! MarkItDown zvládá širokou škálu formátů souborů při zachování klíčové struktury dokumentu, jako jsou nadpisy, seznamy, tabulky a odkazy. Na rozdíl od jednoduchých nástrojů pro extrakci textu se soustředí na udržení sémantického významu a formátování, které je cenné jak pro zpracování AI, tak pro lidskou čitelnost.

**Podporované formáty souborů**:
- **Office dokumenty**: PDF, PowerPoint (PPTX), Word (DOCX), Excel (XLSX/XLS)
- **Mediální soubory**: Obrázky (s EXIF metadaty a OCR), Audio (s EXIF metadaty a přepisem řeči)
- **Webový obsah**: HTML, RSS kanály, URL YouTube, stránky Wikipedie
- **Datové formáty**: CSV, JSON, XML, ZIP soubory (obsah se zpracovává rekurzivně)
- **Publikační formáty**: EPub, Jupyter notebooky (.ipynb)
- **Email**: Outlook zprávy (.msg)
- **Pokročilé**: Integrace Azure Document Intelligence pro vylepšené zpracování PDF

**Pokročilé schopnosti**: MarkItDown podporuje popisy obrázků řízené LLM (pokud je k dispozici OpenAI klient), Azure Document Intelligence pro vylepšené zpracování PDF, přepis audia pro obsah řeči a pluginový systém pro rozšíření o další formáty souborů.

**Reálné použití**: „Převést tuto prezentaci PowerPoint do Markdown pro naše dokumentační stránky“, „Extrahovat text z tohoto PDF se správnou strukturou nadpisů“ nebo „Převést tento Excel list do čitelného formátu tabulky“

**Ukázkový příklad**: Cituji [MarkItDown dokumentaci](https://github.com/microsoft/markitdown#why-markdown):

> Markdown je extrémně blízký prostému textu, s minimálním značením nebo formátováním, ale stále poskytuje způsob, jak reprezentovat důležitou strukturu dokumentu. Hlavní LLM modely, jako je GPT-4o od OpenAI, nativně „mluví“ Markdownem a často bez vyzvání začleňují Markdown do svých odpovědí. To naznačuje, že byly trénovány na obrovském množství textů formátovaných v Markdownu a dobře mu rozumí. Jako vedlejší efekt jsou Markdown konvence také velmi efektivní z hlediska tokenů.

MarkItDown je velmi dobrý v zachování struktury dokumentu, což je důležité pro AI pracovní postupy. Například při převodu prezentace PowerPoint zachovává organizaci snímků s odpovídajícími nadpisy, extrahuje tabulky jako Markdown tabulky, zahrnuje alt text pro obrázky a dokonce zpracovává poznámky přednášejícího. Grafy jsou převedeny do čitelných datových tabulek a výsledný Markdown udržuje logický tok původní prezentace. To je ideální pro předávání obsahu prezentace AI systémům nebo tvorbu dokumentace ze stávajících snímků.

- **🔍 Objevování & nasazení modelů**: Prozkoumávejte katalog modelů Microsoft Foundry, získejte detailní informace s ukázkami kódu a nasazujte modely do služeb Azure AI
- **📚 Správa znalostí**: Vytvářejte a spravujte indexy Azure AI Search, přidávejte dokumenty, konfigurujte indexery a stavte sofistikované RAG systémy
- **⚡ Integrace AI agentů**: Připojujte se k Azure AI agentům, dotazujte se na existující agenty a hodnotte jejich výkon v produkčních scénářích
- **📊 Evaluační rámec**: Proveďte komplexní textové a agentní evaluace, generujte markdown reporty a implementujte zajištění kvality AI aplikací
- **🚀 Nástroje pro prototypování**: Získejte instalační instrukce pro prototypování založené na GitHubu a přístup k Microsoft Foundry Labs pro nejmodernější výzkumné modely

**Reálné použití vývojářem**: „Nasadit model Phi-4 do Azure AI služeb pro mou aplikaci“, „Vytvořit nový vyhledávací index pro můj dokumentační RAG systém“, „Vyhodnotit odpovědi mého agenta vůči metrikám kvality“, nebo „Najít nejlepší model dedukce pro mé složité analytické úkoly“

**Plná demo scénář**: Tady je silný vývojový workflow AI:

> „Stavím zákaznického podporného agenta. Pomoz mi najít dobrý dedukční model z katalogu, nasadit ho do Azure AI služeb, vytvořit znalostní bázi z naší dokumentace, nastavit evaluační rámec pro testování kvality odpovědí a pak mi pomoci s prototypováním integrace s GitHub tokenem pro testování.“

Microsoft Foundry MCP Server:
- Prohledá katalog modelů a doporučí optimální dedukční modely podle vašich požadavků
- Poskytne příkazy pro nasazení a informace o kvótách pro vámi preferovaný region Azure
- Nastaví indexy Azure AI Search s odpovídajícím schématem pro vaši dokumentaci
- Nakonfiguruje evaluační pipelines s metrikami kvality a bezpečnostními kontrolami
- Vygeneruje prototypovací kód s ověřováním GitHubem pro okamžité testování
- Poskytne komplexní instalační návody přizpůsobené vašemu konkrétnímu technologickému zásobníku

**Vybraný příklad**: Jako vývojář jsem měl problém držet krok s různými dostupnými modely LLM. Znám pár hlavních, ale cítil jsem, že mi něco uniká v produktivitě a efektivitě. A tokeny a kvóty jsou stresující a těžké na správu – nikdy nevím, jestli vybírám správný model pro správný úkol nebo jestli neplýtvám rozpočtem. Právě jsem slyšel o tomto MCP Serveru od Jamese Montemagna, když jsem se s týmy radil o doporučeních MCP serveru pro tento příspěvek, a mám radost, že ho budu používat! Schopnosti objevování modelů vypadají obzvlášť působivě pro někoho jako já, kdo chce prozkoumat jiný než obvyklé modely a najít ty, které jsou optimalizované pro konkrétní úkoly. Evaluační rámec by mi měl pomoci ověřit, že skutečně dosahuji lepších výsledků, a ne že jen zkouším něco nového pro samotnou změnu.

## 📋 Závěr

Model Context Protocol (MCP) mění způsob, jakým vývojáři pracují s AI asistenty a externími nástroji. Těchto 10 Microsoft MCP serverů ukazuje sílu standardizované AI integrace, která umožňuje plynulé workflow, drží vývojáře v jejich pracovním flow a zároveň umožňuje přístup k výkonným externím schopnostem.

Od komplexní integrace Azure ekosystému po specializované nástroje jako Playwright pro automatizaci prohlížeče a MarkItDown pro zpracování dokumentů, tyto servery ukazují, jak MCP může zvýšit produktivitu v různorodých vývojových scénářích. Standardizovaný protokol zajišťuje, že tyto nástroje spolupracují bez problémů a vytvářejí soudržný vývojový zážitek.

Jak se MCP ekosystém vyvíjí, je klíčové zůstat v kontaktu s komunitou, objevovat nové servery a vytvářet vlastní řešení pro maximální zvýšení produktivity vývoje. Otevřený standard MCP vám umožní kombinovat nástroje od různých dodavatelů a vytvořit ideální workflow pro vaše specifické potřeby.
