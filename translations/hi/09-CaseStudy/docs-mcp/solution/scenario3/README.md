# Scenario 3: VS Code में MCP Server के साथ इन-एडिटर डॉक्स

## अवलोकन

इस परिदृश्य में, आप सीखेंगे कि कैसे Microsoft Learn Docs को सीधे अपने Visual Studio Code पर्यावरण में MCP सर्वर का उपयोग करके लाया जाए। बार-बार ब्राउज़र टैब स्विच करने के बजाय आधिकारिक दस्तावेज़ों को खोजने, पढ़ने, और संदर्भित करने के लिए आप अपने एडिटर के अंदर सीधे पहुंच सकते हैं। यह तरीका आपके वर्कफ़्लो को सरल बनाता है, आपकी फोकस बनाए रखता है, और GitHub Copilot जैसे उपकरणों के साथ सहज एकीकरण सक्षम करता है।

- VS Code के अंदर कोडिंग वातावरण छोड़े बिना दस्तावेज़ खोजें और पढ़ें।
- दस्तावेज़ का संदर्भ लें और अपने README या कोर्स फ़ाइलों में सीधे लिंक डालें।
- GitHub Copilot और MCP को मिलाकर एक सहज, AI-संचालित दस्तावेज़ वर्कफ़्लो का उपयोग करें।

## सीखने के उद्देश्य

इस अध्याय के अंत तक, आप यह समझ पाएंगे कि VS Code के भीतर MCP सर्वर को सेट अप और उपयोग कैसे किया जाए ताकि आपकी दस्तावेज़ीकरण और विकास वर्कफ़्लो में सुधार हो सके। आप सक्षम होंगे:

- दस्तावेज़ खोज के लिए MCP सर्वर का उपयोग करने के लिए अपने कार्यक्षेत्र को कॉन्फ़िगर करें।
- VS Code के भीतर से सीधे दस्तावेज़ खोजें और जोड़ें।
- GitHub Copilot और MCP की शक्ति को मिलाकर एक अधिक उत्पादक, AI-सहायित वर्कफ़्लो बनाएं।

ये कौशल आपको केंद्रित रहने, दस्तावेज़ की गुणवत्ता सुधारने, और डेवलपर या तकनीकी लेखक के रूप में आपकी उत्पादकता बढ़ाने में मदद करेंगे।

## समाधान

इन-एडिटर दस्तावेज़ीकरण पहुंच प्राप्त करने के लिए, आप MCP सर्वर को VS Code और GitHub Copilot के साथ एकीकृत करने के कई चरणों का पालन करेंगे। यह समाधान कोर्स लेखकों, दस्तावेज़ लेखक और डेवलपर्स के लिए आदर्श है जो एडिटर में रहते हुए दस्तावेज़ और Copilot के साथ काम करना चाहते हैं।

- कोर्स या प्रोजेक्ट दस्तावेज़ीकरण लिखते हुए README में जल्दी से संदर्भ लिंक जोड़ें।
- कोड जेनरेट करने के लिए Copilot का उपयोग करें और प्रासंगिक दस्तावेज़ तुरंत खोजने और उद्धृत करने के लिए MCP का उपयोग करें।
- अपने एडिटर पर केंद्रित रहें और उत्पादकता बढ़ाएं।

### चरण-दर-चरण मार्गदर्शिका

शुरू करने के लिए, इन चरणों का पालन करें। प्रत्येक चरण के लिए, आप प्रक्रिया को विजुअली समझाने के लिए आसेट्स फ़ोल्डर से स्क्रीनशॉट जोड़ सकते हैं।

1. **MCP कॉन्फ़िगरेशन जोड़ें:**
   अपने प्रोजेक्ट रूट में, एक `.vscode/mcp.json` फ़ाइल बनाएं और निम्न कॉन्फ़िगरेशन जोड़ें:
   ```json
   {
     "servers": {
       "LearnDocsMCP": {
         "url": "https://learn.microsoft.com/api/mcp"
       }
     }
   }
   ```
   यह कॉन्फ़िगरेशन VS Code को बताता है कि [`Microsoft Learn Docs MCP server`](https://github.com/MicrosoftDocs/mcp) से कैसे कनेक्ट होना है।
   
   ![Step 1: Add mcp.json to .vscode folder](../../../../../../translated_images/hi/step1-mcp-json.c06a007fccc3edfa.webp)
    
2. **GitHub Copilot चैट पैनल खोलें:**
   यदि आपके पास GitHub Copilot एक्सटेंशन पहले से इंस्टॉल नहीं है, तो VS Code में एक्सटेंशंस व्यू पर जाएं और इसे इंस्टॉल करें। आप इसे सीधे [Visual Studio Code Marketplace](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot-chat) से डाउनलोड कर सकते हैं। फिर, साइडबार से Copilot चैट पैनल खोलें।

   ![Step 2: Open Copilot Chat panel](../../../../../../translated_images/hi/step2-copilot-panel.f1cc86e9b9b8cd1a.webp)

3. **एजेंट मोड सक्षम करें और टूल्स सत्यापित करें:**
   Copilot चैट पैनल में, एजेंट मोड सक्षम करें।

   ![Step 3: Enable agent mode and verify tools](../../../../../../translated_images/hi/step3-agent-mode.cdc32520fd7dd1d1.webp)

   एजेंट मोड सक्षम करने के बाद, सुनिश्चित करें कि MCP सर्वर उपलब्ध टूल्स की सूची में हो। यह सुनिश्चित करता है कि Copilot एजेंट प्रासंगिक जानकारी प्राप्त करने के लिए दस्तावेज़ सर्वर तक पहुँच सकता है।
   
   ![Step 3: Verify MCP server tool](../../../../../../translated_images/hi/step3-verify-mcp-tool.76096a6329cbfecd.webp)
4. **नई चैट शुरू करें और एजेंट को प्रॉम्प्ट करें:**
   Copilot चैट पैनल में नई चैट खोलें। अब आप अपने दस्तावेज़ीकरण संबंधी प्रश्नों के साथ एजेंट को प्रॉम्प्ट कर सकते हैं। एजेंट MCP सर्वर का उपयोग करके प्रासंगिक Microsoft Learn दस्तावेज़ सीधे आपके एडिटर में लाएगा।

   - *"मैं विषय X के लिए एक अध्ययन योजना लिखने की कोशिश कर रहा हूँ। मैं इसे 8 हफ्तों तक अध्ययन करूंगा, हर हफ्ते के लिए सुझाव दें कि मुझे कौन से कंटेंट लेना चाहिए।"*

   ![Step 4: Prompt the agent in chat](../../../../../../translated_images/hi/step4-prompt-chat.12187bb001605efc.webp)

5. **लाइव क्वेरी:**

   > चलिए एक लाइव क्वेरी लेते हैं Microsoft Foundry Discord में [#get-help](https://discord.gg/D6cRhjHWSC) सेक्शन से ([मूल संदेश देखें](https://discord.com/channels/1113626258182504448/1385498306720829572)):
   
   *"मैं Azure AI Foundry पर विकसित AI एजेंट्स के साथ एक मल्टी-एजेंट समाधान तैनात करने के तरीके पर उत्तर खोज रहा हूँ। मैंने देखा कि कॉपिलट स्टूडियो चैनल्स जैसे कोई सीधे तैनाती का तरीका नहीं है। तो, एंटरप्राइज उपयोगकर्ताओं के लिए इस तैनाती को पूरा करने के अलग-अलग तरीके क्या हैं ताकि वे संवाद कर सकें और काम कर सकें?
कई लेख/ब्लॉग उपलब्ध हैं जो कहते हैं कि हम Azure Bot सेवा का उपयोग कर सकते हैं जो MS टीम्स और Azure AI Foundry एजेंट्स के बीच एक पुल का काम कर सकता है, तो क्या यह काम करेगा यदि मैं Azure बोट सेटअप करता हूँ जो Azure AI Foundry के ऑर्केस्ट्रेटर एजेंट से Azure फ़ंक्शन के माध्यम से जुड़ता है या मुझे प्रत्येक AI एजेंट के लिए Azure फ़ंक्शन बनाना होगा जो मल्टी-एजेंट समाधान का हिस्सा हो ताकि बॉट फ्रेमवर्क पर ऑर्केस्ट्रेशन हो सके? कोई अन्य सुझाव स्वागत योग्य हैं।"*

   ![Step 5: Live queries](../../../../../../translated_images/hi/step5-live-queries.49db3e4a50bea273.webp)

   एजेंट प्रासंगिक दस्तावेज़ लिंक और संक्षेप के साथ उत्तर देगा, जिन्हें आप सीधे अपने मार्कडाउन फ़ाइलों में डाल सकते हैं या अपने कोड में संदर्भ के रूप में उपयोग कर सकते हैं।
   
### नमूना क्वेरीज

यहाँ कुछ उदाहरण क्वेरीज हैं जिन्हें आप आज़मा सकते हैं। ये क्वेरीज दिखाएंगे कि कैसे MCP सर्वर और Copilot मिलकर VS Code छोड़ते ही त्वरित, संदर्भ-आधारित दस्तावेज़ीकरण और संदर्भ प्रदान कर सकते हैं:

- "मुझे Azure Functions ट्रिगर्स का उपयोग कैसे करना है दिखाएं।"
- "Azure Key Vault के आधिकारिक दस्तावेज़ के लिए लिंक डालें।"
- "Azure संसाधनों को सुरक्षित करने के सर्वोत्तम अभ्यास क्या हैं?"
- "Azure AI सेवाओं के लिए त्वरित प्रारंभ खोजें।"

ये क्वेरीज दिखाएंगे कि कैसे MCP सर्वर और Copilot मिलकर VS Code छोड़ते ही त्वरित, संदर्भ-आधारित दस्तावेज़ीकरण और संदर्भ प्रदान कर सकते हैं।
