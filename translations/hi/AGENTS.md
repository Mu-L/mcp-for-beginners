# AGENTS.md

## परियोजना अवलोकन

**MCP for Beginners** एक ओपन-सोर्स शैक्षिक पाठ्यक्रम है जो मॉडल कॉन्टेक्स्ट प्रोटोकॉल (MCP) सीखने के लिए है - जो AI मॉडल और क्लाइंट एप्लिकेशन के बीच इंटरैक्शन के लिए एक मानकीकृत फ्रेमवर्क है। यह रिपॉजिटरी कई प्रोग्रामिंग भाषाओं में हैंड्स-ऑन कोड उदाहरणों के साथ व्यापक शिक्षण सामग्री प्रदान करती है।

### मुख्य तकनीकें

- **प्रोग्रामिंग भाषाएँ**: C#, Java, JavaScript, TypeScript, Python, Rust
- **फ्रेमवर्क और SDKs**: 
  - MCP SDK (`@modelcontextprotocol/sdk`)
  - Spring Boot (Java)
  - FastMCP (Python)
  - LangChain4j (Java)
- **डेटाबेस**: pgvector एक्सटेंशन के साथ PostgreSQL
- **क्लाउड प्लेटफॉर्म**: Azure (Container Apps, OpenAI, Content Safety, Application Insights)
- **बिल्ड टूल्स**: npm, Maven, pip, Cargo
- **दस्तावेज़ीकरण**: स्वचालित बहुभाषी अनुवाद के साथ Markdown (48+ भाषाएँ)

### वास्तुकला

- **11 कोर मॉड्यूल (00-11)**: मूल सिद्धांतों से उन्नत विषयों तक अनुक्रमिक शिक्षण मार्ग
- **प्रैक्टिकल लैब्स**: कई भाषाओं में पूर्ण समाधान कोड के साथ व्यावहारिक अभ्यास
- **नमूना परियोजनाएँ**: कार्यशील MCP सर्वर और क्लाइंट कार्यान्वयन
- **अनुवाद प्रणाली**: बहुभाषी सहायता के लिए स्वचालित GitHub Actions वर्कफ़्लो
- **इमेज एसेट्स**: केंद्रीकृत छवियों की निर्देशिका और अनुवादित संस्करण

## सेटअप कमांड्स

यह एक दस्तावेज़ केंद्रित रिपॉजिटरी है। अधिकांश सेटअप व्यक्तिगत नमूना परियोजनाओं और लैब्स में होता है।

### रिपॉजिटरी सेटअप

```bash
# रिपॉजिटरी क्लोन करें
git clone https://github.com/microsoft/mcp-for-beginners.git
cd mcp-for-beginners
```

### नमूना परियोजनाओं के साथ काम करना

नमूना परियोजनाएं निम्नलिखित स्थानों पर स्थित हैं:
- `03-GettingStarted/samples/` - भाषा-विशिष्ट उदाहरण
- `03-GettingStarted/01-first-server/solution/` - पहला सर्वर कार्यान्वयन
- `03-GettingStarted/02-client/solution/` - क्लाइंट कार्यान्वयन
- `11-MCPServerHandsOnLabs/` - व्यापक डेटाबेस इंटीग्रेशन लैब्स

प्रत्येक नमूना परियोजना में अपनी सेटअप निर्देश होते हैं:

#### TypeScript/JavaScript परियोजनाएँ
```bash
cd <project-directory>
npm install
npm start
```

#### Python परियोजनाएँ
```bash
cd <project-directory>
pip install -r requirements.txt
# या
pip install -e .
python main.py
```

#### Java परियोजनाएँ
```bash
cd <project-directory>
mvn clean install
mvn spring-boot:run
```

## विकास कार्यप्रवाह

### दस्तावेज़ीकरण संरचना

- **मॉड्यूल 00-11**: अनुक्रमिक क्रम में मुख्य पाठ्यक्रम सामग्री
- **translations/**: भाषा-विशिष्ट संस्करण (स्वचालित उत्पन्न, सीधे संपादित न करें)
- **translated_images/**: स्थानीयकृत छवि संस्करण (स्वचालित उत्पन्न)
- **images/**: स्रोत छवियाँ और आरेख

### दस्तावेज़ीकरण में परिवर्तन करना

1. केवल रूट मॉड्यूल निर्देशिकाओं (00-11) में अंग्रेजी मार्कडाउन फ़ाइलों को संपादित करें
2. आवश्यक होने पर `images/` निर्देशिका में छवियाँ अपडेट करें
3. co-op-translator GitHub Action स्वतः अनुवाद उत्पन्न करेगा
4. मुख्य शाखा में पुश पर अनुवाद फिर से बनाए जाते हैं

### अनुवादों के साथ काम करना

- **स्वचालित अनुवाद**: GitHub Actions वर्कफ़्लो सभी अनुवादों को संभालता है
- `translations/` निर्देशिका की फ़ाइलों को मैन्युअली संपादित न करें
- अनुवाद मेटाडेटा प्रत्येक अनुवादित फ़ाइल में एम्बेडेड होता है
- समर्थित भाषाएँ: 48+ भाषाएँ जिनमें अरबी, चीनी, फ्रेंच, जर्मन, हिंदी, जापानी, कोरियाई, पुर्तगाली, रूसी, स्पेनिश आदि शामिल हैं

## कोड स्टाइल दिशानिर्देश

### दस्तावेज़ीकरण शैली

- स्पष्ट, शुरुआती के अनुकूल भाषा का उपयोग करें
- जहां लागू हो, कई भाषाओं में कोड उदाहरण शामिल करें
- मार्कडाउन की सर्वोत्तम प्रथाओं का पालन करें:
  - ATX-शैली हेडर (`#` सिंटैक्स) का उपयोग करें
  - भाषा पहचानकर्ताओं के साथ fenced कोड ब्लॉक का उपयोग करें
  - छवियों के लिए वर्णनात्मक alt टेक्स्ट शामिल करें
  - पंक्ति लंबाई उचित रखें (कठोर सीमा नहीं, लेकिन समझदारी से)

### कोड नमूना शैली

#### TypeScript/JavaScript
- ES मॉड्यूल्स (`import`/`export`) का उपयोग करें
- TypeScript सख्त मोड मानदंडों का पालन करें
- प्रकार एनोटेशन शामिल करें
- ES2022 लक्षित करें

#### Python
- PEP 8 शैली निर्देशों का पालन करें
- जहां उचित हो, प्रकार संकेतों का उपयोग करें
- फ़ंक्शंस और क्लासेस के लिए docstrings शामिल करें
- आधुनिक Python फीचर्स (3.8+) का उपयोग करें

#### Java
- Spring Boot मानदंडों का पालन करें
- Java 21 फीचर्स का उपयोग करें
- मानक Maven परियोजना संरचना का पालन करें
- Javadoc टिप्पणियाँ शामिल करें

## योगदान दिशानिर्देश

### पुल अनुरोध प्रक्रिया

1. **Fork और Clone करें**: रिपॉजिटरी का fork करें और अपने fork को स्थानीय रूप से क्लोन करें
2. **एक शाखा बनाएं**: वर्णनात्मक शाखा नामों का उपयोग करें (जैसे, `fix/typo-module-3`, `add/python-example`)
3. **परिवर्तन करें**: केवल अंग्रेजी मार्कडाउन फ़ाइलों को संपादित करें (अनुवाद नहीं)
4. **स्थानीय रूप से परीक्षण करें**: सुनिश्चित करें कि मार्कडाउन सही ढंग से प्रस्तुत हो रहा है
5. **PR सबमिट करें**: स्पष्ट शीर्षक और विवरण के साथ PR प्रस्तुत करें
6. **CLA**: Microsoft Contributor License Agreement पर हस्ताक्षर करें जब संकेत दिया जाए

### PR शीर्षक प्रारूप

स्पष्ट, वर्णनात्मक शीर्षक का उपयोग करें:
- `[Module XX] संक्षिप्त विवरण` मॉड्यूल-विशिष्ट परिवर्तनों के लिए
- `[Samples] विवरण` नमूना कोड परिवर्तनों के लिए
- `[Docs] विवरण` सामान्य दस्तावेज़ीकरण अपडेट के लिए

### क्या योगदान करें

- दस्तावेज़ या कोड नमूनों में बग फिक्स
- अतिरिक्त भाषाओं में नए कोड उदाहरण
- मौजूद सामग्री की स्पष्टिकरण और सुधार
- नए केस स्टडीज या व्यावहारिक उदाहरण
- अस्पष्ट या गलत सामग्री के लिए मुद्दा रिपोर्ट

### क्या न करें

- `translations/` निर्देशिका में सीधे फ़ाइलें संपादित न करें

- बिना चर्चा के बड़े बायनरी फाइल न जोड़ें
