### MCP विनिर्देशन संरेखण और नमूना सत्यापन

हाल के **MCP Specification 2025-11-25** और नवीनतम आधिकारिक SDKs के खिलाफ पाठ्यक्रम का सत्यापन किया गया, फिर पुराने विनिर्देशन संदर्भों को सही किया गया और पुष्टि की गई कि मुख्य नमूने अभी भी बनते और चलते हैं।

#### विनिर्देशन संस्करण सुधार (2025-06-18 / 2025-03-26 → 2025-11-25)

अंग्रेज़ी सामग्री को अपडेट किया गया जहां यह अभी भी पुराने spec संस्करण को *वर्तमान/नवीनतम* मानक के रूप में दर्शा रही थी, और `modelcontextprotocol.io` विनिर्देशन पथों की ओर लिंक दोबारा जोड़े गए:
- **05-AdvancedTopics/mcp-security/README.md**: "Current Standard" बैनर, परिचय, कोर सुरक्षा सिद्धांतों का शीर्षक, अनिवार्य आवश्यकताओं का शीर्षक, Microsoft Entra ID सेक्शन, संदर्भ और संसाधन लिंक, और समापन सुरक्षा नोटिस (8 संदर्भ) को 2025-11-25 में अपडेट किया गया
- **05-AdvancedTopics/mcp-transport/README.md**: अतिरिक्त संसाधनों के spec लिंक और "Current Standard" बैनर को 2025-11-25 में अपडेट किया गया
- **05-AdvancedTopics/mcp-realtimesearch/README.md**: पुराने `2025-03-26` security-and-trust लिंक को वर्तमान 2025-11-25 सुरक्षा सर्वोत्तम प्रथाओं पृष्ठ से बदल दिया गया
- **03-GettingStarted/14-sampling/README.md**: आधिकारिक सैम्पलिंग दस्तावेज़ लिंक को 2025-11-25 में अपडेट किया गया
- **03-GettingStarted/05-stdio-server/README.md**: वर्तमानकालीन "current MCP specification" संदर्भ और अतिरिक्त संसाधनों का spec लिंक 2025-11-25 में अपडेट किया गया (ऐतिहासिक SSE-डिप्रिकेशन नोट्स बिल्कुल यथावत रहें)

#### वर्तमान SDKs के खिलाफ नमूना सत्यापन

- **TypeScript (03-GettingStarted/01-first-server/solution/typescript)**: `npm install` ने `@modelcontextprotocol/sdk@1.29.0` हल किया; `tsc --noEmit` बिना किसी टाइप त्रुटि के सफल रहा — मौजूदा `McpServer`/`StdioServerTransport` API वैध रहें
- **Python (03-GettingStarted/01-first-server/solution/python)**: अलग `.venv` में `mcp[cli]` (1.27.2) के साथ सत्यापित; `py_compile` सफल, और `FastMCP.list_tools()` ने सही ढंग से `add` और `subtract` टूल लौटाए
- पुष्टि की गई कि सभी नमूना `@modelcontextprotocol/sdk` संस्करण सीमा (`>=1.26.0` / `^1.26.0` / `^1.27.0`) वर्तमान `1.29.0` पर साफ-सुथरे हल होते हैं और कोई ब्रेकिंग API परिवर्तन नहीं हुए हैं

- **10-StreamliningAIWorkflows.../lab3/code/weather_mcp/pyproject.toml** और **lab4/code/github_mcp_server/pyproject.toml**: निश्चित पिन `mcp==1.23.0` को `mcp>=1.26.0` से बढ़ाया; दोनों के `uv.lock` फाइलों को पुनः जनरेट किया गया (`uv lock`) ताकि लॉकफाइल्स वर्तमान `mcp 1.27.2` को हल करें और मैनिफेस्ट्स के साथ सिंक में रहें

### सुरक्षा सुदृढ़ीकरण और निर्भरता कमज़ोरी सुधार

हर निर्भरता मैनिफेस्ट और नमूना स्रोत कोड पर संपूर्ण सुरक्षा जाँच चलाई गई, फिर सभी रिपोर्ट की गई npm advisories और एक कोड-स्तरीय खोज को सुधारा गया। सुधार के बाद, `npm audit` प्रत्येक ऑडिट किए गए डायरेक्टरी में **0 कमजोरियों** रिपोर्ट करता है।

#### npm निर्भरता कमजोरियां (अप्रत्यक्ष) — सुधारा गया

सभी 15 कमिट किए गए `package-lock.json` फाइलों का ऑडिट किया गया। कमजोरियां मुख्य रूप से MCP Inspector डिव टूल, OpenAI क्लाइंट, और MCP SDK द्वारा खींचे गए अप्रत्यक्ष निर्भरता में सीमित थीं; सभी को नमूनों को बिना तोड़े हल किया गया:
- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/inspector** और **lab3/code/weather_mcp/inspector**: `@modelcontextprotocol/inspector` (`0.16.6` / `0.14.1` → `0.22.0`) बढ़ाया गया, जिससे बंडल किए गए `ajv`, `brace-expansion`, `diff`, `path-to-regexp` और `ws` advisories साफ हो गए। `shell-quote@1.8.4` पर npm `overrides` प्रविष्टि जोड़ी गई ताकि `concurrently` द्वारा लाई गई शेष गंभीर सलाहकारता समाप्त हो; दोनों लॉकफाइल्स पुनः जनरेट किए गए (अब 0 कमजोरियां)
- **03-GettingStarted/samples/typescript**: `npm audit fix` ने अप्रत्यक्ष `qs` (मध्यम) को पैच किए गए संस्करण में अपडेट किया
- **03-GettingStarted/samples/javascript**: `npm audit fix` ने अप्रत्यक्ष `hono` (मध्यम) को पैच किए गए संस्करण में अपडेट किया
- **03-GettingStarted/03-llm-client/solution/typescript**: `npm audit fix` ने अप्रत्यक्ष `form-data` (उच्च) को पैच किए गए संस्करण में अपडेट किया
- **03-GettingStarted/11-simple-auth/solution/typescript**: गायब `package-lock.json` को जनरेट किया ताकि परियोजना पुनरुत्पाद्य और ऑडिट योग्य हो (0 कमजोरियां)

#### कोड-स्तरीय सुरक्षा सुधार (OWASP A03: Injection)

- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/src/server.py**: `open_in_vscode` टूल से `shell=True` हटाया गया। पिछला `subprocess.run(["start", "", vscode_path, folder_path], shell=True)` ने फोल्डर पथ में शेल मेटाकरेक्टर को `cmd.exe` द्वारा व्याख्यायित होने दिया (कमांड-इंजेक्शन वेक्टर)। अब यह सीधे हल किए गए `Code.exe` को फोल्डर के साथ तर्क के रूप में लॉन्च करता है — बिना शेल के — जो कार्यात्मक रूप से समकक्ष और सुरक्षित है

#### Python निर्भरता ऑडिट

- `pip-audit` के साथ हर Python आवश्यकताएँ सेट का ऑडिट किया गया। `05-AdvancedTopics` और `03-GettingStarted/samples/python` ने **कोई ज्ञात कमजोरियां नहीं** रिपोर्ट कीं (उनके `mcp` / `httpx` / `pydantic` / `python-dotenv` संस्करण सीमाएं वर्तमान पैच रिलीज़ों को हल करती हैं)

#### कोर अवधारणाओं में सुधार (01-CoreConcepts/)
- **नया क्लाइंट प्रिमिटिव - Roots**: Roots क्लाइंट प्रिमिटिव पर व्यापक दस्तावेज़ जोड़ा गया, जो सर्वरों को फाइलसिस्टम सीमाओं और पहुंच अधिकारों को समझने की अनुमति देता है
- **टूल एनोटेशन**: टूल व्यवहार एनोटेशन (`readOnlyHint`, `destructiveHint`) पर दस्तावेज़ जोड़ा गया ताकि बेहतर टूल निष्पादन निर्णय लिए जा सकें

- **URL Mode Elicitation**: सर्वर-प्रेरित बाहरी वेब इंटरैक्शन के लिए URL-आधारित elicitation पर दस्तावेज़ जोड़ा गया

### शुरुआत अनुभाग विस्तार – उन्नत सर्वर उपयोग और सरल प्रमाणीकरण

#### उन्नत सर्वर उपयोग (03-GettingStarted/10-advanced)
- **नया अध्याय जोड़ा गया**: एक व्यापक गाइड प्रस्तुत की गई जिसमें नियमित और निम्न-स्तरीय सर्वर आर्किटेक्चर दोनों के लिए उन्नत MCP सर्वर उपयोग को शामिल किया गया है।
  - **नियमित बनाम निम्न-स्तरीय सर्वर**: दोनों तरीकों के लिए पायथन और टाइपस्क्रिप्ट में विस्तृत तुलना और कोड उदाहरण।
  - **हैंडलर-आधारित डिजाइन**: स्केलेबल, लचीले सर्वर कार्यान्वयन के लिए हैंडलर-आधारित टूल/संसाधन/प्रांप्ट प्रबंधन का स्पष्टीकरण।
  - **व्यावहारिक पैटर्न**: वास्तविक दुनिया के परिदृश्य जहां निम्न-स्तरीय सर्वर पैटर्न उन्नत विशेषताओं और वास्तुकला के लिए लाभकारी हैं।

#### सरल प्रमाणीकरण (03-GettingStarted/11-simple-auth)
- **नया अध्याय जोड़ा गया**: MCP सर्वरों में सरल प्रमाणीकरण लागू करने के लिए चरण-दर-चरण मार्गदर्शिका।
  - **प्रमाणीकरण संकल्पनाएं**: प्रमाणीकरण बनाम अधिकार और क्रेडेंशियल हैंडलिंग की स्पष्ट व्याख्या।
  - **मूलभूत प्रमाणीकरण कार्यान्वयन**: पायथन (Starlette) और टाइपस्क्रिप्ट (Express) में मिडलवेयर-आधारित प्रमाणीकरण पैटर्न, कोड नमूनों के साथ।
  - **उन्नत सुरक्षा की ओर प्रगति**: सरल प्रमाणीकरण से शुरू करने और OAuth 2.1 तथा RBAC तक उन्नत बनाने के लिए मार्गदर्शन, उन्नत सुरक्षा मॉड्यूल के संदर्भ के साथ।

ये जोड़ व्यावहारिक, हाथ-से-निर्देशित मार्गदर्शन प्रदान करते हैं जो अधिक मजबूत, सुरक्षित, और लचीले MCP सर्वर कार्यान्वयन बनाने के लिए नींव के सिद्धांतों को उन्नत उत्पादन पैटर्न से जोड़ते हैं।

- **दृश्य संकेतक**: अनिवार्य आवश्यकताओं और अनुशंसित प्रथाओं का स्पष्ट चिन्हांकन


#### कोर अवधारणाएँ (01-CoreConcepts/) - पूर्ण आधुनिकीकरण
- **प्रोटोकॉल संस्करण अद्यतन**: वर्तमान MCP विशेषण 2025-06-18 के संदर्भ में अद्यतन, तिथि-आधारित संस्करण (YYYY-MM-DD प्रारूप)
- **वास्तुकला परिष्कार**: मेजबानों, क्लाइंट्स, और सर्वरों का वर्णन बेहतर किया गया ताकि वर्तमान MCP वास्तुकला पैटर्न प्रतिबिंबित हों
  - मेजबान अब स्पष्ट रूप से एआई अनुप्रयोग के रूप में परिभाषित, जो कई MCP क्लाइंट कनेक्शन का समन्वय करते हैं
  - क्लाइंट्स को प्रोटोकॉल कनेक्टर्स के रूप में वर्णित, जो एक-से-एक सर्वर संबंध बनाए रखते हैं
  - सर्वरों को स्थानीय और दूरस्थ परिनियोजन परिदृश्यों के साथ बेहतर बनाया गया
- **प्रिमिटिव पुनर्गठन**: सर्वर और क्लाइंट प्रिमिटिव्स का पूर्ण पुनर्गठन
  - सर्वर प्रिमिटिव्स: संसाधन (डेटा स्रोत), प्रॉम्प्ट (टेम्पलेट), टूल (कार्यशील फंक्शन) विस्तृत व्याख्याओं और उदाहरणों के साथ
  - क्लाइंट प्रिमिटिव्स: सैम्पलिंग (LLM पूर्णताएँ), डाटा एकत्रीकरण (यूज़र इनपुट), लॉगिंग (डिबगिंग/निगरानी)
  - वर्तमान खोज (`*/list`), पुनःप्राप्ति (`*/get`), और निष्पादन (`*/call`) विधि पैटर्न के साथ अद्यतन
- **प्रोटोकॉल वास्तुकला**: दो-स्तरीय वास्तुकला मॉडल पेश किया गया
  - डेटा लेयर: JSON-RPC 2.0 आधार के साथ जीवनचक्र प्रबंधन और प्रिमिटिव्स
  - ट्रांसपोर्ट लेयर: STDIO (स्थानीय) और स्ट्रीम करने योग्य HTTP के साथ SSE (डूरस्थ) ट्रांसपोर्ट मैकेनिज़म
- **सुरक्षा फ्रेमवर्क**: स्पष्ट उपयोगकर्ता सहमति, डेटा गोपनीयता संरक्षण, टूल निष्पादन सुरक्षा, और ट्रांसपोर्ट लेयर सुरक्षा सहित व्यापक सुरक्षा सिद्धांत
- **संचार पैटर्न्स**: प्रोटोकॉल संदेशों को अद्यतन किया गया ताकि प्रारंभ, खोज, निष्पादन, और सूचना प्रवाह दिखाए जा सकें
- **कोड उदाहरण**: मौजूदा MCP SDK पैटर्न को प्रतिबिंबित करने के लिए बहुभाषी उदाहरणों (.NET, Java, Python, JavaScript) का नवीनीकरण

#### सुरक्षा (02-Security/) - व्यापक सुरक्षा पुनर्गठन  
- **मानकों का संरेखण**: MCP विशेषण 2025-06-18 सुरक्षा आवश्यकताओं के साथ पूर्ण संरेखण

- **AI-विशिष्ट खतरा विश्लेषण**: आधुनिक AI हमले के वेक्टरों की समृद्ध कवरेज
  - विस्तृत प्रॉम्प्ट इंजेक्शन हमले के परिदृश्य असली जीवन उदाहरणों के साथ
  - टूल विषाक्तता तंत्र और "रग पुल" हमला पैटर्न

  - AI प्रॉम्प्ट शील्ड्स उन्नत पहचान, स्पॉटलाइटिंग, और डिलीमीटर तकनीकों के साथ
  - Azure सामग्री सुरक्षा एकीकरण पैटर्न
  - GitHub एडवांस्ड सुरक्षा के लिए सप्लाई चेन संरक्षण
- **उन्नत खतरा शमन**: विस्तृत सुरक्षा नियंत्रण के लिए
  - MCP-संकेतित सत्र अपहरण हमले परिदृश्य और क्रिप्टोग्राफ़िक सत्र ID आवश्यकताएँ
  - MCP प्रॉक्सी परिदृश्यों में भ्रमित डिप्टी समस्याओं के लिए स्पष्ट सहमति आवश्यकताएं
  - टोकन पासथ्रू कमजोरियों के लिए अनिवार्य सत्यापन नियंत्रण
- **सप्लाई चेन सुरक्षा**: फाउंडेशन मॉडल, एम्बेडिंग सेवा, संदर्भ प्रदाता, और तृतीय-पक्ष API सहित AI सप्लाई चेन कवरेज का विस्तार
- **फाउंडेशन सुरक्षा**: ज़ीरो ट्रस्ट आर्किटेक्चर और Microsoft सुरक्षा पारिस्थितिकी तंत्र सहित एंटरप्राइज़ सुरक्षा पैटर्न के साथ एकीकरण में सुधार
- **संसाधन संगठन**: विस्तृत संसाधन लिंक को प्रकार (आधिकारिक दस्तावेज़, मानक, शोध, Microsoft समाधान, कार्यान्वयन मार्गदर्शिकाएं) के अनुसार वर्गीकृत
