# MCP सुरक्षा: एआई सिस्टम के लिए व्यापक सुरक्षा

[![MCP Security Best Practices](../../../translated_images/hi/03.175aed6dedae133f.webp)](https://youtu.be/88No8pw706o)

_(इस पाठ का वीडियो देखने के लिए ऊपर चित्र पर क्लिक करें)_

सुरक्षा एआई सिस्टम डिजाइन का मूलभूत तत्व है, इसलिए हम इसे अपने दूसरे अनुभाग के रूप में प्राथमिकता देते हैं। यह माइक्रोसॉफ्ट के [Secure Future Initiative](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) के **Secure by Design** सिद्धांत के साथ मेल खाता है।

मॉडल कंटेक्स्ट प्रोटोकॉल (MCP) एआई-संचालित अनुप्रयोगों में शक्तिशाली नई क्षमताएं लाता है, जबकि पारंपरिक सॉफ़्टवेयर जोखिमों से परे अनन्य सुरक्षा चुनौतियां प्रस्तुत करता है। MCP सिस्टमों को स्थापित सुरक्षा चिंताओं (सुरक्षित कोडिंग, न्यूनतम विशेषाधिकार, सप्लाई चेन सुरक्षा) के साथ-साथ नए एआई-विशिष्ट खतरों का सामना करना पड़ता है जिनमें प्रॉम्प्ट इंजेक्शन, टूल विषाक्तता, सत्र हाइजैकिंग, कन्फ्यूज्ड डेपुटी हमले, टोकन पासथ्रू कमजोरियां और गतिशील क्षमता संशोधन शामिल हैं।

यह पाठ MCP कार्यान्वयन में सबसे महत्वपूर्ण सुरक्षा जोखिमों का अन्वेषण करता है—जिसमें प्रमाणीकरण, प्राधिकरण, अत्यधिक अनुमतियाँ, अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन, सत्र सुरक्षा, कन्फ्यूज्ड डेपुटी समस्याएं, टोकन प्रबंधन और सप्लाई चेन कमजोरियां शामिल हैं। आप इन खतरों को कम करने के लिए व्यवहार्य नियंत्रण और सर्वोत्तम प्रथाओं को सीखेंगे और माइक्रोसॉफ्ट समाधानों जैसे कि प्रॉम्प्ट शील्ड्स, Azure कंटेंट सेफ्टी, और GitHub एडवांस्ड सिक्योरिटी का लाभ उठाकर अपनी MCP तैनाती को मजबूत करेंगे।

## सीखने के उद्देश्य

इस पाठ के अंत तक, आप सक्षम होंगे:

- **MCP-विशिष्ट खतरों की पहचान करें**: MCP सिस्टमों में अनन्य सुरक्षा जोखिमों को पहचानें जैसे प्रॉम्प्ट इंजेक्शन, टूल विषाक्तता, अत्यधिक अनुमतियाँ, सत्र हाइजैकिंग, कन्फ्यूज्ड डेपुटी समस्याएं, टोकन पासथ्रू कमजोरियां, और सप्लाई चेन जोखिम
- **सुरक्षा नियंत्रण लागू करें**: मजबूत प्रमाणीकरण, न्यूनतम विशेषाधिकार पहुंच, सुरक्षित टोकन प्रबंधन, सत्र सुरक्षा नियंत्रण, और सप्लाई चेन सत्यापन जैसी प्रभावी रोकथाम लागू करें
- **माइक्रोसॉफ्ट सुरक्षा समाधान का उपयोग करें**: MCP वर्कलोड सुरक्षा के लिए माइक्रोसॉफ्ट प्रॉम्प्ट शील्ड्स, Azure कंटेंट सेफ्टी, और GitHub एडवांस्ड सिक्योरिटी को समझें और तैनात करें
- **टूल सुरक्षा का सत्यापन करें**: टूल मेटाडेटा सत्यापन महत्त्व समझें, गतिशील परिवर्तनों के लिए निगरानी करें, और अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन हमलों से रक्षा करें
- **सर्वोत्तम प्रथाओं को एकीकृत करें**: स्थापित सुरक्षा मूलभूत बातों (सुरक्षित कोडिंग, सर्वर हार्डनिंग, ज़ीरो ट्रस्ट) को MCP-विशिष्ट नियंत्रणों के साथ मिलाएं ताकि व्यापक सुरक्षा सुनिश्चित हो

# MCP सुरक्षा वास्तुकला और नियंत्रण

आधुनिक MCP कार्यान्वयन परतीय सुरक्षा दृष्टिकोणों की मांग करते हैं जो पारंपरिक सॉफ़्टवेयर सुरक्षा और एआई-विशिष्ट खतरों दोनों को संबोधित करते हैं। तेजी से विकसित होने वाली MCP विनिर्देश अपनी सुरक्षा नियंत्रणों को परिपक्व कर रही है, जिससे एंटरप्राइज़ सुरक्षा वास्तुकला और स्थापित सर्वोत्तम प्रथाओं के साथ बेहतर एकीकरण संभव होता है।

[Microsoft Digital Defense Report](https://aka.ms/mddr) से शोध दिखाता है कि **98% रिपोर्ट किए गए उल्लंघन मजबूत सुरक्षा स्वच्छता द्वारा रोके जा सकते हैं**। सबसे प्रभावी सुरक्षा रणनीति में मूल सुरक्षा प्रथाएं और MCP-विशिष्ट नियंत्रण एक साथ होते हैं—प्रमाणित बुनियादी सुरक्षा उपाय कुल सुरक्षा जोखिम को कम करने में सबसे प्रभावी होते हैं।

## वर्तमान सुरक्षा परिदृश्य

> **[!NOTE]** यह सूचना **5 फरवरी, 2026** तक के MCP सुरक्षा मानकों को दर्शाती है, जो **MCP Specification 2025-11-25** के अनुरूप है। MCP प्रोटोकॉल तेजी से विकसित हो रहा है, और भविष्य के कार्यान्वयन नए प्रमाणीकरण पैटर्न और उन्नत नियंत्रण पेश कर सकते हैं। नवीनतम मार्गदर्शन के लिए हमेशा वर्तमान [MCP Specification](https://spec.modelcontextprotocol.io/), [MCP GitHub repository](https://github.com/modelcontextprotocol), और [सुरक्षा सर्वोत्तम प्रथाएं दस्तावेज़](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) देखें।

## 🏔️ MCP सुरक्षा शिखर सम्मेलन कार्यशाला (शेरपा)

**व्यावहारिक सुरक्षा प्रशिक्षण के लिए**, हम अत्यधिक सिफारिश करते हैं **MCP सुरक्षा शिखर सम्मेलन कार्यशाला** (शेरपा) — माइक्रोसॉफ्ट Azure में MCP सर्वरों की सुरक्षा के लिए एक व्यापक मार्गदर्शित अभियान।

### कार्यशाला अवलोकन

[MCP सुरक्षा शिखर सम्मेलन कार्यशाला](https://azure-samples.github.io/sherpa/) एक सिद्ध "असुरक्षित → शोषण → सुधार → सत्यापन" विधि के माध्यम से व्यावहारिक, लागू करने योग्य सुरक्षा प्रशिक्षण प्रदान करती है। आप करेंगे:

- **तोड़फोड़ के माध्यम से सीखें**: जानबूझकर असुरक्षित सर्वरों का शोषण करके कमजोरियों का अनुभव करें
- **Azure-देशी सुरक्षा उपयोग करें**: Azure Entra ID, Key Vault, API Management, और AI Content Safety का लाभ उठाएं
- **रक्षा में गहराई अपनाएं**: शिविरों के माध्यम से व्यापक सुरक्षा परतें बनाएं
- **OWASP मानकों का पालन करें**: हर तकनीक [OWASP MCP Azure Security Guide](https://microsoft.github.io/mcp-azure-security-guide/) से जुड़ी होती है
- **उत्पादन कोड प्राप्त करें**: परीक्षण किया गया काम करने वाला कार्यान्वयन लेकर जाएं

### MCP प्रमाणीकरण का विकास

MCP विनिर्देश प्रमाणीकरण और प्राधिकरण के दृष्टिकोण में काफी विकसित हुआ है:

- **मूल तरीका**: शुरुआती विनिर्देशों में डेवलपर्स को कस्टम प्रमाणीकरण सर्वर लागू करने की आवश्यकता थी, जिसमें MCP सर्वर OAuth 2.0 प्राधिकरण सर्वर के रूप में सीधे उपयोगकर्ता प्रमाणीकरण का प्रबंधन करते थे
- **वर्तमान मानक (2025-11-25)**: अपडेटेड विनिर्देश MCP सर्वरों को बाहरी पहचान प्रदाताओं (जैसे Microsoft Entra ID) को प्रमाणीकरण सौंपने की अनुमति देता है, जो सुरक्षा स्थिति में सुधार करता है और कार्यान्वयन जटिलता को कम करता है
- **ट्रांसपोर्ट लेयर सुरक्षा**: स्थानीय (STDIO) और रिमोट (Streamable HTTP) कनेक्शनों के लिए उचित प्रमाणीकरण पैटर्न के साथ सुरक्षित ट्रांसपोर्ट तंत्र का उन्नत समर्थन

## प्रमाणीकरण और प्राधिकरण सुरक्षा

### वर्तमान सुरक्षा चुनौतियां

आधुनिक MCP कार्यान्वयन कई प्रमाणीकरण और प्राधिकरण चुनौतियों का सामना करते हैं:

### जोखिम और खतरे

- **गलत कॉन्फ़िगर किया गया प्राधिकरण लॉजिक**: MCP सर्वरों में दोषपूर्ण प्राधिकरण कार्यान्वयन संवेदनशील डेटा को उजागर कर सकता है और पहुँच नियंत्रण को गलत लागू कर सकता है
- **OAuth टोकन समझौता**: स्थानीय MCP सर्वर टोकन चोरी हमलावरों को सर्वर के नकली होने और डाउनस्ट्रीम सेवाओं तक पहुंच हासिल करने में सक्षम बनाती है
- **टोकन पासथ्रू कमजोरियां**: गलत टोकन हैंडलिंग सुरक्षा नियंत्रणों को बाईपास करती है और जिम्मेदारी के अंतराल पैदा करती है
- **अत्यधिक अनुमतियाँ**: अधिक विशेषाधिकार प्राप्त MCP सर्वर न्यूनतम विशेषाधिकार सिद्धांत का उल्लंघन करते हैं और हमले की सतह बढ़ाते हैं

#### टोकन पासथ्रू: एक गंभीर विरोधी-पैटर्न

**टोकन पासथ्रू स्पष्ट रूप से वर्तमान MCP प्राधिकरण विनिर्देश में निषिद्ध है क्योंकि इसके गंभीर सुरक्षा निहितार्थ हैं:**

##### सुरक्षा नियंत्रण परामर्श
- MCP सर्वर और डाउनस्ट्रीम API महत्वपूर्ण सुरक्षा नियंत्रण (रेट लिमिटिंग, अनुरोध सत्यापन, ट्रैफिक निगरानी) लागू करते हैं जो उचित टोकन सत्यापन पर निर्भर हैं
- क्लाइंट से सीधे API टोकन उपयोग इन आवश्यक सुरक्षा उपायों को बाईपास करता है, जिससे सुरक्षा वास्तुकला कमजोर होती है

##### जवाबदेही और ऑडिटिंग चुनौतियां
- MCP सर्वर उन क्लाइंट्स को अलग नहीं कर सकते जो अपस्ट्रीम-जारी टोकन का उपयोग करते हैं, जिससे ऑडिट ट्रेल टूट जाता है
- डाउनस्ट्रीम संसाधन सर्वर लॉग्स गलत अनुरोध मूल स्रोत दिखाते हैं, न कि वास्तविक MCP सर्वर मध्यस्थ
- घटना जांच और अनुपालन ऑडिटिंग बहुत कठिन हो जाती है

##### डेटा निकासी जोखिम
- अनसत्यापित टोकन दावों के कारण चोरी किए गए टोकन वाले दुर्भावनापूर्ण अभिनेता MCP सर्वरों का उपयोग डेटा निकासी के लिए प्रॉक्सी के रूप में कर सकते हैं
- विश्वास की सीमाएं उल्लंघन की अनुमति देती हैं जो अभिप्रेत सुरक्षा नियंत्रणों को बाईपास करती हैं

##### मल्टी-सर्विस हमला वेक्टर
- समझौता किए गए टोकन कई सेवाओं द्वारा स्वीकार किए जाते हैं, जिससे जुड़े सिस्टमों में पार्श्व आंदोलन संभव होता है
- टोकन मूल की पुष्टि न होने पर सेवाओं के बीच विश्वास मान्यताएं उल्लंघन हो सकती हैं

### सुरक्षा नियंत्रण और निवारण

**महत्वपूर्ण सुरक्षा आवश्यकताएं:**

> **अनिवार्य**: MCP सर्वरों को कोई भी टोकन स्वीकार नहीं करना चाहिए जो स्पष्ट रूप से MCP सर्वर के लिए जारी नहीं किए गए हों

#### प्रमाणीकरण और प्राधिकरण नियंत्रण

- **कठोर प्राधिकरण समीक्षा**: MCP सर्वर प्राधिकरण लॉजिक का व्यापक ऑडिट करें ताकि केवल इच्छित उपयोगकर्ता और क्लाइंट संवेदनशील संसाधनों तक पहुँच सकें
  - **कार्यान्वयन मार्गदर्शिका**: [MCP सर्वरों के लिए Azure API प्रबंधन को प्रमाणीकरण गेटवे के रूप में उपयोग](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **पहचान एकीकरण**: [MCP सर्वर प्रमाणीकरण के लिए Microsoft Entra ID का उपयोग](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **सुरक्षित टोकन प्रबंधन**: [Microsoft के टोकन सत्यापन और जीवन चक्र सर्वोत्तम प्रथाएं](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens) लागू करें
  - टोकन ऑडियंस दावों का MCP सर्वर पहचान से मेल होना सत्यापित करें
  - उचित टोकन रोटेशन और समाप्ति नीतियां लागू करें
  - टोकन रिप्ले हमलों और अनधिकृत उपयोग को रोकें

- **संरक्षित टोकन भंडारण**: क्षेत्राधिकार और ट्रांजिट दोनों में एन्क्रिप्शन के साथ टोकन भंडारण सुरक्षित करें
  - **सर्वोत्तम प्रथाएं**: [Secure Token Storage and Encryption Guidelines](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### पहुँच नियंत्रण कार्यान्वयन

- **न्यूनतम विशेषाधिकार का सिद्धांत**: MCP सर्वरों को केवल आवश्यक न्यूनतम अनुमतियाँ दें
  - विशेषाधिकार विस्तार को रोकने के लिए नियमित अनुमति समीक्षा और अपडेट
  - **माइक्रोसॉफ्ट दस्तावेज़ीकरण**: [Secure Least-Privileged Access](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **भूमिकाओं पर आधारित पहुँच नियंत्रण (RBAC)**: सूक्ष्म भूमिका असाइनमेंट लागू करें
  - भूमिकाओं को विशिष्ट संसाधनों और क्रियाओं तक सीमित करें
  - ऐसे अनुमतियों से बचें जो हमले की सतह को बढ़ाएं

- **निरंतर अनुमति निगरानी**: चल रहे पहुंच ऑडिट और निगरानी लागू करें
  - असामान्य अनुमति पैटर्न की निगरानी करें
  - अत्यधिक या अप्रयुक्त विशेषाधिकारों को तुरंत ठीक करें

## एआई-विशिष्ट सुरक्षा खतरे

### प्रॉम्प्ट इंजेक्शन और टूल हेरफेर हमले

आधुनिक MCP कार्यान्वयन ऐसे परिष्कृत एआई-विशिष्ट हमले वेक्टरों का सामना करते हैं जिन्हें पारंपरिक सुरक्षा उपाय पूरी तरह से संबोधित नहीं कर सकते:

#### **अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन (क्रॉस-डोमेन प्रॉम्प्ट इंजेक्शन)**

**अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन** MCP-सक्षम एआई सिस्टमों में सबसे गंभीर कमजोरियों में से एक है। हमलावर बाहरी सामग्री—दस्तावेज़, वेब पेज, ईमेल या डेटा स्रोतों—में दुर्भावनापूर्ण निर्देश छिपाते हैं जिन्हें एआई सिस्टम बाद में वैध कमांड के रूप में संसाधित करते हैं।

**हमला परिदृश्य:**
- **दस्तावेज़-आधारित इंजेक्शन**: संसाधित दस्तावेजों में छुपे दुर्भावनापूर्ण निर्देश जो अनपेक्षित एआई क्रियाएं ट्रिगर करते हैं
- **वेब कंटेंट का दुरुपयोग**: बदले गए वेब पेज जिनमें एम्बेडेड प्रॉम्प्ट होते हैं जो स्क्रैपिंग पर एआई व्यवहार को नियंत्रित करते हैं
- **ईमेल-आधारित हमले**: ईमेल में दुर्भावनापूर्ण प्रॉम्प्ट जो एआई सहायक को जानकारी लीक करने या अनधिकृत क्रियाएं करने के लिए प्रेरित करते हैं
- **डेटा स्रोत संदूषण**: दूषित डेटाबेस या API जो एआई सिस्टम को दूषित सामग्री प्रदान करते हैं

**वास्तविक प्रभाव**: ये हमले डेटा निकासी, गोपनीयता उल्लंघन, हानिकारक सामग्री निर्माण, और उपयोगकर्ता इंटरैक्शन के हेरफेर का कारण बन सकते हैं। विस्तृत विश्लेषण के लिए देखें [Prompt Injection in MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/)।

![Prompt Injection Attack Diagram](../../../translated_images/hi/prompt-injection.ed9fbfde297ca877.webp)

#### **टूल विषाक्तता हमले**

**टूल विषाक्तता** MCP टूल को परिभाषित करने वाले मेटाडेटा को लक्षित करता है, जो LLMs द्वारा टूल विवरण और पैरामीटर की व्याख्या के आधार पर निष्पादन निर्णयों को प्रभावित करता है।

**हमला तंत्र:**
- **मेटाडेटा हेरफेर**: हमलावर टूल विवरण, पैरामीटर परिभाषाएँ, या उपयोग उदाहरणों में दुर्भावनापूर्ण निर्देश इंजेक्ट करते हैं
- **अदृश्य निर्देश**: टूल मेटाडेटा में छुपे प्रॉम्प्ट जिन्हें AI मॉडल संसाधित करते हैं पर मानव उपयोगकर्ताओं को दिखाई नहीं देते
- **गतिशील टूल संशोधन ("रग पुल्स")**: उपयोगकर्ता द्वारा अनुमोदित टूल बाद में बिना उपयोगकर्ता की जानकारी के दुर्भावनापूर्ण कार्रवाई के लिए संशोधित किए जाते हैं
- **पैरामीटर इंजेक्शन**: टूल पैरामीटर स्कीमाओं में छुपी दुर्भावनापूर्ण सामग्री जो मॉडल के व्यवहार को प्रभावित करती है

**होस्टेड सर्वर जोखिम**: रिमोट MCP सर्वर विशेष जोखिम प्रस्तुत करते हैं क्योंकि टूल परिभाषाएँ प्रारंभिक उपयोगकर्ता अनुमोदन के बाद अपडेट की जा सकती हैं, जिससे पहले सुरक्षित टूल दुर्भावनापूर्ण बन सकते हैं। व्यापक विश्लेषण के लिए देखें [Tool Poisoning Attacks (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks)।

![Tool Injection Attack Diagram](../../../translated_images/hi/tool-injection.3b0b4a6b24de6bef.webp)

#### **अन्य एआई हमला वेक्टर**

- **क्रॉस-डोमेन प्रॉम्प्ट इंजेक्शन (XPIA)**: परिष्कृत हमले जो कई डोमेनों की सामग्री का उपयोग करके सुरक्षा नियंत्रणों को बाईपास करते हैं
- **डायनेमिक कैपेबिलिटी मॉडिफिकेशन**: उपकरण क्षमताओं में वास्तविक समय में परिवर्तन जो प्रारंभिक सुरक्षा आकलनों से बच जाते हैं
- **कॉन्टेक्स्ट विंडो पॉइजनिंग**: बड़े संदर्भ विंडो को प्रबंधित करने वाले हमले जो दुर्भावनापूर्ण निर्देशों को छुपाते हैं
- **मॉडल कन्फ्यूजन अटैक्स**: मॉडल सीमाओं का शोषण करके अप्रत्याशित या असुरक्षित व्यवहार उत्पन्न करना


### AI सुरक्षा जोखिम प्रभाव

**उच्च-प्रभाव परिणाम:**
- **डेटा एक्सफिल्ट्रेशन**: संवेदनशील एंटरप्राइज या व्यक्तिगत डेटा की अनधिकार पहुंच और चोरी
- **गोपनीयता उल्लंघन**: व्यक्तिगत रूप से पहचानी जाने योग्य जानकारी (PII) और गोपनीय व्यापार डेटा का खुलासा  
- **सिस्टम मैनिपुलेशन**: महत्वपूर्ण सिस्टम और कार्यप्रवाहों में अनपेक्षित परिवर्तन
- **क्रेडेंशियल चोरी**: प्रमाणीकरण टोकन और सेवा क्रेडेन्शियल्स का समझौता
- **लैटरल मूवमेंट**: व्यापक नेटवर्क हमलों के लिए समझौता किए गए AI सिस्टम्स का उपयोग

### Microsoft AI सुरक्षा समाधान

#### **AI प्रॉम्प्ट शील्ड्स: इंजेक्शन हमलों के खिलाफ उन्नत सुरक्षा**

Microsoft के **AI प्रॉम्प्ट शील्ड्स** कई सुरक्षा परतों के माध्यम से सीधे और अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन हमलों के खिलाफ व्यापक सुरक्षा प्रदान करते हैं:

##### **कोर सुरक्षा तंत्र:**

1. **उन्नत डिटेक्शन और फ़िल्टरिंग**
   - मशीन लर्निंग एल्गोरिदम और NLP तकनीकें बाहरी सामग्री में दुर्भावनापूर्ण निर्देशों का पता लगाती हैं
   - दस्तावेज़ों, वेब पेज, ईमेल और डेटा स्रोतों का वास्तविक समय विश्लेषण इसके अंतर्निहित खतरों के लिए
   - वैध और दुर्भावनापूर्ण प्रॉम्प्ट पैटर्न का संदर्भगत समझ

2. **स्पॉटलाइटिंग तकनीकें**  
   - भरोसेमंद सिस्टम निर्देशों और संभावित तौर पर समझौता किए गए बाहरी इनपुट के बीच अंतर करती हैं
   - मॉडल प्रासंगिकता बढ़ाने के लिए टेक्स्ट ट्रांसफॉर्मेशन विधियां जो दुर्भावनापूर्ण सामग्री को अलग करती हैं
   - AI सिस्टमों को उचित निर्देश पदानुक्रम बनाए रखने और इंजेक्ट किए गए आदेशों को अनदेखा करने में मदद करती हैं

3. **डेलीमीटर और डेटा मार्किंग सिस्टम**
   - भरोसेमंद सिस्टम संदेशों और बाहरी इनपुट टेक्स्ट के बीच स्पष्ट सीमा निर्धारण
   - विशेष मार्कर भरोसेमंद और अविश्वसनीय डेटा स्रोतों के बीच सीमाओं को दर्शाते हैं
   - स्पष्ट विभाजन निर्देशों के भ्रम और अनधिकृत कमांड निष्पादन को रोकता है

4. **निरंतर खतरा खुफिया**
   - Microsoft नए उभरते हमले के पैटर्न की लगातार निगरानी करता है और सुरक्षा अपडेट करता है
   - नए इंजेक्शन तकनीकों और हमले के तरीकों के लिए सक्रिय खतरा खोज
   - विकसित होते खतरों के प्रति प्रभावशीलता बनाए रखने के लिए नियमित सुरक्षा मॉडल अपडेट

5. **Azure कंटेंट सेफ्टी एकीकरण**
   - व्यापक Azure AI कंटेंट सेफ्टी सूट का हिस्सा
   - जेलब्रेक प्रयासों, हानिकारक सामग्री, और सुरक्षा नीति उल्लंघनों के लिए अतिरिक्त डिटेक्शन
   - AI एप्लिकेशन घटकों में एकीकृत सुरक्षा नियंत्रण

## उन्नत MCP सुरक्षा खतरे

### सेशन हाइजैकिंग कमजोरियां

**सेशन हाइजैकिंग** राज्यसंपन्न MCP कार्यान्वयन में एक महत्वपूर्ण हमला है जहाँ अनधिकृत पक्ष वैध सेशन पहचानकर्ताओं को प्राप्त करके क्लाइंट्स की नकल करते हैं और अनधिकृत क्रियाएं करते हैं।

#### **हमला परिदृश्य और जोखिम**

- **सेशन हाइजैक प्रॉम्प्ट इंजेक्शन**: चोरी किए गए सेशन आईडी के साथ हमलावर सर्वर पर दुर्भावनापूर्ण घटनाएँ इंजेक्ट करते हैं, जिससे हानिकारक क्रियाएं या संवेदनशील डेटा तक पहुंच संभव हो सकती है
- **प्रत्यक्ष छद्मवेषण**: चोरी किए गए सेशन आईडी के माध्यम से प्रमाणीकरण को बायपास करके सीधे MCP सर्वर कॉल जो हमलावरों को वैध उपयोगकर्ता बनाकर ट्रिट करते हैं
- **समझौता किए गए पुनःआरंभ योग्य स्ट्रीम्स**: हमलावर अनुरोधों को समय से पहले समाप्त कर सकते हैं, जिससे वैध क्लाइंट संभावित तौर पर दुर्भावनापूर्ण सामग्री के साथ पुनः आरंभ करते हैं

#### **सेशन प्रबंधन के लिए सुरक्षा नियंत्रण**

**महत्वपूर्ण आवश्यकताएँ:**
- **अधिकृत सत्यापन**: MCP सर्वर जो अधिकृत करता है, उसे सभी इनबाउंड अनुरोधों की जांच करनी चाहिए और प्रमाणीकरण के लिए सत्रों पर भरोसा नहीं करना चाहिए
- **सुरक्षित सेशन निर्माण**: क्रिप्टोग्राफिक रूप से सुरक्षित, गैर-नियतात्मक सेशन आईडी जो सुरक्षित यादृच्छिक संख्या जनरेटर से बनाई गई हों, का उपयोग करें
- **उपयोगकर्ता-विशिष्ट बंधन**: सेशन आईडी को उपयोगकर्ता-विशिष्ट जानकारी से जोड़ें जैसे फ़ॉर्मेट `<user_id>:<session_id>` ताकि क्रॉस-यूज़र सेशन दुरुपयोग रोक सके
- **सेशन जीवनचक्र प्रबंधन**: उपयुक्त समाप्ति, घुमाव और अमान्यकरण लागू करें ताकि जोखिम विंडो सीमित रहे
- **ट्रांसपोर्ट सुरक्षा**: सेशन आईडी इंटरसेप्शन रोकने के लिए सभी संचार के लिए HTTPS आवश्यक है

### कन्फ्यूज़्ड डिप्टी समस्या

**कन्फ्यूज़्ड डिप्टी समस्या** तब उत्पन्न होती है जब MCP सर्वर क्लाइंट्स और तृतीय-पक्ष सेवाओं के बीच प्रमाणीकरण प्रॉक्सी के रूप में कार्य करते हैं, जिससे स्थैतिक क्लाइंट आईडी शोषण के माध्यम से अधिकृत बाइपास के अवसर बनते हैं।

#### **हमला यांत्रिकी और जोखिम**

- **कुकी-आधारित सहमति बायपास**: पिछले उपयोगकर्ता प्रमाणीकरण से सहमति कुकी बनती है, जिसे हमलावर कस्टम रिडायरेक्ट URI के साथ दुर्भावनापूर्ण अधिकृत अनुरोधों में शोषण करते हैं
- **अधिकृत कोड की चोरी**: मौजूदा सहमति कुकी के कारण अधिकृत सर्वर सहमति स्क्रीन छोड़ सकता है और कोड हमलावर नियंत्रित एंडपॉइंट्स पर पुनर्निर्देशित करता है  
- **अनधिकृत API पहुंच**: चोरी किए गए अधिकृत कोड टोकन विनिमय और उपयोगकर्ता नकल सक्षम करते हैं बिना स्पष्ट अनुमोदन के

#### **रोकथाम रणनीतियाँ**

**अनिवार्य नियंत्रण:**
- **स्पष्ट सहमति आवश्यकताएँ**: स्थैतिक क्लाइंट IDs वाले MCP प्रॉक्सी सर्वरों को प्रत्येक गतिशील पंजीकृत क्लाइंट के लिए उपयोगकर्ता की सहमति प्राप्त करनी चाहिए
- **OAuth 2.1 सुरक्षा कार्यान्वयन**: सभी अधिकृत अनुरोधों के लिए वर्तमान OAuth सुरक्षा सर्वोत्तम प्रथाओं का पालन करें जिनमें PKCE (प्रूफ की फॉर कोड एक्सचेंज) शामिल हैं
- **कड़क क्लाइंट सत्यापन**: शोषण रोकने के लिए रिडायरेक्ट URI और क्लाइंट पहचानकर्ताओं का कड़ा सत्यापन लागू करें

### टोकन पासथ्रू कमजोरियां  

**टोकन पासथ्रू** एक स्पष्ट एंटी-पैटर्न है जहाँ MCP सर्वर उचित सत्यापन के बिना क्लाइंट टोकन स्वीकार करते हैं और उन्हें डाउनस्ट्रीम API को भेज देते हैं, जो MCP प्राधिकरण विनिर्देशों का उल्लंघन है।

#### **सुरक्षा प्रभाव**

- **नियंत्रण निरोध**: सीधे क्लाइंट से API तक टोकन उपयोग से महत्वपूर्ण दर-सीमा, सत्यापन, और निगरानी नियंत्रण बायपास होते हैं
- **ऑडिट ट्रेल भ्रष्टाचार**: अपस्ट्रीम जारी किए गए टोकन क्लाइंट पहचान असंभव बनाते हैं, जिससे घटना जांच क्षमताएं बाधित होती हैं
- **प्रॉक्सी-आधारित डेटा एक्सफिल्ट्रेशन**: अवैध टोकन हमलावरों को सर्वरों का उपयोग अनधिकृत डेटा एक्सेस के लिए प्रॉक्सी के रूप में करने देते हैं
- **ट्रस्ट बाउंड्री उल्लंघन**: डाउनस्ट्रीम सेवाओं के ट्रस्ट मान्यताएँ तब बाधित हो सकती हैं जब टोकन मूल सत्यापित न हों
- **मल्टी-सर्विस हमला विस्तार**: कई सेवाओं में स्वीकार किए गए समझौता किए गए टोकन से लैटरल मूवमेंट संभव होता है

#### **आवश्यक सुरक्षा नियंत्रण**

**बिना छूट वाले आवश्यकताएँ:**
- **टोकन सत्यापन**: MCP सर्वर ऐसे टोकन स्वीकार नहीं करना चाहिए जो विशेष रूप से MCP सर्वर के लिए जारी नहीं किए गए हों
- **ऑडियंस सत्यापन**: हमेशा टोकन ऑडियंस दावों को MCP सर्वर की पहचान से मेल खाता हुआ जांचें
- **उचित टोकन जीवनचक्र**: सुरक्षित घुमाव के साथ अल्पकालिक एक्सेस टोकन लागू करें


## AI सिस्टम के लिए सप्लाई चेन सुरक्षा

सप्लाई चेन सुरक्षा पारंपरिक सॉफ़्टवेयर निर्भरताओं से आगे बढ़कर पूरे AI इकोसिस्टम को कवर करती है। आधुनिक MCP कार्यान्वयन को सभी AI-संबंधित घटकों का कड़ाई से सत्यापन और निगरानी करनी चाहिए, क्योंकि प्रत्येक संभावित कमजोरियों को जन्म देता है जो सिस्टम की अखंडता को खतरे में डाल सकता है।

### विस्तारित AI सप्लाई चेन घटक

**पारंपरिक सॉफ़्टवेयर निर्भरताएं:**
- ओपन सोर्स लाइब्रेरी और फ्रेमवर्क
- कंटेनर इमेज और बेस सिस्टम  
- विकास उपकरण और बिल्ड पाइपलाइन
- इन्फ्रास्ट्रक्चर घटक और सेवाएं

**AI-विशिष्ट सप्लाई चेन तत्व:**
- **फाउंडेशन मॉडल्स**: विभिन्न प्रदाताओं से प्री-ट्रेंड मॉडल जिन्हें स्रोत सत्यापन की आवश्यकता है
- **एम्बेडिंग सर्विसेज़**: बाहरी वेक्टराइज़ेशन और सैमान्टिक सर्च सेवाएं
- **संदर्भ प्रदाता**: डेटा स्रोत, नॉलेज बेस, और दस्तावेज़ रिपॉजिटरीज  
- **तृतीय-पक्ष API**: बाहरी AI सेवाएं, ML पाइपलाइंस, और डेटा प्रोसेसिंग एंडपॉइंट्स
- **मॉडल आर्टिफैक्ट्स**: वेट्स, कॉन्फ़िगरेशन, और फाइन-ट्यून किए गए मॉडल संस्करण
- **प्रशिक्षण डेटा स्रोत**: मॉडल प्रशिक्षण और फाइन-ट्यूनिंग के लिए उपयोग किया जाने वाला डेटा

### व्यापक सप्लाई चेन सुरक्षा रणनीति

#### **घटक सत्यापन और ट्रस्ट**
- **स्रोत सत्यापन**: सभी AI घटकों के मूल, लाइसेंसिंग, और अखंडता को एकीकरण से पहले सत्यापित करें
- **सुरक्षा आकलन**: मॉडल, डेटा स्रोत, और AI सेवाओं के लिए कमजोरियों के स्कैन और समीक्षा करें
- **प्रतिष्ठा विश्लेषण**: AI सेवा प्रदाताओं की सुरक्षा ट्रैक रिकॉर्ड और प्रथाओं का मूल्यांकन करें
- **अनुपालन सत्यापन**: सुनिश्चित करें कि सभी घटक संगठनात्मक सुरक्षा और नियामक आवश्यकताओं को पूरा करें

#### **सुरक्षित डेप्लॉयमेंट पाइपलाइंस**  
- **स्वचालित CI/CD सुरक्षा**: स्वचालित तैनाती पाइपलाइनों में सुरक्षा स्कैनिंग को एकीकृत करें
- **आर्टिफैक्ट अखंडता**: सभी तैनात आर्टिफैक्ट्स (कोड, मॉडल, कॉन्फ़िगरेशन) के लिए क्रिप्टोग्राफिक सत्यापन लागू करें
- **स्टेज्ड डेप्लॉयमेंट**: प्रत्येक चरण में सुरक्षा सत्यापन के साथ प्रगतिशील तैनाती रणनीति अपनाएं
- **विश्वसनीय आर्टिफैक्ट रिपॉजिटरीज**: केवल प्रमाणित, सुरक्षित अभिलेखागार और रिपॉजिटरीज से डेप्लॉय करें

#### **निरंतर निगरानी और प्रतिक्रिया**
- **निर्भरता स्कैनिंग**: सभी सॉफ्टवेयर और AI घटक निर्भरताओं के लिए चल रही कमजोरियों की निगरानी
- **मॉडल निगरानी**: मॉडल व्यवहार, प्रदर्शन विचलन, और सुरक्षा अनियमितताओं का सतत आकलन
- **सेवा स्वास्थ्य ट्रैकिंग**: बाहरी AI सेवाओं की उपलब्धता, सुरक्षा घटनाओं, और नीति परिवर्तनों की निगरानी
- **खतरा खुफिया एकीकरण**: AI और ML सुरक्षा जोखिमों के लिए विशिष्ट खतरा फीड्स को शामिल करना

#### **एक्सेस कंट्रोल और न्यूनतम अधिकार**
- **घटक-स्तर अनुमतियां**: व्यावसायिक आवश्यकताओं के आधार पर मॉडल, डेटा, और सेवाओं की पहुंच प्रतिबंधित करें
- **सेवा खाता प्रबंधन**: न्यूनतम आवश्यक अनुमतियों के साथ समर्पित सेवा खाते लागू करें
- **नेटवर्क सेगमेंटेशन**: AI घटकों को अलग करें और सेवाओं के बीच नेटवर्क पहुंच सीमित करें
- **API गेटवे नियंत्रण**: बाहरी AI सेवाओं की पहुंच को नियंत्रित और मॉनिटर करने के लिए केंद्रीकृत API गेटवे का उपयोग करें

#### **घटना प्रतिक्रिया और पुनर्प्राप्ति**
- **तेजी से प्रतिक्रिया प्रक्रियाएं**: समझौता किए गए AI घटकों के लिए पैचिंग या प्रतिस्थापन के लिए स्थापित प्रक्रियाएं
- **क्रेडेंशियल घुमाव**: रहस्यों, API कुंजियों, और सेवा क्रेडेंशियल्स के घुमाव के लिए स्वचालित सिस्टम
- **रोलबैक क्षमताएं**: AI घटकों के ज्ञात-श्रेष्ठ पूर्व संस्करणों में तेजी से वापस जाने की क्षमता
- **सप्लाई चेन उल्लंघन पुनर्प्राप्ति**: अपस्ट्रीम AI सेवा समझौता के लिए विशिष्ट प्रतिक्रिया प्रक्रियाएं

### Microsoft सुरक्षा उपकरण और एकीकरण

**GitHub Advanced Security** व्यापक सप्लाई चेन सुरक्षा प्रदान करता है जिनमें शामिल हैं:
- **सीक्रेट स्कैनिंग**: रिपॉजिटरीज में क्रेडेंशियल, API कीज़, और टोकन का स्वचालित पता लगाना
- **निर्भरता स्कैनिंग**: ओपन सोर्स निर्भरताओं और लाइब्रेरीज की कमजोरियों का मूल्यांकन
- **CodeQL विश्लेषण**: सुरक्षा कमजोरियों और कोडिंग मुद्दों के लिए स्थैतिक कोड विश्लेषण
- **सप्लाई चेन इंसाइट्स**: निर्भरता स्वास्थ्य और सुरक्षा स्थिति की दृश्यता

**Azure DevOps और Azure Repos एकीकरण:**
- Microsoft विकास प्लेटफॉर्म पर निर्बाध सुरक्षा स्कैनिंग एकीकरण
- AI वर्कलोड के लिए Azure पाइपलाइंस में स्वचालित सुरक्षा जांच
- सुरक्षित AI घटक तैनाती के लिए नीति प्रवर्तन

**Microsoft आंतरिक प्रथाएं:**
Microsoft सभी उत्पादों में व्यापक सप्लाई चेन सुरक्षा प्रथाओं को लागू करता है। इसके सिद्ध दृष्टिकोणों के बारे में जानें [The Journey to Secure the Software Supply Chain at Microsoft](https://devblogs.microsoft.com/engineering-at-microsoft/the-journey-to-secure-the-software-supply-chain-at-microsoft/)।

## फाउंडेशन सुरक्षा सर्वोत्तम प्रथाएं

MCP कार्यान्वयन आपकी संगठन की मौजूदा सुरक्षा स्थिति को अपनाते और उस पर निर्माण करते हैं। मौलिक सुरक्षा प्रथाओं को मजबूत करने से AI सिस्टम और MCP तैनातियों की समग्र सुरक्षा काफी बढ़ती है।

### कोर सुरक्षा मूल बातें

#### **सुरक्षित विकास प्रथाएं**
- **OWASP अनुपालन**: [OWASP Top 10](https://owasp.org/www-project-top-ten/) वेब एप्लिकेशन कमजोरियों से सुरक्षा
- **AI-विशिष्ट सुरक्षा**: [OWASP Top 10 for LLMs](https://genai.owasp.org/download/43299/?tmstv=1731900559) के लिए नियंत्रण लागू करना
- **सुरक्षित सीक्रेट्स प्रबंधन**: टोकन, API कीज, और संवेदनशील कॉन्फ़िगरेशन डेटा के लिए समर्पित वॉल्ट का उपयोग करें
- **एंड-टू-एंड एन्क्रिप्शन**: सभी एप्लिकेशन घटकों और डेटा प्रवाहों के बीच सुरक्षित संचार लागू करें
- **इनपुट मान्यता**: सभी उपयोगकर्ता इनपुट, API पैरामीटर, और डेटा स्रोतों का कठोर सत्यापन

#### **इन्फ्रास्ट्रक्चर हार्डनिंग**
- **मल्टी-फैक्टर ऑथेंटिकेशन**: सभी प्रशासनिक और सेवा खातों के लिए अनिवार्य MFA
- **पैच प्रबंधन**: ऑपरेटिंग सिस्टम, फ्रेमवर्क, और निर्भरताओं के लिए स्वचालित, समयबद्ध पैचिंग  
- **पहचान प्रदाता एकीकरण**: एंटरप्राइज पहचान प्रदाताओं (Microsoft Entra ID, Active Directory) के माध्यम से केंद्रीकृत पहचान प्रबंधन
- **नेटवर्क सेगमेंटेशन**: MCP घटकों का तार्किक पृथक्करण ताकि लैटरल मूवमेंट की संभावना कम हो
- **न्यूनतम आवश्यक अधिकार का सिद्धांत**: सभी सिस्टम घटकों और खातों के लिए न्यूनतम आवश्यक अनुमतियाँ

#### **सुरक्षा निगरानी और पहचान**
- **व्यापक लॉगिंग**: AI एप्लिकेशन गतिविधियों सहित MCP क्लाइंट-सर्वर इंटरैक्शन का विस्तृत लॉगिंग
- **SIEM एकीकरण**: असामान्य व्यवहार पहचान के लिए केंद्रीकृत सुरक्षा सूचना और घटना प्रबंधन
- **व्यवहार विश्लेषण**: सिस्टम और उपयोगकर्ता व्यवहार में असामान्य पैटर्न पहचानने के लिए AI-संचालित निगरानी
- **खतरा खुफिया**: बाहरी खतरा स्रोतों और समझौता सूचकांकों (IOCs) का समावेश
- **घटना प्रतिक्रिया**: सुरक्षा घटना की पहचान, प्रतिक्रिया, और पुनर्प्राप्ति के लिए सुव्यवस्थित प्रक्रियाएं

#### **ज़ीरो ट्रस्ट आर्किटेक्चर**
- **कभी भरोसा न करें, हमेशा सत्यापित करें**: उपयोगकर्ताओं, उपकरणों, और नेटवर्क कनेक्शनों का सतत सत्यापन
- **माइक्रो-सेगमेंटेशन**: व्यक्तिगत वर्कलोड और सेवाओं को पृथक करने वाले सूक्ष्म नेटवर्क नियंत्रण
- **पहचान-केंद्रित सुरक्षा**: नेटवर्क स्थान के बजाय सत्यापित पहचान पर आधारित सुरक्षा नीतियां
- **निरंतर जोखिम मूल्यांकन**: वर्तमान संदर्भ और व्यवहार के आधार पर गतिशील सुरक्षा स्थिति मूल्यांकन
- **सशर्त पहुंच**: जोखिम कारकों, स्थान, और डिवाइस ट्रस्ट के आधार पर अनुकूलित पहुंच नियंत्रण

### एंटरप्राइज एकीकरण पैटर्न

#### **Microsoft सुरक्षा पारिस्थितिकी तंत्र एकीकरण**
- **Microsoft Defender for Cloud**: व्यापक क्लाउड सुरक्षा स्थिति प्रबंधन
- **Azure Sentinel**: AI वर्कलोड संरक्षण के लिए क्लाउड-नेटिव SIEM और SOAR क्षमताएं
- **Microsoft Entra ID**: सशर्त पहुंच नीतियों के साथ एंटरप्राइज पहचान और पहुंच प्रबंधन
- **Azure Key Vault**: हार्डवेयर सुरक्षा मॉड्यूल (HSM) समर्थन के साथ केंद्रीकृत रहस्य प्रबंधन
- **Microsoft Purview**: AI डेटा स्रोतों और कार्यप्रवाहों के लिए डेटा गवर्नेंस और अनुपालन

#### **अनुपालन और प्रशासनिक नियंत्रण**
- **नियमों के अनुकूलन**: सुनिश्चित करें कि MCP कार्यान्वयन उद्योग-विशिष्ट अनुपालन आवश्यकताओं (GDPR, HIPAA, SOC 2) को पूरा करें
- **डेटा वर्गीकरण**: AI सिस्टम द्वारा संसाधित संवेदनशील डेटा की उचित श्रेणीकरण और प्रबंधन
- **ऑडिट ट्रेल्स**: नियामक अनुपालन और फोरेंसिक जांच के लिए व्यापक लॉगिंग
- **गोपनीयता नियंत्रण**: AI सिस्टम आर्किटेक्चर में प्राइवेसी-बाय- डिज़ाइन सिद्धांतों का कार्यान्वयन
- **चेंज मैनेजमेंट**: AI सिस्टम परिवर्तनों की सुरक्षा समीक्षा के लिए औपचारिक प्रक्रिया

ये मौलिक प्रथाएं एक मजबूत सुरक्षा आधार बनाती हैं जो MCP-विशिष्ट सुरक्षा नियंत्रणों की प्रभावशीलता बढ़ाती हैं और AI-संचालित अनुप्रयोगों के लिए व्यापक सुरक्षा प्रदान करती हैं।

## मुख्य सुरक्षा निष्कर्ष  

- **परतदार सुरक्षा दृष्टिकोण**: व्यापक सुरक्षा के लिए मौलिक सुरक्षा प्रथाओं (सुरक्षित कोडिंग, न्यूनतम विशेषाधिकार, आपूर्ति श्रृंखला सत्यापन, निरंतर निगरानी) को AI-विशिष्ट नियंत्रणों के साथ संयोजित करें

- **AI-विशिष्ट खतरा परिदृश्य**: MCP सिस्टमों को अद्वितीय जोखिमों का सामना करना पड़ता है जिनमें प्रॉम्प्ट इंजेक्शन, टूल विषाक्तता, सत्र हाईजैकिंग, कन्फ्यूज्ड डिप्टी समस्याएं, टोकन पासथ्रू कमजोरियां, और अत्यधिक अनुमतियां शामिल हैं जिन्हें विशेष रोधी उपायों की आवश्यकता होती है

- **प्रमाणीकरण और प्राधिकरण उत्कृष्टता**: बाहरी पहचान प्रदाताओं (Microsoft Entra ID) का उपयोग करके मजबूत प्रमाणीकरण लागू करें, उचित टोकन सत्यापन को लागू करें, और कभी भी ऐसे टोकन न स्वीकारें जो आपके MCP सर्वर के लिए स्पष्ट रूप से जारी नहीं किए गए हों

- **AI हमले की रोकथाम**: अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन और टूल विषाक्तता हमलों के खिलाफ सुरक्षा के लिए Microsoft Prompt Shields और Azure Content Safety तैनात करें, साथ ही टूल मेटाडेटा को मान्य करें और गतिशील परिवर्तनों की निगरानी करें

- **सत्र और ट्रांसपोर्ट सुरक्षा**: क्रिप्टोग्राफिक रूप से सुरक्षित, गैर-निर्धारणीय सत्र आईडी का उपयोग करें जो उपयोगकर्ता पहचान से बंधे हों, उचित सत्र जीवनचक्र प्रबंधन लागू करें, और प्रमाणीकरण के लिए सत्रों का उपयोग कभी न करें

- **OAuth सुरक्षा सर्वोत्तम प्रथाएं**: गतिशील रूप से पंजीकृत क्लाइंट्स के लिए स्पष्ट उपयोगकर्ता सहमति के माध्यम से कन्फ्यूज्ड डिप्टी हमलों को रोकें, PKCE के साथ उचित OAuth 2.1 कार्यान्वयन करें, और सख्त रीडायरेक्ट URI सत्यापन करें

- **टोकन सुरक्षा सिद्धांत**: टोकन पासथ्रू विरोधी-पैटर्न से बचें, टोकन ऑडियंस क्लेम को मान्य करें, सुरक्षित रोटेशन के साथ अल्पकालिक टोकन लागू करें, और स्पष्ट ट्रस्ट सीमाएं बनाए रखें

- **व्यापक आपूर्ति श्रृंखला सुरक्षा**: AI पारिस्थितिकी तंत्र के सभी घटकों (मॉडल, एम्बेडिंग, संदर्भ प्रदाता, बाहरी API) को पारंपरिक सॉफ़्टवेयर निर्भरताओं जितनी ही कठोर सुरक्षा के साथ देखें

- **लगातार विकास**: तेजी से विकसित हो रहे MCP विनिर्देशों के साथ अपडेट रहें, सुरक्षा समुदाय के मानकों में योगदान दें, और प्रोटोकॉल के परिपक्व होने के साथ अनुकूलनीय सुरक्षा स्थिरता बनाए रखें

- **Microsoft सुरक्षा एकीकरण**: Microsoft के व्यापक सुरक्षा पारिस्थितिकी तंत्र (Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID) का उपयोग MCP तैनाती सुरक्षा में वृद्धि के लिए करें
