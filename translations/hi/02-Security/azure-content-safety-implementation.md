# MCP के साथ Azure कंटेंट सेफ्टी को लागू करना

> **OWASP MCP जोखिम का समाधान**: [MCP06 - इरादा प्रवाह सबवर्शन](https://microsoft.github.io/mcp-azure-security-guide/mcp/mcp06-prompt-injection/)

प्रॉम्प्ट इंजेक्शन, टूल पॉइज़निंग, और अन्य AI-विशिष्ट कमजोरियों के खिलाफ MCP सुरक्षा को मजबूत करने के लिए, Azure कंटेंट सेफ्टी को एकीकृत करना अत्यंत अनुशंसित है। यह कार्यान्वयन गाइड [MCP सिक्योरिटी समिट वर्कशॉप (शेरपा)](https://azure-samples.github.io/sherpa/) कैंप 3: I/O सिक्योरिटी के अनुरूप है।

## MCP सर्वर के साथ एकीकरण

Azure कंटेंट सेफ्टी को अपने MCP सर्वर के साथ एकीकृत करने के लिए, कंटेंट सेफ्टी फ़िल्टर को अपने अनुरोध प्रसंस्करण पाइपलाइन में मिडलवेयर के रूप में जोड़ें:

1. सर्वर शुरुआत के दौरान फ़िल्टर इनिशियलाइज़ करें
2. प्रसंस्करण से पहले सभी इनकमिंग टूल अनुरोधों का सत्यापन करें
3. क्लाइंट को उत्तर देने से पहले सभी आउटगोइंग प्रतिक्रियाओं की जांच करें
4. सुरक्षा उल्लंघनों पर लॉग करें और अलर्ट जारी करें
5. विफल कंटेंट सेफ्टी जांच के लिए उपयुक्त त्रुटि हैंडलिंग लागू करें

यह निम्नलिखित के खिलाफ एक मजबूत सुरक्षा प्रदान करता है:
- प्रॉम्प्ट इंजेक्शन हमले
- टूल पॉइज़निंग प्रयास
- दुर्भावनापूर्ण इनपुट के माध्यम से डेटा चोरी
- हानिकारक सामग्री का निर्माण

## Azure कंटेंट सेफ्टी एकीकरण के लिए सर्वोत्तम प्रथाएँ

1. **कस्टम ब्लॉकलिस्ट बनाएँ**: विशेष रूप से MCP इंजेक्शन पैटर्न के लिए कस्टम ब्लॉकलिस्ट बनाएं
2. **तीव्रता समायोजन**: अपनी विशिष्ट उपयोग स्थिति और जोखिम सहिष्णुता के आधार पर तीव्रता सीमा समायोजित करें
3. **व्यापक कवरेज**: सभी इनपुट्स और आउटपुट्स पर कंटेंट सेफ्टी जांच लागू करें
4. **प्रदर्शन अनुकूलन**: बार-बार की जा रही कंटेंट सेफ्टी जांच के लिए कैशिंग लागू करने पर विचार करें
5. **फॉलबैक तंत्र**: जब कंटेंट सेफ्टी सेवाएँ अनुपलब्ध हों तो स्पष्ट फॉलबैक व्यवहार परिभाषित करें
6. **उपयोगकर्ता प्रतिक्रिया**: जब सामग्री सुरक्षा कारणों से अवरुद्ध हो, तो उपयोगकर्ताओं को स्पष्ट प्रतिक्रिया प्रदान करें
7. **निरंतर सुधार**: उभरते हुए खतरों के आधार पर ब्लॉकलिस्ट और पैटर्न को नियमित रूप से अपडेट करें
