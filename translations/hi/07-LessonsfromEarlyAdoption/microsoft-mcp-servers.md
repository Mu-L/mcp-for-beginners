# 🚀 10 माइक्रोसॉफ्ट MCP सर्वर जो डेवलपर उत्पादकता को बदल रहे हैं

## 🎯 इस गाइड में आप क्या सीखेंगे

यह व्यावहारिक गाइड दस माइक्रोसॉफ्ट MCP सर्वरों को प्रदर्शित करता है जो सक्रिय रूप से दर्शाते हैं कि डेवलपर AI सहायकों के साथ कैसे काम करते हैं। केवल यह बताने के बजाय कि MCP सर्वर *क्या* कर सकते हैं, हम आपको उन सर्वरों को दिखाएंगे जो माइक्रोसॉफ्ट और उससे आगे के दैनिक विकास वर्कफ़्लो में वास्तविक बदलाव ला रहे हैं।

इस गाइड में प्रत्येक सर्वर वास्तविक दुनिया के उपयोग और डेवलपर फीडबैक के आधार पर चुना गया है। आप न केवल यह जानेंगे कि प्रत्येक सर्वर क्या करता है, बल्कि यह भी कि यह क्यों महत्वपूर्ण है और अपने खुद के प्रोजेक्ट्स में अधिकतम उपयोग कैसे करें। चाहे आप MCP में बिल्कुल नए हों या अपने मौजूदा सेटअप को बढ़ाना चाहते हों, ये सर्वर माइक्रोसॉफ्ट पारिस्थितिकी तंत्र में उपलब्ध सबसे व्यावहारिक और प्रभावशाली उपकरणों का प्रतिनिधित्व करते हैं।

> **💡 त्वरित शुरुआत टिप**
> 
> MCP में नए हैं? चिंता मत कीजिए! यह गाइड शुरुआती लोगों के लिए डिज़ाइन की गई है। हम हलचल के साथ अवधारणाओं को समझाएंगे, और आप हमेशा हमारे [Introduction to MCP](../00-Introduction/README.md) और [Core Concepts](../01-CoreConcepts/README.md) मॉड्यूल्स को गहरी पृष्ठभूमि के लिए देख सकते हैं।

## अवलोकन

यह व्यापक गाइड दस माइक्रोसॉफ्ट MCP सर्वरों की खोज करता है जो इस बात में क्रांति ला रहे हैं कि डेवलपर्स AI सहायकों और बाहरी उपकरणों के साथ कैसे इंटरैक्ट करते हैं। Azure संसाधन प्रबंधन से लेकर दस्तावेज़ प्रक्रिया तक, ये सर्वर मॉडल संदर्भ प्रोटोकॉल की शक्ति को सहज, उत्पादक विकास वर्कफ़्लो बनाने में दिखाते हैं।

## सीखने के उद्देश्य

- माइक्रोसॉफ्ट के सबसे प्रभावशाली MCP सर्वर कार्यान्वयन के बारे में जानेंगे
- प्रत्येक सर्वर के व्यावहारिक उपयोग मामलों की खोज करेंगे
- जानेंगे कि इन सर्वरों को VS Code और Visual Studio में कैसे सेट अप और कॉन्फ़िगर करें
- व्यापक MCP पारिस्थितिकी तंत्र और भविष्य की दिशा का अन्वेषण करेंगे

## 🔧 MCP सर्वर समझना: शुरुआती के लिए गाइड

### MCP सर्वर क्या हैं?

मॉडल संदर्भ प्रोटोकॉल (MCP) में एक शुरुआती के रूप में, आप सोच सकते हैं: "MCP सर्वर वास्तव में क्या है, और मुझे इसकी परवाह क्यों करनी चाहिए?" आइए एक सरल उपमा से शुरू करें।

सोचिए MCP सर्वर विशिष्ट सहायक हैं जो आपके AI कोडिंग साथी (जैसे GitHub Copilot) को बाहरी उपकरणों और सेवाओं से जोड़ने में मदद करते हैं। ठीक उसी तरह जैसे आप अपने फोन पर विभिन्न कार्यों के लिए अलग-अलग ऐप्स का उपयोग करते हैं — एक मौसम के लिए, एक नेविगेशन के लिए, एक बैंकिंग के लिए — MCP सर्वर आपके AI सहायक को विभिन्न विकास उपकरणों और सेवाओं के साथ इंटरैक्ट करने की क्षमता देते हैं।

### MCP सर्वर जो समस्या हल करते हैं

MCP सर्वरों से पहले, अगर आप चाहते थे:
- अपने Azure संसाधनों की जांच करना
- GitHub इश्यू बनाना
- अपने डेटाबेस से क्वेरी करना
- दस्तावेज़ों में खोज करना

तो आपको कोडिंग बंद करनी पड़ती, ब्राउज़र खोलना पड़ता, उपयुक्त वेबसाइट पर जाना पड़ता, और मैन्युअली ये कार्य करना पड़ते। यह लगातार संदर्भ परिवर्तन आपके प्रवाह को तोड़ता है और उत्पादकता को कम करता है।

### MCP सर्वर आपके विकास अनुभव को कैसे बदलते हैं

MCP सर्वरों के साथ, आप अपने विकास वातावरण (VS Code, Visual Studio, आदि) में रह सकते हैं और बस अपने AI सहायक से ये कार्य करने को कह सकते हैं। उदाहरण के लिए:

**इस पारंपरिक वर्कफ़्लो के बजाय:**
1. कोडिंग बंद करें
2. ब्राउज़र खोलें
3. Azure पोर्टल पर जाएं
4. स्टोरेज खाते का विवरण देखें
5. VS Code पर लौटें
6. कोडिंग फिर से शुरू करें

**अब आप यह कर सकते हैं:**
1. AI से पूछें: "मेरे Azure स्टोरेज खातों की स्थिति क्या है?"
2. मिली जानकारी के साथ कोडिंग जारी रखें

### शुरुआती के लिए प्रमुख लाभ

#### 1. 🔄 **अपने फ्लो स्टेट में रहें**
- कई एप्लिकेशन के बीच स्विच करना बंद करें
- आप जिस कोड को लिख रहे हैं उस पर अपना ध्यान बनाएं रखें
- विभिन्न उपकरणों का प्रबंधन करने के मानसिक भार को कम करें

#### 2. 🤖 **जटिल कमांड की जगह प्राकृतिक भाषा का उपयोग करें**
- SQL सिंटैक्स याद रखने के बजाय आवश्यक डेटा का वर्णन करें
- Azure CLI कमांड याद रखने की जगह बताएं कि आप क्या करना चाहते हैं
- AI को तकनीकी विवरण संभालने दें जबकि आप लॉजिक पर ध्यान दें

#### 3. 🔗 **कई उपकरणों को एक साथ जोड़ें**
- विभिन्न सेवाओं को मिलाकर शक्तिशाली वर्कफ़्लो बनाएं
- उदाहरण: "सभी हाल के GitHub इश्यू प्राप्त करें और संबंधित Azure DevOps कार्य आइटम बनाएं"
- जटिल स्क्रिप्ट लिखे बिना ऑटोमेशन बनाएं

#### 4. 🌐 **एक बढ़ती हुई पारिस्थितिकी तंत्र का लाभ उठाएं**
- माइक्रोसॉफ्ट, GitHub, और अन्य कंपनियों द्वारा निर्मित सर्वरों से लाभ उठाएं
- विभिन्न विक्रेताओं के उपकरणों को सहजता से मिलाएं
- एक सुव्यवस्थित पारिस्थितिकी तंत्र में शामिल हों जो विभिन्न AI सहायकों के साथ काम करता है

#### 5. 🛠️ **कर-कर के सीखें**
- अवधारणाओं को समझने के लिए पूर्व-निर्मित सर्वरों से शुरू करें
- जैसे-जैसे आप सहज होते जाएं, अपने स्वयं के सर्वर बनाएं
- उपलब्ध SDKs और दस्तावेज़ का उपयोग करके अपने सीखने में मार्गदर्शन पाएं

### शुरुआती के लिए वास्तविक दुनिया का उदाहरण

मान लीजिए आप वेब विकास में नए हैं और अपनी पहली परियोजना पर काम कर रहे हैं। यहाँ MCP सर्वर आपकी किस प्रकार मदद कर सकते हैं:

**परंपरागत तरीका:**
```
1. Code a feature
2. Open browser → Navigate to GitHub
3. Create an issue for testing
4. Open another tab → Check Azure docs for deployment
5. Open third tab → Look up database connection examples
6. Return to VS Code
7. Try to remember what you were doing
```

**MCP सर्वरों के साथ:**
```
1. Code a feature
2. Ask AI: "Create a GitHub issue for testing this login feature"
3. Ask AI: "How do I deploy this to Azure according to the docs?"
4. Ask AI: "Show me the best way to connect to my database"
5. Continue coding with all the information you need
```

### उद्यम मानक लाभ

MCP एक उद्योग-व्यापी मानक बनता जा रहा है, जिसका अर्थ है:
- **संगतता**: विभिन्न उपकरणों और कंपनियों में समान अनुभव
- **इंटरऑपरेबिलिटी**: विभिन्न विक्रेताओं के सर्वर एक साथ काम करते हैं
- **भविष्य के लिए तैयार**: कौशल और सेटअप विभिन्न AI सहायकों के बीच स्थानांतरित हो सकते हैं
- **समुदाय**: साझा ज्ञान और संसाधनों का बड़ा पारिस्थितिकी तंत्र

## VS Code और Visual Studio में MCP के साथ शुरुआत करें 🚀

यदि आप Visual Studio Code या Visual Studio 2022 को GitHub Copilot के साथ उपयोग कर रहे हैं, तो इन MCP सर्वरों को सेट अप करना सरल है।

### VS Code सेटअप

यहां VS Code के लिए मूल प्रक्रिया है:

1. **एजेंट मोड सक्षम करें**: VS Code में, Copilot Chat विंडो में एजेंट मोड पर स्विच करें
2. **MCP सर्वर कॉन्फ़िगर करें**: अपने VS Code settings.json फ़ाइल में सर्वर कॉन्फ़िगरेशन जोड़ें
3. **सर्वर शुरू करें**: उपयोग करने के लिए प्रत्येक सर्वर के लिए "Start" बटन पर क्लिक करें
4. **उपकरण चुनें**: अपनी वर्तमान सत्र के लिए किस MCP सर्वर को सक्षम करना है चुनें

विस्तृत सेटअप निर्देशों के लिए देखें [VS Code MCP डॉक्यूमेंटेशन](https://code.visualstudio.com/docs/copilot/copilot-mcp)।

> **💡 प्रो टिप: MCP सर्वरों का प्रबंधन पेशेवर की तरह करें!**
> 
> VS Code एक्सटेंशंस व्यू में अब [इंस्टॉल किए गए MCP सर्वरों को प्रबंधित करने के लिए एक नया UI](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode) शामिल है! आप किसी भी इंस्टॉल किए गए MCP सर्वर को शुरू, बंद और प्रबंधित करने के लिए त्वरित पहुंच प्राप्त कर सकते हैं। आज़माएं!

### Visual Studio 2022 सेटअप

Visual Studio 2022 (संस्करण 17.14 या बाद का) के लिए:

1. **एजेंट मोड सक्षम करें**: GitHub Copilot चैट विंडो में "Ask" ड्रॉपडाउन पर क्लिक करें और "Agent" चुनें
2. **कॉन्फ़िगरेशन फ़ाइल बनाएं**: अपने सॉल्यूशन निर्देशिका में `.mcp.json` फ़ाइल बनाएं (सिफारिश की गई जगह: `<SOLUTIONDIR>\.mcp.json`)
3. **सर्वर कॉन्फ़िगर करें**: मानक MCP फॉर्मेट का उपयोग करते हुए अपने MCP सर्वर कॉन्फ़िगरेशन जोड़ें
4. **उपकरण मंजूरी**: जब संकेत दिया जाए, तो उपयोग के लिए आवश्यक स्कोप अनुमतियों के साथ उपकरणों को अनुमोदित करें

विस्तृत Visual Studio सेटअप निर्देशों के लिए देखें [Visual Studio MCP डॉक्यूमेंटेशन](https://learn.microsoft.com/visualstudio/ide/mcp-servers)।

हर MCP सर्वर के अपने कॉन्फ़िगरेशन आवश्यकताएँ होती हैं (जैसे कनेक्शन स्ट्रिंग्स, प्रमाणीकरण, आदि), लेकिन सेटअप पैटर्न दोनों IDE में समान है।

**यह उपयोगी क्यों है**: जबकि यह "केवल दस्तावेज़ीकरण" जैसा लग सकता है, यह सर्वर वास्तव में हर डेवलपर के लिए महत्वपूर्ण है जो माइक्रोसॉफ्ट तकनीकों का उपयोग करता है। AI कोडिंग सहायकों के बारे में .NET डेवलपर्स की सबसे बड़ी शिकायतों में से एक यह है कि वे नवीनतम .NET और C# रिलीज़ के बारे में अपडेट नहीं रहते। माइक्रोसॉफ्ट लर्न डॉक्स MCP सर्वर इसे हल करता है, सबसे वर्तमान दस्तावेज़, API संदर्भ, और सर्वोत्तम प्रथाओं तक वास्तविक समय की पहुंच प्रदान करके। चाहे आप नवीनतम Azure SDK का उपयोग कर रहे हों, नए C# 13 फीचर्स का अन्वेषण कर रहे हों, या अत्याधुनिक .NET Aspire पैटर्न लागू कर रहे हों, यह सर्वर सुनिश्चित करता है कि आपका AI सहायक सटीक, आधुनिक कोड जनरेट करने के लिए अधिकारिक, अद्यतन जानकारी तक पहुंच रखता है।

**वास्तविक दुनिया में उपयोग**: "आधिकारिक माइक्रोसॉफ्ट लर्न दस्तावेज़ीकरण के अनुसार Azure कंटेनर ऐप बनाने के लिए az cli कमांड क्या हैं?" या "मैं ASP.NET Core में Entity Framework को निर्भरता इंजेक्शन के साथ कैसे कॉन्फ़िगर करूं?" या "इस कोड की समीक्षा करें कि क्या यह माइक्रोसॉफ्ट लर्न दस्तावेज़ीकरण में प्रदर्शन अनुशंसाओं के अनुरूप है?" यह सर्वर माइक्रोसॉफ्ट लर्न, Azure डॉक्स, और माइक्रोसॉफ्ट 365 दस्तावेज़ीकरण के पार समृद्ध सेमांटिक खोज का उपयोग करके सबसे सापेक्षिक जानकारी खोजता है। यह लेख शीर्षक और URLs के साथ दस उच्च गुणवत्ता वाले सामग्री टुकड़े प्रदान करता है और हमेशा प्रकाशित होने वाले नवीनतम माइक्रोसॉफ्ट दस्तावेज़ों तक पहुंचता है।

**विशेष उदाहरण**: यह सर्वर `microsoft_docs_search` उपकरण प्रकट करता है जो माइक्रोसॉफ्ट की आधिकारिक तकनीकी दस्तावेज़ीकरण के खिलाफ सेमांटिक खोज करता है। एक बार कॉन्फ़िगर हो जाने पर, आप प्रश्न पूछ सकते हैं जैसे "मैं ASP.NET Core में JWT प्रमाणीकरण कैसे लागू करूं?" और विस्तार से, आधिकारिक उत्तर स्रोत लिंक सहित प्राप्त कर सकते हैं। खोज की गुणवत्ता विशेष है क्योंकि यह संदर्भ को समझता है – Azure संदर्भ में "containers" से पूछा जाए तो Azure Container Instances दस्तावेज़ीकरण लौटाता है, वहीं .NET संदर्भ में वही शब्द प्रासंगिक C# संग्रह जानकारी देता है।

यह विशेष रूप से तेजी से बदलने वाली या हाल ही में अपडेट की गई लाइब्रेरीज़ और उपयोग मामलों के लिए उपयोगी है। उदाहरण के लिए, कुछ हाल के कोडिंग प्रोजेक्ट्स में मैं .NET Aspire और Microsoft.Extensions.AI के नवीनतम रिलीज़ के फीचर्स का लाभ उठाना चाहता था। माइक्रोसॉफ्ट लर्न डॉक्स MCP सर्वर को शामिल करके, मैं केवल API दस्तावेज़ ही नहीं, बल्कि अभी-अभी प्रकाशित किए गए वॉकथ्रू और मार्गदर्शन का भी लाभ उठा सका।

### 2. ☁️ Azure MCP सर्वर
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

**यह क्या करता है**: Azure MCP सर्वर 15+ विशेष Azure सेवा कनेक्टर्स का एक व्यापक सूट है जो पूरे Azure पारिस्थितिकी तंत्र को आपके AI वर्कफ़्लो में लाता है। यह केवल एक सर्वर नहीं है – यह एक शक्तिशाली संग्रह है जिसमें संसाधन प्रबंधन, डेटाबेस कनेक्टिविटी (PostgreSQL, SQL Server), KQL के साथ Azure Monitor लॉग विश्लेषण, Cosmos DB एकीकरण, और बहुत कुछ शामिल है।

**यह क्यों उपयोगी है**: Azure संसाधनों का प्रबंधन करने के अलावा, यह सर्वर Azure SDKs के साथ काम करते समय कोड की गुणवत्ता में नाटकीय सुधार करता है। जब आप Agent मोड में Azure MCP का उपयोग करते हैं, तो यह केवल कोड लिखने में मदद नहीं करता - यह *बेहतर* Azure कोड लिखने में मदद करता है जो वर्तमान प्रमाणीकरण पैटर्न, त्रुटि प्रबंधन सर्वोत्तम प्रथाओं का पालन करता है, और नवीनतम SDK विशेषताओं का लाभ उठाता है। सामान्य कोड प्राप्त करने के बजाय जो शायद काम करे, आपको ऐसा कोड मिलता है जो Azure के उत्पादन कार्यभार के लिए अनुशंसित पैटर्न का पालन करता है।

**मुख्य मॉड्यूल शामिल हैं**:
- **🗄️ डेटाबेस कनेक्टर्स**: Azure Database for PostgreSQL और SQL Server के लिए सीधे प्राकृतिक भाषा पहुंच
- **📊 Azure Monitor**: KQL-संचालित लॉग विश्लेषण और संचालन अंतर्दृष्टि
- **🌐 संसाधन प्रबंधन**: पूर्ण Azure संसाधन जीवनचक्र प्रबंधन
- **🔐 प्रमाणीकरण**: DefaultAzureCredential और प्रबंधित पहचान पैटर्न
- **📦 स्टोरेज सेवाएं**: ब्लॉब स्टोरेज, क्यू स्टोरेज, और टेबल स्टोरेज संचालन
- **🚀 कंटेनर सेवाएं**: Azure कंटेनर ऐप्स, कंटेनर इंस्टेंस, और AKS प्रबंधन
- **और कई अन्य विशेष कनेक्टर्स**

**वास्तविक दुनिया में उपयोग**: "मेरे Azure स्टोरेज खातों की सूची बनाएँ", "पिछले घंटे में त्रुटियों के लिए मेरा Log Analytics कार्यक्षेत्र क्वेरी करें", या "मुझे Node.js का उपयोग करते हुए उचित प्रमाणीकरण के साथ एक Azure एप्लिकेशन बनाने में मदद करें"

**पूर्ण डेमो परिदृश्य**: यहाँ एक पूर्ण वॉकथ्रू है जो दिखाता है कि Azure MCP को GitHub Copilot for Azure एक्सटेंशन के साथ VS Code में मिलाकर कैसे शक्तिशाली बनाया जा सकता है। जब आपके पास दोनों स्थापित होते हैं और आप प्रवाहित करते हैं:

> "एक Python स्क्रिप्ट बनाएँ जो DefaultAzureCredential प्रमाणीकरण का उपयोग करके Azure Blob Storage में फाइल अपलोड करे। स्क्रिप्ट को मेरे Azure स्टोरेज खाते 'mycompanystorage' से कनेक्ट होना चाहिए, 'documents' नामक कंटेनर में अपलोड करना चाहिए, वर्तमान टाइमस्टैम्प के साथ एक टेस्ट फ़ाइल बनानी चाहिए, त्रुटियों को सुचारू रूप से संभालना चाहिए और सूचनात्मक आउटपुट प्रदान करना चाहिए, प्रमाणीकरण और त्रुटि प्रबंधन के लिए Azure की सर्वोत्तम प्रथाओं का पालन करना चाहिए, DefaultAzureCredential प्रमाणीकरण कैसे काम करता है यह समझाते हुए टिप्पणियाँ शामिल करनी चाहिए, और स्क्रिप्ट को उचित फ़ंक्शंस और दस्तावेज़ीकरण के साथ अच्छी तरह से संरचित बनाना चाहिए।"

Azure MCP सर्वर एक पूर्ण, उत्पादन-तैयार Python स्क्रिप्ट जनरेट करेगा जो:
- नवीनतम Azure Blob Storage SDK का उपयोग सही async पैटर्न के साथ करता है
- DefaultAzureCredential को एक व्यापक फॉलबैक चेन व्याख्या के साथ लागू करता है
- विशिष्ट Azure अपवाद प्रकारों के साथ मजबूत त्रुटि प्रबंधन शामिल करता है
- संसाधन प्रबंधन और कनेक्शन प्रबंधन के लिए Azure SDK सर्वोत्तम प्रथाओं का पालन करता है
- विस्तृत लॉगिंग और सूचनात्मक कंसोल आउटपुट प्रदान करता है
- फ़ंक्शंस, दस्तावेज़ीकरण, और टाइप संकेतों के साथ उचित रूप से संरचित स्क्रिप्ट बनाता है

यह असाधारण बात यह है कि Azure MCP के बिना, आपको सामान्य ब्लॉब स्टोरेज कोड मिल सकता है जो काम करता है लेकिन वर्तमान Azure पैटर्न का पालन नहीं करता। Azure MCP के साथ, आपको ऐसा कोड मिलता है जो नवीनतम प्रमाणीकरण विधियों का उपयोग करता है, Azure-विशिष्ट त्रुटि परिदृश्यों को संभालता है, और Microsoft के उत्पादन अनुप्रयोगों के लिए अनुशंसित प्रथाओं का पालन करता है।

**विशेष उदाहरण**: मैं `az` और `azd` CLI के विशिष्ट कमांड याद रखने में संघर्ष करता हूं। मेरे लिए यह हमेशा दो-चरण प्रक्रिया होती है: पहले सिंटैक्स खोजना, फिर कमांड चलाना। मैं अक्सर पोर्टल में जाकर क्लिक करता हूं क्योंकि मैं यह स्वीकार नहीं करना चाहता कि मुझे CLI सिंटैक्स याद नहीं रहता। केवल यह वर्णन करने में सक्षम होना कि मैं क्या चाहता हूं अद्भुत है, और बेहतर यह कि मैं ऐसा अपने IDE को छोड़े बिना कर सकूं!

[Azure MCP रेपोजिटरी](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) में शुरू करने के लिए उपयोग मामलों की एक अच्छी सूची है। व्यापक सेटअप गाइड और उन्नत कॉन्फ़िगरेशन विकल्पों के लिए, [आधिकारिक Azure MCP दस्तावेज़](https://learn.microsoft.com/azure/developer/azure-mcp-server/) देखें।

**यह क्या करता है**: आधिकारिक GitHub MCP सर्वर GitHub के पूरे पारिस्थितिकी तंत्र के साथ सहज एकीकरण प्रदान करता है, जिसमें होस्ट किए गए रिमोट एक्सेस और स्थानीय Docker तैनाती विकल्प दोनों शामिल हैं। यह केवल बेसिक रिपोजिटरी संचालन का मामला नहीं है – यह एक व्यापक टूलकिट है जिसमें GitHub Actions प्रबंधन, पुल रिक्वेस्ट वर्कफ़्लो, इश्यू ट्रैकिंग, सुरक्षा स्कैनिंग, अधिसूचनाएं, और उन्नत ऑटोमेशन क्षमताएं शामिल हैं।

**यह क्यों उपयोगी है**: यह सर्वर GitHub के साथ आपकी बातचीत को इस प्रकार बदल देता है कि पूरा प्लेटफ़ॉर्म अनुभव सीधे आपके डेवलपमेंट वातावरण में आता है। परियोजना प्रबंधन, कोड समीक्षाओं, और CI/CD मॉनिटरिंग के लिए VS Code और GitHub.com के बीच बार-बार स्विच करने के बजाय, आप प्राकृतिक भाषा कमांड के माध्यम से सब कुछ हैंडल कर सकते हैं जबकि अपने कोड पर ध्यान केंद्रित करते हैं।

> **ℹ️ नोट: 'एजेंट्स' के विभिन्न प्रकार**
> 
> इस GitHub MCP सर्वर को GitHub के Coding Agent (जो आप स्वचालित कोडिंग कार्यों के लिए इश्यू असाइन कर सकते हैं) से भ्रमित न करें। GitHub MCP सर्वर VS Code के Agent मोड में GitHub API एकीकरण प्रदान करता है, जबकि GitHub का Coding Agent एक अलग सुविधा है जो GitHub इश्यू पर असाइन होने पर पुल रिक्वेस्ट बनाता है।

**मुख्य क्षमताएँ शामिल हैं**:
- **⚙️ GitHub Actions**: पूर्ण CI/CD पाइपलाइन प्रबंधन, वर्कफ़्लो मॉनिटरिंग, और आर्टिफैक्ट प्रबंधन
- **🔀 पुल रिक्वेस्ट**: PR बनाना, समीक्षा करना, मर्ज करना, और व्यापक स्थिति ट्रैकिंग के साथ प्रबंधन
- **🐛 इश्यूज़**: पूर्ण इश्यू जीवनचक्र प्रबंधन, टिप्पणी, लेबलिंग, और असाइनमेंट
- **🔒 सुरक्षा**: कोड स्कैनिंग अलर्ट, गुप्त पता लगाना, और Dependabot एकीकरण
- **🔔 अधिसूचनाएं**: स्मार्ट अधिसूचना प्रबंधन और रिपोजिटरी सदस्यता नियंत्रण
- **📁 रिपोजिटरी प्रबंधन**: फ़ाइल संचालन, शाखा प्रबंधन, और रिपोजिटरी प्रशासन
- **👥 सहयोग**: उपयोगकर्ता और संगठन खोज, टीम प्रबंधन, और एक्सेस कंट्रोल

**वास्तविक दुनिया में उपयोग**: "मेरी फीचर ब्रांच से एक पुल रिक्वेस्ट बनाएं", "इस सप्ताह मेरी सभी असफल CI रन दिखाएं", "मेरे रिपोजिटरी के लिए खुली सुरक्षा अलर्ट की सूची बनाएं", या "मेरे संगठनों में मुझे असाइन की गई सभी इश्यूज खोजें"

**पूर्ण डेमो परिदृश्य**: यहाँ एक शक्तिशाली वर्कफ़्लो है जो GitHub MCP सर्वर की क्षमताओं को दिखाता है:

> "मुझे हमारी स्प्रिंट समीक्षा की तैयारी करनी है। इस सप्ताह मैं द्वारा बनाई गई सभी पुल रिक्वेस्ट दिखाएं, हमारी CI/CD पाइपलाइनों की स्थिति जांचें, किसी भी सुरक्षा अलर्ट का सारांश बनाएं जिन्हें हमें संबोधित करना है, और 'feature' लेबल वाली मर्ज की गई PRs के आधार पर रिलीज नोट्स बनाने में मदद करें।"

GitHub MCP सर्वर:
- आपकी हाल की पुल रिक्वेस्ट को विस्तृत स्थिति जानकारी के साथ क्वेरी करेगा
- वर्कफ़्लो रन का विश्लेषण करेगा और किसी भी विफलता या प्रदर्शन समस्याओं को उजागर करेगा
- सुरक्षा स्कैनिंग परिणामों को संकलित करेगा और महत्वपूर्ण अलर्ट को प्राथमिकता देगा
- मर्ज की गई PRs से जानकारी निकालकर व्यापक रिलीज नोट्स बनाएगा
- स्प्रिंट योजना और रिलीज तैयारी के लिए क्रियान्वयन योग्य अगले चरण प्रदान करेगा

**विशेष उदाहरण**: मैं कोड समीक्षा वर्कफ़्लो के लिए इसे उपयोग करना पसंद करता हूँ। VS Code, GitHub अधिसूचनाओं, और पुल रिक्वेस्ट पृष्ठों के बीच कूदने की बजाय, मैं कह सकता हूँ "मेरी समीक्षा के लिए सभी PR देखें" और फिर "PR #123 पर प्रमाणीकरण विधि में त्रुटि प्रबंधन के बारे में एक टिप्पणी जोड़ें।" सर्वर GitHub API कॉल संभालता है, चर्चा का संदर्भ बनाए रखता है, और मुझे और अधिक रचनात्मक समीक्षा टिप्पणियाँ बनाने में भी मदद करता है।

**प्रमाणीकरण विकल्प**: सर्वर OAuth (VS Code में निर्बाध) और व्यक्तिगत एक्सेस टोकन दोनों का समर्थन करता है, कॉन्फ़िगर करने योग्य टूलसेट के साथ ताकि आप केवल आवश्यक GitHub कार्यक्षमता सक्षम कर सकें। आप इसे तत्काल सेटअप के लिए रिमोट होस्टेड सेवा के रूप में या पूर्ण नियंत्रण के लिए लोकल Docker के माध्यम से चला सकते हैं।

> **💡 प्रो टिप**
> 
> अपने MCP सर्वर सेटिंग्स में `--toolsets` पैरामीटर कॉन्फ़िगर कर केवल आवश्यक टूलसेट सक्षम करें ताकि संदर्भ आकार कम हो और AI टूल चयन बेहतर हो। उदाहरण के लिए, कोर डेवलपमेंट वर्कफ़्लोज़ के लिए अपने MCP कॉन्फ़िगरेशन args में `"--toolsets", "repos,issues,pull_requests,actions"` जोड़ें, या यदि आप मुख्य रूप से GitHub मॉनिटरिंग क्षमताएं चाहते हैं तो `"--toolsets", "notifications, security"` का उपयोग करें।

### 5. 📝 MarkItDown MCP Server
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_MarkItDown_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_MarkItDown_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/markitdown)

**यह क्या करता है**: MarkItDown एक व्यापक दस्तावेज़ रूपांतरण सर्वर है जो विभिन्न फ़ाइल स्वरूपों को उच्च गुणवत्ता वाले Markdown में परिवर्तित करता है, जिसे LLM उपभोग और टेक्स्ट विश्लेषण कार्यप्रवाह के लिए अनुकूलित किया गया है।

**यह क्यों उपयोगी है**: आधुनिक दस्तावेज़ीकरण कार्यप्रवाहों के लिए आवश्यक! MarkItDown एक प्रभावशाली रेंज के फ़ाइल स्वरूपों को संभालता है जबकि महत्वपूर्ण दस्तावेज़ संरचना जैसे शीर्षक, सूचियाँ, तालिकाएँ, और लिंक संरक्षित रखता है। साधारण टेक्स्ट निष्कर्षण उपकरणों के विपरीत, यह अर्थपूर्ण अर्थ और स्वरूपण बनाए रखने पर ध्यान केंद्रित करता है जो AI प्रोसेसिंग और मानव पठनीयता दोनों के लिए मूल्यवान है।

**समर्थित फ़ाइल स्वरूप**:
- **ऑफिस दस्तावेज़**: PDF, PowerPoint (PPTX), Word (DOCX), Excel (XLSX/XLS)
- **मीडिया फ़ाइलें**: छवियाँ (EXIF मेटाडेटा और OCR के साथ), ऑडियो (EXIF मेटाडेटा और भाषण ट्रांसक्रिप्शन के साथ)
- **वेब सामग्री**: HTML, RSS फ़ीड, YouTube URL, विकिपीडिया पृष्ठ
- **डेटा स्वरूप**: CSV, JSON, XML, ZIP फ़ाइलें (सामग्री को पुनरावृत्ति से संसाधित करता है)
- **प्रकाशन स्वरूप**: EPub, जुपिटर नोटबुक (.ipynb)
- **ईमेल**: Outlook संदेश (.msg)
- **उन्नत**: उन्नत PDF प्रोसेसिंग के लिए Azure Document Intelligence एकीकरण

**उन्नत क्षमताएँ**: MarkItDown LLM-शक्तिशाली छवि विवरण (जब OpenAI क्लाइंट के साथ प्रदान किया जाए), Azure Document Intelligence उन्नत PDF प्रोसेसिंग के लिए, भाषण सामग्री के लिए ऑडियो ट्रांसक्रिप्शन, और अतिरिक्त फ़ाइल स्वरूपों के लिए एक्सटेंशन के रूप में प्लगइन सिस्टम का समर्थन करता है।

**वास्तविक दुनिया में उपयोग**: "इस PowerPoint प्रस्तुति को हमारे दस्तावेज़ीकरण साइट के लिए Markdown में परिवर्तित करें", "इस PDF से उचित शीर्षक संरचना के साथ टेक्स्ट निकालें", या "इस Excel स्प्रेडशीट को पठनीय तालिका स्वरूप में बदलें"

**विशेष उदाहरण**: उद्धृत करने के लिए [MarkItDown दस्तावेज़](https://github.com/microsoft/markitdown#why-markdown) से:

> Markdown सादे टेक्स्ट के बहुत करीबी है, जिसमें न्यूनतम मार्कअप या स्वरूपण होता है, लेकिन फिर भी महत्वपूर्ण दस्तावेज़ संरचना को प्रदर्शित करने का एक तरीका प्रदान करता है। मुख्यधारा के LLMs, जैसे OpenAI का GPT-4o, मूल रूप से "Markdown" बोलते हैं, और अक्सर बिना कहे अपने उत्तरों में Markdown को शामिल करते हैं। यह सुझाव देता है कि उन्हें विशाल मात्रा में Markdown-स्वरूपित टेक्स्ट पर प्रशिक्षित किया गया है, और वे इसे अच्छी तरह समझते हैं। एक अतिरिक्त लाभ के रूप में, Markdown कन्वेंशन्स बहुत टोकन-कुशल भी होते हैं।

MarkItDown दस्तावेज़ संरचना को संरक्षित करने में वास्तव में अच्छा है, जो AI कार्यप्रवाहों के लिए महत्वपूर्ण है। उदाहरण के लिए, जब PowerPoint प्रस्तुति को रूपांतरित किया जाता है, तो यह स्लाइड संगठन को सही शीर्षकों के साथ बनाए रखता है, तालिकाओं को Markdown तालिकाओं के रूप में निकालता है, चित्रों के लिए Alt टेक्स्ट शामिल करता है, और वक्ता नोट्स भी संसाधित करता है। चार्ट को पठनीय डेटा तालिकाओं में परिवर्तित किया जाता है, और परिणामी Markdown मूल प्रस्तुति के तार्किक प्रवाह को बनाए रखता है। यह AI सिस्टम में प्रस्तुति सामग्री को खिलाने या मौजूदा स्लाइड से दस्तावेज़ीकरण बनाने के लिए इसे आदर्श बनाता है।

- **📊 मूल्यांकन फ्रेमवर्क**: व्यापक टेक्स्ट और एजेंट मूल्यांकन चलाएं, मार्कडाउन रिपोर्ट जनरेट करें, और AI एप्लिकेशन के लिए गुणवत्ता आश्वासन लागू करें
- **🚀 प्रोटोटाइपिंग उपकरण**: GitHub-आधारित प्रोटोटाइपिंग के लिए सेटअप निर्देश प्राप्त करें और अत्याधुनिक शोध मॉडलों के लिए Microsoft Foundry Labs तक पहुंचें

**वास्तविक दुनिया के डेवलपर उपयोग**: "अपने एप्लिकेशन के लिए Phi-4 मॉडल को Azure AI Services पर डिप्लॉय करें", "मेरे दस्तावेज़ीकरण RAG सिस्टम के लिए नया सर्च इंडेक्स बनाएं", "मेरे एजेंट की प्रतिक्रियाओं का गुणवत्ता मापदंडों के खिलाफ मूल्यांकन करें", या "मेरे जटिल विश्लेषण कार्यों के लिए सबसे अच्छा तर्क मॉडल खोजें"

**पूरा डेमो परिदृश्य**: यहाँ एक शक्तिशाली AI विकास कार्यप्रवाह है:

> "मैं एक कस्टमर सपोर्ट एजेंट बना रहा हूँ। मुझे कैटलॉग से एक अच्छा तर्क मॉडल खोजने में मदद करें, इसे Azure AI Services पर डिप्लॉय करें, हमारे दस्तावेज़ीकरण से एक ज्ञान आधार बनाएं, प्रतिक्रिया गुणवत्ता का परीक्षण करने के लिए मूल्यांकन फ्रेमवर्क सेट करें, और फिर परीक्षण के लिए GitHub टोकन के साथ इंटीग्रेशन का प्रोटोटाइप बनाने में मेरी मदद करें।"

Microsoft Foundry MCP Server करेगा:
- आपके आवश्यकताओं के आधार पर इष्टतम तर्क मॉडल की सिफारिश के लिए मॉडल कैटलॉग से प्रश्न करेगा
- आपके पसंदीदा Azure क्षेत्र के लिए डिप्लॉयमेंट कमांड और कोटा जानकारी प्रदान करेगा
- आपके दस्तावेज़ीकरण के लिए उचित स्कीमा के साथ Azure AI Search इंडेक्स सेटअप करेगा
- गुणवत्ता मापदंडों और सुरक्षा जांच के साथ मूल्यांकन पाइपलाइनों को कॉन्फ़िगर करेगा
- त्वरित परीक्षण के लिए GitHub प्रमाणीकरण के साथ प्रोटोटाइपिंग कोड जनरेट करेगा
- आपकी विशिष्ट तकनीकी स्टैक के लिए व्यापक सेटअप गाइड प्रदान करेगा

**प्रसिद्ध उदाहरण**: एक डेवलपर के रूप में, मैं विभिन्न LLM मॉडलों के साथ तालमेल बनाए रखने में कठिनाई महसूस करता रहा हूं। मैं कुछ मुख्य मॉडलों को जानता हूँ, लेकिन लगता है कि मैं उत्पादकता और दक्षता के कुछ लाभों से वंचित हूं। टोकन और कोटा प्रबंधन तनावपूर्ण और कठिन है – मैं कभी-कभी नहीं जानता कि क्या मैं सही कार्य के लिए सही मॉडल चुन रहा हूँ या अपने बजट को असंगत रूप से खर्च कर रहा हूँ। मैंने इस MCP सर्वर के बारे में James Montemagno से सुना जब मैं इस पोस्ट के लिए MCP सर्वर सिफारिशों के लिए टीम के साथ बात कर रहा था, और मैं इसे इस्तेमाल करने के लिए उत्साहित हूं! मॉडल खोज क्षमताएं विशेष रूप से प्रभावशाली लगती हैं उन लोगों के लिए जो सामान्य विकल्पों से परे जाकर विशिष्ट कार्यों के लिए अनुकूलित मॉडल खोजने चाहते हैं। मूल्यांकन फ्रेमवर्क यह सुनिश्चित करने में मदद करेगा कि मैं वास्तव में बेहतर परिणाम प्राप्त कर रहा हूँ, न कि केवल कुछ नया आज़माने के लिए।

> **ℹ️ प्रायोगिक स्थिति**
> 
> यह MCP सर्वर प्रायोगिक है और सक्रिय विकास में है। फीचर्स और API में परिवर्तन हो सकते हैं। Azure AI क्षमताओं का अन्वेषण करने और प्रोटोटाइप बनाने के लिए आदर्श, लेकिन उत्पादन उपयोग के लिए स्थिरता आवश्यकताओं को सत्यापित करें।

## अगला क्या है? 🔮

## 📋 निष्कर्ष

Model Context Protocol (MCP) यह बदल रहा है कि डेवलपर्स AI सहायकों और बाहरी उपकरणों के साथ कैसे बातचीत करते हैं। ये 10 Microsoft MCP सर्वर मानकीकृत AI एकीकरण की शक्ति प्रदर्शित करते हैं, जो निर्बाध कार्यप्रवाह सक्षम करते हैं जो डेवलपर्स को उनके फ्लो स्थिति में रखते हैं जबकि शक्तिशाली बाहरी क्षमताओं तक पहुंच प्रदान करते हैं।

व्यापक Azure इकोसिस्टम एकीकरण से लेकर Playwright जैसे ब्राउज़र ऑटोमेशन और MarkItDown जैसे दस्तावेज़ प्रसंस्करण के लिए विशेष उपकरण, ये सर्वर दिखाते हैं कि MCP कैसे विभिन्न विकास परिदृश्यों में उत्पादकता बढ़ा सकता है। मानकीकृत प्रोटोकॉल सुनिश्चित करता है कि ये उपकरण सहजता से एक साथ काम करें, एक समेकित विकास अनुभव बनाते हुए।

जैसे-जैसे MCP इकोसिस्टम विकसित होता रहता है, समुदाय के साथ जुड़ाव बनाए रखना, नए सर्वरों का अन्वेषण करना, और कस्टम समाधान बनाना आपकी विकास उत्पादकता को अधिकतम करने की कुंजी होगी। MCP के खुले मानक स्वभाव का अर्थ है कि आप विभिन्न विक्रेताओं के उपकरणों को मिलाकर अपनी विशिष्ट आवश्यकताओं के लिए आदर्श कार्यप्रवाह बना सकते हैं।
