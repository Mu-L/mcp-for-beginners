## 🔧 Entendiendo los servidores MCP: Guía para principiantes

### ¿Qué son los servidores MCP?

Como principiante en el Protocolo de Contexto de Modelo (MCP), quizá te preguntes: "¿Qué es exactamente un servidor MCP y por qué debería importarme?" Comencemos con una analogía sencilla.

Piensa en los servidores MCP como asistentes especializados que ayudan a tu compañero de codificación con IA (como GitHub Copilot) a conectarse con herramientas y servicios externos. Así como usas diferentes aplicaciones en tu teléfono para distintas tareas—una para el clima, otra para navegación, otra para banca—los servidores MCP le dan a tu asistente IA la capacidad de interactuar con diferentes herramientas y servicios de desarrollo.

### El problema que resuelven los servidores MCP

Antes de los servidores MCP, si querías:
- Revisar tus recursos de Azure
- Crear un issue en GitHub
- Consultar tu base de datos
- Buscar en la documentación

Tenías que dejar de programar, abrir un navegador, ir al sitio web correspondiente y realizar estas tareas manualmente. Este cambio constante de contexto interrumpe tu flujo y reduce la productividad.

### Cómo los servidores MCP transforman tu experiencia de desarrollo

Con los servidores MCP, puedes quedarte en tu entorno de desarrollo (VS Code, Visual Studio, etc.) y simplemente pedirle a tu asistente IA que se encargue de estas tareas. Por ejemplo:

**En lugar de este flujo tradicional:**
1. Dejar de programar  
2. Abrir navegador  
3. Navegar al portal de Azure  
4. Buscar detalles de la cuenta de almacenamiento  
5. Volver a VS Code  
6. Reanudar la codificación  

**Ahora puedes hacer esto:**  
1. Preguntar a la IA: "¿Cuál es el estado de mis cuentas de almacenamiento de Azure?"  
2. Continuar codificando con la información proporcionada  

### Beneficios clave para principiantes

#### 1. 🔄 **Mantente en tu estado de flujo**
- No más cambios entre múltiples aplicaciones  
- Mantén el enfoque en el código que estás escribiendo  
- Reduce la carga mental de gestionar diferentes herramientas  

#### 2. 🤖 **Usa lenguaje natural en lugar de comandos complejos**
- En lugar de memorizar sintaxis SQL, describe qué datos necesitas  
- En lugar de recordar comandos de Azure CLI, explica qué quieres lograr  
- Deja que la IA maneje los detalles técnicos mientras te concentras en la lógica  

#### 3. 🔗 **Conecta múltiples herramientas entre sí**
- Crea flujos de trabajo poderosos combinando diferentes servicios  
- Ejemplo: "Obtener todos los issues recientes de GitHub y crear elementos de trabajo correspondientes en Azure DevOps"  
- Construye automatización sin escribir scripts complejos  

#### 4. 🌐 **Accede a un ecosistema en crecimiento**
- Benefíciate de servidores creados por Microsoft, GitHub y otras compañías  
- Combina herramientas de diferentes proveedores sin problemas  
- Únete a un ecosistema estandarizado que funciona con diferentes asistentes de IA  

#### 5. 🛠️ **Aprende haciendo**
- Empieza con servidores preconstruidos para entender los conceptos  
- Construye tus propios servidores a medida que ganes confianza  
- Usa SDKs y documentación disponibles para guiar tu aprendizaje  

### Ejemplo práctico para principiantes

Supongamos que eres nuevo en el desarrollo web y trabajas en tu primer proyecto. Así es como los servidores MCP pueden ayudar:

**Enfoque tradicional:**
```
1. Code a feature
2. Open browser → Navigate to GitHub
3. Create an issue for testing
4. Open another tab → Check Azure docs for deployment
5. Open third tab → Look up database connection examples
6. Return to VS Code
7. Try to remember what you were doing
```
  
**Con servidores MCP:**
```
1. Code a feature
2. Ask AI: "Create a GitHub issue for testing this login feature"
3. Ask AI: "How do I deploy this to Azure according to the docs?"
4. Ask AI: "Show me the best way to connect to my database"
5. Continue coding with all the information you need
```

## Comenzando con MCP en VS Code y Visual Studio 🚀

Configurar estos servidores MCP es sencillo si usas Visual Studio Code o Visual Studio 2022 con GitHub Copilot.

### Configuración en VS Code

Este es el proceso básico para VS Code:

1. **Habilitar el modo agente**: En VS Code, cambia a modo Agente en la ventana de Copilot Chat  
2. **Configurar servidores MCP**: Agrega configuraciones de servidores a tu archivo settings.json de VS Code  
3. **Iniciar servidores**: Haz clic en el botón "Iniciar" para cada servidor que quieras usar  
4. **Seleccionar herramientas**: Elige qué servidores MCP habilitar para la sesión actual  

Para instrucciones detalladas de configuración, consulta la [documentación de MCP para VS Code](https://code.visualstudio.com/docs/copilot/copilot-mcp).

> **💡 Consejo profesional: ¡Maneja servidores MCP como un experto!**
> 
> La vista de Extensiones en VS Code ahora incluye una [nueva interfaz para gestionar servidores MCP instalados](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode). Tienes acceso rápido para iniciar, detener y administrar cualquier servidor MCP instalado con una interfaz clara y sencilla. ¡Pruébalo!

### Configuración en Visual Studio 2022

Para Visual Studio 2022 (versión 17.14 o posterior):

1. **Habilitar modo agente**: Haz clic en el desplegable "Preguntar" en la ventana de GitHub Copilot Chat y selecciona "Agente"  
2. **Crear archivo de configuración**: Crea un archivo `.mcp.json` en el directorio de tu solución (ubicación recomendada: `<SOLUTIONDIR>\.mcp.json`)  
3. **Configurar servidores**: Agrega tus configuraciones de servidores MCP usando el formato estándar MCP  
4. **Aprobación de herramientas**: Cuando se te solicite, aprueba las herramientas que desees usar con los permisos de alcance apropiados  

Para instrucciones detalladas para Visual Studio, consulta la [documentación MCP para Visual Studio](https://learn.microsoft.com/visualstudio/ide/mcp-servers).

Cada servidor MCP tiene sus propios requisitos de configuración (cadenas de conexión, autenticación, etc.), pero el patrón de configuración es consistente en ambos IDEs.

### 1. 📚 Servidor Microsoft Learn Docs MCP

[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Microsoft_Docs_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Microsoft_Docs_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/mcp)

**Qué hace**: El servidor Microsoft Learn Docs MCP es un servicio alojado en la nube que proporciona a los asistentes IA acceso en tiempo real a la documentación oficial de Microsoft mediante el Protocolo de Contexto de Modelo. Se conecta a `https://learn.microsoft.com/api/mcp` y permite búsquedas semánticas a través de Microsoft Learn, documentación de Azure, documentación de Microsoft 365 y otras fuentes oficiales de Microsoft.

**Por qué es útil**: Aunque pueda parecer "solo documentación", este servidor es crucial para todos los desarrolladores que usan tecnologías Microsoft. Una de las mayores quejas de los desarrolladores .NET sobre los asistentes IA de codificación es que no están actualizados con los últimos lanzamientos de .NET y C#. El servidor Microsoft Learn Docs MCP soluciona esto proporcionando acceso en tiempo real a la documentación más actual, referencias de API y mejores prácticas. Ya sea que estés trabajando con los SDKs más recientes de Azure, explorando nuevas funciones de C# 13 o implementando patrones de vanguardia como .NET Aspire, este servidor garantiza que tu asistente IA tenga acceso a información autoritativa y actualizada para generar código preciso y moderno.

**Uso en el mundo real**: "¿Cuáles son los comandos az cli para crear una aplicación de contenedor Azure según la documentación oficial de Microsoft Learn?" o "¿Cómo configuro Entity Framework con inyección de dependencias en ASP.NET Core?" O qué tal "Revisa este código para asegurar que cumple las recomendaciones de rendimiento en la documentación de Microsoft Learn." El servidor ofrece cobertura completa de Microsoft Learn, docs de Azure y docs de Microsoft 365 usando búsquedas semánticas avanzadas para encontrar la información contextualmente más relevante. Devuelve hasta 10 fragmentos de contenido de alta calidad con títulos de artículos y URLs, siempre accediendo a la documentación actual más reciente al momento de su publicación.

**Ejemplo destacado**: El servidor expone la herramienta `microsoft_docs_search` que realiza búsqueda semántica contra la documentación técnica oficial de Microsoft. Una vez configurado, puedes hacer preguntas como "¿Cómo implemento autenticación JWT en ASP.NET Core?" y obtener respuestas detalladas y oficiales con enlaces a la fuente. La calidad de la búsqueda es excepcional porque entiende el contexto: preguntar sobre "contenedores" en un contexto de Azure retornará documentación sobre Azure Container Instances, mientras que el mismo término en un contexto .NET devuelve información relevante sobre colecciones en C#.

Esto es especialmente útil para bibliotecas y casos de uso que cambian rápidamente o fueron actualizados recientemente. Por ejemplo, en algunos proyectos recientes quise aprovechar características en los lanzamientos más recientes de .NET Aspire y Microsoft.Extensions.AI. Incluyendo el servidor Microsoft Learn Docs MCP, pude aprovechar no solo la documentación API, sino también guías y tutoriales que acababan de publicarse.

Considera agregar un prompt de sistema o [copilot-instructions.md](https://docs.github.com/copilot/how-tos/custom-instructions/adding-repository-custom-instructions-for-github-copilot) como: "Tienes acceso a `microsoft.docs.mcp` – usa esta herramienta para buscar en la documentación oficial más reciente de Microsoft al responder preguntas sobre tecnologías Microsoft como C#, Azure, ASP.NET Core o Entity Framework."

### 2. ☁️ Servidor Azure MCP
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

**Qué hace**: El Azure MCP Server es un conjunto completo de más de 15 conectores especializados para servicios de Azure que lleva todo el ecosistema de Azure a tu flujo de trabajo de IA. Esto no es solo un único servidor, es una colección potente que incluye gestión de recursos, conectividad a bases de datos (PostgreSQL, SQL Server), análisis de registros de Azure Monitor con KQL, integración con Cosmos DB y mucho más.

**Por qué es útil**: Más allá de solo gestionar recursos de Azure, este servidor mejora dramáticamente la calidad del código al trabajar con los SDKs de Azure. Cuando usas Azure MCP en modo Agente, no solo te ayuda a escribir código, te ayuda a escribir *mejor* código para Azure que sigue los patrones actuales de autenticación, las mejores prácticas de manejo de errores y aprovecha las últimas características del SDK. En lugar de recibir código genérico que podría funcionar, obtienes código que sigue los patrones recomendados por Azure para cargas de trabajo de producción.

**Módulos clave incluyen**:
- **🗄️ Conectores de Base de Datos**: Acceso en lenguaje natural directo a Azure Database para PostgreSQL y SQL Server
- **📊 Azure Monitor**: Análisis de registros impulsado por KQL e información operativa
- **🌐 Gestión de Recursos**: Gestión completa del ciclo de vida de recursos de Azure
- **🔐 Autenticación**: Patronés DefaultAzureCredential e identidad administrada
- **📦 Servicios de Almacenamiento**: Operaciones con Blob Storage, Queue Storage y Table Storage
- **🚀 Servicios de Contenedores**: Gestión de Azure Container Apps, Container Instances y AKS
- **Y muchos más conectores especializados**

**Uso en el mundo real**: "Listado de mis cuentas de almacenamiento de Azure", "Consulta mi espacio de trabajo de Log Analytics por errores en la última hora" o "Ayúdame a construir una aplicación de Azure usando Node.js con autenticación adecuada"

**Escenario de demo completo**: Aquí tienes un recorrido completo que muestra el poder de combinar Azure MCP con GitHub Copilot para la extensión de Azure en VS Code. Cuando tienes ambos instalados y das esta instrucción:

> "Crea un script en Python que suba un archivo a Azure Blob Storage usando la autenticación DefaultAzureCredential. El script debe conectarse a mi cuenta de almacenamiento de Azure llamada 'mycompanystorage', subir a un contenedor llamado 'documents', crear un archivo de prueba con la marca de tiempo actual para subir, manejar errores de forma correcta y proporcionar salida informativa, seguir las mejores prácticas de Azure para autenticación y manejo de errores, incluir comentarios que expliquen cómo funciona la autenticación DefaultAzureCredential, y hacer que el script esté bien estructurado con funciones y documentación adecuadas."

El Azure MCP Server generará un script completo en Python listo para producción que:
- Usa el SDK más reciente de Azure Blob Storage con patrones async adecuados
- Implementa DefaultAzureCredential con una explicación detallada de la cadena de respaldo (fallback)
- Incluye un manejo robusto de errores con tipos específicos de excepciones de Azure
- Sigue las mejores prácticas del SDK de Azure para la gestión de recursos y manejo de conexiones
- Proporciona registro detallado y salida informativa en consola
- Crea un script bien estructurado con funciones, documentación y sugerencias de tipos

Lo que hace esto notable es que sin Azure MCP, podrías obtener un código genérico para blob storage que funciona pero no sigue los patrones actuales de Azure. Con Azure MCP recibes código que aprovecha los métodos de autenticación más recientes, maneja escenarios de errores específicos de Azure y sigue las prácticas recomendadas por Microsoft para aplicaciones de producción.

**Ejemplo destacado**: Me ha costado recordar los comandos específicos para los CLI `az` y `azd` para uso rápido. Siempre es un proceso de dos pasos para mí: primero buscar la sintaxis, luego ejecutar el comando. A menudo simplemente entro al portal y hago clic para avanzar porque no quiero admitir que no recuerdo la sintaxis del CLI. ¡Poder simplemente describir lo que quiero es increíble, y mejor aún poder hacerlo sin salir de mi IDE!

Hay una gran lista de casos de uso en el [repositorio Azure MCP](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) para que comiences. Para guías completas de configuración y opciones avanzadas, consulta la [documentación oficial de Azure MCP](https://learn.microsoft.com/azure/developer/azure-mcp-server/).

### 3. 🐙 GitHub MCP Server

[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Server-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=github&config=%7B%22type%22%3A%20%22http%22%2C%22url%22%3A%20%22https%3A%2F%2Fapi.githubcopilot.com%2Fmcp%2F%22%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Server-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=github&config=%7B%22type%22%3A%20%22http%22%2C%22url%22%3A%20%22https%3A%2F%2Fapi.githubcopilot.com%2Fmcp%2F%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/github/github-mcp-server)

**Qué hace**: El GitHub MCP Server oficial provee integración sin fisuras con todo el ecosistema de GitHub, ofreciendo tanto acceso remoto alojado como opciones de despliegue local en Docker. No se trata solo de operaciones básicas de repositorios, es un conjunto completo que incluye gestión de GitHub Actions, flujos de trabajo de pull request, seguimiento de incidencias, escaneo de seguridad, notificaciones y capacidades avanzadas de automatización.

**Por qué es útil**: Este servidor transforma la forma en que interactúas con GitHub al llevar toda la experiencia de la plataforma directamente a tu entorno de desarrollo. En lugar de cambiar constantemente entre VS Code y GitHub.com para gestión de proyectos, revisiones de código y monitoreo de CI/CD, puedes manejar todo mediante comandos en lenguaje natural mientras te mantienes enfocado en tu código.

> **ℹ️ Nota: Diferentes Tipos de 'Agentes'**
> 
> No confundas este GitHub MCP Server con el GitHub Coding Agent (el agente IA al que puedes asignar issues para tareas de codificación automáticas). El GitHub MCP Server funciona dentro del modo Agente de VS Code para proporcionar integración con la API de GitHub, mientras que GitHub Coding Agent es una función separada que crea pull requests cuando se le asignan issues de GitHub.

**Capacidades clave incluyen**:
- **⚙️ GitHub Actions**: Gestión completa de pipelines CI/CD, monitoreo de flujos de trabajo y manejo de artefactos
- **🔀 Pull Requests**: Crear, revisar, fusionar y gestionar PRs con seguimiento completo de estado
- **🐛 Issues**: Gestión completa del ciclo de vida de incidencias, comentarios, etiquetado y asignación
- **🔒 Seguridad**: Alertas de escaneo de código, detección de secretos e integración con Dependabot
- **🔔 Notificaciones**: Gestión inteligente de notificaciones y control de suscripción a repositorios
- **📁 Gestión de Repositorios**: Operaciones con archivos, gestión de ramas y administración de repositorios
- **👥 Colaboración**: Búsqueda de usuarios y organizaciones, gestión de equipos y control de accesos

**Uso en el mundo real**: "Crea un pull request desde mi rama de características", "Muéstrame todas las ejecuciones CI fallidas esta semana", "Lista las alertas de seguridad abiertas para mis repositorios" o "Encuentra todas las issues asignadas a mí en mis organizaciones"

**Escenario de demo completo**: Aquí tienes un flujo poderoso que demuestra las capacidades del GitHub MCP Server:

> "Necesito prepararme para nuestra revisión de sprint. Muéstrame todos los pull requests que he creado esta semana, revisa el estado de nuestros pipelines CI/CD, crea un resumen de alertas de seguridad que necesitamos abordar y ayúdame a redactar las notas de lanzamiento basadas en los PRs fusionados con la etiqueta 'feature'."

El GitHub MCP Server:
- Consulta tus pull requests recientes con información detallada del estado
- Analiza ejecuciones de flujos de trabajo y resalta cualquier fallo o problema de rendimiento
- Compila resultados del escaneo de seguridad y prioriza alertas críticas
- Genera notas de lanzamiento completas extrayendo información de PRs fusionados
- Proporciona pasos accionables para la planificación del sprint y preparación del lanzamiento

**Ejemplo destacado**: Me encanta usar esto para flujos de trabajo de revisión de código. En lugar de saltar entre VS Code, notificaciones de GitHub y páginas de pull request, puedo decir "Muéstrame todos los PRs esperando mi revisión" y luego "Agrega un comentario al PR #123 preguntando sobre el manejo de errores en el método de autenticación." El servidor maneja las llamadas a la API de GitHub, mantiene el contexto de la conversación e incluso me ayuda a redactar comentarios más constructivos para la revisión.

**Opciones de autenticación**: El servidor soporta tanto OAuth (integrado sin fisuras en VS Code) como Tokens de Acceso Personal, con conjuntos de herramientas configurables para habilitar solo la funcionalidad de GitHub que necesitas. Puedes ejecutarlo como un servicio remoto alojado para configuración instantánea o localmente vía Docker para control total.

> **💡 Consejo Profesional**
> 
> Habilita solo los conjuntos de herramientas que necesitas configurando el parámetro `--toolsets` en las opciones de tu servidor MCP para reducir el tamaño del contexto y mejorar la selección de la herramienta de IA. Por ejemplo, añade `"--toolsets", "repos,issues,pull_requests,actions"` a los argumentos de configuración MCP para flujos de trabajo de desarrollo centrales, o usa `"--toolsets", "notifications, security"` si principalmente quieres capacidades de monitoreo de GitHub.

### 5. 📝 MarkItDown MCP Server
[![Instalar en VS Code](https://img.shields.io/badge/VS_Code-Install_MarkItDown_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D) [![Instalar en VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_MarkItDown_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/markitdown)

**Qué hace**: MarkItDown es un servidor integral de conversión de documentos que transforma varios formatos de archivo en Markdown de alta calidad, optimizado para la consumición por LLM y flujos de trabajo de análisis de texto.

**Por qué es útil**: ¡Esencial para flujos de trabajo de documentación modernos! MarkItDown maneja una impresionante variedad de formatos de archivo mientras preserva la estructura crítica del documento como encabezados, listas, tablas y enlaces. A diferencia de las herramientas simples de extracción de texto, se enfoca en mantener el significado semántico y el formato que es valioso tanto para el procesamiento por IA como para la legibilidad humana.

**Formatos de archivo compatibles**:
- **Documentos Office**: PDF, PowerPoint (PPTX), Word (DOCX), Excel (XLSX/XLS)
- **Archivos multimedia**: Imágenes (con metadatos EXIF y OCR), Audio (con metadatos EXIF y transcripción de voz)
- **Contenido web**: HTML, fuentes RSS, URLs de YouTube, páginas de Wikipedia
- **Formatos de datos**: CSV, JSON, XML, archivos ZIP (procesa contenidos recursivamente)
- **Formatos de publicación**: EPub, cuadernos Jupyter (.ipynb)
- **Correo electrónico**: Mensajes Outlook (.msg)
- **Avanzado**: Integración con Azure Document Intelligence para mejor procesamiento de PDFs

**Capacidades avanzadas**: MarkItDown soporta descripciones de imágenes potenciadas por LLM (cuando se dispone de un cliente OpenAI), Azure Document Intelligence para procesamiento mejorado de PDFs, transcripción de audio para contenido hablado y un sistema de plugins para ampliar a formatos de archivo adicionales.

**Uso en el mundo real**: "Convertir esta presentación PowerPoint a Markdown para nuestro sitio de documentación", "Extraer texto de este PDF con la estructura adecuada de encabezados", o "Transformar esta hoja de cálculo Excel en un formato de tabla legible"

**Ejemplo destacado**: Para citar la [documentación de MarkItDown](https://github.com/microsoft/markitdown#why-markdown):

> Markdown está extremadamente cerca del texto plano, con un marcado o formato mínimo, pero aún así proporciona una forma de representar la estructura importante del documento. Los principales LLM, como GPT-4o de OpenAI, "hablan" Markdown de forma nativa y a menudo incorporan Markdown en sus respuestas sin que se les indique. Esto sugiere que han sido entrenados con grandes cantidades de texto formateado en Markdown y lo entienden bien. Como beneficio adicional, las convenciones de Markdown también son altamente eficientes en tokens.

MarkItDown es muy bueno preservando la estructura del documento, lo cual es importante para flujos de trabajo de IA. Por ejemplo, al convertir una presentación PowerPoint, mantiene la organización de las diapositivas con los encabezados correctos, extrae tablas como tablas Markdown, incluye texto alternativo para imágenes e incluso procesa las notas del orador. Los gráficos se convierten en tablas de datos legibles, y el Markdown resultante mantiene el flujo lógico de la presentación original. Esto lo hace perfecto para alimentar contenido de presentaciones en sistemas de IA o crear documentación a partir de diapositivas existentes.

## 📋 Conclusión

El Protocolo de Contexto de Modelos (MCP) está transformando la forma en que los desarrolladores interactúan con asistentes de IA y herramientas externas.
