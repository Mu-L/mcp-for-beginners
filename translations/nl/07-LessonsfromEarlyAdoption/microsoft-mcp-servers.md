## 🔧 MCP-servers begrijpen: een gids voor beginners

### Wat zijn MCP-servers?

Als beginner met het Model Context Protocol (MCP) vraag je je misschien af: "Wat is precies een MCP-server en waarom zou het me iets kunnen schelen?" Laten we beginnen met een eenvoudige analogie.

Zie MCP-servers als gespecialiseerde assistenten die jouw AI-codeermaatje (zoals GitHub Copilot) helpen verbinden met externe tools en diensten. Net zoals je op je telefoon verschillende apps gebruikt voor verschillende taken – één voor het weer, één voor navigatie, één voor bankieren – geven MCP-servers jouw AI-assistent de mogelijkheid te communiceren met verschillende ontwikkeltools en -diensten.

### Het probleem dat MCP-servers oplossen

Voor MCP-servers, als je bijvoorbeeld:
- Je Azure-resources wilde controleren
- Een GitHub-issue wilde aanmaken
- Je database wilde raadplegen
- Documentatie doorzoeken

Moest je stoppen met coderen, een browser openen, naar de juiste website navigeren en deze taken handmatig uitvoeren. Deze constante contextwisselingen onderbreken je flow en verminderen je productiviteit.

### Hoe MCP-servers jouw ontwikkelervaring transformeren

Met MCP-servers kun je in je ontwikkelomgeving blijven (VS Code, Visual Studio, enz.) en gewoon je AI-assistent vragen deze taken uit te voeren. Bijvoorbeeld:

**In plaats van deze traditionele workflow:**
1. Stop met coderen
2. Open browser
3. Navigeer naar Azure-portal
4. Zoek opslagaccountgegevens op
5. Ga terug naar VS Code
6. Ga verder met coderen

**Kun je nu dit doen:**
1. Vraag AI: "Wat is de status van mijn Azure opslagaccounts?"
2. Ga verder met coderen met de verstrekte informatie

### Belangrijkste voordelen voor beginners

#### 5. 🛠️ **Leer door te doen**

## Aan de slag met MCP in VS Code en Visual Studio 🚀

Het instellen van deze MCP-servers is eenvoudig als je Visual Studio Code of Visual Studio 2022 met GitHub Copilot gebruikt.

### VS Code setup

Hier is het basisproces voor VS Code:

1. **Agentmodus inschakelen**: Schakel in VS Code over naar Agent-modus in het Copilot Chat-venster
2. **MCP-servers configureren**: Voeg serverconfiguraties toe aan je settings.json-bestand van VS Code
3. **Servers starten**: Klik op de "Start" knop voor elke server die je wilt gebruiken
4. **Tools selecteren**: Kies welke MCP-servers je in je huidige sessie wilt inschakelen

Voor gedetailleerde installatie-instructies, zie de [VS Code MCP-documentatie](https://code.visualstudio.com/docs/copilot/copilot-mcp).

> **💡 Protip: Beheer MCP-servers als een professional!**
> 
> De VS Code Extensions-weergave bevat nu een [handige nieuwe UI om geïnstalleerde MCP-servers te beheren](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode)! Je hebt snelle toegang om MCP-servers te starten, stoppen en beheren via een duidelijke, eenvoudige interface. Probeer het uit!

### Visual Studio 2022 setup

Voor Visual Studio 2022 (versie 17.14 of later):

1. **Agentmodus inschakelen**: Klik op de dropdown “Ask” in het GitHub Copilot Chat-venster en selecteer “Agent”
2. **Configuratiebestand maken**: Maak een `.mcp.json` bestand aan in je oplossingmap (aanbevolen locatie: `<SOLUTIONDIR>\.mcp.json`)
3. **Servers configureren**: Voeg je MCP-serverconfiguraties toe met het standaard MCP-formaat
4. **Toolgoedkeuring**: Keur de tools goed die je wilt gebruiken met de juiste scope-machtigingen wanneer daarom wordt gevraagd

Voor gedetailleerde Visual Studio-installatie-instructies, zie de [Visual Studio MCP-documentatie](https://learn.microsoft.com/visualstudio/ide/mcp-servers).

Elke MCP-server heeft zijn eigen configuratievereisten (zoals verbindingstrings, authenticatie, enz.), maar het patroon is consistent in beide IDE’s.

### 1. 📚 Microsoft Learn Docs MCP-server

[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Microsoft_Docs_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Microsoft_Docs_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/mcp)

**Wat het doet**: De Microsoft Learn Docs MCP-server is een cloudgehoste dienst die AI-assistenten realtime toegang geeft tot officiële Microsoft-documentatie via het Model Context Protocol. Het maakt verbinding met `https://learn.microsoft.com/api/mcp` en stelt semantisch zoeken mogelijk over Microsoft Learn, Azure-documentatie, Microsoft 365-documentatie en andere officiële Microsoft-bronnen.

**Waarom het nuttig is**: Hoewel het misschien "alleen maar documentatie" lijkt, is deze server eigenlijk cruciaal voor iedere ontwikkelaar die Microsoft-technologieën gebruikt. Eén van de grootste klachten van .NET-ontwikkelaars over AI-codeerassistenten is dat ze niet up-to-date zijn met de nieuwste .NET- en C#-versies. De Microsoft Learn Docs MCP-server lost dit op door realtime toegang te bieden tot de meest actuele documentatie, API-referenties en best practices. Of je nu werkt met de nieuwste Azure SDK’s, nieuwe C# 13-functies verkent of geavanceerde .NET Aspire patronen implementeert, deze server zorgt ervoor dat je AI-assistent beschikt over de gezaghebbende, actuele informatie die nodig is om accurate, moderne code te genereren.

**Praktisch gebruik**: “Wat zijn de az cli-commando’s om een Azure containerapp te maken volgens officiële Microsoft Learn-documentatie?” of “Hoe configureer ik Entity Framework met dependency injection in ASP.NET Core?” Of wat dacht je van: “Bekijk deze code om te controleren of deze voldoet aan de prestatieaanbevelingen in de Microsoft Learn-documentatie.” De server biedt uitgebreide dekking over Microsoft Learn, Azure-docs en Microsoft 365-documentatie met behulp van geavanceerd semantisch zoeken om de meest contextueel relevante informatie te vinden. Het geeft tot 10 kwalitatief hoogwaardige inhoudsfragmenten terug met artikeltitels en URL’s, altijd toegang gevend tot de nieuwste Microsoft-documentatie zodra die wordt gepubliceerd.

**Uitgelicht voorbeeld**: De server maakt de tool `microsoft_docs_search` beschikbaar die semantisch zoekt in Microsofts officiële technische documentatie. Eenmaal geconfigureerd, kun je vragen zoals “Hoe implementeer ik JWT-authenticatie in ASP.NET Core?” stellen en gedetailleerde, officiële antwoorden met bronlinks ontvangen. De zoekkwaliteit is uitstekend omdat het de context begrijpt – het vragen naar “containers” in een Azure-context geeft documentatie over Azure Container Instances, terwijl dezelfde term in een .NET-context relevante C#-collectie-informatie teruggeeft.

Dit is vooral handig voor snel veranderende of recent bijgewerkte bibliotheken en gebruiksscenario’s. Bijvoorbeeld in enkele recente codeerprojecten wilde ik functies gebruiken uit de nieuwste releases van .NET Aspire en Microsoft.Extensions.AI. Door de Microsoft Learn Docs MCP-server mee te nemen, kon ik niet alleen API-docs gebruiken, maar ook pas gepubliceerde walkthroughs en richtlijnen.

### 2. ☁️ Azure MCP-server
[![Installeren in VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![Installeren in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

**Wat het doet**: De Azure MCP Server is een uitgebreide suite van meer dan 15 gespecialiseerde Azure service connectors die het gehele Azure-ecosysteem in je AI-werkstroom brengt. Dit is niet zomaar een enkele server – het is een krachtige verzameling die resourcebeheer, databaseconnectiviteit (PostgreSQL, SQL Server), Azure Monitor loganalyse met KQL, Cosmos DB integratie en nog veel meer omvat.

**Waarom het nuttig is**: Naast het beheren van Azure-resources verbetert deze server aanzienlijk de codekwaliteit bij het werken met Azure SDK's. Wanneer je Azure MCP in Agent-modus gebruikt, helpt het je niet alleen code te schrijven – het helpt je *betere* Azure-code te schrijven die voldoet aan actuele authenticatiepatronen, beste praktijken voor foutafhandeling volgt en gebruikmaakt van de nieuwste SDK-functies. In plaats van algemene code die misschien werkt, krijg je code die de aanbevolen Azure-patronen volgt voor productieomgevingen.

**Belangrijke modules omvatten**:
- **🗄️ Databaseconnectors**: Directe toegang in natuurlijke taal tot Azure Database voor PostgreSQL en SQL Server
- **📊 Azure Monitor**: KQL-aangedreven loganalyse en operationele inzichten
- **🌐 Resourcebeheer**: Volledig beheer van levenscyclus van Azure-resources
- **🔐 Authenticatie**: DefaultAzureCredential en managed identity patronen
- **📦 Opslagservices**: Blob Storage, Queue Storage en Table Storage bewerkingen
- **🚀 Container Services**: Azure Container Apps, Container Instances en AKS beheer
- **En vele andere gespecialiseerde connectors**

**Praktijkvoorbeeld**: "Toon mijn Azure opslagaccounts", "Vraag mijn Log Analytics-werkruimte om fouten in het laatste uur te zoeken" of "Help me een Azure-applicatie te bouwen met Node.js met correcte authenticatie"

**Volledig demo scenario**: Hier is een complete walkthrough die de kracht laat zien van het combineren van Azure MCP met de GitHub Copilot for Azure-extensie in VS Code. Wanneer je beide hebt geïnstalleerd en aanstuurt met:

> "Maak een Python-script dat een bestand uploadt naar Azure Blob Storage met DefaultAzureCredential authenticatie. Het script moet verbinden met mijn Azure opslagaccount genaamd 'mycompanystorage', uploaden naar een container genaamd 'documents', een testbestand maken met de huidige tijdstempel om te uploaden, fouten op een nette manier afhandelen en informatieve output geven, Azure beste praktijken volgen voor authenticatie en foutafhandeling, commentaar bevatten dat uitlegt hoe de DefaultAzureCredential authenticatie werkt, en het script goed gestructureerd maken met juiste functies en documentatie."

De Azure MCP Server genereert een compleet, productieklaar Python-script dat:
- De nieuwste Azure Blob Storage SDK gebruikt met correcte async-patronen
- DefaultAzureCredential implementeert met een uitgebreide fallback-keten uitleg
- Robuuste foutafhandeling bevat met specifieke Azure-exceptietypen
- Azure SDK beste praktijken volgt voor resourcebeheer en verbindingsbeheer
- Gedetailleerde logging en informatieve console-uitvoer levert
- Een goed gestructureerd script creëert met functies, documentatie en type hints

Wat dit opmerkelijk maakt is dat je zonder Azure MCP misschien algemene blob storage code krijgt die werkt maar niet de huidige Azure-patronen volgt. Met Azure MCP krijg je code die de nieuwste authenticatiemethoden gebruikt, Azure-specifieke foutscenario's afhandelt en de aanbevolen Microsoft-praktijken volgt voor productieapplicaties.

**Uitgelicht voorbeeld**: Ik worstel vaak met het herinneren van de specifieke commando's voor de `az` en `azd` CLI's voor ad-hoc gebruik. Het is voor mij altijd een twee-stappen proces: eerst de syntax opzoeken, dan het commando uitvoeren. Vaak ga ik gewoon naar het portal en klik ik rond om werk gedaan te krijgen omdat ik niet wil toegeven dat ik de CLI-syntax niet kan onthouden. Gewoon kunnen beschrijven wat ik wil is geweldig, en nog beter om dat zonder mijn IDE te verlaten te kunnen doen!

Er is een mooie lijst met gebruikscases in de [Azure MCP repository](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) om je op weg te helpen. Voor uitgebreide installatiegidsen en geavanceerde configuratieopties, bekijk de [officiële Azure MCP documentatie](https://learn.microsoft.com/azure/developer/azure-mcp-server/).

### 3. 🐙 GitHub MCP Server

[![Installeren in VS Code](https://img.shields.io/badge/VS_Code-Install_Server-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=github&config=%7B%22type%22%3A%20%22http%22%2C%22url%22%3A%20%22https%3A%2F%2Fapi.githubcopilot.com%2Fmcp%2F%22%7D) [![Installeren in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Server-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=github&config=%7B%22type%22%3A%20%22http%22%2C%22url%22%3A%20%22https%3A%2F%2Fapi.githubcopilot.com%2Fmcp%2F%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/github/github-mcp-server)

**Wat het doet**: De officiële GitHub MCP Server biedt naadloze integratie met het gehele GitHub-ecosysteem en biedt zowel gehoste remote toegang als lokale Docker-implementatie-opties. Dit gaat niet alleen over basale repository-bewerkingen – het is een uitgebreide toolkit die GitHub Actions beheer, pull request workflows, issue tracking, beveiligingsscans, meldingen en geavanceerde automatiseringsmogelijkheden omvat.

**Waarom het nuttig is**: Deze server verandert de manier waarop je met GitHub omgaat door de volledige platformervaring rechtstreeks in je ontwikkelomgeving te brengen. In plaats van constant te schakelen tussen VS Code en GitHub.com voor projectmanagement, code reviews en CI/CD-monitoring, kun je alles behandelen via commando’s in natuurlijke taal terwijl je gefocust blijft op je code.

> **ℹ️ Opmerking: Verschillende soorten 'Agents'**
> 
> Verwissel deze GitHub MCP Server niet met GitHub's Coding Agent (de AI-agent waaraan je issues kunt toewijzen voor geautomatiseerde codeeropdrachten). De GitHub MCP Server werkt binnen VS Code's Agent-modus om GitHub API-integratie te bieden, terwijl GitHub's Coding Agent een aparte functie is die pull requests maakt wanneer die aan GitHub-issues worden toegewezen.

**Belangrijkste mogelijkheden omvatten**:
- **⚙️ GitHub Actions**: Complete CI/CD-pijplijnbeheer, workflow monitoring en artifact handling
- **🔀 Pull Requests**: PR's aanmaken, reviewen, mergen en beheren met uitgebreide statusbewaking
- **🐛 Issues**: Volledig beheer van de levenscyclus van issues, inclusief commentaar, labels en toewijzing
- **🔒 Beveiliging**: Code scan-meldingen, geheimdetectie en Dependabot-integratie
- **🔔 Meldingen**: Slim meldingenbeheer en repository-abonnementscontrole
- **📁 Repositorybeheer**: Bestandsbewerkingen, branchbeheer en repository-administratie
- **👥 Samenwerking**: Gebruiker en organisatie zoeken, teambeheer en toegangscontrole

**Praktijkvoorbeeld**: "Maak een pull request vanuit mijn feature branch", "Toon me alle mislukte CI runs deze week", "Lijst openstaande beveiligingsmeldingen voor mijn repositories", of "Vind alle issues die aan mij zijn toegewezen binnen mijn organisaties"

**Volledig demo scenario**: Hier is een krachtige workflow die de mogelijkheden van de GitHub MCP Server toont:

> "Ik moet me voorbereiden op onze sprint review. Toon me alle pull requests die ik deze week heb gemaakt, controleer de status van onze CI/CD-pijplijnen, maak een samenvatting van eventuele beveiligingsmeldingen die we moeten aanpakken, en help me release-notities op te stellen op basis van gemergde PR's met het label 'feature'."

De GitHub MCP Server zal:
- Je recente pull requests opvragen met gedetailleerde statusinformatie
- Workflow runs analyseren en eventuele fouten of prestatieproblemen benadrukken
- Resultaten van beveiligingsscans samenstellen en kritieke meldingen prioriteren
- Uitgebreide release-notities genereren door informatie uit gemergde PR's te halen
- Actiegerichte volgende stappen bieden voor sprintplanning en releasevoorbereiding

**Uitgelicht voorbeeld**: Ik gebruik dit graag voor code review workflows. In plaats van te springen tussen VS Code, GitHub-meldingen en pull request pagina's, kan ik zeggen "Toon me alle PR's die op mijn review wachten" en daarna "Voeg een commentaar toe aan PR #123 om te vragen naar de foutafhandeling in de authenticatiemethode." De server regelt de GitHub API-aanroepen, houdt de context van de discussie vast en helpt zelfs bij het opstellen van constructievere review-comments.

**Authenticatieopties**: De server ondersteunt zowel OAuth (naadloos in VS Code) als Personal Access Tokens, met configureerbare toolsets om alleen de GitHub-functionaliteit in te schakelen die je nodig hebt. Je kunt het draaien als een remote gehoste service voor directe setup of lokaal via Docker voor volledige controle.

> **💡 Pro tip**
> 
> Schakel alleen de benodigde toolsets in door de parameter `--toolsets` te configureren in je MCP server instellingen om contextgrootte te verkleinen en AI tool-selectie te verbeteren. Voeg bijvoorbeeld `"--toolsets", "repos,issues,pull_requests,actions"` toe aan je MCP configuratie-args voor kernontwikkelworkflows, of gebruik `"--toolsets", "notifications, security"` als je vooral GitHub monitoring mogelijkheden wilt.

**Waarom het nuttig is**: Onmisbaar voor moderne documentatieworkflows! MarkItDown verwerkt een indrukwekkend scala aan bestandsformaten, terwijl het kritieke documentstructuren zoals koppen, lijsten, tabellen en links behoudt. In tegenstelling tot eenvoudige tekstreductietools richt het zich op het behouden van semantische betekenis en opmaak, wat waardevol is voor zowel AI-verwerking als menselijke leesbaarheid.

**Ondersteunde bestandsformaten**:
- **Office-documenten**: PDF, PowerPoint (PPTX), Word (DOCX), Excel (XLSX/XLS)
- **Media-bestanden**: Afbeeldingen (met EXIF-metadata en OCR), Audio (met EXIF-metadata en spraaktranscriptie)
- **Webinhoud**: HTML, RSS-feeds, YouTube-URL’s, Wikipedia-pagina’s
- **Dataformaten**: CSV, JSON, XML, ZIP-bestanden (verwerkt inhoud recursief)
- **Publicatieformaten**: EPub, Jupyter-notebooks (.ipynb)
- **E-mail**: Outlook-berichten (.msg)
- **Geavanceerd**: Azure Document Intelligence-integratie voor geavanceerde PDF-verwerking

**Geavanceerde mogelijkheden**: MarkItDown ondersteunt door LLM aangedreven afbeeldingsbeschrijvingen (wanneer voorzien van een OpenAI-client), Azure Document Intelligence voor verbeterde PDF-verwerking, audiotranscriptie voor spraakinhoud, en een plug-insysteem om extra bestandsformaten toe te voegen.

**Toepassingen in de praktijk**: "Zet deze PowerPoint-presentatie om naar Markdown voor onze documentatiesite", "Extraheer tekst uit deze PDF met de juiste koppenstructuur", of "Transformeer deze Excel-spreadsheet naar een leesbaar tabelformaat".

**Uitgelicht voorbeeld**: Om te citeren uit de [MarkItDown-documentatie](https://github.com/microsoft/markitdown#why-markdown):

> Markdown ligt heel dicht bij gewone tekst, met minimale markup of opmaak, maar biedt toch een manier om belangrijke documentstructuur weer te geven. Mainstream LLM's, zoals OpenAI’s GPT-4o, ‘spreken’ Markdown van nature en verwerken Markdown vaak ongevraagd in hun antwoorden. Dit suggereert dat ze zijn getraind op enorme hoeveelheden Markdown-geformatteerde tekst en dit goed begrijpen. Als bijkomend voordeel zijn Markdown-conventies ook zeer token-efficiënt.

MarkItDown is zeer goed in het behouden van documentstructuur, wat essentieel is voor AI-workflows. Bijvoorbeeld bij het converteren van een PowerPoint-presentatie behoudt het de diapresentatie met de juiste koppen, extraheert het tabellen als Markdown-tabellen, voegt alternatieve tekst toe aan afbeeldingen en verwerkt het zelfs de notities van de spreker. Grafieken worden omgezet naar leesbare datatabellen en de resulterende Markdown behoudt de logische samenhang van de oorspronkelijke presentatie. Dit maakt het perfect om presentatie-inhoud te voeden aan AI-systemen of documentatie te maken van bestaande slides.

## 📋 Conclusie

Het Model Context Protocol (MCP) verandert hoe ontwikkelaars omgaan met AI-assistenten en externe tools. Deze 10 Microsoft MCP-servers tonen de kracht van gestandaardiseerde AI-integratie, waarmee naadloze workflows mogelijk worden gemaakt die ontwikkelaars in hun flow houden terwijl ze krachtige externe mogelijkheden benutten.

Van de uitgebreide integratie met het Azure-ecosysteem tot gespecialiseerde tools zoals Playwright voor browserautomatisering en MarkItDown voor documentverwerking, deze servers illustreren hoe MCP de productiviteit kan verbeteren in diverse ontwikkelscenario’s. Het gestandaardiseerde protocol zorgt ervoor dat deze tools naadloos samenwerken, waardoor een samenhangende ontwikkelervaring ontstaat.

Naarmate het MCP-ecosysteem zich blijft ontwikkelen, is betrokkenheid bij de community, het verkennen van nieuwe servers en het bouwen van maatwerkoplossingen essentieel om je ontwikkelproductiviteit te maximaliseren. Het open standaard karakter van MCP betekent dat je tools van verschillende leveranciers kunt combineren om de perfecte workflow te creëren voor jouw specifieke behoeften.

## 🎯 Oefeningen

1. **Installeren en configureren**: Zet een van de MCP-servers op in je VS Code-omgeving en test de basisfunctionaliteit.
2. **Workflow integratie**: Ontwerp een ontwikkelworkflow die minstens drie verschillende MCP-servers combineert.
3. **Planning van een aangepaste server**: Identificeer een taak in je dagelijkse ontwikkelroutine die kan profiteren van een aangepaste MCP-server en maak er een specificatie voor.
4. **Prestatieanalyse**: Vergelijk de efficiëntie van het gebruik van MCP-servers met traditionele benaderingen voor veelvoorkomende ontwikkeltaken.
5. **Beveiligingsbeoordeling**: Evalueer de beveiligingsimplicaties van het gebruik van MCP-servers in je ontwikkelomgeving en stel best practices voor.
