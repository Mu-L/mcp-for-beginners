# تغییرات: برنامه آموزشی MCP برای مبتدیان

این سند به عنوان یک رکورد از همه تغییرات مهم انجام شده در برنامه آموزشی پروتکل زمینه مدل (MCP) برای مبتدیان عمل می‌کند. تغییرات به ترتیب معکوس زمانی مستندسازی شده‌اند (تازه‌ترین تغییرات ابتدا).

## ۱۶ ژوئن ۲۰۲۶

### هم‌ترازی مشخصات MCP و اعتبارسنجی نمونه‌ها

برنامه آموزشی را با نسخه فعلی **MCP Specification 2025-11-25** و جدیدترین SDKهای رسمی اعتبارسنجی کردیم، سپس مراجعات قدیمی به مشخصات را اصلاح کرده و تأیید کردیم نمونه‌های اصلی همچنان ساخته و اجرا می‌شوند.

#### اصلاح نسخه‌های مشخصات (2025-06-18 / 2025-03-26 → 2025-11-25)

محتوای انگلیسی که هنوز ادعا می‌کرد نسخه‌های قدیمی‌تر مشخصات «رسمی/جدیدترین» استاندارد هستند به‌روزرسانی شده و لینک‌ها به مسیرهای رسمی `modelcontextprotocol.io` اشاره داده شدند:
- **05-AdvancedTopics/mcp-security/README.md**: بروزرسانی بنر «استاندارد فعلی»، مقدمه، عنوان اصول امنیتی اصلی، عنوان الزامات اجباری، بخش Microsoft Entra ID، لینک‌های مراجع و منابع و یادداشت پایانی امنیتی (۸ مرجع) به 2025-11-25
- **05-AdvancedTopics/mcp-transport/README.md**: بروزرسانی لینک منابع اضافی و بنر «استاندارد فعلی» به 2025-11-25
- **05-AdvancedTopics/mcp-realtimesearch/README.md**: جایگزینی لینک قدیمی `2025-03-26` مربوط به امنیت و اعتماد با صفحه بهترین شیوه‌های امنیتی 2025-11-25
- **03-GettingStarted/14-sampling/README.md**: به‌روزرسانی لینک مستندات رسمی نمونه‌گیری به 2025-11-25
- **03-GettingStarted/05-stdio-server/README.md**: به‌روزرسانی ارجاع به «مشخصات MCP فعلی» به زمان حال و لینک منابع اضافی به 2025-11-25 (یادداشت‌های تاریخی مربوط به کنار گذاشتن SSE دست‌نخورده ماندند برای دقت)

#### اعتبارسنجی نمونه‌ها با SDKهای فعلی

- **TypeScript (03-GettingStarted/01-first-server/solution/typescript)**: اجرای `npm install` برای `@modelcontextprotocol/sdk@1.29.0`، اجرای `tsc --noEmit` بدون خطای نوع — APIهای موجود `McpServer`/`StdioServerTransport` همچنان معتبرند
- **Python (03-GettingStarted/01-first-server/solution/python)**: اعتبارسنجی در یک محیط ایزوله `.venv` با `mcp[cli]` نسخه 1.27.2؛ `py_compile` موفقیت‌آمیز بود و `FastMCP.list_tools()` ابزارهای `add` و `subtract` را به درستی برگرداند
- تأیید شد همه نسخه‌های نمونه `@modelcontextprotocol/sdk` (`>=1.26.0` / `^1.26.0` / `^1.27.0`) بدون خطای ناسازگاری API به نسخه جاری `1.29.0` حل می‌شوند

#### هماهنگی پین‌های وابستگی (جبران شکاف نسخه‌ها)

نسخه‌های قدیمی SDKها ارتقا یافتند تا هر نمونه نسخه انتشار MCP فعلی را دنبال کند، مطابق با کنوانسیون کل مخزن:
- **03-GettingStarted/05-stdio-server/solution/typescript/package.json**: ارتقای `@modelcontextprotocol/sdk` از `^1.8.0` به `>=1.26.0` و به‌روزرسانی توصیف بسته «به‌روزرسانی شده برای MCP 2025-06-18» به «هم‌تراز با مشخصات MCP 2025-11-25»
- **10-StreamliningAIWorkflows.../lab3/code/weather_mcp/pyproject.toml** و **lab4/code/github_mcp_server/pyproject.toml**: ارتقاء پین دقیق `mcp==1.23.0` به `mcp>=1.26.0`؛ بازتولید هر دو فایل قفل `uv.lock` (`uv lock`) به گونه‌ای که قفل‌ها به نسخه جدید `mcp 1.27.2` حل شده و با مانیفست‌ها هماهنگ بمانند

### تقویت امنیت و رفع آسیب‌پذیری‌های وابستگی

یک بازبینی کامل امنیتی روی همه مانیفست‌های وابستگی و کد نمونه اجرا شد، سپس تمام هشدارهای npm و یک مشکل سطح کد اصلاح شدند. پس از اصلاح، گزارش `npm audit` نشان‌دهنده **۰ آسیب‌پذیری** در همه دایرکتوری‌های بررسی شده است.

#### آسیب‌پذیری‌های وابستگی npm (انتقالی) — رفع شده

۱۵ فایل `package-lock.json` کامیت شده بررسی شدند. آسیب‌پذیری‌ها محدود به وابستگی‌های انتقالی بودند که توسط ابزار توسعه‌دهنده MCP Inspector، مشتری OpenAI و SDK MCP وارد می‌شوند؛ همه بدون شکستن نمونه‌ها رفع شده‌اند:
- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/inspector** و **lab3/code/weather_mcp/inspector**: ارتقای `@modelcontextprotocol/inspector` (`0.16.6` / `0.14.1` → `0.22.0`) که هشدارهای بسته شده `ajv`، `brace-expansion`، `diff`، `path-to-regexp` و `ws` را پاک کرد. همچنین به npm یک ورودی `overrides` اضافه شد که نسخه اصلاح شده `shell-quote@1.8.4` را اجبار می‌کند تا هشدار حیاتی باقی مانده که توسط `concurrently` آورده شده است حذف شود؛ هر دو فایل قفل بازتولید شدند (اکنون بدون آسیب‌پذیری)
- **03-GettingStarted/samples/typescript**: اجرای `npm audit fix` نسخه انتقالی `qs` (متوسط) را به نسخه اصلاح شده ارتقاء داد
- **03-GettingStarted/samples/javascript**: اجرای `npm audit fix` نسخه انتقالی `hono` (متوسط) را به نسخه اصلاح شده ارتقاء داد
- **03-GettingStarted/03-llm-client/solution/typescript**: اجرای `npm audit fix` نسخه انتقالی `form-data` (بالا) را به نسخه اصلاح شده ارتقاء داد
- **03-GettingStarted/11-simple-auth/solution/typescript**: فایل گمشده `package-lock.json` ایجاد شد تا پروژه قابل بازتولید و بررسی باشد (بدون آسیب‌پذیری)

#### اصلاح امنیتی سطح کد (OWASP A03: تزریق)

- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/src/server.py**: حذف گزینه `shell=True` از ابزار `open_in_vscode`. پیش‌تر `subprocess.run(["start", "", vscode_path, folder_path], shell=True)` اجازه می‌داد کاراکترهای متاداده شل در مسیر پوشه توسط `cmd.exe` تفسیر شود (بردار تزریق فرمان). اکنون مستقیماً `Code.exe` حل‌شده را با مسیر پوشه به عنوان آرگومان اجرا می‌کند — بدون شل — که عملکردی معادل و ایمن است

#### بررسی وابستگی‌های پایتون

- همه مجموعه‌های نیازمندی پایتون با `pip-audit` اعتبارسنجی شدند. `05-AdvancedTopics` و `03-GettingStarted/samples/python` اعلام کردند **هیچ آسیب‌پذیری شناخته شده‌ای ندارند** (دامنه‌های `mcp` / `httpx` / `pydantic` / `python-dotenv` آنها به نسخه‌های اصلاح شده جاری حل می‌شوند)
- **09-CaseStudy/docs-mcp/solution/python/requirements.txt**: `pip-audit` وابستگی انتقالی **`werkzeug` 3.1.1** را با سه هشدار DoS مربوط به `safe_join` در نام دستگاه‌های ویندوز — `CVE-2025-66221`، `CVE-2026-21860` و `CVE-2026-27199` (همه در 3.1.6 اصلاح شده‌اند) نشان داد. پین امنیتی صریح `werkzeug>=3.1.6` اضافه شد تا نسخه اصلاح شده حل شود؛ تأیید شد محدودیت به درستی با استک `chainlit` / `mcp` / `semantic-kernel` حل می‌شود

- **وظایف (آزمایشی)**: بخش جدید درباره ویژگی آزمایشی Tasks برای پردازش‌های دائمی و بازیابی نتایج با تاخیر افزوده شد

#### استفاده پیشرفته از سرور (03-GettingStarted/10-advanced)
- **فصل جدید اضافه شده**: راهنمای جامع در مورد استفاده پیشرفته از سرور MCP، شامل معماری‌های معمول و سطح پایین سرور  
  - **سرور معمولی در برابر سطح پایین**: مقایسه دقیق و نمونه کد در پایتون و تایپ‌اسکریپت برای هر دو رویکرد  
  - **طراحی بر مبنای هندلر**: توضیح مدیریت ابزار/منبع/پرامپت بر اساس هندلر برای پیاده‌سازی‌های مقیاس‌پذیر و انعطاف‌پذیر  
  - **الگوهای کاربردی**: سناریوهای واقعی که در آن الگوهای سرور سطح پایین برای ویژگی‌ها و معماری پیشرفته سودمندند
#### احراز هویت ساده (03-GettingStarted/11-simple-auth)
- **فصل جدید اضافه شده**: راهنمای گام‌به‌گام برای پیاده‌سازی احراز هویت ساده در سرورهای MCP.

  - **پیاده‌سازی پایه احراز هویت**: الگوهای احراز هویت مبتنی بر میان‌افزار در پایتون (Starlette) و تایپ‌اسکریپت (Express)، همراه با نمونه‌های کد.
  - **پیشرفت به امنیت پیشرفته‌تر**: راهنمای شروع با احراز هویت ساده و پیشرفت به OAuth 2.1 و RBAC، همراه با مراجع به ماژول‌های امنیتی پیشرفته.

این افزودنی‌ها راهنمای عملی و کاربردی برای ساخت پیاده‌سازی‌های سرور MCP مقاوم‌تر، امن‌تر و انعطاف‌پذیرتر فراهم می‌کنند که مفاهیم پایه را با الگوهای پیشرفته تولید پیوند می‌دهند.

  - اولیه‌های سرور: منابع (منابع داده)، درخواست‌ها (قالب‌ها)، ابزارها (توابع اجرایی) با توضیحات و مثال‌های دقیق  
  - اولیه‌های مشتری: نمونه‌برداری (تکمیل‌های LLM)، استخراج (ورودی کاربر)، ثبت وقایع (اشکال‌زدایی/نظارت)  
  - به‌روزرسانی شده با الگوهای متد کشف (`*/list`)، بازیابی (`*/get`), و اجرا (`*/call`) فعلی  
- **معماری پروتکل**: معرفی مدل معماری دو لایه  
  - لایه داده: پایه JSON-RPC 2.0 با مدیریت چرخه عمر و اولیه‌ها  
  - لایه انتقال: STDIO (محلی) و HTTP جریان‌پذیر با SSE (راه دور) به عنوان مکانیزم‌های انتقال  
- **چارچوب امنیتی**: اصول جامع امنیتی شامل رضایت صریح کاربر، حفاظت از حریم خصوصی داده‌ها، ایمنی اجرای ابزار، و امنیت لایه انتقال  
- **الگوهای ارتباطی**: به‌روزرسانی پیام‌های پروتکل برای نمایش جریان‌های مقداردهی اولیه، کشف، اجرا و اطلاع‌رسانی  
- **نمونه‌های کد**: به‌روزرسانی مثال‌های چند زبانه (.NET، جاوا، پایتون، جاوااسکریپت) برای بازتاب الگوهای فعلی SDK MCP  

#### امنیت (02-Security/) - بازنگری کامل امنیت  
- **همسویی با استانداردها**: همسویی کامل با الزامات امنیتی مشخصات MCP 2025-06-18  
- **تکامل احراز هویت**: مستندسازی تکامل از سرورهای OAuth سفارشی به واگذاری به ارائه‌دهندگان هویت خارجی (Microsoft Entra ID)  
- **تحلیل تهدیدات ویژه هوش مصنوعی**: پوشش بهبود یافته بردارهای حمله مدرن هوش مصنوعی  
  - سناریوهای حمله تزریق درخواست با مثال‌های واقعی  
  - مکانیزم‌های مسموم‌سازی ابزار و الگوهای حمله "rug pull"  
  - مسموم‌سازی پنجره زمینه و حملات گیج‌کننده مدل  
- **راهکارهای امنیتی هوش مصنوعی مایکروسافت**: پوشش جامع اکوسیستم امنیتی مایکروسافت  
  - سپرهای درخواست هوش مصنوعی با تشخیص پیشرفته، برجسته‌سازی و تکنیک‌های جداکننده  
  - الگوهای یکپارچه‌سازی ایمنی محتوا Azure  
  - امنیت پیشرفته GitHub برای حفاظت از زنجیره تأمین  
- **کاهش تهدید پیشرفته**: کنترل‌های امنیتی دقیق برای  
  - ربودن نشست با سناریوهای حمله ویژه MCP و الزامات رمزنگاری شناسه نشست  
  - مشکلات Confused deputy در سناریوهای پروکسی MCP با نیاز به رضایت صریح  
  - آسیب‌پذیری‌های عبور توکن با کنترل‌های اجباری اعتبارسنجی  
- **امنیت زنجیره تأمین**: گسترش پوشش زنجیره تأمین هوش مصنوعی شامل مدل‌های پایه، خدمات تعبیه، ارائه‌دهندگان زمینه و APIهای شخص ثالث  
- **امنیت پایگاه**: بهبود یکپارچگی با الگوهای امنیت سازمانی شامل معماری اعتماد صفر و اکوسیستم امنیتی مایکروسافت  
- **سازماندهی منابع**: دسته‌بندی لینک‌های منابع جامع بر اساس نوع (مستندات رسمی، استانداردها، پژوهش، راهکارهای مایکروسافت، راهنماهای پیاده‌سازی)
