# سناریو ۳: مستندات درون ویرایشگر با سرور MCP در VS Code

## مرور کلی

در این سناریو، یاد می‌گیرید چگونه مستندات Microsoft Learn را مستقیماً به محیط Visual Studio Code خود با استفاده از سرور MCP بیاورید. به جای اینکه مدام تب‌های مرورگر را عوض کنید تا مستندات را جستجو کنید، می‌توانید مستندات رسمی را مستقیماً درون ویرایشگر خود دسترسی، جستجو و ارجاع دهید. این روش روند کاری شما را هموار می‌کند، تمرکزتان را حفظ می‌کند و امکان یکپارچه‌سازی بی‌وقفه با ابزارهایی مانند GitHub Copilot را فراهم می‌سازد.

- جستجو و خواندن مستندات درون VS Code بدون ترک محیط کدنویسی.
- ارجاع به مستندات و وارد کردن لینک‌ها مستقیماً در فایل‌های README یا دوره‌های آموزشی.
- استفاده همزمان از GitHub Copilot و MCP برای یک روند کاری مستندسازی با قدرت هوش مصنوعی.

## اهداف آموزشی

تا پایان این فصل، خواهید فهمید چگونه سرور MCP را در VS Code راه‌اندازی و استفاده کنید تا مستندسازی و روند توسعه خود را بهبود بخشید. شما قادر خواهید بود:

- فضای کاری خود را برای استفاده از سرور MCP جهت جستجوی مستندات پیکربندی کنید.
- مستندات را مستقیماً از درون VS Code جستجو و وارد کنید.
- قدرت GitHub Copilot و MCP را برای یک روند کاری پربازده‌تر و افزوده به هوش مصنوعی ترکیب کنید.

این مهارت‌ها به شما کمک می‌کنند تمرکز خود را حفظ کنید، کیفیت مستندات را بهبود ببخشید و به عنوان توسعه‌دهنده یا نویسنده فنی بهره‌وری خود را افزایش دهید.

## راه حل

برای دستیابی به دسترسی به مستندات درون ویرایشگر، یک سری مراحل را دنبال خواهید کرد که سرور MCP را با VS Code و GitHub Copilot یکپارچه می‌کند. این راه حل برای نویسندگان دوره، نویسندگان مستندات و توسعه‌دهندگانی ایده‌آل است که می‌خواهند هنگام کار با مستندات و Copilot تمرکز خود را در ویرایشگر حفظ کنند.

- به سرعت لینک‌های ارجاعی به README اضافه کنید در حالی که مستندات دوره یا پروژه را می‌نویسید.
- از Copilot برای تولید کد و MCP برای یافتن و ارجاع فوری به مستندات مرتبط استفاده کنید.
- تمرکز خود را در ویرایشگر حفظ کنید و بهره‌وری خود را افزایش دهید.

### راهنمای مرحله به مرحله

برای شروع، این مراحل را دنبال کنید. برای هر مرحله، می‌توانید از پوشه assets اسکرین شات اضافه کنید تا روند را به صورت تصویری نشان دهید.

1. **افزودن پیکربندی MCP:**
   در ریشه پروژه خود، فایلی به نام `.vscode/mcp.json` ایجاد کرده و پیکربندی زیر را اضافه کنید:
   ```json
   {
     "servers": {
       "LearnDocsMCP": {
         "url": "https://learn.microsoft.com/api/mcp"
       }
     }
   }
   ```
   این پیکربندی به VS Code می‌گوید چگونه به [`Microsoft Learn Docs MCP server`](https://github.com/MicrosoftDocs/mcp) متصل شود.
   
   ![مرحله ۱: افزودن mcp.json به پوشه .vscode](../../../../../../translated_images/fa/step1-mcp-json.c06a007fccc3edfa.webp)
    
2. **باز کردن پنل گفتگو GitHub Copilot:**
   اگر افزونه GitHub Copilot را قبلاً نصب نکرده‌اید، به بخش Extensions در VS Code بروید و آن را نصب کنید. می‌توانید آن را مستقیماً از [بازار Visual Studio Code](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot-chat) دانلود کنید. سپس، پنل گفتگو Copilot را از نوار کناری باز کنید.

   ![مرحله ۲: باز کردن پنل گفتگو Copilot](../../../../../../translated_images/fa/step2-copilot-panel.f1cc86e9b9b8cd1a.webp)

3. **فعال‌سازی حالت عامل و بررسی ابزارها:**
   در پنل گفتگو Copilot، حالت عامل را فعال کنید.

   ![مرحله ۳: فعال‌سازی حالت عامل و بررسی ابزارها](../../../../../../translated_images/fa/step3-agent-mode.cdc32520fd7dd1d1.webp)

   پس از فعال کردن حالت عامل، تأیید کنید که سرور MCP به عنوان یکی از ابزارهای در دسترس فهرست شده باشد. این اطمینان می‌دهد که عامل Copilot می‌تواند به سرور مستندات دسترسی داشته و اطلاعات مرتبط را بازیابی کند.
   
   ![مرحله ۳: تأیید ابزار سرور MCP](../../../../../../translated_images/fa/step3-verify-mcp-tool.76096a6329cbfecd.webp)
4. **شروع گفتگوی جدید و درخواست از عامل:**
   گفتگوی جدیدی را در پنل گفتگو Copilot باز کنید. اکنون می‌توانید درخواست‌های مستندسازی خود را به عامل بدهید. عامل با استفاده از سرور MCP مستندات Microsoft Learn مرتبط را مستقیماً در ویرایشگر شما بازیابی و نمایش می‌دهد.

   - *"می‌خواهم یک برنامه مطالعه برای موضوع X بنویسم. می‌خواهم آن را به مدت ۸ هفته مطالعه کنم، برای هر هفته محتوایی که باید مطالعه کنم را پیشنهاد بده."*

   ![مرحله ۴: درخواست به عامل در گفتگو](../../../../../../translated_images/fa/step4-prompt-chat.12187bb001605efc.webp)

5. **استعلام زنده:**

   > بیایید یک استعلام زنده از بخش [#get-help](https://discord.gg/D6cRhjHWSC) در دیسکورد Microsoft Foundry ([مشاهده پیام اصلی](https://discord.com/channels/1113626258182504448/1385498306720829572)) ببینیم:
   
   *"من دنبال پاسخ‌هایی برای نحوه اجرای راه‌حل چندعاملی با عامل‌های هوش مصنوعی توسعه‌یافته در Azure AI Foundry هستم. می‌بینم که روشی مستقیم مانند کانال‌های Copilot Studio برای استقرار وجود ندارد. پس روش‌های مختلف انجام این استقرار برای کاربران سازمانی که می‌خواهند تعامل داشته باشند و کار را انجام دهند چیست؟
مقالات و وبلاگ‌های متعددی می‌گویند می‌توانیم از سرویس Azure Bot برای انجام این کار استفاده کنیم که پل ارتباطی بین تیم‌های MS و عامل‌های Azure AI Foundry خواهد بود، خب آیا این کار انجام می‌شود اگر یک Azure bot راه‌اندازی کنم که از طریق Azure function به Orchestrator Agent روی Azure AI Foundry متصل شود برای اجرای هماهنگی، یا باید برای هر یک از عامل‌های AI بخشی از راه‌حل چندعاملی، Azure function بسازم تا هماهنگی در Bot framework انجام شود؟ هر پیشنهاد دیگری خیلی خوش‌آمد است.
"*

   ![مرحله ۵: استعلامات زنده](../../../../../../translated_images/fa/step5-live-queries.49db3e4a50bea273.webp)

   عامل با لینک‌ها و خلاصه‌های مستندات مرتبط پاسخ خواهد داد که می‌توانید مستقیماً در فایل‌های مارک‌داون خود وارد کنید یا به عنوان مراجع در کد استفاده نمایید.
   
### نمونه سوالات

در اینجا چند پرسش نمونه وجود دارد که می‌توانید امتحان کنید. این پرسش‌ها نشان می‌دهند چگونه سرور MCP و Copilot می‌توانند با هم کار کنند تا مستندات و ارجاعات آنی و متناسب با زمینه را بدون ترک VS Code فراهم کنند:

- "بهترین روش‌های امنیتی برای منابع Azure چیست؟"
- "یک راهنمای شروع سریع برای خدمات Azure AI پیدا کن."

این پرسش‌ها نشان می‌دهند چگونه سرور MCP و Copilot با هم کار می‌کنند تا مستندات و ارجاعات آنی و متناسب با زمینه را بدون ترک VS Code ارائه دهند.
