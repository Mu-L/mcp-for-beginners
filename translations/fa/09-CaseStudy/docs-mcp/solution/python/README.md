# تولید برنامه مطالعه با Chainlit و مستندات Microsoft Learn Docs MCP

## پیش‌نیازها

- پایتون نسخه ۳.۸ یا بالاتر
- pip (مدیر بسته پایتون)
- دسترسی به اینترنت برای اتصال به سرور Microsoft Learn Docs MCP

## نصب

1. این مخزن را کلون کنید یا فایل‌های پروژه را دانلود کنید.
2. وابستگی‌های مورد نیاز را نصب کنید:

   ```bash
   pip install -r requirements.txt
   ```

## استفاده

### سناریوی ۱: پرسش ساده به Docs MCP
یک کلاینت خط فرمان که به سرور Docs MCP متصل می‌شود، یک پرسش ارسال می‌کند و نتیجه را چاپ می‌کند.

1. اسکریپت را اجرا کنید:
   ```bash
   python scenario1.py
   ```
2. سؤال مستندسازی خود را در پرامپت وارد نمایید.

### سناریوی ۲: تولید برنامه مطالعه (اپ وب Chainlit)
یک رابط وبی (با استفاده از Chainlit) که به کاربران امکان می‌دهد برنامه مطالعه شخصی‌سازی‌شده و هفته به هفته برای هر موضوع فنی ایجاد کنند.

1. اپ Chainlit را اجرا کنید:
   ```bash
   chainlit run scenario2.py
   ```
2. آدرس محلی نمایش داده شده در ترمینال خود (مثلاً http://localhost:8000) را در مرورگر باز کنید.
3. در پنجره چت، موضوع مطالعه و تعداد هفته‌هایی که می‌خواهید مطالعه کنید را وارد کنید (مثلاً "گواهینامه AI-900، ۸ هفته").
4. برنامه با یک برنامه مطالعه هفته به هفته پاسخ خواهد داد که شامل لینک‌هایی به مستندات مربوط Microsoft Learn است.

**متغیرهای محیطی مورد نیاز:**

برای استفاده از سناریوی ۲ (اپ وب Chainlit با Azure OpenAI) باید متغیرهای محیطی زیر را در یک فایل `.env` در دایرکتوری `python` تنظیم کنید:

```
AZURE_OPENAI_CHAT_DEPLOYMENT_NAME=
AZURE_OPENAI_API_KEY=
AZURE_OPENAI_ENDPOINT=
AZURE_OPENAI_API_VERSION=
```

قبل از اجرای برنامه این مقادیر را با جزئیات منبع Azure OpenAI خود پر کنید.

> [!TIP]
> شما می‌توانید به‌راحتی مدل‌های خود را با استفاده از [Microsoft Foundry](https://ai.azure.com/) پیاده‌سازی کنید.

### سناریوی ۳: مستندسازی در ویرایشگر با سرور MCP در VS Code

به جای جابجایی بین تب‌های مرورگر برای جستجوی مستندات، می‌توانید مستندات Microsoft Learn را به‌طور مستقیم در VS Code خود با استفاده از سرور MCP بیاورید. این امکان را به شما می‌دهد که:
- مستندات را مستقیماً در VS Code جستجو و مطالعه کنید بدون ترک محیط کدنویسی خود.
- مستندات را ارجاع داده و لینک‌ها را مستقیماً در فایل README یا فایل‌های دوره خود وارد کنید.
- از GitHub Copilot و MCP به‌صورت همزمان برای تجربه‌ای هموار و مبتنی بر هوش مصنوعی در مستندسازی استفاده کنید.

**نمونه کاربردها:**
- افزودن سریع لینک‌های ارجاعی به README هنگام نوشتن مستندات دوره یا پروژه.
- استفاده از Copilot برای تولید کد و MCP برای سریعاً پیدا کردن و استناد به مستندات مرتبط.
- تمرکز در ویرایشگر و افزایش بهره‌وری.

> [!IMPORTANT]
> مطمئن شوید فایل پیکربندی معتبر [`mcp.json`](../../../../../../09-CaseStudy/docs-mcp/solution/scenario3/mcp.json) در فضای کاری شما وجود دارد (مسیر آن `.vscode/mcp.json` است).

## چرا Chainlit برای سناریوی ۲؟

Chainlit یک چارچوب مدرن متن‌باز برای ساخت برنامه‌های وب مکالمه‌ای است. این ابزار ایجاد رابط‌های کاربری مبتنی بر چت که به سرویس‌های پشتیبان مانند سرور Microsoft Learn Docs MCP متصل می‌شوند را ساده می‌کند. این پروژه از Chainlit برای ارائه روشی آسان و تعاملی جهت تولید برنامه‌های مطالعه شخصی‌سازی‌شده در زمان واقعی استفاده می‌کند. با استفاده از Chainlit می‌توانید به سرعت ابزارهای چت محور را ساخته و منتشر کنید که بهره‌وری و یادگیری را ارتقا می‌دهند.

## این برنامه چه کاری انجام می‌دهد

این برنامه به کاربران اجازه می‌دهد با وارد کردن ساده یک موضوع و مدت زمان، برنامه مطالعه شخصی‌سازی شده ایجاد کنند. برنامه ورودی شما را تجزیه می‌کند، از سرور Microsoft Learn Docs MCP محتوای مرتبط را می‌پرسد و نتایج را به صورت یک برنامه ساختارمند هفته به هفته سازماندهی می‌کند. توصیه‌های هر هفته در چت نمایش داده می‌شوند که دنبال کردن و پیگیری پیشرفت را آسان می‌کند. این یکپارچگی تضمین می‌کند که همیشه به جدیدترین و مرتبط‌ترین منابع یادگیری دسترسی داشته باشید.
