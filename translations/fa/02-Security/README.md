# امنیت MCP: محافظتی جامع برای سیستم‌های هوش مصنوعی

[![بهترین روش‌های امنیت MCP](../../../translated_images/fa/03.175aed6dedae133f.webp)](https://youtu.be/88No8pw706o)

_(برای مشاهده ویدیو این درس روی تصویر بالا کلیک کنید)_

امنیت پایه‌ای در طراحی سیستم‌های هوش مصنوعی است، به همین دلیل آن را به عنوان بخش دوم خود در اولویت قرار داده‌ایم. این با اصل **امنیت به‌وسیله طراحی** مایکروسافت از [ابتکار آینده امن](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) هماهنگ است.

پروتکل مدل زمینه‌ای (MCP) قابلیت‌های جدید و قدرتمندی را به برنامه‌های مبتنی بر هوش مصنوعی می‌آورد و در عین حال چالش‌های امنیتی منحصر به فردی ارائه می‌دهد که فراتر از ریسک‌های سنتی نرم‌افزاری هستند. سیستم‌های MCP هم با نگرانی‌های امنیتی از پیش شناخته شده (کدنویسی امن، حداقل امتیاز، امنیت زنجیره تامین) و هم تهدیدات خاص AI از جمله تزریق پرامپت، مسمومیت ابزار، ربایش جلسه، حملات معاون ناراضی، آسیب‌پذیری‌های عبور توکن، و تغییرات پویا در قابلیت‌ها مواجه می‌شوند.

این درس به بررسی مهم‌ترین ریسک‌های امنیتی در پیاده‌سازی MCP می‌پردازد — شامل تأیید هویت، مجوزدهی، اجازه‌های بیش از حد، تزریق غیرمستقیم پرامپت، امنیت جلسه، مشکلات معاون ناراضی، مدیریت توکن و آسیب‌پذیری‌های زنجیره تأمین. شما کنترل‌ها و بهترین روش‌های کاربردی برای کاهش این ریسک‌ها را خواهید آموخت در حالی که از راه‌حل‌های مایکروسافت مانند سپرهای پرامپت، ایمنی محتوای آزور و امنیت پیشرفته GitHub برای تقویت پیاده‌سازی MCP خود بهره می‌برید.

## اهداف یادگیری

تا پایان این درس، قادر خواهید بود:

- **شناسایی تهدیدات اختصاصی MCP**: شناخت ریسک‌های امنیتی منحصربه‌فرد در سیستم‌های MCP از جمله تزریق پرامپت، مسمومیت ابزار، اجازه‌های بیش از حد، ربایش جلسه، مسائل معاون ناراضی، آسیب‌پذیری عبور توکن و ریسک‌های زنجیره تأمین
- **اعمال کنترل‌های امنیتی**: پیاده‌سازی کاهش‌های مؤثر شامل احراز هویت قوی، دسترسی با حداقل امتیاز، مدیریت امن توکن، کنترل امنیت جلسه و تأیید صحت زنجیره تأمین
- **استفاده از راه‌حل‌های امنیتی مایکروسافت**: درک و استقرار Microsoft Prompt Shields، Azure Content Safety و GitHub Advanced Security برای حفاظت از بار کاری MCP
- **اعتبارسنجی امنیت ابزار**: درک اهمیت اعتبارسنجی متادیتای ابزار، نظارت بر تغییرات پویا و دفاع در برابر حملات تزریق غیرمستقیم پرامپت
- **ادغام بهترین روش‌ها**: ترکیب اصول پایه امنیتی (کدنویسی ایمن، سخت‌افزاری سرور، اعتماد صفر) با کنترل‌های اختصاصی MCP برای حفاظت جامع

# معماری و کنترل‌های امنیت MCP

پیاده‌سازی‌های مدرن MCP نیازمند رویکردهای لایه‌ای امنیتی هستند که هم تهدیدات نرم‌افزاری سنتی و هم تهدیدات مخصوص AI را پوشش دهند. مشخصات MCP که به سرعت در حال تکامل است، کنترل‌های امنیتی خود را تکمیل می‌کند و امکان ادغام بهتر با معماری‌های امنیت سازمانی و بهترین روش‌های تثبیت شده را فراهم می‌آورد.

تحقیقات گزارش دفاع دیجیتال مایکروسافت ([Microsoft Digital Defense Report](https://aka.ms/mddr)) نشان می‌دهد که **۹۸٪ از نفوذهای گزارش شده با رعایت بهداشت امنیتی قوی قابل پیشگیری هستند**. مؤثرترین استراتژی حفاظت ترکیب اقدامات پایه امنیتی با کنترل‌های اختصاصی MCP است — اقدامات امنیتی پایه اثبات شده همچنان بیشترین تأثیر را در کاهش ریسک کلی امنیتی دارند.

## چشم‌انداز فعلی امنیت

> **توجه:** این اطلاعات استانداردهای امنیت MCP تا تاریخ **۵ فوریه ۲۰۲۶** را منعکس می‌کند که مطابق با **مشخصات MCP نسخه ۲۰۲۵-۱۱-۲۵** است. پروتکل MCP همچنان به سرعت در حال توسعه است و پیاده‌سازی‌های آینده ممکن است الگوهای جدید احراز هویت و کنترل‌های پیشرفته‌تری را معرفی کنند. همیشه به [مشخصات MCP](https://spec.modelcontextprotocol.io/)، [مخزن GitHub MCP](https://github.com/modelcontextprotocol)، و [مستندات بهترین روش‌های امنیتی](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) برای راهنمایی به‌روز مراجعه کنید.

[کارگاه قله امنیت MCP](https://azure-samples.github.io/sherpa/) آموزش امنیتی عملی و قابل اقدام را از طریق روش اثبات شده "آسیب‌پذیری → بهره‌برداری → رفع → اعتبارسنجی" فراهم می‌کند. در این کارگاه:

- **با خراب کردن یاد بگیرید**: آسیب‌پذیری‌ها را از نزدیک تجربه کنید با بهره‌برداری از سرورهای عمداً ناامن
- **استفاده از امنیت بومی آزور**: از Azure Entra ID، Key Vault، مدیریت API و ایمنی محتوا بهره ببرید
- **پیروی از دفاع در عمق**: مرحله به مرحله از کمپ‌های مختلف عبور کرده و لایه‌های امنیتی جامع بسازید
- **استفاده از استانداردهای OWASP**: هر تکنیک با [راهنمای امنیت MCP آزور OWASP](https://microsoft.github.io/mcp-azure-security-guide/) هم‌تراز است
- **دریافت کد تولیدی**: با پیاده‌سازی‌های کاری و تست شده کارگاه را ترک کنید

### تکامل احراز هویت MCP

مشخصات MCP به طور قابل توجهی در رویکرد به احراز هویت و مجوزدهی رشد کرده است:

- **رویکرد اولیه**: مشخصات اولیه توسعه‌دهندگان را ملزم می‌کرد سرورهای احراز هویت سفارشی پیاده‌سازی کنند، سرورهای MCP به عنوان سرورهای مجوز OAuth 2.0 با مدیریت مستقیم احراز هویت کاربران عمل می‌کردند
- **استاندارد کنونی (۲۰۲۵-۱۱-۲۵)**: مشخصات به‌روزرسانی شده اجازه می‌دهد سرورهای MCP احراز هویت را به ارائه‌دهندگان هویت خارجی (مانند Microsoft Entra ID) واگذار کنند، که وضعیت امنیتی را بهبود می‌بخشد و پیچیدگی پیاده‌سازی را کاهش می‌دهد
- **امنیت لایه حمل و نقل**: پشتیبانی بهبودیافته از مکانیزم‌های انتقال ایمن با الگوهای احراز هویت مناسب برای اتصالات محلی (STDIO) و راه دور (Streamable HTTP)

## امنیت احراز هویت و مجوزدهی

### چالش‌های امنیتی فعلی

پیاده‌سازی‌های مدرن MCP با چندین چالش در احراز هویت و مجوزدهی مواجه‌اند:

### ریسک‌ها و بردارهای تهدید

- **منطق مجوزدهی پیکربندی شده نادرست**: پیاده‌سازی معیوب مجوزدهی در سرورهای MCP می‌تواند داده‌های حساس را افشا کند و کنترل دسترسی را به اشتباه اعمال نماید
- **نفوذ توکن OAuth**: سرقت توکن سرور MCP محلی به مهاجمان اجازه می‌دهد خود را به جای سرورها جا بزنند و به خدمات پایین‌دست دسترسی پیدا کنند
- **آسیب‌پذیری‌های عبور توکن**: مدیریت نامناسب توکن باعث ایجاد دورزدن کنترل‌های امنیتی و شکاف‌های مسئولیت‌پذیری می‌شود
- **مجوزهای بیش از حد**: سرورهای MCP با امتیاز بیش از حد اصول حداقل امتیاز را نقض می‌کنند و سطح حمله را افزایش می‌دهند

#### عبور توکن: یک الگوی ضد امنیتی بحرانی

**عبور توکن به طور صریح در مشخصات مجوزدهی MCP فعلی ممنوع است** به دلیل پیامدهای شدید امنیتی:

##### دورزدن کنترل‌های امنیتی
- سرورهای MCP و APIهای پایین‌دست کنترل‌های امنیتی حیاتی (محدودیت نرخ، اعتبارسنجی درخواست، پایش ترافیک) را که به تأیید صحیح توکن وابسته‌اند، اجرا می‌کنند
- استفاده مستقیم توکن از مشتری به API باعث دورزدن این محافظت‌های اساسی شده و ساختار امنیتی را تضعیف می‌کند

##### مسائل مسئولیت‌پذیری و حسابرسی  
- سرورهای MCP تفاوت بین مشتریانی که از توکن‌های صادر شده توسط بالادست استفاده می‌کنند را نمی‌توانند تشخیص دهند و ردیابی حسابرسی را تخریب می‌کنند
- لاگ‌های سرورهای منابع پایین‌دست منابع درخواست‌ها را واقعی نشان نمی‌دهند بلکه مبادی نامعتبر MCP را گزارش می‌دهند
- تحقیقات حادثه و حسابرسی انطباق به طور قابل توجهی دشوار می‌شود

##### ریسک‌های خروج داده‌ها
- ادعاهای توکن تأییدنشده امکان استفاده مهاجمان با توکن‌های سرقت‌شده را از سرورهای MCP به عنوان پراکسی برای خروج داده‌ها فراهم می‌آورد
- نقض مرز اعتماد اجازه دسترسی‌های غیرمجاز و دورزدن کنترل‌های امنیتی هدفمند را می‌دهد

##### بردارهای حمله چندخدمتی
- توکن‌های به خطر افتاده که توسط چندین سرویس پذیرفته می‌شوند حرکت جانبی در سیستم‌های متصل را امکان‌پذیر می‌سازند
- فرضیات اعتماد بین سرویس‌ها زمانی که منشأ توکن قابل تأیید نیست، نقض می‌شود

### کنترل‌ها و کاهش‌های امنیتی

**الزامات مهم امنیتی:**

> **الزامی:** سرورهای MCP **نباید** هیچ توکنی را بپذیرند که صریحاً برای سرور MCP صادر نشده باشد

#### کنترل‌های احراز هویت و مجوزدهی

- **بازبینی دقیق مجوزدهی**: حسابرسی جامع منطق مجوزدهی سرور MCP را انجام دهید تا اطمینان حاصل شود فقط کاربران و مشتریان مجاز به منابع حساس دسترسی دارند
  - **راهنمای پیاده‌سازی**: [مدیریت API آزور به عنوان دروازه احراز هویت برای سرورهای MCP](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **ادغام هویت**: [استفاده از Microsoft Entra ID برای احراز هویت سرور MCP](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **مدیریت امن توکن**: پیاده‌سازی [بهترین روش‌های مدیریت و اعتبارسنجی توکن مایکروسافت](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens)
  - اعتبارسنجی ادعاهای مخاطب توکن با هویت سرور MCP
  - پیاده‌سازی سیاست‌های گردش و انقضای توکن مناسب
  - جلوگیری از حملات بازپخش توکن و استفاده غیرمجاز

- **ذخیره‌سازی امن توکن**: ذخیره توکن به صورت رمزنگاری شده در حالت سکون و انتقال
  - **بهترین روش‌ها**: [دستورالعمل‌های ذخیره‌سازی امن توکن و رمزنگاری](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### پیاده‌سازی کنترل دسترسی

- **اصل حداقل امتیاز**: تنها حداقل اجازه‌های لازم به سرورهای MCP اعطا شود
  - بازبینی و به‌روزرسانی مرتب اجازه‌ها برای جلوگیری از گسترش امتیاز
  - **مستندات مایکروسافت**: [دسترسی امن با حداقل امتیاز](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **کنترل دسترسی مبتنی بر نقش (RBAC)**: تخصیص نقش‌های دقیق و مشخص
  - محدود کردن نقش‌ها به منابع و عملکردهای خاص
  - اجتناب از اجازه‌های گسترده یا غیرضروری که سطح حمله را افزایش می‌دهند

- **نظارت مداوم بر اجازه‌ها**: حسابرسی و پایش مداوم دسترسی‌ها
  - پایش الگوهای استفاده از اجازه‌ها برای شناسایی ناهنجاری‌ها
  - رفع سریع اجازه‌های بیش از حد یا استفاده‌نشده

## تهدیدات امنیتی اختصاصی هوش مصنوعی

### حملات تزریق پرامپت و دستکاری ابزار

پیاده‌سازی‌های مدرن MCP با بردارهای حمله پیشرفته اختصاصی AI مواجه‌اند که تدابیر امنیتی سنتی قادر به مقابله کامل با آن‌ها نیستند:

#### **تزریق غیرمستقیم پرامپت (تزریق پرامپت میان‌دامنه‌ای)**

**تزریق غیرمستقیم پرامپت** یکی از بحرانی‌ترین آسیب‌پذیری‌ها در سیستم‌های AI مجهز به MCP است. مهاجمان دستورالعمل‌های مخرب را در محتوای خارجی — اسناد، صفحات وب، ایمیل‌ها یا منابع داده — جاسازی می‌کنند که سیستم‌های AI آن‌ها را به عنوان فرمان‌های مشروع پردازش می‌کنند.

**سناریوهای حمله:**
- **تزریق مبتنی بر سند**: دستورالعمل‌های مخفی در اسناد پردازش شده که باعث رفتار ناخواسته AI می‌شوند
- **سوءاستفاده از محتوای وب**: صفحات وب آلوده با پرامپت‌های جاسازی شده که رفتار AI را هنگام استخراج تغییر می‌دهند
- **حملات مبتنی بر ایمیل**: پرامپت‌های مخرب در ایمیل‌ها که باعث افشای اطلاعات یا انجام اقدامات غیرمجاز توسط دستیارهای AI می‌شوند
- **آلوده‌سازی منابع داده**: پایگاه داده‌ها یا APIهای آلوده که محتواهای آلوده به AI ارائه می‌دهند

**تأثیر واقعی**: این حملات می‌توانند به خروج اطلاعات، نقض حریم خصوصی، تولید محتوای مضر و دستکاری تعاملات کاربری منجر شوند. برای تحلیل دقیق‌تر، به [تزریق پرامپت در MCP (سایمون ویلیسون)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/) مراجعه کنید.

![نمودار حمله تزریق پرامپت](../../../translated_images/fa/prompt-injection.ed9fbfde297ca877.webp)

#### **حملات مسمومیت ابزار**

**مسمومیت ابزار** متادیتایی را که ابزارهای MCP را تعریف می‌کند هدف قرار می‌دهد و نحوه تفسیر ابزار توسط مدل‌های زبان بزرگ برای تصمیم‌گیری‌های اجرایی را به خطر می‌اندازد.

**مکانیزم‌های حمله:**
- **دستکاری متادیتا**: مهاجمان دستورالعمل‌های مخرب را در توصیف ابزار، تعریف پارامترها یا نمونه‌های استفاده تزریق می‌کنند
- **دستورالعمل‌های نامرئی**: پرامپت‌های پنهان در متادیتای ابزار که توسط مدل‌های AI پردازش می‌شوند ولی برای کاربران انسانی قابل مشاهده نیستند
- **تغییر پویا در ابزار ("کلاهبرداری ناگهانی")**: ابزارهایی که توسط کاربران تأیید شده‌اند بعداً به نحوی تغییر می‌یابند که عملکرد مخرب داشته باشند بدون آگاهی کاربران
- **تزریق پارامتر**: محتوای مخرب جاسازی شده در طرح‌واره پارامترهای ابزار که رفتار مدل را تحت تأثیر قرار می‌دهد

**ریسک‌های سرور میزبانی شده**: سرورهای MCP راه دور ریسک بالاتری دارند زیرا تعاریف ابزار می‌توانند پس از تأیید اولیه کاربر به‌روزرسانی شوند و وضعیتی ایجاد کنند که ابزارهای قبلاً ایمن به ابزارهای مخرب تبدیل شوند. برای تحلیل جامع‌تر، به [حملات مسمومیت ابزار (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks) مراجعه کنید.

![نمودار حمله تزریق ابزار](../../../translated_images/fa/tool-injection.3b0b4a6b24de6bef.webp)

#### **بردارهای حمله اضافی AI**

- **تزریق پرامپت میان‌دامنه‌ای (XPIA)**: حملات پیشرفته‌ای که از محتوای چندین دامنه برای دور زدن کنترل‌های امنیتی استفاده می‌کنند
- **تغییر پویا در قابلیت‌ها**: تغییرات آنی در قابلیت‌های ابزار که از ارزیابی‌های امنیتی اولیه فرار می‌کنند  
- **آلوده‌سازی پنجره زمینه**: حملاتی که پنجره‌های زمینه بزرگ را دستکاری می‌کنند تا دستورات مخرب پنهان بمانند  
- **حملات سردرگمی مدل**: سوءاستفاده از محدودیت‌های مدل برای ایجاد رفتارهای غیرقابل پیش‌بینی یا ناامن  


### تأثیر ریسک‌های امنیتی هوش مصنوعی

**عواقب با تأثیر بالا:**  
- **خروج غیرمجاز داده‌ها**: دسترسی و سرقت بدون مجوز به داده‌های حساس سازمانی یا شخصی  
- **نقض حریم خصوصی**: افشای اطلاعات شناسایی فردی (PII) و داده‌های محرمانه کسب‌وکار  
- **دستکاری سیستم**: تغییرات ناخواسته در سیستم‌ها و جریان‌های کاری حساس  
- **سرقت اطلاعات احراز هویت**: به خطر افتادن توکن‌ها و مدارک خدمات  
- **حرکت جانبی**: استفاده از سیستم‌های هوش مصنوعی به خطر افتاده به عنوان نقطه شروع حملات شبکه‌ای گسترده‌تر  


### راهکارهای امنیتی هوش مصنوعی مایکروسافت

#### **سپرهای درخواست هوش مصنوعی: حفاظت پیشرفته در برابر حملات تزریق**

مایکروسافت **سپرهای درخواست هوش مصنوعی** با استفاده از چندین لایه امنیتی، دفاع جامعی در برابر حملات تزریق مستقیم و غیرمستقیم فراهم می‌کنند:

##### **مکانیزم‌های اصلی حفاظت:**

1. **تشخیص و فیلترینگ پیشرفته**  
   - الگوریتم‌های یادگیری ماشین و فنون پردازش زبان طبیعی برای شناسایی دستورات مخرب در محتویات خارجی  
   - تحلیل بلادرنگ اسناد، صفحات وب، ایمیل‌ها و منابع داده برای شناسایی تهدیدهای تعبیه‌شده  
   - درک زمینه‌ای از الگوهای دستورالعمل‌های مشروع در برابر مخرب

2. **تکنیک‌های برجسته‌سازی**  
   - تمایز بین دستورات سیستم قابل اعتماد و ورودی‌های خارجی احتمالا به خطر افتاده  
   - روش‌های تبدیل متن که ارتباط مدل را بهبود می‌بخشد در حالی که محتوی مخرب را جدا می‌کند  
   - کمک به سیستم‌های هوش مصنوعی برای حفظ سلسله‌مراتب دستورالعمل و نادیده گرفتن فرمان‌های تزریق‌شده

3. **سیستم‌های جداکننده و داده‌گذاری**  
   - تعریف صریح مرز بین پیام‌های سیستم قابل اعتماد و متن ورودی خارجی  
   - نشانه‌های خاص برای مشخص کردن مرز بین منابع داده معتبر و نامعتبر  
   - جداسازی واضح برای جلوگیری از سردرگمی دستورالعمل و اجرای دستورات غیرمجاز

4. **اطلاعات تهدید پیوسته**  
   - مایکروسافت به طور مداوم الگوهای حمله نوظهور را نظارت و دفاعیات را به‌روزرسانی می‌کند  
   - شکار تهدید پیشگیرانه برای تکنیک‌ها و بردارهای حمله جدید تزریق  
   - به‌روزرسانی‌های منظم مدل‌های امنیتی برای حفظ اثربخشی در برابر تهدیدات در حال تحول

5. **یکپارچگی Azure Content Safety**  
   - بخشی از مجموعه جامع Azure AI Content Safety  
   - تشخیص اضافی تلاش‌های فرار از محدودیت، محتوای مضر و نقض سیاست‌های امنیتی  
   - کنترل‌های امنیتی متحد در سراسر اجزای کاربردی هوش مصنوعی

## تهدیدات پیشرفته امنیتی MCP

### آسیب‌پذیری سرقت نشست

**سرقت نشست** یک بردار حمله حیاتی در پیاده‌سازی‌های حالت‌دار MCP است که در آن افراد غیرمجاز شناسه‌های نشست معتبر را به دست آورده و سوءاستفاده می‌کنند تا خود را به جای مشتریان جا بزنند و اقدامات غیرمجاز انجام دهند.

#### **سناریوهای حمله و ریسک‌ها**

- **تزریق درخواست سرقت نشست**: مهاجمان با شناسه‌های نشست سرقت شده، رویدادهای مخرب را به سرورهای به اشتراک گذارنده وضعیت نشست تزریق می‌کنند که ممکن است اقدامات مضر را فعال یا به داده‌های حساس دسترسی پیدا کنند  
- **جعل هویت مستقیم**: شناسه‌های نشست سرقت شده امکان تماس مستقیم با سرور MCP بدون احراز هویت را برای مهاجمان فراهم کرده و آن‌ها را به کاربران معتبر تبدیل می‌کند  
- **جریان‌های قابل ادامه به خطر افتاده**: مهاجمان می‌توانند درخواست‌ها را زودتر از موعد خاتمه دهند که باعث می‌شود مشتریان قانونی با محتوای احتمالا مخرب ادامه دهند

#### **کنترل‌های امنیتی مدیریت نشست**

**الزامات حیاتی:**  
- **تأیید مجوز**: سرورهای MCP پیاده‌سازی‌کننده مجوز **باید** همه درخواست‌های ورودی را تأیید کنند و **نباید** فقط به نشست‌ها برای احراز هویت تکیه کنند  
- **تولید نشست امن**: استفاده از شناسه‌های نشست رمزنگاری‌شده، غیرقطعی و ایجاد شده با مولدهای عدد تصادفی امن  
- **پیوند سفارشی به کاربر**: اتصال شناسه‌های نشست به اطلاعات کاربر با قالب‌هایی مانند `<user_id>:<session_id>` برای جلوگیری از سوءاستفاده نشست میان کاربران  
- **مدیریت چرخه عمر نشست**: پیاده‌سازی انقضا، گردش و باطل‌سازی مناسب برای محدود کردن بازه آسیب‌پذیری  
- **امنیت انتقال**: استفاده اجباری از HTTPS برای تمامی ارتباطات جهت جلوگیری از رهگیری شناسه‌های نشست

### مشکل سردرگمی نماینده (Confused Deputy)

مشکل **نماینده سردرگم** زمانی رخ می‌دهد که سرورهای MCP به عنوان نماینده‌های احراز هویت میان مشتریان و خدمات شخص ثالث عمل کنند، و فرصت‌هایی برای دور زدن مجوز از طریق سوءاستفاده از شناسه مشتری ایستا ایجاد شود.

#### **مکانیزم‌ها و ریسک‌های حمله**

- **دور زدن رضایت مبتنی بر کوکی**: احراز هویت پیشین کاربر، کوکی‌های رضایت ایجاد می‌کند که مهاجمان از طریق درخواست‌های مجوز با URI بازگشت دستکاری شده بهره‌برداری می‌کنند  
- **سرقت کد مجوز**: کوکی‌های رضایت موجود ممکن است سرورهای مجوز را وادار کنند که صفحات رضایت را رد کرده و کدها را به نقاط انتهایی تحت کنترل مهاجم هدایت کنند  
- **دسترسی غیرمجاز به API**: کدهای مجوز سرقت شده امکان تبادل توکن و جعل هویت کاربر را بدون نیاز به تأیید صریح فراهم می‌کند

#### **استراتژی‌های کاهش ریسک**

**کنترل‌های ضروری:**  
- **الزامات رضایت واضح**: سرورهای نماینده MCP با شناسه‌های مشتری ایستا **باید** رضایت کاربر را برای هر مشتری ثبت‌شده پویا دریافت کنند  
- **پیاده‌سازی امنیت OAuth 2.1**: پیروی از بهترین شیوه‌های امنیتی OAuth شامل PKCE (اثبات کلید برای تبادل کد) برای کلیه درخواست‌های مجوز  
- **اعتبارسنجی سخت‌گیرانه مشتری**: پیاده‌سازی ارزیابی دقیق URI بازگشت و شناسه‌های مشتری برای جلوگیری از سوءاستفاده

### آسیب‌پذیری عبور توکن  

**عبور توکن** یک الگوی ضدکارکرد صریح است که در آن سرورهای MCP توکن‌های مشتری را بدون اعتبارسنجی مناسب قبول کرده و به APIهای پایین‌دست منتقل می‌کنند که با مشخصات مجوز MCP مغایرت دارد.

#### **پیامدهای امنیتی**

- **دور زدن کنترل‌ها**: استفاده مستقیم توکن مشتری به API، محدودیت‌های نرخ، اعتبارسنجی و کنترل‌های نظارتی را دور می‌زند  
- **خرابی رد صلاحیت وقایع**: توکن‌های صادرشده از سمت بالا امکان شناسایی مشتری را غیرممکن کرده و قابلیت‌های بررسی حادثه را مختل می‌کند  
- **خروج غیرقانونی داده‌ها با واسطه پروکسی**: توکن‌های تأییدنشده به بازیگران مخرب اجازه می‌دهند به عنوان پروکسی برای دسترسی غیرمجاز به داده‌ها از سرورها استفاده کنند  
- **نقض مرزهای اعتماد**: خدمات پایین‌دستی ممکن است فرضیات اعتماد را نقض شده ببینند وقتی منشاء توکن‌ها قابل بررسی نباشد  
- **گسترش حملات چندخدمتی**: توکن‌های به خطر افتاده که در چندین سرویس پذیرفته می‌شوند، امکان حرکت جانبی را فراهم می‌کنند

#### **کنترل‌های امنیتی لازم**

**الزامات غیرقابل مذاکره:**  
- **اعتبارسنجی توکن**: سرورهای MCP **نباید** توکن‌هایی که به صراحت برای سرور MCP صادر نشده‌اند را بپذیرند  
- **تأیید مخاطب**: همواره ادعاهای مخاطب توکن را با هویت سرور MCP تطابق دهید  
- **چرخه عمر مناسب توکن**: استفاده از توکن‌های دسترسی کوتاه‌مدت با روند گردش امن


## امنیت زنجیره تأمین برای سیستم‌های هوش مصنوعی

امنیت زنجیره تأمین فراتر از وابستگی‌های نرم‌افزاری سنتی توسعه یافته و کل اکوسیستم هوش مصنوعی را شامل می‌شود. پیاده‌سازی‌های مدرن MCP باید تمامی اجزای مرتبط با هوش مصنوعی را به دقت تأیید و نظارت کنند، چرا که هر کدام ممکن است آسیب‌پذیری‌هایی ایجاد کنند که به تمامیت سیستم لطمه بزنند.

### اجزای گسترش یافته زنجیره تامین هوش مصنوعی

**وابستگی‌های نرم‌افزاری سنتی:**  
- کتابخانه‌ها و چارچوب‌های متن‌باز  
- ایمیج‌های کانتینر و سیستم‌های پایه  
- ابزارهای توسعه و خطوط ساخت  
- اجزای زیرساخت و سرویس‌ها

**عناصر زنجیره تامین خاص هوش مصنوعی:**  
- **مدل‌های پایه**: مدل‌های پیش‌آموزش‌دیده از ارائه‌دهندگان مختلف که نیازمند تأیید اصالت هستند  
- **سرویس‌های درون‌یابی**: سرویس‌های خارجی برای تبدیل به بردار و جستجوی معنایی  
- **تأمین کنندگان زمینه**: منابع داده، پایگاه‌های دانش و مخازن اسناد  
- **APIهای شخص ثالث**: سرویس‌های هوش مصنوعی خارجی، خطوط پردازش یادگیری ماشین و نقاط پایان داده  
- **مصنوعات مدل**: وزن‌ها، پیکربندی‌ها و انواع مدل‌های دقیق تنظیم‌شده  
- **منابع داده آموزش**: مجموعه داده‌های استفاده شده برای آموزش و تنظیم دقیق مدل

### استراتژی جامع امنیت زنجیره تأمین

#### **تأیید و اعتماد به اجزا**  
- **اعتبارسنجی اصالت**: بررسی منشاء، مجوز و صحت تمام اجزای هوش مصنوعی پیش از ادغام  
- **ارزیابی امنیتی**: انجام اسکن‌های آسیب‌پذیری و مرورهای امنیتی برای مدل‌ها، منابع داده و سرویس‌های AI  
- **تحلیل اعتبار**: ارزیابی سابقه امنیتی و رویه‌های ارائه‌دهندگان خدمات هوش مصنوعی  
- **تأیید انطباق**: اطمینان از رعایت تمام اجزا با الزامات امنیتی سازمانی و مقرراتی

#### **خطوط پیاده‌سازی امن**  
- **امنیت CI/CD خودکار**: ادغام اسکن‌های امنیتی در تمامی مراحل خطوط پیاده‌سازی خودکار  
- **تمامیت مصنوعات**: پیاده‌سازی اعتبارسنجی رمزنگاری برای همه مصنوعات منتشر شده (کد، مدل‌ها، پیکربندی‌ها)  
- **انتشار مرحله‌ای**: استفاده از استراتژی‌های انتشار تدریجی با اعتبارسنجی امنیتی در هر مرحله  
- **مخازن مصنوعات معتبر**: انتشار تنها از مخازن و رژیستری‌های مطمئن و تأیید شده

#### **نظارت و پاسخ مستمر**  
- **اسکن وابستگی‌ها**: نظارت مداوم آسیب‌پذیری برای تمامی وابستگی‌های نرم‌افزاری و اجزای هوش مصنوعی  
- **نظارت مدل**: ارزیابی مستمر رفتار مدل، انحراف عملکرد و ناهنجاری‌های امنیتی  
- **ردیابی سلامت سرویس**: نظارت بر سرویس‌های هوش مصنوعی خارجی برای در دسترس بودن، رخدادهای امنیتی و تغییرات سیاستی  
- **یکپارچگی هوش تهدید**: ادغام خوراک‌های تهدید مخصوص ریسک‌های امنیتی AI و ML

#### **کنترل دسترسی و حداقل امتیاز**  
- **مجوزهای سطح جز**: محدود کردن دسترسی به مدل‌ها، داده‌ها و سرویس‌ها بر اساس نیازهای کسب‌وکار  
- **مدیریت حساب سرویس**: پیاده‌سازی حساب‌های سرویس اختصاصی با کمترین مجوزهای لازم  
- **بخش‌بندی شبکه**: جداسازی اجزای هوش مصنوعی و محدود کردن دسترسی شبکه بین سرویس‌ها  
- **کنترل دروازه API**: استفاده از دروازه‌های API متمرکز برای کنترل و نظارت دسترسی به سرویس‌های هوش مصنوعی خارجی

#### **پاسخ و بازیابی از حادثه**  
- **اقدامات واکنش سریع**: فرایندهای تعیین‌شده برای وصله‌گذاری یا جایگزینی اجزای هوش مصنوعی به خطر افتاده  
- **گردش مدارک**: سیستم‌های خودکار برای گردش اسرار، کلیدهای API و مدارک خدمات  
- **قابلیت بازگردانی**: امکان بازگردانی سریع به نسخه‌های قبلی سالم اجزای هوش مصنوعی  
- **بازیابی نقض زنجیره تأمین**: فرایندهای ویژه پاسخ به به خطر افتادن سرویس‌های بالادستی هوش مصنوعی

### ابزارها و یکپارچگی امنیتی مایکروسافت

**GitHub Advanced Security** حفاظت جامعی از زنجیره تأمین فراهم می‌کند شامل:  
- **اسکن اسرار**: تشخیص خودکار مدارک، کلیدهای API و توکن‌ها در مخازن  
- **اسکن وابستگی‌ها**: ارزیابی آسیب‌پذیری وابستگی‌ها و کتابخانه‌های متن‌باز  
- **تحلیل CodeQL**: تحلیل ایستا کد برای آسیب‌پذیری‌های امنیتی و مسائل برنامه‌نویسی  
- **بینش‌های زنجیره تأمین**: دید کلی به وضعیت سلامت و امنیت وابستگی‌ها

**یکپارچگی Azure DevOps و Azure Repos:**  
- ادغام بی‌درز اسکن امنیتی در تمام پلتفرم‌های توسعه مایکروسافت  
- چک‌های امنیتی خودکار در Azure Pipelines برای بارهای کاری AI  
- اجرای سیاست برای پیاده‌سازی امن اجزای هوش مصنوعی

**روش‌های داخلی مایکروسافت:**  
مایکروسافت روش‌های گسترده امنیت زنجیره تأمین را در همه محصولات اجرا می‌کند. درباره رویکردهای اثبات‌شده در [سفر به سمت تأمین امنیت زنجیره نرم‌افزار در مایکروسافت](https://devblogs.microsoft.com/engineering-at-microsoft/the-journey-to-secure-the-software-supply-chain-at-microsoft/) بیاموزید.


## بهترین شیوه‌های امنیت پایه

پیاده‌سازی‌های MCP بر اساس موضع امنیتی موجود سازمان شما بنا شده و آن را توسعه می‌دهند. تقویت شیوه‌های پایه امنیت به طور قابل توجهی امنیت کلی سیستم‌های هوش مصنوعی و پیاده‌سازی‌های MCP را بهبود می‌بخشد.

### اصول بنیادین امنیت

#### **شیوه‌های توسعه امن**  
- **رعایت OWASP**: محافظت در برابر آسیب‌پذیری‌های وب شناخته شده [OWASP Top 10](https://owasp.org/www-project-top-ten/)  
- **حفاظت‌های خاص هوش مصنوعی**: پیاده‌سازی کنترل‌ها برای [OWASP Top 10 برای مدل‌های زبانی بزرگ (LLM)](https://genai.owasp.org/download/43299/?tmstv=1731900559)  
- **مدیریت امن اسرار**: استفاده از خزانه‌های اختصاصی برای توکن‌ها، کلیدهای API و داده‌های پیکربندی حساس  
- **رمزنگاری انتها به انتها**: پیاده‌سازی ارتباطات امن در سراسر اجزای برنامه و جریان‌های داده  
- **اعتبارسنجی ورودی‌ها**: اعتبارسنجی سختگیرانه تمامی ورودی‌های کاربر، پارامترهای API و منابع داده

#### **سخت‌سازی زیرساخت**  
- **احراز هویت چندمرحله‌ای (MFA)**: MFA اجباری برای کلیه حساب‌های مدیریتی و سرویس  
- **مدیریت وصله‌ها**: وصله‌گذاری خودکار و به موقع برای سیستم‌عامل‌ها، چارچوب‌ها و وابستگی‌ها  
- **یکپارچگی ارائه‌دهنده هویت**: مدیریت هویت متمرکز از طریق ارائه‌دهندگان هویت سازمانی (مایکروسافت انترا ID، Active Directory)  
- **بخش‌بندی شبکه**: جداسازی منطقی اجزای MCP برای محدود کردن حرکت جانبی  
- **اصل حداقل امتیاز**: حداقل مجوزهای مورد نیاز برای تمام اجزا و حساب‌ها

#### **نظارت و تشخیص امنیتی**  
- **ثبت جامع رویدادها**: ثبت دقیق فعالیت‌های برنامه هوش مصنوعی شامل تعاملات کلاینت-سرور MCP  
- **یکپارچگی SIEM**: مدیریت متمرکز اطلاعات امنیتی و رخدادها برای تشخیص ناهنجاری  
- **تحلیل رفتاری**: نظارت مبتنی بر هوش مصنوعی برای شناسایی الگوهای غیرعادی در رفتار سیستم و کاربران  
- **هوش تهدید**: ادغام خوراک‌های تهدید و شاخص‌های نفوذ (IOC)  
- **واکنش به حادثه**: فرایندهای مشخص برای تشخیص، پاسخ و بازیابی از رخدادهای امنیتی

#### **معماری اعتماد صفر**  
- **هرگز اعتماد نکن، همواره تأیید کن**: تأیید مداوم کاربران، دستگاه‌ها و ارتباطات شبکه  
- **ریز‌بندی شبکه**: کنترل‌های شبکه‌ای دقیق که بارهای کاری و سرویس‌های فردی را جدا می‌کند  
- **امنیت مبتنی بر هویت**: سیاست‌های امنیتی بر اساس هویت‌های تأیید شده به جای موقعیت شبکه  
- **ارزیابی ریسک پیوسته**: ارزیابی وضعیت امنیتی پویا بر اساس زمینه و رفتار کنونی  
- **دسترسی مشروط**: کنترل‌های دسترسی که بر اساس فاکتورهای ریسک، مکان و اعتماد دستگاه تطبیق می‌یابند

### الگوهای یکپارچگی سازمانی

#### **یکپارچگی اکوسیستم امنیتی مایکروسافت**  
- **Microsoft Defender for Cloud**: مدیریت جامع وضعیت امنیت فضای ابری  
- **Azure Sentinel**: قابلیت‌های بومی ابری SIEM و SOAR برای حفاظت از بارهای کاری هوش مصنوعی  
- **Microsoft Entra ID**: مدیریت هویت و دسترسی سازمانی با سیاست‌های دسترسی مشروط  
- **Azure Key Vault**: مدیریت متمرکز اسرار با پشتیبانی سخت‌افزاری HSM  
- **Microsoft Purview**: حاکمیت داده و انطباق برای منابع داده و جریان‌های کاری هوش مصنوعی

#### **انطباق و حکمرانی**  
- **تطابق با مقررات**: اطمینان از رعایت الزامات انطباق صنعت مرتبط (GDPR، HIPAA، SOC 2)  
- **دسته‌بندی داده‌ها**: طبقه‌بندی مناسب و مدیریت داده‌های حساس پردازش شده توسط سیستم‌های AI  
- **ردپای حسابرسی**: ثبت جامع برای رعایت مقررات و تحقیقات قانونی  
- **کنترل‌های حفظ حریم خصوصی**: پیاده‌سازی اصول طراحی حریم خصوصی در معماری سیستم AI  
- **مدیریت تغییر**: فرایندهای رسمی برای مرور امنیتی تغییرات سیستم AI

این شیوه‌های بنیادین پایه امنیتی محکمی ایجاد می‌کنند که اثربخشی کنترل‌های امنیتی مخصوص MCP را افزایش داده و حفاظت جامعی برای برنامه‌های کاربردی مبتنی بر هوش مصنوعی فراهم می‌آورد.

## نکات کلیدی امنیتی  

- **رویکرد امنیتی چندلایه**: ترکیب شیوه‌های پایه‌ای امنیتی (کدنویسی امن، حداقل امتیاز، تأیید زنجیره تأمین، پایش مستمر) با کنترل‌های خاص هوش مصنوعی برای حفاظت جامع

- **چشم‌انداز تهدیدات خاص هوش مصنوعی**: سیستم‌های MCP با ریسک‌های منحصر به فردی مانند تزریق دایرکتوری فرمان، مسمومیت ابزار، ربایش جلسه، مشکلات نمایندگی گیج، آسیب‌پذیری‌های عبور توکن و امتیازات بیش از حد مواجه‌اند که به کاهش‌های تخصصی نیاز دارند

- **برتری احراز هویت و مجوزدهی**: پیاده‌سازی احراز هویت قدرتمند با استفاده از ارائه‌دهندگان هویت خارجی (Microsoft Entra ID)، اجرای اعتبارسنجی صحیح توکن و هرگز قبول نکردن توکن‌هایی که صراحتاً برای سرور MCP شما صادر نشده‌اند

- **پیشگیری از حملات هوش مصنوعی**: استقرار Microsoft Prompt Shields و Azure Content Safety برای دفاع در برابر حملات تزریق دایرکتوری فرمان غیرمستقیم و مسمومیت ابزار، در حالی که متاداده ابزار را اعتبارسنجی و تغییرات پویا را پایش می‌کنید

- **امنیت جلسات و انتقال داده‌ها**: استفاده از شناسه‌های جلسه غیرقطعی و رمزنگاری شده که به هویت کاربران متصل است، پیاده‌سازی مدیریت چرخه عمر مناسب جلسه و هرگز استفاده از جلسات برای احراز هویت

- **بهترین شیوه‌های امنیت OAuth**: جلوگیری از حملات نمایندگی گیج با رضایت صریح کاربر برای کلاینت‌های ثبت شده پویا، اجرای صحیح OAuth 2.1 با PKCE و اعتبارسنجی سخت‌گیرانه آدرس‌های بازگشت (redirect URI)

- **اصول امنیت توکن**: اجتناب از الگوهای ضد عبور توکن، اعتبارسنجی ادعای مخاطب توکن، پیاده‌سازی توکن‌های کوتاه‌مدت با چرخش امن و حفظ مرزبندی‌های اعتماد شفاف

- **امنیت جامع زنجیره تأمین**: رفتار با همه اجزای اکوسیستم هوش مصنوعی (مدل‌ها، جاسازی‌ها، تأمین‌کننده‌های زمینه، APIهای خارجی) با همان سختگیری امنیتی که برای وابستگی‌های نرم‌افزاری سنتی وجود دارد

- **تکامل مستمر**: به‌روز نگه داشتن خود با مشخصات سریعاً در حال تکامل MCP، مشارکت در استانداردهای جامعه امنیتی، و حفظ نگرش امنیتی تطبیقی با بلوغ پروتکل

- **یکپارچگی امنیتی مایکروسافت**: بهره‌گیری از اکوسیستم امنیتی جامع مایکروسافت (Prompt Shields، Azure Content Safety، GitHub Advanced Security، Entra ID) برای حفاظت پیشرفته هنگام استقرار MCP
