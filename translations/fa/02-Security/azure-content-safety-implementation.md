# پیاده‌سازی ایمنی محتوای Azure با MCP

> **ریسک MCP مورد بررسی OWASP**: [MCP06 - تغییر نیت در جریان](https://microsoft.github.io/mcp-azure-security-guide/mcp/mcp06-prompt-injection/)

برای تقویت امنیت MCP در برابر تزریق دستور، مسمومیت ابزار و سایر آسیب‌پذیری‌های خاص هوش مصنوعی، توصیه می‌شود که ایمنی محتوای Azure را ادغام کنید. این راهنمای پیاده‌سازی با [کارگاه اجلاس امنیتی MCP (Sherpa)](https://azure-samples.github.io/sherpa/) کمپ ۳: امنیت ورودی/خروجی همراستا است.

## ادغام با سرور MCP

برای ادغام ایمنی محتوای Azure با سرور MCP خود، فیلتر ایمنی محتوا را به‌عنوان نرم‌افزاری میانجی در خط لوله پردازش درخواست‌های خود اضافه کنید:

1. فیلتر را هنگام راه‌اندازی سرور مقداردهی اولیه کنید  
2. تمام درخواست‌های ورودی ابزار را پیش از پردازش اعتبارسنجی کنید  
3. تمامی پاسخ‌های خروجی را پیش از بازگرداندن به کلاینت‌ها بررسی کنید  
4. نقض‌های ایمنی را ثبت و هشدار دهید  
5. کنترل خطای مناسب برای شکست بررسی‌های ایمنی محتوا پیاده‌سازی کنید

این یک دفاع قوی در برابر موارد زیر فراهم می‌کند:  
- حملات تزریق دستور  
- تلاش‌های مسمومیت ابزار  
- استخراج داده‌ها از طریق ورودی‌های مخرب  
- تولید محتوای مضر

## بهترین روش‌ها برای ادغام Azure Content Safety

1. **فهرست‌های مسدودسازی سفارشی**: فهرست‌های مسدودسازی سفارشی را مخصوص الگوهای تزریق MCP ایجاد کنید  
2. **تنظیم شدت**: آستانه‌های شدت را بر اساس مورد استفاده و تحمل ریسک خاص خود تنظیم کنید  
3. **پوشش جامع**: بررسی‌های ایمنی محتوا را روی تمام ورودی‌ها و خروجی‌ها اعمال کنید  
4. **بهینه‌سازی عملکرد**: پیاده‌سازی کش برای بررسی‌های تکراری ایمنی محتوا را مد نظر قرار دهید  
5. **مکانیسم‌های جایگزین**: رفتارهای جایگزین واضح در صورت عدم دسترسی به سرویس‌های ایمنی محتوا تعریف کنید  
6. **بازخورد به کاربران**: هنگام مسدود شدن محتوا به‌دلیل نگرانی‌های ایمنی، به کاربران بازخورد واضح ارائه دهید  
7. **بهبود مستمر**: فهرست‌ها و الگوهای مسدودسازی را بر اساس تهدیدهای جدید به‌طور منظم به‌روزرسانی کنید
