# مقدمه‌ای بر یکپارچه‌سازی پایگاه داده MCP

## 🎯 موضوعات این آزمایشگاه

این آزمایشگاه مقدمه‌ای، مروری جامع بر ساخت سرورهای پروتکل مدل کانتکست (MCP) با یکپارچه‌سازی پایگاه داده ارائه می‌دهد. شما مورد کسب‌وکار، معماری فنی و کاربردهای دنیای واقعی را از طریق استفاده موردی تحلیل خرده‌فروشی زوا در https://github.com/microsoft/MCP-Server-and-PostgreSQL-Sample-Retail درک خواهید کرد.

## مرور کلی

**پروتکل مدل کانتکست (MCP)** به دستیارهای هوش مصنوعی امکان می‌دهد تا به طور امن به منابع داده خارجی در زمان واقعی دسترسی پیدا کرده و با آن‌ها تعامل داشته باشند. هنگامی که این پروتکل با یکپارچه‌سازی پایگاه داده ترکیب می‌شود، قابلیت‌های قدرتمندی برای برنامه‌های هوش مصنوعی مبتنی بر داده فراهم می‌کند.

این مسیر آموزش به شما نحوه ساخت سرورهای MCP آماده تولید را آموزش می‌دهد که دستیارهای هوش مصنوعی را از طریق PostgreSQL به داده‌های فروش خرده‌فروشی متصل می‌کند و الگوهای شرکتی مانند امنیت سطح ردیف، جستجوی معنایی و دسترسی چندفروشگاهی به داده را پیاده‌سازی می‌کند.

## اهداف یادگیری

تا پایان این آزمایشگاه، قادر خواهید بود:

- **تعریف** پروتکل مدل کانتکست و مزایای اصلی آن برای یکپارچه‌سازی پایگاه داده  
- **شناسایی** اجزای کلیدی معماری سرور MCP با پایگاه داده‌ها  
- **درک** مورد استفاده خرده‌فروشی زوا و الزامات کسب‌وکاری آن  
- **تشخیص** الگوهای شرکتی برای دسترسی امن و مقیاس‌پذیر به پایگاه داده  
- **فهرست** ابزارها و فناوری‌های مورد استفاده در سراسر این مسیر یادگیری  

## 🧭 چالش: ادغام هوش مصنوعی با داده‌های دنیای واقعی

### محدودیت‌های هوش مصنوعی سنتی

دستیاران هوش مصنوعی مدرن بسیار قدرتمندند اما هنگام کار با داده‌های کسب‌وکار دنیای واقعی محدودیت‌های قابل توجهی دارند:

| **چالش** | **توضیح** | **تاثیر بر کسب‌وکار** |
|---------------|-----------------|-------------------|
| **دانش ایستا** | مدل‌های هوش مصنوعی آموزش دیده روی مجموعه داده‌های ثابت نمی‌توانند به داده‌های جاری کسب‌وکار دسترسی داشته باشند | بینش‌های منسوخ، فرصت‌های از دست رفته |
| **جزیره‌ای بودن داده‌ها** | اطلاعات در پایگاه داده‌ها، APIها و سیستم‌هایی محبوس شده‌اند که AI نمی‌تواند به آن‌ها دسترسی یابد | تحلیل ناقص، فرآیندهای کاری تکه‌تکه |
| **محدودیت‌های امنیتی** | دسترسی مستقیم به پایگاه داده نگرانی‌های امنیتی و تطابق ایجاد می‌کند | استقرار محدود، آماده‌سازی دستی داده‌ها |
| **پرس‌و‌جوهای پیچیده** | کاربران تجاری برای استخراج بینش داده نیاز به دانش فنی دارند | پذیرش کم، فرآیندهای ناکارآمد |

### راه‌حل MCP

پروتکل مدل کانتکست این چالش‌ها را با ارائه موارد زیر برطرف می‌کند:

- **دسترسی داده در زمان واقعی**: دستیارهای هوش مصنوعی به پایگاه داده‌ها و APIهای زنده پرس‌وجو می‌کنند  
- **یکپارچه‌سازی امن**: دسترسی کنترل‌شده با احراز هویت و مجوزها  
- **رابط زبان طبیعی**: کاربران تجاری سوالات را به زبان ساده می‌پرسند  
- **پروتکل استاندارد شده**: کارایی در پلتفرم‌ها و ابزارهای مختلف AI  

## 🏪 آشنایی با زوا ری‌تیل: مطالعه موردی یادگیری ما https://github.com/microsoft/MCP-Server-and-PostgreSQL-Sample-Retail

در طول این مسیر یادگیری، ما یک سرور MCP برای **زوا ری‌تیل** می‌سازیم، یک زنجیره فروشگاه خرده‌فروشی DIY تخیلی با چندین موقعیت فروشگاهی. این سناریوی واقعی پیاده‌سازی MCP سطح سازمانی را نشان می‌دهد.

### زمینه کسب‌وکار

**زوا ری‌تیل** فعالیت می‌کند:  
- **۸ فروشگاه فیزیکی** در ایالت واشنگتن (سیاتل، بلوویو، تاکوما، اسپوکن، اورِت، رِدموند، کرکلند)  
- **۱ فروشگاه آنلاین** برای فروش الکترونیکی  
- **کاتالوگ متنوع محصول** شامل ابزار، سخت‌افزار، ملزومات باغبانی و مصالح ساختمانی  
- **مدیریت چندسطحی** با مدیران فروشگاه، مدیران منطقه‌ای و مدیران ارشد  

### الزامات کسب‌وکار

مدیران فروشگاه و مدیران ارشد نیازمند تحلیل‌های هوش مصنوعی برای:

1. **تحلیل عملکرد فروش** در فروشگاه‌ها و دوره‌های زمانی  
2. **پیگیری سطح موجودی** و شناسایی نیازهای بازتأمین  
3. **درک رفتار مشتری** و الگوهای خرید  
4. **کشف بینش محصولات** از طریق جستجوی معنایی  
5. **تولید گزارش‌ها** با پرس‌وجوهای زبان طبیعی  
6. **حفظ امنیت داده** با کنترل دسترسی مبتنی بر نقش  

### الزامات فنی

سرور MCP باید فراهم آورد:

- **دسترسی چندمستاجری داده‌ها** که مدیران فروشگاه فقط داده‌های فروشگاه خود را ببینند  
- **پرس‌وجوی منعطف** با پشتیبانی از عملیات پیچیده SQL  
- **جستجوی معنایی** برای کشف و توصیه محصولات  
- **داده‌های زمان واقعی** که وضعیت کسب‌وکار جاری را منعکس کند  
- **احراز هویت امن** با امنیت سطح ردیف  
- **معماری مقیاس‌پذیر** که از کاربران هم‌زمان متعدد پشتیبانی کند

### اجزای کلیدی

#### **۱. لایه سرور MCP**
- **فریم‌ورک FastMCP**: پیاده‌سازی مدرن سرور MCP با پایتون  
- **ثبت ابزارها**: تعاریف اعلامی ابزار با ایمنی نوع  
- **کانتکست درخواست**: مدیریت هویت کاربر و نشست  
- **مدیریت خطا**: مدیریت خطا و لاگ‌گیری مطمئن  

#### **۲. لایه یکپارچه‌سازی پایگاه داده**
- **مدیریت اتصال**: مدیریت کارآمد اتصال asyncpg  
- **ارائه‌دهنده طرح‌واره**: کشف داینامیک طرح‌واره جدول‌ها  
- **اجرای پرس‌وجو**: اجرای امن SQL با کانتکست RLS  
- **مدیریت تراکنش**: تطابق ACID و مدیریت برگشت تغییرات  

#### **۳. لایه امنیت**
- **امنیت سطح ردیف**: PostgreSQL RLS برای جداسازی داده چندمستاجری  
- **هویت کاربر**: احراز هویت و مجوز مدیر فروشگاه  
- **کنترل دسترسی**: مجوزهای دقیق و ردگیری حسابرسی  
- **اعتبارسنجی ورودی**: جلوگیری از تزریق SQL و اعتبارسنجی پرس‌وجو  

#### **۴. لایه افزایش هوش مصنوعی**
- **جستجوی معنایی**: جاسازی‌های برداری برای کشف محصول  
- **یکپارچه‌سازی Azure OpenAI**: تولید جاسازی متنی  
- **الگوریتم‌های مشابهت**: جستجوی شباهت کسینوسی pgvector  
- **بهینه‌سازی جستجو**: فهرست‌بندی و بهینه‌سازی عملکرد

## 🔒 امنیت و چندمستاجری عمیق‌تر

پیاده‌سازی ما اولویت را به امنیت سازمانی می‌دهد:

### امنیت سطح ردیف (RLS)

PostgreSQL RLS جداسازی داده‌ها را تضمین می‌کند:

```sql
-- Store managers see only their store's data
CREATE POLICY store_manager_policy ON retail.orders
  FOR ALL TO store_managers
  USING (store_id = get_current_user_store());

-- Regional managers see multiple stores
CREATE POLICY regional_manager_policy ON retail.orders
  FOR ALL TO regional_managers
  USING (store_id = ANY(get_user_store_list()));
```

### مدیریت هویت کاربر

هر اتصال MCP شامل:  
- **شناسه مدیر فروشگاه**: شناسه یکتا برای کانتکست RLS  
- **اختصاص نقش**: مجوزها و سطوح دسترسی  
- **مدیریت نشست**: توکن‌های احراز هویت امن  
- **ثبت حسابرسی**: تاریخچه کامل دسترسی  

### حفاظت از داده‌ها

چند لایه امنیتی:  
- **رمزگذاری اتصال**: TLS برای همه اتصالات پایگاه داده  
- **جلوگیری از تزریق SQL**: فقط پرس‌وجوهای پارامتری  
- **اعتبارسنجی ورودی**: اعتبارسنجی کامل درخواست‌ها  
- **مدیریت خطا**: بدون نمایش داده‌های حساس در پیام‌های خطا
