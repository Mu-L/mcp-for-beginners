## 🔧 درک سرورهای MCP: راهنمای مبتدیان

### سرورهای MCP چیستند؟

به‌عنوان فردی تازه‌کار با پروتکل مدل کانتکست (MCP)، ممکن است بپرسید: «سرور MCP دقیقاً چیست و چرا باید اهمیت بدهم؟» با یک تشبیه ساده شروع کنیم.

به سرورهای MCP مثل دستیاران تخصصی نگاه کنید که به همراه برنامه‌نویسی هوش مصنوعی شما (مثل GitHub Copilot) کمک می‌کنند به ابزارها و خدمات خارجی متصل شود. درست مثل اینکه برای کارهای مختلف روی گوشی‌تان از برنامه‌های مختلف استفاده می‌کنید—یکی برای هواشناسی، یکی برای مسیریابی، یکی برای بانکداری—سرورهای MCP به دستیار هوش مصنوعی شما این امکان را می‌دهند که با ابزارها و خدمات متنوع توسعه تعامل داشته باشد.

### مشکلی که سرورهای MCP حل می‌کنند

قبل از سرورهای MCP، اگر می‌خواستید:
- منابع Azure خود را بررسی کنید
- مسئله‌ای در GitHub ایجاد کنید
- پایگاه داده خود را پرس‌وجو کنید
- در مستندات جستجو کنید

باید از کد نویسی دست می‌کشیدید، مرورگر باز می‌کردید، به وب‌سایت مربوطه می‌رفتید و این کارها را به‌صورت دستی انجام می‌دادید. این تغییر مکرر زمینه کار، جریان کاری شما را مختل می‌کند و بهره‌وری‌تان را کاهش می‌دهد.

### چگونه سرورهای MCP تجربه توسعه شما را متحول می‌کنند

با سرورهای MCP، می‌توانید در محیط توسعه خود (VS Code، Visual Studio و غیره) بمانید و به دستیار هوش مصنوعی خود بسپارید این کارها را انجام دهد. مثلاً:

**به‌جای جریان کاری سنتی زیر:**
1. دست از کدنویسی بکشید
2. مرورگر را باز کنید
3. به پورتال Azure بروید
4. جزئیات حساب ذخیره‌سازی را بررسی کنید
5. به VS Code برگردید
6. کدنویسی را ادامه دهید

**اکنون می‌توانید این کارها را انجام دهید:**
1. از هوش مصنوعی بپرسید: «وضعیت حساب‌های ذخیره‌سازی Azure من چگونه است؟»
2. با اطلاعات ارائه‌شده به کدنویسی ادامه دهید

### مزایای کلیدی برای مبتدیان

#### 1. 🔄 **در حالت جریان کاری خود بمانید**
- دیگر نیازی به جابه‌جایی بین برنامه‌های مختلف نیست
- تمرکز خود را روی کدی که می‌نویسید حفظ کنید
- بار ذهنی مدیریت ابزارهای مختلف را کاهش دهید

#### 2. 🤖 **از زبان طبیعی به جای دستورات پیچیده استفاده کنید**
- به جای حفظ دستورهای SQL، توضیح دهید چه داده‌ای نیاز دارید
- به جای بخاطر سپردن فرمان‌های Azure CLI، توضیح دهید می‌خواهید چه کار کنید
- بگذارید هوش مصنوعی جزئیات فنی را مدیریت کند در حالی که شما روی منطق تمرکز می‌کنید

#### ۳. 🔗 **ابزارهای مختلف را به هم متصل کنید**
- با ترکیب خدمات مختلف جریان‌های کاری قدرتمند بسازید
- مثال: «تمام مسائل اخیر GitHub را بگیر و برای آن‌ها موارد کاری Azure DevOps بساز»
- خودکارسازی را بدون نوشتن اسکریپت‌های پیچیده بسازید

#### ۴. 🌐 **به یک اکوسیستم رو به رشد دسترسی داشته باشید**
- از سرورهای ساخته شده توسط مایکروسافت، گیت‌هاب و شرکت‌های دیگر بهره ببرید
- ابزارهای متفاوت از فروشندگان مختلف را بدون مشکل ترکیب کنید
- به یک اکوسیستم استاندارد شده بپیوندید که در دستیارهای مختلف هوش مصنوعی کار می‌کند

#### ۵. 🛠️ **با عمل یاد بگیرید**
- با سرورهای آماده شروع کنید تا مفاهیم را درک کنید
- به تدریج سرورهای خود را بسازید زمانی که راحت‌تر شدید
- از SDKها و مستندات موجود برای راهنمایی یادگیری استفاده کنید

### مثال واقعی برای مبتدیان

فرض کنید تازه وارد توسعه وب هستید و روی اولین پروژه خود کار می‌کنید. اینجا چطور سرورهای MCP می‌توانند به شما کمک کنند:

**روش سنتی:**
```
1. Code a feature
2. Open browser → Navigate to GitHub
3. Create an issue for testing
4. Open another tab → Check Azure docs for deployment
5. Open third tab → Look up database connection examples
6. Return to VS Code
7. Try to remember what you were doing
```

**با سرورهای MCP:**
```
1. Code a feature
2. Ask AI: "Create a GitHub issue for testing this login feature"
3. Ask AI: "How do I deploy this to Azure according to the docs?"
4. Ask AI: "Show me the best way to connect to my database"
5. Continue coding with all the information you need
```


### مزیت استاندارد سازمانی

MCP در حال تبدیل شدن به یک استاندارد صنعت است، به این معنا که:
- **یکدستی**: تجربه مشابه در ابزارها و شرکت‌های مختلف
- **تعامل‌پذیری**: سرورهای فروشندگان مختلف با هم کار می‌کنند
- **آینده‌پذیری**: مهارت‌ها و تنظیمات بین دستیارهای مختلف هوش مصنوعی قابل انتقال است
- **جامعه**: اکوسیستم بزرگ دانش و منابع مشترک

## شروع کار با MCP در VS Code و Visual Studio 🚀

راه‌اندازی این سرورهای MCP ساده است اگر از Visual Studio Code یا Visual Studio ۲۰۲۲ با GitHub Copilot استفاده می‌کنید.

### تنظیم VS Code

فرآیند پایه برای VS Code به شرح زیر است:

1. **فعال کردن حالت Agent**: در VS Code در پنجره گفتگو Copilot به حالت Agent سوئیچ کنید
2. **پیکربندی سرورها**: پیکربندی سرورها را در فایل settings.json در VS Code اضافه کنید
3. **شروع به کار سرورها**: روی دکمه «Start» برای هر سروری که می‌خواهید استفاده کنید کلیک کنید
4. **انتخاب ابزارها**: انتخاب کنید کدام سرورهای MCP را برای جلسه فعلی فعال کنید

برای دستورالعمل‌های دقیق تنظیم، به [مستندات MCP در VS Code](https://code.visualstudio.com/docs/copilot/copilot-mcp) مراجعه کنید.

> **💡 نکته حرفه‌ای: سرورهای MCP را مثل حرفه‌ای‌ها مدیریت کنید!**
> 
> نمای افزونه‌های VS Code اکنون [رابط کاربری جدید و کارآمدی برای مدیریت سرورهای MCP نصب‌شده](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode) دارد! شما دسترسی سریع برای شروع، توقف و مدیریت هر سرور MCP نصب‌شده را با رابط کاربری ساده و واضح در اختیار دارید. امتحان کنید!

### تنظیم Visual Studio ۲۰۲۲

برای Visual Studio ۲۰۲۲ (نسخه ۱۷.۱۴ به بعد):

1. **فعال کردن حالت Agent**: در پنجره گفتگو GitHub Copilot، روی منوی کشویی «Ask» کلیک کرده و «Agent» را انتخاب کنید
2. **ایجاد فایل پیکربندی**: یک فایل `.mcp.json` در دایرکتوری پروژه خود بسازید (محل پیشنهادی: `<SOLUTIONDIR>\.mcp.json`)
3. **پیکربندی سرورها**: پیکربندی سرورهای MCP را با فرمت استاندارد MCP اضافه کنید
4. **تأیید ابزارها**: هنگام درخواست، ابزارهایی که می‌خواهید استفاده کنید را با مجوزهای مناسب تأیید کنید

برای دستورالعمل‌های دقیق تنظیم Visual Studio به [مستندات MCP در Visual Studio](https://learn.microsoft.com/visualstudio/ide/mcp-servers) مراجعه کنید.

هر سرور MCP نیازمندی‌های پیکربندی خاص خودش را دارد (رشته‌های اتصال، احراز هویت و غیره)، اما الگوهای راه‌اندازی در هر دو IDE مشابه است.

## درس‌هایی از سرورهای Microsoft MCP 🛠️

### ۱. 📚 سرور Microsoft Learn Docs MCP

[![نصب در VS Code](https://img.shields.io/badge/VS_Code-Install_Microsoft_Docs_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D) [![نصب در VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Microsoft_Docs_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/mcp)

**کارکرد آن چیست**: سرور Microsoft Learn Docs MCP یک سرویس ابری است که به دستیارهای هوش مصنوعی دسترسی به مستندات رسمی مایکروسافت را در زمان واقعی از طریق پروتکل مدل کانتکست فراهم می‌کند. این سرور به آدرس `https://learn.microsoft.com/api/mcp` متصل می‌شود و جستجوی معنایی در بین Microsoft Learn، مستندات Azure، مستندات Microsoft 365 و سایر منابع رسمی مایکروسافت را ممکن می‌سازد.

**چرا مفید است**: گرچه ممکن است به نظر برسد این فقط یک «مستندات» است، این سرور در واقع برای هر توسعه‌دهنده‌ای که از فناوری‌های مایکروسافت استفاده می‌کند حیاتی است. یکی از بزرگ‌ترین شکایات توسعه‌دهندگان .NET از دستیارهای کدنویسی هوش مصنوعی این است که آنها از آخرین نسخه‌های .NET و C# به‌روز نیستند. سرور Microsoft Learn Docs MCP این مشکل را با ارائه دسترسی زمان واقعی به مستندات، مراجع API و بهترین شیوه‌های به‌روز حل می‌کند. چه با جدیدترین SDKهای Azure کار می‌کنید، چه در حال بررسی ویژگی‌های جدید C# 13 باشید یا در حال پیاده‌سازی الگوهای پیشرفته .NET Aspire، این سرور تضمین می‌کند دستیار هوش مصنوعی شما به اطلاعات معتبر، به‌روز و دقیق دسترسی داشته باشد تا کد صحیح و مدرن تولید کند.

**استفاده واقعی**: «فرمان‌های az cli برای ایجاد یک Azure container app طبق مستندات رسمی Microsoft Learn چیست؟» یا «چگونه Ef Core را با تزریق وابستگی در ASP.NET Core پیکربندی کنم؟» یا «این کد را بازبینی کن تا مطمئن شوی با توصیه‌های عملکرد در مستندات Microsoft Learn مطابقت دارد.» این سرور پوشش کاملی در Microsoft Learn، مستندات Azure و مستندات Microsoft 365 با استفاده از جستجوی معنایی پیشرفته دارد تا مرتبط‌ترین اطلاعات را بیابد. تا ۱۰ قطعه محتوای با کیفیت بالا با عنوان مقاله و URL برمی‌گرداند، همیشه به جدیدترین مستندات مایکروسافت که منتشر می‌شود دسترسی دارد.

**مثال برجسته**: این سرور ابزار `microsoft_docs_search` را ارائه می‌دهد که جستجوی معنایی در مستندات فنی رسمی مایکروسافت انجام می‌دهد. پس از پیکربندی، می‌توانید سوالاتی مثل «چگونه JWT Authentication را در ASP.NET Core پیاده‌سازی کنم؟» بپرسید و پاسخ‌های دقیق و رسمی با لینک‌های منبع دریافت کنید. کیفیت جستجو استثنایی است چون زمینه را می‌فهمد – پرسیدن درباره «کانتینرها» در زمینه Azure مستندات Azure Container Instances را برمی‌گرداند، در حالی که همان عبارت در زمینه .NET اطلاعات مرتبط C# collection را نشان می‌دهد.

این به‌ویژه برای کتابخانه‌ها و موارد استفاده‌ای که به سرعت تغییر می‌کنند یا به تازگی به‌روزرسانی شده‌اند، مفید است. مثلاً در برخی پروژه‌های اخیر می‌خواستم از قابلیت‌های نسخه‌های جدید .NET Aspire و Microsoft.Extensions.AI بهره ببرم. با افزودن سرور Microsoft Learn Docs MCP، توانستم نه فقط مستندات API بلکه راهنمایی‌ها و آموزش‌های تازه منتشر شده را نیز استفاده کنم.

### ۲. ☁️ سرور Azure MCP


[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

**چی انجام می‌دهد**: سرور Azure MCP مجموعه‌ای جامع از بیش از ۱۵ کانکتور تخصصی سرویس‌های Azure است که کل اکوسیستم Azure را به جریان کاری هوش مصنوعی شما می‌آورد. این تنها یک سرور نیست – بلکه مجموعه قدرتمندی است که شامل مدیریت منابع، اتصال به پایگاه داده (PostgreSQL، SQL Server)، تحلیل لاگ‌ها با KQL در Azure Monitor، ترکیب Cosmos DB و خیلی موارد دیگر می‌شود.

**چرا مفید است**: علاوه بر مدیریت منابع Azure، این سرور کیفیت کد را هنگام کار با SDKهای Azure به طرز چشمگیری بهبود می‌بخشد. وقتی Azure MCP را در حالت Agent استفاده می‌کنید، فقط به نوشتن کد کمک نمی‌کند – بلکه کمک می‌کند کد *بهتری* بنویسید که الگوهای تایید هویت کنونی، بهترین روش‌های مدیریت خطا و ویژگی‌های جدید SDK را رعایت می‌کند. به جای دریافت کد عمومی که ممکن است کار کند، کدی دریافت می‌کنید که الگوهای توصیه شده Azure برای بارهای کاری تولید را دنبال می‌کند.

**ماژول‌های کلیدی شامل**:
- **🗄️ کانکتورهای پایگاه داده**: دسترسی مستقیم به زبان طبیعی به Azure Database برای PostgreSQL و SQL Server
- **📊 Azure Monitor**: تحلیل لاگ به کمک KQL و دیدهای عملیاتی
- **🌐 مدیریت منابع**: مدیریت کامل چرخه عمر منابع Azure
- **🔐 تایید هویت**: الگوهای DefaultAzureCredential و managed identity
- **📦 سرویس‌های ذخیره‌سازی**: عملیات Blob Storage، Queue Storage و Table Storage
- **🚀 سرویس‌های کانتینر**: مدیریت Azure Container Apps، Container Instances و AKS
- **و بسیاری کانکتورهای تخصصی دیگر**

**استفاده واقعی**: «حساب‌های ذخیره‌سازی Azure من را فهرست کن»، «کاربرد فضای Log Analytics من برای یافتن خطاهای یک ساعت گذشته را پرس و جو کن» یا «کمک کن یک برنامه Azure با Node.js و تایید هویت مناسب بسازم»

**سناریوی دمو کامل**: اینجا یک مرور گام به گام کامل است که قدرت ترکیب Azure MCP با افزونه GitHub Copilot برای Azure در VS Code را نشان می‌دهد. وقتی هر دو را نصب دارید و دستور می‌دهید:

> «یک اسکریپت پایتون ایجاد کن که یک فایل را با استفاده از تایید هویت DefaultAzureCredential به Azure Blob Storage آپلود کند. اسکریپت باید به حساب ذخیره‌سازی Azure من به نام 'mycompanystorage' متصل شود، به کانتینری به نام 'documents' آپلود کند، یک فایل تست با زمان فعلی ایجاد کند و آن را آپلود کند، خطاها را به خوبی مدیریت کند و خروجی آموزنده بدهد، بهترین شیوه‌های تایید هویت و مدیریت خطاهای Azure را رعایت کند، شامل توضیحاتی درباره نحوه عملکرد تایید هویت DefaultAzureCredential باشد، و ساختار خوبی با توابع و مستندسازی مناسب داشته باشد.»

سرور Azure MCP یک اسکریپت پایتون کامل و آماده تولید ایجاد می‌کند که:
- جدیدترین SDK ذخیره‌سازی Azure Blob را با الگوهای صحیح async استفاده می‌کند
- DefaultAzureCredential را با توضیح کامل زنجیره fallback پیاده‌سازی می‌کند
- شامل مدیریت خطای قوی با نوع‌های خاص استثناهای Azure است
- بهترین شیوه‌های SDK Azure را در مدیریت منابع و اتصال رعایت می‌کند
- گزارش‌گیری دقیق و خروجی آموزنده برای کنسول فراهم می‌آورد
- اسکریپتی ساختارمند با توابع، مستندسازی و نشانه‌گذاری نوع‌ها ایجاد می‌کند

چیزی که این را قابل توجه می‌کند این است که بدون Azure MCP، ممکن است کد عمومی blob storage بگیرید که کار کند اما الگوهای جاری Azure را رعایت نکند. با Azure MCP، کدی می‌گیرید که از جدیدترین روش‌های تایید هویت استفاده می‌کند، سناریوهای خطای خاص Azure را مدیریت می‌کند و بهترین شیوه‌های مایکروسافت برای برنامه‌های تولید را دنبال می‌کند.

**مثال برجسته**: من همیشه برای یادآوری دستورات مخصوص CLIهای `az` و `azd` برای استفاده فوری مشکل داشتم. همیشه یک فرآیند دو مرحله‌ای برای من است: اول سینتکس را جست‌وجو کنم، سپس فرمان را اجرا کنم. اغلب به پرتال می‌روم و کلیک می‌کنم تا کارم را انجام دهم چون نمی‌خواهم اعتراف کنم سینتکس CLI را یادم نمی‌آید. توانایی فقط توصیف چیزی که می‌خواهم عالی است، و حتی بهتر است که بتوانم این کار را بدون خارج شدن از IDE انجام دهم!

لیست بسیار خوبی از موارد استفاده در [مخزن Azure MCP](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) برای شروع وجود دارد. برای راهنمای نصب کامل و گزینه‌های پیکربندی پیشرفته، به [مستندات رسمی Azure MCP](https://learn.microsoft.com/azure/developer/azure-mcp-server/) مراجعه کنید.

### 3. 🐙 سرور GitHub MCP

[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Server-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=github&config=%7B%22type%22%3A%20%22http%22%2C%22url%22%3A%20%22https%3A%2F%2Fapi.githubcopilot.com%2Fmcp%2F%22%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Server-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=github&config=%7B%22type%22%3A%20%22http%22%2C%22url%22%3A%20%22https%3A%2F%2Fapi.githubcopilot.com%2Fmcp%2F%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/github/github-mcp-server)

**چی انجام می‌دهد**: سرور رسمی GitHub MCP یکپارچگی کامل با کل اکوسیستم GitHub را فراهم می‌کند، با گزینه‌های دسترسی از راه دور میزبانی شده و همچنین استقرار محلی از طریق Docker. این فقط مربوط به عملیات ساده مخزن نیست – بلکه مجموعه کاملی از ابزارها شامل مدیریت GitHub Actions، جریان‌های کاری pull request، پیگیری مسائل، اسکن امنیتی، اعلان‌ها و قابلیت‌های پیشرفته اتوماسیون است.

**چرا مفید است**: این سرور نحوه تعامل شما با GitHub را تغییر می‌دهد و تجربه کامل پلتفرم را مستقیماً به محیط توسعه می‌آورد. به جای جابجایی مداوم بین VS Code و GitHub.com برای مدیریت پروژه‌ها، بازبینی کد و پایش CI/CD، می‌توانید همه را از طریق دستورات زبان طبیعی و در حین تمرکز روی کد انجام دهید.

> **ℹ️ توجه: انواع مختلف «عامل‌ها»**
> 
> این سرور GitHub MCP را با GitHub Coding Agent (عامل هوش مصنوعی که می‌توانید مسائل را برای انجام خودکار کد به آن اختصاص دهید) اشتباه نگیرید. سرور GitHub MCP در حالت Agent در VS Code کار می‌کند تا یکپارچگی API GitHub را فراهم کند، در حالی که GitHub Coding Agent ویژگی جداگانه‌ای است که هنگام اختصاص به مسائل GitHub، pull request ایجاد می‌کند.

**قابلیت‌های کلیدی شامل**:
- **⚙️ GitHub Actions**: مدیریت کامل خط لوله CI/CD، پایش جریان کاری و مدیریت آثار
- **🔀 Pull Requests**: ایجاد، بازبینی، ادغام و مدیریت PRها با پیگیری وضعیت جامع
- **🐛 Issues**: مدیریت کامل چرخه عمر مسائل، نظر دادن، برچسب‌گذاری و تخصیص
- **🔒 امنیت**: هشدارهای اسکن کد، تشخیص اسرار و یکپارچگی Dependabot
- **🔔 اعلان‌ها**: مدیریت هوشمند اعلان‌ها و کنترل عضویت در مخازن
- **📁 مدیریت مخزن**: عملیات فایل، مدیریت شاخه و مدیریت مخزن
- **👥 همکاری**: جستجوی کاربران و سازمان‌ها، مدیریت تیم و کنترل دسترسی

**استفاده واقعی**: «یک pull request از شاخه feature من ایجاد کن»، «همه اجراهای CI شکست خورده این هفته را نشان بده»، «تمام هشدارهای امنیتی باز برای مخازن من را فهرست کن» یا «همه مسائل تخصیص یافته به من در سراسر سازمان‌هایم را پیدا کن»

**سناریوی دمو کامل**: اینجا یک جریان کاری قدرتمند است که قابلیت‌های GitHub MCP Server را نشان می‌دهد:

> «باید برای بازبینی اسپرینت آماده شوم. همه pull requestهای این هفته را که ایجاد کردم نشان بده، وضعیت خط لوله‌های CI/CD ما را بررسی کن، خلاصه‌ای از هر هشدار امنیتی که باید رسیدگی کنیم بساز، و کمک کن یادداشت‌های انتشار بر اساس PRهای ادغام شده با برچسب "feature" تهیه کنم.»

سرور GitHub MCP:
- درخواست‌های pull request اخیر شما را با اطلاعات دقیق وضعیت جست‌وجو می‌کند
- اجرای جریان کاری را تحلیل کرده و هر گونه خطا یا مشکل عملکردی را برجسته می‌کند
- نتایج اسکن امنیتی را گردآوری کرده و هشدارهای بحرانی را اولویت‌بندی می‌کند
- یادداشت‌های انتشار جامع از اطلاعات استخراج شده از PRهای ادغام شده تولید می‌کند
- گام‌های عملی بعدی برای برنامه‌ریزی اسپرینت و آماده‌سازی انتشار ارائه می‌دهد

**مثال برجسته**: من عاشق استفاده از این برای جریان‌های کاری بازبینی کد هستم. به جای رفت و آمد بین VS Code، اعلان‌های GitHub و صفحات pull request، می‌توانم بگویم «همه PRهایی که منتظر بازبینی من هستند را نشان بده» و سپس «یک نظر به PR شماره ۱۲۳ اضافه کن و درباره مدیریت خطا در روش تایید هویت سؤال بپرس.» سرور تماس‌های API GitHub را انجام می‌دهد، زمینه بحث را حفظ می‌کند و حتی به من کمک می‌کند تا نظرات بازبینی سازنده‌تر بنویسم.

**گزینه‌های تایید هویت**: سرور هر دو روش OAuth (یکپارچه در VS Code) و Personal Access Token را پشتیبانی می‌کند، با مجموعه ابزار قابل پیکربندی که فقط قابلیت‌های مورد نیاز شما از GitHub را فعال می‌کند. می‌توانید آن را به صورت سرویس میزبانی شده راه دور برای راه‌اندازی سریع اجرا کنید یا به صورت محلی با Docker برای کنترل کامل.

> **💡 نکته حرفه‌ای**
> 
> تنها ابزارهای مورد نیاز را با پیکربندی پارامتر `--toolsets` در تنظیمات سرور MCP خود فعال کنید تا اندازه زمینه کاهش یافته و انتخاب ابزار هوش مصنوعی بهبود یابد. برای مثال، `"--toolsets", "repos,issues,pull_requests,actions"` را برای جریان‌های کاری توسعه اصلی به آرگومان‌های پیکربندی MCP خود اضافه کنید یا اگر عمدتاً قابلیت‌های پایش GitHub می‌خواهید، از `"--toolsets", "notifications, security"` استفاده کنید.

**فرمت‌های فایل پشتیبانی شده**:
- **اسناد آفیس**: PDF، پاورپوینت (PPTX)، ورد (DOCX)، اکسل (XLSX/XLS)
- **فایل‌های رسانه‌ای**: تصاویر (با متادیتای EXIF و OCR)، صدا (با متادیتای EXIF و رونویسی گفتار)
- **محتوای وب**: HTML، فیدهای RSS، URLهای یوتیوب، صفحات ویکی‌پدیا
- **فرمت‌های داده‌ای**: CSV، JSON، XML، فایل‌های ZIP (محتویات به صورت بازگشتی پردازش می‌شود)
- **فرمت‌های نشر**: EPub، دفترچه‌های Jupyter (.ipynb)
- **ایمیل**: پیام‌های Outlook (.msg)
- **پیشرفته**: ادغام Azure Document Intelligence برای پردازش پیشرفته PDF

**قابلیت‌های پیشرفته**: MarkItDown از توصیف تصاویر توسط مدل‌های زبان بزرگ (هنگامی که با یک کلاینت OpenAI فراهم شود)، Azure Document Intelligence برای پردازش پیشرفته PDF، رونویسی صوت برای محتوای گفتاری و سیستم افزونه برای گسترش به فرمت‌های فایل اضافی پشتیبانی می‌کند.

**کاربرد دنیای واقعی**: «این ارائه پاورپوینت را به مارک‌داون برای سایت مستندسازی ما تبدیل کن»، «متن این PDF را با ساختار درست سرفصل استخراج کن»، یا «این صفحه گسترده اکسل را به قالب جدول قابل خواندن تبدیل کن»

**مثال برجسته**: نقل قول از [مستندات MarkItDown](https://github.com/microsoft/markitdown#why-markdown):

> مارک‌داون بسیار نزدیک به متن ساده است، با حداقل نشانه‌گذاری یا قالب‌بندی، ولی همچنان راهی برای نمایش ساختار مهم سند فراهم می‌کند. مدل‌های زبان بزرگ اصلی، مانند GPT-4o اوپن‌ای‌آی، به طور بومی «با مارک‌داون حرف می‌زنند» و اغلب بدون فراخوانی این قالب را در پاسخ‌های خود وارد می‌کنند. این نشان می‌دهد که آن‌ها روی مقادیر زیادی متن قالب‌بندی شده با مارک‌داون آموزش دیده‌اند و آن را خوب درک می‌کنند. به عنوان مزیت جانبی، قواعد مارک‌داون همچنین کارایی بالایی از نظر توکن دارند.

MarkItDown در حفظ ساختار سند بسیار خوب عمل می‌کند، که برای جریان‌های کاری هوش مصنوعی اهمیت دارد. برای مثال، هنگام تبدیل یک ارائه پاورپوینت، سازماندهی اسلایدها را با سرفصل‌های مناسب حفظ می‌کند، جداول را به جداول مارک‌داون تبدیل می‌کند، متن جایگزین تصاویر را وارد می‌کند و حتی یادداشت‌های سخنگو را پردازش می‌کند. نمودارها به جداول داده قابل خواندن تبدیل می‌شوند و مارک‌داون حاصل، جریان منطقی ارائه اصلی را حفظ می‌کند. این آن را برای تغذیه محتوای ارائه به سیستم‌های هوش مصنوعی یا ایجاد مستندات از اسلایدهای موجود عالی می‌سازد.

**نمونه برجسته**: به عنوان یک توسعه‌دهنده، در دنبال کردن مدل‌های مختلف LLM مشکل داشتم. چند مدل اصلی را می‌شناسم، اما احساس می‌کردم بهره‌وری و بهینگی را از دست می‌دهم. مدیریت توکن‌ها و سهمیه‌ها استرس‌زا و دشوار بود – نمی‌دانستم آیا مدل مناسب کار را انتخاب می‌کنم یا بی‌مورد بودجه‌ام را هدر می‌دهم. اخیراً از طریق جیمز مونتمگنو درباره این سرور MCP مطلع شدم وقتی داشتم از هم‌تیمی‌ها درباره توصیه‌های MCP Server پرس‌وجو می‌کردم و مشتاقم آن را استفاده کنم! قابلیت‌های کشف مدل برای کسی مثل من که می‌خواهد فراتر از مدل‌های معمولی جستجو کند و مدل‌های بهینه‌شده برای وظایف خاص را پیدا کند، بسیار چشمگیر است. چارچوب ارزیابی هم به من کمک می‌کند تا تأیید کنم که واقعاً نتایج بهتری می‌گیرم، نه فقط برای تنوع کاری امتحان می‌کنم.

## 📋 نتیجه‌گیری

پروتکل زمینه مدل (MCP) نحوه تعامل توسعه‌دهندگان با دستیارهای هوش مصنوعی و ابزارهای خارجی را تغییر می‌دهد. این ۱۰ سرور Microsoft MCP قدرت ادغام استاندارد شده هوش مصنوعی را به نمایش می‌گذارند و امکان جریان‌های کاری یکپارچه را می‌دهند تا توسعه‌دهندگان هنگام دسترسی به قابلیت‌های قدرتمند خارجی در حالت تمرکز باقی بمانند.

از ادغام جامع اکوسیستم Azure تا ابزارهای تخصصی مانند Playwright برای خودکارسازی مرورگر و MarkItDown برای پردازش سند، این سرورها نشان می‌دهند چگونه MCP می‌تواند بهره‌وری را در سناریوهای توسعه متنوع افزایش دهد. پروتکل استانداردشده تضمین می‌کند که این ابزارها به‌طور بی‌نقص با هم کار کنند و تجربه توسعه یکپارچه‌ای ایجاد نمایند.

با ادامه رشد اکوسیستم MCP، مشارکت در جامعه، کاوش سرورهای جدید و ساخت راهکارهای سفارشی کلید حداکثرسازی بهره‌وری توسعه شما خواهد بود. ماهیت استاندارد باز MCP به شما امکان می‌دهد ابزارهای مختلف از فروشندگان متفاوت را ترکیب کرده و جریان کاری ایده‌آل خود را متناسب با نیازهای خاص بسازید.
