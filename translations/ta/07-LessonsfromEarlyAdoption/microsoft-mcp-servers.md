# 🚀 10 மைக்ரோசாஃப்ட் MCP சர்வர்கள் டெவலப்பர் உற்பத்தித்திறனை மாற்றிக் கொண்டிருக்கின்றன

## 🎯 இந்த கையேட்டில் நீங்கள் என்ன கற்கப்போகிறீர்கள்

இந்த செயல்திறன் வழிகாட்டி, AI உதவியாளர்களுடன் பணியாற்றும் முறையில் மாற்றத்தை ஏற்படுத்திக்கொள்ளும் பத்து மைக்ரோசாஃப்ட் MCP சர்வர்களை கண்காணிக்கிறது. MCP சர்வர்கள் *செய்யக்கூடிய* விஷயங்களை விளக்கும் பதிலாக, நாம் மைக்ரோசாஃப့် மற்றும் அதன் அப்பால் தணிக்கையிலும் அன்றாட மேம்பாட்டு பணிகளுக்கு உண்மையாகவே மாற்றம் செய்யும் சர்வர்களை காண்பிப்போம்.

இந்த கையேட்டில் உள்ள ஒவ்வொரு சர்வருமும் உண்மை உலக பயன்பாடு மற்றும் டெவலப்பர் கருத்துக்களின் அடிப்படையில் தேர்ந்தெடுக்கப்பட்டுள்ளது. ஒவ்வொரு சர்வர் செய்யும் பண்பை மட்டுமல்லாது அது எதற்கு முக்கியமானதென்று மற்றும் உங்களுடைய சொந்த திட்டங்களில் அதிலிருந்து அதிகபட்சம் எப்படி பெறுவது என்பதையும் காண்பீர்கள். நீங்கள் MCP யில் முழுமையாக புதியவராக இருக்கிறீர்களோ அல்லது உங்கள் தற்போதைய அமைப்பை விரிவுபடுத்த விரும்புகிறீர்களோ, இந்த சர்வர்கள் மைக்ரோசாஃப்ட் சூழலில் கிடைக்கும் மிகவும் நடைமுறை மற்றும் தாக்கம் மிகுந்த கருவிகளாகும்.

> MCP யில் புதியவரா? தாக்கப்பட்டு விடாதீர்கள்! இந்த கையேடு ஆரம்ப நிலை பயனாளர்கள் கவனத்தில் கொண்டு வடிவமைக்கப்பட்டுள்ளது. நாம் கருத்துக்களை விளக்கப்போகின்றோம், மற்றும் நீங்கள் எப்போதும் [MCP அறிமுகம்](../00-Introduction/README.md) மற்றும் [முக்கிய கருத்துக்கள்](../01-CoreConcepts/README.md) அலகுகளை மேலதிக பின்னணி அறிதலுக்கு பார்க்கலாம்.

## மேலோட்டம்

இந்த விரிவான கையேடு, டெவலப்பர்கள் AI உதவியாளர்களுடன் மற்றும் வெளியூடிய கருவிகளுடன் தொடர்பு கொள்வதில் மாறுதல் கொண்டுவரும் பத்து மைக்ரோசாஃப்ட் MCP சர்வர்களை ஆராய்கிறது. Azure வள மேலாண்மையிலிருந்து கோப்புகளின் செயலாக்கம் வரை, இந்த சர்வர்கள் வடிவமைப்பு சூழ்நிலை நடைமுறையின் சக்தியைக் காண்பிக்கின்றன, இது பிரச்சினைகளை எளிதாக்கி, உற்பத்தித்திறன் மேம்படுத்தும் மேம்பாட்டு பணிகளை உருவாக்குகிறது.

## கற்றல் குறிக்கோள்கள்

இந்த கையேடு முடிவில், நீங்கள்:
- MCP சர்வர்கள் டெவலப்பர் உற்பத்தித்திறனை எவ்வாறு மேம்படுத்துகின்றன என்பதை புரிந்துகொள்வீர்கள்
- மைக்ரோசாஃப்டின் மிகுந்த தாக்கமுள்ள MCP சர்வர் செயல்பாடுகளை கற்றுக்கொள்வீர்கள்
- ஒவ்வொரு சர்வருக்கும் நடைமுறை பயன்பாடுகளை கண்டுபிடிப்பீர்கள்
- VS Code மற்றும் Visual Studio இல் இந்த சர்வர்களை அமைத்து மதிப்பமைப்பது எப்படி என்பதை அறிந்துகொள்வீர்கள்
- MCP சூழல் மற்றும் எதிர்கால வழிகளைக் ஆராய்ப்பீர்கள்

## 🔧 MCP சர்வர்களை புரிந்துகொள்வது: ஒரு ஆரம்பநிலை கையேடு

### MCP சர்வர்களென்ன?

Model Context Protocol (MCP) இல் புதியவராக, "MCP சர்வர் என்றால் என்ன, எனக்கு அதற்குக் காரணம இருக்குமா?" என சந்தேகப்படலாம். ஒரு எளிய உடன்பாடுடன் தொடங்குவோம்.

MCP சர்வர்களை உங்கள் AI குறியீட்டு துணை (GitHub Copilot போன்ற) வெளியர் கருவிகள் மற்றும் சேவைகளுடன் இணைக்க உதவும் சிறப்பு உதவியாளர்களாக நினைத்துக் கொள்ளுங்கள். நீங்கள் ஒருங்கிணைந்த செயலிகளை பயன்படுத்தும் போல் – காலநிலை பார்க்க, வழிசெலுத்த, வங்கி பணியாற்ற – MCP சர்வர்கள் உங்கள் AI உதவியாளருக்கு வெவ்வேறு மேம்பாட்டு கருவிகள் மற்றும் சேவைகளுடன் தொடர்பு கொள்ளும் திறனை வழங்குகின்றன.

### MCP சர்வர்கள் தீர்க்கும் பிரச்சினை

MCP சர்வர்கள் இல்லாமல், நீங்கள்:
- Azure வளங்களைப் பார்க்க விரும்பினால்
- GitHub பிரச்சினை உருவாக்க விரும்பினால்
- உங்கள் தரவுத்தளத்தை கேள்வி கேட்க விரும்பினால்
- ஆவணங்களைத் தேட விரும்பினால்

நீங்கள் குறியீடு எழுத்ததை நிறுத்தி, உலாவியை திறந்து, சரியான இணையதளத்திற்கு செல்ல வேண்டி, இந்த பணிகளை கையாள வேண்டியிருப்பீர்கள். இந்த தொடர்ந்த சுற்றுப்பயணம் உங்கள் ஓட்டத்தை விரட்டிப் productivity யை குறைக்கிறது.

### MCP சர்வர்கள் உங்கள் மேம்பாட்டு அனுபவத்தை மாற்றுவது எப்படி

MCP சர்வர்களுடன், உங்கள் மேம்பாட்டு சூழலில் (VS Code, Visual Studio, மற்றும் பிற) இருந்தே உங்கள் AI உதவியாளரிடம் வேண்டுமென்றும் கேட்டுக் கொண்டு இந்த பணிகளைச் செய்யவும்நீங்கள் முடியும். உதாரணமாக:

**சம்பிரதாய வழிமுறையை மாற்றி:**
1. குறியீடு எழுதுவதை நிறுத்தவும்
2. உலாவியை திறக்கவும்
3. Azure போர்டல் செல்லவும்
4. சேமிப்பு கணக்குத் தகவல்களைப் பார்க்கவும்
5. VS Code இல் திரும்பவும்
6. குறியீடு தொடரவும்

**இப்போது நீங்கள் இதை செய்யலாம்:**
1. AI-யிடம் கேட்கவும்: "என்ன என் Azure சேமிப்பு கணக்குகளின் நிலை?"
2. வழங்கப்பட்ட தகவலுடன் குறியீடு தொடரவும்

### ஆரம்ப நிலை பயனாளர்களுக்கு முக்கிய நன்மைகள்

#### 1. 🔄 **உங்கள் ஓட்ட நிலையைத் தொடர்க**
- பல செயலிகள் மத்தியில் மாறுதல் வேண்டாம்
- நீங்கள் எழுதும் குறியீட்டில் கவனம் செலுத்துங்கள்
- வெவ்வேறு கருவிகளை நிர்வகிப்பதில் உள்ள மன அழுத்தத்தை குறைக்கவும்

#### 2. 🤖 **சிக்கலான கட்டளைகளுக்கு பதிலாக இயல்பான மொழியைப் பயன்படுத்துங்கள்**
- SQL உடன்பாடுகளை நினைவில் வைக்க வேண்டாம், நீங்கள் தேவையான தரவை விவரியுங்கள்
- Azure CLI கட்டளைகளை நினைவில் வைக்க வேண்டாம், நீங்கள் செய்ய விரும்பும் விஷயத்தை விளக்குங்கள்
- AI தொழில்நுட்ப விவரங்களை கையாளட்டும், நீங்கள் தார்க்கिक ளோஜிக் மீது கவனம் செலுத்துங்கள்

#### 3. 🔗 **பல கருவிகளை இணைக்கவும்**
- பல்வேறு சேவைகளை ஒன்றிணைத்து சக்திவாய்ந்த வேலைப்பாடுகளை உருவாக்கவும்
- உதாரணம்: "சமீபத்தில் உள்ள அனைத்து GitHub பிரச்சனைகளையும் பெறுக மற்றும் இணை Azure DevOps வேலை உருப்படிகளை உருவாக்குக"
- சிக்கலான ஸ்கிரிப்ட்களை எழுதாமல் தானாக இயங்க வேலையை அடையலாம்

#### 4. 🌐 **வளரும் சுற்றுச்சூழலை அணுகவும்**
- மைக்ரோசாஃப்ட், GitHub மற்றும் பிற நிறுவனங்கள் உருவாக்கிய சர்வர்களின் நன்மைகளைப் பெறவும்
- வெவ்வேறு விற்பனையாளர்களின் கருவிகளை எளிதாக கலந்து பயன்படுத்தவும்
- வெவ்வேறு AI உதவியாளர்களுக்கு பொருந்தும் ஒரு தரநிலைச் சுற்றுச்சூழலில் இணைவீர்கள்

#### 5. 🛠️ **செய்து கற்றுக்கொள்ளுங்கள்**
- முன் கட்டிய சர்வர்களுடனே ஆரம்பித்து கருத்துக்களை புரிந்து கொள்ளுங்கள்
- மெல்ல மெல்ல உங்களுக்கான சர்வர்களை உருவாக்குங்கள்
- கிடைக்கும் SDKகள் மற்றும் ஆவணங்களைப் பயன்படுத்தி கற்றலை வழிகாட்டு

### ஆரம்ப நிலை பயனாளர்களுக்கான உண்மை உலக உதாரணம்

நீங்கள் இணைய மேம்பாட்டில் புதியவர் என்றும் உங்கள் முதல் திட்டத்தில் பணியாற்றிக் கொண்டுப்போகிறீர்கள் என்று கூறுவோம். MCP சர்வர்கள் உதவும் முறையை இங்கே காணலாம்:

**சம்பிரதாய அணுகுமுறை:**
```
1. Code a feature
2. Open browser → Navigate to GitHub
3. Create an issue for testing
4. Open another tab → Check Azure docs for deployment
5. Open third tab → Look up database connection examples
6. Return to VS Code
7. Try to remember what you were doing
```

**MCP சர்வர்களுடன்:**
```
1. Code a feature
2. Ask AI: "Create a GitHub issue for testing this login feature"
3. Ask AI: "How do I deploy this to Azure according to the docs?"
4. Ask AI: "Show me the best way to connect to my database"
5. Continue coding with all the information you need
```

### நிறுவன தரநிலை நன்மை

MCP ஒரு தொழில்துறை பரவலான தரநிலை ஆகி வருகிறது, அதனால்:
- **தொடர்ச்சியான அனுபவம்**: வெவ்வேறு கருவிகள் மற்றும் நிறுவனங்களில் ஒரே போலியான அனுபவம்
- **இணைவு திறன்**: பல்வேறு விற்பனையாளர்களின் சர்வர்கள் ஒருங்கிணைந்து பணியாற்றுகின்றன
- **எதிர்கால பாதுகாப்பு**: திறன்கள் மற்றும் அமைப்புகள் பல்வேறு AI உதவியாளர்களுக்கு இடையே பரிமாறப்படுகிறது
- **சமூகமூலம்**: பகிரப்பட்ட அறிவும் வளங்களும் நிறைந்த பெரிய சூழல்

இந்த கையேட்டில், அனைத்து நிலைகளிலுள்ள டெவலப்பர்களுக்கு மிக உதவியாக இருக்கும் 10 மைக்ரோசாஃப்ட் MCP சர்வர்களைப் பற்றி பார்ப்போம். ஒவ்வொரு சர்வரின் நோக்கம்:
- பொதுவான மேம்பாட்டு பிரச்சனைகளை தீர்க்க
- முறையே செய்யவேண்டிய பணிகளை குறைக்க
- குறியீட்டின் தரத்தை மேம்படுத்த
- கற்றல் வாய்ப்புகளை வளர்க்க

> **💡 கற்றல் குறிப்புகள்**
> 
> MCP இல் முழுமையாக புதியவராக இருந்தால், முதலில் [MCP அறிமுகம்](../00-Introduction/README.md) மற்றும் [முக்கிய கருத்துக்கள்](../01-CoreConcepts/README.md) அலகுகளைப் படியுங்கள். அதன்பின்பு இங்கு திரும்பி இந்த கருத்துக்களை Microsoft கருவிகளுடன் செயல்படுத்தியதை பாருங்கள்.
>
> MCP-ன் முக்கியத்துவப் பற்றி மேலதிகத் தெளிவுக்கு, Maria Naggaga-வின் பதிவு: [Connect Once, Integrate Anywhere with MCP](https://devblogs.microsoft.com/blog/connect-once-integrate-anywhere-with-mcps) ஐ பாருங்கள்.

## VS Code மற்றும் Visual Studio இல் MCP தொடக்கம் 🚀

MCP சர்வர்களை அமைப்பது Visual Studio Code அல்லது Visual Studio 2022 GitHub Copilot உடன் பயன்படுத்துவோருக்கு எளிதாகும்.

### VS Code அமைப்பு

VS Code இல் அடிப்படை செயல்முறை:

1. **Agent Mode ஐ இயக்கு**: VS Code-இல் Copilot Chat திரையில் Agent mode க்கு மாற்றவும்
2. **MCP சர்வர்கள் அம்மா செய்க**: உங்கள் VS Code settings.json கோப்பில் சர்வர் அமைப்புகளைச் சேர்க்கவும்
3. **சர்வர்களைத் தொடங்கு**: நீங்கள் பயன்படுத்த விரும்பும் ஒவ்வொரு சர்வருக்கும் "Start" பொத்தானை அழுத்தவும்
4. **கருவிகளைத் தேர்ந்தெடு**: தற்போதைய அமர்விற்கான MCP சர்வர்களை இயக்கு

விரிவான அமைப்பு படிகளுக்கு [VS Code MCP ஆவணம்](https://code.visualstudio.com/docs/copilot/copilot-mcp) ஐப் பார்க்கவும்.

> **💡 பரிந்துரையுடன்: MCP சர்வர்களை திறம்பட நிர்வகிக்கவும்!**
> 
> இப்போது VS Code விரிவீன்கள் பார்வையில் [நிறுவப்பட்ட MCP சர்வர்களை நிர்வகிக்க புதிய UI உள்ளது](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode)! இது எளிய மற்றும் தெளிவான முகப்புடன் உங்களுக்கு சர்வர்களை துவக்க, நிறுத்த மற்றும் நிர்வகிக்க விரைவான அணுகலை வழங்குகிறது. உற்றுக்கட்டுங்கள்!

### Visual Studio 2022 அமைப்பு

Visual Studio 2022 (பதிப்பு 17.14 அல்லது அதற்கு மேல்) க்கான படிகள்:

1. **Agent Mode ஐ இயக்கு**: GitHub Copilot Chat திரையில் "Ask" குறுக்குவழிப் பட்டியில் "Agent" தேர்ந்தெடுங்கள்
2. **அமைப்பு கோப்பை உருவாக்கவும்**: உங்கள் தீர்வு அடித்தளத்தில் `.mcp.json` கோப்பை உருவாக்க (பரிந்துரைக்கப்பட்டது: `<SOLUTIONDIR>\.mcp.json`)
3. **சர்வர்களை அமைக்கவும்**: MCP முறைப்படி உங்கள் சர்வர் அமைவுகளைச் சேர்க்கவும்
4. **கருவி அங்கீகாரம்**: கோரப்பட்ட போது, தேவையான உரிமைகளுடன் கருவிகளை பயன்படுத்த அங்கீகாரம் அளிக்கவும்

விரிவான Visual Studio அமைப்பு வழிகாட்டிக்கான [Visual Studio MCP ஆவணம்](https://learn.microsoft.com/visualstudio/ide/mcp-servers) பார்க்கவும்.

ஒவ்வொரு MCP சர்வருக்கும் சொந்த கட்டமைப்பு தேவைகள் உள்ளன (தொடர்பு வரிசைகள், அங்கீகாரம் முதலியன), ஆனால் இரு ஐடிஇக்களில் அமைப்பு நிரம்ப ஒரே மாதிரி உள்ளது.

### 1. 📚 Microsoft Learn Docs MCP சர்வர்

[![VS Code இல் நிறுவுக](https://img.shields.io/badge/VS_Code-Install_Microsoft_Docs_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D) [![VS Code Insiders இல் நிறுவுக](https://img.shields.io/badge/VS_Code_Insiders-Install_Microsoft_Docs_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/mcp)

**இது செய்யும் வேலை**: Microsoft Learn Docs MCP சர்வர் என்பது Model Context Protocol வழியாக AI உதவியாளர்களுக்கு நேரடி அணுகலை வழங்கும் கிளவுட்-хோஸ்ட் சேவையாகும். இது `https://learn.microsoft.com/api/mcp` உடன் இணைகிறது மற்றும் Microsoft Learn, Azure ஆவணங்கள், Microsoft 365 ஆவணங்கள் மற்றும் பிற அதிகாரப்பூர்வ Microsoft மூலங்களில் பொருந்தக்கூடிய செமாண்டிக் தேடலை இயக்கு.

**இதன் பயன் என்ன**: இது "ஆவணங்கள் மட்டும்தான்" என்று தோன்றலாம், ஆனால் Microsoft தொழில்நுட்பங்களைப் பயன்படுத்தும் ஒவ்வொரு டெவலப்பருக்கும் இது அவசியமானது. .NET டெவலப்பர்கள் AI உதவியாளர்களின் புதிய .NET மற்றும் C# வெளியீடுகள் பற்றி புதிய தகவல் இல்லாததற்கு பெரும் குறைபாடுகளைக் கூறுவார்கள். Microsoft Learn Docs MCP சர்வர் இதை தீர்கிறது, மிக சமீபத்திய ஆவணங்கள், API கையேடுகள் மற்றும் சிறந்த நடைமுறைகளை நேரடியாக வழங்குகிறது. நீங்கள் சமீபத்திய Azure SDKகள், புதிய C# 13 அம்சங்கள் அல்லது முன்நோக்கி .NET Aspire மாதிரிகள் பயன்படுத்தினாலும், இந்த சர்வர் உங்கள் AI உதவியாளருக்கு சரியான, நவீன குறியீட்டை உருவாக்க அதிகமான, அதிகாரப்பூர்வ தகவல்களை விரைந்து வழங்குகிறது.

**உண்மை உலக பயன்பாடு**: "அதிகாரப்பூர்வ Microsoft Learn ஆவணத்தின் படி Azure கன்டெய்னர் ஆப் உருவாக்க az cli கட்டளைகள் என்ன?" அல்லது "ASP.NET Core இல் Entity Framework ஐ உணர்வெழுத்துச் சேர்க்கை (dependency injection) உடன் எப்படி அமைக்க?" அல்லது "இந்த குறியீடு Microsoft Learn ஆவணங்களின் செயல்திறன் பரிந்துரைகளுடன் பொருந்துகிறதா என மதிப்பாய்வு செய்க." இந்த சர்வர் Microsoft Learn, Azure ஆவணங்கள் மற்றும் Microsoft 365 ஆவணங்களில் பிடியாகி செமாண்டிக் தேடலை முன்னேற்றி, பொருந்தக்கூடிய 10 தரம் மிக்க உள்ளடக்க துண்டுகளுடன் கட்டுரைகளின் தலைப்புகள் மற்றும் URLகளை வழங்குகிறது. அது வெளியிடப்பட்டவுடன் எப்போதும் சமீபத்திய Microsoft ஆவணங்களை அணுகுகிறது.

**முக்கிய எடுத்துக்காட்டு**: இந்த சர்வர் Microsoft-ன் அதிகாரப்பூர்வ தொழில்நுட்ப ஆவணங்களிடம் செமாண்டிக் தேடல் செய்கின்ற `microsoft_docs_search` கருவியை வெளிப்படுத்துகிறது. ஒருமுறை அமையப்பெற்ற பின், "ASP.NET Core இல் JWT அங்கீகாரம் எப்படி நடைமுறைப்படுத்தல்?" போன்ற கேள்விகளை கேட்டால், துல்லியமான அதிகாரப்பூர்வ பதில்களுடன் மூல இணைப்புகள் கிடைக்கும். தேடல் தரம் சிறந்ததாகும் ஏனெனில் அது சூழலைப் புரிந்துகொள்கிறது – "கன்டெய்னர்கள்" என்ற சொல்லை Azure சூழலில் கேட்கும் போது Azure Container Instances ஆவணம், அதே சொல்லை .NET சூழலில் கேட்கும் போது தொடர்புடைய C# தொகுப்பு தகவல்கள் கிடைக்கும்.

இது வேகமாக மாறும் அல்லது சமீபத்தில் புதுப்பிக்கப்பட்ட நூலகங்களுக்கும் பயன்பாடுகளுக்கும் மிகவும் பயனுள்ளது. சில சமீபத்திய குறியீடு திட்டங்களில் நான் .NET Aspire மற்றும் Microsoft.Extensions.AI இன் சமீபத்திய வெளியீடுகளில் உள்ள அம்சங்களை பயன்படுத்தவேண்டியிருந்தது. Microsoft Learn Docs MCP சர்வரைச் சேர்த்ததால், நான் API ஆவணங்கள் மட்டுமல்லாமல் சமீபத்தில் வெளியிடப்பட்ட ஒவ்வொருவர் படி வழிகாட்டுதல்களையும் பயன்படுத்தி இருக்க முடிந்தது.

**இது என்ன செய்கிறது**: Azure MCP சர்வர் என்பது 15+ சிறப்பு Azure சேவை இணைப்புகளை கொண்ட விரிவான தொகுப்பு ஆகும், இது முழு Azure சூழல் உங்கள் AI வேலைவாய்ப்புக்கு கொண்டு வருகின்றது. இது ஒரே ஒரு சர்வர் அல்ல – இது வள மேலாண்மை, தரவுத்தளம் இணைப்பு (PostgreSQL, SQL Server), KQL உடன் Azure கண்காணிப்பாளர் பதிவு பகுப்பாய்வு, Cosmos DB ஒருங்கிணைப்பு மற்றும் பலவற்றை உள்ளடக்கிய சக்திவாய்ந்த தொகுப்பு ஆகும்.

**இது சக்திவாய்ந்தது ஏன்**: வெறும் Azure வளங்களை நிர்வகிப்பதற்கு மட்டுமல்ல, இந்த சர்வர் Azure SDK களுடன் வேலை செய்யும் போது குறியீடு தரத்தைத் திருப்திப்படுத்துகிறது. Azure MCP ஐ Agent முறையில் பயன்படுத்தினால், இது வெறும் குறியீடு எழுத உதவுவதல்ல – தற்போதைய அங்கீகார முறைகள், பிழை கையாளல் சிறந்த நடைமுறைகள் மற்றும் மேம்பட்ட SDK அம்சங்கள் பின்பற்றப்படும் சிறந்த Azure குறியீட்டை நீங்கள் எழுத உதவுகிறது. வழக்கமான செயல்படும் குறியீட்டிற்கு பதிலாக, தயாரிப்பு வேலைப்பாடுகளுக்கான Azure பரிந்துரைத்துமுறை பின்பற்றும் குறியீடு கிடைக்கும்.

**முக்கிய தொகுதிகள்**:
- **🗄️ தரவுத்தளம் இணைப்பிகள்**: Azure Database for PostgreSQL மற்றும் SQL Server க்கு நேரடியான இயற்கை மொழி அணுகல்
- **📊 Azure கண்காணிப்பாளர்**: KQL-சக்தியூட்டப்பட்ட பதிவு பகுப்பாய்வு மற்றும் இயக்கத் தொழில்நுட்ப அறிவுகள்
- **🌐 வள மேலாண்மை**: முழுமையான Azure வள வாழ்க்கைசுழற்சி மேலாண்மை
- **🔐 அங்கீகாரம்**: DefaultAzureCredential மற்றும் நிர்வகிக்கப்பட்ட அடையாள முறைகள்
- **📦 சேமிப்பு சேவைகள்**: Blob Storage, Queue Storage, மற்றும் Table Storage செயல்பாடுகள்
- **🚀 உள்ளமைப்புகள் சேவைகள்**: Azure Container Apps, Container Instances, மற்றும் AKS மேலாண்மை
- **மற்றும் பல சிறப்பு இணைப்பிகள்**

**வாஸ்தவ உலக பயன்பாடு**: "எனது Azure சேமிப்பு கணக்குகளை பட்டியலிடவும்", "கடந்த ஒரு மணித்தியாலத்தில் பிழைகள் குறிப்பாக என் Log Analytics வேலைவாய்ப்பில் வினவவும்", அல்லது "சரியான அங்கீகாரத்துடன் Node.js பயன்படுத்தி எனக்கு Azure பயன்பாட்டை கட்டிட உதவவும்"

**முழு பிரதிபலிப்பு காட்சி**: Azure MCP ஐ VS Code இல் GitHub Copilot உடன் இணைத்து அதன் சக்தியை காண்பிக்கும் முழுமையான நடைமுறை உள்ளது. இரண்டுமே நிறுவப்பட்டு, நீங்கள்:

> "DefaultAzureCredential அங்கீகாரம் பயன்படுத்தி Azure Blob Storage க்கு கோப்பை பதிவேற்ற ஒரு Python ஸ்கிரிப்ட் உருவாக்கவும். இந்த ஸ்கிரிப்ட் 'mycompanystorage' என்ற Azure சேமிப்பு கணக்குடன் இணைக்க வேண்டும், 'documents' என்ற கன்டெய்னரில் பதிவேற்ற வேண்டும், தற்போதைய காலச்சாய்ந்த ஒரு சோதனை கோப்பை உருவாக்கி பதிவேற்ற வேண்டும், பிழைகளை நுணுக்கமாக கையாள வேண்டும் மற்றும் தகவல் தரும் வெளியீடு வழங்க வேண்டும், Azure இன் அங்கீகாரத்திலும் பிழை கையாளலிலும் சிறந்த நடைமுறைகளை பின்பற்ற வேண்டும், DefaultAzureCredential அங்கீகாரம் எப்படி செயல்படுகிறது என்பதை விளக்கும் கருத்துக்களை அடக்க வேண்டும், மற்றும் ஸ்கிரிப்ட் சரியான செயல்பாடுகளுடன் மற்றும் ஆவணங்களுடன் நன்றாக வடிவமைக்கப்பட்டதாக இருக்க வேண்டும்."

Azure MCP சர்வர் முழுமையான, தயாரிப்பு-தயாரான Python ஸ்கிரிப்டை உருவாக்கும், இது:
- சரியான async முறைகளுடன் புதிய Azure Blob Storage SDK ஐ பயன்படுத்தும்
- விரிவான மாற்று சங்கிலியை விளக்கும் DefaultAzureCredential ஐ அமல்படுத்துகிறது
- Azure தனிப்பட்ட தவறு வகைகள் உடன் கடுமையான பிழை கையாளலை உட்கொள்கிறது
- வள மேலாண்மை மற்றும் இணைப்பு கையாளலில் Azure SDK சிறந்த நடைமுறைகளை பின்பற்றுகிறது
- விரிவான பதிவு மற்றும் தகவல் தரும் கணினி வெளியீட்டை வழங்குகிறது
- செயல்பாடுகள், ஆவணங்கள் மற்றும் வகை குறிப்புகளுடன் சரியான வடிவமைக்கப்பட்ட ஸ்கிரிப்டை உருவாக்குகிறது

இதன் அருமை என்னவென்றால், Azure MCP இல்லாமல், நீங்கள் பொதுவான blob storage குறியீட்டை பெறுவீர்கள், அது தற்போதைய Azure நடைமுறைகளைப் பின்பற்றாதிருக்கலாம். Azure MCP உடன், நீங்கள் அண்மையில் பரிந்துரைக்கப்படும் அங்கீகாரம் முறை, Azure-சார்ந்த பிழை பராமரிப்பு மற்றும் Microsoft பரிந்துரைத்த தயாரிப்பு பயன்பாடுகளை பின்பற்றும் குறியீட்டை பெறுவீர்கள்.

**காட்டிக்கொடுத்த உதாரணம்**: நான் `az` மற்றும் `azd` CLI களுக்கான குறிப்பிட்ட கட்டளைகளை நினைவில் வைக்க சிரமப்பட்டுள்ளேன். எனக்கு எப்போதும் இரண்டு படிகள் உள்ளன: முதலில் சொற்குறிப்பை பார்த்து, பிறகு கட்டளை ஓட்டுகிறேன். CLI சொற்குறிப்பை நினைவில் வைக்க முடியவில்லை என்று ஒப்புக் கொள்ள விரும்பவில்லை என்பதால், நான் அடிக்கடி போர்டல் க்குள் சென்று கிளிக் செய்து வேலை முடிக்கிறேன். நான் என்ன இல்லை என்பதைக் கூறாமல் விரும்புவது தான் மகிழ்ச்சிகரமாக இருக்கிறது; என் IDE விட்டு வெளியேறாமல் விருப்பமுள்ளதை விளக்கச் சொல்லுதல் அதிசயமானது!

[Azure MCP repository](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) இல் ஒரு சிறந்த பயன்பாட்டு பட்டியன்று உள்ளது. முழுமையான அமைப்பு வழிகாட்டிகளுக்கும் மேம்பாடான காட்சி விருப்பங்களுக்கும் [அதிகாரபூர்வ Azure MCP ஆவணத்தை](https://learn.microsoft.com/azure/developer/azure-mcp-server/) பார்வையிடவும்.

### 3. 🐙 GitHub MCP Server

[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Server-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=github&config=%7B%22type%22%3A%20%22http%22%2C%22url%22%3A%20%22https%3A%2F%2Fapi.githubcopilot.com%2Fmcp%2F%22%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Server-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=github&config=%7B%22type%22%3A%20%22http%22%2C%22url%22%3A%20%22https%3A%2F%2Fapi.githubcopilot.com%2Fmcp%2F%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/github/github-mcp-server)

**இது என்ன செய்கிறது**: அதிகாரபூர்வ GitHub MCP Server GitHub முழு சூழலுடன் குறைந்த தூரம் கொண்ட முகப்பை வழங்குகிறது, இதன் மூலம் தொகுதி மறைவான அணுகல் மற்றும் உள்ளூர் Docker வழங்கல் விருப்பங்கள் உள்ளன. இது வெறும் அடிப்படை தொகுதி செயல்பாடுகளுக்கு மட்டுமல்ல – GitHub Actions மேலாண்மை, pull request வேலைப்பாடுகள், பிரச்சினை கண்காணிப்பு, பாதுகாப்பு சோதனை, அறிவிப்புகள் மற்றும் மேம்பட்ட தானியங்கி செயல்பாடுகளை கொண்ட விரிவான கருவிப்பெட்டியாகும்.

**இது சக்திவாய்ந்தது ஏன்**: இந்த சர்வர் GitHub உடன் தொடர்பு கொள்ளும் முறையை மாற்றுகிறது, முழு தள அனுபவத்தை நேரடியாக உங்கள் மேம்பாட்டு சூழலுக்கு கொண்டு வருகிறது. VS Code மற்றும் GitHub.com இடையே இடையூறு இல்லாமல் திட்ட மேலாண்மை, குறியீடு மதிப்பாய்வு மற்றும் CI/CD கண்காணிப்பு போன்றவை இயற்கை மொழி கட்டளைகளில் நிர்வகிக்க முடியும்.

> **ℹ️ குறிப்பு: ‘அஜென்ட்’ வகைகள்**
> 
> இந்த GitHub MCP Server ஐ GitHub இன் Coding Agent (தானியக்க குறியீடு பணிகளுக்கான பிரச்சினைகளை ஒதுக்குக்கூடிய AI அஜெண்ட்) உடன் குழப்பிக் கொள்ளாதீர்கள். GitHub MCP Server VS Code இன் Agent முறையில் GitHub API ஒருங்கிணைப்பை வழங்குகிறது, Coding Agent பிரிவித்துவான அம்சமாக, GitHub பிரச்சினைகளை ஒதுக்கும்போது pull requests உருவாக்குகிறது.

**முக்கிய திறன்கள்**:
- **⚙️ GitHub Actions**: முழுமையான CI/CD குழாய் மேலாண்மை, வேலைப்பாடு கண்காணிப்பு மற்றும் காணொளி கைப்பற்றுதல்
- **🔀 Pull Requests**: PRகளை உருவாக்க, மதிப்பாய்வு செய்ய, ஒருங்கிணைக்க மற்றும் நிலை கண்காணிப்புடன் நிர்வகிக்க
- **🐛 பிரச்சினைகள்**: முழு பிரச்சினை வாழ்க்கைச் சுற்று மேலாண்மை, கருத்திடல், லேபல் மற்றும் ஒதுக்கல்
- **🔒 பாதுகாப்பு**: குறியீடு சோதனை தெரிவிப்புகள், ரகசிய கண்டறிதல் மற்றும் Dependabot ஒருங்கிணைப்பு
- **🔔 அறிவிப்புகள்**: அறிவிப்பு மேலாண்மை மற்றும் தொகுதி சந்தா கட்டுப்பாடு
- **📁 தொகுதி மேலாண்மை**: கோப்பு செயல்பாடுகள், கிளை மேலாண்மை மற்றும் தொகுதி நிர்வகம்
- **👥 ஒத்துழைப்பு**: பயனர் மற்றும் நிறுவன தேடல், குழு மேலாண்மை மற்றும் அணுகல் கட்டுப்பாடு

**வாச்தவ உலக பயன்பாடு**: "எனது செயல்பாட்டு கிளையிலிருந்து pull request உருவாக்கவும்", "இந்த வாரம் தோல்வியடைந்த CI ஓட்டங்களைக் காண்பிக்கவும்", "எனது தொகுதிகளுக்கான திறந்த பாதுகாப்பு அறிவிப்புகளை பட்டியலிடவும்", அல்லது "எனக்கு ஒதுக்கப்பட்ட அனைத்து பிரச்சினைகளையும் என் நிறுவனங்களில் கண்டறியவும்"

**முழு பிரதிபலிப்பு காட்சி**: GitHub MCP Server திறன்களை காட்சி படமாக்கும் சக்திவாய்ந்த வேலைப்பாடு:

> "எங்கள் ஸ்பிரின்ட் மதிப்பாய்வுக்கு தயாராக வேண்டும். இந்த வாரம் நான் உருவாக்கிய pull requests அனைத்தை காண்பிக்கவும், எங்கள் CI/CD குழாய்களின் நிலையை சரிபார்த்து, நாம் முகாமையாக்க வேண்டிய எந்த பாதுகாப்பு அறிவிப்புகளின் சுருக்கத்தையும் உருவாக்கவும், 'feature' லேபிள் கொண்ட ஒருங்கிணைந்த PRகளின் அடிப்படையில் வெளியீட்டு குறிப்புகளை உருவாக்க உதவவும்."

GitHub MCP Server:
- நீண்ட நிலை தகவலுடன் சமீபத்திய pull requests கேள்விப்படையும்
- வேலைப்பாடு ஓட்டங்களை பகுப்பாய்வு செய்து தோல்விகள் மற்றும் செயல்திறன் பிரச்சினைகளை குறிக்கிறது
- பாதுகாப்பு சோதனை முடிவுகளை தொகுத்து முக்கிய அறிவிப்புகளை முன்னுரிமை அளிக்கிறது
- ஒருங்கிணைந்த PRகளில் இருந்து தகவலை வெளியிடி முழுமையான வெளியீட்டு குறிப்புகள் உருவாக்குகிறது
- ஸ்பிரின்ட் திட்டமிடல் மற்றும் வெளியீடு தயாரிப்புக்கு செயல்திறன் அடுத்த படிகள் வழங்குகிறது

**காட்டிக்கொடுத்த உதாரணம்**: குறியீடு மதிப்பாய்வு வேலைப்பாடுகளுக்கு இதை பயன்படுத்துவதை விரும்புகிறேன். VS Code, GitHub அறிவிப்புகள் மற்றும் pull request பக்கங்களுக்கு இடையே மாற வேண்டிய அவசியமில்லை; நான் "எனது மதிப்பாய்விற்கு காத்திருக்கும் அனைத்து PRகளையும் காட்டவும்" என்பதும் பின்னர் "PR #123 இல் Authentication முறையின் தவறு கையாள بارےக் கருத்தைச் சேர்க்கவும்" என்பதும் கூற முடியும். சர்வர் GitHub API அழைப்புகளை நடத்தி, விவாதத்தின் உள்ளடக்கத்தை பராமரித்து, கூடுதலாக உள்ளடக்கமான மதிப்பாய்வு கருத்துக்களை உருவாக உதவுகிறது.

**அங்கீகார விருப்பங்கள்**: சர்வர் OAuth (VS Code இல் இடையூறு இல்லாமல்) மற்றும் Personal Access Tokens இரண்டையும் ஆதரிக்கிறது, உங்கள் தேவைபடி GitHub செயல்பாடு மட்டும் இயக்கப்பட도록 கருவிப்பெட்டுக்கள் கட்டமைக்கப்படலாம். உடனடி அமைப்பிற்கு ரிமோட் ஹோஸ்ட் சேவையாக அல்லது முழு கட்டுப்பாட்டிற்கு உள்ளூரிலான Docker மூலம் இயக்கு.

### 5. 📝 MarkItDown MCP Server
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_MarkItDown_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_MarkItDown_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/markitdown)

**இதன் செயல்**: MarkItDown என்பது பல்வேறு கோப்பு வடிவங்களை உயர் தரமான மார்க்டவுனாக மாற்றும் விரிவான ஆவணம் மாற்றும் சேவையகம், இது LLM பயன்பாட்டிற்கும் உரை பகுப்பாய்வு வேலைநெறிகளுக்கும் சிறப்பாக உருவாக்கப்பட்டுள்ளது.

**ஏன் இது பயன்படுகிறது**: நவீன ஆவண வேலைநெறிகளுக்கு அவசியம்! MarkItDown முக்கியமான ஆவணம் கட்டமைப்புகளைப் பாதுகாத்து தலைப்புகள், பட்டியல்கள், அட்டவணைகள் மற்றும் இணைப்புகள் போன்றவற்றை கையாளும் பிரம்மாண்ட கோப்பு வடிவங்களைக் கையாள்கிறது. எளிய உரை எடுக்கும் கருவிகளுக்கு பதிலாக, இது மனதளவில் பொருளையும் மற்றும் தனியமைப்பை பராமரிக்க கவனம் செலுத்துகிறது, இது AI செயலாக்கத்திற்கும் மனித வாசிப்பிற்கும் பயனுள்ளது.

**ஆதரிக்கப்படும் கோப்பு வடிவங்கள்**:
- **ஆஃபிஸ் ஆவணங்கள்**: PDF, PowerPoint (PPTX), Word (DOCX), Excel (XLSX/XLS)
- **மீடியா கோப்புகள்**: படங்கள் (EXIF மெட்டாடேட்டா மற்றும் OCR உடன்), ஒலி (EXIF மெட்டாடேட்டா மற்றும் பேச்சு உரை மாற்றத்துடன்)
- **இணைய உள்ளடக்கம்**: HTML, RSS ஊட்டங்கள், YouTube URLகள், விக்கிப்பீடியா பக்கங்கள்
- **தரவு வடிவங்கள்**: CSV, JSON, XML, ZIP கோப்புகள் (உள்ளடக்கங்களை மீண்டும் செயலாக்கும்)
- **பதிப்பிப்பு வடிவங்கள்**: EPub, Jupyter நோட்புக்குகள் (.ipynb)
- **மின்னஞ்சல்**: Outlook செய்திகள் (.msg)
- **மேம்பட்டவை**: மேம்பட்ட PDF செயலாக்கத்திற்கு Azure Document Intelligence தொடர்பு

**மேம்பட்ட திறன்கள்**: MarkItDown LLM இயங்கும் படம் விளக்கங்களை ஆதரிக்கிறது (OpenAI கிளையண்ட் வழங்கப்படும்போது), மேம்பட்ட PDF செயலாக்கத்திற்கு Azure Document Intelligence, பேச்சு உள்ளடக்கத்திற்கான ஒலி உரை மாற்றம் மற்றும் கூடுதல் கோப்பு வடிவங்களுக்கு நீட்டிக்கும் பிளகின் அமைப்பை கொண்டுள்ளது.

**பயனுள்ள பயன்பாடு**: "இந்த PowerPoint காட்சிப்பாடலை எங்கள் ஆவண தளத்திற்கு மார்க்டவுனாக மாற்று", "இந்த PDF-இல் தலைப்பு அமைப்புடன் உரையை எடு", அல்லது "இந்த Excel கணினி பட்டியலை வாசிக்கக்கூடிய அட்டவணை வடிவமாக மாற்று"

**முகப்புப் உதாரணம்**: [MarkItDown ஆவணங்களை](https://github.com/microsoft/markitdown#why-markdown) மேற்கோள் காட்டினால்:

### 10. 🏢 Microsoft 365 முகவர்கள் கருவிகள் MCP சர்வர்

**இது செய்கிறது**: Microsoft 365 மற்றும் Microsoft 365 Copilot உடன் ஒருங்கிணைக்கப்படும் AI முகவர்களையும் பயன்பாடுகளையும் உருவாக்க தேவையான கருவிகளை தயாரிப்பவர்களுக்கு வழங்குகிறது, இதில் கட்டுப்பாட்டு சரிபார்ப்பு, எடுத்துக்காட்டு குறியீடு மீட்பு மற்றும் பிழைமறைவு உதவிகள் உள்ளன.

**ஏன் இது பயனுள்ளது**: Microsoft 365 மற்றும் Copilot-க்கு உருவாக்குவது சிக்கலான நிரல் கட்டமைப்புகளை மற்றும் குறிப்பிட்ட வளர்ச்சி மாதிரிகளை தொடர்புபடுத்துகிறது. இந்த MCP சர்வர் உதவியை நேரடியாக உங்கள் குறியீட்டு சூழலுக்கு கொண்டுவருவதால், நீடித்த ஆவணங்களை பார்க்காமல் கட்டுப்பாட்டு சரிபார்ப்பு, எடுத்துக்காட்டு குறியீடு கண்டுபிடிப்பு மற்றும் பொதுவான பிழைகளைத் திருத்த உதவுகிறது.

**இயற்பொருள் பயன்பாடு**: "என் அறிவுறுத்தல் முகவரின் கட்டளை நிலையானது சரிபார்க்கவும், பிழைகளை திருத்தவும்", "Microsoft Graph API பிளக்-இன் உருவாக்கக்கான எடுத்துக்காட்டு குறியீடு காண்பி", அல்லது "என் Teams செயலி அங்கீகார பிரச்சனைகளை தீர்க்க உதவி செய்"

**பிரபலமான எடுத்துக்காட்டு**: நான் Build நிகழ்ச்சியில் M365 முகவர்கள் பற்றி நண்பர் John Miller உடன் பேசியபோது இந்த MCP-வை பரிந்துரைத்தார். இது புதிய M365 முகவர் டெவலப்பர்களுக்கு சிறந்தது, ஏனெனில் இது கட்டமைப்புகள், எடுத்துக்காட்டு குறியீடு மற்றும் அடிப்படை அமைப்புகளை வழங்குகிறது, ஆவணங்களில் மூழ்காமல் துவங்க உதவுகிறது. கட்டளை சரிபார்ப்பு அம்சங்கள் குறிப்பாக கட்டளைகளின் அமைப்பு பிழைகளை தவிர்க்க உதவுகிறது, இது பல மணித்தியால பிழைத்திருத்தத்தைத் தடுக்கும்.

## அடுத்து என்ன? 🔮

## 📋 முடிவு

Model Context Protocol (MCP) டெவலப்பர்கள் AI உதவியாளர்கள் மற்றும் வெளிப்புற கருவிகளுடன் தொடர்புகொள்ளும் முறையை மாற்றிவருகிறது. இந்த 10 Microsoft MCP சர்வர்கள் இணக்கமான AI ஒருங்கிணைப்பின் சக்தியை வெளிப்படுத்தி, டெவலப்பர்களை அவர்களது உருவாக்க சூழலில் வைத்துக் கொண்டு சக்திவாய்ந்த வெளிப்புற திறன்களை எளிதாக அணுக வாய்ப்பு அளிக்கின்றன.

விரிவான Azure சூழல் ஒருங்கிணைப்பு முதல் Playwright போல உலாவி ஆட்டோமேஷன் மற்றும் MarkItDown போன்ற ஆவண செயலாக்க சிறப்பு கருவிகள் வரை, இந்த சர்வர்கள் பல்வேறு உருவாக்க சூழல்களில் MCP எவ்வாறு உற்பத்தித்திறனை மேம்படுத்தும் என்பதை பறைசாற்றுகின்றன. ஒருங்கிணைந்த தொழில்நுட்ப நெறிமுறை, இந்த கருவிகள் ஒன்றோடொன்று இலகுவாக செயல்பட்டு இணைந்த உருவாக்க அனுபவத்தை உருவாக்குகின்றது.

MCP சூழல் வளர்ந்து கொண்டிருக்கும் காரணத்தால், சமுதாயத்துடன் தொடர்பில் இருப்பதும், புதிய சர்வர்களை ஆராய்வதும், தனிப்பயன் தீர்வுகளை உருவாக்குவதும் உங்கள் உருவாக்க உற்பத்தித்திறனை அதிகரிக்க முக்கியமாக இருக்கும். MCP ஓப்பன் ஸ்டாண்டர்ட் இயல்பு என்பதால், பல்வேறு விற்பனையாளர்களின் கருவிகளை கலந்து உங்கள் தேவைக்கேற்ற சிறந்த பணியாற்றலை உருவாக்கலாம்.
