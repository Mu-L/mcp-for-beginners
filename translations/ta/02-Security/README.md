# MCP பாதுகாப்பு: AI அமைப்புகளுக்கான முழுமையான பாதுகாப்பு

[![MCP பாதுகாப்பு சிறந்த நடைமுறைகள்](../../../translated_images/ta/03.175aed6dedae133f.webp)](https://youtu.be/88No8pw706o)

_(இந்த படத்தை கிளிக் செய்து இந்த பாடத்தின் வீடியோவை பார்க்கவும்)_

சுகாதாரம் AI அமைப்பு வடிவமைப்பிற்கு அடிப்படை அம்சமாகும், எனவே நாங்கள் அதை எங்கள் இரண்டாவது பகுதியாக முக்கியத்துவம் தருகிறோம். இது Microsoft-ன் **Secure by Design** 원칙த்துடன் பொருந்துகிறது, [Secure Future Initiative](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) இல் இருந்து.

மாதிரி சூழல் நெறிமுறை (MCP) AI சார்ந்த பயன்பாடுகளுக்கு சக்திவாய்ந்த புதிய திறன்களை அறிமுகப்படுத்துகிறது, ஆனால் பாரம்பரிய மென்பொருள் அபாயங்களை தாண்டி தனித்துவமான பாதுகாப்பு சவால்களை உருவாக்குகிறது. MCP அமைப்புகள் நிலையான பாதுகாப்பு கவலைகளை (பாதுகாப்பான குறியீடு, குறைந்த அனுமதி, வழங்குபவர் சங்கிலி பாதுகாப்பு) மற்றும் புதிய AI-குறையாக்கக் கூடிய ஆபத்துக்களை எதிர்கொள்கிறன, அவை உள்ளது prompt injection, tool poisoning, session hijacking, confused deputy attacks, token passthrough vulnerabilities, மற்றும் dynamic capability modification.

இந்த பாடத்தில், MCP அமல்படுத்தல்களில் மிகவும் முக்கியமான பாதுகாப்பு அபாயங்களை ஆராய்வோம் — அங்கில் அங்கீகாரம், அனுமதி, அதிகப்படியான அனுமதிகள், மறை(prompt) ஊடியச் செய்திகள், அமர்வு பாதுகாப்பு, confused deputy பிரச்சனைகள், குறியாக்க முகாமை மற்றும் வழங்குபவர் சங்கிலி குறைகள் அடங்கும். நீங்கள் Microsoft பொதுவான தீர்வுகளான Prompt Shields, Azure Content Safety, மற்றும் GitHub Advanced Security உடன் இந்த அபாயங்களை சமாளிக்க பயன்பாட்டு கட்டுப்பாடுகள் மற்றும் சிறந்த நடைமுறைகளை கற்றுக்கொள்வீர்கள்.

## கற்றல் நோக்கங்கள்

இந்த பாடம் முடிவடையும் வரை, நீங்கள்:

- **MCP-க்கு உரிய அபாயங்களை அடையாளங்காண்வீர்கள்**: prompt injection, tool poisoning, அதிக அனுமதிகள், session hijacking, confused deputy பிரச்சனைகள், token passthrough குறைகள், மற்றும் வழங்குபவர் சங்கிலி அபாயங்கள் உள்ளிட்ட MCP அமைப்புகளுக்கே உரிய பாதுகாப்பு ஆபத்துக்களை அறிந்துகொள்வீர்கள்
- **பாதுகாப்பு கட்டுப்பாடுகளை பயன்படுத்துவீர்கள்**: வலுவான அங்கீகாரம், குறைந்த அனுமதி அணுகல், பாதுகாப்பான குறியீடு முகாமை, session பாதுகாப்பு கட்டுப்பாடுகள் மற்றும் வழங்குபவர் சங்கிலி சரிபார்ப்பு போன்ற செயல்திறன் குறிப்புகளை அமல்படுத்துவீர்கள்
- **Microsoft பாதுகாப்பு தீர்வுகளை பயன்படுத்துவீர்கள்**: Microsoft Prompt Shields, Azure Content Safety மற்றும் GitHub Advanced Security போன்ற தீர்வுகளை MCP வேலைப்பாடுகளுக்கு பயன்படுத்து கற்றுக்கொள்வீர்கள்
- **கருவி பாதுகாப்பை சரிபார்ப்பீர்கள்**: கருவி தரவுத்தளத்தை சரிபார்க்கும், இயக்கத் திருத்தங்கள் குறித்து கண்காணிப்பதும், மறை(prompt) ஊடியச் செய்திகளை எதிர்க்கும் முக்கியத்துவத்தை புரிந்துகொள்வீர்கள்
- **சிறந்த நடைமுறைகளை ஒருங்கிணைப்பீர்கள்**: நிலையான பாதுகாப்பு அடிப்படைகள் (பாதுகாப்பான குறியீடு, சர்வர் கடுமையாக்கம், பூஜ்ஜிய நம்பிக்கை) உடன் MCP-க்கு நோக்கமான கட்டுப்பாடுகளை இணைத்து முழுமையான பாதுகாப்பை உருவாக்குவீர்கள்

# MCP பாதுகாப்பு கட்டமைப்பு மற்றும் கட்டுப்பாடுகள்

நவீன MCP அமல்படுத்தல்களுக்கு பாரம்பரிய மென்பொருள் பாதுகாப்புக்கும் AI-க்கு உரிய ஆபத்துகளுக்கும் பதிலளிக்கும் பல அடுக்கு பாதுகாப்பு அணுகுமுறைகள் தேவைப்படுகிறது. விரைவாக மேம்படும் MCP குறிப்புத்தகங்கள் அதன் பாதுகாப்பு கட்டுப்பாடுகளை ஒட்டுமொத்தமாக வலுவுபடுத்தி, நிறுவன பாதுகாப்பு கட்டமைப்புகளுடன் சிறப்பாக ஒருங்கிணைக்க உதவுகின்றன.

[Microsoft Digital Defense Report](https://aka.ms/mddr) ஆராய்ச்சி காட்டுகிறது, **98% புகாரான முறைபேறுகள் வலுவான பாதுகாப்பு பராமரிப்பால் தடுப்படைகின்றன**. மிகவும் விளைவு தரும் பாதுகாப்பு நெறிமுறைகள் அத்துடன் MCP-க்கு வல்லுநர்தனமான கட்டுப்பாடுகளுடன் கூடிய அடிப்படை பாதுகாப்பு நடைமுறைகளின் ஒருங்கிணைப்பாகும்.

## தற்போதைய பாதுகாப்பு நிலை

> **குறிப்பு:** இந்த தகவல் **2026 நடுவே 5 ஆம் தேதி** அன்று MCP பாதுகாப்பு தரநிலைகள் மற்றும் **MCP Specification 2025-11-25** என்பதில் ஒத்துப்போக்காக உள்ளது. MCP நெறிமுறை வேகமாக வளர்ந்து கொண்டிருக்கிறது; எதிர்காலத்தில் புதிய அங்கீகார வடிவமைப்புகள் மற்றும் மேம்பட்ட கட்டுப்பாடுகள் உருவாக்கப்படலாம். அறிந்துக்கொள்ள எப்போதும் தற்போதைய [MCP Specification](https://spec.modelcontextprotocol.io/), [MCP GitHub repository](https://github.com/modelcontextprotocol), மற்றும் [பாதுகாப்பு சிறந்த நடைமுறைகள் ஆவணம்](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) ஐ பின்பற்றவும்.

## 🏔️ MCP பாதுகாப்பு உச்சித் திருவிழா பணிமனை (Sherpa)

**கையால் கற்றல் பாதுகாப்பு பயிற்சிக்காக**, நாங்கள் **MCP பாதுகாப்பு உச்சித் திருவிழா பணிமனை** (Sherpa) ஐ பரிந்துரைக்கிறோம் - Microsoft Azure இல் MCP சர்வர்களை பாதுகாப்பதற்கான முழுமையான வழிகாட்டப்படும் பயணம்.

### பணிமனை அறிமுகம்

[MCP பாதுகாப்பு உச்சித் திருவிழா பணிமனை](https://azure-samples.github.io/sherpa/) "உறுதிப்படுத்தப்படாத → பயன்படுத்து → சரி செய் → சரிபார்" என்று செயல்திறன் வாய்ந்த பயிற்சியை வழங்குகிறது. நீங்கள்:

- **தொடர்ச்சியான சிக்கல்களை உடைத்துக் கற்கலாம்**: முனைப்புடன் பாதுகாப்பற்ற சர்வர்களை பயன்படுத்துவதன் மூலம் குறைபாடுகளை அனுபவிப்பீர்கள்
- **Azure-உள்நாட்டு பாதுகாப்பைப் பயன்படுத்தவும்**: Azure Entra ID, Key Vault, API Management மற்றும் AI Content Safety ஆகியவற்றைப் பயன்படுத்து கற்றுக்கொள்வீர்கள்
- **ஆழ்ந்த பாதுகாப்பு நடைமுறையை பின்பற்றவும்**: பாதுகாப்பு அடுக்குகளை கட்டியெழுப்பும் முகாம்களை கடந்து முன்னேறுவீர்கள்
- **OWASP தரநிலைகளை பின்பற்றவும்**: ஒவ்வொரு தொழில்நுட்பமும் [OWASP MCP Azure Security Guide](https://microsoft.github.io/mcp-azure-security-guide/) அட்டவணையை பின்பற்றுகிறது
- **உற்பத்தி குறியீட்டை பெறுங்கள்**: செயல்படக்கூடிய, பரிசோதிக்கப்பட்ட செயல்பாட்டுடன் வெளிவந்து போகலாம்

## OWASP MCP முக்கிய 10 பாதுகாப்பு ஆபத்துகள்

[OWASP MCP Azure Security Guide](https://microsoft.github.io/mcp-azure-security-guide/) MCP அமல்படுத்தல்களின் மிகவும் முக்கிய 10 பாதுகாப்பு அபாயங்களை விளக்குகிறது:

| ஆபத்து | விளக்கம் | Azure முறைகள் |
|------|-------------|------------------|
| **MCP01** | குறியீடு தவறான முகாமை மற்றும் ரகசிய வெளியீடு | Azure Key Vault, Managed Identity |
| **MCP02** | அனுமதி உயர்வு (Scope Creep) | RBAC, Conditional Access |
| **MCP03** | கருவி விஷப்பொருள் | கருவி பரிசோதனை, முழுமைத்திறன் சரிபார்ப்பு |
| **MCP04** | மென்பொருள் வழங்குபவர் சங்கிலி தாக்குதல் மற்றும் சார்பு திருத்தம் | GitHub Advanced Security, சார்பு ஸ்கேனிங் |
| **MCP05** | கட்டளை ஊக்கவியல் மற்றும் அமலாக்கம் | உள்ளீடு சரிபார்ப்பு, sandboxing |
| **MCP06** | நோக்கம் திருப்புதல் | Azure AI Content Safety, Prompt Shields |
| **MCP07** | போதிய அங்கீகாரம் இல்லாமை மற்றும் அங்கீகாரம் | Azure Entra ID, OAuth 2.1 + PKCE |
| **MCP08** | ஆய்வறிக்கை மற்றும் தொலைநுண்கருத்து குறைவு | Azure Monitor, Application Insights |
| **MCP09** | நிழல் MCP சர்வர்கள் | API மைய ஆட்சி, நெட்வொர்க் தனிமைப்படுத்தல் |
| **MCP10** | சூழல் ஊக்கவியல் மற்றும் அதிக பகிர்வு | தரவு வகைப்படுத்தல், குறைந்த வெளிப்படுத்தல் |

### MCP அங்கீகார வளர்ச்சி

MCP குறிப்புத்தகங்கள் அங்கீகாரம் மற்றும் அங்கீகார அணுகுமுறையில் பெரு முன்னேற்றம் கண்டுள்ளன:

- **முந்தைய அணுகுமுறை**: துவக்க குறிப்புகள் வளர்ப்பாளர்களுக்கு தனித்துவமான அங்கீகாரம் சர்வர்களை உருவாக்க வேண்டியிருந்தது, MCP சர்வர்கள் OAuth 2.0 அங்கீகார சர்வராக செயல்பட்டு நேரடியாக ந用户 அங்கீகாரத்தை மேலாண்மை செய்தன
- **தற்போதைய தரநிலை (2025-11-25)**: மேம்பட்ட குறிப்புகள் MCP சர்வர்களுக்கு வெளிப்புற அடையாள வழங்குநர்களுக்கு (Microsoft Entra ID போன்றவை) அங்கீகாரத்தை பணியாற்ற அனுமதிக்கிறது, இது பாதுகாப்பு நிலையை மேம்படுத்துகிறது மற்றும் செயலாக்கக் கடினத்தன்மையை குறைக்கிறது
- **பரிமாற்ற அடுக்கு பாதுகாப்பு**: உள்ளூர் (STDIO) மற்றும் தொலைத்தொடர்பு (Streamable HTTP) இணைப்புகளுக்கு தகுந்த அங்கீகார மோதிரங்களை உடைய பாதுகாப்பு ஆதரவு மேம்படுத்தப்பட்டது

## அங்கீகாரம் மற்றும் அங்கீகார பாதுகாப்பு

### தற்போதைய சவால்கள்

நவீன MCP அமல்படுத்தல்கள் அங்கீகாரம் மற்றும் அங்கீகாரத்தில் பல சவால்களை எதிர்கொள்கின்றன:

### ஆபத்துகள் மற்றும் தாக்குதல் வழிகள்

- **தவறான அங்கீகாரம் நடைமுறை**: MCP சர்வரில் தவறான அங்கீகாரம் செயல்பாடு உணர்வு தரவு வெளிப்பாடு மற்றும் தவறான அணுகல் கட்டுப்பாடுகளை ஏற்படுத்தும்
- **OAuth குறியீடு களவாடல்**: உள்ளூர் MCP சர்வர் குறியீடு திருட்டு தாக்குதலாளர்களுக்கு சர்வர்களை போலியானவராக நுணுக்கம் செய்து கீழ்நிலை சேவைகளை அணுக அனுமதிக்கிறது
- **Token Passthrough குறைபாடுகள்**: தவறான குறியீட்டு மேலாண்மை பாதுகாப்பு கட்டுப்பாடுகளை தவிர்க்கும் மற்றும் பொறுப்புத்தன்மை இடைவெளிகளை உருவாக்கும்
- **அதிக அனுமதிகள்**: அதிக அனுமதியுள்ள MCP சர்வர்கள் குறைந்த அனுமதி கொள்கைகளை மீறி தாக்குதல் மேடைகளை விரிவாக்குகின்றன

#### Token Passthrough: ஒரு கவனிக்கவேண்டிய எதிர்மாதிரி

**தற்போதைய MCP அங்கீகாரக் குறிப்புருவில் Token Passthrough கடுமையான காரணிகளால் தெளிவாக தடுப்புச் செய்திருக்கின்றது**:

##### பாதுகாப்பு கட்டுப்பாடுகளை தவிர்க்குதல்  
- MCP சர்வர்கள் மற்றும் கீழ்தள API களில் முக்கியமான பாதுகாப்பு கட்டுப்பாடுகள் (அளவிடல், கோரிக்கை சரிபார்ப்பு, போக்குவரத்து கண்காணிப்பு) சரியான குறியீடு சரிபார்ப்போடு இணைக்கப்பட்டுள்ளன  
- நேரடி கிளையன்ட்-முதல் API குறியீடு பயன்பாடு இந்த அவசியமான பாதுகாப்புகளை தவிர்க்கின்றது, பாதுகாப்பு கட்டமைப்பை கெடுக்கிறது  

##### பொறுப்புத்தன்மை மற்றும் ஆய்வு சிரமங்கள்  
- MCP சர்வர்கள் மேல் வழங்கப்பட்ட குறியீடுகளை பயன்படுத்தும் கிளையன்ட்களை வேறுபடுத்த முடியாது, இதனால் ஆய்வுவழிகள் பாதிக்கப்படுகின்றன  
- கீழ்த்தள வள இடனர் பதிவுகள் உண்மையான MCP சர்வர் இடைநிலைப்படுத்துநர் இல்லாமல் தவறான கோரிக்கை தோற்றங்களை காட்டுகின்றன  
- நிகழ்வு விசாரணை மற்றும் ஒழுங்கு ஆய்வுகள் மிகவும் கடினமாகும்  

##### தரவு வெளியீடு அபாயங்கள்  
- சரிபார்க்கப்படாத குறியீடு உரிமைகள் திருடப்பட்ட குறியீடுகளை கொண்ட தீமை செய்பவர்களுக்கு MCP சர்வர்களை தரவுத் தொடர்பு முகவர்களாக பயன்படுத்த அனுமதிக்கிறது  
- நம்பிக்கை எல்லையின் மீறல்கள் அனுமதியற்ற அணுகல் மாதிரிகளை இயльக்கின்றன  

##### பல சேவை தாக்குதல் வழிகள்  
- பாதிக்கப்பட்ட குறியீடுகள் பல சேவைகளால் ஏற்றுக்கொள்ளப்பட்டால் இணைக்கப்பட்ட அமைப்புகளுக்கு அகலமான இயக்கம் ஏற்படுகின்றது  
- குறியீடு தோற்றத்தை சரிபார்க்க முடியாதபோது சேவைகளுக்கு இடையேயான நம்பிக்கை முறைகள் மீறப்படலாம்  

### பாதுகாப்பு கட்டுப்பாடுகள் மற்றும் ஈடுசெயல்கள்

**முக்கிய பாதுகாப்பு தேவைகள்:**

> **கட்டாயம்**: MCP சர்வர்கள் குறியீடுகள் **MCP சர்வருக்கு குறிப்பானவையாக இல்லாத** எந்த குறியீடுகளையும் ஏற்கக் கூடாது

#### அங்கீகார & அங்கீகார கட்டுப்பாடுகள்

- **அடிக்கடி அங்கீகார ஆய்வு**: MCP சர்வர் அங்கீகாரம் செயல்பாட்டை விரிவான ஆய்வுகள் மூலம் சரிபார்க்க, குறுகிய நோக்கத்துடனேயே மட்டுமே நியமிக்கப்பட்ட பயனாளர்கள் மற்றும் கிளையன்ட்கள் வளங்கள் அணுக காத்திருக்கின்றனர்  
  - **அமல்படுத்தல் வழிகாட்டி**: [Azure API Management as Authentication Gateway for MCP Servers](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)  
  - **ஆடையாள ஒருங்கிணைப்பு**: [Microsoft Entra ID-ஐ MCP சர்வர் அங்கீகாரத்திற்கு பயன்படுத்தல்](https://den.dev/blog/mcp-server-auth-entra-id-session/)  

- **பாதுகாப்பான குறியீடு முகாமை**: [Microsoft குறியீடு சரிபார்ப்பு மற்றும் வாழ்க்கைச் சுற்று சிறந்த நடைமுறைகள்](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens)

  - சரியான குறியீடு மீள்நுழைவையும் காலாவதியையும் இயக்கவும்  
  - குறியீடு மீள்நடவடிக்கைகளை மற்றும் அனுமதியற்ற பயன்பாடுகளை தடுப்பவை அமல்படுத்தவும்  

- **கடுமையான குறியீடு சேமிப்பு**: குறியீடுகளை இடத்தில் மற்றும் பரிமாற்றத்தின் போது குறியாக்கத்துடன் பாதுகாப்பு  
  - **சிறந்த நடைமுறைகள்**: [Secure Token Storage and Encryption Guidelines](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)  

#### அணுகல் கட்டுப்பாட்டு அமல்படுத்தல்

- **குறைந்த அனுமதி கொள்கை**: MCP சர்வர்களுக்கு தேவையான குறைந்தபட்ச அனுமதிகளை மட்டுமே அளிக்கவும்  
  - அடிக்கடி அனுமதி ஆய்வுகள் மற்றும் புதுப்பிப்புகள் மூலம் அதிக அனுமதி விரிவுகளை தடுக்கவும்  
  - **Microsoft ஆவணம்**: [Secure Least-Privileged Access](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)  

- **பங்கு அடிப்படையிலான அணுகல் கட்டுப்பாடு (RBAC)**: சிறு அளவிலான பங்கு நியமனங்கள் செய்யவும்  
  - குறிப்பிட்ட வளங்கள் மற்றும் செயல்களில் மட்டுமே பங்குகளை அதிகபட்சப்படுத்தவும்  
  - தாக்குதல் மேடைகளை விரிவாக்கும் பரவலான மற்றும் தேவையற்ற அனுமதிகளை தவிர்க்கவும்  

- **தொடர் அனுமதி கண்காணிப்பு**: தொடர்ச்சியான அணுகல் கணக்கெடுப்பு மற்றும் கண்காணிப்பு  
  - அனுமதிப்பயன்பாட்டு முறைமைகளை தவறுகள் குறித்து கண்காணிக்கவும்  
  - அதிக அல்லது பயன்படாத அனுமதிகளை உடனடியாக சரிசெய்யவும்  

## AI-க்கு உரிய பாதுகாப்பு ஆபத்துகள்

### Prompt Injection மற்றும் கருவி களவாடல் தாக்குதல்கள்

நவீன MCP அமல்படுத்தல்கள் பாரம்பரிய பாதுகாப்பு வழிமுறைகள் முழுமையாக சமாளிக்க முடியாத AI-க்கு உரிய தொலைநோக்கிய தாக்குதல்களை சந்திக்கின்றன:

#### **மறை(prompt) ஊடியச் செய்தி (Indirect Prompt Injection / crossing-domain Prompt Injection)**

**மறை(prompt) ஊடியச் செய்தி** MCP இயக்கப்படும் AI அமைப்புகளில் மிகவும் முக்கியமான பாதுகாப்பு குறைபாடுகளில் ஒன்றாகும். தாக்குதலாளர்கள் தீய ஆணைகளை வெளியே உள்ள உள்ளடக்கத் தேதிகளில், ஆவணங்களில், இணையப் பக்கங்களில், மின்னஞ்சல் மற்றும் தரவுத் தளங்களில் உட்படுத்தி, AI அமைப்புகள் அவற்றை சட்டசாரா கட்டளைகள் என்று செயலாற்றும் வகையில் அமைத்துள்ளனர்.

**தாக்குதல் நிலைகள்:**
- **ஆவண அடிப்படையிலான ஊடியச் செய்தி**: செயலாக்கப்பட்ட ஆவணங்களில் மறைக்கப்பட்ட தீவிரமான வழிமுறைகள் AI தவறான செயல்பாடுகளுக்கு வழிவகுக்கின்றன
- **இணைய உள்ளடக்கப் பயன்படுத்தல்**: களவாடப்பட்ட இணையப் பக்கங்களில் மறைந்த prompt கள் AI குணாதிசயத்தை கவரும் முறையில் தாக்கும்
- **மின்னஞ்சல் தாக்குதல்கள்**: மின்னஞ்சலில் உள்ள தீமை prompt கள் AI உதவியாளர்கள் தனிப்பட்ட தகவல்களை வெளியிட அல்லது அனுமதிக்கப்படாத செயல்களை செய்யச் செய்கின்றன
- **தரவு ஆதாரம் மாசுபாடு**: களவாடப்பட்ட தரவுத்தளங்கள் அல்லது API கள் AI அமைப்புகளுக்கு தீவிரமான உள்ளடக்கங்களை வழங்குகின்றன

**உலகளாவிய விளைவுகள்**: இத்தாக்குதல்கள் தரவு வெளியீடு, தனியுரிமை குறைவு, தீய உள்ளடக்க உருவாக்கம், மற்றும் பயனர் தொடர்புகளை மாற்றுதல் போன்ற தீமை விளைவுகளை ஏற்படுத்தக்கூடும். விரிவான ஆய்வுக்காக [Prompt Injection in MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/) ஐ பார்க்கவும்.

![Prompt Injection தாக்கல் வரைபடம்](../../../translated_images/ta/prompt-injection.ed9fbfde297ca877.webp)

#### **கருவி விஷப்பொருள் (Tool Poisoning) தாக்குதல்கள்**

**கருவி விஷப்பொருள்** MCP கருவிகளின் மெட்டா தரவுகளைக் குறிவைத்து LLM களின் கருவி விளக்கங்கள் மற்றும் அளவுரு விளக்கங்களை பயன்படுத்தி செயல்பாட்டில் தீங்குகளை ஏற்படுத்தும்.

**தாக்கல் முறைமைகள்:**
- **மெட்டா தரவு மாற்றம்**: தாக்குதலாளர்கள் கருவி விளக்கங்கள், அளவுரு வரையறைகள் மற்றும் பயன்பாட்டு எடுத்துக்காட்டுகளில் தீய வழிமுறைகளை ஊட்டுகின்றனர்
- **காணாமல் உள்ள வழிமுறைகள்**: மனித பயனர்களுக்கு தெரியாமல் AI மாதிரிகள் செயல்படுத்தும் மறை(prompt) வழிமுறைகள்
- **இயக்க மாற்றங்கள் ("Rug Pulls")**: பயனரின் ஒப்புதலுக்குப் பிறகு கருவிகள் தீய செயல்பாடுகளுக்காக மாற்றப்படுதல்
- **அளவுரு ஊடியச் செய்தி**: கருவி அளவுரு வடிவமைப்புகளில் தீய உள்ளடக்கம் AI குணாதிசயத்தை பாதிக்கும்

**பெருமளவு சேவையக அபாயங்கள்**: தொலை MCP சர்வர்கள் கருவி வரையறைகளை பயனர் முதன்மையாக ஒப்புக்கொண்ட பிறகு மாற்ற முடிந்து, முன்னர் பாதுகாப்பான கருவிகள் தீய செயல்பாடுகளாக மாறும் சூழல்களை உருவாக்குகின்றன. விரிவான ஆய்வுக்கு [Tool Poisoning Attacks (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks) ஐ பார்க்கவும்.

![கருவி ஊடியச் செய்தி தாக்கல் வரைபடம்](../../../translated_images/ta/tool-injection.3b0b4a6b24de6bef.webp)

#### **கூடுதல் AI தாக்குதல் வழிகள்**

- **அடங்கா(prompt) ஊடியச் செய்தி (Cross-Domain Prompt Injection - XPIA)**: பல தளங்களின் உள்ளடக்கத்தை பயன்படுத்தி பாதுகாப்பு கட்டுப்பாடுகளை மீறும் நுணுக்கமான தாக்குதல்கள்
- **டயனமிக் திறன் மாற்றம்**: ஆரம்ப பாதுகாப்பு மதிப்பீடுகளைத் தவிர்க்கும் கருவி திறன்களில் நேரடி மாற்றங்கள்
- **Context Window தொற்றுநோய்**: தீங்கிழக்கும் கட்டளைகளை மறைக்க பெரிய context ஜன்னல்களை மோசடியுடன் மாற்றும் தாக்குதல்கள்
- **மாதிரி குழப்ப தாக்குதல்கள்**: மாதிரிப் பழுதுபார்க்கப்பட்டதைக் கையாள்ந்து அப்படி இல்லாத அல்லது பாதுகாப்பற்ற நடத்தை உருவாக்குதல்


### AI பாதுகாப்பு ஆபத்து பாதிப்பு

**நிலையான தாக்கங்கள்:**
- **தகவல் திருட்டு**: அனுமதியில்லாத அணுகல் மற்றும் நிறுவன அல்லது தனிப்பட்ட விலையுயர்ந்த தரவு திருட்டு
- **தனியுரிமை மீறல்கள்**: தனிப்பட்ட அடையாளத் தரவுகள் (PII) மற்றும் இரகசிய வணிகத் தகவல்களின் வெளிப்பாடு  
- **கணினி நிர்வகிப்பு**: முக்கிய கணினி அமைப்புகள் மற்றும் பணிகள் மீது எதிர்பாராத மாற்றங்கள்
- **அங்கீகாரக் கடவுச்சொல் திருட்டு**: அங்கீகார குறியீடுகளின் பிழைபாடு மற்றும் சேவை அங்கீகாரங்கள்
- **பக்கவாய் நகர்வு**: பாதிக்கப்பட்ட AI அமைப்புகளை அகலமான வலைத்தள தாக்குதல்களுக்கான தொடரமாக பயன்படுத்தல்

### Microsoft AI பாதுகாப்பு தீர்வுகள்

#### **AI Prompt Shields: ஊசி ஊட்டும் தாக்குதல்கள் எதிராக முன்னேறிய பாதுகாப்பு**

Microsoft **AI Prompt Shields** நேரடி மற்றும் நேர்மறை ஊசி ஊட்டும் தாக்குதல்களுக்கு விரிவான பாதுகாப்பை பல பாதுகாப்பு அடுக்குகளின் மூலம் வழங்குகிறது:

##### **முக்கிய பாதுகாப்பு செயல்முறைகள்:**

1. **முன்னேறிய கண்டறிதல் மற்றும் வடிகட்டி செயல்பாடு**
   - இயந்திரக் கற்று அல்கொரிதம்கள் மற்றும் NLP நுட்பங்கள் வெளியால் வரும் தீங்குள்ள கட்டளைகளை கண்டறி
   - ஆவணங்கள், வலைப்பக்கங்கள், மின்னஞ்சல்கள் மற்றும் தரவு மூலங்களின் நேரடி ஆய்வு மூலம் அடங்கும் அபாயங்களை கண்டறிதல்
   - வலது மற்றும் தீங்குள்ள ப்ராம்ட் மாதிரிகள் குறித்து சூழல் புரிதல்

2. **கவனச்சிதறல் நுட்பங்கள்**  
   - நம்பகமான கணினி கட்டளைகள் மற்றும் பாதிக்கப்பட்ட வெளிநுழைவுகளை வேறுபடுத்தல்
   - உரை மாற்று முறைகள், மாதிரியின் பொருத்தத்தை மேம்படுத்தும் போது தீங்குள்ள உள்ளடக்கத்தை தனியாக்குதல்
   - AI அமைப்புகள் சரியான கட்டளை நிலையை பராமரிக்கவும் ஊசி ஊட்டிய கட்டளைகளை புறக்கணிக்கவும் உதவுதல்

3. **வரையறைகள் மற்றும் தரவு குறி அமைப்புகள்**
   - நம்பகமான கணினி செய்திகள் மற்றும் வெளிநுழைவு உரையின் இடையேயுள்ள தெளிவான எல்லைகள்
   - நம்பகமான மற்றும் நம்பமுடியாத தரவு மூலங்களுக்கு இடையேயான எல்லைகளை சிறப்பு குறிகள் காட்டுதல்
   - தெளிவான பிரிவு கட்டளை குழப்பத்தையும் அனுமதியிழந்த கட்டளையையும் தடுக்கும்

4. **தொடர்ச்சி அச்சுறுத்தல் நுண்ணறிவு**
   - Microsoft புதிய தாக்குதல் மாதிரிகளைப் பார்த்து பாதுகாப்புகளை புதுப்பித்தல்
   - புதிய ஊசி ஊட்டும் முறைகள் மற்றும் தாக்குதல் பாதைகளுக்கான முன்னோக்கிய அச்சுறுத்தல் வேட்டையாடல்
   - மாறும் அபாயங்களுக்கு எதிராக பாதுகாப்பு மாதிரிகளை முறையாக புதுப்பித்தல்

5. **Azure உள்ளடக்க பாதுகாப்பு ஒருங்கிணைப்பு**
   - விரிவான Azure AI உள்ளடக்க பாதுகாப்பு தொகுப்பின் பகுதி
   - ஜெயில்நிலையை தவிர்க்கவும், தீங்குள்ள உள்ளடக்கம் மற்றும் பாதுகாப்பு கொள்கை மீறலைக் கண்டறிதல்
   - AI பயன்பாட்டு கூறுகளுக்கிடையில் ஒருங்கிணைந்த பாதுகாப்பு கட்டுப்பாடுகள்

## மேம்பட்ட MCP பாதுகாப்பு அச்சுறுத்தல்கள்

### அமர்வு களவாடல் பாதிப்புகள்

**அமர்வு களவாடல்** என்பது இயங்கும் MCP அமலாக்கங்களில் ஒரு முக்கிய தாக்குதல் பாதையாகும், இதில் அனுமதி இல்லாதவர்கள் செல்லுபடியான அமர்வு அடையாளங்களைப் பெறுவதும், வாடிக்கையாளர்களாக போலி நடிப்பதும் மற்றும் அனுமதியின்றி நடவடிக்கைகளை மேற்கொள்வதும் உள்ளதாகும்.

#### **தாக்குதல் காட்சி நிரல்கள் மற்றும் அபாயங்கள்**

- **அமர்வு களவாடல் ஊசி ஊட்டும் தாக்குதல்**: திருடப்பட்ட அமர்வு அடையாளங்களுடன் தாக்குதலாளர்கள் சேரும் நிலையிலுள்ள சர்வர்களில் தீங்கு விளைவிக்கும் நிகழவழக்குகளை ஊசி ஊட்டுவார்கள் அல்லது முக்கிய தரவுகளை அணுகுவார்கள்
- **நேரடி போலி நடிப்பு**: திருடப்பட்ட அமர்வு அடையாளங்கள் நேரடியான MCP சர்வர் அழைப்புகளை சுயமரியாதையின்றி முடுக்கி, தாக்குதலாளர்களை செல்லுபடி பயனாளர்களாகக் கருதுகின்றன
- **குற்றமுற்ற திரும்பும் ஓட்டங்களின் பாதிப்பு**: தாக்குதலாளர்கள் கோரிக்கைகளை முன்கூட்டியே நிறுத்தி, செல்லுபடியான வாடிக்கையாளர்கள் தீங்குள்ள உள்ளடக்கத்துடன் மீண்டும் தொடங்குவார்கள்

#### **அமர்வு நிர்வாகத்திற்கான பாதுகாப்பு கட்டுப்பாடுகள்**

**வழிகாட்டும் நிபந்தனைகள்:**
- **அங்கீகார சரிபார்ப்பு**: அங்கீகாரத்தை அமுல்படுத்தும் MCP சர்வர்கள் எல்லா வரும் கோரிக்கைகளையும் சரிபார்க்க வேண்டும் மற்றும் அமர்வுகளை அங்கீகாரத்திற்கு நம்பக்கூடாது
- **பாதுகாப்பான அமர்வு உருவாக்கல்**: கிரிப்டோகிராபிக் பாதுகாப்பான, அறிமுகமில்லா அமர்வு அடையாளங்கள் பாதுகாப்பான சீரற்ற எண் உருவாக்கிகளால் உருவாக்கப்பட வேண்டும்
- **பயனர்-செயலாக்கப்பட்டது இணைப்பு**: `<user_id>:<session_id>` போன்ற வடிவத்தில் அமர்வு அடையாளங்களை பயனர் குறிப்பிட்ட தகவலுடன் இணைத்து, வாடிக்கையாளர் இடைவெளி இணைப்பைக் தடுப்பது
- **அமர்வு வாழ்நாள் நிர்வாகம்**: காலாவதி, மாற்றம் மற்றும் செல்லாதாக்கல் முறைகளை சரியாக செயல்படுத்தி பாதிப்பு நேரங்களை குறைப்பது
- **தரவு பரிமாற்ற பாதுகாப்பு**: அனைத்து தொடர்புகளுக்கும் HTTPS கட்டாயம், அமர்வு அடையாளக் கடத்தலை தவிர்க்க

### குழப்பமான துணை பிரச்சனை

**குழப்பமான துணை பிரச்சனை** என்பது MCP சர்வர்கள் வாடிக்கையாளர் மற்றும் மூன்றாம் தரப்பு சேவைகள் இடையேயான அங்கீகார முகவரிகளாக செயல்படும்போது ஏற்படுகிறது. இது நிலையான வாடிக்கையாளர் ஐடியைப் பயன்படுத்தி அங்கீகாரத்தை தவிர்க்கும் வாய்ப்புகளை உருவாக்குகிறது.

#### **தாக்குதல் இயந்திரங்கள் மற்றும் அபாயங்கள்**

- **குக்கீ அடிப்படையிலான சம்மதத் தவிர்ப்புகள்**: முந்தைய பயனர் அங்கீகாரம் சம்மத குக்கீகளை உருவாக்குகிறது, தாக்குதலாளர்கள் தீங்குள்ள அனுமதி கோரிக்கைகளுடன் உருவாக்கப்பட்ட மறுவழிச் URIகளை பயன்படுத்தி அதை அனுமோடுகின்றனர்
- **அங்கீகாரக் குறியீல் திருட்டு**: இருந்த சம்மதக் குக்கீகள் அங்கீகார சர்வர்களால் சம்மத திரைகளை தவிர்க்க உதவுகிறது, குறியீல்களை தாக்குதலாளரின் கட்டுப்பாட்டிலான இடங்களுக்கு மாற்றுகிறது  
- **அனுமதியில்லாத API அணுகல்**: திருடப்பட்ட அங்கீகாரக் குறியீல்களால் குறிகள் பரிமாற்றம் மற்றும் பயனர் போல நடிப்பு அனுமதிக்கப்படுகிறது

#### **தடுப்பு முறைகள்**

**கட்டாய கட்டுப்பாடுகள்:**
- **தெளிவான சம்மதக் கோரிக்கைகள்**: நிலையான வாடிக்கையாளர் ஐடியுடன் MCP முகவரி சர்வர்கள் ஒவ்வொரு இயங்குதளசெயலி வாடிக்கையாளர் அரசு சம்மதத்தை பெற்றிருக்க வேண்டும்
- **OAuth 2.1 பாதுகாப்பு நடைமுறை**: அனைத்து அங்கீகாரக் கோரிக்கைகளிலும் PKCE வழிமுறையுடன் OAuth பாதுகாப்பு சிறந்த நடைமுறைகளை பின்பற்றுங்கள்
- **கண்டிப்பான வாடிக்கையாளர் உறுதிப்படுத்தல்**: மறுவழிச் URI மற்றும் வாடிக்கையாளர் அடையாளங்களை நுணுக்கமாக செகார் செய்து தவறான பயன்பாட்டை தடுப்பது

### குறுந்தொகுப்பு வழியாக கடத்தல் பாதிப்புகள்  

**குறுந்தொகுப்பு வழியாக கடத்தல்** என்பது MCP சர்வர்கள் வாடிக்கையாளர் குறுந்தொகுப்புகளை சரியான சான்றளிப்பின்றி ஏற்றுக்கொண்டு, அதை தாழ்வுச் சேவைகளுக்கு அனுப்பும் விருப்பமற்ற செயல்முறை ஆகும். இது MCP அங்கீகார விதிகளுக்கு எதிராக உள்ளது.

#### **பாதுகாப்பு விளைவுகள்**

- **கட்டுப்பாடு முறிவு**: நேரடி கிளையண்ட்-மூலம் API குறுந்தொகுப்புகள் முக்கிய அளவைத்தேவை மற்றும் கண்காணிப்பு கட்டுப்பாடுகளை தவிர்க்கின்றன
- **ஆடிட்டிங் பாதிப்பு**: மேல் தரப்பில் வழங்கப்பட்ட குறுந்தொகுப்புகள் வாடிக்கையாளர் அடையாளத்தை அந்தரங்கப்படுத்த முடியாமல் செய்து, சம்பவ விசாரணையை முற்றிலும் சீர்குலைக்கும்
- **தடையின்றி தரவு திருட்டு வழியாக சேவைகள்**: சரிபார்க்காத குறுந்தொகுப்புகள் ஹேக்கர்கள் தங்களது நோக்கத்திற்கேற்ப சேவைகளை எதிரியைப் போல பயன்படுத்த இடம் தரும்
- **நம்பிக்கை எல்லை மீறல்கள்**: குறுந்தொகுப்புகளின் தோற்றங்களை சரிபார்க்க முடியாததால் தாழ்ந்த சேவைகள் மீது நம்பிக்கை விதிகள் மீறப்படலாம்
- **பல சேவை தாக்குதலின் விரிவாக்கம்**: பாதிக்கப்பட்ட குறுந்தொகுப்புகளை பல சேவைகளில் ஏற்றுக்கொள்வதால் பக்கவாயுஅகல்ச்சி

#### **தேவையான பாதுகாப்பு கட்டுப்பாடுகள்**

**ஏற்றுக் கொள்ள முடியாத நிபந்தனைகள்:**
- **குறுந்தொகுப்பு சரிபார்ப்பு**: MCP சர்வர்கள் MCP சர்வருக்கே பிரத்தியேகமாக வழங்கப்படாத குறுந்தொகுப்புகளை ஏற்றக்கூடாது
- **கேளிக்கை சரிபார்ப்பு**: குறுந்தொகுப்பு கேள்விகள் MCP சர்வரின் அடையாளத்துடன் பொருந்துவதாக மாறு சரிபார்க்க வேண்டும்
- **சரியான குறுந்தொகுப்பு வாழ்நாள்**: குறுகிய கால அனுமதி குறுந்தொகுப்புகளை பாதுகாப்பான மாற்ற நடைமுறைகளுடன் அமுல்படுத்த வேண்டும்


## AI அமைப்புகளுக்கான வழங்கல் சங்கில் பாதுகாப்பு

வழங்கல் சங்கில் பாதுகாப்பு பாரம்பரிய மென்பொருள் சார்ந்த குறையால் மீறாமல் முழு AI சூழலைப் பராமரிக்க விரிவுசெய்யப்பட்டுள்ளது. நவீன MCP அமலாக்கங்கள் அனைத்து AI சார்ந்த கூறுகளையும் முதன்மையாக நன்கு சரிபார்த்து கண்காணிக்க வேண்டும், ஏனெனில் ஒவ்வொன்றும் அமைப்பு ஒருமித்தத்தன்மையை பாதிக்கும் அபாயங்களை கொண்டு வருகிறது.

### விரிவாக்கப்பட்ட AI வழங்கல் சங்கில் கூறுகள்

**பாரம்பரிய மென்பொருள் சார்ந்தவை:**
- திறந்த மூல நூலகங்கள் மற்றும் சட்டங்கள்
- கொண்டெய்னர் படங்கள் மற்றும் அடிப்படை அமைப்புகள்  
- வளர்ச்சிச் சாதனங்கள் மற்றும் கட்டுமான குழாய்கள்
- அடித்தளம் கூறுகள் மற்றும் சேவைகள்

**AI குறிப்பிட்ட வழங்கல் சங்கில் கூறுகள்:**
- **அடித்தளம் மாதிரிகள்**: பல வழங்குநர்களிடமிருந்து முன்பயிற்சி பெற்ற மாதிரிகள், அறிமுகப்படுத்தல் சரிபார் வேண்டும்
- **ஒட்டுமொத்த சேவைகள்**: வெளியுறவு வெக்டார் வடிவமைப்பு மற்றும் அர்த்த வசூல் சேவைகள்
- **சூழல் வழங்குநர்கள்**: தரவு மூலங்கள், அறிவு தளங்கள் மற்றும் ஆவண சேகரிப்புகள்  
- **மூன்றாம் தரப்பு APIகள்**: வெளியுறவு AI சேவைகள், மெஷின் லெர்னிங் குழாய்கள் மற்றும் தரவு செயலாக்க முனைப்புள்ள இடங்கள்
- **மாதிரி கலைச்சொற்கள்**: எடைகள், கட்டமைப்புகள் மற்றும் சிறப்புசெய்யப்பட்ட மாதிரி வேறுபாடுகள்
- **பயிற்சி தரவு மூலங்கள்**: மாதிரி பயிற்சிக்கான மற்றும் நுண்ணோக்கி பயிற்சி தரவுத் தொகுப்புகள்

### விரிவாக்கப்பட்ட வழங்கல் சங்கில் பாதுகாப்பு செயல்திட்டம்

#### **கூடுதல் சரிபார்ப்பு மற்றும் நம்பிக்கை**
- **ஆரம்ப உதய சரிபார்ப்பு**: அனைத்து AI கூறுகளின் தோற்றம், அனுமதி மற்றும் ஒருமித்தத்தை ஒருங்கிணைப்புக்கு முன் சரிபார்
- **பாதுகாப்பு மதிப்பீடு**: மாதிரிகள், தரவு மூலங்கள் மற்றும் AI சேவைகளுக்கு கருத்திடும் வலுவான பாதுகாப்பு சமீப பார்வைச் செய்யவேண்டும்
- **புகழ்சாற்று பகுப்பு**: AI சேவை வழங்குநர்களின் பாதுகாப்பு சாதனை மற்றும் நடைமுறைகளை மதிப்பாய்வு செய்யவும்
- **காப்பு சரிபார்ப்பு**: அனைத்து கூறுகளும் நிறுவன பாதுகாப்பு மற்றும் கட்டுப்பாட்டு தேவைகளை பூர்த்தி செய்வதைக் உறுதி செய்யுங்கள்

#### **பாதுகாப்பான வெளியீட்டு குழாய்கள்**  
- **தானியங்கி CI/CD பாதுகாப்பு**: தானியங்கி வெளியீட்டு குழாய்களில் பாதுகாப்பு ஆய்வுகளை ஒருங்கிணைத்தல்
- **கலைச்சொற்களின் ஒருமித்த தன்மை**: அனைத்துக் கலைச்சொற்களுக்கும் (குறியீடு, மாதிரிகள், கட்டமைப்புகள்) கிரிப்டோகிராபிக் ஆதாரம் வழங்கல்
- **நிலை சார்ந்த வெளியீடு**: பாதுகாப்பு உறுதிப்பத்திரங்கள் உள்ள நிலைகள் தவிர கிராம்பு வெளியீடு
- **நம்பகமான கலைச்சொல் சேமிப்பகம்**: சரிபார் செய்யப்பட்ட பாதுகாப்பான கலைச்சொல் வெளி சேமிப்பகங்கள் மட்டுமே வெளியீட்டில் பயன்படுத்துதல்

#### **தொடர்ச்சி கண்காணிக்கும் மற்றும் பதில் அளிக்கும்**
- **இணைய சார்ந்த குறைபாடுகளுக்கான கண்காணிப்பு**: அனைத்து மென்பொருள் மற்றும் AI கூறுகளுக்கான குறைபாடுகளை தொடர்ந்து கண்காணித்தல்
- **மாதிரி கண்காணிப்பு**: மாதிரியின் நடத்தை, செயல்பாடு மாற்றங்கள் மற்றும் பாதுகாப்பு விசாரணைகளை தொடர்ச்சி மதிப்பாய்வு

- **மீள்பதிவு திறன்கள்**: தெரிந்த நல்ல பதிப்புக்கு விரைவில் திரும்ப இயலுமாறு ஏற்பாடு

- **தனியுரிமை கட்டுப்பாடுகள்**: AI அமைப்பு வியூகத்தில் தனியுரிமை-வடிவமைப்பு கொள்கைகளை நடைமுறைப்படுத்தல்
- **மாற்றம் மேலாண்மை**: AI அமைப்பு மாற்றங்களுக்கான பாதுகாப்பு ஆய்வு அதிகாரபூர்வ நடைமுறைகள்

இந்த அடித்தள நடைமுறைகள் MCP-க்கு தனித்துவமான பாதுகாப்பு கட்டுப்பாடுகளின் செயல்திறனை உயர் நிலைக்கு கொண்டு சென்று, AI இயக்கப்படும் பயன்பாடுகளுக்கு முழுமையான பாதுகாப்பை வழங்குகின்றன.

## முக்கிய பாதுகாப்பு எடுத்துக்காட்டுகள்
- **பல அடுக்கு பாதுகாப்பு அணுகுமுறை**: அடிப்படை பாதுகாப்பு பயிற்சிகள் (பாதுகாப்பான குறியீட்டல், குறைந்த உரிமைகள், சப்ளை செயின் சோதனை, தொடர்ச்சியான கண்காணிப்பு) மற்றும் AI-ஐ குறிக்குறிஞ்சிய கட்டுப்பாடுகளைக் கொண்டு முழுமையான பாதுகாப்பை ஒன்றிணைக்கவும்

- **AI-குறிந்த அச்சுறுத்தல் நிலம்**: MCP அமைப்புகள் தனித்துவமான ஆபத்துகளை சந்திக்கின்றன, இதில் தூண்டுதல் ஊடுருவல், கருவி விஷபரிசோதனை, அமர்வு கைப்பற்றல், குழப்பப்பட்ட துணை பிரச்சினைகள், டோக்கன் கடத்தல் கடுமைகள் மற்றும் மிகுந்த அனுமதிகள் அடங்கும், இதற்கான சிறப்பு தடுப்பு நடவடிக்கைகள் அவசியம்

- **அங்கீகாரம் & அனுமதி சிறந்ததாக்கல்**: புறஅடையாள வழங்குநர்களை (Microsoft Entra ID) பயன்படுத்தி வலுவான அங்கீகாரத்தை நடைமுறைப்படுத்தவும், சரியான டோக்கன் சரிபார்ப்பை உறுதி செய்யவும், உங்கள் MCP சேவையகத்திற்கு தெளிவாக வழங்கப்படாத டோக்கன்களை ஏற்காதீர்கள்

- **AI தாக்குதல் தடுப்பு**: Microsoft Prompt Shields மற்றும் Azure Content Safety-ஐ செயல்படுத்து, மறைமுக தூண்டுதல் ஊடுருவல் மற்றும் கருவி விஷபரிசோதனை தாக்குதல்களிடமிருந்து பாதுகாக்கவும், கருவி மெட்டாடேட்டாவை சரிபார்க்கவும் மற்றும் மாறுதல்களுக்கு கண்காணிப்பு அமைக்கவும்

- **அமர்வு & போக்குவரத்து பாதுகாப்பு**: பயன்பாட்டாளர் அடையாளத்துடன் பிணைக்கப்பட்ட, குறியீட்டு முறையில் பாதுகாக்கப்பட்ட, தீர்மானிக்க முடியாத அமர்வு ID-களை பயன்படுத்தவும், சரியான அமர்வு வாழ்நாள் மேலாண்மையை நடைமுறைப்படுத்தவும் மற்றும் அங்கீகாரத்திற்கு அமர்வுகளைப் பயன்படுத்தாதீர்கள்

- **OAuth பாதுகாப்பு சிறந்த நடைமுறைகள்**: தூண்டப்பட்ட துணை தாக்குதல்களைக் தடுப்பதற்கான தெளிவான பயனர் ஒப்புதலை இயக்கு, கடுமையான OAuth 2.1 பின்வட்டியுடன் PKCE பயன்பாடு மற்றும் கடுமையான மீள்நிர்ரோதி URI சரிபார்ப்பு

- **டோக்கன் பாதுகாப்பு 원칙ம்**: டோக்கன் கடத்தல் எதிர்முறை மாதிரிகளைத் தவிர்க்கவும், டோக்கன் பார்வையாளர் கோரிக்கைகளை சரிபார்க்கவும், குறுகிய கால டோக்கன்கள் மற்றும் பாதுகாப்பான மறு சுழற்சி அமைக்கவும் மற்றும் தெளிவான நம்பிக்கை எல்லைகளை பராமரிக்கவும்

- **முழுமையான சப்ளை செயின் பாதுகாப்பு**: அனைத்து AI சூழல் கூறுகளையும் (மாடல்கள், திரர்கள், சூழல் வழங்குநர்கள், புற API கள்) பாரம்பரிய மென்பொருள் சார்புகளுப்போல் அதே பாதுகாப்பு தீவிரத்துடன் கையாளவும்

- **தொடர்ச்சியான மேம்பாடு**: வேகமாக மாறும் MCP குறிப்பாடல்களுடனும் பாதுகாப்பு சமூகவியல் தரநிலைகளுடன் தொடர்ந்து இணைந்து, நெறிமுறை மேம்பாட்டுக்கேற்ப பாதுகாப்பு நிலைகளை மாறுபட வைத்திருங்கள்

- **Microsoft பாதுகாப்பு ஒருங்கிணைப்பு**: Microsoft இன் முழுமையான பாதுகாப்பு சூழலை (Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID) பயன்படுத்தி MCP அமைப்பின் பாதுகாப்பை மேம்படுத்தவும்
