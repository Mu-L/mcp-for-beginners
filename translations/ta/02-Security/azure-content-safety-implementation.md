# MCP உடன் Azure உள்ளடக்கம் பாதுகாப்பை நடைமுறைப்படுத்துதல்

> **OWASP MCP அபாயம் முகாமை**: [MCP06 - நோக்க வழி முறைத்தல்](https://microsoft.github.io/mcp-azure-security-guide/mcp/mcp06-prompt-injection/)

MCP பாதுகாப்பை prompt injection, கருவி ஆலைப்போக்கு, மற்றும் பிற AI-சார்ந்த பலவீனங்களுக்காக வலுப்படுத்த Azure Content Safety ஐ ஒருங்கிணைப்பது மிகவும் பரிந்துரைக்கப்படுகிறது. இந்த நடைமுறை வழிகாட்டி [MCP பாதுகாப்பு மாநாடு பணிமனை (Sherpa)](https://azure-samples.github.io/sherpa/) முகாம் 3: I/O பாதுகாப்புடன் ஒத்துழைக்கிறது.

## MCP சேவையகத்துடன் ஒருங்கிணைப்பு

Azure Content Safety ஐ உங்கள் MCP சேவையகத்துடன் ஒருங்கிணைக்க, உங்கள் கோரிக்கை செயலாக்க குழாயில் content safety வடிகாட்டியை இடைநிலை மென்பொருளாகச் சேர்க்கவும்:

1. சேவையகத்திற்குள் துவக்கத்தின் போது வடிகாட்டியை ஆரம்பிக்கவும்
2. செயலாக்கத்திற்கு முன் அனைத்து வருகிற கருவி கோரிக்கைகளையும் சரிபார்க்கவும்
3. வாடிக்கையாளர்களுக்கு திருப்பும் முன் அனைத்து வெளியும் பதில்களையும் பரிசோதிக்கவும்
4. பாதுகாப்பு மீறல்களைக் பதிவு செய்யவும் மற்றும் எச்சரிக்கவும்
5. தோல்வியடைந்த உள்ளடக்கம் பாதுகாப்பு பரிசோதனைகளுக்கான பொருத்தமான பிழை கையாளுதலை நடைமுறைப்படுத்தவும்

இது இடையூறு செயல் தாக்குதல்களுக்கான வலுவான பாதுகாப்பை வழங்குகிறது:
- prompt injection தாக்குதல்கள்
- கருவி ஆலைப்போக்கு முயற்சிகள்
- தீங்கான உள்ளீடுகள் மூலம் தரவு அகற்றல்
- தீங்கான உள்ளடக்கம் உருவாக்குதல்

## Azure Content Safety ஒருங்கிணைப்புக்கான சிறந்த நடைமுறைகள்

1. **அனுவாய் தடுப்புப் பட்டியல்கள்**: MCP இடையூறு மாதிரிகளுக்கான தனிப்பயன் தடுப்பு பட்டியல்களை உருவாக்கவும்
2. **தீவிரத்தன்மை சீரமைப்பு**: உங்கள் குறிப்பிட்ட பயன்பாடு மற்றும் அபாய பொறுத்துதலின் அடிப்படையில் தீவிரத்தன்மை எல்லைகளைக் கட்டுப்படுத்தவும்
3. **பரபரப்பான பாதுகாப்பு**: அனைத்து உள்ளீடுகளுக்கும் மற்றும் வெளிப்பாடுகளுக்கும் content safety பரிசோதனைகளை அமல்படுத்தவும்
4. **செயல்திறன் மேம்பாடு**: முறையே content safety பரிசோதனைகளுக்கு மெமரி சேமிப்பை பரிசீலனை செய்யவும்
5. **பின்வாய்ப்புகள் நடைமுறைப்படுத்தல்**: content safety சேவைகள் கிடைக்காத போது தெளிவான பின்வாய்ப்பு நடத்தை வரையறுக்கவும்
6. **பயனாளர் கருத்து**: பாதுகாப்பு கவலைக்காக உள்ளடக்கம் தடைகளுக்குப் பிறகு பயனாளர்களுக்கு தெளிவான கருத்து வழங்கவும்
7. **தொடர்ந்து மேம்படுத்தல்**: உருவாகும் அபாயங்களை அடிப்படையாகக் கொண்டு தடுப்பு பட்டியல்கள் மற்றும் மாதிரிகள் முறுகிய முறையில் புதுப்பிக்கவும்
