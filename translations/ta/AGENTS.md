# AGENTS.md

## Project Overview

**MCP ஆரம்பத்திற்கானவர்** என்பது மாடல் கன்டெக்ஸ்ட் புரோட்டோக்காலும் (MCP) கற்றுக்கொள்ளும் திறந்த மூல கல்வி பாடத்திட்டமாகும் - செயற்கை நுண்ணறிவு மாதிரிகள் மற்றும் கிளையண்ட் செயலிகளுக்கு இடையிலான தொடர்புகளுக்கான ஒரு தரநிலையான கட்டமைப்பாகும். இந்த ரெப்போசிடோரியில் பல நிரலாக்க மொழிகளில் செய்முறை கோட் உதாரணங்கள் உள்ள விரிவான கற்றல் பொருட்கள் வழங்கப்படுகின்றன.

### Key Technologies

- **நிரலாக்க மொழிகள்**: C#, Java, JavaScript, TypeScript, Python, Rust
- **ஃபிரேம்வொர்க் மற்றும் SDKகள்**: 
  - MCP SDK (`@modelcontextprotocol/sdk`)
  - Spring Boot (Java)
  - FastMCP (Python)
  - LangChain4j (Java)
- **தரவுத்தொகுதிகள்**: pgvector நீட்சியுடன் PostgreSQL
- **கிளவுட் பிளாட்ஃபார்ம்கள்**: Azure (Container Apps, OpenAI, Content Safety, Application Insights)
- **கட்டுமான கருவிகள்**: npm, Maven, pip, Cargo
- **விவரக்குறிப்பு**: பல மொழி தானாகமாற்றத்துடன் Markdown (48+ மொழிகள்)

### Architecture

- **11 அடிப்படை தொகுதிகள் (00-11)**: அடிப்படைகளிலிருந்து முன்னேற்றமான தலைப்புகளுக்கான தொடர்ச்சியான கற்றல் பாதை
- **கையால் செய்யும் ஆய்வுக்கூடங்கள்**: பல மொழிகளில் முழுமையான தீர்வு கோடு உடன் நடைமுறை பயிற்சிகள்
- **மாதிரி திட்டங்கள்**: செயல்பாடுள்ள MCP சர்வர் மற்றும் கிளையண்ட் செயலாக்கங்கள்
- **மாற்று அமைப்பு**: பல மொழி ஆதரவு கொண்ட GitHub செயல்திறன் தானாக இயக்கப்படும்
- **பட ஆட்கள்**: மையப்படம் அடைவு மற்றும் மொழிபெயர்க்கப்பட்ட பதிப்புகள்

## Setup Commands

இது mainly documentation க்கான ரெப்போசிடோரியாகும். பெரும்பான்மையான அமைப்புகள் தனிப்பட்ட மாதிரி திட்டங்கள் மற்றும் ஆய்வுக்கூடங்களில் நேரடியாக நடக்கும்.

```bash
# கிடங்கத்தை நகலெடுக்கவும்
git clone https://github.com/microsoft/mcp-for-beginners.git
cd mcp-for-beginners
```

### மாதிரி திட்டங்களுடன் வேலை செய்வது

மாதிரி திட்டங்கள் இங்கு உள்ளன:
- `03-GettingStarted/samples/` - மொழியின்படி உதாரணங்கள்
- `03-GettingStarted/01-first-server/solution/` - முதல் சர்வர் செயலாக்கங்கள்
- `03-GettingStarted/02-client/solution/` - கிளையண்ட் செயலாக்கங்கள்
- `11-MCPServerHandsOnLabs/` - விரிவான தரவுத்தொகுதி ஒருமையாக்க ஆய்வுக்கூடங்கள்

ஒவ்வொரு மாதிரி திட்டத்திலும் தனிப்பட்ட அமைப்பு வழிகாட்டுதல்கள் உள்ளன:

#### TypeScript/JavaScript திட்டங்கள்
```bash
cd <project-directory>
npm install
npm start
```

#### Python திட்டங்கள்
```bash
cd <project-directory>
pip install -r requirements.txt
# அல்லது
pip install -e .
python main.py
```

#### Java திட்டங்கள்
```bash
cd <project-directory>
mvn clean install
mvn spring-boot:run
```

## Development Workflow

### Documentation Structure

- **தொகுதிகள் 00-11**: அடிப்படை பாடத்திட்ட உள்ளடக்கம் தொடர்ச்சியாக
- **translations/**: மொழி-சிறப்பு பதிப்புகள் (தானாக உருவாக்கப்பட்டவை, நேரடியாக திருத்தப்படக்கூடாது)
- **translated_images/**: உள்ளூர் மொழிபெயர்க்கப்பட்ட படங்கள் (தானாக உருவாக்கப்பட்டவை)
- **images/**: மூல படங்கள் மற்றும் வரைபடங்கள்

### Making Documentation Changes

1. மூல தொகுதி அடைவு (00-11) இல் உள்ள ஆங்கில markdown கோப்புகளை மட்டும் திருத்தவும்
2. தேவையான போது `images/` அடைவில் படங்களை புதுப்பிக்கவும்
3. co-op-translator GitHub செயல்திறன் தானாக மொழிபெயர்ப்புகளை உருவாக்கும்
4. பிரதான கிளையில் புஷ் செய்யும் போதே மொழிபெயர்ப்புகள் புதுப்பிக்கப்படும்

### Working with Translations

- **தானாக முறைமையான மொழிபெயர்ப்பு**: GitHub செயல்திறன் அனைத்து மொழிபெயர்ப்புகளையும் கையாள்கிறது
- **translations/** அடைவில் உள்ள கோப்புகளை கைமுறை திருத்தக்கூடாது
- மொழிபெயர்ப்பு மெட்டாடேட்டா ஒவ்வொரு கோப்பிலும் உள்ளடக்கம்
- ஆதரவு மொழிகள்: 48+ மொழிகள், அரபு, சீன, பிரெஞ்சு, ஜெர்மன், ஹிந்தி, ஜப்பானிஸ், கொரியன், போர்த்துகீசியன், ரஷ்யன், ஸ்பானிஷ் மற்றும் பல.

## Testing Instructions

### Documentation Validation

இது documentation ரெப்போசிடோர் என்பதால், சோதனை கவனம் :

1. **இணைப்பு சரிபார்ப்பு**: உள்ளமைந்த அனைத்து இணைப்புகளும் வேலை செய்கிறதா என உறுதிப்படுத்தவும்
```bash
# உடைந்த மார்க் பதிவியல் இணைப்புகளை பரிசோதிக்கவும்
find . -name "*.md" -type f | xargs grep -n "\[.*\](../../.*)"
```

2. **கோடு உதாரணச் சோதனை**: கோடு உதாரணங்கள் தொகுத்து/நடவடிக்கையாக இயங்குகிறதா பார்க்கவும்
```bash
# குறிப்பிட்ட மாதிரிக்கு செல்லவும் அதன் சோதனைகளை இயக்கவும்
cd 03-GettingStarted/samples/typescript
npm install && npm test
```

3. **Markdown Linting**: வடிவமைப்பு இடஒழுங்குகள் சரிபார்க்கவும்
```bash
# தேவையானபோது markdownlint ஐ பயன்படுத்தவும்
npx markdownlint-cli2 "**/*.md" "#node_modules"
```

## Code Style Guidelines

### Documentation Style

- தெளிவான, ஆரம்பத்திற்கான மொழியை பயன்படுத்துங்கள்
- பல மொழிகளில் கோடு உதாரணங்களை சேர்க்கவும்
- markdown சிறந்த நடைமுறைகளை பின்பற்றவும்:
  - ATX வகை தலைப்புக்கள் (`#` சிந்னிக்ஸ்)
  - மொழி குறிப்பிட்ட fenced code blocks
  - படங்களுக்கு விளக்கமான alt உரை சேர்க்கவும்
  - வரி நீளங்கள் பொருத்தமாக வைத்திருங்கள் (கடுமையான வரம்பு இல்லை, ஆனால் பொருத்தமானது)

### Code Sample Style

#### TypeScript/JavaScript
- ES modules (`import`/`export`) பயன்படுத்தவும்
- TypeScript கடுமையான முறை பின்பற்றவும்
- வகை விளக்கங்களை சேர்க்கவும்
- ES2022 இலக்கு

#### Python
- PEP 8 பாணி வழிமுறைகளை பின்பற்றவும்
- வகை குறிப்புகளைப் பயன்படுத்தவும்
- செயல்கள் மற்றும் வகுப்புகளுக்கான டாக்ஸ்டிரிங்குகள் சேர்க்கவும்
- ఆధুনிக Python அம்சங்கள் பயன்படுத்தவும் (3.8+)

#### Java
- Spring Boot நடைமுறை பின்பற்றவும்
- Java 21 அம்சங்கள் பயன்படுத்தவும்
- பொதுவான Maven திட்ட அமைப்பு பின்பற்றவும்
- Javadoc கருத்துக்கள் சேர்க்கவும்

## Build and Deployment

### Documentation Deployment

இந்த ரெப்போசிடோர் GitHub Pages அல்லது அதிரவை போன்றவற்றைக் கண்காணிக்கிறது (இயக்கப்படுகிறது என்றால்). முக்கிய கிளையில் மாற்றங்கள்:

1. மொழிபெயர்ப்பு வேலைநெறி (`.github/workflows/co-op-translator.yml`)
2. அனைத்து ஆங்கில markdown கோப்புகளின் தானாக மொழிபெயர்ப்பு
3. தேவையான படங்கள் உள்ளூர்மொழிக்கு மாற்றம்

### No Build Process Required

இந்த ரெப்போசிடொரி பெரும்பாலும் markdown விவரக்குறிப்புகளை கொண்டுள்ளது. அடிப்படை பாடத்திட்டம் எதுவும் தொகுப்பு அல்லது கட்டுமான நிலை தேவை இல்லை.

### Sample Project Deployment

தனிப்பட்ட மாதிரி திட்டங்களுக்கு கையாளும் வழிகாட்டல்கள் இருக்கலாம்:
- MCP சர்வர் இடைமுகம் `03-GettingStarted/09-deployment/` ஐ பாருங்கள்

## Contributing Guidelines

### Pull Request Process

1. **ரெப்போசிடோர் Fork செய்து, உங்கள் Fork-ஐ Clone செய்யவும்**
2. **ஒரு கிளை உருவாக்கவும்**: விளக்கமான கிளை பெயர்கள் வைுங்கள் (எ.கா., `fix/typo-module-3`, `add/python-example`)
3. **மாற்றங்கள் செய்யவும்**: ஆங்கில markdown கோப்புகளை மட்டும் திருத்தவும் (மொழிபெயர்ப்புகளை அல்ல)
4. **உள்ளகமாக சோதனை செய்க**: markdown சரி முறையில் காண்பிக்கிறதா என உறுதிசெய்க
5. **PR சமர்ப்பிக்கவும்**: தெளிவான தலைப்புகள் மற்றும் விளக்கங்களுடன் PR ஐ அனுப்பவும்
6. **CLA கையொப்பமிடவும்**: Microsoft Contributor License Agreement க்கு ஒப்புதல் தரவும் தேவையானபோது

### PR Title Format

விளக்கமான தலைப்புகளைப் பயன்படுத்தவும்:
- `[Module XX] சுருக்கமான விளக்கம்` (தொகுதி-சொந்த மாற்றங்களுக்கு)
- `[Samples] விளக்கம்` (மாதிரி கோடு மாற்றங்களுக்கு)
- `[Docs] விளக்கம்` (பொதுவான documentation புதுப்பிப்புகளுக்கு)

### What to Contribute

- விவரக்குறிப்புகள் அல்லது கோடு உதாரணங்களில் பிழை திருத்தங்கள்
- கூடுதல் மொழிகளில் புதிய கோடு உதாரணங்கள்
- உள்ளடக்க தெளிவு மற்றும் மேம்பாடுகள்
- புதிய வழக்கு ஆய்வுகள் அல்லது நடைமுறை உதாரணங்கள்
- தெளிவற்ற அல்லது தவறான உள்ளடக்கங்களுக்கு பிழை அறிக்கைகள்

### What NOT to Do

- `translations/` அடைவில் உள்ள கோப்புகளை நேரடியாக திருத்த வேண்டாம்
- `translated_images/` அடைவை மாற்ற வேண்டாம்
- விரிவான பைனரி கோப்புகளை சேர்க்க வேண்டாம் முன் கலந்துரையாடல்
- மொழிபெயர்ப்பு வேலைநெறி கோப்புகளை ஒத்துழைப்பு இல்லாமல் மாற்ற வேண்டாம்

### Learning Path

சிறந்த கற்றல் ஒன்றியத்திற்காக தொகுதிகள் தொடர்ச்சியாக பின்பற்றவும் (00-11):
1. **00-02**: அடிப்படைகள் (அறிமுகம், முக்கியக் கருத்துக்கள், பாதுகாப்பு)
2. **03**: கையால் செய்யும் நடைமுறைகளைத் தொடங்கல்
3. **04-05**: நடைமுறை செயலாக்கமும் முன்னேற்ற தலைப்புகளும்
4. **06-10**: சமூகம், சிறந்த நடைமுறைகள் மற்றும் அத்தியாவசிய பயன்பாடுகள்
5. **11**: விரிவான தரவுத்தொகுதியின் ஒருமையாக்க செயலிகள் (13 தொடர்ச்சியான ஆய்வுக்கூடங்கள்)

### Common Troubleshooting

**Q: என் PR மொழிபெயர்ப்புச் சோதனையில் தோல்வியடைந்தது**  
A: மூல தொகுதியில் உள்ள ஆங்கில markdown கோப்புகளை மட்டும் திருத்தியுள்ளீர்கள் என்பதை உறுதி செய்யவும், மொழிபெயர்க்கப்பட்ட பதிப்புகளை அல்ல.

**Q: புதிய மொழியை எப்படி சேர்க்கிறேன்?**  
A: மொழியானது co-op-translator வேலைநெறியால் கையாளப்படுகிறது. புதிய மொழிகளுக்கான விவாதத்திற்கு ஒரு issue திறக்கவும்.

**Q: கோடு உதாரணங்கள் வேலை செய்யவில்லை**  
A: குறிப்பிட்ட மாதிரியின் README இல் உள்ள அமைப்பு வழிகாட்டலை பின்பற்றியுள்ளீர்கள் என்பதை உறுதி செய்யவும். தேவையான சார்புகள் சரியாக நிறுவப்பட்டுள்ளன என சரிபார்க்கவும்.

**Q: படங்கள் காணப்படவில்லை**  
A: பட பாதைகள் சுட்டெண்னு நிலையானவையாகவும் முன்னெழுபவைபோல இருப்பதாகவும் பரிசோதிக்கவும். படங்கள் `images/` அல்லது `translated_images/` அடைவுகளில் இருக்க வேண்டும்.
