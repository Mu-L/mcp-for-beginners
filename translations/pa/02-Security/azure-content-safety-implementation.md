# MCP ਨਾਲ Azure ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਲਾਗੂ ਕਰਨਾ

> **OWASP MCP ਖਤਰਾ ਪਤਾ ਲਾਇਆ ਗਿਆ**: [MCP06 - ਮਨਸ਼ਾ ਪ੍ਰਵਾਹ ਛਲ](https://microsoft.github.io/mcp-azure-security-guide/mcp/mcp06-prompt-injection/)

ਪ੍ਰੰਪਟ ਇੰਜੈਕਸ਼ਨ, ਟੂਲ ਪੌਇਜ਼ਨਿੰਗ ਅਤੇ ਹੋਰ AI-ਖਾਸ ਖਤਰਿਆਂ ਤੋਂ MCP ਸੁਰੱਖਿਆ ਨੂੰ ਮਜ਼ਬੂਤ ਕਰਨ ਲਈ, Azure ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਨੂੰ ਸਮੇਤਣਾ ਬਹੁਤ ਹੀ ਸਿਫਾਰਸ਼ੀ ਹੈ। ਇਹ ਲਾਗੂ ਕਰਨ ਦੀ ਮਾਰਗਦਰਸ਼ਿਕਾ [MCP ਸੁਰੱਖਿਆ ਸਮੀਟ ਵਰਕਸ਼ਾਪ (ਸ਼ੇਰਪਾ)](https://azure-samples.github.io/sherpa/) Camp 3: I/O Security ਨਾਲ ਮਿਲਦੀ ਹੈ।

## MCP ਸਰਵਰ ਨਾਲ ਇੰਟੀਗ੍ਰੇਸ਼ਨ

ਆਪਣੇ MCP ਸਰਵਰ ਨਾਲ Azure ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਨੂੰ ਜੋੜਨ ਲਈ, ਆਪਣੇ ਰਿਕਵੇਸਟ ਪ੍ਰੋਸੈਸਿੰਗ ਪਾਈਪਲਾਈਨ ਵਿੱਚ ਮਿਡਲਵੇਅਰ ਵਜੋਂ ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਫਿਲਟਰ ਸ਼ਾਮਲ ਕਰੋ:

1. ਸਰਵਰ ਸ਼ੁਰੂਆਤ ਵੇਲੇ ਫਿਲਟਰ ਨੂੰ ਇਨਿਸ਼ੀਅਲਾਈਜ਼ ਕਰੋ  
2. ਪ੍ਰੋਸੈਸਿੰਗ ਤੋਂ ਪਹਿਲਾਂ ਸਾਰੇ ਆ ਰਹੇ ਟੂਲ ਰਿਕਵੇਸਟਾਂ ਨੂੰ ਵੈਧ ਕਰੋ  
3. ਕਲਾਇੰਟਾਂ ਨੂੰ ਵਾਪਸ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ ਸਾਰੇ ਜਾ ਰਹੇ ਜਵਾਬਾਂ ਦੀ ਜਾਂਚ ਕਰੋ  
4. ਸੁਰੱਖਿਆ ਉੱਲੰਘਣਾਂ 'ਤੇ ਲਾਗ ਕਰੋ ਅਤੇ ਅਲਰਟ ਕਰੋ  
5. ਵਿਫਲ ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਜਾਂਚਾਂ ਲਈ ਉਚਿਤ ਏਰਰ ਹੈਂਡਲਿੰਗ ਲਾਗੂ ਕਰੋ  

ਇਸ ਨਾਲ ਮਜ਼ਬੂਤ ਰਖਿਆ ਪ੍ਰਦਾਨ ਹੁੰਦੀ ਹੈ:  
- ਪ੍ਰੰਪਟ ਇੰਜੈਕਸ਼ਨ ਹਮਲੇ  
- ਟੂਲ ਪੌਇਜ਼ਨਿੰਗ ਕੋਸ਼ਿਸ਼ਾਂ  
- ਖ਼ਤਰਨਾਕ ਇਨਪੁੱਟ ਰਾਹੀਂ ਡੇਟਾ ਚੋਰੀ  
- ਨੁਕਸਾਨਦਾਇਕ ਸਮੱਗਰੀ ਦੀ ਤਿਆਰੀ  

## Azure ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਲਈ ਸਭ ਤੋਂ ਵਧੀਆ ਪ੍ਰਕਟਿਸਾਂ

1. **ਕਸਟਮ ਬਲੌਕਲਿਸਟ**: ਖਾਸ ਕਰਕੇ MCP ਇੰਜੈਕਸ਼ਨ ਨਮੂਨਿਆਂ ਲਈ ਕਸਟਮ ਬਲੌਕਲਿਸਟ ਬਣਾਓ  
2. **ਤਿਵਰਤਾ ਸੈਟਿੰਗ**: ਆਪਣੇ ਵਿਸ਼ੇਸ਼ ਉਪਯੋਗ ਮਾਮਲੇ ਅਤੇ ਖਤਰੇ ਨੂੰ ਸਹੇਣਸ਼ੀਲਤਾ ਅਨੁਸਾਰ ਤਿਵਰਤਾ ਦੇ ਮਿਆਰ ਸੁਧਾਰੋ  
3. **ਵਿਆਪਕ ਕਵਰੇਜ**: ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਜਾਂਚਾਂ ਨੂੰ ਸਾਰੇ ਇਨਪੁੱਟ ਅਤੇ ਆਉਟਪੁੱਟ 'ਤੇ ਲਾਗੂ ਕਰੋ  
4. **ਕਾਰਗੁਜ਼ਾਰੀ ਸੁਧਾਰ**: ਦੁਹਰਾਏ ਜਾਣ ਵਾਲੇ ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਜਾਂਚਾਂ ਲਈ ਕੈਸ਼ਿੰਗ ਲਾਗੂ ਕਰਨ ਬਾਰੇ ਸੋਚੋ  
5. **ਬੈਕਅੱਪ ਮਕੈਨਿਜਮ**: ਜਦੋਂ ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਸੇਵਾਵਾਂ ਉਪਲਬਧ ਨਾ ਹੋਣ, ਤਾਂ ਸਪਸ਼ਟ ਬੈਕਅੱਪ ਬਿਹੈਵੀਅਰ ਪਰਿਭਾਸ਼ਿਤ ਕਰੋ  
6. **ਯੂਜ਼ਰ ਫੀਡਬੈਕ**: ਜਦੋਂ ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਕਾਰਨਾਂ ਕਰਕੇ ਬਲੌਕ ਹੋ ਜਾਏ, ਤਾਂ ਯੂਜ਼ਰਾਂ ਨੂੰ ਸਪਸ਼ਟ ਫੀਡਬੈਕ ਦਿਓ  
7. **ਲਗਾਤਾਰ ਸੁਧਾਰ**: ਨਵੇਂ ਖਤਰਿਆਂ ਦੇ ਆਧਾਰ 'ਤੇ ਬਲੌਕਲਿਸਟ ਅਤੇ ਨਮੂਨਿਆਂ ਨੂੰ ਨਿਯਮਿਤ ਅਪਡੇਟ ਕਰੋ
