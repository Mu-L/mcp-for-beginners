# MCP ਸੁਰੱਖਿਆ: ਏ.ਆਈ. ਸਿਸਟਮਾਂ ਲਈ ਵਿਆਪਕ ਸੁਰੱਖਿਆ

[![MCP Security Best Practices](../../../translated_images/pa/03.175aed6dedae133f.webp)](https://youtu.be/88No8pw706o)

_(ਇਸ ਪਾਠ ਦਾ ਵੀਡੀਓ ਦੇਖਣ ਲਈ ਉੱਪਰ ਦਿੱਤੀ ਤਸਵੀਰ 'ਤੇ ਕਲਿੱਕ ਕਰੋ)_

ਸੁਰੱਖਿਆ ਏ.ਆਈ. ਸਿਸਟਮ ਡਿਜ਼ਾਈਨ ਦਾ ਮੂਲ ਸਤੱਤਾਂ ਹਨ, ਇਸ ਲਈ ਅਸੀਂ ਇਸ ਨੂੰ ਆਪਣੇ ਦੂਜੇ ਹਿੱਸੇ ਵਜੋਂ ਪਹਿਲ ਦਿੱਤੀ ਹੈ। ਇਹ ਮਾਇਕਰੋਸਾਫਟ ਦੇ **ਡਿਜ਼ਾਈਨ ਨਾਲ ਸੁਰੱਖਿਆ** ਸਿਧਾਂਤ ਦੇ ਨਾਲ ਸੰਗਤ ਹੈ, ਜੋ [ਸੁਰੱਖਿਅਤ ਭਵਿੱਖ ਪਹਿਲਕਦਮੀ](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) ਤੋਂ ਹੈ।

ਮਾਡਲ ਸੰਦਰਭ ਪ੍ਰੋਟੋਕੋਲ (MCP) ਏ.ਆਈ.-ਚਾਲਿਤ ਐਪਲੀਕੇਸ਼ਨਜ਼ ਲਈ ਸ਼ਕਤਿਸ਼ਾਲੀ ਨਵੀਆਂ ਸਮਰੱਥਾਵਾਂ ਲਿਆਉਂਦਾ ਹੈ, ਜਦੋਂਕਿ ਇਹ ਵਿਲੱਖਣ ਸੁਰੱਖਿਆ ਚੁਣੌਤੀਆਂ ਪੇਸ਼ ਕਰਦਾ ਹੈ ਜੋ ਪਰੰਪਰੇਗਤ ਸਾਫਟਵੇਅਰ ਖਤਰਿਆਂ ਤੋਂ ਬਾਹਰ ਹਨ। MCP ਸਿਸਟਮ ਪ੍ਰਧਾਨ ਸੁਰੱਖਿਆ ਚਿੰਤਾ (ਸੁਰੱਖਿਅਤ ਕੋਡਿੰਗ, ਘੱਟੋ-ਘੱਟ ਅਧਿਕਾਰ, ਸਪਲਾਈ ਚੇਨ ਸੁਰੱਖਿਆ) ਅਤੇ ਨਵੀਆਂ ਏ.ਆਈ.-ਵਿਸ਼ੇਸ਼ ਧਮਕੀਆਂ ਜਿਵੇਂ ਪ੍ਰਾਟ ਇੰਜੈਕਸ਼ਨ, ਟੂਲ ਜ਼ਹਿਰੀਲਾ ਬਣਾਉਣਾ, ਸੈਸ਼ਨ ਹਾਈਜੈਕਿੰਗ, ਗੁੰਝਲਦਾਰ ਡਿਪਿਊ ਅਟੈਕ, ਟੋਕਨ ਪਾਸਥਰੂ ਖਤਰੇ, ਅਤੇ ਗਤੀਸ਼ੀਲ ਸਮਰੱਥਾ ਸੋਧ ਨੂੰ ਸਾਮ੍ਹਣਾ ਕਰਦੇ ਹਨ।

ਇਸ ਪਾਠ ਵਿੱਚ MCP ਲਾਗੂਆਂ ਵਿੱਚ ਸਭ ਤੋਂ ਮਹੱਤਵਪੂਰਨ ਸੁਰੱਖਿਆ ਖਤਰਿਆਂ ਨੂੰ ਖੰਗਾਲਿਆ ਗਿਆ ਹੈ — ਜਿਨ੍ਹਾਂ ਵਿੱਚ ਪ੍ਰਮਾਣਿਕਤਾ, ਪ੍ਰਾਧਿਕਾਰ, ਵੱਧ ਅਧਿਕਾਰ, ਪੈਰੋਕਸੀ ਦਾ ਪ੍ਰਾਠ ਇੰਜੈਕਸ਼ਨ, ਸੈਸ਼ਨ ਸੁਰੱਖਿਆ, ਗੁੰਝਲਦਾਰ ਡਿਪਿਊ ਸਮੱਸਿਆਵਾਂ, ਟੋਕਨ ਪ੍ਰਬੰਧਨ, ਅਤੇ ਸਪਲਾਈ ਚੇਨ ਖਤਰਿਆਂ ਨੂੰ ਢਾਂਕਿਆ ਗਿਆ ਹੈ। ਤੁਸੀਂ ਐਸੇ ਕਾਰਜ ਕਾਬਲ ਨਿਯੰਤਰਣ ਅਤੇ ਵਧੀਆ ਅਭਿਆਸ ਸਿੱਖੋਗੇ ਜੋ ਇਹਨਾਂ ਖਤਰਿਆਂ ਨੂੰ ਘਟਾਉਂਦੇ ਹਨ ਜਦੋਂ ਕਿ ਮਾਇਕਰੋਸਾਫਟ ਸੁਲੂਸ਼ਨਜ਼ ਜਿਵੇਂ ਕਿ ਪ੍ਰਾਟ ਸ਼ੀਲਡਜ਼, ਏਜ਼ੂਰ ਕੰਟੈਂਟ ਸੇਫਟੀ, ਅਤੇ ਗਿੱਟਹੱਬ ਐਡਵਾਂਸਡ ਸੁਰੱਖਿਆ ਨਾਲ ਤੁਹਾਡੇ MCP ਤਾਇਨਾਤੀ ਨੂੰ ਮਜ਼ਬੂਤ ਬਣਾਉਂਦੇ ਹਨ।

## ਸਿਖਣ ਦੇ ਲੱਖਾਂ

ਇਸ ਪਾਠ ਦੇ ਸਮਾਪਤ 'ਤੇ, ਤੁਸੀਂ ਸਮਰੱਥ ਹੋਵੋਗੇ:

- **MCP-ਵਿਸ਼ੇਸ਼ ਧਮਕੀਆਂ ਦੀ ਪਛਾਣ ਕਰੋ**: MCP ਸਿਸਟਮਾਂ ਵਿੱਚ ਵਿਲੱਖਣ ਸੁਰੱਖਿਆ ਖਤਰਿਆਂ ਦੀ ਪਹਿਚਾਣ ਕਰੋ ਜਿਵੇਂ ਪ੍ਰਾਟ ਇੰਜੈਕਸ਼ਨ, ਟੂਲ ਜ਼ਹਿਰੀਲਾ ਬਣਾਉਣਾ, ਵੱਧ ਅਧਿਕਾਰ, ਸੈਸ਼ਨ ਹਾਈਜੈਕਿੰਗ, ਗੁੰਝਲਦਾਰ ਡਿਪਿਊ ਸਮੱਸਿਆਵਾਂ, ਟੋਕਨ ਪਾਸਥਰੂ ਖਤਰਿਆਂ, ਅਤੇ ਸਪਲਾਈ ਚੇਨ ਖਤਰਿਆਂ
- **ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ ਲਾਗੂ ਕਰੋ**: ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਨਿਯੰਤਰਣ ਜਿਵੇਂ ਕਿ ਮਜ਼ਬੂਤ ਪ੍ਰਮਾਣਿਕਤਾ, ਘੱਟੋ-ਘੱਟ ਅਧਿਕਾਰ ਪਹੁੰਚ, ਸੁਰੱਖਿਅਤ ਟੋਕਨ ਪ੍ਰਬੰਧਨ, ਸੈਸ਼ਨ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ, ਅਤੇ ਸਪਲਾਈ ਚੇਨ ਵੈਰੀਫਿਕੇਸ਼ਨ ਲਾਗੂ ਕਰੋ
- **ਮਾਇਕਰੋਸਾਫਟ ਸੁਰੱਖਿਆ ਸੁਲੂਸ਼ਨ ਲਾਭ ਪ੍ਰਾਪਤ ਕਰੋ**: MCP ਵੋರ್ಕਲੋਡ ਸੁਰੱਖਿਆ ਲਈ ਮਾਇਕਰੋਸਾਫਟ ਪ੍ਰਾਟ ਸ਼ੀਲਡਜ਼, ਏਜ਼ੂਰ ਕੰਟੈਂਟ ਸੇਫਟੀ, ਅਤੇ ਗਿੱਟਹੱਬ ਐਡਵਾਂਸਡ ਸੁਰੱਖਿਆ ਦੀ ਸਮਝ ਅਤੇ ਤੈਨਾਤੀ ਕਰੋ
- **ਟੂਲ ਸੁਰੱਖਿਆ ਦੀ ਪੁਸ਼ਟੀ ਕਰੋ**: ਟੂਲ ਮੈਟਾਡੇਟਾ ਵੈਰੀਫਿਕੇਸ਼ਨ ਮਹੱਤਵ ਪਛਾਣੋ, ਗਤੀਸ਼ੀਲ ਬਦਲਾਵਾਂ ਲਈ ਨਿਗਰਾਨੀ ਕਰੋ, ਅਤੇ ਪੈਰੋਕਸੀ ਪ੍ਰਾਟ ਇੰਜੈਕਸ਼ਨ ਹਮਲਿਆਂ ਵਿਰੁੱਧ ਰੱਖਿਆ ਕਰੋ
- **ਵਧੀਆ ਅਭਿਆਸ ਸਾਂਝੇ ਕਰੋ**: ਇਸਥਾਪਿਤ ਸੁਰੱਖਿਆ ਬੁਨਿਆਦੀਆਂ (ਸੁਰੱਖਿਅਤ ਕੋਡਿੰਗ, ਸਰਵਰ ਸਖ਼ਤੀ, ਜ਼ੀਰੋ ਟਰਸਟ) ਨੂੰ MCP-ਵਿਸ਼ੇਸ਼ ਨਿਯੰਤਰਣ ਨਾਲ ਮਿਲਾ ਕੇ ਵਿਆਪਕ ਸੁਰੱਖਿਆ ਪ੍ਰਦਾਨ ਕਰੋ

# MCP ਸੁਰੱਖਿਆ ਆਰਕੀਟੈਕਚਰ & ਨਿਯੰਤਰਣ

ਆਧੁਨਿਕ MCP ਲਾਗੂਆਂ ਨੂੰ ਪਰੰਪਰਾਗਤ ਸਾਫਟਵੇਅਰ ਸੁਰੱਖਿਆ ਅਤੇ ਏ.ਆਈ.-ਵਿਸ਼ੇਸ਼ ਧਮਕੀਆਂ ਨੂੰ ਪਤਾ ਲਗਾਉਣ ਵਾਲੀਆਂ ਬਹੁ-ਪਹਿਆੜੀ ਸੁਰੱਖਿਆ ਪਹੁੰਚਾਂ ਦੀ ਲੋੜ ਹੁੰਦੀ ਹੈ। ਤੇਜ਼ੀ ਨਾਲ ਵਿਕਸਿਤ ਹੋ ਰਹੀ MCP ਵਿਸ਼ੇਸ਼ਤਾ ਆਪਣੀਆਂ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣਾਂ ਨੂੰ ਮਜ਼ਬੂਤ ਕਰਦੀ ਰਹਿੰਦੀ ਹੈ, ਜੋ ਕਿ ਕਾਰੋਬਾਰੀ ਸੁਰੱਖਿਆ ਆਰਕੀਟੈਕਚਰ ਅਤੇ ਸਥਾਪਿਤ ਵਧੀਆ ਅਭਿਆਸਾਂ ਨਾਲ ਬਿਹਤਰ ਇੰਟੈਗ੍ਰੇਸ਼ਨ ਨੂੰ ਯਕੀਨੀ ਬਣਾਉਂਦੀ ਹੈ।

[Microsoft Digital Defense Report](https://aka.ms/mddr) ਤੋਂ ਖੋਜ ਦਰਸਾਉਂਦੀ ਹੈ ਕਿ **98% ਰਿਪੋਰਟ ਕੀਤੀਆਂ ਭੇਦਭਾਵਰਾਂ ਨੂੰ ਮਜ਼ਬੂਤ ਸੁਰੱਖਿਆ ਸਾਫ਼ ਸੁਥਰੀ ਰੀਤ ਨਾਲ ਰੋਕਿਆ ਜਾ ਸਕਦਾ ਹੈ**। ਸਭ ਤੋਂ ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਸੁਰੱਖਿਆ ਰਣਨੀਤੀ ਮੂਲ ਸੁਰੱਖਿਆ ਅਭਿਆਸਾਂ ਅਤੇ MCP-ਵਿਸ਼ੇਸ਼ ਨਿਯੰਤਰਣਾਂ ਨੂੰ ਮਿਲਾ ਕੇ ਬਣਾਈ ਜਾਂਦੀ ਹੈ — ਸਾਬਤ ਹੋ ਚੁਕੇ ਸੁਰੱਖਿਆ ਮਾਪਦੰਡ ਕੁੱਲ ਸੁਰੱਖਿਆ ਖਤਰੇ ਨੂੰ ਘਟਾਉਣ ਵਿੱਚ ਸਭ ਤੋਂ ਜ਼ਿਆਦਾ ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਬਣੇ ਹੋਏ ਹਨ।

## ਮੌਜੂਦਾ ਸੁਰੱਖਿਆ ਦ੍ਰਿਸ਼

> **ਨੋਟ:** ਇਹ ਜਾਣਕਾਰੀ MCP ਸੁਰੱਖਿਆ ਮਿਆਰਾਂ ਨੂੰ ਦਰਸਾਉਂਦੀ ਹੈ ਜਿਵੇਂ ਕਿ **5 ਫਰਵਰੀ, 2026** ਤੱਕ, ਜੋ ਕਿ **MCP Specification 2025-11-25** ਨਾਲ ਸੰਗਤ ਹੈ। MCP ਪ੍ਰੋਟੋਕੋਲ ਤੇਜ਼ੀ ਨਾਲ ਵਿਕਸਤ ਹੋ ਰਹਾ ਹੈ, ਅਤੇ ਭਵਿੱਖ ਦੇ ਲਾਗੂਆ MCP ਲਈ ਨਵੀਂ ਪ੍ਰਮਾਣਿਕਤਾ ਪੈਟਰਨ ਅਤੇ ਵਧੀਆ ਨਿਯੰਤਰਣ ਪ੍ਰਦਾਨ ਕਰ ਸਕਦੇ ਹਨ। ਹਮੇਸ਼ਾ ਮੌਜੂਦਾ [MCP Specification](https://spec.modelcontextprotocol.io/), [MCP GitHub ਰਿਪੋਜ਼ਟਰੀ](https://github.com/modelcontextprotocol), ਅਤੇ [ਸੁਰੱਖਿਆ ਵਧੀਆ ਅਭਿਆਸ ਦਸਤਾਵੇਜ਼](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) ਦੀ ਜਾਂਚ ਕਰੋ।

## 🏔️ MCP ਸੁਰੱਖਿਆ ਸਮਿੱਟ ਵਰਕਸ਼ਾਪ (ਸ਼ੇਰਪਾ)

**ਹਾਂਡਸ-ਆਨ ਸੁਰੱਖਿਆ ਪ੍ਰਸ਼િક્ષਣ** ਲਈ, ਅਸੀਂ ਬਹੁਤ ਜ਼ੋਰ ਦੇ ਨਾਲ **MCP ਸੁਰੱਖਿਆ ਸਮਿੱਟ ਵਰਕਸ਼ਾਪ** (ਸ਼ੇਰਪਾ) ਦੀ ਸਿਫਾਰਸ਼ ਕਰਦੇ ਹਾਂ - ਮਾਈਕਰੋਸਾਫਟ ਏਜ਼ੂਰ ਵਿੱਚ MCP ਸਰਵਰਾਂ ਨੂੰ ਸੁਰੱਖਿਅਤ ਕਰਨ ਦੀ ਇੱਕ ਵਿਆਪਕ ਦਿਸ਼ਾ ਨਿਰਦੇਸ਼ ਯਾਤਰਾ।

### ਵਰਕਸ਼ਾਪ ਵੇਰਵਾ

[MCP Security Summit Workshop](https://azure-samples.github.io/sherpa/) ਇੱਕ ਪ੍ਰਮਾਣਿਤ "ਜ਼ਰੂਰੀ ਕਮੀ → ਸ਼ੋਧ → ਠੀਕ → ਪੁਸ਼ਟੀ" ਤਰੀਕੇ ਨਾਲ ਪ੍ਰਯੋਗਿਕ ਅਤੇ ਕਾਰਜਕਾਰੀ ਸੁਰੱਖਿਆ ਸਿਖਲਾਈ ਦਿੰਦਾ ਹੈ। ਤੁਸੀਂ:

- **ਤੋੜ ਕੇ ਸਿੱਖੋ**: ਜ਼ਰੂਰਤ ਤੋਂ ਬਾਹਰ ਘੱਟ-ਸੁਰੱਖਿਅਤ ਸਰਵਰਾਂ ਨੂੰ ਨੁਕਸਾਨ ਪਹੁੰਚਾ ਕੇ ਅਨੁਭਵ ਪ੍ਰਾਪਤ ਕਰੋ
- **ਏਜ਼ੂਰ-ਮੂਲ ਸੁਰੱਖਿਆ ਵਰਤੋ**: ਏਜ਼ੂਰ ਏਂਟਰਾ ਆਈ.ਡੀ., ਕੀ ਵੌਲਟ, ਏਪੀਆਈ ਪ੍ਰਬੰਧਨ, ਅਤੇ ਏ.ਆਈ. ਕੰਟੈਂਟ ਸੇਫਟੀ ਦੀ ਵਰਤੋਂ ਕਰੋ
- **ਗਹਿਰੀ ਸੁਰੱਖਿਆ ਅਮਲੀਕਰਨ ਕਰੋ**: ਕੈਂਪਾਂ ਰਾਹੀਂ ਵਧ ਰਹੀਆਂ ਸੁਰੱਖਿਆ ਪਹਿਆੜੀਆਂ ਬਣਾਵੋ
- **OWASP ਮਿਆਰਾਂ ਨੂੰ ਪਾਲਣਾ ਕਰੋ**: ਹਰ ਤਕਨੀਕ [OWASP MCP Azure Security Guide](https://microsoft.github.io/mcp-azure-security-guide/) ਨਾਲ ਨਕਸ਼ਾਬੰਦੀ ਹੈ
- **ਤਿਆਰ ਕੋਡ ਪ੍ਰਾਪਤ ਕਰੋ**: ਚੱਲ ਰਹੀਆਂ, ਪ੍ਰਮਾਣਿਤ ਲਾਗੂਆਂ ਨਾਲ ਜਾਵੋ

### MCP ਪ੍ਰਮਾਣਿਕਤਾ ਦਾ ਵਿਕਾਸ

MCP ਵਿਸ਼ੇਸ਼ਤਾ ਨੇ ਪ੍ਰਮਾਣਿਕਤਾ ਅਤੇ ਪ੍ਰਾਧਿਕਾਰ ਲਈ ਬਹੁਤ ਤਰੱਕੀ ਕੀਤੀ ਹੈ:

- **ਮੂਲ ਤਰੀਕਾ**: ਸ਼ੁਰੂਆਤੀ ਵਿਸ਼ੇਸ਼ਤਾਵਾਂ ਵਿਚ ਵਿਕਾਸਕਾਰਾਂ ਨੂੰ ਕਸਟਮ ਪ੍ਰਮਾਣਿਕਤਾ ਸਰਵਰ ਲਾਗੂ ਕਰਨ ਦੀ ਲੋੜ ਸੀ, MCP ਸਰਵਰ OAuth 2.0 ਪ੍ਰਮਾਣਿਕਤਾ ਸਰਵਰ ਵਜੋਂ ਕੰਮ ਕਰਦੇ ਸੀ ਜੋ ਸਿੱਧਾ ਉਪਭੋਗਤਾ ਪ੍ਰਮਾਣਿਕਤਾ ਸੰਭਾਲਦੇ ਸਨ
- **ਮੌਜੂਦਾ ਮਿਆਰ (2025-11-25)**: ਅਪਡੇਟ ਕੀਤੇ ਵਿਸ਼ੇਸ਼ਤਾ MCP ਸਰਵਰਾਂ ਨੂੰ ਬਾਹਰੀ ਪਹਿਚਾਣ ਪ੍ਰਦਾਤਾਵਾਂ (ਜਿਵੇਂ ਕਿ ਮਾਇਕਰੋਸਾਫਟ ਏਂਟਰਾ ਆਈ.ਡੀ.) ਨੂੰ ਪ੍ਰਮਾਣੀਕਰਨ ਸੌਂਪਣ ਦੀ ਆਗਿਆ ਦਿੰਦੇ ਹਨ, ਸੁਰੱਖਿਆ ਸਥਿਤੀ ਸੁਧਾਰਦੇ ਹਨ ਅਤੇ ਲਾਗੂ ਕਰਨ ਦੀ ਜਟਿਲਤਾ ਘਟਾਉਂਦੇ ਹਨ
- **ਟ੍ਰਾਂਸਪੋਰਟ ਲੇਅਰ ਸੁਰੱਖਿਆ**: ਸਥਾਨਕ (STDIO) ਅਤੇ ਦੂਰੀ ਆਪਣੇ HTTP (Streamable HTTP) ਕਨੈਕਸ਼ਨਾਂ ਲਈ ਢੁਕਵੀਂ ਪ੍ਰਮਾਣਿਕਤਾ ਪੈਟਰਨ ਨਾਲ ਸੁੱਖਿਆ ਦੇ ਢੰਗ ਵਿੱਚ ਵਾਧਾ ਕੀਤਾ ਹੈ

## ਪ੍ਰਮਾਣਿਕਤਾ ਅਤੇ ਪ੍ਰਾਧਿਕਾਰ ਸੁਰੱਖਿਆ

### ਮੌਜੂਦਾ ਸੁਰੱਖਿਆ ਚੁਣੌਤੀਆਂ

ਆਧੁਨਿਕ MCP ਲਾਗੂਆਂ ਨੂੰ ਕਈ ਪ੍ਰਮਾਣਿਕਤਾ ਅਤੇ ਪ੍ਰਾਧਿਕਾਰ ਚੁਣੌਤੀਆਂ ਦਾ ਸਾਹਮਣਾ ਹੈ:

### ਖਤਰਿਆਂ ਅਤੇ ਧਮਕੀ ਵੈਕਟਰ

- **ਪ੍ਰਾਧਿਕਾਰ ਤਰਤੀਬ ਵਿਚ ਗਲਤ ਸਰਵਰ ਚਾਲੂ ਕਰਨਾ**: MCP ਸਰਵਰਾਂ ਵਿੱਚ ਖ਼ਰਾਬ ਪ੍ਰਾਧਿਕਾਰ ਲਾਗੂ ਕਰਨ ਨਾਲ ਸੰਵੇਦਨਸ਼ੀਲ ਡਾਟਾ ਖੁਲਾਸਾ ਅਤੇ ਗਲਤ ਪਹੁੰਚ ਨਿਯੰਤਰਣ ਹੁੰਦੇ ਹਨ
- **OAuth ਟੋਕਨ ਰੁਕਾਵੱਟ**: ਸਥਾਨਕ MCP ਸਰਵਰ ਟੋਕਨ ਦੀ ਚੋਰੀ ਹਮਲਾਵਰਾਂ ਨੂੰ ਸਰਵਰਾਂ ਦਾ ਬੰਦਾ ਬਣਾਕੇ ਡਾਊਨਸਟ੍ਰੀਮ ਸੇਵਾਵਾਂ ਤੱਕ ਪਹੁੰਚ ਦਿੰਦਾ ਹੈ
- **ਟੋਕਨ ਪਾਸਥਰੂ ਖਤਰੇ**: ਗਲਤ ਟੋਕਨ ਹੈਂਡਲਿੰਗ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣਾਂ ਨੂੰ ਬਾਈਪਾਸ ਅਤੇ ਜਵਾਬਦੇਹੀ ਖਾਲੀ ਜਗ੍ਹਾ ਬਣਾਉਂਦਾ ਹੈ
- **ਜ਼ਿਆਦਾ ਅਧਿਕਾਰ**: ਜ਼ਿਆਦਾ ਅਧਿਕਾਰ ਵਾਲੇ MCP ਸਰਵਰ ਘੱਟੋ-ਘੱਟ ਅਧਿਕਾਰ ਸਿਧਾਂਤਾਂ ਦਾ ਉਲੰਘਣ ਕਰਦੇ ਹਨ ਅਤੇ ਹਮਲਿਆਂ ਦੇ ਸਤਹ ਵਧਾਉਂਦੇ ਹਨ

#### ਟੋਕਨ ਪਾਸਥਰੂ: ਇੱਕ ਅਤਿ-ਗੰਭੀਰ ਵਿਰੋਧੀ ਪੈਟਰਨ

**ਟੋਕਨ ਪਾਸਥਰੂ ਵਰਤੋਂ ਸਪਸ਼ਟ ਤੌਰ ਤੇ ਮੌਜੂਦਾ MCP ਪ੍ਰਾਧਿਕਾਰ ਵਿਸ਼ੇਸ਼ਤਾ ਵਿੱਚ ਮਨਾਹੀ ਹੈ**, ਇਸਦੇ ਭਾਰੀ ਸੁਰੱਖਿਆ ਪ੍ਰਭਾਵ ਦੇ ਕਰਕੇ:

##### ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ ਬਾਈਪਾਸਿੰਗ
- MCP ਸਰਵਰ ਅਤੇ ਡਾਊਨਸਟ੍ਰੀਮ API ਅਹੰਕਾਰ-ਨਿਯਤ ਨਿਯੰਤਰਣ ਲਾਗੂ ਕਰਦੇ ਹਨ (ਰੇਟ ਸੀਮਾ, ਬੇਨਤੀ ਵੈਰੀਫਿਕੇਸ਼ਨ, ਟ੍ਰੈਫਿਕ ਮਾਨੀਟਰਿੰਗ) ਜੋ ਢੁਕਵੀਂ ਟੋਕਨ ਵੈਰੀਫਿਕੇਸ਼ਨ 'ਤੇ ਨਿਰਭਰ ਹਨ
- ਸਿੱਧੀ ਕਲਾਇੰਟ-ਟੂ-API ਟੋਕਨ ਵਰਤੋਂ ਇਹਨਾਂ ਜ਼ਰੂਰੀ ਸੁਰੱਖਿਆ ਪਰਦੇ ਨੂੰ ਡਾਗਮਾਗ ਕਰਦੀ ਹੈ

##### ਜਵਾਬਦੇਹੀ ਅਤੇ ਆਡਿਟ ਚੁਣੌਤੀਆਂ  
- MCP ਸਰਵਰ upstream ਜਾਰੀ ਕੀਤੇ ਟੋਕਨਾਂ ਵਾਲੇ ਕਲਾਇੰਟਾਂ ਵਿਚ ਫਰਕ ਨਹੀਂ ਪਾ ਸਕਦੇ, ਜਿਸ ਤੋਂ ਆਡਿਟ ਟਰੇਲ ਟੁੱਟ ਜਾਂਦਾ ਹੈ
- ਡਾਊਨਸਟ੍ਰੀਮ ਸਰੋਤ ਸਰਵਰ ਲੌਗਾਂ ਵਿੱਚ MCP ਸਰਵਰ ਮੱਧਸਥਤਾਂ ਦੀ ਥਾਂ ਗਲਤ ਬੇਨਤੀਆਂ ਦਾ ਸਰੋਤ ਦਿਖਾਇਆ ਜਾਂਦਾ ਹੈ
- ਘਟਨਾ ਜਾਂਚ ਅਤੇ ਅਨੁਕੂਲਤਾ ਆਡਿਟ ਨੂੰ ਕਾਫ਼ੀ ਮੁਸ਼ਕਲ ਕਰਦਾ ਹੈ

##### ਡਾਟਾ ਚੋਰੀ ਖਤਰੇ
- ਬਿਨਾਂ ਵੇਰੀਫਾਈ ਕੀਤੀ ਟੋਕਨ ਦਾਅਵਿਆਂ ਨਾਲ ਚੋਰੀ ਹੋਏ ਟੋਕਨਾਂ ਵਾਲੇ ਮਲਿਸ਼ੀਅਸ ਇਕਾਈਆਂ MCP ਸਰਵਰਾਂ ਨੂੰ ਡਾਟਾ ਚੋਰੀ ਲਈ ਪ੍ਰਾਕਸੀ ਵਜੋਂ ਵਰਤ ਸਕਦੇ ਹਨ
- ਭਰੋਸਾ ਸੀਮਾ ਦੀਆਂ ਉਲੰਘਣਾਂ ਅਣਅਧਿਕਾਰਤ ਪਹੁੰਚ ਪੈਟਰਨ ਬਣਾਉਂਦੀਆਂ ਹਨ ਜੋ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣਾਂ ਨੂੰ ਬਾਈਪਾਸ ਕਰਦੀਆਂ ਹਨ

##### ਬਹੁ-ਸੇਵਾ ਹਮਲੇ ਦੇ ਰਸਤੇ  
- ਕਈ ਸੇਵਾਵਾਂ ਦੁਆਰਾ ਸਵੀਕਾਰ ਕੀਤੇ ਗਏ ਕਮਜ਼ੋਰ ਟੋਕਨ ਸਬੰਧਿਤ ਸਿਸਟਮਾਂ ਵਿੱਚ ਲੈਟਰਲ ਮੂਵਮੈਂਟ ਨੂੰ ਯੋਗ ਬਣਾਉਂਦੇ ਹਨ
- ਟੋਕਨ ਮੂਲਾਂ ਦੀ ਪੁਸ਼ਟੀ ਨਾ ਹੋਣ 'ਤੇ ਸੇਵਾ ਦਰਮਿਆਨ ਭਰੋਸਾ ਸੰਦਰਭਾਂ ਦੀ ਉਲੰਘਣਾ ਹੋ ਸਕਦੀ ਹੈ

### ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ ਅਤੇ ਨਸ਼ਟਕਾਰਤਾ

**ਅਹੰਕਾਰਪੂਰਨ ਸੁਰੱਖਿਆ ਲੋੜਾਂ:**

> **ਜ਼ਰੂਰੀ**: MCP ਸਰਵਰ **ਕਦੇ ਵੀ** ਉਹ ਟੋਕਨ ਸਵੀਕਾਰ ਨਹੀਂ ਕਰਨ ਜੋ ਸਪਸ਼ਟ ਤੌਰ ਤੇ MCP ਸਰਵਰ ਲਈ ਜਾਰੀ ਨਹੀਂ ਹੋਏ

#### ਪ੍ਰਮਾਣਿਕਤਾ ਅਤੇ ਪ੍ਰਾਧਿਕਾਰ ਨਿਯੰਤਰਣ

- **ਸਖ਼ਤ ਪ੍ਰਾਧਿਕਾਰ ਸਮੀਖਿਆ**: MCP ਸਰਵਰ ਪ੍ਰਾਧਿਕਾਰ ਲਾਜਿਕ ਦੀ ਵਿਆਪਕ ਆਡਿਟ ਕਰੋ ਤਾਂ ਜੋ ਸਿਰਫ ਮੰਜ਼ੂਰਸ਼ੁਦਾ ਉਪਯੋਗਕਰਤਾ ਅਤੇ ਕਲਾਇੰਟ ਸਮਕਾਲੀ ਸੰਸਾਧਨਾਂ ਤੱਕ ਪਹੁੰਚ ਪਾ ਸਕਣ
  - **ਲਾਗੂ ਕਰਨ ਦੀ ਮਦਦ**: [Azure API Management as Authentication Gateway for MCP Servers](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **ਪਹਿਚਾਣ ਇੰਟੈਗ੍ਰੇਸ਼ਨ**: [Using Microsoft Entra ID for MCP Server Authentication](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **ਸੁਰੱਖਿਅਤ ਟੋਕਨ ਪ੍ਰਬੰਧਨ**: [ਮਾਇਕਰੋਸਾਫਟ ਦੇ ਟੋਕਨ ਵੈਰੀਫਿਕੇਸ਼ਨ ਅਤੇ ਲਾਈਫਸਾਈਕਲ ਵਧੀਆ ਅਭਿਆਸ](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens) ਲਾਗੂ ਕਰੋ
  - ਟੋਕਨ ਦਰਸ਼ਕ ਦਾਅਵਿਆਂ ਨੂੰ MCP ਸਰਵਰ ਪਹਿਚਾਣ ਨਾਲ ਮੇਲ ਖਾਣਾ ਯਕੀਨੀ ਬਣਾਓ
  - ਟੋਕਨ ਘੁੰਮਾਅ ਅਤੇ ਮਿਆਦ ਪੁੱਗਣ ਦੀਆਂ ਨੀਤੀਆਂ ਲਾਗੂ ਕਰੋ
  - ਟੋਕਨ ਰੀਪਲੇ ਹਮਲਿਆਂ ਅਤੇ ਅਣਅਧਿਕਾਰਤ ਵਰਤੋਂ ਰੋਕੋ

- **ਸੁਰੱਖਿਅਤ ਟੋਕਨ ਸਟੋਰੇਜ**: ਸਟੋਰੇਜ ਨੂੰ ਰਿਹਤ ਅਤੇ ਟ੍ਰਾਂਜ਼ਿਟ ਦੋਹਾਂ ਵਿੱਚ ਇਨਕ੍ਰਿਪਸ਼ਨ ਨਾਲ ਸੁਰੱਖਿਅਤ ਕਰੋ
  - **ਵਧੀਆ ਅਭਿਆਸ**: [Secure Token Storage and Encryption Guidelines](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### ਪਹੁੰਚ ਨਿਯੰਤਰਣ ਲਾਗੂ ਕਰਨਾ

- **ਘੱਟੋ-ਘੱਟ ਅਧਿਕਾਰ ਦਾ ਸਿਧਾਂਤ**: MCP ਸਰਵਰਾਂ ਨੂੰ ਸਿਰਫ ਅਵਸ਼ਕਤ ਕਾਰਜਾਂ ਲਈ ਘੱਟੋ-ਘੱਟ ਅਧਿਕਾਰ ਦਿਓ
  - ਅਧਿਕਾਰ ਸਮੀਖਿਆ ਅਤੇ ਅਪਡੇਟ ਨਿਯਮਤ ਤੌਰ 'ਤੇ ਕਰੋ ਤਾਕਿ ਅਧਿਕਾਰ ਹੋਰ ਨਾ ਵਧਣ
  - **ਮਾਇਕਰੋਸਾਫਟ ਦਸਤਾਵੇਜ਼**: [ਘੱਟੋ-ਘੱਟ ਅਧਿਕਾਰ ਸੁਰੱਖਿਅਤ ਪਹੁੰਚ](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **ਰੋਲ-ਅਧਾਰਤ ਪਹੁੰਚ ਨਿਯੰਤਰਣ (RBAC)**: ਸੁਖੜੇ ਅਤੇ ਵਿਆਪਕ ਰੋਲ ਅਸਾਈਨਮੈਂਟ ਲਾਗੂ ਕਰੋ
  - ਖਾਸ ਸਰੋਤਾਂ ਅਤੇ ਕਾਰਜਾਂ ਲਈ ਰੋਲਾਂ ਨੂੰ ਘੇਰ ਕੇ ਨਿਰਧਾਰਿਤ ਕਰੋ
  - ਵਰਤੋਂ ਵਿੱਚ ਨਾ ਆਉਣ ਵਾਲੇ ਜਾਂ ਅਣਜਰੂਰੀ ਅਧਿਕਾਰਾਂ ਤੋਂ ਬਚੋ ਜੋ ਹਮਲੇ ਦੇ ਸਤਹ ਵਧਾਉਂਦੇ ਹਨ

- **ਨਿਰੰਤਰ ਅਧਿਕਾਰ ਨਿਗਰਾਨੀ**: ਪਹੁੰਚ ਆਡੀਟ ਅਤੇ ਨਿਗਰਾਨੀ ਲਗਾਤਾਰ ਕਰੋ
  - ਨੁਕਸਾਂ ਜਾਂ ਵਿਅਤਕਿਛੜੀਆਂ ਪਹੁੰਚ ਪੈਟਰਨ ਲਈ ਮਾਨੀਟਰਿੰਗ ਕਰੋ
  - ਵੱਧ ਜਾਂ ਅਣਵਰਤੀ ਅਧਿਕਾਰ ਤੁਰੰਤ ਠੀਕ ਕਰੋ

## ਏ.ਆਈ.-ਵਿਸ਼ੇਸ਼ ਸੁਰੱਖਿਆ ਧਮਕੀਆਂ

### ਪ੍ਰਾਟ ਇੰਜੈਕਸ਼ਨ ਅਤੇ ਟੂਲ ਮੈਨਿਪੂਲੇਸ਼ਨ ਹਮਲੇ

ਆਧੁਨਿਕ MCP ਲਾਗੂਆਂ ਵਿੱਚ ਸੁਖੜੇ ਏ.ਆਈ.-ਵਿਸ਼ੇਸ਼ ਹਮਲਾਕਾਰੀ ਰਸਤੇ ਹੁੰਦੇ ਹਨ ਜੋ ਪਰੰਪਰੇਗਤ ਸੁਰੱਖਿਆ ਉਪਾਅ ਪੂਰੀ ਤਰ੍ਹਾਂ ਸਥਾਪਿਤ ਨਹੀਂ ਕਰ ਸਕਦੇ:

#### **ਪੈਰੋਕਸੀ ਪ੍ਰਾਟ ਇੰਜੈਕਸ਼ਨ (ਕ੍ਰਾਸ-ਡੋਮੀਨ ਪ੍ਰਾਟ ਇੰਜੈਕਸ਼ਨ)**

**ਪੈਰੋਕਸੀ ਪ੍ਰਾਟ ਇੰਜੈਕਸ਼ਨ** MCP ਨਿਯੰਤਰਿਤ ਏ.ਆਈ. ਸਿਸਟਮ ਵਿੱਚ ਸਭ ਤੋਂ ਗੰਭੀਰ ਖਤਰਾ ਹੈ। ਹਮਲਾਕਾਰ ਬਾਹਰੀ ਸਮੱਗਰੀ ਜਿਵੇਂ ਦਸਤਾਵੇਜ਼, ਵੈੱਬ ਪੰਨੇ, ਈਮੇਲ ਜਾਂ ਡੇਟਾ ਸਰੋਤਾਂ ਵਿਚ ਖ਼ਤਰਨਾਕ ਹੁਕਮ ਛੁਪਾ ਦਿੰਦੇ ਹਨ ਜਿਹੜੇ ਫਿਰ ਏ.ਆਈ. ਸਿਸਟਮ ਅਸਲ ਹੁਕਮ ਸਮਝ ਕੇ ਪ੍ਰਕਿਰਿਆ ਕਰਦੇ ਹਨ।

**ਹਮਲੇ ਦੇ ਮਾਮਲੇ:**
- **ਦਸਤਾਵੇਜ਼-ਆਧਾਰਿਤ ਇੰਜੈਕਸ਼ਨ**: ਪ੍ਰਕਿਰਿਆ ਵਾਲੇ ਦਸਤਾਵੇਜ਼ਾਂ ਵਿੱਚ ਛੁਪੀ ਖ਼ਤਰਨਾਕ ਹੁਕਮ ਜੋ ਅਣਚਾਹੇ ਏ.ਆਈ. ਕਾਰਜ ਕਾਰਨ ਬਣਾਉਂਦੇ ਹਨ
- **ਵੈੱਬ ਸਮੱਗਰੀ ਲਾਭ ਉਠਾਉਣਾ**: ਸਖ਼ਤ ਕੀਤਾ ਗਿਆ ਵੈੱਬ ਪੰਨਾ ਜਿਸ ਵਿੱਚ ਐम्बੈੱਡਡ ਪ੍ਰਾਟ ਹੁਕਮ ਹੁੰਦੇ ਹਨ ਜੋ ਸਕ੍ਰੈਪਿੰਗ 'ਤੇ ਏ.ਆਈ. ਵਰਤੋਂ ਬਦਲ ਦਿੰਦੇ ਹਨ
- **ਈਮੇਲ-ਆਧਾਰਿਤ ਹਮਲੇ**: ਈਮੇਲਾਂ ਵਿੱਚ ਖ਼ਤਰਨਾਕ ਪ੍ਰਾਟ ਜਿਹੜੇ ਏ.ਆਈ. ਸਹਾਇਕਾਂ ਨੂੰ ਜਾਣਕਾਰੀ ਲੀਕ ਕਰਨ ਜਾਂ ਗੈਰਕਾਨੂੰਨੀ ਕਾਰਵਾਈ ਕਰਨ ਲਈ ਵੱਡੇ ਹੁੰਦੇ ਹਨ
- **ਡੇਟਾ ਸਰੋਤ ਦਾ ਦੂਸ਼ਣ**: ਸਪਿੱਡ ਡਾਟਾਬੇਸ ਜਾਂ ਏਪੀਆਈਆਂ ਜਿਹੜੀਆਂ ਵੰਧੀਆਂ ਸਮੱਗਰੀ ਏ.ਆਈ. ਨੂੰ ਪੇਸ਼ ਕਰਦੀਆਂ ਹਨ

**ਅਸਲੀ ਦੁਨੀਆ ਪ੍ਰਭਾਵ**: ਇਹ ਹਮਲੇ ਡਾਟਾ ਚੁਰੀ, ਨਿੱਜਤਾ ਉਲੰਘਣਾ, ਨੁਕਸਾਨਦਾਇਕ ਸਮੱਗਰੀ ਬਣਾਉਣਾ, ਅਤੇ ਉਪਭੋਗਤਾ ਸੰਵਾਦਾਂ ਵਿੱਚ ਛੇੜਛਾੜ ਕਰ ਸਕਦੇ ਹਨ। ਵਧੇਰੇ ਵਿਸਥਾਰ ਲਈ ਵੇਖੋ [Prompt Injection in MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/)।

![Prompt Injection Attack Diagram](../../../translated_images/pa/prompt-injection.ed9fbfde297ca877.webp)

#### **ਟੂਲ ਜ਼ਹਿਰੀਲਾ ਬਣਾਉਣਾ ਹਮਲੇ**

**ਟੂਲ ਜ਼ਹਿਰੀਲਾ ਬਣਾਉਣਾ** MCP ਟੂਲਾਂ ਦੀ ਮੈਟਾਡੇਟਾ ਨੂੰ ਨਿਸ਼ਾਨਾ ਬਣਾਉਂਦਾ ਹੈ, ਜਿਸ ਨਾਲ LLMs ਟੂਲ ਵਰਣਨਾਂ ਅਤੇ ਪੈਰਾਮੀਟਰਾਂ ਦੀ ਵਿਆਖਿਆ ਕਰਦੇ ਹਨ ਅਤੇ ਜਾਰੀ ਕਰਨ ਵਾਲੀਆਂ ਫੈਸਲਿਆਂ ‘ਤੇ ਪ੍ਰਭਾਵ ਪੈਂਦਾ ਹੈ।

**ਹਮਲਾਕਾਰੀ ਤਰੀਕੇ:**
- **ਮੈਟਾਡੇਟਾ ਤਬਦੀਲੀ**: ਹਮਲਾਕਾਰ ਟੂਲ ਵਰਣਨ, ਪੈਰਾਮੀਟਰ ਪਰਿਭਾਸ਼ਾਵਾਂ ਜਾਂ ਉਦਾਹਰਨਾਂ ਵਿੱਚ ਖ਼ਤਰਨਾਕ ਹੁਕਮ ਛੁਪਾਉਂਦੇ ਹਨ
- **ਅਦਿੱਖੀ ਹਦਾਇਤਾਂ**: ਟੂਲ ਮੈਟਾਡੇਟਾ ਵਿੱਚ ਛੁਪੇ ਪ੍ਰਾਟ ਜਿਹੜੇ ਏ.ਆਈ. ਮਾਡਲ ਦੁਆਰਾ ਪਰਕਿਰਿਆ ਕੀਤੇ ਜਾਂਦੇ ਹਨ ਪਰ ਮਨੁੱਖੀ ਉਪਭੋਗਤਿਆਂ ਲਈ ਅਦਿੱਖੇ ਰਹਿੰਦੇ ਹਨ
- **ਗਤੀਸ਼ੀਲ ਟੂਲ ਸੋਧ ("ਰਗ ਪੂਲਜ਼")**: ਵਰਤੋਂਕਾਰਾਂ ਦੁਆਰਾ ਮਨਜ਼ੂਰ ਕੀਤੇ ਟੂਲਾਂ ਨੂੰ ਬਾਅਦ ਵਿੱਚ ਬਿਨਾਂ ਜਾਣੂ ਕੀਤੀ ਗਈ ਖ਼ਤਰਨਾਕ ਕਾਰਜਾਂ ਦੇ ਲਈ ਸੋਧਿਆ ਜਾ ਸਕਦਾ ਹੈ
- **ਪੈਰਾਮੀਟਰ ਇੰਜੈਕਸ਼ਨ**: ਟੂਲ ਪੈਰਾਮੀਟਰ ਸਕੀਮਾਂ ਵਿੱਚ ਖ਼ਤਰਨਾਕ ਸਮੱਗਰੀ ਜੋ ਮਾਡਲ ਵਿਵਹਾਰ ਤੇ ਪ੍ਰਭਾਵ ਪਾਂਦਾ ਹੈ

**ਹੋਸਟ ਕੀਤੇ ਸਰਵਰ ਖਤਰਿਆਂ**: ਦੂਰੀ MCP ਸਰਵਰਾਂ ਨੂੰ ਵਧੇਰੇ ਖਤਰਾ ਹੁੰਦਾ ਹੈ ਕਿਉਂਕਿ ਟੂਲ ਪਰਿਭਾਸ਼ਾਵਾਂ ਚਾਲੂ ਸਮੇਂ ਬਾਅਦ ਅਪਡੇਟ ਕੀਤੀਆਂ ਜਾਂਦੀਆਂ ਹਨ, ਜਿਸ ਨਾਲ ਪਹਿਲਾਂ ਸੁਰੱਖਿਅਤ ਸਮਝੇ ਜਾਂਦੇ ਟੂਲ ਖ਼ਤਰਨਾਕ ਹੋ ਸਕਦੇ ਹਨ। ਵਿਆਪਕ ਜਾਣਕਾਰੀ ਲਈ ਵੇਖੋ [Tool Poisoning Attacks (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks)।

![Tool Injection Attack Diagram](../../../translated_images/pa/tool-injection.3b0b4a6b24de6bef.webp)

#### **ਵਾਧੂ ਏ.ਆਈ. ਹਮਲੇ ਦੇ ਰਸਤੇ**

- **ਕ੍ਰਾਸ-ਡੋਮੀਨ ਪ੍ਰਾਟ ਇੰਜੈਕਸ਼ਨ (XPIA)**: ਬਹੁ-ਡੋਮੇਨ ਸਮੱਗਰੀ ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਸੁਖੜੇ ਹਮਲੇ ਜੋ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣਾਂ ਨੂੰ ਬਾਈਪਾਸ ਕਰਦੇ ਹਨ
- **ਡਾਇਨਾਮਿਕ ਸਮਰੱਥਾ ਤਬਦੀਲੀ**: ਅਜਿਹੀਆਂ ਸਮਰੱਥਾਵਾਂ ਵਿੱਚ ਰੀਅਲ-ਟਾਈਮ ਬਦਲਾਵ ਜੋ ਆਰੰਭਿਕ ਸੁਰੱਖਿਆ ਜाँचਾਂ ਤੋਂ ਬਚ ਜਾਂਦੇ ਹਨ  
- **ਸੰਦਰਭ ਖਿੜਕੀ ਜ਼ਹਿਰੀਲਾ ਬਣਾਉਣਾ**: ਵੱਡੇ ਸੰਦਰਭ ਖਿੜਕੀਆਂ ਨੂੰ ਮਨਮਦਦਭਰੇ ਨਿਰਦੇਸ਼ਾਂ ਨੂੰ ਛੁਪਾਉਣ ਲਈ ਮੋੜਨ ਵਾਲੇ ਹਮਲੇ  
- **ਮਾਡਲ ਗੁੰਮਰਾਹੀ ਹਮਲੇ**: ਮਾਡਲ ਦੀਆਂ ਸੀਮਾਵਾਂ ਨੂੰ ਲਾਭ ਉਠਾਉ ਕੇ ਅਣਅੰਦਾਜ਼ੇ ਜਾਂ ਅਸੁਰੱਖਿਅਤ ਵਰਤਾਰਿਆਂ ਨੂੰ ਪੈਦਾ ਕਰਨਾ  


### AI ਸੁਰੱਖਿਆ ਖ਼ਤਰਾ ਪ੍ਰਭਾਵ

**ਉੱਚ ਪ੍ਰਭਾਵ ਵਾਲੇ ਨਤੀਜੇ:**  
- **ਡਾਟਾ ਚੋਰੀ**: ਬਿਨਾਂ ਆਗਿਆ ਦੇ ਸੰਵੇਦਨਸ਼ੀਲ, ਉਦਯੋਗਿਕ ਜਾਂ ਨਿੱਜੀ ਡਾਟਾ ਤੱਕ ਪਹੁੰਚ ਅਤੇ ਚੋਰੀ  
- **ਪ੍ਰਾਈਵੇਟਸੀ ਭੰਗ**: ਨਿੱਜੀ ਪਛਾਣਯੋਗ ਜਾਣਕਾਰੀ (PII) ਅਤੇ ਰਾਜ਼ਦਾਰ ਕਾਰੋਬਾਰੀ ਡਾਟਾ ਦਾ ਖੁਲਾਸਾ  
- **ਸਿਸਟਮ ਮੈਨਿਪੁਲੇਸ਼ਨ**: ਮੱਹਤਵਪੂਰਣ ਸਿਸਟਮਾਂ ਅਤੇ ਕਾਰਜਪ੍ਰਣਾਲੀਆਂ ਵਿੱਚ ਅਕਾਲਪਿਤ ਤਬਦੀਲੀ  
- **ਪ੍ਰਮਾਣ ਪੱਤਰ ਚੋਰੀ**: ਪ੍ਰਮਾਣਿਕਤਾ ਟੋਕਨਾਂ ਅਤੇ ਸੇਵਾ ਪ੍ਰਮਾਣ ਪੱਤਰਾਂ ਦਾ ਕਮਜ਼ੋਰ ਹੋਣਾ  
- **ਪਾਰ्शਵਿਕ ਚਲਣ**: ਖਰਾਬ ਕੀਤੇ AI ਸਿਸਟਮਾਂ ਦੀ ਵਰਤੋਂ ਵਿਆਪਕ ਨੈੱਟਵਰਕ ਹਮਲਿਆਂ ਵਾਸਤੇ ਪਿਵਟ ਵਜੋਂ ਕਰਨੀ  

### ਮਾਈਕ੍ਰੋਸੌਫਟ AI ਸੁਰੱਖਿਆ ਹੱਲ

#### **AI ਪ੍ਰਾਂਪਟ ਸ਼ੀਲਡਸ: ਇੰਜੈਕਸ਼ਨ ਹਮਲਿਆਂ ਦੇ ਖ਼ਿਲਾਫ ਉੱਨਤ ਸੁਰੱਖਿਆ**

ਮਾਈਕਰੋਸੌਫਟ **AI ਪ੍ਰਾਂਪਟ ਸ਼ੀਲਡਸ** ਸਿੱਧੇ ਅਤੇ ਪਰੋਖ ਪ੍ਰਾਂਪਟ ਇੰਜੈਕਸ਼ਨ ਹਮਲਿਆਂ ਤੋਂ ਬਹੁ-ਸਤਹੀ ਸੁਰੱਖਿਆ ਮੁਹੱਈਆ ਕਰਵਾਉਂਦੇ ਹਨ:

##### **ਮੁੱਖ ਸੁਰੱਖਿਆ ਤਰੀਕੇ:**

1. **ਆਗਿਆਮਿਤ ਪਛਾਣ ਅਤੇ ਛਾਣ-ਬੀਣ**  
   - ਮਸ਼ੀਨ ਲਰਨਿੰਗ ਅਤੇ NLP ਤਕਨਾਲੋਜੀਆਂ ਦੁਰਾਸ਼ਾ ਨਿਰਦੇਸ਼ਾਂ ਨੂੰ ਬਾਹਰੀ ਸਮੱਗਰੀ ਵਿੱਚ ਪਛਾਣਦੀਆਂ ਹਨ  
   - ਦਸਤਾਵੇਜ਼ਾਂ, ਵੈੱਬ ਪੇਜਾਂ, ਈਮੇਲਾਂ ਅਤੇ ਡਾਟਾ ਸਰੋਤਾਂ ਦੀ ਵਾਸਤਵਿਕ ਸਮੇਂ ਵਿਸ਼ਲੇਸ਼ਣ  
   - ਕਾਨੂੰਨੀ ਅਤੇ ਮਨਮਦਦਭਰੇ ਪ੍ਰਾਂਪਟ ਪੈਟਰਨਾਂ ਦਾ ਸੰਦਰਭਿਕ ਸਮਝਣਾ  

2. **ਸਪੌਟਲਾਈਟਿੰਗ ਤਕਨੀਕਾਂ**  
   - ਭਰੋਸੇਯੋਗ ਸਿਸਟਮ ਨਿਰਦੇਸ਼ਾਂ ਅਤੇ ਸੰਦੇਹਾਸਪਦ ਬਾਹਰੀ ਇਨਪੁਟ ਵਿਚਕਾਰ ਫਰਕ ਕਰਨ ਵਾਲਾ  
   - ਲੇਖ ਤਬਦੀਲੀ ਦੇ ਤਰੀਕੇ ਜੋ ਮਾਡਲ ਦੇ ਸਬੰਧਤਾ ਨੂੰ ਵਧਾਉਂਦੇ ਹਨ ਪਰ ਮਨਮਦਦਭਰੀ ਸਮੱਗਰੀ ਨੂੰ ਵੱਖ ਕਰਦੇ ਹਨ  
   - AI ਸਿਸਟਮਾਂ ਨੂੰ ਉਚਿਤ ਨਿਰਦੇਸ਼ ਅਨੁਕ੍ਰਮ ਬਣਾਈ ਰੱਖਣ ਅਤੇ ਇੰਜੈਕਟ ਕੀਤੇ ਅਦੇਸ਼ਾਂ ਨੂੰ ਅਣਡਿੱਠਾ ਕਰਨ ਵਿੱਚ ਮਦਦ ਕਰਦਾ ਹੈ  

3. **ਡਿਲੀਮੀਟਰ ਅਤੇ ਡਾਟਾਮਾਰਕਿੰਗ ਸਿਸਟਮ**  
   - ਭਰੋਸੇਯੋਗ ਸਿਸਟਮ ਸੰਦੇਸ਼ਾਂ ਅਤੇ ਬਾਹਰੀ ਇਨਪੁਟ ਵਿਚਕਾਰ ਸਪਸ਼ਟ ਸੀਮਾ ਪਰਿਭਾਸ਼ਾ  
   - ਖਾਸ ਚਿੰਨ੍ਹ ਭਰੋਸੇਯੋਗ ਅਤੇ ਅਣਭਰੋਸੇਯੋਗ ਡਾਟਾ ਸਰੋਤਾਂ ਦੀ ਸੀਮਾਵਾਂ ਨੂੰ ਝਲਕਾਉਂਦੇ ਹਨ  
   - ਸਪਸ਼ਟ ਵੰਡ ਨਿਰਦੇਸ਼ਾਂ ਵਿੱਚ ਗੜਬੜ ਅਤੇ ਬਿਨਾਂ ਆਗਿਆ ਦੇ ਹੁਕਮਾਂ ਦੀ ਪਾਲਣਾ ਤੋਂ ਰੋਕਦੀ ਹੈ  

4. **ਲਗਾਤਾਰ ਧਮਕੀ ਬੁੱਝ-ਪ੍ਰਣਾਲੀ**  
   - ਮਾਈਕ੍ਰੋਸੌਫਟ ਤੇਜ਼ੀ ਨਾਲ ਉੱਗ ਰਹੇ ਹਮਲੇ ਦੇ ਪੈਟਰਨਾਂ ਦੀ ਨਿਗਰਾਨੀ ਕਰਦਾ ਹੈ ਅਤੇ ਸੁਰੱਖਿਆ ਅੱਪਡੇਟ ਕਰਦਾ ਹੈ  
   - ਨਵੀਆਂ ਇੰਜੈਕਸ਼ਨ ਤਕਨੀਕਾਂ ਅਤੇ ਹਮਲੇ ਦੇ ਰਸਤੇ ਲੱਭਣ ਲਈ ਦਿਸ਼ਾ ਨਿਰਦੇਸ਼ੀ ਸਰਗਰਮੀ  
   - ਲਗਾਤਾਰ ਸੁਰੱਖਿਆ ਮਾਡਲ ਅੱਪਡੇਟ ਹਮਲਿਆਂ ਦੇ ਖ਼ਿਲਾਫ ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਬਣਾਈ ਰੱਖਣ ਲਈ  

5. **ਅਜ਼ੂਰ ਕੰਟੈਂਟ ਸੁਰੱਖਿਆ ਇंटीਗ੍ਰੇਸ਼ਨ**  
   - ਵਿਆਪਕ ਅਜ਼ੂਰ AI ਕੰਟੈਂਟ ਸੁਰੱਖਿਆ ਸਿਊਟ ਦਾ ਹਿੱਸਾ  
   - ਜੇਲਬ੍ਰੇਕ ਕੋਸ਼ਿਸ਼ਾਂ, ਹਾਨੀਕਾਰਕ ਸਮੱਗਰੀ, ਅਤੇ ਸੁਰੱਖਿਆ ਨੀਤੀ ਉਲੰਘਣਾ ਲਈ ਵਾਧੂ ਪਛਾਣ  
   - AI ਐਪਲੀਕੇਸ਼ਨ ਹਿੱਸਿਆਂ ਵਿੱਚ ਇਕਾਈ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ

## ਉੱਨਤ MCP ਸੁਰੱਖਿਆ ਖਤਰੇ

### ਸੈਸ਼ਨ ਹਾਈਜੈਕਿੰਗ ਕਮਜ਼ੋਰੀਆਂ

**ਸੈਸ਼ਨ ਹਾਈਜੈਕਿੰਗ** ਸਟੇਟਫੁਲ MCP ਇਮਪਲੀਮੈਂਟੇਸ਼ਨਾਂ ਵਿੱਚ ਇੱਕ ਖਤਰਨਾਕ ਹਮਲਾ ਹੈ ਜਿੱਥੇ ਬਿਨਾਂ ਆਗਿਆ ਵਾਲੇ ਪੱਖੀ ਵਾਸ਼ਤਵਿਕ ਸੈਸ਼ਨ ਟਾਕਨ ਜ਼ਬਰਦਸਤੀ ਹਾਸਲ ਕਰਦੇ ਹਨ ਅਤੇ ਗਲਤ ਵਰਤੋਂ ਕਰਦੇ ਹਨ।

#### **ਹਮਲੇ ਦੇ ਸਿਥੁਆਰਥ ਅਤੇ ਖ਼ਤਰੇ**

- **ਸੈਸ਼ਨ ਹਾਈਜੈਕ ਪ੍ਰਾਂਪਟ ਇੰਜੈਕਸ਼ਨ**: ਚੋਰੀ ਕੀਤੇ ਸੈਸ਼ਨ IDs ਨਾਲ ਹਮਲਾਕਾਰ ਸਰਵਰਾਂ ਵਿੱਚ ਮਨਮਦਦ ਭਰੇ ਘਟਨਾ ਭੇਜ ਕੇ ਖਤਰਨਾਕ ਕਾਰਵਾਈਆਂ ਸ਼ੁਰੂ ਕਰ ਸਕਦੇ ਹਨ ਜਾਂ ਸੈਂਸਿਟਿਵ ਡਾਟਾ ਤੱਕ ਪਹੁੰਚ ਸਕਦੇ ਹਨ  
- **ਸਿੱਧਾ ਨਕਲ ਕਰਨ ਵਾਲਾ**: ਚੋਰੀ ਕੀਤੇ ਸੈਸ਼ਨ IDs ਨਾਲ MCP ਸਰਵਰ ਕਾਲਾਂ ਇੱਕ ਸਿੱਧੇ ਰੂਪ ਵਿੱਚ ਪ੍ਰਮਾਣਿਕਤਾ ਤੋਂ ਬਿਨਾਂ ਕਰ ਸਕਦੇ ਹਨ, ਹਮਲਾਕਾਰਾਂ ਨੂੰ ਪੱਕੇ ਯੂਜ਼ਰ ਸਮਝ ਕੇ  
- **ਖਰਾਬ ਰੀਜ਼ੂਮ ਕਰਨ ਵਾਲੇ ਸਟਰੀਮ**: ਹਮਲਾਕਾਰ ਬਿਨਾਂ ਪੂਰੀ ਬੇਨਤੀ ਖਤਮ ਕਰਕੇ ਵਾਸਤਵਿਕ ਕਲਾਇੰਟਾਂ ਨੂੰ ਮਨਮਦਦ ਭਰੇ ਸਮੱਗਰੀ ਨਾਲ ਰੀਜ਼ੂਮ ਕਰਨ ਲਈ ਮਜਬੂਰ ਕਰ ਸਕਦੇ ਹਨ  

#### **ਸੈਸ਼ਨ ਪ੍ਰਬੰਧਨ ਲਈ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ**

**ਅਹੰਕਾਰਪੂਰਨ ਜ਼ਰੂਰਤਾਂ:**  
- **ਪ੍ਰਮਾਣਿਕਤਾ ਜਾਂਚ**: MCP ਸਰਵਰਾਂ ਨੂੰ ਹਰ ਇੱਕ ਆਉਣ ਵਾਲੀ ਬੇਨਤੀ ਦੀ ਜਾਂਚ ਕਰਨੀ **ਜਰੂਰੀ** ਹੈ ਅਤੇ ਪ੍ਰਮਾਣਿਕਤਾ ਲਈ ਸੈਸ਼ਨਾਂ 'ਤੇ ਭਰੋਸਾ ਨਹੀਂ ਕਰਨਾ ਚਾਹੀਦਾ  
- **ਸੁਰੱਖਿਅਤ ਸੈਸ਼ਨ ਬਨਾਉਣਾ**: ਗੁਪਤਕੋਡਤਮਕ ਤੌਰ ਤੇ ਸੁਰੱਖਿਅਤ, ਬੇਨਕਸ਼ ਸੈਸ਼ਨ IDs ਜਿਨ੍ਹਾਂ ਨੂੰ ਸੁਰੱਖਿਅਤ ਰੈਂਡਮ ਨੰਬਰ ਜਨਰੇਟਰ ਨਾਲ ਬਣਾਇਆ ਗਿਆ ਹੋਵੇ  
- **ਇਸਤੋਮਾਲਕ ਕਾਰਤਬੰਧੀ**: ਸੈਸ਼ਨ IDs ਨੂੰ ਉਪਯੋਗਕਰਤਾ ਵਿਸ਼ੇਸ਼ ਜਾਣਕਾਰੀ ਨਾਲ ਜਮ੍ਹਾਂ ਕਰੋ, ਜਿਵੇਂ `<user_id>:<session_id>`, ਤਾਂ ਜੋ ਉਪਭੋਗੀ-ਪਾਰਸੀ ਸੈਸ਼ਨ ਦੁਰਵਿਵਹਾਰ ਤੋਂ ਬਚਿਆ ਜਾ ਸਕੇ  
- **ਸੈਸ਼ਨ ਜੀਵਨਚੱਕਰ ਪ੍ਰਬੰਧਨ**: ਉਚਿਤ ਅਵਧੀ ਖ਼ਤਮ ਕਰਨ, ਘੁੰਮਾਅ ਅਤੇ ਨਿਰਸਤ ਕਰਨ ਨੂੰ ਲਾਗੂ ਕਰਨਾ, ਕਮਜ਼ੋਰੀ ਖਿੜਕੀਆਂ ਨੂੰ ਸੀਮਤ ਕਰਨ ਲਈ  
- **ਟ੍ਰਾਂਸਪੋਰਟ ਸੁਰੱਖਿਆ**: ਸਾਰੇ ਸੰਚਾਰ ਲਈ ਜ਼ਰੂਰੀ HTTPS, ਤਾਂ ਜੋ ਸੈਸ਼ਨ ID ਚੋਰੀ ਤੋਂ ਬਚਾਇਆ ਜਾ ਸਕੇ  

### ਗੁੰਮਰਾਹ ਡਿਪਟੀ ਸਮੱਸਿਆ

**ਗੁੰਮਰਾਹ ਡਿਪਟੀ ਸਮੱਸਿਆ** ਉਸ ਸਮੇਂ ਹੁੰਦੀ ਹੈ ਜਦੋਂ MCP ਸਰਵਰ ਗਾਹਕਾਂ ਅਤੇ ਤੀਜੀ ਪੱਖ ਸਰਵਿਸਾਂ ਵਿੱਚ ਪ੍ਰਮਾਣਿਕਤਾ ਪ੍ਰਾਕਸੀ ਵਜੋਂ ਕੰਮ ਕਰਦੇ ਹਨ, ਜਿਸ ਨਾਲ ਸਥਿਰ ਗਾਹਕ ID ਲਾਭ ਲੈ ਕੇ ਪ੍ਰਮਾਣਿਕਤਾ ਬਾਈਪਾਸ ਹੋ ਜਾਂਦੀ ਹੈ।

#### **ਹਮਲੇ ਦੀਆਂ ਤਕਨੀਕਾਂ ਅਤੇ ਖ਼ਤਰੇ**

- **ਕੂਕੀ-ਅਧਾਰਿਤ ਸਹਿਮਤੀ ਬਾਈਪਾਸ**: ਪਹਿਲਾਂ ਦੀ ਵਰਤੋਂ ਵਾਲੀ ਪ੍ਰਮਾਣਿਕਤਾ ਸਹਿਮਤੀ ਕੂਕੀ ਬਣਾਉਂਦੀ ਹੈ ਜਿਸ ਨੂੰ ਹਮਲਾਕਾਰ crafted redirect URI ਦੇ ਨਾਲ ਮਨਮਦਦ ਭਰੇ ਪ੍ਰਮਾਣਿਕਤਾ ਬੇਨਤੀਆਂ ਦੇ ਜ਼ਰੀਏ ਲਾਭ ਉਠਾਉਂਦੇ ਹਨ  
- **ਪ੍ਰਮਾਣਿਕਤਾ ਕੋਡ ਚੋਰੀ**: ਮੌਜੂਦਾ ਸਹਿਮਤੀ ਕੂਕੀਜ਼ ਕਾਰਨ ਪ੍ਰਮਾਣਿਕਤਾ ਸਰਵਰ ਸਹਿਮਤੀ ਸਕਰੀਨਾਂ ਨੂੰ ਛੱਡ ਸਕਦਾ ਹੈ ਅਤੇ ਕੋਡ ਹਮਲਾਕਾਰ ਦੇ ਕੰਟਰੋਲਡ ਐਂਡਪੌਇੰਟ ਨੂੰ ਭੇਜਦਾ ਹੈ  
- **ਬਿਨਾਂ ਆਗਿਆ ਵਾਲਾ API ਪਹੁੰਚ**: ਚੋਰੀ ਕੀਤੇ ਗਏ ਕੋਡ ਟੋਕਨਾਂ ਦੇ ਤਬਾਦਲੇ ਅਤੇ ਯੂਜ਼ਰ ਨਕਲਾਂ ਬਿਨਾਂ ਖਾਸ ਮਨਜ਼ੂਰੀ ਦੇ ਆਜਾਜ਼ ਦਿੰਦੇ ਹਨ  

#### **ਰੋਕਥਾਮ ਲਈ ਗਤੀਵਿਧੀਆਂ**

**ਜ਼ਰੂਰੀ ਨਿਯੰਤਰਣ:**  
- **ਸਪਸ਼ਟ ਸਹਿਮਤੀ ਲੋੜ**: ਸਥਿਰ ਕਲੀਐਂਟ IDs ਵਾਲੇ MCP ਪ੍ਰਾਕਸੀ ਸਰਵਰ ਹਰੇਕ ਡਾਈਨਾਮਿਕ ਕਲੀਐਂਟ ਲਈ ਸਪਸ਼ਟ ਸਹਿਮਤੀ ਲੈਣ **ਜਰੂਰੀ** ਹੈ  
- **OAuth 2.1 ਸੁਰੱਖਿਆ ਲਾਗੂ ਕਰੋ**: PKCE ਸਮੇਤ ਹਰੇਕ ਪ੍ਰਮਾਣਿਕਤਾ ਬੇਨਤੀ ਲਈ ਮੌਜੂਦਾ OAuth ਸੁਰੱਖਿਆ ਬਿਹਤਰ ਤਰੀਕਿਆਂ ਦੀ ਪਾਲਣਾ ਕਰੋ  
- **ਕਲੀਐਂਟ ਵੈਰੀਫਿਕੇਸ਼ਨ ਦ੍ਰਿੜਤਾ**: ਰੀਡਾਇਰੈਕਟ URI ਅਤੇ ਕਲੀਐਂਟ ਪਹਿਚਾਣ ਵਿੱਚ ਸਖ਼ਤ ਜਾਂਚ ਲਾਗੂ ਕਰੋ ਤਾਂ ਜੋ ਦੁਰਵਿਵਹਾਰ ਰੋਕਿਆ ਜਾ ਸਕੇ  

### ਟੋਕਨ ਪਾਸਥਰੂ ਕਮਜ਼ੋਰੀਆਂ

**ਟੋਕਨ پਾਸਥਰੂ** ਇੱਕ ਖੁਲਾਸਾ ਆਉਂਦਾ ਹੈ ਜਿੱਥੇ MCP ਸਰਵਰ ਬਿਨਾਂ ਸਹੀ ਜਾਂਚ ਦੇ ਗਾਹਕ ਟੋਕਨਾਂ ਨੂੰ ਆਪ੍ਰੇਸ਼ਨਲ APIs ਵੱਲ ਅੱਗੇ ਭੇਜਦੇ ਹਨ, ਜੋ MCP ਪ੍ਰਮਾਣਿਕਤਾ ਨਿਰਦੇਸ਼ਾਂ ਦਾ ਉਲੰਘਣਾ ਹੈ।

#### **ਸੁਰੱਖਿਆ ਪ੍ਰਭਾਵ**

- **ਨਿਯੰਤਰਣ ਚਾਲਾਕੀ**: ਬਿਨਾਂ ਸਹੀ ਪ੍ਰਮਾਣਿਕਤਾ ਵਾਲੇ ਕਲੀਐਂਟ ਤੋਂ API ਸਿੱਧੀ ਟੋਕਨ ਵਰਤੋਂ ਨੂੰ ਰੈਟ ਲਿਮਿਟਿੰਗ ਅਤੇ ਨਿਗਰਾਨੀ ਤੋਂ ਬਚਾਉਂਦਾ ਹੈ  
- **ਆਡਿਟ ਟਰੇਲ ਖਰਾਬ ਹੋਣਾ**: ਅਪਸਟ੍ਰੀਮ ਜਾਰੀ ਕੀਤੇ ਟੋਕਨ ਨਾਲ ਗਾਹਕ ਪਛਾਣ ਅਸੰਭਵ ਬਣ ਜਾਂਦੀ ਹੈ, ਘਟਨਾ ਜਾਂਚ ਖ਼ਰਾਬ ਹੋ ਜਾਂਦੀ ਹੈ  
- **ਪ੍ਰਾਕਸੀ ਆਧਾਰਤ ਡਾਟਾ ਚੋਰੀ**: ਬਿਨਾਂ ਜਾਂਚ ਵਾਲੇ ਟੋਕਨਾਂ ਨਾਲ ਮਨਮਦੀ ਘਟਕ ਸੇਵਾਵਾਂ ਨੂੰ ਪ੍ਰਾਕਸੀ ਵਜੋਂ ਵਰਤ ਸਕਦੇ ਹਨ  
- **ਭਰੋਸਾ ਸੀਮਾ ਉਲੰਘਣਾ**: ਜਦੋਂ ਟੋਕਨ ਦਾ ਮੂਲ ਪਤਾ ਨਹੀਂ ਲੱਗਦਾ ਤਾਂ ਹੇਠਲੇ ਸੇਵਾਵਾਂ ਦੀ ਭਰੋਸੇਯੋਗਤਾ ਭੰਗ ਹੋ ਸਕਦੀ ਹੈ  
- **ਬਹੁ-ਸੇਵਾ ਹਮਲੇ ਦਾ ਵਿਸਤਾਰ**: ਖਰਾਬ ਕੀਤੇ ਟੋਕਨ ਕਈ ਸੇਵਾਵਾਂ ਵਿੱਚ ਮੰਨ ਲਏ ਜਾਣ ਨਾਲ ਪਾਰਸ਼ਵਿਕ ਹਮਲੇ ਵਧਦੇ ਹਨ  

#### **ਜ਼ਰੂਰੀ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ**

**ਨਿਰਦੇਸ਼ਾਂ ਦੀ ਲੋੜ:**  
- **ਟੋਕਨ ਜਾਂਚ**: MCP ਸਰਵਰ ਉਹਨਾਂ ਟੋਕਨਾਂ ਨੂੰ ਮਨਜ਼ੂਰ ਨਹੀਂ ਕਰਨੇ, ਜੋ ਖਾਸ MCP ਸਰਵਰ ਲਈ ਜਾਰੀ ਨਹੀਂ ਕੀਤੇ ਗਏ  
- **ਦਰਸ਼ਕ ਜਾਂਚ**: ਹਮੇਸ਼ਾਂ ਟੋਕਨ ਦਰਸ਼ਕ ਬਿਆਨਾਂ ਦੀ ਜਾਂਚ ਕਰੋ ਕਿ ਉਹ MCP ਸਰਵਰ ਦੀ ਪਛਾਣ ਨਾਲ ਮੇਲ ਖਾਂਦੇ ਹਨ  
- **ਟੋਕਨ ਜੀਵਨ ਚੱਕਰ**: ਛੋਟੀ ਅਵਧੀ ਵਾਲੇ ਐਕਸੈਸ ਟੋਕਨ ਅਤੇ ਸੁਰੱਖਿਅਤ ਰੋਟੇਸ਼ਨ ਅਮਲ ਕਰੋ  

## AI ਸਿਸਟਮਾਂ ਲਈ ਸਪਲਾਈ ਚੇਨ ਸੁਰੱਖਿਆ

ਸਪਲਾਈ ਚੇਨ ਸੁਰੱਖਿਆ ਪਰੰਪਰਾਗਤ ਸਾਫਟਵੇਅਰ ਨਿਰਭਰਤਾਵਾਂ ਤੋਂ ਬਾਹਰ ਨਿਕਲ ਕੇ ਪੂਰੇ AI ਪਰਿਭਾਸ਼ਾ ਤੱਕ ਵਧ ਚੁੱਕੀ ਹੈ। ਆਧੁਨਿਕ MCP ਇੰਪਲੀਮੈਂਟੇਸ਼ਨਾਂ ਨੂੰ ਸਾਰੇ AI ਸੰਬੰਧੀ ਹਿੱਸਿਆਂ ਦੀ ਕੜੀ ਤੌਰ 'ਤੇ ਜਾਂਚ ਅਤੇ ਨਿਗਰਾਨੀ ਕਰਨੀ ਚਾਹੀਦੀ ਹੈ ਕਿਉਂਕਿ ਹਰ ਇੱਕ ਵਲਨਰੇਬਿਲਿਟੀ ਦੇ ਸੰਭਾਵਨਾ ਦੇ ਸਮਰੂਪ ਹੈ ਜੋ ਸਿਸਟਮਣ ਦੀ ਸ਼ੁੱਧਤਾ ਨੂੰ ਪ੍ਰਭਾਵਿਤ ਕਰ ਸਕਦੀ ਹੈ।

### ਵਿਸ਼ਤ੍ਰਿਤ AI ਸਪਲਾਈ ਚੇਨ ਹਿੱਸੇ

**ਪਰੰਪਰਾਗਤ ਸਾਫਟਵੇਅਰ ਨਿਰਭਰਤਾਵਾਂ:**  
- ਖੁੱਲ੍ਹਾ ਸਰੋਤ ਲਾਇਬ੍ਰੇਰੀਆਂ ਅਤੇ ਫਰੇਮਵਰਕ  
- ਕੰਟੇਨਰ ਇਮੇਜ ਅਤੇ ਬੇਸ ਸਿਸਟਮ  
- ਵਿਕਾਸ ਟੂਲ ਅਤੇ ਬਿਲਡ ਪਾਈਪਲਾਈਨ  
- ਢਾਂਚਾਗਤ ਹਿੱਸੇ ਅਤੇ ਸੇਵਾਵਾਂ  

**AI ਵੱਖਰਾ ਸਪਲਾਈ ਚੇਨ ਤੱਤ:**  
- **ਫਾਊਂਡੇਸ਼ਨ ਮਾਡਲ:** ਵੱਖ-ਵੱਖ ਪ੍ਰਦਾਤਾ ਵੱਲੋਂ ਪ੍ਰੀ-ਟ੍ਰੇਨਡ ਮਾਡਲ ਜਿਨ੍ਹਾਂ ਦੀ ਨਸਲ ਜਾਂਚ ਲੋੜੀਂਦੀ ਹੈ  
- **ਇംബੈਡਿੰਗ ਸੇਵਾਵਾਂ:** ਬਾਹਰੀ ਵੈਕਟਰਾਈਜ਼ੇਸ਼ਨ ਅਤੇ ਸੈਮਾਂਟਿਕ ਸੇਰਚ ਸੇਵਾਵਾਂ  
- **ਸੰਦਰਭ ਪ੍ਰਦਾਤਾ:** ਡਾਟਾ ਸਰੋਤ, ਗਿਆਨ ਅਧਾਰ, ਅਤੇ ਦਸਤਾਵੇਜ਼ ਰਿਪੋਜ਼ਿਟਰੀਜ਼  
- **ਤੀਜੀ ਪੱਖ API:** ਬਾਹਰੀ AI ਸੇਵਾਵਾਂ, ML ਪਾਈਪਲਾਈਨ ਅਤੇ ਡਾਟਾ ਪ੍ਰੋਸੈਸਿੰਗ ਐਂਡਪੌਇੰਟ  
- **ਮਾਡਲ ਆਰਟੀਫੈਕਟਸ:** ਵਜ਼ਨ, ਕਨਫਿਗਰੇਸ਼ਨਸ ਅਤੇ ਫਾਈਨ-ਟਿਊਨਡ ਮਾਡਲ ਕਿਸਮਾਂ  
- **ਟ੍ਰੇਨਿੰਗ ਡਾਟਾ ਸਰੋਤ:** ਮਾਡਲ ਟ੍ਰੇਨਿੰਗ ਅਤੇ ਫਾਈਨ-ਟਿਊਨਿੰਗ ਲਈ ਵਰਤੇ ਗਏ ਡੇਟਾਸੈੱਟ  

### ਵਿਸਤ੍ਰਿਤ ਸਪਲਾਈ ਚੇਨ ਸੁਰੱਖਿਆ ਰਣਨੀਤੀ  

#### **ਹਿੱਸਾ ਜਾਂਚ ਅਤੇ ਭਰੋਸਾ**  
- **ਨਸਲ ਦੀ ਜਾਂਚ**: ਸਾਰੇ AI ਹਿੱਸਿਆਂ ਦੀ ਮੂਲ ਸਮੱਗਰੀ, ਲਾਇਸੰਸ ਅਤੇ ਸ਼ੁੱਧਤਾ ਦੀ ਜਾਂਚ ਪ੍ਰ ਇੰਟੈਗਰੇਸ਼ਨ ਤੋਂ ਪਹਿਲਾਂ ਕਰੋ  
- **ਸੁਰੱਖਿਆ ਮੁਲਾਂਕਣ**: ਮਾਡਲਾਂ, ਡਾਟਾ ਸਰੋਤਾਂ ਅਤੇ AI ਸੇਵਾਵਾਂ ਲਈ ਕਮਜ਼ੋਰੀ ਸਕੈਨ ਅਤੇ ਜਾਂਚ  
- **ਖ਼ੁਦਗਿਰਦੀ ਦਾ ਵਿਸ਼ਲੇਸ਼ਣ**: AI ਸੇਵਾ ਪ੍ਰਦਾਤਿਆਂ ਦੇ ਸੁਰੱਖਿਆ ਰਿਕਾਰਡ ਅਤੇ ਅਮਲਾਂ ਦੀ ਸਮੀਖਿਆ  
- **ਅਨੁਕੂਲਤਾ ਜਾਂਚ**: ਯਕੀਨੀ ਬਣਾਓ ਕਿ ਸਾਰੇ ਹਿੱਸੇ ਸੰਗਠਨਾਤਮਕ ਸੁਰੱਖਿਆ ਅਤੇ ਨਿਯਮਾਂ ਨੂੰ ਪੂਰਾ ਕਰਦੇ ਹਨ  

#### **ਸੁਰੱਖਿਅਤ ਤੈਨਾਤੀ ਪਾਈਪਲਾਈਨ**  
- **ਆਟੋਮੈਟਿਕ CI/CD ਸਕੈਨਿੰਗ**: ਆਟੋਮੈਟਿਕ ਤੈਨਾਤੀ ਪਾਈਪਲਾਈਨਾਂ ਵਿੱਚ ਸੁਰੱਖਿਆ ਸਕੈਨਿੰਗ ਸ਼ਾਮਲ ਕਰੋ  
- **ਆਰਟੀਫੈਕਟ ਸ਼ੁੱਧਤਾ**: ਸਾਰੇ ਤੈਨਾਤ ਕੀਤੇ ਆਰਟੀਫੈਕਟਸ (ਕੋਡ, ਮਾਡਲ, ਕਨਫਿਗਰੇਸ਼ਨ) ਲਈ ਗੁਪਤਕੋਡਤਮਕ ਜਾਂਚ ਲਾਗੂ ਕਰੋ  
- **ਦਿੱਤੇ ਗਏ ਤਬਦੀਲੀ ਕਦਮ**: ਹਰੇਕ ਪੜਾਅ 'ਤੇ ਸੁਰੱਖਿਆ ਸਵੀਕਾਰਨਾ ਨਾਲ ਪ੍ਰਗਟ ਪ੍ਰਤੀਰੋਧ  
- **ਭਰੋਸੇਯੋਗ ਆਰਟੀਫੈਕਟ ਰਿਪੋਜ਼ਿਟਰੀਜ਼**: ਸਿਰਫ ਸੁਰੱਖਿਅਤ ਅਤੇ ਸਾਬਤ ਭਰੋਸੇਯੋਗ ਰਜਿਸਟਰੀਜ਼ ਤੋਂ ਹੀ ਤੈਨਾਤ ਕਰੋ  

#### **ਨਿਰੰਤਰ ਨਿਗਰਾਨੀ ਅਤੇ ਜਵਾਬ**  
- **ਨਿਰਭਰਤਾ ਸਕੈਨਿੰਗ**: ਸਾਰੀਆਂ ਸਾਫਟਵੇਅਰ ਅਤੇ AI ਹਿੱਸਿਆਂ ਦੀ ਲਾਗਾਤਾਰ ਕਮਜ਼ੋਰੀ ਨਿਗਰਾਨੀ  
- **ਮਾਡਲ ਮਾਨੀਟਰਿੰਗ**: ਮਾਡਲ ਵਰਤਾਰਾ, ਪ੍ਰਦਰਸ਼ਨ ਡ੍ਰਿਫਟ ਅਤੇ ਸੁਰੱਖਿਆ ਵਿਸੰਗਤੀਆਂ ਦੀ ਲਗਾਤਾਰ ਜाँच  
- **ਸੇਵਾ ਸਿਹਤ ਟ੍ਰැਕਿੰਗ**: ਬਾਹਰੀ AI ਸੇਵਾਵਾਂ ਦੀ ਉਪਲਬਧਤਾ, ਸੁਰੱਖਿਆ ਘਟਨਾਵਾਂ ਅਤੇ ਨੀਤੀ ਬਦਲਾਅ ਦੀ ਨਿਗਰਾਨੀ  
- **ਧਮਕੀ ਜਾਣਕਾਰੀ ਇंटीਗ੍ਰੇਸ਼ਨ**: AI ਅਤੇ ML ਸੁਰੱਖਿਆ ਖ਼ਤਰੇ ਨਾਲ ਸੰਬੰਧਿਤ ਧਮਕੀ ਫੀਡ ਸ਼ਾਮਲ ਕਰੋ  

#### **ਪਹੁੰਚ ਨਿਯੰਤਰਣ ਅਤੇ ਘੱਟੋ-ਘੱਟ ਅਧਿਕਾਰ**  
- **ਹਿੱਸਾ-ਸਤਰ ਅਧਿਕਾਰ**: ਮਾਡਲ, ਡਾਟਾ, ਅਤੇ ਸੇਵਾਵਾਂ ਲਈ ਕਾਰੋਬਾਰੀ ਜ਼ਰੂਰਤ ਅਨੁਸਾਰ ਪਹੁੰਚ ਸੀਮਤ ਕਰੋ  
- **ਸੇਵਾ ਖਾਤਾ ਪ੍ਰਬੰਧਨ**: ਘੱਟੋ-ਘੱਟ ਜ਼ਰੂਰੀ ਅਧਿਕਾਰ ਵਾਲੇ ਸਮਰਪਿਤ ਸੇਵਾ ਖਾਤੇ ਲਾਗੂ ਕਰੋ  
- **ਨੈੱਟਵਰਕ ਵੱਖ-ਵੱਖਤਾ**: AI ਹਿੱਸਿਆਂ ਨੂੰ ਤਨਹਾ ਕਰੋ ਅਤੇ ਸੇਵਾਵਾਂ ਦਰਮਿਆਨ ਨੈੱਟਵਰਕ ਪਹੁੰਚ ਸੀਮਤ ਕਰੋ  
- **API ਗੇਟਵੇ ਨਿਯੰਤਰਣ**: ਬਾਹਰੀ AI ਸੇਵਾਵਾਂ ਲਈ ਕੇਂਦਰਿਤ API ਗੇਟਵੇਜ਼ ਦੇ ਜ਼ਰੀਏ ਪਹੁੰਚ ਨੂੰ ਨਿਯੰਤਰਿਤ ਅਤੇ ਨਿਗਰਾਨੀ ਕਰੋ  

#### **ਘਟਨਾ ਜਵਾਬ ਅਤੇ ਪੁਨਰ ਪ੍ਰਾਪਤੀ**  
- **ਤੇਜ਼ ਜਵਾਬ ਪ੍ਰਕਿਰਿਆਵਾਂ**: ਖਰਾਬ ਕੀਤੇ AI ਹਿੱਸਿਆਂ ਦੀ ਪੈਚਿੰਗ ਜਾਂ ਬਦਲਾਅ ਲਈ ਸਥਾਪਿਤ ਪ੍ਰਕਿਰਿਆਵਾਂ  
- **ਪ੍ਰਮਾਣ ਪੱਤਰ ਘੁੰਮਾਅ**: ਸੈਕਰੇਟ, API ਕੁੰਜੀਆਂ, ਅਤੇ ਸੇਵਾ ਪ੍ਰਮਾਣ ਪੱਤਰਾਂ ਦੀ ਘੁੰਮਾਅ ਲਈ ਆਟੋਮੈਟਿਕ ਪ੍ਰਣਾਲੀ  
- **ਰੋਲਬੈਕ ਸਮਰੱਥਾ**: ਪਿਛਲੇ ਜਾਣੇ-ਪਛਾਣੇ-ਠੀਕ ਸੰਸਕਰਣਾਂ ਵੱਲ ਤੁਰੰਤ ਵਾਪਸੀ  
- **ਸਪਲਾਈ ਚੇਨ ਭੰਗ ਪੁਨਰ ਪ੍ਰਾਪਤੀ**: ਉੱਪਸਟ੍ਰੀਮ AI ਸੇਵਾ ਘਟਕ ਖਰਾਬੀਆਂ ਲਈ ਵਿਸ਼ੇਸ਼ ਪ੍ਰਕਿਰਿਆਵਾਂ  

### ਮਾਈਕ੍ਰੋਸੌਫਟ ਸੁਰੱਖਿਆ ਟੂਲ ਅਤੇ ਇਨਟੀਗ੍ਰੇਸ਼ਨ

- **ਸਪਲਾਈ ਚੇਨ ਜਾਣਕਾਰੀਆਂ**: ਨਿਰਭਰਤਾ ਸਿਹਤ ਅਤੇ ਸੁਰੱਖਿਆ ਹਾਲਤ ਵਿੱਚ ਵੇਖ ਭਾਲ

## ਫਾਊਂਡੇਸ਼ਨ ਸੁਰੱਖਿਆ ਸਰਵੋੱਤਮ ਅਮਲ

MCP ਇੰਪਲੀਮੈਂਟੇਸ਼ਨ ਤੁਹਾਡੇ ਸੰਗਠਨ ਦੀ ਮੌਜੂਦਾ ਸੁਰੱਖਿਆ ਧਾਂਚਾ ਨੂੰ ਵਿਰਾਸਤ ਵਿੱਚ ਲੈਂਦੇ ਅਤੇ ਉਸ ਨੂੰ ਮਜ਼ਬੂਤ ਕਰਦੇ ਹਨ। ਬੁਨਿਆਦੀ ਸੁਰੱਖਿਆ ਅਭਿਆਸਾਂ ਨੂੰ ਮਜ਼ਬੂਤ ਕਰਨਾ AI ਸਿਸਟਮਾਂ ਅਤੇ MCP ਤੈਨਾਤੀਆਂ ਦੀ ਕੁੱਲ ਸੁਰੱਖਿਆ ਨੂੰ ਕਾਫੀ ਹੱਦ ਤੱਕ ਬਹਿਤਰ ਬਣਾਉਂਦਾ ਹੈ।

### ਮੁੱਖ ਸੁਰੱਖਿਆ ਮੂਲ ਤੱਤ

#### **ਸੁਰੱਖਿਅਤ ਵਿਕਾਸ ਅਭਿਆਸ**  
- **OWASP ਪਾਲਣਾ**: [OWASP Top 10](https://owasp.org/www-project-top-ten/) ਵੈੱਬ ਐਪਲੀਕੇਸ਼ਨ ਕਮਜ਼ੋਰੀਆਂ ਤੋਂ ਬਚਾਅ  
- **AI-ਵਿਸ਼ੇਸ਼ ਸੁਰੱਖਿਆ**: [OWASP Top 10 for LLMs](https://genai.owasp.org/download/43299/?tmstv=1731900559) ਲਈ ਨਿਯੰਤਰਣ ਲਾਗੂ ਕਰੋ  
- **ਸੀਕ੍ਰੇਟ ਪ੍ਰਬੰਧਨ**: ਟੋਕਨ, API ਕੁੰਜੀਆਂ ਅਤੇ ਸੰਵੇਦਨਸ਼ੀਲ ਕਨਫਿਗਰੇਸ਼ਨ ਡਾਟਾ ਲਈ ਸਮਰਪਿਤ ਡੂੰਘੇ ਬਟੂੰਨੇ ਵਰਤੋਂ  
- **ਅੰਤ-ਤੋਂ-ਅੰਤ ਇਨਕ੍ਰਿਪਸ਼ਨ**: ਸਾਰੇ ਐਪਲੀਕੇਸ਼ਨ ਹਿੱਸਿਆਂ ਅਤੇ ਡਾਟਾ ਪ੍ਰਵਾਹਾਂ ਵਿੱਚ ਸੁਰੱਖਿਆਤਮਕ ਸੰਚਾਰ  
- **ਇਨਪੁਟ ਜਾਂਚ**: ਸਾਰੇ ਉਪਭੋਗਤਾ ਇਨਪੁਟ, API ਪੈਰਾਮੀਟਰ ਅਤੇ ਡਾਟਾ ਸਰੋਤਾਂ ਦੀ ਕਠੋਰ ਜਾਂਚ  

#### **ਡਾਂਚਾਗਤ ਮਜ਼ਬੂਤੀ**  
- **ਮਲਟੀ-ਫੈਕਟਰ ਪ੍ਰਮਾਣਿਕਤਾ**: ਸਾਰੇ ਪ੍ਰਸ਼ਾਸਕੀ ਅਤੇ ਸੇਵਾ ਖਾਤਿਆਂ ਲਈ ਜ਼ਰੂਰੀ MFA  
- **ਪੈਚ ਪ੍ਰਬੰਧਨ**: ਓਪਰੇਟਿੰਗ ਸਿਸਟਮ, ਫਰੇਮਵਰਕ ਅਤੇ ਨਿਰਭਰਤਾਂ ਲਈ ਆਟੋਮੈਟਿਕ ਤੇ ਸਮੇਂ-ਸਿਰ ਪੈਚਿੰਗ  
- **ਪਹਿਚਾਣ ਪ੍ਰਦਾਤਾ ਇੰਟੀਗ੍ਰੇਸ਼ਨ**: ਐਨਟਰਪ੍ਰਾਈਜ਼ ਪਹਿਚਾਣ ਪ੍ਰਦਾਤਾ (Microsoft Entra ID, ਐਕਟਿਵ ਡਾਇਰੈਕਟਰੀ) ਰਾਹੀਂ ਕੇਂਦਰਿਤ ਪਹਿਚਾਣ ਪ੍ਰਬੰਧਨ  
- **ਨੈੱਟਵਰਕ ਵੱਖਰੇਪਣ**: ਪਾਰਸ਼ਵਿਕ ਚਲਣ ਸੰਭਾਵਨਾਵਾਂ ਨੂੰ ਸੀਮਤ ਕਰਨ ਲਈ MCP ਹਿੱਸਿਆਂ ਦਾ ਤਰਕਿ ਬੱਧ ਵੱਖਰਾ ਕਰਨਾ  
- **ਘੱਟੋ ਘੱਟ ਅਧਿਕਾਰ ਦਾ ਮਨੋਰਥ**: ਸਾਰੇ ਸਿਸਟਮ ਹਿੱਸਿਆਂ ਅਤੇ ਖਾਤਿਆਂ ਵਾਸਤੇ ਘੱਟੋ ਘੱਟ ਲੋੜੀਂਦੇ ਅਧਿਕਾਰ  

#### **ਸੁਰੱਖਿਆ ਮਾਨੀਟਰਨ ਅਤੇ ਪਛਾਣ**  
- **ਵਿਆਪਕ ਲੌਗਿੰਗ**: AI ਐਪਲੀਕੇਸ਼ਨ ਕਾਰਜਕਲਾਪਾਂ ਦਾ ਵਿਸ਼ਤ੍ਰਿਤ ਲੌਗ, ਜਿਸ ਵਿੱਚ MCP ਗਾਹਕ-ਸਰਵਰ ਕਿਰਿਆਵਾਂ ਸ਼ਾਮਲ ਹਨ  
- **SIEM ਇੰਟੀਗ੍ਰੇਸ਼ਨ**: ਵਿਸ਼ੇਸ਼ ਲੱਛਣ ਪਛਾਣ ਲਈ ਕੇਂਦਰੀ ਸੁਰੱਖਿਆ ਜਾਣਕਾਰੀ ਅਤੇ ਘਟਨਾ ਪ੍ਰਬੰਧਨ  
- **ਵਰਤਾਰਾ ਵਿਸ਼ਲੇਸ਼ਣ**: ਸਿਸਟਮ ਅਤੇ ਉਪਭੋਗਤਾ ਵਰਤਾਰਾ ਵਿੱਚ ਅਤਿਅਨੋਖੇ ਰੁੱਖਾਂ ਦੀ ਪਛਾਣ ਲਈ AI-ਚਾਲਿਤ ਮਾਨੀਟਰਨ  
- **ਧਮਕੀ ਖ਼ੁਫੀਆ ਜਾਣਕਾਰੀ**: ਬਾਹਰੀ ਧਮਕੀ ਫੀਡਾਂ ਅਤੇ ਸੰਕਟ ਚਿੰਨ੍ਹਾਂ (IOCs) ਦਾ ਸੰਮੇਲਨ  
- **ਘਟਨਾ ਜਵਾਬ**: ਸੁਰੱਖਿਆ ਘਟਨਾਵਾਂ ਦੀ ਪਛਾਣ, ਜਵਾਬ ਅਤੇ ਪੁਨਰ ਪ੍ਰਾਪਤੀ ਲਈ ਸੁਸਤ ਪ੍ਰਕਿਰਿਆਵਾਂ  

#### **ਜ਼ੀਰੋ ਟਰਸਟ ਆਰਕੀਟੈਕਚਰ**  
- **ਕਦੇ ਭਰੋਸਾ ਨਹੀਂ, ਹਮੇਸ਼ਾ ਜਾਂਚੋ**: ਉਪਭੋਗਤਾਵਾਂ, ਡਿਵਾਇਸਾਂ ਅਤੇ ਨੈੱਟਵਰਕ ਕਨੈਕਸ਼ਨਾਂ ਦੀ ਲਗਾਤਾਰ ਜਾਂਚ  
- **ਮਾਈਕ੍ਰੋ-ਵਭਾਜਨ**: ਵਿਆਪਕ ਕੰਟਰੋਲ ਜੋ ਵਿਅਕਤੀਗਤ ਵਰਕਲੋਡ ਅਤੇ ਸੇਵਾਵਾਂ ਨੂੰ ਤਨਹਾ ਕਰਦੇ ਹਨ  
- **ਪਹਿਚਾਣ-ਕੇਂਦ੍ਰਿਤ ਸੁਰੱਖਿਆ**: ਨੈੱਟਵਰਕ ਸਥਾਨ ਨਾਲ ਨਹੀਂ ਸਗੋਂ ਪਹਿਚਾਣ ਦੀ ਪੁਸ਼ਟੀ ਤੇ ਅਧਾਰਿਤ ਸੁਰੱਖਿਆ ਨੀਤੀਆਂ  
- **ਨਿਰੰਤਰ ਖ਼ਤਰਾ ਮੁਲਾਂਕਣ**: ਮੌਜੂਦਾ ਸੰਦਰਭ ਅਤੇ ਵਰਤਾਰਾ ਅਧਾਰਿਤ ਗਤੀਸ਼ੀਲ ਸੁਰੱਖਿਆ ਅਵਸਥਾ  
- **ਸ਼ਰਤੀ ਪਹੁੰਚ**: ਖ਼ਤਰੇ ਦੇ ਤੱਤਾਂ, ਸਥਾਨਕਤਾ ਅਤੇ ਡਿਵਾਇਸ ਭਰੋਸੇ 'ਤੇ ਅਧਾਰਿਤ ਪਹੁੰਚ ਨਿਯੰਤਰਣ  

### ਉਦਯੋਗਿਕ ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਪਿਰਾਮਿਡ

#### **ਮਾਈਕ੍ਰੋਸੌਫਟ ਸੁਰੱਖਿਆ ਪਰਿਬੇਸ਼ ਇੰਟੀਗ੍ਰੇਸ਼ਨ**  
- **Microsoft Defender for Cloud**: ਵਿਸਤ੍ਰਿਤ ਕਲਾਉਡ ਸੁਰੱਖਿਆ ਧਾਂਚਾ ਪ੍ਰਬੰਧਨ  
- **Azure Sentinel**: AI ਕਾਰਜਲੋਡ ਸੁਰੱਖਿਆ ਵਾਸਤੇ ਕਲਾਉਡ-ਨੈਟਿਵ SIEM ਅਤੇ SOAR ਸਮਰੱਥਾਵਾਂ  
- **Microsoft Entra ID**: ਸ਼ਰਤੀ ਪਹੁੰਚ ਨੀਤੀਆਂ ਨਾਲ ਏਨਟਰਪ੍ਰਾਈਜ਼ ਪਹਿਚਾਣ ਅਤੇ ਪਹੁੰਚ ਪ੍ਰਬੰਧਨ  
- **Azure Key Vault**: ਹਾਰਡਵੇਅਰ ਸੁਰੱਖਿਆ ਯੰਤਰ (HSM) ਸਮਰਥਿਤ ਕੇਂਦਰੀ ਸੀਕ੍ਰੇਟ ਪ੍ਰਬੰਧਨ  
- **Microsoft Purview**: AI ਡਾਟਾ ਸਰੋਤਾਂ ਅਤੇ ਕਾਰਜਪ੍ਰਣਾਲੀਆਂ ਲਈ ਡਾਟਾ ਸ਼ਾਸਨ ਅਤੇ ਅਨੁਕੂਲਤਾ  

#### **ਅਨੁਕੂਲਤਾ ਅਤੇ ਸ਼ਾਸਨ**  
- **ਨਿਯਮਕ ਅਨੁਕੂਲਤਾ**: ਯਕੀਨੀ ਬਣਾਓ ਕਿ MCP ਇੰਪਲੀਮੈਂਟੇਸ਼ਨਾਂ ਉਦਯੋਗ-ਖਾਸ ਅਨੁਕੂਲਤਾ ਮਾਪਦੰਡ (GDPR, HIPAA, SOC 2) ਨੂੰ ਪੂਰਾ ਕਰਦੇ ਹਨ  
- **ਡਾਟਾ ਵਰਗੀਕਰਨ**: AI ਸਿਸਟਮਾਂ ਦੁਆਰਾ ਸੰਪਾਦਿਤ ਸੰਵੇਦਨਸ਼ੀਲ ਡਾਟਾ ਦੀ ਸਹੀ ਵਰਗੀਕਰਨ ਅਤੇ ਸਾਂਭ  
- **ਆਡਿਟ ਟਰੇਲ**: ਨਿਯਮਾਂ ਦੀ ਪਾਲਣਾ ਅਤੇ ਫੌਰੇਨਸਿਕ ਜਾਂਚ ਲਈ ਵਿਸ਼ਤ੍ਰਿਤ ਲੌਗਿੰਗ  
- **ਪ੍ਰਾਈਵੇਟਸੀ ਕੰਟਰੋਲਜ਼**: AI ਸਿਸਟਮ ਆਰਕੀਟੈਕਚਰ ਵਿੱਚ ਡਿਜ਼ਾਈਨ ਤੋਂ ਪ੍ਰਾਈਵੇਟਸੀ ਨੂੰ ਲਾਗੂ ਕਰਨਾ  
- **ਤਬਦੀਲੀ ਪ੍ਰਬੰਧਨ**: AI ਸਿਸਟਮ ਬਦਲਾਅ ਦੀ ਸੁਰੱਖਿਆ ਸਮੀਖਿਆ ਲਈ ਫੌਰਮਲ ਪ੍ਰਕਿਰਿਆਵਾਂ  

ਇਹ ਮੂਲ ਅਭਿਆਸ ਇੱਕ ਮਜ਼ਬੂਤ ਸੁਰੱਖਿਆ ਤਮੱਲੀ ਬਣਾਂਦੇ ਹਨ ਜੋ MCP-ਵਿਸ਼ੇਸ਼ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣਾਂ ਦੀ ਪ੍ਰਭਾਵਸ਼ੀਲਤਾ ਨੂੰ ਵਧਾਉਂਦੇ ਅਤੇ AI-ਚਲਾਏ ਐਪਲੀਕੇਸ਼ਨਾਂ ਲਈ ਵਿਆਪਕ ਸੁਰੱਖਿਆ ਪ੍ਰਦਾਨ ਕਰਦੇ ਹਨ।

## ਮੁੱਖ ਸੁਰੱਖਿਆ ਸੰਖੇਪ
- **ਪਤਿ ਸੁਰੱਖਿਆ ਦ੍ਰਿਸ਼ਟੀਕੋਣ**: ਬੁਨਿਆਦੀ ਸੁਰੱਖਿਆ ਅਮਲਾਂ (ਸੁਰੱਖਿਅਤ ਕੋਡਿੰਗ, ਘੱਟ ਤੋਂ ਘੱਟ ਅਧਿਕਾਰ, ਸਪਲਾਈ ਚੇਨ ਜਾਂਚ, ਲਗਾਤਾਰ ਨਿਗਰਾਨੀ) ਨੂੰ ਏਆਈ-ਖਾਸ ਨੰਟਰੋਲਾਂ ਨਾਲ ਮਿਲਾ ਕੇ ਵਿਸ਼ਤ੍ਰਿਤ ਸੁਰੱਖਿਆ ਮੁਹੱਈਆ ਕਰੋ

- **ਏਆਈ-ਖਾਸ ਖ਼ਤਰੇ ਦਾ ਦ੍ਰਿਸ਼ਟੀਕੋਣ**: MCP ਸਿਸਟਮਾਂ ਨੂੰ ख़ਾਸ ਖ਼ਤਰਿਆਂ ਦਾ ਸਾਹਮਣਾ ਹੈ ਜਿਵੇਂ ਕਿ ਪ੍ਰਾਂਪਟ ਇੰਜੈਕਸ਼ਨ, ਟੂਲ ਜਹਿਰੀਲਾ ਕਰਨ, ਸੈਸ਼ਨ ਹੈਜੈਕਿੰਗ, ਭਰਮਾਤਮਕ ਡਿਪਿਊਟੀ ਮੁੱਦੇ, ਟੋਕਨ ਪਾਸ-ਥਰੂ ਕਮਜ਼ੋਰੀਆਂ ਅਤੇ ਅਧਿਕ ਮੰਜੂਰੀਆਂ, ਜਿਨ੍ਹਾਂ ਲਈ ਵਿਸ਼ੇਸ਼ ਰੋਕਥਾਮ ਦੀ ਲੋੜ ਹੈ

- **ਪਹਿਲਾ ਪਛਾਣ ਅਤੇ ਅਧਿਕਾਰਤਾ ਮਹਾਨਤਾ**: ਬਾਹਰੀ ਪਛਾਣ ਪ੍ਰਦਾਤਾਵਾਂ (Microsoft Entra ID) ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਮਜ਼ਬੂਤ ਪਛਾਣ ਨਿਯਮ ਲਾਗੂ ਕਰੋ, ਟੋਕਨਾਂ ਦੀ ਠੀਕ ਤਸਦੀਕ ਕਰੋ ਅਤੇ ਕਦੇ ਵੀ ਆਪਣੀ MCP ਸਰਵਰ ਲਈ ਖਾਸ ਤੌਰ 'ਤੇ ਜਾਰੀ ਨਾ ਕੀਤੇ ਟੋਕਨ ਨਾ ਮੰਨੋ

- **ਏਆਈ ਹਮਲਿਆਂ ਤੋਂ ਬਚਾਅ**: ਬਿਨਾਂ ਸਿੱਧੇ ਪ੍ਰਾਂਪਟ ਇੰਜੈਕਸ਼ਨ ਅਤੇ ਟੂਲ ਜਹਿਰੀਲਾ ਕਰਨ ਵਾਲੇ ਹਮਲਿਆਂ ਤੋਂ ਬਚाव ਲਈ Microsoft Prompt Shields ਅਤੇ Azure Content Safety ਨੂੰ ਤਾਇਨਾਤ ਕਰੋ, ਜਦੋਂ ਕਿ ਟੂਲ ਮੈਟਾਡੇਟਾ ਦੀ ਤਸਦੀਕ ਕਰੋ ਅਤੇ ਗਤਿਸੀਲ ਬਦਲਾਵਾਂ ਦੀ ਨਿਗਰਾਨੀ ਕਰੋ

- **ਸੈਸ਼ਨ ਅਤੇ ਟਰਾਂਸਪੋਰਟ ਸੁਰੱਖਿਆ**: ਵਿਉਕਤੀ ਪਛਾਣ ਨਾਲ ਬੰਨ੍ਹੇ ਕ੍ਰਿਪਟੋਗ੍ਰਾਫਿਕਲੀ ਸੁਰੱਖਿਅਤ, ਗੈਰ-ਨਿਯਤ ਸੈਸ਼ਨ ਆਈਡੀਜ਼ ਦਾ ਇਸਤੇਮਾਲ ਕਰੋ, ਠੀਕ ਸੈਸ਼ਨ ਜ਼ਿੰਦਗੀ ਚੱਕਰ ਪ੍ਰਬੰਧਨ ਲਾਗੂ ਕਰੋ, ਅਤੇ ਪਛਾਣ ਲਈ ਕਦੇ ਵੀ ਸੈਸ਼ਨ ਨਾ ਵਰਤੋ

- **OAuth ਸੁਰੱਖਿਆ ਦੇ ਬਿਹਤਰ ਅਮਲ**: ਗਤੀਸ਼ੀਲ ਰਜਿਸਟ੍ਰੇਸ਼ਨ ਕੀਤੇ ਗ੍ਰਾਹਕਾਂ ਲਈ ਸਪਸ਼ਟ ਯੂਜ਼ਰ ਸਹਿਮਤੀ ਦੁਆਰਾ ਭਰਮਾਤਮਕ ਡਿਪਿਊਟੀ ਹਮਲਿਆਂ ਨੂੰ ਰੋਕੋ, PKCE ਨਾਲ ਠੀਕ OAuth 2.1 ਲਾਗੂ ਕਰੋ ਅਤੇ ਸਖ਼ਤ redirect URI ਤਸਦੀਕ ਕਰੋ

- **ਟੋਕਨ ਸੁਰੱਖਿਆ ਸਿਧਾਂਤ**: ਟੋਕਨ ਪਾਸ-ਥਰੂ ਵਿਰੋਧਾ ਪੈਟਰਨ ਤੋਂ ਬਚੋ, ਟੋਕਨ ਦਰਸ਼ਕ ਦਾਅਵਿਆਂ ਦੀ ਤਸਦੀਕ ਕਰੋ, ਸੁਰੱਖਿਅਤ ਘੁੰਮਾਅ ਨਾਲ ਛੋਟੀ ਮਿਆਦ ਵਾਲੇ ਟੋਕਨਾਂ ਨੂੰ ਲਾਗੂ ਕਰੋ ਅਤੇ ਸਾਫ਼ ਵਿਸ਼ਵਾਸ ਸੀਮਾਵਾਂ ਦੀ ਰਖਿਆ ਕਰੋ

- **ਵਿਆਪਕ ਸਪਲਾਈ ਚੇਨ ਸੁਰੱਖਿਆ**: ਸਾਰੇ ਏਆਈ ਪਰਿਵਾਰਕ ਭਾਗਾਂ (ਮਾਡਲ, ਇੰਬੈਡਿੰਗਜ਼, ਸੰਦਰਭ ਪ੍ਰਦਾਤਾ, ਬਾਹਰੀ API) ਨੂੰ ਰਵਾਇਤੀ ਸੰਜੀਦਗੀ ਨਾਲ ਸੁਰੱਖਿਅਤ ਸੌਫਟਵੇਅਰ ਨਿਰਭਰਤਾਵਾਂ ਵਾਂਗ ਸਥਿਤ ਕਰੋ

- **ਲਗਾਤਾਰ ਵਿਕਾਸ**: ਤੇਜ਼ੀ ਨਾਲ ਬਦਲ ਰਹੀਆਂ MCP ਵਿਸ਼ੇਸ਼ਤਾਵਾਂ ਨਾਲ ਤਾਜ਼ਾ ਰਹੋ, ਸੁਰੱਖਿਆ ਕਮਿਊਨਿਟੀ ਮਿਆਰਾਂ ਵਿੱਚ ਯੋਗਦਾਨ ਪਾਓ, ਅਤੇ ਪ੍ਰੋਟੋਕਾਲ ਦੀ ਪੱਕਵਾਈ ਦੇ ਨਾਲ ਅਡਾਪਟਿਵ ਸੁਰੱਖਿਆ ਰੂਪਾਂਤਰਿਤ ਕਰਦੇ ਰਹੋ

- **Microsoft ਸੁਰੱਖਿਆ ਏਕੀਕਰਨ**: ਵਧੇਰੇ MCP ਤਾਇਨਾਤੀ ਸੁਰੱਖਿਆ ਲਈ Microsoft ਦੇ ਵਿਸ਼ਤ੍ਰਿਤ ਸੁਰੱਖਿਆ ਪਰਿਸਰ (Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID) ਦਾ ਲਾਭ ਉਠਾਓ
