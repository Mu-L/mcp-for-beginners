# 🚀 10 ਮਾਇਕਰੋਸਾਫਟ MCP ਸਰਵਰ ਜੋ ਵਿਕਾਸਕਾਰਾਂ ਦੀ ਉਤਪਾਦਕਤਾ ਬਦਲ ਰਹੇ ਹਨ

## 🎯 ਇਸ ਗਾਈਡ ਵਿੱਚ ਤੁਸੀਂ ਕੀ ਸਿੱਖੋਗੇ

ਇਹ ਵਿਹੰਗਮ ਗਾਈਡ ਦੱਸਦੀ ਹੈ ਦਸ ਮਾਇਕਰੋਸਾਫਟ MCP ਸਰਵਰਾਂ ਬਾਰੇ ਜੋ ਜੀਵੰਤ ਤੌਰ 'ਤੇ ਇਹ ਬਦਲ ਰਹੇ ਹਨ ਕਿ ਵਿਕਾਸਕਾਰ AI ਸਹਾਇਕਾਂ ਨਾਲ ਕਿਵੇਂ ਕੰਮ ਕਰਦੇ ਹਨ। ਸਿਰਫ ਇਹ ਨਹੀਂ ਦੱਸਿਆ ਜਾ ਰਿਹਾ ਕਿ MCP ਸਰਵਰ ਕੀ ਕਰ ਸਕਦੇ ਹਨ, ਬਲਕਿ ਅਸੀਂ ਤੁਹਾਨੂੰ ਉਹ ਸਰਵਰ ਦਿਖਾਊਂਗੇ ਜੋ ਪਹਿਲਾਂ ਹੀ ਮਾਇਕਰੋਸਾਫਟ ਅਤੇ ਹੋਰ ਥਾਵਾਂ ਤੇ ਰੋਜ਼ਾਨਾ ਵਿਕਾਸਕਾਰੀ ਵਲਕੀਰਾਂ ਨੂੰ ਵਾਸਤੇ ਅਸਲ ਅੰਤਰ ਲਿਆ ਰਹੇ ਹਨ।

ਇਸ ਗਾਈਡ ਵਿੱਚ ਹਰ ਸਰਵਰ ਅਸਲੀ ਵਰਤੋਂ ਅਤੇ ਵਿਕਾਸਕਾਰਾਂ ਦੀ ਸਾਂਝੀ ਰਾਏ ਦੇ ਅਧਾਰ 'ਤੇ ਚੁਣਿਆ ਗਿਆ ਹੈ। ਤੁਸੀਂ ਸਿਰਫ ਹਰ ਸਰਵਰ ਕੀ ਕਰਦਾ ਹੈ, ਇਹ ਨਹੀਂ ਜਾਣੋਗੇ, ਬਲਕਿ ਇਹ ਕਿਉਂ ਮਹੱਤਵਪੂਰਣ ਹੈ ਅਤੇ ਆਪਣੇ ਪ੍ਰਾਜੈਕਟਾਂ ਵਿੱਚ ਇਸ ਤੋਂ ਵੱਧ ਤੋਂ ਵੱਧ ਲਾਭ ਕਿਵੇਂ ਲੈ ਸਕਦੇ ਹੋ ਕੁਝ ਵੀ ਜਾਣੋਗੇ। ਚਾਹੇ ਤੁਸੀਂ MCP ਵਿੱਚ ਨਵੇਂ ਹੋ ਜਾਂ ਆਪਣਾ ਮੌਜੂਦਾ ਸੈਟਅਪ ਵਿਸਥਾਰ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ, ਇਹ ਸਰਵਰ ਮਾਇਕਰੋਸਾਫਟ ਪਰਿਵਾਰ ਵਿੱਚ ਕੁਝ ਸਭ ਤੋਂ ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਅਤੇ ਕਾਰਗਰ ਸੰਦ ਹਨ।

## ਦੇਖਣਾਤਮਕ

ਇਹ ਵਿਸਤਾਰਪੂਰਕ ਗਾਈਡ ਦਸ ਮਾਇਕਰੋਸਾਫਟ MCP ਸਰਵਰਾਂ ਦੀ ਛਾਨਬੀਨ ਕਰਦਾ ਹੈ ਜੋ ਵਿਕਾਸਕਾਰਾਂ ਦੇ AI ਸਹਾਇਕਾਂ ਅਤੇ ਬਾਹਰੀ ਟੂਲਾਂ ਨਾਲ ਸੰਬੰਧਬੱਧ ਹੋਣ ਦੇ ਤਰੀਕੇ ਨੂੰ ਬਦਲ ਰਹੇ ਹਨ। ਏਜ਼ੂਰ ਸਰੋਤ ਪ੍ਰਬੰਧਨ ਤੋਂ ਲੈ ਕੇ ਦਸਤਾਵੇਜ਼ ਪ੍ਰਕਿਰਿਆ ਤੱਕ, ਇਹ ਸਰਵਰ ਮਾਡਲ ਸੰਦੇਸ਼ ਪ੍ਰੋਟੋਕੋਲ ਦੀ ਤਾਕਤ ਦਿਖਾਉਂਦੇ ਹਨ ਜਿਸ ਨਾਲ ਨਿਰੰਤਰ, ਉਤਪਾਦਕ ਵਿਕਾਸਕਾਰੀ ਮਾਹੌਲ ਬਣਦਾ ਹੈ।

## ਸਿੱਖਣ ਦੇ ਲਕਸ਼

ਇਸ ਗਾਈਡ ਦੇ ਅਖੀਰ ਤੱਕ, ਤੁਸੀਂ:
- MCP ਸਰਵਰ ਵਿਕਾਸਕਾਰ ਦੀ ਉਤਪਾਦਕਤਾ ਨੂੰ ਕਿਵੇਂ ਬਹਿਤਰ ਬਣਾਉਂਦੇ ਹਨ ਸਮਝੋਗੇ
- ਮਾਇਕਰੋਸਾਫਟ ਦੇ ਸਭ ਤੋਂ ਪ੍ਰਭਾਵਸ਼ਾਲੀ MCP ਸਰਵਰੀ ਨਿਰਮਾਣ ਬਾਰੇ ਜਾਣੋ ਗੇ
- ਹਰ ਸਰਵਰ ਦੇ ਕਾਰਗਰ ਉਪਯੋਗ ਮਾਮਲੇ ਖੋਜੋਗੇ
- VS ਕੋਡ ਅਤੇ ਵਿਜ਼ੂਅਲ ਸਟੂਡੀਓ ਵਿੱਚ ਇਹਨਾਂ ਸਰਵਰਾਂ ਨੂੰ ਕਿਵੇਂ ਸੈਟਅਪ ਅਤੇ ਕਨਫਿਗਰ ਕਰਨਾ ਹੈ ਸ਼ਿੱਖੋਗੇ
- MCP ਦਾ ਵਿਸਤਾਰ ecosystem ਅਤੇ ਭਵਿੱਖ ਦੀਆਂ ਦਿਸ਼ਾਵਾਂ ਦਾ ਪਤਾ ਲਾੲੋਗੇ

## 🔧 MCP ਸਰਵਰਾਂ ਨੂੰ ਸਮਝਣਾ: ਆਮੜਾਂ ਲਈ ਮਾਰਗਦਰਸ਼ਕ

### MCP ਸਰਵਰ ਕੀ ਹਨ?

Model Context Protocol (MCP) ਵਿੱਚ ਨਵੇਂ ਹੁੰਦਿਆਂ, توهان ਸੋਚੋਗੇ: "ਅਸਲ ਵਿੱਚ MCP ਸਰਵਰ ਕੀ ਹੈ ਅਤੇ ਮੈਂ ਇਸ ਲਈ ਕਿਉਂ ਪਰਵਾਹ ਕਰਾਂ?" ਆਓ ਇੱਕ ਸਧਾਰਣ ਤਰਕ ਨਾਲ ਸ਼ੁਰੂ ਕਰੀਏ।

MCP ਸਰਵਰਾਂ ਨੂੰ ਵਿਸ਼ੇਸ਼ ਸਹਾਇਕਾਂ ਵਾਂਗ ਸਮਝੋ ਜੋ ਤੁਹਾਡੇ AI ਕੋਡਿੰਗ ਸਾਥੀ (ਜਿਵੇਂ GitHub Copilot) ਨੂੰ ਬਾਹਰੀ ਸੰਦ ਅਤੇ ਸੇਵਾਵਾਂ ਨਾਲ ਜੁੜਨ ਵਿੱਚ ਮਦਦ ਕਰਦੇ ਹਨ। ਜਿਵੇਂ ਤੁਸੀਂ ਫੋਨ ਤੇ ਵੱਖ-ਵੱਖ ਕੰਮਾਂ ਲਈ ਵੱਖ ਵੱਖ ਐਪਸ ਵਰਤਦੇ ਹੋ — ਇਕ ਮੌਸਮ ਲਈ, ਦੂਜਾ ਨੇਵੀਗੇਸ਼ਨ ਲਈ, ਤੀਜਾ ਬੈਂਕਿੰਗ ਲਈ — MCP ਸਰਵਰ ਤੁਹਾਡੇ AI ਸਹਾਇਕ ਨੂੰ ਵੱਖ-ਵੱਖ ਵਿਕਾਸਕਾਰੀ ਟੂਲ ਅਤੇ ਸੇਵਾਵਾਂ ਨਾਲ ਗੱਲਬਾਤ ਕਰਨ ਦੀ ਯੋਗਤਾ ਦਿੰਦੇ ਹਨ।

### MCP ਸਰਵਰਾਂ ਦੇ ਸਮੱਸਿਆ ਦਾ ਹੱਲ

MCP ਸਰਵਰਾਂ ਤੋਂ ਪਹਿਲਾਂ ਜੇ ਤੁਹਾਨੂੰ:
- ਆਪਣੇ ਏਜ਼ੂਰ ਸਰੋਤਾਂ ਨੂੰ ਚੈੱਕ ਕਰਨਾ
- ਗਿਟਹੱਬ ਇਸ਼ੂ ਬਣਾਉਣਾ
- ਆਪਣਾ ਡੇਟਾਬੇਸ ਪੁੱਛ ਗਿੱਛਣਾ
- ਦਸਤਾਵੇਜ਼ ਵਿੱਚ ਖੋਜ ਕਰਨੀ ਸੀ

ਤਾਂ ਤੁਹਾਨੂੰ ਕੋਡਿੰਗ ਰੋਕ ਕੇ, ਬਰਾਊਜ਼ਰ ਖੋਲ੍ਹ ਕੇ ਸਹੀ ਵੈੱਬਸਾਈਟ ਤੇ ਜਾ ਕੇ ਹੱਥੋਂ ਇਹ ਕਾਰਜ ਕਰਨੇ ਪੈਂਦੇ ਸਨ। ਇਹ ਲਗਾਤਾਰ ਸੰਦਰਭ ਬਦਲਨਾ ਤੁਹਾਡਾ ਧਿਆਨ ਭੰਗ ਕਰਦਾ ਅਤੇ ਉਤਪਾਦਕਤਾ ਘਟਾਉਂਦਾ ਸੀ।

### MCP ਸਰਵਰ ਤੁਹਾਡੇ ਵਿਕਾਸ ਅਨੁਭਵ ਨੂੰ ਕਿਵੇਂ ਬਦਲਦੇ ਹਨ

MCP ਸਰਵਰਾਂ ਨਾਲ, ਤੁਸੀਂ ਆਪਣੇ ਵਿਕਾਸ ਮਾਹੌਲ (VS ਕੋਡ, ਵਿਜ਼ੂਅਲ ਸਟੂਡੀਓ, ਆਦਿ) ਵਿੱਚ ਹੀ ਰਹਿ ਸਕਦੇ ਹੋ ਅਤੇ ਸਿਰਫ ਆਪਣੇ AI ਸਹਾਇਕ ਨੂੰ ਇਹ ਕੰਮ ਕਰਨ ਲਈ ਕਹਿ ਸਕਦੇ ਹੋ। ਉਦਾਹਰਨ ਲਈ:

**ਪਾਰੰਪਰਿਕ ਕਾਰਜਵਾਹੀ ਦੇ ਬਦਲੇ:**
1. ਕੋਡਿੰਗ ਰੋਕੋ
2. ਬਰਾਊਜ਼ਰ ਖੋਲ੍ਹੋ
3. ਏਜ਼ੂਰ ਪੋਰਟਲ ਤੇ ਜਾਓ
4. ਸਟੋਰੇਜ ਅਕਾਊਂਟ ਡੀਟੇਲ ਵੇਖੋ
5. VS ਕੋਡ ਤੇ ਵਾਪਸ ਆਓ
6. ਕੋਡਿੰਗ ਜਾਰੀ ਰੱਖੋ

**ਹੁਣ ਤੁਸੀਂ ਇਹ ਕਰ ਸਕਦੇ ਹੋ:**
1. AI ਨੂੰ ਪੁੱਛੋ: "ਮੇਰੇ ਏਜ਼ੂਰ ਸਟੋਰੇਜ ਅਕਾਊਂਟਾਂ ਦੀ ਹੈਲਥ ਕੀ ਹੈ?"
2. ਦਿੱਤੀ ਜਾਣਕਾਰੀ ਨਾਲ ਕੋਡਿੰਗ ਜਾਰੀ ਰੱਖੋ

### ਸ਼ੁਰੂਆਤਕਾਰਾਂ ਲਈ ਮੁੱਖ ਲਾਭ

#### 1. 🔄 **ਆਪਣੇ ਧਾਰਾ ਸਥਿਤੀ ਵਿੱਚ ਰਹੋ**
- ਬਹੁਤ ਸਾਰੇ ਐਪਸ ਵਿਚਕਾਰ ਬਦਲਣਾ ਨਹੀਂ
- ਤੁਹਾਡੇ ਲਿਖਣ ਵਾਲੇ ਕੋਡ 'ਤੇ ਧਿਆਨ ਜਾਰੀ ਰੱਖੋ
- ਵੱਖ-ਵੱਖ ਸੰਦਾਂ ਨੂੰ ਸੰਭਾਲਣ ਦੀ ਮਾਨਸਿਕ ਭਾਰ ਘਟਾਓ

#### 2. 🤖 **ਜਟਿਲ ਕਮਾਂਡਾਂ ਦੇ ਬਦਲੇ ਕੁਦਰਤੀ ਭਾਸ਼ਾ ਵਰਤੋਂ**
- SQL ਵਰਗਾ ਸੱਖਣਾ ਯਾਦ ਕਰਨ ਦੀ ਥਾਂ ਤੇ ਦੱਸੋ ਕਿ ਤੁਹਾਨੂੰ ਕਿਹੜੇ ਡੇਟਾ ਦੀ ਲੋੜ ਹੈ
- ਏਜ਼ੂਰ CLI ਕਮਾਂਡਾਂ ਨੂੰ ਯਾਦ ਕਰਨ ਦੀ ਥਾਂ ਤੇ ਦੱਸੋ ਕਿ ਤੁਸੀਂ ਕੀ ਪ੍ਰਾਪਤ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ
- AI ਨੂੰ ਤਕਨਕੀ ਵਿਸਥਾਰ ਸੰਭਾਲਣ ਦਿਓ, ਜਦੋਂ ਕਿ ਤੁਸੀਂ ਲੋਜਿਕ 'ਤੇ ਧਿਆਨ ਦਿੱਤਾ ਕਰੋ

#### 3. 🔗 **ਵੱਖ-ਵੱਖ ਸੰਦਾਂ ਨੂੰ ਇਕਠਾ ਜੋੜੋ**
- ਵੱਖਰੇ ਸੇਵਾਵਾਂ ਮਿਲਾ ਕੇ ਸ਼ਕਤੀਸ਼ਾਲੀ ਕਾਰਜਵਾਹੀਆਂ ਬਣਾਓ
- ਉਦਾਹਰਨ: "ਸਾਰੇ ਹਾਲੀਆ ਗਿਟਹੱਬ ਇਸ਼ੂ ਲਓ ਤੇ ਮੁਕਾਬਲਤ ਏਜ਼ੂਰ ਡਿਵ ਓਪਸ ਵਰਕ ਆਈਟਮ ਬਣਾਓ"
- ਜਟਿਲ ਸਕ੍ਰਿਪਟ ਲਿਖਣ ਤੋਂ ਬਿਨਾਂ ਆਟੋਮੇਸ਼ਨ ਬਣਾਓ

#### 4. 🌐 **ਇੱਕ ਵਧਦਾ ਹੋਇਆ ਪਰਿਵਾਰਤਨ ਪਾਓ**
- ਮਾਇਕਰੋਸਾਫਟ, ਗਿਟਹੱਬ, ਅਤੇ ਹੋਰ ਕੰਪਨੀਆਂ ਦੇ ਸਰਵਰਾਂ ਤੋਂ ਲਾਭ ਉਠਾਓ
- ਵੱਖ ਵੱਖ ਵਿਕਰੇਤਾਵਾਂ ਦੇ ਸੰਦਾਂ ਨੂੰ ਸੁਚੱਜੇ ਤੌਰ ਤੇ ਮਿਲਾਓ
- ਇੱਕ ਮਿਆਰੀਕ੍ਰਿਤ ਪਰਿਵਾਰਤ ਵਿੱਚ ਸ਼ਾਮਿਲ ਹੋ ਜਾਓ ਜੋ ਵੱਖ ਵੱਖ AI ਸਹਾਇਕਾਂ ਤੇ ਕੰਮ ਕਰਦਾ ਹੈ

#### 5. 🛠️ **ਕਰ ਕੇ ਸਿੱਖੋ**
- ਪਹਿਲਾਂ ਮੌਜੂਦ ਸਰਵਰਾਂ ਨਾਲ ਸੰਕਲਪ ਸਮਝੋ
- ਜਿਵੇਂ ਜਿਵੇਂ ਆਰਾਮਦਾਇਕ ਹੋਵੋ, ਆਪਣੇ ਸਰਵਰ ਬਣਾਉਣ ਵੱਲ ਵਧੋ
- ਉਪਲਬਧ SDK ਅਤੇ ਦਸਤਾਵੇਜ਼ਤੋਂ ਸਿੱਖਣ ਲਈ ਮਦਦ ਲਵੋ

### ਸ਼ੁਰੂਆਤਕਾਰਾਂ ਲਈ ਅਸਲੀ ਜਗਤ ਉਦਾਹਰਨ

ਮਾਨ ਲੋ ਤੁਸੀਂ ਵੈੱਬ ਵਿਕਾਸ ਚ ਨਵੇਂ ਹੋ ਅਤੇ ਪਹਿਲਾ ਪ੍ਰਾਜ਼ੈਕਟ ਕਰ ਰਹੇ ਹੋ। MCP ਸਰਵਰ ਅਜਿਹੇ ਮਦਦਗਾਰ ਹੋ ਸਕਦੇ ਹਨ:

**ਰਵਾਇਤੀ ਤਰੀਕਾ:**
```
1. Code a feature
2. Open browser → Navigate to GitHub
3. Create an issue for testing
4. Open another tab → Check Azure docs for deployment
5. Open third tab → Look up database connection examples
6. Return to VS Code
7. Try to remember what you were doing
```

**MCP ਸਰਵਰਾਂ ਨਾਲ:**
```
1. Code a feature
2. Ask AI: "Create a GitHub issue for testing this login feature"
3. Ask AI: "How do I deploy this to Azure according to the docs?"
4. Ask AI: "Show me the best way to connect to my database"
5. Continue coding with all the information you need
```

- **ਸੰਗਤਤਾ**: ਵੱਖ-ਵੱਖ ਸੰਦਾਂ ਅਤੇ ਕੰਪਨੀਆਂ ਵਿੱਚ ਇੱਕੋ ਜਿਹਾ ਅਨੁਭਵ
- **ਪਰਸਪਰ ਕ੍ਰਿਆਸ਼ੀਲਤਾ**: ਵੱਖ-ਵੱਖ ਵਿਕਰੇਤਾਵਾਂ ਦੇ ਸਰਵਰ ਇਕੱਠੇ ਕੰਮ ਕਰਦੇ ਹਨ
- **ਭਵਿੱਖ ਸਬੂਤੀ**: ਕੌਸ਼ਲ ਅਤੇ ਸੈਟਅਪ ਵੱਖ-ਵੱਖ AI ਸਹਾਇਕਾਂ ਵਿਚ ਟ੍ਰਾਂਸਫਰ ਹੁੰਦੇ ਹਨ
- **ਸਮੁਦਾਇ**: ਵੱਡਾ ਪਰਿਵਾਰਤਿਕ ਗਿਆਨ ਅਤੇ ਸਾਂਝਾ ਸਾਧਨ

### ਸ਼ੁਰੂਆਤੀ ਕਦਮ: ਤੁਸੀਂ ਕੀ ਸਿੱਖੋਂਗੇ

ਇਸ ਗਾਈਡ ਵਿੱਚ ਅਸੀਂ 10 ਮਾਇਕਰੋਸਾਫਟ MCP ਸਰਵਰ ਵੇਖਾਂਗੇ ਜੋ ਹਰ ਪੱਧਰ ਦੇ ਵਿਕਾਸਕਾਰਾਂ ਲਈ ਖਾਸ ਤੌਰ 'ਤੇ ਲਾਭਦਾਇਕ ਹਨ। ਹਰ ਸਰਵਰ ਤੈਅ ਕੀਤਾ ਗਿਆ ਹੈ:
- ਆਮ ਵਿਕਾਸਕਾਰੀ ਚੁਣੌਤੀਆਂ ਦਾ ਹੱਲ ਕਰਨ ਲਈ
- ਰੋਜ਼ ਦਾ ਕਰਮ ਘਟਾਉਣ ਲਈ
- ਕੋਡ ਦੀ ਗੁਣਵੱਤਾ ਨੂੰ ਬਹਿਤਰ ਬਣਾਉਣ ਲਈ
- ਸਿੱਖਣ ਦੇ ਮੌਕੇ ਵਧਾਉਣ ਲਈ

> **💡 ਸਿੱਖਣ ਸੁਝਾਅ**
>
> ਜੇ ਤੁਸੀਂ MCP ਵਿੱਚ ਸਿਰਫ ਨਵੇਂ ਹੋ, ਤਾਂ ਪਹਿਲਾਂ ਸਾਡੇ [MCP ਦਾ ਪਰਚਿਆ](../00-Introduction/README.md) ਅਤੇ [ਮੂਲ ਸੰਕਲਪ](../01-CoreConcepts/README.md) ਮੋਡੀਊਲ ਦੇਖੋ। ਫਿਰ ਇੱਥੇ ਵਾਪਸ ਆਓ ਤਾਂ ਜੋ ਤੁਸੀਂ ਇਹ ਸੰਕਲਪ ਅਸਲ ਮਾਇਕਰੋਸਾਫਟ ਸੰਦਾਂ ਨਾਲ ਵੇਖ ਸਕੋ।

> MCP ਦੀ ਮਹੱਤਤਾ ਬਾਰੇ ਹੋਰ ਜਾਣਕਾਰੀ ਲਈ ਮਾਰਿਆ ਨਾਗਾਗਾ ਦੀ ਪੋਸਟ ਵੀ ਵੇਖੋ: [Connect Once, Integrate Anywhere with MCP](https://devblogs.microsoft.com/blog/connect-once-integrate-anywhere-with-mcps)।

## VS ਕੋਡ ਅਤੇ ਵਿਜ਼ੂਅਲ ਸਟੂਡੀਓ ਵਿਚ MCP ਨਾਲ ਸ਼ੁਰੂਆਤ 🚀

ਜੇ ਤੁਸੀਂ Visual Studio Code ਜਾਂ Visual Studio 2022 ਨਾਲ GitHub Copilot ਵਰਤ ਰਹੇ ਹੋ, ਤਾਂ ਇਹ MCP ਸਰਵਰ ਸੈਟਅਪ ਕਰਨਾ ਸਧਾਰਣ ਹੈ।

### VS ਕੋਡ ਸੈਟਅਪ

ਇੱਥੇ VS ਕੋਡ ਲਈ ਮੁੱਢਲਾ ਪ੍ਰਕਿਰਿਆ ਹੈ:

1. **Agent ਮੋਡ ਚਾਲੂ ਕਰੋ**: VS ਕੋਡ ਵਿੱਚ Copilot ਚੈਟ ਵਿੰਡੋ ਵਿੱਚ Agent ਮੋਡ 'ਤੇ ਜਾਓ
2. **MCP ਸਰਵਰ ਕਨਫਿਗਰ ਕਰੋ**: ਆਪਣੇ VS ਕੋਡ settings.json ਫਾਈਲ ਵਿੱਚ ਸਰਵਰ ਵਿਵਰਣ ਸ਼ਾਮਿਲ ਕਰੋ
3. **ਸਰਵਰ ਚਾਲੂ ਕਰੋ**: ਉਹਨਾਂ ਸਰਵਰਾਂ ਲਈ "Start" ਬਟਨ 'ਤੇ ਕਲਿੱਕ ਕਰੋ ਜਿਨ੍ਹਾਂ ਨੂੰ ਵਰਤਣਾ ਚਾਹੁੰਦੇ ਹੋ
4. **ਟੂਲ ਚੁਣੋ**: ਆਪਣੀ ਮੌਜੂਦਾ ਸੈਸ਼ਨ ਲਈ ਕਿੜ੍ਹੇ MCP ਸਰਵਰ ਚਾਲੂ ਕਰਨੇ ਹਨ, ਚੁਣੋ

ਵਿਸਤਾਰ ਨਾਲ ਸੈਟਅਪ ਲਈ, ਵੇਖੋ [VS ਕੋਡ MCP ਦਸਤਾਵੇਜ਼](https://code.visualstudio.com/docs/copilot/copilot-mcp)।

> **💡 ਪ੍ਰੋ ਸੁਝਾਅ: MCP ਸਰਵਰਾਂ ਦਾ ਪ੍ਰਬੰਧਨ ਸੁਗਮ ਬਣਾਓ!**
>
> VS ਕੋਡ ਐਕਸਟੈਂਸ਼ਨ ਦ੍ਰਿਸ਼ теперь содержит [ਹੁਣ ਇੱਕ ਨਵਾਂ ਸਹੂਲਤਪੂਰਕ UI MCP ਸਰਵਰਾਂ ਲਈ](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode)! ਤੁਸੀਂ ਕਈ MCP ਸਰਵਰ ਚਾਲੂ, ਰੋਕਣ ਅਤੇ ਪ੍ਰਬੰਧਨ ਕਰ ਸਕਦੇ ਹੋ ਇੱਕ ਸਾਫ ਅਤੇ ਸਾਦਾ ਇੰਟਰਫੇਸ ਨਾਲ। ਜਰੂਰ ਕੋਸ਼ਿਸ਼ ਕਰੋ!

### ਵਿਜ਼ੂਅਲ ਸਟੂਡੀਓ 2022 ਸੈਟਅਪ

Visual Studio 2022 (ਵਰਜਨ 17.14 ਜਾਂ ਬਾਅਦ):

1. **Agent ਮੋਡ ਚਾਲੂ ਕਰੋ**: GitHub Copilot ਚੈਟ ਵਿੰਡੋ ਵਿੱਚ "Ask" ਡ੍ਰਾਪਡਾਊਨ 'ਤੇ ਕਲਿੱਕ ਕਰੋ ਅਤੇ "Agent" ਚੁਣੋ
2. **ਕਨਫਿਗਰੇਸ਼ਨ ਫਾਈਲ ਬਣਾਓ**: ਆਪਣੇ ਸੋਲੂਸ਼ਨ ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ `.mcp.json` ਫਾਈਲ ਬਣਾਓ (ਸਿਫਾਰਸ਼ੀ ਸਥਾਨ: `<SOLUTIONDIR>\.mcp.json`)
3. **ਸਰਵਰ ਕਨਫਿਗਰ ਕਰੋ**: ਮਿਆਰੀ MCP ਫਾਰਮੈਟ ਵਰਤਕੇ ਆਪਣੇ MCP ਸਰਵਰਾਂ ਦਾ ਵੇਰਵਾ ਦਿਓ
4. **ਟੂਲ ਦੀ ਮਨਜ਼ੂਰੀ**: ਜਦ ਮੰਗਿਆ ਜਾਏ, ਇਜ਼ਾਜ਼ਤ ਦਿਓ ਕਿ ਤੁਸੀਂ ਕਿਹੜੇ ਟੂਲ ਵਰਤਣਾ ਚਾਹੁੰਦੇ ਹੋ

ਵਿਸ਼ੇਸ਼ Visual Studio ਸੈਟਅਪ ਲਈ ਵੇਖੋ [Visual Studio MCP ਦਸਤਾਵੇਜ਼](https://learn.microsoft.com/visualstudio/ide/mcp-servers)।

ਹਰ MCP ਸਰਵਰ ਦੀਆਂ ਆਪਣੀਆਂ ਕਨਫਿਗਰੇਸ਼ਨ ਲੋੜਾਂ (ਕਨੈਕਸ਼ਨ ਸਟਰਿੰਗ, ਪ੍ਰਮਾਣਿਕਤਾ ਆਦਿ) ਹੁੰਦੀਆਂ ਹਨ, ਪਰ ਦੋਹਾਂ IDE ਵਿੱਚ ਸੈੱਟਅਪ ਦਾ ਢਾਂਚਾ ਮਿਲਦਾ ਜੁਲਦਾ ਹੈ।

### 2. ☁️ ਏਜ਼ੂਰ MCP ਸਰਵਰ
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

**ਇਹ ਕੀ ਕਰਦਾ ਹੈ**: Azure MCP ਸਰਵਰ 15+ ਵਿਸ਼ੇਸ਼ਿਤ Azure ਸੇਵਾ ਕਨੈਕਟਰਾਂ ਦਾ ਇੱਕ ਵਿਸਥਾਰਿਤ ਸੂਟ ਹੈ ਜੋ ਸਾਰੇ Azure ਇਕੋਸਿਸਟਮ ਨੂੰ ਤੁਹਾਡੇ AI ਵਰਕਫਲੋ ਵਿੱਚ ਲਿਆਉਂਦਾ ਹੈ। ਇਹ ਸਿਰਫ ਇੱਕ ਸਰਵਰ ਨਹੀਂ ਹੈ – ਇਹ ਇੱਕ ਸ਼ਕਤੀਸ਼ালী ਸੰਗ੍ਰਹਿ ਹੈ ਜਿਸ ਵਿੱਚ ਰਿਸੋਰਸ ਪ੍ਰਬੰਧਨ, ਡੇਟਾਬੇਸ ਕਨੈਕਟੀਵਿਟੀ (PostgreSQL, SQL Server), KQL ਨਾਲ Azure Monitor ਲੌਗ ਵਿਸ਼ਲੇਸ਼ਣ, Cosmos DB ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਅਤੇ ਹੋਰ ਬਹੁਤ ਕੁਝ ਸ਼ਾਮਲ ਹੈ।

**ਇਹ ਕਿਉਂ ਲਾਭਦਾਇਕ ਹੈ**: ਸਿਰਫ Azure ਰਿਸੋਰਸਾਂ ਦਾ ਪ੍ਰਬੰਧਨ ਹੀ ਨਹੀਂ, ਇਸ ਸਰਵਰ ਨਾਲ Azure SDK ਨਾਲ ਕੰਮ ਕਰਦਿਆਂ ਕੋਡ ਦੀ ਗੁਣਵੱਤਾ ਵਿੱਚ ਡ੍ਰੈਮੈਟਿਕ ਬਿਹਤਰੀ ਆਉਂਦੀ ਹੈ। ਜਦੋਂ ਤੁਸੀਂ Agent ਮੋਡ ਵਿੱਚ Azure MCP ਦੀ ਵਰਤੋਂ ਕਰਦੇ ਹੋ, ਇਹ ਸਿਰਫ ਕੋਡ ਲਿਖਣ ਵਿੱਚ ਸਹਾਇਤਾ ਨਹੀਂ ਕਰਦਾ – ਇਹ *ਵਧੀਆ* Azure ਕੋਡ ਲਿਖਣ ਵਿੱਚ ਮਦਦ ਕਰਦਾ ਹੈ ਜੋ ਕਿ ਮੌਜੂਦਾ ਪ੍ਰਮਾਣਿਕਤਾ ਪ੍ਰਣਾਲੀਆਂ, ਗਲਤੀ ਨਿਭਾਉਣ ਦੀਆਂ श्रेष्ठ ਪ੍ਰਥਾਵਾਂ ਦੇ ਪਾਲਣ ਕਰਦਾ ਹੈ ਅਤੇ ਨਵੀਂ SDK ਵਿਸ਼ੇਸ਼ਤਾਵਾਂ ਦਾ ਲਾਭ ਉਠਾਉਂਦਾ ਹੈ। ਬਿਨਾਂDEE generic ਕੋਡ ਦੇ ਤੁਹਾਨੂੰ ਉਹ ਕੋਡ ਮਿਲਦਾ ਹੈ ਜੋ ਕਾਰਗਰ ਹੈ, ਪਰ ਆਪਣੇ ਹੁਣ ਦੇ ਉਤਪਾਦਨ ਵਰਕਲੋਡਾਂ ਲਈ Azure ਦੀ ਸਿਫਾਰਸ਼ੀ ਪੈਟਰਨਾਂ ਦਾ ਪਾਲਣ ਕਰਦਾ ਹੈ।

**ਮੁੱਖ ਮੌਡੀਊਲ ਸ਼ਾਮਲ ਹਨ**:
- **🗄️ ਡੇਟਾਬੇਸ ਕਨੈਕਟਰ**: Azure ਡੇਟਾਬੇਸ ਲਈ PostgreSQL ਅਤੇ SQL Server ਦਾ ਸਿੱਧਾ ਕੁਦਰਤੀ ਭਾਸ਼ਾ ਐਕਸੈੱਸ
- **📊 Azure Monitor**: KQL-ਚਲਿਤ ਲੌਗ ਵਿਸ਼ਲੇਸ਼ਣ ਅਤੇ ਓਪਰੇਸ਼ਨਲ ਦ੍ਰਿਸ਼ਟੀ
- **🌐 ਰਿਸੋਰਸ ਪ੍ਰਬੰਧਨ**: Azure ਰਿਸੋਰਸ ਲਾਈਫਸਾਈਕਲ ਪ੍ਰਬੰਧਨ ਪੂਰਾ
- **🔐 ਪ੍ਰਮਾਣਿਕਤਾ**: DefaultAzureCredential ਅਤੇ ਪ੍ਰਬੰਧਿਤ ਪਛਾਣ ਪੈਟਰਨ
- **📦 ਸਟੋਰੇਜ ਸੇਵਾਵਾਂ**: Blob Storage, Queue Storage, ਅਤੇ Table Storage ਓਪਰੇਸ਼ਨ
- **🚀 ਕੰਟੇਨਰ ਸੇਵਾਵਾਂ**: Azure Container Apps, Container Instances, ਅਤੇ AKS ਪ੍ਰਬੰਧਨ
- **ਅਤੇ ਹੋਰ ਬਹੁਤ ਸਾਰੇ ਵਿਸ਼ੇਸ਼ ਕਨੈਕਟਰ**

**ਅਸਲੀ ਦੁਨੀਆ ਵਿੱਚ ਵਰਤੋਂ**: "ਮੇਰੇ Azure ਸਟੋਰੇਜ ਖਾਤਿਆਂ ਦੀ ਸੂਚੀ ਦਿਖਾਓ", "ਅਖੀਰੀ ਘੰਟੇ ਵਿੱਚ ਮੇਰੇ Log Analytics ਵਰਕਸਪੇਸ ਵਿਚ ਗਲਤੀਆਂ ਲਈ ਪੁੱਛ-ਤੱਛ ਕਰੋ", ਜਾਂ "ਸਹੀ ਪ੍ਰਮਾਣਿਕਤਾ ਦੇ ਨਾਲ Node.js ਵਰਤ ਕੇ Azure ਐਪਲੀਕੇਸ਼ਨ ਬਣਾਉਣ ਵਿੱਚ ਮੇਰੀ ਮਦਦ ਕਰੋ"

**ਪੂਰਾ ਡੈਮੋ ਸीनਾਰਿਓ**: ਇੱਥੇ ਇੱਕ ਪੂਰਾ ਵਾਕਫਲੋ ਹੈ ਜੋ Azure MCP ਨੂੰ GitHub Copilot ਫਾਰ Azure ਐਕਸਟੈਨਸ਼ਨ ਨਾਲ ਝੋਂਕ ਕੇ ਤਾਕਤ ਦਿਖਾਉਂਦਾ ਹੈ। ਜਦੋਂ ਤੁਹਾਡੇ ਕੋਲ ਦੋਹਾਂ ਇੰਸਟਾਲ ਹਨ ਅਤੇ ਤੁਸੀਂ ਪ੍ਰੰਪਟ ਦਿੰਦੇ ਹੋ:

> "ਇੱਕ Python ਸਕ੍ਰਿਪਟ ਬਣਾਓ ਜੋ DefaultAzureCredential ਪ੍ਰਮਾਣਿਕਤਾ ਨਾਲ Azure Blob Storage 'ਤੇ ਫਾਇਲ ਅਪਲੋਡ ਕਰੇ। ਸਕ੍ਰਿਪਟ ਨੂੰ ਮੇਰੇ Azure ਸਟੋਰੇਜ ਖਾਤੇ ਜਿਸਦਾ ਨਾਮ 'mycompanystorage' ਹੈ ਨਾਲ ਜੁੜਨਾ ਚਾਹੀਦਾ ਹੈ, 'documents' ਨਾਮ ਦੇ ਕੰਟੇਨਰ ਵਿੱਚ ਅਪਲੋਡ ਕਰਨਾ ਚਾਹੀਦਾ ਹੈ, ਸਮਕਾਲੀ ਟਾਈਮਸਟੈਂਪ ਨਾਲ ਇੱਕ ਟੈਸਟ ਫਾਇਲ ਬਣਾਉਂਦਾ ਹੋਵੇ, ਗਲਤੀਆਂ ਨੂੰ ਸੁਚੱਜੇ ਢੰਗ ਨਾਲ ਹੱਲ ਕਰਦਾ ਹੋਵੇ ਅਤੇ ਜਾਣਕਾਰੀਦਾਰ ਨਤੀਜੇ ਦਿੰਦਾ ਹੋਵੇ, ਪ੍ਰਮਾਣਿਕਤਾ ਅਤੇ ਗਲਤੀ ਨਿਭਾਉਣ ਲਈ Azure ਦੀਆਂ ਸਰੀਲ ਪ੍ਰਥਾ ਦਾ ਪਾਲਣ ਕਰਦਾ ਹੋਵੇ, DefaultAzureCredential ਪ੍ਰਮਾਣਿਕਤਾ ਕਿਵੇਂ ਕੰਮ ਕਰਦੀ ਹੈ ਇਸ ਬਾਰੇ ਟਿੱਪਣੀਆਂ ਸ਼ਾਮਲ ਕਰਦਾ ਹੋਵੇ, ਅਤੇ ਸਕ੍ਰਿਪਟ ਨੂੰ ਚੰਗੀ ਬਣਤਰ ਅਤੇ ਸਹੀ ਫੰਕਸ਼ਨਾਂ ਅਤੇ ਦਸਤਾਵੇਜ਼ੀਕਰਨ ਨਾਲ ਬਣਾਇਆ ਹੋਵੇ।"

Azure MCP ਸਰਵਰ ਇੱਕ ਪੂਰੀ, ਉਤਪਾਦਨ-ਤਿਆਰ Python ਸਕ੍ਰਿਪਟ ਪੈਦਾ ਕਰੇਗਾ ਜੋ:
- ਤਾਜ਼ਾ Azure Blob Storage SDK ਨੂੰ ਸਹੀ async ਪੈਟਰਨ ਨਾਲ ਵਰਤਦਾ ਹੈ
- DefaultAzureCredential ਨੂੰ ਸਮਪੂਰਨ fallback ਚੇਨ ਵਿਆਖਿਆ ਨਾਲ ਲਾਗੂ ਕਰਦਾ ਹੈ
- ਮਜ਼ਬੂਤ ਗਲਤੀ ਹੱਥ-ਮੱਦਦ ਨਹੀਂ Azure ਖਾਸ ਐਕਸਪਸ਼ਨ ਕਿਸਮਾਂ ਦੀ ਵਰਤੋਂ ਕਰਦਾ ਹੈ
- Azure SDK ਦੀਆਂ ਟੋਲੀ ਵਿਧੀਆਂ ਲਈ ਰਿਸੋਰਸ ਪ੍ਰਬੰਧਨ ਅਤੇ ਕੁਨੈਕਸ਼ਨ ਸੰਭਾਲ ਪਾਲਣਾ ਕਰਦਾ ਹੈ
- ਵਿਸਥਾਰ ਵਿੱਚ ਲੌਗਿੰਗ ਅਤੇ ਜਾਣਕਾਰੀਦਾਰ ਕੰਸੋਲ ਨਤੀਜੇ ਦਿੰਦਾ ਹੈ
- ਚੰਗੇਤਰ ਬਣਤਰ ਵਾਲੀ ਸਕ੍ਰਿਪਟ ਬਣਾਉਂਦਾ ਹੈ ਜਿਸ ਵਿੱਚ ਫੰਕਸ਼ਨ, ਦਸਤਾਵੇਜ਼ੀਕਰਨ ਅਤੇ ਟਾਈਪ ਹਿੰਟ ਸ਼ਾਮਲ ਹਨ

ਇਹ ਗੱਲ ਬਹੁਤ ਮਹੱਤਵਪੂਰਨ ਹੈ ਕਿ Azure MCP ਬਿਨਾਂ ਤੁਸੀਂ ਸੰਭਵਤ: ਸਧਾਰਣ ਬਲੌਬ ਸਟੋਰੇਜ ਕੋਡ ਪ੍ਰਾਪਤ ਕਰ ਸਕਦੇ ਹੋ ਜੋ ਕੰਮ ਤਾਂ ਕਰਦਾ ਹੈ ਪਰ ਮੌਜੂਦਾ Azure ਪੈਟਰਨਾਂ ਦਾ ਪਾਲਣ ਨਹੀਂ ਕਰਦਾ। Azure MCP ਨਾਲ ਤੁਹਾਨੂੰ ਐਜੇਹਾ ਕੋਡ ਮਿਲਦਾ ਹੈ ਜੋ ਨਵੀਂ ਪ੍ਰਮਾਣਿਕਤਾ ਵਿਧੀਆਂ, Azure ਖਾਸ ਗਲਤੀ ਸਥਿਤੀਆਂ ਨੂੰ ਸੰਭਾਲਦਾ ਹੈ ਅਤੇ ਪ੍ਰੋਡਕਸ਼ਨ ਐਪਲੀਕੇਸ਼ਨਾਂ ਲਈ ਮਾਇਕ੍ਰੋਸਾਫਟ ਦੀ ਸਿਫਾਰਸ਼ੀ ਪ੍ਰਥਾਵਾਂ ਦਾ ਪਾਲਣ ਕਰਦਾ ਹੈ।

**ਨਮੂਨਾ**: ਮੈਨੂੰ `az` ਅਤੇ `azd` CLI ਕਮਾਂਡਾਂ ਯਾਦ ਕਰਨ ਵਿੱਚ ਮੁਸ਼ਕਿਲ ਆਉਂਦੀ ਰਹੀ ਹੈ। ਮੇਰੇ ਲਈ ਇਹ ਹਮੇਸ਼ਾਂ ਇੱਕ ਦੋ-ਕਦਮੀ ਪ੍ਰਕਿਰਿਆ ਹੁੰਦੀ ਹੈ: ਪਹਿਲਾਂ ਸਿੰਟੈਕਸ ਨੂੰ ਵੇਖਣਾ, ਫਿਰ ਕਮਾਂਡ ਨੂੰ ਚਲਾਉਣਾ। ਮੈਨੂੰ ਅਕਸਰ ਪੋਰਟਲ ਵਿੱਚ ਜਾ ਕੇ ਕੰਮ ਕਰਨਾ ਪੈਂਦਾ ਹੈ ਕਿਉਂਕਿ ਮੈਂ ਸੀਐਲਆਈ ਸਿੰਟੈਕਸ ਯਾਦ ਨਹੀਂ ਰੱਖ ਸਕਦਾ। ਜੋ ਮੈਂ ਚਾਹੁੰਦਾ ਹਾਂ ਉਸਨੂੰ ਸਿਰਫ਼ ਦੱਸਣਾ ਅਤੇ ਆਪਣੀ IDE ਬਿਨਾ ਛੱਡੇ ਇਹ ਕਰਨਾ ਕਮਾਲ ਦੀ ਗੱਲ ਹੈ!

ਤੁਹਾਨੂੰ ਸ਼ੁਰੂ ਕਰਨ ਲਈ [Azure MCP ਰਿਪੋਜ਼ਟਰੀ](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) ਵਿੱਚ ਵਧੀਆ ਵਰਤੋਂ ਕੇਸਾਂ ਦੀ ਸੂਚੀ ਮਿਲੇਗੀ। ਵਿਸਥਾਰਿਤ ਸੈਟਅਪ ਗਾਈਡਾਂ ਅਤੇ ਉੱਚ ਪੱਧਰੀ ਸੰਰਚਨਾ ਵਿਕਲਪਾਂ ਲਈ [ਅਧਿਕਾਰਿਕ Azure MCP ਦਸਤਾਵੇਜ਼](https://learn.microsoft.com/azure/developer/azure-mcp-server/) ਵੇਖੋ।

- **🔔 ਸੂਚਨਾਵਾਂ**: ਸਮਾਰਟ ਸੂਚਨਾ ਪ੍ਰਬੰਧਨ ਅਤੇ ਰਿਪੋਜ਼ਟਰੀ ਸਬਸਕ੍ਰਿਪਸ਼ਨ ਕੰਟਰੋਲ

**ਅਸਲੀ ਦੁਨੀਆ ਵਿੱਚ ਵਰਤੋਂ**: "ਮੇਰੀ ਫੀਚਰ ਬ੍ਰਾਂਚ ਤੋਂ ਪੁਲ ਰਿਕਵੈਸਟ ਬਣਾਓ", "ਮੈਨੂੰ ਇਸ ਹਫ਼ਤੇ ਦੀਆਂ ਸਾਰੀਆਂ ਨਾਕਾਮ CI ਰਨਜ਼ ਦਿਖਾਓ", "ਮੇਰੇ ਰਿਪੋਜ਼ਟਰੀਜ਼ ਲਈ ਖੁਲ੍ਹੇ ਸੁਰੱਖਿਆ ਚੇਤਾਵਨੀਆਂ ਦੀ ਸੂਚੀ ਦਿਖਾਓ", ਜਾਂ "ਮੇਰੇ ਸਾਰੇ ਓਰਗਨਾਈਜ਼ੇਸ਼ਨਾਂ ਵਿੱਚ ਮੈਨੂੰ ਅਸਾਈਨ ਕੀਤੇ ਗਏ ਸਾਰੇ ਇਸ਼ੂਜ਼ ਲੱਭੋ"

**ਪੂਰਾ ਡੈਮੋ ਸીનਾਰਿਓ**: ਇੱਥੇ ਇੱਕ ਤਾਕਤਵਰ ਵਰਕਫਲੋ ਹੈ ਜੋ GitHub MCP ਸਰਵਰ ਦੀਆਂ ਸਮਰੱਥਾਵਾਂ ਨੂੰ ਦਰਸਾਉਂਦਾ ਹੈ:

> "ਮੈਨੂੰ ਸਪ੍ਰਿੰਟ ਸਮੀਖਿਆ ਲਈ ਤਿਆਰੀ ਕਰਨੀ ਹੈ। ਮੈਨੂੰ ਇਸ ਹਫਤੇ ਬਣਾਏ ਮੇਰੇ ਸਾਰੇ ਪੁਲ ਰਿਕਵੈਸਟ ਦਿਖਾਓ, ਸਾਡੇ CI/CD ਪਾਈਪਲਾਈਨਾਂ ਦੀ ਹਾਲਤ ਚੈੱਕ ਕਰੋ, ਕੋਈ ਵੀ ਸੁਰੱਖਿਆ ਚੇਤਾਵਨੀ ਜੋ ਸਾਨੂੰ ਨਿਭਾਣੀਆਂ ਹਨ ਉਨਾਂ ਦਾ ਇੱਕ ਸਾਰ ਸੰਕਲਪਿਤ ਕਰੋ, ਅਤੇ ਲੇਬਲ 'ਫੀਚਰ' ਵਾਲੇ ਮਿਲੇ ਹੋਏ PRs ਨੂੰ ਧਿਆਨ ਵਿੱਚ ਰੱਖ ਕੇ ਰਿਲੀਜ਼ ਨੋਟਸ ਬਣਾਉਣ ਵਿੱਚ ਮੇਰੀ ਮਦਦ ਕਰੋ।"

GitHub MCP ਸਰਵਰ:
- ਆਪਣੇ ਹਾਲੀਆ ਪੁਲ ਰਿਕਵੈਸਟਾਂ ਦੀ ਹਾਲਤ ਦੇ ਵੇਰਵੇ ਨਾਲ ਪੁੱਛ-ਤੱਛ ਕਰੇਗਾ
- ਵਰਕਫਲੋ ਚਲਾਓ ਦੀ ਵਿਸ਼ਲੇਸ਼ਣ ਕਰੇਗਾ ਅਤੇ ਕਿਸੇ ਵੀ ਅਸਫਲਤਾਵਾਂ ਜਾਂ ਪ੍ਰਵੇਸ਼ ਕਾਰਜਾਂ ਨੂੰ ਉਜਾਗਰ ਕਰੇਗਾ
- ਸੁਰੱਖਿਆ ਸਕੈਨਿੰਗ ਦੇ ਨਤੀਜਿਆਂ ਨੂੰ ਇकटਠਾ ਕਰਕੇ ਜਰੂਰੀ ਚੇਤਾਵਨੀਆਂ ਨੂੰ ਤਰਜੀਹ ਦੇਵੇਗਾ
- ਮਿਲੇ PRs ਤੋਂ ਜਾਣਕਾਰੀ ਕੱਢ ਕੇ ਪੂਰੀ ਰਿਲੀਜ਼ ਨੋਟਸ ਬਣਾਏਗਾ
- ਸਪ੍ਰਿੰਟ ਯੋਜਨਾ ਅਤੇ ਰਿਲੀਜ਼ ਤਿਆਰੀ ਲਈ ਕਾਰਗਰ ਅਗਲੇ ਕਦਮ ਮੁਹੱਈਆ ਕਰਾਵੇਗਾ

**ਨਮੂਨਾ**: ਮੈਂ ਕੋਡ ਸਮੀਖਿਆ ਵਰਕਫਲੋਜ਼ ਲਈ ਇਸ ਨੂੰ ਵਰਤਣਾ ਪਸੰਦ ਕਰਦਾ ਹਾਂ। VS Code, GitHub ਸੂਚਨਾਵਾਂ ਅਤੇ ਪੁਲ ਰਿਕਵੈਸਟ ਪੰਨਿਆਂ ਦੇ ਵਿਚਕਾਰ ਕੂਦਣ ਦੀ ਥਾਂ ਮੈਂ ਕਹਿ ਸਕਦਾ ਹਾਂ "ਮੇਰੀ ਸਮੀਖਿਆ ਲਈ ਸਾਰੇ PR ਦਿਖਾਓ" ਫਿਰ "PR #123 ਵਿੱਚ ਪ੍ਰਮਾਣਿਕਤਾ ਮੀਥਡ ਦੀ ਗਲਤੀ ਨਿਭਾਉਣ ਬਾਰੇ ਟਿੱਪਣੀ ਜੋੜੋ"। ਸਰਵਰ GitHub API ਕਾਲਾਂ ਸਾਂਭਦਾ ਹੈ, ਗੱਲਬਾਤ ਦਾ ਸੰਦਰਭ ਰੱਖਦਾ ਹੈ ਅਤੇ ਮੈਨੂੰ ਹੋਰ ਬਣਾਵਟੀ ਸਮੀਖਿਆ ਟਿੱਪਣੀਆਂ ਲਿਖਣ ਵਿੱਚ ਮਦਦ ਕਰਦਾ ਹੈ।

**ਪ੍ਰਮਾਣਿਕਤਾ ਵਿਕਲਪ**: ਸਰਵਰ OAuth (VS Code ਵਿੱਚ ਬਿਨਾ ਰੁਕਾਵਟ) ਅਤੇ Personal Access Tokens ਦੋਹਾਂ ਦਾ ਸਹਿਯੋਗ ਕਰਦਾ ਹੈ, ਜਿਸ ਨਾਲ ਤੁਸੀਂ ਕੇਵਲ GitHub ਫੰਕਸ਼ਨਲਿਟੀ ਨੂੰ ਹੀ ਯੋਗ ਕਰ ਸਕਦੇ ਹੋ। ਤੁਹਾਡੇ ਲਈ ਰਿਮੋਟ ਹੋਸਟਡ ਸਰਵਿਸ ਵਜੋਂ ਜਲਦੀ ਸੈਟਅਪ ਜਾਂ ਲੋਕਲ Docker ਰਾਹੀਂ ਪੂਰਾ ਕੰਟਰੋਲ ਚਲਾਉਣਾ ਸੰਭਵ ਹੈ।

> **💡 ਪੇਸ਼ੇਵਰ ਸੁਝਾਅ**
> 
> ਆਪਣੇ MCP ਸਰਵਰ ਸੈਟਿੰਗਜ਼ ਵਿੱਚ `--toolsets` ਪੈਰਾਮੀਟਰ ਨੂੰ ਸੰਰਚਿਤ ਕਰਕੇ ਕੇਵਲ ਜਰੂਰੀ ਟੂਲਸੈੱਟਸ ਚਾਲੂ ਕਰੋ ਤਾਂ ਜੋ ਸੰਦਰਭ ਦਾ ਆਕਾਰ ਘਟੇ ਅਤੇ AI ਟੂਲ ਚੋਣ ਵਿੱਚ ਸੁਧਾਰ ਹੋਵੇ। ਉਦਾਹਰਣ ਲਈ, ਆਪਣੀ MCP ਸੰਰਚਨਾ args ਵਿੱਚ `"--toolsets", "repos,issues,pull_requests,actions"` ਸ਼ਾਮਲ ਕਰੋ ਤਾ ਕਿ ਮੁੱਖ ਵਿਕਾਸ ਵਰਕਫਲੋਜ਼ ਦੌਰਾਨ ਇਹ ਕੰਮ ਕਰੇ, ਜਾਂ `"--toolsets", "notifications, security"` ਵਰਤੋਂ ਜੇਕਰ ਤੁਸੀਂ ਮੁੱਖ ਤੌਰ 'ਤੇ GitHub ਨਿਗਰਾਨੀ ਦੀਆਂ ਯੋਗਤਾਵਾਂ ਚਾਹੁੰਦੇ ਹੋ।

### 5. 📝 MarkItDown MCP ਸਰਵਰ

**ਇਹ ਕਿਉਂ ਲਾਭਦਾਇਕ ਹੈ**: ਆਧੁਨਿਕ ਦਸਤਾਵੇਜ਼ ਬਣਾਉਣ ਵਾਲੇ ਵਰਕਫਲੋਜ਼ ਲਈ ਜ਼ਰੂਰੀ! MarkItDown ਅਸਾਧਾਰਣ ਫਾਇਲ ਫਾਰਮੈਟਾਂ ਦੀ ਹਰ ਕਿਸਮ ਦੀ ਸੰਭਾਲ ਕਰਦਾ ਹੈ ਜਦਕਿ ਅਹੰਕਾਰਕ ਦਸਤਾਵੇਜ਼ ਸੰਰਚਨਾ ਜਿਵੇਂ ਕਿ ਸਿਰਲੇਖ, ਸੂਚੀਆਂ, ਟੇਬਲਾਂ ਅਤੇ ਲਿੰਕਾਂ ਨੂੰ ਬਚਾਉਂਦਾ ਹੈ। ਸਾਦੀ ਟੈਕਸਟ ਨਿਕਾਲਣ ਵਾਲੇ ਟੂਲਾਂ ਦੇ ਉਲਟ, ਇਹ ਸੈਮੈਂਟਿਕ ਅਰਥ ਅਤੇ ਫਾਰਮੈਟਿੰਗ ਵੇਵਸਤਿਤ ਕਰਦਾ ਹੈ ਜੋ ਦੋਹਾਂ ਐਆਈ ਪ੍ਰਕਿਰਿਆ ਅਤੇ ਮਾਨਵ ਪੜ੍ਹਨਯੋਗਤਾ ਲਈ ਕੀਮਤੀ ਹੈ।

**ਸਮਰਥਿਤ ਫਾਇਲ ਫਾਰਮੈਟ**:
- **ਦਫਤਰ ਦਸਤਾਵੇਜ਼**: PDF, PowerPoint (PPTX), Word (DOCX), Excel (XLSX/XLS)
- **ਮੀਡੀਆ ਫਾਇਲਾਂ**: ਚਿੱਤਰ (EXIF ਮੈਟਾਡੇਟਾ ਅਤੇ OCR ਦੇ ਨਾਲ), ਆਡੀਓ (EXIF ਮੈਟਾਡੇਟਾ ਅਤੇ ਸਪੀਚ ਟ੍ਰਾਂਸਕ੍ਰਿਪਸ਼ਨ ਦੇ ਨਾਲ)
- **ਵੈੱਬ ਸਮੱਗਰੀ**: HTML, RSS ਫੀਡ, ਯੂਟਿਊਬ URLs, ਵਿਕੀਪੀਡੀਆ ਪੰਨੇ
- **ਡਾਟਾ ਫਾਰਮੈਟ**: CSV, JSON, XML, ZIP ਫਾਇਲਾਂ (ਮੁੜ ਵਾਰ ਸਮੱਗਰੀ ਪ੍ਰੋਸੈਸ ਕਰਦਾ ਹੈ)
- **ਪਬਲਿਸ਼ਿੰਗ ਫਾਰਮੈਟ**: EPub, Jupyter ਨੋਟਬੁੱਕ (.ipynb)
- **ਈਮੇਲ**: Outlook ਸੁਨੇਹੇ (.msg)
- **ਅਡਵਾਂਸਡ**: Azure Document Intelligence ਇੰਟਿਗਰੇਸ਼ਨ ਨਾਲ ਬਿਹਤਰ PDF ਪ੍ਰੋਸੈਸਿੰਗ

**ਅਡਵਾਂਸਡ ਸਮਰੱਥਾ**: MarkItDown LLM-ਚਲਿਤ ਛਬੀਆਂ ਦੇ ਵਰਣਨ ਸਮਰਥਨ ਕਰਦਾ ਹੈ (ਜਦੋਂ OpenAI ਕਲਾਇੰਟ ਉਪਲਬਧ ਹੁੰਦਾ ਹੈ), ਬਿਹਤਰ PDF ਪ੍ਰੋਸੈਸਿੰਗ ਲਈ Azure Document Intelligence, ਸਪੀਚ ਸਮੱਗਰੀ ਲਈ ਆਡੀਓ ਟ੍ਰਾਂਸਕ੍ਰਿਪਸ਼ਨ, ਅਤੇ ਵੱਧ ਫਾਇਲ ਫਾਰਮੈਟਾਂ ਲਈ ਪਲੱਗਇਨ ਸਿਸਟਮ।

**ਅਸਲ ਜ਼ਿੰਦਗੀ ਵਿੱਚ ਵਰਤੋਂ**: "ਇਸ ਪਾਵਰਪਾਇੰਟ ਪ੍ਰਜ਼ੈਂਟੇਸ਼ਨ ਨੂੰ ਸਾਡੇ ਦਸਤਾਵੇਜ਼ੀ ਸਾਈਟ ਲਈ ਮਾਰਕਡਾਊਨ ਵਿੱਚ ਬਦਲੋ", "ਇਸ PDF ਤੋਂ ਠੀਕ ਸਿਰਲੇਖ ਸੰਰਚਨਾ ਨਾਲ ਟੈਕਸਟ ਨਿਕਾਲੋ", ਜਾਂ "ਇਸ Excel ਸ਼ੀਟ ਨੂੰ ਪੜ੍ਹਨਯੋਗ ਟੇਬਲ ਫਾਰਮੈਟ ਵਿੱਚ ਬਦਲੋ"

**ਨਮੂਨਾ**: [MarkItDown docs](https://github.com/microsoft/markitdown#why-markdown) ਤੋਂ ਇਤਹਾਸਤੌਰ 'ਤੇ ਕੋਟ:

MarkItDown ਦਸਤਾਵੇਜ਼ ਬਣਤਰ ਨੂੰ ਸੰਭਾਲਣ ਵਿੱਚ ਬਹੁਤ ਕੁਸ਼ਲ ਹੈ, ਜੋ ਕਿ AI ਵਰਕਫ਼ਲੋਜ਼ ਲਈ ਮਹੱਤਵਪੂਰਨ ਹੈ। ਉਦਾਹਰਨ ਲਈ, ਜਦੋਂ ਇਹ ਪਾਵਰਪਾਇੰਟ ਪ੍ਰਜ਼ੈਂਟੇਸ਼ਨ ਬਦਲਦਾ ਹੈ, ਤਾਂ ਇਹ ਸਹੀ ਸਿਰਲੇਖਾਂ ਨਾਲ ਸਲਾਈਡ ਆਯੋਜਨ ਨੂੰ ਬਣਾਈ ਰੱਖਦਾ ਹੈ, ਟੇਬਲਾਂ ਨੂੰ ਮਾਰਕਡਾਊਨ ਟੇਬਲਾਂ ਵਾਂਗ ਨਿਕਾਲਦਾ ਹੈ, ਚਿੱਤਰਾਂ ਲਈ alt ਟੈਕਸਟ ਸ਼ਾਮਿਲ ਕਰਦਾ ਹੈ, ਅਤੇ ਬੋਲਣ ਵਾਲੇ ਨੋਟ ਵੀ ਪ੍ਰੋਸੈਸ ਕਰਦਾ ਹੈ। ਚਾਰਟ ਸਹੀ ਡਾਟਾ ਟੇਬਲਾਂ ਵਿੱਚ ਬਦਲ ਜਾਂਦੇ ਹਨ, ਅਤੇ ਮਾਰਕਡਾਊਨ ਮੁਲ ਡਾਟਾ ਅਧਾਰ 'ਤੇ ਲਾਜ਼ਮੀ ਲਹਿਰ ਨੂੰ ਰੱਖਦਾ ਹੈ। ਇਹ ਪ੍ਰਜ਼ੈਂਟੇਸ਼ਨ ਸਮੱਗਰੀ ਨੂੰ AI ਸਿਸਟਮਾਂ ਵਿੱਚ ਫीड ਕਰਨ ਜਾਂ ਮੌਜੂਦਾ ਸਲਾਈਡਾਂ ਤੋਂ ਦਸਤਾਵੇਜ਼ ਬਣਾਉਣ ਲਈ ਬਹੁਤ ਉਪਯੋਗ ਹੈ।

**ਇਹ ਕਿਉਂ ਲਾਭਦਾਇਕ ਹੈ**: ਕੁਦਰਤੀ ਭਾਸ਼ਾ ਵਰਣਨਾਂ ਦੁਆਰਾ ਚਲਾਈ ਗਈ ਆਟੋਮੈਟਿੱਕ ਟੈਸਟਿੰਗ ਲਈ ਪਰਫੈਕਟ। AI ਵੈੱਬਸਾਈਟਾਂ ਤੇ ਸਮਰੂਪ ਅਕਸੇਸਯੋਗਤਾ ਸਨੈਪਸ਼ਾਟਾਂ ਰਾਹੀਂ ਟੈਸਟਿੰਗ, ਫਾਰਮ ਭਰਨ ਅਤੇ ਡਾਟਾ ਪ੍ਰਾਪਤ ਕਰ ਸਕਦਾ ਹੈ – ਇਹ ਬਹੁਤ ਹੀ ਸ਼ਕਤੀਸ਼ালী ਹੈ!

**ਅਸਲ ਵਰਤੋਂ**: "ਲੌਗਿਨ ਪ੍ਰਕਿਰਿਆ ਦੀ ਜਾਂਚ ਕਰੋ ਅਤੇ ਟੈਸਟ ਕਰੋ ਕਿ ਡੈਸ਼ਬੋਰਡ ਸਹੀ ਤਰ੍ਹਾਂ ਲੋਡ ਹੁੰਦਾ ਹੈ" ਜਾਂ "ਇਕ ਐਸਾ ਟੈਸਟ ਤਿਆਰ ਕਰੋ ਜੋ ਉਤਪਾਦ ਖੋਜੇ ਅਤੇ ਨਤੀਜੇ ਵਾਲਾ ਪੰਨਾ ਵੈਰੀਫਾਈ ਕਰੇ" – ਇਕ ਵੀ ਕੋਡ ਦੇਖਣ ਦੀ ਲੋੜ ਨਹੀਂ

**ਨਮੂਨਾ**: ਮੇਰੀ ਟੀਮ ਮਿਤ੍ਰ Debbie O'Brien ਨੇ ਹੁਣੇ Playwright MCP Server ਤੇ ਸ਼ਾਨਦਾਰ ਕੰਮ ਕੀਤਾ ਹੈ! ਉਦਾਹਰਨ ਲਈ, ਉਸਨੇ ਹਾਲ ਹੀ ਵਿੱਚ ਦਿਖਾਇਆ ਕਿ ਤੁਸੀਂ ਐਪ ਦਾ ਸੋర్స్ ਕੋਡ ਵੇਖਣ ਬਿਨਾਂ ਪੂਰਨ Playwright ਟੈਸਟ ਬਣਾਈ ਸਕਦੇ ਹੋ। ਉਸਦੇ ਮਾਮਲੇ ਵਿੱਚ, ਉਸਨੇ Copilot ਨੂੰ ਫਿਲਮ ਸਰਚ ਐਪ ਲਈ ਟੈਸਟ ਬਣਾਉਣ ਲਈ ਕਿਹਾ: ਸਾਈਟ ਤੇ ਜਾਓ, "Garfield" ਖੋਜੋ, ਅਤੇ ਫਿਲਮ ਨਤੀਜਿਆਂ ਵਿੱਚ ਹੈ ਕਿ ਨਹੀਂ ਜਾਂਚੋ। MCP ਇੱਕ ਬ੍ਰਾਊਜ਼ਰ ਸੈਸ਼ਨ ਸ਼ੁਰੂ ਕਰਦਾ ਹੈ, DOM ਸਨੈਪਸ਼ਾਟ्स ਨਾਲ ਸਾਈਟ ਢਾਂਚਾ ਸਮਝਦਾ ਹੈ, ਸਹੀ ਸੈਲੈਕਟਰ ਲੱਭਦਾ ਹੈ ਅਤੇ ਪਹਿਲੇ ਹੀ ਦੌਰ 'ਚ ਪਾਸ ਹੋਣ ਵਾਲਾ TypeScript ਟੈਸਟ ਤਿਆਰ ਕਰਦਾ ਹੈ।

ਇਸਦੀ ਖਾਸ ਤਾਕਤ ਇਹ ਹੈ ਕਿ ਇਹ ਕੁਦਰਤੀ ਭਾਸ਼ਾ ਨਿਰਦੇਸ਼ਾਂ ਅਤੇ ਚੱਲਣਯੋਗ ਟੈਸਟ ਕੋਡ ਵਿਚਕਾਰ ਪੁਲ ਬਣਾਉਂਦਾ ਹੈ। ਪਰੰਪਰਾਗਤ ਤਰੀਕਿਆਂ ਵਿੱਚ ਜਾਂ ਤਾਂ ਮੈਨੂਅਲ ਟੈਸਟ ਲੇਖਨ ਹੁੰਦਾ ਹੈ ਜਾਂ ਸੰਦਰਭ ਲਈ ਕੋਡ ਬੇਸ ਦੇਖਣਾ ਪੈਂਦਾ ਹੈ। ਪਰ Playwright MCP ਨਾਲ, ਤੁਸੀਂ ਬਿਹਰਲੀ ਸਾਈਟਾਂ, ਕਲਾਇੰਟ ਐਪਲੀਕੇਸ਼ਨ ਜਾਂ ਬਲੈਕ-ਬਾਕਸ ਟੈਸਟਿੰਗ ਸਸਥਿਤੀਆਂ ਵਿੱਚ ਵੀ ਇਮਤਿਹਾਨ ਲੈ ਸਕਦੇ ਹੋ ਜਿੱਥੇ ਕੋਡ ਤੱਕ ਪਹੁੰਚ ਨਹੀਂ।

**ਇਹ ਕੀ ਕਰਦਾ ਹੈ**: ਕੁਦਰਤੀ ਭਾਸ਼ਾ ਰਾਹੀਂ ਮਾਈਕਰੋਸੌਫਟ Dev Box ਮਾਹੌਲਾਂ ਦਾ ਪ੍ਰਬੰਧਨ ਕਰਦਾ ਹੈ

**ਇਹ ਕਿਉਂ ਲਾਭਦਾਇਕ ਹੈ**: ਵਿਕਾਸ ਦਾ ਮਾਹੌਲ ਪ੍ਰਬੰਧਨ ਬਹੁਤ ਆਸਾਨ ਕਰ ਦਿੰਦਾ ਹੈ! ਖਾਸ ਕਮਾਂਡ ਯਾਦ ਨਾ ਕਰਦੇ ਹੋਏ ਵਿਕਾਸ ਮਾਹੌਲ ਬਣਾਓ, ਸੰਰਚਿਤ ਕਰੋ ਅਤੇ ਮੈਨੇਜ ਕਰੋ।

**ਅਸਲ ਵਰਤੋਂ**: "ਨਵਾਂ Dev Box ਸੈਟ ਕਰੋ ਨਵੀਂ .NET SDK ਨਾਲ ਅਤੇ ਸਾਡੇ ਪ੍ਰੋਜੈਕਟ ਲਈ ਇਸਨੂੰ ਸੰਰਚਿਤ ਕਰੋ", "ਮੇਰੇ ਸਾਰੇ ਵਿਕਾਸ ਮਾਹੌਲਾਂ ਦੀ ਸਥਿਤੀ ਜਾਂਚੋ", ਜਾਂ "ਸਾਡੇ ਟੀਮ ਪ੍ਰਜ਼ੈਂਟੇਸ਼ਨ ਲਈ ਇੱਕ ਮਿਆਰੀਕ੍ਰਿਤ ਡੈਮੋ ਮਾਹੌਲ ਬਣਾਓ"

**ਨਮੂਨਾ**: ਮੈਂ Dev Box ਦੀ ਵੱਖਰੀ ਵਿਕਾਸ ਲਈ ਬਹੁਤ ਵੱਡਾ ਪ੍ਰਸ਼ੰਸਕ ਹਾਂ। ਮੇਰਾ ਰੌਸ਼ਨੀ ਪਲ ਉਹ ਸੀ ਜਦੋਂ James Montemagno ਨੇ ਦੱਸਿਆ ਕਿ ਕਨਫਰੰਸ ਡੈਮੋ ਲਈ Dev Box ਕਿੰਨਾ ਵਧੀਆ ਹੈ, ਕਿਉਂਕਿ ਇਸਦੀਆਂ ਬਹੁਤ ਤੇਜ਼ ਈਥਰਨੈੱਟ ਕਨੈਕਸ਼ਨ ਹਨ, ਚਾਹੇ ਕਨਫਰੰਸ / ਹੋਟਲ / ਹਵਾਈ ਜਹਾਜ਼ ਦਾ ਵਾਈਫਾਈ ਕੁਝ ਵੀ ਹੋਵੇ। ਹਾਲ ਹੀ ਵਿੱਚ ਮੈਂ ਬਸ 'ਚ Bruges ਤੋਂ Antwerp ਜਾਂਦੇ ਸਮੇਂ ਆਪਣੇ ਲੈਪਟਾਪ ਨੂੰ ਫੋਨ ਹੋਟਸਪੱਟ ਨਾਲ ਜੋੜ ਕੇ ਕਨਫਰੰਸ ਡੈਮੋ ਪ੍ਰੈਕਟਿਸ ਵੀ ਕੀਤੀ! ਮੇਰਾ ਅਗਲਾ ਕਦਮ ਟੀਮ ਨੂੰ ਕੁਝ ਵਿਅਕਤੀਗਤ ਵਿਕਾਸ ਮਾਹੌਲਾਂ ਅਤੇ ਮਿਆਰੀ ਡੈਮੋ ਮਾਹੌਲਾਂ ਦੇ ਪ੍ਰਬੰਧਨ ਵਿੱਚ ਡੂੰਘਾਈ ਨਾਲ ਜਾਣਾ ਹੈ। ਹੋਰ ਇਕ ਵੱਡਾ ਵਰਤੋਂ ਵਾਲਾ ਕੇਸ ਜੋ ਮੈਂ ਗ੍ਰਾਹਕਾਂ ਅਤੇ ਸਹਿ-ਕਰਮੀਆਂ ਤੋਂ ਸੁਣਿਆ ਹੈ, ਉਹ ਹੈ Dev Box ਦਾ ਪਹਿਲਾਂ ਤੋਂ ਸੰਰਚਿਤ ਵਿਕਾਸ ਮਾਹੌਲਾਂ ਲਈ ਵਰਤੋਂ। ਦੋਹਾਂ ਸਥਿਤੀਆਂ ਵਿੱਚ, MCP ਨਾਲ Dev Boxes ਸੰਰਚਿਤ ਅਤੇ ਪ੍ਰਬੰਧਿਤ ਕਰਨਾ ਕੁਦਰਤੀ ਭਾਸ਼ਾ ਇੰਟਰੈਕਸ਼ਨ ਦੀ ਆਗਿਆ ਦਿੰਦਾ ਹੈ, ਅਤੇ ਤੁਸੀਂ ਆਪਣੀ ਵਿਕਾਸ ਮਾਹੌਲ ਵਿੱਚ ਹੀ ਰਹਿੰਦੇ ਹੋ।

**ਇਹ ਕੀ ਕਰਦਾ ਹੈ**: ਮਾਈਕਰੋਸਾਫਟ ਫਾਊਂਡਰੀ MCP ਸਰਵਰ ਵਿਕਾਸਕਾਰਾਂ ਨੂੰ ਏਜੁਰ ਦੇ AI ਪਰਿਵਾਰਤਨਿਕ ਪਰਿਸਰ ਨਾਲ ਪੂਰੀ ਤਰ੍ਹਾਂ ਪਹੁੰਚ ਪ੍ਰਦਾਨ ਕਰਦਾ ਹੈ, ਜਿਸ ਵਿੱਚ ਮਾਡਲ ਕੈਟਾਲੌਗ, ਡਿਪਲੋਇਮੈਂਟ ਪ੍ਰਬੰਧਨ, ਏਜੁਰ AI ਸੇਰਚ ਨਾਲ ਗਿਆਨ ਇੰਡੈਕਸਿੰਗ, ਅਤੇ ਮੁੱਲਾਂਕਣ ਸੰਦ ਸ਼ਾਮਲ ਹਨ। ਇਹ ਪ੍ਰਯੋਗਸ਼ਾਲਾ ਸਰਵਰ AI ਵਿਕਾਸ ਅਤੇ ਏਜੁਰ ਦੀ ਸ਼ਕਤੀਸ਼ਾਲੀ AI ਬੁਨਿਆਦ ਦੇ ਵਿਚਕਾਰ ਪੱਲਾ ਬਣਾਉਂਦਾ ਹੈ, ਜਿਸ ਨਾਲ AI ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਤੇਜ਼ੀ ਨਾਲ ਬਣਾਉਣ, ਡਿਪਲੋਇ ਕਰਨ ਅਤੇ ਮੁੱਲਾਂਕਣ ਕਰਨ ਵਿੱਚ ਸਹੂਲਤ ਮਿਲਦੀ ਹੈ।

**ਇਹ ਕਿਉਂ ਲਾਭਦਾਇਕ ਹੈ**: ਇਹ ਸਰਵਰ ਤੁਹਾਡੇ Azure AI ਸੇਵਾਵਾਂ ਨਾਲ ਕੰਮ ਕਰਨ ਦੇ ਢੰਗ ਨੂੰ ਬਦਲ ਦਿੰਦਾ ਹੈ, ਜੋ ਪ੍ਰਤੀਸਥਾਪਿਤ ਗ੍ਰੇਡ ਦੀਆਂ AI ਕਾਬਲੀਅਤਾਂ ਨੂੰ ਸਿੱਧਾ ਤੁਹਾਡੇ ਵਿਕਾਸਕਾਰੀ ਕੰਮ ਦੇ ਦੌਰਾਨ ਲਿਆਉਂਦਾ ਹੈ। Azure ਪੋਰਟਲ, ਦਸਤਾਵੇਜ਼, ਅਤੇ ਤੁਹਾਡੇ IDE ਵਿੱਚ ਬਦਲਾਅ ਕਰਨ ਦੀ ਬਜਾਏ, ਤੁਸੀਂ ਮਾਡਲ ਖੋਜ ਸਕਦੇ ਹੋ, ਸੇਵਾਵਾਂ ਡਿਪਲੋਇ ਕਰ ਸਕਦੇ ਹੋ, ਗਿਆਨੀ ਅਧਾਰਾਂ ਦਾ ਪ੍ਰਬੰਧ ਕਰ ਸਕਦੇ ਹੋ, ਅਤੇ ਕੁਦਰਤੀ ਭਾਸ਼ਾ ਕਮਾਂਡਾਂ ਰਾਹੀਂ AI ਪ੍ਰਦਰਸ਼ਨ ਦਾ ਮੁੱਲਾਂਕਣ ਕਰ ਸਕਦੇ ਹੋ। ਇਹ ਖਾਸ ਕਰਕੇ ਉਹਨਾਂ ਵਿਕਾਸਕਾਰਾਂ ਲਈ ਬਹੁਤ ਸ਼ਕਤੀਸ਼ਾਲੀ ਹੈ ਜੋ RAG (Retrieval-Augmented Generation) ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਬਣਾਉਂਦੇ ਹਨ, ਬਹੁ-ਮਾਡਲ ਡਿਪਲੋਇਮੈਂਟ ਪ੍ਰਬੰਧਿਤ ਕਰਦੇ ਹਨ, ਜਾਂ ਪੂਰੇ AI ਮੁੱਲਾਂਕਣ ਪਾਈਪਲਾਈਨਾਂ ਨੂੰ ਲਾਗੂ ਕਰਦੇ ਹਨ।

ਮਾਡਲ ਕੰਟੈਕਸਟ ਪ੍ਰੋਟੋਕੋਲ (MCP) ਇਹ ਬਦਲ ਰਿਹਾ ਹੈ ਕਿ ਵਿਕਾਸਕਾਰ ਕਿਵੇਂ AI ਸਹਾਇਕ ਅਤੇ ਬਾਹਰੀ ਸੰਦਾਂ ਨਾਲ ਇੰਟਰੇਕਟ ਕਰਦੇ ਹਨ। ਇਹ 10 ਮਾਈਕਰੋਸਾਫਟ MCP ਸਰਵਰ ਸਟੈਂਡਰਡ AI ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਦੀ ਸਮਰੱਥਾ ਦਰਸਾਉਂਦੇ ਹਨ, ਜੋ ਬਿਨਾਂ ਰੁਕਾਵਟ ਤਕਨੀਕੀ ਕਾਮਕਾਜ ਲਈ ਸਮਰੱਥਾ ਦਿੰਦੇ ਹਨ ਜਿਸ ਨਾਲ ਵਿਕਾਸਕਾਰ ਆਪਣੀ ਲਹਿਰ ਵਿੱਚ ਰਹਿੰਦੇ ਹਨ ਤੇ ਸ਼ਕਤੀਸ਼ਾਲੀ ਬਾਹਰੀ ਸਮਰੱਥਾਵਾਂ ਤੱਕ ਪਹੁੰਚ ਕਰਦੇ ਹਨ।

ਵਿਆਪਕ Azure ਪਰਿਵਾਰਤਨਿਕ ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਤੋਂ ਲੈ ਕੇ ਖਾਸ ਸੰਦਾਂ ਵਾਂਗ Playwright ਬਰਾਊਜ਼ਰ ਆਟੋਮੇਸ਼ਨ ਅਤੇ MarkItDown ਦਸਤਾਵੇਜ਼ ਪ੍ਰਕਿਰਿਆ, ਇਹ ਸਰਵਰ ਦਰਸਾਉਂਦੇ ਹਨ ਕਿ MCP ਕਿਵੇਂ ਵੱਖ-ਵੱਖ ਵਿਕਾਸ ਪਰਿਧੀਆਂ ਵਿੱਚ ਉਤਪਾਦਕਤਾ ਵਧਾ ਸਕਦਾ ਹੈ। ਮਿਆਰੀ ਪ੍ਰੋਟੋਕੋਲ ਇਹ ਯਕੀਨੀ ਬਣਾਉਂਦਾ ਹੈ ਕਿ ਇਹ ਸਾਰੇ ਸੰਦ ਇਕੱਠੇ ਸੁਚਾਰੂ ਤਰੀਕੇ ਨਾਲ ਕੰਮ ਕਰਦੇ ਹਨ, ਇੱਕ ਉੱਤਮ ਵਿਕਾਸ ਅਨੁਭਵ ਬਣਾਉਂਦੇ ਹਨ।

ਜਿਵੇਂ ਕਿ MCP ਪਰਿਵਾਰਤਨਿਕ ਵਿਚ ਗਤੀਸ਼ੀਲਤਾ ਆਉਂਦੀ ਰਹਿੰਦੀ ਹੈ, ਕਮੇਊਨਿਟੀ ਨਾਲ ਜੁੜੇ ਰਹਿਣਾ, ਨਵੀਂ ਸਰਵਰਾਂ ਨੂੰ ਖੋਜਣਾ, ਅਤੇ ਕਸਟਮ ਹੱਲ ਬਣਾਉਣਾ ਤੁਹਾਡੇ ਵਿਕਾਸ ਉਤਪਾਦਕਤਾ ਨੂੰ ਵੱਧ ਤੋਂ ਵੱਧ ਕਰਨ ਲਈ ਕੁੰਜੀ ਰਹੇਗਾ। MCP ਖੁੱਲਾ ਮਿਆਰ ਹੋਣ ਦੇ ਨਾਤੇ ਤੁਸੀਂ ਵੱਖ-ਵੱਖ ਵੈਂਡਰਾਂ ਦੇ ਸੰਦ ਮਿਲਾ ਕੇ ਆਪਣੇ ਖਾਸ ਲੋੜਾਂ ਲਈ ਪਰਫੈਕਟ ਕਾਰਜ ਪ੍ਰਣਾਲੀ ਬਣਾ ਸਕਦੇ ਹੋ।

## 🎯 ਅਭਿਆਸ

1. **ਇੰਸਟਾਲ ਅਤੇ ਕਨਫਿਗਰ ਕਰੋ**: ਆਪਣੇ VS ਕੋਡ ਵਾਤਾਵਰਣ ਵਿੱਚ ਕਿਸੇ MCP ਸਰਵਰ ਨੂੰ ਸੈਟਅੱਪ ਕਰੋ ਅਤੇ ਬੁਨਿਆਦੀ ਕਾਰਜਸ਼ੀਲਤਾ ਦੀ ਜਾਂਚ ਕਰੋ।
2. **ਵਰਕਫਲੋ ਇੰਟੀਗ੍ਰੇਸ਼ਨ**: ਤਿੰਨ ਵੱਖ-ਵੱਖ MCP ਸਰਵਰਾਂ ਨੂੰ ਜੋੜ ਕੇ ਇੱਕ ਵਿਕਾਸ ਵਰਕਫਲੋ ਡਿਜ਼ਾਈਨ ਕਰੋ।
3. **ਕਸਟਮ ਸਰਵਰ ਯੋਜਨਾ**: ਆਪਣੇ ਰੋਜ਼ਾਨਾ ਵਿਕਾਸ ਕਾਰਜਾਂ ਵਿੱਚੋਂ ਇੱਕ ਐਸਾ ਟਾਸਕ ਪਛਾਣੋ ਜਿਸ ਨੂੰ ਕਸਟਮ MCP ਸਰਵਰ ਦੀ ਲੋੜ ਹੋ ਸਕਦੀ ਹੈ ਅਤੇ ਇਸਦਾ ਵਿਸ਼ੇਸ਼ਣ ਬਣਾਓ।
4. **ਕਾਰਗੁਜ਼ਾਰੀ ਵਿਸ਼ਲੇਸ਼ਣ**: ਆਮ ਵਿਕਾਸ ਕਾਰਜਾਂ ਲਈ MCP ਸਰਵਰਾਂ ਦੀ ਵਰਤੋਂ ਨੂੰ ਪਾਰੰਪਰਿਕ ਢੰਗ ਨਾਲੋਂ ਤੁਲਨਾ ਕਰੋ।
5. **ਸੁਰੱਖਿਆ ਮੁਲਾਂਕਣ**: ਆਪਣੇ ਵਿਕਾਸ ਵਾਤਾਵਰਣ ਵਿੱਚ MCP ਸਰਵਰਾਂ ਦੀ ਵਰਤੋਂ ਦੇ ਸੁਰੱਖਿਆ ਪ੍ਰਭਾਵਾਂ ਦਾ ਮੁਲਾਂਕਣ ਕਰੋ ਅਤੇ ਵਧੀਆ ਅਭਿਆਸ ਪੇਸ਼ ਕਰੋ।
