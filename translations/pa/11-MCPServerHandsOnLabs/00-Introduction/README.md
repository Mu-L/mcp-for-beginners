# MCP ਡੇਟਾਬੇਸ ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਦਾ ਪਰਿਚਯ

## 🎯 ਇਸ ਲੈਬ ਵਿੱਚ ਕੀ ਕਵਰ ਕੀਤਾ ਗਿਆ ਹੈ

ਇਹ ਪਰਿਚਯ ਲੈਬ ਮਾਡਲ ਸੰਦਰਭ ਪ੍ਰੋਟੋਕੋਲ (MCP) ਸਰਵਰਾਂ ਦੀ ਡੇਟਾਬੇਸ ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਨਾਲ ਬਣਾਉਣ ਦੀ ਇੱਕ ਵਿਸਥਾਰਤ ਜਾਨਕਾਰੀ ਪ੍ਰਦਾਨ ਕਰਦੀ ਹੈ। ਤੁਸੀਂ ਕਾਰੋਬਾਰੀ ਕੇਸ, ਤਕਨਕੀ ਬੁਨਿਆਦ, ਅਤੇ ਵਾਸਤਵਿਕ ਦੁਨੀਆ ਦੇ ਕਾਰਜਾਂ ਨੂੰ https://github.com/microsoft/MCP-Server-and-PostgreSQL-Sample-Retail 'ਤੇ ਜਾਵਾ ਰੀਟੇਲ ਵਿਸ਼ਲੇਸ਼ਣ ਉਪਯੋਗ ਕੇਸ ਰਾਹੀਂ ਸਮਝੋਗੇ।

## ਉਦਘਾਟਨ

**ਮਾਡਲ ਸੰਦਰਭ ਪ੍ਰੋਟੋਕੋਲ (MCP)** AI ਸਹਾਇਕਾਂ ਨੂੰ ਸੁਰੱਖਿਅਤ ਤਰੀਕੇ ਨਾਲ ਵਾਹਰੀ ਡੇਟਾ ਸਰੋਤਾਂ ਤੱਕ ਤੁਰੰਤ ਪਹੁੰਚ ਅਤੇ ਇੰਟਰਐਕਟ ਕਰਨ ਯੋਗ ਬਨਾਉਂਦਾ ਹੈ। ਜਦੋਂ ਇਹ ਡੇਟਾਬੇਸ ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਨਾਲ ਮਿਲਾਇਆ ਜਾਂਦਾ ਹੈ ਤਾਂ MCP ਡੇਟਾ-ਚਲਿਤ AI ਐਪਲੀਕੇਸ਼ਨਾਂ ਲਈ ਸ਼ਕਤਿਸ਼ਾਲੀ ਖ਼ੂਬੀਆਂ ਖੋਲ੍ਹਦਾ ਹੈ।

ਇਹ ਸਿੱਖਿਆ ਪ੍ਰਸਾਰ ਤੁਹਾਨੂੰ ਉਹਨਾਂ ਉਤਪਾਦਨ-ਤਿਆਰ MCP ਸਰਵਰਾਂ ਦੀ ਬਣਾਉਟ ਸਿਖਾਉਂਦਾ ਹੈ ਜੋ AI ਸਹਾਇਕਾਂ ਨੂੰ ਰੀਟੇਲ ਵਿਕਰੀ ਡੇਟਾ ਨਾਲ PostgreSQL ਰਾਹੀਂ ਜੋੜਦੇ ਹਨ, ਅਤੇ ਐਨਟਰਪ੍ਰਾਈਜ਼ ਪੈਟਰਨ ਜਿਵੇਂ ਕਿ ਰੋ ਲੈਵਲ ਸੁਰੱਖਿਆ, ਸੈਮਾਂਟਿਕ ਖੋਜ, ਅਤੇ ਬਹੁ-ਕਿਰਾਏਦਾਰ ਡੇਟਾ ਪਹੁੰਚ ਨੂੰ ਲਾਗੂ ਕਰਦੇ ਹਨ।

## ਸਿੱਖਣ ਦੇ ਉਦੇਸ਼

ਇਸ ਲੈਬ ਦੇ ਅੰਤ ਤੱਕ, ਤੁਸੀਂ ਸਮਰੱਥ ਹੋਵੋਗੇ ਕਿ:

- **ਮਾਡਲ ਸੰਦਰਭ ਪ੍ਰੋਟੋਕੋਲ ਦੀ ਪਰਿਭਾਸ਼ਾ** ਕਰੋ ਅਤੇ ਡੇਟਾਬੇਸ ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਲਈ ਇਸਦੇ ਮੁੱਖ ਫਾਇਦੇ ਵਿਆਖਿਆ ਕਰੋ  
- **ਮੁੱਖ ਹਿੱਸਿਆਂ ਦੀ ਪਛਾਣ** ਕਰੋ ਜੋ MCP ਸਰਵਰ ਆਰਕੀਟੈਕਚਰ ਵਿੱਚ ਡੇਟਾਬੇਸ ਨਾਲ ਹੁੰਦੇ ਹਨ  
- **ਜ਼ਾਵਾ ਰੀਟੇਲ ਉਪਯੋਗ ਕੇਸ ਅਤੇ ਇਸ ਦੀ ਕਾਰੋਬਾਰੀ ਜ਼ਰੂਰਤਾਂ** ਨੂੰ ਸਮਝੋ  
- **ਸੁਰੱਖਿਅਤ, ਵਿਆਪਕ ਡੇਟਾਬੇਸ ਪਹੁੰਚ ਲਈ ਐਨਟਰਪ੍ਰਾਈਜ਼ ਪੈਟਰਨਾਂ ਨੂੰ ਪਛਾਣੋ**  
- ਇਸ ਸਿੱਖਿਆ ਪ੍ਰਸਾਰ ਵਿੱਚ ਵਰਤੀਆਂ ਗਈਆਂ ਉਪਕਾਰਣਾਂ ਅਤੇ ਤਕਨਾਲੋਜੀਆਂ ਦੀ ਸੂਚੀ ਬਣਾਓ

### ਮੁੱਖ ਹਿੱਸੇ

#### **1. MCP ਸਰਵਰ ਪਰਤ**
- **FastMCP ਫਰੇਮਵਰਕ**: ਆਧੁਨਿਕ ਪਾਇਥਨ MCP ਸਰਵਰ ਲਾਗੂ  
- **ਟੂਲ ਰਜਿਸਟ੍ਰੇਸ਼ਨ**: ਪ੍ਰਕਾਰ ਸੁਰੱਖਿਆ ਨਾਲ ਘੋਸ਼ਿਤ ਟੂਲ ਪਰਿਭਾਸ਼ਾਵਾਂ  
- **ਰਿਕਵੇਸਟ ਸੰਦਰਭ**: ਉਪਭੋਗਤਾ ਪਛਾਣ ਅਤੇ ਸੈਸ਼ਨ ਪ੍ਰਬੰਧਨ  
- **ਗਲਤੀ ਮੈਨੇਜਮੈਂਟ**: ਮਜ਼ਬੂਤ ਗਲਤੀ ਸੰਭਾਲ ਅਤੇ ਲੌਗਿੰਗ  

#### **2. ਡੇਟਾਬੇਸ ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਪਰਤ**
- **ਕਨੈਕਸ਼ਨ ਪੂਲਿੰਗ**: ਪ੍ਰਭਾਵਸ਼ਾਲੀ asyncpg ਕਨੈਕਸ਼ਨ ਪ੍ਰਬੰਧਨ  
- **ਸਕੀਮਾ ਪ੍ਰਦਾਤਾ**: ਗਤੀਸ਼ੀਲ ਟੇਬਲ ਸਕੀਮਾ ਖੋਜ  
- **ਕੂਇਰੀ ਐਕਜ਼ਿਕਿਊਟਰ**: RLS ਸੰਦਰਭ ਨਾਲ ਸੁਰੱਖਿਅਤ SQL ਕਾਰਜ  
- **ਟ੍ਰਾਂਜ਼ੈਕਸ਼ਨ ਪ੍ਰਬੰਧਨ**: ACID ਪਾਲਣਾ ਅਤੇ ਰੋਲਬੈਕ ਸੰਭਾਲ  

#### **3. ਸੁਰੱਖਿਆ ਪਰਤ**
- **ਰੋ ਲੈਵਲ ਸੁਰੱਖਿਆ**: PostgreSQL RLS ਦੁਆਰਾ ਬਹੁ-ਕਿਰਾਏਦਾਰ ਡੇਟਾ ਅਲੱਗਾਵ  
- **ਉਪਭੋਗਤਾ ਪਛਾਣ**: ਸਟੋਰ ਮੈਨੇਜਰ ਪ੍ਰਮਾਣਿਕਤਾ ਅਤੇ ਅਧਿਕਾਰ  
- **ਪਹੁੰਚ ਨਿਯੰਤਰਣ**: ਲਚਕੀਲੇ ਅਧਿਕਾਰ ਅਤੇ ਆਡਿਟ ਟਰੇਲ  
- **ਇਨਪੁੱਟ ਵੈਧਤਾ**: SQL ਇੰਜਕਸ਼ਨ ਰੋਕਥਾਮ ਅਤੇ ਕੂਇਰੀ ਵੈਧਤਾ  

#### **4. AI ਸੁਧਾਰ ਪਰਤ**
- **ਸੈਮਾਂਟਿਕ ਖੋਜ**: ਉਤਪਾਦ ਖੋਜ ਲਈ ਵੈਕਟਰ ਐਂਬੈਡਿੰਗ  
- **ਐਜ਼ਯੂਰ OpneAI ਇੰਟੀਗ੍ਰੇਸ਼ਨ**: ਟੈਕਸਟ ਐਂਬੈਡਿੰਗ ਬਣਾਉਣਾ  
- **ਸਮਾਨਤਾ ਅਲਗੋਰਿਦਮ**: pgvector ਕੋਸੀਨ ਸਮਾਨਤਾ ਖੋਜ  
- **ਖੋਜ ਕਲਾ**: ਸੰਕੈਤਨ ਅਤੇ ਕਾਰਗੁਜ਼ਾਰੀ ਵਿੱਚ ਸੁਧਾਰ

## 🎬 ਵਾਸਤਵਿਕ ਦੁਨੀਆ ਉਪਯੋਗ ਹਾਲਾਤ

ਆਓ ਵੇਖੀਏ ਕਿਸ ਤਰ੍ਹਾਂ ਵੱਖ-ਵੱਖ ਉਪਭੋਗਤਾ ਸਾਡੇ MCP ਸਰਵਰ ਨਾਲ ਇੰਟਰਐਕਟ ਕਰਦੇ ਹਨ:

### ਸਿੱਕਾ 1: ਸਟੋਰ ਮੈਨੇਜਰ ਦੀ ਕਾਰਗੁਜ਼ਾਰੀ ਸਮੀਖਿਆ

**ਉਪਭੋਗਤਾ**: ਸਾਰਾਹ, ਸੀਏਟਲ ਸਟੋਰ ਮੈਨੇਜਰ  
**ਲਕਸ਼ਯ**: ਪਿਛਲੇ ਚੌਥਾਈ ਦੀ ਵਿਕਰੀ ਕਾਰਗੁਜ਼ਾਰੀ ਦਾ ਵਿਸ਼ਲੇਸ਼ਣ  

**ਕੁਦਰਤੀ ਭਾਸ਼ਾ ਕੂਇਰੀ**:  
> "ਮੇਰੇ ਸਟੋਰ ਲਈ Q4 2024 ਵਿੱਚ ਦਰਜ ਕੀਤੇ ਪ੍ਰਮੁੱਖ 10 ਉਤਪਾਦ ਦਿਖਾਓ"  

**ਕੀ ਹੁੰਦਾ ਹੈ**:  
1. VS ਕੋਡ AI ਚੈਟ ਕੂਇਰੀ MCP ਸਰਵਰ ਨੂੰ ਭੇਜਦਾ ਹੈ  
2. MCP ਸਰਵਰ ਸਾਰਾਹ ਦੇ ਸਟੋਰ ਸੰਦਰਭ (ਸੀਏਟਲ) ਨੂੰ ਪਛਾਣਦਾ ਹੈ  
3. RLS ਨੀਤੀਆਂ ਡੇਟਾ ਨੂੰ ਸਿਰਫ਼ ਸੀਏਟਲ ਸਟੋਰ ਲਈ ਛਾਣਨੁ ਕਰਦੀਆਂ ਹਨ  
4. SQL ਕੂਇਰੀ ਤਿਆਰ ਅਤੇ ਚਲਾਈ ਜਾਂਦੀ ਹੈ  
5. ਨਤੀਜੇ ਫਾਰਮੈਟ ਕਰਕੇ AI ਚੈਟ ਨੂੰ ਵਾਪਸ ਭੇਜੇ ਜਾਂਦੇ ਹਨ  
6. AI ਵਿਸ਼ਲੇਸ਼ਣ ਅਤੇ ਰਿਪੋਰਟ ਪ੍ਰਦਾਨ ਕਰਦਾ ਹੈ  

### ਸਿੱਕਾ 2: ਸੈਮਾਂਟਿਕ ਖੋਜ ਨਾਲ ਉਤਪਾਦ ਖੋਜ

**ਉਪਭੋਗਤਾ**: ਮਾਈਕ, ਇਨਵੈਂਟਰੀ ਮੈਨੇਜਰ  
**ਲਕਸ਼ਯ**: ਕਿਸੇ ਗਾਹਕ ਦੀ ਮੰਗ ਵਾਂਗ ਉਤਪਾਦ ਲੱਭੋ  

**ਕੁਦਰਤੀ ਭਾਸ਼ਾ ਕੂਇਰੀ**:  
> "ਕਿਹੜੇ ਉਤਪਾਦ ਅਸੀਂ ਵੇਚਦੇ ਹਾਂ ਜੋ 'ਬਾਹਰੀ ਵਰਤੋਂ ਲਈ ਵਾਟਰਪ੍ਰੂਫ਼ ਇਲੈਕਟ੍ਰਿਕਲ ਕਨੈਕਟਰ' ਦੇ ਸਮਾਨ ਹਨ?"  

**ਕੀ ਹੁੰਦਾ ਹੈ**:  
1. ਕੂਇਰੀ ਸੈਮਾਂਟਿਕ ਖੋਜ ਟੂਲ ਦੁਆਰਾ ਪ੍ਰਕਿਰਿਆਤਮਕ ਹੁੰਦੀ ਹੈ  
2. ਐਜ਼ਯੂਰ OpenAI ਐਂਬੈਡਿੰਗ ਵੈਕਟਰ ਤਿਆਰ ਕਰਦਾ ਹੈ  
3. pgvector ਸਮਾਨਤਾ ਖੋਜ ਕਰਦਾ ਹੈ  
4. ਸੰਬੰਧਿਤ ਉਤਪਾਦਾਂ ਨੂੰ ਮੇਲ_VALIDATE ਰਾਹੀਂ ਦਰਜ ਕੀਤਾ ਜਾਂਦਾ ਹੈ  
5. ਨਤੀਜਿਆਂ ਵਿੱਚ ਉਤਪਾਦ ਵੇਰਵਾ ਅਤੇ ਉਪਲਬਧਤਾ ਸ਼ਾਮਲ ਹੁੰਦੀ ਹੈ  
6. AI ਵਿਵਿਕਲਪ ਅਤੇ ਬੰਡਲਿੰਗ ਮੌਕੇ ਸਝਾਅ ਦਿੰਦਾ ਹੈ

**ਉਪਭੋਗਤਾ**: ਜੈਨਿਫਰ, ਖੇਤਰੀ ਮੈਨੇਜਰ  
**ਲਕਸ਼ਯ**: ਸਾਰੇ ਸਟੋਰਾਂ ਦੀ ਕਾਰਗੁਜ਼ਾਰੀ ਦੀ ਤੁਲਨਾ ਕਰੋ

**ਕੀ ਹੁੰਦਾ ਹੈ**:  
1. RLS ਸੰਦਰਭ ਖੇਤਰੀ ਮੈਨੇਜਰ ਪਹੁੰਚ ਲਈ ਸੈੱਟ ਕੀਤਾ ਜਾਂਦਾ ਹੈ  
2. ਜਟਿਲ ਬਹੁ-ਸਟੋਰ ਕੂਇਰੀ ਤਿਆਰ ਹੁੰਦੀ ਹੈ  
3. ਡੇਟਾ ਸਾਰੇ ਸਟੋਰ ਸਥਾਨਾਂ 'ਚ ਇਕੱਠਾ ਹੁੰਦਾ ਹੈ  
4. ਨਤੀਜੇ ਰੁਝਾਨ ਅਤੇ ਤੁਲਨਾਵਾਂ ਦੇ ਨਾਲ ਸ਼ਾਮਲ ਹੁੰਦੇ ਹਨ  
5. AI ਵਿਸ਼ਲੇਸ਼ਣ ਅਤੇ ਸਿਫਾਰਸ਼ਾਂ ਲਾਂਭਦਾ ਹੈ  

## 🔒 ਸੁਰੱਖਿਆ ਅਤੇ ਬਹੁ-ਕਿਰਾਏਦਾਰੀ ਵਿਚ ਡੂੰਘਾਈ

ਸਾਡੀ ਲਾਗੂ ਕਾਰਜਵਾਈ ਏਨਟਰਪ੍ਰਾਈਜ਼-ਗ੍ਰੇਡ ਸੁਰੱਖਿਆ ਨੂੰ ਪ੍ਰਾਥਮਿਕਤਾ ਦਿੰਦੀ ਹੈ।

### ਰੋ ਲੈਵਲ ਸੁਰੱਖਿਆ (RLS)

PostgreSQL RLS ਡੇਟਾ ਅਲੱਗਾਵ ਨੂੰ ਯਕੀਨੀ ਬਨਾਉਂਦਾ ਹੈ:

```sql
-- Store managers see only their store's data
CREATE POLICY store_manager_policy ON retail.orders
  FOR ALL TO store_managers
  USING (store_id = get_current_user_store());

-- Regional managers see multiple stores
CREATE POLICY regional_manager_policy ON retail.orders
  FOR ALL TO regional_managers
  USING (store_id = ANY(get_user_store_list()));
```


### ਉਪਭੋਗਤਾ ਪਛਾਣ ਪ੍ਰਬੰਧਨ

ਹਰ MCP ਕਨੈਕਸ਼ਨ ਵਿੱਚ ਸ਼ਾਮਲ ਹੈ:  
- **ਸਟੋਰ ਮੈਨੇਜਰ ID**: RLS ਸੰਦਰਭ ਲਈ ਯੂਨੀਕ ਪਹਚਾਣ  
- **ਭੂਮਿਕਾ ਨਿਯੁਕਤੀ**: ਅਧਿਕਾਰ ਅਤੇ ਪਹੁੰਚ ਪੱਧਰ  
- **ਸੈਸ਼ਨ ਪ੍ਰਬੰਧਨ**: ਸੁਰੱਖਿਅਤ ਪ੍ਰਮਾਣਿਕਤਾ ਟੋਕਨ  
- **ਆਡਿਟ ਲੌਗਿੰਗ**: ਪੂਰੀ ਪਹੁੰਚ ਇਤਿਹਾਸ  

### ਡੇਟਾ ਸੁਰੱਖਿਆ

ਕਈ ਸੁਰੱਖਿਆ ਪੱਧਰ:  
- **ਕਨੈਕਸ਼ਨ ਐਨਕ੍ਰਿਪਸ਼ਨ**: ਸਾਰੇ ਡੇਟਾਬੇਸ ਕਨੈਕਸ਼ਨ ਲਈ TLS  
- **SQL ਇੰਜੈਕਸ਼ਨ ਰੋਕਥਾਮ**: ਸਿਰਫ਼ ਪੈਰਾਮੀਟਰਾਈਜ਼ਡ ਕੂਇਰੀਆਂ  
- **ਇਨਪੁੱਟ ਵੈਧਤਾ**: ਪੂਰੀ ਰਿਕਵੇਸਟ ਵੈਧਤਾ  
- **ਗਲਤੀ ਸੰਭਾਲ**: ਗਲਤੀ ਸੁਨੇਹਿਆਂ ਵਿੱਚ ਕੋਈ ਸੰਵੇਦਨਸ਼ੀਲ ਡੇਟਾ ਨਹੀਂ
