## 📋 ਸਿੱਖਣ ਦੇ ਉਦੇਸ਼

ਇਸ ਮੋਡਿਊਲ ਦੇ ਅੰਤ ਤੱਕ, ਤੁਸੀਂ ਸਮਰੱਥ ਹੋਵੋਗੇ:
- ✅ ਮਾਇਕ੍ਰੋਸੌਫਟ ਫਾਉਂਡਰੀ ਟੂਲਕਿਟ ਐਕਸਟੇੰਸ਼ਨ ਨੂੰ VS ਕੋਡ ਲਈ ਇੰਸਟਾਲ ਅਤੇ ਸੰਰਚਿਤ ਕਰਨਾ
- ✅ ਮਾਡਲ ਕੈਟਲੌਗ ਨੂੰ ਨੈਵੀਗੇਟ ਕਰਨਾ ਅਤੇ ਵੱਖ-ਵੱਖ ਮਾਡਲ ਸਰੋਤਾਂ ਨੂੰ ਸਮਝਣਾ
- ✅ ਮਾਡਲ ਦੀ ਜਾਂਚ ਅਤੇ ਪ੍ਰਯੋਗ ਲਈ ਪਲੇਗ੍ਰਾਊਂਡ ਦੀ ਵਰਤੋਂ ਕਰਨੀ
- ✅ ਏਜੰਟ ਬਿਲਡਰ ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਕਸਟਮ ਏਆਈ ਏਜੰਟ ਬਣਾਉਣਾ
- ✅ ਵੱਖ-ਵੱਖ ਪ੍ਰਦਾਤਾਵਾਂ ਦੇ ਮਾਡਲ ਪ੍ਰਦਰਸ਼ਨ ਦੀ ਤੁਲਨਾ ਕਰਨੀ
- ✅ ਪ੍ਰਾਂਪਟ ਇੰਜੀਨੀਅਰਿੰਗ ਲਈ ਸਰਵੋਤਮ ਅਭਿਆਸ ਲਾਗੂ ਕਰਨਾ

## 🧠 ਮਾਇਕ੍ਰੋਸੌਫਟ ਫਾਉਂਡਰੀ ਟੂਲਕਿਟ ਦਾ ਪਰਿਚਯ

**ਮਾਇਕ੍ਰੋਸੌਫਟ ਫਾਉਂਡਰੀ ਟੂਲਕਿਟ ਐਕਸਟੇੰਸ਼ਨ ਫਾਰ VS ਕੋਡ** ਮਾਇਕ੍ਰੋਸੌਫਟ ਦੀ ਮੁੱਖ ਐਕਸਟੇੰਸ਼ਨ ਹੈ ਜੋ VS ਕੋਡ ਨੂੰ ਇੱਕ ਵਿਸ਼ਤ੍ਰਿਤ AI ਵਿਕਾਸ ਵਾਤਾਵਰਨ ਵਿੱਚ ਬਦਲਦਾ ਹੈ। ਇਹ AI ਖੋਜ ਅਤੇ ਵਿਹਾਰਕ ਐਪਲੀਕੇਸ਼ਨ ਵਿਕਾਸ ਦਰਮਿਆਨ ਪੁਲ ਦਾ ਕੰਮ ਕਰਦਾ ਹੈ, ਜੋ ਹਰ ਕਿਸੇ ਤਰ੍ਹਾਂ ਦੇ ਵਿਕਾਸਕਾਰਾਂ ਲਈ ਜੇਨਰੇਟਿਵ AI ਨੂੰ ਪਹੁੰਚਯੋਗ ਬਣਾਉਂਦਾ ਹੈ।

### 📊 ਕਦਮ 1: ਮਾਡਲ ਕੈਟਲੌਗ 'ਤੇ ਜਾਓ

ਮਾਡਲ ਕੈਟਲੌਗ ਤੁਹਾਡਾ AI ਪਰਿਵਾਰ ਦਾ ਦਰਵਾਜ਼ਾ ਹੈ। ਇਹ ਕਈ ਪ੍ਰਦਾਤਾਵਾਂ ਦੇ ਮਾਡਲ ਸੰਗ੍ਰਹਿਤ ਕਰਦਾ ਹੈ, ਜਿਸ ਨਾਲ ਮਾਡਲਾਂ ਦੀ ਖੋਜ ਅਤੇ ਤੁਲਨਾ ਆਸਾਨ ਬਣ ਜਾਂਦੀ ਹੈ।

**🔍 ਨੈਵੀਗੇਸ਼ਨ ਗਾਈਡ:**

Microsoft Foundry Toolkit ਸਾਈਡਬਾਰ ਵਿੱਚ **MODELS - Catalog** 'ਤੇ ਕਲਿੱਕ ਕਰੋ

![Model Catalog](../../../../translated_images/pa/aimodel.263ed2be013d8fb0.webp)

**💡 ਪ੍ਰੋ ਟਿਪ**: ਆਪਣੇ ਵਰਤੋਂ ਦੇ ਕੇਸ ਨਾਲ ਮੇਲ ਖਾਂਦੇ ਹੋਏ ਖਾਸ ਸਮਰੱਥਾਵਾਂ ਵਾਲੇ ਮਾਡਲਾਂ ਨੂੰ ਲੱਭੋ (ਜਿਵੇਂ ਕੋਡ ਜਨਰੇਸ਼ਨ, ਸ੍ਰਿਜਨਾਤਮਕ ਲਿਖਾਈ, ਵਿਸ਼ਲੇਸ਼ਣ)।

**⚠️ ਨੋਟ**: GitHub ਉੱਤੇ ਹੋਸਟ ਕੀਤੇ ਮਾਡਲ (ਜਿਵੇਂ GitHub ਮਾਡਲ) ਵਰਤੋਂ ਲਈ ਮੁਫ਼ਤ ਹਨ ਪਰ ਅਰਜ਼ੀਆਂ ਅਤੇ ਟੋਕਨਾਂ 'ਤੇ ਰੇਟ ਲਿਮਿਟਾਂ ਦਾ ਪਾਲਣ ਕਰਦੇ ਹਨ। ਜੇ ਤੁਸੀਂ ਗਿਟਹੁਬ ਨਾਲ ਸੰਬੰਧਿਤ ਨਾ ਹੋਣ ਵਾਲੇ ਮਾਡਲ (ਜਿਵੇਂ Azure AI ਜਾਂ ਹੋਰ ਐਂਡਪੌਇੰਟਾਂ ਵੱਲੋਂ ਹੋਸਟ ਕੀਤੇ ਗਏ) ਤੱਕ ਪਹੁੰਚ ਕਰਨੀ ਹੈ, ਤਾਂ ਤੁਹਾਨੂੰ ਢੰਗ ਨਾਲ API ਕੁੰਜੀ ਜਾਂ ਪ੍ਰਮਾਣਿਕਤਾ ਦਿੱਤੀਆਂ ਦੇਣੀਆਂ ਪੈਣਗੀਆਂ।

### 🚀 ਕਦਮ 2: ਆਪਣਾ ਪਹਿਲਾ ਮਾਡਲ ਸ਼ਾਮਿਲ ਅਤੇ ਸੰਰਚਿਤ ਕਰੋ

### 🎯 ਕਦਮ 3: ਪਲੇਗ੍ਰਾਊਂਡ ਇੰਟਰਫੇਸ ਵਿੱਚ ਮਾਹਰ ਬਣੋ

ਪਲੇਗ੍ਰਾਊਂਡ ਤੁਹਾਡਾ AI ਪਰਯੋਗਸ਼ਾਲਾ ਹੈ। ਇਸਦੀ ਸਮਰੱਥਾ ਨੂੰ ਵਧਾਉਣ ਲਈ ਇੱਥੇ ਕਦਮ ਹਨ:

**🎨 ਪ੍ਰਾਂਪਟ ਇੰਜੀਨੀਅਰਿੰਗ ਲਈ ਸਰਵੋਤਮ ਅਭਿਆਸ:**

2. **ਸੰਦਰਭ ਦਿਓ**: ਲਾਗੂ ਵਿੱਚ ਸਬੰਧਤ ਜਾਣਕਾਰੀ ਸ਼ਾਮਿਲ ਕਰੋ
3. **ਉਦਾਹਰਨ ਦੀ ਵਰਤੋਂ ਕਰੋ**: ਮਾਡਲ ਨੂੰ ਆਪਣੇ ਵੇਖਾਣ ਮੰਗ ਵਰਗੇ ਉਦਾਹਰਨ ਦਿਓ
4. **ਦੁਹਰਾਓ**: ਸ਼ੁਰੂਆਤੀ ਨਤੀਜਿਆਂ ਦੇ ਅਧਾਰ 'ਤੇ ਪ੍ਰਾਂਪਟ ਸੁਧਾਰੋ

```markdown
"Analyze this sales data and provide insights: [paste your data]. Focus on trends, anomalies, and actionable recommendations."
```

### 🏗️ ਕਦਮ 1: ਏਜੰਟ ਬਿਲਡਰ ਨੂੰ ਸਮਝਨਾ

ਏਜੰਟ ਬਿਲਡਰ ਉਹਥੇ ਹੈ ਜਿੱਥੇ Microsoft Foundry Toolkit ਅਸਲ ਵਿੱਚ ਚਮਕਦਾ ਹੈ। ਇਹ ਤੁਹਾਨੂੰ ਐਸੇ AI ਸਹਾਇਕ ਬਣਾਉਣ ਦਿੰਦਾ ਹੈ ਜਿਸ ਵਿੱਚ ਵੱਡੇ ਲੈਂਗਵੇਜ ਮਾਡਲਾਂ ਦੀ ਸ਼ਕਤੀ ਕਸਟਮ ਹੁਕਮਾਂ, ਖਾਸ ਪੈਰਾਮੀਟਰਾਂ ਅਤੇ ਵਿਸ਼ੇਸ਼ ਗਿਆਨ ਨਾਲ ਜੋੜੀ ਜਾਂਦੀ ਹੈ।

**🧠 ਏਜੰਟ ਆਰਕੀਟੈਕਚਰ ਦੇ ਹਿੱਸੇ:**
- **ਕੋਰ ਮਾਡਲ**: ਬੁਨਿਆਦ LLM (GPT-4, Groks, Phi ਆਦਿ)
- **ਸਿਸਟਮ ਪ੍ਰਾਂਪਟ**: ਏਜੰਟ ਦੀ ਸ਼ਖਸੀਅਤ ਅਤੇ ਵਿਹਾਰ ਨਿਰਧਾਰਤ ਕਰਦਾ ਹੈ
- **ਪੈਰਾਮੀਟਰ**: ਉਦਾਤਮਕ ਪ੍ਰਦਰਸ਼ਨ ਲਈ ਬਿਹਤਰੀਨ ਵਿਵਸਥਾ
- **ਟੂਲਜ਼ ਇੰਟੀਗ੍ਰੇਸ਼ਨ**: ਬਾਹਰੀ API ਅਤੇ MCP ਸੇਵਾਵਾਂ ਨਾਲ ਜੁੜਨਾ
- **ਮੈਮੋਰੀ**: ਗੱਲਬਾਤ ਦੀ ਸੰਦਰਭ ਅਤੇ ਸੈਸ਼ਨ ਦੀ ਸਥਿਰਤਾ

### ⚙️ ਕਦਮ 2: ਏਜੰਟ ਸੰਰਚਨਾ ਦੀ ਵਿਆਖਿਆ

**🎨 ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਸਿਸਟਮ ਪ੍ਰਾਂਪਟ ਬਣਾਉਣਾ:**
```markdown
# Template Structure:
## Role Definition
You are a [specific role] with expertise in [domain].

## Capabilities
- List specific abilities
- Define scope of knowledge
- Clarify limitations

## Behavior Guidelines
- Response style (formal, casual, technical)
- Output format preferences
- Error handling approach

## Examples
Provide 2-3 examples of ideal interactions
```

*ਬੇਸ਼ੱਕ, ਤੁਸੀਂ Generate System Prompt ਦੀ ਵਰਤੋਂ ਕਰਕੇ AI ਦੀ مدد ਨਾਲ ਪ੍ਰਾਂਪਟ ਬਣਾਉਣ ਅਤੇ ਸਭ ਤੋਂ ਵਧੀਆ ਬਣਾਉਣ ਲਈ ਵੀ ਵਰਤ ਸਕਦੇ ਹੋ*

**🔧 ਪੈਰਾਮੀਟਰ ਬਿਹਤਰੀ:**
| ਪੈਰਾਮੀਟਰ | ਸਿਫਾਰਸ਼ੀ ਰੇਂਜ | ਵਰਤੋਂ ਦਾ ਕੇਸ |
|-----------|------------------|----------|
| **Temperature** | 0.1-0.3 | ਤਕਨੀਕੀ/ਤਥਿਆਤਮਕ ਜਵਾਬ |
| **Temperature** | 0.7-0.9 | ਰਚਨਾਤਮਕ/ਦਿਮਾਗ਼ੀ ਕਾਰਜ |
| **Max Tokens** | 500-1000 | ਸੰਖੇਪ ਜਵਾਬ |
| **Max Tokens** | 2000-4000 | ਵਿਸਥਾਰ ਭਰਪੂਰ ਵਿਆਖਿਆ |

### 🐍 ਕਦਮ 3: ਪ੍ਰਯੋਗ - ਪਾਈਥਨ ਪ੍ਰੋਗ੍ਰਾਮਿੰਗ ਏਜੰਟ

**🎯 ਮਿਸ਼ਨ**: ਖਾਸ ਤੌਰ 'ਤੇ ਪਾਈਥਨ ਕੋਡਿੰਗ ਸਹਾਇਕ ਬਣਾਉਣਾ

**📋 ਸੰਰਚਨਾ ਕਦਮ:**

1. **ਮਾਡਲ ਚੋਣ**: ਚੁਣੋ **Claude 3.5 Sonnet** (ਕੋਡ ਲਈ ਸ਼ਾਨਦਾਰ)

2. **ਸਿਸਟਮ ਪ੍ਰਾਂਪਟ ਡਿਜ਼ਾਈਨ**:
```markdown
# Python Programming Expert Agent

## Role
You are a senior Python developer with 10+ years of experience. You excel at writing clean, efficient, and well-documented Python code.

## Capabilities
- Write production-ready Python code
- Debug complex issues
- Explain code concepts clearly
- Suggest best practices and optimizations
- Provide complete working examples

## Response Format
- Always include docstrings
- Add inline comments for complex logic
- Suggest testing approaches
- Mention relevant libraries when applicable

## Code Quality Standards
- Follow PEP 8 style guidelines
- Use type hints where appropriate
- Handle exceptions gracefully
- Write readable, maintainable code
```

3. **ਪੈਰਾਮੀਟਰ ਸੰਰਚਨਾ**:
   - Temperature: 0.2 (ਲਗਾਤਾਰ, ਭਰੋਸੇਯੋਗ ਕੋਡ ਲਈ)
   - Max Tokens: 2000 (ਵਿਸਥਾਰ ਲਈ ਵਿਆਖਿਆਵਾਂ)
   - Top-p: 0.9 (ਤੁਲਨਾਤਮਕ ਰਚਨਾਤਮਕਤਾ)
