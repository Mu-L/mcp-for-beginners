# Scenario 3: In-Editor Docs with MCP Server in VS Code

## Overview

ਇਸ ਸਿਨਾਰਿਓ ਵਿੱਚ, ਤੁਸੀਂ ਸਿੱਖੋਗੇ ਕਿ ਮਾਈਕ੍ਰੋਸੌਫਟ ਲਰਨ ਡੌਕਸ ਨੂੰ ਸਿੱਧਾ ਆਪਣੇ ਵਿਜ਼ੂਅਲ ਸਟੂਡੀਓ ਕੋਡ ਮਾਹੌਲ ਵਿੱਚ MCP ਸਰਵਰ ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਕਿਵੇਂ ਲਿਆ ਜਾ ਸਕਦਾ ਹੈ। ਡਾਕੂਮੈਂਟੇਸ਼ਨ ਲਈ ਬ੍ਰਾਊਜ਼ਰ ਟੈਬਾਂ ਬਦਲਣ ਦੇ ਥਾਂ, ਤੁਸੀਂ ਆਪਣੇ ਏਡੀਟਰ ਦੇ ਅੰਦਰ ਹੀ ਅਧਿਕਾਰਕ ਡੌਕਸ ਨੂੰ ਐਕਸੈਸ, ਖੋਜ ਅਤੇ ਹਵਾਲਾ ਦੇ ਸਕਦੇ ਹੋ। ਇਹ ਤਰੀਕਾ ਤੁਹਾਡੇ ਕਾਰਜਪ੍ਰਵਾਹ ਨੂੰ ਸੁਗਮ ਬਣਾਉਂਦਾ ਹੈ, ਤੁਹਾਨੂੰ ਧਿਆਨ ਕੇਂਦਰਿਤ ਰੱਖਦਾ ਹੈ ਅਤੇ GitHub Copilot ਵਰਗੇ ਟੂਲਾਂ ਨਾਲ ਬਿਨਾਂ ਕਿਸੇ ਰੁਕਾਵਟ ਦੇ ਇੰਟਿਗ੍ਰੇਸ਼ਨ ਮੁਹੱਈਆ ਕਰਵਾਉਂਦਾ ਹੈ।

- VS Code ਦੇ ਅੰਦਰ ਹੀ ਖੋਜੋ ਅਤੇ ਡੌਕਸ ਪੜ੍ਹੋ ਬਿਨਾਂ ਆਪਣੇ ਕੋਡਿੰਗ ਮਾਹੌਲ ਨੂੰ ਛੱਡੇ।
- ਡੌਕੂਮੈਂਟੇਸ਼ਨ ਦਾ ਹਵਾਲਾ ਦਿਓ ਅਤੇ ਲਿੰਕਸ ਨੂੰ ਸਿੱਧਾ README ਜਾਂ ਕੋਰਸ ਫਾਇਲਾਂ ਵਿੱਚ ਸ਼ਾਮਲ ਕਰੋ।
- GitHub Copilot ਅਤੇ MCP ਨੂੰ ਇਕੱਠੇ ਵਰਤੋਂ ਕਰਕੇ ਇੱਕ ਬਿਨਾਂ ਰੁਕਾਵਟ, ਏਆਈ-ਚਾਲਤ ਡੌਕੂਮੈਂਟੇਸ਼ਨ ਕਾਰਜਪ੍ਰਵਾਹ ਪ੍ਰਾਪਤ ਕਰੋ।

## Learning Objectives

ਇਸ ਅਧਿਆਇ ਦੇ ਅੰਤ ਤੱਕ, ਤੁਹਾਨੂੰ ਸਮਝ ਆ ਜਾਵੇਗਾ ਕਿ MCP ਸਰਵਰ ਨੂੰ VS Code ਦੇ ਅੰਦਰ ਕਿਵੇਂ ਸੈਟਅਪ ਅਤੇ ਵਰਤਣਾ ਹੈ ਤਾਂ ਜੋ ਤੁਹਾਡੇ ਡੌਕੂਮੈਂਟੇਸ਼ਨ ਅਤੇ ਵਿਕਾਸ ਕਾਰਜਪ੍ਰਵਾਹ ਨੂੰ ਬਹਿਤਰ ਬਣਾਇਆ ਜਾ ਸਕੇ। ਤੁਸੀਂ ਇਹ ਕਰਨ ਦੇ ਸਮਰੱਥ ਹੋਵੋਗੇ:

- MCP ਸਰਵਰ ਨੂੰ ਡੌਕੂਮੈਂਟੇਸ਼ਨ ਲੁਕਅਪ ਲਈ ਵਰਤਣ ਲਈ ਆਪਣੇ ਵਰਕਸਪੇਸ ਨੂੰ ਕਨਫਿਗਰ ਕਰੋ।
- VS Code ਦੇ ਅੰਦਰੋਂ ਸਿੱਧਾ ਡੌਕੂਮੈਂਟੇਸ਼ਨ ਖੋਜੋ ਅਤੇ ਸ਼ਾਮਲ ਕਰੋ।
- GitHub Copilot ਅਤੇ MCP ਦੀ ਤਾਕਤ ਨੂੰ ਮਿਲਾ ਕੇ ਇੱਕ ਵਧੇਰੇ ਉਤਪਾਦਕ, ਏਆਈ-ਵਧੀਕ ਕਾਰਜਪ੍ਰਵਾਹ ਬਣਾਓ।

ਇਹ ਹੁਨਰ ਤੁਹਾਨੂੰ ਧਿਆਨ ਕੇਂਦਰਿਤ ਰਹਿਣ, ਡੌਕੂਮੈਂਟੇਸ਼ਨ ਗੁਣਵੱਤਾ ਵਿੱਚ ਸੁਧਾਰ ਕਰਨ ਅਤੇ ਵਿਕਾਸਕਾਰ ਜਾਂ ਤਕਨੀਕੀ ਲੇਖਕ ਵਜੋਂ ਉਤਪਾਦਕਤਾ ਵਧਾਉਣ ਵਿੱਚ ਸਹਾਇਤਾ ਕਰਨਗੇ।

## Solution

ਇਨ-ਏਡੀਟਰ ਡੌਕੂਮੈਂਟੇਸ਼ਨ ਐਕਸੈਸ ਪ੍ਰਾਪਤ ਕਰਨ ਲਈ, ਤੁਸੀਂ MCP ਸਰਵਰ ਨੂੰ VS Code ਅਤੇ GitHub Copilot ਨਾਲ ਇਕੱਠੇ ਜੋੜਨ ਵਾਲੇ ਕਈ ਕਦਮਾਂ ਦੀ ਪਾਲਣਾ ਕਰੋਗੇ। ਇਹ ਹੱਲ ਕੋਰਸ ਲੇਖਕਾਂ, ਡੌਕੂਮੈਂਟੇਸ਼ਨ ਲੇਖਕਾਂ ਅਤੇ ਵਿਕਾਸਕਾਰਾਂ ਲਈ ਉਚਿਤ ਹੈ, ਜੋ ਡੌਕਸ ਅਤੇ Copilot ਨਾਲ ਕੰਮ ਕਰਦੇ ਸਮੇਂ ਆਪਣੇ ਧਿਆਨ ਨੂੰ ਏਡੀਟਰ ਵਿੱਚ ਰੱਖਣਾ ਚਾਹੁੰਦੇ ਹਨ।

- ਕੋਰਸ ਜਾਂ ਪ੍ਰੋਜੈਕਟ ਡੌਕੂਮੈਂਟੇਸ਼ਨ ਲਿਖਦੇ ਸਮੇਂ README ਵਿੱਚ ਤੇਜ਼ੀ ਨਾਲ ਹਵਾਲਾ ਲਿੰਕ ਸ਼ਾਮਲ ਕਰੋ।
- ਕੋਡ ਤਿਆਰ ਕਰਨ ਲਈ Copilot ਦਾ ਵਰਤੋਂ ਕਰੋ ਅਤੇ MCP ਨਾਲ ਸੰਬੰਧਿਤ ਡੌਕਸ ਤੁਰੰਤ ਲੱਭੋ ਅਤੇ ਹਵਾਲਾ ਦਿਓ।
- ਆਪਣੇ ਏਡੀਟਰ ਵਿੱਚ ਧਿਆਨ ਕੇਂਦਰਿਤ ਰਹੋ ਅਤੇ ਉਤਪਾਦਕਤਾ ਵਧਾਓ।

### Step-by-Step Guide

ਸ਼ੁਰੂ ਕਰਨ ਲਈ, ਹੇਠਾਂ ਦਿੱਤੇ ਕਦਮਾਂ ਦੀ ਪਾਲਣਾ ਕਰੋ। ਹਰ ਕਦਮ ਲਈ, ਤੁਸੀਂ ਪ੍ਰਕਿਰਿਆ ਦੀ ਵਿਜ਼ੂਅਲ ਵਿਆਖਿਆ ਲਈ ਐਸੈਟਸ ਫੋਲਡਰ ਵਿੱਚੋਂ ਸਕ੍ਰੀਨਸ਼ਾਟ ਸ਼ਾਮਲ ਕਰ ਸਕਦੇ ਹੋ।

1. **MCP ਸੰਰਚਨਾ ਸ਼ਾਮਲ ਕਰੋ:**
   ਆਪਣੇ ਪ੍ਰੋਜੈਕਟ ਰੂਟ ਵਿੱਚ `.vscode/mcp.json` ਫਾਇਲ ਬਣਾਓ ਅਤੇ ਹੇਠਾਂ ਦਿੱਤੀ ਸੰਰਚਨਾ ਸ਼ਾਮਲ ਕਰੋ:
   ```json
   {
     "servers": {
       "LearnDocsMCP": {
         "url": "https://learn.microsoft.com/api/mcp"
       }
     }
   }
   ```
   ਇਹ ਸੰਰਚਨਾ VS Code ਨੂੰ ਦੱਸਦੀ ਹੈ ਕਿ [`Microsoft Learn Docs MCP server`](https://github.com/MicrosoftDocs/mcp) ਨਾਲ ਕਿਵੇਂ ਜੁੜਨਾ ਹੈ।
   
   ![Step 1: Add mcp.json to .vscode folder](../../../../../../translated_images/pa/step1-mcp-json.c06a007fccc3edfa.webp)
    
2. **GitHub Copilot ਚੈਟ ਪੈਨਲ ਖੋਲ੍ਹੋ:**
   ਜੇ ਤੁਹਾਡੇ ਕੋਲ ਪਹਿਲਾਂ ਹੀ GitHub Copilot ਐਕਸਟੈਂਸ਼ਨ ਇੰਸਟਾਲ ਨਹੀਂ ਹੈ, ਤਾਂ VS Code ਵਿੱਚ ਇੱਕਸਟੈਂਸ਼ਨ ਨੂੰਖੋਜ ਕੇ ਇਸਨੂੰ ਇੰਸਟਾਲ ਕਰੋ। ਤੁਸੀਂ ਇਸ ਨੂੰ ਸਿੱਧਾ [Visual Studio Code Marketplace](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot-chat) ਤੋਂ ਡਾਊਨਲੋਡ ਕਰ ਸਕਦੇ ਹੋ। ਫਿਰ, ਸਾਈਡਬਾਰ ਤੋਂ Copilot ਚੈਟ ਪੈਨਲ ਖੋਲ੍ਹੋ।

   ![Step 2: Open Copilot Chat panel](../../../../../../translated_images/pa/step2-copilot-panel.f1cc86e9b9b8cd1a.webp)

3. **ਏਜੰਟ ਮੋਡ ਚਾਲੂ ਕਰੋ ਅਤੇ ਟੂਲਾਂ ਦੀ ਤਸਦੀਕ ਕਰੋ:**
   Copilot ਚੈਟ ਪੈਨਲ ਵਿੱਚ, ਏਜੰਟ ਮੋਡ ਚਾਲੂ ਕਰੋ।

   ![Step 3: Enable agent mode and verify tools](../../../../../../translated_images/pa/step3-agent-mode.cdc32520fd7dd1d1.webp)

   ਏਜੰਟ ਮੋਡ ਚਾਲੂ ਕਰਨ ਤੋਂ ਬਾਅਦ, ਜ਼ਰੂਰੀ ਹੈ ਕਿ MCP ਸਰਵਰ ਨੂੰ ਉਪਲਬਧ ਟੂਲਾਂ ਵਿੱਚ ਦਰਸਾਇਆ ਜਾਵੇ। ਇਹ ਯਕੀਨੀ ਬਣਾਉਂਦਾ ਹੈ ਕਿ Copilot ਏਜੰਟ ਡੌਕੂਮੈਂਟੇਸ਼ਨ ਸਰਵਰ ਤੱਕ ਪਹੁੰਚ ਸਕਦਾ ਹੈ ਅਤੇ ਸੰਬੰਧਿਤ ਜਾਣਕਾਰੀ ਪ੍ਰਦਾਨ ਕਰ ਸਕਦਾ ਹੈ।
   
   ![Step 3: Verify MCP server tool](../../../../../../translated_images/pa/step3-verify-mcp-tool.76096a6329cbfecd.webp)
4. **ਨਵਾਂ ਚੈਟ ਸ਼ੁਰੂ ਕਰੋ ਅਤੇ ਏਜੰਟ ਨੂੰ ਪ੍ਰੇਰणा ਦਿਓ:**
   Copilot ਚੈਟ ਪੈਨਲ ਵਿੱਚ ਨਵਾਂ ਚੈਟ ਖੋਲ੍ਹੋ। ਹੁਣ ਤੁਸੀਂ ਡੌਕੂਮੈਂਟੇਸ਼ਨ ਸਵਾਲਾਂ ਨਾਲ ਏਜੰਟ ਨੂੰ ਪ੍ਰੇਰਿਤ ਕਰ ਸਕਦੇ ਹੋ। ਏਜੰਟ MCP ਸਰਵਰ ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਸੰਬੰਧਿਤ Microsoft Learn ਡੌਕੂਮੈਂਟੇਸ਼ਨ ਨੂੰ ਸਿੱਧਾ ਤੁਹਾਡੇ ਏਡੀਟਰ ਵਿੱਚ ਲਿਆਵੇਗਾ।

   - *"ਮੈਂ ਵਿਸ਼ਾ X ਲਈ ਇੱਕ ਅਧਿਐਨ ਯੋਜਨਾ ਲਿਖਣ ਦੀ ਕੋਸ਼ਿਸ਼ ਕਰ ਰਿਹਾ ਹਾਂ। ਮੈਂ ਇਸਨੂੰ 8 ਹਫਤਿਆਂ ਤੱਕ ਪੜ੍ਹਨਾ ਚਾਹੁੰਦਾ ਹਾਂ, ਹਰ ਹਫ਼ਤੇ ਲਈ, ਮੈਨੂੰ ਸੁਝਾਅ ਦਿਓ ਕਿ ਮੈਨੂੰ ਕਿਹੜਾ ਸਮੱਗਰੀ ਲੈਣੀ ਚਾਹੀਦੀ ਹੈ।"*

   ![Step 4: Prompt the agent in chat](../../../../../../translated_images/pa/step4-prompt-chat.12187bb001605efc.webp)

5. **ਲਾਈਵ ਕੂਐਰੀ:**

   > ਆਓ Microsoft Foundry Discord ਦੇ [#get-help](https://discord.gg/D6cRhjHWSC) ਸੈਕਸ਼ਨ ਤੋਂ ਇੱਕ ਲਾਈਵ ਕੂਐਰੀ ਲੈਈਏ ([ਅਸਲ ਸੁਨੇਹਾ ਵੇਖੋ](https://discord.com/channels/1113626258182504448/1385498306720829572)):
   
   *"ਮੈਂ ਏਜ਼ਯੂਰ ਏਆਈ ਫਾਉਂਡਰੀ 'ਤੇ ਵਿਕਸਤ ਕੀਆ ਗਿਆ ਬਹੁ-ਏਜੰਟ ਹੱਲ ਨੂੰ ਡਿਪਲੌਇ ਕਰਨ ਦੇ ਤਰੀਕੇ ਬਾਰੇ ਉੱਤਰ ਲੱਭ ਰਹਿਆ ਹਾਂ। ਮੈਨੂੰ ਦਿੱਖਦਾ ਹੈ ਕਿ ਕੋਈ ਸਿੱਧਾ ਡਿਪਲੌਇਮੈਂਟ ਤਰੀਕਾ ਨਹੀਂ ਹੈ, ਜਿਵੇਂ ਕਿ Copilot Studio ਚੈਨਲ। ਤਾਂ, ਵੱਖ-ਵੱਖ ਤਰੀਕੇ ਕੀ ਹਨ ਜਿਨ੍ਹਾਂ ਨਾਲ ਐਂਟਰਪ੍ਰਾਈਜ਼ ਉਪਭੋਗਤਾ ਇੰਟਰੈਕਟ ਕਰ ਸਕਦੇ ਹਨ ਅਤੇ ਕੰਮ ਪੂਰਾ ਕਰ ਸਕਦੇ ਹਨ?
ਬਹੁਤ ਸਾਰੇ ਲੇਖ/ਬਲੌਗ ਹਨ ਜੋ ਕਹਿੰਦੇ ਹਨ ਕਿ ਅਸੀਂ Azure Bot ਸੇਵਾ ਦੀ ਵਰਤੋਂ ਕਰ ਸਕਦੇ ਹਾਂ ਜੋ MS Teams ਅਤੇ Azure AI Foundry ਏਜੰਟਾਂ ਵਿੱਚ ਇੱਕ ਪੁਲ ਵਜੋਂ ਕੰਮ ਕਰ ਸਕਦਾ ਹੈ, ਕੀ ਇਹ ਕੰਮ ਕਰੇਗਾ ਜੇ ਮੈਂ ਇੱਕ Azure bot ਸੈੱਟਅਪ ਕਰਾਂ ਜੋ Azure Function ਰਾਹੀਂ Azure AI Foundry ਦੇ Orchestrator Agent ਨਾਲ ਜੁੜਦਾ ਹੈ ਜਾਂ ਕੀ ਮੈਨੂੰ ਹਰ ਇੱਕ AI ਏਜੰਟ ਲਈ Azure Function ਬਣਾਉਣੀ ਪਵੇਗੀ ਜੋ ਬਹੁ-ਏਜੰਟ ਹੱਲ ਦਾ ਹਿੱਸਾ ਹੈ ਤਾਂ ਜੋ Bot Framework ਵਿੱਚ orchestration ਕਰ ਸਕੀਏ? ਹੋਰ ਸਝਾਵਾਂ ਵੀ ਸਵਾਗਤਯੋਗ ਹਨ।"*

   ![Step 5: Live queries](../../../../../../translated_images/pa/step5-live-queries.49db3e4a50bea273.webp)

   ਏਜੰਟ ਸੰਬੰਧਿਤ ਡੌਕੂਮੈਂਟੇਸ਼ਨ ਲਿੰਕ ਅਤੇ ਸਾਰਾਂਸ਼ ਨਾਲ ਜਵਾਬ ਦੇਵੇਗਾ, ਜਿਸਨੂੰ ਤੁਸੀਂ ਸਿੱਧਾ ਆਪਣੇ ਮਾਰਕਡਾਊਨ ਫਾਇਲਾਂ ਵਿੱਚ ਸ਼ਾਮਲ ਕਰ ਸਕਦੇ ਹੋ ਜਾਂ ਆਪਣੇ ਕੋਡ ਵਿੱਚ ਹਵਾਲਿਆਂ ਵਜੋਂ ਵਰਤ ਸਕਦੇ ਹੋ।
   
### Sample Queries

ਇੱਥੇ ਕੁਝ ਉਦਾਹਰਣ ਸਵਾਲ ਹਨ ਜੋ ਤੁਸੀਂ ਟਰਾਈ ਕਰ ਸਕਦੇ ਹੋ। ਇਹ ਸਵਾਲ ਵੇਖਾਊਂਂਦੇ ਹਨ ਕਿ MCP ਸਰਵਰ ਅਤੇ Copilot ਇਕੱਠੇ ਕਿਵੇਂ ਤੁਰੰਤ, ਸੰਦਰਭ-ਜਾਣੂ ਡੌਕੂਮੈਂਟੇਸ਼ਨ ਅਤੇ ਹਵਾਲੇ ਪ੍ਰਦਾਨ ਕਰ ਸਕਦੇ ਹਨ ਬਿਨਾਂ VS Code ਛੱਡੇ:

- "ਮੈਨੂੰ ਦੱਸੋ ਕਿ Azure Functions ਟ੍ਰਿਗਰ ਵਰਤ ਕੇ ਕਿਵੇਂ ਕਰਨਾ ਹੈ।"
- "ਆਫੀਸ਼ੀਅਲ ਡੌਕੂਮੈਂਟੇਸ਼ਨ ਲਈ Azure Key Vault ਦਾ ਲਿੰਕ ਸ਼ਾਮਲ ਕਰੋ।"
- "Azure ਸਰੋਤਾਂ ਨੂੰ ਸੁਰੱਖਿਅਤ ਕਰਨ ਦੀਆਂ ਸਭ ਤੋਂ ਵਧੀਆ ਪ੍ਰਥਾਵਾਂ ਕੀ ਹਨ?"
