# 🚀 10 مائیکروسافٹ MCP سرورز جو ڈویلپر کی پیداواریت کو تبدیل کر رہے ہیں

## 🎯 اس رہنما میں آپ کیا سیکھیں گے

یہ عملی گائیڈ دس مائیکروسافٹ MCP سرورز کو دکھاتی ہے جو فعال طور پر اس بات کو بدل رہے ہیں کہ ڈویلپرز AI اسسٹنٹس کے ساتھ کس طرح کام کرتے ہیں۔ صرف یہ سمجھانے کے بجائے کہ MCP سرورز *کیا* کر سکتے ہیں، ہم آپ کو ایسے سرورز دکھائیں گے جو مائیکروسافٹ اور اس سے آگے روزمرہ ترقیاتی ورک فلو میں واقعی فرق لا رہے ہیں۔

اس گائیڈ میں ہر سرور کو حقیقی دنیا کے استعمال اور ڈویلپرز کی رائے کی بنیاد پر منتخب کیا گیا ہے۔ آپ نہ صرف یہ جانیں گے کہ ہر سرور کیا کرتا ہے، بلکہ یہ بھی کہ یہ کیوں اہم ہے اور اسے اپنے پراجیکٹس میں زیادہ سے زیادہ کس طرح استعمال کیا جا سکتا ہے۔ چاہے آپ MCP میں بالکل نئے ہوں یا اپنے موجودہ سیٹ اپ کو بڑھانا چاہتے ہوں، یہ سرورز مائیکروسافٹ ایکو سسٹم میں دستیاب سب سے عملی اور مؤثر ٹولز کی نمائندگی کرتے ہیں۔

> **💡 فوری شروع کرنے کا ٹپ**
> 
> MCP میں نئے ہیں؟ فکر نہ کریں! یہ گائیڈ ابتدائی افراد کے لیے ڈیزائن کی گئی ہے۔ ہم تصورات کو ساتھ ساتھ سمجھائیں گے، اور آپ ہمیشہ ہماری [MCP تعارف](../00-Introduction/README.md) اور [بنیادی تصورات](../01-CoreConcepts/README.md) ماڈیولز کو گہرائی سے سمجھنے کے لیے دیکھ سکتے ہیں۔

## جائزہ

یہ جامع گائیڈ دس مائیکروسافٹ MCP سرورز کا جائزہ لیتی ہے جو ڈویلپرز کے AI اسسٹنٹس اور خارجی ٹولز کے ساتھ تعامل کرنے کے طریقے کو انقلاب کی شکل دے رہے ہیں۔ ازور ریسورس مینیجمنٹ سے لے کر دستاویزات کی پروسیسنگ تک، یہ سرورز ماڈل کانٹیکسٹ پروٹوکول کی طاقت کو ظاہر کرتے ہیں جو ہموار، پیداواری ترقیاتی ورک فلو تخلیق کرتے ہیں۔

## سیکھنے کے مقاصد

اس گائیڈ کے آخر تک، آپ:
- سمجھیں گے کہ MCP سرورز ڈویلپر کی پیداواری صلاحیت کو کیسے بہتر بناتے ہیں
- مائیکروسافٹ کے سب سے مؤثر MCP سرور نفاذ کے بارے میں جانیں گے
- ہر سرور کے عملی استعمال کے کیسز دریافت کریں گے
- VS کوڈ اور ویژول سٹوڈیو میں ان سرورز کو سیٹ اپ اور ترتیب دینا سیکھیں گے
- MCP کے وسیع تر ایکو سسٹم اور مستقبل کے رجحانات کو جانیں گے

## 🔧 MCP سرورز کی سمجھ: ابتدائی افراد کے لیے گائیڈ

### MCP سرورز کیا ہیں؟

ماڈل کانٹیکسٹ پروٹوکول (MCP) کے لیے ایک ابتدائی کے طور پر، آپ سوچ سکتے ہیں: "اصل میں MCP سرور کیا ہے، اور مجھے اس کی پرواہ کیوں کرنی چاہیے؟" آئیے ایک آسان تمثیل سے شروع کرتے ہیں۔

MCP سرورز کو ایسے خصوصی اسسٹنٹس سمجھیں جو آپ کے AI کوڈنگ ساتھی (جیسے GitHub Copilot) کو بیرونی ٹولز اور سروسز سے جوڑنے میں مدد دیتے ہیں۔ جس طرح آپ اپنے فون پر مختلف کاموں کے لیے مختلف ایپس استعمال کرتے ہیں — موسم کے لیے ایک، نیویگیشن کے لیے دوسرا، اور بینکنگ کے لیے تیسرا — MCP سرورز آپ کے AI اسسٹنٹ کو مختلف ترقیاتی ٹولز اور خدمات کے ساتھ تعامل کرنے کی صلاحیت دیتے ہیں۔

### وہ مسئلہ جو MCP سرورز حل کرتے ہیں

MCP سرورز سے پہلے، اگر آپ کو:
- اپنے ازور ریسورسز چیک کرنے ہوتے
- GitHub مسئلہ بنانا ہوتا
- اپنے ڈیٹا بیس سے سوال کرنا ہوتا
- دستاویزات میں تلاش کرنا ہوتی

تو آپ کو کوڈنگ روکنی پڑتی، براؤزر کھولنا پڑتا، متعلقہ ویب سائٹ پر جانا پڑتا، اور دستی طور پر یہ کام کرنے پڑتے۔ یہ مسلسل سیاق و سباق بدلنا آپ کے بہاؤ کو توڑتا اور پیداواری صلاحیت کم کرتا تھا۔

### MCP سرورز آپ کے ترقیاتی تجربے کو کیسے بدلتے ہیں

MCP سرورز کے ساتھ، آپ اپنے ترقیاتی ماحول (VS کوڈ، ویژول سٹوڈیو وغیرہ) میں رہ کر بس اپنے AI اسسٹنٹ سے یہ کام کرنے کو کہہ سکتے ہیں۔ مثلاً:

**اس روایتی ورک فلو کے بجائے:**
1. کوڈنگ روک دیں
2. براؤزر کھولیں
3. ازور پورٹل پر جائیں
4. اسٹوریج اکاؤنٹ کی تفصیلات دیکھیں
5. VS کوڈ پر واپس آئیں
6. کوڈنگ جاری رکھیں

**آپ اب یہ کر سکتے ہیں:**
1. AI سے پوچھیں: "میرے Azure اسٹوریج اکاؤنٹس کی صورتحال کیا ہے؟"
2. فراہم کردہ معلومات کے ساتھ کوڈنگ جاری رکھیں

### ابتدائیوں کے لیے کلیدی فوائد

#### 1. 🔄 **اپنے بہاؤ کی حالت میں رہیں**
- مختلف ایپلیکیشنز کے درمیان سوئچنگ بند کریں
- اپنے کوڈ پر توجہ مرکوز رکھیں جو آپ لکھ رہے ہیں
- مختلف ٹولز کے انتظام کا ذہنی بوجھ کم کریں

#### 2. 🤖 **پیچیدہ کمانڈز کی بجائے قدرتی زبان استعمال کریں**
- SQL نحو یاد رکھنے کی بجائے بتائیں کہ آپ کو کیا ڈیٹا چاہیے
- ازور CLI کمانڈز یاد رکھنے کی بجائے وضاحت کریں کہ آپ کیا کرنا چاہتے ہیں
- AI کو تکنیکی تفصیلات سنبھالنے دیں جبکہ آپ منطق پر توجہ دیں

#### 3. 🔗 **متعدد ٹولز کو آپس میں جوڑیں**
- مختلف خدمات کو ملا کر طاقتور ورک فلو بنائیں
- مثال: "تمام تازہ ترین GitHub مسائل حاصل کریں اور ان کے مطابق Azure DevOps ورک آئٹمز بنائیں"
- پیچیدہ اسکرپٹس لکھے بغیر خودکار نظام بنائیں

#### 4. 🌐 **ایک بڑھتے ہوئے ایکو سسٹم تک رسائی حاصل کریں**
- مائیکروسافٹ، GitHub، اور دیگر کمپنیوں کے بنائے ہوئے سرورز کا فائدہ اٹھائیں
- مختلف فروشندوں کے ٹولز کو بغیر رکاوٹ ملائیں
- ایک ایسا معیاری ایکو سسٹم جو مختلف AI اسسٹنٹس میں کام کرتا ہے

#### 5. 🛠️ **عملی طور پر سیکھیں**
- تصورات کو سمجھنے کے لیے پہلے سے بنے ہوئے سرورز سے شروع کریں
- جیسے جیسے آپ زیادہ پر اعتماد ہوں، اپنے سرورز بنانا شروع کریں
- دستیاب SDKs اور دستاویزات کا استعمال کریں تاکہ آپ کی سیکھنے میں رہنمائی ہو

### ابتدائیوں کے لیے حقیقی دنیا کی مثال

فرض کریں آپ ویب ڈویلپمنٹ میں نئے ہیں اور اپنا پہلا پراجیکٹ کر رہے ہیں۔ یہ ہے کہ MCP سرورز آپ کی مدد کیسے کر سکتے ہیں:

**روایتی طریقہ:**
```
1. Code a feature
2. Open browser → Navigate to GitHub
3. Create an issue for testing
4. Open another tab → Check Azure docs for deployment
5. Open third tab → Look up database connection examples
6. Return to VS Code
7. Try to remember what you were doing
```

**MCP سرورز کے ساتھ:**
```
1. Code a feature
2. Ask AI: "Create a GitHub issue for testing this login feature"
3. Ask AI: "How do I deploy this to Azure according to the docs?"
4. Ask AI: "Show me the best way to connect to my database"
5. Continue coding with all the information you need
```


### انٹرپرائز اسٹینڈرڈ کا فائدہ

MCP ایک صنعت گیر معیار بنتا جا رہا ہے، جس کا مطلب ہے:
- **ثبات**: مختلف ٹولز اور کمپنیوں میں یکساں تجربہ
- **انٹرآپریبلٹی**: مختلف فروشندوں کے سرورز ایک ساتھ کام کرتے ہیں
- **مستقبل بینی**: مہارتیں اور سیٹ اپ مختلف AI اسسٹنٹس کے درمیان منتقل ہوتے ہیں
- **کمیونٹی**: مشترکہ علم اور وسائل کا ایک بڑا ایکو سسٹم

## VS کوڈ اور ویژول سٹوڈیو میں MCP کے ساتھ شروعات 🚀

اگر آپ Visual Studio Code یا Visual Studio 2022 کو GitHub Copilot کے ساتھ استعمال کر رہے ہیں تو ان MCP سرورز کو سیٹ اپ کرنا آسان ہے۔

### VS کوڈ سیٹ اپ

VS کوڈ کے لیے بنیادی عمل یہ ہے:

1. **ایجنٹ موڈ فعال کریں**: VS کوڈ میں Copilot Chat ونڈو میں ایجنٹ موڈ پر سوئچ کریں
2. **MCP سرورز کی ترتیب دیں**: اپنی VS کوڈ settings.json فائل میں سرور کنفیگریشنز شامل کریں
3. **سرورز شروع کریں**: ہر سرور کے لیے "Start" بٹن کلک کریں جو آپ استعمال کرنا چاہتے ہیں
4. **ٹولز منتخب کریں**: اپنے موجودہ سیشن کے لیے کون سے MCP سرورز فعال کرنے ہیں منتخب کریں

تفصیلی سیٹ اپ کے لیے [VS کوڈ MCP دستاویزات](https://code.visualstudio.com/docs/copilot/copilot-mcp) دیکھیں۔

> **💡 پرو ٹپ: MCP سرورز کو پرو کی طرح منظم کریں!**
> 
> VS کوڈ ایکسٹینشنز ویو میں اب [انسٹال شدہ MCP سرورز کو منظم کرنے کے لیے نیا آسان UI](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode) شامل ہے! آپ کسی بھی انسٹالڈ MCP سرور کو شروع، روکنے، اور منظم کرنے کے لیے واضح اور سادہ انٹرفیس استعمال کر سکتے ہیں۔ اسے ضرور آزمائیں!

### ویژول سٹوڈیو 2022 سیٹ اپ

Visual Studio 2022 (ورژن 17.14 یا بعد کے لیے):

1. **ایجنٹ موڈ فعال کریں**: GitHub Copilot Chat ونڈو میں "Ask" ڈراپ ڈاؤن پر کلک کریں اور "Agent" منتخب کریں
2. **کنفیگریشن فائل بنائیں**: اپنی سلوشن ڈائریکٹری میں `.mcp.json` فائل بنائیں (تجویز کردہ مقام: `<SOLUTIONDIR>\.mcp.json`)
3. **سرورز کی ترتیب دیں**: MCP سرورز کی کنفیگریشنز معیاری MCP فارمیٹ میں شامل کریں
4. **ٹول کی منظوری**: جب پوچھا جائے تو مناسب اسکوپ اجازتوں کے ساتھ استعمال کرنے والے ٹولز کی منظوری دیں

تفصیلی ویژول سٹوڈیو سیٹ اپ ہدایات کے لیے [ویژول سٹوڈیو MCP دستاویزات](https://learn.microsoft.com/visualstudio/ide/mcp-servers) دیکھیں۔

ہر MCP سرور کی اپنی کنفیگریشن ضروریات ہوتی ہیں (کنکشن سٹرنگز، تصدیق، وغیرہ)، لیکن سیٹ اپ کا نمونہ دونوں IDEs میں یکساں ہے۔

**حقیقی دنیا میں استعمال**: "مائیکروسافٹ Learn دستاویزات کے مطابق Azure کنٹینر ایپ بنانے کے لیے az cli کمانڈز کیا ہیں؟" یا "ASP.NET Core میں Entity Framework کو dependency injection کے ساتھ کیسے ترتیب دوں؟" یا "اس کوڈ کا جائزہ لیں کہ یہ Microsoft Learn Documentation کے کارکردگی کی سفارشات سے میل کھاتا ہو۔" یہ سرور Microsoft Learn، Azure دستاویزات، اور Microsoft 365 دستاویزات کے اوپر معنوی تلاش کے ذریعے سب سے زیادہ متعلقہ معلومات تلاش کرتا ہے۔ یہ 10 اعلی معیار کے مواد کے حصے، آرٹیکل ٹائٹلز اور URLs کے ساتھ واپس کرتا ہے، ہمیشہ تازہ ترین مائیکروسافٹ دستاویزات تک رسائی حاصل کرتا ہے۔

**نمایاں مثال**: سرور `microsoft_docs_search` ٹول فراہم کرتا ہے جو مائیکروسافٹ کی سرکاری تکنیکی دستاویزات کے خلاف معنوی تلاش کرتا ہے۔ ایک بار کنفیگر ہونے کے بعد، آپ سوالات پوچھ سکتے ہیں جیسے "ASP.NET Core میں JWT اتھینٹیکیشن کیسے نافذ کروں؟" اور تفصیلی، سرکاری جوابات سورس لنکس کے ساتھ حاصل کریں۔ تلاش کا معیار استثنائی ہے کیونکہ یہ سیاق و سباق کو سمجھتا ہے — Azure کے تناظر میں "containers" کے بارے میں پوچھنے پر Azure Container Instances کی دستاویزات ملتی ہیں، جبکہ .NET کے تناظر میں یہی اصطلاح C# کے متعلقہ کلیکشن معلومات دیتی ہے۔

### 2. ☁️ Azure MCP Server
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

**یہ کیا کرتا ہے**: Azure MCP سرور 15+ خصوصی Azure سروس کنیکٹرز کا ایک جامع مجموعہ ہے جو پورے Azure ایکو سسٹم کو آپ کے AI ورک فلو میں لاتا ہے۔ یہ صرف ایک سرور نہیں – یہ ایک طاقتور مجموعہ ہے جس میںوسائل کا انتظام، ڈیٹا بیس کنیکٹیویٹی (PostgreSQL، SQL Server)، Azure Monitor لاگ تجزیہ KQL کے ساتھ، Cosmos DB انضمام، اور بہت کچھ شامل ہے۔

**یہ کیوں مفید ہے**: Azure وسائل کے انتظام سے آگے، یہ سرور Azure SDKs کے ساتھ کام کرتے وقت کوڈ کے معیار کو نمایاں طور پر بہتر بناتا ہے۔ جب آپ Azure MCP کو Agent موڈ میں استعمال کرتے ہیں، تو یہ صرف کوڈ لکھنے میں مدد نہیں کرتا – بلکہ آپ کو *بہتر* Azure کوڈ لکھنے میں مدد دیتا ہے جو موجودہ توثیقی نمونوں، غلطی سنبھالنے کی بہترین مشقوں کی پیروی کرتا ہے، اور تازہ ترین SDK خصوصیات کا فائدہ اٹھاتا ہے۔ آپ کو صرف عام کوڈ نہیں ملتا جو کام کر سکتا ہے، بلکہ وہ کوڈ ملتا ہے جو پروڈکشن ورک لوڈز کے لیے Azure کی تجویز کردہ نمونوں کی پیروی کرتا ہے۔

**کلیدی ماڈیولز میں شامل ہیں**:
- **🗄️ ڈیٹا بیس کنیکٹرز**: Azure Database for PostgreSQL اور SQL Server تک براہ راست قدرتی زبان میں رسائی
- **📊 Azure Monitor**: KQL سے چلنے والا لاگ تجزیہ اور آپریشنل بصیرت
- **🌐 وسائل کا انتظام**: مکمل Azure وسائل کی زندگی کے چکر کا انتظام
- **🔐 توثیق**: DefaultAzureCredential اور managed identity کے نمونے
- **📦 اسٹوریج سروسز**: Blob Storage، Queue Storage، اور Table Storage عملیاتی امور
- **🚀 کنٹینر سروسز**: Azure Container Apps، Container Instances، اور AKS کا انتظام
- **اور بہت سے خصوصی کنیکٹرز**

**حقیقی دنیا میں استعمال**: "میری Azure اسٹوریج اکاؤنٹس کی فہرست دکھائیں"، "میرے Log Analytics ورک اسپیس میں پچھلے ایک گھنٹے کی غلطیوں کے لیے سوال کریں"، یا "Node.js کے استعمال سے ایک Azure ایپلیکیشن بنانے میں میری مدد کریں جس میں مناسب توثیق ہو"

**مکمل ڈیمو منظرنامہ**: یہاں ایک مکمل تعارفی عمل ہے جو ظاہر کرتا ہے کہ Azure MCP کو GitHub Copilot for Azure ایکسٹینشن کے ساتھ VS Code میں ملانے کی طاقت کیا ہے۔ جب آپ دونوں انسٹال کر لیتے ہیں اور پرامپٹ کرتے ہیں:

> "ایک Python اسکرپٹ بنائیں جو فائل Azure Blob Storage پر DefaultAzureCredential تصدیق کے ساتھ اپ لوڈ کرے۔ اسکرپٹ میری Azure اسٹوریج اکاؤنٹ 'mycompanystorage' سے جڑے، 'documents' نامی کنٹینر میں اپ لوڈ کرے، موجودہ ٹائم اسٹیمپ کے ساتھ ایک ٹیسٹ فائل بنائے، غلطیوں کو اچھی طرح ہینڈل کرے اور معلوماتی آؤٹ پٹ دے، توثیق اور غلطی سنبھالنے کے لیے Azure کی بہترین مشقوں کی پیروی کرے، DefaultAzureCredential کے کام کرنے کے طریقہ کار کی وضاحت کریں، اور اسکرپٹ کو صحیح فنکشنز اور دستاویزات کے ساتھ منظم بنائے۔"

Azure MCP سرور ایک مکمل، پروڈکشن کے لیے تیار Python اسکرپٹ تیار کرے گا جو:
- تازہ ترین Azure Blob Storage SDK کو مناسب async پیٹرنز کے ساتھ استعمال کرتا ہے
- DefaultAzureCredential کو جامع fallback چین کی وضاحت کے ساتھ نافذ کرتا ہے
- مخصوص Azure استثناء کی اقسام کے ساتھ مضبوط غلطی سنبھالنے کو شامل کرتا ہے
- Azure SDK بہترین مشقوں کی پیروی کرتا ہے، خصوصاً وسائل کے انتظام اور کنیکشن ہینڈلنگ میں
- تفصیلی لاگنگ اور معلوماتی کنسول آؤٹ پٹ فراہم کرتا ہے
- فنکشنز، دستاویزات، اور قسم کی ہدایات کے ساتھ درست اسکرپٹ بناتا ہے

یہ خاص بات یہ ہے کہ Azure MCP کے بغیر، آپ ممکنہ طور پر ایک عام blob storage کوڈ حاصل کرتے جو کام تو کرتا ہے لیکن موجودہ Azure نمونوں کی پیروی نہیں کرتا۔ Azure MCP کے ساتھ، آپ کو ایسا کوڈ ملتا ہے جو تازہ ترین توثیقی طریقوں کا استعمال کرتا ہے، Azure مخصوص غلطی کے حالات کو سنبھالتا ہے، اور Microsoft کی پروڈکشن ایپلیکیشنز کے لیے تجویز کردہ بہترین طریقوں کی پیروی کرتا ہے۔

**نمایاں مثال**: مجھے `az` اور `azd` CLI کمانڈز یاد رکھنے میں مشکل پیش آتی ہے۔ میرے لیے یہ ہمیشہ دو قدمی عمل ہوتا ہے: پہلے سنٹیکس دیکھنا، پھر کمانڈ چلانا۔ میں اکثر پورٹل میں جا کر کام کرنا ترجیح دیتا ہوں کیونکہ مجھے تسلیم کرنا مشکل ہوتا ہے کہ CLI سنٹیکس یاد نہیں رہتی۔ یہ بہت زبردست ہے کہ میں جو چاہتا ہوں اسے بیان کر سکتا ہوں، اور بہتر بات یہ ہے کہ یہ سب کچھ اپنے IDE چھوڑے بغیر کر سکتا ہوں!

شروع کرنے کے لیے [Azure MCP repository](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) میں استعمال کے کیسز کی ایک عمدہ فہرست موجود ہے۔ جامع سیٹ اپ گائیڈز اور جدید کنفیگریشن اختیارات کے لیے [سرکاری Azure MCP دستاویزات](https://learn.microsoft.com/azure/developer/azure-mcp-server/) ملاحظہ کریں۔

**یہ کیوں مفید ہے**: یہ سرور GitHub کے ساتھ آپ کے رابطے کو بدل دیتا ہے اور پورے پلیٹ فارم کا تجربہ براہ راست آپ کے ڈیولپمنٹ ماحول میں لے آتا ہے۔ VS Code اور GitHub.com کے مابین بار بار سوئچ کرنے کی جگہ، آپ قدرتی زبان کے کمانڈز کے ذریعے سب کچھ سنبھال سکتے ہیں جبکہ آپ اپنے کوڈ پر توجہ مرکوز رکھیں۔

> **ℹ️ نوٹ: 'Agents' کی مختلف اقسام**
> 
> اس GitHub MCP سرور کو GitHub کے Coding Agent (جو AI ایجنٹ ہے اور آپ اسے خودکار کوڈنگ ٹاسکس کے لیے GitHub ایشوز پر تفویض کر سکتے ہیں) سے مت بھولیں۔ GitHub MCP سرور VS Code کے Agent موڈ میں GitHub API انٹیگریشن فراہم کرتا ہے، جبکہ GitHub کا Coding Agent ایک علیحدہ فیچر ہے جو GitHub ایشوز پر تفویض کیے جانے پر پل ریکویسٹ بناتا ہے۔

**کلیدی صلاحیتیں شامل ہیں**:
- **⚙️ GitHub Actions**: مکمل CI/CD پائپ لائن کا انتظام، ورک فلو کی نگرانی اور آرٹی فیکٹ ہینڈلنگ
- **🔀 پل ریکویسٹس**: PRs بنائیں، جائزہ لیں، ضم کریں، اور جامع اسٹیٹس ٹریکنگ کے ساتھ مینج کریں
- **🐛 ایشوز**: مکمل ایشو زندگی کے چکر کا انتظام، تبصرے، لیبلنگ، اور تفویض
- **🔒 سیکیورٹی**: کوڈ اسکیننگ الرٹس، خفیہ اشیاء کی دریافت، اور Dependabot انضمام
- **🔔 اطلاع رسانی**: اسمارٹ نوٹیفیکیشن مینجمنٹ اور ریپوزٹری سبسکرپشن کنٹرول
- **📁 ریپوزٹری مینجمنٹ**: فائل آپریشنز، برانچ مینجمنٹ، اور ریپوزٹری انتظامیہ
- **👥 تعاون**: صارف اور تنظیم کی تلاش، ٹیم مینجمنٹ، اور رسائی کنٹرول

**حقیقی دنیا میں استعمال**: "میری فیچر برانچ سے پل ریکویسٹ بنائیں"، "مجھے اس ہفتے کے تمام ناکام CI رنز دکھائیں"، "میری ریپوزٹریز کے لیے تمام کھلے سیکیورٹی الرٹس کی فہرست دکھائیں"، یا "میرے تفویض کیے ہوئے تمام ایشوز میرے تنظیموں میں تلاش کریں"

**مکمل ڈیمو منظرنامہ**: یہاں ایک طاقتور ورک فلو ہے جو GitHub MCP سرور کی صلاحیتوں کا مظاہرہ کرتا ہے:

> "مجھے اپنی اسپرنٹ ریویو کی تیاری کرنی ہے۔ مجھے اس ہفتے کے تمام پل ریکویسٹس دکھائیں جو میں نے بنائے، ہمارے CI/CD پائپ لائنز کی اسٹیٹس چیک کریں، کسی بھی سیکیورٹی الرٹس کا خلاصہ تیار کریں جنہیں حل کرنا ہے، اور 'feature' لیبل والے ضم شدہ PRs کی بنیاد پر ریلیز نوٹس کا مسودہ بنانے میں مدد کریں۔"

GitHub MCP سرور یہ کرے گا:
- آپ کے حالیہ پل ریکویسٹس کو تفصیلی اسٹیٹس معلومات کے ساتھ تلاش کرے
- ورک فلو رنز کا تجزیہ کرے اور کسی بھی ناکامیوں یا کارکردگی کے مسائل کی نشاندہی کرے
- سیکیورٹی اسکیننگ کے نتائج مرتب کرے اور اہم الرٹس کو ترجیح دے
- ضم شدہ PRs سے معلومات نکال کر مکمل ریلیز نوٹس تیار کرے
- اسپرنٹ کی منصوبہ بندی اور ریلیز کی تیاری کے لیے قابل عمل اگلے اقدامات فراہم کرے

**نمایاں مثال**: میں کوڈ ریویو ورک فلو کے لیے اسے بہت پسند کرتا ہوں۔ VS Code، GitHub نوٹیفیکیشنز، اور پل ریکویسٹ صفحات کے درمیان چھلانگ لگانے کی بجائے، میں کہہ سکتا ہوں "میرے ریویو کے انتظار میں تمام PRs دکھائیں" اور پھر "PR #123 پر ایک تبصرہ شامل کریں جو توثیق کے طریقہ کار میں غلطی سنبھالنے کے بارے میں پوچھے۔" سرور GitHub API کالز سنبھالتا ہے، مباحثے کا سیاق و سباق رکھتا ہے، اور مجھے زیادہ تعمیری ریویو تبصرے بنانے میں بھی مدد دیتا ہے۔

**تصدیقی اختیارات**: سرور OAuth (VS Code میں بے دردی سے) اور Personal Access Tokens دونوں کی حمایت کرتا ہے، نوجوان ٹولز کو صرف آپ کی مطلوبہ GitHub فعالیت فعال کرنے کے لیے ترتیب دیا جا سکتا ہے۔ آپ اسے فوری سیٹ اپ کے لیے ریموٹ ہوسٹڈ سروس کی حیثیت سے چلا سکتے ہیں یا مکمل کنٹرول کے لیے مقامی طور پر Docker کے ذریعے چلا سکتے ہیں۔

> **💡 پرو ٹپ**
> 
> صرف وہی ٹول سیٹس فعال کریں جن کی آپ کو ضرورت ہے تاکہ سیاق و سباق کا سائز کم ہو اور AI ٹول کے انتخاب میں بہتری آئے۔ مثال کے طور پر، بنیادی ڈیولپمنٹ ورک فلو کے لیے اپنے MCP کنفیگریشن args میں `"--toolsets", "repos,issues,pull_requests,actions"` شامل کریں، یا اگر آپ بنیادی طور پر GitHub مانیٹرنگ صلاحیتیں چاہتے ہیں تو `"--toolsets", "notifications, security"` استعمال کریں۔

### 5. 📝 MarkItDown MCP Server
[![VS کوڈ میں انسٹال کریں](https://img.shields.io/badge/VS_Code-Install_MarkItDown_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D) [![VS کوڈ انسائڈرز میں انسٹال کریں](https://img.shields.io/badge/VS_Code_Insiders-Install_MarkItDown_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D&quality=insiders) [![گیٹ ہب](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/markitdown)

**یہ کیا کرتا ہے**: MarkItDown ایک جامع دستاویز تبدیل کرنے والا سرور ہے جو مختلف فائل فارمیٹس کو اعلیٰ معیار کے مارک ڈاؤن میں تبدیل کرتا ہے، جو LLM کے استعمال اور متن کے تجزیے کے ورک فلو کے لیے بہتر بنایا گیا ہے۔

**یہ کیوں مفید ہے**: جدید دستاویزی ورک فلو کے لیے ضروری! MarkItDown فائل فارمیٹس کی ایک متاثر کن رینج کو ہینڈل کرتا ہے جبکہ اہم دستاویزاتی ڈھانچے جیسے ہیڈنگز، فہرستیں، جدولیں اور لنکس کو محفوظ رکھتا ہے۔ سادہ متن نکالنے والے آلات کے برعکس، یہ معنوی معنی اور فارمیٹنگ کو برقرار رکھنے پر توجہ دیتا ہے جو AI پروسیسنگ اور انسانی پڑھائی دونوں کے لیے قیمتی ہے۔

**حمایت یافتہ فائل فارمیٹس**:
- **آفس دستاویزات**: پی ڈی ایف، پاورپوائنٹ (PPTX)، ورڈ (DOCX)، ایکسل (XLSX/XLS)
- **میڈیا فائلیں**: تصاویر (EXIF میٹا ڈیٹا اور OCR کے ساتھ)، آڈیو (EXIF میٹا ڈیٹا اور تقریر کی تحریری نقل کے ساتھ)
- **ویب مواد**: HTML، آر ایس ایس فیڈز، یوٹیوب URLs، وکی پیڈیا صفحات
- **ڈیٹا فارمیٹس**: CSV، JSON، XML، ZIP فائلیں (مواد کو تکراری طور پر پروسیس کرتا ہے)
- **پبلشنگ فارمیٹس**: EPub، جیوپیٹر نوٹ بکس (.ipynb)
- **ای میل**: آؤٹ لک پیغامات (.msg)
- **ایڈوانسڈ**: پرفتن پی ڈی ایف پروسیسنگ کے لیے Azure Document Intelligence انٹیگریشن

**ایڈوانسڈ خصوصیات**: MarkItDown LLM سے چلنے والی تصویر کی تفصیل (جب OpenAI کلائنٹ مہیا کیا جائے)، Azure Document Intelligence برائے بہتر PDF پروسیسنگ، تقریر کے مواد کی نقل کاری، اور اضافی فائل فارمیٹس کے لیے پلگ ان سسٹم کی سپورٹ فراہم کرتا ہے۔

**حقیقی دنیا میں استعمال**: "اس پاورپوائنٹ پریزنٹیشن کو ہماری دستاویزی سائٹ کے لیے مارک ڈاؤن میں تبدیل کریں"، "اس PDF سے مناسب ہیڈنگ ساخت کے ساتھ متن نکالیں"، یا "اس ایکسل اسپریڈشیٹ کو پڑھنے کے قابل جدول کے فارمیٹ میں تبدیل کریں"

**نمایاں مثال**: [MarkItDown دستاویزات](https://github.com/microsoft/markitdown#why-markdown) سے اقتباس:

> مارک ڈاؤن بالکل سادہ متن کے بہت قریب ہے، بہت کم مارک اپ یا فارمیٹنگ کے ساتھ، لیکن پھر بھی اہم دستاویزاتی ڈھانچے کو ظاہر کرنے کا طریقہ فراہم کرتا ہے۔ جدید LLMs، جیسے OpenAI کا GPT-4o، نیٹیو طور پر "مارک ڈاؤن" بولتے ہیں، اور اکثر بغیر پوچھے اپنے جوابات میں مارک ڈاؤن شامل کرتے ہیں۔ یہ ظاہر کرتا ہے کہ انہیں بڑی مقدار میں مارک ڈاؤن فارمیٹ شدہ متن پر تربیت دی گئی ہے اور وہ اسے اچھی طرح سمجھتے ہیں۔ ایک ضمنی فائدہ کے طور پر، مارک ڈاؤن کنونشنز بھی انتہائی ٹوکن مؤثر ہیں۔

MarkItDown واقعی دستاویزی ڈھانچے کو برقرار رکھنے میں ماہر ہے، جو AI ورک فلو کے لیے اہم ہے۔ مثال کے طور پر، پاورپوائنٹ پریزنٹیشن کو تبدیل کرتے وقت، یہ سلائیڈ کی تنظیم کو درست ہیڈنگز کے ساتھ رکھتا ہے، جدولوں کو مارک ڈاؤن جدولوں کے طور پر نکالتا ہے، تصاویر کے لیے متبادل متن شامل کرتا ہے، اور اسپیکر نوٹس کو بھی پروسیس کرتا ہے۔ گراف کو پڑھنے کے قابل ڈیٹا ٹیبلز میں تبدیل کیا جاتا ہے، اور پیدا ہونے والا مارک ڈاؤن اصل پریزنٹیشن کے منطقی بہاؤ کو برقرار رکھتا ہے۔ یہ پیشکش کے مواد کو AI نظاموں میں فیڈ کرنے یا موجودہ سلائیڈز سے دستاویزات بنانے کے لیے بالکل موزوں بناتا ہے۔

**نمونہ مثال**: میں نے اپنے دوست John Miller سے رابطہ کیا جب میں نے Build پر M365 Agents کے بارے میں بات کی، اور انہوں نے یہ MCP تجویز کیا۔ یہ M365 Agents کے لیے نئے ڈویلپرز کے لیے بہت مفید ہو سکتا ہے کیونکہ یہ ٹیمپلیٹس، نمونہ کوڈ، اور شروع کرنے کے لیے اسکیفولڈنگ فراہم کرتا ہے، تاکہ دستاویزات میں غرق ہوئے بغیر کام شروع کیا جا سکے۔ اسکیمہ کی توثیق کی خصوصیات خاص طور پر مینیفیسٹ ڈھانچے کی غلطیوں سے بچنے کے لیے مفید ہیں جو گھنٹوں کی ڈیبگنگ کا باعث بن سکتی ہیں۔

> **💡 پرو ٹپ**
> 
> اس سرور کو Microsoft Learn Docs MCP سرور کے ساتھ استعمال کریں تاکہ M365 ڈیولپمنٹ کی مکمل مدد حاصل ہو سکے – ایک سرکاری دستاویزات فراہم کرتا ہے جبکہ یہ عملی ترقیاتی ٹولز اور مسئلہ حل کرنے میں مدد دیتا ہے۔

## آگے کیا ہے؟ 🔮

## 📋 نتیجہ

Model Context Protocol (MCP) اس بات کو تبدیل کر رہا ہے کہ ڈویلپرز AI اسسٹنٹس اور بیرونی ٹولز کے ساتھ کیسے تعامل کرتے ہیں۔ یہ 10 Microsoft MCP سرورز معیاری AI انضمام کی طاقت کو ظاہر کرتے ہیں، جو بغیر رکاوٹ کے ورک فلو مہیا کرتے ہیں جو ڈویلپرز کو ان کی پیداوری کی حالت میں رکھ کر طاقتور بیرونی صلاحیتوں تک رسائی فراہم کرتے ہیں۔

جامع Azure ماحولیاتی نظام کے انضمام سے لے کر Playwright جیسے براؤزر آٹومیشن اور MarkItDown جیسے دستاویز پراسیسنگ کے خصوصی ٹولز تک، یہ سرورز دکھاتے ہیں کہ MCP کیسے مختلف ترقیاتی منظرناموں میں پیداوری کو بڑھا سکتا ہے۔ معیاری پروٹوکول اس بات کو یقینی بناتا ہے کہ یہ ٹولز ہم آہنگی سے کام کریں، جو ایک مربوط ترقیاتی تجربہ پیدا کرتے ہیں۔

جیسے جیسے MCP ماحولیاتی نظام ترقی کرتا ہے، کمیونٹی کے ساتھ منسلک رہنا، نئے سرورز تلاش کرنا، اور کسٹم حل بنانا آپ کی ترقیاتی پیداوری کو زیادہ سے زیادہ کرنے کی کلید ہو گا۔ MCP کی اوپن اسٹینڈرڈ نوعیت کا مطلب ہے کہ آپ مختلف وینڈرز کے ٹولز کو مکس اور میچ کر سکتے ہیں تاکہ اپنی مخصوص ضروریات کے لیے بہترین ورک فلو تیار کیا جا سکے۔
