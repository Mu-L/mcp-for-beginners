# اسٹڈی پلان جنریٹر وِد چین لِٹ اور مائیکروسافٹ لرن ڈاکس MCP

## پیشگی شرائط

- پائتھون 3.8 یا اس سے اوپر
- pip (پائتھون پیکیج مینیجر)
- مائیکروسافٹ لرن ڈاکس MCP سرور سے جڑنے کے لیے انٹرنیٹ کی رسائی

## تنصیب

1. اس رپوزیٹری کو کلون کریں یا پراجیکٹ فائلز ڈاؤن لوڈ کریں۔
2. مطلوبہ dependencies انسٹال کریں:

   ```bash
   pip install -r requirements.txt
   ```

## استعمال

### منظر نامہ 1: ڈاکس MCP کو سادہ سوال
ایک کمانڈ لائن کلائنٹ جو Docs MCP سرور سے جڑتا ہے، سوال بھیجتا ہے، اور نتیجہ پرنٹ کرتا ہے۔

1. اسکرپٹ چلائیں:
   ```bash
   python scenario1.py
   ```
2. پرامپٹ پر اپنی دستاویزی سوال درج کریں۔

### منظر نامہ 2: اسٹڈی پلان جنریٹر (چین لِٹ ویب ایپ)
ایک ویب پر مبنی انٹرفیس (چین لِٹ کا استعمال کرتے ہوئے) جو صارفین کو کسی بھی تکنیکی موضوع کے لیے ذاتی نوعیت کا ہفتہ بہ ہفتہ اسٹڈی پلان تیار کرنے کی اجازت دیتا ہے۔

1. چین لِٹ ایپ شروع کریں:
   ```bash
   chainlit run scenario2.py
   ```
2. اپنی ٹرمینل میں دی گئی مقامی یو آر ایل (مثلاً http://localhost:8000) اپنے براؤزر میں کھولیں۔
3. چیٹ ونڈو میں اپنا مطالعہ موضوع اور مطالعہ کے لیے مطلوبہ ہفتوں کی تعداد درج کریں (مثلاً "AI-900 certification, 8 weeks")۔
4. ایپ آپ کو ہفتہ بہ ہفتہ اسٹڈی پلان کے ساتھ جواب دے گی، جس میں متعلقہ مائیکروسافٹ لرن دستاویزات کے لنکس شامل ہوں گے۔

**ضروری ماحولیاتی متغیرات:**

منظر نامہ 2 (Azure OpenAI کے ساتھ چین لِٹ ویب ایپ) استعمال کرنے کے لیے، آپ کو `python` ڈائریکٹری میں `.env` فائل میں درج ذیل ماحولیاتی متغیرات سیٹ کرنے ہوں گے:

```
AZURE_OPENAI_CHAT_DEPLOYMENT_NAME=
AZURE_OPENAI_API_KEY=
AZURE_OPENAI_ENDPOINT=
AZURE_OPENAI_API_VERSION=
```

ایپ چلانے سے پہلے اپنے Azure OpenAI وسائل کی تفصیلات کے ساتھ ان ویلیوز کو پُر کریں۔

> [!TIP]
> آپ [Microsoft Foundry](https://ai.azure.com/) کا استعمال کرتے ہوئے آسانی سے اپنے ماڈلز تعینات کر سکتے ہیں۔

### منظر نامہ 3: VS Code میں MCP سرور کے ساتھ ان ایڈیٹر ڈاکس

دستاویزات تلاش کرنے کے لیے براؤزر کے ٹیبز تبدیل کرنے کی بجائے، آپ Microsoft Learn Docs کو براہ راست اپنے VS Code میں MCP سرور کے ذریعے لا سکتے ہیں۔ اس سے آپ کو اجازت ملتی ہے:
- VS Code میں رہتے ہوئے دستاویزات تلاش اور پڑھیں بغیر اپنے کوڈنگ ماحول کو چھوڑے۔
- دستاویزات کا حوالہ دیں اور لنکس براہ راست اپنے README یا کورس فائلوں میں داخل کریں۔
- GitHub Copilot اور MCP کا استعمال کرکے ایک مربوط، AI سے چلنے والا دستاویزات کا ورک فلو بنائیں۔

**مثالی استعمال کی صورتیں:**
- کورس یا پروجیکٹ کی دستاویزات لکھتے ہوئے فوری طور پر README میں حوالہ کے لنکس شامل کریں۔
- کوڈ بنانے کے لیے Copilot اور متعلقہ دستاویزات فوراً تلاش کرنے کے لیے MCP استعمال کریں۔
- اپنے ایڈیٹر میں توجہ مرکوز رکھیں اور پیداواری صلاحیت میں اضافہ کریں۔

> [!IMPORTANT]
> یقینی بنائیں کہ آپ کے ورک اسپیس میں ایک درست [`mcp.json`](../../../../../../09-CaseStudy/docs-mcp/solution/scenario3/mcp.json) کنفیگریشن موجود ہے (مقام `.vscode/mcp.json` ہے)۔

## منظر نامہ 2 کے لیے چین لِٹ کیوں؟

چین لِٹ ایک جدید اوپن سورس فریم ورک ہے جو بات چیت پر مبنی ویب ایپلیکیشنز بنانے کے لیے ہے۔ یہ آسان بناتا ہے چیٹ بیسڈ یوزر انٹرفیسز تخلیق کرنا جو Microsoft Learn Docs MCP سرور جیسے بیک اینڈ سروسز سے جڑیں۔ یہ پروجیکٹ چین لِٹ کو استعمال کرتا ہے تاکہ ذاتی نوعیت کے اسٹڈی پلان حقیقی وقت میں تیار کرنے کا ایک سادہ، انٹرایکٹو طریقہ فراہم کیا جا سکے۔ چین لِٹ کا فائدہ اٹھاتے ہوئے، آپ تیزی سے چیٹ بیسڈ ٹولز بنا اور تعینات کر سکتے ہیں جو پیداواری صلاحیت اور سیکھنے کو بڑھائیں۔

## یہ کیا کرتا ہے

یہ ایپ صارفین کو اجازت دیتی ہے کہ وہ صرف ایک موضوع اور دورانیہ درج کرکے ذاتی نوعیت کا اسٹڈی پلان بنائیں۔ ایپ آپ کے ان پٹ کو پارس کرتی ہے، متعلقہ مواد کے لیے Microsoft Learn Docs MCP سرور سے سوالات بھیجتی ہے، اور نتائج کو ایک منظم ہفتہ بہ ہفتہ پلان میں ترتیب دیتی ہے۔ ہر ہفتے کی سفارشات چیٹ میں دکھائی جاتی ہیں، جس سے آپ آسانی سے اپنی پیشرفت کو فالو اور ٹریک کر سکتے ہیں۔ انضمام یقینی بناتا ہے کہ آپ کو ہمیشہ تازہ ترین، سب سے متعلقہ لرننگ وسائل ملیں۔
