# AGENTS.md

## پروجیکٹ کا جائزہ

**MCP برائے مبتدیان** ماڈل کانٹیکسٹ پروٹوکول (MCP) سیکھنے کے لیے ایک اوپن سورس تعلیمی نصاب ہے - جو AI ماڈلز اور کلائنٹ ایپلیکیشنز کے مابین تعاملات کے لیے ایک معیاری فریم ورک ہے۔ یہ ریپوزٹری مختلف پروگرامنگ زبانوں میں عملی کوڈ کی مثالوں کے ساتھ جامع تعلیمی مواد فراہم کرتی ہے۔

### کلیدی ٹیکنالوجیز

- **پروگرامنگ زبانیں**: C#, Java, JavaScript, TypeScript, Python, Rust  
- **فریم ورکس اور SDKs**:  
  - MCP SDK (`@modelcontextprotocol/sdk`)  
  - Spring Boot (Java)  
  - FastMCP (Python)  
  - LangChain4j (Java)  
- **ڈیٹابیسز**: PostgreSQL بمع pgvector توسیع  
- **کلاؤڈ پلیٹ فارمز**: Azure (Container Apps, OpenAI, Content Safety, Application Insights)  
- **بلڈ ٹولز**: npm, Maven, pip, Cargo  
- **دستاویزات**: مارکڈاؤن بمع خودکار کثیر لسانی ترجمہ (48+ زبانیں)  

### ساخت

- **11 بنیادی ماڈیولز (00-11)**: بنیادیات سے لے کر اعلیٰ موضوعات تک متسلسل تعلیمی راہ  
- **عملی تجرباتی لیبز**: مکمل حل کے کوڈ کے ساتھ عملی مشقیں متعدد زبانوں میں  
- **نمونہ پروجیکٹس**: کام کرنے والا MCP سرور اور کلائنٹ نفاذ  
- **ترجمہ نظام**: کثیراللسانی معاونت کے لیے خودکار GitHub Actions ورک فلو  
- **تصویری اثاثے**: مرکزیت شدہ تصاویر کا فولڈر مع ترجمہ شدہ ورژنز  

## سیٹ اپ کمانڈز

یہ ایک دستاویزی مرکزیت والی ریپوزٹری ہے۔ زیادہ تر سیٹ اپ نمونہ پروجیکٹس اور لیبز کے اندر ہوتا ہے۔

### ریپوزٹری سیٹ اپ

```bash
# ریپوزٹری کو کلون کریں
git clone https://github.com/microsoft/mcp-for-beginners.git
cd mcp-for-beginners
```
  
### نمونہ پروجیکٹس کے ساتھ کام کرنا

نمونہ پروجیکٹس درج ذیل جگہوں پر موجود ہیں:  
- `03-GettingStarted/samples/` - زبان مخصوص مثالیں  
- `03-GettingStarted/01-first-server/solution/` - پہلے سرور کے نفاذ  
- `03-GettingStarted/02-client/solution/` - کلائنٹ نفاذ  
- `11-MCPServerHandsOnLabs/` - جامع ڈیٹابیس انٹیگریشن لیبز  

ہر نمونہ پروجیکٹ میں اپنے سیٹ اپ ہدایات موجود ہیں:

#### TypeScript/JavaScript پروجیکٹس  
```bash
cd <project-directory>
npm install
npm start
```
  
#### Python پروجیکٹس  
```bash
cd <project-directory>
pip install -r requirements.txt
# یا
pip install -e .
python main.py
```
  
#### Java پروجیکٹس  
```bash
cd <project-directory>
mvn clean install
mvn spring-boot:run
```

3. co-op-translator GitHub Action خود بخود ترجمے بنائے گا  
4. ترجمے مین برانچ پر push ہونے پر دوبارہ تخلیق کیے جاتے ہیں

- ترجمہ میٹا ڈیٹا ہر ترجمہ شدہ فائل میں شامل ہوتا ہے

## کوڈ اسٹائل رہنما اصول

### دستاویزی انداز

- واضح، مبتدی دوست زبان استعمال کریں  
- جہاں مناسب ہو، متعدد زبانوں میں کوڈ مثالیں شامل کریں  
- مارکڈاؤن کی بہترین مشقیں اپنائیں:  
  - ATX اسٹائل ہیڈرز (`#` سینٹیکس)  
  - زبان کے شناخت کنندگان کے ساتھ fenced کوڈ بلاکس  
  - تصاویر کے لیے وضاحتی alt متن شامل کریں  
  - لائن کی لمبائی مناسب رکھیں (سخت حد نہیں لیکن معقول ہوں)  

### کوڈ سمپل اسٹائل

#### TypeScript/JavaScript  
- ES ماڈیولز (`import`/`export`) استعمال کریں  
- TypeScript کے سخت موڈ اصول اپنائیں  
- قسم کے تشریحات شامل کریں  
- ES2022 کو ہدف بنائیں  

#### Python  
- PEP 8 اسٹائل گائیڈ لائنز کی پیروی کریں  
- جہاں مناسب ہو ٹائپ ہنٹس استعمال کریں  
- فنکشنز اور کلاسز کے لیے docstrings شامل کریں  
- جدید Python خصوصیات (3.8+) استعمال کریں  

#### Java  
- Spring Boot کنونشنز کی پیروی کریں  
- Java 21 خصوصیات استعمال کریں  
- معیاری Maven پروجیکٹ ڈھانچہ اپنائیں  
- Javadoc تبصرے شامل کریں

## تعمیری اور تعیناتی

### دستاویزی تعیناتی

اگر لاگو ہو، تو یہ ریپوزٹری GitHub Pages یا مماثل سروسز دستاویزات کی میزبانی کے لیے استعمال کرتی ہے۔ مین برانچ میں تبدیلیاں مندرجہ ذیل کو متحرک کرتی ہیں:

## تعاون کے اصول

### پل ریکویسٹ کا عمل

1. **فورک اور کلون کریں**: ریپوزٹری کو فورک کریں اور اپنی فورک کو مقامی طور پر کلون کریں  
2. **برانچ بنائیں**: وضاحتی برانچ نام استعمال کریں (مثلاً `fix/typo-module-3`, `add/python-example`)  
3. **تبدیلیاں کریں**: صرف انگریزی مارکڈاؤن فائلوں میں ترمیم کریں (ترجموں میں نہیں)  
4. **مقامی طور پر ٹیسٹ کریں**: تصدیق کریں کہ مارکڈاؤن صحیح طریقے سے رینڈرز ہو  
5. **PR جمع کروائیں**: واضح PR عنوانات اور وضاحتیں استعمال کریں  
6. **CLA**: جب پوچھا جائے تو Microsoft Contributor License Agreement پر دستخط کریں  

### PR عنوان کی شکل

واضح اور وضاحتی عنوانات استعمال کریں:  
- `[Module XX] مختصر وضاحت` ماڈیول مخصوص تبدیلیوں کے لیے  
- `[Samples] وضاحت` نمونہ کوڈ کی تبدیلیوں کے لیے  
- `[Docs] وضاحت` عمومی دستاویزی اپ ڈیٹس کے لیے  

### کیا تعاون کریں

- دستاویزات یا کوڈ نمونہ جات میں بگ فکسز  
- اضافی زبانوں میں نئے کوڈ مثالیں  
- موجودہ مواد کی وضاحت اور اصلاحات  
- نئے کیس اسٹڈیز یا عملی مثالیں  
- غیر واضح یا غلط مواد کی رپورٹیں  

### کیا نہ کریں

- `translations/` فولڈر کی فائلوں میں مستقیم ترمیم نہ کریں  
- `translated_images/` فولڈر میں ترمیم نہ کریں  
- بغیر گفت و شنید کے بڑے بائنری فائلز نہ شامل کریں  
- ترجمہ ورک فلو فائلوں کو بغیر ہم آہنگی کے تبدیل نہ کریں
