# ایم سی پی سیکیورٹی: اے آئی سسٹمز کے لیے جامع تحفظ

[![ایم سی پی سیکیورٹی بہترین طریقے](../../../translated_images/ur/03.175aed6dedae133f.webp)](https://youtu.be/88No8pw706o)

_(اس سبق کی ویڈیو دیکھنے کے لیے اوپر دی گئی تصویر پر کلک کریں)_

سیکیورٹی اے آئی سسٹم ڈیزائن کا بنیادی جزو ہے، اسی لیے ہم اسے اپنے دوسرے حصے میں ترجیح دیتے ہیں۔ یہ مائیکروسافٹ کے **Secure by Design** اصول کے مطابق ہے جو [Secure Future Initiative](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) سے لیا گیا ہے۔

ماڈل کانٹیکسٹ پروٹوکول (MCP) اے آئی پر مبنی ایپلیکیشنز میں طاقتور نئی صلاحیتیں لاتا ہے، لیکن ساتھ ہی ایسے منفرد سیکیورٹی مسائل بھی جن کی روایتی سوفٹ ویئر خطرات سے آگے بڑھ کر روک تھام کرنی پڑتی ہے۔ MCP سسٹمز کو روایتی سیکیورٹی خدشات (محفوظ کوڈنگ، کم سے کم اختیارات، سپلائی چین سیکیورٹی) کے ساتھ ساتھ نئے AI-مخصوص خطرات کا بھی سامنا ہے جن میں پرامپٹ انجیکشن، ٹول زہر آلودگی، سیشن ہائی جیکنگ، کنفیوزڈ ڈیپٹی حملے، ٹوکن پاس تھرو کمزوریاں، اور متحرک صلاحیتوں کی تبدیلی شامل ہیں۔

یہ سبق MCP نفاذ میں سب سے اہم حفاظتی خطرات کو اجاگر کرتا ہے—جن میں تصدیق، اجازت، ضرورت سے زیادہ اختیارات، بالواسطہ پرامپٹ انجیکشن، سیشن سیکیورٹی، کنفیوزڈ ڈیپٹی مسائل، ٹوکن مینجمنٹ، اور سپلائی چین کی کمزوریاں شامل ہیں۔ آپ ان خطرات کو کم کرنے کے لیے عملی کنٹرولز اور بہترین طریقے سیکھیں گے، اور مائیکروسافٹ کے حل جیسے کہ Prompt Shields، Azure Content Safety، اور GitHub Advanced Security کا استعمال کر کے اپنی MCP تنصیب کو مضبوط بنائیں گے۔

## سیکھنے کے مقاصد

اس سبق کے اختتام تک، آپ قابل ہو جائیں گے کہ:

- **MCP-مخصوص خطرات کی شناخت کریں**: MCP سسٹمز میں منفرد سیکیورٹی خطرات جیسے کہ پرامپٹ انجیکشن، ٹول زہر آلودگی، ضرورت سے زیادہ اختیارات، سیشن ہائی جیکنگ، کنفیوزڈ ڈیپٹی مسائل، ٹوکن پاس تھرو کمزوریاں، اور سپلائی چین کے خطرات کو پہچانیں
- **سیکیورٹی کنٹرولز کا اطلاق کریں**: مؤثر حفاظتی تدابیر نافذ کریں جن میں مضبوط تصدیق، کم سے کم اختیارات تک رسائی، محفوظ ٹوکن مینجمنٹ، سیشن سیکیورٹی کنٹرولز، اور سپلائی چین کی تصدیق شامل ہیں
- **مائیکروسافٹ کے سیکیورٹی حل استعمال کریں**: MCP ورک لوڈ کی حفاظت کے لیے Microsoft Prompt Shields، Azure Content Safety، اور GitHub Advanced Security کو سمجھیں اور نافذ کریں
- **ٹول سیکیورٹی کی توثیق کریں**: ٹول میٹاڈیٹا کی توثیق کی اہمیت کو سمجھیں، متحرک تبدیلیوں کی نگرانی کریں، اور بالواسطہ پرامپٹ انجیکشن حملوں سے دفاع کریں
- **بہترین طریقے شامل کریں**: روایتی سیکیورٹی بنیادی اصولوں (محفوظ کوڈنگ، سرور ہارڈننگ، زیرو ٹرسٹ) کو MCP-مخصوص کنٹرولز کے ساتھ ملا کر مکمل تحفظ فراہم کریں

# MCP سیکیورٹی فن تعمیر اور کنٹرولز

جدید MCP نفاذ میں سطح بہ سطح سیکیورٹی طریقے کی ضرورت ہوتی ہے جو روایتی سوفٹ ویئر سیکیورٹی اور AI-مخصوص خطرات دونوں کو حل کریں۔ تیزی سے ترقی پذیر MCP وضاحت اپنے سیکیورٹی کنٹرولز کو بہتر کر رہی ہے، جس سے انٹرپرائز سیکیورٹی فن تعمیر اور مستند بہترین طریقوں کے ساتھ بہتر انضمام ممکن ہوتا ہے۔

[Microsoft Digital Defense Report](https://aka.ms/mddr) کی تحقیق ظاہر کرتی ہے کہ **98% رپورٹس شدہ خلاف ورزیاں مضبوط سیکیورٹی حفظان صحت سے روکی جا سکتی ہیں**۔ سب سے مؤثر حفاظتی حکمت عملی بنیادی سیکیورٹی طریقہ کار کو MCP-مخصوص کنٹرولز کے ساتھ ملانا ہے—ثابت شدہ بنیادی حفاظتی تدابیر مجموعی سیکیورٹی خطرے میں نمایاں کمی لاتی ہیں۔

## موجودہ حفاظتی منظرنامہ

> **نوٹ:** یہ معلومات MCP سیکیورٹی معیارات کی عکاسی کرتی ہے جو **5 فروری 2026** کو، **MCP Specification 2025-11-25** کے مطابق ہیں۔ MCP پروٹوکول تیزی سے ترقی کر رہا ہے، اور آئندہ نفاذ نئی تصدیقی پیٹرنز اور بہتر کنٹرولز متعارف کرو سکتے ہیں۔ تازہ ترین رہنمائی کے لیے ہمیشہ موجودہ [MCP Specification](https://spec.modelcontextprotocol.io/)، [MCP GitHub repository](https://github.com/modelcontextprotocol)، اور [سیکیورٹی بہترین طریقوں کی دستاویزات](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) کا حوالہ دیں۔

## 🏔️ MCP سیکیورٹی سمٹ ورکشاپ (شرپا)

**ہاتھوں ہاتھ سیکیورٹی کی تربیت** کے لیے، ہم **MCP سیکیورٹی سمٹ ورکشاپ** (شرپا) کی سختی سے سفارش کرتے ہیں — مائیکروسافٹ ایزور میں MCP سرورز کو محفوظ کرنے کے لیے مکمل رہنمائی یافتہ مہم۔

### ورکشاپ کا جائزہ

[MCP سیکیورٹی سمٹ ورکشاپ](https://azure-samples.github.io/sherpa/) ایک "کمزوری → استحصال → درستگی → توثیق" طریقہ کار کے ذریعے عملی، قابل عمل سیکیورٹی تربیت فراہم کرتا ہے۔ آپ کریں گے:

- **خرابیوں کے ذریعے سیکھیں**: جان بوجھ کر کمزور سرورز کو استحصال کر کے نقائص کا تجربہ کریں
- **ایزور نیٹیو سیکیورٹی استعمال کریں**: Azure Entra ID، Key Vault، API Management، اور AI Content Safety کا فائدہ اٹھائیں
- **ڈیفنس ان ڈیپتھ پر عمل کریں**: کیمپوں کے ذریعے مرحلہ وار مکمل سیکیورٹی تہوں کی تعمیر کریں
- **اوواسپ معیارات اپنائیں**: ہر تکنیک [OWASP MCP Azure Security Guide](https://microsoft.github.io/mcp-azure-security-guide/) سے مماثل ہوتی ہے
- **پیداواری کوڈ حاصل کریں**: کام کرنے والے، جانچے ہوئے نفاذ کے ساتھ راستہ ہموار کریں

| خطرہ | وضاحت | ایزور میں حفاظت |
|------|-------------|------------------|
| **MCP01** | ٹوکن کی غلط مینجمنٹ اور راز فاش ہونا | Azure Key Vault، Managed Identity |

### MCP تصدیق کا ارتقاء

MCP تفصیل نے تصدیق اور اجازت کے طریقوں میں نمایاں تبدیلی کی ہے:

- **اصل طریقہ کار**: پرانے معیارات میں ڈویلپرز کو اپنی مرضی کے مطابق تصدیقی سرور بنانے کی ضرورت تھی، جہاں MCP سرورز نے صارف کی تصدیق براہ راست OAuth 2.0 آتھورائزیشن سرور کے طور پر کی
- **موجودہ معیار (2025-11-25)**: اپ ڈیٹ شدہ وضاحت MCP سرورز کو باہر کے شناختی فراہم کنندگان (جیسے Microsoft Entra ID) کو تصدیق تفویض کرنے کی اجازت دیتی ہے، جس سے سیکیورٹی بہتر اور نفاذ آسان ہو جاتا ہے
- **ٹرانسپورٹ لیئر سیکیورٹی**: مقامی (STDIO) اور دور دراز (Streamable HTTP) کنکشنز کے لیے مناسب تصدیقی پیٹرنز کے ساتھ محفوظ ٹرانسپورٹ میکانزم کی بہتر حمایت

## تصدیق اور اجازت سیکیورٹی

### موجودہ حفاظتی چیلنجز

جدید MCP نفاذ چند اہم تصدیقی اور اجازت کے مسائل کا سامنا کرتے ہیں:

### خطرات اور حملے کے ذرائع

- **غلط طور پر ترتیب دی گئی اجازت کا منطق**: MCP سرورز میں خراب اجازت کا نفاذ حساس ڈیٹا کو بے نقاب کر سکتا ہے اور غلط رسائی کنٹرول لگا سکتا ہے
- **OAuth ٹوکن کا سمجھوتہ**: مقامی MCP سرور کے ٹوکن چوری ہو جانے سے حملہ آور سرور کا نقشہ بنا کر نیچے کی خدمات تک رسائی حاصل کر سکتے ہیں
- **ٹوکن پاس تھرو کمزوریاں**: نامناسب ٹوکن ہینڈلنگ سیکیورٹی کنٹرولز کو نظر انداز کرنے اور جوابدہی کے خلا پیدا کرنے کی اجازت دیتی ہے
- **ضرورت سے زیادہ اختیارات**: اوور پرولیجڈ MCP سرورز کم از کم اختیارات کے اصول کی خلاف ورزی کرتے ہیں اور حملے کی سطح کو بڑھاتے ہیں

#### ٹوکن پاس تھرو: ایک اہم اینٹی پیٹرن

**ٹوکن پاس تھرو موجودہ MCP اجازت کی وضاحت میں سختی سے ممنوع ہے** کیونکہ اس کے سنگین سیکیورٹی اثرات ہیں:

##### سیکیورٹی کنٹرول کی بائی پاسنگ
- MCP سرورز اور نیچے کی APIs اہم سیکیورٹی کنٹرولز (رفتار کی حد بندی، درخواست کی توثیق، ٹریفک کی نگرانی) لاگو کرتے ہیں جو صحیح ٹوکن ویریفکیشن پر منحصر ہیں
- کلائنٹ سے براہ راست API کو ٹوکن کا استعمال ان ضروری پروٹیکشنز کو بائی پاس کر دیتا ہے، اور سیکیورٹی فن تعمیر کو کمزور کرتا ہے

##### جوابدہی اور آڈٹ کے مسائل
- MCP سرورز اپ اسٹریم جاری کردہ ٹوکن استعمال کرنے والے کلائنٹس میں فرق نہیں کر سکتے، جس سے آڈٹ ٹریلز ٹوٹ جاتے ہیں
- نیچے کی ریسورس سرور لاگز غلط درخواست کے ماخذ ظاہر کرتے ہیں بجائے حقیقی MCP سرور کے
- واقعہ کی تحقیقات اور کمپلائنس آڈٹنگ بہت مشکل ہو جاتی ہے

##### ڈیٹا چوری کے خطرات
- غیر تصدیق شدہ ٹوکن دعوے چوری شدہ ٹوکن رکھنے والے بد نیتی افراد کو MCP سرورز کو بطور پراکسی استعمال کرنے کی اجازت دیتے ہیں
- اعتماد کی حدوں کی خلاف ورزی غیر مجاز رسائی کے نمونے پیدا کرتی ہے جو سیکیورٹی کنٹرولز کو بائی پاس کرتی ہے

##### کثیر سروس حملے کے ذرائع
- متعدد خدمات کی جانب سے قبول شدہ سمجھوتہ شدہ ٹوکنز مربوط نظاموں میں مجموعی نقل و حرکت کو ممکن بناتے ہیں
- جب ٹوکن کے ماخذ کی تصدیق نہ ہو سکے تو خدمات کے درمیان اعتماد کی مفروضات شکسته ہو سکتی ہیں

### حفاظتی کنٹرولز اور کمی

**اہم حفاظتی تقاضے:**

> **ضروری**: MCP سرورز **کبھی بھی** ایسے ٹوکن قبول نہیں کریں جو واضح طور پر سرور کے لیے جاری نہ کیے گئے ہوں

#### تصدیق اور اجازت کے کنٹرولز

- **سخت اجازت کا جائزہ**: MCP سرور کی اجازت منطق کا جامع آڈٹ کریں تاکہ صرف مطلوبہ صارفین اور کلائنٹس حساس وسائل تک رسائی حاصل کر سکیں
  - **عمل درآمد کی رہنمائی**: [Azure API Management as Authentication Gateway for MCP Servers](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **شناخت کا انضمام**: [Microsoft Entra ID کا استعمال MCP سرور کی توثیق کے لیے](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **محفوظ ٹوکن مینجمنٹ**: [Microsoft کے ٹوکن کی توثیق اور لائف سائیکل کے بہترین طریقے](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens) نافذ کریں
  - ٹوکن کے آڈینس کلیمز کی توثیق کریں جو MCP سرور کی شناخت سے میل کھاتے ہوں
  - مناسب ٹوکن روٹیشن اور اختتام کی پالیسیاں نافذ کریں
  - ٹوکن کی دوبارہ حملہ اور غیر مجاز استعمال کو روکیں

- **محفوظ ٹوکن اسٹوریج**: آرام اور ترسیل کے دوران انکرپشن کے ساتھ محفوظ اسٹوریج
  - **بہترین طریقے**: [محفوظ ٹوکن اسٹوریج اور انکرپشن کی رہنمائی](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### رسائی کنٹرول کا نفاذ

- **کم از کم مراعات کا اصول**: MCP سرورز کو صرف وہ کم از کم اجازت دیں جو مطلوبہ فعالیت کے لیے ضروری ہو
  - مراعات کے جائزے اور تازہ کاریوں کے ذریعے مراعات کی حد بندی کو یقینی بنائیں
  - **مائیکروسافٹ دستاویزات**: [محفوظ کم از کم مراعات کی رسائی](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **رول کی بنیاد پر رسائی کا کنٹرول (RBAC)**: باریک بینی سے رول تفویض کریں
  - مخصوص وسائل اور سرگرمیوں کے لیے رولز کو محدود رکھیں
  - وسیع یا غیر ضروری اختیارات سے گریز کریں جو حملے کی سطح کو بڑھاتے ہیں

- **مستقل اجازت مانیٹرنگ**: رسائی کے آڈٹ اور نگرانی کا تسلسل رکھیں
  - اجازت کے استعمال کے رجحانات میں غیر معمولی چیزوں پر نظر رکھیں
  - ضرورت سے زیادہ یا استعمال نہ ہونے والی مراعات کو فوری درست کریں

## AI-مخصوص سیکیورٹی خطرات

### پرامپٹ انجیکشن اور ٹول میں مداخلت کے حملے

جدید MCP نفاذ میں پیچیدہ AI-مخصوص حملے شامل ہیں جو روایتی سیکیورٹی اقدامات پوری طرح سے قابو نہیں پا سکتے:

#### **بالواسطہ پرامپٹ انجیکشن (کراس ڈومین پرامپٹ انجیکشن)**

**بالواسطہ پرامپٹ انجیکشن** MCP پر مبنی AI سسٹمز میں ایک سب سے زیادہ اہم کمزوری ہے۔ حملہ آور بیرونی مواد—دستاویزات، ویب صفحات، ای میلز، یا ڈیٹا سورسز—میں بدنیتی پر مبنی ہدایات داخل کرتے ہیں جو AI سسٹمز بعد میں جائز احکامات کے طور پر عمل کرتے ہیں۔

**حملے کے منظرنامے:**
- **دستاویز کی بنیاد پر انجیکشن**: پروسیس کی گئی دستاویزات میں چھپی ہوئی بدنیتی پر مبنی ہدایات جو AI کو غیر ارادی کارروائیاں کرنے پر مجبور کرتی ہیں
- **ویب مواد کا استحصال**: متاثر شدہ ویب صفحات میں شامل پرامپٹس جو AI رویے میں مداخلت کرتے ہیں جب وہ اسکرپ کیے جاتے ہیں
- **ای میل پر مبنی حملے**: ای میلز میں بدنیتی پر مبنی پرامپٹس جو AI اسسٹنٹس کو معلومات افشا کرنے یا غیر مجاز عمل کرنے پر مجبور کرتے ہیں
- **ڈیٹا سورس کی آلودگی**: متاثرہ ڈیٹا بیس یا APIs جو بدنیتی پر مبنی مواد AI سسٹمز کو فراہم کرتے ہیں

**حقیقی دنیا کا اثر:** یہ حملے ڈیٹا کی چوری، پرائیویسی کی خلاف ورزی، نقصان دہ مواد کی تخلیق، اور صارف کے تعاملات میں مداخلت کا سبب بن سکتے ہیں۔ تفصیلی تجزیے کے لیے، دیکھیں [Prompt Injection in MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/)۔

![پرامپٹ انجیکشن حملے کا خاکہ](../../../translated_images/ur/prompt-injection.ed9fbfde297ca877.webp)

#### **ٹول زہر آلودگی کے حملے**

**ٹول زہر آلودگی** MCP ٹولز کو بیان کرنے والے میٹاڈیٹا کو نشانہ بناتی ہے، جو LLMs کے ذریعے ٹول کی وضاحت اور پیرامیٹرز کو سمجھ کر عمل درآمد کے فیصلے کرتی ہے۔

**حملے کے طریقہ کار:**
- **میٹاڈیٹا میں مداخلت**: حملہ آور ٹول کی وضاحتوں، پیرامیٹر کی تعریفوں، یا استعمال کی مثالوں میں بدنیتی پر مبنی ہدایات داخل کرتے ہیں
- **غیر مرئی ہدایات**: ٹول میٹاڈیٹا میں پوشیدہ پرامپٹس جو AI ماڈلز پراسیس کرتے ہیں لیکن انسان صارفین کے لیے غیر مرئی ہوتے ہیں
- **متحرک ٹول تبدیلی ("رگ پلز")**: صارفین کی منظوری کے بعد ٹولز کو بدنیتی پر مبنی کارروائیاں کرنے کے لیے تبدیل کیا جاتا ہے بغیر صارف کی معلومات کے
- **پیرامیٹر انجیکشن**: ٹول کے پیرامیٹر اسکیموں میں بدنیتی پر مبنی مواد جو ماڈل کے رویے کو متاثر کرتا ہے

**میزبانی سرور کے خطرات:** دور دراز کے MCP سرورز زیادہ خطرات پیش کرتے ہیں کیوں کہ ٹول کی وضاحتیں بعد میں اپ ڈیٹ کی جا سکتی ہیں، جس سے پہلے محفوظ ٹولز بدنیتی پر مبنی ہو سکتے ہیں۔ جامع تجزیہ کے لیے دیکھیں [Tool Poisoning Attacks (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks)۔

![ٹول انجیکشن حملے کا خاکہ](../../../translated_images/ur/tool-injection.3b0b4a6b24de6bef.webp)

#### **اضافی AI حملے کے ذرائع**

- **کراس ڈومین پرامپٹ انجیکشن (XPIA):** پیچیدہ حملے جو متعدد ڈومینز کے مواد کا استعمال کر کے سیکیورٹی کنٹرولز کو بائی پاس کرتے ہیں
- **متحرک صلاحیت میں تبدیلی**: اوزار کی صلاحیتوں میں حقیقی وقت میں تبدیلیاں جو ابتدائی سیکیورٹی جائزوں سے بچ نکلتی ہیں  
- **سیاق و سباق ونڈو کا زہر آلود کرنا**: حملے جو وسیع سیاق و سباق کی ونڈوز کو قابو پانے کے لئے استعمال ہوتے ہیں تاکہ نقصان دہ ہدایات چھپائی جا سکیں  
- **ماڈل الجھن کے حملے**: ماڈل کی محدودیتوں کا فائدہ اٹھا کر غیر متوقع یا غیر محفوظ رویے پیدا کرنا  

### AI سیکیورٹی رسک کا اثر

**اعلی اثرات کے نتائج:**  
- **ڈیٹا کی چوری**: غیر مجاز رسائی اور حساس ادارتی یا ذاتی ڈیٹا کی چوری  
- **رازداری کی خلاف ورزیاں**: ذاتی طور پر شناخت پذیر معلومات (PII) اور خفیہ کاروباری ڈیٹا کا افشاء  
- **نظام میں مداخلت**: اہم نظاموں اور ورک فلو میں غیر ارادی تبدیلیاں  
- **شناختی اسناد کی چوری**: تصدیقی علامات اور سروس کی شناختی اسناد کا سمجھوتہ  
- **افقی نقل و حرکت**: متاثرہ AI نظاموں کو وسیع نیٹ ورک حملوں کے لئے محوری نقطہ کے طور پر استعمال کرنا  

### مائیکروسافٹ AI سیکیورٹی حل

#### **AI پرامپٹ شیلڈز: انجیکشن حملوں کے خلاف جدید تحفظ**

مائیکروسافٹ **AI پرامپٹ شیلڈز** کئی سیکیورٹی تہوں کے ذریعے براہ راست اور بالواسطہ پرامپٹ انجیکشن حملوں کے خلاف جامع دفاع فراہم کرتی ہیں:

##### **مرکزی حفاظتی میکانزم:**  

1. **جدید دریافت اور فلٹرنگ**  
   - مشین لرننگ الگوردمز اور NLP تکنیکیں خارجی مواد میں نقصان دہ ہدایات کو شناخت کرتی ہیں  
   - دستاویزات، ویب صفحات، ای میلز، اور ڈیٹا ذرائع کی حقیقی وقت میں تجزیہ برائے مخفی خطرات  
   - جائز اور نقصان دہ پرامپٹ پیٹرنز کی سیاق و سباق کی تفہیم  

2. **اسپٹلائٹنگ تکنیکیں**  
   - معتبر نظام کی ہدایات اور ممکنہ طور پر متاثرہ خارجی ان پٹس میں تمیز کرتی ہیں  
   - متن کی تبدیلی کے طریقے جو ماڈل کی مطابقت میں اضافہ کرتے ہیں اور نقصان دہ مواد کو الگ کرتے ہیں  
   - AI نظاموں کو مناسب ہدایتی درجہ بندی برقرار رکھنے اور انجیکٹ کی گئی کمانڈز کو نظرانداز کرنے میں مدد دیتی ہیں  

3. **ڈیلمیٹر اور ڈیٹا مارکنگ سسٹمز**  
   - معتبر نظام پیغامات اور خارجی ان پٹ کے درمیان واضح حد بندی  
   - مخصوص نشانات جو معتبر اور غیر معتبر ڈیٹا ذرائع کے درمیان حد بندی کو نمایاں کرتے ہیں  
   - واضح علیحدگی ہدایات کی الجھن اور غیر مجاز کمانڈز کی انجام دہی کو روکتی ہے  

4. **مسلسل خطرہ انٹیلی جنس**  
   - مائیکروسافٹ ابھرتے ہوئے حملے کے نمونوں کی نگرانی مسلسل کرتا ہے اور دفاع کو اپ ڈیٹ کرتا ہے  
   - نئے انجیکشن تکنیکوں اور حملہ آور راستوں کے لئے فعال خطرہ تلاش  
   - بدلتے ہوئے خطرات کے خلاف مؤثر رہنے کے لئے باقاعدہ سیکیورٹی ماڈل اپ ڈیٹس  

5. **Azure مواد کی حفاظت کا انضمام**  
   - جامع Azure AI مواد کی حفاظت کے سوئٹ کا حصہ  
   - جیل بریک کوششوں، نقصان دہ مواد، اور سیکیورٹی پالیسی کی خلاف ورزیوں کے لئے اضافی دریافت  
   - AI ایپلیکیشن کے اجزاء میں متحدہ سیکیورٹی کنٹرولز

## جدید MCP سیکیورٹی خطرات

### سیشن ہائی جیکنگ کی کمزوریاں

**سیشن ہائی جیکنگ** ایک اہم حملہ کا راستہ ہے جو Stateful MCP نفاذ میں ہوتا ہے جہاں غیر مجاز فریق جائز سیشن شناختی کو حاصل کر کے کلائنٹس کی صورت دھار کر غیر مجاز کارروائیاں انجام دیتے ہیں۔  

#### **حملے کے منظرنامے اور خطرات**

- **سیشن ہائی جیک پرامپٹ انجیکشن**: چوری شدہ سیشن شناختی کے حامل حملہ آور سرورز میں نقصان دہ واقعات انجیکٹ کرتے ہیں جو سیشن کی حالت شیئر کرتے ہیں، جو ممکنہ طور پر نقصان دہ کارروائیاں یا حساس ڈیٹا تک رسائی کا سبب بنتا ہے  
- **براہ راست بہروپ**: چوری شدہ سیشن شناختی سے بغیر تصدیق کے سیدھے MCP سرور کالز ممکن ہوتی ہیں جو حملہ آور کو جائز صارف سمجھتی ہیں  
- **متاثرہ ریزومیبل سلسلے**: حملہ آور درخواستوں کو قبل از وقت ختم کر سکتے ہیں، جس سے جائز کلائنٹس کو ممکنہ طور پر نقصان دہ مواد کے ساتھ دوبارہ شروع کرنا پڑ سکتا ہے  

#### **سیشن مینجمنٹ کے لئے حفاظتی کنٹرولز**

**اہم ضروریات:**  
- **اختیارات کی توثیق**: MCP سرورز جو اجازت نامہ نافذ کرتے ہیں **ضروری ہے** کہ تمام آنے والی درخواستیں توثیق کریں اور سیشن پر تصدیق کے لئے انحصار نہ کریں  
- **محفوظ سیشن جنریشن**: کرپٹوگرافک طور پر محفوظ، غیر متعین شدہ سیشن شناختی استعمال کریں جو Secured Random Number Generators سے پیدا ہوں  
- **صارف مخصوص بندھن**: سیشن شناختی کو صارف مخصوص معلومات سے جوڑیں جیسا کہ `<user_id>:<session_id>` تاکہ صارفین کے درمیان سیشن کے غلط استعمال سے بچا جا سکے  
- **سیشن لائف سائیکل مینجمنٹ**: مناسب ختم ہونے، گردش، اور منسوخی کے عمل کا نفاذ تاکہ کمزوری کی مدت کو محدود کیا جا سکے  
- **ٹرانسپوٹ سیکیورٹی**: تمام مواصلات کے لئے لازمی HTTPS تاکہ سیشن شناختی کی چوری کو روکا جا سکے  

### الجھے ہوئے ڈیپٹی کا مسئلہ

**الجھے ہوئے ڈیپٹی کا مسئلہ** اس وقت وقوع پزیر ہوتا ہے جب MCP سرورز کلائنٹس اور تھرڈ پارٹی سروسز کے درمیان توثیق پراکسی کے طور پر کام کرتے ہیں، جس سے جامد کلائنٹ ID کے استحصال کے ذریعے اجازت نامہ کی بائی پاس کے مواقع پیدا ہوتے ہیں۔  

#### **حملے کے طریقہ کار اور خطرات**

- **کوکی پر مبنی رضامندی بائی پاس**: سابقہ صارف کی توثیق رضامندی کی کوکیز پیدا کرتی ہے جنہیں حملہ آور نقصان دہ اجازت نامہ درخواستوں کے ذریعے تیار کردہ ری ڈائریکٹ URI کے ذریعے استحصال کرتے ہیں  
- **اجازت نامہ کوڈ کی چوری**: موجودہ رضامندی کی کوکیز اجازت نامہ سرورز کو اجازت اسکرینز چھوڑنے پر مجبور کر سکتی ہیں، کوڈز کو حملہ آور کے کنٹرول والے اینڈ پوائنٹس کی طرف ری ڈائریکٹ کرتی ہیں  
- **غیر مجاز API رسائی**: چوری شدہ اجازت نامہ کوڈز کی مدد سے بغیر واضح منظوری کے ٹوکن ایکسچینج اور صارف کی بہروپی ممکن ہے  

#### **تدارکی حکمت عملیاں**

**لازمی کنٹرول:**  
- **واضح رضامندی کی ضرورت**: MCP پراکسی سرورز جو جامد کلائنٹ IDs استعمال کرتے ہیں **ضروری ہے** کہ ہر متحرک رجسٹرڈ کلائنٹ کے لئے صارف کی رضامندی حاصل کریں  
- **OAuth 2.1 سیکیورٹی نفاذ**: موجودہ OAuth سیکیورٹی بہترین عمل بشمول PKCE (پروف کی) تمام اجازت نامہ درخواستوں میں نافذ کریں  
- **سخت کلائنٹ تصدیق**: ری ڈائریکٹ URI اور کلائنٹ شناخت کنندگان کی سخت توثیق تاکہ استحصال کو روکا جا سکے  

### ٹوکن پاس تھرو کی کمزوریاں  

**ٹوکن پاس تھرو** ایک واضح خلاف ورزی ہے جہاں MCP سرورز مناسب توثیق کے بغیر کلائنٹ ٹوکن قبول کرتے ہیں اور انہیں ڈاؤن سٹریم APIs کو بھیج دیتے ہیں، جو MCP اجازت نامہ وضاحتوں کی خلاف ورزی ہے۔  

#### **سیکیورٹی مضمرات**

- **کنٹرول سے بچ نکلنا**: کلائنٹ سے API تک براہ راست ٹوکن کا استعمال اہم ریٹ لمٹنگ، توثیق، اور مانیٹرنگ کنٹرولز کو بائی پاس کرتا ہے  
- **آڈٹ ٹریل کی خرابی**: اپ اسٹریم جاری کردہ ٹوکنز کلائنٹ کی شناخت ناممکن بناتے ہیں، حادثہ کی تفتیش کی صلاحیت کو نقصان پہنچاتے ہیں  
- **پراکسی پر مبنی ڈیٹا چوری**: غیر توثیق شدہ ٹوکنز سے نقصان دہ فریقین کو غیر مجاز ڈیٹا تک رسائی کے لئے سرورز کا پراکسی کے طور پر استعمال ممکن ہوتا ہے  
- **اعتماد کی حد کی خلاف ورزیاں**: جب ٹوکن کے ماخذ کی تصدیق ممکن نہ ہو تو نیچے کی طرف سروسز کی اعتماد کی مفروضات متاثر ہو سکتی ہیں  
- **کئی خدمات پر حملے کی توسیع**: متاثرہ ٹوکنز جو متعدد خدمات میں قبول کیے جاتے ہیں افقی نقل و حرکت کو ممکن بناتے ہیں  

#### **لازمی حفاظتی کنٹرولز**

**ناقابلِ مذاکرات تقاضے:**  
- **ٹوکن کی توثیق**: MCP سرورز کو **ضروری ہے** کہ وہ صرف وہی ٹوکن قبول کریں جو مخصوص طور پر MCP سرور کے لیے جاری کیے گئے ہوں  
- **مستمع کی تصدیق**: ہمیشہ ٹوکن کے مستمع کے دعووں کی توثیق کریں کہ وہ MCP سرور کی شناخت سے میل کھاتے ہوں  
- **صحیح ٹوکن لائف سائیکل**: مختصر مدت کے ایکسیس ٹوکنز اور محفوظ گردش کے عمل نافذ کریں  

## AI نظاموں کے لئے سپلائی چین سیکیورٹی

سپلائی چین سیکیورٹی روایتی سافٹ ویئر انحصار سے آگے بڑھ کر پورے AI ماحول کو شامل کرتی ہے۔ جدید MCP نفاذ سختی سے تمام AI متعلقہ اجزاء کی تصدیق اور نگرانی کریں، کیونکہ ہر ایک ممکنہ کمزوری لے کر آتا ہے جو نظام کی سالمیت کو مجروح کر سکتی ہے۔  

### توسیع شدہ AI سپلائی چین کے اجزاء

**روایتی سافٹ ویئر انحصار:**  
- اوپن سورس لائبریریز اور فریم ورک  
- کنٹینر امیجز اور بنیادی نظام  
- ترقیاتی آلات اور بلڈ پائپ لائنز  
- انفراسٹرکچر اجزاء اور خدمات  

**AI مخصوص سپلائی چین کے عناصر:**  
- **بنیادیاتی ماڈلز**: مختلف فراہم کنندگان کے پیشگی تربیت یافتہ ماڈلز جن کی ماخذ کی تصدیق ضروری ہے  
- **ایمبیڈنگ خدمات**: خارجی ویکٹرائزیشن اور معنوی تلاش کی خدمات  
- **سیاق و سباق فراہم کرنے والے**: ڈیٹا ذرائع، علم کے ذخائر، اور دستاویزی مخزن  
- **تیسری پارٹی APIs**: خارجی AI خدمات، مشین لرننگ پائپ لائنز، اور ڈیٹا پراسیسنگ اینڈ پوائنٹس  
- **ماڈل آرٹی فیکٹس**: وزن، کنفیگریشنز، اور فائن ٹیون شدہ ماڈل مختلف اقسام  
- **تربیتی ڈیٹا ذرائع**: وہ ڈیٹاسیٹس جو ماڈل کی تربیت اور بہتری کے لئے استعمال ہوتے ہیں  

### جامع سپلائی چین سیکیورٹی حکمت عملی

#### **جزو کی تصدیق اور اعتماد**  
- **ماخذ کی توثیق**: انضمام سے پہلے تمام AI اجزاء کے ماخذ، لائسنسنگ، اور سالمیت کی تصدیق  
- **سیکیورٹی کا جائزہ**: ماڈلز، ڈیٹا ذرائع، اور AI خدمات کے لئے کمزوریوں کے اسکین اور سیکیورٹی جائزے  
- **شہرت کا تجزیہ**: AI سروس فراہم کنندگان کے سیکیورٹی ریکارڈ اور طریقہ کار کی جانچ  
- **تعمیل کی تصدیق**: تمام اجزاء کی تنظیمی سیکیورٹی اور ضابطہ کی ضروریات کے مطابق جانچ  

#### **محفوظ نافذ کاری پائپ لائنز**  
- **خودکار CI/CD سیکیورٹی**: خودکار نافذکاری پائپ لائنز میں سیکیورٹی اسکین شامل کریں  
- **آرٹیفیکٹ کی سالمیت**: تمام آرٹیفیکٹس (کوڈ، ماڈلز، کنفیگریشنز) کی کرپٹوگرافک تصدیق  
- **مرحلہ وار نفاذ**: ہر مرحلے پر سیکیورٹی کی توثیق کے ساتھ تدریجی نفاذ کی حکمت عملی  
- **معتبر آرٹیفیکٹ ذخائر**: صرف جانچے ہوئے، محفوظ آرٹیفیکٹ ریجسٹریز اور ذخائر سے تعیناتی  

#### **مسلسل نگرانی اور جواب**  
- **انحصاری اسکیننگ**: تمام سافٹ ویئر اور AI اجزاء کی کمزوریوں کی مسلسل نگرانی  
- **ماڈل مانیٹرنگ**: ماڈل کے رویے، کارکردگی میں تبدیلی، اور سیکیورٹی انومالیوں کا مسلسل جائزہ  
- **سروس صحت کی نگرانی**: خارجی AI خدمات کی دستیابی، سیکیورٹی واقعات، اور پالیسی تبدیلیوں کی نگرانی  
- **خطرہ انٹیلی جنس انضمام**: AI اور مشین لرننگ سیکیورٹی خطرات کے لئے مخصوص خطرے کے فیڈز کو شامل کرنا  

#### **رسائی کنٹرول اور کم از کم مراعات**  
- **جزو کی سطح کی اجازتیں**: کاروباری ضرورت کی بنیاد پر ماڈلز، ڈیٹا، اور خدمات کی رسائی کو محدود کرنا  
- **سروس اکاؤنٹ مینجمنٹ**: مخصوص خدمات کے اکاؤنٹس جن کے پاس انتہائی کم حقوق ہوں  
- **نیٹ ورک تقسیم**: AI اجزاء کو الگ رکھیں اور خدمات کے درمیان نیٹ ورک کی رسائی محدود کریں  
- **API گیٹ وے کنٹرولز**: خارجی AI خدمات تک رسائی کو کنٹرول اور مانیٹر کرنے کے لئے مرکزی API گیٹ وے استعمال کریں  

#### **حوادث کا جواب اور بحالی**  
- **تیز رفتار جواب کے عمل**: متاثرہ AI اجزاء کے پیچنگ یا تبدیلی کے لئے قائم شدہ عمل  
- **شناختی کو تبدیل کرنا**: راز، API کلیدوں، اور سروس شناخت کو خودکار طور پر تبدیل کرنے کے نظام  
- **واپس لانے کی صلاحیت**: AI اجزاء کے پچھلے معلوم ٹھیک ورژن پر فوری واپسی کی قابلیت  
- **سپلائی چین کی خلاف ورزی سے بازیابی**: اپ اسٹریم AI سروس کے متاثرہ ہونے پر مخصوص ردعمل کے طریقہ کار  

### مائیکروسافٹ سیکیورٹی ٹولز اور انضمام

**GitHub Advanced Security** جامع سپلائی چین تحفظ فراہم کرتا ہے جس میں:  
- **راز اسکیننگ**: ذخائر میں اسناد، API کلیدوں، اور ٹوکنز کی خودکار دریافت  
- **انحصار اسکیننگ**: اوپن سورس انحصارات اور لائبریریز کے لئے کمزوریوں کا جائزہ  
- **CodeQL تجزیہ**: سیکیورٹی کی خامیاں اور کوڈنگ مسائل کے لئے جامد کوڈ تجزیہ  
- **سپلائی چین ان سائٹس**: انحصار کی صحت اور سیکیورٹی کی صورتحال پر نظر  

**Azure DevOps اور Azure Repos انضمام:**  
- مائیکروسافٹ ترقیاتی پلیٹ فارمز میں بآسانی سیکیورٹی اسکین کا انضمام  
- AI کام کے بوجھ کے لئے Azure پائپ لائنز میں خودکار سیکیورٹی چیکس  
- محفوظ AI اجزاء کی تعیناتی کے لیے پالیسی نفاذ  

**مائیکروسافٹ اندرونی طریقہ کار:**  
مائیکروسافٹ تمام مصنوعات میں وسیع سپلائی چین سیکیورٹی طریقہ کار نافذ کرتا ہے۔ [The Journey to Secure the Software Supply Chain at Microsoft](https://devblogs.microsoft.com/engineering-at-microsoft/the-journey-to-secure-the-software-supply-chain-at-microsoft/) میں ثابت شدہ طریقوں کا مطالعہ کریں۔  

## بنیادی سیکیورٹی بہترین عوامل

MCP نفاذ آپ کی تنظیم کی موجودہ سیکیورٹی حالت کو وراثت میں لیتے ہیں اور اس کو مضبوط کرتے ہیں۔ بنیادی سیکیورٹی عوامل کو مضبوط بنانے سے AI نظاموں اور MCP تعیناتیوں کی مجموعی سیکیورٹی میں نمایاں اضافہ ہوتا ہے۔  

### بنیادی سیکیورٹی اصول

#### **محفوظ ترقیاتی طریقے**  
- **OWASP کی تعمیل**: [OWASP Top 10](https://owasp.org/www-project-top-ten/) ویب ایپلیکیشن کمزوریوں کے خلاف تحفظ  
- **AI مخصوص تحفظات**: [LLMs کے لئے OWASP Top 10](https://genai.owasp.org/download/43299/?tmstv=1731900559) کے کنٹرولز نافذ کریں  
- **محفوظ راز انتظام**: ٹوکنز، API چابیاں، اور حساس کنفیگریشن ڈیٹا کے لئے مخصوص والٹ  
- **اختتامی سے اختتامی انکرپشن**: تمام ایپلیکیشن کمپونینٹس اور ڈیٹا فلو کے ذریعے محفوظ مواصلات  
- **ان پٹ کی توثیق**: تمام صارف ان پٹس، API پیرا میٹرز، اور ڈیٹا ذرائع کی سخت توثیق  

#### **بنیادی ڈھانچے کی مضبوطی**  
- **کثیر عوامل کی تصدیق**: تمام ایڈمنسٹریٹو اور سروس اکاؤنٹس کے لیے لازمی MFA  
- **پیچ مینجمنٹ**: آپریٹنگ سسٹمز، فریم ورکز، اور انحصار کے لئے خودکار بروقت پیچنگ  
- **شناخت فراہم کنندہ کا انضمام**: مرکزی شناختی انتظام (Microsoft Entra ID, Active Directory)  
- **نیٹ ورک تقسیم**: MCP اجزاء کی منطقی علیحدگی تاکہ افقی نقل و حرکت کو محدود کیا جا سکے  
- **کم از کم ضروری مراعات کا اصول**: تمام نظامی اجزاء اور اکاؤنٹس کے لئے کم سے کم ضروری اجازتیں  

#### **سیکیورٹی نگرانی اور دریافت**  
- **جامع لاگنگ**: AI ایپلیکیشن سرگرمیوں کی تفصیلی لاگنگ بشمول MCP کلائنٹ-سرور تعاملات  
- **SIEM انضمام**: مرکزیت والی سیکیورٹی معلومات اور واقعہ انتظام برائے انومالی دریافت  
- **رویے کی تجزیہ کاری**: نظام اور صارف رویے میں غیر معمولی پیٹرنز کا AI کی مدد سے پتہ لگانا  
- **خطرہ انٹیلی جنس**: بیرونی خطرہ فیڈز اور کمپرمائز کے اشاریے (IOCs) کا انضمام  
- **حوادث کا جواب**: سیکیورٹی حادثات کی دریافت، جواب، اور بازیابی کے واضح قواعد  

#### **زیرو ٹرسٹ آرکیٹیکچر**  
- **کبھی نہ بھروسہ کرو، ہمیشہ تصدیق کرو**: صارفین، آلات، اور نیٹ ورک کنکشن کی مسلسل تصدیق  
- **مائیکرو سیگمنٹیشن**: انفرادی کاموں اور خدمات کو الگ کرنے والے تفصیلی نیٹ ورک کنٹرولز  
- **شناختی مرکزیت والی سیکیورٹی**: نیٹ ورک مقام کی بجائے تصدیق شدہ شناختوں پر مبنی سیکیورٹی پالیسیاں  
- **مسلسل رسک جائزہ**: موجودہ سیاق و سباق اور رویے کی بنیاد پر متحرک سیکیورٹی حالت کا جائزہ  
- **مشروط رسائی**: رسائی کنٹرولز جو خطرہ عوامل، مقام، اور آلہ کے اعتماد کی بنیاد پر مطابقت پذیر ہوں  

### ادارہ جاتی انضمام کے نمونے

#### **مائیکروسافٹ سیکیورٹی ماحولیاتی نظام کا انضمام**  
- **Microsoft Defender for Cloud**: جامع کلاؤڈ سیکیورٹی پوزیشن مینجمنٹ  
- **Azure Sentinel**: کلاؤڈ نیٹِو SIEM اور SOAR صلاحیتیں AI کام کے بوجھ کے تحفظ کے لئے  
- **Microsoft Entra ID**: ادارہ جاتی شناخت اور رسائی مینجمنٹ بشمول مشروط رسائی پالیسیاں  
- **Azure Key Vault**: مرکزی راز مینجمنٹ جس میں ہارڈ ویئر سیکیورٹی ماڈیول (HSM) کی حمایت  
- **Microsoft Purview**: AI ڈیٹا ذرائع اور ورک فلو کے لئے ڈیٹا گورننس اور تعمیل  

#### **تعمیل اور گورننس**  
- **ضابطہ کی مطابقت**: MCP نفاذ کو صنعت مخصوص تعمیل (GDPR, HIPAA, SOC 2) کی ضروریات پوری کرنی چاہئیں  
- **ڈیٹا کی درجہ بندی**: AI نظاموں کے ذریعہ پراسیس ہونے والے حساس ڈیٹا کی مناسب اقسام اور ہینڈلنگ  
- **آڈٹ ٹریلز**: ضابطہ کی تعمیل اور قانونی تفتیش کے لئے جامع لاگنگ  
- **رازداری کنٹرولز**: AI نظام آرکیٹیکچر میں پرائیویسی بائی ڈیزائن اصولوں کا نفاذ  
- **تبدیلی مینجمنٹ**: AI نظام میں تبدیلیوں کے لئے رسمی سیکیورٹی جائزے کے عمل  

یہ بنیادی عوامل ایک مضبوط سیکیورٹی معیاد تیار کرتے ہیں جو MCP مخصوص سیکیورٹی کنٹرولز کی مؤثریت کو بڑھاتے ہیں اور AI پر مبنی ایپلیکیشنز کے لیے جامع تحفظ فراہم کرتے ہیں۔  

## کلیدی سیکیورٹی نکات  
- **سطح بندی شدہ حفاظتی طریقہ کار**: بنیادی حفاظتی طریقہ کار (محفوظ کوڈنگ، کم از کم اختیارات، سپلائی چین کی تصدیق، مسلسل نگرانی) کو AI سے متعلق کنٹرولز کے ساتھ ملائیں تاکہ جامع تحفظ فراہم کیا جا سکے

- **AI سے متعلق مخصوص خطرات کا منظرنامہ**: MCP سسٹمز کو منفرد خطرات کا سامنا ہوتا ہے جن میں پرامپٹ انجیکشن، ٹول زہر آلودگی، سیشن ہائی جیکنگ، الجھن میں مبتلا معاون کے مسائل، ٹوکن پاس تھرو کمزوریاں، اور ضرورت سے زیادہ اجازتیں شامل ہیں جن کے لیے مخصوص حفاظتی اقدامات کی ضرورت ہوتی ہے

- **تصدیق اور اجازت میں بہتری**: بیرونی شناختی فراہم کنندگان (Microsoft Entra ID) کا استعمال کرتے ہوئے مضبوط تصدیق نافذ کریں، مناسب ٹوکن کی تصدیق کو یقینی بنائیں، اور کبھی بھی ایسے ٹوکن قبول نہ کریں جو خاص طور پر آپ کے MCP سرور کے لیے جاری نہ کیے گئے ہوں

- **AI حملوں کی روک تھام**: Microsoft Prompt Shields اور Azure Content Safety نافذ کریں تاکہ بالواسطہ پرامپٹ انجیکشن اور ٹول زہر آلودگی کے حملوں سے دفاع کیا جا سکے، ساتھ ہی ٹول میٹاڈیٹا کی تصدیق اور متحرک تبدیلیوں کی نگرانی کریں

- **سیشن اور ٹرانسپورٹ سیکیورٹی**: کرپٹوگرافک طور پر محفوظ، غیر متعین سیشن IDs استعمال کریں جو صارف کی شناخت سے منسلک ہوں، مناسب سیشن لائف سائیکل مینجمنٹ نافذ کریں، اور سیشنز کو تصدیق کے لیے استعمال نہ کریں

- **OAuth حفاظتی بہترین طریقے**: ڈائنامک طور پر رجسٹرڈ کلائنٹس کے لیے صریح صارف کی رضامندی کے ذریعے الجھن میں مبتلا معاون حملوں کی روک تھام کریں، PKCE کے ساتھ OAuth 2.1 کا مناسب نفاذ کریں، اور سخت ری ڈائریکٹ URI کی تصدیق کریں

- **ٹوکن حفاظتی اصول**: ٹوکن پاس تھرو اینٹی پیٹرنز سے بچیں، ٹوکن کے سامعین کے دعوے کی تصدیق کریں، محفوظ گردش کے ساتھ مختصر مدت کے ٹوکن نافذ کریں، اور واضح اعتماد کی حد بندی برقرار رکھیں

- **جامع سپلائی چین سیکیورٹی**: AI ماحولیاتی نظام کے تمام اجزاء (ماڈلز، ایمبیڈنگز، کانٹیکسٹ فراہم کنندگان، بیرونی APIs) کو روایتی سافٹ ویئر انحصارات کی طرح سخت حفاظتی معیار کے ساتھ برتاؤ کریں

- **مسلسل ارتقاء**: تیزی سے ترقی پاتا ہوا MCP وضاحتوں کے ساتھ تازہ رہیں، حفاظتی کمیونٹی کے معیارات میں حصہ لیں، اور پروٹوکول کی ترقی کے ساتھ موافق حفاظتی رویے برقرار رکھیں

- **Microsoft حفاظتی انضمام**: Microsoft کے جامع حفاظتی ماحولیاتی نظام (Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID) کو زیادہ بہتر MCP تعیناتی کے تحفظ کے لیے استعمال کریں
