## आढावा

**मॉडेल कॉन्टेक्स्ट प्रोटोकॉल (MCP)** AI सहाय्यकांना बाह्य डेटा स्रोतांशी सुरक्षितपणे प्रवेश करण्याची आणि संवाद साधण्याची रिअल-टाईम क्षमता देते. डेटाबेस एकीकरणासह संयुक्त झाल्यास, MCP डेटा संचालित AI अनुप्रयोगांसाठी शक्तिशाली क्षमता अनलॉक करते.

हा शिक्षण मार्ग तुम्हाला उत्पादन-तयार MCP सर्व्हर्स तयार करायला शिकवतो जे AI सहाय्यकांना PostgreSQL च्या माध्यमातून रिटेल विक्री डेटाशी जोडतात, ज्यात रो लेव्हल सिक्युरिटी, सेंसिक सर्च आणि मल्टी-टेनेन्ट डेटा प्रवेश यांसारखी एंटरप्राइझ पद्धती समाविष्ट आहेत.

### मुख्य घटक

#### **1. MCP सर्व्हर लेयर**
- **FastMCP Framework**: आधुनिक Python MCP सर्व्हर अंमलबजावणी
- **टूल नोंदणी**: प्रकार सुरक्षिततेसह घोषणात्मकरित्या टूल परिभाषा
- **रिक्वेस्ट कॉन्टेक्स्ट**: वापरकर्ता ओळख आणि सत्र व्यवस्थापन
- **त्रुटी हाताळणी**: मजबूत त्रुटी व्यवस्थापन आणि लॉगिंग

#### **2. डेटाबेस एकीकरण लेयर**
- **कनेक्शन पूलिंग**: कार्यक्षम asyncpg कनेक्शन व्यवस्थापन
- **स्कीमा प्रदाता**: डायनॅमिक टेबल स्कीमा शोध
- **क्वेरी अंमलकारक**: RLS कॉन्टेक्स्टसह सुरक्षित SQL अंमलबजावणी
- **ट्रान्झॅक्शन व्यवस्थापन**: ACID अनुपालन आणि रोलबॅक हाताळणी

#### **3. सुरक्षा लेयर**
- **रो लेव्हल सिक्युरिटी**: PostgreSQL RLS मल्टी-टेनेन्ट डेटा पृथक्करणासाठी
- **वापरकर्ता ओळख**: स्टोअर मॅनेजर प्रमाणीकरण आणि अधिकृतता
- **अॅक्सेस कंट्रोल**: सूक्ष्म परवानग्या आणि ऑडिट ट्रेल्स
- **इनपुट व्हॅलिडेशन**: SQL इंजेक्शन प्रतिबंध आणि क्वेरी प्रमाणीकरण

#### **4. AI वाढवणी लेयर**
- **सेंसिक सर्च**: उत्पादन शोधासाठी व्हेक्टर एम्बेडिंग्ज
- **Azure OpenAI एकीकरण**: टेक्स्ट एम्बेडिंग निर्मिती
- **समानता अल्गोरिदम**: pgvector कोसाईन समानता शोध
- **शोध ऑप्टिमायझेशन**: इंडेक्सिंग आणि कार्यक्षमता सुधारणा

### परिस्थिति 1: स्टोअर मॅनेजर कार्यक्षमता पुनरावलोकन

**वापरकर्ता**: सारा, सिएटल स्टोअर मॅनेजर  
**उद्देश**: मागील तिमाहीची विक्री कार्यक्षमता विश्लेषित करा

**नैसर्गिक भाषा क्वेरी**:
> "माझ्या स्टोअरमधील Q4 2024 च्या शीर्ष 10 उत्पादनांची महसुलानुसार यादी दाखवा"

**घटनेची मालिका**:
1. VS Code AI Chat MCP सर्व्हरला क्वेरी पाठवतो
2. MCP सर्व्हर सारा चा स्टोअर कॉन्टेक्स्ट (सिएटल) ओळखतो
3. RLS धोरणे सिएटल स्टोअरचा डेटा फिल्टर करतात
4. SQL क्वेरी तयार केली जाते आणि अंमलात आणली जाते
5. निकाल स्वरूपित करून AI चॅटला परत पाठवला जातो
6. AI विश्लेषण आणि अंतर्दृष्टी प्रदान करतो

### परिस्थिति 2: सेंसिक सर्चसह उत्पादन शोध

**वापरकर्ता**: माइक, इन्व्हेंटरी मॅनेजर  
**उद्देश**: ग्राहक विनंतीसारखे उत्पादन शोधा

**नैसर्गिक भाषा क्वेरी**:
> "आम्ही कोणती उत्पादने विकतो जी 'बाहेर वापरासाठी जलरोधक इलेक्ट्रिकल कनेक्टर्स' सारख्या आहेत?"

**घटनेची मालिका**:
1. क्वेरी सेंसिक सर्च टूलने प्रक्रिया केली जाते
2. Azure OpenAI एम्बेडिंग व्हेक्टर तयार करतो
3. pgvector समानता शोध पार पाडतो
4. संबंधित उत्पादने त्यांच्या सान्निध्यामुळे क्रमवारी लावली जातात
5. निकालांमध्ये उत्पादन तपशील आणि उपलब्धता असते
6. AI पर्यायी आणि बंडल पर्याय सुचवते

### परिस्थिति 3: क्रॉस-स्टोर अॅनालिटिक्स

**वापरकर्ता**: जेनिफर, प्रादेशिक मॅनेजर  
**उद्देश**: सर्व स्टोर्समधील कार्यक्षमता तुलना करा

**नैसर्गिक भाषा क्वेरी**:

**घटनेची मालिका**:

## 🔒 सुरक्षा आणि मल्टी-टेनेन्सी सखोल माहिती

आमची अंमलबजावणी एंटरप्राइझ दर्जाच्या सुरक्षेला प्राधान्य देते:

### रो लेव्हल सिक्युरिटी (RLS)

PostgreSQL RLS डेटा पृथक्करण सुनिश्चित करते:

```sql
-- Store managers see only their store's data
CREATE POLICY store_manager_policy ON retail.orders
  FOR ALL TO store_managers
  USING (store_id = get_current_user_store());

-- Regional managers see multiple stores
CREATE POLICY regional_manager_policy ON retail.orders
  FOR ALL TO regional_managers
  USING (store_id = ANY(get_user_store_list()));
```


### वापरकर्ता ओळख व्यवस्थापन

प्रत्येक MCP कनेक्शनमध्ये असते:
- **स्टोअर मॅनेजर आयडी**: RLS कॉन्टेक्स्टसाठी अद्वितीय ओळख
- **भूमिका नेमणूक**: परवानग्या आणि प्रवेश पातळ्या
- **सत्र व्यवस्थापन**: सुरक्षित प्रमाणीकरण टोकन्स
- **ऑडिट लॉगिंग**: संपूर्ण प्रवेश इतिहास

### डेटा संरक्षण

सुरक्षेच्या अनेक स्तरांवर:
- **कनेक्शन एन्क्रिप्शन**: सर्व डेटाबेस कनेक्शनसाठी TLS
- **SQL इंजेक्शन प्रतिबंध**: फक्त पॅरामीटराइज्ड क्वेरीज
- **इनपुट व्हॅलिडेशन**: सर्वसमावेशक विनंती प्रमाणीकरण
- **त्रुटी हाताळणी**: त्रुटी संदेशांमध्ये संवेदनशील डेटा नाही
