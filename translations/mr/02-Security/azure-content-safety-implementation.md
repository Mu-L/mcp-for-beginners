# MCP सह Azure कंटेंट सेफ्टीची अंमलबजावणी

> **OWASP MCP धोका निवारण**: [MCP06 - Intent Flow Subversion](https://microsoft.github.io/mcp-azure-security-guide/mcp/mcp06-prompt-injection/)

प्रॉम्प्ट इंजेक्शन, टूल विषबाधा आणि इतर AI-विशिष्ट असुरक्षा विरुद्ध MCP सुरक्षेला मजबूत करण्यासाठी Azure कंटेंट सेफ्टीचे एकत्रीकरण अत्यंत शिफारसीय आहे. ही अंमलबजावणी मार्गदर्शिका [MCP Security Summit Workshop (Sherpa)](https://azure-samples.github.io/sherpa/) कॅम्प 3: I/O सुरक्षा यासाठी अनुरूप आहे.

## MCP सर्व्हरशी एकत्रीकरण

Azure कंटेंट सेफ्टी तुमच्या MCP सर्व्हरमध्ये एकत्रित करण्यासाठी, रिक्वेस्ट प्रोसेसिंग पाईपलाइनमध्ये कंटेंट सेफ्टी फिल्टर मिडलवेअर म्हणून जोडा:

1. सर्व्हर सुरू होताना फिल्टर प्रारंभ करा
2. सर्व येणाऱ्या टूल विनंत्यांची प्रक्रिया करण्यापूर्वी वैधता तपासा
3. सर्व जात असलेल्या प्रतिसादांची ग्राहकांना परत करण्यापूर्वी तपासणी करा
4. सुरक्षा उल्लंघने नोंदवा आणि सतर्क रहा
5. कंटेंट सेफ्टी तपासणी अयशस्वी झाल्यास योग्य त्रुटी हाताळणी अंमलात आणा

हे सुरक्षिततेचे मजबूत संरक्षण प्रदान करते:
- प्रॉम्प्ट इंजेक्शन हल्ले
- टूल विषबाधा प्रयत्न
- द्वेषपूर्ण इनपुटद्वारे डेटाचा चोरी करणे
- घातक सामग्री निर्मिती

## Azure कंटेंट सेफ्टी एकत्रीकरणासाठी सर्वोत्तम सराव

1. **कस्टम ब्लॉकलिस्ट**: MCP इंजेक्शन नमुन्यांसाठी विशेष कस्टम ब्लॉकलिस्ट तयार करा
2. **गंभीरता समायोजन**: तुमच्या वापर प्रकरण आणि धोका सहनशक्तीवर आधारित गंभीरतेच्या मर्यादांना समायोजित करा
3. **संपूर्ण कव्हरेज**: सर्व इनपुट आणि आउटपुटवर कंटेंट सेफ्टी तपासण्या लागू करा
4. **प्रदर्शन ऑप्टिमायझेशन**: पुनरावृत्ती होणाऱ्या कंटेंट सेफ्टी तपासण्या साठी कॅशिंगचा विचार करा
5. **फॉलबॅक यंत्रणा**: कंटेंट सेफ्टी सेवा अनुपलब्ध असताना स्पष्ट फॉलबॅक वर्तन परिभाषित करा
6. **वापरकर्ता अभिप्राय**: जेव्हा सुरक्षा कारणास्तव सामग्री अवरोधित केली जाते तेव्हा वापरकर्त्यांना स्पष्ट अभिप्राय द्या
7. **सतत सुधारणा**: उदयोन्मुख धमक्यांवर आधारित ब्लॉकलिस्ट आणि नमुने नियमित अद्यतनित करा
