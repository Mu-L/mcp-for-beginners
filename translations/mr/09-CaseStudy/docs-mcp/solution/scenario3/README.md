# Scenario 3: MCP सर्व्हरसह VS कोडमधील इन-एडिटर दस्तऐवज

## विहंगावलोकन

या परिस्थितीत, तुम्हाला Microsoft Learn Docs थेट तुमच्या Visual Studio Code पर्यावरणात MCP सर्व्हर वापरून कसे आणायचे हे शिकवले जाईल. दस्तऐवज शोधण्यासाठी सतत ब्राउझर टॅब स्विच करण्याऐवजी, तुम्ही अधिकृत दस्तऐवज थेट तुमच्या संपादकाच्या आत प्रवेश करू शकता, शोधू शकता आणि संदर्भ घेऊ शकता. या पद्धतीने तुमचा कार्यप्रवाह सुलभ होतो, तुम्ही लक्ष केंद्रित ठेवू शकता आणि GitHub Copilot सारख्या साधनांशी सहज एकत्रीकरण शक्य होते.

- VS कोडमध्ये तुमच्या कोडिंग वातावरणातून न निघता दस्तऐवज शोधा आणि वाचा.
- दस्तऐवजाचा संदर्भ घ्या आणि लिंक थेट तुमच्या README किंवा कोर्स फाइल्समध्ये टाका.
- GitHub Copilot आणि MCP एकत्र वापरून एका अखंड, AI-चालित दस्तऐवज कार्यप्रवाहाचा अनुभव घ्या.

## शिकण्याचे उद्दिष्टे

या प्रकरणाच्या शेवटी, तुम्हाला MCP सर्व्हर VS कोडमध्ये कसा सेटअप करायचा आणि वापरायचा हे समजलेले असेल जेणेकरून दस्तऐवज आणि विकास कार्यप्रवाह सुधारू शकतील. तुम्ही सक्षम असाल:

- तुमच्या वर्कस्पेसला MCP सर्व्हर दस्तऐवज शोधासाठी कसे कॉन्फिगर करायचे.
- VS कोडमधून थेट दस्तऐवज शोधणे आणि समाविष्ट करणे.
- एक अधिक आधुनिक, AI-समृद्ध कार्यप्रवाहासाठी GitHub Copilot आणि MCP यांचा वापर करणे.

हे कौशल्य तुम्हाला केंद्रित राहायला मदत करतील, दस्तऐवजाची गुणवत्ता सुधारतील आणि विकासक किंवा तांत्रिक लेखक म्हणून तुमची उत्पादकता वाढवतील.

## उपाय

इन-एडिटर दस्तऐवज प्रवेश साध्य करण्यासाठी, तुम्ही MCP सर्व्हर VS कोड आणि GitHub Copilot सोबत एकत्र करण्यासाठी काही चरणांचे पालन कराल. हा उपाय कोर्स लेखक, दस्तऐवज लेखक आणि विकासकांसाठी आदर्श आहे जे दस्तऐवज आणि Copilot सह काम करताना संपादकावर लक्ष केंद्रित ठेवू इच्छितात.

- कोर्स किंवा प्रोजेक्ट दस्तऐवज लिहिताना README मध्ये संदर्भ लिंक त्वरेने जोडा.
- कोड तयार करण्यासाठी Copilot वापरा आणि संबंधित दस्तऐवज ताबडतोब शोधण्यासाठी MCP वापरा.
- तुमच्या संपादकात लक्ष केंद्रित ठेवा आणि उत्पादकता वाढवा.

### चरण-दर-चरण मार्गदर्शक

प्रारंभ करण्यासाठी, खालील चरणांचे पालन करा. प्रत्येक चरणासाठी, प्रक्रियेचे दृश्यरूप दाखवण्यासाठी तुम्ही assets फोल्डरमधून स्क्रीनशॉट जोडू शकता.

1. **MCP कॉन्फिगरेशन जोडा:**
   तुमच्या प्रोजेक्टच्या मूळ फोल्डरात `.vscode/mcp.json` फाइल तयार करा आणि खालील कॉन्फिगरेशन जोडा:
   ```json
   {
     "servers": {
       "LearnDocsMCP": {
         "url": "https://learn.microsoft.com/api/mcp"
       }
     }
   }
   ```
   ही कॉन्फिगरेशन VS कोडला [`Microsoft Learn Docs MCP सर्व्हर`](https://github.com/MicrosoftDocs/mcp) शी कसे कनेक्ट करायचे ते सांगते.
   
   ![Step 1: Add mcp.json to .vscode folder](../../../../../../translated_images/mr/step1-mcp-json.c06a007fccc3edfa.webp)
    
2. **GitHub Copilot Chat पॅनेल उघडा:**
   जर तुमच्याकडे अद्याप GitHub Copilot एक्सटेंशन इन्स्टॉल नसेल, तर VS कोडमधील एक्सटेंशन्स दृश्यावर जा आणि ते इन्स्टॉल करा. तुम्ही ते थेट [Visual Studio Code Marketplace](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot-chat) वरून डाउनलोड करू शकता. नंतर साइडबारमधून Copilot Chat पॅनेल उघडा.

   ![Step 2: Open Copilot Chat panel](../../../../../../translated_images/mr/step2-copilot-panel.f1cc86e9b9b8cd1a.webp)

3. **एजंट मोड सक्रिय करा आणि साधने तपासा:**
   Copilot Chat पॅनेलमध्ये एजंट मोड सक्षम करा.

   ![Step 3: Enable agent mode and verify tools](../../../../../../translated_images/mr/step3-agent-mode.cdc32520fd7dd1d1.webp)

   एजंट मोड सक्रिय केल्यावर, सक्षम साधनांमध्ये MCP सर्व्हर यादीत आहे की नाही हे तपासा. यामुळे Copilot एजंटला संबंधित माहिती मिळविण्यासाठी दस्तऐवज सर्व्हरवर प्रवेश करता येतो याची खात्री होते.
   
   ![Step 3: Verify MCP server tool](../../../../../../translated_images/mr/step3-verify-mcp-tool.76096a6329cbfecd.webp)

4. **नवीन चॅट सुरू करा आणि एजंटला प्रश्न विचारा:**
   Copilot Chat पॅनेलमध्ये नवीन चॅट उघडा. तुम्ही आता एजंटला तुमचे दस्तऐवज प्रश्न विचारू शकता. एजंट MCP सर्व्हर वापरून संबंधित Microsoft Learn दस्तऐवज थेट संपादकात आणेल.

   - *"मी विषय X साठी अभ्यास योजना लिहित आहे. मी ती 8 आठवडे अभ्यासणार आहे, प्रत्येक आठवड्यासाठी मला कोणते विषय घेणे आवश्यक आहे ते सुचवा."*

   ![Step 4: Prompt the agent in chat](../../../../../../translated_images/mr/step4-prompt-chat.12187bb001605efc.webp)

5. **लाइव क्वेरी:**

   > चला Microsoft Foundry Discord मधील [#get-help](https://discord.gg/D6cRhjHWSC) विभागातून एक थेट क्वेरी घेऊया ([मूळ संदेश पहा](https://discord.com/channels/1113626258182504448/1385498306720829572)):
   
   *"मी Azure AI Foundry वर विकसित केलेल्या AI एजंटांसह मल्टी-एजंट सोल्यूशन कसे डिप्लॉय करायचे याबाबत उत्तर शोधत आहे. मला दिसते की Copilot Studio चॅनेल्ससारखा थेट डिप्लॉयमेंट मार्ग नाहीये. तर, एंटरप्राइझ वापरकर्त्यांनी इंटरॅक्ट करण्यासाठी आणि काम पूर्ण करण्यासाठी या डिप्लॉयमेंटचे विविध मार्ग कोणते आहेत?
बरेच लेख/ब्लॉग्स म्हणतात की Azure Bot सेवा वापरून MS Teams आणि Azure AI Foundry एजंट्समधील पूल म्हणून काम करणे शक्य आहे. पण जर मी Azure bot सेटअप केला जो Azure AI foundry वरील Orchestrator एजंटशी Azure function द्वारा कनेक्ट करतो, तर ते कार्य करेल का? किंवा मला प्रत्येक AI एजंटसाठी Azure function तयार करावा लागेल जे मल्टी एजंट सोल्यूशनचा भाग आहे ज्यायोगे Bot फ्रेमवर्कवर ऑर्केस्ट्रेशन करता येईल? दुसरे काही सल्ले असतील तर ते देखील स्वागतार्ह आहेत."*

   ![Step 5: Live queries](../../../../../../translated_images/mr/step5-live-queries.49db3e4a50bea273.webp)

   एजंट संबंधित दस्तऐवज लिंक आणि सारांशांसह प्रतिसाद देईल, जे तुम्ही थेट तुमच्या मार्कडाउन फाइल्समध्ये समाविष्ट करू शकता किंवा कोडमध्ये संदर्भ म्हणून वापरू शकता.
   
### नमुना प्रश्न

खाली काही उदाहरण प्रश्न आहेत जे तुम्ही वापरून पाहू शकता. हे प्रश्न MCP सर्व्हर आणि Copilot कसे एकत्र काम करू शकतात हे दाखवतील, जेणेकरून तुमच्या VS कोडमधून निघता त्वरित, संदर्भ-साक्षर दस्तऐवज आणि संदर्भ मिळवता येतील:

- "Azure Functions ट्रिगर कसे वापरायचे ते दाखवा."
- "Azure Key Vault साठी अधिकृत दस्तऐवजांची लिंक समाविष्ट करा."
- "Azure संसाधने सुरक्षित करण्यासाठी सर्वोत्तम पद्धती काय आहेत?"
- "Azure AI सेवा साठी एक क्विकस्टार्ट शोधा."

हे प्रश्न MCP सर्व्हर आणि Copilot कसे एकत्र काम करू शकतात हे दाखवतील, जेणेकरून तुमच्या VS कोडमधून निघता त्वरित, संदर्भ-साक्षर दस्तऐवज आणि संदर्भ मिळवता येतील.
