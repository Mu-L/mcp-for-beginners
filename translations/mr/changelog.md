# बदलनोंचा नोंद: MCP for Beginners अभ्यासक्रम

हा दस्तऐवज Model Context Protocol (MCP) for Beginners अभ्यासक्रमात केलेल्या सर्व महत्त्वाच्या बदलांचा नोंद म्हणून वापरला जातो. बदल उलट कालानुक्रमाने (नवीनतम बदल प्रथम) दस्तऐवज केले आहेत.

## १६ जून, २०२६

### MCP तपशील संरेखन & नमुना पडताळणी

सध्याच्या **MCP Specification 2025-11-25** आणि नवीनतम अधिकृत SDKs विरुद्ध अभ्यासक्रमाची पडताळणी केली, नंतर उरलेल्या जुना तपशील संदर्भ दुरुस्त केले आणि मुख्य नमुने अजूनही तयार होतात आणि चालतात हे पुष्टी केले.

#### तपशील आवृत्ती दुरुस्ती (2025-06-18 / 2025-03-26 → 2025-11-25)

जिथे इंग्रजी मजकूर अजून जुना तपशील आवृत्ती "*सध्याचा/नवीनतम*" मानक म्हणून दावा करत होता तेथे अद्ययावत केले आणि `modelcontextprotocol.io` अधिकृत तपशील मार्गांकडे दुवे पुनर्निर्देशित केले:
- **05-AdvancedTopics/mcp-security/README.md**: "सध्याचा मानक" बॅनर, परिचय, मुख्य सुरक्षा तत्त्वे शीर्षक, अनिवार्य गरजा शीर्षक, Microsoft Entra ID विभाग, संदर्भ व संसाधने दुवे, आणि समाप्त सुरक्षा सूचना (८ संदर्भ) 2025-11-25 पर्यंत अद्ययावत केले
- **05-AdvancedTopics/mcp-transport/README.md**: अतिरिक्त संसाधने तपशील लिंक आणि "सध्याचा मानक" बॅनर 2025-11-25 पर्यंत अद्ययावत केले
- **05-AdvancedTopics/mcp-realtimesearch/README.md**: जुन्या `2025-03-26` security-and-trust लिंकची जागा सध्याच्या 2025-11-25 सुरक्षा सर्वोत्तम पद्धतींच्या पेजने घेतली
- **03-GettingStarted/14-sampling/README.md**: अधिकृत नमुना दस्तऐवज लिंक 2025-11-25 वर अद्ययावत केले
- **03-GettingStarted/05-stdio-server/README.md**: वर्तमान काळातील "सध्याचा MCP तपशील" संदर्भ आणि अतिरिक्त संसाधने तपशील लिंक 2025-11-25 पर्यंत अद्ययावत केले (ऐतिहासिक SSE-अपवाद नोट्स अक्युरेटसाठी ठेवलेल्या)

#### सध्याच्या SDK विरुद्ध नमुना पडताळणी

- **TypeScript (03-GettingStarted/01-first-server/solution/typescript)**: `npm install` ने `@modelcontextprotocol/sdk@1.29.0` समाधान केले; `tsc --noEmit` कोणतीही प्रकार त्रुटीशिवाय पास झाला — विद्यमान `McpServer`/`StdioServerTransport` API अजूनही वैध
- **Python (03-GettingStarted/01-first-server/solution/python)**: वेगळ्या `.venv` मध्ये `mcp[cli]` (1.27.2) सह पडताळले; `py_compile` पास आणि `FastMCP.list_tools()` योग्यरित्या `add` आणि `subtract` साधने परत केली
- सर्व नमुना `@modelcontextprotocol/sdk` आवृत्ती श्रेणी (`>=1.26.0` / `^1.26.0` / `^1.27.0`) सध्याच्या `1.29.0` शी स्वच्छ रित्या सुसंगत आणि कोणतेही API तोडफोड न करता ठरवले

- **03-GettingStarted/05-stdio-server/solution/typescript/package.json**: `@modelcontextprotocol/sdk` साठी `^1.8.0` → `>=1.26.0` बंप केला आणि जुना `"updated for MCP 2025-06-18"` पॅकेज वर्णन `"aligned with MCP Specification 2025-11-25"` असं अपडेट केलं

### सुरक्षा कडक करणे आणि अवलंबित्व कमकुवतपणा सुधारणा

प्रत्येक अवलंबित्व घोषणापत्र आणि नमुना स्रोत कोडमध्ये संपूर्ण सुरक्षा तपासणी केली, नंतर सर्व नोंदलेल्या npm सल्ल्यांचे व एका कोड-स्तरीय शोधाचे निराकरण केले. सुधारानंतर, प्रत्येक तपासलेल्या निर्देशिकेत `npm audit` रिपोर्ट्स **0 कमकुवतपणा** दर्शविले.

#### npm अवलंबित्व कमकुवतपणा (परुष्यत) — दुरुस्त

सर्व १५ कमिट केलेल्या `package-lock.json` फाइलचे ऑडिट केले. कमकुवतपणा MCP Inspector डेव्ह टूल, OpenAI क्लायंट, आणि MCP SDK या परुष्यत अवलंबित्वांमध्ये मर्यादित होत्या; सर्व आता नमुन्यांना तोडफोड न करता निराकरण केले:
- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/inspector** व **lab3/code/weather_mcp/inspector**: `@modelcontextprotocol/inspector` (`0.16.6` / `0.14.1` → `0.22.0`) बंप केला, ज्यामुळे `ajv`, `brace-expansion`, `diff`, `path-to-regexp` आणि `ws` सल्ले दूर झाले. npm `overrides` नोंद ऐड केली जी पटलेल्या `shell-quote@1.8.4` ला जबरदस्ती वापरते, त्यामुळे `concurrently` कडून बाकी असलेला महत्त्वाचा सल्ला समाप्त झाला; दोन्ही लॉकफाइल्स पुनर्निर्मित केल्या (आता 0 कमकुवतपणा)
- **03-GettingStarted/samples/typescript**: `npm audit fix` ने माध्यमिक `qs` (मध्यम) चे पटलेले प्रकाशन अपडेट केले
- **03-GettingStarted/samples/javascript**: `npm audit fix` ने माध्यमिक `hono` (मध्यम) चे पटलेले प्रकाशन अपडेट केले
- **03-GettingStarted/03-llm-client/solution/typescript**: `npm audit fix` ने माध्यमिक `form-data` (उच्च) चे पटलेले प्रकाशन अपडेट केले
- **03-GettingStarted/11-simple-auth/solution/typescript**: हरवलेले `package-lock.json` तयार केले ज्यामुळे प्रकल्प पुन्हा तयार करण्यायोग्य आणि तपासण्यायोग्य

#### कोड-स्तरीय सुरक्षा दुरुस्ती (OWASP A03: Injection)

- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/src/server.py**: `open_in_vscode` टूलमधून `shell=True` काढले. आधीच्या `subprocess.run(["start", "", vscode_path, folder_path], shell=True)` मध्ये फोल्डर पथातील शेल मेटाकरॅक्टर्स `cmd.exe` ने समजले जात होते (कमांड-इंजेक्शन धोका). आता तो न थांबणाऱ्या `Code.exe` ला थेट आणि फोल्डरच्या आर्ग्युमेंटसह लाँच करतो — कोणतेही शेल नसल्याने सुरक्षीत आणि कार्यक्षम

#### Python अवलंबित्व ऑडिट

- प्रत्येक Python आवश्यकता संचात `pip-audit` ने ऑडिट केले. `05-AdvancedTopics` आणि `03-GettingStarted/samples/python` **कोणतेही ज्ञात कमकुवतपणा आढळले नाही** (त्यांचे `mcp` / `httpx` / `pydantic` / `python-dotenv` श्रेणी पटलेले प्रकाशने देते)

- **MCP टूल लूप वायरिंग**: Python debate orchestrator ला `AsyncAnthropic` क्लायंट वापरण्यासाठी अपडेट केले (ब्लॉकिंग सिंक `Anthropic` च्या ऐवजी), प्रत्येक एजंट टर्नला थेट लाईव्ह `ClientSession` पास केला, प्रत्येक टर्नला `session.list_tools()` द्वारे टूल डिफिनिशन्स मिळवले, आणि `session.call_tool()` द्वारे `tool_use` ब्लॉक्स डिस्पॅच केले लूपमध्ये जोपर्यंत मॉडेल अंतिम टेक्स्ट प्रतिसाद देत नाही

#### कोर संकल्पना सुधारणा (01-CoreConcepts/)
- **नवीन क्लायंट प्रिमिटिव्ह - Roots**: Roots क्लायंट प्रिमिटिव्हसाठी व्यापक दस्तऐवज जोडला, ज्यामुळे सर्व्हर्सना फाइलसिस्टम सीमारेषा आणि प्रवेश परवानग्या समजतील
- **टूल अनोटेशन्स**: टूल वर्तनात्मक अनोटेशन्स (`readOnlyHint`, `destructiveHint`) वर दस्तऐवज जोडलं, ज्यामुळे टूल कार्यान्वयनासाठी योग्य निर्णयांना मदत होईल
- **सॅम्पलिंगमध्ये टूल कॉलिंग**: सॅम्पलिंग दस्तऐवजात `tools` आणि `toolChoice` पॅरामीटर्स टाकले, मॉडेल-चालित टूल कॉलिंगसाठी
- **URL मोड एलिसिटेशन**: सर्व्हर-प्रेरित बाह्य वेब संवादासाठी URL-आधारित एलिसिटेशनचे दस्तऐवज जोडले
- **टास्क्स (प्रायोगिक)**: टिकाऊ एक्झिक्युशन व्रॅपर्स आणि स्थगित निकाल मिळविण्यासाठी प्रायोगिक टास्क वैशिष्ट्याचे नवीन प्रकरण
- **आयकॉन्स समर्थन**: आता टूल्स, रिसोर्सेस, रिसोर्स टेम्प्लेट्स आणि प्रॉम्प्ट्समध्ये अतिरिक्त मेटाडेटा म्हणून आयकॉन्स असू शकतात याची नोंद

  - सर्व्हर्समध्ये स्थानिक व दूरस्थ नियोजन परिस्थिती सुधारित
- **प्राथमिक पुनररचना**: सर्व्हर आणि क्लायंट प्राथमिक घटकांची पूर्णपणे सुधारणा
  - सर्व्हर प्राथमिक घटक: संसाधने (डेटा स्त्रोत), प्रॉम्प्ट्स (टेम्पलेट), साधने (कार्यकारी फंक्शन्स) सविस्तर स्पष्टीकरणांसह आणि उदाहरणांसह
  - क्लायंट प्राथमिक घटक: नमुना घेणे (LLM पूर्णता), माहिती गोळा करणे (वापरकर्ता इनपुट), लॉगिंग (डिबगिंग/मॉनिटरिंग)
  - सध्याच्या शोध (`*/list`), पुनर्प्राप्ती (`*/get`), आणि अंमलबजावणी (`*/call`) पद्धतींचा समावेश
- **प्रोटोकॉल आर्किटेक्चर**: दोन-स्तरीय आर्किटेक्चर मॉडेल सादर
  - डेटा स्तर: JSON-RPC 2.0 पाया जीवनचक्र व्यवस्थापन आणि प्राथमिक घटकांसह
  - ट्रान्सपोर्ट स्तर: STDIO (स्थानिक) आणि Streamable HTTP SSE (दूरस्थ) ट्रान्सपोर्ट यंत्रणा
- **सुरक्षा फ्रेमवर्क**: स्पष्ट वापरकर्ता संमती, डेटा गोपनीयता संरक्षण, साधन अंमलबजावणी सुरक्षा, आणि ट्रान्सपोर्ट स्तर सुरक्षा यांचा समृद्ध समावेश
- **संवाद नमुने**: प्रोटोकॉल संदेशांचे प्रारंभ, शोध, अंमलबजावणी आणि सूचना प्रवाह अद्यतनित
- **कोड उदाहरणे**: सध्याच्या MCP SDK पॅटर्ननुसार बहुभाषिक उदाहरणे (.NET, Java, Python, JavaScript) नूतनीकरण

#### सुरक्षा (02-Security/) - सखोल सुरक्षा पुनर्रचना  
- **मानके जुळणी**: MCP Specification 2025-06-18 सुरक्षा आवश्यकता पूर्णपणे जुळवली
- **प्रमाणीकरण विकास**: कस्टम OAuth सर्व्हर पासून बाह्य ओळख प्रदाता प्रतिनिधित्व (Microsoft Entra ID) पर्यंतचा विकास दस्तऐवजीकृत
- **AI-विशिष्ट धोका विश्लेषण**: आधुनिक AI हल्ल्यांच्या मार्गदर्शनातील सुधारणा 
  - सविस्तर प्रॉम्प्ट इंजेक्शन हल्ल्यांचे वास्तव जीवनातील उदाहरणांसह वर्णन
  - साधन विषबाधित करणारे यंत्रणा आणि "रग पुल" हल्ले
  - संदर्भ विंडो विषबाधित करणे आणि मॉडेल गोंधळ हल्ले
- **Microsoft AI सुरक्षा उपाय**: Microsoft सुरक्षा इकोसिस्टमचा सखोल समावेश
  - AI प्रॉम्प्ट शील्ड्सच्या उन्नत शोध, स्पॉटलाइट आणि डेलिमीटर तंत्रांसह
  - Azure Content Safety एकात्मिक नमुने
  - पुरवठा साखळी संरक्षणासाठी GitHub Advanced Security
- **उन्नत धोका प्रतिबंध**: सत्र हायजॅकिंगसाठी सुरक्षा नियंत्रणांसह
  - MCP-विशिष्ट हल्ला परिस्थिती आणि क्रिप्टोग्राफिक सत्र आयडी आवश्यकता
  - MCP प्रॉक्सीतील गोंधळलेले प्रतिनिधी समस्या तसेच स्पष्ट संमती आवश्यकता
  - टोकन पासथ्रू दुर्बलता व वैधता नियंत्रण
- **पुरवठा साखळी सुरक्षा**: आधारभूत मॉडेल्स, एम्बेडिंग सेवा, संदर्भ प्रदाता आणि तृतीय-पक्ष API सहित AI पुरवठा साखळी वाढवलेली आढावा
- **मूल सुरक्षा**: एंटरप्राइझ सुरक्षा पॅटर्नसह जीरो ट्रस्ट आर्किटेक्चर आणि Microsoft सुरक्षा इकोसिस्टम एकत्रिकरण सुधारित
- **संसाधन संघटन**: प्रकारानुसार (अधिकृत दस्तऐवज, मानके, संशोधन, Microsoft उपाय, अंमलबजावणी मार्गदर्शक) संसाधन दुवा व्यवस्थापन

### दस्तऐवज गुणवत्ता सुधारणा
- **संरचित शिक्षण उद्दिष्टे**: विशिष्ट आणि क्रियाशील परिणामांसह शिक्षण उद्दिष्टे सुधारित
- **आंतर-संदर्भ**: संबंधित सुरक्षा आणि मुख्य संकल्पना विषयांमध्ये दुवे जोडले
- **सध्याची माहिती**: सर्व दिनांक संदर्भ आणि निर्दिष्टीकरण दुवे सध्याच्या मानकांनुसार अद्यतनित
- **अंमलबजावणी मार्गदर्शन**: दोन्ही विभागांमध्ये विशिष्ट, कार्यक्षम अंमलबजावणी मार्गदर्शक जोडले

### README आणि नॅव्हिगेशन सुधारणा

- हा व्यापक बदलनोंदी देखील जोडली
