सध्याच्या **MCP Specification 2025-11-25** आणि नवीनतम अधिकृत SDKs विरुद्ध अभ्यासक्रमाची पडताळणी केली, नंतर उरलेले जुने तपशील संदर्भ दुरुस्त केले आणि मुख्य नमुने अजूनही तयार होतात आणि चालतात याची खात्री केली.

- **05-AdvancedTopics/mcp-security/README.md**: "चालू मानक" बॅनर, प्रस्तावना, मुख्य सुरक्षा सिद्धांत शीर्षक, अनिवार्य आवश्यकतांचा हेडिंग, Microsoft Entra ID विभाग, संदर्भ आणि संसाधने दुवे, आणि बंदी सुरक्षा सूचना (८ संदर्भ) 2025-11-25 ला अद्यतनित केले

- **03-GettingStarted/14-sampling/README.md**: अधिकृत सॅम्पलिंग दस्तऐवज लिंक 2025-11-25 ला अद्यतनित केले

### सुरक्षा कडक करणे आणि अवलंबित्व असुरक्षा दुरुस्ती

प्रत्येक अवलंबित्व मॅनिफेस्ट आणि नमुना स्रोत कोडवर संपूर्ण सुरक्षा तपासणी केली, नंतर सर्व नोंदलेल्या npm सल्ला सूचना आणि कोड स्तरावर एक भेटलेली समस्या दुरुस्त केली. दुरुस्ती नंतर, `npm audit` प्रत्येक तपासलेल्या डायरेक्टरीत **0 असुरक्षा** सांगते.

#### npm अवलंबित्व असुरक्षा (ट्रान्झिटिव्ह) — दुरुस्त केले

सर्व १५ परवानगी मिळालेल्या `package-lock.json` फाइल्स तपासल्या. असुरक्षा फक्त MCP Inspector डेव्हलपर टूल, OpenAI क्लायंट, आणि MCP SDK कडून बाह्य अवलंबित्वांमध्ये होत्या; सर्व आता नमुन्यांना तडा न पोहचवता सोडवल्या गेल्या:
- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/inspector** आणि **lab3/code/weather_mcp/inspector**: `@modelcontextprotocol/inspector` (`0.16.6` / `0.14.1` → `0.22.0`) वाढवले, ज्याने `ajv`, `brace-expansion`, `diff`, `path-to-regexp` आणि `ws` सल्ला सूचनांस मोकळे केले. npm `overrides` मध्ये `shell-quote@1.8.4` नवीन पॅच लावून, `concurrently` कडून उरलेली गंभीर सल्ला सूचना हटवण्यात आली; दोन्ही लॉकफाइल्स पुनर्जन्मित केल्या (आता 0 असुरक्षा)
- **03-GettingStarted/samples/typescript**: `npm audit fix` ने ट्रान्झिटिव्ह `qs` (मध्यम) पॅच आवृत्तीवर अद्यतनित केले
- **03-GettingStarted/samples/javascript**: `npm audit fix` ने ट्रान्झिटिव्ह `hono` (मध्यम) पॅच आवृत्तीवर अद्यतनित केले
- **03-GettingStarted/03-llm-client/solution/typescript**: `npm audit fix` ने ट्रान्झिटिव्ह `form-data` (उच्च) पॅच आवृत्तीवर अद्यतनित केले
- **03-GettingStarted/11-simple-auth/solution/typescript**: गहाळ `package-lock.json` फायली तयार केल्या, ज्यामुळे प्रकल्प पुनरुत्पादनयोग्य आणि तपासणीयोग्य झाला (0 असुरक्षा)

#### कोड स्तर सुरक्षा दुरुस्ती (OWASP A03: Injection)

- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/src/server.py**: `open_in_vscode` टूलमधून `shell=True` काढले. आधीचा `subprocess.run(["start", "", vscode_path, folder_path], shell=True)` फोल्डर पाथमध्ये शेल मेटा-कॅरॅक्टर्स `cmd.exe` कडून समजून घेणे परवानगी देत होता (कमांड इंजेक्शन धोका). आता थेट `Code.exe` फोल्डर आर्ग्युमेंटसह सुरू होते — शेलशिवाय — जे कार्यक्षमतेने बरोबरीचे व सुरक्षित आहे

#### कोर संकल्पना सुधारणा (01-CoreConcepts/)
- **नवीन क्लायंट प्रिमिटिव - Roots**: Roots क्लायंट प्रिमिटिवचे सर्वसमावेशक दस्तऐवजीकरण जोडले, ज्यामुळे सर्व्हर्सना फाइलसिस्टम सीमा आणि प्रवेश परवानग्या समजतात
- **टूल अ‍ॅनोटेशन्स**: टूल वर्तनशील अ‍ॅनोटेशन्स (`readOnlyHint`, `destructiveHint`) बद्दल दस्तऐवज जोडले जे टूल अंमलबजावणी निर्णय सुधारतात
- **सॅम्पलिंगमध्ये टूल कॉलिंग**: सॅम्पलिंग दस्तऐवज सुधारित करून `tools` आणि `toolChoice` पॅरामीटर्स समाविष्ट केले, जे मॉडेल-चालित टूल विनंत्यांसाठी वापरले जातील
- **URL मोड elicitation**: सर्व्हर-प्रेरित बाह्य वेब संवादांसाठी URL-आधारित elicitation दस्तऐवज जोडले
- **टास्क्स (प्रायोगिक)**: टिकाऊ अंमलबजावणी रॅपर्स आणि नंतर निकाल पुनर्प्राप्तीसाठी प्रयोगशाळेतील टास्क वैशिष्ट्यासाठी नवीन विभाग जोडला
- **आयकॉन्स समर्थन**: आता टूल्स, संसाधने, संसाधन टेम्प्लेट आणि प्रॉम्प्ट्समध्ये अतिरिक्त मेटाडेटा म्हणून आयकॉन्स समाविष्ट करता येतात

### प्रारंभ सेक्शन विस्तार – प्रगत सर्व्हर वापर व सोपा प्रमाणीकरण

#### प्रगत सर्व्हर वापर (03-GettingStarted/10-advanced)
- **नवीन प्रकरण जोडले**: नियमित आणि कमी-स्तरीय सर्व्हर आर्किटेक्चर दोन्ही कव्हर करणारा प्रगत MCP सर्व्हर वापरासाठी सर्वसमावेशक मार्गदर्शक सादर
  - **नियमित vs. लो-लेव्हल सर्व्हर**: दोन्ही दृष्टिकोनांसाठी Python आणि TypeScript मध्ये सविस्तर तुलना आणि कोड उदाहरणे.
  - **हँडलर-आधारित डिझाइन**: स्केलेबल, लवचिक सर्व्हर अंमलबजावणीसाठी हँडलर-आधारित टूल/स्रोत/प्रॉम्प्ट व्यवस्थापनाचे स्पष्टीकरण.
  - **व्यवहार्य पॅटर्न्स**: प्रगत वैशिष्ट्ये आणि आर्किटेक्चरसाठी लो-लेव्हल सर्व्हर पॅटर्न्स उपयुक्त असलेल्या प्रत्यक्ष जगातील परिस्थिती.

#### सोपे प्रमाणीकरण (03-GettingStarted/11-simple-auth)
- **नवीन प्रकरण जोडले**: MCP सर्व्हरमध्ये सोपे प्रमाणीकरण कसे राबवायचे यासाठी स्टेप-बाय-स्टेप मार्गदर्शक.
  - **ऑथ संकल्पना**: प्रमाणीकरण विरुद्ध प्राधिकरण आणि क्रेडेन्शियल हाताळणीचे स्पष्ट स्पष्टीकरण.
  - **मूलभूत ऑथ अंमलबजावणी**: Python (Starlette) आणि TypeScript (Express) मधील मिडलबेअर-आधारित प्रमाणीकरण पॅटर्न्स, कोड नमुन्यांसह.
  - **प्रगत सुरक्षा कडे प्रगती**: सोप्या ऑथपासून सुरू करणे आणि OAuth 2.1 आणि RBAC पर्यंत प्रगती करण्यामध्ये मार्गदर्शन, प्रगत सुरक्षा मॉड्यूल्स संदर्भांसह.

या वाढीमुळे MCP सर्व्हर अंमलबजावणी अधिक मजबूतीने, सुरक्षितपणे आणि लवचिकपणे तयार करण्यासाठी व्यावहारिक, हाताळणी मार्गदर्शन मिळते, ज्यामुळे मूलभूत संकल्पना आणि प्रगत उत्पादन पॅटर्न्स यामध्ये सेतू तयार होते.

- **प्रिमिटिव्ह्स पुनर्रचना**: सर्व्हर आणि क्लायंट प्रिमिटिव्ह्सचे संपूर्ण पुनर्गठन
  - सर्व्हर प्रिमिटिव्ह्स: रिसोर्सेस (डेटा स्रोत), प्रॉम्प्ट्स (टेम्पलेट्स), टूल्स (कार्यकारी फंक्शन्स) सविस्तर स्पष्टीकरण आणि उदाहरणांसह
  - क्लायंट प्रिमिटिव्ह्स: सॅम्पलिंग (LLM पूर्णता), एलिसिटेशन (वापरकर्ता इनपुट), लॉगिंग (डिबगिंग/मॉनिटरिंग)
  - सध्याच्या शोध (`*/list`), प्राप्ती (`*/get`), आणि कार्यान्वयन (`*/call`) पद्धतींच्या नमुन्यांसह अद्ययावत
- **प्रोटोकॉल आर्किटेक्चर**: दोन स्तरांची आर्किटेक्चर मॉडेल सादर केली
  - डेटा स्तर: JSON-RPC 2.0 पाया, जीवनचक्र व्यवस्थापन आणि प्रिमिटिव्ह्ससह
  - ट्रान्सपोर्ट स्तर: STDIO (स्थानिक) आणि Streamable HTTP सह SSE (रिमोट) ट्रान्सपोर्ट यंत्रणा
- **सुरक्षा फ्रेमवर्क**: स्पष्ट वापरकर्ता संमती, डेटा गोपनीयता संरक्षण, टूल कार्यान्वयन सुरक्षा, आणि ट्रान्सपोर्ट लेयर सुरक्षा यांसह व्यापक सुरक्षा तत्त्वे
- **संवाद नमुने**: प्रोटोकॉल संदेशांमध्ये प्रारंभ, शोध, कार्यान्वयन व सूचना प्रवाह दर्शविले
- **कोड उदाहरणे**: बहुभाषिक (.NET, Java, Python, JavaScript) उदाहरणे सध्याच्या MCP SDK पॅटर्न्सनुसार पुनरुद्धृत

#### सुरक्षा (02-Security/) - सर्वसमावेशक सुरक्षा पुनर्रचना  
- **मानके सुसंगतता**: MCP स्पेसिफिकेशन 2025-06-18 सुरक्षा गरजांशी पूर्ण सुसंगतता
- **प्रमाणीकरण विकास**: कस्टम OAuth सर्व्हर्सकडून बाह्य ओळख प्रदाता प्रतिनिधीत्व (Microsoft Entra ID) या विकासाचा दस्तऐवज
- **AI-विशिष्ट धमकी विश्लेषण**: आधुनिक AI हल्ल्यांवरील वाढलेली माहिती
  - वास्तविक उदाहरणांसह सविस्तर प्रॉम्प्ट इंजेक्शन हल्ल्यांचे वर्णन
  - टूल विषबाधा यंत्रणा आणि "रग पुल" हल्ले नमुने
  - संदर्भ विंडो विषबाधा आणि मॉडेल गोंधळ हल्ले
- **Microsoft AI सुरक्षा उपाययोजना**: Microsoft सुरक्षा परिसंस्थेचा सर्वसमावेशक आढावा
  - AI प्रॉम्प्ट शिल्ड्स, प्रगत शोध, स्पॉटलाईटिंग, आणि डेलिमिटर तंत्रे
  - Azure Content Safety समाकलन नमुने
  - GitHub Advanced Security पुरवठा साखळी सुरक्षेसाठी
- **प्रगत धमकी निवारण**: 
  - MCP-विशिष्ट सत्र हायजॅकिंग हल्ल्यांसाठी सत्र ओळख क्रमांकांच्या क्रिप्टोग्राफिक गरजा
  - MCP प्रॉक्सीच्या संदर्भात गोंधळलेले डिप्युटी समस्या स्पष्ट संमती आवश्यकतांसह
  - टोकन पासथ्रू धोके आणि अनिवार्य प्रमाणीकरण नियंत्रण
- **पुरवठा साखळी सुरक्षा**: कॉन्टेक्स्ट प्रदाते, फाउंडेशन मॉडेल्स, एमबेडिंग सेवा आणि तृतीय-पक्ष API समाविष्ट करीत AI पुरवठा साखळीचे विस्तारित कव्हरेज
- **मुळ सुरक्षा**: शून्य ट्रस्ट आर्किटेक्चर आणि Microsoft सुरक्षा परिसंस्थेसह एंटरप्राइझ सुरक्षा पॅटर्नमध्ये सुधारित समाकलन
- **स्रोत संघटन**: प्रकारानुसार (अधिकृत दस्तऐवज, मानके, संशोधन, Microsoft उपाययोजना, अंमलबजावणी मार्गदर्शक) संपूर्ण स्त्रोत दुव्यांचे वर्गीकरण

### दस्तऐवज गुणवत्ता सुधारणा

- **क्रॉस-रेफरन्सेस**: संबंधित सुरक्षा व मुख्य संकल्पना विषयांमध्ये दुवे जोडले
- **सध्याची माहिती**: सर्व तारखांचे संदर्भ आणि स्पेसिफिकेशन दुवे सध्याच्या मानकांनुसार अद्ययावत
- **अंमलबजावणी मार्गदर्शन**: दोन्ही विभागांमध्ये विशिष्ट, कृतीक्षम अंमलबजावणी मार्गदर्शक जोडले

## जुलै १६, २०२५

### README आणि नेव्हिगेशन सुधारणा
- README.md मधील अभ्यासक्रम नेव्हिगेशन पूर्णपणे पुनर्निर्मित केले
- `<details>` टॅग्जच्या जागी अधिक प्रवेशयोग्य टेबल-आधारित स्वरूप वापरले
- नवीन "alternative_layouts" फोल्डरमध्ये पर्यायी लेआउट पर्याय तयार केले
- कार्ड-आधारित, टॅब्ड-शैली, आणि अॅकॉर्डियन-शैली नेव्हिगेशन उदाहरणे जोडली
- संचिका रचना विभाग अद्ययावत करून सर्व नवीनतम फाइल्स समाविष्ट केले
- "या अभ्यासक्रमाचा कसा वापर करावा" विभाग स्पष्ट शिफारसींसह सुधारित
- MCP स्पेसिफिकेशन दुवे योग्य URL वर अद्ययावत
- अभ्यासक्रम रचनेत Context Engineering विभाग (5.14) जोडले

### अभ्यास मार्गदर्शक अद्ययावत
- सध्याच्या संचिका रचनेशी अनुरूप अभ्यास मार्गदर्शक पूर्णपणे पुनर्रचित
- MCP क्लायंट आणि टूल्ससाठी नवीन विभाग, तसेच लोकप्रिय MCP सर्व्हर्सचे परिचय दिले
- व्हिज्युअल अभ्यासक्रम नकाशा सर्व विषयांशी अचूक जुळवले
- प्रगत विषयांचे वर्णन वाढवून सर्व विशेष क्षेत्रांना व्यापले
- केस स्टडीज विभाग प्रत्यक्ष उदाहरणांसह अद्ययावत
- हा सर्वसमावेशक बदल इतिहास जोडला

### समुदाय योगदान (06-CommunityContributions/)
- प्रतिमा निर्मितीसाठी MCP सर्व्हर्स विषयी सविस्तर माहिती जोडली

- योगदानासाठी अधिक अचूक कोड नमुने सुधारित
