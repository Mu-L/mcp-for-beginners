# 🚀 10 मायक्रोसॉफ्ट MCP सर्व्हर जे विकसकांच्या उत्पादकतेत क्रांती घडवत आहेत

## 🎯 या मार्गदर्शनात तुम्हाला काय शिकायला मिळेल

हे व्यावहारिक मार्गदर्शन दहा मायक्रोसॉफ्ट MCP सर्व्हर्सची ओळख करून देते जे सक्रियपणे विकसकांच्या AI सहाय्यकांसोबत काम करण्याच्या पद्धतीत बदल घडवत आहेत. MCP सर्व्हर्स काय करू शकतात यावर फक्त स्पष्टीकरण देण्याऐवजी, आम्ही असे सर्व्हर दाखवणार आहोत जे मायक्रोसॉफ्ट आणि त्यापलीकडील दैनंदिन विकास कार्यप्रवाहात खऱ्या अर्थाने फरक आणत आहेत.

या मार्गदर्शनातील प्रत्येक सर्व्हर प्रत्यक्ष वापर आणि विकसकांच्या अभिप्रायावर आधारित निवडण्यात आले आहेत. तुम्हाला फक्त प्रत्येक सर्व्हर काय करतो ते नाही तर तो का महत्वाचा आहे आणि तुमच्या स्वतःच्या प्रकल्पांमध्ये त्याचा कसा सर्वोत्तम वापर करायचा हे देखील समजेल. तुम्ही MCP मध्ये पूर्णपणे नवीन असाल किंवा तुमचे विद्यमान सेटअप वाढवू इच्छित असाल, हे सर्व्हर्स मायक्रोसॉफ्ट परिसंस्थेतील सर्वात व्यावहारिक आणि प्रभावी साधने दर्शवतात.

> **💡 जलद सुरुवातीचा टीप**
>
> MCP मध्ये नवीन आहात? काळजी करू नका! हे मार्गदर्शन नवशिक्यांसाठी सोप्या भाषेत तयार केले आहे. आम्ही संकल्पना समजावून सांगू आणि तुम्ही कधीही आमच्या [MCP परिचय](../00-Introduction/README.md) आणि [कोर संकल्पना](../01-CoreConcepts/README.md) मॉड्यूलकडे अधिक सखोल पार्श्वभूमीसाठी परत जाऊ शकता.

## अवलोकन

हे सर्वसमावेशक मार्गदर्शन दहा मायक्रोसॉफ्ट MCP सर्व्हर्सचे शोध घेत आहे जे विकसक AI सहाय्यक आणि बाह्य साधनांसोबत संवाद साधण्यासाठी कसे बदल घडवत आहेत हे दाखवते. Azure संसाधन व्यवस्थापनापासून दस्तऐवज प्रक्रिया पर्यंत, हे सर्व्हर्स मॉडेल संदर्भ प्रोटोकॉलचा वापर करून सुरळीत आणि उत्पादक विकास कार्यप्रवाह कसे तयार केले जातात हे दर्शवतात.

## शिकण्याच्या उद्दिष्टे

- मायक्रोसॉफ्टच्या सर्वात प्रभावी MCP सर्व्हर अंमलबजावण्या जाणून घेणे
- प्रत्येक सर्व्हसाठी व्यवहार्य वापर प्रकरणे शोधणे
- VS Code आणि Visual Studio मध्ये हे सर्व्हर्स कसे सेटअप व कॉन्फिगर करायचे ते शिकणे
- विस्तृत MCP परिसंस्था आणि भविष्यातील दिशा ओळखणे

## 🔧 MCP सर्व्हर्स समजून घेणे: नवशिक्यांसाठी मार्गदर्शक

### MCP सर्व्हर्स म्हणजे काय?

मॉडेल संदर्भ प्रोटोकॉल (MCP) मध्ये नवशिक्या म्हणून, तुम्हाला वाटू शकते: "MCP सर्व्हर म्हणजे नेमके काय, आणि मला त्याची गरज का आहे?" तर एक साधे उदाहरण घेऊ या.

MCP सर्व्हर्सला असा समजा की ते विशेष सहाय्यक आहेत जे तुमच्या AI कोडिंग साथीदाराला (जसे GitHub Copilot सारखे) बाह्य साधने आणि सेवा जोडण्यास मदत करतात. जसे तुम्ही तुमच्या फोनवर विविध कामांसाठी वेगवेगळ्या अ‍ॅप्स वापरता—एक हवामानासाठी, एक नॅव्हिगेशनसाठी, एक बँकिंगसाठी—असेच MCP सर्व्हर्स तुमच्या AI सहाय्यकाला वेगवेगळ्या विकास साधनांशी आणि सेवांशी संवाद साधण्याची क्षमता देतात.

### MCP सर्व्हर्स कोणता प्रश्न सोडवतात

MCP सर्व्हर्सपूर्वी, जर तुम्हाला हे करायचे असेल:
- तुमचे Azure संसाधने तपासणे
- एक GitHub समस्या तयार करणे
- तुमचा डेटाबेस क्वेरी करणे
- दस्तऐवज शोधणे

तर तुम्हाला कोडिंग थांबवावे लागे, ब्राऊझर उघडावा लागे, योग्य वेबसाइटवर जावे लागे आणि हे काम हाताने करावे लागे. हा सतत संदर्भ बदल तुमचा प्रवाह मोडतो आणि उत्पादकता कमी करतो.

### MCP सर्व्हर्स तुमचा विकास अनुभव कसा बदलतात

MCP सर्व्हर्ससह, तुम्ही तुमच्या विकास पर्यावरणातच (VS Code, Visual Studio इत्यादी) राहू शकता आणि फक्त तुमच्या AI सहाय्यकाला या कामांशी सांगा. उदाहरणार्थ:

**यापूर्वी असा पारंपरिक कार्यप्रवाह होता:**
1. कोडिंग थांबवा  
2. ब्राउझर उघडा  
3. Azure पोर्टलवर जा  
4. स्टोरेज खात्याची माहिती पाहा  
5. VS Code मध्ये परत या  
6. कोडिंग सुरू ठेवा  

**आता तुम्ही असे करू शकता:**
1. AI ला विचारा: "माझ्या Azure स्टोरेज खात्यांची स्थिती काय आहे?"
2. मिळालेल्या माहितीने कोडिंग सुरू ठेवा

### नवशिक्यांसाठी महत्वाचे फायदे

#### 1. 🔄 **तुमच्या प्रवाह स्थितीत राहा**
- अनेक अ‍ॅप्समध्ये बदल करण्याची गरज नाही
- तुम्ही लिहित असलेल्या कोडवर लक्ष केंद्रित ठेवा
- वेगवेगळ्या साधनांचा व्यवस्थापन करताना मनाची थकवा कमी करा

#### 2. 🤖 **संकูล आदेशांच्या ऐवजी नैसर्गिक भाषा वापरा**
- SQL सिंटॅक्स आठवण्याच्या ऐवजी, तुम्हाला हवे असलेले डेटा वर्णन करा
- Azure CLI आदेश आठवण्यापेक्षा काय साध्य करायचे ते स्पष्ट करा
- AI तांत्रिक तपशील सांभाळू द्या, तुम्ही लॉजिकवर लक्ष द्या

#### 3. 🔗 **अनेक साधने एकत्र जोडा**
- वेगवेगळ्या सेवांचा एकत्रित वापर करून शक्तिशाली कार्यप्रवाह तयार करा
- उदाहरण: "सर्व अलीकडील GitHub समस्या मिळवा आणि त्यानुसार Azure DevOps कार्य गोष्टी तयार करा"
- जटिल स्क्रिप्ट न लिहिता ऑटोमेशन तयार करा

#### 4. 🌐 **वाढती परिसंस्था वापरा**
- मायक्रोसॉफ्ट, GitHub आणि इतर कंपन्यांनी तयार केलेल्या सर्व्हर्सचा फायदा घ्या
- विविध विक्रेत्यांच्या साधनांचा मिश्रण सहज करा
- एक मानकीकृत परिसंस्थेत सामील व्हा जी वेगवेगळ्या AI सहाय्यकांसोबत कार्य करते

#### 5. 🛠️ **प्रात्यक्षिकातून शिका**
- सुरुवातीला तयार केलेल्या सर्व्हर्ससह संकल्पना समजून घ्या
- जसजसा तुम्हाला आत्मविश्वास येतो तसतसे स्वतःचे सर्व्हर्स तयार करा
- उपलब्ध SDK आणि दस्तऐवज वापरून तुमचे शिक्षण प्रगती करा

### नवशिक्यांसाठी प्रत्यक्ष उदाहरण

धरा तुम्ही वेब विकासात नवीन आहात आणि तुमचा पहिला प्रकल्प करत आहात. MCP सर्व्हर्स कसे मदत करू शकतात ते अशी:

**पारंपरिक पद्धत:**
```
1. Code a feature
2. Open browser → Navigate to GitHub
3. Create an issue for testing
4. Open another tab → Check Azure docs for deployment
5. Open third tab → Look up database connection examples
6. Return to VS Code
7. Try to remember what you were doing
```
  
**MCP सर्व्हर्ससह:**
```
1. Code a feature
2. Ask AI: "Create a GitHub issue for testing this login feature"
3. Ask AI: "How do I deploy this to Azure according to the docs?"
4. Ask AI: "Show me the best way to connect to my database"
5. Continue coding with all the information you need
```
  
### एंटरप्राइझ मानकाचा फायदा

MCP उद्योगभरात प्रमाणित मानक बनत आहे, म्हणजे:  
- **सुसंगतता**: वेगवेगळ्या साधनांमध्ये आणि कंपन्यांमध्ये समान अनुभव  
- **परस्परसंवाद**: वेगवेगळ्या विक्रेत्यांच्या सर्व्हर्स एकत्र काम करतात  
- **भविष्यसंधी**: कौशल्ये आणि सेटअप विविध AI सहाय्यकांमध्ये हस्तांतरित होतात  
- **समुदाय**: मोठ्या परिसंस्थेमध्ये ज्ञान आणि संसाधनांचा सामायिक उपयोग

## VS Code आणि Visual Studio मध्ये MCP सह सुरुवात 🚀

जर तुम्ही Visual Studio Code किंवा Visual Studio 2022 GitHub Copilot सह वापरत असाल, तर हे MCP सर्व्हर्स सेट अप करणे सोपे आहे.

### VS Code सेटअप

VS Code साठी मूलभूत प्रक्रिया खालीलप्रमाणे:

1. **एजंट मोड सक्षम करा**: VS Code मध्ये Copilot Chat विंडोमध्ये एजंट मोडवर स्विच करा  
2. **MCP सर्व्हर कॉन्फिगर करा**: तुमच्या VS Code च्या settings.json फाइलमध्ये सर्व्हर कॉन्फिगरेशन जोडा  
3. **सर्व्हर्स सुरू करा**: तुम्हाला वापरायचे सर्व्हर "Start" बटणावर क्लिक करून सुरु करा  
4. **साधने निवडा**: सध्याच्या सत्रासाठी कोणती MCP सर्व्हर्स सक्षम करायची ते निवडा  

तपशीलवार सेटअप सूचना पाहण्यासाठी [VS Code MCP दस्तऐवज](https://code.visualstudio.com/docs/copilot/copilot-mcp) पहा.

> **💡 प्रो टीप: MCP सर्व्हर व्यावसायिक सारखे व्यवस्थापित करा!**  
>  
> VS Code Extensions नऊ आता [स्थापित MCP सर्व्हर्सच्या व्यवस्थापनासाठी सोपी नवी यूआय](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode) देते! तुम्हाला कोणतेही स्थापित MCP सर्व्हर सुरु, थांबवणे आणि व्यवस्थापित करणे सोपे झाले आहे. वापरून पहा!

### Visual Studio 2022 सेटअप

Visual Studio 2022 (आवृत्ती 17.14 किंवा नंतर) साठी:

1. **एजंट मोड सक्षम करा**: GitHub Copilot Chat विंडोतील "Ask" ड्रॉपडाउनमध्ये क्लिक करा आणि "Agent" निवडा  
2. **कॉन्फिगरेशन फाइल तयार करा**: तुमच्या सोल्यूशन डिरेक्टरीमध्ये `.mcp.json` फाइल तयार करा (शिफारस केलेला मार्ग: `<SOLUTIONDIR>\.mcp.json`)  
3. **सर्व्हर्स कॉन्फिगर करा**: प्रमाणित MCP फॉर्मॅट वापरून तुमचे MCP सर्व्हर कॉन्फिगरेशन जोडा  
4. **साधन मंजुरी द्या**: विचारल्यावर वापरायची साधने योग्य स्कोप परवानग्यांसह मंजूर करा  

Visual Studio सेटअपसाठी अधिक तपशीलवार सूचना पाहण्यासाठी [Visual Studio MCP दस्तऐवज](https://learn.microsoft.com/visualstudio/ide/mcp-servers) पहा.

प्रत्येक MCP सर्व्हरची स्वतःची कॉन्फिगरेशन आवश्यकताएँ (कनेक्शन स्ट्रिंग्ज, प्रमाणीकरण इ.) असल्या तरी, दोन्ही IDE मध्ये सेटअप पॅटर्न सारखेच आहे.

**हे काय करते**: Microsoft Learn Docs MCP Server हा एक क्लाउड-होस्ट केलेला सेवा आहे जो AI सहाय्यकांना Model Context Protocol च्या माध्यमातून अधिकृत मायक्रोसॉफ्ट दस्तऐवजांमध्ये वास्तविक-वेळ प्रवेश प्रदान करतो. हे `https://learn.microsoft.com/api/mcp` शी जोडलेले आहे आणि Microsoft Learn, Azure दस्तऐवज, Microsoft 365 दस्तऐवज आणि इतर अधिकृत मायक्रोसॉफ्ट स्रोतांमध्ये अर्थपूर्ण शोध सक्षम करते.

**हे का उपयुक्त आहे**: हे फक्त "दस्तऐवज" म्हणून दिसले तरीही, हा सर्व्हर प्रत्येक मायक्रोसॉफ्ट तंत्रज्ञान वापरणाऱ्या विकसकांसाठी अत्यंत महत्त्वाचा आहे. .NET विकसकांकडून AI कोडिंग सहाय्यकांच्या बाबतीत मोठ्या प्रमाणात तक्रार असते की ते नवीनतम .NET आणि C# रीलिझबद्दल अद्ययावत नाहीत. Microsoft Learn Docs MCP Server हे समस्यांचे निराकरण करतो, त्यामुळे AI सहाय्यकाला सर्वात अलीकडील दस्तऐवज, API संदर्भ आणि सर्वोत्तम पद्धतींना वास्तविक-वेळ प्रवेश मिळतो. तुम्ही नवीनतम Azure SDKs सह काम करत असाल, C# 13 फिचर्स एक्सप्लोर करत असाल किंवा कटिंग-एज .NET Aspire पॅटर्न्स अंमलात आणत असाल, हा सर्व्हर AI सहाय्यकाला अधिकृत, अद्ययावत माहिती उपलब्ध करून देतो जे अचूक आणि आधुनिक कोड तयार करू शकतो.

**प्रत्यक्ष वापर**: "Microsoft Learn अधिकृत दस्तऐवजांनुसार Azure कंटेनर अ‍ॅप तयार करण्यासाठी az cli कमांड्स काय आहेत?" किंवा "ASP.NET Core मध्ये dependency injection सह Entity Framework कसे कॉन्फिगर करायचे?" किंवा "हाच कोड तपासा की तो Microsoft Learn Documentation मध्ये दिलेल्या कार्यक्षमता सूचना जुळतो का." हा सर्व्हर Microsoft Learn, Azure दस्तऐवज आणि Microsoft 365 दस्तऐवजांमध्ये प्रगत अर्थपूर्ण शोध वापरून सर्वसमावेशक माहिती पुरवतो. तो 10 उच्च-गुणवत्तेच्या लेख सामग्री संचाचा प्रतिसाद देतो, ज्यात लेख शीर्षके आणि URL असतात, आणि नेहमी नवीनतम अधिकृत दस्तऐवज उपलब्ध असताना तो वापरतो.

**विशिष्ट उदाहरण**: हा सर्व्हर `microsoft_docs_search` साधन प्रदान करतो जे मायक्रोसॉफ्टच्या अधिकृत तांत्रिक दस्तऐवजांवर अर्थपूर्ण शोध करतो. एकदा कॉन्फिगर केल्यावर, तुम्ही विचारू शकता "ASP.NET Core मध्ये JWT प्रमाणीकरण कसे अंमलात आणायचे?" आणि विस्तृत अधिकृत उत्तरासह स्रोत लिंक मिळवा. शोधाची गुणवत्ता अपवादात्मक आहे कारण तो संदर्भ समजतो – उदा., Azure संदर्भात "containers" बद्दल विचारल्यास Azure Container Instances दस्तऐवज परत करतो, तर .NET संदर्भात त्याच शब्दाने संबंधित C# संग्रह माहिती परत करते.

हे विशेषतः जलद बदलणाऱ्या किंवा अलीकडे अपडेट झालेल्या ग्रंथालये आणि वापर प्रकरणांसाठी उपयुक्त आहे. उदाहरणार्थ, काही अलीकडील कोडिंग प्रकल्पांमध्ये मला नवीनतम .NET Aspire आणि Microsoft.Extensions.AI या रीलिझमधील वैशिष्ट्ये वापरायची होती. Microsoft Learn Docs MCP सर्व्हरचा समावेश केल्यामुळे मला API दस्तऐवजांशिवाय walkthroughs आणि मार्गदर्शनाचा फायदा मिळाला जे नुकतेच प्रकाशित झाले होते.

### 2. ☁️ Azure MCP Server
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

**हे काय करते**: Azure MCP Server हा 15+ विशेष Azure सेवा कनेक्टर्सचा संपूर्ण संच आहे जो तुमच्या AI कार्यप्रवाहात संपूर्ण Azure परिसंस्था आणतो. हा फक्त एक सर्व्हर नाही – हा एक शक्तिशाली संग्रह आहे ज्यात रिसोर्स व्यवस्थापन, डेटाबेस कनेक्टिव्हिटी (PostgreSQL, SQL Server), KQL सह Azure Monitor लॉग विश्लेषण, Cosmos DB एकत्रीकरण, आणि बरेच काही समाविष्ट आहे.

**हे का उपयुक्त आहे**: Azure संसाधने फक्त व्यवस्थापित करण्याबरोबरच, हा सर्व्हर Azure SDK वापरताना कोडची गुणवत्ता नाट्यमयरीत्या सुधारतो. जेव्हा तुम्ही Azure MCP एजंट मोडमध्ये वापरता, तेव्हा तो फक्त कोड लिहिण्यास मदत करत नाही – तो *बेहतर* Azure कोड लिहिण्यास मदत करतो जो सद्य प्रमाणीकरण पद्धती, त्रुटी हाताळण्याच्या सर्वोत्तम पद्धती आणि नवीनतम SDK वैशिष्ट्यांचा वापर करतो. सर्वसाधारण काम करणारा कोड मिळण्याऐवजी, तुम्हाला उत्पादन कार्यांसाठी Azure च्या शिफारस केलेल्या पद्धतींचे पालन करणारा कोड मिळतो.

**महत्वाचे मॉड्यूल्स यामध्ये आहेत**:
- **🗄️ डेटाबेस कनेक्टर्स**: Azure Database for PostgreSQL आणि SQL Server साठी थेट नैसर्गिक भाषा प्रवेश
- **📊 Azure Monitor**: KQL-शक्तियुक्त लॉग विश्लेषण आणि ऑपरेशनल अंतर्दृष्टी
- **🌐 रिसोर्स व्यवस्थापन**: संपूर्ण Azure रिसोर्स जीवनचक्र व्यवस्थापन
- **🔐 प्रमाणीकरण**: DefaultAzureCredential आणि व्यवस्थापित ओळखण्याच्या पद्धती
- **📦 स्टोरेज सेवा**: Blob Storage, Queue Storage, आणि Table Storage ऑपरेशन्स
- **🚀 कंटेनर सेवा**: Azure Container Apps, Container Instances, आणि AKS व्यवस्थापन
- **आणि बरेच विशेष कनेक्टर्स**

**प्रत्यक्ष वापर**: "माझी Azure स्टोरेज अकाउंट्स सूचीबद्ध करा", "माझ्या लॉग अ‍ॅनालिटिक्स कार्यक्षेत्राचा मागील तासातील त्रुटींसाठी विचार करा", किंवा "योग्य प्रमाणीकरणासह Node.js वापरून Azure अ‍ॅप्लिकेशन तयार करण्यात माझी मदत करा"

**पूर्ण डेमो परिस्थिती**: येथे एक पूर्ण वॉकथ्रू आहे जे Azure MCP आणि GitHub Copilot for Azure विस्तार यांचे संयोजन शक्ती दाखवते. जेव्हा तुम्ही दोन्ही स्थापित करता आणि प्रॉम्प्ट देता:

> "DefaultAzureCredential प्रमाणीकरण वापरून Azure Blob Storage मध्ये फाइल अपलोड करणारी एक Python स्क्रिप्ट तयार करा. स्क्रिप्टने 'mycompanystorage' नावाच्या माझ्या Azure स्टोरेज खात्याला कनेक्ट करणे, 'documents' नावाच्या कंटेनरमध्ये अपलोड करणे, सध्याच्या टाइमस्टॅम्पसह चाचणी फाइल तयार करणे, त्रुटी सौम्यपणे हाताळणे आणि माहितीपूर्ण आउटपुट देणे, प्रमाणीकरण आणि त्रुटी हाताळणीसाठी Azure च्या सर्वोत्तम पद्धतींचे पालन करणे, DefaultAzureCredential प्रमाणीकरण कसे कार्य करते हे समजावून सांगणारी टिप्पणी समाविष्ट करणे, आणि योग्य फंक्शन व दस्तऐवजीकरणासह स्क्रिप्ट व्यवस्थितरित्या तयार करणे अपेक्षित आहे."

Azure MCP Server एक संपूर्ण, उत्पादन-तयार Python स्क्रिप्ट निर्माण करेल जी:
- योग्य असिंक्रोनस पॅटर्नसह नवीनतम Azure Blob Storage SDK वापरते
- व्यापक फॉलबॅक साखळी स्पष्टीकरणासह DefaultAzureCredential अमलात आणते
- विशिष्ट Azure अपवाद प्रकारांसह मजबूत त्रुटी हाताळणी समाविष्ट करते
- रिसोर्स व्यवस्थापन आणि कनेक्शन हाताळणीसाठी Azure SDK च्या सर्वोत्तम पद्धतींचे पालन करते
- सविस्तर लॉगिंग आणि माहितीपूर्ण कन्सोल आउटपुट प्रदान करते
- फंक्शन्स, दस्तऐवजीकरण आणि प्रकार संकेतांसह व्यवस्थितरित्या स्क्रिप्ट तयार करते

विशेष म्हणजे Azure MCP शिवाय तुम्हाला कदाचित सामान्य ब्लॉब स्टोरेज कोड मिळेल जो कार्य करतो पण सद्य Azure पॅटर्नचे पालन करत नाही. Azure MCP सह, तुम्हाला नवीनतम प्रमाणीकरण पद्धती वापरणारा, Azure-विशिष्ट त्रुटी परिस्थिती हाताळणारा, आणि Microsoft च्या उत्पादन अनुप्रयोगांसाठी शिफारस केलेल्या पद्धतींचे पालन करणारा कोड मिळतो.

**विशिष्ट उदाहरण**: मला नेहमी `az` आणि `azd` CLI चे विशिष्ट कमांड लक्षात ठेवण्यात अडचण होते. माझ्यासाठी नेहमी दोन टप्प्यांचा प्रक्रिया आहे: प्रथम सिंटॅक्स शोधणे, नंतर कमांड चालवणे. मी अनेकदा फक्त पोर्टलमध्ये जाऊन क्लिक करून काम पूर्ण करतो कारण मला CLI सिंटॅक्स लक्षात नाही राहत हे मान्य करायला आवडत नाही. मला काय हवे ते फक्त वर्णन करू शकणे आश्चर्यकारक आहे, आणि ते माझ्या IDE सोडल्याशिवाय करता येणे अजून चांगले!

शुरू करण्यासाठी [Azure MCP रेपॉझिटरी](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) मध्ये छान वापर केसेसची यादी आहे. सर्वसमावेशक सेटअप मार्गदर्शक आणि प्रगत कॉन्फिगरेशन पर्यायांसाठी, [अधिकृत Azure MCP दस्तऐवज](https://learn.microsoft.com/azure/developer/azure-mcp-server/) पाहा.

**हे काय करते**: अधिकृत GitHub MCP Server GitHub च्या संपूर्ण परिसंस्थेबरोबर अखंड एकात्मता प्रदान करतो, ज्यात होस्टेड रिमोट अॅक्सेस आणि स्थानिक Docker तैनाती पर्याय दोन्ही आहेत. हा फक्त मूलभूत रेपॉझिटरी ऑपरेशन्ससाठी नाही – हा एक सर्वसमावेशक उपकरण संच आहे ज्यात GitHub Actions व्यवस्थापन, पुल रिक्वेस्ट वर्कफ्लोज, इश्यू ट्रॅकिंग, सुरक्षा स्कॅनिंग, सूचना आणि प्रगत ऑटोमेशन क्षमता समाविष्ट आहेत.

**हे का उपयुक्त आहे**: हा सर्व्हर GitHub सोबत तुमचा संवाद कसा साधलाय याचं रूपांतर करतो, पूर्ण प्लॅटफॉर्म अनुभव तुमच्या विकास पर्यावरणात थेट आणतो. VS Code आणि GitHub.com दरम्यान प्रोजेक्ट व्यवस्थापन, कोड पुनरावलोकन, आणि CI/CD देखरेखीकरता वारंवार परत जाण्याऐवजी, तुम्ही नैसर्गिक भाषा आदेशांद्वारे सर्व काही हाताळू शकता आणि तुमच्या कोडवर लक्ष केंद्रित ठेवू शकता.

> **ℹ️ नोंद: 'एजंट्स' चे वेगवेगळे प्रकार**

GitHub MCP Server VS Code च्या एजंट मोडमध्ये काम करतो आणि GitHub API एकात्मता पुरवतो, तर GitHub चा कोडिंग एजंट हा वेगळा फीचर आहे जो GitHub इश्यूजना असाईन केल्यावर पुल रिक्वेस्ट तयार करतो.

- मर्ज केलेल्या PRs मधून माहिती काढून सविस्तर रिलीझ नोट्स तयार करेल

VS Code, GitHub नोटिफिकेशन्स, आणि पुल रिक्वेस्ट पृष्ठांमध्ये उड्या मारण्याऐवजी, मी "माझ्या पुनरावलोकनासाठी सर्व PR दाखवा" असे म्हणू शकतो आणि नंतर "PR #123 वर प्रमाणीकरण पद्धतीतील त्रुटी हाताळणीबद्दल टिप्पणी जोडा." सर्व्हर GitHub API कॉल्स हाताळतो, चर्चेचा संदर्भ राखतो, आणि मला अधिक रचनात्मक पुनरावलोकन टिप्पण्या तयार करण्यात मदत करतो.

**हे का उपयुक्त आहे**: आधुनिक दस्तऐवज कार्यप्रवाहांसाठी अत्यावश्यक! MarkItDown विविध फाईल फॉरमॅट्स हाताळतो आणि मुख्य दस्तऐवज रचना जशी की शीर्षके, यादी, तक्ते आणि दुवे जपतो. साध्या मजकूर काढण्याच्या साधनांपेक्षा वेगळे, हे सांकेतिक अर्थ आणि स्वरूपणा जपण्यावर भर देतो, जे AI प्रक्रिया आणि मानवी वाचनासाठी तसेच उपयुक्त आहे.

**समर्थित फाईल फॉरमॅट्स**:
- **ऑफिस दस्तऐवज**: PDF, PowerPoint (PPTX), Word (DOCX), Excel (XLSX/XLS)
- **मिडिया फाईल्स**: प्रतिमा (EXIF मेटाडेटा आणि OCR सहित), ऑडिओ (EXIF मेटाडेटा आणि स्पीच ट्रान्स्क्रिप्शनसहित)
- **वेब सामग्री**: HTML, RSS फीड्स, YouTube URLs, Wikipedia पृष्ठे
- **डेटा फॉरमॅट्स**: CSV, JSON, XML, ZIP फाईल्स (आतील सामग्री पुनरावृत्ती करून प्रक्रिया करणे)
- **प्रकाशन फॉरमॅट्स**: EPub, Jupyter नोटबुक (.ipynb)
- **ई-मेल**: Outlook संदेश (.msg)
- **प्रगत**: PDF प्रक्रियेसाठी Azure Document Intelligence एकत्रीकरण

**प्रगत क्षमता**: MarkItDown LLM-शक्तीने चालणाऱ्या प्रतिमा वर्णनांना (OpenAI क्लायंट उपलब्ध असल्यास), Azure Document Intelligence द्वारे PDF प्रक्रिया सुधारित करण्यासाठी, स्पीच सामग्रीसाठी ऑडिओ ट्रान्स्क्रिप्शन आणि अतिरिक्त फाईल फॉरमॅट्ससाठी प्लगइन सिस्टमला समर्थन देतो.

**वास्तविक वापर**: "या PowerPoint सादरीकरणाला Markdown मध्ये रूपांतर करा आमच्या दस्तऐवजीकरण साइटसाठी", "या PDF मधून योग्य शीर्षक रचनेसह मजकूर काढा", किंवा "हा Excel स्प्रेडशीट वाचनीय तक्ता स्वरूपात रूपांतरित करा".

**वैशिष्ट्यीकृत उदाहरण**: [MarkItDown दस्तऐवज](https://github.com/microsoft/markitdown#why-markdown) मधून उद्धृत केले:

> Markdown हा साध्या मजकूरास जवळचा आहे, ज्यामध्ये किमान मार्कअप किंवा स्वरूपणा असतो, तरीही महत्त्वपूर्ण दस्तऐवज रचना दर्शविण्याचा मार्ग पुरवतो. मुख्य प्रवाही LLMs, जसे की OpenAI चा GPT-4o, स्वाभाविकरित्या "Markdown" बोलतात, आणि बर्‍याचदा त्याच्या प्रतिसादांमध्ये Markdown सहज समाविष्ट करतात. याचा अर्थ असा की त्यांना प्रचंड प्रमाणात Markdown-स्वरूपित मजकूरावर प्रशिक्षण दिले गेले आहे आणि ते ते उत्तम समजतात. एक पर्यायी फायदेशीर बाब म्हणजे, Markdown च्या पारंपारिक नियमांमुळे ती टिकाऊ टोकन कार्यक्षमही आहे.

MarkItDown दस्तऐवज रचना जपण्यात अत्यंत कुशल आहे, जे AI कार्यप्रवाहांसाठी महत्त्वाचे आहे. उदाहरणार्थ, PowerPoint सादरीकरण रूपांतरित करताना, ते योग्य शीर्षकांसह स्लाइड संघटना जपते, तक्ते Markdown तक्त्यांमध्ये काढते, प्रतिमांसाठी alt मजकूर समाविष्ट करते, आणि उद्घोषक नोट्सही प्रक्रिया करते. चार्ट्स वाचनीय डेटा तक्त्यांमध्ये रूपांतरित होतात, आणि परिणामी Markdown मूळ सादरीकरणाच्या तार्किक प्रवाहाला टिकवून ठेवतो. त्यामुळे हे AI प्रणालींसाठी सादरीकरण सामग्री पुरवण्यासाठी किंवा विद्यमान स्लाइड्समधून दस्तऐवजीकरण तयार करण्यासाठी उत्तम आहे.

**प्रत्यक्ष विकास वापर**: "माझ्या अनुप्रयोगासाठी Phi-4 मॉडेल Azure AI सेवांवर डिप्लॉय करा", "माझ्या दस्तऐवज RAG सिस्टीमसाठी नवीन शोध निर्देशांक तयार करा", "माझ्या एजंटच्या प्रतिसादांचे गुणवत्ता मेट्रिक्सशी मूल्यांकन करा", किंवा "माझ्या गुंतागुंतीच्या विश्लेषण कार्यांसाठी सर्वोत्तम तर्क मॉडेल शोधा"

**पूर्ण डेमो परिस्थिती**: येथे एक शक्तिशाली AI विकास कार्यप्रवाह आहे:

> "मी ग्राहक समर्थन एजंट तयार करत आहे. मला कॅटलॉगमधून एक चांगला तर्क मॉडेल शोधण्यात मदत करा, तो Azure AI सेवांवर डिप्लॉय करा, आमच्या दस्तऐवजांमधून ज्ञान आधार तयार करा, प्रतिसाद गुणवत्तेची चाचणी करण्यासाठी मूल्यांकन फ्रेमवर्क सेट करा, आणि नंतर GitHub टोकनसोबत समाकलनासाठी प्रोटोटायपिंगमध्ये मदत करा."

मायक्रोसॉफ्ट फाउंड्री MCP सर्व्हर हे करेल:
- आपल्या आवश्यकतांनुसार सर्वोत्तम तर्क मॉडेल शिफारस करण्यासाठी मॉडेल कॅटलॉगची चौकशी करा
- आपल्या पसंतीच्या Azure प्रदेशासाठी डिप्लॉयमेंट आदेश व कोटा माहिती प्रदान करा
- आपल्या दस्तऐवजांसाठी योग्य स्कीमासह Azure AI Search निर्देशांक सेट करा
- गुणवत्ता मेट्रिक्स व सुरक्षितता तपासणींसह मूल्यांकन पाईपलाईन्स कॉन्फिगर करा
- त्वरित परीक्षणासाठी GitHub प्रमाणीकरणासह प्रोटोटायपिंग कोड जनरेट करा
- आपल्या विशिष्ट तंत्रज्ञान संचासाठी सानुकूल सेटअप मार्गदर्शक प्रदान करा

**वैशिष्ट्यीकृत उदाहरण**: विकासक म्हणून, मी वेगवेगळ्या LLM मॉडेल्सशी जुळवून घेणे कठीण वाटले आहे. काही मुख्य मॉडेल्स मला माहित आहेत, पण मला वाटतंय की मी काही उत्पादनक्षमता आणि कार्यक्षमतेतील लाभ गमावत आहे. तसेच टोकन्स आणि कोटा व्यवस्थापनाचा त्रास आणि ताण आहे – मला कधीच खात्री नसते की मी योग्य कामासाठी योग्य मॉडेल निवडत आहे का किंवा माझा बजेट अकारण वापरत आहे का. मी या MCP सर्व्हरबद्दल जेम्स मोंटेमॅग्नो यांच्याकडून ऐकलं जेव्हा मी या पोस्टसाठी टीम मेंबर्सकडे सल्ला घेत होतो, आणि मला ते वापरण्यास उत्सुकता आहे! माझ्यासारख्या एखाद्यासाठी ज्याला सामान्य मॉडेलपेक्षा पुढे शोधायचं आहे आणि विशिष्ट कामांसाठी ऑप्टिमाइझ्ड मॉडेल्स शोधायचे आहेत, मॉडेल शोध क्षमता विशेष आकर्षक आहेत. मूल्यांकन फ्रेमवर्क मला प्रत्यक्षात चांगले परिणाम मिळत आहेत की नाही हे पडताळण्यास मदत करेल, फक्त नवीन काहीतरी करून पाहण्यासाठी नाही.

> **ℹ️ प्रयोगात्मक स्थिती**
> 
> हा MCP सर्व्हर प्रयोगात्मक असून सक्रिय विकासात आहे. वैशिष्ट्ये आणि API बदलू शकतात. Azure AI क्षमता एक्सप्लोर करण्यासाठी आणि प्रोटोटायप तयार करण्यासाठी उत्तम, पण उत्पादनासाठी स्थिरता गरजा तपासाव्यात.
### 10. 🏢 Microsoft 365 एजंट्स टूलकिट MCP सर्व्हर

**हे काय करते**: Microsoft 365 आणि Microsoft 365 Copilot सह एकत्रित AI एजंट्स आणि अनुप्रयोग तयार करण्यासाठी विकासकांना आवश्यक साधने उपलब्ध करुन देते, ज्यात स्कीमा पडताळणी, नमुना कोड पुनर्प्राप्ती, आणि समस्या निवारण सहाय्य यांचा समावेश आहे.

**हे का उपयुक्त आहे**: Microsoft 365 आणि Copilot साठी विकास करताना क्लिष्ट मॅनिफेस्ट स्कीमास आणि विशिष्ट विकास नमुन्यांचा समावेश असतो. हा MCP सर्व्हर आवश्यक विकास संसाधने थेट आपल्या कोडिंग वातावरणात आणतो, ज्यामुळे आपण स्कीमा पडताळणी करू शकता, नमुना कोड शोधू शकता, आणि सामान्य समस्या सोडवू शकता, वारंवार दस्तऐवज पाहण्याची गरज न पडता.

**प्रत्यक्ष वापर**: "माझा डिक्लेरटिव एजंट मॅनिफेस्ट पडताळा आणि स्कीमा त्रुटी दुरुस्त करा", "Microsoft Graph API प्लगइन अंमलात आणण्यासाठी नमुना कोड दाखवा", किंवा "माझ्या Teams अ‍ॅपचे प्रमाणीकरण समस्यांचे निराकरण करण्यात मदत करा"

**वैशिष्ट्यीकृत उदाहरण**: Build येथे M365 एजंट्स विषयी चर्चा करून माझ्या मित्र John Miller यांच्याशी संपर्क साधला आणि त्यांनी हा MCP शिफारस केला. M365 एजंट्ससाठी नवीन विकासकांसाठी हा उत्तम ठरू शकतो कारण तो टेम्प्लेट्स, नमुना कोड, आणि स्कॅफोल्डिंग प्रदान करतो ज्यामुळे दस्तऐवजांमध्ये अडकल्याशिवाय सुरुवात करता येते. मॅनिफेस्ट संरचनेतील त्रुटी टाळण्यासाठी स्कीमा पडताळणीची सुविधा विशेषतः उपयुक्त दिसते जी तासांनाच डिबगिंग वाचवते.

## पुढे काय? 🔮

## 📋 निष्कर्ष

मॉडेल कंटेक्स्ट प्रोटोकॉल (MCP) विकसित करण्याचा पद्धत बदलत आहे जिथे विकासक AI सहाय्यकांशी आणि बाह्य साधनांशी संवाद साधतात. हे १० मायक्रोसॉफ्ट MCP सर्व्हर्स एआय इंटिग्रेशनमध्ये मानकीकृत पद्धत वापरून सहज कार्यप्रवाह राबवतात, जे विकासकांना त्यांच्या कामाच्या प्रवाहात राहून शक्तिशाली बाह्य क्षमता वापरता येतात.

संपूर्ण Azure परिसंस्था एकत्रिकरणापासून ते Playwright सारख्या ब्राऊजर ऑटोमेशन आणि MarkItDown सारख्या दस्तऐवज प्रक्रियेसाठी विशेष साधने, हे सर्व्हर्स विविध विकास परिस्थितींमध्ये MCP कशी उत्पादनक्षमता वाढवू शकते हे दाखवतात. मानकीकृत प्रोटोकॉलमुळे या साधनांचा अनुभव सुसंगत होतो, ज्यामुळे एक सुसंगत विकास अनुभव निर्माण होतो.

MCP परिसंस्था जसजशी विकसित होत आहे, समुदायात सहभागी राहणे, नवीन सर्व्हर्सचा शोध घेणे, आणि सानुकूल सोल्यूशन्स तयार करणे हे विकासकांकरिता अधिकाधिक उत्पादकता वाढवण्याचे प्रमुख मार्ग असतील. MCP चे खुले मानक असल्यामुळे आपण वेगवेगळ्या विक्रेत्यांकडून साधने एकत्र करून आपल्या गरजेनुसार उत्तम कार्यप्रवाह तयार करू शकता.
