# AGENTS.md

## प्रोजेक्ट परिचय

**MCP for Beginners** हे मॉडेल संदर्भ प्रोटोकॉल (MCP) शिकण्यासाठी एक मुक्त-स्रोत शैक्षणिक अभ्यासक्रम आहे - जे एआय मॉडेल आणि क्लायंट अनुप्रयोगांमधील संवादासाठी एक मानकीकृत फ्रेमवर्क आहे. या रिपॉझिटरीमध्ये अनेक प्रोग्रामिंग भाषांमध्ये हाताळणी उदाहरणांसह सर्वसमावेशक शिक्षण साहित्य दिले आहे.

### मुख्य तंत्रज्ञान

- **प्रोग्रामिंग भाषा**: C#, Java, JavaScript, TypeScript, Python, Rust
- **फ्रेमवर्क्स आणि SDKs**: 
  - MCP SDK (`@modelcontextprotocol/sdk`)
  - Spring Boot (Java)
  - FastMCP (Python)
  - LangChain4j (Java)
- **डेटाबेस**: PostgreSQL सह pgvector विस्तार
- **क्लाउड प्लॅटफॉर्म्स**: Azure (Container Apps, OpenAI, Content Safety, Application Insights)
- **बिल्ड टूल्स**: npm, Maven, pip, Cargo
- **डॉक्युमेंटेशन**: Markdown सह स्वयंचलित बहुभाषिक भाषांतर (48+ भाषा)

### आर्किटेक्चर

- **11 मुख्य मॉड्यूल्स (00-11)**: मुलभूतांपासून प्रगत विषयांपर्यंत सलग शिक्षण मार्ग
- **हाताळणी प्रयोगशाळा**: अनेक भाषांमध्ये पूर्ण उत्तरांसह व्यावहारिक उदाहरणे
- **नमुना प्रोजेक्ट्स**: चालू MCP सर्व्हर व क्लायंट अंमलबजावणी
- **भाषांतर प्रणाली**: बहुभाषिक समर्थनासाठी स्वयंचलित GitHub Actions कार्यप्रवाह
- **प्रतिमा साधने**: केंद्रित प्रतिमा निर्देशिका भाषांतरित आवृत्त्यांसह

## सेटअप कमांड्स

ही एक डॉक्युमेंटेशन-केंद्रित रिपॉझिटरी आहे. बहुतेक सेटअप वैयक्तिक नमुना प्रोजेक्ट्स व प्रयोगशाळांमध्ये होतो.

### रिपॉझिटरी सेटअप

```bash
# रेपॉझिटरी क्लोन करा
git clone https://github.com/microsoft/mcp-for-beginners.git
cd mcp-for-beginners
```

### नमुना प्रोजेक्ट्ससह काम

नमुना प्रोजेक्ट्स या ठिकाणी आहेत:
- `03-GettingStarted/samples/` - भाषा-विशिष्ट उदाहरणे
- `03-GettingStarted/01-first-server/solution/` - पहिले सर्व्हर अंमलबजावणी
- `03-GettingStarted/02-client/solution/` - क्लायंट अंमलबजावणी
- `11-MCPServerHandsOnLabs/` - सर्वसमावेशक डेटाबेस एकत्रीकरण प्रयोगशाळा

प्रत्येक नमुना प्रोजेक्टमध्ये स्वतःचे सेटअप सूचनाः

#### TypeScript/JavaScript प्रोजेक्ट्स
```bash
cd <project-directory>
npm install
npm start
```

#### Python प्रोजेक्ट्स
```bash
cd <project-directory>
pip install -r requirements.txt
# किंवा
pip install -e .
python main.py
```

#### Java प्रोजेक्ट्स
```bash
cd <project-directory>
mvn clean install
mvn spring-boot:run
```

## विकास कार्यप्रवाह

### डॉक्युमेंटेशन रचना

- **मॉड्यूल 00-11**: सलग क्रमाने मुख्य अभ्यासक्रम सामग्री

- **images/**: मूळ प्रतिमा व आरेखन

### डॉक्युमेंटेशनमध्ये बदल करणे

1. फक्त इंग्रजी markdown फायली मूळ मॉड्यूल निर्देशिकांमध्ये (00-11) संपादित करा
2. `images/` निर्देशिकेत प्रतिमा आवश्यक असल्यास अद्ययावत करा
3. सहकारी-भाषांतर GitHub Action स्वयंचलितपणे भाषांतर निर्माण करेल
4. मुख्य शाखेत पुश केल्यावर भाषांतराची नवीन निर्मिती होईल

### भाषांतरांसह काम

- **स्वयंचलित भाषांतर**: सर्व भाषांतर GitHub Actions कार्यप्रवाहाद्वारे केले जाते
- **translations/** निर्देशिकेतील फायली थेट संपादित करू नका
- प्रत्येक भाषांतरित फाईलमध्ये भाषांतर मेटाडेटा अंतर्भूत आहे
- समर्थित भाषा: 48+ भाषा जसे की अरेबिक, चायनीज, फ्रेंच, जर्मन, हिंदी, जपानीज, कोरियन, पोर्तुगीज, रशियन, स्पॅनिश आणि बरेच काही

## कोड शैली मार्गदर्शक

### डॉक्युमेंटेशन स्टाईल

- स्पष्ट, नवशिकासाठी सुलभ भाषा वापरा
- कोड उदाहरणे अनेक भाषांमध्ये समाविष्ट करा जिथे लागू असेल
- markdown सर्वोत्तम पद्धतींचे पालन करा:
  - ATX-शैली हेडर्स (`#` सिंटॅक्स) वापरा
  - भाषा ओळखसहित fenced कोड ब्लॉक्स वापरा
  - प्रतिमांसाठी वर्णनात्मक alt मजकूर जोडा
  - ओळींची लांबी योग्य ठेवा (कठोर मर्यादा नाही, पण समजूतदारपणा ठेवा)

### कोड नमुना शैली

#### TypeScript/JavaScript
- ES मॉड्यूल्स (`import`/`export`) वापरा
- TypeScript कडक मोड नियमांचे पालन करा
- प्रकार सूचनांसहित कोड लिहा
- ES2022 लक्ष्य ठेवा

#### Python
- PEP 8 शैली नियमांचे पालन करा
- जिथे योग्य असेल तिथे प्रकार सूचनांचा वापर करा
- फंक्शन्स व क्लासेससाठी docstrings समाविष्ट करा
- आधुनिक Python वैशिष्ट्ये वापरा (3.8+)

#### Java
- Spring Boot नियमांचे पालन करा
- Java 21 वैशिष्ट्ये वापरा
- Maven चा मानक प्रोजेक्ट स्ट्रक्चर वापरा
- Javadoc टिप्पण्या जोडा

## योगदान मार्गदर्शक

### पुल विनंती प्रक्रिया

1. **Fork आणि Clone करा**: रिपॉझिटरी Fork करा आणि आपला Fork स्थानिकपणे क्लोन करा
2. **ब्रांच तयार करा**: वर्णनात्मक ब्रांच नावे वापरा (उदा., `fix/typo-module-3`, `add/python-example`)
3. **बदल करा**: केवळ इंग्रजी markdown फायली संपादित करा (भाषांतर नाही)
4. **स्थानिक चाचणी करा**: markdown योग्यरित्या दर्शवली जाते याची खात्री करा
5. **PR सबमिट करा**: स्पष्ट PR शीर्षक व वर्णन वापरा
6. **CLA**: आवश्यक तेव्हा Microsoft Contributor License Agreement वर स्वाक्षरी करा

### PR शीर्षक स्वरूप

स्पष्ट व वर्णनात्मक शीर्षक वापरा:
- `[Module XX] संक्षिप्त वर्णन` मॉड्यूल-विशिष्ट बदलासाठी
- `[Samples] वर्णन` नमुना कोड बदलांसाठी
- `[Docs] वर्णन` सामान्य डॉक्युमेंटेशन अद्ययतनांसाठी

### काय योगदान करावे

- डॉक्युमेंटेशन किंवा कोड उदाहरणातील बग दुरुस्त्या
- अतिरिक्त भाषांमध्ये नवीन कोड उदाहरणे
- विद्यमान सामग्रीतील स्पष्टता व सुधारणा
- नवीन केस स्टडीज किंवा व्यावहारिक उदाहरणे
- अस्पष्ट किंवा चुकीच्या सामग्रीसाठी इश्यू रिपोर्ट

### काय करू नये

- `translations/` निर्देशिकेतील फायली थेट संपादित करू नका

### सामान्य समस्या निराकरण

**Q: माझ्या PR ला भाषांतर तपासणीमध्ये अडचण येत आहे**  
A: खात्री करा की तुम्ही फक्त मुख्य मॉड्यूल निर्देशिकेतील इंग्रजी markdown फायली संपादित केल्या आहेत, भाषांतरित आवृत्त्या नाहीत.

**Q: नवीन भाषा कशी जोडावी?**  
A: भाषांतर कार्यप्रवाह मार्फत भाषा समर्थन व्यवस्थापित होते. नवीन भाषा जोडण्यासंबंधी चर्चा करण्यासाठी इश्यू उघडा.

**Q: कोड नमुने काम करत नाहीत**

**Q: प्रतिमा दर्शविल्या जात नाहीत**  
A: प्रतिमांचे मार्ग सापेक्ष व पुढील सापेक्ष स्लीशसह असल्याची खात्री करा. प्रतिमा `images/` निर्देशिका किंवा स्थानिकृत `translated_images/` मध्ये असाव्यात.
