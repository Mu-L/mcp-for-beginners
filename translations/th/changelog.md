# Changelog: หลักสูตร MCP สำหรับผู้เริ่มต้น

เอกสารนี้เป็นบันทึกของการเปลี่ยนแปลงที่สำคัญทั้งหมดที่ทำกับหลักสูตร Model Context Protocol (MCP) สำหรับผู้เริ่มต้น การเปลี่ยนแปลงจะถูกบันทึกในลำดับย้อนกลับตามเวลา (การเปลี่ยนแปลงใหม่สุดอยู่ด้านบน)

## 16 มิถุนายน 2026

### การปรับให้สอดคล้องกับข้อกำหนด MCP และการตรวจสอบตัวอย่าง

ตรวจสอบหลักสูตรเทียบกับ **ข้อกำหนด MCP 2025-11-25** ปัจจุบันและ SDK ทางการล่าสุด แล้วแก้ไขข้อมูลอ้างอิงข้อกำหนดที่ล้าสมัยที่เหลืออยู่และยืนยันว่าตัวอย่างหลักยังคงสร้างและรันได้

- **05-AdvancedTopics/mcp-security/README.md**: อัปเดตแบนเนอร์ "มาตรฐานปัจจุบัน" บทนำ หัวข้อหลักการความปลอดภัย หัวข้อข้อกำหนดบังคับ ส่วน Microsoft Entra ID ลิงก์อ้างอิง & ทรัพยากร และประกาศความปลอดภัยปิดท้าย (8 รายการอ้างอิง) เป็น 2025-11-25
- **05-AdvancedTopics/mcp-transport/README.md**: อัปเดตลิงก์สเปคทรัพยากรเพิ่มเติมและแบนเนอร์ "มาตรฐานปัจจุบัน" เป็น 2025-11-25
- **05-AdvancedTopics/mcp-realtimesearch/README.md**: แทนที่ลิงก์ `2025-03-26` ด้านความปลอดภัยและความน่าเชื่อถือด้วยหน้าปฏิบัติที่ดีที่สุดด้านความปลอดภัย 2025-11-25
- **03-GettingStarted/14-sampling/README.md**: อัปเดตลิงก์เอกสารอย่างเป็นทางการสำหรับการสุ่มตัวอย่างเป็น 2025-11-25

#### การตรวจสอบตัวอย่างเทียบกับ SDK ปัจจุบัน

- **TypeScript (03-GettingStarted/01-first-server/solution/typescript)**: รัน `npm install` ได้ `@modelcontextprotocol/sdk@1.29.0` ผ่าน `tsc --noEmit` โดยไม่มีข้อผิดพลาดประเภท — API `McpServer`/`StdioServerTransport` ที่มีอยู่ยังใช้ได้
- **Python (03-GettingStarted/01-first-server/solution/python)**: ทดสอบใน `.venv` แยกด้วย `mcp[cli]` (1.27.2); ผ่าน `py_compile` และ `FastMCP.list_tools()` ส่งกลับเครื่องมือ `add` และ `subtract` อย่างถูกต้อง
- ยืนยันว่าช่วงเวอร์ชันตัวอย่างทั้งหมดของ `@modelcontextprotocol/sdk` (`>=1.26.0` / `^1.26.0` / `^1.27.0`) สามารถลงตัวกับเวอร์ชันปัจจุบัน `1.29.0` ได้โดยไม่มีการเปลี่ยนแปลง API ที่ทำให้ใช้งานไม่ได้

### การเสริมความปลอดภัยและการแก้ไขช่องโหว่ Dependency

ดำเนินการตรวจสอบความปลอดภัยแบบเต็มในทุกไฟล์ manifest ของ dependency และซอร์สโค้ดตัวอย่างแล้วแก้ไข npm advisory ทั้งหมดและปัญหาระดับโค้ด 1 รายการ หลังแก้ไข, `npm audit` รายงาน **0 ช่องโหว่** ในทุกไดเรกทอรีที่ตรวจสอบ

#### ช่องโหว่ npm Dependency (แบบทรานสิติฟ) — แก้ไขแล้ว

ตรวจสอบไฟล์ `package-lock.json` จำนวน 15 ไฟล์ที่คอมมิต พบช่องโหว่จำกัดเฉพาะ dependency ที่ถูกใช้งานแบบทรานสิติฟโดย MCP Inspector dev tool, ลูกค้า OpenAI และ MCP SDK; ทั้งหมดได้รับการแก้ไขโดยไม่กระทบตัวอย่าง:
- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/inspector** และ **lab3/code/weather_mcp/inspector**: อัปเกรด `@modelcontextprotocol/inspector` (`0.16.6` / `0.14.1` → `0.22.0`), ซึ่งล้าง advisory ของ `ajv`, `brace-expansion`, `diff`, `path-to-regexp` และ `ws` ที่มาพร้อมแพ็กเกจ เพิ่ม entry `overrides` ของ npm บังคับใช้การแพตช์ `shell-quote@1.8.4` เพื่อลบ advisory สำคัญที่เหลือจาก `concurrently`; สร้างไฟล์ล็อกใหม่ (ตอนนี้ 0 ช่องโหว่)
- **03-GettingStarted/samples/typescript**: `npm audit fix` อัปเดต `qs` แบบทรานสิติฟ (ระดับกลาง) เป็นเวอร์ชันแพตช์แล้ว
- **03-GettingStarted/samples/javascript**: `npm audit fix` อัปเดต `hono` แบบทรานสิติฟ (ระดับกลาง) เป็นเวอร์ชันแพตช์แล้ว
- **03-GettingStarted/03-llm-client/solution/typescript**: `npm audit fix` อัปเดต `form-data` แบบทรานสิติฟ (ระดับสูง) เป็นเวอร์ชันแพตช์แล้ว
- **03-GettingStarted/11-simple-auth/solution/typescript**: สร้าง `package-lock.json` ที่ขาดหายไปเพื่อให้โปรเจกต์ทำใหม่และตรวจสอบได้ (0 ช่องโหว่)

#### การแก้ไขปัญหาความปลอดภัยระดับโค้ด (OWASP A03: การฉีดคำสั่ง)

- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/src/server.py**: ลบ `shell=True` จากเครื่องมือ `open_in_vscode` คำสั่งก่อนหน้านี้ `subprocess.run(["start", "", vscode_path, folder_path], shell=True)` อนุญาตให้มีเมตาแคแรคเตอร์เชลล์ในเส้นทางโฟลเดอร์ถูกตีความโดย `cmd.exe` (เวกเตอร์ฉีดคำสั่ง) ตอนนี้โค้ดเรียกใช้งาน `Code.exe` ที่แก้ไขแล้วโดยตรงพร้อมส่งโฟลเดอร์เป็นอาร์กิวเมนต์ — ไม่มีเชลล์ — เทียบเท่าทางฟังก์ชันและปลอดภัย

- **การเดินสายลูปเครื่องมือ MCP**: อัปเดต orchestration โต้ตอบ Python ให้ใช้ไคลเอนต์ `AsyncAnthropic` (แทน `Anthropic` แบบ synchronous ที่บล็อก), ส่งต่อ `ClientSession` ที่ใช้งานอยู่โดยตรงไปยังแต่ละรอบของเอเย่นต์, ดึงคำจำกัดความเครื่องมือผ่าน `session.list_tools()` ในแต่ละรอบ, และส่งบล็อก `tool_use` ผ่าน `session.call_tool()` ในลูปจนกว่ารุ่นจะส่งข้อความตอบกลับสุดท้ายออกมา

#### การพัฒนาคอนเซปต์หลัก (01-CoreConcepts/)
- **Primitive Client ใหม่ - Roots**: เพิ่มเอกสารครอบคลุมเกี่ยวกับ primitive client Roots เพื่อให้เซิร์ฟเวอร์เข้าใจขอบเขตไฟล์ซิสเต็มและสิทธิ์การเข้าถึง  
- **คำอธิบายเครื่องมือ**: เพิ่มเอกสารเกี่ยวกับคำอธิบายพฤติกรรมเครื่องมือ (`readOnlyHint`, `destructiveHint`) เพื่อการตัดสินใจใช้เครื่องมือที่ดียิ่งขึ้น  
- **การเรียกใช้งานเครื่องมือใน Sampling**: อัปเดตเอกสาร Sampling ให้รวมพารามิเตอร์ `tools` และ `toolChoice` สำหรับการเรียกใช้เครื่องมือโดยโมเดลระหว่างการขอ sampling  
- **การเปิดใช้งานโหมด URL**: เพิ่มเอกสารเกี่ยวกับการเปิดใช้งานบนพื้นฐาน URL สำหรับการโต้ตอบเว็บภายนอกที่เซิร์ฟเวอร์เริ่มต้น  
- **Tasks (ทดลอง)**: เพิ่มส่วนใหม่ที่อธิบายฟีเจอร์ทดลอง Tasks สำหรับห่อหุ้มการทำงานที่ทนทานและการดึงผลลัพธ์ล่าช้า  
- **รองรับไอคอน**: ระบุว่าเครื่องมือ, ทรัพยากร, เทมเพลตทรัพยากร, และ prompt สามารถเพิ่มไอคอนเป็น metadata เสริมได้

- **อัปเดตเวอร์ชันโปรโตคอล**: อัปเดตให้อ้างอิง MCP Specification 2025-11-25 ล่าสุด (เผยแพร่ 25 พฤศจิกายน 2025)  
  - เปลี่ยนการอ้างอิงเวอร์ชันทั้งหมดจาก 2025-06-18 เป็น 2025-11-25  
  - อัปเดตวันที่เอกสารจาก 18 สิงหาคม 2025 เป็น 18 ธันวาคม 2025  
  - ตรวจสอบ URL ข้อกำหนดให้ชี้ไปยังเอกสารปัจจุบันทั้งหมด

  - **โซลูชันความปลอดภัย Microsoft**: ตรวจสอบคำศัพท์และลิงก์ปัจจุบันสำหรับ Prompt Shields (ก่อนหน้านี้เรียกว่า "การตรวจจับความเสี่ยง jailbreak"), Azure Content Safety, Microsoft Entra ID, และ Azure Key Vault  
  - **OAuth 2.1 ความปลอดภัย**: ยืนยันความสอดคล้องกับแนวทางปฏิบัติความปลอดภัย OAuth ล่าสุด  
  - **มาตรฐาน OWASP**: ตรวจสอบว่าแหล่งข้อมูล OWASP Top 10 สำหรับ LLMs ยังคงทันสมัย  
  - **บริการ Azure**: ตรวจสอบลิงก์เอกสาร Microsoft Azure และแนวทางปฏิบัติที่ดีที่สุดทั้งหมด  
- **ความสอดคล้องมาตรฐาน**: ยืนยันว่ามาตรฐานความปลอดภัยทั้งหมดที่อ้างอิงยังคงเป็นปัจจุบัน  
  - NIST AI Risk Management Framework  
  - ISO 27001:2022  
  - แนวทางปฏิบัติความปลอดภัย OAuth 2.1  
  - กรอบงานความปลอดภัยและการยืนยันของ Azure  
- **แหล่งข้อมูลการใช้งาน**: ตรวจสอบลิงก์คู่มือการใช้งานและทรัพยากรทั้งหมด  
  - รูปแบบการพิสูจน์ตัวตน API การจัดการ Azure  
  - คู่มือการผนวกรวม Microsoft Entra ID  
  - การบริหารจัดการความลับด้วย Azure Key Vault  
  - ท่อการส่ง CI/CD DevSecOps และโซลูชันการตรวจสอบ

### การประกันคุณภาพเอกสาร  
- **ความสอดคล้องข้อกำหนด**: รับรองว่าเงื่อนไขความปลอดภัย MCP ที่จำเป็น (MUST/MUST NOT) สอดคล้องกับข้อกำหนดล่าสุด  
- **ความทันสมัยของแหล่งข้อมูล**: ตรวจสอบลิงก์ภายนอกทั้งหมดไปยังเอกสาร Microsoft, มาตรฐานความปลอดภัย และคู่มือการใช้งาน  
- **ครอบคลุมแนวทางปฏิบัติที่ดีที่สุด**: รับรองว่าครอบคลุมการพิสูจน์ตัวตน, การอนุญาต, ภัยคุกคามเฉพาะ AI, ความปลอดภัยสายการผลิต, และรูปแบบองค์กรอย่างครบถ้วน

## 6 ตุลาคม 2025

### การขยายส่วน Getting Started – การใช้งานเซิร์ฟเวอร์ขั้นสูง & การพิสูจน์ตัวตนง่ายๆ

#### การใช้งานเซิร์ฟเวอร์ขั้นสูง (03-GettingStarted/10-advanced)  
- **เพิ่มบทใหม่**: แนะนำคู่มือครอบคลุมสำหรับการใช้งานเซิร์ฟเวอร์ MCP ขั้นสูง ครอบคลุมทั้งสถาปัตยกรรมเซิร์ฟเวอร์ทั่วไปและระดับต่ำ  
  - **เซิร์ฟเวอร์ทั่วไป vs ระดับต่ำ**: เปรียบเทียบรายละเอียดและตัวอย่างโค้ด Python และ TypeScript สำหรับทั้งสองแนวทาง  
  - **ออกแบบด้วย Handlers**: อธิบายแนวทางจัดการเครื่องมือ/ทรัพยากร/prompt ด้วย handler เพื่อการใช้งานเซิร์ฟเวอร์ที่ยืดหยุ่นและขยายได้  
  - **รูปแบบการใช้งานจริง**: กรณีศึกษาจริงที่แสดงประโยชน์ของรูปแบบเซิร์ฟเวอร์ระดับต่ำสำหรับฟีเจอร์ขั้นสูงและสถาปัตยกรรม
#### การตรวจสอบสิทธิ์อย่างง่าย (03-GettingStarted/11-simple-auth)
- **เพิ่มบทใหม่**: คู่มือทีละขั้นตอนสำหรับการใช้งานการตรวจสอบสิทธิ์อย่างง่ายในเซิร์ฟเวอร์ MCP
  - **แนวคิดเกี่ยวกับการตรวจสอบสิทธิ์**: การอธิบายที่ชัดเจนเกี่ยวกับการตรวจสอบสิทธิ์เทียบกับการอนุญาต และการจัดการข้อมูลรับรอง
  - **การใช้งานการตรวจสอบสิทธิ์พื้นฐาน**: รูปแบบการตรวจสอบสิทธิ์ผ่านมิดเดิลแวร์ในภาษาไพธอน (Starlette) และไทป์สคริปต์ (Express) พร้อมตัวอย่างโค้ด
  - **การก้าวสู่ความปลอดภัยขั้นสูง**: คำแนะนำเกี่ยวกับการเริ่มต้นด้วยการตรวจสอบสิทธิ์อย่างง่ายและก้าวสู่ OAuth 2.1 และ RBAC พร้อมการอ้างอิงถึงโมดูลความปลอดภัยขั้นสูง

การเพิ่มเติมเหล่านี้ให้คำแนะนำเชิงปฏิบัติที่เป็นประโยชน์สำหรับการสร้างเซิร์ฟเวอร์ MCP ที่แข็งแกร่ง ปลอดภัย และยืดหยุ่นมากขึ้น เชื่อมโยงแนวคิดพื้นฐานกับรูปแบบการใช้งานระดับผลิตภัณฑ์ขั้นสูง

  - **โครงสร้างกรณีศึกษาที่ปรับปรุง**: ปรับรูปแบบให้สอดคล้องและให้คำอธิบายครอบคลุมในกรณีศึกษาทั้งเจ็ด
    - ตัวแทนท่องเที่ยว Azure AI: เน้นการประสานงานเอเจนต์หลายตัว
    - การบูรณาการ Azure DevOps: เน้นการทำงานอัตโนมัติของเวิร์กโฟลว์

#### MCP Custom Transports (05-AdvancedTopics/mcp-transport/) – คู่มือการใช้งานขั้นสูงใหม่
- **README.md**: คู่มือการใช้งานครบถ้วนสำหรับกลไกการขนส่ง MCP ที่กำหนดเอง
  - **Azure Event Grid Transport**: การใช้งานการขนส่งแบบเซิร์ฟเวอร์เลสแบบเหตุการณ์ขับเคลื่อนอย่างครบถ้วน
    - ตัวอย่าง C#, TypeScript และ Python พร้อมการผสาน Azure Functions
    - รูปแบบสถาปัตยกรรมเหตุการณ์ขับเคลื่อนสำหรับโซลูชัน MCP ที่ปรับขนาดได้
    - ตัวรับ webhook และการจัดการข้อความแบบ push
  - **Azure Event Hubs Transport**: การใช้งานการขนส่งสตรีมที่มี Throughput สูง
    - ความสามารถสตรีมแบบเรียลไทม์สำหรับสถานการณ์หน่วงต่ำ
    - กลยุทธ์การแบ่งพาร์ติชันและการจัดการ checkpoint
    - การจัดกลุ่มข้อความและการปรับแต่งประสิทธิภาพ
  - **รูปแบบการบูรณาการระดับองค์กร**: ตัวอย่างสถาปัตยกรรมพร้อมใช้งานสำหรับการผลิต
    - การประมวลผล MCP แบบกระจายผ่านหลาย Azure Functions
    - สถาปัตยกรรมการขนส่งแบบไฮบริดรวมหลายประเภทการขนส่ง
    - ความทนทาน ความน่าเชื่อถือ และกลยุทธ์การจัดการข้อผิดพลาดของข้อความ
  - **ความปลอดภัยและการตรวจสอบ**: การบูรณาการ Azure Key Vault และรูปแบบการสังเกตการณ์
    - การตรวจสอบสิทธิ์ผ่าน managed identity และการเข้าถึงโดยสิทธิ์น้อยที่สุด
    - การตรวจวัด Application Insights และการตรวจสอบประสิทธิภาพ
    - ตัวตัดวงจรและรูปแบบความทนทานต่อข้อผิดพลาด
  - **กรอบงานทดสอบ**: กลยุทธ์การทดสอบครบถ้วนสำหรับการขนส่งที่กำหนดเอง
    - การทดสอบหน่วยด้วย test doubles และ mocking frameworks
    - การทดสอบการผสานกลุ่มกับ Azure Test Containers
    - การพิจารณาด้านประสิทธิภาพและการทนต่อภาระงาน

#### วิศวกรรมบริบท (05-AdvancedTopics/mcp-contextengineering/) – สาขา AI ที่เกิดใหม่
- **README.md**: การสำรวจอย่างครบถ้วนของวิศวกรรมบริบทในฐานะสาขาที่เกิดใหม่
  - **หลักการสำคัญ**: การแบ่งปันบริบทครบถ้วน ความรับรู้ในการตัดสินใจดำเนินการ และการจัดการบริบทวินโดวส์
  - **ความสอดคล้องกับโปรโตคอล MCP**: วิธีที่การออกแบบ MCP ตอบโจทย์ความท้าทายทางวิศวกรรมบริบท
    - ข้อจำกัดของบริบทวินโดวส์และกลยุทธ์การโหลดแบบค่อยเป็นค่อยไป
    - การกำหนดความเกี่ยวข้องและการดึงบริบทแบบไดนามิก
    - การจัดการบริบทหลายโหมดและข้อพิจารณาด้านความปลอดภัย
  - **แนวทางการใช้งาน**: สถาปัตยกรรมแบบเธรดเดียวเทียบกับหลายเอเจนต์
    - เทคนิคการแบ่งชิ้นบริบทและการจัดลำดับความสำคัญ
    - การโหลดบริบทแบบก้าวหน้าและกลยุทธ์บีบอัด
    - แนวทางบริบทแบบหลายชั้นและการปรับแต่งการดึงข้อมูล
  - **กรอบการวัดผล**: เมตริกที่เกิดใหม่สำหรับประเมินประสิทธิภาพบริบท
    - ประสิทธิภาพการป้อนข้อมูล ประสิทธิภาพ การคุณภาพ และประสบการณ์ผู้ใช้
    - แนวทางทดลองสำหรับการปรับบริบท
    - การวิเคราะห์ความล้มเหลวและวิธีการปรับปรุง

    - ความปลอดภัยโทเค็น & การตรวจสอบสิทธิ์ด้วยการผสานผู้ให้บริการตัวตนภายนอก

    - การปฏิบัติตามข้อกำหนด & การกำกับดูแลตามระเบียบ
    - การควบคุมความปลอดภัยขั้นสูงด้วยสถาปัตยกรรม zero trust
    - การบูรณาการระบบความปลอดภัย Microsoft อย่างครอบคลุม
    - การพัฒนาด้านความปลอดภัยอย่างต่อเนื่องด้วยแนวทางปรับตัว
  - **โซลูชันความปลอดภัย Microsoft**: คำแนะนำการบูรณาการที่ได้รับการปรับปรุงสำหรับ Prompt Shields, Azure Content Safety, Entra ID และ GitHub Advanced Security

#### แนวคิดหลัก (01-CoreConcepts/) - ปรับปรุงใหม่ทั้งหมด
- **อัปเดตเวอร์ชันโปรโตคอล**: ปรับปรุงให้อ้างอิง MCP Specification 2025-06-18 ปัจจุบัน พร้อมระบบเวอร์ชันโดยวันที่ (รูปแบบ YYYY-MM-DD)
- **การปรับแต่งสถาปัตยกรรม**: ขยายคำอธิบายของ Hosts, Clients, และ Servers ให้สอดคล้องกับรูปแบบสถาปัตยกรรม MCP ปัจจุบัน
  - Hosts ถูกกำหนดชัดเจนว่าเป็นแอป AI ที่ประสานงานการเชื่อมต่อ Client MCP หลายรายการ
  - Clients อธิบายว่าเป็นตัวเชื่อมโปรโตคอลที่รักษาความสัมพันธ์แบบหนึ่งต่อหนึ่งกับเซิร์ฟเวอร์
  - Servers ปรับปรุงให้รองรับสถานการณ์ใช้งานแบบโลคอลกับระยะไกล
- **การจัดระเบียบ Primitive ใหม่**: ปรับโครงสร้างโดยสมบูรณ์ของ server และ client primitives
  - Server Primitives: แหล่งข้อมูล (data sources), prompt (เทมเพลต), เครื่องมือ (ฟังก์ชันที่เรียกใช้ได้) พร้อมคำอธิบายและตัวอย่างอย่างละเอียด
  - Client Primitives: การสุ่มตัวอย่าง (completion ของ LLM), การเรียกข้อมูล (ข้อมูลผู้ใช้), การบันทึก (ดีบัก/การตรวจสอบ)
  - อัปเดตด้วยรูปแบบเมธอดสำหรับค้นหา (`*/list`), ดึงข้อมูล (`*/get`), และเรียกใช้งาน (`*/call`) ที่เป็นปัจจุบัน
- **สถาปัตยกรรมโปรโตคอล**: นำเสนอสองชั้นของโมเดลสถาปัตยกรรม
  - ชั้นข้อมูล: พื้นฐาน JSON-RPC 2.0 พร้อมการจัดการวงจรชีวิตและ primitives
  - ชั้นขนส่ง: STDIO (ในเครื่อง) และ HTTP แบบสตรีมพร้อม SSE (ระยะไกล)
- **กรอบความปลอดภัย**: หลักการความปลอดภัยที่ครอบคลุมรวมถึงความยินยอมจากผู้ใช้ที่ชัดเจน การปกป้องความเป็นส่วนตัวของข้อมูล ความปลอดภัยในการใช้งานเครื่องมือ และความปลอดภัยของชั้นขนส่ง
- **รูปแบบการสื่อสาร**: อัปเดตข้อความโปรโตคอลเพื่อแสดงกระบวนการเริ่มต้น ค้นหา เรียกใช้ และแจ้งเตือน
- **ตัวอย่างโค้ด**: ปรับปรุงตัวอย่างหลายภาษา (.NET, Java, Python, JavaScript) ให้สอดคล้องกับรูปแบบ SDK MCP ปัจจุบัน

#### ความปลอดภัย (02-Security/) - ปรับโครงสร้างความปลอดภัยโดยสมบูรณ์  
- **ความสอดคล้องกับมาตรฐาน**: ปรับให้สอดคล้องเต็มที่กับข้อกำหนดความปลอดภัย MCP Specification 2025-06-18
- **วิวัฒนาการการพิสูจน์ตัวตน**: บันทึกวิวัฒนาการจากเซิร์ฟเวอร์ OAuth แบบกำหนดเองไปสู่การมอบหมายผ่านผู้ให้บริการตัวตนภายนอก (Microsoft Entra ID)
- **การวิเคราะห์ภัยคุกคามเฉพาะ AI**: ขยายเนื้อหาครอบคลุมช่องทางการโจมตี AI สมัยใหม่
  - สถานการณ์โจมตี prompt injection พร้อมตัวอย่างที่เกิดขึ้นจริง
  - กลไกการวางยาพิษเครื่องมือและรูปแบบการโจมตีแบบ "rug pull"
  - การวางยาพิษบริบทหน้าต่างและการโจมตีทำให้สับสนโมเดล
- **โซลูชันความปลอดภัย AI ของ Microsoft**: ครอบคลุมระบบนิเวศความปลอดภัยของ Microsoft อย่างครบถ้วน
  - AI Prompt Shields พร้อมเทคนิคการตรวจจับขั้นสูง การเน้น และการกำหนดขอบเขต
  - รูปแบบการผสานรวม Azure Content Safety
  - GitHub Advanced Security สำหรับการปกป้องซัพพลายเชน
- **มาตรการป้องกันภัยขั้นสูง**: ควบคุมความปลอดภัยอย่างละเอียดสำหรับ
  - การแฮ็กเซสชันพร้อมสถานการณ์การโจมตีเฉพาะ MCP และข้อกำหนดรหัสเซสชันทางคริปโตกราฟี
  - ปัญหา confused deputy ในเซ็นเซอร์ MCP proxy พร้อมข้อกำหนดยินยอมชัดเจน
  - ช่องโหว่ token passthrough พร้อมข้อควบคุมการตรวจสอบที่บังคับใช้
- **ความปลอดภัยซัพพลายเชน**: ขยายความครอบคลุมซัพพลายเชน AI รวมโมเดลพื้นฐาน บริการ embeddings ผู้ให้บริบท และ API จากบุคคลที่สาม
- **ความปลอดภัยพื้นฐาน**: ปรับปรุงการผสานรวมกับรูปแบบความปลอดภัยองค์กร รวมสถาปัตยกรรม zero trust และระบบนิเวศความปลอดภัยของ Microsoft
- **การจัดระเบียบทรัพยากร**: จัดหมวดหมู่ลิงก์ทรัพยากรอย่างครบถ้วนตามประเภท (เอกสารทางการ, มาตรฐาน, งานวิจัย, โซลูชัน Microsoft, คู่มือการใช้งาน)

### การปรับปรุงคุณภาพเอกสาร
- **เป้าหมายการเรียนรู้ที่มีโครงสร้าง**: ขยายเป้าหมายการเรียนรู้ให้เฉพาะเจาะจงและปฏิบัติได้จริง
- **การอ้างอิงข้าม**: เพิ่มลิงก์ระหว่างหัวข้อความปลอดภัยกับแนวคิดหลักที่เกี่ยวข้อง
- **ข้อมูลที่เป็นปัจจุบัน**: ปรับปรุงการอ้างอิงวันที่และลิงก์สเปคให้เป็นมาตรฐานปัจจุบันทั้งหมด
- **คำแนะนำการใช้งาน**: เพิ่มแนวทางการใช้งานที่เจาะจงและปฏิบัติได้ในส่วนต่าง ๆ ทั้งสองหัวข้อ

## 16 กรกฎาคม 2025

### README และการปรับปรุงการนำทาง
- ออกแบบใหม่ระบบนาเวชันหลักสูตรใน README.md อย่างสมบูรณ์
- แทนที่แท็ก `<details>` ด้วยรูปแบบตารางที่เข้าถึงง่ายกว่า
- สร้างตัวเลือกเลย์เอาต์ทางเลือกในโฟลเดอร์ "alternative_layouts"
- เพิ่มตัวอย่างนำทางแบบการ์ด, แบบแท็บ, และแบบแอคคอร์เดียน
- อัปเดตส่วนโครงสร้างรีโพสิตอรีให้รวมไฟล์ล่าสุดทั้งหมด
- ปรับปรุงส่วน "วิธีใช้หลักสูตรนี้" ด้วยคำแนะนำที่ชัดเจน
- ปรับลิงก์สเปค MCP ให้ชี้ไปยัง URL ที่ถูกต้อง
- เพิ่มหัวข้อ Context Engineering (5.14) ในโครงสร้างหลักสูตร

### การปรับปรุงคู่มือการศึกษา
- ปรับปรุงคู่มือการศึกษาให้สอดคล้องกับโครงสร้างรีโพสิตอรีปัจจุบันอย่างสมบูรณ์
- เพิ่มหัวข้อใหม่สำหรับ MCP Clients และ Tools รวมถึง MCP Servers ยอดนิยม
- อัปเดตแผนที่หลักสูตรภาพรวมให้สะท้อนหัวข้อทั้งหมดอย่างแม่นยำ
- ขยายคำอธิบายหัวข้อขั้นสูงครอบคลุมทุกพื้นที่เฉพาะทาง
- อัปเดตส่วนกรณีศึกษาให้สะท้อนตัวอย่างจริง
- เพิ่มบันทึกการเปลี่ยนแปลงฉบับครบถ้วนนี้

### การมีส่วนร่วมของชุมชน (06-CommunityContributions/)
- เพิ่มข้อมูลรายละเอียดเกี่ยวกับเซิร์ฟเวอร์ MCP สำหรับสร้างภาพ
- เพิ่มส่วนครอบคลุมการใช้ Claude ใน VSCode
- เพิ่มคำแนะนำการตั้งค่าและใช้งานไคลเอนต์เทอร์มินัล Cline
- อัปเดตส่วน MCP client รวมตัวเลือกไคลเอนต์ยอดนิยมทั้งหมด
- ขยายตัวอย่างการมีส่วนร่วมพร้อมตัวอย่างโค้ดที่ถูกต้องมากขึ้น

### หัวข้อขั้นสูง (05-AdvancedTopics/)
- จัดระเบียบโฟลเดอร์หัวข้อเฉพาะทั้งหมดด้วยการตั้งชื่อที่สม่ำเสมอ
- เพิ่มเนื้อหาและตัวอย่างด้านวิศวกรรมบริบท
- เพิ่มเอกสารการผสานรวมเอเจนต์ Foundry
- ขยายเอกสารการผสานรวมความปลอดภัย Entra ID
