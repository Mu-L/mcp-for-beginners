# บทนำสู่การผสานฐานข้อมูล MCP

## 🎯 สิ่งที่ห้องปฏิบัติการนี้ครอบคลุม

ห้องปฏิบัติการบทนำนี้ให้ภาพรวมที่ครอบคลุมในการสร้างเซิร์ฟเวอร์ Model Context Protocol (MCP) โดยผสานฐานข้อมูล คุณจะเข้าใจกรณีธุรกิจ สถาปัตยกรรมเชิงเทคนิค และการใช้งานจริงผ่านกรณีวิเคราะห์ Zava Retail ที่ https://github.com/microsoft/MCP-Server-and-PostgreSQL-Sample-Retail

## ภาพรวม

**Model Context Protocol (MCP)** ช่วยให้ผู้ช่วย AI เข้าถึงและโต้ตอบกับแหล่งข้อมูลภายนอกได้อย่างปลอดภัยแบบเรียลไทม์ เมื่อรวมกับการผสานฐานข้อมูล MCP จะเปิดใช้งานความสามารถอันทรงพลังสำหรับแอปพลิเคชัน AI ขับเคลื่อนด้วยข้อมูล

เส้นทางการเรียนรู้นี้สอนให้คุณสร้างเซิร์ฟเวอร์ MCP ที่พร้อมสำหรับการใช้งานจริงที่เชื่อมต่อผู้ช่วย AI กับข้อมูลยอดขายค้าปลีกผ่าน PostgreSQL โดยใช้รูปแบบองค์กรเช่น Row Level Security, การค้นหาเชิงความหมาย และการเข้าถึงข้อมูลแบบหลายผู้เช่า

## วัตถุประสงค์การเรียนรู้

เมื่อสิ้นสุดห้องปฏิบัติการนี้ คุณจะสามารถ:

- **นิยาม** Model Context Protocol และประโยชน์หลักสำหรับการผสานฐานข้อมูล
- **ระบุ** ส่วนประกอบหลักของสถาปัตยกรรมเซิร์ฟเวอร์ MCP ที่มีฐานข้อมูล
- **เข้าใจ** กรณีใช้งาน Zava Retail และข้อกำหนดทางธุรกิจ
- **รับรู้** รูปแบบองค์กรสำหรับการเข้าถึงฐานข้อมูลอย่างปลอดภัยและขยายขนาดได้
- **ระบุ** เครื่องมือและเทคโนโลยีที่ใช้ตลอดเส้นทางการเรียนรู้นี้

## 🧭 ความท้าทาย: AI พบกับข้อมูลโลกจริง

### ข้อจำกัด AI แบบดั้งเดิม

ผู้ช่วย AI สมัยใหม่มีพลังอย่างมากแต่เผชิญข้อจำกัดสำคัญเมื่อต้องทำงานกับข้อมูลธุรกิจโลกจริง:

| **ความท้าทาย** | **คำอธิบาย** | **ผลกระทบทางธุรกิจ** |
|---------------|-----------------|-------------------|
| **ความรู้แบบคงที่** | โมเดล AI ที่ฝึกด้วยชุดข้อมูลคงที่ไม่สามารถเข้าถึงข้อมูลธุรกิจปัจจุบันได้ | ข้อมูลล้าสมัย โอกาสสูญหาย |
| **ข้อมูลแยกส่วน** | ข้อมูลถูกเก็บไว้ในฐานข้อมูล APIs และระบบที่ AI ไม่สามารถเข้าถึงได้ | การวิเคราะห์ไม่ครบถ้วน งานไหลเวียนไม่ต่อเนื่อง |
| **ข้อจำกัดด้านความปลอดภัย** | การเข้าถึงฐานข้อมูลโดยตรงก่อให้เกิดความกังวลด้านความปลอดภัยและความสอดคล้อง | การปรับใช้งานจำกัด การเตรียมข้อมูลด้วยมือ |
| **คำสั่งซับซ้อน** | ผู้ใช้ธุรกิจต้องมีความรู้ทางเทคนิคเพื่อสกัดข้อมูลเชิงลึก | การยอมรับน้อย กระบวนการไม่มีประสิทธิภาพ |

### ทางออก MCP

Model Context Protocol แก้ไขความท้าทายเหล่านี้โดยการให้:

- **การเข้าถึงข้อมูลแบบเรียลไทม์**: ผู้ช่วย AI สืบค้นฐานข้อมูลและ API แบบสด
- **การผสานที่ปลอดภัย**: การควบคุมการเข้าถึงด้วยการรับรองตัวตนและสิทธิ์
- **อินเทอร์เฟซภาษาธรรมชาติ**: ผู้ใช้ธุรกิจสามารถถามคำถามด้วยภาษาอังกฤษธรรมดา
- **โปรโตคอลมาตรฐาน**: ทำงานร่วมกับแพลตฟอร์ม AI และเครื่องมือต่าง ๆ ได้

6. **รักษาความปลอดภัยข้อมูล** ด้วยการควบคุมการเข้าถึงตามบทบาท

- **การเข้าถึงข้อมูลแบบหลายผู้เช่า** ที่ผู้จัดการร้านเห็นข้อมูลเฉพาะร้านของตน
- **การสืบค้นที่ยืดหยุ่น** รองรับการดำเนินงาน SQL ที่ซับซ้อน
- **การค้นหาเชิงความหมาย** สำหรับค้นพบสินค้าและข้อแนะนำ
- **ข้อมูลแบบเรียลไทม์** ที่สะท้อนสถานะธุรกิจปัจจุบัน
- **การรับรองตัวตนที่ปลอดภัย** ด้วย row-level security
- **สถาปัตยกรรมที่สามารถปรับขนาดได้** รองรับผู้ใช้จำนวนมากพร้อมกัน

### ส่วนประกอบหลัก

#### **1. เลเยอร์เซิร์ฟเวอร์ MCP**
- **FastMCP Framework**: การนำเสนอเซิร์ฟเวอร์ MCP ด้วย Python รุ่นใหม่
- **การลงทะเบียนเครื่องมือ**: การกำหนดเครื่องมือแบบประกาศด้วยความปลอดภัยของชนิดข้อมูล
- **บริบทคำร้อง**: การจัดการตัวตนผู้ใช้และเซสชัน
- **การจัดการข้อผิดพลาด**: การจัดการข้อผิดพลาดที่แข็งแกร่งและการบันทึก

#### **2. เลเยอร์ผสานฐานข้อมูล**
- **การจัดกลุ่มการเชื่อมต่อ**: การจัดการการเชื่อมต่อ asyncpg อย่างมีประสิทธิภาพ
- **ผู้ให้บริการสกีมา**: การค้นพบโครงสร้างตารางแบบไดนามิก
- **เครื่องมือสั่งคำสั่ง**: การดำเนินการ SQL อย่างปลอดภัยพร้อมบริบท RLS
- **การจัดการธุรกรรม**: การปฏิบัติตาม ACID และการจัดการ rollback

#### **3. เลเยอร์ความปลอดภัย**
- **Row Level Security**: PostgreSQL RLS สำหรับแยกข้อมูลหลายผู้เช่า
- **ตัวตนผู้ใช้**: การรับรองตัวตนและการอนุญาตผู้จัดการร้าน
- **การควบคุมการเข้าถึง**: สิทธิ์แบบละเอียดและการติดตามการตรวจสอบ
- **การตรวจสอบข้อมูลนำเข้า**: การป้องกัน SQL injection และการตรวจสอบคำสั่ง

#### **4. เลเยอร์เสริม AI**
- **การค้นหาเชิงความหมาย**: การฝังเวกเตอร์สำหรับการค้นพบสินค้า
- **การผสาน Azure OpenAI**: การสร้าง embedding จากข้อความ
- **อัลกอริทึมความคล้ายคลึงกัน**: การค้นหา similarity ด้วย pgvector cosine similarity
- **การปรับแต่งการค้นหา**: การจัดทำดัชนีและการปรับปรุงประสิทธิภาพ

### สถานการณ์ 2: การค้นพบสินค้าด้วยการค้นหาเชิงความหมาย

**เป้าหมาย**: ค้นหาสินค้าที่คล้ายกับคำร้องขอลูกค้า

**คำสั่งภาษาธรรมชาติ**:
> "สินค้าที่เราขายที่คล้ายกับ 'ขั้วต่อไฟฟ้ากันน้ำสำหรับใช้งานกลางแจ้ง' มีอะไรบ้าง?"

**สิ่งที่เกิดขึ้น**:
1. คำสั่งถูกประมวลผลโดยเครื่องมือค้นหาเชิงความหมาย  
2. Azure OpenAI สร้างเวกเตอร์ embedding  
3. pgvector ค้นหาความคล้ายคลึงกัน  
4. สินค้าที่เกี่ยวข้องเรียงอันดับตามความเกี่ยวข้อง  
5. ผลลัพธ์รวมถึงรายละเอียดสินค้าและสถานะพร้อมจำหน่าย  
6. AI แนะนำทางเลือกและโอกาสจับคู่สินค้า

### สถานการณ์ 3: การวิเคราะห์ข้ามร้าน

**ผู้ใช้**: Jennifer, ผู้จัดการภูมิภาค  
**เป้าหมาย**: เปรียบเทียบประสิทธิภาพของร้านค้าทั้งหมด

**คำสั่งภาษาธรรมชาติ**:
> "เปรียบเทียบยอดขายตามหมวดหมู่ของร้านทั้งหมดใน 6 เดือนที่ผ่านมา"

**สิ่งที่เกิดขึ้น**:
1. ตั้งบริบท RLS สำหรับผู้จัดการภูมิภาค  
2. สร้างคำสั่งซับซ้อนข้ามหลายร้าน  
3. รวมรวมข้อมูลจากตำแหน่งร้านค้า  
4. ผลลัพธ์รวมถึงแนวโน้มและการเปรียบเทียบ  
5. AI ระบุข้อมูลเชิงลึกและข้อแนะนำ

## 🔒 เจาะลึกความปลอดภัยและมัลติเทนันซี

การใช้งานของเราเน้นความปลอดภัยระดับองค์กร:

### Row Level Security (RLS)

PostgreSQL RLS รับประกันการแยกข้อมูล:

```sql
-- Store managers see only their store's data
CREATE POLICY store_manager_policy ON retail.orders
  FOR ALL TO store_managers
  USING (store_id = get_current_user_store());

-- Regional managers see multiple stores
CREATE POLICY regional_manager_policy ON retail.orders
  FOR ALL TO regional_managers
  USING (store_id = ANY(get_user_store_list()));
```

### การจัดการตัวตนผู้ใช้

การเชื่อมต่อ MCP แต่ละรายการประกอบด้วย:  
- **รหัสผู้จัดการร้าน**: ตัวระบุเฉพาะสำหรับบริบท RLS  
- **การกำหนดบทบาท**: สิทธิ์และระดับการเข้าถึง  
- **การจัดการเซสชัน**: โทเค็นการรับรองตัวตนที่ปลอดภัย  
- **การบันทึกตรวจสอบ**: ประวัติการเข้าถึงอย่างครบถ้วน

### การปกป้องข้อมูล

หลายชั้นของความปลอดภัย:  
- **การเข้ารหัสการเชื่อมต่อ**: TLS สำหรับทุกการเชื่อมต่อฐานข้อมูล  
- **การป้องกัน SQL injection**: ใช้คำสั่งที่มีพารามิเตอร์เท่านั้น  
- **การตรวจสอบข้อมูลนำเข้า**: การตรวจสอบคำร้องอย่างครอบคลุม  
- **การจัดการข้อผิดพลาด**: ไม่มีข้อมูลสำคัญในข้อความข้อผิดพลาด
