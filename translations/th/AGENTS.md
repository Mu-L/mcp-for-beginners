# AGENTS.md

## ภาพรวมโครงการ

**MCP สำหรับผู้เริ่มต้น** เป็นหลักสูตรการศึกษาระบบเปิดเพื่อเรียนรู้ Model Context Protocol (MCP) - กรอบมาตรฐานสำหรับการโต้ตอบระหว่างโมเดล AI และแอปพลิเคชันของลูกค้า ที่เก็บนี้ให้วัสดุการเรียนรู้ครบถ้วนพร้อมตัวอย่างโค้ดปฏิบัติในหลายภาษาโปรแกรม

### เทคโนโลยีสำคัญ

- **ภาษาโปรแกรม**: C#, Java, JavaScript, TypeScript, Python, Rust
- **เฟรมเวิร์ก & SDK**: 
  - MCP SDK (`@modelcontextprotocol/sdk`)
  - Spring Boot (Java)
  - FastMCP (Python)
  - LangChain4j (Java)
- **ฐานข้อมูล**: PostgreSQL พร้อมส่วนขยาย pgvector
- **แพลตฟอร์มคลาวด์**: Azure (Container Apps, OpenAI, Content Safety, Application Insights)
- **เครื่องมือสร้าง**: npm, Maven, pip, Cargo
- **เอกสาร**: Markdown พร้อมการแปลภาษาอัตโนมัติหลายภาษา (กว่า 48 ภาษา)

### สถาปัตยกรรม

- **11 โมดูลหลัก (00-11)**: เส้นทางการเรียนรู้อย่างต่อเนื่องจากพื้นฐานถึงหัวข้อขั้นสูง
- **ห้องปฏิบัติการปฏิบัติจริง**: แบบฝึกหัดพร้อมโค้ดคำตอบครบถ้วนในหลายภาษา
- **โครงการตัวอย่าง**: ตัวอย่างเซิร์ฟเวอร์และไคลเอนต์ MCP ที่ทำงานได้
- **ระบบแปลภาษา**: กระบวนการทำงาน GitHub Actions อัตโนมัติสำหรับรองรับหลายภาษา
- **ทรัพยากรภาพ**: โฟลเดอร์ภาพรวมศูนย์พร้อมเวอร์ชันแปลภาษา

## คำสั่งการตั้งค่า

นี่คือที่เก็บที่เน้นเอกสาร ส่วนใหญ่การตั้งค่าจะเกิดขึ้นในโครงการตัวอย่างและห้องปฏิบัติการแต่ละรายการ

### การตั้งค่ารีโพซิทอรี

```bash
# โคลนที่เก็บข้อมูล
git clone https://github.com/microsoft/mcp-for-beginners.git
cd mcp-for-beginners
```

### การทำงานกับโครงการตัวอย่าง

โครงการตัวอย่างอยู่ที่:
- `03-GettingStarted/samples/` - ตัวอย่างเฉพาะภาษา
- `03-GettingStarted/01-first-server/solution/` - ตัวอย่างเซิร์ฟเวอร์แรก
- `03-GettingStarted/02-client/solution/` - ตัวอย่างไคลเอนต์
- `11-MCPServerHandsOnLabs/` - ห้องปฏิบัติการผสานรวมฐานข้อมูลครบวงจร

แต่ละโครงการตัวอย่างมีคำแนะนำการตั้งค่าของตนเอง:

#### โครงการ TypeScript/JavaScript
```bash
cd <project-directory>
npm install
npm start
```

#### โครงการ Python
```bash
cd <project-directory>
pip install -r requirements.txt
# หรือ
pip install -e .
python main.py
```

#### โครงการ Java
```bash
cd <project-directory>
mvn clean install
mvn spring-boot:run
```

## กระบวนการพัฒนา

### โครงสร้างเอกสาร

- **โมดูล 00-11**: เนื้อหาหลักของหลักสูตรตามลำดับ
- **translations/**: เวอร์ชันภาษาเฉพาะ (สร้างอัตโนมัติ ห้ามแก้ไขโดยตรง)
- **translated_images/**: ภาพที่ถูกแปลแล้ว (สร้างอัตโนมัติ)
- **images/**: ภาพต้นฉบับและแผนภาพ

### การแก้ไขเอกสาร

1. แก้ไขไฟล์ markdown ภาษาอังกฤษในโฟลเดอร์โมดูลหลัก (00-11) เท่านั้น
2. อัปเดตภาพในโฟลเดอร์ `images/` ตามต้องการ
3. แอ็คชัน co-op-translator บน GitHub จะสร้างการแปลให้โดยอัตโนมัติ
4. แปลจะถูกสร้างใหม่เมื่อมีการ push เข้าสาขา main

### การทำงานกับงานแปล

- **แปลอัตโนมัติ**: กระบวนการทำงาน GitHub Actions จัดการแปลทั้งหมด
- **ห้ามแก้ไขด้วยมือ** ไฟล์ในโฟลเดอร์ `translations/`
- เมตาดาต้าแปลฝังอยู่ในแต่ละไฟล์แปล
- รองรับมากกว่า 48 ภาษา เช่น อาหรับ, จีน, ฝรั่งเศส, เยอรมัน, ฮินดี, ญี่ปุ่น, เกาหลี, โปรตุเกส, รัสเซีย, สเปน และอื่นๆ อีกมาก

## แนวทางสไตล์โค้ด

### สไตล์เอกสาร

- ใช้ภาษาชัดเจนและเหมาะสมสำหรับผู้เริ่มต้น
- รวมตัวอย่างโค้ดในหลายภาษาตามที่เหมาะสม
- ปฏิบัติตามแนวทาง markdown ที่ดี:
  - ใช้หัวเรื่องแบบ ATX (`#` syntax)
  - ใช้บล็อกโค้ดมีรั้วพร้อมตัวบ่งชี้ภาษา
  - ใส่ข้อความ alt อธิบายภาพ
  - จำกัดความยาวบรรทัดอย่างเหมาะสม (ไม่มีขีดจำกัดตายตัวแต่ต้องสมเหตุสมผล)

### สไตล์ตัวอย่างโค้ด

#### TypeScript/JavaScript
- ใช้โมดูล ES (`import`/`export`)
- ปฏิบัติตามข้อกำหนด strict mode ของ TypeScript
- ใส่คำชี้แจงประเภทข้อมูล (type annotations)
- กำหนดเป้าหมาย ES2022

#### Python
- ปฏิบัติตามแนวทาง PEP 8
- ใช้ type hints เมื่อเหมาะสม
- มี docstrings สำหรับฟังก์ชันและคลาส
- ใช้ฟีเจอร์ Python รุ่นใหม่ (3.8+)

#### Java
- ปฏิบัติตามแนวทาง Spring Boot
- ใช้ฟีเจอร์ Java 21
- โครงสร้างโครงการ Maven มาตรฐาน
- รวมความเห็น Javadoc

## แนวทางการมีส่วนร่วม

### กระบวนการ Pull Request

1. **Fork และ Clone**: Fork ที่เก็บนี้แล้ว clone มายังเครื่องของคุณ
2. **สร้างสาขา**: ใช้ชื่อสาขาที่อธิบายได้ (เช่น `fix/typo-module-3`, `add/python-example`)
3. **ทำการเปลี่ยนแปลง**: แก้ไขไฟล์ markdown ภาษาอังกฤษเท่านั้น (ไม่แก้ไขไฟล์แปล)
4. **ทดสอบในเครื่อง**: ตรวจสอบว่า markdown แสดงผลถูกต้อง
5. **ส่ง PR**: ใช้หัวข้อและคำอธิบาย PR ที่ชัดเจน
6. **CLA**: ลงชื่อในข้อตกลงใบอนุญาตผู้ร่วมพัฒนา Microsoft เมื่อได้รับแจ้ง

### รูปแบบหัวข้อ PR

ใช้หัวข้อที่ชัดเจนและอธิบายได้:
- `[Module XX] คำอธิบายสั้น` สำหรับการเปลี่ยนแปลงเฉพาะโมดูล
- `[Samples] คำอธิบาย` สำหรับการเปลี่ยนแปลงตัวอย่างโค้ด
- `[Docs] คำอธิบาย` สำหรับการอัปเดตเอกสารทั่วไป

### สิ่งที่ควรมีส่วนร่วม

- แก้ไขบั๊กในเอกสารหรือโค้ดตัวอย่าง
- ตัวอย่างโค้ดใหม่ในภาษาที่เพิ่มขึ้น
- ชี้แจงและปรับปรุงเนื้อหาที่มีอยู่
