# กรณีที่ 3: เอกสารในตัวแก้ไขพร้อม MCP เซิร์ฟเวอร์ใน VS Code

## ภาพรวม

ในกรณีนี้ คุณจะได้เรียนรู้วิธีนำเอกสาร Microsoft Learn Docs เข้าสู่สภาพแวดล้อม Visual Studio Code ของคุณโดยตรงโดยใช้ MCP เซิร์ฟเวอร์ แทนที่จะต้องเปลี่ยนแท็บเบราว์เซอร์ไปมาเพื่อค้นหาเอกสาร คุณสามารถเข้าถึง ค้นหา และอ้างอิงเอกสารอย่างเป็นทางการได้ภายในตัวแก้ไขของคุณ วิธีนี้ช่วยให้การทำงานของคุณราบรื่นขึ้น ทำให้คุณโฟกัสกับงาน และสามารถรวมการทำงานกับเครื่องมืออย่าง GitHub Copilot ได้อย่างไร้รอยต่อ

- ค้นหาและอ่านเอกสารภายใน VS Code โดยไม่ต้องออกจากสภาพแวดล้อมการเขียนโค้ดของคุณ
- อ้างอิงเอกสารและแทรกลิงก์ลงใน README หรือไฟล์คอร์สของคุณโดยตรง
- ใช้ GitHub Copilot และ MCP ร่วมกันเพื่อการทำงานเอกสารที่ขับเคลื่อนด้วย AI อย่างไร้รอยต่อ

## วัตถุประสงค์การเรียนรู้

เมื่อจบบทนี้ คุณจะเข้าใจวิธีตั้งค่าและใช้งาน MCP เซิร์ฟเวอร์ภายใน VS Code เพื่อเสริมประสิทธิภาพการทำงานด้านเอกสารและการพัฒนา คุณจะสามารถ:

- กำหนดค่าพื้นที่ทำงานของคุณให้ใช้ MCP เซิร์ฟเวอร์สำหรับการค้นหาเอกสาร
- ค้นหาและแทรกเอกสารโดยตรงจากภายใน VS Code
- รวมพลังของ GitHub Copilot และ MCP เพื่อการทำงานที่มีประสิทธิภาพมากขึ้น เสริมด้วย AI

ทักษะเหล่านี้จะช่วยให้คุณรักษาการโฟกัส ปรับปรุงคุณภาพเอกสาร และเพิ่มผลผลิตในฐานะนักพัฒนาหรือนักเขียนเทคนิค

## วิธีแก้ปัญหา

เพื่อให้เข้าถึงเอกสารภายในตัวแก้ไขได้ คุณจะปฏิบัติตามขั้นตอนหลายประการที่รวม MCP เซิร์ฟเวอร์กับ VS Code และ GitHub Copilot วิธีแก้ปัญหานี้เหมาะสำหรับผู้แต่งคอร์ส นักเขียนเอกสาร และนักพัฒนาที่ต้องการรักษาการโฟกัสในตัวแก้ไขขณะทำงานกับเอกสารและ Copilot

- เพิ่มลิงก์อ้างอิงอย่างรวดเร็วไปยัง README ในขณะที่เขียนเอกสารคอร์สหรือโปรเจค
- ใช้ Copilot สร้างโค้ด และ MCP ค้นหาและอ้างอิงเอกสารที่เกี่ยวข้องทันที
- รักษาการโฟกัสในตัวแก้ไขและเพิ่มผลผลิต

### คู่มือทีละขั้นตอน

เพื่อเริ่มต้น ให้ทำตามขั้นตอนเหล่านี้ สำหรับแต่ละขั้นตอน คุณสามารถเพิ่มภาพหน้าจอจากโฟลเดอร์ assets เพื่อแสดงขั้นตอนอย่างชัดเจน

1. **เพิ่มการกำหนดค่า MCP:**  
   ในโฟลเดอร์รากของโปรเจคของคุณ สร้างไฟล์ `.vscode/mcp.json` และเพิ่มการกำหนดค่าดังนี้  
   ```json
   {
     "servers": {
       "LearnDocsMCP": {
         "url": "https://learn.microsoft.com/api/mcp"
       }
     }
   }
   ```
   การกำหนดค่านี้บอก VS Code ว่าจะเชื่อมต่อกับ [`Microsoft Learn Docs MCP server`](https://github.com/MicrosoftDocs/mcp) อย่างไร
   
   ![ขั้นตอนที่ 1: เพิ่ม mcp.json ไปยังโฟลเดอร์ .vscode](../../../../../../translated_images/th/step1-mcp-json.c06a007fccc3edfa.webp)
    
2. **เปิดแผง GitHub Copilot Chat:**  
   หากคุณยังไม่มีส่วนขยาย GitHub Copilot ติดตั้งอยู่ ให้ไปที่มุมมอง Extensions ใน VS Code และติดตั้ง คุณสามารถดาวน์โหลดได้โดยตรงจาก [Visual Studio Code Marketplace](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot-chat) จากนั้น เปิดแผง Copilot Chat จากแถบด้านข้าง

   ![ขั้นตอนที่ 2: เปิดแผง Copilot Chat](../../../../../../translated_images/th/step2-copilot-panel.f1cc86e9b9b8cd1a.webp)

3. **เปิดใช้งานโหมด agent และตรวจสอบเครื่องมือ:**  
   ในแผง Copilot Chat เปิดใช้งานโหมด agent

   ![ขั้นตอนที่ 3: เปิดใช้งานโหมด agent และตรวจสอบเครื่องมือ](../../../../../../translated_images/th/step3-agent-mode.cdc32520fd7dd1d1.webp)

   หลังจากเปิดใช้งานโหมด agent ให้ตรวจสอบว่า MCP เซิร์ฟเวอร์ปรากฏเป็นหนึ่งในเครื่องมือที่ใช้งานได้ เพื่อให้แน่ใจว่า agent ของ Copilot สามารถเข้าถึงเซิร์ฟเวอร์เอกสารเพื่อดึงข้อมูลที่เกี่ยวข้อง
   
   ![ขั้นตอนที่ 3: ตรวจสอบเครื่องมือ MCP server](../../../../../../translated_images/th/step3-verify-mcp-tool.76096a6329cbfecd.webp)
4. **เริ่มการสนทนาใหม่และส่งคำถามให้ agent:**  
   เปิดการสนทนาใหม่ในแผง Copilot Chat คุณสามารถส่งคำถามเกี่ยวกับเอกสารของคุณได้ทันที Agent จะใช้ MCP เซิร์ฟเวอร์เพื่อดึงและแสดงเอกสาร Microsoft Learn ที่เกี่ยวข้องโดยตรงในตัวแก้ไขของคุณ

   - *"ผมกำลังจะเขียนแผนการศึกษาในหัวข้อ X ผมจะเรียนเป็นเวลา 8 สัปดาห์ ขอแนะนำเนื้อหาที่ควรเรียนแต่ละสัปดาห์ด้วยครับ"*

   ![ขั้นตอนที่ 4: ส่งคำถามให้ agent ในแชท](../../../../../../translated_images/th/step4-prompt-chat.12187bb001605efc.webp)

5. **คำถามสด:**

   > มาดูคำถามสดจากส่วน [#get-help](https://discord.gg/D6cRhjHWSC) ใน Microsoft Foundry Discord ([ดูข้อความต้นฉบับ](https://discord.com/channels/1113626258182504448/1385498306720829572)):
   
   *"ผมกำลังหาคำตอบเกี่ยวกับการปรับใช้โซลูชันหลาย agent โดยใช้ agent AI ที่พัฒนาบน Azure AI Foundry ผมเห็นว่ามีวิธีการปรับใช้โดยตรงเช่นช่องทาง Copilot Studio ไม่มากนัก ดังนั้น วิธีการต่าง ๆ ที่จะทำให้ผู้ใช้ระดับองค์กรสามารถโต้ตอบและทำงานได้สำเร็จมีอะไรบ้าง?  
มีบทความ/บล็อกมากมายที่บอกว่าเราสามารถใช้บริการ Azure Bot ทำหน้าที่เป็นสะพานระหว่าง MS Teams กับ Azure AI Foundry Agents ได้ ถ้าเช่นนั้นจะใช้ได้ไหมหากผมตั้งค่า Azure bot ที่เชื่อมต่อกับ Orchestrator Agent บน Azure AI Foundry ผ่าน Azure function เพื่อจัดการ orchestrate หรือว่าผมต้องสร้าง Azure function สำหรับแต่ละ agent ที่เป็นส่วนหนึ่งของโซลูชันหลาย agent เพื่อทำ orchestration ที่ Bot framework? ข้อเสนอแนะอื่น ๆ ยินดีรับครับ"*

   ![ขั้นตอนที่ 5: คำถามสด](../../../../../../translated_images/th/step5-live-queries.49db3e4a50bea273.webp)

   Agent จะตอบกลับด้วยลิงก์เอกสารและสรุปที่เกี่ยวข้อง ซึ่งคุณสามารถแทรกลงในไฟล์ markdown ของคุณโดยตรงหรือใช้เป็นข้อมูลอ้างอิงในโค้ดของคุณ

### ตัวอย่างคำถาม

นี่คือตัวอย่างคำถามที่คุณสามารถลองใช้ คำถามเหล่านี้จะสาธิตวิธีที่ MCP เซิร์ฟเวอร์และ Copilot ทำงานร่วมกันเพื่อให้เอกสารและการอ้างอิงที่รับรู้บริบทได้ทันทีโดยไม่ต้องออกจาก VS Code:

- "แสดงวิธีใช้ทริกเกอร์ Azure Functions ให้ผมหน่อย"
- "แทรกลิงก์ไปยังเอกสารทางการสำหรับ Azure Key Vault"
- "แนวปฏิบัติที่ดีที่สุดสำหรับการรักษาความปลอดภัยทรัพยากร Azure คืออะไร"
- "หาคู่มือเริ่มต้นเร็วสำหรับบริการ Azure AI ให้หน่อย"

คำถามเหล่านี้จะแสดงให้เห็นวิธีที่ MCP เซิร์ฟเวอร์และ Copilot ทำงานร่วมกันเพื่อให้เอกสารและการอ้างอิงที่รับรู้บริบทได้ทันทีโดยไม่ต้องออกจาก VS Code
