# 🚀 10 เซิร์ฟเวอร์ Microsoft MCP ที่กำลังเปลี่ยนแปลงประสิทธิภาพการทำงานของนักพัฒนา

## 🎯 สิ่งที่คุณจะได้เรียนรู้ในคู่มือนี้

คู่มือเชิงปฏิบัตินี้นำเสนอเซิร์ฟเวอร์ Microsoft MCP สิบตัวที่กำลังเปลี่ยนแปลงวิธีที่นักพัฒนาทำงานร่วมกับผู้ช่วย AI แทนที่จะอธิบายแค่สิ่งที่เซิร์ฟเวอร์ MCP *สามารถ* ทำได้ เราจะแสดงให้เห็นถึงเซิร์ฟเวอร์ที่กำลังสร้างผลกระทบจริงในเวิร์กโฟลว์การพัฒนาประจำวันที่ Microsoft และที่อื่นๆ

เซิร์ฟเวอร์แต่ละตัวในคู่มือนี้ได้รับการคัดเลือกโดยอิงจากการใช้งานจริงและความคิดเห็นจากนักพัฒนา คุณจะได้ค้นพบไม่เพียงแต่สิ่งที่เซิร์ฟเวอร์แต่ละตัวทำ แต่ทำไมมันจึงสำคัญและวิธีใช้ประโยชน์สูงสุดในโครงการของคุณเอง ไม่ว่าคุณจะเป็นมือใหม่กับ MCP หรือกำลังมองหาขยายการตั้งค่าที่มีอยู่ เซิร์ฟเวอร์เหล่านี้เป็นเครื่องมือที่ใช้งานได้จริงและมีผลกระทบมากที่สุดในระบบนิเวศของ Microsoft

> **💡 เคล็ดลับเริ่มต้นอย่างรวดเร็ว**
> 
> ใหม่กับ MCP? ไม่ต้องกังวล! คู่มือนี้ออกแบบมาให้เหมาะกับผู้เริ่มต้น เราจะอธิบายแนวคิดไปเรื่อย ๆ และคุณสามารถกลับไปดูโมดูล [แนะนำ MCP](../00-Introduction/README.md) และ [แนวคิดหลัก](../01-CoreConcepts/README.md) สำหรับพื้นฐานที่ลึกกว่าได้เสมอ

## ภาพรวม

คู่มือที่ครอบคลุมนี้สำรวจเซิร์ฟเวอร์ Microsoft MCP สิบตัวที่ปฏิวัติการที่นักพัฒนาสื่อสารกับผู้ช่วย AI และเครื่องมือต่างประเทศ ตั้งแต่การจัดการทรัพยากร Azure ไปจนถึงการประมวลผลเอกสาร เซิร์ฟเวอร์เหล่านี้แสดงพลังของ Model Context Protocol ในการสร้างเวิร์กโฟลว์การพัฒนาที่ราบรื่นและมีประสิทธิภาพ

## วัตถุประสงค์การเรียนรู้

เมื่อจบคู่มือนี้ คุณจะ:
- เข้าใจว่าเซิร์ฟเวอร์ MCP ช่วยเพิ่มประสิทธิภาพนักพัฒนาอย่างไร
- เรียนรู้การใช้งานเซิร์ฟเวอร์ MCP ที่มีผลกระทบสูงจาก Microsoft
- ค้นพบกรณีใช้งานจริงของแต่ละเซิร์ฟเวอร์
- รู้วิธีตั้งค่าและกำหนดค่าเซิร์ฟเวอร์เหล่านี้ใน VS Code และ Visual Studio
- สำรวจระบบนิเวศ MCP ที่กว้างขึ้นและทิศทางในอนาคต

## 🔧 เข้าใจเซิร์ฟเวอร์ MCP: คู่มือสำหรับผู้เริ่มต้น

### เซิร์ฟเวอร์ MCP คืออะไร?

ในฐานะผู้เริ่มต้นกับ Model Context Protocol (MCP) คุณอาจสงสัยว่า: "เซิร์ฟเวอร์ MCP คืออะไร และทำไมต้องสนใจ?" มาเริ่มด้วยคำอุปมาง่ายๆ

นึกภาพเซิร์ฟเวอร์ MCP เป็นผู้ช่วยเฉพาะทางที่ช่วยให้ผู้ช่วยเขียนโค้ด AI ของคุณ (เหมือนกับ GitHub Copilot) เชื่อมต่อกับเครื่องมือและบริการภายนอก เช่นเดียวกับที่คุณใช้แอปต่างๆ บนโทรศัพท์เพื่อทำงานต่างกัน — หนึ่งแอปสำหรับสภาพอากาศ หนึ่งแอปสำหรับนำทาง หนึ่งแอปสำหรับธนาคาร — เซิร์ฟเวอร์ MCP ช่วยให้ผู้ช่วย AI ของคุณสามารถโต้ตอบกับเครื่องมือและบริการพัฒนาต่างๆ ได้

### ปัญหาที่เซิร์ฟเวอร์ MCP แก้ไข

ก่อนจะมีเซิร์ฟเวอร์ MCP หากคุณต้องการ:
- ตรวจสอบทรัพยากร Azure ของคุณ
- สร้างปัญหาใน GitHub
- สืบค้นฐานข้อมูลของคุณ
- ค้นหาเอกสารอ้างอิง

คุณจะต้องหยุดเขียนโค้ด เปิดเว็บเบราว์เซอร์ ไปยังเว็บไซต์ที่เกี่ยวข้อง และทำงานเหล่านี้ด้วยตนเอง การสลับบริบทบ่อยๆ นี้ขัดจังหวะการทำงานและลดประสิทธิภาพ

### เซิร์ฟเวอร์ MCP เปลี่ยนประสบการณ์การพัฒนาของคุณอย่างไร

ด้วยเซิร์ฟเวอร์ MCP คุณสามารถอยู่ในสภาพแวดล้อมการพัฒนาของคุณ (VS Code, Visual Studio เป็นต้น) และเพียงแค่ขอให้ผู้ช่วย AI จัดการงานเหล่านี้ ตัวอย่างเช่น:

**แทนที่จะเป็นเวิร์กโฟลว์แบบดั้งเดิมนี้:**
1. หยุดเขียนโค้ด
2. เปิดเบราว์เซอร์
3. ไปที่พอร์ทัล Azure
4. ค้นหาข้อมูลบัญชีเก็บข้อมูล
5. กลับไปที่ VS Code
6. เขียนโค้ดต่อ

**ตอนนี้คุณสามารถทำแบบนี้ได้:**
1. ถาม AI: "สถานะของบัญชีเก็บข้อมูล Azure ของฉันเป็นอย่างไร?"
2. เขียนโค้ดต่อโดยใช้ข้อมูลที่ได้รับ

### ประโยชน์สำคัญสำหรับผู้เริ่มต้น

#### 1. 🔄 **รักษาสภาพความไหลของคุณไว้**
- ไม่ต้องสลับไปมาระหว่างแอปพลิเคชันหลายๆ ตัว
- มุ่งเน้นที่โค้ดที่เขียน
- ลดภาระทางจิตใจในการจัดการเครื่องมือต่างๆ

#### 2. 🤖 **ใช้ภาษาธรรมชาติแทนคำสั่งที่ซับซ้อน**
- แทนที่จะจำไวยากรณ์ SQL ให้บรรยายว่าคุณต้องการข้อมูลอะไร
- แทนที่จะจำคำสั่ง Azure CLI ให้บอกว่าสิ่งที่คุณอยากทำคืออะไร
- ให้ AI จัดการรายละเอียดทางเทคนิคในขณะที่คุณโฟกัสกับตรรกะ

#### 3. 🔗 **เชื่อมต่อเครื่องมือหลายอย่างเข้าด้วยกัน**
- สร้างเวิร์กโฟลว์ที่ทรงพลังโดยการรวมบริการต่างๆ
- ตัวอย่าง: "ดึงปัญหาใหม่ทั้งหมดใน GitHub และสร้างงานที่เกี่ยวข้องใน Azure DevOps"
- สร้างระบบอัตโนมัติโดยไม่ต้องเขียนสคริปต์ซับซ้อน

#### 4. 🌐 **เข้าถึงระบบนิเวศที่กำลังเติบโต**
- ได้ประโยชน์จากเซิร์ฟเวอร์ที่สร้างโดย Microsoft, GitHub, และบริษัทอื่นๆ
- ผสมผสานเครื่องมือจากผู้ขายต่างๆ ได้อย่างราบรื่น
- เข้าร่วมระบบนิเวศมาตรฐานที่ใช้งานได้กับผู้ช่วย AI ต่างๆ

#### 5. 🛠️ **เรียนรู้โดยการลงมือทำ**
- เริ่มต้นด้วยเซิร์ฟเวอร์สำเร็จรูปเพื่อเข้าใจแนวคิด
- ค่อยๆ สร้างเซิร์ฟเวอร์ของคุณเองเมื่อคุณคุ้นเคยมากขึ้น
- ใช้ SDK และเอกสารที่มีเพื่อแนะนำการเรียนรู้ของคุณ

### ตัวอย่างในโลกจริงสำหรับผู้เริ่มต้น

สมมติว่าคุณเพิ่งเริ่มพัฒนาเว็บและกำลังทำโปรเจกต์แรก นี่คือวิธีที่เซิร์ฟเวอร์ MCP สามารถช่วยคุณได้:

**วิธีแบบดั้งเดิม:**
```
1. Code a feature
2. Open browser → Navigate to GitHub
3. Create an issue for testing
4. Open another tab → Check Azure docs for deployment
5. Open third tab → Look up database connection examples
6. Return to VS Code
7. Try to remember what you were doing
```

**ด้วยเซิร์ฟเวอร์ MCP:**
```
1. Code a feature
2. Ask AI: "Create a GitHub issue for testing this login feature"
3. Ask AI: "How do I deploy this to Azure according to the docs?"
4. Ask AI: "Show me the best way to connect to my database"
5. Continue coding with all the information you need
```

### ข้อได้เปรียบมาตรฐานสำหรับองค์กร

MCP กำลังกลายเป็นมาตรฐานในอุตสาหกรรม ซึ่งหมายความว่า:
- **ความสม่ำเสมอ**: ประสบการณ์ที่คล้ายกันในเครื่องมือและบริษัทต่างๆ
- **ความสามารถในการทำงานร่วมกันได้**: เซิร์ฟเวอร์จากผู้ขายต่างๆ ทำงานร่วมกันได้
- **รองรับอนาคต**: ทักษะและการตั้งค่าถ่ายโอนระหว่างผู้ช่วย AI ต่างๆ
- **ชุมชน**: ระบบนิเวศขนาดใหญ่ของความรู้และทรัพยากรที่แบ่งปันกัน

## เริ่มต้นกับ MCP ใน VS Code และ Visual Studio 🚀

การตั้งค่าเซิร์ฟเวอร์ MCP เหล่านี้ไม่ซับซ้อนหากคุณใช้ Visual Studio Code หรือ Visual Studio 2022 ร่วมกับ GitHub Copilot

### การตั้งค่า VS Code

นี่คือขั้นตอนพื้นฐานสำหรับ VS Code:

1. **เปิดใช้งานโหมด Agent**: ใน VS Code ให้เปลี่ยนเป็นโหมด Agent ในหน้าต่าง Copilot Chat
2. **กำหนดค่าเซิร์ฟเวอร์ MCP**: เพิ่มการตั้งค่าเซิร์ฟเวอร์ไปยังไฟล์ settings.json ของ VS Code
3. **เริ่มเซิร์ฟเวอร์**: คลิกปุ่ม "เริ่ม" สำหรับแต่ละเซิร์ฟเวอร์ที่คุณต้องการใช้
4. **เลือกเครื่องมือ**: เลือกเซิร์ฟเวอร์ MCP ที่จะเปิดใช้งานสำหรับเซสชันปัจจุบัน

ดูคำแนะนำการตั้งค่าอย่างละเอียดได้ที่ [เอกสาร MCP ของ VS Code](https://code.visualstudio.com/docs/copilot/copilot-mcp)

> **💡 เคล็ดลับมือโปร: จัดการเซิร์ฟเวอร์ MCP เหมือนมือโปร!**
> 
> มุมมองส่วนขยายของ VS Code ตอนนี้รวม [อินเทอร์เฟซใหม่ที่ใช้งานง่ายสำหรับจัดการเซิร์ฟเวอร์ MCP ที่ติดตั้งแล้ว](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode)! คุณสามารถเข้าถึงอย่างรวดเร็วเพื่อเริ่ม หยุด และจัดการเซิร์ฟเวอร์ MCP ที่ติดตั้งทั้งหมดโดยใช้อินเทอร์เฟซที่ชัดเจนและเรียบง่าย ลองใช้ดู!

### การตั้งค่า Visual Studio 2022

สำหรับ Visual Studio 2022 (เวอร์ชัน 17.14 หรือใหม่กว่า):

1. **เปิดใช้งานโหมด Agent**: คลิกเมนู "ถาม" ในหน้าต่าง GitHub Copilot Chat และเลือก "Agent"
2. **สร้างไฟล์กำหนดค่า**: สร้างไฟล์ `.mcp.json` ในไดเรกทอรีโซลูชันของคุณ (ตำแหน่งแนะนำ: `<SOLUTIONDIR>\.mcp.json`)
3. **กำหนดค่าเซิร์ฟเวอร์**: เพิ่มการตั้งค่าเซิร์ฟเวอร์ MCP ของคุณโดยใช้รูปแบบ MCP มาตรฐาน
4. **อนุมัติเครื่องมือ**: เมื่อมีการแจ้งอนุมัติ ให้อนุมัติเครื่องมือที่คุณต้องการใช้พร้อมกับสิทธิ์ในขอบเขตที่เหมาะสม

ดูคำแนะนำการตั้งค่า Visual Studio อย่างละเอียดได้ที่ [เอกสาร MCP ของ Visual Studio](https://learn.microsoft.com/visualstudio/ide/mcp-servers)

เซิร์ฟเวอร์ MCP แต่ละตัวมีข้อกำหนดการกำหนดค่าของตัวเอง (สตริงเชื่อมต่อ การรับรองความถูกต้อง ฯลฯ) แต่รูปแบบการตั้งค่าจะเหมือนกันทั้งสอง IDE

**ทำไมจึงมีประโยชน์**: แม้มันจะดูเหมือน "แค่เอกสาร" แต่เซิร์ฟเวอร์นี้สำคัญมากสำหรับนักพัฒนาทุกคนที่ใช้เทคโนโลยี Microsoft หนึ่งในข้อร้องเรียนหลักจากนักพัฒนา .NET เกี่ยวกับผู้ช่วยเขียนโค้ด AI คือพวกเขาไม่ทันกับการอัปเดตล่าสุดของ .NET และ C# Microsoft Learn Docs MCP Server แก้ไขปัญหานี้ด้วยการให้การเข้าถึงเอกสารที่เป็นปัจจุบันที่สุด, เอกสารอ้างอิง API และแนวปฏิบัติที่ดีที่สุดแบบเรียลไทม์ ไม่ว่าคุณจะทำงานกับ Azure SDK เวอร์ชันล่าสุด, สำรวจฟีเจอร์ใหม่ใน C# 13 หรือใช้รูปแบบ .NET Aspire ที่ล้ำสมัย เซิร์ฟเวอร์นี้ช่วยให้ผู้ช่วย AI ของคุณมีข้อมูลที่ถูกต้องและทันสมัยเพื่อสร้างโค้ดที่แม่นยำและสมัยใหม่

**การใช้งานจริง**: "คำสั่ง az cli สำหรับสร้าง Azure container app ตามเอกสาร Microsoft Learn อย่างเป็นทางการมีอะไรบ้าง?" หรือ "ฉันจะกำหนดค่า Entity Framework กับ dependency injection ใน ASP.NET Core อย่างไร?" หรือ "ช่วยตรวจสอบโค้ดนี้เพื่อให้แน่ใจว่าสอดคล้องกับคำแนะนำด้านประสิทธิภาพในเอกสาร Microsoft Learn" เซิร์ฟเวอร์นี้ให้การค้นหาที่ครอบคลุมทั้ง Microsoft Learn, เอกสาร Azure และเอกสาร Microsoft 365 โดยใช้การค้นหาเชิงความหมายขั้นสูงเพื่อค้นหาข้อมูลที่เกี่ยวข้องที่สุดในบริบท มันคืนค่าชิ้นเนื้อหาคุณภาพสูงสูงสุด 10 ชิ้นพร้อมชื่อบทความและ URL โดยจะเข้าถึงเอกสาร Microsoft ใหม่ล่าสุดเสมอเมื่อมีการเผยแพร่

**ตัวอย่างเด่น**: เซิร์ฟเวอร์นี้เผยแพร่เครื่องมือ `microsoft_docs_search` ที่ทำการค้นหาเชิงความหมายในเอกสารเทคนิคทางการของ Microsoft เมื่อกำหนดค่าแล้ว คุณสามารถถามคำถามเช่น "ฉันจะทำการพิสูจน์ตัวตน JWT ใน ASP.NET Core อย่างไร?" และได้รับคำตอบอย่างละเอียด มีแหล่งข้อมูลอ้างอิง คุณภาพของการค้นหานี้ยอดเยี่ยมเพราะมันเข้าใจบริบท — ถ้าถามเรื่อง "containers" ในบริบท Azure จะคืนข้อมูลเอกสาร Azure Container Instances ขณะที่คำเดียวกันในบริบท .NET จะคืนข้อมูลชุดข้อมูลเกี่ยวกับ C# collection ที่เกี่ยวข้อง

นี่เป็นประโยชน์อย่างยิ่งสำหรับไลบรารีและกรณีใช้งานที่เปลี่ยนแปลงเร็วหรืออัปเดตล่าสุด ตัวอย่างเช่น ในโปรเจกต์โค้ดดิ้งล่าสุด ฉันต้องการใช้ฟีเจอร์ในเวอร์ชันล่าสุดของ .NET Aspire และ Microsoft.Extensions.AI ด้วยการรวมเซิร์ฟเวอร์ Microsoft Learn Docs MCP ฉันจึงสามารถใช้ประโยชน์จากเอกสาร API และคำแนะนำที่เพิ่งเผยแพร่ได้

### 2. ☁️ Azure MCP Server
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

**สิ่งที่มันทำ**: Azure MCP Server คือชุดเชื่อมต่อบริการ Azure ที่ครอบคลุมมากกว่า 15 รายการ ซึ่งนำระบบนิเวศ Azure มาใช้กับเวิร์กโฟลว์ AI ของคุณ นี่ไม่ใช่แค่เซิร์ฟเวอร์เดียว – แต่เป็นชุดรวมพลังที่ประกอบด้วยการจัดการทรัพยากร การเชื่อมต่อฐานข้อมูล (PostgreSQL, SQL Server) การวิเคราะห์ล็อกด้วย KQL ผ่าน Azure Monitor การผนวกรวม Cosmos DB และอื่นๆ อีกมากมาย

**ทำไมมันจึงมีประโยชน์**: นอกจากการจัดการทรัพยากร Azure แล้ว เซิร์ฟเวอร์นี้ช่วยปรับปรุงคุณภาพโค้ดอย่างมากเมื่อนำไปใช้กับ Azure SDKs เมื่อคุณใช้ Azure MCP ในโหมด Agent มันไม่ได้แค่ช่วยเขียนโค้ด – แต่มันช่วยเขียนโค้ด Azure ที่ *ดีกว่า* ซึ่งปฏิบัติตามรูปแบบการตรวจสอบตัวตนในปัจจุบัน วิธีจัดการข้อผิดพลาดที่ดีที่สุด และใช้ฟีเจอร์ล่าสุดของ SDK แทนที่จะได้โค้ดทั่วไปที่อาจใช้งานได้ คุณจะได้โค้ดที่เป็นไปตามรูปแบบที่แนะนำโดย Azure สำหรับงานในสภาพแวดล้อมการผลิต

**โมดูลสำคัญรวมถึง**:
- **🗄️ ตัวเชื่อมต่อฐานข้อมูล**: เข้าถึงฐานข้อมูล Azure for PostgreSQL และ SQL Server ผ่านภาษาธรรมชาติโดยตรง
- **📊 Azure Monitor**: การวิเคราะห์ล็อกด้วย KQL และข้อมูลเชิงลึกเชิงปฏิบัติการ
- **🌐 การจัดการทรัพยากร**: การจัดการวงจรชีวิตทรัพยากร Azure แบบเต็มรูปแบบ
- **🔐 การตรวจสอบตัวตน**: รูปแบบ DefaultAzureCredential และ managed identity
- **📦 บริการเก็บข้อมูล**: การดำเนินงาน Blob Storage, Queue Storage และ Table Storage
- **🚀 บริการคอนเทนเนอร์**: การจัดการ Azure Container Apps, Container Instances และ AKS
- **และตัวเชื่อมต่อพิเศษอื่นๆ อีกมากมาย**

**กรณีการใช้งานจริง**: "แสดงรายการบัญชีเก็บข้อมูล Azure ของฉัน", "ค้นหาใน Log Analytics workspace ของฉันเพื่อดูข้อผิดพลาดในชั่วโมงที่ผ่านมา" หรือ "ช่วยฉันสร้างแอป Azure โดยใช้ Node.js พร้อมการตรวจสอบตัวตนอย่างถูกต้อง"

**สถานการณ์สาธิตเต็มรูปแบบ**: นี่คือการสาธิตแบบสมบูรณ์ที่แสดงพลังของการรวม Azure MCP กับส่วนขยาย GitHub Copilot for Azure ใน VS Code เมื่อคุณติดตั้งทั้งสองและส่งคำสั่งว่า:

> "สร้างสคริปต์ Python ที่อัปโหลดไฟล์ไปยัง Azure Blob Storage ด้วยการตรวจสอบตัวตน DefaultAzureCredential สคริปต์ควรเชื่อมต่อกับบัญชีเก็บข้อมูล Azure ของฉันชื่อ 'mycompanystorage' อัปโหลดไปยังคอนเทนเนอร์ชื่อ 'documents' สร้างไฟล์ทดสอบที่มีเวลาปัจจุบันสำหรับอัปโหลด จัดการข้อผิดพลาดอย่างราบรื่นและแสดงผลลัพธ์ที่ให้ข้อมูล ปฏิบัติตามแนวปฏิบัติที่ดีที่สุดของ Azure สำหรับการตรวจสอบตัวตนและการจัดการข้อผิดพลาด รวมถึงคอมเมนต์อธิบายวิธีการทำงานของ DefaultAzureCredential และจัดโครงสร้างสคริปต์อย่างดีด้วยฟังก์ชันและเอกสารประกอบ"

Azure MCP Server จะสร้างสคริปต์ Python ที่พร้อมใช้งานจริงสำหรับการผลิตที่:
- ใช้ SDK Azure Blob Storage ล่าสุดพร้อมรูปแบบ async ที่ถูกต้อง
- ใช้ DefaultAzureCredential พร้อมคำอธิบายโซ่สำรองอย่างครบถ้วน
- มีการจัดการข้อผิดพลาดอย่างมั่นคงพร้อมประเภทข้อยกเว้นเฉพาะ Azure
- ปฏิบัติตามแนวปฏิบัติที่ดีที่สุดของ Azure SDK สำหรับการจัดการทรัพยากรและการเชื่อมต่อ
- ให้บันทึกข้อมูลโดยละเอียดและแสดงผลในคอนโซลอย่างให้ข้อมูล
- สร้างสคริปต์ที่มีโครงสร้างเหมาะสมรวมฟังก์ชัน เอกสาร และคำแนะนำประเภทข้อมูล

สิ่งที่น่าทึ่งคือหากไม่มี Azure MCP คุณอาจได้โค้ด blob storage ที่เหมือนกันซึ่งใช้งานได้แต่ไม่ปฏิบัติตามรูปแบบของ Azure ปัจจุบัน แต่ด้วย Azure MCP คุณจะได้โค้ดที่ใช้วิธีการตรวจสอบตัวตนล่าสุด จัดการกับสถานการณ์ข้อผิดพลาดเฉพาะ Azure และทำตามแนวทางที่ Microsoft แนะนำสำหรับแอปพลิเคชันในสภาพแวดล้อมจริง

**ตัวอย่างเด่น**: ฉันเคยลำบากกับการจำคำสั่งเฉพาะของ CLI `az` และ `azd` สำหรับใช้ชั่วคราว มันเป็นกระบวนการสองขั้นตอนสำหรับฉัน: ดูวากยสัมพันธ์ จากนั้นรันคำสั่ง ส่วนใหญ่มักจะเปิดพอร์ทัลแล้วคลิกทำงานเพราะไม่อยากยอมรับว่าจำวากยสัมพันธ์ CLI ไม่ได้ การที่สามารถบรรยายสิ่งที่ต้องการได้เพียงอย่างเดียวมันน่าทึ่งมาก และดียิ่งกว่าที่ทำแบบนั้นได้โดยไม่ต้องออกจาก IDE!

มีรายการกรณีใช้งานยอดเยี่ยมใน [คลัง Azure MCP](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) เพื่อช่วยเริ่มต้น สำหรับคำแนะนำการตั้งค่าโดยละเอียดและตัวเลือกการกำหนดค่าขั้นสูง ให้ดูที่ [เอกสาร Azure MCP อย่างเป็นทางการ](https://learn.microsoft.com/azure/developer/azure-mcp-server/)

> **ℹ️ หมายเหตุ: ประเภทของ 'Agents' ที่แตกต่างกัน**
> 
> อย่าสับสน GitHub MCP Server นี้กับ GitHub Coding Agent (เอเจนต์ AI ที่คุณสามารถมอบหมายปัญหาให้ทำงานเขียนโค้ดอัตโนมัติ) GitHub MCP Server ทำงานในโหมด Agent ของ VS Code เพื่อให้บริการการรวม API GitHub ขณะที่ GitHub Coding Agent เป็นฟีเจอร์แยกต่างหากที่สร้างคำขอดึงเมื่อถูกมอบหมายกับปัญหาใน GitHub

**ความสามารถหลักรวมถึง**:
- **⚙️ GitHub Actions**: การจัดการ pipeline CI/CD แบบเต็มรูปแบบ การตรวจสอบเวิร์กโฟลว์ และการจัดการงาน artifacts
- **🔀 คำขอดึง (Pull Requests)**: สร้าง ทบทวน ผสาน และจัดการ PR พร้อมติดตามสถานะอย่างครบถ้วน
- **🐛 ปัญหา (Issues)**: การจัดการวงจรชีวิตปัญหาแบบเต็ม รูปแบบคอมเมนต์ การติดป้าย และการมอบหมาย
- **🔒 ความปลอดภัย**: การแจ้งเตือนการสแกนโค้ด การค้นหาความลับ และการรวม Dependabot
- **🔔 การแจ้งเตือน**: การจัดการการแจ้งเตือนอย่างชาญฉลาดและการควบคุมการสมัครรับข้อมูลรีโพสิโทรี
- **📁 การจัดการรีโพสิโทรี**: การดำเนินการไฟล์ การจัดการสาขา และการบริหารรีโพสิโทรี
- **👥 การทำงานร่วมกัน**: การค้นหาผู้ใช้และองค์กร การจัดการทีม และการควบคุมการเข้าถึง

**กรณีการใช้งานจริง**: "สร้างคำขอดึงจากสาขาฟีเจอร์ของฉัน", "แสดงงาน CI ที่ล้มเหลวทั้งหมดในสัปดาห์นี้", "แสดงรายการการแจ้งเตือนความปลอดภัยที่เปิดอยู่สำหรับรีโพสิโทรีของฉัน" หรือ "ค้นหาปัญหาที่มอบหมายให้ฉันทั้งหมดในทุกองค์กรของฉัน"

**สถานการณ์สาธิตเต็มรูปแบบ**: นี่คือเวิร์กโฟลว์ทรงพลังที่แสดงความสามารถของ GitHub MCP Server:

> "ฉันต้องเตรียมตัวสำหรับการทบทวนสปรินต์ของเรา แสดงคำขอดึงทั้งหมดที่ฉันสร้างในสัปดาห์นี้ ตรวจสอบสถานะ pipeline CI/CD ของเรา สร้างสรุปการแจ้งเตือนความปลอดภัยที่ต้องจัดการ และช่วยฉันร่างบันทึกการปล่อยฟีเจอร์จาก PR ที่ผสานพร้อมป้าย 'feature'"

GitHub MCP Server จะ:
- สืบค้นคำขอดึงล่าสุดของคุณพร้อมข้อมูลสถานะโดยละเอียด
- วิเคราะห์การทำงานของเวิร์กโฟลว์และเน้นความล้มเหลวหรือปัญหาด้านประสิทธิภาพใดๆ
- รวบรวมผลลัพธ์การสแกนความปลอดภัยและจัดลำดับความสำคัญการแจ้งเตือนที่สำคัญ
- สร้างบันทึกการปล่อยอย่างครบถ้วนโดยดึงข้อมูลจาก PR ที่ผสานแล้ว
- ให้ขั้นตอนถัดไปที่นำไปปฏิบัติได้สำหรับการวางแผนสปรินต์และการเตรียมการปล่อย

**ตัวอย่างเด่น**: ฉันชอบใช้สำหรับเวิร์กโฟลว์รีวิวโค้ด แทนที่จะต้องสลับไปมาระหว่าง VS Code, การแจ้งเตือน GitHub และหน้าคำขอดึง ฉันก็เพียงแค่บอกว่า "แสดง PR ทั้งหมดที่รอการรีวิวของฉัน" แล้วตามด้วย "เพิ่มคอมเมนต์ใน PR #123 ถามเกี่ยวกับการจัดการข้อผิดพลาดในวิธีการตรวจสอบตัวตน" เซิร์ฟเวอร์จะจัดการเรียก API GitHub รักษาบริบทการสนทนา และช่วยฉันร่างคอมเมนต์รีวิวที่สร้างสรรค์ขึ้นได้อีกด้วย

**ตัวเลือกการตรวจสอบตัวตน**: เซิร์ฟเวอร์รองรับทั้ง OAuth (ใช้งานง่ายใน VS Code) และ Personal Access Tokens พร้อมชุดเครื่องมือที่สามารถกำหนดค่าเพื่อเปิดเฉพาะฟีเจอร์ GitHub ที่จำเป็น คุณสามารถรันเป็นบริการโฮสต์ระยะไกลสำหรับการตั้งค่าทันที หรือรันบนเครื่องผ่าน Docker เพื่อควบคุมอย่างเต็มที่

> **💡 เคล็ดลับมือโปร**
> 
> เปิดใช้เฉพาะชุดเครื่องมือที่ต้องการโดยกำหนดค่าพารามิเตอร์ `--toolsets` ในการตั้งค่าเซิร์ฟเวอร์ MCP ของคุณเพื่อลดขนาดบริบทและปรับปรุงการเลือกเครื่องมือ AI ตัวอย่างเช่น เพิ่ม `"--toolsets", "repos,issues,pull_requests,actions"` ใน args การตั้งค่า MCP สำหรับเวิร์กโฟลว์การพัฒนาหลัก หรือใช้ `"--toolsets", "notifications, security"` หากคุณเน้นไปที่ความสามารถการตรวจสอบ GitHub เป็นหลัก

### 5. 📝 MarkItDown MCP Server
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_MarkItDown_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_MarkItDown_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/markitdown)

**สิ่งที่ทำได้**: MarkItDown เป็นเซิร์ฟเวอร์แปลงเอกสารที่ครอบคลุมซึ่งแปลงไฟล์หลายรูปแบบเป็น Markdown คุณภาพสูง ปรับแต่งสำหรับการใช้งาน LLM และเวิร์กโฟลว์การวิเคราะห์ข้อความ

**ทำไมถึงมีประโยชน์**: จำเป็นสำหรับเวิร์กโฟลว์เอกสารสมัยใหม่! MarkItDown รองรับไฟล์หลากหลายประเภทพร้อมรักษาโครงสร้างเอกสารที่สำคัญ เช่น หัวเรื่อง รายการ ตาราง และลิงก์ ต่างจากเครื่องมือดึงข้อความธรรมดา ที่เน้นการรักษาความหมายเชิงสัญลักษณ์และรูปแบบที่มีคุณค่าแก่ทั้งการประมวลผล AI และการอ่านของมนุษย์

**รูปแบบไฟล์ที่รองรับ**:
- **เอกสารสำนักงาน**: PDF, PowerPoint (PPTX), Word (DOCX), Excel (XLSX/XLS)
- **ไฟล์สื่อ**: รูปภาพ (พร้อมข้อมูล EXIF และ OCR), เสียง (พร้อมข้อมูล EXIF และแปลงข้อความพูด)
- **เนื้อหาเว็บ**: HTML, ฟีด RSS, URL YouTube, หน้า Wikipedia
- **รูปแบบข้อมูล**: CSV, JSON, XML, ไฟล์ ZIP (ประมวลผลเนื้อหาแบบวนซ้ำ)
- **รูปแบบสิ่งพิมพ์**: EPub, Jupyter notebooks (.ipynb)
- **อีเมล**: ข้อความ Outlook (.msg)
- **ขั้นสูง**: การผสานรวม Azure Document Intelligence สำหรับการประมวลผล PDF ขั้นสูง

**ความสามารถขั้นสูง**: MarkItDown รองรับคำบรรยายภาพด้วย LLM (เมื่อใช้กับไคลเอนต์ OpenAI), Azure Document Intelligence สำหรับการประมวลผล PDF ขั้นสูง, การถอดเสียงไฟล์เสียง และระบบปลั๊กอินสำหรับเพิ่มรูปแบบไฟล์เพิ่มเติม

**การใช้งานจริง**: “แปลงงานนำเสนอ PowerPoint เป็น Markdown สำหรับเว็บไซต์เอกสารของเรา” “ดึงข้อความจาก PDF พร้อมรักษาโครงสร้างหัวเรื่องอย่างถูกต้อง” หรือ “แปลงสเปรดชีต Excel เป็นรูปแบบตารางที่อ่านได้”

**ตัวอย่างที่โดดเด่น**: เพื่อยกคำพูดจาก [เอกสาร MarkItDown](https://github.com/microsoft/markitdown#why-markdown):

> Markdown ใกล้เคียงกับข้อความธรรมดามาก มีเครื่องหมายหรือรูปแบบน้อย แต่ยังคงมีวิธีนำเสนอโครงสร้างเอกสารสำคัญ LLM ที่แพร่หลาย เช่น GPT-4o ของ OpenAI "พูด" Markdown โดยธรรมชาติและมักจะรวม Markdown ในการตอบสนองโดยไม่ได้รับคำสั่งเพิ่มเติม ซึ่งบ่งบอกว่าพวกเขาฝึกฝนด้วยข้อความที่จัดรูปแบบ Markdown จำนวนมากและเข้าใจมันได้ดี อีกทั้งข้อดีเพิ่มเติมคือ การใช้ Markdown มีประสิทธิภาพสูงในแง่การใช้โทเคน

MarkItDown มีความสามารถดีเยี่ยมในการรักษาโครงสร้างเอกสาร ซึ่งสำคัญสำหรับเวิร์กโฟลว์ AI เช่น เมื่อแปลงงานนำเสนอ PowerPoint จะคงการจัดสไลด์ด้วยหัวเรื่องที่ถูกต้อง ดึงตารางให้อยู่ในรูปแบบตาราง Markdown ใส่ข้อความแทนภาพ รวมถึงประมวลผลบันทึกของผู้บรรยาย แผนภูมิจะถูกแปลงเป็นตารางข้อมูลที่อ่านได้ และ Markdown ที่ได้จะรักษาเนื้อหาตรรกะตามงานนำเสนอเดิม เหมาะอย่างยิ่งสำหรับการป้อนเนื้อหางานนำเสนอเข้าสู่ระบบ AI หรือสร้างเอกสารจากสไลด์ที่มีอยู่แล้ว

**ตัวอย่างที่โดดเด่น**: เมื่อตั้งค่าการเชื่อมต่อฐานข้อมูลเสร็จแล้ว คุณสามารถเริ่มสนทนากับข้อมูลได้ทันที บล็อกโพสต์นี้แสดงตัวอย่างด้วยคำถามง่ายๆ: “คุณเชื่อมต่อกับฐานข้อมูลไหน?” เซิร์ฟเวอร์ MCP ตอบโดยเรียกใช้เครื่องมือฐานข้อมูลที่เหมาะสม เชื่อมต่อกับอินสแตนซ์ SQL Server ของคุณ และส่งคืนรายละเอียดเกี่ยวกับการเชื่อมต่อฐานข้อมูลปัจจุบัน ทั้งหมดโดยไม่ต้องเขียน SQL เลย เซิร์ฟเวอร์รองรับการดำเนินการฐานข้อมูลอย่างครบถ้วนตั้งแต่การจัดการสคีมาไปจนถึงการจัดการข้อมูล ผ่านคำสั่งภาษาธรรมชาติ สำหรับคำแนะนำการตั้งค่าและตัวอย่างร่วมกับ VS Code และ Claude Desktop ดูที่: [Introducing MSSQL MCP Server (Preview)](https://devblogs.microsoft.com/azure-sql/introducing-mssql-mcp-server/).

**สิ่งที่ทำได้**: ให้เครื่องมือจำเป็นสำหรับนักพัฒนาในการสร้าง AI agents และแอปพลิเคชันที่รวมกับ Microsoft 365 และ Microsoft 365 Copilot รวมถึงการตรวจสอบสคีมา ดึงตัวอย่างโค้ด และช่วยแก้ไขปัญหา

**เหตุผลที่เป็นประโยชน์**: การพัฒนาสำหรับ Microsoft 365 และ Copilot มีสคีมา Manifest ที่ซับซ้อนและรูปแบบการพัฒนาเฉพาะ เซิร์ฟเวอร์ MCP นี้นำทรัพยากรการพัฒนาที่สำคัญเข้าสู่สภาพแวดล้อมการเขียนโค้ดของคุณ ช่วยตรวจสอบสคีมา ค้นหาตัวอย่างโค้ด และแก้ไขปัญหาที่พบบ่อยโดยไม่ต้องอ้างอิงเอกสารอย่างต่อเนื่อง

**การใช้งานจริง**: "ตรวจสอบ manifest ของเอเจนต์แบบ declarative และแก้ไขข้อผิดพลาดสคีมา", "แสดงตัวอย่างโค้ดสำหรับการใช้งาน Microsoft Graph API plugin" หรือ "ช่วยแก้ปัญหาการยืนยันตัวตนของแอป Teams ของฉัน"

**ตัวอย่างเด่น**: ฉันติดต่อเพื่อน John Miller หลังจากคุยกับเขาที่งาน Build เรื่อง M365 Agents และเขาแนะนำ MCP นี้ อาจเป็นประโยชน์สำหรับนักพัฒนาที่ใหม่กับ M365 Agents เพราะมีเทมเพลต ตัวอย่างโค้ด และโครงสร้างพื้นฐานให้เริ่มต้นโดยไม่ต้องจมกับเอกสาร ฟีเจอร์ตรวจสอบสคีมาดูน่าสนใจมากเพื่อหลีกเลี่ยงข้อผิดพลาดโครงสร้าง manifest ที่ใช้เวลาหลายชั่วโมงในการดีบัก

## 📋 บทสรุป

Model Context Protocol (MCP) กำลังเปลี่ยนวิธีที่นักพัฒนาปฏิสัมพันธ์กับผู้ช่วย AI และเครื่องมือภายนอก เซิร์ฟเวอร์ Microsoft MCP ทั้ง 10 ตัวนี้แสดงให้เห็นถึงพลังของการรวม AI แบบมาตรฐาน ทำให้เวิร์กโฟลว์เป็นไปอย่างไร้รอยต่อ ช่วยให้นักพัฒนายังคงอยู่ในสภาพสมาธิพร้อมเข้าถึงความสามารถภายนอกที่ทรงพลัง

ตั้งแต่การรวมระบบนิเวศ Azure อย่างครบถ้วนไปจนถึงเครื่องมือเฉพาะอย่าง Playwright สำหรับอัตโนมัติบนเว็บเบราว์เซอร์ และ MarkItDown สำหรับประมวลเอกสาร เซิร์ฟเวอร์เหล่านี้แสดงให้เห็นว่า MCP ช่วยเพิ่มผลิตภาพในสถานการณ์การพัฒนาที่หลากหลายอย่างไร โปรโตคอลมาตรฐานทำให้เครื่องมือเหล่านี้ทำงานร่วมกันได้อย่างราบรื่น สร้างประสบการณ์การพัฒนาที่กลมกลืน

ในขณะที่ระบบนิเวศ MCP ยังคงพัฒนาอย่างต่อเนื่อง การมีส่วนร่วมกับชุมชน การสำรวจเซิร์ฟเวอร์ใหม่ๆ และการสร้างโซลูชันเฉพาะจะเป็นกุญแจสำคัญในการเพิ่มประสิทธิภาพการพัฒนาของคุณ ธรรมชาติแบบเปิดของ MCP หมายความว่าคุณสามารถผสมผสานเครื่องมือจากผู้ขายต่างๆ เพื่อสร้างเวิร์กโฟลว์ที่สมบูรณ์แบบสำหรับความต้องการเฉพาะของคุณ

## 🎯 แบบฝึกหัด

1. **ติดตั้งและตั้งค่า**: ตั้งค่า MCP server ตัวใดตัวหนึ่งในสภาพแวดล้อม VS Code ของคุณและทดสอบฟังก์ชันพื้นฐาน
2. **ผสานเวิร์กโฟลว์**: ออกแบบเวิร์กโฟลว์การพัฒนาที่รวม MCP server อย่างน้อยสามตัว
3. **วางแผนเซิร์ฟเวอร์เฉพาะ**: ระบุงานในกิจวัตรประจำวันของคุณที่อาจได้รับประโยชน์จาก MCP server แบบกำหนดเอง และสร้างข้อกำหนดให้กับมัน
4. **วิเคราะห์ประสิทธิภาพ**: เปรียบเทียบประสิทธิภาพการใช้ MCP server กับวิธีดั้งเดิมสำหรับงานพัฒนาทั่วไป
5. **ประเมินความปลอดภัย**: ประเมินผลกระทบด้านความปลอดภัยของการใช้ MCP server ในสภาพแวดล้อมการพัฒนาของคุณและเสนอแนวปฏิบัติที่ดีที่สุด
