# AGENTS.md

## Επισκόπηση Έργου

**MCP για Αρχάριους** είναι ένα ανοικτού κώδικα εκπαιδευτικό πρόγραμμα για την εκμάθηση του Model Context Protocol (MCP) - ένα τυποποιημένο πλαίσιο για αλληλεπιδράσεις μεταξύ μοντέλων AI και πελατειακών εφαρμογών. Αυτό το αποθετήριο παρέχει ολοκληρωμένο εκπαιδευτικό υλικό με πρακτικά παραδείγματα κώδικα σε πολλές γλώσσες προγραμματισμού.

### Κύριες Τεχνολογίες

- **Γλώσσες Προγραμματισμού**: C#, Java, JavaScript, TypeScript, Python, Rust
- **Πλαίσια & SDKs**: 
  - MCP SDK (`@modelcontextprotocol/sdk`)
  - Spring Boot (Java)
  - FastMCP (Python)
  - LangChain4j (Java)
- **Βάσεις Δεδομένων**: PostgreSQL με επέκταση pgvector
- **Πλατφόρμες Cloud**: Azure (Container Apps, OpenAI, Content Safety, Application Insights)
- **Εργαλεία Ανάπτυξης**: npm, Maven, pip, Cargo
- **Τεκμηρίωση**: Markdown με αυτοματοποιημένη μετάφραση σε πολλές γλώσσες (48+ γλώσσες)

### Αρχιτεκτονική

- **11 Βασικές Μονάδες (00-11)**: Ακολουθιακή διαδρομή εκμάθησης από τα θεμελιώδη μέχρι προχωρημένα θέματα
- **Πρακτικά Εργαστήρια**: Ασκήσεις με ολοκληρωμένο κώδικα λύσης σε πολλές γλώσσες
- **Δείγματα Έργων**: Λειτουργικές υλοποιήσεις MCP server και client
- **Σύστημα Μετάφρασης**: Αυτοματοποιημένη ροή εργασίας GitHub Actions για υποστήριξη πολλαπλών γλωσσών
- **Αρχεία Εικόνων**: Κεντρικός φάκελος εικόνων με μεταφρασμένες εκδόσεις

## Εντολές Ρύθμισης

Αυτό είναι ένα αποθετήριο με επίκεντρο την τεκμηρίωση. Η περισσότερη ρύθμιση γίνεται μέσα σε μεμονωμένα δείγματα έργων και εργαστήρια.

### Ρύθμιση Αποθετηρίου

```bash
# Κλωνοποιήστε το αποθετήριο
git clone https://github.com/microsoft/mcp-for-beginners.git
cd mcp-for-beginners
```

### Εργασία με Δείγματα Έργων

Τα δείγματα έργων βρίσκονται στις εξής θέσεις:
- `03-GettingStarted/samples/` - Παραδείγματα συγκεκριμένα για κάθε γλώσσα
- `03-GettingStarted/01-first-server/solution/` - Πρώτες υλοποιήσεις server
- `03-GettingStarted/02-client/solution/` - Υλοποιήσεις client
- `11-MCPServerHandsOnLabs/` - Ολοκληρωμένα εργαστήρια ενσωμάτωσης βάσης δεδομένων

Κάθε δείγμα έργου περιέχει τις δικές του οδηγίες ρύθμισης:

#### Έργα TypeScript/JavaScript
```bash
cd <project-directory>
npm install
npm start
```

#### Έργα Python
```bash
cd <project-directory>
pip install -r requirements.txt
# ή
pip install -e .
python main.py
```

#### Έργα Java
```bash
cd <project-directory>
mvn clean install
mvn spring-boot:run
```

## Ροή Εργασίας Ανάπτυξης

### Δομή Τεκμηρίωσης

- **Μονάδες 00-11**: Βασικό περιεχόμενο προγράμματος με αλληλουχία
- **translations/**: Εκδόσεις ανά γλώσσα (αυτοπαραγόμενες, μη επεξεργαστείτε απευθείας)
- **translated_images/**: Τοπικοποιημένες εκδόσεις εικόνων (αυτοπαραγόμενες)
- **images/**: Πρωτότυπες εικόνες και διαγράμματα

### Δημιουργία Αλλαγών στην Τεκμηρίωση

1. Επεξεργαστείτε μόνο τα αγγλικά αρχεία markdown στους ριζικούς καταλόγους μονάδων (00-11)
2. Ενημερώστε εικόνες στον φάκελο `images/` αν χρειάζεται
3. Η ενέργεια co-op-translator στο GitHub δημιουργεί αυτόματα τις μεταφράσεις
4. Οι μεταφράσεις ανανεώνονται με κάθε push στον κλάδο main

### Εργασία με τις Μεταφράσεις

- **Αυτοματοποιημένη Μετάφραση**: Η ροή εργασίας GitHub Actions διαχειρίζεται όλες τις μεταφράσεις
- **Μην επεξεργάζεστε χειροκίνητα** τα αρχεία στο φάκελο `translations/`
- Τα μεταδεδομένα μετάφρασης είναι ενσωματωμένα σε κάθε μεταφρασμένο αρχείο
- Υποστηριζόμενες γλώσσες: 48+ γλώσσες, συμπεριλαμβανομένων των Αραβικών, Κινέζικων, Γαλλικών, Γερμανικών, Χίντι, Ιαπωνικών, Κορεάτικων, Πορτογαλικών, Ρωσικών, Ισπανικών και άλλων

## Οδηγίες Στυλ Κώδικα

### Στυλ Τεκμηρίωσης

- Χρησιμοποιήστε σαφή, φιλική προς αρχάριους γλώσσα
- Συμπεριλάβετε παραδείγματα κώδικα σε πολλές γλώσσες όπου είναι απαραίτητο
- Ακολουθήστε τις βέλτιστες πρακτικές markdown:
  - Χρησιμοποιήστε επικεφαλίδες ATX (`#` σύνταξη)
  - Χρησιμοποιήστε fenced code blocks με αναγνωριστικό γλώσσας
  - Περιλάβετε περιγραφικό alt κείμενο για εικόνες
  - Κρατήστε ορθό το μήκος γραμμής (χωρίς αυστηρό όριο, αλλά με λογική)

### Στυλ Δείγματος Κώδικα

#### TypeScript/JavaScript
- Χρησιμοποιήστε ES modules (`import`/`export`)
- Ακολουθήστε τις αυστηρές πρακτικές TypeScript
- Συμπεριλάβετε δηλώσεις τύπων
- Στοχεύστε ES2022

#### Python
- Ακολουθήστε τις οδηγίες στυλ PEP 8
- Χρησιμοποιήστε τύπους όπου απαιτείται
- Συμπεριλάβετε docstrings για συναρτήσεις και κλάσεις
- Χρησιμοποιήστε σύγχρονες δυνατότητες Python (3.8+)

#### Java
- Ακολουθήστε τις συμβάσεις Spring Boot
- Χρησιμοποιήστε δυνατότητες Java 21
- Ακολουθήστε την τυπική δομή Maven
- Συμπεριλάβετε σχόλια Javadoc

## Κατασκευή και Ανάπτυξη

### Ανάπτυξη Τεκμηρίωσης

Το αποθετήριο χρησιμοποιεί GitHub Pages ή παρόμοιο για τη φιλοξενία της τεκμηρίωσης (αν ισχύει). Οι αλλαγές στον κλάδο main ενεργοποιούν:

Αυτό το αποθετήριο περιέχει κυρίως τεκμηρίωση markdown. Δεν απαιτείται βήμα μεταγλώττισης ή κατασκευής για το βασικό περιεχόμενο προγράμματος.

Τα μεμονωμένα δείγματα έργων μπορεί να έχουν οδηγίες ανάπτυξης:

## Οδηγίες Συνεισφοράς

### Διαδικασία Pull Request

1. **Fork και Clone**: Κάντε fork το αποθετήριο και κατεβάστε το τοπικά
2. **Δημιουργία branch**: Χρησιμοποιήστε περιγραφικά ονόματα branch (π.χ. `fix/typo-module-3`, `add/python-example`)
3. **Κάντε αλλαγές**: Επεξεργαστείτε μόνο τα αγγλικά αρχεία markdown (όχι μεταφράσεις)
4. **Δοκιμάστε τοπικά**: Βεβαιωθείτε ότι τα markdown αποδίδονται σωστά
5. **Υποβάλετε PR**: Χρησιμοποιήστε ξεκάθαρους τίτλους και περιγραφές PR
6. **CLA**: Υπογράψτε τη Microsoft Contributor License Agreement όταν ζητηθεί

### Μορφή Τίτλου PR

Χρησιμοποιήστε καθαρούς, περιγραφικούς τίτλους:
- `[Module XX] Σύντομη περιγραφή` για αλλαγές σε μονάδες
- `[Samples] Περιγραφή` για αλλαγές σε δείγματα κώδικα
- `[Docs] Περιγραφή` για γενικές ενημερώσεις τεκμηρίωσης

### Τι να Συνεισφέρετε

- Διορθώσεις σφαλμάτων στην τεκμηρίωση ή παραδείγματα κώδικα
- Νέα παραδείγματα κώδικα σε επιπλέον γλώσσες
- Διευκρινίσεις και βελτιώσεις στο υπάρχον περιεχόμενο
- Νέα μελέτες περίπτωσης ή πρακτικά παραδείγματα
- Αναφορές θεμάτων για ασαφές ή λανθασμένο περιεχόμενο

### Τι να ΜΗΝ Κάνετε

- Μην επεξεργάζεστε απευθείας αρχεία στο φάκελο `translations/`
- Μην επεξεργάζεστε το φάκελο `translated_images/`
- Μην προσθέτετε μεγάλα δυαδικά αρχεία χωρίς συνεννόηση
- Μην αλλάζετε αρχεία ροής εργασίας μετάφρασης χωρίς συντονισμό
