# Σενάριο 3: Έγγραφα μέσα στον Επεξεργαστή με MCP Server στο VS Code

## Επισκόπηση

Σε αυτό το σενάριο, θα μάθετε πώς να φέρνετε τα Microsoft Learn Docs απευθείας στο περιβάλλον σας του Visual Studio Code χρησιμοποιώντας τον MCP server. Αντί να αλλάζετε συνεχώς καρτέλες στον περιηγητή για να αναζητήσετε τεκμηρίωση, μπορείτε να έχετε πρόσβαση, να ψάχνετε και να αναφέρετε τα επίσημα έγγραφα μέσα στον επεξεργαστή σας. Αυτή η προσέγγιση απλοποιεί τη ροή εργασίας σας, διατηρεί την εστίασή σας και επιτρέπει απρόσκοπτη ενσωμάτωση με εργαλεία όπως το GitHub Copilot.

- Αναζητήστε και διαβάστε έγγραφα μέσα στο VS Code χωρίς να φύγετε από το περιβάλλον κωδικοποίησής σας.
- Αναφερθείτε σε τεκμηρίωση και εισάγετε συνδέσμους απευθείας στα README ή στα αρχεία των μαθημάτων σας.
- Χρησιμοποιήστε το GitHub Copilot και τον MCP μαζί για μια ομαλή ροή εργασίας τεκμηρίωσης με υποστήριξη AI.

## Στόχοι Μάθησης

Στο τέλος αυτού του κεφαλαίου, θα κατανοείτε πώς να ρυθμίσετε και να χρησιμοποιήσετε τον MCP server μέσα στο VS Code για να βελτιώσετε τη ροή εργασίας σας στην τεκμηρίωση και ανάπτυξη. Θα μπορείτε να:

- Ρυθμίσετε τον χώρο εργασίας σας για να χρησιμοποιεί τον MCP server για αναζήτηση τεκμηρίωσης.
- Αναζητάτε και να εισάγετε τεκμηρίωση απευθείας από το VS Code.
- Συνδυάσετε τη δύναμη του GitHub Copilot και του MCP για μια πιο παραγωγική ροή εργασίας με υποστήριξη AI.

Αυτές οι δεξιότητες θα σας βοηθήσουν να παραμείνετε συγκεντρωμένοι, να βελτιώσετε την ποιότητα της τεκμηρίωσης και να αυξήσετε την παραγωγικότητά σας ως προγραμματιστής ή τεχνικός συγγραφέας.

## Λύση

Για να επιτύχετε πρόσβαση στην τεκμηρίωση μέσα στον επεξεργαστή, θα ακολουθήσετε μια σειρά βημάτων που ενσωματώνουν τον MCP server με το VS Code και το GitHub Copilot. Αυτή η λύση είναι ιδανική για συγγραφείς μαθημάτων, συγγραφείς τεκμηρίωσης και προγραμματιστές που θέλουν να διατηρήσουν την εστίασή τους στον επεξεργαστή ενώ εργάζονται με έγγραφα και Copilot.

- Προσθέστε γρήγορα συνδέσμους αναφοράς σε ένα README ενώ γράφετε τεκμηρίωση μαθήματος ή έργου.
- Χρησιμοποιήστε το Copilot για να δημιουργήσετε κώδικα και τον MCP για να βρείτε και να επικαλεστείτε άμεσα συναφή έγγραφα.
- Παραμείνετε συγκεντρωμένοι στον επεξεργαστή σας και αυξήστε την παραγωγικότητα.

### Οδηγός Βήμα-Βήμα

Για να ξεκινήσετε, ακολουθήστε αυτά τα βήματα. Για κάθε βήμα, μπορείτε να προσθέσετε ένα στιγμιότυπο οθόνης από τον φάκελο assets για να εικονογραφήσετε οπτικά τη διαδικασία.

1. **Προσθέστε τη ρύθμιση MCP:**
   Στη ρίζα του έργου σας, δημιουργήστε ένα αρχείο `.vscode/mcp.json` και προσθέστε την ακόλουθη ρύθμιση:
   ```json
   {
     "servers": {
       "LearnDocsMCP": {
         "url": "https://learn.microsoft.com/api/mcp"
       }
     }
   }
   ```
   Αυτή η ρύθμιση λέει στο VS Code πώς να συνδεθεί με τον [`Microsoft Learn Docs MCP server`](https://github.com/MicrosoftDocs/mcp).
   
   ![Step 1: Add mcp.json to .vscode folder](../../../../../../translated_images/el/step1-mcp-json.c06a007fccc3edfa.webp)
    
2. **Ανοίξτε τον πίνακα GitHub Copilot Chat:**
   Αν δεν έχετε ήδη εγκαταστήσει την επέκταση GitHub Copilot, μεταβείτε στην προβολή Επεκτάσεων στο VS Code και εγκαταστήστε την. Μπορείτε να την κατεβάσετε απευθείας από το [Visual Studio Code Marketplace](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot-chat). Στη συνέχεια, ανοίξτε τον πίνακα Copilot Chat από το πλευρικό μενού.

   ![Step 2: Open Copilot Chat panel](../../../../../../translated_images/el/step2-copilot-panel.f1cc86e9b9b8cd1a.webp)

3. **Ενεργοποιήστε τη λειτουργία agent και επαληθεύστε τα εργαλεία:**
   Στον πίνακα Copilot Chat, ενεργοποιήστε τη λειτουργία agent.

   ![Step 3: Enable agent mode and verify tools](../../../../../../translated_images/el/step3-agent-mode.cdc32520fd7dd1d1.webp)

   Μετά την ενεργοποίηση της λειτουργίας agent, επαληθεύστε ότι ο MCP server εμφανίζεται ως ένα από τα διαθέσιμα εργαλεία. Αυτό εξασφαλίζει ότι ο agent του Copilot μπορεί να έχει πρόσβαση στον server τεκμηρίωσης για να ανακτήσει σχετικές πληροφορίες.
   
   ![Step 3: Verify MCP server tool](../../../../../../translated_images/el/step3-verify-mcp-tool.76096a6329cbfecd.webp)

4. **Ξεκινήστε μια νέα συνομιλία και προτροπήστε τον agent:**
   Ανοίξτε μια νέα συνομιλία στον πίνακα Copilot Chat. Τώρα, μπορείτε να προτροπήσετε τον agent με τις ερωτήσεις τεκμηρίωσης σας. Ο agent θα χρησιμοποιήσει τον MCP server για να αναζητήσει και να εμφανίσει σχετική τεκμηρίωση Microsoft Learn απευθείας στον επεξεργαστή σας.

   - *"Προσπαθώ να γράψω ένα σχέδιο μελέτης για το θέμα X. Θα το μελετήσω για 8 εβδομάδες, για κάθε εβδομάδα προτείνε περιεχόμενο που θα πρέπει να καλύψω."*

   ![Step 4: Prompt the agent in chat](../../../../../../translated_images/el/step4-prompt-chat.12187bb001605efc.webp)

5. **Ζωντανές Ερωτήσεις:**

   > Ας πάρουμε μια ζωντανή ερώτηση από την ενότητα [#get-help](https://discord.gg/D6cRhjHWSC) στο Microsoft Foundry Discord ([δείτε το αρχικό μήνυμα](https://discord.com/channels/1113626258182504448/1385498306720829572)):
   
   *"Ψάχνω απαντήσεις για το πώς να αναπτύξω μια λύση πολλαπλών agents με agents AI που αναπτύχθηκαν στο Azure AI Foundry. Βλέπω ότι δεν υπάρχει άμεσος τρόπος ανάπτυξης, όπως κανάλια Copilot Studio. Ποιες είναι οι διαφορετικές μέθοδοι για αυτή την ανάπτυξη ώστε οι εταιρικοί χρήστες να μπορούν να αλληλεπιδρούν και να ολοκληρώσουν τη δουλειά; Υπάρχουν πολλά άρθρα/ιστολόγια που λένε ότι μπορούμε να χρησιμοποιήσουμε το Azure Bot service για αυτή τη δουλειά το οποίο μπορεί να λειτουργεί ως γέφυρα ανάμεσα σε MS teams και Azure AI Foundry Agents, θα δουλέψει αυτό αν ρυθμίσω ένα Azure bot που συνδέεται με τον Orchestrator Agent στο Azure AI Foundry μέσω λειτουργίας Azure για την εκτέλεση της ορχήστρωσης ή χρειάζεται να δημιουργήσω Azure function για κάθε AI agent που αποτελεί μέρος της λύσης πολλαπλών agents για να γίνει η ορχήστρωση στο Bot framework; Οποιαδήποτε άλλη πρόταση είναι ευπρόσδεκτη."*

   ![Step 5: Live queries](../../../../../../translated_images/el/step5-live-queries.49db3e4a50bea273.webp)

   Ο agent θα απαντήσει με σχετικούς συνδέσμους τεκμηρίωσης και συνοπτικές περιλήψεις, τις οποίες μπορείτε στη συνέχεια να εισάγετε απευθείας στα αρχεία markdown σας ή να χρησιμοποιήσετε ως αναφορές στον κώδικά σας.
