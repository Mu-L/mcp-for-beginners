# Εισαγωγή στην Ενσωμάτωση Βάσης Δεδομένων MCP

## 🎯 Τι καλύπτει αυτό το εργαστήριο

Αυτό το εισαγωγικό εργαστήριο παρέχει μια ολοκληρωμένη επισκόπηση για την κατασκευή διακομιστών Model Context Protocol (MCP) με ενσωμάτωση βάσης δεδομένων. Θα κατανοήσετε την επιχειρηματική υπόθεση, την τεχνική αρχιτεκτονική και τις πραγματικές εφαρμογές μέσω της περίπτωσης χρήσης ανάλυσης λιανικής πώλησης Zava Retail στο https://github.com/microsoft/MCP-Server-and-PostgreSQL-Sample-Retail.

## Επισκόπηση

**Το Model Context Protocol (MCP)** επιτρέπει στους βοηθούς AI να έχουν ασφαλή πρόσβαση και αλληλεπίδραση με εξωτερικές πηγές δεδομένων σε πραγματικό χρόνο. Όταν συνδυάζεται με ενσωμάτωση βάσης δεδομένων, το MCP ξεκλειδώνει ισχυρές δυνατότητες για εφαρμογές AI που βασίζονται σε δεδομένα.

Αυτή η διαδρομή μάθησης σας διδάσκει πώς να δημιουργείτε διακομιστές MCP έτοιμους για παραγωγή που συνδέουν βοηθούς AI με δεδομένα πωλήσεων λιανικής μέσω PostgreSQL, υλοποιώντας επιχειρηματικά πρότυπα όπως η Ασφάλεια Επιπέδου Γραμμής (Row Level Security), η εννοιολογική αναζήτηση και η πρόσβαση σε δεδομένα πολλαπλών ενοικιαστών.

## 🧭 Η πρόκληση: Η AI συναντά δεδομένα του πραγματικού κόσμου

### Παραδοσιακοί περιορισμοί της AI

Οι σύγχρονοι βοηθοί AI είναι εξαιρετικά ισχυροί αλλά αντιμετωπίζουν σημαντικούς περιορισμούς όταν εργάζονται με πραγματικά επιχειρηματικά δεδομένα:

| **Πρόκληση** | **Περιγραφή** | **Επιχειρηματικός αντίκτυπος** |
|---------------|-----------------|-------------------|
| **Στατική γνώση** | Τα μοντέλα AI εκπαιδεύονται σε σταθερά σύνολα δεδομένων και δεν έχουν πρόσβαση σε τρέχοντα επιχειρηματικά δεδομένα | Καθυστερημένες πληροφορίες, χαμένες ευκαιρίες |
| **Απομονωμένες πηγές δεδομένων** | Πληροφορίες κλειδωμένες σε βάσεις δεδομένων, APIs και συστήματα μη προσβάσιμα από την AI | Ατελής ανάλυση, κατακερματισμένες ροές εργασίας |
| **Περιορισμοί ασφάλειας** | Η άμεση πρόσβαση στη βάση δεδομένων εγείρει ζητήματα ασφάλειας και συμμόρφωσης | Περιορισμένη ανάπτυξη, χειροκίνητη προετοιμασία δεδομένων |
| **Σύνθετα ερωτήματα** | Οι επιχειρηματικοί χρήστες χρειάζονται τεχνικές γνώσεις για να εξαγάγουν δεδομένα | Μειωμένη υιοθέτηση, αναποτελεσματικές διαδικασίες |

### Η λύση MCP

Το Model Context Protocol αντιμετωπίζει αυτές τις προκλήσεις παρέχοντας:

- **Πρόσβαση δεδομένων σε πραγματικό χρόνο**: Οι βοηθοί AI εκτελούν ερωτήματα σε ζωντανές βάσεις δεδομένων και APIs
- **Ασφαλή ενσωμάτωση**: Ελεγχόμενη πρόσβαση με αυθεντικοποίηση και δικαιώματα
- **Φυσική γλώσσα**: Οι επιχειρηματικοί χρήστες κάνουν ερωτήσεις σε απλή αγγλική γλώσσα
- **Τυποποιημένο πρωτόκολλο**: Λειτουργεί σε διαφορετικές πλατφόρμες και εργαλεία AI

## 🏪 Γνωρίστε το Zava Retail: Η μαθησιακή μας περίπτωση https://github.com/microsoft/MCP-Server-and-PostgreSQL-Sample-Retail

Σε όλη αυτή τη διαδρομή μάθησης, θα δημιουργήσουμε έναν διακομιστή MCP για το **Zava Retail**, μια φανταστική αλυσίδα λιανικής αυτοεξυπηρέτησης με πολλαπλές καταστήματα. Το ρεαλιστικό αυτό σενάριο παρουσιάζει μια υλοποίηση MCP επιπέδου επιχείρησης.

### Επιχειρηματικό πλαίσιο

Το **Zava Retail** λειτουργεί:
- **8 φυσικά καταστήματα** σε όλη την πολιτεία της Ουάσιγκτον (Seattle, Bellevue, Tacoma, Spokane, Everett, Redmond, Kirkland)
- **1 ηλεκτρονικό κατάστημα** για πωλήσεις ηλεκτρονικού εμπορίου
- **Ποικιλόμορφο κατάλογο προϊόντων** που περιλαμβάνει εργαλεία, υλικά, εφόδια κήπου και οικοδομικά υλικά
- **Πολυεπίπεδη διοίκηση** με διευθυντές καταστημάτων, περιφερειακούς διευθυντές και στελέχη

### Επιχειρηματικές απαιτήσεις

Οι διευθυντές καταστημάτων και τα στελέχη χρειάζονται αναλύσεις με τη βοήθεια AI για:

1. **Ανάλυση απόδοσης πωλήσεων** ανά καταστήματα και χρονικές περιόδους
2. **Παρακολούθηση επιπέδων αποθεμάτων** και εντοπισμός αναγκών ανανέωσης
3. **Κατανόηση συμπεριφοράς πελατών** και αγοραστικών προτύπων
4. **Ανακάλυψη πληροφοριών για προϊόντα** μέσω εννοιολογικής αναζήτησης
5. **Δημιουργία αναφορών** με ερωτήματα σε φυσική γλώσσα
6. **Διατήρηση ασφάλειας δεδομένων** με έλεγχο πρόσβασης βάσει ρόλων

### Τεχνικές απαιτήσεις

Ο διακομιστής MCP πρέπει να παρέχει:

- **Πρόσβαση σε δεδομένα πολλαπλών ενοικιαστών** όπου οι διευθυντές καταστημάτων βλέπουν μόνο τα δεδομένα του καταστήματός τους
- **Ευέλικτα ερωτήματα** που υποστηρίζουν σύνθετες λειτουργίες SQL
- **Εννοιολογική αναζήτηση** για ανακάλυψη προϊόντων και προτάσεις
- **Δεδομένα σε πραγματικό χρόνο** που αντανακλούν την τρέχουσα επιχειρηματική κατάσταση
- **Ασφαλή αυθεντικοποίηση** με ασφάλεια επιπέδου γραμμής
- **Κλιμακωτή αρχιτεκτονική** που υποστηρίζει πολλούς ταυτόχρονους χρήστες

## 🔒 Βαθιά ματιά στην ασφάλεια και την υποστήριξη πολλαπλών ενοικιαστών

Η υλοποίησή μας δίνει προτεραιότητα στην ασφάλεια επιπέδου επιχείρησης:

### Ασφάλεια Επιπέδου Γραμμής (RLS)

Το PostgreSQL RLS εξασφαλίζει απομόνωση δεδομένων:

```sql
-- Store managers see only their store's data
CREATE POLICY store_manager_policy ON retail.orders
  FOR ALL TO store_managers
  USING (store_id = get_current_user_store());

-- Regional managers see multiple stores
CREATE POLICY regional_manager_policy ON retail.orders
  FOR ALL TO regional_managers
  USING (store_id = ANY(get_user_store_list()));
```

### Διαχείριση ταυτότητας χρήστη

Κάθε σύνδεση MCP περιλαμβάνει:
- **ID διευθυντή καταστήματος**: Μοναδικός αναγνωριστικός για το πλαίσιο RLS
- **Ανάθεση ρόλου**: Δικαιώματα και επίπεδα πρόσβασης
- **Διαχείριση συνεδρίας**: Ασφαλείς διακριτές αυθεντικοποιήσεις
- **Καταγραφή ελέγχου**: Πλήρες ιστορικό πρόσβασης

### Προστασία δεδομένων

Πολλαπλά επίπεδα ασφάλειας:
- **Κρυπτογράφηση σύνδεσης**: TLS για όλες τις συνδέσεις βάσης δεδομένων
- **Πρόληψη SQL Injection**: Μόνο παραμετροποιημένα ερωτήματα
- **Επικύρωση εισόδου**: Πλήρης επικύρωση αιτημάτων
- **Διαχείριση σφαλμάτων**: Χωρίς ευαίσθητα δεδομένα σε μηνύματα λάθους
