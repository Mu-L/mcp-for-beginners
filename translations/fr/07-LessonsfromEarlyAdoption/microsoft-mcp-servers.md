## 🔧 Comprendre les serveurs MCP : un guide pour débutants

### Qu’est-ce qu’un serveur MCP ?

En tant que débutant au Model Context Protocol (MCP), vous vous demandez peut-être : « Qu’est-ce exactement qu’un serveur MCP, et pourquoi devrais-je m’y intéresser ? » Commençons par une analogie simple.

Considérez les serveurs MCP comme des assistants spécialisés qui aident votre compagnon de codage IA (comme GitHub Copilot) à se connecter à des outils et services externes. Tout comme vous utilisez différentes applications sur votre téléphone pour diverses tâches — une pour la météo, une pour la navigation, une pour la banque — les serveurs MCP donnent à votre assistant IA la capacité d’interagir avec différents outils et services de développement.

### Le problème que les serveurs MCP résolvent

Avant les serveurs MCP, si vous vouliez :
- Vérifier vos ressources Azure
- Créer un problème GitHub 
- Interroger votre base de données
- Chercher dans la documentation

Vous deviez arrêter de coder, ouvrir un navigateur, naviguer vers le site approprié, et effectuer ces tâches manuellement. Ce changement constant de contexte brise votre concentration et diminue votre productivité.

### Comment les serveurs MCP transforment votre expérience de développement

Avec les serveurs MCP, vous pouvez rester dans votre environnement de développement (VS Code, Visual Studio, etc.) et simplement demander à votre assistant IA d’effectuer ces tâches. Par exemple :

**Au lieu de ce flux de travail traditionnel :**
1. Arrêter de coder
2. Ouvrir un navigateur
3. Naviguer vers le portail Azure
4. Rechercher les détails du compte de stockage
5. Retourner à VS Code
6. Reprendre le codage

**Vous pouvez maintenant faire ceci :**
1. Demander à l’IA : « Quel est le statut de mes comptes de stockage Azure ? »
2. Continuer à coder avec les informations fournies

## Démarrage avec MCP dans VS Code et Visual Studio 🚀

Configurer ces serveurs MCP est simple si vous utilisez Visual Studio Code ou Visual Studio 2022 avec GitHub Copilot.

### Configuration dans VS Code

Voici le processus basique pour VS Code :

1. **Activer le mode Agent** : Dans VS Code, passez en mode Agent dans la fenêtre Copilot Chat
2. **Configurer les serveurs MCP** : Ajoutez les configurations des serveurs dans votre fichier settings.json de VS Code
3. **Lancer les serveurs** : Cliquez sur le bouton « Start » pour chaque serveur que vous souhaitez utiliser
4. **Sélectionner les outils** : Choisissez quels serveurs MCP activer pour votre session en cours

Pour des instructions détaillées, consultez la [documentation MCP pour VS Code](https://code.visualstudio.com/docs/copilot/copilot-mcp).

> **💡 Astuce pro : gérez les serveurs MCP comme un expert !**
> 
> La vue Extensions de VS Code inclut désormais une [interface utilisateur pratique pour gérer les serveurs MCP installés](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode) ! Vous avez un accès rapide pour démarrer, arrêter et gérer tous les serveurs MCP installés via une interface claire et simple. Essayez dès maintenant !

### Configuration dans Visual Studio 2022

Pour Visual Studio 2022 (version 17.14 ou ultérieure) :

1. **Activer le mode Agent** : Cliquez sur le menu déroulant « Ask » dans la fenêtre GitHub Copilot Chat et sélectionnez « Agent »
2. **Créer un fichier de configuration** : Créez un fichier `.mcp.json` dans votre répertoire solution (emplacement recommandé : `<SOLUTIONDIR>\.mcp.json`)
3. **Configurer les serveurs** : Ajoutez vos configurations de serveur MCP en utilisant le format MCP standard
4. **Approbation des outils** : Validez les outils que vous souhaitez utiliser avec les permissions de portée appropriées

Pour des instructions détaillées sur Visual Studio, consultez la [documentation MCP pour Visual Studio](https://learn.microsoft.com/visualstudio/ide/mcp-servers).

Chaque serveur MCP a ses propres exigences de configuration (chaînes de connexion, authentification, etc.), mais le modèle d’installation est cohérent pour les deux IDE.

### 1. 📚 Serveur Microsoft Learn Docs MCP

[![Installer dans VS Code](https://img.shields.io/badge/VS_Code-Install_Microsoft_Docs_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D) [![Installer dans VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Microsoft_Docs_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/mcp)

**Ce que ça fait** : Le serveur Microsoft Learn Docs MCP est un service hébergé dans le cloud qui offre aux assistants IA un accès en temps réel à la documentation officielle Microsoft via le Model Context Protocol. Il se connecte à `https://learn.microsoft.com/api/mcp` et permet une recherche sémantique à travers Microsoft Learn, la documentation Azure, la documentation Microsoft 365 et d’autres sources officielles Microsoft.

**Pourquoi c’est utile** : Bien que cela puisse sembler être « juste de la documentation », ce serveur est en fait crucial pour tout développeur utilisant des technologies Microsoft. Une des principales plaintes des développeurs .NET à propos des assistants de codage IA est qu’ils ne sont pas à jour des dernières versions de .NET et C#. Le serveur Microsoft Learn Docs MCP résout cela en fournissant un accès en temps réel à la documentation, aux références API et aux meilleures pratiques les plus récentes. Que vous travailliez avec les derniers SDK Azure, exploriez les nouvelles fonctionnalités de C# 13, ou mettiez en œuvre des patterns .NET Aspire à la pointe, ce serveur garantit que votre assistant IA a accès à l’information officielle, à jour et fiable nécessaire pour générer un code précis et moderne.

**Usage concret** : « Quelles sont les commandes az cli pour créer une application conteneur Azure selon la documentation officielle Microsoft Learn ? » ou « Comment configurer Entity Framework avec l’injection de dépendances dans ASP.NET Core ? » Ou encore « Vérifiez ce code pour vous assurer qu’il respecte les recommandations de performance dans la documentation Microsoft Learn. » Le serveur couvre de manière exhaustive Microsoft Learn, la documentation Azure, et Microsoft 365 grâce à une recherche sémantique avancée pour trouver l’information contextuellement la plus pertinente. Il retourne jusqu’à 10 extraits de contenu de haute qualité avec titres d’article et URLs, accédant toujours à la documentation Microsoft la plus récente dès sa publication.

**Exemple mis en avant** : Le serveur expose l’outil `microsoft_docs_search` qui réalise une recherche sémantique dans la documentation technique officielle Microsoft. Une fois configuré, vous pouvez poser des questions comme « Comment implémenter l’authentification JWT dans ASP.NET Core ? » et obtenir des réponses détaillées officielles avec liens sources. La qualité de la recherche est exceptionnelle car elle comprend le contexte — une question sur les « containers » dans un contexte Azure retournera la documentation Azure Container Instances, tandis que le même terme dans un contexte .NET reviendra sur des informations pertinentes sur les collections C#.

Ceci est particulièrement utile pour les bibliothèques ou cas d’usage qui évoluent rapidement ou récemment mis à jour. Par exemple, dans certains projets de codage récents, je voulais exploiter des fonctionnalités des dernières versions de .NET Aspire et Microsoft.Extensions.AI. En incluant le serveur Microsoft Learn Docs MCP, j’ai pu exploiter non seulement la documentation API, mais aussi des tutoriels et des guides tout juste publiés.

### 2. ☁️ Serveur Azure MCP
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

**Ce que ça fait** : Le serveur Azure MCP est une suite complète de plus de 15 connecteurs spécialisés pour les services Azure qui intègre tout l’écosystème Azure dans votre flux de travail IA. Ce n’est pas juste un serveur unique – c’est une collection puissante qui inclut la gestion des ressources, la connectivité aux bases de données (PostgreSQL, SQL Server), l’analyse des logs Azure Monitor avec KQL, l’intégration Cosmos DB, et bien plus encore.

**Pourquoi c’est utile** : Au-delà de la simple gestion des ressources Azure, ce serveur améliore considérablement la qualité du code lors de l’utilisation des SDK Azure. Lorsque vous utilisez Azure MCP en mode Agent, il ne vous aide pas seulement à écrire du code – il vous aide à écrire un code *Azure meilleur* qui suit les modèles actuels d’authentification, les meilleures pratiques de gestion des erreurs, et exploite les dernières fonctionnalités des SDK. Au lieu d’obtenir un code générique qui pourrait fonctionner, vous obtenez un code qui suit les recommandations Azure pour les charges de travail en production.

**Les modules clés incluent** :
- **🗄️ Connecteurs de bases de données** : Accès en langage naturel direct à Azure Database pour PostgreSQL et SQL Server
- **📊 Azure Monitor** : Analyse des logs et insights opérationnels alimentés par KQL
- **🌐 Gestion des ressources** : Gestion complète du cycle de vie des ressources Azure
- **🔐 Authentification** : Patterns DefaultAzureCredential et identité managée
- **📦 Services de stockage** : Opérations Blob Storage, Queue Storage et Table Storage
- **🚀 Services de conteneurs** : Gestion d’Azure Container Apps, Container Instances et AKS
- **Et bien d’autres connecteurs spécialisés**

**Exemple concret** : « Liste mes comptes de stockage Azure », « Interroge mon espace Workspaces Log Analytics pour les erreurs de la dernière heure », ou « Aide-moi à créer une application Azure en Node.js avec une authentification appropriée »

**Scénario complet de démonstration** : Voici une présentation complète qui montre la puissance de la combinaison Azure MCP avec l’extension GitHub Copilot for Azure dans VS Code. Lorsque vous avez les deux installés et que vous demandez :

> "Crée un script Python qui télécharge un fichier vers Azure Blob Storage en utilisant l’authentification DefaultAzureCredential. Le script doit se connecter à mon compte de stockage Azure nommé 'mycompanystorage', téléverser dans un conteneur nommé 'documents', créer un fichier test avec l’horodatage actuel à téléverser, gérer les erreurs proprement et fournir une sortie informative, suivre les meilleures pratiques Azure pour l’authentification et la gestion des erreurs, inclure des commentaires expliquant le fonctionnement de l’authentification DefaultAzureCredential, et structurer bien le script avec des fonctions appropriées et de la documentation."

Le serveur Azure MCP générera un script Python complet prêt pour la production qui :
- Utilise le dernier SDK Azure Blob Storage avec des patterns async appropriés
- Implémente DefaultAzureCredential avec une explication complète de la chaîne de secours
- Inclut une gestion robuste des erreurs avec des types d’exceptions Azure spécifiques
- Suit les meilleures pratiques des SDK Azure pour la gestion des ressources et des connexions
- Fournit un logging détaillé et une sortie console informative
- Crée un script structuré avec fonctions, documentation et annotations de types

Ce qui rend cela remarquable, c’est qu’en absence d’Azure MCP, vous pourriez obtenir un code générique pour Blob Storage qui fonctionne mais ne suit pas les patterns Azure actuels. Avec Azure MCP, vous obtenez un code qui exploite les dernières méthodes d’authentification, gère les scénarios d’erreurs spécifiques à Azure et suit les pratiques recommandées par Microsoft pour les applications en production.

**Exemple mis en avant** : J’ai eu du mal à me souvenir des commandes spécifiques pour les CLI `az` et `azd` pour un usage ponctuel. C’est toujours un processus en deux étapes pour moi : d’abord chercher la syntaxe, puis lancer la commande. Je finis souvent par aller sur le portail Azure et cliquer un peu partout pour faire mon travail car je ne veux pas admettre que je ne me souviens pas de la syntaxe du CLI. Pouvoir simplement décrire ce que je veux est incroyablement pratique, et encore mieux de pouvoir le faire sans quitter mon IDE !

Il existe une excellente liste de cas d’usage dans le [dépôt Azure MCP](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) pour vous aider à démarrer. Pour des guides d’installation complets et des options de configuration avancées, consultez la [documentation officielle Azure MCP](https://learn.microsoft.com/azure/developer/azure-mcp-server/).

### 3. 🐙 Serveur GitHub MCP

[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Server-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=github&config=%7B%22type%22%3A%20%22http%22%2C%22url%22%3A%20%22https%3A%2F%2Fapi.githubcopilot.com%2Fmcp%2F%22%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Server-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=github&config=%7B%22type%22%3A%20%22http%22%2C%22url%22%3A%20%22https%3A%2F%2Fapi.githubcopilot.com%2Fmcp%2F%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/github/github-mcp-server)

**Ce que ça fait** : Le serveur officiel GitHub MCP offre une intégration fluide avec l’écosystème complet de GitHub, proposant à la fois l’accès distant hébergé et des options de déploiement local via Docker. Il ne s’agit pas juste d’opérations basiques sur les dépôts – c’est un kit d’outils complet qui inclut la gestion des GitHub Actions, les workflows de pull request, le suivi des issues, les analyses de sécurité, les notifications et des capacités avancées d’automatisation.

**Pourquoi c’est utile** : Ce serveur transforme votre manière d’interagir avec GitHub en intégrant toute l’expérience de la plateforme directement dans votre environnement de développement. Au lieu de passer constamment de VS Code à GitHub.com pour la gestion de projet, les revues de code et le suivi CI/CD, vous pouvez tout gérer via des commandes en langage naturel tout en restant concentré sur votre code.

> **ℹ️ Note : Différents types d’« agents »**  
>  
> Ne confondez pas ce serveur GitHub MCP avec le Coding Agent de GitHub (l’agent IA que vous pouvez assigner aux issues pour des tâches de codage automatisées). Le serveur GitHub MCP fonctionne dans le mode Agent de VS Code pour fournir une intégration avec l’API GitHub, tandis que le Coding Agent de GitHub est une fonctionnalité distincte qui crée des pull requests lorsqu’il est assigné aux issues GitHub.

**Fonctionnalités clés** :
- **⚙️ GitHub Actions** : Gestion complète des pipelines CI/CD, surveillance des workflows et gestion des artefacts
- **🔀 Pull Requests** : Création, revue, fusion et gestion des PR avec suivi d’état complet
- **🐛 Issues** : Gestion complète du cycle de vie des issues, commentaires, étiquetage et assignation
- **🔒 Sécurité** : Alertes de scan de code, détection de secrets et intégration Dependabot
- **🔔 Notifications** : Gestion intelligente des notifications et contrôle des abonnements aux dépôts
- **📁 Gestion des dépôts** : Opérations sur fichiers, gestion des branches et administration des dépôts
- **👥 Collaboration** : Recherche d’utilisateurs et d’organisations, gestion des équipes et contrôle des accès

**Exemple concret** : « Crée une pull request depuis ma branche feature », « Montre-moi tous les runs CI qui ont échoué cette semaine », « Liste les alertes de sécurité ouvertes pour mes dépôts », ou « Trouve toutes les issues qui me sont assignées dans mes organisations »

**Scénario complet de démonstration** : Voici un workflow puissant qui montre les capacités du serveur GitHub MCP :

> « Je dois préparer notre revue de sprint. Montre-moi toutes les pull requests que j’ai créées cette semaine, vérifie le statut de nos pipelines CI/CD, crée un résumé des alertes de sécurité à traiter, et aide-moi à rédiger les notes de version basées sur les PR fusionnées avec le label 'feature'. »

Le serveur GitHub MCP va :
- Interroger vos pull requests récentes avec des détails sur leur statut
- Analyser les exécutions de workflow et mettre en avant les échecs ou problèmes de performance
- Compiler les résultats des scans de sécurité et prioriser les alertes critiques
- Générer des notes de version complètes en extrayant les infos des PR fusionnées
- Fournir des étapes concrètes pour la planification du sprint et la préparation de la release

**Exemple mis en avant** : J’adore utiliser cela pour les workflows de revue de code. Au lieu de sauter entre VS Code, les notifications GitHub et les pages de pull request, je peux dire « Montre-moi toutes les PR en attente de ma revue » puis « Ajoute un commentaire à la PR #123 demandant des précisions sur la gestion des erreurs dans la méthode d’authentification. » Le serveur gère les appels API GitHub, maintient le contexte des discussions, et m’aide même à rédiger des commentaires de revue plus constructifs.

**Options d’authentification** : Le serveur supporte à la fois OAuth (transparent dans VS Code) et les tokens d’accès personnel, avec des ensembles d’outils configurables pour activer uniquement les fonctionnalités GitHub dont vous avez besoin. Vous pouvez l’exécuter comme service distant hébergé pour une installation instantanée ou localement via Docker pour un contrôle total.

> **💡 Astuce Pro**  
>  
> Activez uniquement les ensembles d’outils dont vous avez besoin en configurant le paramètre `--toolsets` dans vos réglages du serveur MCP pour réduire la taille du contexte et améliorer la sélection des outils IA. Par exemple, ajoutez `"--toolsets", "repos,issues,pull_requests,actions"` à vos arguments de configuration MCP pour les workflows de développement principaux, ou utilisez `"--toolsets", "notifications, security"` si vous voulez surtout les capacités de surveillance GitHub.

### 5. 📝 Serveur MarkItDown MCP
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_MarkItDown_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_MarkItDown_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/markitdown)

**Ce que cela fait** : MarkItDown est un serveur de conversion de documents complet qui transforme divers formats de fichiers en Markdown de haute qualité, optimisé pour la consommation par les LLM et les flux de travail d'analyse de texte.

**Pourquoi c'est utile** : Essentiel pour les flux de documentation modernes ! MarkItDown gère une gamme impressionnante de formats de fichiers tout en préservant la structure critique du document comme les titres, listes, tableaux et liens. Contrairement aux outils simples d’extraction de texte, il se concentre sur le maintien du sens sémantique et du formatage, précieux à la fois pour le traitement IA et la lisibilité humaine.

**Formats de fichiers supportés** :
- **Documents Office** : PDF, PowerPoint (PPTX), Word (DOCX), Excel (XLSX/XLS)
- **Fichiers multimédias** : Images (avec métadonnées EXIF et OCR), Audio (avec métadonnées EXIF et transcription vocale)
- **Contenu web** : HTML, flux RSS, URLs YouTube, pages Wikipédia
- **Formats de données** : CSV, JSON, XML, fichiers ZIP (traite récursivement le contenu)
- **Formats de publication** : EPub, notebooks Jupyter (.ipynb)
- **Email** : Messages Outlook (.msg)
- **Avancé** : Intégration Azure Document Intelligence pour un traitement PDF amélioré

**Capacités avancées** : MarkItDown supporte la description d’images assistée par LLM (lorsqu’un client OpenAI est fourni), Azure Document Intelligence pour un traitement PDF avancé, la transcription audio pour le contenu parlé, ainsi qu’un système de plugins pour étendre à d’autres formats de fichiers.

**Usage réel** : « Convertir cette présentation PowerPoint en Markdown pour notre site de documentation », « Extraire le texte de ce PDF avec la structure correcte des titres », ou « Transformer ce tableau Excel en un format de tableau lisible »

**Exemple mis en avant** : Pour citer la [documentation de MarkItDown](https://github.com/microsoft/markitdown#why-markdown) :

> Markdown est extrêmement proche du texte brut, avec un balisage ou un formatage minimal, mais offre quand même un moyen de représenter une structure documentaire importante. Les LLM grand public, tels que GPT-4o d’OpenAI, "parlent" nativement Markdown, et intègrent souvent du Markdown dans leurs réponses sans invitation. Cela suggère qu’ils ont été entraînés sur d’énormes quantités de texte formaté en Markdown et le comprennent bien. En plus, les conventions Markdown sont très efficaces en termes de jetons.

MarkItDown est vraiment doué pour préserver la structure des documents, ce qui est important pour les flux de travail IA. Par exemple, lors de la conversion d’une présentation PowerPoint, il conserve l’organisation des diapositives avec les bons titres, extrait les tableaux en tables Markdown, inclut le texte alternatif des images et même traite les notes du présentateur. Les graphiques sont convertis en tableaux de données lisibles et le Markdown résultant maintient le flux logique de la présentation originale. Cela le rend parfait pour alimenter le contenu des présentations dans des systèmes IA ou pour créer la documentation à partir des diapositives existantes.

## 📋 Conclusion

Le Model Context Protocol (MCP) transforme la manière dont les développeurs interagissent avec les assistants IA et outils externes. Ces 10 serveurs MCP Microsoft démontrent la puissance d’une intégration IA standardisée, permettant des flux de travail fluides qui maintiennent les développeurs dans leur état de concentration tout en accédant à des capacités externes puissantes.

De l’intégration complète à l’écosystème Azure aux outils spécialisés comme Playwright pour l’automatisation du navigateur et MarkItDown pour le traitement documentaire, ces serveurs illustrent comment MCP peut accroître la productivité dans divers scénarios de développement. Le protocole standardisé garantit que ces outils fonctionnent ensemble harmonieusement, créant une expérience de développement cohérente.

À mesure que l’écosystème MCP évolue, rester engagé avec la communauté, explorer de nouveaux serveurs, et construire des solutions personnalisées seront les clés pour maximiser votre productivité en développement. La nature open source du standard MCP vous permet de combiner des outils de différents fournisseurs pour créer le flux de travail parfait pour vos besoins spécifiques.
