# MCP తో Azure కంటెంట్ సేఫ్టీ అమలుచేయడం

> **OWASP MCP ప్రమాదం పరిష్కారం**: [MCP06 - ఉద్దేశ్యం ప్రవాహ వ్యతిరేకత](https://microsoft.github.io/mcp-azure-security-guide/mcp/mcp06-prompt-injection/)

ప్రాంప్ట్ ఇంజెక్షన్, టూల్ విషపరచడం, ఇంకా ఇతర AI-ఆధారిత భద్రతా లోపాలపై MCP భద్రతను బలోపేతం చేసేందుకు, Azure కంటెంట్ సేఫ్టీ ఇంటిగ్రేషన్ చేయడం అత్యంత సిఫార్సు చేయబడింది. ఈ అమలు గైడ్ [MCP Security Summit Workshop (Sherpa)](https://azure-samples.github.io/sherpa/) క్యాంబ్ 3: I/O భద్రత కూడా అనుసరిస్తుంది.

## MCP సర్వర్‌తో ఇంటిగ్రేషన్

మీ MCP సర్వర్‌తో Azure కంటెంట్ సేఫ్టీని ఇంటిగ్రేట్ చేయడానికి, మీ అభ్యర్థన ప్రాసెసింగ్ పైప్‌లైన్లో కంటెంట్ సేఫ్టీ ఫిల్టర్‌ను మిడిల్‌వేర్‌గా జోడించండి:

1. సర్వర్ ప్రారంభ సమయంలో ఫిల్టర్‌ను ప్రారంభించండి  
2. ప్రాసెస్ చేయడం ముందు అన్ని టూల్ అభ్యర్థనలను ధృవీకరించండి  
3. క్లయింట్లకు తిరిగి పంపేముందు బాహ్య ప్రతిస్పందనలు తనిఖీ చేయండి  
4. సేఫ్టీ ఉల్లంఘనలపై లాగ్ మరియు అలర్ట్ చేయండి  
5. కంటెంట్ సేఫ్టీ తనిఖీలు విఫలమైనప్పుడు తగిన తప్పిద నిర్వహణ అమలు చేయండి  

ఇది క్రమంగా రక్షణను అందిస్తుంది:  
- ప్రాంప్ట్ ఇంజెక్షన్ దాడులు  
- టూల్ విషపరచడమైన యత్నాలు  
- దుష్టి ఇన్పుట్ల ద్వారా డేటా ఎక్స్‌ఫిల్ట్రేషన్  
- హానికర కంటెంట్ తీయడము  

## Azure కంటెంట్ సేఫ్టీ ఇంటిగ్రేషన్ ఉత్తమ పద్ధతులు

1. **కస్టమ్ బ్లాక్‌లిస్ట్‌లు**: MCP ఇంజెక్షన్ నమూనాల కోసం ప్రత్యేక కస్టమ్ బ్లాక్‌లిస్ట్‌లను సృష్టించండి  
2. **తీవ్రత ట్యూనింగ్**: మీ నిర్దిష్ట ఉపయోగానికి మరియు ప్రమాద అనుభవానికి అనుగుణంగా తీవ్రత పరిమితులను సర్దుబాటు చేయండి  
3. **సమగ్ర కవరేజ్**: అన్ని ఇన్పుట్లు మరియు అవుట్‌పుట్లకు కంటెంట్ సేఫ్టీ తనిఖీలను వర్తింపజేయండి  
4. **పర్ఫార్మెన్స్ ఆప్టిమైజేషన్**: పునరావృత కంటెంట్ సేఫ్టీ తనిఖీల కోసం క్యాచింగ్ అమలుపరచడానికి పరిశీలించండి  
5. **ఫాల్బ్యాక్ meకానిజంలు**: కంటెంట్ సేఫ్టీ సేవలు అందుబాటులో లేనప్పుడు స్పష్టమైన ఫాల్బ్యాక్ ప్రవర్తనలను నిర్వచించండి  
6. **వినియోగదారు ప్రతిస్పందన**: సేఫ్టీ సమస్యల కారణంగా కంటెంట్ నిరోధింపబడినప్పుడు వినియోగదారులకు స్పష్టమైన ఫీడ్‌బ్యాక్ ఇవ్వండి  
7. **నిరంతర సవరణ**: కొత్త ప్రమాదాల ఆధారంగా బ్లాక్‌లిస్ట్‌లు మరియు నమూనాలను సకాలంలో నవీకరించండి
