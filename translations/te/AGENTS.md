# AGENTS.md

## ప్రాజెక్ట్ అవలోకనం

**MCP సులభతరుల కోసం** అనేది మోడల్ కాంటెక్స్ట్ ప్రోటోకాల్ (MCP) నేర్చుకునేందుకు ఓపెన్ సోర్స్ విద్యా పాఠ్యాంశం - ఇది ఏఐ మోడల్స్ మరియు కస్టమర్ అప్లికేషన్ల మధ్య పరస్పర చర్యలకు ఒక ప్రామాణిక పద్ధతి. ఈ రిపాజిటరీలో అనేక కార్యక్రమ భాషలలో అనుప్రయోగ కోడ్ ఉదాహరణల సహా సమగ్రమైన శిక్షణ సామగ్రి అందుబాటులో ఉంది.

### ముఖ్య సాంకేతికతలు

- **ప్రోగ్రామింగ్ భాషలు**: C#, జావా, జావాస్క్రిప్ట్, టైప్స్క్రిప్ట్, పైథాన్, రస్ట్
- **ఫ్రేమ్‌వర్క్‌లు & SDKలు**: 
  - MCP SDK (`@modelcontextprotocol/sdk`)
  - స్ర్పింగ్ బూట్ (జావా)
  - ఫాస్ట్MCP (పైథాన్)
  - లాంగ్‌చెయిన్4j (జావా)
- **డేటాబేస్‌లు**: pgvector ఎక్స్‌టెన్షన్‌తో పోస్ట్‌గ్రెఎస్‌క్యూఎల్
- **క్లౌడ్ ప్లాట్‌ఫారమ్స్**: అజ్యూర్ (కంటైనర్ యాప్స్, ఓపెన్‌ఏఐ, కంటెంట్ సేఫ్టీ, అప్లికేషన్ ఇన్సైట్స్)
- **బిల్డ్ టూల్స్**: npm, మావెన్, పిప్, కార్గో
- **డాక్యుమెంటేషన్**: మార్క్‌డౌన్ తో స్వయంచాలక బహుభాషా అనువాదం (48+ భాషలు)

### వాస్తుశ్రేణి

- **11 కోర్ మాడ్యూల్స్ (00-11)**: మౌలికాల నుండి అభివృద్ధి వ్యాసాలకు వరుసగా నేర్చుకునే మార్గం
- **హ్యాండ్స్-ఆన్ లాబ్స్**: అనేక భాషలలో పూర్తి పరిష్కార కోడ్‌తో ప్రాక్టికల్ వ్యాయామాలు
- **నమూనా ప్రాజెక్టులు**: MCP సర్వర్ మరియు క్లయింట్ అమల్లు
- **అనువాద వ్యవస్థ**: బహుభాషా మద్దతు కోసం ఆటోమేటెడ్ GitHub యాక్షన్స్ వర్క్‌ఫ్లో
- **చిత్ర ఆస్తులు**: అనువాద సంస్కరణలతో కేంద్రీకృత చిత్రం డైరెక్టరీ

## సెట్‌అప్ కమాండ్లు

ఇది డాక్యుమెంటేషన్-కేంద్రిత రిపాజిటరీ. ఎక్కువ సెట్‌అప్ వ్యక్తిగత నమూనా ప్రాజెక్టులు మరియు లాబ్స్‌లో జరుగుతుంది.

### రిపాజిటరీ సెట్‌అప్

```bash
# రిపోజిటరీని క్లోన్ చేయండి
git clone https://github.com/microsoft/mcp-for-beginners.git
cd mcp-for-beginners
```

### నమూనా ప్రాజెక్టులతో పని చేయడం

నమూనా ప్రాజెక్టులు ఈ చోట్ల కలవు:
- `03-GettingStarted/samples/` - భాష ఆధారిత ఉదాహరణలు
- `03-GettingStarted/01-first-server/solution/` - మొదటి సర్వర్ అమల్లు
- `03-GettingStarted/02-client/solution/` - క్లయింట్ అమల్లు
- `11-MCPServerHandsOnLabs/` - సమగ్ర డేటాబేస్ ఇంటిగ్రేషన్ లాబ్స్

ప్రతి నమూనా ప్రాజెక్టులో స్వంత సెట్‌అప్ సూచనలు ఉంటాయి:

#### టైప్స్క్రిప్ట్/జావాస్క్రిప్ట్ ప్రాజెక్టులు
```bash
cd <project-directory>
npm install
npm start
```

#### పైథాన్ ప్రాజెక్టులు
```bash
cd <project-directory>
pip install -r requirements.txt
# లేదా
pip install -e .
python main.py
```

#### జావా ప్రాజెక్టులు
```bash
cd <project-directory>
mvn clean install
mvn spring-boot:run
```

## అభివృద్ధి వర్క్‌ఫ్లో

### డాక్యుమెంటేషన్ నిర్మాణం

- **మాడ్యూల్స్ 00-11**: వరుసగా ముఖ్య పాఠ్యాంశం
- **translations/**: భాష ఆధారిత సంస్కరణలు (స్వయంచాలితంగా ఉత్పత్తి, నేరుగా మార్చవద్దు)
- **translated_images/**: స్థానికీకరించిన చిత్రం సంస్కరణలు (స్వయంచాలితంగా ఉత్పత్తి)
- **images/**: మూల చిత్రం మరియు చిత్రణలు

### డాక్యుమెంటేషన్ మార్పులు చేయడం

1. రూట్ మాడ్యూల్ డైరెక్టరీలలో (00-11) లోని ఇంగ్లీష్ మార్క్‌డౌన్ ఫైళ్లను మాత్రమే సవరించండి
2. అవసరమైతే `images/` డైరెక్టరీలో చిత్రాలను నవీకరించండి
3. కో- ఆప్-ట్రాన్స్‌లేటర్ GitHub యాక్షన్ ఆటోమేటిక్ అనువాదాలు ఉత్పత్తి చేస్తుంది
4. అనువాదాలు మెయిన్ బ్రాంచ్ కు పుష్ చేసినపుడు మళ్లీ ఉత్పత్తి అవుతాయి

### అనువాదాలతో పని చేయడం

- **ఆటోమేటెడ్ అనువాదం**: GitHub Actions వర్క్‌ఫ్లో అన్ని అనువాదాలు నిర్వహిస్తుంది
- **`translations/` డైరెక్టరీ ఫైళ్లు మానవీయంగా మార్చవద్దు**
- ప్రతీ అనువాద ఫైలులో అనువాద మెటాడేటా ఉంచడం జరుగుతుంది
- మద్దతు ఉన్న భాషలు: 48+ భాషలు, వాటిలో అరేబిక్, చైనీస్, ఫ్రెంచ్, జర్మన్, హిందీ, జపనీస్, కొరియన్, పోర్చుగీసు, రష్యన్, స్పానిష్ సహా మరిన్ని ఉన్నాయి

## పరీక్ష సూచనలు

### డాక్యుమెంటేషన్ ధృవీకరణ

ఇది ప్రధానంగా డాక్యుమెంటేషన్ రిపాజిటరీ కావడంతో, పరీక్ష కిందివి ఉంటాయి:

1. **లింక్ ధృవీకరణ**: అన్ని అంతర్గత లింకులు పనిచేస్తున్నాయో చూసుకోండి
```bash
# విరిగిన మార్క్డౌన్ లింకులను తనిఖీ చేయండి
find . -name "*.md" -type f | xargs grep -n "\[.*\](../../.*)"
```

2. **కోడ్ నమూనాలను ధృవీకరణ**: కోడ్ ఉదాహరణలు కంపైల్/రన్ అవుతాయో పరీక్షించండి
```bash
# నిర్దిష్ట నమూనాకు వెళ్లి దాని పరీక్షలను నిర్వహించండి
cd 03-GettingStarted/samples/typescript
npm install && npm test
```

3. **మార్క్‌డౌన్ లింటింగ్**: ఫార్మాటింగ్ సరిగ్గా ఉందా జాగ్రత్త
```bash
# అవసరమైతే markdownlint ఉపయోగించండి
npx markdownlint-cli2 "**/*.md" "#node_modules"
```

#### టైప్స్క్రిప్ట్/జావాస్క్రిప్ట్

mvn test

## కోడ్ శైలి మార్గదర్శకాలు

### డాక్యుమెంటేషన్ శైలి

- స్పష్టమైన, ప్రారంభ దశలో ఉన్నవారికి సులభమైన భాష ఉపయోగించండి
- అనేక భాషల్లో కోడ్ ఉదాహరణలు చేర్చండి ఎక్కడ అవసరమో
- మార్క్‌డౌన్ మంచి ఆచారాలు పాటించండి:
  - ATX-శైలి హెడర్లు (`#` సింటాక్స్) ఉపయోగించండి
  - భాష గుర్తింపు ఉన్న ఫెన్స్‌డ్ కోడ్ బ్లాక్స్ వాడండి
  - చిత్రాలకు వివరణాత్మక ఆల్‌టెక్స్ట్ ఉండాలి
  - లైన్ పొడవులను రూపొందించండి (కఠిన పరిమితి లేదు కానీ సంభ్రమంతో ఉండాలి)

### కోడ్ నమూనా శైలి

#### టైప్స్క్రిప్ట్/జావాస్క్రిప్ట్
- ES మాడ్యూల్స్ (`import`/`export`) ఉపయోగించండి
- టైప్స్క్రిప్ట్ కఠిన మోడ్ నియమాలు పాటించండి
- టైప్ అనోటేషన్లు చేర్చండి
- ES2022 టార్గెట్ చేయండి

#### పythాన్
- PEP 8 స్టైల్ మార్గదర్శకాలు పాటించండి
- నాగరైతమైన చోట టైపు హింట్లు వాడండి
- ఫంక్షన్స్ మరియు క్లాసులకు డోక్‌స్ట్రింగ్స్ చేర్చండి
- ఆధునిక పythాన్ ఫీచర్స్ (3.8+) వాడండి

#### జావా
- స్ర్పింగ్ బూట్ నియమాలు పాటించండి
- జావా 21 ఫీచర్లు వాడండి
- సాంప్రదాయ మావెన్ ప్రాజెక్ట్ నిర్మాణం పాటించండి
- జావడాక్ కామెంట్లను చేర్చండి

## నిర్మాణం మరియు ప్రయోగం

### డాక్యుమెంటేషన్ ప్రయోగం

రిపాజిటరీ డాక్యుమెంటేషన్ హోస్టింగ్ కోసం GitHub పేజీస్ లేదా మరొకటి ఉపయోగిస్తుంది (ఉల్లంఘన ఉంటే). మెయిన్ బ్రాంచ్ లో మార్పులు:

1. అనువాద వర్క్‌ఫ్లో (`.github/workflows/co-op-translator.yml`)
2. అన్ని ఇంగ్లీష్ మార్క్‌డౌన్ ఫైల్స్ యొక్క ఆటోమేటెడ్ అనువాదం
3. అవసరానికి చిత్ర స్థానికీకరణ

### నిర్మాణం అవసరం లేదు

ఈ రిపాజిటరీని ప్రధానంగా మార్క్‌డౌన్ డాక్యుమెంటేషన్ కలిగి ఉంది. కోర్ పాఠ్యాంశానికి కంపైల్ లేదా బిల్డ్ దశ అవసరం లేదు.

### నమూనా ప్రాజెక్ట్ ప్రయోగం

ప్రత్యేక నమూనా ప్రాజెక్టులకు ప్రయోగ సూచనలు ఉండకూడవచ్చు:
- MCP సర్వర్ ప్రయోగ సూచనలు కోసం `03-GettingStarted/09-deployment/` చూడండి
- అజ్యూర్ కంటైనర్ యాప్స్ ప్రయోగ ఉదాహరణలు `11-MCPServerHandsOnLabs/` లో ఉన్నాయి

3. **మార్పులు చేయండి**: ఇంగ్లీష్ మార్క్‌డౌన్ ఫైల్స్ మాత్రమే సవరించండి (అనువాదాలు కాదు)
4. **స్థానికంగా పరీక్షించండి**: మార్క్‌డౌన్ సరిగా ప్రదర్శించబడుతున్నది నిర్ధారించండి
5. **పుల్ రిక్వెస్ట్ సబ్మిట్ చేయండి**: స్పష్టమైన శీర్షికలు మరియు వివరణలు ఇచ్చి PR పంపండి
6. **CLA**: మైక్రోసాఫ్ట్ కాంట్రిబ్యూటర్ లైసెన్స్ అగ్రిమెంట్‌కు సంతకం చేయండి

### PR శీర్షిక నమూనా

స్పష్టమైన, వివరణాత్మక శీర్షికలు ఉపయోగించండి:
- `[Module XX] సంక్షిప్త వివరణ` మాడ్యూల్-నిర్దిష్ట మార్పుల కోసం
- `[Samples] వివరణ` నమూనా కోడ్ మార్పుల కోసం
- `[Docs] వివరణ` సాధారణ డాక్యుమెంటేషన్ అప్‌డేట్ల కోసం

### ఏం పంపాలి

- డాక్యుమెంటేషన్ లేదా కోడ్ నమూనాల్లో బగ్ సరిదిద్దులు
- అదనపు భాషల్లో కొత్త కోడ్ ఉదాహరణలు
- ఈ ఇప్పటికే ఉన్న కంటెంట్ లో స్పష్టత మరియు మెరుగుదలలు
- కొత్త కేసు స్టడీస్ లేదా ప్రాక్టికల్ ఉదాహరణలు
- అర్థం కాని లేదా తప్పు సమాచారంపై ఇష్యూలు అందించడం

### ఏమి చేయకూడదు

- `translations/` డైరెక్టరీలో ఉన్న ఫైళ్లను నేరుగా మార్చవద్దు
- `translated_images/` డైరెక్టరీలో మార్పులు చేయవద్దు
- చర్చ లేకుండా పెద్ద బైనరీ ఫైళ్లు జోడించవద్దు
- అనువాద వర్క్‌ఫ్లో ఫైళ్లను సమన్వయం లేకుండా మార్చవద్దు

### సాధారణ సమస్యలు పరిష్కారం

**Q: నా PR అనువాద తనిఖీ విఫలమవుతోంది**
A: మీరు రూట్ మాడ్యూల్ డైరెక్టరీలలోని ఇంగ్లీష్ మార్క్‌డౌన్ ఫైళ్లనే సవరించారని నిర్ధారించండి, అనువాదాల కాలేదు.

**Q: కొత్త భాష ఎలా జోడించాలి?**
A: భాష మద్దతు కో-ఆప్-ట్రాన్స్‌లేటర్ వర్క్‌ఫ్లో ద్వారా నిర్వహించబడుతుంది. కొత్త భాషల జోడింపును చర్చించేందుకు ఇష్యూ తెరువు.

**Q: కోడ్ నమూనాలు పని చేయడం లేదు**
A: మీరు ప్రత్యేక నమూనా README లో పేర్కొన్న సెట్‌అప్ నియమాలను పాటించారా చూసుకోండి. అవసరమైన డిపెండెన్సీలు సరిగా ఇన్‌స్టాల్ చేశారని తనిఖీ చేయండి.

**Q: చిత్రాలు కనిపించడం లేదు**
A: చిత్రం పాత్‌లు రిలేటివ్ గా ఉండాలి మరియు ఫార్వర్డ్ స్లాష్‌లు ఉపయోగించండి. చిత్రాలు `images/` లేదా స్థానికీకృతకరణ కోసం `translated_images/` డైరెక్టరీలలో ఉండాలి.
