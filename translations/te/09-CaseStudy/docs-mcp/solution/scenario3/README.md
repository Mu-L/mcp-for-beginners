# అంకం 3: MCP సర్వర్‌తో VS కోడ్‌లో ఇన్-ఎడిటర్ డాక్స్

## అవగాహనా

ఈ అంకంలో, మీరు Microsoft Learn Docs ని MCP సర్వర్ ఉపయోగించి నేరుగా మీ Visual Studio Code వాతావరణంలో ఎలా దారితీయాలో నేర్చుకుంటారు. డాక్యుమెంటేషన్ కోసం బ్రౌజర్ ట్యాబ్‌లను తరచుగా మార్చుకోవడానికి బదులుగా, మీరు అధికారిక డాక్స్‌ని ఎడిటర్‌లోనే యాక్సెస్ చేయగలుగుతారు, శోధించగలుగుతారు మరియు సూచించగలుగుతారు. ఈ విధానం మీ పనితీరు సులభమవుతుంది, మీరు దృష్టిని నిలుపుకుంటారు, మరియు GitHub Copilot వంటి సాధనాలతో సుగమమైన సమగ్రీకరణను సాధిస్తుంది.

- కోడ్ వాతావరణాన్ని verlassen చేయకుండా VS కోడ్ లోనే డాక్స్ శోధించండి మరియు చదవండి.
- README లేదా కోర్స్ ఫైళ్లలో నేరుగా డాక్యుమెంటేషన్ సూచనలని చేర్చండి.
- GitHub Copilot మరియు MCP కలిపి AI ఆధారిత డాక్యుమెంటేషన్ వర్క్‌ఫ్లో కోసం ఉపయోగించండి.

## నేర్చుకునే లక్ష్యాలు

ఈ అధ్యాయాన్ని పూర్తి చేసిన తరువాత, మీరు MCP సర్వర్ ని VS కోడ్ లో సెట్ చేయడం, ఉపయోగించడం ఎలా అనేది అర్థం చేసుకుంటారు. మీరు చేయగలుగుతారు:

- డాక్యుమెంటేషన్ కోసం MCP సర్వర్ ఉపయోగించునది పని స్థలాన్ని కాంఫిగర్ చేయండి.
- VS కోడ్ లోనే నేరుగా డాక్యుమెంటేషన్ శోధించండి మరియు చేర్చండి.
- మరింత ఉత్పాదకతతో AI-ఆధారిత వర్క్‌ఫ్లో కోసం GitHub Copilot మరియు MCP శక్తిని కలిపి ఉపయోగించండి.

ఈ నైపుణ్యాలు మీ దృష్టిని నిలుపుకోవడానికి, డాక్యుమెంటేషన్ నాణ్యతను మెరుగుపరచడానికి, మీ ఉత్పాదకతను అభివృద్ధి చేయడానికి సహాయపడతాయి.

## పరిష్కారం

ఇన్-ఎడిటర్ డాక్యుమెంటేషన్ యాక్సెస్ కోసం, మీరు MCP సర్వర్ ని VS కోడ్ మరియు GitHub Copilot తో సమగ్రపరిచే ఒక సరళమైన దశలను అనుసరించండి. ఈ పరిష్కారం కోర్స్ రచయితలు, డాక్యుమెంటేషన్ రైటర్లు మరియు డెవలపర్లు కోసం అనుకూలం, వారు ఎడిటర్ లోనే ఫోకస్ ఉంచుకొని docs మరియు Copilot తో పని చేయగలుగుతారు.

- కోర్స్ లేదా ప్రాజెక్ట్ డాక్యుమెంటేషన్ రాయుతూ READMEకి సూచనల లింకులను త్వరగా చేర్చండి.
- కోడ్ జనరేట్ చేయడానికి Copilot ఉపయోగించండి మరియు సంబంధిత డాక్స్ తక్షణం కనుగొనడానికి MCP ఉపయోగించండి.
- ఎడిటర్ లో ఫోకస్ నిలుపుకుని ఉత్పాదకతను పెంచండి.

### దశల వారీ గైడ్

ప్రారంభించడానికి, ఈ దశలను అనుసరించండి. ప్రతి దశ కోసం, మీరు ఆ దశని దృశ్యంగా వివరించడానికి అసెట్ ఫోల్డర్ నుండి స్క్రీన్షాట్ చేర్చవచ్చు.

1. **MCP కాన్ఫిగరేషన్ జోడించండి:**
   మీ ప్రాజెక్ట్ రూట్ లో `.vscode/mcp.json` ఫైల్ సృష్టించి క్రింది కాన్ఫిగరేషన్ చేర్చండి:
   ```json
   {
     "servers": {
       "LearnDocsMCP": {
         "url": "https://learn.microsoft.com/api/mcp"
       }
     }
   }
   ```
   ఈ కాన్ఫిగరేషన్ VS కోడ్ MCP సర్వర్ తో ఎలా కనెక్ట్ అవ్వాలో తెలుపుతుంది: [`Microsoft Learn Docs MCP server`](https://github.com/MicrosoftDocs/mcp).
   
   ![Step 1: Add mcp.json to .vscode folder](../../../../../../translated_images/te/step1-mcp-json.c06a007fccc3edfa.webp)
    
2. **GitHub Copilot చాట్ ప్యానల్ తెరవండి:**
   మీరు ముందే GitHub Copilot ఎక్స్‌టెన్షన్ ను ఇన్‌స్టాల్ చేసుకోకపోతే, VS కోడ్‌లో ఎక్స్‌టెన్షన్స్ వీక్షణకు వెళ్లి దాన్ని ఇన్‌స్టాల్ చేయండి. మీరు [Visual Studio Code Marketplace](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot-chat) నుండి నేరుగా డౌన్లోడ్ చేసుకోవచ్చు. తరువాత, సైడ్‌బార్ నుండి Copilot చాట్ ప్యానల్ తెరవండి.

   ![Step 2: Open Copilot Chat panel](../../../../../../translated_images/te/step2-copilot-panel.f1cc86e9b9b8cd1a.webp)

3. **ఏజెంట్ మోడ్ ప్రారంభించి టూల్స్‌ను ధృవీకరించండి:**
   Copilot చాట్ ప్యానల్ లో ఏజెంట్ మోడ్ ప్రారంభించండి.

   ![Step 3: Enable agent mode and verify tools](../../../../../../translated_images/te/step3-agent-mode.cdc32520fd7dd1d1.webp)

   ఏజెంట్ మోడ్ ప్రారంభించిన తరువాత, MCP సర్వర్ అందుబాటులో ఉన్న టూల్స్ జాబితాలో ఉన్నదో లేదో తనిఖీ చేయండి. దీని ద్వారా Copilot ఏజెంట్ డాక్యుమెంటేషన్ సర్వర్ యాక్సెస్ చేసి సంబంధిత సమాచారాన్ని తెచ్చుకోగలుగుతుంది.
   
   ![Step 3: Verify MCP server tool](../../../../../../translated_images/te/step3-verify-mcp-tool.76096a6329cbfecd.webp)
4. **కొత్త చాట్‌ను ప్రారంభించి ఏజెంట్‌ను ప్రాంప్ట్ చేయండి:**
   Copilot చాట్ ప్యానల్ లో కొత్త చాట్ తెరవండి. ఇప్పుడు మీరు మీ డాక్యుమెంటేషన్ ప్రశ్నలతో ఏజెంట్‌ను ప్రాంప్ట్ చేయవచ్చు. ఏజెంట్ MCP సర్వర్ ను ఉపయోగించి సంబంధిత Microsoft Learn డాక్యుమెంటేషన్ ను నేరుగా మీ ఎడిటర్ లో తెప్పిస్తుంది.

   - *"నేను అంశం X కోసం అధ్యయన ప్రణాళిక రాయాలనుకుంటున్నాను. నేను ప్రతి వారానికి 8 వారాలు అధ్యయనం చేస్తాను, ప్రతి వారానికి ఉపయుక్తమైన కంటెంట్ను సూచించండి."*

   ![Step 4: Prompt the agent in chat](../../../../../../translated_images/te/step4-prompt-chat.12187bb001605efc.webp)

5. **లైవ్ క్వెరీ:**

   > Microsoft Foundry Discordలోని [#get-help](https://discord.gg/D6cRhjHWSC) విభాగం నుండి ఒక ప్రస్తుత క్వెరీ తీసుకుందాం ([మూల సందేశాన్ని వీక్షించండి](https://discord.com/channels/1113626258182504448/1385498306720829572)):
   
   *"నేను Azure AI Foundryపై అభివృద్ధి చేసిన AI ఏజెంట్లతో మల్టీ-ఏజెంట్ సొల్యూషన్ ఎలా డిప్లాయ్ చేయాలో సమాధానాల కోసం చూస్తున్నాను. Copilot Studio ఛానళ్ల వంటి ప్రత్యక్ష డిప్లాయ్ విధానం లేదు. కాబట్టి, ఎంటర్ప్రైజ్ వినియోగదారులు ప‌ని చేయడానికి ఈ డిప్లాయ్‌ల కోసం భిన్న మార్గాలు ఏమిటి?
MS Teams మరియు Azure AI Foundry ఏజెంట్ల మధ్య బ్రిడ్జ్ గా పనిచేసే Azure బాట్ సర్విస్ ఉపయోగించాలని అనేక వ్యాసాలు/బ్లాగ్లు చెప్తున్నాయి, అయితే నేను Azure ఫంక్షన్ ద్వారా Azure AI Foundryలోని Orchestrator Agent కి కనెక్ట్ అయ్యే Azure బాట్ సెట్ చేస్తే ఇది పని చేస్తుందా? లేక బాట్ ఫ్రేమ్‌వర్క్‌లో ఆర్కెస్ట్రేషన్ కోసం ప్రతి AI ఏజెంట్ భాగం కోసం Azure ఫంక్షన్ సృష్టించాల్సి ఉంటుందా? ఇతర సూచనలు స్వాగతం."*

   ![Step 5: Live queries](../../../../../../translated_images/te/step5-live-queries.49db3e4a50bea273.webp)

   ఏజెంట్ సంబంధిత డాక్యుమెంటేషన్ లింకులు మరియు సారాంశాలతో స్పందిస్తుంది, వాటిని మీరు నేరుగా మీ మార్క్డౌన్ ఫైళ్లలో చేర్చుకోవచ్చు లేదా కోడ్‌లో సూచనలకు ఉపయోగించవచ్చు.
   
### నమూనా క్వెరీలు

ఇవి మీరు ప్రయత్నించగల కొన్ని ఉదాహరణ ప్రశ్నలు. ఈ ప్రశ్నలు MCP సర్వర్ మరియు Copilot కలిపి ఎలా తక్షణ, సందర్భభైత డాక్యుమెంటేషన్ మరియు సూచనలను అందిస్తాయో చూపిస్తాయి, VS కోడ్ వదలకుండా:

- "Azure Functions ట్రిగ్గర్లు ఎలా ఉపయోగించాలో చూపించండి."
- "Azure Key Vault అధికారిక డాక్యుమెంటేషన్ కి లింక్ చేర్చండి."
- "Azure వనరులను భద్రపరచడానికి ఉత్తమ పద్ధతులు 무엇?"
- "Azure AI సర్వీసుల కోసం క్విక్‌స్టార్ట్ కనుగొనండి."

ఈ ప్రశ్నలు MCP సర్వర్ మరియు Copilot కలిపి కంటెక్ట్-అవేర్ డాక్యుమెంటేషన్ మరియు సూచనలను ఎలా అందిస్తాయో చూపిస్తాయి, VS కోడ్ వదలకుండా.
