#### ప్రస్తుత SDKలతో నమూనా ధృవీకరణ

- **TypeScript (03-GettingStarted/01-first-server/solution/typescript)**: `npm install` ద్వారా `@modelcontextprotocol/sdk@1.29.0` అనుమానాలు తొలగిపోయినవి; `tsc --noEmit` లో ఎటువంటి టైప్ ఎర్రర్స్ లేవు — ప్రస్తుత `McpServer`/`StdioServerTransport` APIs ఉపయోగించదగినవే

- అన్ని నమూనాలు `@modelcontextprotocol/sdk` వెర్షన్ రేంజ్‌లు (`>=1.26.0` / `^1.26.0` / `^1.27.0`) ప్రస్తుత `1.29.0`కి సాఫీగా పరిష్కరించబడినవి, ఎటువంటి API బ్రేక్ లేవు

### సెక్యూరిటీ హార్డెనింగ్ & డిపెండెన్సీ సున్నితత్వ పరిష్కారం

ప్రతి డిపెండెన్సీ మేనిఫెస్ట్ మరియు నమూనా సోర్స్ కోడ్ పై పూర్తి సెక్యూరిటీ తనిఖీ నిర్వహించి, అన్ని npm అలర్ట్‌లు మరియు ఒక కోడ్-స్థాయి ఫైండ్ పరిష్కరించబడింది. పరిష్కారం తర్వాత, `npm audit` ప్రతి ఆడిట్ చేసిన డైరక్టరీలో **0 సున్నితత్వాలు** చూపిస్తుంది.

#### npm డిపెండెన్సీ సున్నితత్వాలు (సంచారుగా) — పరిష్కరించబడింది

- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/inspector** మరియు **lab3/code/weather_mcp/inspector**: `@modelcontextprotocol/inspector` (`0.16.6` / `0.14.1` → `0.22.0`) పెంచబడింది, ఇది bundled `ajv`, `brace-expansion`, `diff`, `path-to-regexp` మరియు `ws` అలర్ట్‌లు తొలగించింది. npm `overrides` ఎంట్రీ జత చేసి `shell-quote@1.8.4` ప్యాచ్‌ను బలవంతపరచడంతో `concurrently`కి సంబంధించి మిగిలిన క్రిటికల్ అలర్ట్ తొలగింది; రెండు లాక్ ఫైళ్లు తిరిగి జనరేట్ అయి (ఇప్పుడు 0 సున్నితత్వాలు)

#### కోర్ కాన్సెప్ట్స్ అభివృద్ధి (01-CoreConcepts/)
- **కొత్త క్లయింట్ ప్రిమిటివ్ - Roots**: Roots క్లయింట్ ప్రిమిటివ్ పై సవ్యమైన డాక్యుమెంటేషన్ జోడించడం ద్వారా సర్వర్లు ఫైల్ సిస్టమ్ సరిహద్దులు మరియు ప్రాప్యత అనుమతులను అర్థం చేసుకోవడానికి సదుపాయం కల్పించింది
- **టూల్ అనోటేషన్లు**: మైరుగైన టూల్ అమలు నిర్ణయాల కోసం సాధనపు ప్రవర్తనా అనోటేషన్ల (`readOnlyHint`, `destructiveHint`) డాక్యుమెంటేషన్ జోడించడం
- **నమూనా తీసుకోవడంలో టూల్ కాలింగ్**: నమూనా డాక్యుమెంటేషన్‌లో `tools` మరియు `toolChoice` పారామితులను జోడించడం, నమూనా అభ్యర్థనల సమయంలో మోడల్-నడిపే టూల్ పిలుపు కోసం
- **URL మోడ్ ఎలిసిటేషన్**: సర్వర్ ప్రారంభించిన బాహ్య వెబ్ సంబంధాల కోసం URL ఆధారిత ఎలిసిటేషన్ పై డాక్యుమెంటేషన్ జోడింపు
- **టాస్కులు (ప్రయోగాత్మక)**: బహుళ అమలులు ప్యాకేజీ మరియు వాయిదా ఫలితాల కోసం ప్రయోగాత్మక టాస్క్‌ ఫీచర్ పై కొత్త విభాగం
- **ఐకాన్ల మద్దతు**: టూల్స్, వనరులు, వనరు టెంప్లేట్లు మరియు ప్రాంప్ట్‌లకు అదనపు మెటాడేటాగా ఐకాన్లు చేర్చగలడని గమనిక

#### డాక్యుమెంటేషన్ అప్డేట్లు
- **README.md**: MCP స్పెసిఫికేషన్ 2025-11-25 వెర్షన్ రిఫరెన్స్ మరియు తేదీ ఆధారిత వెర్షనింగ్ వివరాలు జోడించడం
- **study_guide.md**: కోర్ కాన్సెప్ట్స్ విభాగంలో టాస్కులు మరియు టూల్ అనోటేషన్లు జోడించి విద్యాక్రమం నవీకరింపు; డాక్యుమెంట్ టైమ్‌స్పామ్ అప్డేట్

#### స్పెసిఫికేషన్ అనుగుణ పట్టింపు ధృవీకరణ
- **ప్రోటోకాల్ వెర్షన్**: అన్ని డాక్యుమెంటేషన్ ప్రస్తుత MCP స్పెసిఫికేషన్ 2025-11-25 ను సూచిస్తున్నదని ధృవీకరించడం
- **అధార శాస్త్ర సర్దుబాటు**: ఇద్దరు-పట్టాల నిర్మాణం (డేటా లేయర్ + ట్రాన్స్‌పోర్ట్ లేయర్) డాక్యుమెంటేషన్ ఖచ్చితత్వం నిర్ధారించడం
- **ప్రిమిటివ్స్ డాక్యుమెంటేషన్**: సర్వర్ ప్రిమిటివ్స్ (వనరులు, ప్రాంప్ట్స్, టూల్స్) మరియు క్లయింట్ ప్రిమిటివ్స్ (నమూనా, ఎలిసిటేషన్, లాగింగ్, రూట్స్) ధృవీకరణ  
- **ట్రాన్స్‌పోర్ట్ యంత్రాంగం**: STDIO మరియు స్ట్రీమబుల్ HTTP ట్రాన్స్‌పోర్ట్ డాక్యుమెంటేషన్ ఖచ్చితత్వం  
- **భద్రతా మార్గదర్శకత్వం**: ప్రస్తుత MCP భద్రతా ఉత్తమమైన పద్ధతులకు అనుగుణంగా ధృవీకరణ  

#### MCP 2025-11-25 ముఖ్య లక్షణాలు డాక్యుమెంటెడ్  
- **OpenID Connect డిస్కవరీ**: OIDC ద్వారా ఆస్‌థర్ సర్వర్ డిస్కవరీ  
- **OAuth క్లయింట్ ID మెటాడేటా డాక్యుమెంట్లు**: సిఫార్సు చేసిన క్లయింట్ రిజిస్ట్రేషన్ యంత్రాంగం  
- **JSON స్కీమా 2020-12**: MCP స్కీమా నిర్వచనాల కోసం డిఫాల్ట్ డైలెక్ట్  
- **SDK టియర్ సిస్టమ్**: SDK ఫీచర్ మద్దతు మరియు నిర్వహణ కోసం ఫార్మలైజ్డ్ అవసరాలు  
- **పాలన నిర్మాణం**: MCP పాలనలో వర్కింగ్ గ్రూప్స్ మరియు ఇంట్రెస్ట్ గ్రూప్స్ ఫార్మల్ నిర్మాణం

### విద్యాక్రమ విస్తృత MCP స్పెసిఫికేషన్ 2025-11-25 అనుసరణ

#### మాడ్యూల్ 03 - ప్రారంభం
- **SDK డాక్యుమెంటేషన్**: అధికారిక SDK జాబితాలో Go SDK జోడింపు; అన్ని SDK సూచనలు MCP స్పెసిఫికేషన్ 2025-11-25కి అనుగుణంగా నవీకరణ  
- **ట్రాన్స్‌పోర్ట్ స్పష్టత**: STDIO మరియు HTTP స్ట్రీమింగ్ ట్రాన్స్‌పోర్ట్ వివరణలను స్పెసిఫికేషన్ సూచనలతో అప్డేట్ చేయడం  

#### మాడ్యూల్ 04 - ప్రత్యక్ష అమలు
- **SDK నవీకరణలు**: Go SDK జోడించడం; స్పెసిఫికేషన్ వెర్షన్ సూచనతో SDK జాబితా నవీకరణ  
- **అధికార Spec**: MCP Authorization స్పెసిఫికేషన్ లింక్ 2025-11-25 ప్రస్తుత వెర్షన్‌కు అప్డేట్  

#### మాడ్యూల్ 05 - అధునాతన అంశాలు
- **కొత్త ఫీచర్ల గమనిక**: MCP స్పెసిఫికేషన్ 2025-11-25 కొత్త లక్షణాలు (టాస్కులు, టూల్ అనోటేషన్లు, URL మోడ్ ఎలిసిటేషన్, రూట్స్)పై గమనిక జోడింపు  
- **భద్రతా వనరులు**: OWASP MCP టాప్ 10 మరియు Sherpa వర్క్‌షాప్ లింకులు అదనపు వనరులుగా జోడించడం  

#### మాడ్యూల్ 06 - కమ్యూనిటీ కాంట్రిబ్యూషన్స్
- **SDK జాబితా**: Swift మరియు Rust SDKలను జోడించడం; స్పెసిఫికేషన్ లింకును 2025-11-25కి నవీకరించడం  
- **స్పెసిఫికేషన్ సూచన**: MCP స్పెక్ లింకు ప్రత్యక్ష స్పెసిఫికేషన్ URLకి అప్డేట్  

#### మాడ్యూల్ 07 - తొలి ఆమోదం నుండి పాఠాలు
- **వనరు నవీకరణలు**: MCP స్పెసిఫికేషన్ 2025-11-25 లింకు మరియు OWASP MCP టాప్ 10 అదనపు వనరులకు జోడింపు  

#### మాడ్యూల్ 08 - ఉత్తమ పద్ధతులు
- **స్పెసిఫికేషన్ వెర్షన్**: MCP స్పెసిఫికేషన్ సూచనను 2025-11-25 కి నవీకరణ  
- **భద్రతా వనరులు**: OWASP MCP టాప్ 10 మరియు Sherpa వర్క్‌షాప్ లింకులు అదనపు వనరులుగా జోడింపు

### ప్రారంభ విభాగం విస్తరణ – ఆధునిక సర్వర్ ఉపయోగం & సాధారణ ధృవీకరణ

#### ఆధునిక సర్వర్ ఉపయోగం (03-GettingStarted/10-advanced)
- **కొత్త అధ్యాయం జోడింపు**: సాధారణ మరియు లో-లెవల్ సర్వర్ నిర్మాణాలను కవర్ చేసే MCP సర్వర్ ఆధునిక వినియోగంపై విస్తృత మార్గదర్శకం ప్రవేశపెట్టబడింది.
  - **సాధారణ vs. లో-లెవల్ సర్వర్**: రెండు విధానాల కోసం Python మరియు TypeScript లో వివరణాత్మక పోలిక మరియు కోడ్ ఉదాహరణలు.
  - **హ్యాండ్లర్ ఆధారిత డిజైన్**: విస్తరించదగిన, సరళమైన సర్వర్ అమలుల కోసం హ్యాండ్లర్-ఆధారిత టూల్/రీసోర్స్/ప్రాంప్ట్ నిర్వహణ వివరణ.
  - **ప్రాక్టికల్ ప్యాటర్న్‌లు**: లో-లెవల్ సర్వర్ ప్యాటర్న్‌లు ఆధునిక ఫీచర్లు మరియు ఆర్కిటెక్చర్‌కు ఎప్పుడు ఉపయోగకరమవుతాయో వాస్తవ ప్రపంచ సందర్భాలు.

#### సాదా ఆథెంటికేషన్ (03-GettingStarted/11-simple-auth)
- **కొత్త అధ్యాయం జోడించబడింది**: MCP సర్వర్‌లలో సాదా ఆథెంటికేషన్ అమలు కోసం దశ దశ గైడ్.
  - **ఆథ్ కాన్సెప్ట్స్**: ఆథెంటికేషన్ మరియు ఆథరైజేషన్, క్రెడెన్షియల్ నిర్వహణపై స్పష్టమైన వివరణ.
  - **బేసిక్ ఆథ్ అమలాపరచడం**: Python (Starlette) మరియు TypeScript (Express) లో మిడిల్వేర్ ఆధారిత ఆథెంటికేషన్ ప్యాటర్న్స్, కోడ్ ఉదాహరణలతో.
  - **ఆధునిక సెక్యూరిటీ వైపు ప్రగతి**: సాదా ఆథ్ తో ప్రారంభించి OAuth 2.1 మరియు RBAC వరకు ఎలా అభివృద్ధి చెందాలో మార్గదర్శకత్వం, అధునాతన సెక్యూరిటీ మాడ్యూల్స్ సూచనలతో.

ఈ మార్పులు మున్నాది కాన్సెప్ట్లను ఆధునిక ఉత్పత్తి ప్యాటర్న్‌లతో కలిపి మరింత బలమైన, సురక్షితమైన, మరియు లైఫ్లెక్స్ MCP సర్వర్ అమలులకు ప్రాక్టికల్ మరియు చేతులకు అనువైన మార్గదర్శకత్వాన్ని అందిస్తాయి.

  - **Azure ఈవెంట్ గ్రిడ్ ట్రాన్స్‌పోర్ట్**: సమగ్ర సర్వర్‌లెస్ ఈవెంట్ ఆధారిత ట్రాన్స్‌పోర్ట్ అమలు

  - క్లయింట్ ప్రిమిటివ్స్: శాంప్లింగ్ (LLM పూర్తి), ఇలిసిటేషన్ (వినియోగదారు ఇన్‌పుట్), లాగింగ్ (డీబగ్గింగ్/మానిటరింగ్)
  - ప్రస్తుత డిస్కవరీ (`*/list`), రీట్రీవల్ (`*/get`), మరియు ఎగ్జిక్యూషన్ (`*/call`) పద్ధతుల నమూనాలతో నవీకరించబడింది
- **ప్రోటోకాల్ వాస್ತು శిల్పం**: రెండు-పొరల వాస್ತು శిల్ప మోడల్ ప్రవేశపెట్టబడింది
  - డేటా పొర: JSON-RPC 2.0 పునాది, జీవన చక్ర నిర్వహణ మరియు ప్రిమిటివ్స్
  - రవాణా పొర: STDIO (స్థానిక) మరియు స్ట్రీమబుల్ HTTP SSE (దూర) రవాణా విధానాలు
- **భద్రతా ఫ్రేమ్‌వర్క్**: స్పష్టమైన వినియోగదారు అంగీకారం, డేటా గోప్యత రక్షణ, టూల్ నిర్వహణ భద్రత, మరియు రవాణా పొర భద్రతతో సమగ్ర భద్రతా సూత్రాలు
- **సంచారం నమూనాలు**: ప్రారంభం, డిస్కవరీ, ఎగ్జిక్యూషన్, మరియు నోటిఫికేషన్ ప్రవాహాలను చూపించడానికి ప్రోటోకాల్ సందేశాలు నవీకరించబడ్డాయి
- **కోడ్ ఉదాహరణలు**: ప్రస్తుత MCP SDK నమూనాలను ప్రతిబింబించడానికి బహుభాషా (.NET, జావా, పython, జావాస్క్రిప్ట్) ఉదాహరణలు నవీకరించబడ్డాయి

#### సెక్యూరిటీ (02-Security/) - సమగ్ర భద్రతా పునఃస్థాపన  
- **ప్రమాణాల సర్దుబాటు**: MCP విశేషణ 2025-06-18 భద్రతా అవసరాల పూర్తి సర్దుబాటు
- **ఆథెంటికేషన్ అభివృద్ధి**: కస్టమ్ OAuth సర్వర్ల నుండి బాహ్య ఐడెంటిటీ ప్రొవైడర్ ప్రతినిధిత్వం (Microsoft Entra ID) వరకు పరిణామాన్ని డాక్యుమెంటేషన్ చేసినది
- **AI-విశేష ముప్పు విశ్లేషణ**: ఆధునిక AI దాడి దిశలపై మరింత కవర్ చేయబడింది
  - రియల్-వర్చువల్ ఉదాహరణలతో వివరమైన ప్రాంప్ట్ ఇంజెక్షన్ దాడి వర్గాలు
  - టూల్ విషపూరణ క్రియావిధానాలు మరియు "రగ్ పుల్" దాడి నమూనాలు
  - సందర్భ విండో విషపూరణ మరియు మోడల్ గందరగోళ దాడులు
- **Microsoft AI సెక్యూరిటీ పరిష్కారాలు**: Microsoft సెక్యూరిటీ ఇకోసిస్టం సమగ్ర ప్రదర్శన
  - AI ప్రాంప్ట్ షీల్డ్స్ అధునాతన గుర్తింపు, స్పాట్‌లైట్, మరియు డెలిమిటర్ సాంకేతికతలతో
  - Azure కంటెంట్ సేఫ్టీ ఇంటిగ్రేషన్ నమూనాలు
  - గిట్‌హబ్ అడ్వాన్స్డ్ సెక్యూరిటీ సరఫరా గొలుసు రక్షణ కోసం
- **అధునాతన ముప్పు నివారణ**: 
  - MCP ప్రత్యేక దాడి సన్నివేశాలతో సెషన్ హైజాకింగ్ కోసం విశదీకరించిన భద్రతా నియంత్రణలు మరియు క్రిప్టోగ్రాఫిక్ సెషన్ ID అవసరం
  - MCP ప్రాక్సీ సందర్భాల్లో గందరగోళ డెప్యూటీ సమస్యలకు స్పష్టమైన అంగీకార అవసరాలు
  - టోకెన్ పాస్త్‌త్రూ పాపదృష్టులు మరియు తప్పనిసరి ఆమోద నియంత్రణలు
- **సరఫరా గొలుసు భద్రత**: ఫౌండేషన్ మోడల్స్, ఎంబెడ్డింగ్ సేవలు, సందర్భ ప్రొవైడర్లు, మరియు మూడవ పార్టీ API లతో AI సరఫరా గొలుసు సమగ్ర కవరేజి
- **ఫౌండేషన్ భద్రత**: జీరో ట్రస్టు వాస್ತು శిల్ప మరియు Microsoft సెక్యూరిటీ ఇకోసిస్టమ్ సహా ఎంటర్ప్రైజ్ భద్రతా నమూనాల ఇంటిగ్రేషన్ మెరుగుపరచబడింది
- **వనరు ఏర్పాటు**: రకాలు ప్రకారం (అధికారిక డాక్స్, ప్రమాణాలు, పరిశోధన, Microsoft పరిష్కారాలు, అమలుదారుల మార్గదర్శకాలు) విస్తృత వనరు లింకులను వర్గీకరించారు

### పత్రికా నాణ్యత మెరుగుదలలు
- **పరిపాటీబద్ధమైన అభ్యాస లక్ష్యాలు**: నిర్దిష్ట, కార్యాచరణ ఫలితాలతో అభ్యాస లక్ష్యాలు మెరుగుపరచబడ్డాయి
- **క్రాస్-రెఫరెన్సులు**: సంబంధిత భద్రత మరియు కోర్ సూత్రాంశాల మధ్య లింకులు జోడించబడ్డాయి
- **ప్రస్తుత సమాచారం**: అన్ని తేదీ సూచనలు మరియు విశేషణ లింకులు ప్రస్తుత ప్రమాణాలకు నవీకరించబడ్డాయి
- **అమలు మార్గదర్శకత్వం**: రెండు విభాగాలలో నిర్దిష్ట, కార్యాచరణ మార్గదర్శకాలు జోడించబడ్డాయి
