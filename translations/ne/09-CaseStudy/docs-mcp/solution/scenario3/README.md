# परिदृश्य ३: VS Code मा MCP सर्भरसहित इन-एडिटर डकहरू

## अवलोकन

यस परिदृश्यमा, तपाईंले MCP सर्भर प्रयोग गरेर Microsoft Learn डकहरूलाई सिधै आफ्नो Visual Studio Code वातावणमा ल्याउन कसरी गर्ने सिक्नुहुनेछ। कागजात खोज्न बारम्बार ब्राउजर ट्याबहरू साटासाट गर्ने सट्टा, तपाईं आफ्नो सम्पादक भित्रै आधिकारिक डकहरू पहुँच, खोज, र सन्दर्भ गर्न सक्नुहुन्छ। यो तरिका तपाईंको कार्यप्रवाहलाई सरल बनाउँछ, तपाईंलाई केन्द्रित राख्छ, र GitHub Copilot जस्ता उपकरणहरूसँग सहज अन्तरक्रिया सक्षम बनाउँछ।

- VS Code भित्रै डकहरू खोज्नुहोस् र पढ्नुहोस्, आफ्नो कोडिङ्ग वातावरण छोड्न नपर्ने।
- कागजात सन्दर्भ गर्नुहोस् र लिंकहरू सिधै तपाईंको README वा कोर्स फाइलहरूमा राख्नुहोस्।
- GitHub Copilot र MCP सँगै प्रयोग गरेर सहज, AI-चालित कागजात कार्यप्रवाह बनाउनुहोस्।

## सिकाइ उद्देश्यहरू

यस अध्यायको अन्त्यमा, तपाईंले VS Code भित्र MCP सर्भर सेट अप गर्नु र प्रयोग गरी आफ्नो कागजात र विकास कार्यप्रवाह कसरी सुधार गर्ने बुझ्नु हुनेछ। तपाईं सक्षम हुनुहुनेछ:

- कागजात खोजको लागि MCP सर्भर प्रयोग गर्न आफ्नो कार्यक्षेत्र कन्फिगर गर्न।
- VS Code भित्रैबाट कागजात खोज्न र राख्न।
- GitHub Copilot र MCP को शक्ति संयोजन गरी थप उत्पादनशिल, AI-संवर्धित कार्यप्रवाह बनाउन।

यी सीपहरूले तपाईंलाई ध्यान केन्द्रित राख्न, कागजात गुणस्तर सुधार गर्न, र विकासकर्ता वा प्राविधिक लेखकको रूपमा तपाईंको उत्पादकत्व बढाउन मद्दत गर्नेछन्।

## समाधान

इन-एडिटर कागजात पहुँच प्राप्त गर्न, तपाईंले MCP सर्भरलाई VS Code र GitHub Copilot सँग एकीकृत गर्ने चरणहरू पालना गर्नु हुनेछ। यो समाधान कोर्स लेखक, कागजात लेखक, र विकासकर्ताहरूका लागि उपयुक्त छ जसले सम्पादकमा ध्यान केन्द्रित राखेर कागजात र Copilot सँग काम गर्न चाहन्छन्।

- कोर्स वा परियोजना कागजात लेख्दा README मा चाँडो सन्दर्भ लिंक थप्नुहोस्।
- कोड उत्पादन गर्न Copilot र तत्काल सम्बद्ध डकहरू फेला पार्न MCP प्रयोग गर्नुहोस्।
- आफ्नो सम्पादकमा ध्यान केन्द्रित राख्नुहोस् र उत्पादनशिलता बढाउनुहोस्।

### चरण-द्वारा-चरण मार्गदर्शन

सुरु गर्नका लागि यी चरणहरू पालना गर्नुहोस्। प्रत्येक चरणका लागि तपाईंसँग छविहरू थपेर प्रक्रिया देखाउन सक्नुहुन्छ।

1. **MCP कन्फिगरेसन थप्नुहोस्:**
   तपाईंको परियोजना रुटमा, `.vscode/mcp.json` फाइल बनाउनुहोस् र निम्न कन्फिगरेसन थप्नुहोस्:
   ```json
   {
     "servers": {
       "LearnDocsMCP": {
         "url": "https://learn.microsoft.com/api/mcp"
       }
     }
   }
   ```
   यस कन्फिगरेसनले VS Code लाई [`Microsoft Learn Docs MCP server`](https://github.com/MicrosoftDocs/mcp) सँग कसरी जडान गर्ने भन्ने बताउँछ।
   
   ![चरण १: .vscode फोल्डरमा mcp.json थप्नुहोस्](../../../../../../translated_images/ne/step1-mcp-json.c06a007fccc3edfa.webp)
    
2. **GitHub Copilot Chat प्यानल खोल्नुहोस्:**
   यदि तपाईंसँग GitHub Copilot विस्तार पहिले स्थापना गरिएको छैन भने, VS Code मा Extensions भ्यूमा जानुहोस् र यसलाई स्थापना गर्नुहोस्। तपाईं यसलाई सिधै [Visual Studio Code Marketplace](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot-chat) बाट डाउनलोड गर्न सक्नुहुन्छ। त्यसपछि, साइडबारबाट Copilot Chat प्यानल खोल्नुहोस्।

   ![चरण २: Copilot Chat प्यानल खोल्नुहोस्](../../../../../../translated_images/ne/step2-copilot-panel.f1cc86e9b9b8cd1a.webp)

3. **एजेन्ट मोड सक्षम गर्नुहोस् र उपकरणहरू जाँच्नुहोस्:**
   Copilot Chat प्यानलमा एजेन्ट मोड सक्षम गर्नुहोस्।

   ![चरण ३: एजेन्ट मोड सक्षम गर्नुहोस् र उपकरणहरू जाँच गर्नुहोस्](../../../../../../translated_images/ne/step3-agent-mode.cdc32520fd7dd1d1.webp)

   एजेन्ट मोड सक्षम गरेपछि, MCP सर्भर उपलब्ध उपकरणहरूको रूपमा सूचीकृत छ कि छैन भनेर जाँच गर्नुहोस्। यसले सुनिश्चित गर्दछ कि Copilot एजेन्ट कागजात सर्भरसँग जडान हुन सक्छ र सम्बन्धित जानकारी ल्याउन सक्छ।
   
   ![चरण ३: MCP सर्भर उपकरण जाँच गर्नुहोस्](../../../../../../translated_images/ne/step3-verify-mcp-tool.76096a6329cbfecd.webp)
4. **नयाँ च्याट सुरु गर्नुहोस् र एजेन्टलाई सोध्नुहोस्:**
   Copilot Chat प्यानलमा नयाँ च्याट खोल्नुहोस्। अब तपाईं आफ्नो कागजात प्रश्नहरू एजेन्टलाई सोध्न सक्नुहुन्छ। एजेन्टले MCP सर्भर प्रयोग गरी सम्बन्धित Microsoft Learn कागजातहरू सिधै तपाईंको सम्पादकमा ल्याउनेछ।

   - *"म विषय X को लागि अध्ययन योजना तयार गर्दैछु। म यसलाई ८ हप्ता अध्ययन गर्नेछु, प्रत्येक हप्ताका लागि आवश्यक सामग्री सिफारिस गर्नुहोस्।"*

   ![चरण ४: च्याटमा एजेन्टलाई सोध्नुहोस्](../../../../../../translated_images/ne/step4-prompt-chat.12187bb001605efc.webp)

5. **प्रत्यक्ष प्रश्न:**

   > हामी माइक्रोसफ्ट फाउन्ड्री डिस्कोर्डको [#get-help](https://discord.gg/D6cRhjHWSC) सेक्सनबाट प्रत्यक्ष प्रश्न लिऔं ([मूल सन्देश हेर्नुहोस्](https://discord.com/channels/1113626258182504448/1385498306720829572)):
   
   *"म Azure AI Foundry मा विकसित गरिएको बहु-एजेन्ट समाधान कसरी डिप्लोय गर्ने बारे जवाफ खोज्दैछु। मलाई थाहा छ कि Copilot Studio च्यानल जस्ता सिधा डिप्लोय विधि छैन। त्यसैले, एंटरप्राइज प्रयोगकर्ताहरूसँग कुराकानी गर्न र काम सम्पन्न गर्न यो डिप्लोय कसरी गर्ने विभिन्न तरिकाहरू के-के छन्?
धेरै लेख/ब्लगहरू देखिन्छन् जसले भन्छन् कि MS Teams र Azure AI Foundry एजेन्टबीच पुलको रूपमा Azure Bot सेवा प्रयोग गर्न सकिन्छ, के यो काम गर्नेछ यदि म Azure Function प्रयोग गरेर Azure AI Foundry को Orchestrator Agent मा कनेक्ट हुने Azure Bot सेटअप गर्छु, वा मलाई प्रत्येक AI एजेन्ट पार्टको लागि Azure Function बनाउनुपर्नेछ जसले Bot फ्रेमवर्कमा समन्वय गर्ने? अन्य सल्लाह पनि स्वागतयोग्य छन्।"*

   ![चरण ५: प्रत्यक्ष प्रश्नहरू](../../../../../../translated_images/ne/step5-live-queries.49db3e4a50bea273.webp)

   एजेन्टले सम्बन्धित कागजात लिंक र सारांशहरू प्रदान गर्नेछ, जुन तपाईंले सिधै आफ्नो मार्कडाउन फाइलहरूमा राख्न वा कोडमा सन्दर्भको रूपमा प्रयोग गर्न सक्नुहुन्छ।
   
### नमूना प्रश्नहरू

यहाँ केही उदाहरण प्रश्नहरू छन् जुन तपाईं प्रयास गर्न सक्नुहुन्छ। यी प्रश्नहरूले कसरी MCP सर्भर र Copilot सँगै काम गरी सन्दर्भ, प्रसंग-जानकारीयुक्त कागजातहरू र सन्दर्भहरू तुरुन्त उपलब्ध गराउन सक्छन् भन्ने देखाउनेछन्:

- "मलाई Azure Functions ट्रिगरहरू कसरी प्रयोग गर्ने देखाउनुहोस्।"
- "Azure Key Vault को आधिकारिक कागजातको लिंक राख्नुहोस्।"
- "Azure स्रोतहरू सुरक्षित गर्न उत्तम अभ्यासहरू के-के हुन्?"
- "Azure AI सेवाहरूको लागि क्विकस्टार्ट फेला पार्नुहोस्।"

यी प्रश्नहरूले देखाउनेछन् कि MCP सर्भर र Copilot कसरी सँगै काम गरेर तुरुन्त, प्रसंग-जानकारीयुक्त कागजातहरू र सन्दर्भहरू प्रदान गर्न सक्छन्, र तपाईंले VS Code छोड्नु आवश्यक छैन।
