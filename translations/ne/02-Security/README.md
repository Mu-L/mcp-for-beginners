# MCP सुरक्षा: एआई प्रणालीहरूको लागि व्यापक संरक्षण

[![MCP सुरक्षा उत्तम अभ्यासहरू](../../../translated_images/ne/03.175aed6dedae133f.webp)](https://youtu.be/88No8pw706o)

_(यो पाठको भिडियो हेर्न माथिको छवि क्लिक गर्नुहोस्)_

सुरक्षा एआई प्रणाली डिजाइनको आधारभूत कुरा हो, यसैले हामी यसलाई हाम्रो दोस्रो भागको रूपमा प्राथमिकता दिन्छौं। यो माइक्रोसफ्टको [Secure Future Initiative](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) बाट लिइएको **Secure by Design** सिद्धान्तसँग मेल खान्छ।

मोडेल कन्टेक्स्ट प्रोटोकल (MCP) ले एआई-चालित अनुप्रयोगहरूमा शक्तिशाली नयाँ क्षमता ल्याउँछ र परम्परागत सफ्टवेयर जोखिमहरूभन्दा बाहिरी विशिष्ट सुरक्षा चुनौतीहरू प्रस्तुत गर्छ। MCP प्रणालीहरूले स्थापित सुरक्षा चिन्ताहरू (सुरक्षित कोडिङ, न्यूनतम अधिकार, सुप्लाई चेन सुरक्षा) र नयाँ एआई-विशिष्ट खतराहरू जस्तै प्रॉम्प्ट इन्जेक्शन, उपकरण विषाक्तता, सेसन हाइज्याकिंग, कन्फ्युज्ड डेपुटी आक्रमणहरू, टोकन पासथ्रू कमजोरिहरू, र गतिशील क्षमता परिमार्जन सामना गर्छन्।

यस पाठले MCP कार्यान्वयनहरूमा सबैभन्दा महत्वपूर्ण सुरक्षा जोखिमहरू अन्वेषण गर्छ—प्रमाणीकरण, अधिकार, अत्यधिक अनुमतिहरू, अप्रत्यक्ष प्रॉम्प्ट इन्जेक्शन, सेसन सुरक्षा, कन्फ्युज्ड डेपुटी समस्याहरू, टोकन व्यवस्थापन, र सुप्लाई चेन कमजोरिहरूको समावेशमा। तपाईंले यी जोखिमहरूलाई कम गर्नका लागि क्रियाशील नियन्त्रणहरू र उत्तम अभ्यासहरू सिक्नुहुनेछ र माइक्रोसफ्ट समाधानहरू जस्तै प्रॉम्प्ट शिल्ड्स, Azure Content Safety, र GitHub Advanced Security प्रयोग गरी आफ्नो MCP परिनियोजनलाई बलियो बनाउन सक्नुहुनेछ।

## सिकाइ उद्देश्यहरू

यस पाठको अन्त्यसम्म, तपाईं सक्षम हुनुहुनेछ:

- **MCP-विशिष्ट खतराहरूको पहिचान**: MCP प्रणालीमा अद्वितीय सुरक्षा जोखिमहरू जस्तै प्रॉम्प्ट इन्जेक्शन, उपकरण विषाक्तता, अत्यधिक अनुमतिहरू, सेसन हाइज्याकिंग, कन्फ्युज्ड डेपुटी समस्या, टोकन पासथ्रू कमजोरिहरू, र सुप्लाई चेन जोखिमहरूको चिनारी गर्नुहोस्
- **सुरक्षा नियन्त्रणहरूको आवेदन**: शक्तिशाली प्रमाणीकरण, न्यूनतम अधिकार पहुँच, सुरक्षित टोकन व्यवस्थापन, सेसन सुरक्षा नियन्त्रणहरू, र सुप्लाई चेन प्रमाणीकरण लगायत प्रभावकारी निवारणहरू कार्यान्वयन गर्नुहोस्
- **माइक्रोसफ्ट सुरक्षा समाधानहरूको उपयोग**: MCP कार्यभार संरक्षणका लागि माइक्रोसफ्ट प्रॉम्प्ट शिल्ड्स, Azure Content Safety, र GitHub Advanced Security लाई बुझ्नुहोस् र परिनियोजन गर्नुहोस्
- **उपकरण सुरक्षा प्रमाणीकरण गर्नुहोस्**: उपकरण मेटाडाटा प्रमाणीकरणको महत्त्व, गतिशील परिवर्तनहरूको अनुगमन, र अप्रत्यक्ष प्रॉम्प्ट इन्जेक्शन आक्रमणहरू विरुद्ध सुरक्षा गर्नुहोस्
- **उत्तम अभ्यासहरू समेकन गर्नुहोस्**: स्थापित सुरक्षा आधारहरू (सुरक्षित कोडिङ, सर्भर कडा बनाउने, शून्य ट्रस्ट) लाई MCP-विशिष्ट नियन्त्रणहरूसँग संयुक्त गरेर व्यापक संरक्षण प्राप्त गर्नुहोस्

# MCP सुरक्षा वास्तुकला र नियन्त्रणहरू

आधुनिक MCP कार्यान्वयनहरूले परम्परागत सफ्टवेयर सुरक्षा र एआई-विशिष्ट खतराहरू दुबैलाई सम्बोधन गर्ने तहमा आधारित सुरक्षा दृष्टिकोणहरू आवश्यक पर्छ। छिटो विकास भइरहेका MCP विशिष्टताले यसको सुरक्षा नियन्त्रणहरूलाई परिपक्व बनाउँदैछ, जसले एण्टरप्राइज सुरक्षा वास्तुकलाहरू र स्थापित उत्तम अभ्यासहरूसँग राम्रो एकीकरण सक्षम बनाउँछ।

[Microsoft Digital Defense Report](https://aka.ms/mddr) बाट आएको अनुसन्धानले देखाएको छ कि **९८% रिपोर्ट गरिएको भाँचाहरूलाई कडा सुरक्षा हाइजिनले रोक्न सकिन्छ**। सबैभन्दा प्रभावकारी सुरक्षा रणनीतिले आधारभूत सुरक्षा अभ्यासहरूलाई MCP-विशिष्ट नियन्त्रणहरूसँग जोड्ने गर्दछ—प्रमाणित आधारभूत सुरक्षा उपायहरूले कुल सुरक्षा जोखिम घटाउन सबैभन्दा ठूलो प्रभाव पार्छ।

## वर्तमान सुरक्षा परिदृश्य

> **नोट:** यो जानकारी MCP सुरक्षा मापदण्डहरूलाई **फेब्रुअरी ५, २०२६** सम्म प्रतिबिम्बित गर्दछ, जुन **MCP Specification 2025-11-25** सँग मेल खान्छ। MCP प्रोटोकल तीव्र गतिमा विकास हुँदैछ, र भविष्यका कार्यान्वयनहरूले नयाँ प्रमाणीकरण ढाँचाहरू र सुधारिएको नियन्त्रणहरू समावेश गर्न सक्छन्। सदैव सबैभन्दा हालको निर्देशिका लागि [MCP Specification](https://spec.modelcontextprotocol.io/), [MCP GitHub repository](https://github.com/modelcontextprotocol), र [security best practices documentation](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) हेर्नुहोस्।

- **विषयहरू तोडेर सिक्नुहोस्**: जानाजानी कमजोर सर्भरहरू शोषण गरेर जोखिमहरूको अनुभव गर्नुहोस्

- **रक्षा-इन-डेप्थ पालना गर्नुहोस्**: तहहरू निर्माण गर्दै व्यापक सुरक्षा तहहरू सिर्जना गर्नुहोस्
- **OWASP मापदण्डहरू लागू गर्नुहोस्**: प्रत्येक प्रविधि [OWASP MCP Azure Security Guide](https://microsoft.github.io/mcp-azure-security-guide/) सँग मेल खान्छ
- **उत्पादनयोग्य कोड प्राप्त गर्नुहोस्**: काम गर्ने, परीक्षण गरिएका कार्यान्वयनहरू साथमा लैजानुहोस्

### MCP प्रमाणीकरणको विकास

MCP विशिष्टताले प्रमाणीकरण र अधिकारमा उल्लेखनीय विकास गरेको छ:

- **मूल दृष्टिकोण**: प्रारम्भिक विशिष्टताले विकासकर्ताहरूलाई कस्टम प्रमाणीकरण सर्भरहरू कार्यान्वयन गर्न आवश्यक पारे थियो, जसमा MCP सर्भरहरूले प्रयोगकर्ता प्रमाणीकरण सिधा OAuth 2.0 अधिकृत सर्भरको रूपमा व्यवहार गर्थे
- **वर्तमान मापदण्ड (2025-11-25)**: नयाँ विशिष्टताले MCP सर्भरहरूलाई बाह्य पहिचान प्रदायकहरू (जस्तै माइक्रोसफ्ट Entra ID) लाई प्रमाणीकरण प्रस्तुत गर्न अनुमति दिन्छ, जसले सुरक्षा अवस्था सुधार्छ र जटिलता घटाउँछ
- **ट्रान्सपोर्ट लेयर सुरक्षा**: स्थानीय (STDIO) र दुरवर्ती (Streamable HTTP) जडानहरूका लागि उपयुक्त प्रमाणीकरण ढाँचाहरू सहित सुरक्षित ट्रान्सपोर्ट संयन्त्रहरूमा सुधारिएको समर्थन

## प्रमाणीकरण र अधिकार सुरक्षा

### वर्तमान सुरक्षा चुनौतीहरू

आधुनिक MCP कार्यान्वयनहरूले विभिन्न प्रमाणीकरण र अधिकार चुनौतीहरू सामना गर्छन्:

### जोखिम र खतरा भेक्टरहरू

- **गलत रूपमा कन्फिगर गरिएको अधिकार तर्क**: MCP सर्भरहरूमा त्रुटिपूर्ण अधिकार लागू गर्दा संवेदनशील डाटा खुल्ने र पहुँच नियन्त्रणहरू गलत लागू हुन सक्छन्
- **OAuth टोकन चोरी**: स्थानीय MCP सर्भर टोकन चोरीले आक्रमणकारीहरूलाई सर्भरहरूको भेषमा पहुँच प्राप्त गर्ने मौका दिन्छ
- **टोकन पासथ्रू कमजोरिहरू**: गलत टोकन प्रयोगले सुरक्षा नियन्त्रणलाई बाईपास गर्ने र जवाफदेहिताको कमी ल्याउने गर्दछ
- **अत्यधिक अनुमतिहरू**: अधिक अधिकारप्राप्त MCP सर्भरहरूले न्यूनतम अधिकार सिद्धान्त उल्लंघन गरी आक्रमण सतहहरू बढाउँछन्

#### टोकन पासथ्रू: एक गम्भीर सुरक्षा दोष

**टोकन पासथ्रू हालको MCP अधिकार विशिष्टतामा स्पष्ट रूपमा निषेध गरिएको छ किनकि यसका गम्भीर सुरक्षा असरहरू छन्:**

##### सुरक्षा नियन्त्रण बाईपास
- MCP सर्भरहरू र तल्लो सेवाहरूले क्रिटिकल सुरक्षा नियन्त्रणहरू (दर सीमा, अनुरोध प्रमाणीकरण, ट्राफिक अनुगमन) लागू गर्छन् जुन सही टोकन प्रमाणीकरणमा निर्भर हुन्छन्
- क्लाइन्टदेखि-API सिधा टोकन प्रयोगले यी सुरक्षा तहहरूलाई बाइपास गर्छ र सुरक्षा वास्तुकलालाई कमजोर बनाउँछ

##### जवाफदेहिता र लेखा समस्याहरू
- MCP सर्भरहरूले माथिल्लो टोकन प्रयोग गर्ने क्लाइन्टहरू छुट्याउन सक्दैनन् र लेखा ट्रेलहरू भाँचिन्छन्
- तल्लो सेवा लगहरू वास्तविक MCP सर्भरहरू नभएर अनौठा अनुरोध स्रोत देखाउँछन्
- घटना अनुसन्धान र अनुपालन लेखापरीक्षण निकै समस्याग्रस्त हुन्छ

##### डाटा चोरी जोखिमहरू
- प्रमाणीकरण नगरेका टोकन दाबीहरूले चोरी गरिएका टोकनहरूसँग भएका दुष्टहरूले MCP सर्भरलाई डाटा चोर्न प्रयोग गर्न अनुमति दिन्छन्
- आत्मविश्वास सिमाना उल्लंघनका कारण अवैध पहुँच ढाँचाहरू सुरक्षा नियन्त्रणबाट बच्छन्

##### बहु-सेवा आक्रमण भेक्टरहरू
- संलग्न सेवाहरूमा स्वीकारिएका जोखिमयुक्त टोकनहरूले नेटवर्कमा समग्र रूपमा चल्ने क्षमतामा वृद्धि गर्छन्
- टोकन उत्पत्तिको प्रमाणीकरण नगर्दा सेवाहरूबीचको विश्वास उल्लंघन हुन सक्छ

### सुरक्षा नियन्त्रणहरू र निवारणहरू

**महत्वपूर्ण सुरक्षा आवश्यकताहरू:**

> **आदेशयोग्य**: MCP सर्भरहरूले ती टोकनहरू स्वीकार गर्नु हुँदैनन् जुन स्पष्ट रूपमा MCP सर्भरका लागि जारी गरिएको छैन

#### प्रमाणीकरण र अधिकार नियन्त्रणहरू

- **कडा अधिकार समीक्षा**: MCP सर्भर अधिकार तर्कहरूको विस्तृत अडिट गरेर केवल इच्छित प्रयोगकर्ता र क्लाइन्टहरूलाई संवेदनशील स्रोतहरूमा पहुँच सुनिश्चित गर्नुहोस्
  - **कार्यान्वयन गाइड**: [Azure API Management as Authentication Gateway for MCP Servers](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **पहिचान एकीकरण**: [Using Microsoft Entra ID for MCP Server Authentication](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **सुरक्षित टोकन व्यवस्थापन**: [माइक्रोसफ्टको टोकन प्रमाणीकरण र जीवनचक्र उत्तम अभ्यासहरू](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens) कार्यान्वयन गर्नुहोस्
  - टोकनका दर्शक दाबीहरूले MCP सर्भर पहिचानसँग मेल खान्छन् भनि प्रमाणीकरण गर्नुहोस्
  - उपयुक्त टोकन घुमाउना र समाप्ति नीतिहरू लागू गर्नुहोस्
  - टोकन पुनःप्रयोग आक्रमणहरू र अनधिकार प्रयोग रोक्नुहोस्

- **सुरक्षित टोकन भण्डारण**: आराममा र ट्रान्जिटमा दुवैमा इन्क्रिप्सन सहित टोकन सुरक्षित गर्नुहोस्
  - **उत्तम अभ्यासहरू**: [Secure Token Storage and Encryption Guidelines](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### पहुँच नियन्त्रण कार्यान्वयन

- **न्यूनतम अधिकार सिद्धान्त**: MCP सर्भरहरूलाई मात्र निर्धारित कार्यका लागि न्यूनतम अनुमतिहरू दिनुहोस्
  - नियमित अनुमतिहरू समीक्षा र अद्यावधिक गरी अधिकार सीमा बढ्नु नदिई रोक्नुहोस्
  - **माइक्रोसफ्ट दस्तावेजीकरण**: [Secure Least-Privileged Access](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **भूमिका-आधारित पहुँच नियन्त्रण (RBAC)**: छिटो र सटिक भूमिका असाइन गरिएका प्रणाली लागू गर्नुहोस्
  - भूमिका निश्चित स्रोत र कार्यहरूको लागि कडा रूपमा सीमित गर्नुहोस्
  - व्यापक वा अनावश्यक अनुमतिहरू जोगाएर आक्रमण सतहहरू नबढाउनुहोस्

- **लगातार अनुमति अनुगमन**: पहुँचमा निरन्तर लेखा परीक्षण र अनुगमन कार्यान्वयन गर्नुहोस्
  - अनुमति प्रयोग ढाँचाहरूमा असामान्यता निगरानी गर्नुहोस्
  - अत्यधिक वा प्रयोग नगरिएका अनुमतिहरू तुरुन्त सुधार गर्नुहोस्

## एआई-विशिष्ट सुरक्षा खतरा

### प्रॉम्प्ट इन्जेक्शन र उपकरण हेरफेर आक्रमणहरू

आधुनिक MCP कार्यान्वयनहरूले परम्परागत सुरक्षा उपायहरूले पूरै सम्बोधन गर्न नसक्ने परिष्कृत एआई-विशिष्ट आक्रमण भेक्टरहरूको सामना गर्छन्:

#### **अप्रत्यक्ष प्रॉम्प्ट इन्जेक्शन (क्रस-डोमेन प्रॉम्प्ट इन्जेक्शन)**

**अप्रत्यक्ष प्रॉम्प्ट इन्जेक्शन** MCP सक्षम एआई प्रणालीहरूमा सबैभन्दा महत्वपूर्ण कमजोरीहरूमध्ये एक हो। आक्रमणकारीहरूले बाह्य कन्टेन्ट—कागजातहरू, वेब पृष्ठहरू, इमेलहरू, वा डाटा स्रोतहरूमा दुष्ट निर्देशनहरू प्याक गर्ने गर्छन् जुन एआई प्रणालीहरूले बादमा वैध आदेशको रूपमा प्रक्रिया गर्छ।

**आक्रमण परिदृश्यहरू:**
- **कागजात-आधारित इन्जेक्शन**: प्रक्रिया गरिएको कागजातमा लुकाइएको दुष्ट निर्देशनहरूले अनपेक्षित एआई क्रियाकलापहरू ट्रिगर गर्छ
- **वेब कन्टेन्ट शोषण**: घातक वेब पृष्ठहरू जसमा समावेश प्रॉम्प्टहरूले स्क्र्याप गर्दा एआई व्यवहारलाई नियन्त्रण गर्छ
- **इमेल-आधारित आक्रमणहरू**: इमेलमा रहेको दुष्ट प्रॉम्प्टहरूले एआई सहायकहरूलाई जानकारी चुहाउन वा अनधिकृत कार्य गर्न लगाउँछ
- **डाटा स्रोत प्रदूषण**: दूषित डेटाबेस वा एपीआईहरूले एआई प्रणालीहरूमा दूषित कन्टेन्ट प्रदान गर्छ

**वास्तविक प्रभाव**: यी आक्रमणहरूले डाटा चोरी, गोपनीयता उल्लंघन, हानिकारक सामग्री उत्पाद, र प्रयोगकर्ता अन्तरक्रियाहरूको मनिपुलेसन गर्न सक्छन्। विस्तृत विश्लेषणका लागि [Prompt Injection in MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/) हेर्नुहोस्।

![Prompt Injection Attack Diagram](../../../translated_images/ne/prompt-injection.ed9fbfde297ca877.webp)

#### **उपकरण विषाक्तता आक्रमणहरू**

**उपकरण विषाक्तता**ले MCP उपकरणहरू परिभाषित गर्ने मेटाडाटा लक्ष्य बनाउछ, जसमा LLMs ले उपकरण वर्णन र प्यारामिटरहरूलाई कसरी व्याख्या गर्छन् त्यसको फाइदा उठाउँछ।

**आक्रमण मेकानिज्महरू:**
- **मेटाडाटा हेरफेर**: आक्रमणकारीहरूले उपकरण वर्णन, प्यारामिटर परिभाषा, वा प्रयोग उदाहरणहरूमा दुष्ट निर्देशनहरू समावेश गर्छन्
- **अदृश्य निर्देशनहरू**: उपकरण मेटाडाटामा लुकेको प्रॉम्प्टहरू जुन AI मोडेलहरूले प्रक्रिया गर्छ तर मानव प्रयोगकर्ताहरूले देख्दैनन्
- **गतिशील उपकरण परिमार्जन ("रग पुल्स")**: प्रयोगकर्ताहरूले अनुमोदित उपकरणहरू पछि दुष्ट कार्य गर्ने गरी परिमार्जित हुन्छन्, बिना प्रयोगकर्ताको जानकारी
- **प्यारामिटर इन्जेक्शन**: उपकरण प्यारामिटर स्किमाहरूमा दुष्ट सामग्री समावेश जसले मोडेल व्यवहारलाई प्रभाबित गर्छ

**होस्ट गरिएको सर्भर जोखिमहरू**: रिमोट MCP सर्भरहरू उच्च जोखिममा हुन्छन् किनकि उपकरण परिभाषाहरू प्रयोक्ताको अनुमोदनपछि अपडेट गर्न सकिन्छ, जसबाट पहिले सुरक्षित उपकरणहरूले दुष्ट व्यवहार प्रदर्शन गर्न सक्छन्। विस्तृत विश्लेषणका लागि [Tool Poisoning Attacks (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks) हेर्नुहोस्।

![Tool Injection Attack Diagram](../../../translated_images/ne/tool-injection.3b0b4a6b24de6bef.webp)

#### **थप एआई आक्रमण भेक्टरहरू**

- **क्रस-डोमेन प्रॉम्प्ट इन्जेक्शन (XPIA)**: धेरै डोमेनहरूबाट सामग्री उपयोग गरेर सुरक्षा नियन्त्रणहरू बाइपास गर्ने परिष्कृत आक्रमणहरू
- **गतिशील क्षमता संशोधन**: प्रारम्भिक सुरक्षा मूल्यांकनहरूबाट बच्ने उपकरण क्षमताहरूमा वास्तविक समयमा परिवर्तनहरू
- **सन्दर्भ विन्डो विषाक्तता**: दुर्भावनापूर्ण निर्देशनहरू लुकाउन ठूलो सन्दर्भ विन्डोलाई हेरफेर गर्ने आक्रमणहरू
- **मोडेल भ्रम आक्रमणहरू**: मोडेल सीमाहरूको दुरुपयोग गरेर अप्रत्याशित वा असुरक्षित व्यवहार सिर्जना गर्नु


### AI सुरक्षा जोखिम प्रभाव

**उच्च प्रभावका परिणामहरू:**
- **डाटा चोरी**: संवेदनशील उद्यम वा व्यक्तिगत डेटामा अनधिकृत पहुँच र चोरी
- **गोपनीयता उल्लङ्घनहरू**: व्यक्तिगत रूपमा पहिचानयोग्य जानकारी (PII) र गोप्य व्यापार डाटा सार्वजनिक हुनु  
- **प्रणाली हेरफेर**: महत्वपूर्ण प्रणाली र कार्यप्रवाहहरूमा अनपेक्षित संशोधनहरू
- **पहिचान चोरी**: प्रमाणीकरण टोकनहरू र सेवा प्रमाणपत्रहरूको उल्लङ्घन
- **पहिचान फैलावट**: व्यापक नेटवर्क आक्रमणहरूको लागि दुरुपयोग गरिएका AI प्रणालीहरूको प्रयोग

### Microsoft AI सुरक्षा समाधानहरू

#### **AI प्रॉम्प्ट शिल्ड: इन्जेक्शन आक्रमणविरुद्ध उन्नत सुरक्षा**

Microsoft **AI प्रॉम्प्ट शिल्ड** ले प्रत्यक्ष र अप्रत्यक्ष प्रॉम्प्ट इन्जेक्शन आक्रमणहरू विरुद्ध बहु सुरक्षा तहहरू मार्फत व्यापक सुरक्षा प्रदान गर्दछ:

##### **मुख्य सुरक्षा संयन्त्रहरू:**

1. **उन्नत पत्ता लगाउने र फिल्टर गर्ने**
   - मेसिन लर्निङ एल्गोरिदमहरू र NLP प्रविधिहरूले बाह्य सामग्रीमा दुर्भावनापूर्ण निर्देशनहरू पत्ता लगाउँछन्
   - कागजातहरू, वेब पेजहरू, इमेलहरू, र डेटा स्रोतहरूमा एम्बेडेड खतरा हरूको वास्तविक समय विश्लेषण
   - वैध र दुर्भावनापूर्ण प्रॉम्प्ट ढाँचाहरूको सन्दर्भगत बुझाइ

2. **स्पटलाइटिङ प्रविधिहरू**  
   - विश्वसनीय प्रणाली निर्देशनहरू र सम्भावित रूपमा दुरुपयोग गरिएका बाह्य इनपुटहरू बीच फरक पार्ने
   - पाठ रूपान्तरण विधिहरू जसले मोडेल सान्दर्भिकता सुधार गर्छ र दुर्भावनापूर्ण सामग्री अलग गर्छ
   - AI प्रणालीहरूलाई उचित निर्देशन अनुक्रम कायम गर्न मद्दत गर्ने र इन्जेक्ट गरिएको आदेशहरू बेवास्ता गर्ने

3. **डेलिमिटर र डाटामार्किङ प्रणालीहरू**
   - विश्वसनीय प्रणाली सन्देशहरू र बाह्य इनपुट पाठबीच स्पष्ट सिमाना परिभाषा
   - विशेष मार्करहरूले विश्वसनीय र अविश्वसनीय डाटा स्रोतहरूका बीच सिमानाहरू उजागर गर्छन्
   - स्पष्ट पृथक्करणले निर्देशन भ्रम र अनधिकृत आदेश कार्यान्वयन रोक्छ

4. **लगातार खतरा बुद्धिमत्ता**
   - Microsoft ले लगातार नयाँ आक्रमण ढाँचाहरू अनुगमन गर्दछ र सुरक्षा अद्यावधिकहरू जारी गर्दछ
   - नयाँ इन्जेक्शन प्रविधिहरू र आक्रमण मार्गहरूका लागि सक्रिय खतरा खोजी
   - विकासशील खतराहरु विरुद्ध प्रभावकारिता कायम राख्न नियमित सुरक्षा मोडेल अपडेटहरू

5. **Azure सामग्री सुरक्षा एकीकरण**
   - समग्र Azure AI सामग्री सुरक्षा सूटको एक हिस्सा
   - जेलब्रेक प्रयासहरू, हानिकारक सामग्री, र सुरक्षा नीति उल्लङ्घनहरूको अतिरिक्त पत्ता लगाउने
   - AI अनुप्रयोग कम्पोनेन्टहरूमा एकीकृत सुरक्षा नियन्त्रणहरू

## उन्नत MCP सुरक्षा खतराहरू

### सत्र अपहरण कमजोरिहरू

**सत्र अपहरण** राज्यीय MCP कार्यान्वयनहरूमा एक गम्भीर आक्रमण मार्ग हो जहाँ अनधिकृत पक्षहरूले वैध सत्र परिचायकहरू प्राप्त गरी क्लाइन्टको नक्कल गरी अनधिकृत कार्यहरू गर्न सक्छन्।

#### **आक्रमण परिदृश्यहरू र जोखिमहरू**

- **सत्र अपहरण प्रॉम्प्ट इन्जेक्शन**: चोरी गरिएका सत्र ID भएका आक्रमणकर्ताहरूले सत्र अवस्था साझा गर्ने सर्भरहरूमा दुर्भावनापूर्ण घटनाहरू इन्जेक्ट गर्छन् जसले हानिकारक कार्यहरू गर्न वा संवेदनशील डाटा पहुँच गर्न सक्छ
- **प्रत्यक्ष नक्कली व्यक्ति परिचय**: चोरी गरिएका सत्र IDहरूले प्रमाणीकरण छली बिना सिधा MCP सर्भर कल गर्न सक्षम बनाउँछन्, जसले आक्रमणकर्तालाई वैध प्रयोगकर्ता जस्तै व्यवहार गराउँछ
- **दुर्व्यवस्थित पुनःसञ्चार स्ट्रीमहरू**: आक्रमणकर्ताहरूले अनुरोधहरू पूर्वमै समाप्त गराउन सक्छन्, जसले वैध क्लाइन्टहरूलाई सम्भावित रूपमा दुर्भावनापूर्ण सामग्रीसँग पुनः सुरु गराउँछ

#### **सत्र व्यवस्थापनको लागि सुरक्षा नियन्त्रणहरू**

**महत्वपूर्ण आवश्यकताहरू:**
- **अधिकार प्रमाणीकरण जाँच**: MCP सर्भरहरूले सबै आउने अनुरोधहरूको प्रमाणीकरण गर्नुपर्छ र प्रमाणीकरणका लागि सत्रमा निर्भर हुनु हुँदैन
- **सुरक्षित सत्र सिर्जना**: क्रिप्टोग्राफिक रूपमा सुरक्षित, अनियत सत्र ID हरू प्रयोग गर्ने जुन सुरक्षित र्यान्डम नम्बर जेनेरेटरहरूले बनाईएका हुन्छन्
- **प्रयोगकर्ता-विशिष्ट बाँधिने**: सत्र ID लाई प्रयोगकर्ता-विशिष्ट जानकारीसँग `<user_id>:<session_id>` जस्ता ढाँचामा बाँध्नु जसले अन्तर-प्रयोगकर्ता सत्र दुरुपयोग रोक्छ
- **सत्र जीवनचक्र व्यवस्थापन**: उपयुक्त समाप्ति, घुमाउने, र अमान्य गर्ने प्रक्रिया कार्यान्वयन गर्न ताकि जोखिम सीमित होस्
- **परिवहन सुरक्षा**: सबै संचारका लागि अवश्य HTTPS प्रयोग गर्नुपर्ने ताकि सत्र ID चोरी रोकियोस

### भ्रमित डिपुटी समस्या

**भ्रमित डिपुटी समस्या** तब देखा पर्छ जब MCP सर्भरहरू क्लाइन्ट र तेस्रो-पक्ष सेवाहरूबीच प्रमाणीकरण प्रोक्सीका रूपमा काम गर्छन्, जसले स्थिर क्लाइन्ट ID प्रयोग गरेर अधिकार उल्लंघन गर्ने अवसरहरू सिर्जना गर्छ।

#### **आक्रमण संयन्त्रहरू र जोखिमहरू**

- **कुकी-आधारित सहमति छली**: पहिलेको प्रयोगकर्ता प्रमाणीकरणले सहमति कुकीहरू सिर्जना गर्छ, जुन आक्रमणकर्ताहरूले दुर्भावनापूर्ण अधिकार अनुरोधहरू र कस्टम रिडाइरेक्ट URI हरू मार्फत दुरुपयोग गर्छन्
- **अधिकार कोड चोरी**: विद्यमान सहमति कुकीहरूले अधिकार सर्भरहरूलाई सहमति स्क्रिनहरू छोड्न अनुमति दिन सक्छ, जसले कोडहरू आक्रमणकर्ता नियन्त्रणमा रहेको अन्त्यबिन्दुमा पठाउँछ  
- **अनधिकृत API पहुँच**: चोरी गरिएका अधिकार कोडहरूले टोकन विनिमय गर्न र प्रयोगकर्ता नक्कली बनाउन अनुमति दिन्छ, निर्दिष्ट स्वीकृति बिना

#### **निवारण रणनीतिहरू**

**अनिवार्य नियन्त्रणहरू:**
- **स्पष्ट सहमति आवश्यकताहरू**: स्थिर क्लाइन्ट ID भएका MCP प्रोक्सी सर्भरहरूले प्रत्येक गतिशील रूपमा दर्ता भएका क्लाइन्टका लागि प्रयोगकर्ताको सहमति लिनुपर्छ
- **OAuth 2.1 सुरक्षा कार्यान्वयन**: सबै अधिकार अनुरोधहरूमा PKCE (Proof Key for Code Exchange) सहित हालको OAuth सुरक्षा सर्वोत्तम अभ्यासहरू पालन गर्ने
- **कडा क्लाइन्ट प्रमाणीकरण**: दुरुपयोग रोक्न रिडाइरेक्ट URI र क्लाइन्ट पहिचानकर्ताहरूको कडाइका साथ प्रमाणीकरण गर्ने

### टोकन पासथ्रू कमजोरिहरू  

**टोकन पासथ्रू** एक स्पष्ट विरोधाभास प्रदर्शन गर्छ जहाँ MCP सर्भरहरूले क्लाइन्ट टोकन बिना उचित प्रमाणीकरण स्वीकृति लिए न्यान्ध्य API हरूमा अग्रेषण गर्छन्, जसले MCP अधिकार विशेषताहरू उल्लङ्घन गर्छ।

#### **सुरक्षा प्रभावहरू**

- **नियन्त्रण घुमाउने**: सिधा क्लाइन्ट-बाट-API टोकन प्रयोगले महत्वपूर्ण दर सीमांकन, प्रमाणीकरण, र अनुगमन नियन्त्रणहरू च्यात्छ
- **अडिट ट्रेल बिग्रावट**: माथि जारी गरिएको टोकनहरूले क्लाइन्ट पहिचान असम्भव बनाउँछ, जसले घटना जाँच क्षमताहरूलाई तोड्छ
- **प्रोक्सी-आधारित डाटा चोरी**: अमान्य गरिएको टोकनहरूले दुर्भावनापूर्ण पक्षलाई सर्भरहरूलाई अनधिकृत डाटा पहुँचको प्रोक्सीको रूपमा प्रयोग गर्न अनुमति दिन्छ
- **विश्वास सीमा उल्लंघन**: डाउनस्ट्रीम सेवाहरूको विश्वास मान्यताहरू टोकनका मूलहरू प्रमाणीकरण गर्न नसक्दा उल्लङ्घित हुन सक्छन्
- **बहु-सेवा आक्रमण विस्तार**: धेरै सेवाहरूमा स्वीकार गरिएका दुरुपयोग गरिएका टोकनहरूले फैलावट सक्षम पार्छ

#### **आवश्यक सुरक्षा नियन्त्रणहरू**

**अपरिहार्य आवश्यकताहरू:**
- **टोकन प्रमाणीकरण**: MCP सर्भरहरूले स्पष्ट रूपमा जारी नभएको टोकनहरू स्वीकार्नु हुँदैन
- **प्रेक्षक प्रमाणीकरण**: सदैव टोकनको प्रेक्षक दाबीहरू MCP सर्भरको पहिचानसँग मेल खान्छन् भनी जाँच गर्ने
- **उपयुक्त टोकन जीवनचक्र**: छोटो अवधिका पहुँच टोकनहरू प्रयोग र सुरक्षित घुमाउने अभ्यास कार्यान्वयन गर्ने


## AI प्रणालीहरूका लागि सप्लाई चेन सुरक्षा

सप्लाई चेन सुरक्षा परम्परागत सफ्टवेयर निर्भरताहरूबाट परे गएर सम्पूर्ण AI पारिस्थितिकी प्रणाली समेट्ने गरी विकास भएको छ। आधुनिक MCP कार्यान्वयनहरूले सबै AI सम्बन्धित कम्पोनेन्टहरू कठोर रूपमा प्रमाणीकरण र अनुगमन गर्नुपर्छ, किनकि प्रत्येकले प्रणाली अखण्डता जोखिममा पार्न सक्छ।

### विस्तार गरिएको AI सप्लाई चेन कम्पोनेन्टहरू

**परम्परागत सफ्टवेयर निर्भरताहरू:**
- खुला स्रोत पुस्तकालयहरू र फ्रेमवर्कहरू
- कन्टेनर छविहरू र आधार प्रणालीहरू  
- विकास उपकरण र निर्माण पाइपलाइनहरू
- पूर्वाधार कम्पोनेन्टहरू र सेवा

**AI-विशिष्ट सप्लाई चेन तत्वहरू:**
- **फाउन्डेशन मोडेलहरू**: विभिन्न प्रदायकहरूबाट पूर्व-प्रशिक्षित मोडेलहरू जसको स्रोत सुनिश्चित गर्नुपर्छ
- **एम्बेडिङ सेवाहरू**: बाह्य वेक्टराइजेशन र सेमेन्टिक खोज सेवाहरू
- **सन्दर्भ प्रदायकहरू**: डेटा स्रोतहरू, ज्ञान भण्डारहरू, र कागजात रिपोजिटरीहरू  
- **तेस्रो-पक्ष API हरू**: बाह्य AI सेवा, ML पाइपलाइनहरू, र डाटा प्रशोधन अन्तबिन्दुहरू
- **मोडेल कलाकृतिहरू**: वजनहरू, कन्फिगरेसनहरू, र फाइन-ट्युन गरिएको मोडेल भेरियन्टहरू
- **प्रशिक्षण डेटा स्रोतहरू**: मोडेल प्रशिक्षण र फाइन-ट्युनिंगका लागि प्रयोग गरिएका डाटासेटहरू

### व्यापक सप्लाई चेन सुरक्षा रणनीति

#### **कम्पोनेन्ट प्रमाणीकरण र विश्वास**
- **मूल प्रमाणिकरण**: सबै AI कम्पोनेन्टहरूको उत्पत्ति, लाइसेन्सिङ, र अखण्डता सुनिश्चित गरी समायोजन गर्नु
- **सुरक्षा मूल्यांकन**: मोडेलहरू, डाटा स्रोतहरू, र AI सेवाहरूको सुरक्षा समीक्षा र कमजोरी स्क्यान गर्नु
- **प्रतिष्ठा विश्लेषण**: AI सेवा प्रदायकहरूको सुरक्षा ट्रयाक रेकर्ड र अभ्यासहरू मुल्यांकन गर्नु
- **अनुपालन प्रमाणीकरण**: सबै कम्पोनेन्टहरूले संस्थागत सुरक्षा र नियामक आवश्यकताहरू पूरा गर्छन् भनी सुनिश्चित गर्नु

#### **सुरक्षित तैनाती पाइपलाइनहरू**  
- **स्वचालित CI/CD सुरक्षा**: स्वचालित तैनाती पाइपलाइनहरूभर सुरक्षा स्क्यानिङ समावेश गर्नु
- **कलाकृति अखण्डता**: सबै तैनाथ कलाकृतिहरू (कोड, मोडेल, कन्फिगरेसन) को क्रिप्टोग्राफिक प्रमाणीकरण कार्यान्वयन गर्न
- **चरणबद्ध तैनाती**: प्रत्येक चरणमा सुरक्षा प्रमाणीकरणसहित प्रगतिशील तैनाती रणनीतिहरू प्रयोग गर्नु
- **विश्वसनीय कलाकृति रिपोजिटरीहरू**: प्रमाणित, सुरक्षित कलाकृति भण्डार र रिपोजिटरीहरूबाट मात्र तैनाती

#### **लगातार अनुगमन र जवाफ**
- **निर्भरता स्क्यानिङ**: सबै सफ्टवेयर र AI कम्पोनेन्ट निर्भरतालाई निरन्तर कमजोरि अनुगमन
- **मोडेल अनुगमन**: मोडेल व्यवहार, प्रदर्शन विचलन, र सुरक्षा विषमताहरूको निरन्तर मूल्यांकन
- **सेवा स्वास्थ्य ट्र्याकिङ**: बाह्य AI सेवाहरूको उपलब्धता, सुरक्षा घटना र नीति परिवर्तनहरूको अनुगमन
- **खतरा बुद्धिमत्ता एकीकरण**: AI र ML सुरक्षा जोखिमहरूका लागि विशिष्ट खतरा फिडहरू समावेश

#### **पहुँच नियन्त्रण र न्यूनतम विशेषाधिकार**
- **कम्पोनेन्ट-स्तरीय अनुमति**: मोडेलहरू, डाटा, र सेवाहरूमा व्यवसाय आवश्यकताअनुसार पहुँच सीमित गर्नु
- **सेवा खाता व्यवस्थापन**: न्यूनतम आवश्यक अनुमति भएका समर्पित सेवा खाता कार्यान्वयन गर्नु
- **नेटवर्क खण्डीकरण**: AI कम्पोनेन्टहरू अलग पारेर सेवाहरूबीच नेटवर्क पहुँच सीमित गर्नु
- **API गेटवे नियन्त्रणहरू**: बाह्य AI सेवाहरूको पहुँच नियन्त्रण र अनुगमनका लागि केन्द्रित API गेटवेहरू प्रयोग गर्नु

#### **घटना प्रतिक्रिया र पुन: प्राप्ति**
- **द्रुत प्रतिक्रिया प्रक्रिया**: दुरुपयोग गरिएका AI कम्पोनेन्टहरूको लागि छिटो प्याच वा प्रतिस्थापन प्रक्रिया लागू गर्नु
- **प्रमाणपत्र घुमाव**: गोप्य जानकारी, API कुञ्जीहरू, र सेवा प्रमाणपत्रहरू स्वचालित रूपमा घुमाउने प्रणालीहरू
- **रोलब्याक क्षमता**: पहिलेको ज्ञात राम्रो संस्करणमा छिटो फिर्ता जानसक्ने क्षमता
- **सप्लाई चेन उल्लंघन पुन: प्राप्ति**: माथिल्लो स्रोत AI सेवा उल्लंघनहरूको प्रतिक्रिया प्रक्रियाहरू

Microsoft ले सबै उत्पादनहरूमा व्यापक सप्लाई चेन सुरक्षा अभ्यासहरू कार्यान्वयन गर्छ। [The Journey to Secure the Software Supply Chain at Microsoft](https://devblogs.microsoft.com/engineering-at-microsoft/the-journey-to-secure-the-software-supply-chain-at-microsoft/) मा प्रमाणित दृष्टिकोणहरूबारे जान्नुहोस्।


## आधारभूत सुरक्षा सर्वोत्तम अभ्यासहरू

MCP कार्यान्वयनहरूले तपाईंको संस्थाको विद्यमान सुरक्षा स्थिति ग्रहण गर्छ र त्यसमा निर्माण गर्छ। आधारभूत सुरक्षा अभ्यासहरू सुदृढ पार्दा AI प्रणालीहरू र MCP तैनातीहरूको समग्र सुरक्षा अत्यधिक सुधार हुन्छ।

### मुख्य सुरक्षा आधारहरू

#### **सुरक्षित विकास अभ्यासहरू**
- **OWASP अनुपालन**: [OWASP शीर्ष १०](https://owasp.org/www-project-top-ten/) वेब अनुप्रयोग कमजोरिहरूबाट सुरक्षा
- **AI-विशिष्ट सुरक्षा**: [OWASP शीर्ष १० LLMs](https://genai.owasp.org/download/43299/?tmstv=1731900559) को लागि नियन्त्रणहरू कार्यान्वयन
- **सुरक्षित गोप्य व्यवस्थापन**: टोकन, API कुञ्जी, र संवेदनशील कन्फिगरेसन डाटाका लागि समर्पित भल्टहरू प्रयोग
- **एन्ड-टु-एन्ड इन्क्रिप्सन**: सबै अनुप्रयोग कम्पोनेन्टहरू र डाटा प्रवाहहरूमा सुरक्षित संचार कार्यान्वयन
- **इनपुट प्रमाणीकरण**: सबै प्रयोगकर्ता इनपुट, API प्यारामिटरहरू, र डेटा स्रोतहरू कडाइका साथ जाँच

#### **पूर्वाधार कठोरता**
- **बहु-कारक प्रमाणीकरण**: सबै प्रशासकीय र सेवा खाताहरूका लागि अनिवार्य MFA
- **प्याच व्यवस्थापन**: अपरेटिङ सिस्टम, फ्रेमवर्क, र निर्भरतालाई स्वचालित, समयमा प्याच गर्न  
- **पहिचान प्रदायक एकीकरण**: संस्थागत पहिचान प्रदायकहरू (Microsoft Entra ID, Active Directory) मार्फत केन्द्रित पहिचान व्यवस्थापन
- **नेटवर्क खण्डीकरण**: MCP कम्पोनेन्टहरूको तार्किक अलगावले दिगो आन्दोलन सीमित गर्नु
- **न्यूनतम विशेषाधिकार सिद्धान्त**: सबै प्रणाली कम्पोनेन्ट र खाताहरूमाझ न्यूनतम आवश्यक अनुमति

#### **सुरक्षा अनुगमन र पत्ता लगाउने**
- **सम्पूर्ण लॉगिङ**: AI अनुप्रयोग गतिविधिहरूको विस्तृत लॉगिङ, MCP क्लाइन्ट-सर्भर अन्तरक्रियाहरू समेटेर
- **SIEM एकीकरण**: असामान्य गतिविधि पत्ता लगाउन केन्द्रित सुरक्षा सूचना र घटना व्यवस्थापन
- **व्यावहारिक विश्लेषण**: प्रणाली र प्रयोगकर्ता व्यवहारमा अनौठा ढाँचाहरू पत्ता लगाउन AI-सञ्चालित अनुगमन
- **खतरा बुद्धिमत्ता**: बाह्य खतरा फीडहरू र कम्प्रोमाइज सूचकहरू समावेश
- **घटना प्रतिक्रिया**: सुरक्षा घटना पत्ता लगाउने, प्रतिक्रिया गर्ने, र पुन: प्राप्ति प्रक्रिया

#### **शून्य विश्वास वास्तुकला**
- **कहिल्यै विश्वास नगर्नुहोस्, सधैं जाँच गर्नुहोस्**: प्रयोगकर्ता, उपकरण, र नेटवर्क जडानहरूको निरन्तर प्रमाणीकरण
- **माइक्रो-सेग्मेन्टेसन**: व्यक्तिगत कार्यभार र सेवाहरू अलग राख्ने सूक्ष्म नेटवर्क नियन्त्रणहरू
- **पहिचान-केन्द्रित सुरक्षा**: नेटवर्क स्थान होइन, प्रमाणित पहिचानमा आधारित सुरक्षा नीतिहरू
- **निरन्तर जोखिम मूल्यांकन**: वर्तमान सन्दर्भ र व्यवहारअनुसार गतिशील सुरक्षा स्थिति मूल्यांकन
- **सशर्त पहुँच**: जोखिम कारक, स्थान, र उपकरण विश्वासमा आधारित अनुकूल पहुँच नियन्त्रणहरू

### उद्यम एकीकरण ढाँचाहरू

#### **Microsoft सुरक्षा पारिस्थितिकी प्रणाली एकीकरण**
- **Microsoft Defender for Cloud**: व्यापक क्लाउड सुरक्षा स्थिति व्यवस्थापन
- **Azure Sentinel**: AI कार्यभार सुरक्षाका लागि क्लाउड-आधारित SIEM र SOAR क्षमता
- **Microsoft Entra ID**: सशर्त पहुँच नीतिहरू सहित उद्यम पहिचान र पहुँच व्यवस्थापन
- **Azure Key Vault**: हार्डवेयर सुरक्षा मोड्युल (HSM) समर्थित केन्द्रित गोपनीय व्यवस्थापन
- **Microsoft Purview**: AI डाटा स्रोतहरू र कार्यप्रवाहहरूको डाटा शासन र अनुपालन

#### **अनुपालन र सरकार व्यवस्था**
- **नियमक समन्वय**: MCP कार्यान्वयनहरूले उद्योग-विशिष्ट अनुपालन आवश्यकताहरू (GDPR, HIPAA, SOC 2) पूरा गरुन्
- **डाटा वर्गीकरण**: AI प्रणालीहरूले प्रशोधन गर्ने संवेदनशील डाटाको उचित वर्गीकरण र व्यवस्थापन
- **लेखा परीक्षा ट्रेल**: नियामक अनुपालन र फोरेंसिक अनुगमनका लागि व्यापक लॉगिङ
- **गोपनीयता नियन्त्रण**: AI प्रणाली वास्तुकलामा गोपनीयता-द्वारा-डिजाइन सिद्धान्तहरूको कार्यान्वयन
- **परिवर्तन व्यवस्थापन**: AI प्रणाली संशोधनहरूको सुरक्षा समीक्षाका लागि औपचारिक प्रक्रिया

यी आधारभूत अभ्यासहरूले MCP-विशिष्ट सुरक्षा नियन्त्रणहरूको प्रभावकारितामा वृद्धि गर्छ र AI-चालित अनुप्रयोगहरूको सुसंगत सुरक्षा सुनिश्चित गर्छ।

## मुख्य सुरक्षा सङ्क्षेपहरू
- **स्तरित सुरक्षा दृष्टिकोण**: आधारभूत सुरक्षा अभ्यासहरू (सुरक्षित कोडिङ, न्यूनतम विशेषाधिकार, आपूर्ति श्रृंखला सत्यापन, निरन्तर अनुगमन) लाई AI-विशेष नियन्त्रणहरूसँग संयोजन गरी व्यापक सुरक्षा प्रदान गर्नुहोस्

- **AI-विशेष खतराको परिदृश्य**: MCP प्रणालीहरूले विशेष जोखिमहरू जस्तै प्रॉम्प्ट इन्जेक्शन, उपकरण विषाक्तता, सेसन अपहरण, भ्रमित उपप्रहरी समस्या, टोकन पासथ्रू कमजोरियाँ, र अत्यधिक अनुमति जसको लागि विशेष सुरक्षा उपायहरू आवश्यक हुन्छन्, सामना गर्छन्

- **प्रमाणीकरण र अधिकार प्राप्ति उत्कृष्टता**: बाह्य पहिचान प्रदायकहरू (Microsoft Entra ID) प्रयोग गरी बलियो प्रमाणीकरण लागू गर्नुहोस्, उचित टोकन मान्यकरण कडाइका साथ गर्नुहोस्, र कहिल्यै पनि त्यो टोकनहरू स्वीकार नगर्नुहोस् जुन तपाईंको MCP सर्भरका लागि खुल्ला रूपमा जारी नगरिएका हुन्

- **AI आक्रमण रोकथाम**: Microsoft Prompt Shields र Azure Content Safety प्रयोग गरी अप्रत्यक्ष प्रॉम्प्ट इन्जेक्शन र उपकरण विषाक्तता आक्रमणबाट रक्षा गर्नुहोस्, उपकरण मेटाडाटा प्रमाणीकरण गर्नुहोस् र गतिशील परिवर्तनहरूको निगरानी गर्नुहोस्

- **सेसन र ट्रान्सपोर्ट सुरक्षा**: क्रिप्टोग्राफिक रूपमा सुरक्षित, गैर-निर्धारित सेसन ID हरू प्रयोग गर्नुहोस् जुन प्रयोगकर्ता पहिचानसँग बाँधिएका हुन्छन्, सेसन जीवनचक्र व्यवस्थापन ठीकसँग लागू गर्नुहोस्, र प्रमाणीकरणका लागि कहिल्यै सेसनहरू प्रयोग नगर्नुहोस्

- **OAuth सुरक्षा उत्कृष्ट अभ्यासहरू**: गतिशील रूपमा दर्ता गरिएका क्लाइन्टहरूको लागि स्पष्ट प्रयोगकर्ता सहमतिको माध्यमबाट भ्रमित उपप्रहरी आक्रमणहरू रोक्नुहोस्, PKCE सहित OAuth 2.1 कडाइका साथ लागू गर्नुहोस्, र रिडिरेक्ट URI कडाइका साथ मान्य गर्नुहोस्

- **टोकन सुरक्षा सिद्धान्तहरू**: टोकन पासथ्रू एन्टी-प्याटर्नहरूबाट बच्नुहोस्, टोकन दर्शक दाबीहरू मान्य गर्नुहोस्, सुरक्षित घुमावसहित छोटो अवधिका टोकनहरू लागू गर्नुहोस्, र स्पष्ट विश्वास सीमाहरू कायम राख्नुहोस्

- **व्यापक आपूर्ति श्रृंखला सुरक्षा**: सबै AI पारिस्थितिकी तन्त्र कम्पोनेन्टहरू (मोडेलहरू, एम्बेडिङहरू, सन्दर्भ प्रदायकहरू, बाह्य API हरू) लाई परम्परागत सफ्टवेयर निर्भरता जस्तै नै सुरक्षा कडाइका साथ व्यवहार गर्नुहोस्

- **निरन्तर विकास**: छिटो विकसित हुँदै गरेका MCP विशिष्टताहरू अनुसार अद्यावधिक रहनुहोस्, सुरक्षा समुदायका मापदण्डहरूमा योगदान गर्नुहोस्, र प्रोटोकल परिपक्व हुँदै जाँदा अनुकुलनीय सुरक्षा अवस्थाहरू कायम राख्नुहोस्

- **Microsoft सुरक्षा एकीकरण**: Microsoft को व्यापक सुरक्षा पारिस्थितिकी तन्त्र (Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID) लाई MCP परिनियोजन संरक्षणका लागि उपयोग गर्नुहोस्
