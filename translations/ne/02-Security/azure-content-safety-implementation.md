# MCP सँग Azure Content Safety कार्यान्वयन गर्नुहोस्

> **OWASP MCP जोखिम सम्बोधन**: [MCP06 - Intent Flow Subversion](https://microsoft.github.io/mcp-azure-security-guide/mcp/mcp06-prompt-injection/)

प्रॉम्प्ट इन्जेक्शन, टुल पोइसनिङ, र अन्य AI-विशिष्ट जोखिमहरू विरुद्ध MCP सुरक्षा बलियो बनाउन, Azure Content Safety एकीकृत गर्नु अत्यन्त सिफारिस गरिएको छ। यो कार्यान्वयन गाइड [MCP सुरक्षा सम्मेलन कार्यशाला (Sherpa)](https://azure-samples.github.io/sherpa/) क्याम्प ३: I/O सुरक्षा सँग मेल खान्छ।

## MCP सर्भरसँग एकीकरण

Azure Content Safety लाई तपाईको MCP सर्भरमा एकीकृत गर्न, तपाईको अनुरोध प्रक्रिया पाइपलाइनमा कन्टेन्ट सेफ्टी फिल्टरलाई मिडलवेयरको रूपमा थप्नुहोस्:

1. सर्भर सुरु हुँदा फिल्टरलाई आरम्भ गर्नुहोस्
2. प्रशोधन गर्नुअघि सबै आउने टुल अनुरोधहरू मान्य गर्नुहोस्
3. क्लाइन्टहरूलाई फर्काउनु अघि सबै बाहिरिने प्रतिक्रियाहरू जाँच्नुहोस्
4. सुरक्षा उल्लङ्घनहरूमा लग र सचेत गर्नुहोस्
5. असफल कन्टेन्ट सेफ्टी जाँचका लागि उपयुक्त त्रुटि ह्यान्डलिङ कार्यान्वयन गर्नुहोस्

यसले निम्नविपक्षहरू विरुद्ध मजबुत सुरक्षा प्रदान गर्छ:
- प्रॉम्प्ट इन्जेक्शन आक्रमणहरू
- टुल पोइसनिङ प्रयासहरू
- दुष्ट इनपुटहरू मार्फत डेटा चोरी
- हानिकारक कन्टेन्ट उत्पादन

## Azure Content Safety एकीकरणका लागि उत्कृष्ट अभ्यासहरू

1. **कस्टम ब्लकलिस्टहरू**: MCP इन्जेक्शन ढाँचाहरूका लागि विशेष कस्टम ब्लकलिस्टहरू बनाउनुहोस्
2. **गम्भीरता सन्तुलन**: तपाईको विशेष प्रयोग केस र जोखिम सहिष्णुताका आधारमा गम्भीरता सीमा समायोजन गर्नुहोस्
3. **व्यापक आवरण**: सबै इनपुट र आउटपुटहरूमा कन्टेन्ट सेफ्टी जाँचहरू लागू गर्नुहोस्
4. **प्रदर्शन अनुकूलन**: पुनरावृत्ति कन्टेन्ट सेफ्टी जाँचहरूका लागि क्यासिङ लागू गर्ने सोच्नुहोस्
5. **फलब्याक मेकानिजमहरू**: कन्टेन्ट सेफ्टी सेवाहरू अनुपलब्ध हुँदा स्पष्ट फलब्याक व्यवहार परिभाषित गर्नुहोस्
6. **प्रयोगकर्ता प्रतिक्रियाहरू**: जब कन्टेन्ट सुरक्षाका कारण ब्लक हुन्छ, प्रयोगकर्ताहरूलाई स्पष्ट प्रतिक्रिया प्रदान गर्नुहोस्
7. **लगातार सुधार**: उदाउने खतराहरूको आधारमा ब्लकलिस्ट र ढाँचाहरू नियमित रूपमा अपडेट गर्नुहोस्
