# 🚀 १० वटा माइक्रोसफ्ट MCP सर्भरहरू जसले विकासकर्ता उत्पादनशीलता परिवर्तन गर्दैछन्

## 🎯 तपाईं यस मार्गदर्शनमा के सिक्नुहुनेछ

यो व्यावहारिक मार्गदर्शनले दस माइक्रोसफ्ट MCP सर्भरहरू प्रदर्शन गर्दछ जुन सक्रिय रूपमा विकासकर्ताले AI सहायकहरू सँग कसरी काम गर्छन् भन्ने तरिकामा रूपान्तरण गर्दैछन्। केबल MCP सर्भरहरूले *के* गर्न सक्छन् भनेर मात्र व्याख्या गर्नुको सट्टा, हामी तपाईंलाई ती सर्भरहरू देखाउँछौं जुन माईक्रोसफ्टमा दैनिक विकास कार्यप्रवाहहरूमा र यसको बाहिर पहिले नै वास्तविक प्रभाव पारिरहेका छन्।

यस मार्गदर्शनमा प्रत्येक सर्भर वास्तविक-विश्व प्रयोग र विकासकर्ता प्रतिक्रिया अनुसार छानिएको छ। तपाईं प्रत्येक सर्भरले के गर्छ भन्ने मात्र होइन, किन यो महत्वपूर्ण छ र कसरी तपाईं आफ्नै परियोजनाहरूमा यसको अधिकतम लाभ लिन सक्नुहुन्छ भन्ने पनि पत्ता लगाउनुहुनेछ। तपाईं MCP मा पूरै नयाँ हुनुहुन्छ वा तपाईंको विद्यमान सेटअप विस्तार गर्न चाहानुहुन्छ भने यी सर्भरहरूले माइक्रोसफ्ट इकोसिस्टममा उपलब्ध सबैभन्दा व्यावहारिक र प्रभावकारी उपकरणहरू मध्ये केही प्रतिनिधित्व गर्दछन्।

> **💡 छिटो सुरुवात सुझाव**
> 
> MCP मा नयाँ हुनुहुन्छ? चाँडो चिन्ता नगर्नुहोस्! यो मार्गदर्शन शुरुवातीहरूको लागि डिजाइन गरिएको छ। हामी जाँदाँ अवधारणाहरू व्याख्या गर्छौं, र तपाईं सँधै हाम्रो [MCP परिचय](../00-Introduction/README.md) र [मूल अवधारणाहरू](../01-CoreConcepts/README.md) मोड्युलहरूमा फिर्ता सन्दर्भ गर्न सक्नुहुन्छ गहिरो पृष्ठभूमिको लागि।

## अवलोकन

यो व्यापक मार्गदर्शनले दस माइक्रोसफ्ट MCP सर्भरहरूको अन्वेषण गर्दछ जसले विकासकर्ताहरूलाई AI सहायकहरू र बाह्य उपकरणहरूसँग अन्तरक्रिया गर्ने तरिकामा क्रान्ति ल्याइरहेको छ। Azure स्रोत व्यवस्थापनबाट लिएर दस्तावेज प्रक्रिया सम्म, यी सर्भरहरूले विकासकार्य प्रक्रियाहरूलाई सहज र उत्पादनशील बनाउन Model Context Protocol को शक्तिलाई प्रदर्शन गर्छन्।

## सिकाइ उद्देश्यहरू

यो मार्गदर्शनको अन्त्य सम्म, तपाईंले:
- MCP सर्भरहरूले विकासकर्ता उत्पादनशीलता कसरी बढाउँछन् बुझ्ने
- माइक्रोसफ्टका सबैभन्दा प्रभावकारी MCP सर्भर कार्यान्वयनहरूका बारेमा सिक्ने
- प्रत्येक सर्भरका व्यवहारिक प्रयोगहरूको पत्ता लगाउने
- VS Code र Visual Studio मा यी सर्भरहरू कसरी सेटअप र कन्फिगर गर्ने थाहा पाउने
- व्यापक MCP इकोसिस्टम र भविष्यका दिशाहरू अन्वेषण गर्ने

## 🔧 MCP सर्भरहरू बुझ्ने: शुरुवातीहरूको मार्गदर्शक

### MCP सर्भरहरू के हुन्?

Model Context Protocol (MCP) मा नयाँ हुनुहुन्छ भने, तपाईंलाई लाग्न सक्छ: "MCP सर्भर के हो, र यो किन महत्वपूर्ण छ?" साधारण उपमा देखि शुरू गरौं।

MCP सर्भरहरूलाई विशेष सहायकहरूका रूपमा सोच्नुहोस् जसले तपाईंको AI कोडिङ साथी (जस्तै GitHub Copilot) लाई बाह्य उपकरणहरू र सेवासँग जोड्न मद्दत गर्छ। जस्तै तपाईं आफ्नो फोनमा विभिन्न कार्यका लागि विभिन्न एपहरू प्रयोग गर्नुहुन्छ—एक मौसमको लागि, अर्को नेभिगेसनको लागि, अर्को बैंकिङको लागि—MCP सर्भरहरूले तपाईंको AI सहायकलाई विभिन्न विकास उपकरण र सेवासँग अन्तरक्रिया गर्ने क्षमता दिन्छ।

### MCP सर्भरहरूले समाधान गर्ने समस्या

MCP सर्भरहरू आउनुअघि, यदि तपाईंले:
- आफ्नो Azure स्रोतहरू जाँच्न
- GitHub मुद्दा सिर्जना गर्न
- आफ्नो डेटाबेस सोध्न
- दस्तावेजीकरणमा खोजी गर्न

चाहनुभएको थियो भने, तपाईंले कोडिङ रोक्नुपर्ने, ब्राउजर खोल्नुपर्ने, उपयुक्त वेबसाइटमा जानुपर्ने र यी कार्यहरू म्यानुअली गर्नु पर्ने थियो। यो निरन्तर सन्दर्भ स्विचिङले तपाईंको प्रवाह भङ्ग गर्छ र उत्पादनशीलता घटाउँछ।

### MCC सर्भरहरूले तपाईंको विकास अनुभव कसरी रूपान्तरण गर्छन्

MCP सर्भरहरूसँग, तपाईं आफ्नो विकास वातावरण (VS Code, Visual Studio, आदि) मा बस्न सक्नुहुन्छ र सजिलै आफ्नो AI सहायकलाई यी कार्यहरू सुम्पन सक्नुहुन्छ। उदाहरणका लागि:

**परम्परागत कार्यप्रवाहको सट्टा:**
1. कोडिङ रोक्नुहोस्
2. ब्राउजर खोल्नुहोस्
3. Azure पोर्टलमा जानुहोस्
4. भण्डारण खाता विवरण हेर्नुहोस्
5. VS Code मा फर्किनुहोस्
6. कोडिङ पुन: सुरु गर्नुहोस्

**तपाईं अब गर्न सक्नुहुन्छ:**
1. AI लाई सोध्नुहोस्: "मेरा Azure भण्डारण खाताहरूको स्थिति के छ?"
2. दिइएको जानकारीसहित कोडिङ जारी राख्नुहोस्

### शुरुवातीहरूको लागि मुख्य लाभहरू

#### 1. 🔄 **तपाईंको प्रवाह अवस्थामा बस्न सक्नुहुन्छ**
- धेरै अनुप्रयोगहरू बीच स्विच गर्न आवश्यक छैन
- तपाईं लेखिरहेको कोडमा ध्यान केन्द्रित गर्नुहोस्
- विभिन्न उपकरणहरूको व्यवस्थापन गर्ने मानसिक भार कम गर्नुहोस्

#### 2. 🤖 **जटिल आदेशहरूको सट्टा प्राकृतिक भाषा प्रयोग गर्नुहोस्**
- SQL सिन्ट्याक्स सम्झिनुको सट्टा चाहिने डाटा वर्णन गर्नुहोस्
- Azure CLI आदेशहरू सम्झिनुको सट्टा के गर्न चाहनुहुन्छ व्याख्या गर्नुहोस्
- प्राविधिक विवरण AI लाई ह्यान्डल गर्न दिनुहोस् र तपाईं तर्कमा ध्यान दिनुहोस्

#### 3. 🔗 **विभिन्न उपकरणहरूलाई जोड्नुहोस्**
- विभिन्न सेवाहरूलाई संयोजन गरेर शक्तिशाली कार्यप्रवाहहरू बनाउनुहोस्
- उदाहरण: "हालैका सबै GitHub मुद्दाहरू प्राप्त गर्नुहोस् र सम्बन्धित Azure DevOps कार्य वस्तुहरू सिर्जना गर्नुहोस्"
- जटिल स्क्रिप्ट बिना स्वचालन निर्माण गर्नुहोस्

#### 4. 🌐 **विकसित हुँदै गरेको इकोसिस्टममा पहुँच पाउनुहोस्**
- माइक्रोसफ्ट, GitHub, र अन्य कम्पनीहरूले बनाएको सर्भरहरूबाट लाभ उठाउनुहोस्
- विभिन्न विक्रेताहरूका उपकरणहरू सहजै मिलाउनुहोस्
- विभिन्न AI सहायकहरूमा काम गर्ने मानकीकृत इकोसिस्टममा सामेल हुनुहोस्

#### 5. 🛠️ **गर्न सिक्नुहोस्**
- अवधारणाहरू बुझ्न पूर्व-निर्मित सर्भरहरूबाट सुरु गर्नुहोस्
- आरामदायक भएपछि आफ्नै सर्भरहरू क्रमशः निर्माण गर्नुहोस्
- उपलब्ध SDKs र दस्तावेजहरूको प्रयोगले सिकाइलाई मार्गनिर्देशन गर्नुहोस्

### शुरुवातीहरूको लागि वास्तविक उदाहरण

मानौं तपाईं वेब विकासमा नयाँ हुनुहुन्छ र पहिलो परियोजनामा काम गर्दै हुनुहुन्छ। यहाँ MCP सर्भरहरूले कसरी मद्दत गर्न सक्छन्:

**परम्परागत विधि:**
```
1. Code a feature
2. Open browser → Navigate to GitHub
3. Create an issue for testing
4. Open another tab → Check Azure docs for deployment
5. Open third tab → Look up database connection examples
6. Return to VS Code
7. Try to remember what you were doing
```

**MCP सर्भरहरूसँग:**
```
1. Code a feature
2. Ask AI: "Create a GitHub issue for testing this login feature"
3. Ask AI: "How do I deploy this to Azure according to the docs?"
4. Ask AI: "Show me the best way to connect to my database"
5. Continue coding with all the information you need
```

### एन्त्रप्राइज स्ट्यान्डर्ड लाभ

MCP उद्योगव्यापी मानक बन्दैछ, जसको अर्थ:
- **संगतता**: विभिन्न उपकरण र कम्पनीहरूमा समान अनुभव
- **अन्तःक्रियाशीलता**: विभिन्न विक्रेताहरूका सर्भरहरू सँगै काम गर्छन्
- **भविष्य सुनिश्चितता**: सीप र सेटअपहरू विभिन्न AI सहायकहरूमा स्थानान्तरण हुन्छन्
- **समुदाय**: साझा ज्ञान र स्रोतहरूको ठूलो इकोसिस्टम

- सामान्य विकास चुनौतीहरू समाधान गर्न

## VS Code र Visual Studio मा MCP सर्भरहरू सेटअप गर्ने 🚀

यदि तपाईं Visual Studio Code वा Visual Studio 2022 GitHub Copilot सँग प्रयोग गर्दै हुनुहुन्छ भने यी MCP सर्भरहरू सेटअप गर्न सजिलो छ।

### VS Code सेटअप

VS Code का लागि आधारभूत प्रक्रिया यसप्रकार छ:

1. **एजेन्ट मोड सक्षम गर्नुहोस्**: VS Code मा Copilot Chat विन्डोमा एजेन्ट मोडमा स्विच गर्नुहोस्  
2. **MCP सर्भरहरू कन्फिगर गर्नुहोस्**: तपाईंको VS Code settings.json फाइलमा सर्भर कन्फिगरेसनहरू थप्नुहोस्  
3. **सर्भरहरू सुरु गर्नुहोस्**: तपाईं प्रयोग गर्न चाहने प्रत्येक सर्भरको लागि "Start" बटन क्लिक गर्नुहोस्  
4. **औजारहरू चयन गर्नुहोस्**: तपाईंको वर्तमान सत्रका लागि सक्रिय गर्न चाहिने MCP सर्भरहरू छान्नुहोस्  

विशिष्ट सेटअप निर्देशनहरूको लागि हेर्नुहोस् [VS Code MCP दस्तावेज](https://code.visualstudio.com/docs/copilot/copilot-mcp)।

> **💡 प्रो सुझाव: MCP सर्भरहरूलाई प्रबन्ध गर्न प्रो जस्तो!**  
> 
> VS Code एक्सटेन्सन भ्युमा अहिले [स्थापित MCP सर्भरहरू प्रबन्ध गर्न नयाँ सरल UI](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode) उपलब्ध छ! तपाईंले कुनै पनि स्थापित MCP सर्भरहरू सुरु, रोक्न, र व्यवस्थापन गर्न छिटो पहुँच पाउनुहुनेछ। अवश्य प्रयास गर्नुहोस्!

### Visual Studio 2022 सेटअप

Visual Studio 2022 (संस्करण १७.१४ वा पछि) का लागि:

1. **एजेन्ट मोड सक्षम गर्नुहोस्**: GitHub Copilot Chat विन्डोमा "Ask" ड्रपडाउनमा क्लिक गरी "Agent" चयन गर्नुहोस्  
2. **कन्फिगरेसन फाइल सिर्जना गर्नुहोस्**: तपाईंको समाधान निर्देशिका मा `.mcp.json` फाइल बनाउनुहोस् (सुझाव गरिएको स्थान: `<SOLUTIONDIR>\.mcp.json`)  
3. **सर्भरहरू कन्फिगर गर्नुहोस्**: मानक MCP ढाँचामा तपाईंका सर्भर कन्फिगरेसनहरू थप्नुहोस्  
4. **औजार स्वीकृति दिनुहोस्**: तपाईंले प्रयोग गर्न चाहने उपकरणहरूका लागि उपयुक्त स्कोप अनुमति स्वीकृत गर्नुहोस्  

विशिष्ट Visual Studio सेटअप निर्देशनहरूको लागि हेर्नुहोस् [Visual Studio MCP दस्तावेज](https://learn.microsoft.com/visualstudio/ide/mcp-servers)।

प्रत्येक MCP सर्भर आफ्नै कन्फिगुरेशन आवश्यकताहरू (कनेक्शन स्ट्रिङहरू, प्रमाणीकरण आदि) राख्दछ, तर दुबै IDE मा सेटअप ढाँचा समान छ।

[![VS Code मा स्थापना गर्नुहोस्](https://img.shields.io/badge/VS_Code-Install_Microsoft_Docs_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D) [![VS Code Insiders मा स्थापना गर्नुहोस्](https://img.shields.io/badge/VS_Code_Insiders-Install_Microsoft_Docs_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/mcp)

**यसले के गर्छ**: Microsoft Learn Docs MCP Server एक क्लाउड-होस्ट गरिएको सेवा हो जसले AI सहायकहरूलाई Model Context Protocol मार्फत आधिकारिक माइक्रोसफ्ट दस्तावेजीकरणमा वास्तविक-समय पहुँच प्रदान गर्दछ। यो `https://learn.microsoft.com/api/mcp` सँग जडान हुन्छ र Microsoft Learn, Azure दस्तावेजीकरण, Microsoft 365 दस्तावेजीकरण, र अन्य आधिकारिक माइक्रोसफ्ट स्रोतहरूमा सेम्यान्टिक खोज सक्षम गर्छ।

**किन उपयोगी छ**: यो "केवल दस्तावेजीकरण" जस्तो लाग्न सक्छ, तर यो सर्भर माइक्रोसफ्ट प्रविधिहरू प्रयोग गर्ने प्रत्येक विकासकर्ताको लागि अत्यन्त महत्वपूर्ण छ। .NET विकासकर्ताहरूको AI कोडिङ सहायकहरूका बारेमा सबैभन्दा ठूलो गुनासो हो कि तिनीहरू नवीनतम .NET र C# रिलीजहरूमा अद्यावधिक छैनन्। Microsoft Learn Docs MCP Server ले यसको समाधान गर्छ वास्तविक-समय पहुँच प्रदान गरेर सबैभन्दा वर्तमान दस्तावेजीकरण, API सन्दर्भहरू, र सर्वोत्तम अभ्यासहरूमा। तपाईं नवीनतम Azure SDKहरू, नयाँ C# 13 सुविधाहरू अन्वेषण गर्दै हुनुहुन्छ वा नयाँ .NET Aspire ढाँचा लागू गर्दै हुनुहुन्छ भने पनि, यसले तपाईंको AI सहायकलाई त्यतिन्जेल उपलब्ध अधिकारिक, अद्यावधिक जानकारीसम्म पहुँच सुनिश्चित गर्दछ जुन सही र आधुनिक कोड उत्पादन गर्न आवश्यक छ।

**वास्तविक-विश्व प्रयोग**: "आधिकारिक Microsoft Learn दस्तावेज अनुसार Azure कन्टेनर एप बनाउनका लागि az cli आदेशहरू के हुन्?" वा "ASP.NET Core मा Dependency Injection सँग Entity Framework कसरी कन्फिगर गर्ने?" वा "यो कोड Microsoft Learn दस्तावेजीकरणका प्रदर्शन सिफारिसहरूसँग मेल खान्छ कि छैन समीक्षा गर्नुहोस्।" सर्भरले Microsoft Learn, Azure docs, र Microsoft 365 दस्तावेजीकरणमा व्यापक समेटिएको छ, उन्नत सेम्यान्टिक खोज प्रयोग गरी सबैभन्दा सान्दर्भिक जानकारी फेला पार्दछ। यो लेख शीर्षकहरू र URL सहित १० भन्दा बढी गुणस्तरीय सामग्री भागहरू फिर्ता दिन्छ, सधैं नवीनतम माइक्रोसफ्ट दस्तावेज प्रकाशन हुँदा पहुँच गर्न सक्षम।

**विशेष उदाहरण**: सर्भरले `microsoft_docs_search` उपकरणलाई एक्सपोज गर्छ जुन माइक्रोसफ्टको आधिकारिक प्राविधिक दस्तावेजीकरणमा सेम्यान्टिक खोज प्रदर्शन गर्दछ। कन्फिगर गरेपछि, तपाईं यसरी प्रश्नहरू सोध्न सक्नुहुन्छ: "ASP.NET Core मा JWT प्रमाणीकरण कसरी कार्यान्वयन गर्ने?" र विस्तृत, आधिकारिक जवाफहरू स्रोत लिङ्कहरूसहित प्राप्त गर्नुहोस्। खोज गुणस्तर असाधारण छ किनभने यसले सन्दर्भ बुझ्दछ – Azure सन्दर्भमा "कन्टेनरहरू" को बारेमा प्रश्न गर्दा Azure Container Instances दस्तावेज फर्काउँछ, जबकि .NET सन्दर्भमा उही शब्द प्रयोग गर्दा सम्बन्धित C# सङ्कलन जानकारी प्राप्त हुन्छ।

यो तीव्र रूपमा परिवर्तन हुँदै गरेको वा भर्खरै अपडेट गरिएको पुस्तकालयहरू र प्रयोगहरूका लागि विशेष उपयोगी छ। उदाहरणका लागि, केही भर्खरका कोडिङ परियोजनाहरूमा मैले .NET Aspire र Microsoft.Extensions.AI का नवीनतम रिलिजका सुविधाहरूलाई उपयोग गर्न चाहन्थेँ। Microsoft Learn Docs MCP सर्भर समावेश गरेर, म केवल API डकहरू मात्र हैन, ताजा प्रकाशित मार्गदर्शन र वाकथ्रूहरू पनि उपयोग गर्न सक्षम भएँ।

### २. ☁️ Azure MCP Server
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

**के यो गर्छ**: Azure MCP सर्भर 15+ विशेष Azure सेवा कनेक्टर्सहरूको व्यापक सेट हो जसले सम्पूर्ण Azure पारिस्थितिकी तंत्रलाई तपाईंको AI workflow मा ल्याउँछ। यो केवल एउटा सर्भर होइन – यो शक्तिशाली संग्रह हो जसमा स्रोत व्यवस्थापन, डेटाबेस कनेक्टिभिटी (PostgreSQL, SQL Server), KQL सँग Azure Monitor लग विश्लेषण, Cosmos DB एकीकरण, र धेरै थप समावेश छन्।

**किन यो उपयोगी छ**: Azure स्रोतहरू मात्र व्यवस्थापन गर्नु बाहेक, यो सर्भर Azure SDKs सँग काम गर्दा कोड गुणस्तरलाई नाटकीय रूपमा सुधार गर्छ। जब तपाईं Agent मोडमा Azure MCP प्रयोग गर्नुहुन्छ, यसले केवल कोड लेख्न सहयोग गर्दैन – यो तपाईंलाई *राम्रो* Azure कोड लेख्न मद्दत गर्छ जुन वर्तमान प्रमाणीकरण ढाँचाहरू, त्रुटि ह्यान्डलिङ सर्वश्रेष्ठ अभ्यासहरू पालना गर्छ र नवीनतम SDK सुविधाहरूको उपयोग गर्छ। सामान्य कोड पाउनु भन्दा तपाईंसँग उत्पादन भारहरूको लागि Azure सिफारिस गरिएको ढाँचाहरू पालना गर्ने कोड हुनेछ।

**मुख्य मोड्युलहरूमा समावेश छन्**:
- **🗄️ डेटाबेस कनेक्टर्स**: Azure Database for PostgreSQL र SQL Server मा प्राकृतिक भाषा पहुँच प्रत्यक्ष
- **📊 Azure Monitor**: KQL-संचालित लग विश्लेषण र सञ्चालन जानकारीहरू
- **🌐 स्रोत व्यवस्थापन**: पूर्ण Azure स्रोत जीवनचक्र व्यवस्थापन
- **🔐 प्रमाणीकरण**: DefaultAzureCredential र प्रबन्धित पहिचान ढाँचाहरू
- **📦 भण्डारण सेवाहरू**: Blob Storage, Queue Storage, र Table Storage अपरेशन्स
- **🚀 कन्टेनर सेवाहरू**: Azure Container Apps, Container Instances, र AKS व्यवस्थापन
- **र धेरै थप विशेष कनेक्टर्स**

**वास्तविक-विश्व प्रयोग**: "मेरो Azure storage खाताहरू सूचीबद्ध गर्नुहोस्", "पछिल्लो घण्टामा मेरो Log Analytics कार्यक्षेत्रका त्रुटिहरू सोधपुछ गर्नुहोस्", वा "Node.js प्रयोग गरेर उपयुक्त प्रमाणीकरणसहित Azure अनुप्रयोग निर्माण गर्न मद्दत गर्नुहोस्"

**पूरा डेमो परिदृश्य**: यहाँ पूर्ण वाक्पथ छ जसले Azure MCP र VS Code मा GitHub Copilot for Azure एक्सटेन्सनलाई संयोजन गरेर शक्ति देखाउँछ। जब तपाईं दुबै स्थापना गर्नुहुन्छ र निम्न कुरा पूरै गर्नुहुन्छ:

> "एउटा Python स्क्रिप्ट बनाउनुहोस् जुन DefaultAzureCredential प्रमाणीकरण प्रयोग गरेर Azure Blob Storage मा फाइल अपलोड गर्दछ। स्क्रिप्टले मेरो Azure storage खाता 'mycompanystorage' सँग जडान हुनुपर्छ, 'documents' नामक कन्टेनरमा अपलोड गर्नुपर्छ, हालको टाइमस्ट्याम्प सहित एउटा परीक्षण फाइल सिर्जना गर्नुपर्छ, त्रुटिहरू सुचारु रूपमा ह्यान्डल गर्नुपर्छ र सूचना दिने नतिजा दिनुपर्छ, प्रमाणीकरण र त्रुटि ह्यान्डलिङ्को लागि Azure का सर्वश्रेष्ठ अभ्यासहरू पालन गर्नुपर्छ, DefaultAzureCredential प्रमाणीकरण कसरी काम गर्छ भनी व्याख्या गर्ने टिप्पणीहरू समावेश गर्नुपर्छ, र स्क्रिप्टलाई उचित कार्यहरु र प्रलेखनसहित राम्रो संरचना बनाइनुपर्छ।"

Azure MCP Server ले पूर्ण, उत्पादन-तयार Python स्क्रिप्ट सिर्जना गर्नेछ जुन:
- नवीनतम Azure Blob Storage SDK सही async ढाँचाहरू सहित प्रयोग गर्छ
- DefaultAzureCredential लाई व्यापक fallback श्रृंखलासहित कार्यान्वयन गर्छ
- विशिष्ट Azure अपवाद प्रकारहरू सहित मजबुद त्रुटि ह्यान्डलिङ गर्छ
- Azure SDK का स्रोत व्यवस्थापन र जडान ह्यान्डलिङका लागि सर्वश्रेष्ठ अभ्यासहरू पालना गर्छ
- विस्तृत लगिङ र सूचनात्मक कन्सोल परिणाम प्रदान गर्छ
- उचित संरचित स्क्रिप्ट बनाउँछ जसमा कार्यहरू, प्रलेखन र प्रकार संकेतहरू छन्

यो उल्लेखनीय के हो भने Azure MCP बिना तपाईं सायद सामान्य blob storage कोड पाउनुहुन्छ जुन काम गर्छ तर हालको Azure ढाँचाहरू पालना गर्दैन। Azure MCP सँग, तपाईंले नवीनतम प्रमाणीकरण विधिहरू प्रयोग गरेर, Azure-विशिष्ट त्रुटि परिदृश्यहरू ह्यान्डल गरेर, र उत्पादन अनुप्रयोगहरूको लागि Microsoft ले सिफारिस गरेको अभ्यासहरू पालना गर्ने कोड पाउनुहुन्छ।

**विशेष उदाहरण**: म `az` र `azd` CLIहरूका विशेष आदेशहरू सम्झन संघर्ष गरिरहेको छु। मेरो लागि यो सधैं दुई चरणको प्रक्रिया हुन्छ: पहिलो सिन्ट्याक्स हेर्‍नु, त्यसपछि आदेश चलाउनु। म प्रायः पोर्टलमा गएर काम गर्छु किनकि CLI सिन्ट्याक्स सम्झन नसक्नु स्वीकार्न चाहन्न। जे पनि चाहिन्छ वर्णन गर्न सक्नु मलाई अद्भुत लाग्छ, र अझ राम्रो त यो मेरो IDE छोडे बिना नै गर्न सक्नु हो!

[Azure MCP repository](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) मा सुरूवातको लागि उत्कृष्ट प्रयोग सूची छ। समग्र सेटअप मार्गनिर्देशन र उन्नत कन्फिगरेसन विकल्पहरूको लागि [आधिकारिक Azure MCP कागजात](https://learn.microsoft.com/azure/developer/azure-mcp-server/) हेर्नुहोस्।

**के यो गर्छ**: आधिकारिक GitHub MCP Server ले GitHub को सम्पूर्ण पारिस्थितिकी तंत्रसँग सहज एकीकरण प्रदान गर्छ, होस्ट गरिएको रिमोट पहुँच र स्थानीय Docker डिप्लोयमेन्ट विकल्पहरू दुबै समावेश गर्दै। यो केवल आधारभूत रिपोजिटरी अपरेशन्स मात्र होइन – यो कार्यान्वयन, पुल अनुरोध वर्कफ्लोज, मुद्दा ट्र्याकिङ, सुरक्षा स्क्यानिङ, सूचनाहरू, र उन्नत स्वचालन क्षमताहरू सहित व्यापक उपकरण किट हो।

**किन यो उपयोगी छ**: यो सर्भरले GitHub सँग तपाईंको अन्तरक्रियालाई रूपान्तरण गर्छ र सम्पूर्ण प्लेटफर्म अनुभवलाई तपाईंको विकास वातावरणमा लिएर आउँछ। परियोजना व्यवस्थापन, कोड समीक्षा, र CI/CD अनुगमनका लागि VS Code र GitHub.com बीच बारम्बार सर्नुको सट्टा, तपाईं प्राकृतिक भाषा आदेशहरू मार्फत सबै कुरा तपाईंको कोडमा ध्यान दिँदै व्यवस्थापन गर्न सक्नुहुन्छ।

> **ℹ️ नोट: फरक फरक प्रकारका 'Agents'**
> 
> GitHub MCP Server लाई GitHub को Coding Agent (आइश्यूहरूलाई स्वचालित कोडिङ कार्यको लागि तोकिएको AI एजेण्ट) सँग नबुझ्नुहोस्। GitHub MCP Server VS Code को Agent मोडमा GitHub API एकीकरण प्रदान गर्छ, जबकि GitHub को Coding Agent छुट्टै सुविधा हो जुन GitHub मुद्दामा तोक्दा पुल अनुरोधहरू सिर्जना गर्दछ।

**मुख्य क्षमताहरू समावेश छन्**:
- **⚙️ GitHub Actions**: पूर्ण CI/CD पाइपलाइन व्यवस्थापन, वर्कफ्लो अनुगमन, र आर्टिफ्याक्ट ह्यान्डलिङ
- **🔀 पुल अनुरोधहरू**: PR बनाउने, समीक्षा गर्ने, मर्ज गर्ने, र व्यापक स्थिति ट्र्याकिङ सहित व्यवस्थापन गर्ने
- **🐛 मुद्दाहरू**: पूर्ण मुद्दा जीवनचक्र व्यवस्थापन, टिप्पणी, लेबलिङ, र कार्य सौंपन
- **🔒 सुरक्षा**: कोड स्क्यानिङ चेतावनीहरू, गोप्य जानकारी पहिचान, र Dependabot एकीकरण
- **🔔 सूचनाहरू**: स्मार्ट सूचनाको व्यवस्थापन र रिपोजिटरी सदस्यता नियन्त्रण
- **📁 रिपोजिटरी व्यवस्थापन**: फाइल अपरेशन्स, शाखा व्यवस्थापन, र रिपोजिटरी प्रशासन
- **👥 सहकार्य**: प्रयोगकर्ता र संगठन खोज, टोली व्यवस्थापन, र पहुँच नियन्त्रण

**वास्तविक-विश्व प्रयोग**: "मेरो फीचर शाखाबाट पुल अनुरोध सिर्जना गर्नुहोस्", "यस हप्तामा सबै असफल CI रनहरू देखाउनुहोस्", "मेरो रिपोजिटरीहरूको लागि खुला सुरक्षा चेतावनीहरूको सूची दिनुहोस्", वा "मेरो संगठनहरूमा मलाई तोकिएका सबै मुद्दाहरू पत्ता लगाउनुहोस्"

**पूरा डेमो परिदृश्य**: यहाँ GitHub MCP Server को क्षमता प्रदर्शन गर्ने शक्तिशाली कार्यप्रवाह छ:

> "हामीलाई हाम्रो स्प्रिन्ट समीक्षा तयारी गर्नु छ। यस हप्तामा मैले सिर्जना गरेका सबै पुल अनुरोधहरू देखाउनुहोस्, हाम्रो CI/CD पाइपलाइनहरूको स्थिति जाँच गर्नुहोस्, कुनै पनि सुरक्षा चेतावनीहरूको सारांश बनाउनुहोस् जुन समाधान गर्न आवश्यक छ, र 'feature' लेबल भएका मर्ज गरिएको PR हरूको आधारमा रिलीज नोटहरू तयार गर्न मद्दत गर्नुहोस्।"

GitHub MCP Server ले:
- तपाईंका हालका पुल अनुरोधहरू विस्तृत स्थिति जानकारीसहित सोधपुछ गर्नेछ
- वर्कफ्लो रनहरू विश्लेषण गर्ने र कुनै पनि असफलता वा प्रदर्शन समस्याहरू हाइलाइट गर्नेछ
- सुरक्षा स्क्यानिङ परिणामहरू संकलन गर्ने र महत्वपूर्ण चेतावनीहरूलाई प्राथमिकता दिनेछ
- मर्ज गरिएको PR बाट जानकारी निकालेर व्यापक रिलीज नोटहरू सिर्जना गर्नेछ
- स्प्रिन्ट योजना र रिलिज तयारीका लागि कार्यान्वयन योग्य अर्को कदमहरू प्रदान गर्नेछ

**विशेष उदाहरण**: म यो कोड समीक्षा वर्कफ्लोजका लागि प्रयोग गर्न मन पराउँछु। VS Code, GitHub सूचनाहरू र पुल अनुरोध पृष्ठहरू बीच जम्प गर्नुभन्दा,"मेरी समीक्षा पर्खिरहेका सबै PR हरू देखाउनुहोस्" भनी म भन्न सक्छु र त्यसपछि "PR #123 मा प्रमाणीकरण विधिमा त्रुटि ह्यान्डलिङको बारेमा टिप्पणी थप्नुहोस्" भनी भन्न सक्छु। सर्भरले GitHub API कलहरू ह्यान्डल गर्छ, छलफलको सन्दर्भ राख्छ, र मसँग अझ रचनात्मक समीक्षा टिप्पणीहरू तयार गर्न मद्दत गर्छ।

**प्रमाणीकरण विकल्पहरू**: सर्भरले OAuth (VS Code मा सहज) र व्यक्तिगत पहुँच टोकनहरू दुवैलाई समर्थन गर्दछ, तपाईंलाई आवश्यक GitHub कार्यक्षमता मात्रै सक्षम गर्न कन्फिगरयोग्य उपकरण सेटहरूसहित। तपाईं यसलाई तुरुन्तै सेटअपका लागि रिमोट होस्टेड सेवा रूपमा वा पूर्ण नियन्त्रणका लागि लोकल Docker मार्फत चलाउन सक्नुहुन्छ।

> **💡 प्रो सुझाव**
> 
> सन्दर्भ आकार घटाउन र AI उपकरण चयन सुधार गर्न MCP सर्भर सेटिङमा `--toolsets` प्यारामिटर कन्फिगर गरेर तपाईंलाई आवश्यक उपकरण सेटहरू मात्र सक्षम गर्नुहोस्। उदाहरणका लागि, मुख्य विकास वर्कफ्लोजको लागि `"--toolsets", "repos,issues,pull_requests,actions"` थप्नुहोस्, वा तपाईं मुख्य रूपमा GitHub निगरानी क्षमताहरू चाहनुभएको खण्डमा `"--toolsets", "notifications, security"` प्रयोग गर्नुहोस्।

### 5. 📝 MarkItDown MCP Server
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_MarkItDown_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_MarkItDown_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/markitdown)

**यसले के गर्छ**: MarkItDown एक व्यापक कागजात रूपान्तरण सर्भर हो जसले विभिन्न फाइल ढाँचाहरूलाई उच्च गुणस्तरको Markdown मा परिवर्तन गर्छ, LLM उपभोग र पाठ विश्लेषण कार्यप्रवाहहरूको लागि अनुकूलित।

**किन यो उपयोगी छ**: आधुनिक कागजात कार्यप्रवाहहरूको लागि आवश्यक! MarkItDown ले शीर्षकहरू, सूचीहरू, तालिका र लिंक जस्ता महत्वपूर्ण कागजात संरचनालाई कायम राख्दै विभिन्न फाइल ढाँचाहरूलाई प्रभावकारी रूपमा व्यवस्थापन गर्दछ। साधारण पाठ निक्षेप उपकरणहरू भन्दा फरक, यो सेमेन्टिक अर्थ र फर्म्याटिङमा विशेष ध्यान दिन्छ जुन AI प्रक्रियाकरण र मानवीय पठनीयताको लागि उपयोगी हुन्छ।

**समर्थित फाइल ढाँचाहरू**:
- **कार्यालय दस्तावेजहरू**: PDF, PowerPoint (PPTX), Word (DOCX), Excel (XLSX/XLS)
- **मिडिया फाइलहरू**: छविहरू (EXIF मेटाडाटा र OCR सहित), अडियो (EXIF मेटाडाटा र भाषण ट्रान्सक्रिप्सन सहित)
- **वेब सामग्री**: HTML, RSS फिडहरू, YouTube URL हरू, विकिपीडिया पृष्ठहरू
- **डेटा ढाँचाहरू**: CSV, JSON, XML, ZIP फाइलहरू (पुनरावृत्ति गर्दै सामग्रीहरूलाई प्रक्रिया गर्छ)
- **प्रकाशन ढाँचाहरू**: EPub, Jupyter नोटबुक (.ipynb)
- **इमेल**: Outlook सन्देशहरू (.msg)
- **उन्नत**: PDF प्रशोधनको लागि Azure Document Intelligence इम्बेड

**उन्नत क्षमताहरू**: MarkItDown ले LLM-सञ्चालित छवि विवरण (OpenAI क्लाएन्ट उपलब्ध हुँदा), PDF प्रशोधनका लागि Azure Document Intelligence, भाषण सामग्रीको लागि अडियो ट्रान्सक्रिप्सन, र थप फाइल ढाँचाहरू विस्तार गर्ने प्लगइन प्रणालीलाई समर्थन गर्छ।

**वास्तविक प्रयोग**: "यस PowerPoint प्रस्तुतीकरणलाई हाम्रो कागजात साइटको लागि Markdown मा रूपान्तरण गर्नुहोस्", "यो PDF बाट शीर्षक संरचना सही तरिकाले सहित पाठ निक्षेप गर्नुहोस्", वा "यो Excel स्प्रेडशीटलाई पढ्न मिल्ने तालिका ढाँचामा परिवर्तन गर्नुहोस्"

**विशेष उदाहरण**: [MarkItDown कागजातहरू](https://github.com/microsoft/markitdown#why-markdown) लाई उद्धृत गर्दा:

> Markdown निकै नजिक छ सामान्य टेक्स्टसँग, न्यूनतम मार्कअप वा फर्म्याटिङ सहित, तर अझै महत्वपूर्ण कागजात संरचना प्रतिनिधित्व गर्ने तरिका प्रदान गर्दछ। प्रमुख LLM हरू, जस्तै OpenAI को GPT-4o, स्वाभाविक रूपमा Markdown "बोल्छन्", र प्रायः Markdown लाई उनीहरूको प्रतिक्रियामा स्वतः समावेश गर्छन्। यसले देखाउँछ कि तिनीहरूलाई ठूलो मात्रामा Markdown-फर्म्याट गरिएको पाठमा प्रशिक्षित गरिएको छ, र यसलाई राम्ररी बुझ्छन्। साइड फाइदाको रूपमा, Markdown का नियमहरू पनि अत्यन्त टोकन-कुशल छन्।

MarkItDown कागजात संरचनाको संरक्षणमा साँच्चै राम्रो छ, जुन AI कार्यप्रवाहहरूको लागि महत्त्वपूर्ण हुन्छ। उदाहरणका लागि, PowerPoint प्रस्तुतीकरण रूपान्तरण गर्दा, यो स्लाइड संगठनलाई सही शीर्षकहरूसहित कायम राख्छ, तालिकाहरूलाई Markdown तालिकाका रूपमा निकाल्छ, छविहरूको लागि alt पाठ समावेश गर्दछ, र स्पिकर नोटहरू समेत प्रक्रिया गर्छ। चार्टहरू पढ्न मिल्ने डाटा तालिकामा परिणत हुन्छन्, र परिणामी Markdown ले मूल प्रस्तुतीकरणको तार्किक प्रवाह कायम राख्छ। यसले AI प्रणालीहरूलाई प्रस्तुतीकरण सामग्री फ़िड गर्न वा अवस्थित स्लाइडहरूबाट कागजात तयार पार्न उपयुक्त बनाउँछ।

- **⚡ AI एजेन्ट एकीकरण**: Azure AI एजेन्टहरूसँग जडान हुनुहोस्, अवस्थित एजेन्टहरू सोध्नुहोस्, र उत्पादन परिदृश्यहरूमा एजेन्ट प्रदर्शन मूल्यांकन गर्नुहोस्
- **📊 मूल्यांकन फ्रेमवर्क**: व्यापक पाठ र एजेन्ट मूल्यांकन चलाउनुहोस्, मार्कडाउन रिपोर्टहरू सिर्जना गर्नुहोस्, र AI अनुप्रयोगहरूको लागि गुणस्तर आश्वासन कार्यान्वयन गर्नुहोस्
- **🚀 प्रोटोटाइपिङ उपकरणहरू**: GitHub-आधारित प्रोटोटाइपिङका लागि सेटअप निर्देशनहरू प्राप्त गर्नुहोस् र अत्याधुनिक अनुसन्धान मोडलहरूको लागि Microsoft Foundry Labs मा पहुँच पाउनुहोस्

- तुरुन्त परीक्षणका लागि GitHub प्रमाणीकरणको साथ प्रोटोटाइपिङ कोड सिर्जना गर्नेछ
- तपाइँको विशेष प्राविधिक स्ट्याक अनुरूप व्यापक सेटअप मार्गदर्शकहरू प्रदान गर्नेछ

## 📋 निष्कर्ष

Model Context Protocol (MCP) ले विकासकर्ताहरू कसरी AI सहायकहरू र बाह्य उपकरणहरूसँग अन्तरक्रिया गर्छन् भन्ने तरिका रूपान्तरण गर्दैछ। यी १० Microsoft MCP सर्भरहरूले मानकीकृत AI एकीकरणको शक्ति प्रदर्शित गर्छन्, जसले विकासकर्ताहरूलाई उत्पादनशील अवस्थामै राख्दै शक्तिशाली बाह्य क्षमता पहुँचाउने सहज कार्यप्रवाह सक्षम पार्दछ।

व्यापक Azure पारिस्थितिकी तन्त्र एकीकरण देखि ब्राउजर स्वचालनका लागि Playwright र कागजात प्रशोधनका लागि MarkItDown जस्ता विशेष उपकरणहरू सम्म, यी सर्भरहरूले विविध विकास परिदृश्यहरूमा MCP ले कसरी उत्पादकता बढाउन सक्छ भन्ने देखाउँछन्। मानकीकृत प्रोटोकलले यी उपकरणहरूलाई सहज रूपमा एकसाथ काम गर्न सक्षम पार्दछ, जसले एक समेकित विकास अनुभव बनाउँछ।

MCP पारिस्थितिकी तन्त्र बढ्न जारी रहेको हुँदा, समुदायसँग संलग्न रहनु, नयाँ सर्भरहरू अन्वेषण गर्नु, र अनुकूल समाधानहरू निर्माण गर्नु तपाइँको विकास उत्पादकता अधिकतम बनाउन प्रमुख हुनेछ। MCP को खुला मानक प्रकृतिले तपाइँलाई विभिन्न विक्रेताबाट उपकरणहरू मिलाएर तपाईँको विशेष आवश्यकताहरूको लागि उत्तम कार्यप्रवाह सिर्जना गर्न अनुमति दिन्छ।
