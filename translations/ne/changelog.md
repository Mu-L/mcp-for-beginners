# परिवर्तनावली: MCP सुरु गर्नेहरूको पाठ्यक्रम

यो कागजात मोडेल कण्टेक्स्ट प्रोटोकल (MCP) सुरु गर्नेहरूको पाठ्यक्रममा गरिएका सबै महत्वपूर्ण परिवर्तनहरूको अभिलेखको रूपमा सेवा गर्छ। परिवर्तनहरू उल्टो कालानुक्रमिक क्रममा दर्ता गरिएका छन् (सबैभन्दा नयाँ परिवर्तनहरू पहिले)।

## जुन १६, २०२६

### MCP विनिर्देश सन्तुलन र नमूना प्रमाणीकरण

पाठ्यक्रमलाई वर्तमान **MCP विनिर्देश २०२५-११-२५** र पछिल्लो आधिकारिक SDK हरूसँग प्रमाणीकरण गरियो, त्यसपछि बाँकी रहेका पुराना विनिर्देश सन्दर्भहरू सच्याइयो र मुख्य नमूनाहरू अझै निर्माण र चल्ने सुनिश्चित गरियो।

#### विनिर्देश संस्करण सुधारहरू (२०२५-०६-१८ / २०२५-०३-२६ → २०२५-११-२५)

अंग्रेजी सामग्री अद्यावधिक गरियो जहाँ यो अझै पुरानो विनिर्देश संस्करणलाई *वर्तमान/नवीनतम* मानक भनेर दाबी गरिरहेको थियो र लिंकहरूलाई आधिकारिक `modelcontextprotocol.io` विनिर्देश पथहरूमा पुनःतिहाइ गरियो:
- **05-AdvancedTopics/mcp-security/README.md**: "वर्तमान मानक" ब्यानेर, परिचय, मुख्य सुरक्षा सिद्धान्त शीर्षक, अनिवार्य आवश्यकताहरू शीर्षक, Microsoft Entra ID खण्ड, सन्दर्भ र स्रोत लिंकहरू, र समापन सुरक्षा सूचना (८ सन्दर्भहरू) लाई २०२५-११-२५ मा अद्यावधिक गरियो
- **05-AdvancedTopics/mcp-transport/README.md**: थप स्रोतहरू विनिर्देश लिंक र "वर्तमान मानक" ब्यानेरलाई २०२५-११-२५ मा अद्यावधिक गरियो
- **05-AdvancedTopics/mcp-realtimesearch/README.md**: पुरानो `२०२५-०३-२६` सुरक्षा-र-विश्वास लिंकलाई हालको २०२५-११-२५ सुरक्षा उत्तम अभ्यास पृष्ठसँग प्रतिस्थापन गरियो
- **03-GettingStarted/14-sampling/README.md**: आधिकारिक नमूना कागजात लिंक २०२५-११-२५ मा अद्यावधिक गरियो
- **03-GettingStarted/05-stdio-server/README.md**: वर्तमान कालको "हालको MCP विनिर्देश" सन्दर्भ र थप स्रोतहरू विनिर्देश लिंक २०२५-११-२५ मा अद्यावधिक गरियो (ऐतिहासिक SSE-विषयक नोटहरू निश्चितता का लागि यथावत राखियो)

#### वर्तमान SDK हरू विरुद्ध नमूना प्रमाणीकरण

- **TypeScript (03-GettingStarted/01-first-server/solution/typescript)**: `npm install` ले `@modelcontextprotocol/sdk@1.29.0` समाधान गर्‍यो; `tsc --noEmit` बिना कुनै प्रकार त्रुटिहरू पास भयो — अवस्थित `McpServer`/`StdioServerTransport` API हरू मान्य छन्
- **Python (03-GettingStarted/01-first-server/solution/python)**: अलग `.venv` मा `mcp[cli]` (1.27.2) सँग प्रमाणीकरण गरियो; `py_compile` पास भयो र `FastMCP.list_tools()` ले सही रूपमा `add` र `subtract` उपकरणहरू फर्कायो
- सबै नमूना `@modelcontextprotocol/sdk` संस्करण दायराहरू (`>=1.26.0` / `^1.26.0` / `^1.27.0`) ले वर्तमान `1.29.0` लाई सफा रूपमा समाधान गर्छन् र कुनै API परिवर्तन तोडेकै छैनन्

### सुरक्षा कडा पार्ने र निर्भरताका कमजोरीहरू सुधार

हरेक निर्भरता म्यानिफेस्ट र नमूना स्रोत कोडमा पूर्ण सुरक्षा परीक्षण चलाइयो, त्यसपछि सबै रिपोर्ट गरिएका npm सल्लाहहरू र एक कोड-स्तरको समस्यालाई सुधार गरियो। सुधारपछि, `npm audit` ले हरेक जाँचिएको निर्देशिकामा **० जोखिमहरू** रिपोर्ट गर्यो।

#### npm निर्भरता जोखिमहरू (अप्रत्यक्ष) — समाधान गरियो

१५ वटा प्रतिबद्ध `package-lock.json` फाइलहरू परीक्षण गरियो। जोखिमहरू केवल MCP Inspector विकास उपकरण, OpenAI क्लाइन्ट, र MCP SDK द्वारा ल्याइएका अप्रत्यक्ष निर्भरता सीमित थिए; सबै नमूनाहरूलाई तोड्ने बिना समाधान गरियो:
- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/inspector** र **lab3/code/weather_mcp/inspector**: `@modelcontextprotocol/inspector` (`0.16.6` / `0.14.1` → `0.22.0`) बढाइयो, जसले समावेश `ajv`, `brace-expansion`, `diff`, `path-to-regexp`, र `ws` सल्लाहहरू हटायो। `concurrently` द्वारा राखिएको बाँकी महत्त्वपूर्ण सल्लाह हटाउन npm `overrides` मा `shell-quote@1.8.4` मजबुरी गरिएको छ; दुबै लकफाइलहरू पुनःउत्पन्न (अहिले ० जोखिमहरू)
- **03-GettingStarted/samples/typescript**: `npm audit fix` ले अप्रत्यक्ष `qs` (मध्यम) लाई सुधारिएको संस्करणमा अपडेट गर्यो
- **03-GettingStarted/samples/javascript**: `npm audit fix` ले अप्रत्यक्ष `hono` (मध्यम) लाई सुधारिएको संस्करणमा अपडेट गर्यो
- **03-GettingStarted/03-llm-client/solution/typescript**: `npm audit fix` ले अप्रत्यक्ष `form-data` (उच्च) लाई सुधारिएको संस्करणमा अपडेट गर्यो
- **03-GettingStarted/11-simple-auth/solution/typescript**: हराएको `package-lock.json` सिर्जना गरियो ताकि परियोजना पुनरुत्पादनयोग्य र जाँचयोग्य होस् (० जोखिमहरू)

#### कोड-स्तर सुरक्षा सुधार (OWASP A03: इन्जेक्सन)

- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/src/server.py**: `open_in_vscode` टूलबाट `shell=True` हटाइयो। पहिलेको `subprocess.run(["start", "", vscode_path, folder_path], shell=True)` ले `cmd.exe` मार्फत फोल्डर पथका शेल मेटा अवयवहरू व्याख्या गर्न दिन्थ्यो (कमाण्ड इन्जेक्सन भेक्टर)। अहिले यसले सोझै समाधान गरिएका `Code.exe` लाई फोल्डरलाई तर्कको रूपमा दिँदै सुरु गर्छ — कुनै शेल छैन — जुन कार्यात्मक रूपमा समान र सुरक्षित छ

#### मुख्य अवधारणा सुधार (01-CoreConcepts/)
- **नयाँ क्लाइन्ट प्रिमिटिभ - Roots**: Roots क्लाइन्ट प्रिमिटिभको व्यापक कागजात थपियो, जसले सर्भरहरूलाई फाइलसिस्टम सीमाना र पहुँच अनुमति बुझ्न सक्षम पार्छ
- **उपकरण एनोटेसनहरू**: उपकरण व्यवहार एनोटेसनहरू (`readOnlyHint`, `destructiveHint`) को कागजात थपियो, जसले उपकरण कार्यान्वयन निर्णय सुधार गर्दछ

- **नयाँ अध्याय थपियो**: उन्नत MCP सर्भर प्रयोगको लागि व्यापक गाइड थपियो, जुन दुवै नियमित र निम्न-स्तरीय सर्भर वास्तुकलाहरू समेट्छ।  
  - **नियमित बनाम निम्न-स्तरीय सर्भर**: दुवै विधिहरूका लागि पायथन र टाइपस्क्रिप्टमा विस्तृत तुलना र कोड उदाहरणहरू

  - **व्यावहारिक ढाँचाहरू**: जहाँ निम्न-स्तरीय सर्भर ढाँचाहरू उन्नत सुविधाहरू र वास्तुकलाका लागि उपयुक्त छन् त्यस्ता वास्तविक परिदृश्यहरू
#### सरल प्रमाणीकरण (03-GettingStarted/11-simple-auth)
- **नयाँ अध्याय थपिएको**: MCP सर्भरमा सरल प्रमाणीकरण लागू गर्ने चरण-दर-चरण मार्गदर्शन।
  - **प्रमाणीकरण अवधारणाहरू**: प्रमाणीकरण र अनुमति बीच स्पष्ट व्याख्या, र प्रमाणपत्र ह्यान्डलिङ।
  - **मूलभूत प्रमाणीकरण कार्यान्वयन**: Python (Starlette) र TypeScript (Express) मा मिडलवेयर-आधारित प्रमाणीकरण ढाँचा, कोड नमूनासहित।
  - **उन्नत सुरक्षा तर्फको प्रगति**: सरल प्रमाणीकरणबाट OAuth 2.1 र RBAC मा उन्नति गर्ने मार्गदर्शन, उन्नत सुरक्षा मोड्युलहरूको सन्दर्भसहित।

यी थपहरूले व्यावहारिक, हातमा काम गर्ने मार्गदर्शन प्रदान गर्छन् जसले थप बलियो, सुरक्षित र लचीला MCP सर्भर कार्यान्वयनहरू निर्माण गर्न मद्दत गर्छ, आधारभूत अवधारणाहरूलाई उन्नत उत्पादन ढाँचासँग जोड्दै।
