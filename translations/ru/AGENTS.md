# AGENTS.md

## Обзор проекта

**MCP для начинающих** — это открытая учебная программа для изучения Model Context Protocol (MCP) — стандартизированного фреймворка для взаимодействия между AI-моделями и клиентскими приложениями. В этом репозитории представлены комплексные учебные материалы с практическими примерами кода на нескольких языках программирования.

### Ключевые технологии

- **Языки программирования**: C#, Java, JavaScript, TypeScript, Python, Rust
- **Фреймворки и SDK**: 
  - MCP SDK (`@modelcontextprotocol/sdk`)
  - Spring Boot (Java)
  - FastMCP (Python)
  - LangChain4j (Java)
- **Базы данных**: PostgreSQL с расширением pgvector
- **Облачные платформы**: Azure (Container Apps, OpenAI, Content Safety, Application Insights)
- **Инструменты сборки**: npm, Maven, pip, Cargo
- **Документация**: Markdown с автоматическим многоязычным переводом (более 48 языков)

### Архитектура

- **11 основных модулей (00-11)**: Последовательный учебный путь от основ к продвинутым темам
- **Практические лаборатории**: Практические задания с полным кодом решений на нескольких языках
- **Примерные проекты**: Рабочие реализации MCP-сервера и клиента
- **Система перевода**: Автоматизированный рабочий процесс GitHub Actions для многоязычной поддержки
- **Графические ресурсы**: Централизованная директория изображений с переведёнными версиями

## Рабочий процесс разработки

### Структура документации

- **Модули 00-11**: Основной учебный материал в последовательном порядке
- **translations/**: Языковые версии (автоматически сгенерированные, не редактировать вручную)
- **translated_images/**: Локализованные версии изображений (автоматически сгенерированные)
- **images/**: Исходные изображения и диаграммы

### Внесение изменений в документацию

1. Редактируйте только английские markdown-файлы в корневых директориях модулей (00-11)
2. При необходимости обновляйте изображения в каталоге `images/`
3. GitHub Action co-op-translator автоматически создаст переводы
4. Переводы обновляются при пуше в ветку main

### Работа с переводами

- **Автоматический перевод**: workflow GitHub Actions обрабатывает все переводы
- **НЕ редактируйте вручную** файлы в директории `translations/`
- Метаданные перевода встроены в каждый переведённый файл
- Поддерживаемые языки: более 48 языков, включая арабский, китайский, французский, немецкий, хинди, японский, корейский, португальский, русский, испанский и многие другие

## Руководство по стилю кода

### Стиль документации

- Используйте понятный, дружественный для новичков язык
- Включайте примеры кода на нескольких языках по возможности
- Следуйте лучшим практикам markdown:
  - Используйте заголовки ATX-стиля (`#`)
  - Используйте блоки кода с указанием языка
  - Включайте описательные alt-тексты для изображений
  - Сохраняйте разумную длину строк (жёстких ограничений нет, но будьте разумны)

### Стиль примеров кода

#### TypeScript/JavaScript
- Используйте ES-модули (`import`/`export`)
- Следуйте строгому режиму TypeScript
- Включайте аннотации типов
- Цель — ES2022

#### Python
- Следуйте стилю PEP 8
- Используйте подсказки типов там, где это уместно
- Включайте докстринги для функций и классов
- Используйте современные возможности Python (3.8+)

#### Java
- Следуйте конвенциям Spring Boot
- Используйте возможности Java 21
- Соблюдайте стандартную структуру проектов Maven
- Включайте комментарии Javadoc

### Процесс pull request

1. **Форк и клонирование**: Форкните репозиторий и клонируйте форк локально
2. **Создайте ветку**: Используйте описательные имена веток (например, `fix/typo-module-3`, `add/python-example`)
3. **Внесите изменения**: Редактируйте только английские markdown-файлы (не переводы)
4. **Тестируйте локально**: Убедитесь, что markdown отображается корректно
5. **Отправьте PR**: Используйте понятные заголовки и описания PR
6. **CLA**: Подпишите Microsoft Contributor License Agreement при запросе

Используйте ясные, описательные заголовки:
- `[Module XX] Краткое описание` для изменений в модулях

### Распространённые вопросы и решения

**В: Мой PR не проходит проверку перевода**  
О: Убедитесь, что вы изменяли только английские markdown-файлы в корневых директориях модулей, а не переводы.

**В: Как добавить новый язык?**  
О: Поддержка языков управляется через workflow co-op-translator. Откройте issue для обсуждения добавления новых языков.

**В: Примеры кода не работают**  
О: Убедитесь, что вы следовали инструкциям по настройке в README конкретного примера. Проверьте версии зависимостей.

**В: Изображения не отображаются**  
О: Убедитесь, что пути к изображениям относительные и используют прямые слэши. Изображения должны находиться в `images/` или `translated_images/` для локализованных версий.
