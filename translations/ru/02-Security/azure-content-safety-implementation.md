# Реализация Azure Content Safety с MCP

> **Риск OWASP MCP, рассматриваемый здесь**: [MCP06 - Подмена потока намерений](https://microsoft.github.io/mcp-azure-security-guide/mcp/mcp06-prompt-injection/)

Для усиления безопасности MCP против инъекций подсказок, отравления инструментов и других уязвимостей, специфичных для ИИ, настоятельно рекомендуется интегрировать Azure Content Safety. Это руководство по реализации согласуется с [MCP Security Summit Workshop (Sherpa)](https://azure-samples.github.io/sherpa/) Лагерь 3: Безопасность ввода/вывода.

## Интеграция с сервером MCP

Чтобы интегрировать Azure Content Safety с вашим сервером MCP, добавьте фильтр безопасности контента в виде промежуточного ПО в ваш конвейер обработки запросов:

1. Инициализируйте фильтр во время запуска сервера
2. Проверяйте все входящие запросы инструментов перед обработкой
3. Проверяйте все исходящие ответы перед возвратом их клиентам
4. Ведите журнал и оповещения о нарушениях безопасности
5. Реализуйте соответствующую обработку ошибок для неудачных проверок безопасности контента

Это обеспечивает надежную защиту от:
- Атак инъекций подсказок
- Попыток отравления инструментов
- Вывода данных через вредоносные вводы
- Генерации вредоносного контента

## Лучшие практики интеграции Azure Content Safety

1. **Пользовательские блок-листы**: Создавайте пользовательские блок-листы специально для шаблонов инъекций MCP
2. **Настройка уровней серьезности**: Регулируйте пороги серьезности в зависимости от вашего конкретного использования и уровня риска
3. **Комплексное покрытие**: Применяйте проверки безопасности контента ко всем входным и выходным данным
4. **Оптимизация производительности**: Рассмотрите возможность внедрения кэширования для повторяющихся проверок безопасности контента
5. **Механизмы резервного копирования**: Определите четкие действия при недоступности служб безопасности контента
6. **Обратная связь пользователям**: Обеспечьте понятную обратную связь пользователям, когда контент блокируется из-за проблем с безопасностью
7. **Постоянное улучшение**: Регулярно обновляйте блок-листы и шаблоны на основе новых угроз
