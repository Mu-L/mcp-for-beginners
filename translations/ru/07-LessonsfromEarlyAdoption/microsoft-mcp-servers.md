## 🔧 Понимание серверов MCP: руководство для начинающих

### Что такое серверы MCP?

Если вы новичок в Протоколе Контекста Модели (MCP), вы, возможно, задаётесь вопросом: «Что такое сервер MCP и почему это важно?» Начнём с простой аналогии.

Представьте серверы MCP как специализированных помощников, которые помогают вашему AI-помощнику по программированию (например, GitHub Copilot) подключаться к внешним инструментам и сервисам. Как вы можете использовать различные приложения на телефоне для разных задач — одно для погоды, другое для навигации, третье для банковских операций — серверы MCP дают вашему AI-помощнику возможность взаимодействовать с различными инструментами и сервисами для разработки.

### Какую проблему решают серверы MCP

До появления серверов MCP, если вы хотели:
- Проверить ресурсы Azure
- Создать issue в GitHub
- Выполнить запрос к базе данных
- Поискать информацию в документации

вам приходилось прерывать кодирование, открывать браузер, переходить на нужный сайт и выполнять эти действия вручную. Такое постоянное переключение контекста нарушает ваш поток работы и снижает продуктивность.

### Как серверы MCP меняют ваш опыт разработки

С MCP серверами вы можете оставаться в своей среде разработки (VS Code, Visual Studio и т.д.) и просто попросить AI-помощника выполнить эти задачи. Например:

**Вместо традиционного рабочего процесса:**
1. Прервать кодирование
2. Открыть браузер
3. Перейти в портал Azure
4. Просмотреть детали хранилища
5. Вернуться в VS Code
6. Продолжить кодирование

**Вы теперь можете сделать так:**
1. Спросить AI: «Каково состояние моих аккаунтов хранения Azure?»
2. Продолжить кодирование с полученной информацией

### Ключевые преимущества для новичков

#### 1. 🔄 **Оставайтесь в состоянии потока**
- Больше не нужно переключаться между множеством приложений

- Снижайте умственную нагрузку от управления разными инструментами

#### 2. 🤖 **Используйте естественный язык вместо сложных команд**
- Не нужно запоминать синтаксис SQL, просто опишите нужные данные
- Не требуется знать команды Azure CLI, просто объясните задачу
- Пусть AI разбирается с техническими деталями, а вы сосредоточьтесь на логике

- Создавайте мощные рабочие процессы, комбинируя сервисы
- Пример: «Получить все последние issues из GitHub и создать соответствующие задачи в Azure DevOps»
- Автоматизируйте без написания сложных скриптов

#### 4. 🌐 **Доступ к расширяющейся экосистеме**
- Пользуйтесь серверами от Microsoft, GitHub и других компаний
- Смешивайте и сочетайте инструменты от разных поставщиков без проблем
- Присоединяйтесь к стандартизованной экосистеме, которая работает с разными AI-помощниками

#### 5. 🛠️ **Обучайтесь на практике**
- Начинайте с готовых серверов, чтобы понять концепции
- Постепенно создавайте собственные серверы по мере роста уверенности
- Используйте доступные SDK и документацию для поддержки обучения

## Начало работы с MCP в VS Code и Visual Studio 🚀

Настройка этих серверов MCP проста, если вы используете Visual Studio Code или Visual Studio 2022 с GitHub Copilot.

### Настройка VS Code

Базовый процесс для VS Code:

1. **Включите режим агента**: в VS Code переключитесь на режим агента в окне Copilot Chat
2. **Настройка серверов MCP**: добавьте конфигурации серверов в файл settings.json VS Code
3. **Запустите серверы**: нажмите кнопку «Старт» для каждого сервера, который хотите использовать
4. **Выберите инструменты**: выберите, какие серверы MCP включить для текущей сессии

Подробные инструкции по настройке смотрите в [документации VS Code по MCP](https://code.visualstudio.com/docs/copilot/copilot-mcp).

> **💡 Профессиональный совет: управляйте MCP серверами как эксперт!**
> 
> Теперь в представлении расширений VS Code есть [удобный новый интерфейс для управления установленными MCP серверами](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode)! У вас есть быстрый доступ к запуску, остановке и управлению любыми MCP серверами через понятный и простой интерфейс. Попробуйте!

### Настройка Visual Studio 2022

Для Visual Studio 2022 (версия 17.14 и новее):

1. **Включите режим агента**: нажмите выпадающий список «Спросить» в окне GitHub Copilot Chat и выберите «Агент»
2. **Создайте конфигурационный файл**: создайте файл `.mcp.json` в директории вашего решения (рекомендуемое расположение: `<SOLUTIONDIR>\.mcp.json`)
3. **Настройте серверы**: добавьте конфигурации серверов MCP в стандартном формате MCP
4. **Одобрение инструментов**: при запросе одобрите использование инструментов с соответствующими правами доступа

Подробные инструкции по настройке Visual Studio смотрите в [документации Visual Studio по MCP](https://learn.microsoft.com/visualstudio/ide/mcp-servers).

Каждый сервер MCP имеет свои требования к конфигурации (строки подключения, аутентификация и т.п.), но шаблон настройки одинаков как для VS Code, так и для Visual Studio.

### 1. 📚 Сервер Microsoft Learn Docs MCP

[![Установить в VS Code](https://img.shields.io/badge/VS_Code-Install_Microsoft_Docs_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D) [![Установить в VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Microsoft_Docs_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/mcp)

**Что делает**: Сервер Microsoft Learn Docs MCP — это облачный сервис, который обеспечивает AI-помощников доступом в реальном времени к официальной документации Microsoft через Протокол Контекста Модели. Он подключается к `https://learn.microsoft.com/api/mcp` и обеспечивает семантический поиск по Microsoft Learn, документации Azure, документации Microsoft 365 и другим официальным источникам Microsoft.

**Почему это полезно**: Хотя это может казаться «просто документацией», этот сервер крайне важен для каждого разработчика, работающего с технологиями Microsoft. Одним из самых больших нареканий со стороны .NET-разработчиков по AI-помощникам было то, что они не знают последних выпусков .NET и C#. Сервер Microsoft Learn Docs MCP решает эту проблему, предоставляя доступ в реальном времени к самым актуальным документам, API-справочникам и лучшим практикам. Независимо от того, работаете ли вы с последними SDK для Azure, изучаете новые возможности C# 13 или реализуете передовые паттерны .NET Aspire, этот сервер гарантирует, что у вашего AI-помощника есть доступ к авторитетной и обновлённой информации для генерации точного, современного кода.

**Реальное применение**: «Какие команды az cli использовать для создания контейнерного приложения Azure согласно официальной документации Microsoft Learn?» или «Как настроить Entity Framework с внедрением зависимостей в ASP.NET Core?» Или: «Проверь этот код, чтобы убедиться, что он соответствует рекомендациям по производительности из Microsoft Learn Documentation.» Сервер охватывает Microsoft Learn, документацию Azure и Microsoft 365, используя продвинутый семантический поиск для нахождения наиболее релевантной информации. Возвращает до 10 высококачественных фрагментов контента с названиями статей и URL, всегда получая последние опубликованные документы.

**Пример использования**: Сервер предоставляет инструмент `microsoft_docs_search`, который выполняет семантический поиск по официальной технической документации Microsoft. После настройки вы можете задавать вопросы типа «Как реализовать JWT-аутентификацию в ASP.NET Core?» и получать детальные официальные ответы с ссылками на источники. Качество поиска исключительное, так как учитывается контекст — запрос про «контейнеры» в контексте Azure вернёт документацию по Azure Container Instances, а в контексте .NET — информацию о коллекциях C#.

Это особенно полезно для быстро меняющихся или недавно обновлённых библиотек и сценариев. Например, в некоторых последних проектах я хотел использовать возможности последних выпусков .NET Aspire и Microsoft.Extensions.AI. Включив сервер Microsoft Learn Docs MCP, я смог использовать не только API-документацию, но и недавно опубликованные пошаговые руководства и советы.

### 2. ☁️ Azure MCP Server
[![Установить в VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![Установить в VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

**Что это делает**: Azure MCP Server — это комплексный набор из более чем 15 специализированных коннекторов для сервисов Azure, который интегрирует всю экосистему Azure в ваш AI-рабочий процесс. Это не просто один сервер – это мощный набор, включающий управление ресурсами, подключение к базам данных (PostgreSQL, SQL Server), анализ логов Azure Monitor с помощью KQL, интеграцию Cosmos DB и многое другое.

**Почему это полезно**: Помимо управления ресурсами Azure, этот сервер значительно улучшает качество кода при работе с SDK Azure. Используя Azure MCP в режиме агента, он не просто помогает писать код — он помогает писать *лучший* код для Azure, который следует современным паттернам аутентификации, лучшим практикам обработки ошибок и использует последние возможности SDK. Вместо получения общего кода, который может работать, вы получаете код, соответствующий рекомендуемым Azure паттернам для производственных нагрузок.

**Основные модули включают**:
- **🗄️ Коннекторы баз данных**: Прямой доступ на естественном языке к Azure Database для PostgreSQL и SQL Server
- **📊 Azure Monitor**: Анализ логов и операционные инсайты с помощью KQL
- **🌐 Управление ресурсами**: Полное управление жизненным циклом ресурсов Azure
- **🔐 Аутентификация**: Паттерны DefaultAzureCredential и управляемой идентичности
- **📦 Службы хранения**: Операции с Blob Storage, Queue Storage и Table Storage
- **🚀 Контейнерные сервисы**: Управление Azure Container Apps, Container Instances и AKS
- **И множество других специализированных коннекторов**

**Реальное использование**: "Перечисли мои аккаунты хранения Azure", "Запроси в моей рабочей области Log Analytics ошибки за последний час" или "Помоги мне создать Azure-приложение на Node.js с корректной аутентификацией"

**Полный демонстрационный сценарий**: Вот полный пример, демонстрирующий возможности сочетания Azure MCP с расширением GitHub Copilot для Azure в VS Code. При наличии обеих установок и запросе:

> "Создай Python-скрипт, который загружает файл в Azure Blob Storage, используя аутентификацию DefaultAzureCredential. Скрипт должен подключаться к моему аккаунту хранения Azure 'mycompanystorage', загружать в контейнер 'documents', создать тестовый файл с текущей отметкой времени для загрузки, корректно обрабатывать ошибки и предоставлять информативный вывод, следовать лучшим практикам Azure по аутентификации и обработке ошибок, включать комментарии, объясняющие работу DefaultAzureCredential, а также быть хорошо структурированным с правильными функциями и документацией."

Azure MCP Server сгенерирует полный, готовый к производству Python-скрипт, который:
- Использует последнюю версию Azure Blob Storage SDK с правильными асинхронными паттернами
- Реализует DefaultAzureCredential с подробным объяснением цепочки откатов
- Включает надежную обработку ошибок с учётом специфических типов исключений Azure
- Следует лучшим практикам SDK Azure для управления ресурсами и обработки подключений
- Предоставляет подробный журнал действий и информативный вывод в консоль
- Создаёт правильно структурированный скрипт с функциями, документацией и аннотациями типов

Особенность в том, что без Azure MCP вы могли бы получить общий код для blob storage, который работает, но не соответствует актуальным паттернам Azure. С Azure MCP вы получаете код, который использует современные методы аутентификации, учитывает специфические ошибки Azure и следует рекомендованным Microsoft практикам для производственных приложений.

### 3. 🐙 GitHub MCP Server

[![Установить в VS Code](https://img.shields.io/badge/VS_Code-Install_Server-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=github&config=%7B%22type%22%3A%20%22http%22%2C%22url%22%3A%20%22https%3A%2F%2Fapi.githubcopilot.com%2Fmcp%2F%22%7D) [![Установить в VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Server-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=github&config=%7B%22type%22%3A%20%22http%22%2C%22url%22%3A%20%22https%3A%2F%2Fapi.githubcopilot.com%2Fmcp%2F%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/github/github-mcp-server)

Это не просто базовые операции с репозиториями — это полный набор инструментов, включая управление GitHub Actions, рабочие процессы pull request, отслеживание задач, проверку безопасности, уведомления и расширенные возможности автоматизации.

**Почему это полезно**: Этот сервер меняет способ взаимодействия с GitHub, предоставляя полный опыт работы с платформой прямо в вашей среде разработки. Вместо постоянного переключения между VS Code и GitHub.com для управления проектом, просмотра кода и мониторинга CI/CD, вы можете делать всё с помощью команд на естественном языке, оставаясь сосредоточенным на коде.

> **ℹ️ Примечание: Различные типы 'агентов'**  
>  
> Не путайте этот GitHub MCP Server с GitHub Coding Agent (ИИ-агентом, которому можно назначать задачи для автоматического кодирования). GitHub MCP Server работает в режиме агента VS Code для интеграции с GitHub API, в то время как GitHub Coding Agent — это отдельная функция, создающая pull request, когда ей назначают задачи GitHub issues.

**Основные возможности включают**:
- **⚙️ GitHub Actions**: Полное управление CI/CD пайплайнами, мониторинг рабочих процессов и обработка артефактов
- **🔀 Pull Requests**: Создание, просмотр, слияние и управление PR с полным отслеживанием статусов
- **🐛 Задачи (Issues)**: Полный цикл управления задачами, комментирование, маркировка и назначение
- **🔒 Безопасность**: Оповещения сканирования кода, обнаружение секретов и интеграция с Dependabot
- **🔔 Уведомления**: Интеллектуальное управление уведомлениями и подписками на репозитории
- **📁 Управление репозиториями**: Операции с файлами, управление ветками и администрирование репозиториев
- **👥 Сотрудничество**: Поиск пользователей и организаций, управление командами и контроль доступа

**Реальное использование**: "Создай pull request из моей feature-ветки", "Покажи все неудачные CI-запуски на этой неделе", "Перечисли открытые оповещения безопасности в моих репозиториях" или "Найди все задачи, назначенные мне во всех моих организациях"

**Полный демонстрационный сценарий**: Вот мощный рабочий процесс, демонстрирующий возможности GitHub MCP Server:

> "Мне нужно подготовиться к обзору спринта. Покажи все pull request, созданные мной на этой неделе, проверь статус наших CI/CD пайплайнов, создай сводку по оповещениям безопасности, которые нам надо обработать, и помоги составить релизные заметки на основе объединённых PR с меткой 'feature'."

GitHub MCP Server:
- Запросит недавние pull request с подробной информацией о статусах
- Проанализирует запуски рабочих процессов и выделит ошибки или проблемы с производительностью
- Скомпилирует результаты сканирования безопасности и приоретизирует критические оповещения
- Создаст комплексные релизные заметки, извлекая информацию из объединённых PR
- Предоставит действенные рекомендации для планирования спринта и подготовки релиза

**Пример из жизни**: Мне нравится использовать это для рабочих процессов код-ревью. Вместо того чтобы переключаться между VS Code, уведомлениями GitHub и страницами pull request, я могу сказать: "Покажи мне все PR, ожидающие моего обзора", а затем "Добавь комментарий к PR #123 с вопросом об обработке ошибок в методе аутентификации." Сервер выполняет GitHub API-запросы, поддерживает контекст обсуждения и даже помогает формировать более конструктивные комментарии к ревью.

**Опции аутентификации**: Сервер поддерживает как OAuth (бесшовно в VS Code), так и Personal Access Tokens, с возможностью выбора только нужных функций GitHub. Можно запускать как удалённый хостинг для мгновенной настройки, либо локально через Docker для полного контроля.

> **💡 Полезный совет**  
>  
> Включайте только необходимые наборы инструментов, настраивая параметр `--toolsets` в конфигурации MCP сервера, чтобы уменьшить размер контекста и улучшить выбор AI-инструментов. Например, добавьте `"--toolsets", "repos,issues,pull_requests,actions"` для основных рабочих процессов разработки, или используйте `"--toolsets", "notifications, security"`, если вам важен преимущественно мониторинг GitHub.

### 5. 📝 MarkItDown MCP Server
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_MarkItDown_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_MarkItDown_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/markitdown)

**Что это делает**: MarkItDown — это комплексный сервер конвертации документов, который преобразует различные форматы файлов в высококачественный Markdown, оптимизированный для обработки LLM и рабочих процессов анализа текста.

**Почему это полезно**: Необходим для современных рабочих процессов документации! MarkItDown обрабатывает впечатляющий спектр форматов файлов, при этом сохраняет важную структуру документа — такие элементы, как заголовки, списки, таблицы и ссылки. В отличие от простых инструментов извлечения текста, он сосредоточен на сохранении семантического смысла и форматирования, которые важны как для обработки искусственным интеллектом, так и для удобочитаемости человеком.

**Поддерживаемые форматы файлов**:
- **Офисные документы**: PDF, PowerPoint (PPTX), Word (DOCX), Excel (XLSX/XLS)
- **Медиафайлы**: изображения (с EXIF-метаданными и OCR), аудио (с EXIF-метаданными и транскрипцией речи)
- **Веб-контент**: HTML, RSS-ленты, URL-адреса YouTube, страницы Википедии
- **Данные**: CSV, JSON, XML, ZIP-файлы (рекурсивная обработка содержимого)
- **Публикации**: EPub, Jupyter ноутбуки (.ipynb)
- **Электронная почта**: сообщения Outlook (.msg)
- **Расширено**: интеграция Azure Document Intelligence для улучшенной обработки PDF

**Продвинутые функции**: MarkItDown поддерживает описания изображений с использованием LLM (при наличии клиента OpenAI), Azure Document Intelligence для улучшенной обработки PDF, транскрипцию аудио для речевого контента, а также систему плагинов для расширения поддержки дополнительных форматов файлов.

**Практическое применение**: «Преобразовать эту презентацию PowerPoint в Markdown для нашего сайта документации», «Извлечь текст из этого PDF с корректной структурой заголовков» или «Преобразовать эту таблицу Excel в читаемый формат таблицы».

**Пример из документации**: Цитируя [документацию MarkItDown](https://github.com/microsoft/markitdown#why-markdown):

> Markdown чрезвычайно близок к простому тексту, с минимальной разметкой или форматированием, но при этом предоставляет способ представления важной структуры документа. Основные LLM, такие как GPT-4o от OpenAI, «говорят» Markdown нативно и часто включают Markdown в свои ответы без запроса. Это говорит о том, что они обучались на огромных объемах текста в формате Markdown и хорошо его понимают. Дополнительным преимуществом является высокая эффективность использования токенов при работе с Markdown.

MarkItDown отлично сохраняет структуру документа, что важно для рабочих процессов с ИИ. Например, при конвертации презентации PowerPoint он сохраняет организацию слайдов с правильными заголовками, извлекает таблицы в формате Markdown, включает альтернативный текст для изображений и даже обрабатывает заметки докладчика. Графики преобразуются в читаемые таблицы данных, а итоговый Markdown сохраняет логическую структуру оригинальной презентации. Это делает его идеальным для подачи содержимого презентаций в ИИ-системы или создания документации по существующим слайдам.

Традиционно требовалась ручная разработка тестов или доступ к коду для контекста. Но с Playwright MCP можно тестировать внешние сайты, клиентские приложения или работать в сценариях black-box тестирования, где доступ к коду невозможен.

**Практическое применение**: «Настроить новый Dev Box с последним SDK .NET и подготовить его для нашего проекта», «Проверить статус всех моих сред разработки» или «Создать стандартизированную демонстрационную среду для презентаций команды»

**Пример из реальной жизни**: Я большой поклонник использования Dev Box для личной разработки. Момент прозрения у меня случился, когда Джеймс Монтеманьо сказал, как классно использовать Dev Box для демонстраций на конференциях, ведь там всегда есть очень быстрый Ethernet, несмотря на конференц-зал, отель или Wi-Fi в самолёте. Кстати, недавно я тренировался делать демодемонстрации на конференции, пока мой ноутбук был подключён к точке доступа телефона в автобусе из Брюгге в Антверпен! Следующий шаг — изучить командное управление множеством сред разработки и стандартизированные демонстрационные среды. Ещё один важный кейс, который я слышал от клиентов и коллег, — использование Dev Box для преднастроенных сред разработки. В обоих случаях использование MCP для настройки и управления Dev Box позволяет работать с ними через естественный язык, оставаясь в вашей среде разработки.

### 9. 🤖 Microsoft Foundry MCP Server
[![Установить в VS Code](https://img.shields.io/badge/VS_Code-Install_Microsoft_Foundry_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20Foundry%20MCP%20Server&config=%7B%22type%22%3A%22stdio%22%2C%22command%22%3A%22uvx%22%2C%22args%22%3A%5B%22--prerelease%3Dallow%22%2C%22--from%22%2C%22git%2Bhttps%3A%2F%2Fgithub.com%2Fazure-ai-foundry%2Fmcp-foundry.git%22%2C%22run-azure-ai-foundry-mcp%22%5D%7D) [![Установить в VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Microsoft_Foundry_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20Foundry%20MCP%20Server&config=%7B%22type%22%3A%22stdio%22%2C%22command%22%3A%22uvx%22%2C%22args%22%3A%5B%22--prerelease%3Dallow%22%2C%22--from%22%2C%22git%2Bhttps%3A%2F%2Fgithub.com%2Fazure-ai-foundry%2Fmcp-foundry.git%22%2C%22run-azure-ai-foundry-mcp%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/azure-ai-foundry/mcp-foundry)

**Что он делает**: Сервер Microsoft Foundry MCP предоставляет разработчикам полный доступ к экосистеме AI Azure, включая каталоги моделей, управление развертыванием, индексирование знаний с помощью Azure AI Search и инструменты оценки. Этот экспериментальный сервер устраняет разрыв между разработкой AI и мощной инфраструктурой AI Azure, упрощая создание, развертывание и оценку AI-приложений.

**Почему это полезно**: Этот сервер меняет способ работы с сервисами Azure AI, интегрируя возможности корпоративного уровня AI прямо в ваш процесс разработки. Вместо переключения между порталом Azure, документацией и IDE, вы можете находить модели, разворачивать сервисы, управлять базами знаний и оценивать производительность AI с помощью команд на естественном языке. Особенно полезен для разработчиков, создающих RAG (Retrieval-Augmented Generation) приложения, управляющих многомодельными развертываниями или реализующих комплексные цепочки оценки AI.

**Ключевые возможности для разработчиков**:
- **🔍 Поиск и развертывание моделей**: Исследуйте каталог моделей Microsoft Foundry, получайте подробную информацию с примерами кода и разворачивайте модели в Azure AI Services
- **📚 Управление знаниями**: Создавайте и управляйте индексами Azure AI Search, добавляйте документы, настраивайте индексаторы и создавайте сложные RAG-системы
- **⚡ Интеграция AI-агентов**: Подключайтесь к агентам Azure AI, запрашивайте существующих агентов и оценивайте их работу в производственных сценариях
- **📊 Фреймворк оценки**: Запускайте полную оценку текста и агентов, генерируйте отчёты в формате markdown и реализуйте контроль качества AI-приложений
- **🚀 Инструменты прототипирования**: Получите инструкции по настройке прототипирования на GitHub и доступ к Microsoft Foundry Labs для передовых исследовательских моделей

**Примеры реального использования**: «Развернуть модель Phi-4 в Azure AI Services для моего приложения», «Создать новый поисковый индекс для системы RAG в моей документации», «Оценить ответы моего агента по показателям качества», или «Найти лучшую модель рассуждения для моих сложных аналитических задач»

**Полный демонстрационный сценарий**: Вот мощный рабочий процесс разработки AI:

> «Я создаю агента поддержки клиентов. Помоги мне найти хорошую модель рассуждения в каталоге, развернуть её в Azure AI Services, создать базу знаний из нашей документации, настроить фреймворк оценки для проверки качества ответов и затем помочь спрототипировать интеграцию с токеном GitHub для тестирования.»

Сервер Microsoft Foundry MCP выполнит:
- Запрос к каталогу моделей для рекомендации оптимальных моделей рассуждения на основе ваших требований
- Предоставит команды по развертыванию и информацию о квотах для выбранного региона Azure
- Настроит индексы Azure AI Search с нужной схемой для вашей документации
- Сконфигурирует пайплайны оценки с метриками качества и проверками безопасности
- Сгенерирует код прототипирования с аутентификацией GitHub для мгновенного тестирования
- Предоставит полные руководства по настройке, адаптированные под ваш стек технологий

**Рекомендуемый пример**: Как разработчику, мне сложно уследить за множеством доступных моделей LLM. Я знаю несколько основных, но чувствовал, что упускаю выгоды по производительности и эффективности. Управление токенами и квотами вызывает стресс – я не всегда уверен, правильно ли выбираю модель для задачи и не трачу ли бюджет впустую. Я только что узнал об этом MCP от Джеймса Монтеманьо, когда общался с коллегами по рекомендациям MCP Server для этого поста, и я с радостью начну его использовать! Возможности поиска моделей выглядят особенно впечатляющими для таких, как я, кто хочет исследовать не только распространённые варианты, но и модели, оптимизированные для конкретных задач. Фреймворк оценки поможет убедиться, что результаты действительно улучшаются, а не просто пробуется что-то новое ради эксперимента.

> **ℹ️ Статус эксперимента**
