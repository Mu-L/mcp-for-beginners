# Сценарий 3: Встроенная документация с сервером MCP в VS Code

## Обзор

В этом сценарии вы узнаете, как интегрировать Microsoft Learn Docs непосредственно в вашу среду Visual Studio Code с помощью сервера MCP. Вместо постоянного переключения вкладок браузера для поиска документации вы сможете получать доступ, искать и ссылаться на официальные документы прямо внутри редактора. Этот подход оптимизирует ваш рабочий процесс, помогает оставаться сосредоточенным и обеспечивает бесшовную интеграцию с такими инструментами, как GitHub Copilot.

- Искать и читать документацию внутри VS Code, не покидая среды кодирования.
- Ссылаться на документацию и вставлять ссылки непосредственно в README или файлы курса.
- Использовать GitHub Copilot и MCP вместе для бесшовного рабочего процесса с поддержкой ИИ.

## Цели обучения

По окончании этой главы вы поймете, как настроить и использовать сервер MCP в VS Code для улучшения работы с документацией и разработки. Вы сможете:

- Настроить рабочее пространство для использования сервера MCP для поиска документации.
- Искать и вставлять документацию непосредственно из VS Code.
- Совместно использовать возможности GitHub Copilot и MCP для более продуктивного рабочего процесса с поддержкой ИИ.

Эти навыки помогут вам оставаться сосредоточенным, улучшить качество документации и повысить вашу продуктивность как разработчика или технического писателя.

## Решение

Для доступа к документации внутри редактора вы выполните серию шагов по интеграции сервера MCP с VS Code и GitHub Copilot. Это решение идеально подходит для авторов курсов, технических писателей и разработчиков, которые хотят сосредоточиться на работе в редакторе, используя документацию и Copilot.

- Быстро добавляйте ссылочные ссылки в README при написании курса или проектной документации.
- Используйте Copilot для генерации кода и MCP для мгновенного поиска и цитирования релевантной документации.
- Оставайтесь сосредоточенными в редакторе и повышайте продуктивность.

### Пошаговое руководство

Чтобы начать, выполните следующие шаги. Для каждого шага вы можете добавить скриншот из папки assets для наглядной иллюстрации процесса.

1. **Добавьте конфигурацию MCP:**
   В корне вашего проекта создайте файл `.vscode/mcp.json` и добавьте следующую конфигурацию:
   ```json
   {
     "servers": {
       "LearnDocsMCP": {
         "url": "https://learn.microsoft.com/api/mcp"
       }
     }
   }
   ```
   Эта конфигурация сообщает VS Code, как подключиться к [`Microsoft Learn Docs MCP server`](https://github.com/MicrosoftDocs/mcp).
   
   ![Шаг 1: Добавьте mcp.json в папку .vscode](../../../../../../translated_images/ru/step1-mcp-json.c06a007fccc3edfa.webp)
    
2. **Откройте панель GitHub Copilot Chat:**
   Если расширение GitHub Copilot у вас еще не установлено, откройте представление расширений в VS Code и установите его. Скачать можно напрямую с [Visual Studio Code Marketplace](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot-chat). Затем откройте панель Copilot Chat в боковой панели.

   ![Шаг 2: Откройте панель Copilot Chat](../../../../../../translated_images/ru/step2-copilot-panel.f1cc86e9b9b8cd1a.webp)

3. **Включите режим агента и проверьте инструменты:**
   В панели Copilot Chat включите режим агента.

   ![Шаг 3: Включите режим агента и проверьте инструменты](../../../../../../translated_images/ru/step3-agent-mode.cdc32520fd7dd1d1.webp)

   После включения режима агента убедитесь, что сервер MCP указан среди доступных инструментов. Это позволит агенту Copilot получать доступ к серверу документации для поиска релевантной информации.
   
   ![Шаг 3: Проверьте инструмент сервера MCP](../../../../../../translated_images/ru/step3-verify-mcp-tool.76096a6329cbfecd.webp)
4. **Начните новый чат и задайте вопросы агенту:**
   Откройте новый чат в панели Copilot Chat. Теперь вы можете задавать агенту вопросы по документации. Агент будет использовать сервер MCP для поиска и отображения соответствующей документации Microsoft Learn прямо в вашем редакторе.

   - *"Я пытаюсь составить учебный план по теме X. Планирую изучать её в течение 8 недель, предложи, какой контент мне стоит проходить каждую неделю."*

   ![Шаг 4: Задайте вопрос агенту в чате](../../../../../../translated_images/ru/step4-prompt-chat.12187bb001605efc.webp)

5. **Живой запрос:**

   > Возьмём живой запрос из раздела [#get-help](https://discord.gg/D6cRhjHWSC) в Microsoft Foundry Discord ([просмотреть оригинальное сообщение](https://discord.com/channels/1113626258182504448/1385498306720829572)):
   
   *"Я ищу ответы о том, как развернуть мультиагентное решение с ИИ-агентами, разработанными на Azure AI Foundry. Понимаю, что нет прямого метода развертывания, такого как каналы Copilot Studio. Какие существуют способы развертывания для корпоративных пользователей, чтобы взаимодействовать и выполнять задачи? Есть много статей/блогов, которые говорят, что можно использовать Azure Bot Service, который будет мостом между MS Teams и Azure AI Foundry Agents. Сработает ли это, если я создам Azure bot, подключённый к Orchestrator Agent на Azure AI Foundry через Azure function для оркестрации, или нужно создавать Azure function для каждого из ИИ-агентов мультиагентного решения для оркестрации в Bot framework? Любые другие предложения приветствуются."*

   ![Шаг 5: Живые запросы](../../../../../../translated_images/ru/step5-live-queries.49db3e4a50bea273.webp)

   Агент ответит релевантными ссылками на документацию и резюме, которые вы сможете вставить прямо в markdown-файлы или использовать в качестве ссылок в коде.
   
### Примеры запросов

Вот некоторые примеры запросов, которые вы можете попробовать. Они демонстрируют, как сервер MCP и Copilot могут совместно обеспечивать мгновенную, контекстно-зависимую документацию и ссылки без выхода из VS Code:

- "Покажи, как использовать триггеры Azure Functions."

Эти запросы демонстрируют, как MCP сервер и Copilot работают вместе, чтобы обеспечить мгновенную, контекстно-зависимую документацию и ссылки непосредственно в VS Code.
