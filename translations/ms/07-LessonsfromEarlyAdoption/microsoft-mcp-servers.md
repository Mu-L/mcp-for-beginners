## 🔧 Memahami Pelayan MCP: Panduan untuk Pemula

### Apa Itu Pelayan MCP?

Sebagai pemula kepada Model Context Protocol (MCP), anda mungkin tertanya-tanya: "Apa sebenarnya pelayan MCP, dan mengapa saya patut peduli?" Mari mulakan dengan analogi mudah.

Fikirkan pelayan MCP seperti pembantu khusus yang membantu pembantu kod AI anda (seperti GitHub Copilot) berhubung dengan alat dan perkhidmatan luar. Sama seperti anda menggunakan aplikasi yang berbeza pada telefon untuk tugasan berbeza—satu untuk cuaca, satu untuk navigasi, satu untuk perbankan—pelayan MCP memberi pembantu AI anda keupayaan untuk berinteraksi dengan pelbagai alat dan perkhidmatan pembangunan.

### Masalah Yang Diselesaikan Pelayan MCP

Sebelum wujudnya pelayan MCP, jika anda mahu:  
- Semak sumber Azure anda  
- Cipta isu GitHub  
- Tanya pangkalan data anda  
- Cari dalam dokumentasi  

Anda perlu berhenti menulis kod, buka pelayar, pergi ke laman web yang sesuai, dan lakukan tugasan ini secara manual. Peralihan kontekstual yang kerap ini memecahkan fokus dan mengurangkan produktiviti anda.

### Bagaimana Pelayan MCP Mengubah Pengalaman Pembangunan Anda

Dengan pelayan MCP, anda boleh kekal dalam persekitaran pembangunan anda (VS Code, Visual Studio, dll.) dan hanya minta pembantu AI mengendalikan tugasan ini. Contohnya:

**Daripada aliran kerja tradisional ini:**  
1. Berhenti menulis kod  
2. Buka pelayar  
3. Pergi ke portal Azure  
4. Cari butiran akaun storan  
5. Kembali ke VS Code  
6. Sambung semula menulis kod  

**Anda kini boleh buat ini:**  
1. Tanya AI: "Apakah status akaun storan Azure saya?"  
2. Terus menulis kod dengan maklumat yang diberikan

### Manfaat Utama untuk Pemula

#### 1. 🔄 **Kekal dalam Keadaan Fokus Anda**  
- Tiada lagi tukar antara pelbagai aplikasi  
- Kekalkan fokus pada kod yang anda tulis  
- Kurangkan beban mental mengurus pelbagai alat

#### 2. 🤖 **Gunakan Bahasa Semula Jadi Daripada Arahan Kompleks**  
- Daripada menghafal sintaks SQL, terangkan data yang anda perlukan  
- Daripada mengingati arahan Azure CLI, jelaskan apa yang hendak dicapai  
- Biarkan AI uruskan butiran teknikal sementara anda fokus pada logik

#### 3. 🔗 **Sambungkan Pelbagai Alat Bersama**  
- Cipta aliran kerja berkuasa dengan menggabungkan perkhidmatan berbeza  
- Contoh: "Dapatkan semua isu GitHub terkini dan cipta item kerja Azure DevOps berkaitan"  
- Bina automasi tanpa menulis skrip rumit

#### 4. 🌐 **Akses Ekosistem Yang Semakin Berkembang**  
- Manfaatkan pelayan yang dibina oleh Microsoft, GitHub, dan syarikat lain  
- Campur dan padan alat dari vendor berlainan dengan lancar  
- Sertai ekosistem piawai yang berfungsi merentas pembantu AI berbeza

#### 5. 🛠️ **Belajar Dengan Melakukan**  
- Mulakan dengan pelayan siap untuk faham konsep  
- Secara beransur-ansur bina pelayan anda sendiri apabila lebih selesa  
- Gunakan SDK dan dokumentasi tersedia untuk panduan pembelajaran

### Contoh Dunia Sebenar untuk Pemula

Katakan anda baru dalam pembangunan web dan sedang mengusahakan projek pertama. Berikut cara pelayan MCP boleh membantu:

**Pendekatan tradisional:**  
```
1. Code a feature
2. Open browser → Navigate to GitHub
3. Create an issue for testing
4. Open another tab → Check Azure docs for deployment
5. Open third tab → Look up database connection examples
6. Return to VS Code
7. Try to remember what you were doing
```
  
**Dengan pelayan MCP:**  
```
1. Code a feature
2. Ask AI: "Create a GitHub issue for testing this login feature"
3. Ask AI: "How do I deploy this to Azure according to the docs?"
4. Ask AI: "Show me the best way to connect to my database"
5. Continue coding with all the information you need
```

## Memulakan Dengan MCP Dalam VS Code dan Visual Studio 🚀

Menyediakan pelayan MCP ini mudah jika anda menggunakan Visual Studio Code atau Visual Studio 2022 dengan GitHub Copilot.

### Persediaan VS Code

Proses asas untuk VS Code adalah:  

1. **Aktifkan Mod Ejen**: Dalam VS Code, tukar ke mod Ejen di tetingkap GitHub Copilot Chat  
2. **Konfigurasikan Pelayan MCP**: Tambah konfigurasi pelayan dalam fail settings.json VS Code anda  
3. **Mulakan Pelayan**: Klik butang "Mula" untuk setiap pelayan yang ingin anda gunakan  
4. **Pilih Alat**: Pilih pelayan MCP mana yang hendak diaktifkan untuk sesi anda kini  

Untuk arahan persediaan terperinci, lihat [dokumentasi MCP VS Code](https://code.visualstudio.com/docs/copilot/copilot-mcp).

> **💡 Petua Pro: Urus Pelayan MCP Seperti Profesional!**  
>  
> Paparan Extensions VS Code kini termasuk [UI baru untuk urus pelayan MCP yang dipasang](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode)! Anda mempunyai akses pantas untuk mula, berhenti, dan urus apa sahaja pelayan MCP yang dipasang menggunakan antara muka yang jelas dan mudah. Cuba sekarang!

### Persediaan Visual Studio 2022

Untuk Visual Studio 2022 (versi 17.14 atau lebih baru):  

1. **Aktifkan Mod Ejen**: Klik dropdown "Ask" di tetingkap GitHub Copilot Chat dan pilih "Agent"  
2. **Cipta Fail Konfigurasi**: Buat fail `.mcp.json` dalam direktori penyelesaian anda (lokasi disarankan: `<SOLUTIONDIR>\.mcp.json`)  
3. **Konfigurasikan Pelayan**: Tambah konfigurasi pelayan MCP menggunakan format MCP standard  
4. **Kelulusan Alat**: Apabila diminta, luluskan alat yang ingin digunakan dengan skop kebenaran yang sesuai  

Untuk arahan persediaan Visual Studio terperinci, lihat [dokumentasi MCP Visual Studio](https://learn.microsoft.com/visualstudio/ide/mcp-servers).

Setiap pelayan MCP datang dengan syarat konfigurasi tersendiri (rentetan sambungan, pengesahan, dll.), tetapi corak persediaannya seragam di kedua-dua IDE.

[![Pasang dalam VS Code](https://img.shields.io/badge/VS_Code-Install_Microsoft_Docs_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D) [![Pasang dalam VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Microsoft_Docs_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/mcp)

**Apa yang dilakukan**: Pelayan MCP Dokumen Microsoft Learn ialah perkhidmatan berasaskan awan yang menyediakan pembantu AI dengan akses masa nyata kepada dokumentasi rasmi Microsoft melalui Model Context Protocol. Ia berhubung dengan `https://learn.microsoft.com/api/mcp` dan membolehkan carian semantik merentas Microsoft Learn, dokumentasi Azure, dokumentasi Microsoft 365, dan sumber rasmi Microsoft lain.

**Mengapa ia berguna**: Walaupun nampak seperti "hanya dokumentasi," pelayan ini sangat penting untuk setiap pembangun yang menggunakan teknologi Microsoft. Salah satu rungutan terbesar daripada pembangun .NET tentang pembantu kod AI ialah mereka tidak dikemas kini dengan keluaran .NET dan C# terkini. Pelayan MCP Dokumen Microsoft Learn menyelesaikan masalah ini dengan menyediakan akses masa nyata kepada dokumentasi paling terkini, rujukan API, dan amalan terbaik. Sama ada anda bekerja dengan SDK Azure terbaru, meneroka ciri C# 13 baru, atau melaksanakan corak terkini .NET Aspire, pelayan ini memastikan pembantu AI anda mempunyai akses kepada maklumat rasmi dan terkini yang diperlukan untuk menjana kod yang tepat dan moden.

**Penggunaan dunia sebenar**: "Apakah arahan az cli untuk mencipta aplikasi kontena Azure menurut dokumentasi rasmi Microsoft Learn?" atau "Bagaimana saya mengkonfigurasi Entity Framework dengan penyuntik ketergantungan dalam ASP.NET Core?" Atau bagaimana dengan "Semak kod ini untuk memastikan ia selaras dengan cadangan prestasi dalam Dokumentasi Microsoft Learn." Pelayan menyediakan liputan komprehensif merentas Microsoft Learn, dokumen Azure, dan dokumentasi Microsoft 365 menggunakan carian semantik lanjutan untuk mencari maklumat paling relevan secara kontekstual. Ia memulangkan sehingga 10 kandungan berkualiti tinggi dengan tajuk artikel dan URL, sentiasa mengakses dokumentasi Microsoft terkini sebaik sahaja diterbitkan.

**Contoh unggul**: Pelayan mendedahkan alat `microsoft_docs_search` yang melakukan carian semantik terhadap dokumentasi teknikal rasmi Microsoft. Setelah dikonfigurasikan, anda boleh bertanya soalan seperti "Bagaimana saya melaksanakan pengesahan JWT dalam ASP.NET Core?" dan mendapat respons terperinci, rasmi dengan pautan sumber. Kualiti carian sangat baik kerana ia memahami konteks – bertanya tentang "kontena" dalam konteks Azure akan mengembalikan dokumentasi Azure Container Instances, manakala istilah sama dalam konteks .NET mengembalikan maklumat koleksi C# yang relevan.

Ini sangat berguna untuk perpustakaan dan kes penggunaan yang berubah cepat atau baru dikemas kini. Sebagai contoh, dalam beberapa projek pengkodan baru-baru ini saya ingin memanfaatkan ciri dalam keluaran terkini .NET Aspire dan Microsoft.Extensions.AI. Dengan memasukkan pelayan MCP Dokumen Microsoft Learn, saya dapat menggunakan bukan sahaja dokumen API, tetapi juga panduan dan panduan yang baru diterbitkan.

### 2. ☁️ Pelayan MCP Azure
[![Pasang dalam VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![Pasang dalam VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

**Apa fungsinya**: Azure MCP Server adalah satu suite menyeluruh dengan lebih daripada 15 penyambung perkhidmatan Azure khusus yang membawa keseluruhan ekosistem Azure ke dalam aliran kerja AI anda. Ini bukan sekadar satu pelayan tunggal – ia adalah koleksi berkuasa yang merangkumi pengurusan sumber, sambungan pangkalan data (PostgreSQL, SQL Server), analisis log Azure Monitor dengan KQL, integrasi Cosmos DB, dan banyak lagi.

**Mengapa ia berguna**: Selain daripada hanya mengurus sumber Azure, pelayan ini secara dramatik meningkatkan kualiti kod apabila bekerja dengan Azure SDK. Apabila anda menggunakan Azure MCP dalam mod Ejen, ia bukan sahaja membantu anda menulis kod – ia membantu anda menulis kod Azure *yang lebih baik* yang mengikuti pola pengesahan semasa, amalan pengendalian ralat terbaik, dan memanfaatkan ciri SDK terkini. Daripada mendapat kod generik yang mungkin berfungsi, anda mendapat kod yang mengikuti pola disyorkan Azure untuk beban kerja pengeluaran.

**Modul utama termasuk**:
- **🗄️ Penyambung Pangkalan Data**: Akses bahasa semula jadi terus ke Azure Database untuk PostgreSQL dan SQL Server
- **📊 Azure Monitor**: Analisis log berkuasa KQL dan wawasan operasi
- **🌐 Pengurusan Sumber**: Pengurusan kitar hayat sumber Azure sepenuhnya
- **🔐 Pengesahan**: Pola DefaultAzureCredential dan identiti terurus
- **📦 Perkhidmatan Simpanan**: Operasi Blob Storage, Queue Storage, dan Table Storage
- **🚀 Perkhidmatan Kontena**: Pengurusan Azure Container Apps, Container Instances, dan AKS
- **Dan banyak penyambung khusus lain**

**Penggunaan dunia sebenar**: "Senaraikan akaun storan Azure saya", "Kueri ruang kerja Log Analytics saya untuk ralat dalam sejam terakhir", atau "Bantu saya membina aplikasi Azure menggunakan Node.js dengan pengesahan yang betul"

**Senario demo penuh**: Berikut adalah panduan lengkap yang menunjukkan kekuatan gabungan Azure MCP dengan peluasan GitHub Copilot untuk Azure dalam VS Code. Apabila kedua-duanya dipasang dan arahan diberikan:

> "Cipta skrip Python yang memuat naik fail ke Azure Blob Storage menggunakan pengesahan DefaultAzureCredential. Skrip harus disambungkan ke akaun storan Azure saya bernama 'mycompanystorage', muat naik ke kontena bernama 'documents', buat fail ujian dengan cap masa semasa untuk dimuat naik, kendalikan ralat dengan lancar serta berikan output yang informatif, ikut amalan terbaik Azure untuk pengesahan dan pengendalian ralat, sertakan komen yang menerangkan bagaimana pengesahan DefaultAzureCredential berfungsi, dan pastikan skrip tersusun dengan fungsi dan dokumentasi yang betul."

Azure MCP Server akan menjana skrip Python lengkap siap digunakan untuk pengeluaran yang:
- Menggunakan SDK Azure Blob Storage terkini dengan pola async yang betul
- Melaksanakan DefaultAzureCredential dengan penjelasan rantai fallback yang menyeluruh
- Termasuk pengendalian ralat yang kukuh dengan jenis pengecualian Azure khusus
- Mengikuti amalan terbaik Azure SDK untuk pengurusan sumber dan sambungan
- Menyediakan log terperinci dan output konsol yang informatif
- Mencipta skrip tersusun dengan fungsi, dokumentasi, dan petunjuk jenis

Yang luar biasa adalah tanpa Azure MCP, anda mungkin mendapat kod penyimpanan blob generik yang berfungsi tetapi tidak mengikuti pola Azure semasa. Dengan Azure MCP, anda mendapat kod yang memanfaatkan kaedah pengesahan terkini, mengendalikan senario ralat khusus Azure, dan mengikuti amalan yang disyorkan Microsoft untuk aplikasi pengeluaran.

**Contoh yang diketengahkan**: Saya pernah bergelut mengingati perintah khusus untuk CLI `az` dan `azd` untuk kegunaan ad-hoc. Ia sentiasa proses dua langkah bagi saya: pertama cari sintaksis, kemudian jalankan perintah. Saya sering masuk ke portal dan klik di sana sini untuk menyiapkan kerja kerana saya tidak mahu mengaku saya tidak dapat ingat sintaks CLI. Mampu hanya menerangkan apa yang saya mahu adalah sesuatu yang menakjubkan, malah lebih bagus dapat melakukan itu tanpa meninggalkan IDE saya!

Terdapat senarai kegunaan hebat dalam [repo Azure MCP](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) untuk memulakan anda. Untuk panduan pemasangan menyeluruh dan pilihan konfigurasi lanjutan, rujuk [dokumentasi rasmi Azure MCP](https://learn.microsoft.com/azure/developer/azure-mcp-server/).

### 3. 🐙 GitHub MCP Server

[![Pasang dalam VS Code](https://img.shields.io/badge/VS_Code-Install_Server-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=github&config=%7B%22type%22%3A%20%22http%22%2C%22url%22%3A%20%22https%3A%2F%2Fapi.githubcopilot.com%2Fmcp%2F%22%7D) [![Pasang dalam VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Server-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=github&config=%7B%22type%22%3A%20%22http%22%2C%22url%22%3A%20%22https%3A%2F%2Fapi.githubcopilot.com%2Fmcp%2F%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/github/github-mcp-server)

**Apa fungsinya**: GitHub MCP Server rasmi menyediakan integrasi lancar dengan keseluruhan ekosistem GitHub, menawarkan pilihan capaian jauh dihos dan penyebaran Docker tempatan. Ini bukan hanya operasi repositori asas – ia juga kit alat menyeluruh yang merangkumi pengurusan GitHub Actions, aliran kerja pull request, penjejakan isu, imbasan keselamatan, pemberitahuan, dan keupayaan automasi lanjutan.

**Mengapa ia berguna**: Pelayan ini mengubah cara anda berinteraksi dengan GitHub dengan membawa pengalaman platform penuh terus ke persekitaran pembangunan anda. Daripada sentiasa bertukar antara VS Code dan GitHub.com untuk pengurusan projek, semakan kod, dan pemantauan CI/CD, anda boleh mengendalikan semuanya melalui arahan bahasa semula jadi sambil kekal fokus pada kod anda.

> **ℹ️ Nota: Jenis 'Ejen' Berbeza**
> 
> Jangan kelirukan GitHub MCP Server ini dengan GitHub Coding Agent (ejen AI yang anda boleh tugaskan isu untuk tugas pengekodan automatik). GitHub MCP Server berfungsi dalam mod Ejen VS Code untuk menyediakan integrasi API GitHub, manakala GitHub Coding Agent adalah ciri berasingan yang mencipta pull request apabila ditugaskan ke isu GitHub.

**Keupayaan utama termasuk**:
- **⚙️ GitHub Actions**: Pengurusan lengkap saluran CI/CD, pemantauan aliran kerja, dan pengendalian artifak
- **🔀 Pull Requests**: Cipta, semak, gabung, dan urus PR dengan penjejakan status menyeluruh
- **🐛 Isu**: Pengurusan kitar hayat isu sepenuhnya, komen, pelabelan, dan penugasan
- **🔒 Keselamatan**: Amaran imbas kod, pengesanan rahsia, dan integrasi Dependabot
- **🔔 Pemberitahuan**: Pengurusan pemberitahuan pintar dan kawalan langganan repositori
- **📁 Pengurusan Repositori**: Operasi fail, pengurusan cawangan, dan pengurusan repositori
- **👥 Kolaborasi**: Carian pengguna dan organisasi, pengurusan pasukan, dan kawalan akses

**Penggunaan dunia sebenar**: "Cipta pull request dari cawangan ciri saya", "Tunjukkan semua larian CI gagal minggu ini", "Senaraikan amaran keselamatan terbuka untuk repositori saya", atau "Cari semua isu yang ditugaskan kepada saya dalam organisasi saya"

**Senario demo penuh**: Berikut adalah aliran kerja berkuasa yang menunjukkan keupayaan GitHub MCP Server:

> "Saya perlu bersedia untuk ulasan sprint kami. **Apa yang ia lakukan**: MarkItDown ialah pelayan penukaran dokumen yang komprehensif yang mengubah pelbagai format fail menjadi Markdown berkualiti tinggi, dioptimumkan untuk penggunaan LLM dan aliran kerja analisis teks.

**Mengapa ia berguna**: Penting untuk aliran kerja dokumentasi moden! MarkItDown mengendalikan pelbagai format fail yang mengagumkan sambil memelihara struktur dokumen penting seperti tajuk, senarai, jadual, dan pautan. Berbeza dengan alat ekstrak teks ringkas, ia fokus pada mengekalkan makna semantik dan pemformatan yang bernilai untuk pemprosesan AI dan keterbacaan manusia.

**Format fail yang disokong**:
- **Dokumen Pejabat**: PDF, PowerPoint (PPTX), Word (DOCX), Excel (XLSX/XLS)
- **Fail Media**: Imej (dengan metadata EXIF dan OCR), Audio (dengan metadata EXIF dan transkripsi ucapan)
- **Kandungan Web**: HTML, suapan RSS, URL YouTube, halaman Wikipedia
- **Format Data**: CSV, JSON, XML, fail ZIP (memproses kandungan secara rekursif)
- **Format Penerbitan**: EPub, buku nota Jupyter (.ipynb)
- **Emel**: Mesej Outlook (.msg)
- **Lanjutan**: Integrasi Kecerdasan Dokumen Azure untuk pemprosesan PDF yang dipertingkatkan

**Keupayaan lanjutan**: MarkItDown menyokong penerangan imej dipacu LLM (apabila disediakan klien OpenAI), Kecerdasan Dokumen Azure untuk pemprosesan PDF yang dipertingkatkan, transkripsi audio untuk kandungan ucapan, dan sistem plugin untuk meluaskan kepada format fail tambahan.

**Penggunaan di dunia sebenar**: "Tukarkan pembentangan PowerPoint ini ke Markdown untuk laman dokumentasi kami", "Ekstrak teks dari PDF ini dengan struktur tajuk yang betul", atau "Tukarkan helaian Excel ini ke format jadual yang boleh dibaca"

**Contoh kes terpilih**: Untuk memetik [dokumentasi MarkItDown](https://github.com/microsoft/markitdown#why-markdown):

> Markdown sangat hampir dengan teks biasa, dengan markup atau pemformatan yang minima, tetapi masih menyediakan cara untuk mewakili struktur dokumen penting. LLM arus perdana, seperti GPT-4o OpenAI, secara semulajadi "berbicara" Markdown, dan sering memasukkan Markdown ke dalam respons mereka tanpa disuruh. Ini menunjukkan bahawa mereka telah dilatih dengan sejumlah besar teks berformat Markdown, dan memahaminya dengan baik. Sebagai faedah sampingan, konvensyen Markdown juga sangat cekap token.

MarkItDown sangat baik dalam memelihara struktur dokumen, yang penting untuk aliran kerja AI. Sebagai contoh, apabila menukar pembentangan PowerPoint, ia mengekalkan organisasi slaid dengan tajuk yang betul, mengekstrak jadual sebagai jadual Markdown, menyertakan teks alt untuk imej, dan bahkan memproses nota penceramah. Carta ditukar menjadi jadual data yang boleh dibaca, dan Markdown yang terhasil mengekalkan aliran logik pembentangan asal. Ini menjadikannya sesuai untuk memberi isian kandungan pembentangan ke dalam sistem AI atau mencipta dokumentasi dari slaid sedia ada.

## 📋 Kesimpulan

Model Context Protocol (MCP) mengubah cara pembangun berinteraksi dengan pembantu AI dan alat luaran. 10 pelayan Microsoft MCP ini menunjukkan kuasa integrasi AI yang distandardisasi, membolehkan aliran kerja yang lancar yang mengekalkan pembangun dalam keadaan fokus sambil mengakses kebolehan luaran yang hebat.

Daripada integrasi ekosistem Azure yang komprehensif hingga ke alat khusus seperti Playwright untuk automasi pelayar dan MarkItDown untuk pemprosesan dokumen, pelayan-pelayan ini mempamerkan bagaimana MCP boleh meningkatkan produktiviti dalam pelbagai senario pembangunan. Protokol standard memastikan alat-alat ini berfungsi bersama dengan lancar, mewujudkan pengalaman pembangunan yang padu.

Semasa ekosistem MCP terus berkembang, kekal berhubung dengan komuniti, meneroka pelayan baru, dan membina penyelesaian tersuai akan menjadi kunci untuk memaksimumkan produktiviti pembangunan anda. Sifat standard terbuka MCP bermakna anda boleh mencampur dan padankan alat daripada pelbagai vendor untuk mencipta aliran kerja sempurna mengikut keperluan khusus anda.
