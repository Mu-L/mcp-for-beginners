# Scenario 3: In-Editor Docs with MCP Server in VS Code

## အနှစ်ချုပ်

ဤအခြေအနေတွင် သင်သည် MCP ဆာဗာကို အသုံးပြု၍ Microsoft Learn စာရွက်စာတမ်းများကို Visual Studio Code ပတ်ဝန်းကျင်အတွင်းတွင် တိုက်ရိုက် ရယူနိုင်သည်မှာ လေ့လာသင်ယူမည်ဖြစ်သည်။ Documentation ရှာဖွေရန်အတွက် browser tab များကို ဆက်တိုက်လွှဲပြောင်းရန် အစား သင့် editor အတွင်းမှ တိုက်ရိုက် စာရွက်စာတမ်းများကို ရယူ၊ ရှာဖွေပြီး ကိုးကားနိုင်ပါသည်။ ဤနည်းလမ်းသည် သင့်လုပ်ငန်းစဉ်ကို လျင်မြန်စေပြီး၊ အာရုံစိုက်စေပြီး၊ GitHub Copilot ကဲ့သို့သော ကိရိယာများနှင့် အဆက်အသွယ် ပေါင်းစည်းမှုအတွက် သက်ပြန့်စေသည်။

- VS Code တွင် coding ပတ်ဝန်းကျင်ကို မထွက်ဘဲ စာရွက်စာတမ်းများကို ရှာဖွေနိုင်ပြီး ဖတ်ရှုနိုင်သည်။
- README သို့မဟုတ် သင်တန်းဖိုင်များထဲသို့ စာရွက်စာတမ်းများကို ကိုးကားခြင်းနှင့် link များထည့်သွင်းနိုင်သည်။
- GitHub Copilot နှင့် MCP ကို ပေါင်းစပ်အသုံးပြု၍ AI အားဖြင့် စိစစ်ထားသော စာရွက်စာတမ်း လုပ်ငန်းစဉ်ကို ဖန်တီးနိုင်သည်။

## သင်ယူရမည့် ရည်မှန်းချက်များ

ဤအခန်းအဆုံးတွင် MCP ဆာဗာအား VS Code ထဲတွင် တပ်ဆင်ခြင်းနှင့် အသုံးပြုခြင်းနည်းလမ်းများကို နားလည်ရမည်ဖြစ်ပြီး စာရွက်စာတမ်းနှင့် ဖွံ့ဖြိုးတိုးတက်မှုလုပ်ငန်းစဉ်များကို မြှင့်တင်နိုင်မည်ဖြစ်သည်။

- MCP ဆာဗာကို အသုံးပြုရန် workspace ကို ဖန်တီးတပ်ဆင်နိုင်ရန်
- VS Code အတွင်းမှ တိုက်ရိုက် စာရွက်စာတမ်းများ ရှာဖွေပြီး ထည့်သွင်းနိုင်ရန်
- GitHub Copilot နှင့် MCP ၏ စွမ်းအားများကို ပေါင်းစပ်အသုံးပြု၍ ထုတ်လုပ်မှု မြှင့်တင်နိုင်ရန်

ဤကျွမ်းကျင်မှုများသည် သင့်အာရုံစူးစိုက်မှုကိုထိန်းသိမ်းပေးကာ စာရွက်စာတမ်းအရည်အသွးကို တိုးတက်စေပြီး developer သို့မဟုတ် နည်းပညာရေးသူအဖြစ် ထုတ်လုပ်မှုကို မြှင့်တင်ပေးမည် ဖြစ်သည်။

## ဖြေရှင်းနည်း

In-editor စာရွက်စာတမ်း လက်တွေ့အသုံးပြုမှုရရှိရန် MCP ဆာဗာကို VS Code နှင့် GitHub Copilot နှင့်ပေါင်းစည်းပါမည်။ ဒီဖြေရှင်းနည်းသည် သင်တန်းရေးသူများ၊ စာရွက်စာတမ်းရေးသူများနှင့် developer များအတွက် စာရွက်စာတမ်းနဲ့ Copilot နှစ်ခုစလုံးကို editor အတွင်းနေထိုင်၍ အသုံးပြုလိုသူများအတွက် အထူးကောင်းမွန်ပါသည်။

- သင်တန်း သို့မဟုတ် ပရောဂျက် စာရွက်စာတမ်းရေးစဉ် README ဖိုင်တွင် ဖော်ပြချက် link များကို အမြန်ထည့်သွင်းနိုင်သည်။
- Copilot သည် ကုဒ်ထုတ်ဖန်တီးပြီး MCP သည် သက်ဆိုင်ရာ စာရွက်စာတမ်းကို ချက်ချင်း ရှာဖွေနိုင်သည်။
- Editor အတွင်းမှာပဲ အာရုံစိုက် နှင့် ထုတ်လုပ်မှုမြှင့်တင်ခြင်း။

### အဆင့်ဆင့် လမ်းညွှန်ချက်

စတင်ရန်၊ အောက်ပါ အဆင့်များကို လိုက်နာပါ။ အဆင့်တိုင်းတွင် assets ဖိုင်ထဲမှ screenshot တစ်ပုံထည့်၍ လုပ်ငန်းစဉ်ကို ရုပ်ပြနိုင်သည်။

1. **MCP configuration ထည့်ပါ:**
   သင့်ပရောဂျက် ၏ root တွင် `.vscode/mcp.json` ဖိုင်ကို ဖန်တီးပြီး အောက်ပါ configuration ကိုထည့်ပါ။
   ```json
   {
     "servers": {
       "LearnDocsMCP": {
         "url": "https://learn.microsoft.com/api/mcp"
       }
     }
   }
   ```
   ဒီ configuration က VS Code ကို [`Microsoft Learn Docs MCP server`](https://github.com/MicrosoftDocs/mcp) နှင့် မည်သို့ ချိတ်ဆက်ရမည်ကို ပြသသည်။
   
   ![Step 1: Add mcp.json to .vscode folder](../../../../../../translated_images/my/step1-mcp-json.c06a007fccc3edfa.webp)
    
2. **GitHub Copilot Chat panel ကိုဖွင့်ပါ:**
   GitHub Copilot extension ကို မထည့်သွင်းထားသောသူများအနေဖြင့် VS Code ၏ Extensions မျက်နှာပြင်သို့ သွား၍ ထည့်သွင်းပါ။ [Visual Studio Code Marketplace](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot-chat) မှ တိုက်ရိုက်ဒေါင်းလုပ်လည်း ရနိုင်ပါသည်။ ထို့နောက် Sidebar မှ Copilot Chat panel ကို ဖွင့်ပါ။

   ![Step 2: Open Copilot Chat panel](../../../../../../translated_images/my/step2-copilot-panel.f1cc86e9b9b8cd1a.webp)

3. **Agent mode ကိုဖွင့်ပြီး ကိရိယာများကို သေချာစစ်ဆေးပါ:**
   Copilot Chat panel တွင် agent mode ကို ဖွင့်ပါ။

   ![Step 3: Enable agent mode and verify tools](../../../../../../translated_images/my/step3-agent-mode.cdc32520fd7dd1d1.webp)

   agent mode ဖွင့်ပြီးနောက် MCP ဆာဗာသည် အသုံးပြုနိုင်သောကိရိယာများစာရင်းတွင် ပါရှိနေသည်ကို စစ်ဆေးပါ။ ၎င်းက Copilot agent သည် စာရွက်စာတမ်းဆာဗာမှ သက်ဆိုင်ရာ သတင်းအချက်အလက်များ ရယူနိုင်ရန် သေချာစေသည်။
   
   ![Step 3: Verify MCP server tool](../../../../../../translated_images/my/step3-verify-mcp-tool.76096a6329cbfecd.webp)
4. **Chat အသစ် ပြုလုပ်ပြီး agent ကို မေးမြန်းပါ:**
   Copilot Chat panel တွင် chat အသစ် ဖွင့်ပြီး စာရွက်စာတမ်း မေးခွန်းများကို agent ကို ဖြေပေးရန် မေးမြန်းနိုင်ပါပြီ။ Agent သည် MCP ဆာဗာကို အသုံးပြုကာ သင့် editor အတွင်း Microsoft Learn စာရွက်စာတမ်းကို ရယူပြသလိမ့်မည်။

   - *"Topic X အတွက် သင်ယူအစီအစဉ်ရေးချင်ပါတယ်။ ၈ ပတ် တစ်ပတ်စီအတွက် အကြောင်းအရာများကို အကြံပြုပါ။"*

   ![Step 4: Prompt the agent in chat](../../../../../../translated_images/my/step4-prompt-chat.12187bb001605efc.webp)

5. **တိုက်ရိုက် မေးခွန်းများ (Live Query):**

   > Microsoft Foundry Discord ၏ [#get-help](https://discord.gg/D6cRhjHWSC) အပိုင်းမှ တိုက်ရိုက်မေးခွန်းတစ်ခုကို ကြည့်ပါ ([မူရင်းစာတမ်းကြည့်ရန်](https://discord.com/channels/1113626258182504448/1385498306720829572))။
   
   *"Azure AI Foundry မှ တီထွင်ထားသော AI agent များနှင့် အတူ မျိုးစုံ agent ဖြေရှင်းနည်း တပ်ဆင်ခြင်းအတွက် ဖြေရှင်းနည်းများ ကို လေ့လာလိုပါသည်။ Copilot Studio channels အတိုင်း တိုက်ရိုက် တပ်ဆင်နည်း မရှိပေမယ့် enterprise အသုံးပြုသူများ စီမံဆောင်ရွက်နိုင်ရန် မည်သည့် နည်းလမ်းများ ရှိသနည်း? Azure Bot service ကို MS teams နှင့် Azure AI Foundry Agents တို့ကို ချိတ်ဆက်ရန် တံတားအဖြစ် အသုံးပြုနိုင်သည်ဆိုသော်လည်း Orchestrator Agent သို့ Azure function မှတဆင့် ချိတ်ဆက်ပါက ရှေ့ဆောင်မလုပ် ရသလား? မျိုးစုံ agent တစ်ခုချင်း ရဲ့ orchestration အတွက် Azure function ကို တစ်ခုချင်းစီ ဖန်တီးရမလား? အခြားအကြံပြုချက်များလည်း ကြိုဆိုပါသည်။"*

   ![Step 5: Live queries](../../../../../../translated_images/my/step5-live-queries.49db3e4a50bea273.webp)

   Agent သည် သက်ဆိုင်ရာ စာရွက်စာတမ်း link များနှင့် အကျဥ်းချုပ်များ ဖြင့် တုံ႔ပြန်မည်ဖြစ်ပြီး ၎င်းတို့ကို markdown ဖိုင်များသို့ တိုက်ရိုက်ထည့်သွင်းသုံးစွဲနိုင်သည်။

### နမူနာ မေးခွန်းများ

ဤနေရာတွင် သင်စမ်းကြည့်နိုင်သည့် မေးခွန်းများဖြစ်သည်။ ထိုမေးခွန်းများသည် MCP ဆာဗာနှင့် Copilot တို့ပေါင်းစပ်ကာ VS Code မှ ထွက်မနေဘဲ အချက်ချင်း၊ ပတ်ဝန်းကျင်အလိုက် စာရွက်စာတမ်းများနှင့် ကိုးကားချက်များ ထည့်သွင်းပေးနိုင်မှုကို ပြသမည်။

- "Azure Functions triggers ကို ဘယ်လိုသုံးရမလဲ ပြပါ။"
- "Azure Key Vault အတွက် တရားဝင် စာရွက်စာတမ်း link ကို ထည့်ပါ။"
- "Azure အရင်းအမြစ်များအတွက် အကောင်းဆုံး လုံခြုံရေး လေ့လာမှုနည်းလမ်းများ ဘာတွေလဲ?"
- "Azure AI services အတွက် quickstart ရှာပါ။"

ဤ မေးခွန်းများသည် MCP ဆာဗာနှင့် Copilot တို့ပေါင်းစပ်က VS Code မှ ထွက်မနေဘဲ စာရွက်စာတမ်းများနှင့် ကိုးကားချက်များ ချက်ချင်း ရယူနိုင်မည့် နည်းလမ်းကို ပြသပါမည်။
