### Посилення безпеки та усунення вразливостей залежностей

Виконано повний аудит безпеки всіх маніфестів залежностей та коду прикладів, після чого усунено всі виявлені npm зауваження та одну вразливість на рівні коду. Після усунення `npm audit` повідомляє про **0 вразливостей** у всіх перевірених директоріях.

#### Вразливості залежностей npm (транзитивні) — виправлено

Аудитовано всі 15 зафіксованих `package-lock.json` файлів. Вразливості обмежувались транзитивними залежностями, які підтягуються інструментом MCP Inspector, клієнтом OpenAI та SDK MCP; тепер вони усі вирішені без порушення роботи зразків:
- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/inspector** і **lab3/code/weather_mcp/inspector**: оновлено `@modelcontextprotocol/inspector` (`0.16.6` / `0.14.1` → `0.22.0`), що усунуло згадані у комплекті `ajv`, `brace-expansion`, `diff`, `path-to-regexp` та `ws` зауваження. Додано npm `overrides`, який примусово задає запатчений `shell-quote@1.8.4` для усунення критичного зауваження від `concurrently`; повторно згенеровано обидва lock-файли (тепер 0 вразливостей)
- **03-GettingStarted/samples/typescript**: `npm audit fix` оновив транзитивний `qs` (помірний рівень) до запатченої версії
- **03-GettingStarted/samples/javascript**: `npm audit fix` оновив транзитивний `hono` (помірний рівень) до запатченої версії
- **03-GettingStarted/03-llm-client/solution/typescript**: `npm audit fix` оновив транзитивний `form-data` (високий рівень) до запатченої версії
- **03-GettingStarted/11-simple-auth/solution/typescript**: створено відсутній `package-lock.json` для відтворюваності та аудитуємісті (0 вразливостей)

#### Виправлення безпеки на рівні коду (OWASP A03: Injection)

- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/src/server.py**: Видалено `shell=True` з інструменту `open_in_vscode`. Попередній виклик `subprocess.run(["start", "", vscode_path, folder_path], shell=True)` дозволяв інтерпретацію shell-метасимволів у шляху до папки через `cmd.exe` (вектор командної ін’єкції). Тепер запускається безпосередньо виконуваний `Code.exe` з аргументом папки — без shell, що функціонально еквівалентно та безпечно

#### Аудит залежностей Python

- Аудитовано всі sets вимог Python за допомогою `pip-audit`. `05-AdvancedTopics` та `03-GettingStarted/samples/python` повідомили про **відсутність відомих вразливостей** (їхні діапазони `mcp`, `httpx`, `pydantic`, `python-dotenv` рҳезолвляться до актуальних патчованих релізів)
- **09-CaseStudy/docs-mcp/solution/python/requirements.txt**: `pip-audit` виявив транзитивну залежність **`werkzeug` 3.1.1** із трьома зауваженнями DoS через `safe_join` для імен пристроїв Windows — `CVE-2025-66221`, `CVE-2026-21860` та `CVE-2026-27199` (усі виправлені в 3.1.6). Додано явне обмеження безпеки `werkzeug>=3.1.6` для резолва до запатченого релізу; перевірено, що обмеження коректно працює з ланцюгом `chainlit` / `mcp` / `semantic-kernel`
