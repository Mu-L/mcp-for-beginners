# Сценарій 3: Документація в редакторі з MCP сервером у VS Code

## Огляд

У цьому сценарії ви дізнаєтесь, як інтегрувати Microsoft Learn Docs безпосередньо у ваше середовище Visual Studio Code за допомогою MCP сервера. Замість постійного перемикання між вкладками браузера для пошуку документації, ви можете отримувати доступ, шукати та посилатися на офіційні документи прямо в редакторі. Такий підхід оптимізує ваш робочий процес, допомагає залишатися зосередженим та дає змогу безшовно інтегруватися з інструментами, як GitHub Copilot.

- Пошук та читання документації всередині VS Code без виходу з середовища кодування.
- Посилання на документацію та вставлення гіперпосилань безпосередньо до README або навчальних файлів.
- Використання GitHub Copilot і MCP разом для безшовного робочого процесу з підтримкою штучного інтелекту.

## Навчальні цілі

До кінця цього розділу ви навчитеся налаштовувати та використовувати MCP сервер у VS Code для покращення процесу документування та розробки. Ви зможете:

- Налаштовувати робочий простір для використання MCP сервера для пошуку документації.
- Шукати та вставляти документацію безпосередньо у VS Code.
- Поєднувати можливості GitHub Copilot та MCP для більш продуктивного робочого процесу з підтримкою ШІ.

Ці навички допоможуть вам залишатися зосередженими, покращити якість документації та підвищити продуктивність як розробника або технічного письменника.

## Рішення

Для отримання доступу до документації всередині редактора ви виконаєте ряд кроків, які інтегрують MCP сервер з VS Code та GitHub Copilot. Це рішення ідеально підходить для авторів курсів, письменників документації та розробників, які хочуть зберігати фокус у редакторі під час роботи з документацією і Copilot.

- Швидко додавати посилання на README під час написання курсу або проектної документації.
- Використовувати Copilot для генерації коду, а MCP — для миттєвого пошуку та цитування відповідних документів.
- Залишатися зосередженим у редакторі та підвищувати продуктивність.

### Детальний покроковий посібник

Щоб розпочати, виконайте наступні кроки. Для кожного кроку ви можете додати скріншот із папки assets для візуалізації процесу.

1. **Додайте конфігурацію MCP:**
   У корені вашого проєкту створіть файл `.vscode/mcp.json` і вставте наступну конфігурацію:
   ```json
   {
     "servers": {
       "LearnDocsMCP": {
         "url": "https://learn.microsoft.com/api/mcp"
       }
     }
   }
   ```
   Ця конфігурація повідомляє VS Code, як підключитися до [`Microsoft Learn Docs MCP server`](https://github.com/MicrosoftDocs/mcp).
   
   ![Крок 1: Додайте mcp.json у папку .vscode](../../../../../../translated_images/uk/step1-mcp-json.c06a007fccc3edfa.webp)
    
2. **Відкрийте панель GitHub Copilot Chat:**
   Якщо у вас ще не встановлено розширення GitHub Copilot, перейдіть до перегляду розширень у VS Code та встановіть його. Ви можете завантажити його безпосередньо з [Visual Studio Code Marketplace](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot-chat). Потім відкрийте панель Copilot Chat у бічній панелі.

   ![Крок 2: Відкрийте панель Copilot Chat](../../../../../../translated_images/uk/step2-copilot-panel.f1cc86e9b9b8cd1a.webp)

3. **Увімкніть агентський режим і перевірте інструменти:**
   У панелі Copilot Chat увімкніть агентський режим.

   ![Крок 3: Увімкніть агентський режим і перевірте інструменти](../../../../../../translated_images/uk/step3-agent-mode.cdc32520fd7dd1d1.webp)

   Після увімкнення агентського режиму переконайтеся, що MCP сервер зазначений серед доступних інструментів. Це гарантує, що Copilot агент може отримувати інформацію з сервера документації.

   ![Крок 3: Перевірте інструмент MCP сервер](../../../../../../translated_images/uk/step3-verify-mcp-tool.76096a6329cbfecd.webp)
4. **Почніть новий чат і задавайте запити агенту:**
   Відкрийте новий чат у панелі Copilot Chat. Тепер ви можете ставити агенту запити щодо документації. Агент буде використовувати MCP сервер для отримання та відображення відповідної документації Microsoft Learn безпосередньо у вашому редакторі.

   - *"Я намагаюся написати навчальний план для теми X. Я планую вивчати її протягом 8 тижнів, для кожного тижня запропонуй контент, який слід опрацювати."*

   ![Крок 4: Задайте запит агенту у чаті](../../../../../../translated_images/uk/step4-prompt-chat.12187bb001605efc.webp)

5. **Запит у реальному часі:**

   > Розглянемо живий запит із розділу [#get-help](https://discord.gg/D6cRhjHWSC) у Microsoft Foundry Discord ([переглянути оригінальне повідомлення](https://discord.com/channels/1113626258182504448/1385498306720829572)):
   
   *"Я шукаю відповіді на питання, як розгорнути мультиагентське рішення з AI агентами, розробленими на Azure AI Foundry. Бачу, що немає прямого методу розгортання, такого як канали Copilot Studio. Тож які є різні способи розгортання для корпоративних користувачів, щоб взаємодіяти і виконувати завдання? Існує багато статей/блогів, в яких говориться, що можна використати Azure Bot service для цього, що може слугувати мостом між MS Teams та Azure AI Foundry Agents. Чи працюватиме це, якщо я налаштую Azure bot, який підключатиметься до Orchestrator Agent на Azure AI Foundry через Azure function для оркестрації, або мені потрібно створювати Azure function для кожного AI агента у мультиагентському рішенні, щоб робити оркестрацію на Bot framework? Буду вдячний за будь-які інші пропозиції."*

   ![Крок 5: Запити у реальному часі](../../../../../../translated_images/uk/step5-live-queries.49db3e4a50bea273.webp)

   Агент надасть відповіді з посиланнями на релевантну документацію та підсумками, які ви можете вставити безпосередньо у ваші markdown файли або використовувати як посилання у коді.
   
### Приклади запитів

Ось декілька прикладів запитів, які можна спробувати. Ці запити демонструють, як MCP сервер і Copilot працюють разом, щоб надати миттєву документацію з контекстом і посиланнями без виходу з VS Code:

- "Покажи, як використовувати тригери Azure Functions."
- "Встав посилання на офіційну документацію для Azure Key Vault."
- "Які кращі практики для забезпечення безпеки ресурсів Azure?"
- "Знайди quickstart для Azure AI services."

Ці запити показують, як MCP сервер і Copilot можуть працювати разом, щоб надавати миттєву документацію з контексту та посиланнями без виходу з VS Code.
