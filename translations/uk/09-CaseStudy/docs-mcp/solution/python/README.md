# Генератор плану навчання з Chainlit та Microsoft Learn Docs MCP

## Вимоги

- Python 3.8 або новіша версія
- pip (менеджер пакетів Python)
- Доступ до Інтернету для підключення до сервера Microsoft Learn Docs MCP

## Встановлення

1. Клонуйте це репозиторій або завантажте файли проекту.
2. Встановіть необхідні залежності:

   ```bash
   pip install -r requirements.txt
   ```

## Використання

### Сценарій 1: Простий запит до Docs MCP
Клієнт командного рядка, який підключається до сервера Docs MCP, надсилає запит і виводить результат.

1. Запустіть скрипт:
   ```bash
   python scenario1.py
   ```
2. Введіть ваше запитання до документації у запиті.

### Сценарій 2: Генератор плану навчання (веб-застосунок Chainlit)
Веб-інтерфейс (на базі Chainlit), який дозволяє користувачам створювати персоналізований покроковий план навчання на будь-яку технічну тему.

1. Запустіть застосунок Chainlit:
   ```bash
   chainlit run scenario2.py
   ```
2. Відкрийте локальну URL-адресу, вказану у вашому терміналі (наприклад, http://localhost:8000) у браузері.
3. У вікні чату введіть тему для навчання і кількість тижнів для вивчення (наприклад, "AI-900 сертифікація, 8 тижнів").
4. Застосунок надасть план навчання по тижнях з посиланнями на відповідну документацію Microsoft Learn.

**Необхідні змінні середовища:**

Щоб використовувати Сценарій 2 (веб-застосунок Chainlit з Azure OpenAI), потрібно встановити такі змінні середовища у файлі `.env` в каталозі `python`:

```
AZURE_OPENAI_CHAT_DEPLOYMENT_NAME=
AZURE_OPENAI_API_KEY=
AZURE_OPENAI_ENDPOINT=
AZURE_OPENAI_API_VERSION=
```

Заповніть ці значення даними вашого ресурсу Azure OpenAI перед запуском застосунку.

> [!TIP]
> Ви можете легко розгорнути власні моделі за допомогою [Microsoft Foundry](https://ai.azure.com/).

### Сценарій 3: Документація в редакторі з MCP сервером у VS Code

Замість перемикання вкладок браузера для пошуку документації ви можете інтегрувати Microsoft Learn Docs прямо у VS Code за допомогою MCP сервера. Це дозволяє:
- Шукати та читати документацію у VS Code, не виходячи з редактора.
- Посилатися на документацію та вставляти посилання безпосередньо в README або файли курсу.
- Використовувати GitHub Copilot і MCP разом для безшовної роботи з документацією з підтримкою ШІ.

**Приклади використання:**
- Швидко додавати посилання на джерела у README під час написання документації курсу чи проєкту.
- Використовувати Copilot для генерації коду й MCP для миттєвого пошуку та цитування релевантної документації.
- Залишатися сфокусованим в редакторі та підвищувати продуктивність.

> [!IMPORTANT]
> Переконайтеся, що у вашому робочому просторі є дійсний файл конфігурації [`mcp.json`](../../../../../../09-CaseStudy/docs-mcp/solution/scenario3/mcp.json) (розташування: `.vscode/mcp.json`).

## Чому Chainlit для Сценарію 2?

Chainlit — це сучасний open-source фреймворк для створення конверсійних веб-застосунків. Він спрощує створення чат-інтерфейсів користувача, які підключаються до бекенд-сервісів, таких як Microsoft Learn Docs MCP сервер. Цей проєкт використовує Chainlit, щоб запропонувати простий і інтерактивний спосіб створення персоналізованих планів навчання в режимі реального часу. Використовуючи Chainlit, ви можете швидко будувати та розгортати чат-інструменти, які підвищують продуктивність та навчання.

## Що це робить

Цей застосунок дозволяє користувачам створювати персоналізований план навчання, просто ввівши тему та тривалість. Застосунок аналізує ваш запит, надсилає запити на сервер Microsoft Learn Docs MCP для пошуку релевантного контенту та організовує результати у структурований покроковий план по тижнях. Рекомендації кожного тижня відображаються в чаті, що полегшує слідкування й відстеження прогресу. Інтеграція забезпечує, що ви завжди отримуєте найновіші і найбільш релевантні навчальні матеріали.
