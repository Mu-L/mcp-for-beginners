# AGENTS.md

## Огляд Проєкту

**MCP для Початківців** — це відкритий освітній курс для вивчення Model Context Protocol (MCP) — стандартизованої рамки для взаємодії між AI-моделями та клієнтськими додатками. Цей репозиторій містить комплексні навчальні матеріали з практичними прикладами коду на різних мовах програмування.

### Ключові Технології

- **Мови програмування**: C#, Java, JavaScript, TypeScript, Python, Rust
- **Фреймворки та SDK**: 
  - MCP SDK (`@modelcontextprotocol/sdk`)
  - Spring Boot (Java)
  - FastMCP (Python)
  - LangChain4j (Java)
- **Бази даних**: PostgreSQL з розширенням pgvector
- **Хмарні платформи**: Azure (Container Apps, OpenAI, Content Safety, Application Insights)
- **Інструменти збірки**: npm, Maven, pip, Cargo
- **Документація**: Markdown з автоматизованим багатомовним перекладом (понад 48 мов)

### Архітектура

- **11 основних модулів (00-11)**: Послідовний шлях навчання від базових до просунутих тем
- **Практичні лабораторії**: Завдання з повними рішеннями на кількох мовах програмування
- **Прикладні проєкти**: Робочі реалізації MCP-сервера та клієнта
- **Система перекладів**: Автоматизований конвеєр GitHub Actions для підтримки багатьох мов
- **Зображення**: Централізована папка із зображеннями та їх перекладеними версіями

## Команди для Налаштування

Цей репозиторій орієнтований на документацію. Більшість налаштувань відбувається у окремих прикладах та лабораторіях.

### Налаштування Репозиторію

```bash
# Клонуйте репозиторій
git clone https://github.com/microsoft/mcp-for-beginners.git
cd mcp-for-beginners
```

### Робота з Прикладними Проєктами

Прикладні проєкти розташовані в:
- `03-GettingStarted/samples/` — приклади для конкретних мов
- `03-GettingStarted/01-first-server/solution/` — реалізації першого сервера
- `03-GettingStarted/02-client/solution/` — реалізації клієнта
- `11-MCPServerHandsOnLabs/` — лабораторії з інтеграції баз даних

Кожен приклад має свої інструкції з налаштування:

#### Проєкти TypeScript/JavaScript
```bash
cd <project-directory>
npm install
npm start
```

#### Проєкти на Python
```bash
cd <project-directory>
pip install -r requirements.txt
# або
pip install -e .
python main.py
```

#### Проєкти на Java
```bash
cd <project-directory>
mvn clean install
mvn spring-boot:run
```

## Робочий Процес Розробки

### Структура Документації

- **Модулі 00-11**: Основний навчальний матеріал у послідовному порядку
- **translations/**: Версії для різних мов (генеруються автоматично, не редагувати вручну)
- **translated_images/**: Локалізовані версії зображень (генеруються автоматично)
- **images/**: Початкові зображення та схеми

### Внесення Змін до Документації

1. Редагуйте лише англомовні markdown-файли у кореневих папках модулів (00-11)
2. При потребі оновлюйте зображення в папці `images/`
3. Автоматизований co-op-translator GitHub Action генерує переклади
4. Переклади оновлюються щоразу при push у гілку main

### Робота з Перекладами

- **Автоматичний переклад**: Конвеєр GitHub Actions виконує всі переклади
- **Не редагуйте вручну** файли у папці `translations/`
- Метадані перекладу вбудовані в кожен перекладений файл
- Підтримується понад 48 мов, включно з арабською, китайською, французькою, німецькою, хінді, японською, корейською, португальською, російською, іспанською та ін.

## Керівництво зі Стилю Коду

### Стиль Документації

- Використовуйте чітку, зрозумілу мову для початківців
- Включайте приклади коду на кількох мовах, де це доречно
- Дотримуйтеся найкращих практик markdown:
  - Заголовки ATX (`#` синтаксис)
  - Використовуйте fenced code blocks з позначенням мови
  - Включайте описовий alt-текст для зображень
  - Тримайте довжину рядків розумною (без жорстких обмежень, але логічно)

### Стиль Прикладів Коду

#### TypeScript/JavaScript
- Використовуйте ES-модулі (`import`/`export`)
- Дотримуйтеся суворого режиму TypeScript
- Включайте анотації типів
- Цільова версія ES2022

#### Python
- Виконуйте правила стилю PEP 8
- Використовуйте типові підказки (type hints) де доречно
- Включайте докстрінги для функцій і класів
- Використовуйте сучасні можливості Python (3.8+)

#### Java
- Дотримуйтеся конвенцій Spring Boot
- Використовуйте можливості Java 21
- Дотримуйтеся стандартної структури проекту Maven
- Включайте коментарі Javadoc

## Правила Співпраці

### Процес Pull Request

1. **Форк і клацання**: Форкніть репозиторій та клоніть свій форк локально
2. **Створіть гілку**: Використовуйте описові імена гілок (наприклад, `fix/typo-module-3`, `add/python-example`)
3. **Внесіть зміни**: Редагуйте лише англомовні markdown-файли (не переклади)
4. **Тестуйте локально**: Переконайтеся, що markdown відображається коректно
5. **Надішліть PR**: Використовуйте чіткі назви та описи PR
6. **CLA**: Підпишіть Microsoft Contributor License Agreement при запиті

### Формат Назви PR

Використовуйте чіткі, описові назви:
- `[Module XX] Короткий опис` для змін у модулі
- `[Samples] Опис` для змін у прикладах коду
- `[Docs] Опис` для загальних оновлень документації

### Поширені Проблеми

**Q: Мій PR не проходить перевірку перекладу**  
A: Переконайтеся, що редагували лише англомовні markdown-файли у кореневих директоріях модулів, не переклади.

**Q: Як додати нову мову?**  
A: Підтримка мов керується через конвеєр co-op-translator. Відкрийте issue для обговорення додавання нових мов.

**Q: Приклади коду не працюють**  
A: Переконайтеся, що дотрималися інструкцій встановлення у README конкретного прикладу. Перевірте правильність версій залежностей.

**Q: Зображення не відображаються**  
A: Перевірте, що шляхи до зображень відносні та використовують прямі слеші. Зображення повинні бути у `images/` або `translated_images/` для локалізованих версій.
