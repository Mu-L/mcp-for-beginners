## 🔧 Розуміння серверів MCP: посібник для початківців

### Що таке сервери MCP?

Як початківець у Model Context Protocol (MCP), ви, можливо, замислюєтеся: "Що саме таке сервер MCP і чому це важливо?" Почнемо з простої аналогії.

Уявіть, що сервери MCP — це спеціалізовані помічники, які допомагають вашому AI-компаньйону для кодування (наприклад, GitHub Copilot) підключатися до зовнішніх інструментів і сервісів. Так само, як ви використовуєте різні додатки на телефоні для різних завдань — один для погоди, інший для навігації, третій для банкінгу — сервери MCP дають вашому AI-помічнику можливість взаємодіяти з різними інструментами розробки та сервісами.

### Проблема, яку вирішують сервери MCP

До появи серверів MCP, якщо ви хотіли:
- Перевірити ваші ресурси Azure
- Створити GitHub issue
- Запитати базу даних
- Шукати у документації

Вам доводилося припиняти кодування, відкривати браузер, переходити на відповідний сайт і виконувати ці завдання вручну. Постійне переключення контексту порушує ваш потік та знижує продуктивність.

### Як сервери MCP трансформують ваш досвід розробки

За допомогою серверів MCP ви можете залишатися у вашому середовищі розробки (VS Code, Visual Studio тощо) і просто попросити AI-помічника виконати ці завдання. Наприклад:

**Замість такого традиційного робочого процесу:**
1. Припинити кодування
2. Відкрити браузер
3. Зайти в портал Azure
4. Перевірити дані облікового запису зберігання
5. Повернутися до VS Code
6. Продовжити кодування

**Ви можете робити так:**
1. Запитати AI: "Який стан моїх облікових записів зберігання Azure?"
2. Продовжити кодувати, отримавши потрібну інформацію

## Початок роботи з MCP у VS Code і Visual Studio 🚀

Налаштувати ці сервери MCP просто, якщо ви використовуєте Visual Studio Code або Visual Studio 2022 з GitHub Copilot.

### Налаштування VS Code

Основні кроки для VS Code:

1. **Увімкнути режим агента**: у VS Code перемкніться в режим агента у вікні Copilot Chat
2. **Налаштувати сервери MCP**: додайте конфігурації серверів у файл settings.json VS Code
3. **Запустити сервери**: натисніть кнопку «Start» для кожного потрібного сервера
4. **Вибрати інструменти**: увімкніть потрібні сервери MCP для поточної сесії

Для детальної інструкції дивіться [документацію MCP для VS Code](https://code.visualstudio.com/docs/copilot/copilot-mcp).

> **💡 Професійна порада: Керуйте серверами MCP як профі!**
> 
> Тепер у в'ю Extensions VS Code є [зручний новий інтерфейс для керування встановленими серверами MCP](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode)! Ви можете швидко запускати, зупиняти та керувати будь-якими серверами MCP за допомогою простого і зрозумілого інтерфейсу. Спробуйте!

### Налаштування Visual Studio 2022

Для Visual Studio 2022 (версія 17.14 або новіша):

1. **Увімкнути режим агента**: натисніть випадаюче меню «Ask» у вікні GitHub Copilot Chat і виберіть «Agent»
2. **Створити конфігураційний файл**: створіть файл `.mcp.json` у директорії рішення (рекомендоване розташування: `<SOLUTIONDIR>\.mcp.json`)
3. **Налаштувати сервери**: додайте конфігурації серверів MCP за стандартним форматом MCP
4. **Затвердження інструментів**: коли буде запит, дозвольте інструментам працювати з відповідними дозволами області

Для детальної інструкції дивіться [документацію MCP для Visual Studio](https://learn.microsoft.com/visualstudio/ide/mcp-servers).

Кожен сервер MCP має власні вимоги до конфігурації (рядки підключення, автентифікація тощо), але схема налаштування послідовна у обох IDE.

### 1. 📚 Сервер Microsoft Learn Docs MCP

[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Microsoft_Docs_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Microsoft_Docs_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/mcp)

**Що робить**: Сервер Microsoft Learn Docs MCP — це хмарний сервіс, який надає AI-помічникам доступ у реальному часі до офіційної документації Microsoft через Model Context Protocol. Він підключається до `https://learn.microsoft.com/api/mcp` і забезпечує семантичний пошук по Microsoft Learn, документації Azure, документації Microsoft 365 та інших офіційних джерел Microsoft.

**Чому це корисно**: Хоча може здатися, що це «просто документація», цей сервер насправді є критично важливим для кожного розробника, що працює з технологіями Microsoft. Одним із найбільших скарг розробників .NET на AI-помічників кодування є їх застарілі знання щодо останніх версій .NET і C#. Сервер Microsoft Learn Docs MCP вирішує цю проблему, надаючи доступ у реальному часі до найновішої документації, API-довідників і найкращих практик. Незалежно від того, чи ви працюєте з останніми SDK Azure, досліджуєте нові можливості C# 13 або впроваджуєте передові патерни .NET Aspire, цей сервер гарантує, що ваш AI-помічник має доступ до авторитетної, актуальної інформації для генерації точного, сучасного коду.

**Приклад реального використання**: «Які команди az cli для створення контейнерного додатка Azure відповідно до офіційної документації Microsoft Learn?» або «Як налаштувати Entity Framework з впровадженням залежностей в ASP.NET Core?» Або «Перевір цей код, щоб впевнитися, що він відповідає рекомендаціям з продуктивності з документації Microsoft Learn». Сервер забезпечує всебічне охоплення Microsoft Learn, документації Azure та документації Microsoft 365, використовуючи просунутий семантичний пошук для знаходження найбільш контекстно релевантної інформації. Він повертає до 10 високоякісних фрагментів контенту із заголовками статей і URL-адресами, завжди звертаючись до найновішої документації Microsoft одразу після її публікації.

**Виділений приклад**: Сервер надає інструмент `microsoft_docs_search`, який виконує семантичний пошук по офіційній технічній документації Microsoft. Після налаштування можна ставити запитання, наприклад, «Як імплементувати автентифікацію JWT у ASP.NET Core?» і отримувати докладні офіційні відповіді з посиланнями на джерела. Якість пошуку виключна, бо він розуміє контекст — запит про «контейнери» у контексті Azure поверне документацію про Azure Container Instances, тоді як той самий термін у .NET контексті поверне інформацію про колекції C#.

Це особливо корисно для швидкозмінних або нещодавно оновлених бібліотек і випадків використання. Наприклад, в деяких моїх недавніх проєктах я хотів використати можливості останніх релізів .NET Aspire і Microsoft.Extensions.AI. Додавши сервер Microsoft Learn Docs MCP, я зміг скористатися не тільки API-документацією, а й керівництвами та інструкціями, які щойно були опубліковані.

### 2. ☁️ Сервер Azure MCP
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

**Що це робить**: Azure MCP Server — це комплексний набір із понад 15 спеціалізованих конекторів сервісів Azure, які інтегрують всю екосистему Azure у ваш AI-робочий процес. Це не просто один сервер — це потужна колекція, що включає управління ресурсами, підключення до баз даних (PostgreSQL, SQL Server), аналіз журналів Azure Monitor за допомогою KQL, інтеграцію Cosmos DB та багато іншого.

**Чому це корисно**: Окрім управління ресурсами Azure, цей сервер істотно покращує якість коду при роботі з Azure SDK. Використовуючи Azure MCP в режимі агента, ви не просто отримуєте допомогу у написанні коду — ви отримуєте *кращий* код Azure, який відповідає актуальним шаблонам автентифікації, найкращим практикам обробки помилок і використовує найновіші можливості SDK. Замість загального коду, який може працювати, ви отримуєте код, що слідує рекомендованим Azure паттернам для продуктивних навантажень.

**Основні модулі включають**:
- **🗄️ Підключення до баз даних**: Прямий доступ природною мовою до Azure Database для PostgreSQL та SQL Server
- **📊 Azure Monitor**: Аналіз журналів на основі KQL та операційні інсайти
- **🌐 Управління ресурсами**: Повний життєвий цикл управління ресурсами Azure
- **🔐 Автентифікація**: Шаблони DefaultAzureCredential та керованої ідентичності
- **📦 Служби зберігання**: Операції з Blob Storage, Queue Storage та Table Storage
- **🚀 Контейнерні служби**: Управління Azure Container Apps, Container Instances та AKS
- **Й багато інших спеціалізованих конекторів**

**Приклади використання в реальному житті**: "Покажи мої облікові записи зберігання в Azure", "Запитай мій простір Log Analytics щодо помилок за останню годину", або "Допоможи мені створити Azure-додаток на Node.js з правильною автентифікацією"

**Повний демонстраційний сценарій**: Ось повний покроковий приклад, який демонструє потужність поєднання Azure MCP з розширенням GitHub Copilot для Azure у VS Code. Якщо у вас встановлені обидва інструменти, і ви вводите:

> "Створи Python-скрипт, який завантажує файл в Azure Blob Storage, використовуючи автентифікацію DefaultAzureCredential. Скрипт повинен підключитись до мого облікового запису зберігання Azure з назвою 'mycompanystorage', завантажити в контейнер під назвою 'documents', створити тестовий файл з поточним часом, коректно обробляти помилки і виводити інформативні повідомлення, слідувати найкращим практикам Azure для автентифікації та обробки помилок, містити коментарі про роботу автентифікації DefaultAzureCredential, і бути добре структурованим із належними функціями та документацією."

Azure MCP Server згенерує повний, готовий до продуктивного використання Python-скрипт, який:
- Використовує останній SDK Azure Blob Storage з правильними асинхронними паттернами
- Реалізує DefaultAzureCredential з докладним описом ланцюга відмов
- Містить надійну обробку помилок із конкретними типами виключень Azure
- Слідує найкращим практикам SDK Azure для управління ресурсами та з'єднаннями
- Забезпечує детальне логування та інформативний вивід в консоль
- Створює належним чином структурований скрипт з функціями, документацією та підказками типів

Це особливо вражає, тому що без Azure MCP ви могли б отримати загальний код Blob Storage, який може працювати, але не слідує сучасним Azure паттернам. З Azure MCP ви отримуєте код, що використовує найновіші методи автентифікації, обробляє специфічні для Azure сценарії помилок і слідує рекомендованим Microsoft практикам для продуктивних додатків.

Для мене це завжди двоетапний процес: спершу шукати синтаксис, потім запускати команду. Часто я просто заходжу в портал і клацаю по інтерфейсу, бо не хочу визнавати, що не пам'ятаю синтаксис CLI. Це не лише базові операції з репозиторіями — це комплексний набір засобів, що включає управління GitHub Actions, workflow pull request, відстеження issue, сканування безпеки, повідомлення та розширені можливості автоматизації.

**Чому це корисно**: Цей сервер трансформує вашу взаємодію з GitHub, перенісши повний досвід платформи безпосередньо у ваше середовище розробки. Замість постійного перемикання між VS Code та GitHub.com для управління проєктами, рев’ю коду та моніторингу CI/CD, ви можете керувати всіма процесами через команди природною мовою, залишаючись сфокусованими на коді.

> **ℹ️ Примітка: Різні типи «агентів»**
> 
> Не плутайте цей GitHub MCP Server з GitHub Coding Agent (AI-агентом, якому можна призначати issues для автоматизованого кодування). GitHub MCP Server працює у режимі агента VS Code для інтеграції із GitHub API, тоді як GitHub Coding Agent — це окрема функція, яка створює pull request при призначенні на GitHub issues.

**Ключові можливості включають**:
- **⚙️ GitHub Actions**: Повне управління CI/CD конвеєрами, моніторинг workflow і обробка артефактів
- **🔀 Pull Requests**: Створення, перегляд, злиття та керування PR з докладним відстеженням статусу
- **🐛 Issues**: Повне життєве коло issue — керування, коментування, маркування та призначення
- **🔒 Безпека**: Сповіщення про сканування коду, виявлення секретів та інтеграція з Dependabot
- **🔔 Повідомлення**: Розумне управління сповіщеннями та підписками на репозиторії
- **📁 Управління репозиторіями**: Операції з файлами, керування гілками та адміністрування репозиторіїв
- **👥 Співпраця**: Пошук користувачів і організацій, управління командами та контроль доступу

**Приклади використання в реальному житті**: "Створити pull request з моєї feature-гілки", "Показати всі помилки CI за цей тиждень", "Відобразити всі відкриті сповіщення безпеки для моїх репозиторіїв", або "Знайти всі issues, призначені мені в межах моїх організацій"

**Повний демонстраційний сценарій**: Ось потужний робочий процес для демонстрації можливостей GitHub MCP Server:

> "Мені потрібно підготуватись до спринт-рев’ю. Покажи всі pull request, які я створював цього тижня, перевір статус наших CI/CD pipeline, створи зведення несправностей безпеки, які треба опрацювати, і допоможи написати нотатки до релізу на основі злитих PR з міткою ‘feature’."

GitHub MCP Server:
- Запитає ваші останні pull request із докладною інформацією про статус
- Проаналізує виконання workflow та виділить будь-які помилки чи проблеми з продуктивністю
- Скомпілює результати сканування безпеки та виокремить критичні сповіщення
- Згенерує вичерпні нотатки до релізу, витягуючи інформацію зі злитих PR
- Надасть конкретні кроки для планування спринту і підготовки релізу

**Приклад із життя**: Я люблю використовувати це для робочих процесів рев’ю коду. Замість перемикання між VS Code, сповіщеннями GitHub і сторінками pull request, я можу сказати: "Покажи мені всі PR, які очікують на моє рев’ю", а потім: "Додай коментар до PR #123, питання щодо обробки помилок у методі автентифікації." Сервер опрацьовує виклики GitHub API, зберігає контекст розмови і навіть допомагає мені складати більш конструктивні коментарі.

**Параметри автентифікації**: Сервер підтримує як OAuth (безшовну авторизацію у VS Code), так і Personal Access Tokens, з можливістю налаштування лише потрібних функцій GitHub. Його можна запускати як віддалену хостингову службу для швидкого налаштування або локально через Docker для повного контролю.

### 5. 📝 MarkItDown MCP Server
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_MarkItDown_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_MarkItDown_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/markitdown)

**Що це робить**: MarkItDown — це комплексний сервер конвертації документів, який перетворює різні формати файлів у високоякісний Markdown, оптимізований для споживання LLM та робочих процесів аналізу тексту.

**Чому це корисно**: Незамінний інструмент для сучасних робочих процесів документації! MarkItDown підтримує вражаючий спектр форматів файлів, зберігаючи критичну структуру документа, таку як заголовки, списки, таблиці та посилання. На відміну від простих інструментів вилучення тексту, він зосереджується на підтримці семантичного змісту та форматування, які цінні як для обробки штучним інтелектом, так і для зручності читання людиною.

**Підтримувані формати файлів**:
- **Офісні документи**: PDF, PowerPoint (PPTX), Word (DOCX), Excel (XLSX/XLS)
- **Медіафайли**: Зображення (з EXIF-метаданими та OCR), Аудіо (з EXIF-метаданими та транскрипцією мови)
- **Веб-контент**: HTML, RSS-стрічки, URL-адреси YouTube, сторінки Wikipedia
- **Формати даних**: CSV, JSON, XML, ZIP-файли (рекурсивна обробка вмісту)
- **Публікаційні формати**: EPub, блокноти Jupyter (.ipynb)
- **Електронна пошта**: Повідомлення Outlook (.msg)
- **Розширені**: інтеграція Azure Document Intelligence для покращеної обробки PDF

**Розширені можливості**: MarkItDown підтримує опис зображень за допомогою LLM (за наявності клієнта OpenAI), Azure Document Intelligence для покращеної обробки PDF, транскрипцію аудіо для мовного контенту та систему плагінів для розширення підтримки додаткових форматів файлів.

**Використання в реальному житті**: «Перетворити цю презентацію PowerPoint у Markdown для нашого сайту документації», «Вилучити текст з цього PDF із правильною структурою заголовків» або «Перетворити цю таблицю Excel у зручний для читання формат таблиці».

**Приклад із фокусом**: Цитуючи [документацію MarkItDown](https://github.com/microsoft/markitdown#why-markdown):

> Markdown надзвичайно близький до звичайного тексту, з мінімальним розміткою або форматуванням, але все ж забезпечує спосіб подання важливої структури документів. Головні LLM, такі як GPT-4o від OpenAI, рідною мовою "говорять" Markdown і часто включають Markdown у свої відповіді без запиту. Це свідчить про те, що вони були навчені на величезних обсягах текстів у форматі Markdown і добре його розуміють. Додатково, конвенції Markdown є дуже економними щодо кількості токенів.

MarkItDown дуже добре зберігає структуру документів, що важливо для робочих процесів ШІ. Наприклад, при конвертації презентації PowerPoint він зберігає організацію слайдів із правильними заголовками, вилучає таблиці у форматі Markdown, додає альтернативний текст для зображень і навіть обробляє примітки доповідача. Діаграми перетворюються у читаємі таблиці даних, і отриманий Markdown підтримує логічний порядок оригінальної презентації. Це робить його ідеальним для подавання презентаційного контенту в системи ШІ або створення документації з існуючих слайдів.

Це надзвичайно потужно, бо поєднує інструкції природною мовою з виконуваним тестовим кодом. Традиційні підходи потребують або ручного написання тестів, або доступу до коду для контексту. Але з Playwright MCP можна тестувати зовнішні сайти, клієнтські додатки або працювати в сценаріях чорної скриньки, де доступ до коду недоступний.

Цей експериментальний сервер створює міст між розробкою AI та потужною інфраструктурою Azure AI, спрощуючи створення, розгортання та оцінювання AI-застосунків.

Замість перемикання між порталом Azure, документацією та IDE, ви можете знаходити моделі, розгортати сервіси, управляти базами знань та оцінювати продуктивність AI за допомогою команд природною мовою. Особливо корисний для розробників, які створюють застосунки RAG (Retrieval-Augmented Generation), керують розгортанням мульти-модельних систем або реалізують комплексні конвеєри оцінки AI.

**Ключові можливості для розробників**:
- **🔍 Виявлення та розгортання моделей**: Досліджуйте каталог моделей Microsoft Foundry, отримуйте детальну інформацію про моделі з прикладами коду та розгортайте моделі в Azure AI Services
- **📚 Управління знаннями**: Створюйте та керуйте індексами Azure AI Search, додавайте документи, налаштовуйте індексатори та створюйте складні RAG-системи
- **⚡ Інтеграція AI агентів**: Підключайтеся до Azure AI Agents, запитуйте існуючих агентів та оцінюйте продуктивність агентів у продуктивних сценаріях
- **📊 Фреймворк оцінки**: Виконуйте комплексну оцінку тексту та агентів, генеруйте звіти у форматі markdown і реалізуйте контроль якості AI-застосунків
- **🚀 Інструменти для прототипування**: Отримуйте інструкції з налаштування, базовані на GitHub, та доступ до Microsoft Foundry Labs для передових дослідницьких моделей

**Приклад реального використання**: «Розгорнути модель Phi-4 в Azure AI Services для мого застосунку», «Створити новий індекс пошуку для системи RAG на основі моєї документації», «Оцінити відповіді мого агента за метриками якості» або «Знайти найкращу модель для логічного мислення для моїх складних аналітичних задач»

**Повний сценарій демонстрації**: Ось потужний робочий процес розробки AI:

Сервер Microsoft Foundry MCP зробить:
- Запити до каталогу моделей для рекомендації оптимальних моделей логічного мислення на основі ваших вимог
- Надання команд розгортання та інформації про квоти для вашого регіону Azure
- Налаштування індексів Azure AI Search з правильною схемою для вашої документації
- Конфігурацію конвеєрів оцінки з метриками якості та перевірками безпеки
- Генерацію коду для прототипування з автентифікацією GitHub для негайного тестування
- Надання всебічних інструкцій налаштування, адаптованих до вашого технологічного стека

Токени та квоти стресують і складні у керуванні – я ніколи не знаю, чи обираю правильну модель для конкретного завдання, чи не марную бюджет. Можливості виявлення моделей особливо вражають для когось, хто хоче виходити за межі звичних варіантів і знаходити моделі, оптимізовані для конкретних завдань. ## 📋 Висновок

Протокол Model Context (MCP) змінює спосіб взаємодії розробників з AI-помічниками та зовнішніми інструментами. Ці 10 серверів Microsoft MCP демонструють потужність стандартизованої інтеграції AI, забезпечуючи безперебійні робочі процеси, які зберігають концентрацію розробника при доступі до потужних зовнішніх можливостей.

Від всебічної інтеграції в екосистему Azure до спеціалізованих інструментів, як Playwright для автоматизації браузера і MarkItDown для обробки документів, ці сервери показують, як MCP може підвищити продуктивність у різноманітних сценаріях розробки. Стандартизований протокол забезпечує, щоб ці інструменти працювали разом органічно, створюючи узгоджений досвід розробки.

У міру розвитку екосистеми MCP, підтримка зв’язку з спільнотою, вивчення нових серверів і створення власних рішень будуть ключем до максимізації продуктивності розробки. Відкритий стандарт MCP означає, що ви можете поєднувати інструменти різних постачальників, щоб створити ідеальний робочий процес для ваших особливих потреб.
