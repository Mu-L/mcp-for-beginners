# AGENTS.md

## പ്രോജക്ട് അവലോകനം

**MCP for Beginners** എന്നത് മോഡൽ കോൺടെക്സ്റ്റ് പ്രോട്ടോകോൾ (MCP) പഠിക്കാൻ ഉള്ള ഒരു ഓപ്പൺ-സോഴ്‌സ് വിദ്യാഭ്യാസ പാഠ്യപദ്ധതിയാണ് - ഇത് AI മോഡലുകളും ക്ലയന്റ് അപ്ലിക്കേഷനുകളും തമ്മിലുള്ള ഇടപെടലുകൾക്കുള്ള ഒരു സ്റ്റാൻഡേർഡൈസ്ഡ് ഫ്രെയിംവർക്ക് ആണ്. ഈ റിപോസിറ്ററി വിവിധ പ്രോഗ്രാമിങ് ഭാഷകളിൽ ഹാൻഡ്‌സ്-ഓൺ കോഡ് ഉദാഹരണങ്ങളുമായി സമഗ്രമായ പഠന സാമഗ്രികൾ നൽകുന്നു.

### പ്രധാന സാങ്കേതികവിദ്യകൾ

- **പ്രോഗ്രാമിംഗ് ഭാഷകൾ**: C#, ജാവ, ജാവാസ്ക്രിപ്റ്റ്, ടൈപ്പ്എസ്ക്രിപ്റ്റ്, പൈത്തൺ, റസ്റ്റ്
- **ഫ്രെയിംവർക്ക് & SDKs**: 
  - MCP SDK (`@modelcontextprotocol/sdk`)
  - സ്പ്രിംഗ് ബൂട്ട് (ജാവ)
  - ഫാസ്റ്റ്MCP (പൈത്തൺ)
  - ലാങ്‌ചെയിൻ4j (ജാവ)
- **ഡാറ്റാബേസുകൾ**: pgvector എക്സ്റ്റൻഷനോടുകൂടിയ പോസ്റ്റ്‌ഗ്രെഎസ്‌ക്യൂഎൽ
- **ക്ലൗഡ് പ്ലാറ്റ്ഫോമുകൾ**: അസ്യൂർ (കൺറ്റെയ്‌നർ ആപ്പ്‌സ്, ഓപ്പൺഎഐ, കണ്ടന്റ് സെഫ്റ്റി, ആപ്ലിക്കേഷൻ ഇൻസൈറ്റ്‌സ്)
- **ബിൽഡ് ടൂളുകൾ**: npm, മാവൻ, പിപ്, കാർഗോ
- **ഡോക്യുമെന്റേഷൻ**: ഓട്ടോമേറ്റ് ചെയ്യുന്ന മൾട്ടി-ലാംഗ്വേജ് വിവർത്തനത്തോടെ മാർക്‌ഡൗൺ (48+ ഭാഷകൾ)

### ആർക്കിടെക്ചർ

- **11 കോർ മൊഡ്യൂളുകൾ (00-11)**: അടിസ്ഥാനങ്ങളിൽ നിന്നും പരിപാർപ്പിത വിഷയങ്ങളിലേക്കുള്ള പരമ്പരാഗത പഠന പാത
- **ഹാൻഡ്‌സ്-ഓൺ ലാബുകൾ**: പല ഭാഷകളിൽ പൂർത്തിയായ പരിഹാര കോഡോടെയുള്ള പ്രായോഗിക വ്യായാമങ്ങൾ
- **സാമ്പിൾ പ്രോജക്ടുകൾ**: പ്രവർത്തിക്കുന്ന MCP സെർവർ & ക്ലയന്റ് ഇംപ്ലിമെൻറേഷനുകൾ
- **വിവർത്തന സംവിധാനം**: മൾട്ടി-ലാംഗ്വേജ് പിന്തുണയ്ക്കുള്ള ഓട്ടോമേറ്റഡ് GitHub ആക്ഷൻസ് വർക്ക്‌ഫ്ലോ
- **ഇമേജ് അസറ്റ്സ്**: വിവർത്തനമുള്ള പതിപ്പുകളുമായി കേന്ദ്രകൃതമായ ഇമേജ് ഡയറക്ടറി

## സെറ്റപ്പ് കമാൻഡുകൾ

ഇത് ഡോക്യുമെന്റേഷൻ-കേന്ദ്രമായ ഒരു റിപോസിറ്ററിയാണ്. പ്രധാനം സെറ്റപ്പ് ഓരോ സാമ്പിൾ പ്രജക്ടുകളിലും ലാബുകളിലും നടക്കും.

### റിപോസിറ്ററി സെറ്റപ്പ്

```bash
# റിപ്പോസിറ്ററി ക്ലോൺ ചെയ്യുക
git clone https://github.com/microsoft/mcp-for-beginners.git
cd mcp-for-beginners
```

### സാമ്പിൾ പ്രോജക്ടുകളുമായി പ്രവർത്തിക്കൽ

സാമ്പിൾ പ്രോജക്ടുകൾ എന്നവിടെ സ്ഥിതിചെയ്യുന്നു:
- `03-GettingStarted/samples/` - ഭാഷാനുസരിച്ചുള്ള ഉദാഹരണങ്ങൾ
- `03-GettingStarted/01-first-server/solution/` - ആദ്യ സെർവർ ഇംപ്ലിമെൻറേഷനുകൾ
- `03-GettingStarted/02-client/solution/` - ക്ലയന്റ് ഇംപ്ലിമെൻറേഷനുകൾ
- `11-MCPServerHandsOnLabs/` - സമഗ്രമായ ഡാറ്റാബേസ് ഇന്റഗ്രേഷനുള്ള ലാബുകൾ

ഓരോ സാമ്പിൾ പ്രോജക്ടിനും തനിക്ക് സ്വന്തമായ സെറ്റപ്പ് നിർദ്ദേശങ്ങൾ ഉണ്ട്:

#### ടൈപ്പ്എസ്ക്രിപ്റ്റ്/ജാവാസ്ക്രിപ്റ്റ് പ്രോജക്ടുകൾ
```bash
cd <project-directory>
npm install
npm start
```

#### പൈത്തൺ പ്രോജക്ടുകൾ
```bash
cd <project-directory>
pip install -r requirements.txt
# അല്ലെങ്കിൽ
pip install -e .
python main.py
```

#### ജാവ പ്രോജക്ടുകൾ
```bash
cd <project-directory>
mvn clean install
mvn spring-boot:run
```

## ഡെവലപ്പ്മെന്റ് വർക്‌ഫ്ലോ

### ഡോക്യുമെന്റേഷൻ ഘടന

- **മൊഡ്യൂളുകൾ 00-11**: കോർ പാഠ്യപദ്ധതിയുടെ പരമ്പരാഗത ഉള്ളടക്കം
- **translations/**: ഭാഷാനുസരിച്ചുള്ള പതിപ്പുകൾ (ഓട്ടോ-ജനറേറ്റ്, നേരിട്ട് തിരുത്തരുത്)
- **translated_images/**: ലൊക്കലൈസ്ഡ് ഇമേജ് പതിപ്പുകൾ (ഓട്ടോ-ജനറേറ്റ്)
- **images/**: ഉറവിട ഇമേജുകളും ഡയഗ്രാമുകളും

### ഡോക്യുമെന്റേഷനിൽ മാറ്റങ്ങൾ വരുത്തൽ

1.  മൂലമായ മെട്യൂളാംഗളിൽ (00-11) ഉള്ള ആംഗലേയം മാർക്‌ഡൗൺ ഫയലുകൾ മാത്രം തിരുത്തുക
2.  ആവശ്യമായ പക്ഷം `images/` ഫോളഡറിലുള്ള ചിത്രങ്ങൾ അപ്ഡേറ്റ് ചെയ്യുക
3.  co-op-translator GitHub ആക്ഷൻ ഓട്ടോമാറ്റിക്ക് വിവർത്തനങ്ങൾ സൃഷ്ടിക്കും
4.  മെയിൻ ബ്രാഞ്ചിലേക്ക് നൽകുമ്പോൾ വിവർത്തനങ്ങൾ പുനഃ സൃഷ്ടിക്കപ്പെടും

### വിവർത്തനങ്ങളുമായി പ്രവർത്തിക്കൽ

- **ഓട്ടോമേറ്റഡ് വിവർത്തനം**: GitHub ആക്ഷനുകൾ എല്ലാ വിവർത്തനങ്ങളും കൈകാര്യം ചെയ്യും
- **translations/** ഡയറക്ടറിയിലുള്ള ഫയലുകൾ **നേര്‍ക്കണ്ടി** തിരുത്തരുത്
- ഓരോ വിവർത്തന ഫയലിന്റെ ഉള്ളിൽ വിവർത്തന മെറ്റാഡേറ്റ ഇടുന്നു
- പിന്തുണയുള്ള ഭാഷകൾ: അറബിക്, ചൈനീസ്, ഫ്രഞ്ച്, ജർമൻ, ഹിന്ദി, ജാപ്പനീസ്, കൊറിയൻ, പോർച്ചുഗീസ്, റഷ്യൻ, സ്പാനിഷ് തുടങ്ങിയ 48+ ഭാഷകൾ ഉൾപ്പെടെ

## ടെസ്റ്റിങ് നിർദ്ദേശങ്ങൾ

### ഡോക്യുമെന്റേഷൻ പരിശോധന

ഈ റിപോസിറ്ററി പ്രധാനമായും ഡോക്യുമെന്റേഷനാകയാൽ, പരിശോധന താഴെപ്പറയുന്ന കാര്യങ്ങളിലാണ്:

1. **ലിങ്ക് സാധുത പരിശോധന**: എല്ലാ ആന്തരിക ലിങ്കുകളും പ്രവർത്തിക്കുന്നുണ്ടോ എന്ന് ഉറപ്പാക്കുക
```bash
# തകർന്ന മാർക്ക്ഡൗൺ ലിങ്കുകൾ പരിശോധിക്കുക
find . -name "*.md" -type f | xargs grep -n "\[.*\](../../.*)"
```

2. **കോടിംഗ് സൈമ്പിൾ പരിശോധന**: കോഡ് ഉദാഹരണങ്ങൾ കംപൈൽ/റൺ ചെയ്യാമോ പരിശോധിക്കുക
```bash
# പ്രത്യേക സാമ്പിളിലേക്ക് നാവിഗേറ്റ് ചെയ്ത് അതിന്റെ പരിശോധനകൾ നടത്തുക
cd 03-GettingStarted/samples/typescript
npm install && npm test
```

3. **മാർക്‌ഡൗൺ ലിന്റിംഗ്**: ഫോർമാറ്റിംഗ് കൃത്യത പരിശോധിക്കുക
```bash
# ആവശ്യമെങ്കിൽ markdownlint ഉപയോഗിക്കുക
npx markdownlint-cli2 "**/*.md" "#node_modules"
```

### സാമ്പിൾ പ്രോജക്ട് ടെസ്റ്റിങ്

ഓരോ ഭാഷാനുസരിച്ചുള്ള സാമ്പിൾ പ്രോജക്ടിനും തങ്ങളുടെ ടെസ്റ്റിംഗ് രീതി ഉണ്ട്:

#### ടൈപ്പ്എസ്ക്രിപ്റ്റ്/ജാവാസ്ക്രിപ്റ്റ്
```bash
npm test
npm run build
```

#### പൈത്തൺ
```bash
pytest
python -m pytest tests/
```

#### ജാവ
```bash
mvn test
mvn verify
```

## കോഡ് സ്റ്റൈൽ മാർഗനിർദ്ദേശങ്ങൾ
