# മാറ്റങ്ങൾ: MCP ആരംഭക്കാർക്കുള്ള പാഠ്യപദ്ധതി

Model Context Protocol (MCP) ആരംഭക്കാർക്കുള്ള പാഠ്യപദ്ധതിയിലുള്ള എല്ലാ പ്രധാന മാറ്റങ്ങളും രേഖപ്പെടുത്തുന്നതിന് ഈ ഡോക്യുമെന്റ്用. പുതിയ മാറ്റങ്ങൾ ആദ്യം വരുത്തി വിപരീത ക്രമത്തിൽ രേഖപ്പെടുത്തിയിട്ടുണ്ട്.

## ജൂൺ 16, 2026

### MCP സ്പെസിഫിക്കേഷൻ ചേര്ക്കൽ & സാമ്പിൾ സ്ഥിരീകരണം

നിലവിലെ **MCP Specification 2025-11-25**ക്കും ഏറ്റവും പുതിയ ഔദ്യോഗിക SDKകളും അടിസ്ഥാനമാക്കി പാഠ്യപദ്ധതി പരിശോദിച്ച്, പഴയ സ്പെസിഫിക്കേഷൻ റഫറൻസുകൾ ശരിയാക്കി, കോർ സാമ്പിൾസ് ഇനിയും ബിൽഡ് ചെയ്ത് പ്രവർത്തിക്കുന്നുവെന്ന് ഉറപ്പു ചെയ്തു.

#### സ്പെസിഫിക്കേഷൻ വേർഷൻ പരിഷ്കാരങ്ങൾ (2025-06-18 / 2025-03-26 → 2025-11-25)

പഴയ സ്പെസിഫിക്കേഷൻ റിവിഷൻ എന്നാൽ *നിലവിലെ/സമ്മതിച്ച* സ്റ്റാൻഡാർഡ് എന്ന് ഇപ്പോഴും പറയുന്ന ഇംഗ്ലീഷ് ഉള്ളടക്കം പുതുക്കി, canonical `modelcontextprotocol.io` സ്പെസിഫിക്കേഷൻ പാതകളിലേക്ക് ലിങ്കുകൾ തിരുത്തി:
- **05-AdvancedTopics/mcp-security/README.md**: "Current Standard" ബാനർ, പരിചയം, കോർ സെക്യൂരിറ്റി സിദ്ധാന്തങ്ങളുടെ തലവാചകം, നിർബന്ധമായ ആവശ്യകതകൾ, Microsoft Entra ID, റഫറൻസുകളും സംഭരണങ്ങളും ലിങ്കുകൾ, സെക്യൂരിറ്റി നോട്ടീസ് (8 റഫറൻസുകൾ) 2025-11-25ല്‍ പുതുക്കി
- **05-AdvancedTopics/mcp-transport/README.md**: അധിക സംഭരണങ്ങളുടെ സ്പെക് ലിങ്കും "Current Standard" ബാനറും 2025-11-25ല്‍ പുതുക്കി
- **05-AdvancedTopics/mcp-realtimesearch/README.md**: പഴയ `2025-03-26` സെക്യൂരിറ്റി-ആൻഡ്-ട്രസ്റ്റ് ലിങ്ക് ഇപ്പോഴത്തെ 2025-11-25 സെക്യൂരിറ്റി മികച്ച രീതികളുടെ പേജിൽ മാറ്റി
- **03-GettingStarted/14-sampling/README.md**: ഔദ്യോഗിക സാമ്പിൾ ഡോക്‌സ് ലിങ്ക് 2025-11-25ല്‍ പുതുക്കി
- **03-GettingStarted/05-stdio-server/README.md**: "നിലവിലെ MCP സ്പെസിഫിക്കേഷൻ" റഫറൻസ് പ്രസന്റ്-ടെൻസ് വാക്കിൽ ശരിയാക്കി, അധിക സംഭരണങ്ങളുടെ സ്പെക് ലിങ്കും 2025-11-25ല്‍ പുതുക്കി (ചരിത്രപരമായ SSE-ഡിപ്രിക്കേഷൻ കുറിപ്പുകൾ ഒതുക്കാതെ)

#### നിലവിലെ SDK-കളെ അടിസ്ഥാനമാക്കി സാമ്പിൾ സ്ഥിരീകരണം

- **TypeScript (03-GettingStarted/01-first-server/solution/typescript)**: `npm install` വഴി `@modelcontextprotocol/sdk@1.29.0` നിശ്ചയിച്ച്; `tsc --noEmit` യാതൊരു ടൈപ്പ് പിഴവുകളും കൂടാതെ പാസ്സായി — നിലവിലുള്ള `McpServer` / `StdioServerTransport` APIകൾ സാധുവായി നിലനിൽക്കുന്നു
- **Python (03-GettingStarted/01-first-server/solution/python)**: `.venv`ഉള്ളിൽ `mcp[cli]` (1.27.2) ഉപയോഗിച്ച് പരിശോധന നടത്തി; `py_compile` പാസ്സായി, `FastMCP.list_tools()` ശരിയായി `add`യും `subtract`വും ടൂളുകൾ കൊടുത്തു
- എല്ലാ സാമ്പിൾ `@modelcontextprotocol/sdk` വേർഷൻ പരിധികളും (`>=1.26.0` / `^1.26.0` / `^1.27.0`) ഇപ്പൊഴത്തെ `1.29.0` ലേക്ക് പിരിച്ച് മാറ്റാതെ സുസ്ഥിരമായി പരിഹരിച്ചു

#### ഡിപെൻഡൻസി പിൻ അടുക്കൽ (വർഷൻ ഗ്യാപ്‌സ് അടയ്ക്കൽ)

ഏതു സാമ്പിൾ MCP റിലീസിനോട് പൊരുത്തപ്പെടാൻ പഴയ SDK പിനുകൾ അണിഞ്ഞു, റീപ്പോ മുഴുവൻ കൺവെൻഷനുമായി പൊരുത്തപ്പെടുത്തി:
- **03-GettingStarted/05-stdio-server/solution/typescript/package.json**: `@modelcontextprotocol/sdk` പിന് ഒഴിച്ച് `^1.8.0` → `>=1.26.0` എന്നും പഴയ `"updated for MCP 2025-06-18"` പാക്കേജ് വിവരണവും `"aligned with MCP Specification 2025-11-25"` ആയി പുതുക്കി
- **10-StreamliningAIWorkflows.../lab3/code/weather_mcp/pyproject.toml** & **lab4/code/github_mcp_server/pyproject.toml**: നിർദ്ദിഷ്ട `mcp==1.23.0` → `mcp>=1.26.0` ആയി പിന് മാറ്റിയതും, രണ്ട് `uv.lock` ഫയലുകളും പുന:രേസിപ്പിച്ചും (`uv lock`) അതോടെ ലോക്ക്ഫയലുകൾ ഇപ്പോഴത്തെ `mcp 1.27.2` ലേക്ക് മറിച്ചു സങ്കേതപരമായി സമമാണ്

#### പാഠ്യപദ്ധതി ഗ്യാപ് വിശകലനം — പുതിയ സ്പെക് സവിശേഷതകൾ ഉൾപ്പെടൽ

MCP 2025-11-25ൽ അവതരിപ്പിച്ച/വികസിപ്പിച്ച എല്ലാ പ്രാഥമിക ഘടകങ്ങളും പാഠ്യപദ്ധതി എളുപ്പത്തിൽ ഉൾക്കൊള്ളിച്ചിട്ടുള്ളത് ഉറപ്പാക്കിയതിനാൽ ഉള്ളടക്കം ഗ്യാപ് ഇല്ല:
- **Sampling**: പാഠം 03-GettingStarted/14-sampling കൂടാതെ 05-AdvancedTopics/mcp-sampling
- **Elicitation (URL മോഡ് ഉൾപ്പെടെ)**: 01-CoreConcepts, 05-AdvancedTopics/mcp-protocol-features-ൽ രേഖപ്പെടുത്തിയിട്ടുണ്ട്
- **Roots**: 00-Introduction, 01-CoreConcepts, 05-AdvancedTopics/mcp-root-contexts-ൽ രേഖപ്പെടുത്തിയിട്ടുണ്ട്
- **Tasks (പ്രയോഗപരമായ, നീണ്ടകാല പ്രവർത്തനങ്ങൾ)**: 01-CoreConcepts, 05-AdvancedTopics/mcp-protocol-features-ൽ രേഖപ്പെടുത്തിയിട്ടുണ്ട്
- **Tool Annotations** (`readOnlyHint` / `destructiveHint`): 01-CoreConcepts, 05-AdvancedTopics/mcp-protocol-features-ൽ രേഖപ്പെടുത്തിയിട്ടുണ്ട്

### സെക്യൂരിറ്റി ഹാർഡനിംഗ് & ഡിപെൻഡൻസി ഭീഷണി പരിഹാരം

ഓരോ ഡിപെൻഡൻസി മാനിഫെസ്റ്റ്, സാമ്പിൾ സോഴ്സ് കോഡുകൾ എല്ലാം മുഴുവനായി സെക്യൂരിറ്റി പരിശോധന നടത്തി, npm അഡ്‌വൈസറികളും ഒരു കോഡ്-ലവല്‍ കണ്ടെത്തലും പരിഹരിച്ചു. തുടർന്ന്, `npm audit` പ്രവൃത്തിപരിധിയിൽ **0 ഭീഷണികൾ** എന്ന് റിപ്പോർട്ട് ചെയ്തു.

#### npm ഡിപെൻഡൻസി ഭീഷണികൾ (സമാന്തരമായത്) — പരിഹരിച്ചു

അടിസ്ഥാനമായ 15 കൊടുത്ത `package-lock.json` ഫയലുകൾ ഓഡിറ്റ് ചെയ്തു. ഭീഷണികൾ MCP ഇൻസ്പെക്ടർ dev ടൂൾ, OpenAI ക്ലയന്റ്, MCP SDK എന്നിവയുടെ ട്രാൻസിറ്റീവ് ഡിപെൻഡൻസികളിൽ മാത്രം ഉണ്ടായിരുന്നു; എല്ലാം സാംപിൾ ഹാനികരമാകാതെ പരിഹരിച്ചു:
- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/inspector** & **lab3/code/weather_mcp/inspector**: `@modelcontextprotocol/inspector` (`0.16.6` / `0.14.1` → `0.22.0`) അപ്ഗ്രേഡ് ചെയ്തു. ഇതോടെ ബണ്ടിൽ ചെയ്‌ത `ajv`, `brace-expansion`, `diff`, `path-to-regexp` ഉം `ws` ഉം അപ്ഡേറ്റുകൾ തീർന്നു. npm `overrides` ലിസ്റ്റിൽ പാടിച്ച `shell-quote@1.8.4` പാടിച്ച് `concurrently`ല്‍ നിലനിർത്തപ്പെട്ട അവസാന ക്രിറ്റിക്കൽ അഡ്‌വൈസർ നീക്കി; രണ്ട് ലോക്ക്ഫയലുകളും പുനഃസൃഷ്ടിച്ചു (ഇപ്പോൾ 0 ഭീഷണികൾ)
- **03-GettingStarted/samples/typescript**: `npm audit fix` ട്രാൻസിറ്റീവ് `qs` (മിതമായ) അപ്ഡേറ്റ്
- **03-GettingStarted/samples/javascript**: `npm audit fix` ട്രാൻസിറ്റീവ് `hono` (മിതമായ) അപ്ഡേറ്റ്
- **03-GettingStarted/03-llm-client/solution/typescript**: `npm audit fix` ട്രാൻസിറ്റീവ് `form-data` (ഉയർന്ന) അപ്ഡേറ്റ്
- **03-GettingStarted/11-simple-auth/solution/typescript**: നഷ്ടമായ `package-lock.json` സൃഷ്ടിച്ച് പ്രോജക്ട് പുനരുജ്ജീവനം കഴിയും വിധം

#### കോഡ്-ലവല്‍ സെക്യൂരിറ്റി പരിഹാരം (OWASP A03: ഇജക്ഷൻ)

- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/src/server.py**: `open_in_vscode` ടൂളിൽ നിന്ന് `shell=True` നീക്കം ചെയ്തു. മുമ്പത്തെ `subprocess.run(["start", "", vscode_path, folder_path], shell=True)` പ്രകാരമുള്ള ഷെൽ മെടാക്‌ളർ പാത്തിലുള്ള ഫോൾഡർ പാത്തിന് `cmd.exe` വഴി ഇജക്ഷൻ സാധ്യത ഉണ്ടാകുന്നുണ്ടായിരുന്നു. ഇപ്പോൾ `Code.exe` നേരിട്ട് ഫോൾഡർ ആർഗുമെന്റായി ഉപയോഗിച്ച് ലയിക്കുന്നു — ഷെൽ നിർബന്ധമില്ല — ഇത് ഫങ്ഷണലി സമാനവും സുരക്ഷിതവുമാണ്

#### Python ഡിപെൻഡൻസി ഓഡിറ്റ്

#### കോർ ആശയങ്ങളുടെ മെച്ചപ്പെടുത്തൽ (01-CoreConcepts/)
- **പുതിയ ക്ലയന്റ് പ്രിമിറ്റീവ് - Roots**: ഫയൽസിസ്റ്റം അതിരുകൾ, ആക്‌സസ് അധികാരം മനസ്സിലാക്കാൻ സർവർക്ക് സഹായിക്കുന്ന Roots ക്ലയന്റ് പ്രിമിറ്റീവ് സംബന്ധം സമഗ്ര രേഖപ്പെടുത്തൽ ചേർത്തു
- **ടൂൾ അനോട്ടേഷനുകൾ**: ടൂൾ പെരുമാറ്റ അനോട്ടേഷനുകൾ (`readOnlyHint`, `destructiveHint`) സാധുവായ ടൂൾ പ്രവർത്തന തീരുമാനം നൽകാൻ ചേർത്തു
- **സാമ്പിളിംഗ് ടൂളുകൾ വിളിക്കൽ**: സാമ്പിളിംഗ് ഡോക്യുമെന്റേഷനിൽ മോഡൽ ഡ്രിവൻ ടൂൾ വിളിക്കുവാനുള്ള `tools` ഉം `toolChoice` ഉം ഉൾപ്പെടുത്തിയത് പുതുക്കി
- **URL മോഡ് പ്രേരണം**: സർവർ ആരംഭിക്കുന്ന ബാഹ്യ വെബ് ഇടപെടലിനുള്ള URL അടിസ്ഥാന elicitation ഡോക്യുമെന്റേഷൻ ചേർത്തു
- **ടാസ്കുകൾ (പരീക്ഷണവസ്‌തു)**: ദൃഢമായ നടപ്പാക്കൽ വ്രാപ്പറുകളായും അപലബ്ധ ഫലം സ്വീകരണത്തിനും പുതിയ ടാസ്ക് ഫോം ഫീച്ചർ രേഖപ്പെടുത്തി
- **ഐക്കൺ പിന്തുണ**: ഇപ്പോൾ ടൂളുകൾ, വിഭവങ്ങൾ, വിഭവ ടെംപ്ലേറ്റുകൾ, പ്രോമ്പ്റ്റുകൾ ഐക്കൺ들을 ചേർക്കാൻ കഴിയുന്നതായി കുറിപ്പിടുന്നു

- **study_guide.md**: ടാസ്കുകളും ടൂൾ അനോട്ടേഷനുകളും കോർ ആശയങ്ങളായി ഉൾപ്പെടുത്തി; ഡോക്യുമെന്റ് ടൈംസ്റ്റാമ്പ് പുതുക്കി

#### സ്പെസിഫിക്കേഷൻ അനുരൂപത സ്ഥിരീകരണം
- **പ്രോട്ടോക്കോൾ പതിപ്പ്**: എല്ലാ ഡോക്യുമെന്റേഷൻ MCP സ്പെസിഫിക്കേഷൻ 2025-11-25 നിലവിലുള്ളതായി പരിശോധിച്ചു
- **ആർക്കിടെക്ചർ പൊരുത്തം**: രണ്ട് ലെയർ ആർക്കിടെക്ചർ (ഡേറ്റ ലെയർ + ട്രാൻസ്പോർട്ട് ലെയർ) ഡോക്യമെന്റ് ശരിയെന്ന് സ്ഥിരീകരിച്ചു
- **പ്രിമിറ്റീവ് ഡോക്യുമെന്റേഷനുകൾ**: സർവർ പ്രിമിറ്റീവുകൾ (Resources, Prompts, Tools) കൾയും ക്ലൈന്റ് പ്രിമിറ്റീവുകൾ (Sampling, Elicitation, Logging, Roots) എന്നിവ പരിശോധിച്ചു
- **ട്രാൻസ്പോർട്ട് സംവിധാനം**: STDIO, Streamable HTTP ട്രാൻസ്പോർട്ട് ഡോക്യുമെന്റേഷൻ ശരിയെന്ന് ഉറപ്പിച്ചു
- **സുരക്ഷാ മാർഗ്ഗ നിർദ്ദേശങ്ങൾ**: MCP സുരക്ഷ ഏറ്റവും പുത്തൻ മാർഗ്ഗ നിർദ്ദേശത്തിനോട് പൊരുത്തപ്പെടുന്നതിനായി സ്ഥിരീകരിച്ചു

  - **ഹാൻഡ്ലർ അധിഷ്ഠിത ഡിസൈൻ**: സ്കേലബിളും ഫ്ലെക്സിബിളുമായ ടൂൾ/വിഭവ/പ്രോമ്പ്റ്റ് മാനേജ്മെൻറ് വിശദീകരണം
  - **പ്രായോഗിക മാതൃകകൾ**: ഉയർന്ന നൽകുന്ന പ്രത്യേകതകളും ആർക്കിടെക്ചർക്കും അനുയോജ്യമായ താഴ്ന്ന നില സർവർ മാതൃകകൾ അനുഭവ കാഴ്ചപ്പാട്
#### ലളിതമായ അംഗീകാരം (03-GettingStarted/11-simple-auth)
- **പുതിയ അധ്യായം ചേർത്തു**: MCP സർവറുകളിൽ ലളിതമായ അംഗീകാരം നടപ്പിലാക്കുന്നതിനുള്ള ഘട്ടംഘട്ടമായ ഗൈഡ്.
  - **അംഗീകാരം ആശയങ്ങൾ**: അംഗീകാരം vs. മുഴുവൻ അനുമതി വ്യത്യാസവും ക്രെഡൻഷ്യൽ കൈകാര്യം വ്യക്തമാക്കി.
  - **മൂലഭൂത അംഗീകാരം നടപ്പാക്കൽ**: Python (Starlette)യും TypeScript (Express)ഉം അടിസ്ഥാനമാക്കിയ മിഡിൽവെയർ ആയി അംഗീകാരം മാതൃകകൾ, കോഡ് സാമ്പിളുകൾ सहित.

ഇവ ചേർക്കലുകൾ ശക്തമായ, സുരക്ഷിതമായ, ലളിതമായ MCP സർവർ നടപ്പാക്കലുകൾ നിർമ്മിക്കാൻ പ്രായോഗികമായ കൈകാര്യം ചെയ്യുന്ന മാർഗദർശനം നൽകുന്നു, അടിസ്ഥാന ആശയങ്ങൾ ഉന്നത നിലവാരമുള്ള പ്രോഡക്ഷൻ മാതൃകകളുമായി കണക്ട് ചെയ്യുന്നു.

    - സ്കെയിലബിൾ MCP പരിഹാരങ്ങൾക്ക് ഇവന്റ്-ഡ്രിവൻ ആർക്കിടെക്ചർ മാതൃകകൾ
    - വെബ്‌ഹുക്ക് റിസീവേഴ്സും പുഷ്-പ്യാസഡ് മെസ്സേജ് കൈകാര്യം ചെയ്യൽ
  - **Azure ഇവന്റ് ഹബ്സ് ട്രാൻസ്‌പോർട്ട്**: ഉയർന്ന സ്രാവൽ തൊഴിലൻട്രണം സ്ട്രീമിംഗ് ട്രാൻസ്പോർട്ട്
    - കുറഞ്ഞ വൈകിപ്പാട് മാർഗ്ഗങ്ങളിലുള്ള റിയൽ-ടൈം സ്ട്രീമിംഗ് കഴിവ്

    - Azure Functions അടിസ്ഥാനമാക്കിയ MCP പ്രോസസ്സിംഗ് വിതരണ ശൃംഖല
    - ഒരുമിച്ച് മിക്കവാറും ട്രാൻസ്പോർട്ട് തരം മിശ്രിതമായ ഹൈബ്രിഡ് ട്രാൻസ്പോർട്ട് ആർക്കിടെക്ചറുകൾ
    - മെസ്സേജ് ദൃഢത, വിശ്വാസ്യത, തെറ്റു കൈകാര്യം തന്ത്രങ്ങൾ
  - **സുരക്ഷ & നിരീക്ഷണം**: Azure കീ വോൾട്ട് സംയോജനം, ദൃശ്യമാക്കൽ മാതൃകകൾ

    - അപ്ലിക്കേഷൻ ഇൻസൈറ്റ്സ് ടെലിമെട്രി, പ്രകടന നിരീക്ഷണം
    - സർക്യൂട്ട് ബ്രേക്കേഴ്സ്, ദോഷങ്ങൾ തളയ്ക്കൽ മാതൃകകൾ
  - **ടെസ്റ്റിങ് ഫ്രെയിംവർക്കുകൾ**: കസ്റ്റം ട്രാൻസ്പോർട്ടുകൾക്കുള്ള സമഗ്ര ടെസ്റ്റിംഗ് തന്ത്രങ്ങൾ
    - ടെസ്റ്റ് ഡബിൾസും മോകിംഗ് ഫ്രെയിംവർക്കുകളും ഉപയോഗിച്ച യൂണിറ്റ് ടെസ്റ്റിംഗ്
    - Azure ടെസ്റ്റ് കണ്ടെയ്‌നറുകൾ ഉപയോഗിച്ച ഇന്റഗ്രേഷൻ ടെസ്റ്റിംഗ്
    - പ്രകടനവും ലോഡ് ടെസ്റ്റിങ്ങും പരിഗണനകൾ

    - ലെയർഡ് കോൺടക്സ്‌റ്റ് സമീപനങ്ങൾ, പിൻവലിക്കൽ ഓപ്റ്റിമൈസേഷൻ

#### കോർ ആശയങ്ങൾ (01-CoreConcepts/) - സമ്പൂർണ നവീകരണം
- **പ്രോട്ടോക്കോൾ പതിപ്പ് അപ്‌ഡേറ്റ്**: നിലവിലെ MCP സ്പെസിഫിക്കേഷൻ 2025-06-18ന് തിയതി അടിസ്ഥാനത്തിലുള്ള പതിപ്പിന് (YYYY-MM-DD ഫോർമാറ്റ്) അപ്ഡേറ്റ് ചെയ്തു
- **ആർക്കിടെക്ചർ പുനർനിർമ്മാണം**: ഹോസ്റ്റുകൾ, ക്ലൈന്റുകൾ, സെർവറുകൾ എന്നിവയുടെ വിവരണങ്ങൾ മെച്ചപ്പെടുത്തി നിലവിലെ MCP ആർക്കിടെക്ചർ പാറ്റേണുകൾ പ്രതിഫലിപ്പിക്കാൻ
  - ഹോസ്റ്റുകൾ ഒരേ സമയം പല MCP ക്ലൈന്റ് കണക്ഷനുകൾ ഏകീകരിക്കുന്ന AI അപ്ലിക്കേഷനുകളായി വ്യക്തമായി നിർവചിച്ചു
  - ക്ലൈന്റുകൾ പ്രോട്ടോക്കോൾ കണക്റ്ററുകൾ ആയി ഒരൊറ്റ സെർവറുമായി ബന്ധം പരിപാലിക്കുന്നു എന്ന് വിവരിച്ചു
  - സെർവറുകൾ ലൊക്കൽ ಮತ್ತು റിമോട്ട് വിന്യാസ സന്നിവേശങ്ങളോടൊപ്പം മെച്ചപ്പെടുത്തി
- **പ്രിമിറ്റീവ് പുനരുലേഖനം**: സെർവർ, ക്ലൈന്റ് പ്രിമിറ്റീവുകളുടെ സമഗ്ര പുനർനിർമ്മാണം
  - സെർവർ പ്രിമിറ്റീവുകൾ: റിസോഴ്‌സുകൾ (ഡാറ്റ സോഴ്‌സുകൾ), പ്രോമ്പ്റ്റുകൾ (ടെംപ്ലേറ്റുകൾ), ടൂൾസ് (പ്രവർത്തനക്ഷമമായ ഫംഗ്ഷനുകൾ) വിശദീകരണങ്ങളോടെയും ഉദാഹരണങ്ങളോടെയും
  - ക്ലൈന്റ് പ്രിമിറ്റീവുകൾ: സാമ്പ്ലിംഗ് (LLM പൂർത്തീകരണങ്ങൾ), എലിസിറ്റേഷൻ (ഉപയോക്തൃ ഇൻപുട്ട്), ലോഗിംഗ് (ഡീബഗ്ഗിംഗ്/മോണിറ്ററിംഗ്)
  - നിലവിലെ ഡിസ്കവർറ് (`*/list`), റിട്രീവൽ (`*/get`), എക്സിക്യൂഷൻ (`*/call`) കണ്ടെത്തൽ പാറ്റേണ்களുമായി അപ്ഡേറ്റ് ചെയ്തു
- **പ്രോട്ടോക്കോൾ ആർക്കിടെക്ചർ**: രണ്ടാം-ലേയർ ആർക്കിടെക്ചർ മോഡൽ അവതരിപ്പിച്ചു
  - ഡാറ്റ ലെയർ: JSON-RPC 2.0 അടിസ്ഥാനത്തിലുള്ള ലൈഫ്സൈക്കിൾ മാനേജുമെന്റ്, പ്രിമിറ്റീവുകൾ
  - ട്രാൻസ്പോർട്ട് ലെയർ: STDIO (ലൊക്കൽ) және Streamable HTTP ഒപ്പം SSE (റിമോട്ട്) ട്രാൻസ്പോർട്ട് മെക്കാനിസങ്ങൾ
- **സെക്യൂരിറ്റി ഫ്രെയിംവർക്ക്**: വ്യക്തമായ ഉപയോക്തൃ സമ്മതം, ഡാറ്റ പ്രൈവസി സംരക്ഷണം, ടൂൾ പ്രവർത്തനസുരക്ഷ, ട്രാൻസ്പോർട്ട് ലെയർ സുരക്ഷ തുടങ്ങിയ സമഗ്ര സുരക്ഷാ സിദ്ധാന്തങ്ങൾ
- **സംവേദന പാറ്റേണുകൾ**: പ്രോട്ടോക്കോൾ സന്ദേശങ്ങളുടെ ഇൻഷിയലൈസേഷൻ, ഡിസ്കവർ, എക്സിക്യൂഷൻ, നോട്ടിഫിക്കേഷൻ ഫ്ലോകൾ അപ്‌ഡേറ്റ് ചെയ്തു
- **കോഡ് ഉദാഹരണങ്ങൾ**: ബഹു-ഭാഷാ ഉദാഹരണങ്ങൾ (.NET, ജാവ, പൈത്തൺ, ജാവാസ്ക്രിപ്റ്റ്) ഇപ്പോഴത്തെ MCP SDK പാറ്റേണുകൾ പ്രതിഫലിപ്പിക്കുന്ന തരത്തിൽ പുതുക്കി

- **റിസോഴ്‌സ് സംഘാടനം**: തരം അനുസരിച്ച് (ഔദ്യോഗിക ഡോക്സ്, സ്റ്റാൻഡേർഡുകൾ, റിസർച്ച്, Microsoft പരിഹാരങ്ങൾ, നടപ്പാക്കൽ മാർഗ്ഗനിർദ്ദേശങ്ങൾ) സമഗ്ര റിസോഴ്‌സ് ലിങ്കുകൾ പിരിച്ചുവിട്ടു

### ഡോക്യുമെന്റേഷൻ ഗുണനിലവാര മെച്ചപ്പെടുത്തലുകൾ

- **ക്രോസ്-റഫറൻസുകൾ**: ബന്ധപ്പെട്ട സെക്യൂരിറ്റി, കോർ ആശയ വിഷയം തമ്മിൽ ലിങ്കുകൾ ചേർത്തു
- **നിലവിലെ വിവരങ്ങൾ**: എല്ലാ തീയതി റഫറൻസുകളും സ്പെസിഫിക്കേഷൻ ലിങ്കുകളും നിലവിലെ സ്റ്റാൻഡേർഡുകൾ പ്രകാരം അപ്ഡേറ്റ് ചെയ്തു
