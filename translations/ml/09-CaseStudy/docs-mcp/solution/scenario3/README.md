# സീനാരിയോ 3: VS കോഡിലെ MCP സെർവർ ഉപയോഗിച്ച് ഇൻ-എഡിറ്റർ ഡോക്സ്

## അവലോകനം

ഈ സീനാരിയോയയിൽ, MCP സെർവർ ഉപയോഗിച്ച് മൈക്രോസോഫ്റ്റ് ലേൺ ഡോക്സ് നേരിട്ട് നിങ്ങളുടെ Visual Studio Code പരിസ്ഥിതിയിലേക്ക് എങ്ങനെ കൊണ്ടുവരാമെന്നു നിങ്ങൾക്ക് പഠിക്കാം. ഡോക്യുമെന്റേഷൻ തിരയാൻ സ്ഥിരമായി ബ്രൗസർ ടാബുകൾ മാറേണ്ടതിന്റെ പകരം, നിങ്ങളുടെ എഡിറ്ററിനകത്ത് ഔദ്യോഗിക ഡോക്സ് ആക്‌സസ് ചെയ്യുകയും തിരയുകയും റഫറൻസ് ചെയ്യുകയും ചെയ്യാനാകും. ഈ സമീപനം നിങ്ങളുടെ പ്രവൃത്തി ശ്രുതി കൃത്യവും GitHub Copilot പോലുള്ള ഉപകരണങ്ങളോടൊത്ത് സാന്നിഹിതവും ആക്കുന്നു.

- നിങ്ങളുടെ കോഡിംഗ് പരിസ്ഥിതി വിട്ട് പോകാതെയാണ് VS കോഡിനകത്ത് ഡോക്സ് തിരയുകയും വായിക്കുകയും ചെയ്യുക.
- README അല്ലെങ്കിൽ കോഴ്സ് ഫയൽസിൽ നേരിട്ട് ഡോക്യുമെന്റേഷൻ റഫറൻസ് ഉൾപ്പെടുത്തുക.
- GitHub Copilot, MCP എന്നിവ ഒരുമിച്ച് ഉപയോഗിച്ച് എളുപ്പമുള്ള, AI-ശക്തിയുള്ള ഡോക്യുമെന്റേഷൻ പ്രവൃത്തി പ്രവാഹം സജ്ജമാക്കുക.

## പഠനലക്ഷ്യങ്ങൾ

ഈ അധ്യായം അവസാനിക്കുന്നത് നിങ്ങളുടെ ഡോക്യുമെന്റേഷൻ വികസന പ്രവൃത്തി പ്രവാഹം മെച്ചപ്പെടുത്താൻ MCP സെർവർ VS കോഡിനകത്ത് എങ്ങനെ ക്രമീകരിച്ച് ഉപയോഗിക്കാമെന്ന് നിങ്ങൾക്ക് മനസിലാകും. നിങ്ങൾക്ക് ഇങ്ങനെ ചെയ്യാൻ കഴിയും:

- ഡോക്യുമെന്റേഷൻ ശേഖരണത്തിനായി MCP സെർവർ ഉപയോഗിച്ച് നിങ്ങളുടെ വർക്ക്സ്പെയ്സ് ക്രമീകരിക്കുക.
- VS കോഡിനകത്ത് നിന്നു തന്നെ ഡോക്യുമെന്റേഷൻ തിരയുകയും ഉളളപ്പെടുത്തുകയും ചെയ്യുക.
- കൂടുതൽ ഉത്പാദകത്വമുള്ള, AI-സംയോജിത പ്രവൃത്തി പ്രവാഹത്തിനായി GitHub Copilot, MCP ഒന്നിച്ചുചേർത്തുകൊണ്ട് പ്രവർത്തിക്കുക.

ഇവയും നിങ്ങൾക്ക് ശ്രദ്ധ കേന്ദ്രീകരിക്കാൻ, ഡോക്യുമെന്റേഷൻ ഗുണനിലവാരം മെച്ചപ്പെടുത്താൻ, ഡെവലപ്പർ അല്ലെങ്കിൽ സാങ്കേതിക എഴുത്തുകാരനായി നിങ്ങളുടെ ഉത്പാദകത്വം വർധിപ്പിക്കാൻ സഹായിക്കും.

## പരിഹാരം

ഇൻ-എഡിറ്റർ ഡോക്യുമെന്റേഷൻ ആക്‌സസ് നേടാൻ, MCP സെർവർ VS കോഡ്, GitHub Copilot എന്നിവയുമായി സംയോജിപ്പിക്കുന്ന postupiv series പിന്തുടരുക. കോഴ്സ് എഴുത്തുകാരും, ഡോക്യുമെന്റേഷൻ എഴുത്തുകാരും, ഡെവലപ്പർമാരും ഇവിടെയാണ് ലക്ഷ്യം — അവർ റിപ്പോർട്ട് ചെയ്യുമ്പോഴും ഡോക്സ്, Copilot എന്നിവയുമായി പ്രവർത്തിക്കുമ്പോഴും എഡിറ്ററിൽ തന്നെ ശ്രദ്ധ കേന്ദ്രീകരിക്കാം.

- കോഴ്സ് അല്ലെങ്കിൽ പ്രോജക്ട് ഡോക്യുമെന്റേഷൻ എഴുതുമ്പോൾ README-യിലേക്ക് എളുപ്പത്തിൽ റഫറൻസ് ലിങ്കുകൾ ചേർക്കുക.
- കോഡ് ജനറേറ്റ് ചെയ്യാൻ Copilot ഉപയോഗിച്ച് ഡോക്യുമെന്റേഷൻ അതിവേഗം കണ്ടെത്താനും MCP ഉപയോഗിക്കുക.
- എഡിറ്ററിൽ ശ്രദ്ധ കേന്ദ്രീകരിക്കുകയും ഉത്പാദകത്വം വർധിപ്പിക്കുകയും ചെയ്യുക.

### ഘട്ടം ഘട്ടമായ ഗൈഡ്

ആരംഭിക്കാൻ, ഈ ഘട്ടങ്ങൾ പാലിക്കുക. ഓരോ ഘട്ടത്തിനും പ്രക്രിയ ദൃശ്യവൽക്കരിക്കാൻ ആസറ്റ് ഫോൾഡറിൽ നിന്നും സ്ക്രീൻഷോട്ട് ചേർക്കാം.

1. **MCP കോൺഫിഗറേഷൻ ചേർക്കുക:**
   നിങ്ങളുടെ പ്രോജക്ട് റൂട്ടിൽ `.vscode/mcp.json` ഫയൽ സൃഷ്ടിച്ച് താഴെയുള്ള കോൺഫിഗറേഷൻ ചേർക്കുക:
   ```json
   {
     "servers": {
       "LearnDocsMCP": {
         "url": "https://learn.microsoft.com/api/mcp"
       }
     }
   }
   ```
   ഈ കോൺഫിഗറേഷൻ VS Code-ന് [`Microsoft Learn Docs MCP server`](https://github.com/MicrosoftDocs/mcp) ഇനി എങ്ങനെ കണക്ട് ചെയ്യണമെന്നു പറയുന്നതാണ്.
   
   ![Step 1: Add mcp.json to .vscode folder](../../../../../../translated_images/ml/step1-mcp-json.c06a007fccc3edfa.webp)
    
2. **GitHub Copilot ചാറ്റ് പാനൽ തുറക്കുക:**
   GitHub Copilot എക്സ്റ്റൻഷൻ ഇൻസ്റ്റാൾ ചെയ്തിട്ടില്ലെങ്കിൽ, VS കോഡിൽ എക്സ്റ്റൻസ് വിഖ്യാനം വഴി ഇൻസ്റ്റാൾ ചെയ്യുക. നിങ്ങള്‍ നിശ്ചയമായി [Visual Studio Code Marketplace](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot-chat) വഴി നേരിട്ട് ഡൗൺലോഡ് ചെയ്യാം. പിന്നെ സൈഡ്‌ബാറിൽ നിന്ന് Copilot Chat പാനൽ തുറക്കുക.

   ![Step 2: Open Copilot Chat panel](../../../../../../translated_images/ml/step2-copilot-panel.f1cc86e9b9b8cd1a.webp)

3. **ഏജന്റ് മോഡ് കൈമാറാൻ, ഉപകരണങ്ങൾ പരിശോധിക്കുക:**
   Copilot Chat പാനലിൽ ഏജന്റ് മോഡ് സജ്ജീകരിക്കുക.

   ![Step 3: Enable agent mode and verify tools](../../../../../../translated_images/ml/step3-agent-mode.cdc32520fd7dd1d1.webp)

   ഏജന്റ് മോഡ് സജീവമാക്കിയതിനു ശേഷം MCP സെർവർ ലഭ്യമായ ഉപകരണങ്ങളിൽ ഒന്നാണെന്ന് സ്ഥിരീകരിക്കുക. ഇത് Copilot ഏജന്റ് ഡോക്യമെന്റേഷൻ സെർവർ പ്രാപിച്ച് ബന്ധപ്പെട്ട വിവരങ്ങൾ നേടാൻ സഹായിക്കും.
   
   ![Step 3: Verify MCP server tool](../../../../../../translated_images/ml/step3-verify-mcp-tool.76096a6329cbfecd.webp)
4. **പുതിയ ചാറ്റ് ആരംഭിച്ച് ഏജന്റിനെ പ്രോംപ്റ്റ് ചെയ്യുക:**
   Copilot Chat പാനലിൽ പുതിയ ചാറ്റ് തുറക്കുക. ഇപ്പോൾ നിങ്ങളുടെ ഡോക്യമെന്റേഷൻ ചോദനകൾ ഏജന്റിന് നൽകാം. MCP സെർവർ ഉപയോഗിച്ച് അതിന്റെ അനുയോജ്യമായ മൈക്രോസോഫ്റ്റ് ലേൺ ഡോക്സുകൾ നേരിട്ട് എഡിറ്ററിൽ പ്രദർശിപ്പിക്കും.

   - *"ഞാൻ X വിഷയത്തിനായുള്ള പഠന പദ്ധതി രചിക്കാൻ ശ്രമിക്കുന്നു. 8 ആഴ്ചകളായി പഠിക്കാനാണ്. ഓരോ ആഴ്ചയ്ക്കും എടുക്കേണ്ട ഉള്ളടക്കം നിർദ്ദേശിക്കൂ."*

   ![Step 4: Prompt the agent in chat](../../../../../../translated_images/ml/step4-prompt-chat.12187bb001605efc.webp)

5. **ലൈവ് ക്വറി:**

   > Microsoft Foundry Discord ലെ [#get-help](https://discord.gg/D6cRhjHWSC) വിഭാഗത്തിൽ നിന്നുള്ള ലൈവ് ക്വറി (ഒറിജിനൽ സന്ദേശം കാണാൻ [ഇവിടെ ക്ലിക്ക് ചെയ്യുക](https://discord.com/channels/1113626258182504448/1385498306720829572)):
   
   *"ഞാൻ Azure AI Foundry-ല്‍ വികസിപ്പിച്ച മൾട്ടി-ഏജന്റ് സോള്യൂഷൻ എങ്ങനെ ഡിപ്ലോയ് ചെയ്യാമെന്ന് ഉത്തരം തേടുകയാണ്. Copilot Studio ചാനലുകൾ പോലുള്ള നേരിട്ടുള്ള ഡിപ്ലോയ്‌മെന്റ് രീതി ലഭ്യമല്ല. എന്റർപ്രൈസ് ഉപഭോക്താക്കൾക്ക് പരസ്പരം പ്രതികരിച്ചും ജോലി പൂർത്തിയാക്കുകയും ചെയ്യാൻ ഈ ഡിപ്ലോയ്‌മെന്റ് വിവിധ രീതികൾ എത്രവെറും? MS Teams ഉം Azure AI Foundry ഏജന്റുമാരും തമ്മിൽ അധാരമായി പ്രവർത്തിക്കുന്ന Azure Bot സേവനം ഉപയോഗിക്കാമെന്ന പല ലേഖനങ്ങളും ബ്ലോഗുകളും ഉണ്ട്, എന്നാൽ ഇത് ശരിയായ മാർഗമാണോ? Orchestrator ഏജന്റുമായി Azure ഫംഗ്ഷൻ വഴി ബന്ധിപ്പിക്കുന്ന Azure ബോട്ട് സെറ്റപ്പും, അല്ലെങ്കിൽ ഓരോ AI ഏജന്റിനും പ്രത്യേകം Azure ഫംഗ്ഷൻ നിർമ്മിക്കേണ്ടതുണ്ടോ? മറ്റേതെങ്കിലും നിർദ്ദേശങ്ങൾ അഭിനന്ദനാർഹമാണ്."*

   ![Step 5: Live queries](../../../../../../translated_images/ml/step5-live-queries.49db3e4a50bea273.webp)

   ഏജന്റ് അനുയോജ്യമായ ഡോക്യമെന്റേഷൻ ലിങ്കുകൾ, സംഗ്രഹങ്ങൾ എന്നിവ നൽകും, അവ നിങ്ങൾക്ക് നേരിട്ട് മാർക്ഡൗൺ ഫയലുകളിലേക്ക് ചേർക്കാം അല്ലെങ്കിൽ കോഡിൽ റഫറന്‍സായി ഉപയോഗിക്കാം.
   
### ഉദാഹരണ ക്വെറികൾ

നിങ്ങൾ ശ്രമിക്കാവുന്ന ഉദാഹരണ ക്വെറിയിലെ ചിലത് ഇവയാണ്. MCP സെർവർ, Copilot ഒരുമിച്ച് പ്രവർത്തിച്ചു സൂക്ഷ്മവും പ്രാസംഗികവുമായ ഡോക്യുമെന്റേഷൻ അതിവേഗം നൽകുന്നത് ഈ ചോദനകൾ കാണിക്കും:

- "Azure Functions ട്രിഗ്ഗറുകൾ എങ്ങനെ ഉപയോഗിക്കാമെന്ന് കാണിക്കുക."
- "Azure Key Vaultന് ഔദ്യോഗിക ഡോക്യുമെന്റേഷനിലേക്ക് ലിങ്ക് ചേർക്കുക."
- "Azure വనരസാധനങ്ങൾ സുരക്ഷിതമാക്കാനുള്ള മികച്ച പരിശീലാനങ്ങൾ എന്തെല്ലാമാണ്?"
- "Azure AI സർവീസുകൾക്ക് വേണ്ടി ഒരു ക്വിക്ക് സ്റ്റാർട്ട് കണ്ടെത്തുക."

ഈ ക്വെറികൾ MCP സെർവർ, Copilot എന്നിവർ ഒരുമിച്ച് സംയോജിപ്പിച്ച് VS കോഡ് വിട്ടുപോകാതെ തന്നെ ശരിയായ, സാന്ദർശ്യപരമായ ഡോക്യുമെന്റേഷൻ നൽകുന്നതിന് വ്യക്തമായി ഉദാഹരണം നൽകും.
