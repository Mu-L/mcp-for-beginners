# 🚀 ഡെവലപ്പർ ഉത്പാദകക്ഷമത തിരുത്തി മാറ്റിവെക്കുന്ന 10 മൈക്രോസോഫ്റ്റ് MCP സർവറുകൾ

## 🎯 ഈ ഗൈഡിൽ നിങ്ങൾക്ക് പഠിക്കാനിരിക്കുന്നത്

ഈ പ്രായോഗിക ഗൈഡ് എഐ അസിസ്റ്റന്റുകളുമായി ഡെവലപ്പർമാർ എങ്ങനെ ജോലി ചെയ്യുന്നതിൽ സജീവമായി മാറ്റം ഉണ്ടാക്കുന്ന പത്തു മൈക്രോസോഫ്റ്റ് MCP സർവറുകൾ പ്രദർശിപ്പിക്കുന്നു. MCP സർവറുകൾ *ഏതു* കാര്യങ്ങൾക്ക് കഴിവുള്ളവയെന്ന് വെറും വിശദീകരിക്കുന്നതിന് പകരം, മൈക്രോസോഫ്റ്റിലും അതിനപ്പുറവും ദിവസേനയുടെ വികസന പ്രവൃത്തി പ്രവാഹങ്ങളിൽ യഥാർത്ഥ വ്യത്യാസം സൃഷ്ടിച്ച് കൊണ്ടിരിക്കുന്ന സർവറുകൾ കാണിക്കുന്നു.

ഈ ഗൈഡിലുള്ള ഓരോ സർവറും യാഥാർത്ഥ്യ ഉപയോഗവും ഡെവലപ്പർ പ്രതികരണവും അടിസ്ഥാനമാക്കി തിരഞ്ഞെടുത്തതാണ്. ഓരോ സർവറും എന്തൊക്കെയാണ് ചെയ്യുന്നത് എന്നതിൽ മാത്രമല്ല, അതിൻ്റെ പ്രാധാന്യം എന്ത്, നിങ്ങളുടെ സ്വന്തം പ്രോജക്റ്റുകളിൽ അതിൽ നിന്ന് കൂടുതലെടുക്കാൻ എങ്ങനെ എന്നതും നിങ്ങൾ കണ്ടെത്തും. നിങ്ങൾ MCP-യിൽ പൂർണ്ണമായും പുതിയവരാണ് എങ്കിൽ അല്ലെങ്കിൽ നിലവിലെ സജ്ജീകരണം വിപുലീകരിക്കാൻ ആഗ്രഹിക്കുന്നവരാണ് എങ്കിൽ, ഈ സർവറുകൾ মൈക്രോസോഫ്റ്റ് എക്കോസിസ്റ്റത്തിൽ ലഭ്യമായ ഏറ്റവും പ്രായോഗികവും കാര്യക്ഷമവുമായ ഉപകരണങ്ങളിലൊന്നാണ്.

> **💡 എളുപ്പമേറിയ തുടക്കം**
> 
> MCP-യിൽ പുതിയവരാണോ? വിഷമിക്കേണ്ട! ഈ ഗൈഡ് തുടങ്ങാനുണ്ടായിരിക്കുന്നത് തുടങ്ങുന്നവർക്ക് അനുയോജ്യമായ രീതിയിലാണ്. ഞങ്ങൾ സ്‌പഷ്ടീകരിക്കുന്ന ആശയങ്ങൾ എല്ലാ ഘട്ടത്തിലും നിർവഹിക്കും, നിങ്ങൾക്ക് എപ്പോഴും നമ്മുടെ [MCP-ൽ പരിചയം](../00-Introduction/README.md)യും [പ്രധാന ആശയങ്ങൾ](../01-CoreConcepts/README.md) എന്ന മോഡ്യൂളുകൾക്കു തിരികെ പോകാം കൂടുതൽ ആഴത്തിലുള്ള പശ്ചാത്തലം കാണാൻ.

## അവലോകനം

ഈ സമഗ്ര ഗൈഡ് എഐ അസിസ്റ്റന്റുകളുമായി ഡെവലപ്പർമാർ ഇടപഴകുന്നത് എങ്ങനെ വിപ്ലവകരമായി മാറ്റിവെക്കുകയാണെന്ന് തിരഞ്ഞെടുത്ത പത്തു മൈക്രോസോഫ്റ്റ് MCP സർവറുകൾ പരിശോധിക്കുന്നു. ആസ്യൂർ റിസോഴ്‌സ് മാനേജ്മെന്റ് മുതൽ ഡോക്യുമെന്റ് പ്രോസസിംഗ് വരെ, ഈ സർവറുകൾ മോഡൽ കോൺടക്സ് പ്രോട്ടോകോൾ ഉപയോഗിച്ച് സ്മூത്, ഉത്പാദകമായ വികസന പ്രവൃത്തി പ്രവാഹങ്ങൾ സൃഷ്ടിക്കുന്നതിൽ പ്രതിഭാസമാണ്.

## പഠന ലക്ഷ്യങ്ങൾ

ഈ ഗൈഡ് അവസാനിക്കുന്നതിന് മുമ്പ്, നിങ്ങൾക്ക്:
- MCP സർവറുകൾ ഡെവലപ്പർ ഉത്പാദകക്ഷമത എങ്ങനെ മെച്ചപ്പെടുത്തുന്നു എന്നത് മനസ്സിലാക്കുക
- മൈക്രോസോഫ്റ്റിന്റേതായ ഏറ്റവും സ്വാധീനമുള്ള MCP സർവർ നടപ്പാക്കലുകളെക്കുറിച്ച് അറിയുക
- ഓരോ സർവറിനും പ്രായോഗിക ഉപയോഗ കേസുകൾ കണ്ടുപിടിക്കുക
- VS കോഡ്, വിസ്വൽ സ്റ്റുഡിയോ എന്നിവയിൽ ഈ സർവറുകൾ എങ്ങനെ ക്രമീകരിക്കാമെന്ന് അറിയുക
- MCP എക്കോസിസ്റ്റം കൂടുതൽ വികസിപ്പിച്ചറിയുക, ഭാവിയിൽ പോകുന്ന ദിശകൾ അന്വേഷിക്കുക

## 🔧 MCP സർവറുകൾ മനസ്സിലാക്കുക: ആരംഭകർക്കുള്ള ഗൈഡ്

### MCP സർവറുകൾ എന്തൊക്കെയാണ്?

മോഡൽ കോൺടക്സ് പ്രോട്ടോകോൾ (MCP) ഇപ്പോഴെത്തുന്നതിന് പുതിയ വ്യക്തിയായ നിങ്ങൾ ആലോചിക്കാം: "MCP സർവർ എന്നത് എന്താണ്, എനിക്ക് അതിൽ എങ്ങനെ പ്രാധാന്യമുണ്ടാകണം?" നൂതന ഉപമ ഉപയോഗിച്ച് തുടങ്ങാം.

MCP സർവറുകൾ എഐ കോഡിംഗ് കൂട്ടാളി (GitHub Copilot പോലുള്ള) പുറത്ത് ഉള്ള ഉപകരണങ്ങളുമായും സേവനങ്ങളുമായും ബന്ധപ്പെടാൻ സഹായിക്കുന്ന പ്രത്യേകസഹായികളായി കരുതുക. നിങ്ങൾ ഫോണിൽ വേറിട്ട ആപ്പുകൾ വിവിധ കാര്യങ്ങൾക്കായി ഉപയോഗിക്കുന്നത് പോലെ — കാലാവസ്ഥയ്ക്ക് ഒരു ആപ്പ്, നവിഗേഷനിന് മറ്റൊരു ആപ്പ്, ബാങ്ക് സേവനങ്ങൾക്ക് മറ്റൊരു — MCP സർവറുകൾ നിങ്ങളുടെ എഐ അസിസ്റ്റന്റിന് വ്യത്യസ്ത വികസന ഉപകരണങ്ങളുമായും സേവനങ്ങളുമായും ഇടപഴകാനുള്ള കഴിവ് നൽകുന്നു.

### MCP സർവറുകൾ പരിഹരിക്കുന്ന പ്രശ്നം

MCP സർവറുകൾ വരുന്നതിന് മുമ്പ്, നിങ്ങൾ ചെയ്തിരിക്കേണ്ടത്:
- നിങ്ങളുടെ ആസ്യൂർ റിസോഴ്‌സുകൾ പരിശോധിക്കുക
- GitHub ഇഷ്യൂ സൃഷ്ടിക്കുക
- ഡാറ്റാബേസ് ക്വറി ചെയുക
- ഡോക്യുമെന്റേഷൻ തിരയുക

ഈ പ്രവർത്തനങ്ങൾക്കായി കോഡിംഗ് നിർത്തി ബ്രൗസർ തുറന്ന് പടിയേറ്റ് യുക്തമായ വെബ്സൈറ്റ് സന്ദർശിച്ച് മാന്വൽ ആയി ചെയ്യേണ്ടതായിരുന്നു. ഈ സ്ഥിരം കോൺ‌ടെക്‌സ്‌റ്റ് മാറ്റം നിങ്ങളുടെ പ്രവാഹം തകരാറിലാക്കുകയും ഉത്പാദകത കുറയ്ക്കുകയും ചെയ്തു.

### MCP സർവറുകൾ നിങ്ങളുടെ വികസന അനുഭവം എങ്ങനെ മാറ്റിമറിക്കുന്നു

MCP സർവറുകളോടെ, നിങ്ങൾ നിങ്ങളുടെ വികസന പരിസ്ഥിതിയിലേക്കു (VS കോഡ്, വിസ്വൽ സ്റ്റുഡിയോ തുടങ്ങിയവ) പിന്നീടായും തുടരാം, നിങ്ങളുടെ എഐ അസിസ്റ്റന്റോട് ഈ പ്രവർത്തനങ്ങൾ കൈകാര്യം ചെയ്യാൻ ചോദിക്കാം. ഉദാഹരണത്തിന്:

**പരമ്പരാഗത പ്രവൃത്തി ശ്രേണിയുടെ പകരം:**
1. കോഡിംഗ് നിർത്തുക
2. ബ്രൗസർ തുറക്കുക
3. ആസ്യൂർ പോർട്ടലിലേക്ക് പോകുക
4. സ്റ്റോറേജ് അക്കൗണ്ട് വിശദാംശങ്ങൾ നോക്കുക
5. VS കോഡിലേക്ക് മടങ്ങുക
6. കോഡിംഗ് തുടരണം

**ഇപ്പോൾ നിങ്ങൾക്ക് ഇത് ചെയ്യാം:**
1. എഐ-നോട് ചോദിക്കുക: "എന്റെ ആസ്യൂർ സ്റ്റോറേജ് അക്കൗണ്ടുകളുടെ നില എന്താണ്?"
2. ലഭിച്ച വിവരങ്ങല്‍ക്ക് ഒപ്പം കോഡിംഗ് തുടരണം

### തുടക്കക്കാർക്കുള്ള പ്രധാന ഗുണങ്ങൾ

#### 1. 🔄 **നിങ്ങളുടെ പ്രവാഹ അവസ്ഥയിൽ തുടരുക**
- പല ആപ്രണക്കലുകൾക്കിടയിൽ മാറേണ്ട ആവശ്യം ഇല്ല
- നിങ്ങൾ എഴുതുന്ന കോഡിൽ കേന്ദ്രീകരിക്കുക
- വ്യത്യസ്ത ഉപകരണങ്ങൾ കൈകാര്യം ചെയ്യുന്നതിന്റെ മാനസിക ഭാര軽мен ചെയ്യുക

#### 2. 🤖 **കമ്പ്ലെക്സ് കമാൻഡുകളുടെ പകരം സ്വാഭാവിക ഭാഷ ഉപയോഗിക്കുക**
- SQL സിന്താക്സ് ഓർക്കാനല്ല, നിങ്ങൾക്ക് വേണമെന്ന ഡാറ്റ വിശകലനം ചെയ്യുക
- ആസ്യൂർ CLI കമാൻഡുകൾ ഓർക്കാതെ, നിങ്ങൾ ചെയ്യാൻ ആഗ്രഹിക്കുന്നത് വിശദീകരിക്കുക
- സാങ്കേതിക വിശദാംശങ്ങൾ എഐ കൈകാര്യം ചെയ്യട്ടെ, നിങ്ങൾ ലാജിക്-ൽ കേന്ദ്രീകരിക്കുക

#### 3. 🔗 **പല ഉപകരണങ്ങളും ബന്ധിപ്പിക്കുക**
- വ്യത്യസ്ത സേവനങ്ങൾ സംയോജിപ്പിച്ച് ശക്തമായ പ്രവൃത്തി പ്രവാഹങ്ങൾ സൃഷ്ടിക്കുക
- ഉദാഹരണം: "എല്ലാ പുതിയ GitHub ഇഷ്യൂകളും പിടിച്ച് ബന്ധപ്പെട്ട ആസ്യൂർ ഡെവ്ഒപ്സ് ജോലി ഐറ്റങ്ങൾ സൃഷ്ടിക്കുക"
- കോമ്പ്ലെക്സ് സ്ക്രിപ്റ്റുകൾ എഴുതാതെ ഓട്ടോമേഷൻ നിർമ്മിക്കുക

#### 4. 🌐 **വരെച്ചുകൊണ്ടിരിക്കുന്ന ഇക്കോസിസ്റ്റത്തിലെ പ്രവേശനം**
- മൈക്രോസോഫ്റ്റും GitHub-ഉം മറ്റ് കമ്പനികളും നിർമ്മിച്ച സർവറുകളിൽ നിന്നുള്ള ഗുണം നേടുക
- വിവിധ വിൽപ്പനക്കാരുടെ ഉപകരണങ്ങൾ എളുപ്പത്തിൽ കൂർത്തുക
- വിവിധ എഐ അസിസ്റ്റന്റുകളുമായി പ്രവർത്തിക്കുന്ന ഒരു സ്റ്റാൻഡേർഡ് ഇക്കോസിസ്റ്റത്തിലേക്ക് ചേർക്കുക

#### 5. 🛠️ **പ്രായോഗിക പരിചയത്തിലൂടെ പഠിക്കുക**
- ആശയങ്ങൾ മനസിലാക്കാൻ പ്രീ-ബിൽറ്റ് സർവറുകളിൽ ആരംഭിക്കുക
- കൂടുതൽ ആശ്വാസമുള്ളപ്പോൾ നിങ്ങളുടെ സ്വന്തം സർവറുകൾ നിർമ്മിക്കുക
- ലഭ്യമായ SDKകളും ഡോക്യുമെന്റേഷനും പഠനം മാർഗ്ഗനിർദ്ദേശിക്കാൻ ഉപയോഗിക്കുക

### ആരംഭകർക്കുള്ള യഥാർത്ഥ ലോക ഉദാഹരണം

നിങ്ങൾ വെബ് വികസനത്തിൽ പുതിയവനാണെന്നും നിങ്ങളുടെ ആദ്യ പ്രോജക്റ്റിൽ ജോലി ചെയ്യുകയാണെന്നും പറയാം. MCP സർവറുകൾ എങ്ങനെ സഹായിക്കും:

**പരമ്പരാഗത സമീപനം:**
```
1. Code a feature
2. Open browser → Navigate to GitHub
3. Create an issue for testing
4. Open another tab → Check Azure docs for deployment
5. Open third tab → Look up database connection examples
6. Return to VS Code
7. Try to remember what you were doing
```

**MCP സർവറുകളോടുകൂടെ:**
```
1. Code a feature
2. Ask AI: "Create a GitHub issue for testing this login feature"
3. Ask AI: "How do I deploy this to Azure according to the docs?"
4. Ask AI: "Show me the best way to connect to my database"
5. Continue coding with all the information you need
```

### എന്റർപ്രൈസ് സ്റ്റാൻഡേർഡ് നേട്ടം

MCP ഒരു വ്യവസായ വ്യാപക നിലവാരമാകുകയാണ്, അതുകൊണ്ട്:
- **സ്ഥിരത**: വ്യത്യസ്ത ഉപകരണങ്ങളിലും കമ്പനികളിലും സമാന അനുഭവം
- **ഇന്റർഓപ്പറബിലിറ്റി**: വ്യത്യസ്ത വിൽപ്പനക്കാരുടെ സർവറുകൾ തമ്മിൽ പ്രവർത്തിക്കുന്നു
- **ഭാവിയെ ഉറപ്പാക്കൽ**: വ്യത്യസ്ത എഐ അസിസ്റ്റന്റുകൾക്കിടയിലെ കഴിവുകളും സജ്ജീകരണങ്ങളും കൈമാറാം
- **സാമൂഹ്യത്തിൽ**: പങ്കുവയ്ക്കപ്പെട്ട അറിവും വിഭവങ്ങളും വലിയ ഇക്കോസിസ്റ്റം

### ആരംഭിക്കുന്നത്: നിങ്ങൾക്ക് എന്ത് പഠിക്കാനാണ്

ഈ ഗൈഡിൽ, ഡെവലപ്പർമാരുടെ എല്ലാ തലങ്ങളിൽ പ്രത്യേകിച്ച് പ്രയോജനകരമായ 10 മൈക്രോസോഫ്റ്റ് MCP സർവറുകൾ പരിശോധിക്കും. ഓരോ സർവറും ഒരുങ്ങുന്നത്:
- പൊതു വികസന വെല്ലുവിളികൾ പരിഹരിക്കാൻ
- ആവർത്തന പ്രവൃത്തികൾ കുറയ്ക്കാൻ
- കോഡ് ഗുണനിലവാരം മെച്ചപ്പെടുത്താൻ
- പഠന അവസരങ്ങൾ വളർത്താൻ

> **💡 പഠന നിർദ്ദേശം**
> 
> MCP-യിൽ പൂർണ്ണമായി പുതിയവരായാൽ, ആദ്യം നമ്മുടെ [MCP പരിചയം](../00-Introduction/README.md)യും [പ്രധാന ആശയങ്ങൾ](../01-CoreConcepts/README.md) മോഡ്യൂളുകളും പഠിക്കുക. അതിന് ശേഷം ഇവിടെ തിരിച്ചുവന്നു ഈ ആശയങ്ങൾ യഥാർഥ മൈക്രോസോഫ്റ്റ് ടൂളുകളോടും കൂടി പ്രവർത്തനത്തിൽ കാണുക.
>
> MCP-യുടെ പ്രാധാന്യത്തെക്കുറിച്ച് കൂടുതൽ പശ്ചാത്തലത്തിന്, Maria Naggagaയുടെ പോസ്റ്റ് കാണുക: [Connect Once, Integrate Anywhere with MCP](https://devblogs.microsoft.com/blog/connect-once-integrate-anywhere-with-mcps).

## VS കോഡിലും വിസ്വൽ സ്റ്റുഡിയോയിലുമായുള്ള MCP ആരംഭിക്കൽ 🚀

GitHub Copilot ഉപയോഗിക്കുന്ന വിസ്വൽ സ്‌ടുഡിയോ കോഡ് അല്ലെങ്കിൽ വിസ്വൽ സ്റ്റുഡിയോ 2022 ഉപയോഗിക്കുന്ന പക്ഷം ഈ MCP സർവറുകൾ ക്രമീകരിക്കുന്നത് എളുപ്പമാണ്.

### VS കോഡ് ക്രമീകരണം

VS കോഡിനുള്ള അടിസ്ഥാന പ്രക്രിയ ഇതാ:

1. **ഏജന്റ് മോഡ് സജീവമാക്കുക**: VS കോഡിൽ Copilot ചാറ്റ് വിൻഡോയിൽ ഏജന്റ് മോഡിലേക്ക് മാറുക
2. **MCP സർവറുകൾ ക്രമീകരിക്കുക**: നിങ്ങളുടെ VS കോഡ് settings.json ഫയലിലേക്ക് സർവർ ക്രമീകരണങ്ങൾ ചേർക്കുക
3. **സർവറുകൾ ആരംഭിക്കുക**: ഉപയോഗിക്കാൻ ആഗ്രഹിക്കുന്ന ഓരോ സർവറിനും "Start" ബട്ടൺ ക്ലിക്കു ചെയ്യുക
4. **ടൂളുകൾ തിരഞ്ഞെടുക്കുക**: നിങ്ങളുടെ പ്രస్తుత സെഷനിൽ സജീവമാക്കേണ്ട MCP സർവറുകൾ തിരഞ്ഞെടുക്കുക

വിശദരൂപത്തിലുള്ള ക്രമീകരണ നിർദ്ദേശങ്ങൾക്ക് [VS Code MCP ഡോക്യുമെന്റേഷൻ](https://code.visualstudio.com/docs/copilot/copilot-mcp) കാണുക.

> **💡 പ്രോ ടിപ്പ്: MCP സർവറുകൾ പ്രൊഫഷണല്മാരെപ്പോലെ മാനേജ് ചെയ്യൂ!**
> 
> VS കോഡ് എക്സ്റ്റൻഷൻസ് ദൃശ്യം ഇപ്പോൾ ഇൻസ്റ്റാൾ ചെയ്ത MCP സർവർ മാനേജ്മെന്റിനായുള്ള [സഹായകമായ പുതിയ UI](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode) ഉൾക്കൊള്ളുന്നു! നിങ്ങൾക്ക് ഏതെങ്കിലും ഇൻസ്റ്റാൾ ചെയ്ത MCP സർവർ സുതാര്യമായ, ലളിതമായ ഇന്റർഫേസിൽ നിന്ന് "Start", "Stop" എന്നിവ ചെയ്യാനും നിയന്ത്രിക്കാനും അവസരം ഉണ്ട്. പരീക്ഷിച്ച് നോക്കൂ!

### വിസ്വൽ സ്റ്റുഡിയോ 2022 ക്രമീകരണം

17.14 അല്ലെങ്കിൽ അതിന്റെ മുമ്പത്തെ പതിപ്പുകളിൽ വിസ്വൽ സ്റ്റുഡിയോ 2022 ഉപയോഗിക്കുന്നവർക്ക്:

1. **ഏജന്റ് മോഡ് സജീവമാക്കുക**: GitHub Copilot ചാറ്റ് വിൻഡോയിലുള്ള "Ask" ഡ്രോപ്പ്‌ഡൗണിൽ നിന്ന് "Agent" തിരഞ്ഞെടുക്കുക
2. **ക്രമീകരണ ഫയൽ സൃഷ്ടിക്കുക**: നിങ്ങളുടെ സൊല്യൂഷൻ ഡയറക്ടറിയിൽ `.mcp.json` ഫയൽ സൃഷ്ടിക്കുക (ശുപാർശ ചെയ്ത സ്ഥലം: `<SOLUTIONDIR>\.mcp.json`)
3. **സർവറുകൾ ക്രമീകരിക്കുക**: സാധാരണ MCP ഫോർമാറ്റിൽ നിങ്ങളുടെ MCP സർവർ ക്രമീകരണങ്ങൾ ചേർക്കുക
4. **ടൂൾ അംഗീകാരം**: ആവശ്യപ്പെട്ടാൽ ഉപയോഗിക്കാൻ ആഗ്രഹിക്കുന്ന ടൂളുകൾക്ക് അനുയോജ്യമായ സ്കോപ്പ് അനുമതികളോടെ അംഗീകാരം നൽകുക

വിശദമായ വിസ്വൽ സ്റ്റുഡിയോ ക്രമീകരണ നിർദ്ദേശങ്ങൾക്കായി [Visual Studio MCP ഡോക്യുമെന്റേഷൻ](https://learn.microsoft.com/visualstudio/ide/mcp-servers) കാണുക.

ഓരോ MCP സർവറിനും സ്വന്തം ക്രമീകരണ ആവശ്യകതകൾ (കണക്ഷൻ സ്റ്റ്രിംഗുകൾ, ഓത്തൻ‌റിക്കേഷൻ, മുതലായവ) ഉണ്ടെങ്കിലും ഇരുവിധ ഐഡിയും സമമായ ക്രമീകരണ രീതിയിലാണ്.

**പ്രധാന ഉദാഹരണം**: സർവർ `microsoft_docs_search` ടൂൾ എന്നതിനെ എക്സ്പോസ് ചെയ്യുന്നു, ഇത് മൈക്രോസോഫ്റ്റിന്റെ ഔദ്യോഗിക സാങ്കേതിക ഡോക്യുമെന്റേഷനിൽ സെമാന്റിക് സേർച്ച് നടത്തുന്നു. ക്രമീകരിച്ചതിനു ശേഷം "ASP.NET കോറിൽ JWT ഓത്തൻറിക്കേഷൻ എങ്ങനെ നടപ്പാക്കാം?" പോലുള്ള ചോദ്യങ്ങൾ ചോദിച്ച് വിശദമായ, ഔദ്യോഗിക മറുപടികളും ഉറവിട കണക്ഷനുകളും ലഭിക്കും. ഈ തിരച്ചിൽ നിലവാരം അപൂർവ്വമാണ് കാരണം ഇത് കോൺടെക്‌സ്‌റ്റ് മനസ്സിലാക്കുന്നു — "കൺടെയ്‌നറുകൾ" ആസ്യൂർ കോൺടെക്‌സ്‌റ്റിൽ ചോദിച്ചാൽ ആസ്യൂർ കണ്ടെയ്‌നർ ഇൻസ്റ്റൻസസ് ഡോക്യുമെന്റേഷൻ ലഭിക്കും, പക്ഷെ .NET കോൺടെക്‌സ്‌റ്റിൽ എങ്കിൽ പ്രസക്തമായ C# ശേഖരണ വിവരങ്ങൾ ലഭിക്കും.

പുതിയത് മാറുകയോ ഇടയ്ക്ക് പുതുക്കപ്പെടുകയോ ചെയ്യുന്ന ലൈബ്രറികളും ഉപയോഗകേസുകളും ഇതിന് പ്രത്യേക പ്രയോജനമാണ്. അടുത്തിടെ ചില കോഡിംഗ് പ്രോജക്റ്റുകളിൽ .NET Aspire, Microsoft.Extensions.AI ആയുള്ള ഏറ്റവും പുതിയ റിലീസുകളുടെ ഫീച്ചറുകൾ ഉപയോഗിക്കാനായി ഞാൻ ശ്രമിക്കുകയായിരുന്നു. **ഇത് ചെയ്യുന്ന കാര്യം**: Azure MCP സർവർ 15+ പ്രത്യേക Azure സേവന കണക്ടറുകളുടെ ഒരു സമഗ്ര സ്യൂട്ട് ആണ്, ഇത് നിങ്ങളുടെ AI വർക്ക്‌ഫ്ലോയിൽ മുഴുവൻ Azure ഇക്കോസിസ്റ്റമും കൊണ്ടുവരുന്നു. ഇത് ഒരു ഒറ്റ സർവർ അല്ല – ഇത് റിസോഴ്‌സ് മാനേജുമെന്റ്, ഡാറ്റാബേസ് കണക്റ്റിവിറ്റി (PostgreSQL, SQL Server), KQL ഉപയോഗിച്ചുള്ള Azure Monitor ലോഗ് വിശകലനം, Cosmos DB ഇന്റഗ്രേഷൻ എന്നിവ ഉൾപ്പെടുന്ന ശക്തമായ ഒരു ശേഖരമാണ്.

**ഇത് ഉപകാരപ്രദമായത് എന്തുകൊണ്ട്**: Azure റിസോഴ്‌സുകൾ നിയന്ത്രിക്കുന്നതിലുപരി, Azure SDKs-ഉടൻ പ്രവർത്തിക്കുമ്പോൾ കോഡ് ഗുണനിലവാരം ഗണ്യമായി മെച്ചപ്പെടുത്തുന്നു. Agent മോഡിൽ Azure MCP ഉപയോഗിക്കുമ്പോൾ, ഇത് നിങ്ങൾക്ക് കോഡ് എഴുതാൻ സഹായിക്കുന്നതോടൊപ്പം *മികച്ച* Azure കോഡ് എഴുതാൻ സഹായിക്കുന്നു, കൂടാതെ നിലവിലുള്ള അഥന്റിക്കേഷന്‍ മാതൃകകൾ, പിശക് കൈകാര്യം ചെയ്യല്‍, ലേറ്റസ്റ്റ് SDK ഫീച്ചറുകൾ എന്നിവ പിന്തുടരുന്നു. പ്രവർത്തനക്ഷമമായ സാധാരണ കോഡ് നേടുന്നതിനുപകരം, ഉത്പാദന പ്രവര്‍ത്തനങ്ങൾക്ക് Azure നല്കുന്ന ശുപാർശ ചെയ്ത മാതൃകകൾ അനുസരിക്കുന്ന കോഡ് നിങ്ങൾക്ക് ലഭിക്കും.

**പ്രധാന ഘടകങ്ങൾ ഉൾപ്പെടുന്നു**:
- **🗄️ ഡാറ്റാബേസ് കണക്ടറുകൾ**: PostgreSQL, SQL Server എന്നിവയ്ക്കുള്ള Azure ഡാറ്റാബേസ് നേരിട്ട് ശരീരഭാഷാ ആക്‌സസ്
- **📊 Azure Monitor**: KQL-ഉൽപാദിത ലോഗ് വിശകലനം, ഓപ്പറേഷന്‍ ഇന്‍സൈറ്റുകൾ
- **🌐 റിസോഴ്‌സ് മാനേജുമെന്റ്**: പൂർണ്ണ Azure റിസോഴ്‌സ് ലൈഫ്സൈക്കിൾ മാനേജുമെന്റ്
- **🔐 അഥന്റിക്കേഷൻ**: DefaultAzureCredential, മാനേജുചെയ്ത ഐഡന്റിറ്റി മാതൃകകൾ
- **📦 സ്റ്റോറേജ് സേവനങ്ങൾ**: ബ്ലോബ്സ്, ക്യൂ, ടേബിൾ സ്റ്റോറേജ് ഓപ്പറേഷനുകൾ

- **മറ്റും പ്രത്യേകമായ കണക്ടറുകൾ**

**പ്രായോഗിക ഉപയോഗം**: "എന്റെ Azure സ്റ്റോറേജ് അക്കൗണ്ടുകൾ ലിസ്റ്റ് ചെയ്യുക", "കഴിഞ്ഞ മണിക്കൂറിലെ പിശകുകൾക്കായി എന്റെ Log Analytics വർക്ക്‌സ്പെയ്സ് ക്വറി ചെയ്യുക", അല്ലെങ്കിൽ "Node.js ഉപയോഗിച്ച് സൗകര്യപ്രദമായ അഥന്റിക്കേഷന്‍ ഉപയോഗിച്ച് Azure അപ്ലിക്കേഷൻ നിർമ്മിക്കാൻ സഹായിക്കുക"

**പൂർണ്ണ ഡെമോ സീനരിപ്പ്പാട്**: Azure MCP നും VS Code-ലുള്ള GitHub Copilot for Azure എക്സ്റ്റൻഷനും സംയോജിപ്പിക്കുന്നതിന്റെ ശക്തി കാണിക്കുന്ന ഒരു കംപ്ലീറ്റ് വാക്ക്ത്രൂ ഇവിടെ ഉണ്ട്. നിങ്ങൾ ഒരുമിച്ച് ഇൻസ്റ്റാൾ ചെയ്തിരിക്കുന്നത് എങ്കിൽ, ഇത് ചോദിച്ച് ചെയ്യാം:

> "DefaultAzureCredential അഥന്റിക്കേഷൻ ഉപയോഗിച്ച് Azure Blob Storage-ൽ ഫയൽ അപ്‌ലോഡ് ചെയ്യുന്ന Python സ്‌ക്രിപ്റ്റ് സൃഷ്ടിക്കൂ. 'mycompanystorage' എന്ന എന്റെ Azure സ്റ്റോറേജ് അക്കൗണ്ടിലേക്ക് കണക്ട് ചെയ്യണം, 'documents' എന്ന കണ്ടെയ്‌നറിൽ അപ്‌ലോഡ് ചെയ്യണം, ഇപ്പോഴത്തെ ടൈംസ്റ്റാമ്പ് ഉപയോഗിച്ച് ഒരു ടെസ്റ്റ് ഫയൽ സൃഷ്ടിക്കുകയും ചെയ്യണം, പിശകുകൾ സൌഹൃദമായി കൈകാര്യം ചെയ്യുകയും വിവരപ്രദമായ ഔട്ട്‌പുട്ടു നൽകുകയും ചെയ്യണം, അഥന്റിക്കേഷനും പിശക് കൈകാര്യം ചെയ്യലും Azure-യുടെ മികച്ച രീതികൾ അനുസരിച്ച് ആണെന്ന് ഉറപ്പാക്കുക, DefaultAzureCredential അഥന്റിക്കേഷൻ എങ്ങനെ പ്രവർത്തിക്കുന്നതാണെന്ന് വിശദീകരിക്കുന്ന കമന്റുകളും ഉൾപ്പെടുത്തുക, ശരിയായ ഫങ്ഷനുകളും ഡോക്ക്യുമെന്റേഷനും ഉൾപ്പെടുത്തി സ്‌ക്രിപ്റ്റ് നല്ല രീതിയിൽ ഘടിപ്പിക്കുക."

Azure MCP Server ഒരു മുഴുവൻ ഉത്പാദനക്ഷമമായ Python സ്‌ക്രിപ്റ്റ് തയാറാക്കും:
- പുതിയ Azure Blob Storage SDK ഏറ്റവും പുതിയ async മാതൃകകൾ ഉപയോഗിച്ച്
- DefaultAzureCredential അതിന്റെ പരിപൂർണ ബാക്ക്‌ഫാൾ ചെയിൻ വിശദീകരണത്തോടെ നടപ്പിലാക്കുന്നു
- പ്രത്യേക Azure exceptiosntypes-ഉം ഉൾപ്പെടുന്ന ശക്തമായ പിശക് കൈകാര്യം ചെയ്യല്
- Azure SDK ലെ നല്ല രീതികൾ അനുസരിച്ചുള്ള റിസോഴ്‌സ് മാനേജുമെന്റ്, കണക്ഷൻ കൈകാര്യം ചെയ്യൽ
- വിശദമായ ലോഗിംഗ്, വിവരപ്രദമായ കൺസോൾ ഔട്ട്‌പുട്ട്
- ഫങ്ഷനുകൾ, ഡോക്യുമെന്റേഷൻ, ടൈപ്പ് ഹിന്റുകൾ എന്നിവയോടെയുള്ള ശരിയായ ഘടനയുള്ള സ്‌ക്രിപ്റ്റ്

Azure MCP ഇല്ലാതെ സാധാരണ ബ്ലോബ്സ് സ്റ്റോറേജ് കോഡ് നിങ്ങൾക്ക് ലഭിക്കുന്നുണ്ടെങ്കിലും അത് നിലവിലെ Azure മാതൃകകൾ അനുസരിക്കാറില്ല. Azure MCP ഉപയോഗിക്കുമ്പോൾ, ഏറ്റവും പുതിയ അഥന്റിക്കേഷൻ രീതികൾ പ്രയോജനപ്പെടുത്തുന്ന, Azure-സവിശേഷമായ പിശക് സന്നിവേശങ്ങൾ കൈകാര്യം ചെയ്യുന്ന, പ്രൊഡക്ഷൻ അപ്ലിക്കേഷനുകൾക്കുള്ള Microsoft ശുപാർശകൾ പിന്തുടരുന്ന കോഡ് നിങ്ങൾക്ക് കിട്ടും.

**പ്രധാന ഉദാഹരണം**: `az` , `azd` CLI-കളുടെ പ്രത്യേക കമാൻഡുകൾ ഓർക്കുന്നതിൽ ഞാൻ പതിവായി ബുദ്ധിമുട്ടുന്നു. എനിക്ക് എല്ലായ്പ്പോഴും രണ്ട് ഘട്ട പ്രക്രിയ: ആദ്യം സിന്റക്സ്സ് നോക്കുക, പിന്നെ കമാൻഡ് എക്‌സിക്യൂട്ട് ചെയ്യുക. CLI സിന്റക്സ്സ് ഓർക്കാൻ കഴിയില്ലെന്ന് സമ്മതിക്കാൻ ഇച്ഛിക്കാതെ വേണ്ട വേഗത്തിൽ പോർട്ടലിൽ പ്രവേശിച്ച് ക്ലിക്ക് ചെയ്യുന്നതാണ് എനിക്ക് പതിവ്. എന്നാൽ ഞാൻ വേണ്ടുന്നത് വിവരണം നൽകുന്നതു മാത്രമാണ്, അതും IDE വിട്ടുവിട്ടോ വേണ്ടാതെ ചെയ്യാൻ കഴിയും എന്നത് അത്ഭുതകരമാണ്!

ആരംഭിക്കാൻ സഹായിക്കുന്ന മികച്ച ഉപയോഗകേസുകൾ [Azure MCP രെപ്പോസിറ്ററിയിൽ](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) ഉണ്ട്. സമഗ്ര സെറ്റപ്പ് ഗൈഡുകൾക്കും ഉയർന്ന നിലവാരത്തിലുള്ള കോൺഫിഗറേഷൻ ഓപ്ഷനുകൾക്കുമായി [അധികാരിക Azure MCP ഡോക്യുമെന്റേഷൻ](https://learn.microsoft.com/azure/developer/azure-mcp-server/) കാണുക.

### 3. 🐙 GitHub MCP Server

**ഇത് ചെയ്യുന്ന കാര്യം**: ഔദ്യോഗിക GitHub MCP Server GitHub-ന്റെ മുഴുവൻ ഇക്കോസിസ്റ്റത്തിനൊപ്പം സമന്വയിക്കുന്നുണ്ടെന്ന് ഉറപ്പാക്കുന്നു, ഹോസ്റ്റഡ് റിമോട്ട് ആക്‌സസ് കൂടാതെ ലോക്കൽ ഡോക്കർ ഡിപ്ലോയ്മെന്റ് ഓപ്ഷനുകളും നൽകുന്നു. അടിസ്ഥാന റിപോസിറ്ററി പ്രവർത്തനങ്ങൾക്കപ്പുറം, GitHub Actions മാനേജുമെന്റ്, പുൾ റിക്വസ്റ്റ് വർക്‌ഫ്ലോകൾ, ഇഷ്യൂ ട്രാക്കിങ്, സുരക്ഷാ സ്കാനിംഗ്, നോട്ടിഫിക്കേഷനുകൾ, ആഡ്‌വാൻസ്ഡ് ഓട്ടോമേഷൻ എന്നിവ ഉൾപ്പെട്ട സവിശേഷ ഉപകരണങ്ങളാണ് ഇത്.

**ഇത് ഉപകാരപ്രദമായത് എന്തുകൊണ്ട്**: GitHub ന്റെ മുഴുവൻ പ്ലാറ്റ്ഫോം അനുഭവം നിങ്ങളുടെ ഡവലപ്പ്മെന്റ് പരിസരത്തേക്ക് നേരിട്ടെത്തിക്കുന്നതിനാൽ GitHub നുമായി സംവദിക്കുന്ന വിധം മാറ്റുന്നു. പ്രോജക്ട് മാനേജുമെന്റ്, കോഡ് റിവ്യൂസ്, CI/CD നിരീക്ഷണം എന്നിവയ്ക്കായി VS Code-നും GitHub.com-നും ഇടയ്ക്ക് മാറുന്നതിനുപകരം, സ്വാഭാവിക ഭാഷാ കമാൻഡുകൾ വഴിയും എല്ലാം കൈകാര്യം ചെയ്യാം, കൂടെ നിങ്ങളുടെ കോഡിൽ കേന്ദ്രീകരിച്ചുകൊണ്ട്.

> **ℹ️ ശ്രദ്ധിക്കുക: വ്യത്യസ്ത 'Agent' തരം**
> 
> GitHub MCP Server-നെ GitHub-യുടെ Coding Agent-എൻറെ (സ്വയംരൂപീകരിച്ച കോഡിംഗ് ടെസ്കുകൾക്ക് ഇഷ്യൂകൾ നിയോഗിക്കാൻ കഴിയുന്ന AI ഏജന്റ്) കൂട്ടിച്ചേർക്കരുത്. GitHub MCP Server VS Code-ലെ Agent മോഡിൽ GitHub API ഇന്റഗ്രേഷൻ നൽകുന്നു, GitHub-യുടെ Coding Agent GitHub ഇഷ്യൂകൾക്ക് നിയോഗിക്കുമ്പോൾ പുൾ റിക്വസ്റ്റുകൾ സൃഷ്ടിക്കുന്ന വേറെ ഫീച്ചറാണ്.

**പ്രധാന സവിശേഷതകൾ**:
- **⚙️ GitHub Actions**: പൂർണ്ണ CI/CD പൈപ്പ്‌ലൈൻ മാനേജുമെന്റ്, വർക്‌ഫ്ലോ നിരീക്ഷണം, ആർട്ടിഫാക്ട് കൈകാര്യം
- **🔀 പുൾ റിക്വസ്റ്റുകൾ**: പുൾ റിക്വസ്റ്റുകൾ സൃഷ്ടിക്കൽ, റിവ്യൂ, മർജ്, ഘടനാപൂർവ്വക സ്റ്റാറ്റസ് ട്രാക്കിംഗ്
- **🐛 ഇഷ്യൂസ്**: പൂര്‍ണ്ണ ഇഷ്യൂ ലൈഫ്‌സൈക്കിൾ മാനേജുമെന്റ്, കമന്റിംഗ്, ലേബലിംഗ്, നിയമനം
- **🔒 സുരക്ഷ**: കോഡ് സ്കാനിംഗ് അലർട്ടുകൾ, രഹസ്യ കണ്ടെത്തൽ, Dependabot ഇന്റഗ്രേഷൻ
- **🔔 നോട്ടിഫിക്കേഷനുകൾ**: ബുദ്ധിമാനായ നോട്ടിഫിക്കേഷൻ മാനേജുമെന്റ്, റിപോസിറ്ററി സബ്സ്ക്രിപ്ഷൻ നിയന്ത്രണം
- **📁 റിപോസിറ്ററി മാനേജുമെന്റ്**: ഫയൽ ഓപ്പറേഷനുകൾ, ബ്രാഞ്ച് മാനേജുമെന്റ്, റിപോസിറ്ററി അഡ്മിനിസ്ട്രേഷൻ
- **👥 സഹകരണം**: ഉപയോക്താക്കൾ, ഓർഗനൈസേഷൻ സെർച്ച്, ടീം മാനേജുമെന്റ്, ആക്‌സസ് നിയന്ത്രണം

**പ്രായോഗിക ഉപയോഗം**: "എന്റെ ഫീച്ചർ ബ്രാഞ്ചിൽ നിന്ന് പുൾ റിക്വസ്റ്റ് സൃഷ്ടിക്കൂ", "ഈ ആഴ്ച എല്ലാ പരാജയപ്പെട്ട CI ഓട്ടോമേറ്റഡ് റൺസും കാണിക്കൂ", "എന്റെ റിപോസിറ്ററികളിൽ തുറന്നിരിക്കുന്ന എല്ലാ സുരക്ഷാ അലർട്ടുകളും ലിസ്റ്റ് ചെയ്യൂ", അല്ലെങ്കിൽ "എന്റെ സംഘടനകളിൽ എനിക്കായി നിയോഗിച്ച എല്ലാ ഇഷ്യൂകളും കണ്ടെത്തൂ"

**പൂർണ്ണ ഡെമോ സീനരിപ്പ്പാട്**: GitHub MCP Server-ന്റെ ശേഷികളെ കാണിക്കുന്ന ശക്തമായ വർക്‌ഫ്ലോ ഇവിടെ:

>  "ഞങ്ങളുടെ സ്പ്രിന്റ് റിവ്യൂക്കായി തയ്യാറെടുക്കണം. ഈ ആഴ്ച ഞാൻ സൃഷ്ടിച്ച എല്ലാ പുൾ റിക്വസ്റ്റുകളും കാണിക്കൂ, നമ്മുടെ CI/CD പൈപ്പ്‌ലൈൻ സ്റ്റാറ്റസ് പരിശോധിക്കൂ, പരിഹരിക്കേണ്ടതായ സുരക്ഷാ അലർട്ടുകളുടെ സംഗ്രഹം സൃഷ്ടിക്കൂ, 'feature' ലേബൽ ഉള്ള മർജ് ചെയ്ത PR-കളുടെ അടിസ്ഥാനത്തിൽ റിലീസ് നോട്ടുകൾ തയ്യാറാക്കാൻ സഹായിക്കൂ."

- പുതിയ പുൾ റിക്വസ്റ്റുകളുടെ സ്റ്റാറ്റസ് വിവരങ്ങൾ വിശദമായി ചോദ്യം ചെയ്യും
- വർക്‌ഫ്ലോ റൺസുകൾ വിശകലനം ചെയ്ത് പരാജയങ്ങൾയും പ്രകടന പ്രശ്നങ്ങളും ഹൈലൈറ്റ് ചെയ്യും
- സുരക്ഷാ സ്കാനിംഗ് ഫലങ്ങൾ സംഗ്രഹിച്ച് ഗുരുതര അലർട്ടുകൾ മുൻഗണന നൽകും
- മർ‍ജ് ചെയ്ത പുൾ റിക്വസ്റ്റ് വിവരങ്ങൾ എടുത്ത് സമഗ്ര റിലീസ് നോട്ടുകൾ സൃഷ്ടിക്കും
- സ്പ്രിന്റ് രൂപരേഖാ ഒരുക്കം, റിലീസ് തയ്യാറെടുപ്പിന് നടപ്പിലാക്കാവുന്ന അടുത്ത ഘടകങ്ങൾ നൽകും

**പ്രധാന ഉദാഹരണം**: കോഡ് റിവ്യൂ വർക്‌ഫ്ലോകൾക്കായി ഇത് ഉപയോഗിക്കുന്നത് എനിക്ക് വല്ലാതെ ഇഷ്ടമാണ്. VS Code, GitHub നോട്ടിഫിക്കേഷനുകൾ, പുൾ റിക്വസ്റ്റ് പേജുകൾ എന്നിവ ഇടക്ക് മാറുന്നതിന് പകരം, "എന്റെ റിവ്യൂക്കായി കാത്തിരിക്കുന്ന എല്ലാ PR-കളും കാണിക്കൂ" എന്നും "അഥന്റിക്കേഷൻ രീതിയിൽ പിശക് കൈകാര്യം ചെയ്യലിനെക്കുറിച്ച് PR #123-ൽ ഒരു കമന്റ് ചേർക്കൂ" എന്നും പറയാം. സർവർ GitHub API കോളുകൾ കൈകാര്യം ചെയ്‌തു, ചർച്ചയുടെ കോൺടെക്സ്റ്റ് നിലനിർത്തുകയും, കൂടുതൽ നിർമ്മാണաքինമായ റിവ്യൂ കമന്റുകൾ തയ്യാറാക്കാൻ സഹായിക്കുകയും ചെയ്യും.

**അഥന്റിക്കേഷൻ ഓപ്ഷനുകൾ**: ഈ സർവർ OAuth (VS Code-ൽ seamless) മായും Personal Access Tokens മായും പിന്തുണയുള്ളതാണ്, നിങ്ങൾക്ക് ആവശ്യമുള്ള GitHub ഫംഗ്ഷൻ മാത്രമേ പ്രവർത്തിക്കൂ എന്ന് കൺഫിഗർ ചെയ്യാൻ ടൂൾസെറ്റുകൾ ആണ്. ഇൻസ്റ്റന്റ് സെറ്റപ്പിനായി റിമോട്ട് ഹോസ്റ്റഡ് സർവീസ് ആയി ഓടിക്കാം, അല്ലെങ്കിൽ ഡോക്കർ മുഖേന ലൊക്കലായി ഓടിച്ച് പൂർണ്ണ നിയന്ത്രണവും ലഭിക്കും.

**ഇത് ചെയ്യുന്നതു**: MarkItDown ഒരു സമഗ്രമായ ഡോക്യുമെന്റ് പരിവർത്തന സര്‍വറാണ്, വിവിധ ഫയല്‍ ഫോർമാറ്റുകൾ ഉയർന്ന നിലവാരമുള്ള മാർക്ക്ഡൗണിലേക്ക്, LLM ഉപഭോഗത്തിനും വാചക വിശകലന പ്രവൃത്തികള്‍ക്കും അനുയോജ്യമായി പരിവർത്തനം ചെയ്യുന്നു.

**എന്തിനാണ് ഇത് പ്രയോജനം**: ആധുനിക രേഖപത്രപ്രവർത്തനങ്ങൾക്ക് അനിവാര്യമാണ്! MarkItDown പ്രധാന തലക്കെട്ടുകൾ, പട്ടികകൾ, പട്ടികകൾ, ലിങ്കുകൾ തുടങ്ങിയ അനിവാര്യമായ രേഖാ ഘടനകൾ സംരക്ഷിക്കുന്നതായി വിവിധ ഫയൽ ഫോർമാറ്റുകൾ കൈകാര്യം ചെയ്യുന്നു. സാധാരണ വാചക ഊര്‍ജ്ജം പകരാനുള്ള ഉപകരണങ്ങളെതിരെ, ഇത് സാങ്കേതിക അർത്ഥവും സജ്ജീകരണവും സൂക്ഷിക്കുന്നതിൽ കേന്ദ്രീകരിക്കുന്നു, അത് AI പ്രോസസ്സിംഗിനും മനുഷ്യ വായനാസാധ്യതയ്ക്കും മൂല്യമുണ്ടാക്കുന്നു.

**സഹായിക്കുന്ന ഫയല്‍ ഫോർമാറ്റുകൾ**:
- **ഓഫീസ് ഡോക്യുമെന്റുകൾ**: PDF, PowerPoint (PPTX), Word (DOCX), Excel (XLSX/XLS)
- **മീഡിയ ഫയലുകൾ**: ചിത്രംകൾ (EXIF മെറ്റഡേറ്റയും OCR ഉം ഉൾപ്പെടെ), ശബ്‌ദം (EXIF മെറ്റഡേറ്റയും ശബ്ദ ലിപ്യന്തരണം ഉൾപ്പെടെ)
- **വെബ് ഉള്ളടക്കം**: HTML, RSS ഫീഡുകൾ, YouTube URLs, Wikipedia പേജുകൾ
- **ഡേറ്റാ ഫോർമാറ്റുകൾ**: CSV, JSON, XML, ZIP ഫയലുകൾ (പരിഭവവിവരം പുനരാവൃത്തി ചെയ്ത് പ്രോസസ്സ് ചെയ്യുന്നു)
- **പബ്ലിഷിംഗ് ഫോർമാറ്റുകൾ**: EPub, Jupyter നോട്ട്ബുക്കുകൾ (.ipynb)
- **ഇമെയിൽ**: Outlook സന്ദേശങ്ങൾ (.msg)
- **ഉന്നതത്തെന്നുള്ളത്**: മെച്ചപ്പെടുത്തിയ PDF പ്രോസസ്സിംഗിനായി Azure Document Intelligence സംയോജനം

**ഉന്നത കഴിവുകള്‍**: OpenAI ക്ലയന്റ് നൽകിയാൽ LLM-നെ പിന്തുണച്ച് ചിത്രം വിവരണങ്ങള്‍, മെച്ചപ്പെടുത്തിയ PDF പ്രോസസ്സിംഗിനുള്ള Azure Document Intelligence, ശബ്ദ ലിപ്യന്തരം, കൂടാതെ അധിക ഫയൽ ഫോർമാറ്റുകൾക്കായി പ്ലഗിൻ സിസ്റ്റം MarkItDown പിന്തുണക്കുന്നു.

**വാസ്തവ ലോകത്തിലെ ഉപയോഗം**: "ഈ PowerPoint പ്രეზന്റേഷന് മാർക്ക്ഡൗണിലേക്ക് പരിവർത്തനം ചെയ്യുക", "ഈ PDF-യിൽ നിന്നുള്ള വാചകം ശരിയായ തലക്കെട്ട് ഘടനയോടുകൂടെ നീക്കം ചെയ്യുക", അല്ലെങ്കിൽ "ഈ Excel സ്പ്രെഡ്‌ഷീറ്റിനെ വായനാസാമർത്ഥ്യമുള്ള പട്ടിക ഫോർമാറ്റിലേക്ക് മാറ്റുക"

> മാർക്ക്ഡൗൺ ഒപറിച്ചവാക്യം അടച്ചിരിക്കുന്ന പെൺവാചകത്തെ വളരെ അടുത്ത്, കുറഞ്ഞ മാർക്കപ്പ് അല്ലെങ്കിൽ ഫോർമാറ്റിംഗോടെ, എന്നാൽ പ്രധാനപ്പെട്ട രേഖാ ഘടന പ്രതിനിധാനം ചെയ്യാനുള്ള മാർഗം നൽകുന്നു. പ്രധാന സ്രോതസ്സായ LLM-കൾ, ഉദാഹരണത്തിന് OpenAI യുടെ GPT-4o, സ്വാഭാവികമായി "മാർക്ക്ഡൗൺ സംസാരിക്കുന്നു," അനേകം തവണ അയച്ച മറുപടികളിൽ Markdown ഉൾപ്പെടുത്തുന്നു. ഇത് അവരാണ് വിപുലമായ Markdown-ഫോർമാറ്റ് ചെയ്ത വാചകത്തിൽ പരിശീലിപ്പിക്കപ്പെട്ടത്, അതും നന്നായി മനസ്സിലാക്കുന്നതും സൂചിപ്പിക്കുന്നു. ഒരു സൈഡ്ബിനിഫിറ്റായി, Markdown സങ്കേതങ്ങളും ടോക്കൺ കാര്യക്ഷമത വർദ്ധിപ്പിക്കുന്നു.

MarkItDown രേഖാ ഘടനയ്ക്ക് നല്ല പരിരക്ഷ നൽകുന്നു, അത് AI പ്രവൃത്തികൾക്ക് പ്രധാനമാണ്. ഉദാഹരണത്തിന്, PowerPoint പ്രეზന്റേഷൻ പരിവർത്തനം ചെയ്യുന്നപ്പോൾ അത് സ്ക്ലൈഡ് ക്രമീകരണം ശരിയായ തലക്കെട്ടുകളോടെ സൂക്ഷിക്കുന്നു, ടെബിളുകൾ Markdown പട്ടികകളായി നീക്കുന്നു, ചിത്രങ്ങൾക്ക് alt ടെക്സ്റ്റ് ഉൾപ്പെടുത്തുന്നു, വക്തൃ നോട്ടുകളും പ്രോസസ് ചെയ്യുന്നു. ചാർട്ടുകൾ വായനാസാധ്യമായ ഡാറ്റ പട്ടികകളിലേക്ക് പരിവർത്തനം ചെയ്യുന്നു, ഫലമായി Markdown പ്രഥമ പ്രეზന്റേഷൻയുടെ താൽപര്യമുള്ള പ്രവാഹം സൂക്ഷിക്കുന്നു. ഇത് AI സിസ്റ്റങ്ങളിലേക്കു പ്രეზന്റേഷൻ ഉള്ളടക്കം നൽകുന്നതിനോ നിലവിലുള്ള സ്‌ലൈഡുകളിൽ നിന്നുള്ള ഡോക്യുമെന്റേഷൻ സൃഷ്ടിയ്ക്കുന്നതിനോ പൂർണ്ണമായും അനുയോജ്യമാണ്.

James Montemagno सम्मेलन ഡെമോസിന് Dev Box എത്രമേൽ മികച്ചത് എന്നത് വിശദീകരിച്ചപ്പോൾ എന്റെ മനസ്സിലായത് പ്രസങ്കംന്റെ വലിയ ലോഡ്സൂത്രം അലവൽ എടുത്തിട്ടും മിന്നും ഇഥർനെറ്റ് കണക്ഷൻ ഉള്ളതാണ്. അതേസമയം, ഞാൻ ബ്രുഗെസ് മുതൽ ആന്റ്വർപ് വരെ ബസിൽ യാത്ര ചെയ്ത് എന്റെ ലാപ്‌ടോപ്പ് എന്റെ ഫോൺ ഹോട്ട്‌സ്‌പോട്ടിലേക്ക് കണക്ട് ചെയ്ത അവസ്ഥയിൽ കോൺഫറൻസ് ഡെമോ പ്രാക്ടീസ് നടത്തി! അടുത്ത ഘട്ടം, പല ഡെവലപ്പർ പരിതസ്ഥിതികൾ മാനേജുചെയ്യാനും ടീമിനായി ഏകീകൃത ഡെമോ പരിതസ്ഥിതികൾ എങ്ങനെ സൃഷ്ടിക്കാമെന്ന് നോക്കുകയാണ്. ഉപഭോക്താക്കളിൽ നിന്നും കൂട്ടുകാർക്കു നിന്നും കേട്ട വലിയ മറ്റൊരു ഉപയോഗം Dev Box പ്ര_CONFIGured ഡെവലപ്പ്മെന്റ് പരിതസ്ഥിതികളായി ഉപയോഗപ്പെടുത്തലാണ്. ഇത്തരമൊരു MCP ഉപയോഗിച്ച് Dev Box ന്റെ സജ്ജീകരണം വേഗത്തിൽ നാച്ചുറൽ ഭാഷാ ഇടപാടിലൂടെ കൈകാര്യം ചെയ്യാനാകും, നിങ്ങൾ എപ്പോഴും നിങ്ങളുടെ ഡവലപ്പ്മെന്റ് പരിതസ്ഥിതിയിൽ തന്നെ ഇരിക്കുന്നു.

### 9. 🤖 Microsoft Foundry MCP Server

**What it does**: മൈക്രോസോഫ്റ്റ് ഫൗണ്ട്രി MCP സർവർ ഡെവലപ്പർമാർക്ക് ആസ്യൂർ AI എക്കോസിസ്റ്റത്തിൽ സമഗ്രമായ ആക്‌സസ് നൽകുന്നു, മോഡൽ കാറ്റലോഗുകൾ, പ്രയോഗം മാനേജ്മെന്റ്, ആസ്യൂർ AI സെർച്ചിന്റെ ജ്ഞാന ഇൻഡക്സ്, വിലയിരുത്തൽ ഉപകരണങ്ങൾ എന്നിവ ഉൾപ്പെടെ. AI ഡെവലപ്പ്മെന്റ് മൈക്രോസോഫ്റ്റിന്റെ ശക്തമായ AI യന്ത്രവിഭവങ്ങളുമായി ബന്ധിപ്പിക്കുന്ന ഈ പരീക്ഷണ സർവർ AI ആപ്ലിക്കേഷനുകൾ എളുപ്പത്തിൽ നിർമിക്കാൻ, വിനിയോഗിക്കാനും വിലയിരുത്താനും സഹായിക്കുന്നു.

**Why it's useful**: ഈ സർവർ ആസ്യൂർ AI സേവനങ്ങളെ നിങ്ങളുടെ ഡെവലപ്പമെന്റ് പ്രവാഹത്തിലേക്ക് നೇರമായി എത്തിച്ച് എന്റർപ്രൈസ് തലത്തിലുള്ള AI കഴിവുകൾ നേടാൻ സഹായിക്കുന്നു. ആസ്യൂർ പോർട്ടൽ, ഡോക്യുമെന്റേഷൻ, IDE എന്നിവ തമ്മിൽ മാറിവരുന്നതിന് പകരം, മോഡലുകൾ കണ്ടെത്തുകയും സേവനങ്ങൾ വിനിയോഗിക്കുകയും, ജ്ഞാന കേന്ദ്രങ്ങൾ നിയന്ത്രിക്കുകയും, പ്രകൃതിപാട ഊര്‍ജ്ജസ്വലമായ കമാൻഡുകൾ ഉപയോഗിച്ച് AI പ്രകടനം വിലയിരുത്തുകയും ചെയ്യാം. RAG (Retrieval-Augmented Generation) ആപ്ലിക്കേഷനുകൾ നിർമ്മിക്കുന്നവർക്കും, ബഹുമുഖ മോഡൽ വിനിയോഗങ്ങൾ നിയന്ത്രിക്കുന്നവർക്കും, സമഗ്ര AI വിലയിരുത്തൽ പൈപ്പ്‌ലൈൻ നടപ്പിലാക്കുന്നതിന് ശക്തമായതാണ്.

**Key developer capabilities**:
- **🔍 മോഡൽ കണ്ടെത്തൽ & വിനിയോഗം**: മൈക്രോസോഫ്റ്റ് ഫൗണ്ട്രിയുടെ മോഡൽ കാറ്റലോഗ് പര്യവേക്ഷണം ചെയ്യുക, കോഡ് സാമ്പിളുകൾ ഉൾപ്പെടെയുള്ള വിശദമായ മോഡൽ വിവരങ്ങൾ ലഭിക്കുക, മോഡലുകൾ ആസ്യൂർ AI സേവനങ്ങളിൽ വിനിയോഗിക്കുക
- **📚 ജ്ഞാന മാനേജ്മെന്റ്**: ആസ്യൂർ AI സെർച്ചിന്റെ ഇൻഡക്സുകൾ സൃഷ്ടിക്കുകയും, ഡോക്യുമെന്റുകൾ ചേർക്കുകയും, ഇൻഡക്സ് ക്രമീകരണങ്ങൾ നിർവഹിക്കുകയും, സങ്കീർണ്ണ RAG സംവിധാനങ്ങൾ നിർമ്മിക്കുകയും ചെയ്യുക
- **⚡ AI ഏജൻറ് ഇന്റഗ്രേഷൻ**: ആസ്യൂർ AI ഏജൻറുകളുമായി ബന്ധപ്പെടുക, നിലവിലുള്ള ഏജൻറുകളെ ചോദ്യം ചെയ്യുക, പ്രോഡക്ഷൻ സന്നിവേശങ്ങളിൽ ഏജൻറ് പ്രകടനം വിലയിരുത്തുക
- **📊 വിലയിരുത്തൽ ഫ്രെയിംവർക്ക്**: സമഗ്രമായ ടെക്സ്റ്റ്, ഏജൻറ് വിലയിരുത്തലുകൾ നടത്തുക, മാർക്ക്‌ഡൗൺ റിപ്പോർട്ടുകൾ സൃഷ്ടിക്കുക, AI ആപ്ലിക്കേഷനുകളുടെ ഗുണനിലവാരം ഉറപ്പാക്കൽ നടപ്പിലാക്കുക
- **🚀 പ്രോട്ടോടൈപ്പിംഗ് ടൂൾസ്**: GitHub അടിസ്ഥാനമാക്കിയുള്ള പ്രോട്ടോടൈപ്പിങ്ങിനുള്ള സെറ്റപ്പ് നിർദ്ദേശങ്ങൾ ലഭിക്കുക, മൈക്രോസോഫ്റ്റ് ഫൗണ്ട്രി ലാബ്സ് വഴി നൂതന ഗവേഷണ മോഡലുകൾക്ക് ആക്‌സസ് നേടുക

**Featured example**: ഞാൻ ഒരു ഡെവലപ്പർ ആയിട്ടാണ് വിവിധ LLM മോഡലുകളിൽ ഒറ്റനോട്ടം പിടിക്കാൻ ശ്രമിക്കുന്നത്. ചില പ്രധാന മോഡലുകൾ അറിയാമെങ്കിലും, ചില ഉൽപ്പാദനക്ഷമതകളും ഫലപ്രദതയും നഷ്ടപ്പെടുന്നതിനാകാമെന്നാണ് തോന്നുന്നത്. ടോക്കണുകളും ക്വოტകളും മാനേജുചെയ്യുന്നത് തീവ്രമായ സമ്മർദ്ദമാണ് - ശരിയായ ജോലി നിർവഹിക്കാൻ ശരിയായ മോഡൽ തിരഞ്ഞെടുക്കുന്നു എന്ന് എനിക്കറിയാതെ ചിലപ്പോൾ ബജറ്റ് വഴക്കാർന്നുപോകുന്നു. ഈ MCP സർവറിനെ കുറിച്ച് James Montemagno-വിനോട് കേട്ട്, ടീംമേറ്റ്‌സിനൊപ്പം സർവർ പരിചയപ്പെടുത്തൽ അന്വേഷിച്ചതിൽ നിന്ന് ഇത് എനിക്ക് ഉപകാരപ്പെടുമെന്ന് നിജമാണ്! മോഡൽ കണ്ടെത്തൽ കഴിവുകൾ പ്രത്യേകിച്ച് ഏറ്റവും പ്രചാരമില്ലാത്ത മോഡലുകൾ ആവിശ്യങ്ങളനുസരിച്ച് കണ്ടെത്താൻ സഹായിക്കുന്നവർക്കായി വളരെ ഗുണകരമായിരിക്കും. വിലയിരുത്തൽ ഫ്രെയിംവർക്ക് എനിക്ക് യാഥാർത്ഥത്തിൽ നല്ല ഫലം ലഭിക്കുന്നുണ്ടോ എന്നാണ് ഉറപ്പാക്കാൻ സഹായിക്കും, വെറും പുതിയതിനു വേണ്ടി പരീക്ഷിക്കുന്നതല്ലെന്ന്.

> **ℹ️ പരീക്ഷണ നില**
> 
> ഈ MCP സർവർ പരീക്ഷണാത്മകവും സജീവ വികസനത്തിനും കീഴിൽ ഉള്ളതുമാണ്. ഫീച്ചറുകളും API-കളും മാറാൻ സാധ്യതയുണ്ട്. ആസ്യൂർ AI കഴിവുകൾ പര്യവേക്ഷിക്കാൻ, പ്രോട്ടോട്ടൈപ്പ് നിർമ്മിക്കാൻ ഇത് അനുയോജ്യമാണ്, എന്നാൽ പ്രൊഡക്ഷൻ ഉപയോഗത്തിനായി സ്റ്റേബിലിറ്റിയുടെ ആവശ്യകതകൾ പരിശോധിക്കുക.
### 10. 🏢 Microsoft 365 ഏജൻറ്സ് ടൂള്കിറ്റ് MCP സർവർ

[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_M365_Agents_Toolkit-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=M365AgentsToolkit%20Server&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22@microsoft%2Fm365agentstoolkit-mcp%40latest%22%2C%22server%22%2C%22start%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_M365_Agents_Toolkit-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=M365AgentsToolkit%20Server&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22@microsoft%2Fm365agentstoolkit-mcp%40latest%22%2C%22server%22%2C%22start%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/OfficeDev/microsoft-365-agents-toolkit)

**What it does**: മൈക്രോസോഫ്റ്റ് 365, മൈക്രോസോഫ്റ്റ് 365 കോപൈലറ്റുമായി ഇന്റഗ്രേറ്റ് ചെയ്യാവുന്ന AI ഏജൻറുകളും ആപ്ലിക്കേഷനുകളും നിർമ്മിക്കുന്നതിന് അനിവാര്യമായ ഉപകരണങ്ങൾ ഡെവലപ്പർമാർക്ക് നൽകുന്നു, സ്കീമാ വെരിഫിക്കേഷൻ, സാമ്പിൾ കോഡ് ലഭ്യത, ട്രബിൾഷൂട്ടിംഗ് സഹായം തുടങ്ങി.

**Why it's useful**: മൈക്രോസോഫ്റ്റ് 365, കോപൈലറ്റ് ഡെവലപ്പ്മെന്റ് സങ്കീർണ്ണമായ മാനിഫെസ്റ്റ് സ്കീമകളും പ്രത്യേക ഡവലപ്പ്മെന്റ് മാതൃകകളും ഉൾക്കൊള്ളുന്നു. ഈ MCP സർവർ കോഡിംഗ് പരിസരത്തേയ്ക്ക് തന്നെ അനിവാര്യ ഡെവലപ്പ്മെന്റ് ഉപകരണങ്ങൾ കൊണ്ട് എത്തിച്ച്, നിങ്ങൾക്ക് സ്കീമകൾ സാധുതയുള്ളതെന്ന് ഉറപ്പാക്കാനും, സാമ്പിൾ കോഡ് കണ്ടെത്താനും, സാധാരണ പ്രശ്നങ്ങൾ പരിഹരിക്കാനും സഹായിക്കുന്നു.

**Real-world use**: "എന്റെ ഡിക്ലാരേറ്റീവ് ഏജൻറ് മാനിഫെസ്റ്റ് സാധുത പരിശോധിച്ച് സ്കീമാ പിശകുകൾ മാറ്റുക", "Microsoft Graph API പ്ലഗിൻ സഹായത്തിനുള്ള സാമ്പിൾ കോഡ് കാണിക്കുക", "എന്റെ ടീംസിന്റെ ആപ്പ് ഓഥന്റിക്കേഷൻ പ്രശ്നങ്ങൾ ട്രബിൾഷൂട്ടുചെയ്യാൻ സഹായിക്കുക"

**Featured example**: Build മീറ്റിംഗിൽ എൻ്റെ സുഹൃത്തായ John Miller-നെ സമീപിച്ചതിനു ശേഷം, M365 ഏജൻറുകൾക്കായി ഈ MCP ഞാൻ കണ്ടുപിടിച്ചു. ഡെവലപ്പർമാർക്കായി ഇത് വളരെ സഹായകമാണ്, പ്രത്യേകിച്ച് M365 ഏജൻറുകളിൽ പുതിയവർക്കായി, കാരണം ടെംപ്ലേറ്റുകൾ, സാമ്പിൾ കോഡ്, സ്കാഫോൾഡിങ്ങ് എന്നിവ ഇത് സൗകര്യമാകും, ഡോക്യുമെന്റേഷൻ കൂട്ടമായിരിക്കാൻ ആവശ്യമില്ലാതെ. മാനിഫെസ്റ്റ് ഘടനാ പിഴവുകൾ അവോയ്‌ഡ് ചെയ്യുന്നതിന് സ്കീമാ വെരിഫിക്കേഷൻ ഫീച്ചറുകൾ ഏറെ പ്രയോജനപ്രദമാണെന്ന് കാണുന്നു, കാരണം അതവർക്ക് മണിക്കൂറുകൾ നീണ്ട ബഗ് തിരയൽ ഒഴിവാക്കാം.

## 📋 ഉപസംഹാരം

Model Context Protocol (MCP) ഡെവലപ്പർമാർ എങ്ങനെ AI സഹായി, ബാഹ്യ ഉപകരണങ്ങൾ എന്നിവയെ സംയോജിപ്പിക്കുന്നു എന്നത് മാറ്റുന്നുണ്ട്. ഈ 10 മൈക്രോസോഫ്റ്റ് MCP സർവർ AI ഇന്റഗ്രേഷൻക്ക് സ്റ്റാൻഡേർഡൈസ്ഡ് കരാർ എങ്ങനെ ശക്തമായ പ്രവാഹങ്ങൾക്ക് സാധ്യത നൽകുന്നു എന്ന് തെളിയിക്കുന്നു, ശക്തമായ ബാഹ്യ കഴിവുകൾ എക്‌സസ് ചെയ്യുമ്പോഴും ഡെവലപ്പർമാരെ ഫ്ലോാവില്‍ സൂക്ഷിക്കുന്നു.

ആസ്യൂർ ഇക്കോസിസ്റ്റം സംയോജനം മുതൽ Playwright പോലുള്ള ബ്രൗസർ ഓട്ടോമേഷൻ ഉപകരണങ്ങൾ, MarkItDown പോലുള്ള ഡോക്യുമെന്റ് പ്രോസസിംഗ് ടൂളുകൾ വരെ, ഈ സർവർകൾ MCP എങ്ങനെ مختلف ഡെവലപ്പ്മെന്റ് സാഹചര്യങ്ങളിൽ ഉത്പാദകത്വം വർദ്ധിപ്പിക്കുന്നു എന്ന് കാണിക്കുന്നു. സ്റ്റാൻഡേർഡൈസ്ഡ് പ്രോട്ടോക്കോൾ ഈ ഉപകരണങ്ങൾ ട്രൂട്ടിയായി മൃദുവായി ചേർന്ന്, ഒറ്റത്തവണമുള്ള ഡെവലപ്പ്മെന്റ് അനുഭവം സൃഷ്ടിക്കുന്നു.

MCP എക്കോസിസ്റ്റം വികസനത്തിലിറങ്ങിയിരിക്കുന്നതിനാൽ, കമ്മ്യൂണിറ്റിയുമായി ബന്ധം സൂക്ഷിച്ചുകൊണ്ട്, പുതിയ സർവർ പിന്തുടരുകയും, കസ്റ്റം പരിഹാരങ്ങൾ നിർമ്മിക്കുകയും ചെയ്യുക ഡെവലപ്പ്മെന്റ് ഉത്പാദകത്വം വളർത്താൻ നിർണായകം. MCP ഓപ്പൺ സ്റ്റാൻഡേർഡ് ആയതിനാൽ വിവിധ വെണ്ടർമാരിൽ നിന്നുള്ള ഉപകരണങ്ങൾ സംയോജിപ്പിച്ച് നിങ്ങളുടെ പ്രത്യേക ആവശ്യങ്ങൾക്ക് പരിപൂണമായ പ്രവാഹം സൃഷ്ടിക്കാൻ കഴിയും.

4. **പ്രകടന വിശകലനം**: MCP സർവറുകൾ ഉപയോഗിക്കൽ സാങ്കേതിക പ്രശ്നങ്ങളിൽ സാധാരണ രീതികളുമായി താരതമ്യം ചെയ്തു പ്രവർത്തനക്ഷമത വിലയിരുത്തുക.
5. **സുരക്ഷാ വിലയിരുത്തൽ**: MCP സർവറുകളുടെ ഉപയോഗത്തിന്റെ സുരക്ഷാ പ്രഭാവങ്ങൾ വിലയിരുത്തി മികച്ച പ്രവൃത്തിപരമായ നിശ്ചയങ്ങൾ നിർദ്ദേശിക്കുക.
