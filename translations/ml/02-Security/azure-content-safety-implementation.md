# MCP-യോടുള്ള Azure കോൺടെന്റ് സുരക്ഷ നടപ്പിലാക്കൽ

> **OWASP MCP റിസ്ക് പരിഹരിച്ചതായി**: [MCP06 - Intent Flow Subversion](https://microsoft.github.io/mcp-azure-security-guide/mcp/mcp06-prompt-injection/)

പ്രോസസ്സിംഗ് പൈപ്പ്ലൈനിൽ Azure കോൺറന്റ് സുരക്ഷാ ഫിൽറ്റർ മിഡിൽവെയർ ആക്കിവെച്ച് MCP prompt injection, ടൂൾ വിഷം, മറ്റ് AI-സവിശേഷ വിഭിന്നതകൾക്കെതിരായ സുരക്ഷ ശക്തിപ്പെടുത്താൻ ഇത് ഏറെ നിർദ്ദേശിക്കപ്പെട്ടതാണ്. ഈ നടപ്പാക്കൽ മാർഗ്ഗനിർദ്ദേശം [MCP Security Summit Workshop (Sherpa)](https://azure-samples.github.io/sherpa/) ക്യാമ്പ് 3: I/O Security-യുമായി പൊരുത്തപ്പെടുന്നു.

## MCP സെർവർ ഇന്റഗ്രേഷൻ

Azure കോൺടെന്റ് സുരക്ഷ MCP സെർവറുമായി ഇന്റഗ്രേറ്റ് ചെയ്യാൻ, നിങ്ങളുടെ അഭ്യർത്ഥന പ്രോസസ്സിംഗ് പൈപ്പ്ലൈനിൽ കോൺടെന്റ് സുരക്ഷാ ഫിൽറ്റർ മിഡിൽവെയർ ആയി ചേർക്കുക:

1. സെർവർ സ്റ്റാർട്ടപ്പിൽ ഫിൽറ്റർ ഇൻഷ്യലൈസ് ചെയ്യുക  
2. പ്രോസസ്സിങ്ങിനുമുൻപ് എല്ലാ അംഗമാകുന്ന ടൂൾ അഭ്യർത്ഥനകളും പരിശോധിക്കുക  
3. ക്ളയന്റുകളിൽ വിടുന്ന മുമ്പ് എല്ലാ പരിഹാരങ്ങളും പരിശോധിക്കുക  
4. സുരക്ഷ ലംഘനങ്ങൾ ഉണ്ടാകുമ്പോൾ ലോഗ് ചെയ്യുകയും അലർട്ട് ചെയ്യുകയും ചെയ്യുക  
5. കോൺടെന്റ് സുരക്ഷാ പരിശോധനകളിൽ പരാജയപ്പെട്ടപ്പോൾ ഉചിതമായ പിശക് കൈകാര്യം നടപ്പിലാക്കുക  

ഇത് ശക്തമായ സംരക്ഷണം നൽകുന്നു:  
- പ്രോംപ്റ്റ് ഇൻജക്ഷൻ ആക്രമണങ്ങൾ  
- ടൂൾ വിഷം ശ്രമങ്ങൾ  
- ദുഷ്പ്രവർത്തനമായ ഇൻപുട്ടുകൾ വഴി ഡേറ്റാ പുറത്തുവിടൽ  
- ഹാനികരമായ ഉള്ളടക്കം സൃഷ്ടിക്കൽ  

## Azure കോൺടെന്റ് സുരക്ഷ ഇന്റഗ്രേഷനിൽ മികച്ച പ്രയോഗങ്ങൾ

1. **കസ്റ്റം ബ്ലോക്ക്ലിസ്റ്റുകൾ**: MCP ഇൻജക്ഷൻ മാതൃകകൾക്കായി പ്രത്യേകമായി കസ്റ്റം ബ്ലോക്ക്ലിസ്റ്റുകൾ സൃഷ്ടിക്കുക  
2. **തീവ്രത ക്രമീകരണം**: നിങ്ങളുടെ ഉപയോഗ കാര്യങ്ങൾക്കും റിസ്ക് ടോളറൻസ് അടിസ്ഥാനമാക്കി തീവ്രത പരിധികൾ ക്രമീകരിക്കുക  
3. **പരിപൂർണ കവറേജ്**: എല്ലാ ഇൻപുട്ടുകൾക്കും ഔട്ട്പുട്ടുകൾക്കും കോൺടെന്റ് സുരക്ഷ പരിശോധനകൾ പ്രയോഗിക്കുക  
4. **പ്രകടന മെച്ചപ്പെടുത്തൽ**: ആവർത്തിക്കുന്ന കോൺറന്റ് സുരക്ഷ പരിശോധനകൾക്കായി ക്യാഷിംഗ് നടപ്പിലാക്കാൻ പരിഗണിക്കുക  
5. **ഫാൾബാക് സജ്ജീകരണങ്ങൾ**: കോൺറന്റ് സുരക്ഷാ സേവനങ്ങൾ ലഭ്യമല്ലാതിരിക്കുമ്പോൾ വ്യക്തമായ ഫാൾബാക് പെരുമാറ്റങ്ങൾ നിർവചിക്കുക  
6. **ഉപയോ​ഗ​​ത്താക്കൾക്ക് പ്രതികരണം**: സുരക്ഷാ കാരണങ്ങളാൽ ഉള്ളടക്കം തടയപ്പെട്ടപ്പോൾ വ്യക്തമായ ഫീഡ്ബാക്ക് നൽകുക  
7. **തുടർച്ചയായ മെച്ചപ്പെടുത്തൽ**: പുതിയ ഭീഷണികളെ ആശ്രയിച്ച് ബ്ലോക്ക്ലിസ്റ്റുകളും മാതൃകകളും ഇടക്കിടെ പുതുക്കുക
