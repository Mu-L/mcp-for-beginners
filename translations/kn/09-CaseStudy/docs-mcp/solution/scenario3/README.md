# ದೃಶ್ಯಾವಳಿ 3: VS ಕೋಡ್‌ನಲ್ಲಿನ MCP ಸರ್ವರ್ ಜೊತೆಗೆ ಎಡಿಟರ್ ಒಳಗಿನ ಡಾಕ್ಸ್

## ಅವಲೋಕನ

ಈ ದೃಶ್ಯಾವಳಿಯಲ್ಲಿ, ನೀವು Microsoft Learn Docs ಅನ್ನು ನೇರವಾಗಿ ನಿಮ್ಮ Visual Studio Code ಪರಿಸರಕ್ಕೆ MCP ಸರ್ವರ್ ಬಳಸಿ ಹೇಗೆ ತಂದೊಯ್ಯುವುದು ಎಂದು ತಿಳಿಯುತ್ತೀರಿ. ದಸ್ತಾವೇಜೆಯನ್ನು ಹುಡುಕಲು ಬ್ರೌಸರ್ ಟ್ಯಾಬ್‌ಗಳನ್ನು ನಿರಂತರವಾಗಿ ಬದಲಾಯಿಸುವ ಬದಲು, ನೀವು ನಿಮ್ಮ ಎಡಿಟರ್ ಒಳಗೆಯೇ ಅಧಿಕೃತ ಡಾಕ್ಯೂಮೆಂಟ್‌ಗಳನ್ನು ಪ್ರವೆಶ ಮಾಡಿ, ಹುಡುಕಿ ಮತ್ತು ಉಲ್ಲೇಖಿಸಬಹುದು. ಈ ವಿಧಾನ ನಿಮ್ಮ ಕೆಲಸದ ಹರಿವನ್ನು ಸರಳಗೊಳಿಸುತ್ತದೆ, ನಿಮ್ಮ ದೃಷ್ಟಿಯನ್ನು ಕೇಂದ್ರೀಕರಿಸುತ್ತದೆ, ಮತ್ತು GitHub Copilot ಅಂತಹ ಸಾಧನಗಳೊಂದಿಗೆ ನಿರಂತರ ಸಂಯೋಜನೆಯನ್ನು ಸಾದ್ಯಮಾಡುತ್ತದೆ.

- VS Code ಒಳಗೇ ಡಾಕ್ಸ್ ಹುಡುಕಿ ಮತ್ತು ಓದಿ, ನಿಮ್ಮ ಕೋಡಿಂಗ್ ಪರಿಸರವನ್ನು ತೊರೆದೆ ಇಲ್ಲದೆ.
- ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಅನ್ನು ಉಲ್ಲೇಖಿಸಿ ಮತ್ತು ಲಿಂಕ್‌ಗಳನ್ನು ನೇರವಾಗಿ ನಿಮ್ಮ README ಅಥವಾ ಕೋರ್ಸ್ ಫೈಲ್‌ಗಳಲ್ಲಿ ಸೇರಿಸಿ.
- GitHub Copilot ಮತ್ತು MCP ಅನ್ನು ಜೊತೆಗೆ ಬಳಸಿಕೊಂಡು ನಿರಂತರ, AI ಚಾಲಿತ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಕಾರ್ಯಪ್ರವಾಹವನ್ನು ಅನುಭವಿಸಿ.

## ಕಲಿಕ transient ಗುರಿಗಳು

ಈ ಅಧ್ಯಾಯದ ಕೊನೆಯಲ್ಲಿ, ನೀವು VS Code ಒಳಗೆ MCP ಸರ್ವರ್‌ನ್ನು ಸೆಟ್ ಅಪ್ ಮಾಡಿ, ನಿಮ್ಮ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಮತ್ತು ಅಭಿವೃದ್ಧಿ ಕಾರ್ಯಪ್ರವಾಹವನ್ನು ಸುಧಾರಿಸಲು ಹೇಗೆ ಬಳಸಬೇಕು ಎಂದು ತಿಳಿದುಕೊಳ್ಳುತ್ತೀರಿ. ನೀವು ಮುಂದೆ ಈ ಕಾರ್ಯಗಳನ್ನು ಮಾಡಲು ಸಿದ್ಧರಾಗುತ್ತೀರಿ:

- ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಹುಡುಕಲು MCP ಸರ್ವರ್ ಬಳಕೆಗೆ ನಿಮ್ಮ ಕಾರ್ಯಕ್ಷೇತ್ರವನ್ನು ಹೊಂದಿಸಿಕೊಳ್ಳಿ.
- VS Code ಒಳಗಿಂದಲೇ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಹುಡುಕಿ ಮತ್ತು ಸೇರಿಸಿ.
- ಹೆಚ್ಚು ಉತ್ಪಾದಕತೆ, AI-ಸಹಾಯಿತ ಕಾರ್ಯಪ್ರವಾಹಕ್ಕಾಗಿ GitHub Copilot ಮತ್ತು MCP ಶಕ್ತಿಗಳನ್ನು ಸಂಯೋಜಿಸಿ.

ಈ ಕೌಶಲ್ಯಗಳು ನಿಮಗೆ ಕೇಂದ್ರೀಕೃತವಾಗಿಯೇ ಇರುವುದು, ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಗುಣಮಟ್ಟವನ್ನು ಸುಧಾರಿಸುವುದು ಮತ್ತು ಅಭಿವೃದ್ಧಿ ಅಥವಾ ತಾಂತ್ರಿಕ ಲೇಖಕರಾಗಿ ನಿಮ್ಮ ಉತ್ಪಾದಕತೆಯನ್ನು ವೃದ್ಧಿಸಿಕೊಳ್ಳುವಲ್ಲಿ ಸಹಾಯ ಮಾಡುತ್ತವೆ.

## ಪರಿಹಾರ

ಎಡಿಟರ್ ಒಳಗಿನ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಪ್ರವೇಶವನ್ನು ಸಾಧಿಸಲು, ನೀವು MCP ಸರ್ವರ್ ಅನ್ನು VS Code ಮತ್ತು GitHub Copilot ಜೊತೆಗೆ ಸಂಯೋಜಿಸುವ ಕ್ರಮಗಳನ್ನು ಅನುಸರಿಸುವಿರಿ. ಈ ಪರಿಹಾರವು ಕೋರ್ಸ್ ಲೇಖಕರು, ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಬರಹಗಾರರು ಮತ್ತು ಡೆವಲಪರ್‌ಗಳಿಗೆ ತಕ್ಕದು, ಯಾರು ಎಡಿಟರ್‌ನಲ್ಲಿ ಗಮನವಿಟ್ಟು ಇದ್ದು ಡಾಕ್ಸ್ ಮತ್ತು Copilot ಜೊತೆಗೆ ಕೆಲಸ ಮಾಡಲು ಬಯಸುತ್ತಾರೆ.

- ಕೋರ್ಸ್ ಅಥವಾ ಯೋಜನೆ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಬರೆಯುತ್ತಿರುವಾಗ README ಗೆ ಉಲ್ಲೇಖ ಲಿಂಕ್‌ಗಳನ್ನು ತ್ವರಿತವಾಗಿ ಸೇರಿಸಿ.
- ಕೋಡ್ ರಚಿಸಲು Copilot ಬಳಸಿ, ಮತ್ತು MCP ಬಳಸಿ ತಕ್ಷಣ ಸಂಬಂಧಿತ ಡಾಕ್ಸ್ ಹುಡುಕಿ ಉಲ್ಲೇಖಿಸಿ.
- ನಿಮ್ಮ ಎಡಿಟರ್‌ನಲ್ಲಿ ಕೇಂದ್ರೀಕೃತವಾಗಿರಿ ಮತ್ತು ಉತ್ಪಾದಕತೆಯನ್ನು ಹೆಚ್ಚಿಸಿಕೊಳ್ಳಿ.

### ಹಂತ-ಹಂತದ ಮಾರ್ಗದರ್ಶಿಕೆ

ಪ್ರಾರಂಭಿಸಲು, ಈ ಹಂತಗಳನ್ನು ಅನುಸರಿಸಿ. ಪ್ರತಿ ಹಂತಕ್ಕೆ, ಪ್ರಕ್ರಿಯೆಯನ್ನು ದೃಶ್ಯಾತ್ಮಕವಾಗಿ ವಿವರಿಸಲು assets ಫೋಲ್ಡರ್‌ನಿಂದ ಸ್ಕ್ರೀನ್ಶಾಟ್ ಸೇರಿಸಬಹುದು.

1. **MCP ಕಾನ್ಫಿಗರೇಶನ್ ಸೇರಿಸಿ:**
   ನಿಮ್ಮ ಯೋಜನೆಯ ಮೂಲದಲ್ಲಿ `.vscode/mcp.json` ಫೈಲ್ ರಚಿಸಿ ಮತ್ತು ಕೆಳಗಿನ ಕಾನ್ಫಿಗರೇಶನ್ ಸೇರಿಸಿ:
   ```json
   {
     "servers": {
       "LearnDocsMCP": {
         "url": "https://learn.microsoft.com/api/mcp"
       }
     }
   }
   ```
   ಈ ಕಾನ್ಫಿಗರೇಶನ್ VS Codeಗೆ [`Microsoft Learn Docs MCP ಸರ್ವರ್`](https://github.com/MicrosoftDocs/mcp) ಗೆ ಹೇಗೆ ಸಂಪರ್ಕಿಸಬೇಕೆಂದು ಸೂಚಿಸುತ್ತದೆ.
   
   ![ಹಂತ 1: mcp.json ಅನ್ನು .vscode ಫೋಲ್ಡರ್‌ಗೆ ಸೇರಿಸಿ](../../../../../../translated_images/kn/step1-mcp-json.c06a007fccc3edfa.webp)
    
2. **GitHub Copilot ಚಾಟ್ ಪ್ಯಾನೆಲ್ ತೆರೆಯಿರಿ:**
   ನೀವು GitHub Copilot ವಿಸ್ತರಣೆ ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಿಲ್ಲದಿದ್ದರೆ, VS Code ನಲ್ಲಿ Extensions ವೀಕ್ಷಣೆಗೆ ಹೋಗಿ ಅದನ್ನು ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಿ. ನೇರವಾಗಿ [Visual Studio Code Marketplace](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot-chat) ನಿಂದ ಡೌನ್‌ಲೋಡ್ ಮಾಡಬಹುದು. ನಂತರ, ಸೈಡ್‌ಬಾರ್‌ನಿಂದ Copilot ಚಾಟ್ ಪ್ಯಾನೆಲ್ ತೆರೆಯಿರಿ.

   ![ಹಂತ 2: Copilot Chat ಪ್ಯಾನೆಲ್ ತೆರೆಯಿರಿ](../../../../../../translated_images/kn/step2-copilot-panel.f1cc86e9b9b8cd1a.webp)

3. **ಏಜೆಂಟ್ ಮೋಡ್ ಸಕ್ರಿಯಮಾಡಿ ಮತ್ತು ಸಾಧನಗಳನ್ನು ಪರಿಶೀಲಿಸಿ:**
   Copilot ಚಾಟ್ ಪ್ಯಾನೆಲ್‌ನಲ್ಲಿ ಏಜೆಂಟ್ ಮೋಡ್ ಸಕ್ರಿಯಮಾಡಿ.

   ![ಹಂತ 3: ಏಜೆಂಟ್ ಮೋಡ್ ಸಕ್ರಿಯ ಮಾಡಿ ಮತ್ತು ಸಾಧನಗಳನ್ನು ಪರಿಶೀಲಿಸಿ](../../../../../../translated_images/kn/step3-agent-mode.cdc32520fd7dd1d1.webp)

   ಏಜೆಂಟ್ ಮೋಡ್ ಸಕ್ರಿಯ ಮಾಡಿದ ನಂತರ, MCP ಸರ್ವರ್ ಲಭ್ಯವಿರುವ ಸಾಧನಗಳಲ್ಲಿ ಒಂದೆಯೆಂದು ಪರಿಶೀಲಿಸಿ. ಇದರಿಂದ Copilot ಏಜೆಂಟ್ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಸರ್ವರ್ ಅನ್ನು ಪ್ರವೇಶಿಸಲು ಸಾಧ್ಯವಾಗುತ್ತದೆ.
   
   ![ಹಂತ 3: MCP ಸರ್ವರ್ ಸಾಧನ ಪರಿಶೀಲಿಸಿ](../../../../../../translated_images/kn/step3-verify-mcp-tool.76096a6329cbfecd.webp)
4. **ಹೊಸ ಚಾಟ್ ಆರಂಭಿಸಿ ಮತ್ತು ಏಜೆಂಟ್ ಅನ್ನು ಪ್ರಶ್ನಿಸಿ:**
   Copilot ಚಾಟ್ ಪ್ಯಾನೆಲ್‌ನಲ್ಲಿ ಹೊಸ ಚಾಟ್ ತೆರೆಯಿರಿ. ಈಗ ನೀವು ನಿಮ್ಮ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಪ್ರಶ್ನೆಗಳನ್ನು ಏಜೆಂಟ್‌ಗೆ ಕೇಳಬಹುದು. ಏಜೆಂಟ್ MCP ಸರ್ವರ್ ಬಳಸಿ ಸಂಬಂಧಿಸಿದ Microsoft Learn ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಅನ್ನು ತಂದು ನಿಮ್ಮ ಎಡಿಟರ್‌ನಲ್ಲಿ ಪ್ರದರ್ಶಿಸುವುದು.

   - *"ನಾನು X ವಿಷಯದ ಅಧ್ಯಯನ ಯೋಜನೆಯನ್ನು ಬರೆಯಲು ಪ್ರಯತ್ನಿಸುತ್ತಿದ್ದೇನೆ. ನಾನು ಇದನ್ನು 8 ವಾರಗಳ ಕಾಲ ಅಧ್ಯಯನ ಮಾಡಲಿದ್ದೇನೆ, ಪ್ರತಿಯೊಂದು ವಾರಿಗೆ, ಯಾವ ವಿಷಯವನ್ನು ತೆಗೆದುಕೊಳ್ಳಬೇಕು ಎಂದು ಸಲಹೆ ನೀಡಿ."*

   ![ಹಂತ 4: ಚಾಟ್‌ನಲ್ಲಿ ಏಜೆಂಟ್‌ಗೆ ಪ್ರಶ್ನೆಯನ್ನು ಕೇಳಿ](../../../../../../translated_images/kn/step4-prompt-chat.12187bb001605efc.webp)

Copilot Studio ಚಾನೆಲ್‌ಗಳಂತೆ ನೇರ ನಿಯೋಜನೆಯ ವಿಧಾನ ಇಲ್ಲ ಎಂದು ನಾನು ನೋಡುತ್ತಿದ್ದೇನೆ. ಆದ್ದರಿಂದ, ಈ ನಿಯೋಜನೆಯನ್ನು ಎಂಟರ್‌ಪ್ರೈಸ್ ಬಳಕೆದಾರರು ಪರಸ್ಪರ ಇಂಟರಾಕ್ಟ್ ಮಾಡು ಮತ್ತು ಕಾರ್ಯವನ್ನು ಮುಗಿಸಲು ವಿವಿಧ ವಿಧಾನಗಳು ಯಾವುವು?

### ಮಾದರಿ ಪ್ರಶ್ನೆಗಳು

ನೀವು ಪ್ರಯತ್ನಿಸಬಹುದಾದ ಕೆಲವು ಮಾದರಿ ಪ್ರಶ್ನೆಗಳು ಇಲ್ಲಿವೆ. ಈ ಪ್ರಶ್ನೆಗಳು MCP ಸರ್ವರ್ ಮತ್ತು Copilot‍ನ ಜೊತೆಗೆ ನೇರ, ಸ೦ದರ್ಭೋಚಿತ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಮತ್ತು ಉಲ್ಲೇಖಗಳನ್ನು VS Code ಬಿಟ್ಟು ಹೊರಹೋಗದೆ ಒದಗಿಸುವ ವಿಧಾನವನ್ನು ತೋರಿಸುತ್ತವೆ:

- "Azure Functions ಟ್ರಿಗರ್‌ಗಳು ಹೇಗೆ ಬಳಸಬೇಕು ಎಂದು ತೋರಿಸಿ."
