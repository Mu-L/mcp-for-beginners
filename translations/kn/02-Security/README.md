# MCP ಭದ್ರತೆ: AI ವ್ಯವಸ್ಥೆಗಳಿಗಾಗಿ ಸಮಗ್ರ ರಕ್ಷಣೆ

[![MCP Security Best Practices](../../../translated_images/kn/03.175aed6dedae133f.webp)](https://youtu.be/88No8pw706o)

_(ಈ ಪಾಠದ ವೀಡಿಯೊವನ್ನು ನೋಡಲು ಮೇಲಿನ ಚಿತ್ರದ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ)_

ಭದ್ರತೆ AI ವ್ಯವಸ್ಥೆಯ ವಿನ್ಯಾಸಕ್ಕೆ ಮೂಲಭೂತವಾಗಿದೆ, ಹಾಗೆಯೇ ನಾವು ಇದನ್ನು ನಮ್ಮ ಎರಡನೇ ವಿಭಾಗವೆಂದಾಗಿ ಪ್ರಮುಖತೆ ನೀಡುತ್ತೇವೆ. ಇದು Microsoft-ನ **Secure by Design** ತತ್ವದೊಂದಿಗೆ ಹೊಂದಿಕೊಳ್ಳುತ್ತದೆ, ಇದು [Secure Future Initiative](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) ಯಿಂದ ಬಂದಿದೆ.

ಮಾದರಿ ಸಾಂದರ್ಭಿಕ ಪ್ರೋಟೋಕಾಲ್ (MCP) ಸಶಕ್ತ ಹೊಸ ಸಾಮರ್ಥ್ಯಗಳನ್ನು AI ಚಾಲಿತ ಅನ್ವಯಿಕೆಗೆ ತರಲಿದೆ, ಆದರೆ ಸಾಂಪ್ರದಾಯಿಕ ಸಾಫ್ಟ್‌ವೇರ್ ಅಪಾಯಗಳನ್ನು ಮೀರಿದ ವಿಭಿನ್ನ ಭದ್ರತಾ ಸವಾಲುಗಳನ್ನು ಪರಿಚಯಿಸುತ್ತದೆ. MCP ವ್ಯವಸ್ಥೆಗಳು ಸ್ಥಾಪಿತ ಭದ್ರತಾ ಸಮಸ್ಯೆಗಳನ್ನು (ಸುರಕ್ಷಿತ ಕೋಡಿಂಗ್, ಕನಿಷ್ಟ ಆಧಿಕಾರ, ಪೂರೈಕೆ ಸರಪಳಿ ಭದ್ರತೆ) ಮತ್ತು ಹೊಸ AI-ನಿರ್ದಿಷ್ಟ ಅಪಾಯಗಳನ್ನು ಎದುರಿಸುತ್ತವೆ, ಅವುಗಳಲ್ಲಿ ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್, ಉಪಕರಣ ವಿಷಾಕರಣ, ಸೆಷನ್ ಹೈಜ್ಯಾಕಿಂಗ್, ಗೊಂದಲವಾದ ಡೆಪ್ಯೂಟಿ ದಾಳಿ, ಟೋಕನ್ ಪಾಸ್‌ಥ್ರು ದುರ್ಬಲತೆಗಳು ಮತ್ತು ಡೈನಾಮಿಕ್ ಸಾಮರ್ಥ್ಯ ಪರಿಷ್ಕರಣೆ ಸೇರಿವೆ.

ಈ ಪಾಠವು MCP ಅನ್ವಯಿಕೆಗಳಲ್ಲಿ ಅತಿ ಗಂಭೀರ ಭದ್ರತಾ ಅಪಾಯಗಳನ್ನು ವಿಶ್ಲೇಷಿಸುತ್ತದೆ—ಪ್ರಮಾಣೀಕರಣ, ಪ್ರಾಧಿಕಾರ, ಅತಿಕ್ರಮಿತ ಅನುಮತಿಗಳು, ಅಪ್ರತಕ್ಷ ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್, ಸೆಷನ್ ಭದ್ರತೆ, ಗೊಂದಲವಾದ ಡೆಪ್ಯೂಟಿ ಸಮಸ್ಯೆಗಳು, ಟೋಕನ್ ನಿರ್ವಹಣೆ ಮತ್ತು ಪೂರೈಕೆ ಸರಪಳಿ ದುರ್ಬಲತೆಗಳನ್ನು ಒಳಗೊಂಡು. ನೀವು ಕ್ರಿಯಾತ್ಮಕ ನಿಯಂತ್ರಣಗಳು ಮತ್ತು ಉತ್ತಮ ಪದ್ಧತಿಗಳನ್ನು ಕಲಿತುಕೊಳ್ಳುತ್ತೀರಿ, ಜೊತೆಗೆ Microsoft ಪರಿಹಾರಗಳನ್ನು ಬಳಸಿಕೊಂಡು ನಿಮ್ಮ MCP ನಿಯೋಜನೆ ಶಕ್ತಿಶಾಲಿಯಾಗಿಸುತ್ತದೆ, ಅವುಗಳಲ್ಲಿ Prompt Shields, Azure Content Safety ಮತ್ತು GitHub Advanced Security ಮುಂತಾದವುಗಳಿವೆ.

## ಕಲಿಕೆ ಉದ್ದೇಶಗಳು

ಈ ಪಾಠದ ಅಂತ್ಯಕ್ಕೆ ನೀವು:

- **MCP-ನಿರ್ದಿಷ್ಟ ಅಪಾಯಗಳನ್ನು ಗುರುತಿಸು**: MCP ವ್ಯವಸ್ಥೆಗಳಲ್ಲಿ ವಿಶಿಷ್ಟ ಭದ್ರತಾ ಅಪಾಯಗಳನ್ನು ಗುರುತಿಸಿ; ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್, ಉಪಕರಣ ವಿಷಾಕರಣ, ಅತಿಕ್ರಮಿತ ಅನುಮತಿಗಳು, ಸೆಷನ್ ಹೈಜ್ಯಾಕಿಂಗ್, ಗೊಂದಲವಾದ ಡೆಪ್ಯೂಟಿ ಸಮಸ್ಯೆಗಳು, ಟೋಕನ್ ಪಾಸ್‌ಥ್ರು ದುರ್ಬಲತೆಗಳು ಮತ್ತು ಪೂರೈಕೆ ಸರಪಳಿ ಅಪಾಯಗಳು ಸೇರಿ
- **ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳನ್ನು ಅನ್ವಯಿಸು**: ಬಲವಾದ ಪ್ರಮಾಣೀಕರಣ, ಕನಿಷ್ಟ ಆಧಿಕಾರ ಹಕ್ಕು, ಸುಸ್ಥಿರ ಟೋಕನ್ ನಿರ್ವಹಣೆ, ಸೆಷನ್ ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳು ಮತ್ತು ಪೂರೈಕೆ ಸರಪಳಿ ದೃಢೀಕರಣ ಸೇರಿದಂತೆ ಪರಿಣಾಮಕಾರಿ ತಡೆಗಟ್ಟುವಿಕೆಗಳನ್ನು ಜಾರಿಗೆ ತರುವುದನ್ನು
- **Microsoft ಭದ್ರತಾ ಪರಿಹಾರಗಳನ್ನು ಉಪಯೋಗಿಸು**: MCP ಕಾರ್ಯಭಾರ ರಕ್ಷಣೆಗೆ Microsoft Prompt Shields, Azure Content Safety, GitHub Advanced Security ಅನ್ನು ಅರ್ಥಮಾಡಿ ಮತ್ತು ಜಾರಿಗೊಳಿಸು
- **ಉಪಕರಣ ಭದ್ರತೆಯನ್ನು ಮಾನ್ಯಗೊಳಿಸು**: ಉಪಕರಣ ಮೆಟಾ ಡೇಟಾ ಮಾನ್ಯಗೊಳಿಸುವಿಕೆಯ ಮಹತ್ವವನ್ನು, ಡೈನಾಮಿಕ್ ಬದಲಾವಣೆಗಳ ಮೇಲ್ವಿಚಾರಣೆ ಮತ್ತು ಅಪ್ರತಕ್ಷ ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್ ದಾಳಿಗಳ ವಿರುದ್ಧ ರಕ್ಷಣೆ ಕಲಿಯಿರಿ
- **ಉತ್ತಮ ಪದ್ಧತಿಗಳನ್ನು ಸಂಯೋಜಿಸು**: ಸ್ಥಾಪಿತ ಭದ್ರತಾ ಮೂಲತತ್ವಗಳು (ಸುರಕ್ಷಿತ ಕೋಡಿಂಗ್, ಸರ್ವರ್ ಬಲವರ್ಧನೆ, ಶೂನ್ಯ ನಂಬಿಕೆಯೊಳಗೆ) ಮತ್ತು MCP-ನಿರ್ದಿಷ್ಟ ನಿಯಂತ್ರಣಗಳನ್ನು ಸಮಗ್ರ ರಕ್ಷಣೆಗೆ ಒಗ್ಗೂಡಿಸಿ

# MCP ಭದ್ರತಾ ನಿರ್ಮಾಣ ಮತ್ತು ನಿಯಂತ್ರಣಗಳು

ಆಧುನಿಕ MCP ಅನ್ವಯಿಕೆಗಳಿಗೆ ಸಾಂಪ್ರದಾಯಿಕ ಸಾಫ್ಟ್‌ವೇರ್ ಭದ್ರತೆ ಮತ್ತು AI-ನಿರ್ದಿಷ್ಟ ಅಪಾಯಗಳನ್ನು ತಲುಪುವ ಪದರದ ಭದ್ರತಾ ವಿಧಾನಗಳು ಅಗತ್ಯವಿವೆ. ತ್ವರಿತವಾಗಿ ಅಭಿವೃದ್ಧಿಯಾಗುತ್ತಿರುವ MCP ನಿರ್ದಿಷ್ಟರೇಖೆ ತನ್ನ ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳನ್ನು ಸದೃಢಗೊಳಿಸುತ್ತಿದೆ, ಇದರಿಂದ ಉದ್ಯಮ ಭದ್ರತಾ ರೂಪರೇಖೆಗಳಿಗೆ ಮತ್ತು ಸ್ಥಾಪಿತ ಉತ್ತಮ ಕ್ರಮಗಳಿಗೆ ಉತ್ತಮ ಏಕೀಕರಣ ಸಾಧ್ಯವಾಗುತ್ತದೆ.

[Microsoft ಡಿಜಿಟಲ್ ಡಿಫೆನ್ಸ್ ವರದಿ](https://aka.ms/mddr) ನಿಂದ ತಿಳಿದುಬಂದಿರುವಂತೆ **ವರದಿಪಡಿಸಿದ ಉಲ್ಲಂಘನೆಗಳ 98% ತುಮಕ ಭದ್ರತಾ ಶೌಚಾಲಯದಿಂದ ತಡೆಯಬಹುದು**. ಅತ್ಯುತ್ತಮ ರಕ್ಷಣಾ ತಂತ್ರ ಮುಂದಿನ MCP-ನಿರ್ದಿಷ್ಟ ನಿಯಂತ್ರಣಗಳೊಂದಿಗೆ ಮೂಲಭೂತ ಭದ್ರತಾ ಪದ್ಧತಿಗಳನ್ನು ಸಂಯೋಜಿಸುವುದು—ಸ್ಥಾಪಿತ ಸ್ಥೂಲ ಭದ್ರತಾ ಕ್ರಮಗಳು ಒಟ್ಟಾರೆ ಅಪಾಯವನ್ನು ಕಡಿಮೆ ಮಾಡಲು ಅತ್ಯಂತ ಪರಿಣಾಮಕಾರಿಯಾಗಿದೆ.

## ಪ್ರಸ್ತುತ ಭದ್ರತಾ ಪರಿಸರ

> **ಟಿಪ್ಪಣಿ:** ಈ ಮಾಹಿತಿಯು MCP ಭದ್ರತಾ ಪ್ರಾಮಾಣಿಕತೆಗಳನ್ನು **ಫೆಬ್ರವರಿ 5, 2026** ರ ಮೂಲಕ ಪರೀಕ್ಷಿಸಲಾಗಿದೆ, ಇದು **MCP ನಿರ್ದಿಷ್ಟರೇಖೆ 2025-11-25** ಗಾಗಿ ಹೊಂದಿಕೆ ಮಾಡಿದೆ. MCP ಪ್ರೋಟೋಕಾಲ್ ದ್ರುಮುಖವಾಗಿ ಮುಂದುವರೆಯುತ್ತಿದೆ ಮತ್ತು ಭವಿಷ್ಯದ ಅನ್ವಯಿಕೆಗಳು ಹೊಸ ಪ್ರಮಾಣೀಕರಣ ಮಾದರಿಗಳನ್ನು ಮತ್ತು ಸುಧಾರಿತ ನಿಯಂತ್ರಣಗಳನ್ನು ಪರಿಚಯಿಸಬಹುದು. ದಯವಿಟ್ಟು ಸದ್ಯದ [MCP ನಿರ್ದಿಷ್ಟರೇಖೆ](https://spec.modelcontextprotocol.io/), [MCP GitHub ರೆಪೊ](https://github.com/modelcontextprotocol), ಮತ್ತು [ಭದ್ರತಾ ಉತ್ತಮ ಕ್ರಮಗಳ ಪ್ರമാണಿತ ದಾಖಲೆ](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices)ಗಳನ್ನು ಅನುಸರಿಸಿ.

## 🏔️ MCP ಭದ್ರತಾ ಶಿಖರ ಶಿಬಿರ (ಶರ್ಫಾ)

**ಹಾಸು-ಹಸ್ತ ಭದ್ರತಾ ತರಬೇತಿಗಾಗಿ**, ನಾವು **MCP ಭದ್ರತಾ ಶಿಖರ ಶಿಬಿರ** (ಶರ್ಫಾ)ನ್ನು ಶಿಫಾರಸು ಮಾಡುತ್ತೇವೆ - Microsoft Azure ನಲ್ಲಿ MCP ಸರ್ವರ್‌ಗಳನ್ನು ಸುರಕ್ಷಿತಗೊಳಿಸುವ ಸಮಗ್ರ ಮಾರ್ಗಸೂಚಿಯೊಡನೆ ಮಾರ್ಗದರ್ಶನಗೊಳಿಸುವ ಯಾತ್ರೆ.

### ಶಿಬಿರ ಅವಲೋಕನ

[MCP ಭದ್ರತಾ ಶಿಖರ ಶಿಬಿರ](https://azure-samples.github.io/sherpa/) ವ್ಯಾಪಕವಾಗಿ "ದುರ್ಬಲ → ಅಪಾಯ → ಸರಿಪಡಿಸು → ಮಾನ್ಯಗೊಳಿಸು" ವಿಧಾನವನ್ನು ಅನುಸರಿಸಿ ಪ್ರಾಯೋಗಿಕ ಮತ್ತು ಕಾರ್ಯನಿರ್ವಹಣಾ ಭದ್ರತಾ ತರಬೇತಿಯನ್ನು ಒದಗಿಸುತ್ತದೆ. ನೀವು:

- **ಬ್ರೇಕ್ ಮಾಡಿ ಕಲಿಯಿರಿ**: ಉದ್ದೇಶಪೂರ್ವಕವಾಗಿ ದುರ್ಬಲ ಸರ್ವರ್‌ಗಳನ್ನು ದುರುಪಯೋಗ ಪಡಿಸಿ ಅಪಾಯಗಳನ್ನು ಅನುಭವ ಮಾಡಿ
- **Azure-ಸ್ವದೇಶೀ ಭದ್ರತೆ ಉಪಯೋಗಿಸಿ**: Azure Entra ID, Key Vault, API Management, ಮತ್ತು AI Content Safety ಬಳಸಿರಿ
- **ರಕ್ಷಣೆ-ಮಧ್ಯಮ ಸ್ಥಿತಿ ಅನುಸರಿಸಿ**: ಶಿಬಿರಗಳ ಮೂಲಕ ಸಾಗುತ್ತಾ ಸಮಗ್ರ ಭದ್ರತಾ ಪದರಗಳನ್ನು ನಿರ್ಮಿಸಿ
- **OWASP ಪ್ರಮಾಣಿತಗಳನ್ನು ಅನ್ವಯಿಸಿ**: ಪ್ರತಿ ತಂತ್ರ [OWASP MCP Azure Security Guide](https://microsoft.github.io/mcp-azure-security-guide/) ಗೆ ಹೊಂದಿಕೆ
- **ಕಾರ್ಯನಿರ್ವಹಣೆಯ ಕೋಡ್ ಪಡೆಯಿರಿ**: ಕಾರ್ಯನಿರ್ವಹಣೆಯ ಮತ್ತು ಪರೀಕ್ಷಿತ ಅನ್ವಯಿಕೆಗಳನ್ನು ಎತ್ತಿಬಂದುಕೊಳ್ಳಿ

| **MCP02** | ವ್ಯಾಪ್ತಿಯಲ್ಲಿ ಹಕ್ಕು ವೃದ್ಧಿ | RBAC, ಶರತ್ತುಬದ್ಧ ಪ್ರವೇಶ |
| **MCP03** | ಉಪಕರಣ ವಿಷಾಕರಣೆ | ಉಪಕರಣ ಮಾನ್ಯಗೊಳಿಸುವಿಕೆ, ಗುಣಮಟ್ಟ ಪರಿಶೀಲನೆ |
| **MCP04** | ಸಾಫ್ಟ್‌ವೇರ್ ಪೂರೈಕೆ ಸರಪಳಿ ದಾಳಿಗಳು ಮತ್ತು ಅವಲಂಬನೆ ಹಾನಿ | GitHub Advanced Security, ಅವಲಂಬನೆ ಪರಿಶೀಲನೆ |
| **MCP05** | ಆಜ್ಞೆ ಇಂಜೆಕ್ಷನ್ ಮತ್ತು ಕಾರ್ಯಾಚರಣೆ | ಇನ್‌ಪುಟ್ ಮಾನ್ಯಗೊಳಿಸುವಿಕೆ, ಸ್ಯಾಂಡ್‌ಬಾಕ್ಸಿಂಗ್ |
| **MCP06** | ಉದ್ದೇಶದ ಪ್ರವಾಹ ವಂಚನೆ | Azure AI Content Safety, Prompt Shields |
| **MCP07** | ಅಪರ್ಯाप्त ಪ್ರಮಾಣೀಕರಣ ಮತ್ತು ಪ್ರಾಧಿಕಾರ | Azure Entra ID, PKCE ಜೊತೆಗೆ OAuth 2.1 |
| **MCP08** | ಆಡಿಟ್ ಮತ್ತು ಟೆಲಿಮೆಟ್ರಿ ಕೊರತೆ | Azure Monitor, Application Insights |
| **MCP09** | ಛಾಯಾಚಿತ್ರ MCP ಸರ್ವರ್‌ಗಳು | API ಸೆಂಟರ್ ಆಡಳಿತ, ನೆಟ್ವರ್ಕ್ ಒಡೆಯಿಕೆ |
| **MCP10** | ಸಾಂದರ್ಭಿಕ ಇಂಜೆಕ್ಷನ್ ಮತ್ತು ಅಧಿಕ ಹಂಚಿಕೆ | ಡೇಟಾ ವರ್ಗೀಕರಣ, ಕನಿಷ್ಠ ಬಹಿರಂಗತೆ |

### MCP ಪ್ರಮಾಣೀಕರಣದ ಬೆಳವಣಿಗೆ

MCP ನಿರ್ದಿಷ್ಟರೇಖೆ ಪ್ರಮಾಣೀಕರಣ ಮತ್ತು ಪ್ರಾಧಿಕಾರಗಳ ದೃಷ್ಟಿಯಿಂದ ಪ್ರಮುಖ ಬದಲಾವಣೆಗಳನ್ನು ಅನುಭವಿಸಿದೆ:

- **ಆರಂಭಿಕ ಪರಿಕಲ್ಪನೆ**: ಪ್ರಾರಂಭದ ನಿರ್ದಿಷ್ಟರೇಖೆಗಳು ಡೆವಲಪರ್‌ಗಳಿಗೆ ಕಸ್ಟಮ್ ಪ್ರಮಾಣೀಕರಣ ಸರ್ವರ್‌ಗಳನ್ನು ಜಾರಿಗೊಳಿಸುವಂತೆ ಮಾಡುತ್ತಿದ್ದು, MCP ಸರ್ವರ್‌ಗಳು OAuth 2.0 ಪ್ರಾಧಿಕಾರ ಸರ್ವರ್‌ಗಳಾಗಿ ನೇರವಾಗಿ ಬಳಕೆದಾರರನ್ನು ನಿರ್ವಹಿಸುತ್ತಿದ್ದವು
- **ಪ್ರಸ್ತುತ ಮಾನದಂಡ (2025-11-25)**: ನವೀಕರಿಸಿದ ನಿರ್ದಿಷ್ಟರೇಖೆ MCP ಸರ್ವರ್‌ಗಳು ಬಾಹ್ಯ ಗುರುತು ಒದಗಿಸುವವರ (ಉದಾ: Microsoft Entra ID) ನಗರಿಕತೆ ಪ್ರಮಾಣೀಕರಣಕ್ಕೆ ಹಸ್ತಾಂತರಿಸಬಹುದು, ಇದರಿಂದ ಭದ್ರತೆ ಮತ್ತು ಜಾರಿಗೆ ಕಡಿಮೆ ಸಂಕೋಚ ನೀಡುತ್ತದೆ
- **ಸंचಾರ ಪದರ ಭದ್ರತೆ**: ಸ್ಥಳೀಯ (STDIO) ಮತ್ತು ದೂರಸ್ಥ (Streamable HTTP) ಸಂಪರ್ಕಗಳಿಗೆ ಸೂಕ್ತ ಪ್ರಮಾಣೀಕರಣ ಮಾದರಿ ಹೊಂದಿರುವ ಸುರಕ್ಷಿತ ಸಂಪರ್ಕ ವಿಧಾನಗಳು

## ಪ್ರಮಾಣೀಕರಣ ಮತ್ತು ಪ್ರಾಧಿಕಾರ ಭದ್ರತೆ

### ಪ್ರಸ್ತುತ ಭದ್ರತಾ ಸವಾಲುಗಳು

ಆಧುನಿಕ MCP ಅನ್ವಯಿಕೆಗಳು ಹಲವು ಪ್ರಮಾಣೀಕರಣ ಮತ್ತು ಪ್ರಾಧಿಕಾರ ಸವಾಲುಗಳನ್ನು ಎದುರಿಸುತ್ತಿವೆ:

### ಅಪಾಯ ಮತ್ತು ಹಾನಿ ಮಾರ್ಗಗಳು

- **ಪ್ರಾಧಿಕಾರ ಲಾಜಿಕ್ ದೋಷ**: MCP ಸರ್ವರ್‌ಗಳಲ್ಲಿ ಅಂಗೀಕರಿಸಿದ ದೃಷ್ಟಿಕೋನವಿಲ್ಲದ ಪ್ರಾಧಿಕಾರ ಜಾರಿಗೆ ಅಪಾಯಕಾರಿ ಮಾಹಿತಿಯನ್ನು ಬಹಿರ್ ಮಾಡಬಹುದು ಮತ್ತು ತಪ್ಪಾಗಿ ಪ್ರವೇಶ ನಿಯಂತ್ರಣ ಅನ್ವಯಿಸಬಹುದು
- **OAuth ಟೋಕನ್ ವರದಿ**: ಸ್ಥಳೀಯ MCP ಸರ್ವರ್ ಟೋಕನ್ ದುರ್ಲಭರಹಿತತೆ ದಾಳಿಕ್ತಾರರು ಸರ್ವರ್‌ಗಳನ್ನು ನಕಲಿ ಸಾಗಿ ಅದುಡಲು ಅವಕಾಶ ಕೊಡುತ್ತದೆ
- **ಟೋಕನ್ ಪಾಸ್‌ಥ್ರು ದುರ್ಬಲತೆಗಳು**: ಅಸಮರ್ಥ ಟೋಕನ್ ನಿರ್ವಹಣೆ ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳನ್ನು ದಾಟುವಿಕೆಗೆ ಮತ್ತು ಹೊಣೆಗಾರಿಕೆಯ ಕೊರತೆಯನ್ನುಂಟುಮಾಡುತ್ತದೆ
- **ಅತಿಕ್ರಮಿತ ಅನುಮತಿಗಳು**: ಹೆಚ್ಚು ಹಕ್ಕು ಪಡೆದ MCP ಸರ್ವರ್‌ಗಳು ಕನಿಷ್ಟ ಹಕ್ಕು ತತ್ವವನ್ನು ಉಲ್ಲಂಘಿಸಿ ದಾಳಿ ಮೇಲೆ ಬಲವಾದ ಭೂಮಿಯನ್ನು ವಿಸ್ತರಿಸುತ್ತವೆ

#### ಟೋಕನ್ ಪಾಸ್‌ಥ್ರು: ಗಂಭೀರ ತಿರಸ್ಕಾರ ಮಾದರಿ

**ಪ್ರಸ್ತುತ MCP ಪ್ರಾಧಿಕಾರ ನಿರ್ದಿಷ್ಟರೇಖೆಯಲ್ಲಿ ಟೋಕನ್ ಪಾಸ್‌ಥ್ರು ಸ್ಪಷ್ಟವಾಗಿ ನಿಷೇಧಿಸಲಾಗಿದೆ, ಇದರಕಡುತ್ತೀರ್ಥ ಭದ್ರತಾ ಪರಿಣಾಮಗಳು ಇದ್ದವು:**

##### ಭದ್ರತಾ ನಿಯಂತ್ರಣ ಸುತ್ತುವೂವಿಕೆ  
- MCP ಸರ್ವರ್‌ಗಳು ಮತ್ತು ಡೌನ್‌ಸ್ಟ್ರೀಮ್ API ಗಳು ಪ್ರಮುಖ ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳನ್ನು (दर ಮಾನದಂಡ, ವಿನಂತಿ ಪರಿಶೀಲನೆ, ಸಂಚಾರ ಮೇಲ್ವಿಚಾರಣೆ) ಜಾರಿಗೊಳಿಸುತ್ತವೆ, ಮತ್ತು ಅವುಗಳಿಗೆ ಸೂಕ್ತ ಟೋಕನ್ ಮಾನ್ಯತೆ ಅಗತ್ಯ
- ಕ್ಲೈಂಟ್ ನೇರವಾಗಿ APIಗೆ ಟೋಕನ್ ಬಳಸುವುದು ಈ ರಕ್ಷಣೆಗಳನ್ನು ತಡೆದು ಡಿಫೆನ್ಸ್ ಶಿಲ್ಪಕ್ಕೆ ಹಾನಿ ಮಾಡುತ್ತದೆ

##### ಹೊಣೆಗಾರಿಕೆ ಮತ್ತು ಆಡಿಟ್ ಸವಾಲುಗಳು  
- MCP ಸರ್ವರ್‌ಗಳು ಅಮೂಲ್ಯ ಗ್ರಾಹಕರಿಂದ ಬರುವ ಟೋಕನ್‌ಗಳಿಂದ ವ್ಯತ್ಯಾಸವನ್ನು ಮಾಡಲಾಗದು, ಅಂದರೆ ಆಡಿಟ್ ಟ್ರೇಸ್ಗಳು ಕದಿಯುತ್ತದೆ
- ಡೌನ್‌ಸ್ಟ್ರೀಮ್ ಸಂಪನ್ಮೂಲ ಸರ್ವರ್ ಲಾಗ್‌ಗಳು ತಪ್ಪು ಮೂಲಗಳನ್ನು ತೋರಿಸುತ್ತವೆ, ವಾಸ್ತವ MCP ಸರ್ವರ್ ಮಧ್ಯವರ್ತಿಗಳನ್ನು ತೋರಿಸದು
- ಘಟನೆ ತನಿಖೆ ಮತ್ತು ಅನುಕೂಲ ಪರಿಶೀಲನೆ ಬಹಳ ಕಷ್ಟವಾಗುತ್ತದೆ

##### ಡೇಟಾ ಹೊರಹಾಕುವ ಅಪಾಯಗಳು  
- ಮಾನ್ಯತೆ ಇಲ್ಲದ ಟೋಕನ್ ಹಕ್ಕುಗಳು ಕಳ್ಳರ ಕೈಗೆ ಬಂದು MCP ಸರ್ವರ್‌ಗಳನ್ನು ಪ್ರೋಕ್ಸಿ ಮಾಡಿ ಡೇಟಾ ಹೊರಹಾಕಲು ಉಪಯೋಗಿಸಬಹುದು
- ನಂಬಿಕೆ ಗಡಿಗಳನ್ನು ಉಲ್ಲಂಘಿಸಿ ಅನಧಿಕೃತ ಪ್ರವೇಶ ರಸ್ತೆಗಳಿಗೆ ಅವಕಾಶ ನೀಡುತ್ತದೆ

##### ಬಹು ಸೇವಾ ದಾಳಿ ಮಾರ್ಗಗಳು  
- ಕದಿದ ಟೋಕನ್‌ಗಳ ಮೂಲಕ ಅನೇಕ ಸೇವೆಗಳು ಸ್ವೀಕರಿಸಿ ಜೋಡಣಾ ವ್ಯವಸ್ಥೆಯಲ್ಲಿ ಅಡ್ಡ ҳಲನ ಮರುಚಲನೆ ಸಾಧ್ಯ
- ಮೂಲ ಪರಿಶೀಲನೆ ಸಾಧ್ಯವಾಗದಾಗ ಸೇವೆಗಳ ಮಧ್ಯೆ ನಂಬಿಕೆ ಗಡಿಗಳು ಉಲ್ಲಂಘಿಸುವ ಸಂಕಷ್ಟವುಂಟಾಗುತ್ತದೆ

### ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳು ಮತ್ತು ತಡೆಗಟ್ಟುವಿಕೆಗಳು

**ಅತಿಮೂಲ್ಯ ಭದ್ರತಾ ಅಗತ್ಯ:**

> **ಅನಿವಾರ್ಯ**: MCP ಸರ್ವರ್‌ಗಳು **ಮಾತ್ರ** ತಮ್ಮದೇಿಗಾಗಿ ವಿಶೇಷವಾಗಿ ಜಾರಿಗೊಳಿಸಿದ ಟೋಕನ್‌ಗಳನ್ನು ಮಾತ್ರ ಸ್ವೀಕರಿಸಬೇಕು

#### ಪ್ರಮಾಣೀಕರಣ ಮತ್ತು ಪ್ರಾಧಿಕಾರ ನಿಯಂತ್ರಣ

- **ಕಠಿಣ ಪ್ರಾಧಿಕಾರ ವಿಮರ್ಶೆ**: MCP ಸರ್ವರ್ ಪ್ರಾಧಿಕಾರ ಲಾಜಿಕ್ ಸಂಪೂರ್ಣವಾಗಿ ಪರಿಶೀಲಿಸಿ, ಮಾತ್ರ ನಿರ್ಧಿಷ್ಟ ಬಳಕೆದಾರರು ಮತ್ತು ಗ್ರಾಹಕರು ವಿಶಿಷ್ಟ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಪ್ರವೇಶಿಸುತ್ತಾರೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
  - **ಜಾರಿಗೊಳಿಸುವಿಕೆ ಮಾರ್ಗದರ್ಶಿ**: [MCP ಸರ್ವರ್‌ಗಳಿಗೆ Azure API ಮ್ಯಾನೇಜ್ಮೆಂಟ್ ಅನ್ನು ಪ್ರಮಾಣೀಕರಣ ಗೇಟ್ವೇ ಆಗಿ ಉಪಯೋಗಿಸುವುದು](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **ಗುರುತು ಏಕರೂಪಗೊಳಿಸುವಿಕೆ**: [MCP ಸರ್ವರ್ ಪ್ರಮಾಣೀಕರಣಕ್ಕಾಗಿ Microsoft Entra ID ಉಪಯೋಗಿಸುವುದು](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **ಸುಸ್ಥಿರ ಟೋಕನ್ ನಿರ್ವಹಣೆ**: [Microsoft-ನ ಟೋಕನ್ ಮಾನ್ಯತೆ ಮತ್ತು ಜೀವಚಕ್ರ ಉತ್ತಮ ಕ್ರಮಗಳು](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens) ಜಾರಿಗೊಳಿಸಿ
  - ಟೋಕನ್ ದರ್ಶನದ ಹಕ್ಕುಗಳು MCP ಸರ್ವರ್ ಗುರುತಿಗೆ ಹೊಂದಾಣಿಕೆ ಹೊಂದಿವೆ ಎಂದು ಪರಿಶೀಲಿಸಿ
  - ಸೂಕ್ತ ಟೋಕನ್ ರೋಟೇಶನ್ ಮತ್ತು ಅವಧಿಯ ನಿಯಮಗಳನ್ನು ಜಾರಿಗೊಳಿಸಿ
  - ಟೋಕನ್ ಮರುಬಳಕೆ ದಾಳಿಗಳನ್ನು ಮತ್ತು ಅನಧಿಕೃತ ಬಳಕೆಯನ್ನು ತಡೆಯಿರಿ

- **ರಕ್ಷಿತ ಟೋಕನ್ ಸಂಗ್ರಹಣೆ**: ಡೇಟಾವನ್ನು ಸ್ಟೋರೇಜ್ ಮತ್ತು ಸಂಚಾರದ ಸಮಯದಲ್ಲಿ ಎನ್‌ಕ್ರಿಪ್ಷನ್ ಮೂಲಕ ಸುರಕ್ಷಿತವಾಗಿರಿಸಿ
  - **ಉತ್ತಮ ಪದ್ಧತಿಗಳು**: [ರಕ್ಷಿತ ಟೋಕನ್ ಸಂಗ್ರಹಣೆ ಮತ್ತು ಎನ್‌ಕ್ರಿಪ್ಷನ್ ಮಾರ್ಗದರ್ಶಿ](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### ಪ್ರವೇಶ ನಿಯಂತ್ರಣ ಜಾರಿಗೊಳಿಸುವಿಕೆ

- **ಕನಿಷ್ಟ ಆಧಿಕಾರ ತತ್ವ**: MCP ಸರ್ವರ್‌ಗಳಿಗೆ ಅದರ ಕಾರ್ಯಕ್ಷಮತೆಯಿಗಾಗಿ ಅಗತ್ಯವೇನಾದರೂ ಕನಿಷ್ಟ ಹಕ್ಕುಗಳನ್ನು ಮಾತ್ರ ನೀಡಿ
  - ನಿಯಮಿತ ಅನುಮತಿ ವಿಮರ್ಶೆಗಳು ಮತ್ತು ಪರಿಷ್ಕರಣೆ privilege creep ತಡೆಯಲು
  - **Microsoft ದಾಖಲೆ**: [ಕನಿಷ್ಠ-ಅಧಿಕೃತ ಪ್ರವೇಶ ಭದ್ರತೆ](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **ಪಾತ್ರ ಆಧಾರಿತ ಪ್ರವೇಶ ನಿಯಂತ್ರಣ (RBAC)**: ಸೂಕ್ಷ್ಮ ಪಾತ್ರ ನಿಯೋಜನೆಗಳನ್ನು ಜಾರಿಗೊಳಿಸಿ
  - ನಿರ್ದಿಷ್ಟ ಸಂಪನ್ಮೂಲಗಳು ಮತ್ತು ಕ್ರಿಯೆಗಳಿಗಷ್ಟೇ ಪಾತ್ರಗಳನ್ನು ಕಠಿಣವಾಗಿ ಮಿತಿ ಮಾಡಿ
  - ವ್ಯಾಪಕ ಅಥವಾ ಅನಗತ್ಯ ಅನುಮತಿಗಳನ್ನು ತಪ್ಪಿಸಿ, ದಾಳಿ ಸದನ ವಿಸ್ತರಣೆ ತಡೆಯಿರಿ

- **ನಿರಂತರ ಅನುಮತಿ ಮೇಲ್ವಿಚಾರಣೆ**: ಪ್ರವೇಶವನ್ನು ನಿರಂತರವಾಗಿ ಪರಿಶೀಲಿಸಿ ಮತ್ತು ಮೇಲ್ವಿಚಾರಣೆ ಮಾಡಿರಿ
  - ಅನುಮತಿ ಬಳಕೆಯ ಮಾದರಿಗಳನ್ನು ವಿಶ್ಲೇಷಿಸಿ ಅಸಾಮಾನ್ಯತೆಗಳನ್ನು ಪತ್ತೆ ಹೊತ್ತು
  - ಹೆಚ್ಚು ಅಥವಾ ಬಳಕೆಯಿಲ್ಲದ ಹಕ್ಕುಗಳನ್ನು ತ್ವರಿತವಾಗಿ ಸರಿಪಡಿಸಿ

## AI-ನಿರ್ದಿಷ್ಟ ಭದ್ರತಾ ಅಪಾಯಗಳು

### ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್ ಮತ್ತು ಉಪಕರಣ ಸಂಚಲನೆ ದಾಳಿಗಳು

ಆಧುನಿಕ MCP ಅನ್ವಯಿಕೆಗಳು ಸಾಂಪ್ರದಾಯಿಕ ಭದ್ರತಾ ಕ್ರಮಗಳು ಪೂರ್ಣವಾಗಿ ನಿಭಾಯಿಸಲಾಗದ ಸುಕ್ಷ್ಮ AI-ನಿರ್ದಿಷ್ಟ ದಾಳಿ ಮಾರ್ಗಗಳನ್ನು ಎದುರಿಸುತ್ತಿವೆ:

#### **ಅಪ್ರತಕ್ಷ ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್ (ಕ್ರಾಸ್-ಡೊಮೇನ್ ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್)**

**ಅಪ್ರತಕ್ಷ ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್** MCP ಸಕ್ರಿಯ AI ವ್ಯವಸ್ಥೆಯ ಅತ್ಯಂತ ಗಂಭೀರ ದುರ್ಬಲತೆಗಳಲ್ಲೊಂದಾಗಿದೆ. ದಾಳಿಕ್ತಾರರು ಹೊರಗಿನ ವಿಷಯಗಳಲ್ಲಿ - ಡಾಕ್ಯುಮೆಂಟ್‌ಗಳು, ವೆಬ್ ಪುಟಗಳು, ಇಮೇಲ್‌ಗಳು ಅಥವಾ ಡೇಟಾ ಮೂಲಗಳು - ನೀತಿಪರ ನಿಯಮಗಳಂತೆ AI ವ್ಯವಸ್ಥೆಗಳು ಪ್ರಕ್ರಿಯೆ ಮಾಡುವ ದುಷ್ಟ ನಿರ್ದೇಶನಗಳನ್ನು ಸೇರಿಸುತ್ತಾರೆ.

**ದಾಳಿ ಪ್ರಕರಣಗಳು:**
- **ಡಾಕ್ಯುಮೆಂಟ್ ಆಧಾರಿತ ಇಂಜೆಕ್ಷನ್**: ಪ್ರಕ್ರಿಯೆಗೆ ಒಳಾದ ಡಾಕ್ಯುಮೆಂಟ್‌ಗಳಲ್ಲಿ ಮೌಢ್ಯಾದೇಶಿತ ನಿರ್ದೇಶನಗಳು AI ನಿಂದ ಅನಿರೀಕ್ಷಿತ ಕ್ರಮಗಳನ್ನು ಪ್ರೇರೇಪಿಸುತ್ತವೆ
- **ವೆಬ್ ವಿಷಯ ದುರುಪಯೋಗ**: ಸಂಸ್ಕೃತಿದ್ದ ವೆಬ್ ಪುಟಗಳಲ್ಲಿ ತಂತ್ರಜ್ಞಾನದ ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ಸೇರಿಸುವ ಮೂಲಕ AI ನ ನಡೆಯುವಿಕೆಯನ್ನು ಪ್ರಭಾವಗೊಳಿಸುವುದು
- **ಇಮೇಲ್ ಆಧಾರಿತ ದಾಳಿಗಳು**: ಇಮೇಲ್‌ಗಳಲ್ಲಿ ದುಷ್ಟ ಪ್ರಾಂಪ್ಟ್‌ಗಳು AI ಸಹಾಯಕರಿಂದ ಮಾಹಿತಿ ಹೊರಡಿಸುವುದಕ್ಕೆ ಅಥವಾ ಅನಧಿಕೃತ ಕಾರ್ಯಗಳು ನಡೆಸುವಂತೆ ಮಾಡುತ್ತವೆ
- **ಡೇಟಾ ಮೂಲಗಳು ಮಾಲಿನ್ಯಗೊಳಿಸುವಿಕೆ**: ಕೆಟ್ಟಡಿತ ಡೇಟಾಬೇಸ್ ಅಥವಾ API ಗಳು AI ವ್ಯವಸ್ಥೆಗಳಿಗೆ ಬವಣೆಗೊಂಡ ವಿಷಯಗಳನ್ನು ನೀಡುತ್ತವೆ

**ವಾಸ್ತವ ಪ್ರಭಾವ**: ಈ ದಾಳಿಗಳು ಡೇಟಾ ಹೊರಪಡಿಸುವಿಕೆ, ಗೌಪ್ಯತಾ ಉಲ್ಲಂಘನೆ, ಹಾನಿಕರ ವಿಷಯ ಸೃಷ್ಟಿ ಮತ್ತು ಬಳಕೆದಾರ ಸಂವಾದಗಳ ನಿಯಂತ್ರಣಕ್ಕೆ ಕಾರಣವಾಗಬಹುದು. ವಿವರಕ್ಕೆ [MCP ನಲ್ಲಿ ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್ (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/) ನೋಡಿ.

![Prompt Injection Attack Diagram](../../../translated_images/kn/prompt-injection.ed9fbfde297ca877.webp)

#### **ಉಪಕರಣ ವಿಷಾಕರಣೆ ದಾಳಿಗಳು**

**ಉಪಕರಣ ವಿಷಾಕರಣೆ** MCP ಉಪಕರಣಗಳನ್ನು นิರ್ದರಿಸುವ ಮೆಟಾ ಡೇಟಾದ ಮೇಲೆ ದಾಳಿ ಮಾಡುತ್ತದೆ, ಇದು LLM ಗಳು ಉಪಕರಣ ವಿವರಣೆಗಳು ಮತ್ತು ನಿಯಮಾವಳಿಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವ ಮೂಲಕ ಕಾರ್ಯಾಚರಣೆ ನಿರ್ಣಯ ಮಾಡುತ್ತದೆ.

**ದಾಳಿ ಯಾವುದೇ ವಿಧಾನಗಳು:**
- **ಮೆಟಾ ಡೇಟಾ ಚರಿತ್ರಣ**: ದಾಳಿಕತಾರರು ಉಪಕರಣ ವಿವರಣೆಗಳು, ನಿಯಮಾವಳಿ ನಿರ್ಧಾರಗಳು ಅಥವಾ ಉಪಯೋಗ ಉದಾಹರಣೆಗಳಲ್ಲಿ ದುಷ್ಟ ನಿರ್ದೇಶನಗಳನ್ನು ಸೇರಿಸುತ್ತಾರೆ
- **ಅದೃಶ್ಯ ನಿರ್ದೇಶನಗಳು**: ಮಾನವನಿಗೆ ಕಾಣದಂತೆ AI ಮಾದರಿಗಳು ಪ್ರಕ್ರಿಯೆ ಮಾಡುವ ಸಚಿವಾಧೀನ ಪ್ರಾಂಪ್ಟ್‌ಗಳು
- **ಡೈನಾಮಿಕ್ ಉಪಕರಣ ಪರಿಷ್ಕರಣೆ ("ರಗ್ ಪಲ್ಲು")**: ಬಳಕೆದಾರರು ಅನುಮೋದಿಸಿರುವ ಉಪಕರಣಗಳನ್ನು ನಂತರ ದುಷ್ಟ ಕೆಲಸ ಮಾಡಲು ಬದಲಾಯಿಸಲಾಗುವುದು, ಬಳಕೆದಾರರಿಗೆ ತಿಳಿಯದೆ
- **ಪ್ಯಾರಾಮೀಟರ್ ಇಂಜೆಕ್ಷನ್**: ಮಾದರಿಯ ನಡೆಗೆ ಪ್ರಭಾವ ಬೀರುವುದಕ್ಕಾಗಿ ಉಪಕರಣ ನಿಯಮಾವಳಿ ಸ್ಕೀಮಾಗಳಲ್ಲಿ ದುಷ್ಟ ವಿಷಯ ಸೇರಿಸುವುದು

**ಹೋಸ್ಟ್ ಮಾಡಲಾದ ಸರ್ವರ್ ಅಪಾಯಗಳು**: ದೂರಸ್ಥ MCP ಸರ್ವರ್‌ಗಳಲ್ಲಿ ಉಪಕರಣ ವಿವರಣೆಗಳನ್ನು ಬಳಕೆದಾರ ಅನುಮೋದನೆಯ ನಂತರ ನವೀಕರಿಸಬಹುದು, ಇದರಿಂದ ಮೊದಲಿಗೆ ಸುರಕ್ಷಿತವಾಗಿದ್ದ ಉಪಕರಣಗಳು ನಂತರ ದುಷ್ಟವಾಗುವ ಸಂದರ್ಭಗಳು ಸೃಷ್ಟಿಯಾಗುತ್ತವೆ. ವಿವರಕ್ಕಾಗಿ [ಉಪಕರಣ ವಿಷಾಕರಣೆ ದಾಳಿಗಳು (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks) ನೋಡಿ.

![Tool Injection Attack Diagram](../../../translated_images/kn/tool-injection.3b0b4a6b24de6bef.webp)

#### **ಹೆಚ್ಚುವರಿ AI ದಾಳಿ ಮಾರ್ಗಗಳು**

- **ಕ್ರಾಸ್-ಡೊಮೇನ್ ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್ (XPIA)**: ಹಲವಾರು ಡೊಮೇನ್‌ಗಳ ವಿಷಯವನ್ನು ಬಳಸಿ ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳನ್ನು ತೊಲಗಿಸುವ ಜಟಿಲ ದಾಳಿಗಳು
- **ಡೈನಾಮಿಕ್ ಸಾಮರ್ಥ್ಯ ಬದಲಾವಣೆ**: ಪ್ರಾಥಮಿಕ ಭದ್ರತಾ ಮುಲ್ಯಾಂಕನವನ್ನು ತಪ್ಪಿಸಿ ಸಾಧನದ ಸಾಮರ್ಥ್ಯಗಳಲ್ಲಿ ರಿಯಲ್-ಟೈಮ್ ಬದಲಾವಣೆಗಳು  
- **ಕಾಂಟೆಕ್ಸ್ಟ್ ವಿಂಡೋ ವಿಷಪಡಿಸುವಿಕೆ**: ದ್ರೋಷಕಾರ инструк्शನ್ ಗಳನ್ನು ಅಡಗಿಸಲು ಬೃಹತ್ ಕಾಂಟೆಕ್ಸ್ಟ್ ವಿಂಡೋಗಳನ್ನು ನಿಯಂತ್ರಿಸುವ ದಾಳಿಗಳು  
- **ಮಾದರಿ ಗೊಂದಲ ದಾಳಿ**: ಮಾದರಿಗಳ ಮಿತಿಗಳನ್ನು ದುರುಪಯೋಗಿಸಿ ಅನಿರೀಕ್ಷಿತ ಅಥವಾ ಅಸುರಕ್ಷಿತ ವರ್ತನೆಗಳನ್ನು ನಿರ್ಮಿಸುವಿಕೆ  


### AI ಭದ್ರತಾ ಅಪಾಯ ಪರಿಣಾಮ

**ಹೆಚ್ಚು ಪರಿಣಾಮಕಾರಿ ಪರಿಣಾಮಗಳು:**  
- **ಡೇಟಾ ದೋಲನ**: ಅನಧಿಕೃತ ಪ್ರವೇಶ ಮತ್ತು ಸಂವೇದನಶೀಲ ಎಂಟರ್‌ಪ್ರೈಸ್ ಅಥವಾ ವೈಯಕ್ತಿಕ ಡೇಟಾ ಕಳ್ಳತನ  
- **ಗೋಪ್ಯತೆ ಲೋಪಗಳು**: ವೈಯಕ್ತಿಕವಾಗಿ ಗುರುತಿಸಬಹುದಾದ ಮಾಹಿತಿ (PII) ಮತ್ತು ಗೌಪ್ಯ ವ್ಯಾಪಾರ ಡೇಟಾರ ಬಹಿರಂಗತೆ  
- **ಸಿಸ್ಟಮ್ ತಿದ್ದುಪಡಿ**: ಪ್ರಮುಖ ವ್ಯವಸ್ಥೆಗಳು ಮತ್ತು ಕಾರ್ಯವಿಧಾನಗಳಿಗೆ ಅಕಾಲಿಕ ಬದಲಾವಣೆಗಳು  
- **ಪ್ರಮಾಣಪತ್ರ ಕಳ್ಳತನ**: ದೃಢೀಕರಣ ಟೋಕನ್‌ಗಳು ಮತ್ತು ಸೇವಾ認ಪತ್ರಗಳ ಪಾಳು  
- **ಕಾಂತಾರ ಚಲನೆ**: ಅಪಹೃತ AI ವ್ಯವಸ್ಥೆಗಳನ್ನು ವ್ಯಾಪಕ ಜಾಲದ ಮೇಲೆ ದಾಳಿಗಳಿಗಾಗಿ ಬಳಕೆ ಮಾಡಿಕೊಳ್ಳುವುದು  

### Microsoft AI ಭದ್ರತಾ ಪರಿಹಾರಗಳು

#### **AI ಪ್ರಾಂಪ್ಟ್ ಶೀಲ್ಡ್‌ಗಳು: ಇಂಜೆಕ್ಷನ್ ದಾಳಿಗಳ ವಿರುದ್ಧ ಅಭಿವೃದ್ಧಿಗೊಡ್ಡಿದ ರಕ್ಷಣಾ ಕ್ರಮಗಳು**

Microsoft **AI ಪ್ರಾಂಪ್ಟ್ ಶೀಲ್ಡ್‌ಗಳು** ನೇರ ಮತ್ತು ಅಪ್ರತಕ್ಷ ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್ ದಾಳಿಗಳ ವಿರುದ್ಧ ವಿವಿಧ ಭದ್ರತಾ ಹಿನ್ನಲೆಗಳ ಮುಖಾಂತರ ಸಂಪೂರ್ಣ ರಕ್ಷಣೆಯನ್ನು ಒದಗಿಸುತ್ತವೆ:

##### **ಮೂಲ ರಕ್ಷಣಾ ಯಂತ್ರಗಳು:**

1. **ಅತ್ಯಾವಶ್ಯಕ ಪತ್ತೆ & ಫಿಲ್ಟರಿಂಗ್**  
   - ಯಂತ್ರ ಕಲಿಕೆಯ ಅಾಳ್ಗೋರಿ ಥಮ್‌ಗಳು ಮತ್ತು NLP ತಂತ್ರಗಳು ಹೊರಗಿನ ವಿಷಯದಲ್ಲಿ ವಿದ್ಯುಕ್ತ ದೋಷಕಾರ инструк್ಷನ್‌ಗಳನ್ನು ಪತ್ತೆ ಹೊಡೆದುಕೊಳ್ಳುತ್ತವೆ  
   - ದಾಖಲೆಗಳು, ವೆಬ್ ಪುಟಗಳು, ಇಮೇಲ್‌ಗಳು ಮತ್ತು ಡೇಟಾ ಮೂಲಗಳಲ್ಲಿ ನೆಲಸಿದ ಭೀತಿಾಶಂಕೆಗಳ ರಿಯಲ್-ಟೈಮ್ ವಿಶ್ಲೇಷಣೆ  
   - ಮಾನ್ಯ ಮತ್ತು ದೋಷಕಾರ ಪ್ರಾಂಪ್ಟ್ ಮಾದರಿಗಳ ಪೈಕಿ ಪ್ರಾಸಂಗಿಕ ವಿವರಣೆ  

2. **ಸ್ಪಾಟ್‌ಲೈಟಿಂಗ್ ತಂತ್ರಗಳು**  
   - ನಂಬಿಕೆಯ 시스템 ಸೂಚನೆಗಳು ಮತ್ತು ಸಾಧ್ಯತೆಯಾಗಿ ಹಾನಿಕಾರಕ ಹೊರಗಿನ ಇನ್‌ಪುಟ್‌ಗಳನ್ನು ವಿಭಜಿಸುತ್ತದೆ  
   - ಪಠ್ಯ ಪರಿವರ್ತನ ವಿಧಾನಗಳು ಮಾದರಿಯ ಪ್ರಸ್ತುತತೆ ಹೆಚ್ಚಿಸುತ್ತದೆ ಮತ್ತು ದೋಷಕಾರ ವಿಷಯವನ್ನು ಪ್ರತ್ಯೇಕಿಸುತ್ತದೆ  
   - AI ವ್ಯವಸ್ಥೆಗಳಿಗೂ ಸೂಕ್ತ ಸೂಚನೆ ಕ್ರಮವನ್ನು ಉಳಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ ಮತ್ತು ಇಂಜೆಕ್ಟ್ ಆದ ಆಜ್ಞೆಗಳನ್ನು ನಿರ್ಲಕ್ಷಿಸುತ್ತದೆ  

3. **ಡೆಲಿಮಿಟರ್ ಮತ್ತು ಡೇಟಾಮಾರ್ಕಿಂಗ್ ವ್ಯವಸ್ಥೆಗಳು**  
   - ನಂಬಿಕೆಯಾಗಿರುವ ಸಿಸ್ಟಮ್ ಸಂದೇಶಗಳು ಮತ್ತು ಹೊರಗಿನ ಇನ್‌ಪುಟ್ ಪಠ್ಯದ ನಡುವೆ ಸ್ಪಷ್ಟ ಗಡಿಹೊರಹೋಗುವಿಕೆ  
   - ವಿಶೇಷ ಗುರುತುಗಳು ನಂಬಿಕೆಯ ಮತ್ತು ಅನಂಬೋಗ್ಯ ಡೇಟಾ ಮೂಲಗಳ ಮಧ್ಯೆ ಗಡಿ ಲಕ್ಷಣಗಳನ್ನು ಹೈಲೈಟ್ ಮಾಡುತ್ತದೆ  
   - ಸ್ಪಷ್ಟ ಅಗತ್ಯವಿಲ್ಲದ混淆 ತಪ್ಪಿಸಿ ಮತ್ತು ಅನಧಿಕೃತ ಆಜ್ಞೆ ಜಾರಿಗೆ ತಡೆಯುತ್ತದೆ  

4. **ನಿರಂತರ ಭೀತಿ ಬೂತ ಮಾಹಿತಿ**  
   - Microsoft ನಿರಂತರವಾಗಿ ಮೇಲ್ಪರುವ ದಾಳಿ ಮಾದರಿಗಳನ್ನು ಮೇಲ್ವಿಚಾರಣೆ ಮಾಡುತ್ತದೆ ಮತ್ತು ರಕ್ಷಣೆಯನ್ನು ನವೀಕರಿಸುತ್ತದೆ

## ಆಧುನಿಕ MCP ಭದ್ರತಾ ಹಾನಿ

### ಸೆಷನ್ ಹ್ಯಾಜಕಿಂಗ್ ದುರ್ಬಲತೆಗಳು

**ಸೆಷನ್ ಹ್ಯಾಜಕಿಂಗ್** ರಾಜ್ಯ ಸಂರಕ್ಷಿತ MCP ಅನುಷ್ಠಾನಗಳಲ್ಲಿನ ಪ್ರಮುಖ ದಾಳಿ ಮಾರ್ಗವನ್ನು ಪ್ರತಿನಿಧಿಸುತ್ತದೆ, ಅಲ್ಲಿ ಅನಧಿಕೃತ ಪಕ್ಷಿಗಳು ಮಾನ್ಯ ಸೆಷನ್ ಗುರುತುಗಳನ್ನು ಪಡೆಯುತ್ತವೆ ಮತ್ತು ದುರುಪಯೋಗಿಸಿ ಗ್ರಾಹಕರ ಮರುಪ್ರತಿನಿಧಿತ್ವ ಮಾಡಿಕೊಂಡು ಅನಧಿಕೃತ ಕ್ರಿಯೆಗಳನ್ನೂ ಮಾಡುತ್ತವೆ.

#### **ದಾಳಿ ದೃಷ್ಠಾಂತಗಳು ಮತ್ತು ಅಪಾಯಗಳು**

- **ಸೆಷನ್ ಹ್ಯಾಜಕ್ ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್**: ಕಳಪೆ ಸೆಷನ್  ID ಗಳೊಂದಿಗೆ ದಾಳಿಗಾರರು ಸೆಷನ್ ಸ್ಥಿತಿಯನ್ನು ಹಂಚಿಕೊಳ್ಳುವ ಸರ್ವರ್‌ಗಳಲ್ಲಿ ಹಾನಿಕಾರಕ ಘಟನೆಗಳನ್ನು ಆಳ.embed ಮಾಡಲು ಸಾಧ್ಯ, ಹಾನಿಕಾರಕ ಕ್ರಿಯೆಗಳನ್ನು ಪ್ರೇರೇಪಿಸಬಹುದು ಅಥವಾ ಸಂವೇದನಶೀಲ ಡೇಟಾ ಪ್ರವೇಶಿಸಲು ಸಾಧ್ಯವಿದೆ  
- **ನೇರ ಮರುಪ್ರತಿನಿಧಿತ್ವ**: ಕಳಪೆ ಸೆಷನ್ ID ಗಳು ದೃಢೀಕರಣವನ್ನು ತತ್ತರಿಸುವ ನೇರ MCP ಸರ್ವರ್ ಕರೆಗಳನ್ನು ಸಕ್ರಿಯಗೊಳಿಸುತ್ತವೆ, ಹೀಗೆ ದಾಳಿಗಾರರನ್ನು ಮಾನ್ಯ ಬಳಕೆದಾರರಾಗಿ ಪರಿಗಣಿಸುತ್ತದೆ  
- **ದೋಷಪೂರ್ಣ ಪುನಃಪ್ರಾರಂಭ ಸ್ಟ್ರೀಮ್ಗಳು**: ದಾಳಿಗಾರರು ವಿನಂತಿಗಳನ್ನು ಮೊದಲೇ ನಿಲ್ಲಿಸಿ, ಮಾನ್ಯ ಗ್ರಾಹಕರು ಹಾನಿಕಾರಕ ವಿಷಯದೊಂದಿಗೆ ಪುನಃ ಪ್ರಾರಂಭ ಮಾಡುವಂತೆ ಮಾಡಬಹುದು  

#### **ಸೆಷನ್ ನಿರ್ವಹಣೆಯ ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳು**

**ಪ್ರಮುಖ ಅಗತ್ಯಗಳೆಂದರೆ:**  
- **ಅಧಿಕೃತ ಪರಿಶೀಲನೆ**: MCP ಸರ್ವರ್‌ಗಳು ಇನ್ಬೌಂಡ್ ಎಲ್ಲಾ ವಿನಂತಿಗಳ ಪರಿಷೀಲನೆ ಮಾಡಬೇಕು ಮತ್ತು ದೃಢೀಕರಣಕ್ಕಾಗಿ ಸೆಷನ್‌ಗಳ ಮೇಲೆ ಆಸರೆಯುಳ್ಳುದಿಲ್ಲ  
- **ಭದ್ರಿತ ಸೆಷನ್ ರಚನೆ**: ಕ್ರಿಪ್ಟೋಗ್ರಾಫಿಕ್ ಭದ್ರಿತ, ಅಸ್ಥಿರ ಸೆಷನ್ ID ಗಳಿಗಾಗಿ ಭದ್ರವಂತ ಯಾದೃಚ್ಛಿಕ ಸಂಖ್ಯೆ ಉತ್ಪನ್ನಗಳನ್ನು ಬಳಸಿ  
- **ಬಳಕೆದಾರ-ನಿರ್ದಿಷ್ಟ ಬಾಂಧನ**: ಸೆಷನ್ ID ಗಳನ್ನು ಬಳಕೆದಾರರ ವಿಶಿಷ್ಟ ಮಾಹಿತಿಗಳಿಗೆ ಬಂಧಿಸಿ ಹಾಗೆ `<user_id>:<session_id>` ಮಾದರಿಯಲ್ಲಿ, ಬಳಕೆದಾರರ ನಡುವೆ ಸೆಷನ್ ದುರ್ಬಳಿಕೆಯನ್ನು ತಡೆಯಲು  
- **ಸೆಷನ್ ಜೀವನಚಕ್ರ ನಿರ್ವಹಣೆ**: ಸರಿಯಾದ ಅವಧಿ ನಿರ್ಧಾರ, ತಿರುಗಿಸುವಿಕೆ ಮತ್ತು ಅಮಾನ್ಯಗೊಳಿಸುವಿಕೆ ಮೂಲಕ ದುರ್ಬಲತೆ ಕಾಲವನ್ನು ಮಿತಿಗೊಳಿಸು  
- **ಸಂಪ್ರೇಷಣಾ ಭದ್ರತೆ**: ಸೆಷನ್ ID ಲು ಕುಪ್ಪಿಸುವಿಕೆ ತಡೆಗಾಗಿ ಎಲ್ಲಾ ಸಂವಹನಗಳಿಗೆ ಕಡ್ಡಾಯ HTTPS ಬಳಸಿ  

### ಗೊಂದಲಗೊಳ್ಳಿರುವ ದಿಪತಿ ಸಮಸ್ಯೆ

**ಗೊಂದಲಗೊಳ್ಳುವ ದಿಪತಿ ಸಮಸ್ಯೆ** MCP ಸರ್ವರ್‌ಗಳು ಗ್ರಾಹಕರ ಮತ್ತು ಮೂರನೇ ಪಕ್ಷದ ಸೇವೆಗಳ ಮಧ್ಯದಲ್ಲಿ ದೃಢೀಕರಣ ಪ್ರಾಕ್ಸಿಗಳು ಆಗಿರುವಾಗ ಸಂಭವಿಸುತ್ತದೆ. ಇದರ ಮೂಲಕ ಸ್ಥಿರ ಗ್ರಾಹಕ ID ದುರುಪಯೋಗದಿಂದ ಅಧಿಕೃತತೆ ಮೀರುವ ಅವಕಾಶಗಳು ಸೃಷ್ಟಿಯಾಗುತ್ತವೆ.

#### **ದಾಳಿ ಯಂತ್ರಗಳ ಮತ್ತು ಅಪಾಯಗಳು**

- **ಕೂಕಿ ಆಧಾರಿತ ಅನುಮತಿ ಪರಿಗಣನೆ ತಪ್ಪಿಸುವಿಕೆ**: ಹಿಂದಿನ ಬಳಕೆದಾರ ದೃಢೀಕರಣವು ಅನುಮತಿ ಕೂಕಿಗಳನ್ನು ರಚಿಸುತ್ತದೆ, ಅವುಗಳನ್ನು ದಾಳಿಗಾರರು ಕಪಟ授权 ವಿನಂತಿಯೊಂದಿಗೆ ವಿನ್ಯಾಸಿತ ರೀಡೈರೆಕ್ಟ್ URI ಗಳ ಮೂಲಕ ದುರುಪಯೋಗ ಮಾಡುತ್ತಾರೆ  
- **ಅಧಿಕೃತ ಕೋಡ್ ಕಳ್ಳತನ**: ಇರುವ ಅನುಮತಿ ಕookyಗಳು ಅನುಮತಿ ಸ್ಕ್ರೀನ್ ಗಳನ್ನು ಬಿಟ್ಟುಹೋಗಿ, ದಾಳಿಗಾರಿಗೆ ನಿಯಂತ್ರಿತ ಅ೦ತಿಮವೊಂದಕ್ಕೆ ಕೋಡ್ ಗಳನ್ನು ರೀಡೈರೆಕ್ಟ್ ಮಾಡಬಹುದು  
- **ಅನಧಿಕೃತ API ಪ್ರವೇಶ**: ಕಳ್ಳತನವಾದ ಅನುಮತಿಸಿದ್ದ ಕೋಡ್‌ಗಳು ಟೋಕನ್ ವಿನಿಮಯ ಮತ್ತು ಬಳಕೆದಾರ ಮರುಪ್ರತಿನಿಧತ್ತನ್ನು ಸ್ಪಷ್ಟ ಅನುಮೋದನೆಯನ್ನು ಇಲ್ಲದೆ ಮಾಡಬಹುದು  

#### **ತಡೆ ತಂತ್ರಗಳು**

**ಕಡ್ಡಾಯ ನಿಯಂತ್ರಣಗಳು:**  
- **ಸ್ಪಷ್ಟ ಅನುಮತಿ ಅವಶ್ಯಕತೆಗಳು**: ಸ್ಥಿರ ಗ್ರಾಹಕ ID ಗಳನ್ನು ಬಳಸಿ MCP ಪ್ರಾಕ್ಸಿ ಸರ್ವರ್‌ಗಳು ಪ್ರತಿ ಡೈನಾಮಿಕ್ ನೋಂದಾಯಿತ ಗ್ರಾಹಕದಿಂದ ಬಳಕೆದಾರ ಅನುಮತಿಯನ್ನು 반드시 ಪಡೆಯಬೇಕು  
- **OAuth 2.1 ಭದ್ರತೆ ಅನುಷ್ಠಾನ**: ಪ್ರತಿ ಅನುಮತಿ ವಿನಂತಿಗೆ PKCE (ಕೋಡ್ ವಿನಿಮಯದ ಸಾಬೀತು ಕೀಲಿಯ ಪ್ರೂಫ್) ಸೇರಿದಂತೆ ಪ್ರಸ್ತುತ OAuth ಭದ್ರತಾ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳನ್ನು ಅನುಸರಿಸಿ  
- **ಕಟ್ಟುನಿಟ್ಟಾದ ಗ್ರಾಹಕ ಪರಿಶೀಲನೆ**: ದುರುಪಯೋಗ ತಡೆಗಾಗಿ ರೀಡೈರೆಕ್ಟ್ URI ಗಳು ಮತ್ತು ಗ್ರಾಹಕ ಗುರುತುಗಳ ಕಠಿಣ ಪರಿಶೀಲನೆ ಮಾಡು  

### ಟೋಕನ್ ಪಾಸ್ತ್‌ಥ್ರೂ ದುರ್ಬಲತೆಗಳು  

**ಟೋಕನ್ ಪಾಸ್ತ್‌ಥ್ರೂ** MCP ಸರ್ವರ್‌ಗಳು ಗ್ರಾಹಕ ಟೋಕನ್‌ಗಳನ್ನು ಸರಿಯಾದ ಪರಿಶೀಲನೆ ಇಲ್ಲದೆ ಅಂಗೀಕರಿಸಿ, ಅವುಗಳನ್ನು ನಂತರದ API ಗೆ ಮುಂದಿಸುವುದು ಎಂಬ ಸ್ಪಷ್ಟ ವಿರೋಧ ಮೌಲ್ಯ ತತ್ವವಾಗಿದೆ, ಇದು MCP ಪ್ರಮಾಣೀಕರಣ ನಿರ್ದಿಷ್ಟತೆಗಳನ್ನು ಉಲ್ಲಂಘಿಸುತ್ತದೆ.

#### **ಭದ್ರತಾ ಪರಿಣಾಮಗಳು**

- **ನಿಯಂತ್ರಣ ತಿರಸ್ಕರಣೆ**: ನೇರ ಗ್ರಾಹಕ-API ಟೋಕನ್ ಬಳಕೆ ಪ್ರಮುಖ ಪ್ರಮಾಣಿತ ಗಡಿಬಂಧನೆಗಳು, ಪರಿಶೀಲನೆ, ಮತ್ತು ಮೇಲ್ವಿಚಾರಣೆ ನಿಯಂತ್ರಣಗಳನ್ನು ತಪ್ಪಿಸುತ್ತದೆ  
- **ಆಡಿಟ್ ಟ್ರೇಲ್ ಕುಗ್ಗಿಸುವಿಕೆ**: ಮೇಲ್ದಿಕ್ಕಿನ ಟೋಕನ್ಮೂಲಗಳಿಂದ ಗ್ರಾಹಕ ಗುರುತಿಸುವಿಕೆ ಅಸಾಧ್ಯವಾಗುತ್ತದೆ, ಇದು ಘಟನೆಗಳ ತನಿಖೆಯನ್ನು ತಡೆಹಿಡಿಯುತ್ತದೆ  
- **ಪ್ರಾಕ್ಸಿ ಆಧಾರಿತ ಡೇಟಾ ದೋಲನ**: ಪರಿಶೀಲನೆ ಇಲ್ಲದ ಟೋಕನಗಳು ದೋಷಕಾರರಾಗಿ ಸರ್ವರ್‌ಗಳನ್ನು ಅನಧಿಕೃತ ಡೇಟಾ ಪ್ರವೇಶಕ್ಕೆ ಪ್ರಾಕ್ಸಿಗಳಾಗಿ ಉಪಯೋಗಿಸುವುದಕ್ಕೆ ಅವಕಾಶ ನೀಡುತ್ತವೆ  
- **ನಂಬಿಕೆ ಗಡಿಬಂಧನೆ ಉಲ್ಲಂಘನೆಗಳು**: ಟೋಕನ್ ಮೂಲವನ್ನು ಪರಿಶೀಲಿಸಲು ಸಾಧ್ಯವಾಗದಾಗ, ನಂತರದ ಸೇವೆಗಳ ನಂಬಿಕೆ ಊಹೆಗಳನ್ನು ಉಲ್ಲಂಘಿಸಬಹುದು  
- **ಬಹು-ಸೇವಾ ದಾಳಿ ವಿಸ್ತರಣೆ**: ಅಪಹೃತ ಟೋಕನ್ಗಳನ್ನು ಅನೇಕ ಸೇವೆಗಳಷ್ಟು ಒಪ್ಪಿಕೊಂಡರೆ ವ್ಯಾಪಕ ಚಲನೆ ಸಾದ್ಯವಿದೆ  

#### **ಅಗತ್ಯ ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳು**

**ಪರಿಹಾರ ವಿನಿಮಯಕ್ಕಿಲ್ಲದ ಅಗತ್ಯಗಳು:**  
- **ಟೋಕನ್ ಪರಿಶೀಲನೆ**: ಹಕ್ಕುಸ್ಥಾಪಿತವಾಗಿ MCP ಸರ್ವರ್ ಗಾಗಿ ನೀಡಲ್ಪಟ್ಟ ಟೋಕನ್ಗಳನ್ನೇ ಮಾತ್ರ ಒಪ್ಪಿಕೊಳ್ಳಬೇಕು, ಇತರೆಗಳು ಖಂಡಿತವಾಗಿ ತಿರಸ್ಕರಿಸಬೇಕು  
- **ಪ್ರೇಕ್ಷಕ ಖಾತ್ರಿಗೊಳಿಕೆ**: ಟೋಕನ್ ಪ್ರೇಕ್ಷಕರ ಹಕ್ಕು MCP ಸರ್ವರ್ ಗುರುತಿನ ಜೊತೆಗೆ ಹೊಂದಿಕೆಯಾಗಿದೆಯೇ ಎಂದು ಶಾಶ್ವತ ಪರಿಶೀಲನೆ ಮಾಡಬೇಕು  
- **ಸರಿಯಾದ ಟೋಕನ್ ಜೀವನಚಕ್ರ**: ಸುರಕ್ಷಿತ ತಿರುಗಿಸುವಿಕೆಯೊಂದಿಗೆ ಸಟಾವುಕಾಲಾವಧಿ ಹೊಂದಿರುವ ಪ್ರವೇಶ ಟೋಕನ್ಗಳನ್ನು ಅನುಷ್ಠಾನ ಮಾಡು  


## AI ವ್ಯವಸ್ಥೆಗಳಿಗೆ ಸರಬರಾಜು ಸರಪಳಿ ಭದ್ರತೆ

ಸರಬರಾಜು ಸರಪಳಿಯ ಭದ್ರತೆ ಸಾಂಪ್ರದಾಯಿಕ ಸಾಫ್ಟ್‌ವೇರ್ ಅವಲಂಬನೆಗಳಿಗೆ ಮೀರಿ ಸಂಪೂರ್ಣ AI ಪರಿಸರವನ್ನು ವ್ಯಾಪಿಸಿದೆ. ಆಧುನಿಕ MCP ಅನುಷ್ಠಾನಗಳು ಎಲ್ಲಾ AI ಸಂಬಂಧಿತ ಘಟಕಗಳನ್ನು ಕಟ್ಟುನಿಟ್ಟಾಗಿ ಪರಿಶೀಲಿಸಿ ಮೇಲ್ವಿಚಾರಣೆ ಮಾಡಬೇಕು, ಏಕೆಂದರೆ ಪ್ರತಿ ಘಟಕವು ಸಂಸ್ಥಾನದ ಸಮಗ್ರತೆಯನ್ನು ಹಾನಿಪಡಿಸುವ ಸಾಧ್ಯತೆಗಳನ್ನು ತರುತ್ತದೆ.

### ವಿಸ್ತರಿಸಿದ AI ಸರಬರಾಜು ಸರಪಳಿ ಘಟಕಗಳು

**ಸಾಂಪ್ರದಾಯಿಕ ಸಾಫ್ಟ್‌ವೇರ್ ಅವಲಂಬನೆಗಳು:**  
- ಓಪನ್ ಸೋರ್ಸ್ ಲೈಬ್ರರಿಗಳು ಮತ್ತು ಫ್ರೇಮ್‌ವರ್ಕ್‌ಗಳು  
- ಕಂಟೈನರ್ ಚಿತ್ರಗಳು ಮತ್ತು ಮೂಲ ವ್ಯವಸ್ಥೆಗಳು  
- ಅಭಿವೃದ್ಧಿ ಉಪಕರಣಗಳು ಮತ್ತು ನಿರ್ಮಾಣ ಮೇಲ್ವಿಚಾರಣೆ  
- ಮೂಲಸೌಕರ್ಯ ಘಟಕಗಳು ಮತ್ತು ಸೇವೆಗಳು  

**AI-ನಿರ್ದಿಷ್ಟ ಸರಬರಾಜು ಸರಪಳಿ ಅಂಶಗಳು:**  
- **ಅಡಿದ ಮಾದರಿಗಳು**: ವಿವಿಧ ಪ್ರೊವೈಡರ್‌ಗಳಿಂದ ಪೂರ್ವ-ತರಬೇತಿ ನೀಡಲಾದ ಮಾದರಿಗಳು, ಮೂಲಸ್ಥಿತಿಯನ್ನು ಪರಿಶೀಲಿಸುವ ಅಗತ್ಯವಿದೆ  
- **ಎಂಬೆಡಿಂಗ್ ಸೇವೆಗಳು**: ಬಾಹ್ಯ ವೆಕ್ಟರೀಕರಣ ಮತ್ತು ಶಾಂಕೂಮಂತ್ರಣ ಸೇವೆಗಳು  
- **ಸಂದರ್ಭ ಪ್ರೊವೈಡರ್‌ಗಳು**: ಡೇಟಾ ಮೂಲಗಳು, ಜ್ಞಾನ ಆಧಾರಗಳು ಮತ್ತು ದಾಖಲಾತಿ ಸಂಗ್ರಹಾಲಯಗಳು  
- **ಮೂರನೇ ಪಕ್ಷದ API ಗಳು**: ಹೊರಗಿನ AI ಸೇವೆಗಳು, ML ಪೈಪ್‌ಲೈನ್‌ಗಳು, ಮತ್ತು ಡೇಟಾ ಪ್ರಕ್ರಿಯೆ.Endpoint‌ಗಳು  
- **ಮಾದರಿ ಕಲೆಗಳು**: ತೂಕ, ಸಂರಚನೆಗಳು, ಮತ್ತು ಸೂಕ್ಷ್ಮ ನಿಗದಿತ ಮಾದರಿ ಬದಲಾವಣೆಗಳು  
- **ತರಬೇತಿ ಡೇಟಾ ಮೂಲಗಳು**: ಮಾದರಿ ತರಬೇತಿಗೆ ಬಳಸಲಾದ ಡೇಟಾಸೆಟ್‌ಗಳು  

### ಸಂಪೂರ್ಣ ಸರಬರಾಜು ಸರಪಳಿ ಭದ್ರತಾ ತಂತ್ರಜ್ಞಾನ

#### **ಘಟಕ ಪರಿಶೀಲನೆ ಮತ್ತು ನಂಬಿಕೆಗೆಣಿಕೆ**  
- **ಮೂಲಸ್ಥಿತಿ ಪರಿಶೀಲನೆ**: ಆಧಾರ, ಪರವಾನಗಿ, ಮತ್ತು ಒಟ್ಟು AI ಘಟಕಗಳ ಶುದ್ಧತೆಯನ್ನು ಆಗಮಿಸುವ ಮೊದಲು ಪರಿಶೀಲಿಸು  
- **ಭದ್ರತಾ ವಿಮರ್ಶೆ**: ಮಾದರಿ, ಡೇಟಾ ಮೂಲಗಳು ಮತ್ತು AI ಸೇವೆಗಳ ಭದ್ರತಾ ದುರ್ಬಲತೆಗಳ ಸ್ಕ್ಯಾನ್ ಮತ್ತು ವಿಮರ್ಶೆ  
- **ಪ್ರತಿಷ್ಠೆ ವಿಶ್ಲೇಷಣೆ**: AI ಸೇವಾ ಪ್ರೊವೈಡರ್‌ಗಳ ಭದ್ರತಾ ದಾಖಲೆಯಿವು ಮತ್ತು ಅಭ್ಯಾಸಗಳು  
- **ಅನುಕೂಲತೆ ಖಾತ್ರಿ**: ಎಲ್ಲಾ ಘಟಕಗಳು ಸಂಸ್ಥೆಗಳ ಭದ್ರತಾ ಮತ್ತು ನಿಯಂತ್ರಣ ಅವಶ್ಯಕತೆಗಳನ್ನು ಪೂರೈಸುತ್ತವೆ ಎಂದು ಖಚಿತಪಡಿಸು  

#### **ಭದ್ರಿತ ಜಾರಿಗೊಳಿಸುವಿಕೆ ಪೈಪ್‌ಲೈನ್‌ಗಳು**  
- **ಸ್ವಯಂಚಾಲಿತ CI/CD ಭದ್ರತೆ**: ಸ್ವಯಂಚಾಲಿತ ಜಾರಿಗೊಳಿಸುವಿಕೆಯ ಪೈಪ್‌ಲೈನ್ ಗಳಲ್ಲಿ ಭದ್ರತಾ ಸ್ಕ್ಯಾನಿಂಗ್ ಏಕೀಕರಣ  
- **ಕಲೆ ಶುದ್ಧತೆ**: ಎಲ್ಲಾ ಜಾರಿಗೊಳ್ಳುವ ಕಲೆಗಳ (ಕೋಡ್, ಮಾದರಿ, ಸಂರಚನೆಗಳು) ಕ್ರಿಪ್ಟೋಗ್ರಾಫಿಕ್ ಪರಿಶೀಲನೆ  
- **ದರ್ಜೆಯ ಜಾರಿಗೊಳಿಸುವಿಕೆ**: ಪ್ರತಿಯೊಂದರ ಹಂತದಲ್ಲಿ ಭದ್ರತಾ ಪರಿಶೀಲನೆ ಸೇರಿದಂತೆ ಕ್ರಮೇಣ ಜಾರಿಗೊಳಿಸುವಿಕೆ ಮಾರ್ಗಗಳು  
- **ಭದ್ರಿತ ಕಲೆ ಸಂಗ್ರಹಾಲಯಗಳು**: ಖಚಿತಪಡಿಸಿದ, ಭದ್ರವಾದ ಕಲೆ ಸಂಗ್ರಹಾಲಯಗಳಿಂದ ಮಾತ್ರ ಜಾರಿಗೊಳಿಸುವಿಕೆ  

#### **ನಿರಂತರ ಮೇಲ್ವಿಚಾರಣೆ ಮತ್ತು ಪ್ರತಿಕ್ರಿಯೆ**  
- **ಅವಲಂಬನೆ ಸ್ಕ್ಯಾನಿಂಗ್**: ಎಲ್ಲಾ ಸಾಫ್ಟ್‌ವೇರ್ ಮತ್ತು AI ಘಟಕ ಅವಲಂಬನೆಗಳ ಮೇಲೆ ನಿರಂತರ ದುರ್ಬಲತೆ ತಕ್ಷಣ ರಿಪೋರ್ಟ್ ಮಾಡುವುದು  
- **ಮಾದರಿ ಮೇಲ್ವಿಚಾರಣೆ**: ಮodelನ ವರ್ತನೆ, ಕಾರ್ಯಕ್ಷಮತೆ ಡ್ರಿಫ್ಟ್, ಮತ್ತು ಭದ್ರತಾ ಅಸಾಮಾನ್ಯತೆಗಳ ನಿರಂತರ ವಿಶ್ಲೇಷಣೆ  
- **ಸೇವೆ ಆರೋಗ್ಯ ಟ್ರ್ಯಾಕಿಂಗ್**: ಬಾಹ್ಯ AI ಸೇವೆಗಳ ಲಭ್ಯತೆ, ಭದ್ರತಾ ಘಟನೆಗಳು ಮತ್ತು ನೀತಿ ಬದಲಾವಣೆಗಳ ಮೇಲ್ವಿಚಾರಣೆ  
- **ಭೀತಿ ಬೂತ ಮಾಹಿತಿ ಏಕೀಕರಣ**: AI ಮತ್ತು ML ಭದ್ರತಾ ಅಪಾಯಗಳಿಗೆ ನಿರ್ದಿಷ್ಟ ಭೀತಿದಾಯಕ ಫೀಡ್ಸ್‌ಗಳನ್ನು ಸಂಯೋಜನೆ  

#### **ಪ್ರವೇಶ ನಿಯಂತ್ರಣ ಮತ್ತು ಕನಿಷ್ಟ ಹಕ್ಕುಪ್ರದಾನ**  
- **ಘಟಕ ಮಟ್ಟದ ಅನುಮೋದನೆಗಳು**: ವ್ಯಾಪಾರದ ಅವಶ್ಯಕತೆಯ ಮೇರೆಗೆ ಮಾದರಿ, ಡೇಟಾ ಮತ್ತು ಸೇವೆಗಳ ಪ್ರವೇಶವನ್ನು ನಿಯಂತ್ರಿಸಿ  
- **ಸೇವಾ ಖಾತೆ ನಿರ್ವಹಣೆ**: ಕನಿಷ್ಠ ಅಗತ್ಯ ಅನುಮತಿಗಳೊಂದಿಗೆ ಸಮರ್ಪಕ ವ್ಯವಸಾಯ ಖಾತೆಗಳ ಅನುಷ್ಠಾನ  
- **ಜಾಲ ವಿಭಾಗೀಕರಣ**: AI ಘಟಕಗಳನ್ನು ನಿರ್ಬಂಧಿಸಿ ಮತ್ತು ಸೇವೆಗಳ ನಡುವೆ ಜಾಲ ಪ್ರವೇಶಗಳನ್ನು ಕಡಿಮೆ ಮಾಡಿ  
- **API ಗೇಟ್ವೇ ನಿಯಂತ್ರಣಗಳು**: ಬಾಹ್ಯ AI ಸೇವೆಗಳ ಪ್ರವೇಶ ನಿಯಂತ್ರಣ ಮತ್ತು ಮೇಲ್ವಿಚಾರಣೆಗಾಗಿ ಕೇಂದ್ರಿತ API ಗೇಟ್ವೇಗಳನ್ನು ಬಳಸಿ  

#### **ಘಟನೆ ಪ್ರತಿಕ್ರಿಯೆ ಮತ್ತು ಪುನಃಪ್ರಾಪ್ತಿ**  
- **ತ್ವರಿತ ಪ್ರತಿಕ್ರಿಯೆ ಪ್ರಕ್ರಿಯೆಗಳು**: ಕುಗ್ಗಿದ AI ಘಟಕಗಳನ್ನು ತಡೆಬಿಡುವ ಅಥವಾ ಬದಲಾಯಿಸುವ ಸ್ಥಾಪಿತ ಪ್ರಕ್ರಿಯೆಗಳು  
- **ಪ್ರಮಾಣಪತ್ರ ತಿರುಗಿಸುವಿಕೆ**: ರಹಸ್ಯಗಳು, API ಕೀಲಿಗಳು ಮತ್ತು ಸೇವಾ認ಪತ್ರಗಳನ್ನು ತಿರುಗಿಸುವ ಸ್ವಯಂಚಾಲಿತ ವ್ಯವಸ್ಥೆಗಳು  
- **ರೋಲ್ಬ್ಯಾಕ್ ಸಾಮರ್ಥ್ಯ**: ಯಶಸ್ವಿಯಾಗಿ ಪೂರ್ವದ ತಿಳಿದ-ಉತ್ತಮ ಸಂಸ್ಕರಣೆಗಳಿಗೆ ತ್ವರಿತವಾಗಿ ಹಿಂದಕ್ಕೆ ಹೋಗುವ ಸಾಮರ್ಥ್ಯ  
- **ಸರಬರಾಜು ಸರಪಳಿ ಲೋಪ ಪುನಃಪ್ರಾಪ್ತಿ**: ಮೇಲ್ದಿಕ್ಕಿನ AI ಸೇವೆಗಳ ಕುಗ್ಗುವಿಕೆಗಳಿಗೆ ಸ್ಪಷ್ಟ ಪ್ರತ್ಯುತ್ತರ ಕ್ರಮಗಳು

- **AI-ನಿರ್ದಿಷ್ಟ ರಕ್ಷಣೆಗಳು**: [OWASP Top 10 for LLMs](https://genai.owasp.org/download/43299/?tmstv=1731900559) ನಿಯಂತ್ರಣಗಳ ಅನುಷ್ಠಾನ  
- **ಭದ್ರಿತ ರಹಸ್ಯ ನಿರ್ವಹಣೆ**: ಟೋಕನ್, API ಕೀಲಿಗಳು ಮತ್ತು ಸಂವೇದನಶೀಲ ಸಂರಚನಾ ಡೇಟಾಗಾಗಿ ವ್ಯವಹಾರ ವಾಲ್‌ಟ್‌ಗಳನ್ನು ಬಳಸಿ  
- **ಅಂತ್ಯದಿಂದ ಅಂತ್ಯದEncryption**: ಎಲ್ಲಾ ಅಪ್ಲಿಕೇಶನ್ ಘಟಕಗಳು ಮತ್ತು ಡೇಟಾ ಹರಿವುಗಳ ಭದ್ರ ಸಂವಹನಗಳ ಅನುಷ್ಠಾನ  
- **ಇನ್‌ಪುಟ್ ಪರಿಶೀಲನೆ**: ಎಲ್ಲಾ ಬಳಕೆದಾರ ಇನ್‌ಪುಟ್‌ಗಳು, API ಪ್ಯಾರಾಮೀಟರ್‌ಗಳು ಮತ್ತು ಡೇಟಾ ಮೂಲಗಳ ಕಟ್ಟುನಿಟ್ಟಾದ ಪರಿಶೀಲನೆ  

#### **ಮೂಲಸೌಕರ್ಯ ಮುಡುಪಾಗಿಸುವಿಕೆ**  
- **ಬಹು-ಘಟಕ ಪರಿಶೀಲನೆ**: ಎಲ್ಲಾ ನಿರ್ವಾಹಕ ಮತ್ತು ಸೇವಾ ಖಾತೆಗಳಿಗೆ ಕಡ್ಡಾಯ MFA  
- **ಪ್ಯಾಚ್ ನಿರ್ವಹಣೆ**: ನಿರಂತರ, ಸಮಯೋಚಿತ ಪ್ಯಾಚಿಂಗ್ ಮಾಡಿ ಸಂಚಿಕೆಗಳು, ಫ್ರೇಮ್‌ವರ್ಕ್ ಗಳ ಮತ್ತು ಅವಲಂಬನೆಗಳಿಗೆ  
- **ಗುರುತಿನ ಪ್ರೊವೈಡರ್ ಏಕೀಕರಣ**: ಸಂಸ್ಥೆ ಗುರುತಿನ ಪ್ರೊವೈಡರ್‌ಗಳ ಮೂಲಕ ಕೇಂದ್ರಿತ ಗುರುತಿನ ನಿರ್ವಹಣೆ (Microsoft Entra ID, ಆಕ್ಟಿವ್ ಡೈರೆಕ್ಟರಿ)  
- **ಜಾಲ ವಿಭಾಗೀಕರಣ**: MCP ಘಟಕಗಳನ್ನು ತರ್ಕಸಮ್ಮತವಾಗಿ ಪ್ರತ್ಯೇಕಿಸಿ ದಪ್ಪಕಕಾಲ ಚಲನೆ ಅನ್ನು ನಿಯಂತ್ರಿಸು  
- **ಕನಿಷ್ಟ ಹಕ್ಕು ತತ್ವ**: ಎಲ್ಲಾ ವ್ಯವಸ್ಥಾ ಘಟಕಗಳು ಮತ್ತು ಖಾತೆಗಳ ಕನಿಷ್ಟ ಅಗತ್ಯ ಅನುಮತಿಗಳು  

#### **ಭದ್ರತಾ ಮೇಲ್ವಿಚಾರಣೆ ಮತ್ತು ಪತ್ತೆ**  
- **ಸಂಕೀರ್ಣ ಲಾಗಿಂಗ್**: AI ಅಪ್ಲಿಕೇಶನ್ ಚಟುವಟಿಕೆಗಳ ನಿಖರ ಲಾಗಿಂಗ್, MCP ಗ್ರಾಹಕ-ಸರ್ವರ್ ಸಂವಹನಗಳು ಸೇರಿ  
- **SIEM ಏಕೀಕರಣ**: ವಿಶಿಷ್ಟತೆಯನ್ನು ಪತ್ತೆ ಮಾಡಲು ಕೇಂದ್ರಿತ ಭದ್ರತಾ ಮಾಹಿತಿ ಮತ್ತು ಘಟನೆ ನಿರ್ವಹಣೆ  
- **ವ್ಯವಹಾರ ವಿಶ್ಲೇಷಣೆ**: ವ್ಯವಸ್ಥೆ ಮತ್ತು ಬಳಕೆದಾರ ವರ್ತನೆಯ ಅಸಾಮಾನ್ಯ ಮಾದರಿಗಳನ್ನು ಪತ್ತೆ ಮಾಡಲಿರುವ AI ಚಾಲಿತ ಮೇಲ್ವಿಚಾರಣೆ  
- **ಭೀತಿದಾಯಕ ಮಾಹಿತಿ**: ಬಾಹ್ಯ ಭೀತಿದಾಯಕ ಫೀಡ್ಸ್ ಮತ್ತು ದೃಢೀಕರಣ ಸೂಚಕಗಳ ಏಕೀಕರಣ  
- **ಘಟನೆ ಪ್ರತಿಕ್ರಿಯೆ**: ಭದ್ರತಾ ಘಟನೆ ಪತ್ತೆ, ಪ್ರತಿಕ್ರಿಯೆ ಮತ್ತು ಪುನಃಪ್ರಾಪ್ತಿಗೆ ಸ್ಪಷ್ಟ ಪ್ರಕ್ರಿಯೆಗಳು  

#### **ಶೂನ್ಯ ನಂಬಿಕೆ ಗಥನಶಿಲ್ಪ**  
- **ಯಾವಾಗಲೂ ನಂಬಿಕೆಯ ಇಲ್ಲ, ಯಾವಾಗಲೂ ಪರಿಶೀಲನೆ**: ಬಳಕೆದಾರರು, ಸಾಧನಗಳು ಮತ್ತು ಜಾಲ ಸಂಪರ್ಕಗಳ ನಿರಂತರ ಪರಿಶೀಲನೆ  
- **ಮೈಕ್ರೋ-ವಿಭಾಗೀಕರಣ**: ವೈಯಕ್ತಿಕ ಕಾರ್ಯಭಾರಗಳು ಮತ್ತು ಸೇವೆಗಳ ಪ್ರತ್ಯೇಕ ಈತನಂತ್ರ ನಿಯಂತ್ರಣಗಳು  
- **ಗುರುತಿನ ಕೇಂದ್ರಿತ ಭದ್ರತೆ**: ಜಾಲ ಸ್ಥಳವಲ್ಲದೆ ಪರಿಶೀಲಿಸಿದ ಗುರುತಿನ ಮೇರೆಗೆ ರಕ್ಷಣಾ ನೀತಿಗಳು  
- **ನಿರಂತರ ಅಪಾಯ ಮೌಲ್ಯಾಂಕನ**: ಪ್ರಸ್ತುತ ಸಂಕಲಿತ ಸಂದರ್ಭ ಮತ್ತು ವರ್ತನೆಯ ಆಧಾರದ ಮೇಲೆ ಸಮರ್ಥ ಭದ್ರತಾ ಸ್ಥಿತಿ ಅಂದಾಜು  
- **ಶರತ್ವರಿತ ಪ್ರವೇಶ ನಿಯಂತ್ರಣ**: ಅಪಾಯಗಳ, ಸ್ಥಳ, ಮತ್ತು ಸಾಧನ ನಂಬಿಕೆಯನ್ನು ಆಧರಿಸಿ ಪ್ರವೇಶ ನಿಯಂತ್ರಣಗಳು  

### ಎಂಟರ್‌ಪ್ರೈಸ್ ಏಕೀಕರಣ ಮಾದರಿಗಳು

#### **Microsoft ಭದ್ರತಾ ಪರಿಸರ ಏಕೀಕರಣ**  
- **Microsoft Defender for Cloud**: ಸಂಪೂರ್ಣ ಕ್ಲೌಡ್ ಭದ್ರತಾ ಸ್ಥಿತಿ ನಿರ್ವಹಣೆ  
- **Azure Sentinel**: AI ಕಾರ್ಯಭಾರ ರಕ್ಷಣೆಗೆ ಕ್ಲೌಡ್-ಸ್ವಂತ SIEM ಮತ್ತು SOAR ಸಾಮರ್ಥ್ಯಗಳು  
- **Microsoft Entra ID**: ಉನ್ನತ ಉದ್ಯಮ ಗುರುತು ಮತ್ತು ಪ್ರವೇಶ ನಿರ್ವಹಣೆ ಶರತ್ವರಿತ ಪ್ರವೇಶ ನಿಯಮಗಳೊಂದಿಗೆ  
- **Azure Key Vault**: ಹಾರ್ಡ್‌ವೇರ್ ಭದ್ರತಾ ಘಟಕ ಬೆಂಬಲದ ಕೇಂದ್ರಿತ ರಹಸ್ಯ ನಿರ್ವಹಣೆ  
- **Microsoft Purview**: AI ಡೇಟಾ ಮೂಲಗಳು ಮತ್ತು ಕಾರ್ಯವಿಧಾನಗಳಿಗಾಗಿ ಡೇಟಾ ಆಡಳಿತ ಮತ್ತು ಅನುಕೂಲನೆ  

#### **ಅನುಕೂಲತೆ ಮತ್ತು ಆಡಳಿತ**  
- **ನಿಯಂತ್ರಣಕರ ಧ್ವನಿ ಹೊಂದಿರುವ ಕೈಗಾರಿಕಾ ಅನುಕೂಲತೆ**: MCP ಅನುಷ್ಠಾನಗಳು ಉದ್ಯಮ-ವಿಶೇಷ ಅನುಕೂಲನೆ ಅವಶ್ಯಕತೆಗಳಿಗೆ (GDPR, HIPAA, SOC 2) ಅನುಗುಣವಾಗಿರಬೇಕು  
- **ಡೇಟಾ ವರ್ಗೀಕರಣ**: AI ವ್ಯವಸ್ಥೆಗಳ ಮೂಲಕ ಪ್ರಕ್ರಿಯೆಗೊಳ್ಳುವ ಸಂವೇದನಶೀಲ ಡೇಟಾಗಳ ಸರಿಯಾದ ವರ್ಗೀಕರಣ ಮತ್ತು ನಿರ್ವಹಣೆ  
- **ಆಡಿಟ್ ಟ್ರೇಲ್ಸ್**: ನಿಯಂತ್ರಣ ಅನುಕೂಲತೆ ಮತ್ತು ಫೋರೆನ್ಸಿಕ್ ತನಿಖೆಗೆ ಸಂಪೂರ್ಣ ಲಾಗಿಂಗ್  
- **ಗೋಪ್ಯತೆ ನಿಯಂತ್ರಣಗಳು**: AI ವ್ಯವಸ್ಥೆಯ ವಿನ್ಯಾಸದಲ್ಲಿ ಗೌಪ್ಯತೆ-ದೃಷ್ಟಿ ತತ್ವಗಳ ಅನುಷ್ಠಾನ  
- **ಬದಲಾವಣೆ ನಿರ್ವಹಣೆ**: AI ವ್ಯವಸ್ಥೆ ತಿದ್ದುಪಡಿ ಭದ್ರತಾ ವಿಮರ್ಶೆಗಳಿಗೆ ಅಧಿಕೃತ ಪ್ರಕ್ರಿಯೆಗಳು  

ಈ ಆಧಾರಭೂತ ಅಭ್ಯಾಸಗಳು MCP-ನಿರ್ದಿಷ್ಟ ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳ ಪರಿಣಾಮಕಾರಿತ್ವವನ್ನು ಹೆಚ್ಚಿಸುವುದು ಮತ್ತು AI ಚಾಲಿತ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ ಸಮಗ್ರ ರಕ್ಷಣೆಯನ್ನು ಒದಗಿಸುವುದು.  

## ಪ್ರಮುಖ ಭದ್ರತಾ ಸಂಕ್ಷಿಪ್ತ ವಿವರಣೆಗಳು  

- **ತರತಮದ ಸುರಕ್ಷತಾ ಕ್ರಮ**: ಮೂಲಭೂತ ಸುರಕ್ಷತಾ ಅಭ್ಯಾಸಗಳನ್ನು (ಸುರಕ್ಷಿತ ಕೋಡಿಂಗ್, ಕನಿಷ್ಠ ಹಕ್ಕುಗಳು, ಸರಪಳಿ ಪರಿಶೀಲನೆ, ನಿರಂತರ ನಿಗಾದರ್ಶಣೆ) AI-ನಿರ್ದಿಷ್ಟ ನಿಯಂತ್ರಣಗಳೊಂದಿಗೆ ಸಂಯೋಜಿಸಿ ಸಂಪೂರ್ಣ ರಕ್ಷಣೆಗೆ

- **AI-ನಿರ್ದಿಷ್ಟ ಅಪಾಯ ಭೂದೃಶ್ಯ**: MCP ವ್ಯವಸ್ಥೆಗಳು ಅನನ್ಯ ಅಪಾಯಗಳನ್ನು ಎದುರಿಸುತ್ತವೆ, ಇದರಲ್ಲಿ ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್, ಸಾಧನ ವಿಷಪ್ರಯೋಗ, ಸೆಷನ್ ಹೈಜೆಕಿಂಗ್, ಗೊಂದಲದಲ್ಲಿರುವ ಡೆಪ್ಯೂಟಿ ಸಮಸ್ಯೆಗಳು, ಟೋಕನ್ ಪಾಸ್ತ್ರೂ ವಿಬಲತೆಗಳು ಮತ್ತು ಹೆಚ್ಚು ಹಕ್ಕುಗಳಿಗೆ ಮುಂಚಿತ ಪರಿಹಾರಗಳು ಅಗತ್ಯವಿವೆ

- **ಪ್ರಮಾಣೀಕರಣ ಮತ್ತು ಹಕ್ಕು ನಿರ್ವಹಣೆಯ ಶ್ರೇಷ್ಠತೆ**: ಬಾಹ್ಯ ಗುರುತಿಸುವಿಕೆ ಪ್ರದಾತರೆ (Microsoft Entra ID) ಬಳಸುವವುದು, ಸೂಕ್ತ ಟೋಕನ್ ಮಾನ್ಯತೆ ಜಾರಿಗೊಳಿಸುವುದು ಮತ್ತು ನಿಮ್ಮ MCP ಸರ್ವರ್‌ಗಾಗಿ ಸ್ಪಷ್ಟವಾಗಿ ನೀಡದ ಟೋಕನ್‌ಗಳನ್ನು ಎಂದೂ ಸ್ವೀಕರಿಸಬೇಡಿ

- **AI ದಾಳಿ ತಡೆಗಟ್ಟುವಿಕೆ**: Microsoft Prompt Shields ಹಾಗೂ Azure Content Safety ಅನ್ನು ಅನುಪರೋಕ್ಷ ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್ ಮತ್ತು ಸಾಧನ ವಿಷಪ್ರಯೋಗ ದಾಳಿಗಳಿಗೆ ವಿರುದ್ಧವಾಗಿ ನಿಯೋಜಿಸಿ, ಸಾಧನ ಮೆಟಾಡೇಟಾ ಮಾನ್ಯತೆ ಮಾಡು ಹಾಗೂ ಪ್ರತ್ಯೇಕ ಬದಲಾವಣೆಯ ಮೇಲ್ವಿಚಾರಣೆ ನಡಿಸಿ

- **ಸೆಷನ್ ಮತ್ತು ಸಾರಿಗೆ ಸುರಕ್ಷತೆ**: ಕ್ರಿಪ್ಟೋಗ್ರಾಫಿಕ್ ಸುರಕ್ಷಿತ, ಅನಿರ್ಣಾಯಕ ಸೆಷನ್ ಐಡಿಗಳನ್ನು ಬಳಸಿ, ಅವುಗಳನ್ನು ಬಳಕೆದಾರ ಗುರುತಿನೊಂದಿಗೆ ಬಂಧಿಸಿ, ಸೂಕ್ತ ಸೆಷನ್ ಜೀವನಚರ್ಯಾ ನಿರ್ವಹಣೆ ಜಾರಿಗೊಳಿಸಿ ಮತ್ತು ಪ್ರಮಾಣೀಕರಣಕ್ಕಾಗಿ ಸೆಷನ್‌ಗಳನ್ನು ಬಳಸಬೇಡಿ

- **OAuth ಸುರಕ್ಷತಾ ಶ್ರೇಷ್ಠ ಅಭ್ಯಾಸಗಳು**: ಸ್ಪಷ್ಟ ಬಳಕೆದಾರ ಅನುಮತಿಯನ್ನು ಸಕ್ರೀಯವಾಗಿ ನೋಂದಾಯಿತ ಕ್ಲೈಯಿಂಟ್‌ಗಳಿಗೆ ಒದಗಿಸುವ ಮೂಲಕ ಗೊಂದಲದಲ್ಲಿರುವ ಡೆಪ್ಯೂಟಿ ದಾಳಿಗಳನ್ನು ತಡೆಗಟ್ಟುವುದು, PKCE ಒಳಗೊಂಡ ಸೂಕ್ತ OAuth 2.1 ಜಾರಿಗೆ ಹಾಗೂ ಕಠಿಣ ರೀಡೈರೆಕ್ಟ್ URI ಮಾನ್ಯತೆ

- **ಟೋಕನ್ ಸುರಕ್ಷತಾ ಸ 원ರಗಳು**: ಟೋಕನ್ ಪಾಸ್ತ್ರೂ ವಿರೋಧಾಭ್ಯಾಸಗಳನ್ನು ತಪ್ಪಿಸಿ, ಟೋಕನ್ ಪ್ರೇಕ್ಷಣಾ ಹಕ್ಕುಗಳನ್ನು ಮಾನ್ಯಗೊಳಿಸಿ, ಸಣ್ಣ ಆಯುಷ್ಯ ಹೊಂದಿದ ಟೋಕನ್‌ಗಳನ್ನು ಸುರಕ್ಷಿತ ರೋಟೇಶನ್ ಸಹಿತ ಜಾರಿಗೊಳಿಸಿ ಮತ್ತು ಸ್ಪಷ್ಟ ವಿಶ್ವಾಸ ಗಡಿಗಳನ್ನು ಕಾಪಾಡಿ

- **ಸಂಪೂರ್ಣ ಸರಪಳಿ ಸುರಕ್ಷತೆ**: AI ಪರಿಸರದ ಎಲ್ಲಾ ಘಟಕಗಳನ್ನು (ಮಾದರಿಗಳು, ಹೆಮೆಯಿಂಗ್ಗಳು, ಸಂದರ್ಭ ಪ್ರದಾನಕರು, ಬಾಹ್ಯ APIಗಳು) ಸಂಪ್ರದಾಯಬದ್ಧ ಸಾಫ್ಟ್ವೆರ್ ಅವಲಂಬನೆಗಳಷ್ಟೇ ಗಂಭೀರವಾಗಿ ರಕ್ಷಿಸಿ

- **ತಿರುಗಿ ಬೆಳವಣಿಗೆ**: ವೇಗವಾಗಿ ಬೆಳೆದಿರುವ MCP ನಿರ್ದಿಷ್ಟತೆಯೊಂದಿಗೆ ನವೀಕೃತವಾಗಿರಿ, ಸುರಕ್ಷತಾ ಸಮುದಾಯ ಸ್ಥಿತಿಗತಿಗಳಲ್ಲಿ ಪಾಲ್ಗೊಳ್ಳಿ ಮತ್ತು ಪ್ರೋಟೋಕಾಲ್ ಅಭಿವೃದ್ಧಿಯಾಗುವಂತೆ অভিযೋಜಿತ ಸುರಕ್ಷತಾ ದೃಷ್ಟಿಕೋನಗಳನ್ನು ಕಾಪಾಡಿಕೊಳ್ಳಿ

- **Microsoft ಸುರಕ್ಷತಾ ಏಕೀಕರಣ**: Microsoft ನ ಸಮಗ್ರ ಸುರಕ್ಷತಾ ಪರಿಸರವನ್ನು (Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID) MCP ನಿಯೋಜನೆ ರಕ್ಷಣೆಗೆ ಬಳಸಿಕೊಳ್ಳಿ
