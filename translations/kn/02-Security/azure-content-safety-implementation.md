# MCP ಜೊತೆಗೆ Azure ವಿಷಯ ಸುರಕ್ಷತೆ ಅನುಷ್ಟಾನಗೊಳಿಸುವುದು

> **OWASP MCP ಅಪಾಯ ನಿರ್ವಹಣೆ**: [MCP06 - ಬಯಕೆಪ್ರವಾಹ ಹೇರಳಿಕೆ](https://microsoft.github.io/mcp-azure-security-guide/mcp/mcp06-prompt-injection/)

ಪ್ರಾಂಪ್ಟ್ ಸursorsion, ಉಪಕರಣ ವಿಷಜಡಿಕೆ, ಮತ್ತು ಇತರ AI-ನಿರ್ದಿಷ್ಟ ಭದ್ರತಾ ದುರ್ಬಲತೆಗಳ ವಿರುದ್ಧ MCP ಭದ್ರತೆಯನ್ನು ಮಗ್ಗುಗೊಳಿಸಲು, Azure ವಿಷಯ ಸುರಕ್ಷತೆಯನ್ನು ಜೋಡಿಸುವುದು ಬಹಳ ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ. ಈ ಅನುಷ್ಟಾನ ಮಾರ್ಗದರ್ಶಿ [MCP Security Summit Workshop (Sherpa)](https://azure-samples.github.io/sherpa/) ಕ್ಯಾಂಪ್ 3: I/O ಭದ್ರತೆ ಜೊತೆ ಹೊಂದಿಕೊಂಡಿದೆ.

## MCP ಸರ್ವರ್ ಜೊತೆ ಸಂಯೋಜನೆ

ನಿಮ್ಮ MCP ಸರ್ವರ್ ಜೊತೆ Azure ವಿಷಯ ಸುರಕ್ಷತೆಯನ್ನು ಸಂಯೋಜಿಸಲು, ವಿಷಯ ಸುರಕ್ಷತಾ ಫಿಲ್ಟರ್ ಅನ್ನು ಮಧ್ಯವರ್ತಿಯಾಗಿ ನಿಮ್ಮ ವಿನಂತಿ ಪ್ರಕ್ರಿಯೆ ಪೈಪ್ಲೈನಿನಲ್ಲಿ ಸೇರಿಸಿ:

1. ಸರ್ವರ್ ಪ್ರಾರಂಭಿಸುವಾಗ ಫಿಲ್ಟರ್ ಅನ್ನು ಆರಂಭಿಸಿ  
2. ಪ್ರತಿ ತಲುಪುವ ಉಪಕರಣ ವಿನಂತಿಯನ್ನು ಪ್ರಕ್ರಿಯೆಮಾಡುವ ಮುಂಚೆ ಪರಿಶೀಲಿಸಿ  
3. ಬಿಟ್ಟುಹೋಗುವ ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನೆಲ್ಲಾ ಕ್ಲೈಂಟ್‌ಗಳಿಗೆ ಮರಳಿಸುವಿಂದ ಮುಂಚೆ ಪರಿಶೀಲಿಸಿ  
4. ಸುರಕ್ಷತಾ ಉಲ್ಲಂಘನೆಗಳ ಮೇಲೆ ಲಾಗ್ ಮಾಡಿ ಮತ್ತು ಆಲರ್ಟ್ ನೀಡಿ  
5. ವಿಷಯ ಸುರಕ್ಷತಾ ಪರಿಶೀಲನೆಗಳಲ್ಲಿ ವೈಫಲ್ಯಗಳಿಗಾಗಿ ಸೂಕ್ತ ದೋಷ ನಿರ್ವಹಣೆ ಅನುಷ್ಟಾನಗೊಳಿಸಿ

ಇದರಿಂದ ಕೆಳಗಿನರಿಂದ ಬಲಿಷ್ಠ ರಕ್ಷಣೆಯನ್ನು ಒದಗಿಸುತ್ತದೆ:  
- ಪ್ರಾಂಪ್ಟ್ ಸ_cursorion ದಾಳಿ ಗಳು  
- ಉಪಕರಣ ವಿಷಜಡಿಕೆ ಪ್ರಯತ್ನಗಳು  
- ದುಷ್ಟ ಇನ್ಪುಟ್‌ಗಳರಿಂದ ಡೇಟಾ ಗಂಡಾಂತರ  
- ಹಾನಿಕಾರಕ ವಿಷಯ ಉತ್ಪಾದನೆ

## Azure ವಿಷಯ ಸುರಕ್ಷತಾ ಸಂಯೋಜನೆಗೆ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು

1. **ಕಸ್ಟಮ್ ಬ್ಲಾಕ್‌ಲಿಸ್ಟ್‌ಗಳು**: MCP ಇಂಜೆಕ್ಷನ್ ಮಾದರಿಗಳಿಗೆ ವಿಶೇಷವಾಗಿ ಕಸ್ಟಮ್ ಬ್ಲಾಕ್‌ಲಿಸ್ಟ್‌ಗಳನ್ನು ರಚಿಸಿ  
2. **ತೀವ್ರತೆ ಹೊಂದಿಕೆ**: ನಿಮ್ಮ ವಿಶಿಷ್ಟ ಬಳಕೆ ಮತ್ತು ಅಪಾಯ ಸಹನೆಗೆ ಅನುಗುಣವಾಗಿ ತೀವ್ರತೆ ಗಡಿಗಳನ್ನು ಸರಿಹೊಂದಿಸಿ  
3. **ಸಂಪೂರ್ಣ ವ್ಯಾಪ್ತಿ**: ಎಲ್ಲಾ ಇನ್ಪುಟ್ ಮತ್ತು ಔಟ್‌ಪುಟ್‌ಗಳಿಗೆ ವಿಷಯ ಸುರಕ್ಷತಾ ಪರಿಶೀಲನೆಗಳನ್ನು ಅನ್ವಯಿಸಿ  
4. **ಕಾರ್ಯಕ್ಷಮತೆ ಸರಳಗೊಳಿಕೆ**: ಪುನರಾವರ್ತಿತ ವಿಷಯ ಸುರಕ್ಷತಾ ಪರಿಶೀಲನೆಗಳಿಗೆ ಕેચಿಂಗ್ ಅನುಷ್ಟಾನಗೊಳಿಸುವುದನ್ನು ಪರಿಗಣಿಸಿ  
5. **ಫಾಲ್ಬ್ಯಾಕ್ ವ್ಯವಸ್ಥೆಗಳು**: ವಿಷಯ ಸುರಕ್ಷತಾ ಸೇವೆಗಳು ಲಭ್ಯವಿಲ್ಲದಾಗ ಸ್ಪಷ್ಟವಾದ ಫಾಲ್ಬ್ಯಾಕ್ ವರ್ತನೆಗಳನ್ನು ವ್ಯಾಖ್ಯಾನಿಸಿ  
6. **ಬಳಕೆದಾರ ಪ್ರತಿಕ್ರಿಯೆ**: ವಿಷಯದ ಸುರಕ್ಷತೆಯ ಕಾರಣದಿಂದ ಅಡ್ಡಿಪಡಿಸಿದಾಗ ಬಳಕೆದಾರರಿಗೆ ಸ್ಪಷ್ಟ ಪ್ರತಿಕ್ರಿಯೆ ಒದಗಿಸಿ  
7. **ನಿರಂತರ ಸುಧಾರಣೆ**: ಮುಂದುವರಿದ ಬೆದರಿಕೆಗಳ ಆಧಾರದಲ್ಲಿ ಬ್ಲಾಕ್‌ಲಿಸ್ಟ್‌ಗಳು ಮತ್ತು ಮಾದರಿಗಳನ್ನು ನಿಯಮಿತವಾಗಿ ನವಿಕರಣಗೊಳಿಸಿ
