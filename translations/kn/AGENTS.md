# AGENTS.md

## Project Overview

**MCP for Beginners** ಎಂಬುದು ಮೂಲಭೂತವಾಗಿ ಮಾದರಿ ಸಂಧರ್ಭ ಪ್ರೋಟೋಕಾಲ್ (MCP) ಅನ್ನು ಕಲಿಯಲು ಬಳಸಿಕೊಳ್ಳಬಹುದಾದ ಓಪನ್-ಸೋರ್ಸ್ ಶಿಕ್ಷಣದ ಪಠ್ಯಕ್ರಮವಾಗಿದ್ದು, ಇದು AI ಮಾದರಿಗಳು ಮತ್ತು ಕ್ಲೈಂಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳ ನಡುವೆ ಸಂವಹನವಿಗಾಗಿ ಮಾನ್ಯಮಾಡಲಾದ ರೂಪರೆಖೆ. ಈ ಸಂಗ್ರಹಣೆಯಲ್ಲಿ ಹಲವಾರು ಪ್ರೋಗ್ರಾಮಿಂಗ್ ಭಾಷೆಗಳಲ್ಲಿನ ಕೈಯಿಂದ ಕಾರ್ಯಮಾಡುವ ಕೋಡ್ ಉದಾಹರಣೆಗಳೊಂದಿಗೆ ಸಮಗ್ರ ಅಧ್ಯಯನ ಸಾಮಗ್ರಿಗಳನ್ನು ಒದಗಿಸಲಾಗುತ್ತದೆ.

### ಪ್ರಮುಖ ತಂತ್ರಜ್ಞಾನಗಳು

- **ಪ್ರೋಗ್ರಾಮಿಂಗ್ ಭಾಷೆಗಳು**: C#, ಜಾವಾ, ಜಾವಾಸ್ಕ್ರಿಪ್ಟ್, ಟೈಪ್‌ಸ್ಕ್ರಿಪ್ಟ್, ಪೈಥನ್, ರಸ್ಟ್  
- **ಫ್ರೇಮ್‌ವರ್ಕ್‌ಗಳು ಮತ್ತು SDKಗಳು**:  
  - MCP SDK (`@modelcontextprotocol/sdk`)  
  - ಸ್ಪ್ರಿಂಗ್ ಬುಟ್ (ಜావಾ)  
  - ಫಾಸ್ಟ್MCP (ಪೈಥನ್)  
  - ಲಾಂಗ್‌ಚೈನ್4j (ಜಾವಾ)  
- **ಡೇಟಾಬೇಸ್‌ಗಳು**: pgvector ವಿಸ್ತರಣೆ ಸಹಿತ PostgreSQL  
- **ಮেঘ ಪ್ಲಾಟ್‌ಫಾರ್ಮ್‌ಗಳು**: ಅಜೂರ್ (ಕಂಟೈನರ್ ಅಪ್ಲಿಕೇಶನ್ಸ್, ಓಪನ್‌ಎಐ, ಕಂಟೆಂಟ್ ಸೆಫ್ಟಿ, ಅಪ್ಲಿಕೇಶನ್ ಇನ್ಸೈಟ್‌ಗಳು)  
- **ಬಿಲ್ಡ್ ಉಪಕರಣಗಳು**: npm, Maven, pip, Cargo  
- **ದಾಖಲೆಗಳ ನಿರ್ವಹಣೆ**: ಬಹುಭಾಷಾ ಸ್ವಯಂಚಾಲಿತ ಅನುವಾದದೊಂದಿಗೆ Markdown

### ವಾಸ್ತುಶಿಲ್ಪ

- **೧೧ ಮೂಲ ಮೋಡ್ಯೂಲ್ಗಳು (00-11)**: ಆಧಾರದ ವಿಷಯದಿಂದ ಪ್ರಗತಿಪರ ವಿಷಯಗಳ ತನಕ ಕ್ರಮಬದ್ಧ ಅಧ್ಯಯನ ಮಾರ್ಗ  
- **ಕೈಯಿಂದ ಕಾರ್ಯಮಾಡುವ ಪ್ರಯೋಗಾಲಯಗಳು**: ಹಲವಾರು ಭಾಷೆಗಳ ಪೂರ್ಣ ಪರಿಹಾರ ಕೋಡ್ ಜೊತೆಗೆ ಉಪಯೋಗಾತ್ಮಕ ಅಭ್ಯಾಸಗಳು  
- **ನಮೂನಾ ಪ್ರಾಜೆಕ್ಟ್‌ಗಳು**: ಕಾರ್ಯನಿರ್ವಹಿಸುವ MCP ಸರ್ವರ್ ಮತ್ತು ಕ್ಲೈಂಟ್ ಅನುಷ್ಠಾನಗಳು  
- **ಅನುವಾದ ವ್ಯವಸ್ಥೆ**: GitHub ಆಕ್ಷನ್ಸ್ ಸ್ವಯಂಚಾಲಿತ ಬಹುಭಾಷಾ ಸಹಾಯಕಾರ  
- **ಚಿತ್ರ ಸಂಪತ್ತುಗಳು**: ಅನುವಾದಿತ ಆವೃತ್ತಿಗಳೊಂದಿಗೆ ಕೇಂದ್ರಿಕೃತ ಚಿತ್ರ ಸಂಗ್ರಹಣೆ

## Development Workflow

### Documentation Structure

- **ಮೋಡ್ಯೂಲ್ಗಳು 00-11**: ಕ್ರಮಬದ್ಧ ಮೂಲ ಪಠ್ಯಕ್ರಮ ವಿಷಯ  
- **translations/**: ಭಾಷಾ-ನಿರ್ದಿಷ್ಟ ಆವೃತ್ತಿಗಳು (ಸ್ವಯಂ ರಚಿತ, ದಯವಿಟ್ಟು ನೇರವಾಗಿ ಸಂಪಾದಿಸಬಾರದು)  
- **translated_images/**: ಸ್ಥಳೀಯೀಕೃತ ಚಿತ್ರ ಆವೃತ್ತಿಗಳು (ಸ್ವಯಂ ರಚಿತ)  
- **images/**: ಮೂಲ ಚಿತ್ರಗಳು ಮತ್ತು ಆಕೃತಿಗಳು

### Making Documentation Changes

1. ಮೂಲ ಮೋಡ್ಯೂಲ್ ಡೈರೆಕ್ಟರಿಗಳಲ್ಲಿ (00-11) ಇಂಗ್ಲಿಷ್ ಮಾರ್ಕ್‌ಡೌನ್ ಫೈಲ್‌ಗಳನ್ನು ಮಾತ್ರ ಸಂಪಾದಿಸಿ  
2. ಅಗತ್ಯವಿದ್ದರೆ `images/` ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ ಚಿತ್ರದ ನವೀಕರಣ ಮಾಡಿ  
3. ಸಹಯೋಗ-ಅನುವಾದಕ GitHub ಆಕ್ಷನ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಅನುವಾದಗಳನ್ನು ರಚಿಸುತ್ತದೆ  
4. ಮುಖ್ಯ ಶಾಖೆಗೆ ಪುಷ್ ಮಾಡಿದಾಗ ಅನುವಾದಗಳು ಪುನರುತ್ಪಾದನೆಯಾಗುತ್ತವೆ

### Working with Translations

- **ಸ್ವಯಂಚಾಲಿತ ಅನುವಾದ**: GitHub Actions ವರ್ಕ್ಫ್ಲೋ ಎಲ್ಲಾ ಅನುವಾದಗಳನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ

## Testing Instructions

### Documentation Validation

ಇದು ಪ್ರಧಾನವಾಗಿ ದಸ್ತಾವೇಜಿನ ಸಂಗ್ರಹಣೆ ಆಗಿರುವುದರಿಂದ, ಪರೀಕ್ಷೆಯು ಕೆಳಕಂಡವನ್ನೇ ಗಮನಿಸುತ್ತದೆ:  

1. **ಲಿಂಕ್ ಪರಿಶೀಲನೆ**: ಎಲ್ಲಾ ಆಂತರಿಕ ಲಿಂಕ್‌ಗಳು ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತವೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ  
```bash
# ಮುರಿದ ಮಾರ್ಕ್‌డೌನ್ ಲಿಂಕ್‌ಗಳು ಇದ್ದಾವೇ ಎಂದು ಪರಿಶೀಲಿಸಿ
find . -name "*.md" -type f | xargs grep -n "\[.*\](../../.*)"
```
  
2. **ಕೋಡ್ ಉದಾಹರಣೆಯ ಪರಿಶೀಲನೆ**: ಕೋಡ್ ಉದಾಹರಣೆಗಳು ಸಂकलಿತ/ಕಾರ್ಯನಿರ್ವಹಿಸುತಿವೆ ಎಂದು ಪರೀಕ್ಷಿಸಿ  
```bash
# ನಿರ್ದಿಷ್ಟ ಮಾದರಿಗೆ ನಾವಿಗೇಟ್ ಮಾಡಿ ಅದರ ಪರೀಕ್ಷೆಗಳನ್ನು ನಡೆಸಿ
cd 03-GettingStarted/samples/typescript
npm install && npm test
```
  
3. **Markdown ಲಿಂಟಿಂಗ್**: ಸ್ವರೂಪದ ಸಡಿಲಿತ ವಿವರಣೆ ಮಾಡುವುದು  
```bash
# ಅಗತ್ಯವಿದ್ದರೆ markdownlint ಅನ್ನು ಬಳಸಿರಿ
npx markdownlint-cli2 "**/*.md" "#node_modules"
```

## Code Style Guidelines

### Documentation Style

- ಸ್ಪಷ್ಟ ಮತ್ತು ಪ್ರಾರಂಭಿಕರಿಗೆ ಹೊಂದಾಣಿಕೆಯ ಭಾಷೆಯನ್ನು ಬಳಸಿರಿ  
- ಅನೇಕ ಭಾಷೆಗಳ ಕೋಡ್ ಉದಾಹರಣೆಗಳನ್ನು ಸೇರಿಸಿ  
- Markdown ಉತ್ತಮ ಆಚಾರಗಳನ್ನು ಅನುಸರಿಸಿ:  
  - ATX-ಶೈಲಿ ಹೆಡರ್‌ಗಳನ್ನು ಉಪಯೋಗಿಸಿ (`#` ಸಂಶ್ಲೇಷಣ)  
  - ಭಾಷಾ ಗುರುತು ಹೊಂದಿದ fenced code blocks ಬಳಸಿರಿ  
  - ಚಿತ್ರಗಳಿಗೆ ವಿವರಣಾತ್ಮಕ ಆಲ್ಟ್ ಪಠ್ಯ ಸೇರಿಸಿ  
  - ಸಾಲಿನ ಉದ್ದವನ್ನು ಫಲಾನುಪಾತವಾಗಿ ಇಡಿ (ಹೆಚ್ಚು ಪ್ರಮಾಣದ ನಿರ್ಬಂಧವಿಲ್ಲ, ಆದರೆ ಜಾಗರೂಕತೆ ಇರಲಿ)

### Code Sample Style

#### TypeScript/JavaScript  
- ES ಮೋಡ್ಯೂಲ್‌ಗಳನ್ನು ಬಳಸಿರಿ (`import`/`export`)  
- ಟೈಪ್‌ಸ್ಕ್ರಿಪ್ಟ್ ಸ್ಟ್ರಿಕ್ಟ್ ಮೋಡ್ ನಿಯಮ ಪಾಲಿಸಿ  
- ಪ್ರಕಾರ ಸೂಚಿಕೆಗಳನ್ನು ಸೇರಿಸಿ  
- ES2022 ಗುರಿಯಾಗಿರಲಿ  

#### Python  
- PEP 8 ಶೈಲಿ ಮಾರ್ಗಸೂಚಿಗಳನ್ನು ಅನುಸರಿಸಿ  
- ಅನುಕೂಲವಾದಲ್ಲಿ ಪ್ರಕಾರ ಸೂಚಕಗಳನ್ನು ಬಳಸಿರಿ  
- ಕ್ರಿಯೆಗಳು ಮತ್ತು ಶ್ರೇಣಿಗಳಿಗೆ ಡಾಕ್ಸ್ಟ್ರಿಂಗ್ ನೀಡಿರಿ  
- ಆಧುನಿಕ Python ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಬಳಸಿರಿ (3.8+)

#### Java  
- ಸ್ಪ್ರಿಂಗ್ ಬುಟ್ ನಿಯಮಗಳನ್ನು ಅನುಸರಿಸಿ  
- ಜಾವಾ 21 ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಬಳಸಿ  
- ಮ್ಯಾವೆನ್ ಪ್ರಾಜೆಕ್ಟ್ ಸರಂಜಾಮು ಅನುಸರಿಸಿ  
- ಜಾವಡಾಕ್ ಕಾಮೆಂಟ್‌ಗಳನ್ನು ಸೇರಿಸಿ

## Build and Deployment

### Documentation Deployment

ದಾಖಲೆ ಅನ್ವಯ GitHub Pages ಅಥವಾ ಸಮಾನ ಸೇವೆಯನ್ನು ಬಳಸುತ್ತದೆ (ಅಗತ್ಯವಿದ್ದರೆ). ಮುಖ್ಯ ಶಾಖೆಯಲ್ಲಿನ ಬದಲಾವಣೆಗಳು ಕೆಳಕಂಡ ಕ್ರಮವನ್ನು ಪ್ರೇರೇಪಿಸುತ್ತವೆ:

1. ಅನುವಾದ ಕಾರ್ಯಪ್ರವಾಹ (`.github/workflows/co-op-translator.yml`)  
2. ಎಲ್ಲಾ ಇಂಗ್ಲಿಷ್ ಮಾರ್ಕ್‌ಡೌನ್ ಫೈಲ್‌ಗಳ ಸ್ವಯಂಚಾಲಿತ ಅನುವಾದ  
3. ಅವಶ್ಯಕತೆ ಇದ್ದಲ್ಲಿ ಚಿತ್ರಸ್ಥಾನೀಕರಣ  

### No Build Process Required

ಈ ಸಂಗ್ರಹಣೆ ಮುಖ್ಯವಾಗಿ Markdown ದಾಖಲೆಗಳನ್ನು ಹೊಂದಿದೆ. ಕೋರ್ ಪಠ್ಯಕ್ರಮದ ವಿಷಯಕ್ಕಾಗಿ ಸಂಯೋಜನೆ ಅಥವಾ ಬಿಲ್ಡ್ ಪ್ರಕ್ರಿಯೆ ಅಗತ್ಯವಿಲ್ಲ.

### Sample Project Deployment

ಪ್ರತ್ಯೇಕ نمೂನಾ ಪ್ರಾಜೆಕ್ಟ್ ಗಾಗಿ ಬಿಡುಗಡೆ ಸೂಚನೆಗಳು ಇರಬಹುದು:  
- MCP ಸರ್ವರ್ ಬಿಡುಗಡೆಗೆ `03-GettingStarted/09-deployment/` ನೋಡಿ  
- ಅಜೂರ್ ಕಂಟೈನರ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳ ಬಿಡುಗಡೆ ಉದಾಹರಣೆಗಳು `11-MCPServerHandsOnLabs/` ನಲ್ಲಿ

## Contributing Guidelines

1. **ಫೋರ್ಕ್ ಮತ್ತು ಕ್ಲೋನ್**: ಸಂಗ್ರಹಣೆಯನ್ನು ಫೋರ್ಕ್ ಮಾಡಿ, ನಿಮ್ಮ ಫೋರ್ಕ್ ಅನ್ನು ಸ್ಥಳೀಯವಾಗಿ ಕ್ಲೋನ್ ಮಾಡಿಕೊಳ್ಳಿ  
2. **ಶಾಖೆ ರಚನೆ**: ಸ್ಪಷ್ಟವಾಗಿ ಶಾಖೆ ಹೆಸರನ್ನು ನೀಡಿ (ಉದಾ: `fix/typo-module-3`, `add/python-example`)  
3. **ಬದಲಾವಣೆ ಮಾಡುವುದು**: ಇಂಗ್ಲಿಷ್ ಮಾರ್ಕ್‌ಡೌನ್ ಫೈಲ್‌ಗಳನ್ನು ಮಾತ್ರ ಸಂಪಾದಿಸಿ (ಅನುವಾದಗಳನ್ನು ಅಲ್ಲ)  
4. **ಸ್ಥಳೀಯ ಪರೀಕ್ಷೆ**: ಮಾರ್ಕ್‌ಡೌನ್ ಸರಿಯಾಗಿ ಪ್ರದರ್ಶಿಸುತ್ತಿದೆಯೇ ಪರೀಕ್ಷಿಸಿ  
5. **PR ಸಲ್ಲಿಸಿ**: ಸ್ಪಷ್ಟ PR ಶೀರ್ಷಿಕೆ ಮತ್ತು ವಿವರಣೆ ಬಳಸಿ  
6. **CLA**: ಸೂಚಿಸಿದಾಗ Microsoft Contributor License Agreementಗೆ ಸಹಿ ಮಾಡಿ

### Common Troubleshooting

**Q: ನನ್ನ PR ಅನುವಾದ ಪರಿಶೀಲನೆನಲ್ಲಿ ವಿಫಲವಾಗುತ್ತಿದೆ**  
A: ದಯವಿಟ್ಟು ನೀವು ಮೂಲ ಮೋಡ್ಯೂಲ್ ಡೈರೆಕ್ಟರಿಗಳ ಇಂಗ್ಲಿಷ್ ಮಾರ್ಕ್‌ಡೌನ್ ಫೈಲ್‌ಗಳನ್ನು ಮಾತ್ರ ಸಂಪಾದಿಸಿದ್ದೀರಾ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ, ಅನುವಾದಿತ ಆವೃತ್ತಿಗಳನ್ನು ಅಲ್ಲ.

**Q: ನಾನು ಹೊಸ ಭಾಷೆ ಸೇರಿಸಬೇಕೆಂದು ಹೋಂಗುತ್ತೋಡಿದೆ**  
A: ಭಾಷಾ ಬೆಂಬಲವನ್ನು ಸಹಯೋಗ-ಅನುವಾದಕ ವರ್ಕ್ಫ್ಲೋ ಮೂಲಕ ನಿರ್ವಹಿಸಲಾಗುತ್ತದೆ. ಹೊಸ ಭಾಷೆ ಸೇರಿಸುವ ಬಗ್ಗೆ ಚರ್ಚಿಸಲು ಇಶ್ಯು ತೆಗೆಯಿರಿ.

**Q: ಕೋಡ್ نمೂನಾ ಕಾರ್ಯನಿರ್ವಹಿಸುವುದಿಲ್ಲ**  
A: ನಿಮಗೆ ಕಾರಣಿಯಾದ نمೂನಾದ README ನಲ್ಲಿ ಸೂಚನೆಗಳನ್ನು ಅನುಸರಿಸಿದ್ದೀರಾ ಎಂದು ಪರಿಶೀಲಿಸಿ. ಅವಶ್ಯಕ ಅವಲಂಬನೆಗಳನ್ನು ಸರಿಯಾದ ಆವೃತ್ತಿಯಲ್ಲಿ ಹೊಂದಿದೆ ಎಂದು ಖಾತ್ರಿಪಡಿಸಿಕೊಳ್ಳಿ.

**Q: ಚಿತ್ರಗಳು ಪ್ರದರ್ಶಿಸಲಾಗುತ್ತಿಲ್ಲ**  
A: ಚಿತ್ರದ ಮಾರ್ಗಗಳು ಸಂಬಂಧಿಸಿದವೆಯೇ ಮತ್ತು ಫಾರ್ವರ್ಡ್ ಸ್ಲಾಶ್‌ಗಳಿವೆ ಎಂದು ತಪಾಸಿಸಿ. ಚಿತ್ರಗಳೂ `images/` ಅಥವಾ ಸ್ಥಳೀಯೀಕೃತ ಆವೃತ್ತಿಗೆ `translated_images/` ನಲ್ಲಿ ಇರಬೇಕು.
