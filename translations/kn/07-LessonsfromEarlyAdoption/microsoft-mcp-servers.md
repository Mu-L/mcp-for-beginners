# 🚀 10 ಮೈಕ್ರೋಸಾಫ್ಟ್ MCP ಸರ್ವರ್‌ಗಳು ಡೆವಲಪರ್ ಉತ್ಪಾದಕತೆಯನ್ನು ಪರಿವರ್ತಿಸುತ್ತಿವೆ

## 🎯 ಈ ಮಾರ್ಗಸೂಚಿಯಲ್ಲಿ ನೀವು ಕಲಿಯುವುದೆನಿದು

ಈ ಪ್ರಾಯೋಗಿಕ ಮಾರ್ಗಸೂಚಿ ಆತನಾರೂಢವಾಗಿ ಕೆಲಸ ಮಾಡುತ್ತಿರುವ ಹತ್ತು ಮೈಕ್ರೋಸಾಫ್ಟ್ MCP ಸರ್ವರ್‌ಗಳನ್ನು ಪರಿಚಯಿಸುತ್ತದೆ, ಅವು ಡೆವಲಪರ್‌ಗಳು AI ಸಹಾಯಕರೊಂದಿಗೆ ಕೆಲಸ ಮಾಡುವ ರೀತಿಯನ್ನು ಪರಿವರ್ತಿಸುತ್ತಿವೆ. MCP ಸರ್ವರ್‌ಗಳು *ಏನು ಮಾಡಿದಹಾಗೆ* ಮಾತ್ರವಲ್ಲ, ahubwo ನಾವು ನಿಮಗೆ ಈಗಾಗಲೇ ಮೈಕ್ರೋಸಾಫ್ಟ್ ಮತ್ತು ಅದರ ಹೊರಗಿನ ದೈನಂದಿನ ಅಭಿವೃದ್ಧಿ ಕಾರ್ಯಪ್ರವೃತ್ತಿಗಳಲ್ಲಿ ನಿಜವಾದ ಪ್ರಭಾವ ಬೀರುತ್ತಿರುವ ಸರ್ವರ್‌ಗಳನ್ನು ತೋರಿಸುತ್ತೇವೆ.

ಈ ಮಾರ್ಗಸೂಚಿಯ ಪ್ರತಿಯೊಂದು ಸರ್ವರ್ ನಿಜವಾದ ಬಳಕೆ ಮತ್ತು ಡೆವಲಪರ್ ಪ್ರತಿಕ್ರೀಯೆ ಆಧರಿಸಿ ಆಯ್ಕೆ ಮಾಡಲಾಗಿದೆ. ನೀವು ಪ್ರತಿಯೊಂದು ಸರ್ವರ್ ಏನು ಮಾಡುತ್ತದೆ ಮಾತ್ರವಲ್ಲ, ಅದು ಏಕೆ ಮುಖ್ಯ ಮತ್ತು ನಿಮ್ಮ ಸ್ವಂತ ಪ್ರಾಜೆಕ್ಟ್‌ಗಳಲ್ಲಿ ಅದರಿಂದ ಹೆಚ್ಚು ಲಾಭ ಪಡೆಯುವುದು ಹೇಗೆ ಎಂಬುದನ್ನು ಅರಿತುಕೊಳ್ಳುತ್ತೀರಿ. ನೀವು MCP ಗೆ ಸಂಪೂರ್ಣ ಹೊಸದಾದರೂ ಅಥವಾ ಈಗಾಗಲೇ ಇರುವ ಬಳಕೆಯನ್ನು ವಿಸ್ತರಿಸಿಕೊಳ್ಳಬೇಕಾದರೂ, ಈ ಸರ್ವರ್‌ಗಳು ಮೈಕ್ರೋಸಾಫ್ಟ್ ಪರಿಸರದಲ್ಲಿ ಲಭ್ಯವಿರುವ ಅತ್ಯಂತ ಪ್ರಾಯೋಗಿಕ ಮತ್ತು ಪರಿಣಾಮಕಾರಿಯಾದ ಸಾಧನಗಳನ್ನು ಪ್ರತಿನಿಧಿಸುತ್ತವೆ.

> MCP ಗೆ ಹೊಸದಾಗಿದ್ದೀರಾ? ಚಿಂತಿಸಬೇಡಿ! ಈ ಮಾರ್ಗಸೂಚಿ ಪ್ರಾರಂಭಿಕರಿಗು ಅನುಕೂಲಕರವಾಗಿದ್ದು, ನಾವು ಧಾರ್ಮಿಕ ವಿಚಾರಗಳನ್ನು ವಿವರಿಸುತ್ತೇವೆ ಮತ್ತು ನೀವು ಯಾವಾಗಲಾದರೂ ನಮ್ಮ [MCP ಪರಿಚಯ](../00-Introduction/README.md) ಮತ್ತು [ಮೂಲತತ್ವಗಳು](../01-CoreConcepts/README.md) ಘಟಕಗಳಿಗೆ ಹಿಂತಿರುಗಿ, ಆಳವಾದ ಹಿನ್ನೆಲೆ ತಿಳಿಯಬಹುದು.

## ಅವಲೋಕನ

ಈ ಸಮಗ್ರ ಮಾರ್ಗಸೂಚಿ, ಡೆವಲಪರ್‌ಗಳು AI ಸಹಾಯಕರು ಮತ್ತು ಬಾಹ್ಯ ಉಪಕರಣಗಳೊಡನೆ ಸಂಪರ್ಕ ಹೊಂದುವುದನ್ನು ಪರಿವರ್ತಿಸಲಾಗುತ್ತಿರುವ 10 ಮೈಕ್ರೋಸಾಫ್ಟ್ MCP ಸರ್ವರ್‌ಗಳನ್ನು ಅನ್ವೇಷಿಸುತ್ತದೆ. ಅಜೂರ್ ಸಂಪನ್ಮೂಲ ನಿರ್ವಹಣೆಯಿಂದ ಡಾಕ್ಯುಮೆಂಟ್ ಸಂಸ್ಕರಣೆಯವರೆಗೆ, ಈ ಸರ್ವರ್‌ಗಳು ಮಾದರಿ_Context ಪ್ರೋಟೋಕಾಲ್ ನ ಶಕ್ತಿಯನ್ನು ಪ್ರದರ್ಶಿಸುತ್ತವೆ, ಅಂಥ seamless ಮತ್ತು ಉತ್ಪಾದಕತೆಯುತ ಅಭಿವೃದ್ಧಿ ಕಾರ್ಯಪ್ರವೃತ್ತಿಗಳನ್ನು ನಿರ್ಮಿಸುವಲ್ಲಿ.

## ಕಲಿಕೆಯ ಗುರಿಗಳು

ಈ ಮಾರ್ಗಸೂಚಿಯ ಕೊನೆಯಲ್ಲಿ ನೀವು:
- MCP ಸರ್ವರ್‌ಗಳು ಹೇಗೆ ಡೆವಲಪರ್ ಉತ್ಪಾದಕತೆಯನ್ನು ಹೆಚ್ಚಿಸುತ್ತವೆ ಮನಸ್ಸಾಗಿಸುವಿರಿ
- ಮೈಕ್ರೋಸಾಫ್ಟ್‌ನ ಅತಿ ಪರಿಣಾಮಕಾರಿ MCP ಸರ್ವರ್ ಅನುಷ್ಠಾನಗಳನ್ನು ತಿಳಿಹೀರಿ
- ಪ್ರತಿಯೊಂದು ಸರ್ವರ್‌ಗಾಗಿ ಪ್ರಾಯೋಗಿಕ ಬಳಕೆಯ ಪ್ರಕರಣಗಳನ್ನು ಕಂಡುಕೊಳ್ಳುವಿರಿ
- VS ಕೋಡ್ ಮತ್ತು Visual Studio ನಲ್ಲಿ ಈ ಸರ್ವರ್‌ಗಳನ್ನು ಸೆಟ್ ಮಾಡುವ ಮತ್ತು ಹೊಂದಿಸುವ ವಿಧಾನವನ್ನು ತಿಳಿಯುವಿರಿ
- MCP ವಿಸ್ತೃತ ಪರಿಸರ ಮತ್ತು ಭವಿಷ್ಯ ದಿಕ್ಕುಗಳನ್ನು ಅನ್ವೇಷಿಸುವಿರಿ

## 🔧 MCP ಸರ್ವರ್‌ಗಳನ್ನು ಅರಿದುಕೊಳ್ಳುವಿಕೆ: ಪ್ರಾರಂಭಿಕರಿಗಾಗಿ ಮಾರ್ಗದರ್ಶಿ

### MCP ಸರ್ವರ್ ಎಂದರೆ ಏನು?

ಮಾದರಿ_Context ಪ್ರೋಟೋಕಾಲ್ (MCP) ನಲ್ಲಿ ಹೊಸದಾಗಿ ಇದ್ದರೆ, "MCP ಸರ್ವರ್ ಎಂದರೆ ಏನು, ಮತ್ತು ಇದಕ್ಕೆ ನಾನು ಯಾಕೆ ಗಮನ ಕೊಡಬೇಕಾಗುತ್ತದೆ?" ಎಂದು ನೀವು ಆಲೋಚಿಸಬಹುದು. ಸರಳ ಉದಾಹರಣೆಯಿಂದ ಪ್ರಾರಂಭಿಸೋಣ.

MCP ಸರ್ವರ್‌ಗಳನ್ನು ನಿಮ್ಮ AI ಕೋಡಿಂಗ್ ಸಹಾಯಕ (GitHub Copilot ಹೋಲಿಸಿದಂತೆ) ಬಾಹ್ಯ ಉಪಕರಣಗಳು ಮತ್ತು ಸೇವೆಗಳೊಡನೆ ಸಂಪರ್ಕ ಸಾಧಿಸಲು ಸಹಾಯ ಮಾಡುವ ವಿಶೇಷ ನೆರವಿನವರು ಎಂದು ಭಾವಿಸಿ. ನೀವು ಫೋನ್ ನಲ್ಲಿ ಹವಾಮಾನ ನೋಡಲು, ನ್ಯಾವಿಗೇಶನ್ ಮಾಡಲೂ, ಬ್ಯಾಂಕಿಂಗ್ ಕೆಲಸಕ್ಕೆ ವಿಭಿನ್ನ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಬಳಸುತ್ತೀರಿ. ಹಾಗೆ MCP ಸರ್ವರ್‌ಗಳು ನಿಮ್ಮ AI ಸಹಾಯಕರಿಗೆ ವಿಭಿನ್ನ ಡೆವಲಪ್ಮೆಂಟ್ ಉಪಕರಣಗಳು ಮತ್ತು ಸೇವೆಗಳನ್ನು ಸಂಪರ್ಕಿಸಲು ಸಾಮರ್ಥ್ಯ ನೀಡುತ್ತವೆ.

### MCP ಸರ್ವರ್‌ಗಳು ಯಾವ ಸಮಸ್ಯೆಗಳನ್ನು ಪರಿಹರಿಸುತ್ತವೆ

MCP ಸರ್ವರ್‌ಗಳ ಮೊದಲಿನ ಕಾಲದಲ್ಲಿ, ನೀವು:
- ನಿಮ್ಮ ಅಜೂರ್ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಪರೀಕ್ಷಿಸುವುದು
- GitHub issue ಸೃಷ್ಟಿಸುವುದು
- ನಿಮ್ಮ ಡೇಟಾಬೇಸ್‌ನಲ್ಲಿ ವಿಚಾರಿಸುವುದು
- ಡಾಕ್ಯುಮೆಂಟೇಷನ್ ಹುಡುಕುವುದು

ಎಲ್ಲವನ್ನು ಕೋಡಿಂಗ್ ನಿಲ್ಲಿಸಿ, ಬ್ರೌಸರ್ ತೆರೆಯಬೇಕು, ಸೂಕ್ತ ವೆಬ್‌ಸೈಟ್‌ಗೆ ಹೋಗಿ ಈ ಕೆಲಸಗಳನ್ನು ಕೈಗೊಳ್ಳಬೇಕು. ಈ ನಿರಂತರ ಎಂದರೆ ಹಿನ್ನೆಲೆಯನ್ನು ಬದಲಾಯಿಸುವಿಕೆ ನಿಮ್ಮ ಕಾರ್ಯ ಶ್ರೇಣಿ ಮುರಿಯಿಸುತ್ತದೆ ಮತ್ತು ಉತ್ಪಾದಕತೆಯನ್ನು ಕಡಿಮೆ ಮಾಡುತ್ತದೆ.

### MCP ಸರ್ವರ್‌ಗಳು ನಿಮ್ಮ ಅಭಿವೃದ್ದಿ ಅನುಭವವನ್ನು ಹೇಗೆ ಪರಿವರ್ತಿಸುತ್ತವೆ

MCP ಸರ್ವರ್‌ಗಳೊಂದಿಗೆ ನೀವು ನಿಮ್ಮ ಅಭಿವೃದ್ಧಿ ಪರಿಸರದ (VS ಕೋಡ್, Visual Studio, ಇತ್ಯಾದಿ) ಒಳಗೇ ಇರುತ್ತೀರಿ ಮತ್ತು ಚರಿತ್ರೆ ಪ್ರಕಾರ, AI ಸಹಾಯಕರಿಗೆ ಈ ಕೆಲಸಗಳನ್ನು ಕೈಕೊಂಡು ಹೋಗು ಎಂದು ಕೇಳಬಹುದು. ಉದಾಹರಣೆಗೆ:

**ಪೂರ್ವಭಾವಿಯಾಗಿ ಈ ಪರಂಪರೆಯ ಕಾರ್ಯಪ್ರವೃತ್ತಿ:**
1. ಕೋಡಿಂಗ್ ನಿಲ್ಲಿಸಿ
2. ಬ್ರೌಸರ್ ತೆರೆಯಿರಿ
3. ಅಜೂರ್ ಪೋರ್ಟಲ್‌ಗೆ ಹೋಗಿ
4. ಸ್ಟೋರೇಜ್ ಖಾತೆಯ ವಿವರಗಳನ್ನು ನೋಡಿ
5. VS ಕೋಡ್ ಗೆ ಹಿಂದಿರುಗಿ
6. ಕೋಡಿಂಗ್ ಮುಂದುವರಿಸು

**ಈಗ ನೀವು ಇತರಾಯಿಸಬಹುದು:**
1. AI ಗೆ ಕೇಳಿ: "ನನ್ನ ಅಜೂರ್ ಸ್ಟೋರೇಜ್ ಖಾತೆಗಳ ಸ್ಥಿತಿಯನ್ನು ತಿಳಿಸಿ?"
2. ತಿಳಿವಳಿಕೆ ಪಡೆಯುತ್ತ ಕೋಡಿಂಗ್ ಮುಂದುವರಿಸು

### ಪ್ರಾರಂಭಿಕರಿಗಾಗಿನ ಕೀಳು ಲಾಭಗಳು

#### 1. 🔄 **ನಿಮ್ಮ ಕ್ರಮಸ್ಥಿತಿಯಲ್ಲಿ ಇರಲು ಸಹಾಯ**
- ಅನೇಕ ಅಪ್ಲಿಕೇಶನ್‌ಗಳ ನಡುವೆ ಸ್ವಿಚ್ ಮಾಡುವುದಿಲ್ಲ
- ನೀವು ಬರೆಯುತ್ತಿರುವ ಕೋಡ್ ಮೇಲೆ ಕೇಂದ್ರಿತವಾಗಿರಿ
- ವಿಭಿನ್ನ ಸಾಧನಗಳನ್ನು ನಿರ್ವಹಿಸುವ ಮಾನಸಿಕ ಭಾರ ಕಡಿಮೆ ಮಾಡು

#### 2. 🤖 **ಸಂಕೀರ್ಣ ಆಜ್ಞೆಗಳ ಬದಲು ಸ್ವಾಭಾವಿಕ ಭಾಷೆಯನ್ನು ಬಳಸು**
- SQL ವಾಕ್ಯರಚನೆಯ ನೆನಪಿನ ಬದಲು ನೀವು ಬೇಕಾದ ಮಾಹಿತಿಯನ್ನು ವಿವರಿಸಿ
- ಅಜೂರ್ CLI ಆಜ್ಞೆಗಳನ್ನು ನೆನಪಿನಲ್ಲಿ ಇಡುವ ಬದಲು ನೀವು ಸಾಧಿಸುವುದೇನು ಎಂದು ಹೇಳಿ
- ತಾಂತ್ರಿಕ ವಿವರಗಳನ್ನು AI ಗೆ ಬಳಸುವುದು, ನೀವು ತರ್ಕದಲ್ಲಿ ಕೇಂದ್ರೀಕರಿಸಬಹುದು

#### 3. 🔗 **ಬಹುಸಂಖ್ಯಕ ಸಾಧನಗಳನ್ನು ಸಂಪರ್ಕ ಮಾಡಿ**
- ವಿಭಿನ್ನ ಸೇವೆಗಳನ್ನು ಒಟ್ಟುಗೂಡಿಸುವ ಮೂಲಕ ಶಕ್ತಿಶಾಲಿ ಕಾರ್ಯಾಚರಣೆಗಳನ್ನು ರಚಿಸಿ
- ಉದಾಹರಣೆ: "ಇತ್ತೀಚಿನ GitHub issues ಎಲ್ಲವನ್ನು ಪಡೆದು ಇಲ್ಲಿ ಹೊಂದಿಕೊಂಡ Azure DevOps ಕೆಲಸದ ಐಟಂಗಳನ್ನು ರಚಿಸಿ"
- ಸಂಕೀರ್ಣ ಸ್ಕ್ರಿಪ್ಟ್ ಬರೆಯದೆ ಸ್ವಯಂಚಾಲಿತವನ್ನು ನಿರ್ಮಿಸಿ

#### 4. 🌐 **ವರ್ತಮಾನದಲ್ಲಿ ಅಭಿವೃದ್ಧಿಯಾಗುತ್ತಿರುವ ಪರಿಕಲ್ಪನೆಗೆ ಪ್ರವೇಶ**
- ಮೈಕ್ರೋಸಾಫ್ಟ್, GitHub ಮತ್ತು ಇತರ ಸಂಸ್ಥೆಗಳ ಮೂಲಕ ನಿರ್ಮಿತ ಸರ್ವರ್‌ಗಳಿಂದ ಲಾಭ ಪಡೆಯಿರಿ
- ವಿಭಿನ್ನ ಮಾರಾಟಗಾರರ ಸಾಧನಗಳನ್ನು ಹೇರಳಿಸಿ ಮಿಶ್ರಣ ಮಾಡಿ
- ವಿಭಿನ್ನ AI ಸಹಾಯಕರ ಕೆಲಸ ಮಾಡುವ ಸ್ಥಿರಂತ ಪರಿಸರ ಸೇರಿ ಸೇರಿಕೊಳ್ಳಿ

#### 5. 🛠️ **ಪ್ರಾಯೋಗಿಕವಾಗಿ ಕಲಿಯಿರಿ**
- ಮೂಲಭೂತ ಸರ್ವರ್‌ಗಳಿಂದ ಪ್ರಾರಂಭಿಸಿ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಿ
- ಸ್ವಂತ ಸರ್ವರ್ ಗಳನ್ನು ಕೊಂಡು ಹೋಗಿ ಶುರುಮಾಡಿ
- ಲಭ್ಯವಿರುವ SDK ಗಳು ಮತ್ತು ಡಾಕ್ಯುಮೆಂಟೇಷನ್ ನಿಮ್ಮ ಕಲಿಕೆಯನ್ನು ಮಾರ್ಗದರ್ಶಿಸುತ್ತವೆ

### ಪ್ರಾರಂಭಿಕರಿಗಾಗಿ ನಿಜವಿರುವ ಉದಾಹರಣೆ

ನೀವು ವೆಬ್ ಅಭಿವೃದ್ಧಿಯಲ್ಲಿ ಸಂಪೂರ್ಣ ಹೊಸದಾಗಿದ್ದು ನಿಮ್ಮ ಮೊದಲ ಪ್ರಾಜೆಕ್ಟ್‌ನಲ್ಲಿ ಕೆಲಸ ಮಾಡುತ್ತಿದ್ದೀರಿ ಅಂತಾಳೋಣ. MCP ಸರ್ವರ್‌ಗಳು ಹೇಗೆ ಸಹಾಯ ಮಾಡುತ್ತವೆ:

**ಪಾರಂಪರಿಕ ವಿಧಾನ:**
```
1. Code a feature
2. Open browser → Navigate to GitHub
3. Create an issue for testing
4. Open another tab → Check Azure docs for deployment
5. Open third tab → Look up database connection examples
6. Return to VS Code
7. Try to remember what you were doing
```

**MCP ಸರ್ವರ್‌ಗಳಿಗೆ:**
```
1. Code a feature
2. Ask AI: "Create a GitHub issue for testing this login feature"
3. Ask AI: "How do I deploy this to Azure according to the docs?"
4. Ask AI: "Show me the best way to connect to my database"
5. Continue coding with all the information you need
```

### ಎಂಟರ್ಪ್ರೈಸ್ ಸ್ಟ್ಯಾಂಡರ್ಡ್ ಮೇಲುವಳಿಕೆ

MCP ಒಂದು ಉದ್ಯಮ-ವ್ಯಾಪ್ತ ಸ್ಥಿರಂತವಾಗುತ್ತಿದ್ದು:
- **ಸಾಮರಸ್ಯ**: ವಿಭಿನ್ನ ಸಾಧನಗಳು ಮತ್ತು ಕಂಪನಿಗಳ ನಡುವೆ ಸಮಾನ ಅನುಭವ
- **ಆಂತರಿಕ ಸಂವಹನ**: ವಿಭಿನ್ನ ಮಾರಾಟಗಾರರಿಂದ ಸರ್ವರ್‌ಗಳು ಒಟ್ಟಿಗೆ ಕೆಲಸ ಮಾಡುತ್ತವೆ
- **ಭವಿಷ್ಯ-ನಿರೋಧಕತೆ**: ವಿಭಿನ್ನ AI ಸಹಾಯಕರ ನಡುವೆ ಕೌಶಲ್ಯಗಳು ಮತ್ತು ಸೆಟ್ಟಪ್ಪ್‌ಗಳು ವರ್ಗಾಯಿಸಬಹುದು
- **ಸಮುದಾಯ**: ಹಂಚಿಕೆ ಹೊಂದಿದ ಜ್ಞಾನ ಹಾಗೂ ಸಂಪನ್ಮೂಲಗಳ ದೊಡ್ಡ ಪರಿಸರ

### ಪ್ರಾರಂಭಿಸುವುದು: ನೀವು ಕಲಿಯುವುದು ಏನು

ಈ ಮಾರ್ಗಸೂಚಿಯಲ್ಲಿ, ಎಲ್ಲಾ ಮಟ್ಟದ ಡೆವಲಪರ್‌ಗಳಿಗೆ ವಿಶೇಷವಾಗಿ ಉಪಯುಕ್ತವಾಗಿರುವ 10 ಮೈಕ್ರೋಸಾಫ್ಟ್ MCP ಸರ್ವರ್‌ಗಳನ್ನು ನಾವು ಅನುಷ್ಠಾನ ಮಾಡುತ್ತಿದ್ದೇವೆ. ಪ್ರತಿಯೊಂದು ಸರ್ವರ್:
- ಸಾಮಾನ್ಯ ಅಭಿವೃದ್ಧಿ ಸವಾಲುಗಳನ್ನು ಪರಿಹರಿಸುತ್ತದೆ
- ಪುನರಾವರ್ತಿತ ಸಮಸ್ಯೆಗಳನ್ನು ಕಡಿಮೆ ಮಾಡುತ್ತದೆ
- ಕೋಡ್ ಗುಣಮಟ್ಟವನ್ನು ಉತ್ತಮಗೊಳಿಸುತ್ತದೆ
- ಕಲಿಕೆಯ ಅವಕಾಶಗಳನ್ನು ಹೆಚ್ಚಿಸುತ್ತದೆ

> **💡 ಕಲಿಕೆಯ ಸಲಹೆ**
> 
> ನೀವು MCP ಗೆ ಸಂಪೂರ್ಣ ಹೊಸದಾದರೆ, ಮೊದಲು ನಮ್ಮ [MCP ಪರಿಚಯ](../00-Introduction/README.md) ಮತ್ತು [ಮೂಲತತ್ವಗಳು](../01-CoreConcepts/README.md) ಘಟಕಗಳನ್ನು ಓದಿ. ನಂತರ ಇಲ್ಲಿ ಬಂದು ಈ ತತ್ವಗಳನ್ನು ನಾವು ಮೈಕ್ರೋಸಾಫ್ಟ್ ಸಾಧನಗಳೊಂದಿಗೆ ನಿಜವಾಗಿ ಹೇಗೆ ಬಳಸುತ್ತಿದ್ದೇವೆ ನೋಡಬಹುದು.
>
> MCP ಮಹತ್ವದ ಬಗ್ಗೆ ಹೆಚ್ಚುವರಿ ಹಿನ್ನೆಲೆಗೆ, ಮಾರಿಯಾ ನ್ಯಾಗಗ ಅವರ ಪೋಸ್ಟ್ ನೋಡಿ: [ಒಮ್ಮೆ ಸಂಪರ್ಕಿಸಿ, ಎಲ್ಲೆಡೆ ಸಂಯೋಜಿಸಿ MCP ಸಹಿತ](https://devblogs.microsoft.com/blog/connect-once-integrate-anywhere-with-mcps).

## VS ಕೋಡ್ ಮತ್ತು Visual Studio ನಲ್ಲಿ MCP ಪ್ರಾರಂಭಿಸುವುದು 🚀

ಈ MCP ಸರ್ವರ್‌ಗಳನ್ನು Visual Studio Code ಅಥವಾ Visual Studio 2022 ಗಿಟ್‌ಹಬ್ ಕೋಪೈಲಟ್‌ ಜೊತೆಗೆ ಬಳಸುತ್ತಿರುವ ವೇಳೆ ಸುಲಭವಾಗಿ ಸೆಟ್ ಅಪ್ ಮಾಡಬಹುದು.

### VS ಕೋಡ್ ಸೆಟ್ ಅಪ್

VS ಕೋಡ್‌ನಲ್ಲಿ ಮೂಲ ಪ್ರಕ್ರಿಯೆ:

1. **ಏಜೆಂಟ್ ಮೋಡ್ ಸಕ್ರಿಯಗೊಳಿಸಿ**: VS ಕೋಡ್ ನಲ್ಲಿ ಕೋಪೈಲಟ್ ಚಾಟ್ ವಿಂಡೋದಲ್ಲಿ ಏಜೆಂಟ್ ಮೋಡ್ ಗೆ ಬದಲಾಯಿಸಿ
2. **MCP ಸರ್ವರ್‌ಗಳ ಸಂರಚನೆ**: ನಿಮ್ಮ VS ಕೋಡ್ settings.json ಫೈಲ್ ಗೆ ಸರ್ವರ್ ಸಂರಚನೆ ಸೇರಿಸಿ
3. **ಸರ್ವರ್ ಪ್ರಾರಂಭಿಸಿ**: ನೀವು ಬಳಸಬೇಕಾದ ಪ್ರತಿಯೊಂದು ಸರ್ವರ್‌ಗಾಗಿ "ಪ್ರಾರಂಭಿಸಿ" ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಿ
4. **ಉಪಕರಣಗಳನ್ನು ಆಯ್ಕೆಮಾಡಿ**: ಪ್ರಸ್ತುತ ಸೆಷನ್ ಮೇಲೆ ನೀವು ಸಕ್ರಿಯಗೊಳಿಸುವ MCP ಸರ್ವರ್‌ಗಳ ಆಯ್ಕೆ ಮಾಡಿ

ವಿಸ್ತೃತ ಸ್ಥಾಪನೆ ಸೂಚನೆಗಾಗಿ ನೋಡಿ [VS ಕೋಡ್ MCP ಡಾಕ್ಯುಮೆಂಟೇಷನ್](https://code.visualstudio.com/docs/copilot/copilot-mcp).

> **💡 ತಜ್ಞರ ಸಲಹೆ: MCP ಸರ್ವರ್‌ಗಳನ್ನು ನಿಪುಣವಾಗಿ ನಿರ್ವಹಿಸಿ!**
> 
> VS ಕೋಡ್ ಎಕ್ಸ್ಟೆನ್ಶನ್ ನೋಟದಲ್ಲಿ ಈಗ [ಸ್ಥಾಪಿತ MCP ಸರ್ವರ್‌ಗಳನ್ನು ನಿರ್ವಹಿಸಲು ಹೊಸ UI ಇದೆ](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode)! ಯಾವುದೇ ಸ್ಥಾಪಿತ MCP ಸರ್ವರ್ ಅನ್ನು ತಡಪಡಿಸದೇ ಪ್ರಾರಂಭಿಸಲು, ನಿಲ್ಲಿಸಲು ಮತ್ತು ನಿರ್ವಹಿಸುವುದಕ್ಕೆ ಸುಲಭವಾದ ಇಂಟರ್ಫೇಸ್ ಲಭ್ಯವಿದೆ. ಪ್ರಯತ್ನಿಸಿ!

### Visual Studio 2022 ಸೆಟ್ ಅಪ್

Visual Studio 2022 (ಆವೃತ್ತಿ 17.14 ಅಥವಾ ಮೇಲು)ಗಾಗಿ:

1. **ಏಜೆಂಟ್ ಮೋಡ್ ಸಕ್ರಿಯಗೊಳಿಸಿ**: GitHub Copilot ಚಾಟ್ ವಿಂಡೋದಲ್ಲಿನ "Ask" ಡ್ರಾಪ್‌ಡೌನ್ ಕ್ಲಿಕ್ ಮಾಡಿ "Agent" ಆಯ್ಕೆಮಾಡಿ
2. **ಸಂರಚನಾ ಫೈಲ್ ರಚಿಸಿ**: ನಿಮ್ಮ ಪರಿಹಾರ ಡೈರಕ್ಟರಿಯಲ್ಲಿ `.mcp.json` ಫೈಲ್ ರಚಿಸಿ (ಶಿಫಾರಸು: `<SOLUTIONDIR>\.mcp.json`)
3. **ಸರ್ವರ್‌ಗಳನ್ನು ಸಂರಚಿಸಿ**: ಸಾಮಾನ್ಯ MCP ರೂಪದಲ್ಲಿ ನಿಮ್ಮ ಸರ್ವರ್ ಕಾನ್ಫಿಗರೇಶನ್‌ಗಳನ್ನು ಸೇರಿಸಿ
4. **ಉಪಕರಣ ಅನುಮೋದನೆ**: ನಿಮ್ಮ ಬಳಕೆಗಾಗಿ ಅನುಕೂಲ ಮಾಡಿಕೊಡಬೇಕಾದ ಉಪಕರಣಗಳಿಗೆ ಅನುಮತಿ ನೀಡಿ

ವಿಸ್ತೃತ Visual Studio ಸೆಟ್ ಅಪ್ ಸೂಚನೆಗಾಗಿ [Visual Studio MCP ಡಾಕ್ಯುಮೆಂಟೇಷನ್](https://learn.microsoft.com/visualstudio/ide/mcp-servers) ನೋಡಿ.

ಪ್ರತಿಯೊಂದು MCP ಸರ್ವರ್ ತನ್ನದೇ ಆದ ಸಂರಚನಾ ಅಗತ್ಯಗಳನ್ನು ಹೊಂದಿದೆ (ಕನೆಕ್ಷನ್ ಸ್ಟ್ರಿಂಗ್‌ಗಳು, ಪ್ರಮಾಣೀಕರಣ, ಇತ್ಯಾದಿ) ಆದರೆ ಎರಡೂ IDE‌ಗಳಲ್ಲಿ ಸ್ಥಾಪನೆ ಮಾದರಿಯು ಸವಿಸ್ತಾರವಾಗಿದೆ.

**ಯಾಕೆ ಇದು ಉಪಯುಕ್ತ**: ಇದು “ಕೆವಲ ಡಾಕ್ಯುಮೆಂಟೇಷನ್” ಎಂದು ತೋರುವಾಗಿದೆ ಆದರೂ, ಮೈಕ್ರೋಸಾಫ್ಟ್ ತಂತ್ರಜ್ಞಾನಗಳನ್ನು ಬಳಸುತ್ತಿರುವ ಪ್ರತಿಯೊಬ್ಬ ಡೆವಲಪರ್ಸ್‌ಗಾಗಿ ಇದು ಮಹತ್ವಪೂರ್ಣವಾಗಿದೆ. .NET ಡೆವಲಪರ್‌ಗಳ ಇತ್ತೀಚಿನ .NET ಮತ್ತು C# ಬಿಡುಗಡೆಗಳ ಬಗ್ಗೆ AI ಕೋಡಿಂಗ್ ಸಹಾಯಕರು ತಾಜಾ ಮಾಹಿತಿ ಹೊಂದಿಲ್ಲವೆಂಬ ಅತಿದೊಡ್ಡ ಪ್ರತಿಕೂಲತೆಯ ಸಮಾಧಾನ ಬಂತು ಈ ಸರ್ವರ್. ಈ ಸರ್ವರ್ ನಿಂದ ನವೀನ ಡಾಕ್ಯುಮೆಂಟೇಷನ್, API ಉಲ್ಲೇಖಗಳು, ಉತ್ತಮ ಆಚಾರ ಸಂಹಿತೆಯನ್ನು ರಿಯಲ್-ಟೈಮ್ ನಲ್ಲಿ ಪಡೆಯಬಹುದು. ನೀವು ಇತ್ತೀಚಿನ ಅಜೂರ್ SDK ಗಳೊಂದಿಗೆ ಕೆಲಸ ಮಾಡುತ್ತಿದ್ದರೂ, ಹೊಸ C# 13 ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಅನ್ವೇಷಿಸುತ್ತಿದ್ದರೂ, ಅಥವಾ ಮೊಡರ್ನ್ .NET Aspire ರೂಪರೇಖೆಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸುತ್ತಿದ್ದರೂ ಈ ಸರ್ವರ್ ನಿಮ್ಮ AI ಸಹಾಯಕರಿಗೆ ವಿಶ್ವಾಸಾರ್ಹ ಮತ್ತು ನವೀಕೃತ ಮಾಹಿತಿಯನ್ನು ನೀಡುತ್ತದೆ, ಹಾಗಾಗಿ ನಿಖರ, ಆಧುನಿಕ ಕೋಡ್ ರಚನೆ ಸಾದ್ಯ.

**ವಾಸ್ತವಿಕ ಬಳಕೆ**: "ಮೈಕ್ರೋಸಾಫ್ಟ್ ಲರ್ನ್ ಅಧಿಕೃತ ಡಾಕ್ಯುಮೆಂಟೇಷನ್ ಪ್ರಕಾರ ಅಜೂರ್ ಕಂಟೈನರ್ ಅಪ್ ರಚಿಸಲು az cli ಆಜ್ಞೆಗಳು ಯಾವುವು?" ಅಥವಾ "ASP.NET ಕೊರ್ನಲ್ಲಿ Entity Framework ಡಿಪೆಂಡೆನ್ಸಿ ಇಂಜೆಕ್ಷನ್ ಅನ್ನು ಹೇಗೆ ಸಂರಚಿಸುವುದು?" ಅಥವಾ "ಈ ಕೋಡ್ ಅನ್ನು ಪರಿಶೀಲಿಸಿ ಮೈಕ್ರೋಸಾಫ್ಟ್ ಲರ್ನ್ ಡಾಕ್ಯುಮೆಂಟೇಷನ್ ಸಲ್ಲಿಸುವ ಪ್ರದರ್ಶನ ಶಿಫಾರಸುಗಳಿಗೆ ಹೊಂದಿಕೊಳ್ಳೋಕೆ". ಈ ಸರ್ವರ್ ಮೈಕ್ರೋಸಾಫ್ಟ್ ಲರ್ನ್, ಅಜೂರ್ ಡಾಕ್ಯುಮೆಂಟ್ಸ್ ಮತ್ತು Microsoft 365 ಡಾಕ್ಯುಮೆಂಟೇಷನ್ ಗಳಲ್ಲಿ ಅಗತ್ಯ ಪ್ರಶ್ನೆಗಳಿಗನುಗುಣವಾಗಿ ಅತ್ಯಾಧುನಿಕ Semantik ಶೋಧ ಮೂಲಕ ಉಪಯುಕ್ತ ಮಾಹಿತಿಯನ್ನು ತರುತ್ತದೆ. ಇದು ಹೆಚ್ಚಿನ ಮಟ್ಟದ ವಿಷಯಗ್ರಂಥಗಳನ್ನು, ಲೇಖನ ಶೀರ್ಷಿಕೆಗಳು ಮತ್ತು URL ಗಳೊಂದಿಗೆ 10 ಕ್ಕಷ್ಟು ವಿಷಯದ ತುಂಡುಗಳನ್ನು ಇಟ್ಟುಕೊಳ್ಳುತ್ತದೆ, ಸದಾ ಅತ್ಯಾದುನಿಕ ಮೈಕ್ರೋಸಾಫ್ಟ್ ಡಾಕ್ಯುಮೆಂಟೇಷನ್ ಪ್ರವೇಶಿಸುತ್ತದೆ.

**ಪ್ರಮುಖ ಉದಾಹರಣೆ**: ಈ ಸರ್ವರ್ `microsoft_docs_search` ಉಪಕರಣವನ್ನು ಹೊರತರುತ್ತದೆ, ಇದು ಮೈಕ್ರೋಸಾಫ್ಟ್ ಅಧಿಕೃತ ತಾಂತ್ರಿಕ ಡಾಕ್ಯುಮೆಂಟೇಷನ್ ವಿರುದ್ಧ Semantik ಶೋಧ ನಿರ್ವಹಿಸುತ್ತದೆ. ಒಂದು ಬಾರಿ ಸಂರಚಿಸಿದ ನಂತರ ನೀವು "ASP.NET ಕೊರ್ನಲ್ಲಿ JWT ಪ್ರಮಾಣೀಕರಣವನ್ನು ಹೇಗೆ ಅನುಷ್ಠಾನಗೊಳಿಸುವುದು?" ಅಂತ ಪ್ರಶ್ನೆ ಕೇಳಬಹುದು ಮತ್ತು ಮೂಲ ಲಿಂಕ್‌ಗಳೊಂದಿಗೆ ವಿವರವಾದ ಅಧಿಕೃತ ಉತ್ತರ ಪಡೆಯಬಹುದು. ಶೋಧದ ಗುಣಮಟ್ಟ ಅತ್ಯುತ್ತಮವಾಗಿದ್ದು, ಅರ್ಥ ಗ್ರಹಣೆಯಲ್ಲಿ ವಿಭಿನ್ನ-ಸಂದರ್ಭ-ಪರಿಸರಕ್ಕೆ ಹೊಂದುವಂತೆ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ – ಉದಾಹರಣೆಗೆ, "ಕಂಟೈನರ್" ಅಜೂರ್ ಸಂಸ್ಥಾನದ ನೋಟದಲ್ಲಿ ತಿಳಿಸುವಾಗ ಅಜೂರ್ ಕಂಟೈನರ್ ಇನ್ಸ್ಟಾನ್ಸಸ್ ಡಾಕ್ಯುಮೆಂಟೇಷನ್ ಬರುತ್ತದೆ, ಆದರೆ .NET ಸನ್ನಿವೇಶದಲ್ಲಿ ಸಂಬಂಧಿತ C# ಸಂಗ್ರಹ ಮಾಹಿತಿ ಬರುತ್ತದೆ.

ಇದು ವೇಗವಾಗಿ ಬದಲಾಗುತ್ತಿರುವ ಅಥವಾ ಇತ್ತೀಚೆಗೆ ನವೀಕರಿಸಿದ ಲೈಬ್ರರಿಗಳು ಮತ್ತು ಬಳಕೆಯ ಪರಿಸರಗಳಿಗಾಗಿ ವಿಶೇಷ ಉಪಯುಕ್ತವಾಗಿದೆ. ಉದಾಹರಣೆಗೆ, ಇತ್ತೀಚಿನ ಕೆಲವು ಕೋಡಿಂಗ್ ಪ್ರಾಜೆಕ್ಟ್‌ಗಳಲ್ಲಿ ನಾನು .NET Aspire ಮತ್ತು Microsoft.Extensions.AI ನ ಇತ್ತೀಚಿನ ಬಿಡುಗಡೆಗಳ ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಅಳವಡಿಸಲು ಬಯಸಿದೆ. > "DefaultAzureCredential ದೃಢೀಕರಣ ಬಳಸಿ Azure Blob Storage ಗೆ ಒಂದು ಫೈಲ್ ಅಪ್ಲೋಡ್ ಮಾಡುವ ಪೈಥಾನ್ ಸ್ಕ್ರಿಪ್ಟ್ ಸೃಷ್ಟಿಸು. ಸ್ಕ್ರಿಪ್ಟ್ ನನ್ನ 'mycompanystorage' ಎಂಬ Azure ಸ್ಟೋರೆಜ್ ಖಾತೆಗೆ ಸಂಪರ್ಕ ಮಾಡಬೇಕು, 'documents' ಎಂಬ ಕಂಟೈನರಿಗೆ ಅಪ್ಲೋಡ್ ಮಾಡಬೇಕು, ಪರಿಕ್ಷಣಕ್ಕಾಗಿ ಪ್ರಸ್ತುತ ಟೈಮ್‌ಸ್ಟ್ಯಾಂಪ್ ಹೊಂದಿರುವ ಟೆಸ್ಟ್ ಫೈಲ್ ಸೃಷ್ಟಿಸಬೇಕು, ದೋಷಗಳನ್ನು ಒಯ್ಯಿಸಿ ಆರೈಕೆ ಮಾಡಲು ಸಾಧ್ಯವಾಗಬೇಕು ಮತ್ತು ಮಾಹಿತಿಪೂರ್ಣ ಔಟ್‌ಪುಟ್ ನೀಡಬೇಕು, ದೃಢೀಕರಣ ಮತ್ತು ದೋಷ ನಿರ್ವಹಣೆಯಲ್ಲಿ Azure ಉತ್ತಮ ಅಭ್ಯಾಸಗಳನ್ನು ಅನುಸರಿಸಬೇಕು, DefaultAzureCredential ದೃಢೀಕರಣ ಹೇಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆಯೆಂಬುದನ್ನು ತಿಳಿಸುವ ಕಾಮೆಂಟ್‌ಗಳನ್ನು ಒಳಗೊಂಡಿರಬೇಕು, ಸ್ಕ್ರಿಪ್ಟ್ ಒಳ್ಳೆಯ ರಚನೆಯಿಂದ, ಸರಿಯಾದ ಫಂಕ್ಷನ್ಗಳು ಮತ್ತು ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಜೊತೆಗೆ ಇರಬೇಕು."

Azure MCP ಸರ್ವರ್ ಪೂರ್ಣ, ಉತ್ಪಾದನಾ ರೆಡಿಯಾಗಿ ಇರುವ ಪೈಥಾನ್ ಸ್ಕ್ರಿಪ್ಟ್ ರಚಿಸುತ್ತದೆ:
- ಇತ್ತೀಚಿನ Azure Blob Storage SDK ಕ್ಕೆ ಸರಿಯಾದ async ಮಾದರಿಗಳನ್ನು ಬಳಸಿ
- DefaultAzureCredential ಅನ್ನು ಸಂಕೀರ್ಣ ಬ್ಯಾಕ್ಅಪ್ ಸರಣಿಯ ವಿವರಣೆಗಳೊಂದಿಗೆ ಜಾರಿಗೆ ತಂದಿದೆ
- ನಿರ್ದಿಷ್ಟ Azure исключенияಗಳ ಜೊತೆಗೆ ದೃಢವಾದ ದೋಷ ನಿರ್ವಹಣೆಯನ್ನು ಒಳಗೊಂಡಿರುತ್ತದೆ
- ಸಂಪನ್ಮೂಲ ನಿರ್ವಹಣೆ ಮತ್ತು ಸಂಪರ್ಕ ನಿರ್ವಹಣೆಯ Azure SDK ಉತ್ತಮ ಅಭ್ಯಾಸಗಳನ್ನು ಅನುಸರಿಸುತ್ತದೆ
- ವಿವರವಾದ ಲಾಗಿಂಗ್ ಮತ್ತು ಮಾಹಿತಿಪೂರ್ಣ ಕನ್ಸೋಲ್ ಔಟ್‌ಪುಟ್ ನೀಡುತ್ತದೆ
- ಫಂಕ್ಷನ್ಗಳು, ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಮತ್ತು ಟೈಪ್ ಸೂಚನೆಗಳೊಂದಿಗೆ ಸರಿಯಾದ ರಚನೆಯ ಸ್ಕ್ರಿಪ್ಟ್ ಸೃಷ್ಟಿಸುತ್ತದೆ

Azure MCP ಇಲ್ಲದಿದ್ದರೆ ಸಾಮಾನ್ಯ ಬ್ಲೋಬ್ ಸ್ಟೋರೆಜ್ ಕೋಡ್ ದೊರೆಯಬಹುದು ಆದರೆ ಅದು ಇತ್ತೀಚಿನ Azure ಮಾದರಿಗಳನ್ನು ಅನುಸರಿಸುವುದಿಲ್ಲ. ಆದರೆ Azure MCP ಜೊತೆಗೆ ನೀವು ಇತ್ತೀಚಿನ ದೃಢೀಕರಣ ವಿಧಾನಗಳನ್ನು ಬಳಸುವ, Azure ಖಾಸಗಿ ದೋಷ ಪರಿಸ್ಥಿತಿಗಳನ್ನು ನಿರ್ವಹಿಸುವ ಮತ್ತು Microsoft ಶಿಫಾರಸುಗಳನ್ನು ಅನುಸರಿಸುವ ಕೋಡ್ ಅನ್ನು ಪಡೆಯುತ್ತೀರಿ.

**ಪ್ರಮುಖ ಉದಾಹರಣೆ**: ನಾನು `az` ಮತ್ತು `azd` CLI ಗಾಗಿ ನಿರ್ದಿಷ್ಟ ಕಮಾನ್ಡ್‌ಗಳನ್ನು ನೆನಪಿರಿಸಲು ಕಷ್ಟಪಟ್ಟು ಬರುತ್ತಿದೆ. ನನಗೆ ಯಾವಾಗಲೂ ಎರಡು ಹಂತಗಳ ಪ್ರಕ್ರಿಯೆ: ಮೊದಲಿಗೆ ಸಿಂಟ್ಯಾಕ್ಸ್ ಅನ್ನು ಪರಿಶೀಲಿಸಿ, ನಂತರ ಕಮಾನ್ಡ್ ಅನ್ನು ರನ್ ಮಾಡು. ನಾನು ಅಡೆತಡೆ ಹೊಂದದೆ ಕೆಲಸ ಮಾಡಲು ಪೋರ್ಟಲ್‌ಗೆ ಹೋಗಿ ಕ್ಲಿಕ್ ಮಾಡುವೆನು. ನಾನು ಏನು ಬಯಸುತ್ತೇನೆ ಎಂದು ವಿವರಣೆ ಮಾಡುವುದು ಅದ್ಭುತ, ಮತ್ತು ಅದನ್ನು ನನ್ನ IDE ಬಿಟ್ಟು ಹೊರಡುವದಿಲ್ಲದೇ ಮಾಡಬಹುದಾದುದು ಇನ್ನಷ್ಟು ಚೆನ್ನಾಗಿದೆ!

ನಿಮಗೆ ಪ್ರಾರಂಭಿಸಲು [Azure MCP ರೆಪೊಸಿಟರಿ](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) ನಲ್ಲಿ ಉತ್ತಮ ಬಳಕೆ ಪ್ರಕರಣಗಳ ಪಟ್ಟಿ ಇದೆ. ಸಂಪೂರ್ಣ ಸ್ಥಾಪನೆ ಮಾರ್ಗದರ್ಶಿಗಳು ಮತ್ತು ಸುಧಾರಿತ ಸಂರಚನೆ ಆಯ್ಕೆಗಳಿಗಾಗಿ, [ಅಧಿಕೃತ Azure MCP ಡಾಕ್ಯುಮೆಂಟೇಶನ್](https://learn.microsoft.com/azure/developer/azure-mcp-server/) ಅನ್ನು ಪರಿಶೀಲಿಸಿ.

- ಕಾರ್ಯಪ್ರವಾಹ ಚಾಲನೆಗಳನ್ನು ವಿಶ್ಲೇಷಿಸಿ ಯಾವುದೇ ವೈಫಲ್ಯ ಅಥವಾ ಕಾರ್ಯಕ್ಷಮತಾ ಸಮಸ್ಯೆಗಳನ್ನು ಗುರುತಿಸುತ್ತದೆ
- ಭದ್ರತಾ ಸ್ಕ್ಯಾನ್ ಫಲಿತಾಂಶಗಳನ್ನು ಸಂಗ್ರಹಿಸಿ ಪ್ರಮುಖ ಸೂಚನೆಗಳನ್ನು ಆದ್ಯತೆ ನೀಡುತ್ತದೆ
- ಮರ್ಜಿ PR ಗಳಿಂದ ಮಾಹಿತಿ ಹೊರತೆಗೆಯುವ ಮೂಲಕ ಪೂರ್ಣ ಬಿಡುಗಡೆ ನೋಟ್ಸ್ ರಚಿಸುತ್ತದೆ
- ಸ್ಪ್ರಿಂಟ್ ಯೋಜನೆ ಮತ್ತು ಬಿಡುಗಡೆ ಸಿದ್ಧತೆಗೆ ಕೈಗೊಂಡ ಕ್ರಮಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ

**ಪ್ರಮುಖ ಉದಾಹರಣೆ**: ಕೋಡ್ ವಿಮರ್ಶಾ ಕಾರ್ಯಪ್ರವಾಹದಲ್ಲಿ ಇದನ್ನು ಬಳಸಲು ನನಗೆ ತುಂಬಾ ಇಷ್ಟ. VS Code, GitHub ಸೂಚನೆಗಳು, ಮತ್ತು ಪುಲ್ ರಿಕ್ವೆಸ್ಟ್ ಪುಟಗಳ ನಡುವೆ ಜಿಗಿದಾಡುವ ಬದಲು ನಾನು "ನನ ದಿನನಿಕ ವಿಮರ್ಶೆಗೆ ಕಾಯುತ್ತಿರುವ ಎಲ್ಲಾ PR ಗಳವನ್ನು ತೋರಿಸು" ಮತ್ತು ನಂತರ "PR #123 ನಲ್ಲಿ ದೃಢೀಕರಣ ವಿಧಾನದಲ್ಲಿ ದೋಷ ನಿರ್ವಹಣೆ ಬಗ್ಗೆ ಪ್ರಶ್ನಿಸುವ ಕಾಮೆಂಟ್ ಸೇರಿಸು" ಎಂದು ಹೇಳಬಹುದು. ಸರ್ವರ್ GitHub API ಕರೆಗಳನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ, ಚರ್ಚೆಯ ಸ೦ದರ್ಭ ಉಳಿಸಿಕೊಂಡಿದೆ ಮತ್ತು ವಿಮರ್ಶಾ ಕಾಮೆಂಟ್‌ಗಳನ್ನು ರಚಿಸುವಲ್ಲಿ ಸಹಾಯ ಮಾಡುತ್ತದೆ.

**ದೃಢೀಕರಣ ಆಯ್ಕೆಗಳನ್ನು**: ಸರ್ವರ್ OAuth (VS Code ನಲ್ಲಿನ ಸಡಿಲೀಕರಣ) ಮತ್ತು ವೈಯಕ್ತಿಕ ಪ್ರವೇಶ ಟೋಕನ್ಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ, ನೀವು ಬೇಕಾದ GitHub ಕಾರ್ಯಚಟುವಟಿಕೆಗಳಿಗೆ ಮಾತ್ರ ಟೂಲ್ ಸೆಟ್‌ಗಳನ್ನು ಹೊಂದಿಕೊಳಗಿನ ಮೂಲಕ. ನೀವು ಅದನ್ನು ತಕ್ಷಣದ ಸಂರಚನೆಗಾಗಿ ದೂರದ ಹೋಸ್ಟ್ ಸೇವೆಯಂತೆ ಅಥವಾ ಪೂರ್ಣ ನಿಯಂತ್ರಣಕ್ಕಾಗಿ ಸ್ಥಳೀಯ ಡಾಕರ್ ಮೂಲಕ ಚಾಲನೆ ಮಾಡಬಹುದು.

> **💡 ಪ್ರೋ ಸಲಹೆ**
> 
> ನಿಮ್ಮ MCP ಸರ್ವರ್ ಸೆಟ್ಟಿಂಗ್‌ಗಳಲ್ಲಿ `--toolsets` ಪರಿಮಾಣವನ್ನು ಹೊಂದಿಸಿ ನೀವು ಬೇಕಾದ ಟೂಲ್‌ಸೆಟ್‌ಗಳನ್ನು ಮಾತ್ರ ಸಕ್ರಿಯ ಮಾಡಿ, ಇದರಿಂದ ಸ೦ದರ್ಭದ ಗಾತ್ರವನ್ನು ಕಡಿಮೆ ಮಾಡುವುದರ ಜೊತೆಗೆ AI ಟೂಲ್ ಆಯ್ಕೆಮಾಡುವಿಕೆಯೂ ಸುಧಾರಿಸಬಹುದು. ಉದಾಹರಣೆಗೆ, ಕೋರ್ ಡೆವಲಪ್ಮೆಂಟ್ ಕಾರ್ಯಪ್ರವಾಹಗಳಿಗೆ `"--toolsets", "repos,issues,pull_requests,actions"` ಅನ್ನು ನಿಮ್ಮ MCP ಸಂರಚನೆ ಆರ್ಗ್‌ಗಳಲ್ಲಿ ಸೇರಿಸಿ, ಅಥವಾ ನೀವು ಮುಖ್ಯವಾಗಿ GitHub ಮೇಲ್ವಿಚಾರಣಾ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಬಯಸಿದರೆ `"--toolsets", "notifications, security"` ಉಪಯೋಗಿಸಿ.

**ಇದು ಏನು ಮಾಡುತ್ತದೆ**: MarkItDown ವಿಭಿನ್ನ ಫೈಲ್ ಫಾರ್ಮ್ಯಾಟ್‌ಗಳನ್ನು ಉನ್ನತ ಗುಣಮಟ್ಟದ ಮಾರ್ಕ್‌ಡೌನ್‌ಗೆ ಪರಿವರ್ತಿಸುವ ಸಮಗ್ರ ಡಾಕ್ಯುಮೆಂಟ್ ಕಾಂವರ್ಝನ್ ಸರ್ವರ್ ಆಗಿದ್ದು, LLM ಉಪಯೋಗ ಮತ್ತು ಪಠ್ಯ ವಿಶ್ಲೇಷಣಾ ಕಾರ್ಯಪ್ರವಾಹಗಳಿಗೆ ಪೂರಕವಾಗಿದ್ದು, ಅದನ್ನು ಪರಿಪಾಟಿ ಮಾಡುತ್ತದೆ.

**ಇದು ಯಾಕೆ ಉಪಯುಕ್ತ**: ಆಧುನಿಕ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಕಾರ್ಯಪ್ರವಾಹಗಳಿಗಾಗಿ ಅಗತ್ಯ! MarkItDown ಪ್ರಮುಖ ಡಾಕ್ಯುಮೆಂಟ್ ರಚನೆಗಳನ್ನು (ಹೆಡಿಂಗ್ಸ್, ಪಟ್ಟಿ, ಟೇಬಲ್ಸ್, ಹಾಗೂ ಲಿಂಕ್ಸ್) ಕಾಪಾಡಿಕೊಳ್ಳುತ್ತಾ ವಿಶಾಲ ಫೈಲ್ ಫಾರ್ಮ್ಯಾಟ್‌ಗಳನ್ನೂ ನಿಭಾಯಿಸುತ್ತದೆ. ಸರಳ ಪಠ್ಯ ಎಷ್ಟ್ಯಾಂಕ್ಷನ್ ಸಾಧನಗಳಿಗಿಂತ ವಿಭಿನ್ನವಾಗಿ, ಇದು ಅರ್ಥಮಯ ಅರ್ಥ ಮತ್ತು ಫಾರ್ಮ್ಯಾಟಿಂಗ್ ಅನ್ನು ಕಾಪಾಡಿಕೊಳ್ಳುವುದರಲ್ಲಿ ಗಮನ ಕೇಂದ್ರಿತವಾಗಿದ್ದು, ಇದು AI ಪ್ರಕ್ರಿಯೆ ಹಾಗೂ ಮಾನವ ಓದಲಿಕ್ಕೆ ಮೌಲ್ಯದಾಗಿದೆ.

**ಬೆಂಬಲಿತ ಫೈಲ್ ಫಾರ್ಮ್ಯಾಟ್‌ಗಳು**:
- **ಆಪೀಸ್ ಡಾಕ್ಯುಮೇಂಟ್ಸ್**: PDF, ಪವರ್‌ಪಾಯಿಂಟ್ (PPTX), ವರ್ಡ್ (DOCX), ಎಕ್ಸೆಲ್ (XLSX/XLS)
- **ಮೀಡಿಯಾ ಫೈಲ್‌ಗಳು**: ಚಿತ್ರಗಳು (EXIF ಮೆಟಾಡೇಟಾ ಮತ್ತು OCR ಜೊತೆಗೆ), ಧ್ವನಿ (EXIF ಮೆಟಾಡೇಟಾ ಮತ್ತು ಭಾಷಣ ಲಿಪ್ಯಂತರಣದೊಂದಿಗೆ)
- **ವೆಬ್ ವಿಷಯ**: HTML, RSS ಫೀಡ್‌ಗಳು, YouTube URL ಗಳು, ವಿಕಿಪೀಡಿಯಾ ಪುಟಗಳು
- **ಡೇಟಾ ಫಾರ್ಮ್ಯಾಟ್‌ಗಳು**: CSV, JSON, XML, ZIP ಫೈಲ್‌ಗಳು (ವಿಷಯವಸ್ತುಗಳನ್ನು ಪುನರಾವರ್ತನೆಗೂಡಿಸಿ ಪ್ರಕ್ರಿಯೆ ಮಾಡುತ್ತದೆ)
- **ಪ್ರಕಾಶನ ಫಾರ್ಮ್ಯಾಟ್‌ಗಳು**: EPub, ಜುಪಿಟರ್ ನೋಟ್ಬುಕ್‌ಗಳು (.ipynb)
- **ಇಮೇಲ್**: ಔಟ್‌ಲುಕ್ ಸಂದೇಶಗಳು (.msg)
- **ಉನ್ನತತಮ**: ಹೆಚ್ಚಳವಾದ PDF ಪ್ರಕ್ರಿಯೆಗಾಗಿ Azure ಡಾಕ್ಯುಮೆಂಟ್ ಇಂಟೆಲಿಜೆನ್ಸ್ ಒಗ್ಗಟ್ಟಿನೊಂದಿಗೆ

**ಉನ್ನತ ಸಾಮರ್ಥ್ಯಗಳು**: OpenAI ಕ್ಲೈಂಟ್ ಒದಗಿಸಿದಾಗ LLM-ಚಾಲಿತ ಚಿತ್ರ ವಿವರಣೆಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ, ಹೆಚ್ಚಳವಾದ PDF ಪ್ರಕ್ರಿಯೆಗೆ Azure ಡಾಕ್ಯುಮೆಂಟ್ ಇಂಟೆಲಿಜೆನ್ಸ್, ಭಾಷಣವಿಷಯದ ಧ್ವನಿಗಳನ್ನು ಲಿಪ್ಯಂತರಿಸುವಿಕೆ, ಮತ್ತು ಹೆಚ್ಚುವರಿ ಫೈಲ್ ಫಾರ್ಮ್ಯಾಟ್‌ಗಳಿಗೆ ವಿಸ್ತರಿಸಲು ಪ್ಲಗಿನ್ ವ್ಯವಸ್ಥೆ.

MarkItDown ಡಾಕ್ಯುಮೆಂಟ್ ರಚನೆಯನ್ನು ಅತ್ಯುತ್ತಮವಾಗಿ ಉಳಿಸುವ ಕೆಲಸದಲ್ಲಿ ಪರಿಣತವಾಗಿದೆ, ಇದು AI ಕಾರ್ಯಪ್ರವಾಹಗಳಿಗಾಗಿ ಮುಖ್ಯವಾದುದು. ಉದಾಹರಣೆಗೆ, ಪವರ್‌ಪಾಯಿಂಟ್ ಪ್ರಸ್ತುತಿಕರಣವನ್ನು ಪರಿವರ್ತಿಸುವಾಗ ಎಸ್‌ಲೈಡ್‌ಗಳ ಸಂಘಟನೆ ಸರಿ ಹೊಂದಿರುವ ಹೆಡಿಂಗ್‌ಗಳೊಂದಿಗೆ ಉಳಿಯುತ್ತದೆ, ಟೇಬಲ್‌ಗಳನ್ನು ಮಾರ್ಕ್‌ಡೌನ್ ಟೇಬಲ್‌ಗಳಾಗಿ ಎಷ್ಟ್ಯಾಂಕ್ಷನ್ ಮಾಡುತ್ತದೆ, ಚಿತ್ರಗಳಿಗಾಗಿ ಅಲ್ಟ್ ಪಠ್ಯ ಸೇರಿಸುತ್ತದೆ, ಹಾಗೆಯೇ ಸ್ಪೀಕರ್ ನೋಟ್‌ಗಳನ್ನು ಸಹ ಪ್ರಕ್ರಿಯೆ ಮಾಡುತ್ತದೆ. ಚಾರ್ಟ್‌ಗಳನ್ನು ಓದಲು ಸುಲಭವಾಗುವ ಡೇಟಾ ಟೇಬಲ್‌ಗಳಿಗೆ ಪರಿವರ್ತಿಸಲಾಗುತ್ತದೆ ಮತ್ತು ಫಲಿತಾಂಶ ಮಾರ್ಕ್‌ಡೌನ್ ಮೂಲ ಪ್ರಸ್ತುತಿಕರಣದ ತರ್ಕಸಂಬಂಧಿ ಹರಿವು ಉಳಿಸುತ್ತದೆ. ಇದು AI ವ್ಯವಸ್ಥೆಗಳಿಗೆ ಪ್ರಸ್ತುತಿಕರಣ ವಿಷಯವನ್ನು ನೀಡಲು ಅಥವಾ ಇರುವ ಎಸ್‌ಲೈಡ್‌ಗಳಿಂದ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಸೃಷ್ಟಿಸಲು ಪರಿಪೂರ್ಣವಾಗಿದೆ.

> ಸಮಗ್ರ M365 ಅಭಿವೃದ್ಧಿ ಬೆಂಬಲಕ್ಕಾಗಿ ಈ ಸರ್ವರ್ ಅನ್ನು ಮೈಕ್ರೋಸಾಫ್ಟ್ ಲರ್ನ್ ಡಾಕ್ಸ್ MCP ಸರ್ವರ್ ಜೊತೆಗೆ ಬಳಸಿರಿ – ಒಂದು ಅಧಿಕೃತ ಡಾಕ್ಯುಮೆಂಟೇಷನ್ ಒದಗಿಸುತ್ತದೆ, ಮತ್ತು ಇನ್ನೊಂದು ಪ್ರಾಯೋಗಿಕ ಅಭಿವೃದ್ಧಿ ಸಾಧನಗಳು ಹಾಗೂ ಸಮಸ್ಯೆ ಪರಿಹಾರವನ್ನು ಒದಗಿಸುತ್ತದೆ.


## ಮುಂದೇನು? 🔮

## 📋 ಸಾರಾಂಶ

ಮಾಡೆಲ್ ಕಂಟೆಕ್ಸ್ಟ್ ಪ್ರೋಟೋಕಾಲ್ (MCP) ಅಭಿವೃದ್ಧಿಪಡಿಸುವವರು AI ಸಹಾಯಕರ ಮತ್ತು ಹೊರಗಿನ ಸಾಧನಗಳೊಂದಿಗೆ ಸಂವಹನ ಮಾಡುವ ರೀತಿಯನ್ನು ಪರಿವರ್ತಿಸುತ್ತಿದೆ. ಈ 10 ಮೈಕ್ರೋಸಾಫ್ಟ್ MCP ಸರ್ವರ್‌ಗಳು AI ಸಂಯೋಜನೆಯ ಮಾನ್ಯತೆಯ ಶಕ್ತಿಯನ್ನು ತೋರಿಸುತ್ತಿವೆ, ಶಕ್ತಿಶಾಲಿ ಹೊರಗಿನ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಪ್ರವೇಶಿಸುವಾಗ ನಿರಂತರ ವಹಿವಾಟುಗಳನ್ನು ಸುಗಮಗೊಳಿಸುತ್ತವೆ.

ಸಂಪೂರ್ಣ ಆಜೂರ್ ಪರಿಸರದ ಸಂಯೋಜನೆಯಿಂದ ನಿಯೋಜನೆಗಾಗಿ ಪ್ಲೇ ರೈಟ್‌ನಂತಹ ವಿಶೇಷ ಸಾಧನಗಳು ಮತ್ತು ಡಾಕ್ಯುಮೆಂಟ್ ಪ್ರಕ್ರಿಯೆಗೆ ಮಾರ್ಕ್ ಇಟ್ ಡೌನ್ ವರೆಗೆ, ಈ ಸರ್ವರ್‌ಗಳು ವಿಭಿನ್ನ ಅಭಿವೃದ್ಧಿ ದೃಶ್ಯಾವಳಿಗಳಲ್ಲಿ ಉತ್ಪಾದಕತೆಯನ್ನು ಹೆಚ್ಚಿಸುವಲ್ಲಿ MCP ಹೇಗೆ ಸಹಾಯಕವಾಗಬಹುದು ಎಂಬುದನ್ನು ತೋರಿಸುತ್ತವೆ. ಮಾನ್ಯ ಪ್ರೋಟೋಕಾಲ್ ಈ ಸಾಧನಗಳನ್ನು ಹೊಂದಾಣಿಕೆ ಮಾಡಿಕೊಂಡು ಒಟ್ಟಾರೆ ವಿಕಸನ ಅನುಭವವನ್ನು ನಿರ್ಮಿಸುತ್ತಿದೆ.

MCP ಪರಿಸರ ಮುಂದುವರೆಯುತ್ತಿದ್ದಂತೆ, ಸಮುದಾಯದೊಂದಿಗೆ ತೊಡುಗುವಿಕೆ, ಹೊಸ ಸರ್ವರ್‌ಗಳನ್ನು ಅನ್ವೇಷಣೆ ಮಾಡುವುದು ಮತ್ತು ಸ್ವಂತ ಪರಿಹಾರಗಳನ್ನು ನಿರ್ಮಿಸುವುದು ನಿಮ್ಮ ಅಭಿವೃದ್ಧಿ ಉತ್ಪಾದಕತೆಯನ್ನು ಗರಿಷ್ಠೀಕರಿಸುವ ಪ್ರಮುಖ ಅಂಶಗಳಾಗಿರುತ್ತದೆ. MCP ಸಂಪೂರ್ಣವಾಗಿ ತೆರೆಯಾದ ಮಾನ್ಯತೆಯ ಪ್ರೋಟೋಕಾಲು ಆದ್ದರಿಂದ ವಿವಿಧ ಮಾರಾಟಿಗರು ನೀಡುವ ಸಾಧನಗಳನ್ನು ಮಿಶ್ರಣ ಮಾಡಿ ನಿಮ್ಮ ನಿರ್ದಿಷ್ಟ ಅಗತ್ಯಗಳಿಗೆ ತಕ್ಕಂತೆ ಆದರ್ಶ ಕಾರ್ಯಪ್ರವಾಹವನ್ನು ಸೃಷ್ಟಿಸಬಹುದು.

## 🎯 ಅಭ್ಯಾಸಗಳು

1. **ಸ್ಥಾಪನೆ ಮತ್ತು ಸಂರಚನೆ**: ನಿಮ್ಮ VS ಕೋಡ್ ಪರಿಸರದಲ್ಲಿ ಒಂದುವರ MCP ಸರ್ವರ್ ಸ್ಥಾಪಿಸಿ ಮತ್ತು ಮೂಲ ಕಾರ್ಯಕ್ಷಮತೆಯನ್ನು ಪರೀಕ್ಷಿಸಿ.
2. **ಕಾರ್ಯಪ್ರವಾಹ ಸಂಯೋಜನೆ**: ಕನీసು ಮೂರು ವಿಭಿನ್ನ MCP ಸರ್ವರ್‌ಗಳನ್ನು ಸಂಯೋಜಿಸುವ ಅಭಿವೃದ್ಧಿ ಕಾರ್ಯಪ್ರವಾಹವನ್ನು ರೂಪಿಸಿ.
3. **ಕಸ್ಟಮ್ ಸರ್ವರ್ ಯೋಜನೆ**: ನಿಮ್ಮ ದೈನಂದಿನ ಅಭಿವೃದ್ಧಿ ದಿನಚರಿಯಲ್ಲಿ ಲಾಭದಾಯಕವಾಗುವ ಕಾರ್ಯವನ್ನು ಗುರುತಿಸಿ ಮತ್ತು ಅದರಕ್ಕೆ ಸ್ಪೆಸಿಫಿಕೇಶನ್ ಅನ್ನು ರಚಿಸಿ.
4. **ಕಾರ್ಯಕ್ಷಮತೆ ವಿಶ್ಲೇಷಣೆ**: ಸಾಮಾನ್ಯ ಅಭಿವೃದ್ಧಿ ಕಾರ್ಯಗಳಿಗೆ MCP ಸರ್ವರ್‌ಗಳನ್ನು ಬಳಕೆಯು ಅನ್ವಯಿಕ ವಿಧಾನಗಳೊಂದಿಗೆ ಹೋಲಿಸಿ.
5. **ಭದ್ರತಾ ಅಂಕಿಅಂಶಗಳ ವಿಮರ್ಶೆ**: ನಿಮ್ಮ ಅಭಿವೃದ್ಧಿ ಪರಿಸರದಲ್ಲಿ MCP ಸರ್ವರ್‌ಗಳ ಬಳಕೆಯ ಭದ್ರತಾ ಪರಿಣಾಮಗಳನ್ನು ಮೌಲ್ಯಮಾಪನ ಮಾಡಿ ಮತ್ತು ಉತ್ತಮ ಅಭ್ಯಾಸಗಳನ್ನು ಪ್ರಸ್ತಾವಿಸಿ.
