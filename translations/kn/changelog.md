# ಅಪ್ಡೇಟ್ ದಾಖಲೆ: MCP ಗಾಗಿ ಆರಂಭಿಕರ ಪಠ್ಯಕ್ರಮ

## ಜೂನ್ 16, 2026

### MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ ಹೊಂದಾಣಿಕೆ ಮತ್ತು ನಿದರ್ಶನ ಪರಿಶೀಲನೆ

ಪಠ್ಯಕ್ರಮವನ್ನು ಪ್ರಸ್ತುತ **MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ 2025-11-25** ಹಾಗೂ ಇತ್ತೀಚಿನ ಅಧಿಕೃತ SDK ಗಳ ವಿರುದ್ಧ ಪರಿಶೀಲಿಸಿ, ಉಳಿದಿರುವ ಹಳೆಯ ಸ್ಪೆಸಿಫಿಕೇಶನ್ ಉಲ್ಲೇಖಗಳನ್ನು ಸರಿಪಡಿಸಿ, ಮತ್ತು ಮೂಲ ನಿದರ್ಶನಗಳು ಇನ್ನೂ ಕಟ್ಟಿಹಾಕಬಲ್ಲ ಮತ್ತು ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತವೆ ಎಂದು ದೃಢಪಡಿಸಲಾಗಿದೆ.

#### ಸ್ಪೆಸಿಫಿಕೇಶನ್ ಆವೃತ್ತಿ ಸರಿಪಡಣೆಗಳು (2025-06-18 / 2025-03-26 → 2025-11-25)

ಹಳೆಯ ಸ್ಪೆಸಿಫಿಕೇಶನ್ ನ ಪ್ರಸ್ತುತ/ಇತ್ತೀಚಿನ* ಮಾನದಂಡ ಎಂದು ಎತ್ತಿಹಿಡಿದಿರುವ ಇಂಗ್ಲಿಷ್ ವಿಷಯವನ್ನು ಅಪ್‌ಡೇಟ್ ಮಾಡಲಾಗಿದೆ ಮತ್ತು ಲಿಂಕ್‌ಗಳನ್ನು ಕಾನೋನಿಕಲ್ `modelcontextprotocol.io` ಸ್ಪೆಸಿಫಿಕೇಶನ್ ಪಥಗಳಿಗೆ ಮರುನಿರ್ದೇಶಿಸಲಾಗಿದೆ:
- **05-AdvancedTopics/mcp-security/README.md**: "ಪ್ರಸ್ತುತ ಮಾನದಂಡ" ಬ್ಯಾನರ್, ಪರಿಚಯ, ಪ್ರಮುಖ ಭದ್ರತಾ ತತ್ವಗಳು, ಆವಶ್ಯಕ ಅಗತ್ಯಗಳು, Microsoft Entra ID ವಿಭಾಗ, ಉಲ್ಲೇಖಗಳು ಮತ್ತು ಸಂಪನ್ಮೂಲಗಳ ಲಿಂಕ್ ಮತ್ತು ಮುಕ್ತಾಯ ಭದ್ರತಾ ಸೂಚನೆ (8 ಉಲ್ಲೇಖಗಳು) 2025-11-25 ಗೆ ಅಪ್‌ಡೇಟ್ ಮಾಡಲಾಗಿದೆ
- **05-AdvancedTopics/mcp-transport/README.md**: ಹೆಚ್ಚುವರಿ ಸಂಪನ್ಮೂಲಗಳ ಸ್ಪೆಸಿಫಿಕೇಶನ್ ಲಿಂಕ್ ಮತ್ತು "ಪ್ರಸ್ತುತ ಮಾನದಂಡ" ಬ್ಯಾನರ್ 2025-11-25 ಗೆ ಅಪ್‌ಡೇಟ್
- **05-AdvancedTopics/mcp-realtimesearch/README.md**: ಹಳೆಯ `2025-03-26` ಭದ್ರತೆ ಮತ್ತು ನಂಬಿಕೆ ಲಿಂಕ್ 2025-11-25 ಭದ್ರತಾ ಉತ್ತಮ ಅಭ್ಯಾಸ ಪುಟದೊಂದಿಗೆ ಬದಲಾಯಿಸಲಾಗಿದೆ
- **03-GettingStarted/14-sampling/README.md**: ಅಧಿಕೃತ ನಿದರ್ಶನ ಡಾಕ್ಸ್ ಲಿಂಕ್ 2025-11-25 ಗೆ ಅಪ್‌ಡೇಟ್ ಮಾಡಲಾಗಿದೆ
- **03-GettingStarted/05-stdio-server/README.md**: ಪ್ರಸ್ತುತ "ಪ್ರಸ್ತುತ MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್" ಉಲ್ಲೇಖ ಮತ್ತು ಹೆಚ್ಚುವರಿ ಸಂಪನ್ಮೂಲಗಳ ಲಿಂಕ್ 2025-11-25 ಗೆ ಅಪ್‌ಡೇಟ್ (ವೈದ್ಯಮಾನ SSE-ಪರಿಹಾರ ಟಿಪ್ಪಣಿಗಳು ಸರಿಯಾದಂತೆ ಉಳಿಸಲಾಗಿದೆ)

#### ಪ್ರಸ್ತುತ SDK ಗಳ ವಿರುದ್ಧ ನಿದರ್ಶನ ಪರಿಶೀಲನೆ

- **TypeScript (03-GettingStarted/01-first-server/solution/typescript)**: `npm install` ಮೂಲಕ `@modelcontextprotocol/sdk@1.29.0` ಅನ್ನು ಸ್ಥಾಪಿಸಲಾಗಿದೆ; `tsc --noEmit` ಯಾವುದೇ ಪ್ರಕಾರ ದೋಷವಿಲ್ಲದೇ ಪಾಸಾಗಿದೆ — ಇರುವ `McpServer`/`StdioServerTransport` APIs ಮಾನ್ಯವಾಗಿವೆ
- **Python (03-GettingStarted/01-first-server/solution/python)**: ತಾರತಮ್ಯ `.venv` ನಲ್ಲಿ `mcp[cli]` (1.27.2) ಯೊಂದಿಗೆ ಪರಿಶೀಲಿಸಲಾಗಿದೆ; `py_compile` ಪಾಸಾಗಿದೆ ಮತ್ತು `FastMCP.list_tools()` ಸರಿಯಾಗಿ `add` ಮತ್ತು `subtract` ಉಪಕರಣಗಳನ್ನು ಹಿಂತಿರುಗಿಸಿದೆ
- ಎಲ್ಲಾ `@modelcontextprotocol/sdk` ಆವೃತ್ತಿ ಶ್ರೇಣಿಗಳು (`>=1.26.0` / `^1.26.0` / `^1.27.0`) ಪ್ರಸ್ತುತ `1.29.0` ಗೆ ಸುಗಮವಾಗಿ ಮ್ಯಾಪ್ ಆಗುತ್ತದೆ ಮತ್ತು ಯಾವುದೆ API ಮುರಿಯುವ ಬದಲಾವಣೆಗಳಿಲ್ಲ

#### ಅವಲಂಬನೆ ಪಿನ್ ಹೊಂದಾಣಿಕೆ (ಆವೃತ್ತಿ ಜಾಸ್ತಿ ತಪ್ಪಿಸಿದೆಗಳ ಮುಚ್ಚುವಿಕೆ)

ಹಳೆಯ SDK ಪಿನ್‌ಗಳನ್ನು ತಿದ್ದಿ ಪ್ರತಿ ನಿದರ್ಶನವು ಪ್ರಸ್ತುತ MCP ಬಿಡುಗಡೆಯ ನಿಯಂತ್ರಣ ಅನುಸರಿಸುತ್ತದೆ:

- **10-StreamliningAIWorkflows.../lab3/code/weather_mcp/pyproject.toml** ಮತ್ತು **lab4/code/github_mcp_server/pyproject.toml**: ಖಾಸಗಿ ಪಿನ್ `mcp==1.23.0` ನಿಂದ `mcp>=1.26.0` ಗೆ ವೃದ್ಧಿಸಲಾಗಿದೆ; ಎರಡು `uv.lock` ಫೈಲ್‌ಗಳ್ನೂ ಪುನರ್ ರಚಿಸಲಾಗಿದೆ `uv lock` ಮೂಲಕ, ಲಾಕ್‌ಫೈಲ್‌ಗಳು ಪ್ರಸ್ತುತ `mcp 1.27.2` ಗೆ ಸುಗ್ಗತೆಯಿಂದ ಸರಿ ಹೊಂದಿವೆ ಮತ್ತು ಮ್ಯಾನಿಫೆಸ್ಟ್‌ಗಳೊಂದಿಗೆ ಸಮನ್ವಯದಲ್ಲಿವೆ

### ಭದ್ರತಾ ಬಲವರ್ಧನೆ ಮತ್ತು ಅವಲಂಬನೆ ಪ್ರಬಲತೆ ಪರಿಹಾರ

ಪ್ರತಿ ಅವಲಂಬನೆ ಮ್ಯಾನಿಫೆಸ್ಟ್ ಮತ್ತು ನಿದರ್ಶನ ಮೂಲ ಕೋಡ್ ಮೇಲೆ ಸಂಪೂರ್ಣ ಭದ್ರತಾ ಪರೀಕ್ಷೆ ನಡೆಸಿ, ಎಲ್ಲಾ ವರದಿಯಾದ npm ಸಲಹೆಗಳ ಮತ್ತು ಒಂದು ಕೋಡ್ ಮಟ್ಟದ ಕಂಡುಬಂದ ತಪ್ಪನ್ನು ಸರಿಪಡಿಸಿದ್ದಾರೆ. ಪರಿಹಾರದ ನಂತರ, `npm audit` ಪ್ರತಿಯೊಂದು ಪರಿಶೀಲಿಸಿದ ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ **0 ದುರ್ಬಲತೆಗಳನ್ನು** ವರದಿ ಮಾಡುತ್ತದೆ.

#### npm ಅವಲಂಬನೆ ದುರ್ಬಲತೆಗಳು (ಪರೋಕ್ಷ) — ಸರಿಪಡಿಸಲಾಗಿದೆ

ನಿಮಿಷ 15 ಪ್ರತಿಮಾದ `package-lock.json` ಫೈಲ್‌ಗಳನ್ನು ಅಡ್ಡಪರೀಕ್ಷೆ ಮಾಡಲಾಗಿದೆ. ದುರ್ಬಲತೆಗಳು MCP ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಡೆವ್ ಟೂಲ್, OpenAI ಕ್ಲೈಂಟ್, ಮತ್ತು MCP SDK ಯಿಂದ ಟರಾಂಸಿಟಿವ್ ಅವಲಂಬನೆಗಳಾಗಿದ್ದು; ಇವುನ್ಮುಖವಾಗಿ ಪರಿಹರಿಸಲ್ಪಟ್ಟಿದ್ದು ನಿದರ್ಶನಗಳನ್ನು ಮುರಿಯದು:
- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/inspector** ಮತ್ತು **lab3/code/weather_mcp/inspector**: `@modelcontextprotocol/inspector` (`0.16.6` / `0.14.1` → `0.22.0`) ಮೂರುಿಕರಣದ ಮೇರೆಗೆ `ajv`, `brace-expansion`, `diff`, `path-to-regexp` ಮತ್ತು `ws` ಸಲಹೆಗಳು ನಿವಾರಿಸಲ್ಪಟ್ಟವು. `concurrently` ಮೂಲಕ ಬಾಕಿ ಇರುವ ತೀವ್ರ ಸಲಹೆಯನ್ನು ಕೊನೆಗೊಳ್ಳಿಸುವುದಕ್ಕಾಗಿ `shell-quote@1.8.4` ಫಿಕ್ಸ್ ಜೋಡಿಸಿದ npm `overrides` ದಾಖಲಿಸುವುದು ಸೇರಿಸಲಾಗಿದೆ; ಎರಡೂ ಲಾಕ್‌ಫೈಲ್‌ಗಳು ಪುನರ್ ರಚಿಸಲಾಗಿದೆ (ಯಾವುದೇ ದುರ್ಬಲತೆ ಇಲ್ಲ)
- **03-GettingStarted/samples/typescript**: `npm audit fix` ಮೂಲಕ ಟರಾಂಸಿಟಿವ್ `qs` (ಮಧ್ಯಮ) ನವೀಕರಿಸಲಾಗಿದೆ
- **03-GettingStarted/samples/javascript**: `npm audit fix` ಮೂಲಕ ಟರಾಂಸಿಟಿವ್ `hono` (ಮಧ್ಯಮ) ನವೀಕರಿಸಲಾಗಿದೆ
- **03-GettingStarted/03-llm-client/solution/typescript**: `npm audit fix` ಮೂಲಕ ಟರಾಂಸಿಟಿವ್ `form-data` (ಎತ್ತರ) ನವೀಕರಿಸಲಾಗಿದೆ
- **03-GettingStarted/11-simple-auth/solution/typescript**: `package-lock.json` ಸೃಷ್ಟಿಸಿರುವುದರಿಂದ ಪ್ರಾಜೆಕ್ಟ್ ಪುನರ್ಜನನೀಯ ಮತ್ತು ಪರಿಶೀಲನೀಯವಾಗಿದೆ (0 ದುರ್ಬಲತೆ)

#### ಕೋಡ್ ಮಟ್ಟದ ಭದ್ರತಾ ದೋಷ (OWASP A03: ಅಥವಾಚನೆ)

- **10-StreamliningAIWorkflows.../lab4/code/github_mcp_server/src/server.py**: `open_in_vscode` ಟೂಲ್ ನಲ್ಲಿ `shell=True` ತೆಗೆದುಹಾಕಿತು. ಹಿಂದಿನ `subprocess.run(["start", "", vscode_path, folder_path], shell=True)` ಫೋಲ್ಡರ್ ಪಥದ ಶೆಲ್ ಮೆಟಾಕ್ಯಾರೆಕ್ಟರ್‌ಗಳು `cmd.exe` ಮೂಲಕ ವ್ಯಾಖ್ಯಾನಿಸಬಹುದು (ಕಮಾಂಡ್-ಇಂಜೆಕ್ಶನ್ ಸೇತು). ಈಗ ನೇರವಾಗಿ ಪರಿಹರಿಸಿದ `Code.exe` ಅನ್ನು ಫೋಲ್ಡರ್ ಅರ್ಗ್ಯುಮೆಂಟ್ ಸಹಿತ ಲಾಂಚ್ ಮಾಡುತ್ತದೆ — ಶೆಲ್ ಇಲ್ಲದೆ — ಕ್ರಿಯಾಶೀಲರೂಪದಲ್ಲಿ ಸಮಾನ ಮತ್ತು ಸುರಕ್ಷಿತ

#### ಮೌಲಿಕ ಕಲ್ಪನೆಗಳ ವೃದ್ಧಿ (01-CoreConcepts/)
- **ಹೊಸ ಕ್ಲೈಂಟ್ ಪ್ರಿಮಿಟಿವ್ - ರೂಟ್ಸ್**: ಫೈಲ್‌ಸಿಸ್ಟಮ್ ಗಡಿಗಳನ್ನು ತಿಳಿದುಕೊಳ್ಳಲು ಮತ್ತು ಪ್ರವೇಶ ಅನುಮತಿಗಳನ್ನು ನಿರ್ಧರಿಸಲು ಸರ್ವರ್‌ಗಳಿಗೆ ಸಾಧ್ಯವಾಗುವ ರೂಟ್ಸ್ ಕ್ಲೈಂಟ್ ಪ್ರಿಮಿಟಿವ್‍‌ನ ಸಮಗ್ರ ಡಾಕ್ಯುಮೆಂಟೇಷನ್ ಸೇರಿಸಲಾಗಿದೆ
- **ಸಾಧನ ಟಿಪ್ಪಣಿಗಳು**: ಸಾಧನ ಚಟುವಟಿಕೆ ನಿರ್ಧಾರಗಳಿಗೆ ಉತ್ತಮ ಮೌಲ್ಯ ನೀಡಲು `readOnlyHint`, `destructiveHint` ಟಿಪ್ಪಣಿಗಳ ಬಗ್ಗೆ ಡಾಕ್ಯುಮೆಂಟೇಷನ್ ಸೇರಿಸಲಾಗಿದೆ
- **ಸ್ಯಾಂಪ್ಲಿಂಗ್‌ನಲ್ಲಿ ಸಾಧನ ಕರೆ**: ಮಾದರಿ ಚಾಲಿತ ಸಾಧನ ಕರೆಗಾಗಿ `tools` ಮತ್ತು `toolChoice` ಪರಿಮಾಣಗಳನ್ನು ಒಳಗೊಂಡಂತೆ ಸ್ಯಾಂಪ್ಲಿಂಗ್ ಡಾಕ್ಯುಮೆಂಟ್ ನವೀಕರಿಸಲಾಗಿದೆ
- **URL ಮೋಡ್ ಎಲಿಸಿಟೇಶನ್**: ಸರ್ವರ್ ಆರಂಭಿಸುವ ಬಾಹ್ಯ ವೆಬ್ ಸಂವಹನಕ್ಕಾಗಿ URL ಆಗಿರುವ ಎಲಿಸಿಟೇಶನ್ ಬಗ್ಗೆ ಡಾಕ್ಯುಮೆಂಟ್ ಸೇರಿಸಲಾಗಿದೆ
- **ಕಾರ್ಯಗಳು (ಪ್ರಾಯೋಗಿಕ)**: ದುರ್ಬಲ ಕಾರ್ಯ ಕಾರ್ಯಾಚರಣೆ ಸುತ್ತಲು ನಿರಂತರ ಫಲಿತಾಂಶ ಪಡೆದುಕೊಳ್ಳುವ ಪ್ರಾಯೋಗಿಕ ಕಾರ್ಯಗಳ ನವೀಕರಣ ಸექ್ಷನ್ ಸೇರಿಸಲಾಗಿದೆ
- **ಐಕಾನ್ಸ್ ಬೆಂಬಲ**: ಸಾಧನಗಳು, ಸಂಪನ್ಮೂಲಗಳು, ಸಂಪನ್ಮೂಲ ಟೆಂಪ್ಲೇಟ್‌ಗಳು ಮತ್ತು ಪ್ರಾಂಪ್ಟ್‌ಗಳಲ್ಲಿ ಐಕಾನ್‌ಗಳು ಸೇರಿಸಲಾಗಿವೆ ಎಂಬುದನ್ನು ಗಮನಿಸಿದಂತೆ

#### ಸ್ಪೆಸಿಫಿಕೇಶನ್ ಅನುಕೂಲತಾ ದೃಢೀಕರಣ
- **ಪ್ರೋಟೋಕಾಲ್ ಆವೃತ್ತಿ**: ಎಲ್ಲಾ ಡಾಕ್ಯುಮೆಂಟ್ಸ್ ಇತ್ತೀಚಿನ MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ 2025-11-25 ನಂತಹವೋ ಎಂದು ಪರಿಶೀಲಿಸಲಾಗಿದೆ
- **ವಾಸ್ತುಶಿಲ್ಪ ಹೊಂದಾಣಿಕೆ**: ಎರಡು-ಮಟ್ಟದ ವಾಸ್ತುಶಿಲ್ಪ (ಡೇಟಾ ಲೇಯರ್ + ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಲೇಯರ್) ಡಾಕ್ಯುಮೆಂಟ್ ಸರಿಹೊಂದಿರುವುದು ಖಚಿತಪಡಿಸಲಾಗಿದೆ
- **ಪ್ರಿಮಿಟಿವ್ಸ್ ಡಾಕ್ಯುಮೆಂಟೇಷನ್**: ಸರ್ವರ್ ಪ್ರಿಮಿಟಿವ್ಸ್ (ಸಂಪನ್ಮೂಲಗಳು, ಪ್ರಾಂಪ್ಟ್‌ಗಳು, ಸಾಧನಗಳು) ಮತ್ತು ಕ್ಲೈಂಟ್ ಪ್ರಿಮಿಟಿವ್ಸ್ (ಸ್ಯಾಂಪ್ಲಿಂಗ್, ಎಲಿಸಿಟೇಶನ್, ಲಾಗಿಂಗ್, ರೂಟ್ಸ್) ಪರಿಶೀಲಿಸಲಾಗಿದೆ
- **ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ವ್ಯವಸ್ಥೆಗಳು**: STDIO ಮತ್ತು ಸ್ಟ್ರೀಮಬಲ್ HTTP ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ಗಳ ಸರಿಯಾದತೆ ಪರಿಶೀಲಿಸಲಾಗಿದೆ
- **ಸುರಕ್ಷತಾ ಮಾರ್ಗದರ್ಶನ**: ಇತ್ತೀಚಿನ MCP ಸುರಕ್ಷತಾ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳೊಂದಿಗೆ ಹೊಂದಾಣಿಕೆ ದೃಢಪಟ್ಟಿದೆ

### ಪಾಠಕ್ರಮ ವ್ಯಾಪ್ತಿಯ MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ 2025-11-25 ಹೊಂದಾಣಿಕೆ

#### ಮಾಡ್ಯೂಲ್ 03 - ಪ್ರಾರಂಭಿಸುವುದು
- **SDK ಡಾಕ್ಯುಮೆಂಟೇಷನ**: ಅಧಿಕೃತ SDK ಪಟ್ಟಿಯಲ್ಲಿ Go SDK ಸೇರಿಸಲಾಗಿದೆ; ಎಲ್ಲಾ SDK ಉಲ್ಲೇಖಗಳನ್ನು MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ 2025-11-25ಗೆ ಹೊಂದುವಂತೆ ನವೀನಗೊಳಿಸಲಾಗಿದೆ
- **ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಸ್ಪಷ್ಟತೆ**: STDIO ಮತ್ತು HTTP ಸ್ಟ್ರೀಮಿಂಗ್ ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ವಿವರಣೆಗೆ ಸ್ಪಷ್ಟ ಮತ್ತು ಅಧಿಕೃತ ಸ್ಪೆಕ್ ಉಲ್ಲೇಖಗಳನ್ನು ಸೇರಿಸಲಾಗಿದೆ

#### ಮಾಡ್ಯೂಲ್ 04 - ಪ್ರಾಯೋಗಿಕ ಅನುಷ್ಠಾನ
- **SDK ನವೀಕರಣಗಳು**: Go SDK ಸೇರಿಸಲಾಗಿದೆ; SDK ಪಟ್ಟಿಯ ಸ್ಪೆಸಿಫಿಕೇಶನ್ ಆವೃತ್ತಿ ಉಲ್ಲೇಖವನ್ನು ನವೀಕರಿಸಲಾಗಿದೆ
- **ಅಧಿಕಾರ ಒಪ್ಪಿಗೆಯ ಸ್ಪೆಸಿಫಿಕೇಶನ್**: MCP ಅಧಿಕೃತ 2025-11-25 ಆವೃತ್ತಿಯ ಸಂಪರ್ಕ ಸರಿಪಡಿಸಲಾಗಿದೆ

#### ಮಾಡ್ಯೂಲ್ 05 - ಉನ್ನತ ವಿಷಯಗಳು
- **ಹೊಸ ವೈಶಿಷ್ಟ್ಯಗಳು**: MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ 2025-11-25 ಹೊಸ ವೈಶಿಷ್ಟ್ಯಗಳ ಬಗ್ಗೆ ಟಿಪ್ಪಣಿ ಸೇರಿಸಲಾಗಿದೆ (ಕಾರ್ಯಗಳು, ಸಾಧನ ಟಿಪ್ಪಣಿಗಳು, URL ಮೋಡ್ ಎಲಿಸಿಟೇಶನ್, ರೂಟ್ಸ್)
- **ಸುರಕ್ಷತಾ ಸಂಪನ್ಮೂಲಗಳು**: OWASP MCP Top 10 ಮತ್ತು ಶೆರ್ಪಾ ಕಾರ್ಯಾಗಾರದ ಲಿಂಕ್‌ಗಳು ಹೆಚ್ಚಿಸಲಾಗಿದೆ

#### ಮಾಡ್ಯೂಲ್ 06 - ಸಮುದಾಯ ಕೊಡುಗೆಗಳು
- **SDK ಪಟ್ಟಿ**: Swift ಮತ್ತು Rust SDK ಸೇರಿಸಲಾಗಿದೆ; MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ 2025-11-25 ಆವೃತ್ತಿಯ ನವೀಕರಣ ಉಲ್ಲೇಖ
- **ಸ್ಪೆಕ್ ಉಲ್ಲೇಖ**: MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ ನೇರ URLಗೆ ಲಿಂಕ್ ನವೀಕರಿಸಲಾಗಿದೆ

#### ಮಾಡ್ಯೂಲ್ 07 - ಆರಂಭಿಕ ಸ್ವೀಕಾರದಿಂದ ಪಾಠಗಳು
- **ಸಂಪನ್ಮೂಲ ನವೀಕರಣಗಳು**: MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ 2025-11-25 ಮತ್ತು OWASP MCP Top 10 ಅಪಾಯ ಸಂಪನ್ಮೂಲ ಸೇರಿಸಲಾಗಿದೆ

#### ಮಾಡ್ಯೂಲ್ 08 - ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು
- **ಸ್ಪೆಕ್ ಆವೃತ್ತಿ**: MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ ಅನ್ನು 2025-11-25ಕ್ಕೆ ನವೀಕರಿಸಲಾಗಿದೆ
- **ಸುರಕ್ಷತಾ ಸಂಪನ್ಮೂಲಗಳು**: OWASP MCP Top 10 ಮತ್ತು ಶೆರ್ಪಾ ಕಾರ್ಯಾಗಾರ ಸೇರಿಸಲಾಗಿದೆ

#### ಮಾಡ್ಯೂಲ್ 10 - AI ಕಾರ್ಯಪ್ರವಾಹಗಳನ್ನು ಸುಧಾರಿಸುವುದು
- **ಬ್ಯಾಡ್ಜ್ ನವೀಕರಣ**: MCP ಆವೃತ್ತಿ ಬ್ಯಾಡ್ಜ್ SDK ಆವೃತ್ತಿಯಿಂದ (1.9.3) ಸ್ಪೆಸಿಫಿಕೇಶನ್ ಆವೃತ್ತಿ (2025-11-25)ಗೆ ಬದಲಿಸಲಾಗಿದೆ
- **ಸಂಪನ್ಮೂಲ ಲಿಂಕ್ಸ್**: MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ ಲಿಂಕ್ ನವೀಕರಿಸಿ OWASP MCP Top 10 ಸೇರಿಸಲಾಗಿದೆ

#### ಮಾಡ್ಯೂಲ್ 11 - MCP ಸರ್ವರ್ ಹ್ಯಾಂಡ್ಸ್-ಆನ್ ಪ್ರಯೋಗಾಲಯಗಳು
- **ಸ್ಪೆಕ್ ಉಲ್ಲೇಖ**: MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ 2025-11-25 ಆವೃತ್ತಿಗೆ ಲಿಂಕ್ ನವೀಕರಿಸಲಾಗಿದೆ
- **ಸುರಕ್ಷತಾ ಸಂಪನ್ಮೂಲಗಳು**: ಅಧಿಕೃತ ಸಂಪನ್ಮೂಲಗಳಿಗೆ OWASP MCP Top 10 ಸೇರಿಸಲಾಗಿದೆ

#### MCP ಸುರಕ್ಷತಾ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು (02-Security/mcp-best-practices.md) - ಸ್ಪೆಸಿಫಿಕೇಶನ್ ಆವೃತ್ತಿ ನವೀಕರಣ
- **ಪ್ರೋಟೋಕಾಲ್ ಆವೃತ್ತಿ ನವೀಕರಣ**: ಇತ್ತೀಚಿನ MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ 2025-11-25 (ನವೆಂಬರ್ 25, 2025 ಬಿಡುಗಡೆ)ಗೆ ಉಲ್ಲೇಖ ನವೀಕರಿಸಲಾಗಿದೆ
  - ಎಲ್ಲಾ ಸ್ಪೆಸಿಫಿಕೇಶನ್ ಆವೃತ್ತಿ ಉಲ್ಲೇಖಗಳನ್ನು 2025-06-18 ರಿಂದ 2025-11-25 ಗೆ ನವೀಕರಿಸಲಾಗಿದೆ
  - ಡಾಕ್ಯುಮೆಂಟ್ ದಿನಾಂಕ ಉಲ್ಲೇಖಗಳನ್ನು ಆಗಸ್ಟ್ 18, 2025 ರಿಂದ ಡಿಸೆಂಬರ್ 18, 2025 ಗೆ ನವೀಕರಿಸಲಾಗಿದೆ
  - ಎಲ್ಲಾ ಸ್ಪೆಸಿಫಿಕೇಶನ್ URL ಗಳು ಇತ್ತೀಚಿನ ಡಾಕ್ಯುಮೆಂಟೇಷನ್‌ಗೆ ಸೂಚಿಸುತ್ತವೆ ಎಂದು ದೃಢೀಕರಿಸಲಾಗಿದೆ
- **ವಿಷಯ ಪರಿಶೀಲನೆ**: ಇತ್ತೀಚಿನ ಮಾನದಂಡಗಳಿಗೆ ಅನುಗುಣವಾಗಿ ಸುರಕ್ಷತಾ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳ ಸಮಗ್ರ ಪರಿಶೀಲನೆ
  - **Microsoft ಸುರಕ್ಷತಾ ಪರಿಹಾರಗಳು**: ಪ್ರಾಂಪ್ಟ್ ಶೀಲ್ಡ್ಸ್ (ಹಿಂದಿನ "ಜೈಲ್ ಬ್ರೇಕ್ ಅಪಾಯ ಪತ್ತೆ"), Azure ವಿಷಯ ಸುರಕ್ಷತೆ, Microsoft Entra ID, ಮತ್ತು Azure ಕೀ ვಾಲ್ಟ್‌ಗೆ ಇತ್ತೀಚಿನ ಪದವಿ ಮತ್ತು ಲಿಂಕ್‌ಗಳನ್ನು ಪರಿಶೀಲಿಸಲಾಗಿದೆ
  - **OAuth 2.1 ಸುರಕ್ಷತೆ**: ಇತ್ತೀಚಿನ OAuth ಸುರಕ್ಷತಾ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳಿಗೆ ಅನುಗುಣತೆ ದೃಢೀಕರಿಸಲಾಗಿದೆ

  - **Azure ಸೇವೆಗಳು**: ಎಲ್ಲಾ Microsoft Azure ಡಾಕ್ಯುಮೆಂಟ್ ಲಿಂಕ್‌ಗಳು ಮತ್ತು ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು ಪರಿಶೀಲಿಸಲ್ಪಟ್ಟಿವೆ
- **ಮಾನದಂಡ ಹೊಂದಾಣಿಕೆ**: ಎಲ್ಲ ಉಲ್ಲೇಖಿತ ಸುರಕ್ಷತಾ ಮಾನದಂಡಗಳು ಇತ್ತೀಚಿನವು ಎಂದು ದೃಢೀಕರಿಸಲಾಗಿದೆ

### ಡಾಕ್ಯುಮೆಂಟ್ ಗುಣಮಟ್ಟದ ಖಾತ್ರಿ

- **ಸ್ಪೆಸಿಫಿಕೇಶನ್ ಅನುಕೂಲತೆ**: ಎಲ್ಲಾ MCP ಸುರಕ್ಷತಾ ಅಗತ್ಯತೆಯುಗಳು (ಮუს್ಟ್/ಕಡಿವ) ಇತ್ತೀಚಿನ ಸ್ಪೆಸಿಫಿಕೇಶನ್‌ಗೆ ಅನುಗುಣವಾಗಿವೆ

- **ಉತ್ತಮ ಅಭ್ಯಾಸಗಳ ವ್ಯಾಪ್ತಿ**: ಪ್ರಮಾಣೀಕರಣ, ಅಧಿಕಾರದ ನಿರ್ವಹಣೆ, AI-ಸಂಬಂಧಿತ ಅಪಾಯಗಳು, ಸರಬರಾಜು ಸರಪಳಿ ಸುರಕ್ಷತೆ ಮತ್ತು ಎಂಟರ್ಪ್ರೈಸ್ ಮಾದರಿಗಳ ಸಂಪೂರ್ಣ ವ್ಯಾಪ್ತಿಯನ್ನು ದೃಢೀಕರಿಸಲಾಗಿದೆ

- **ಹೊಸ ಅಧ್ಯಾಯ ಸೇರಿಸಲಾಗಿದೆ**: ಸಾಮಾನ್ಯ ಮತ್ತು ಕಡಿಮೆ ಮಟ್ಟದ ಸರ್ವರ್ ವಾಸ್ತುಶಿಲ್ಪಗಳನ್ನು ಒಳಗೊಂಡ ಸಮಗ್ರ ಮಾರ್ಗದರ್ಶಿಯನ್ನು ಪರಿಚಯಿಸಲಾಗಿದೆ.
  - **ನಿಯಮಿತ vs. ಕಡಿಮೆ-ಮಟ್ಟದ ಸರ್ವರ್**: ಎರಡೂ ವಿಧಾನಗಳಿಗಾಗಿ ಪೈಥಾನ್ ಮತ್ತು ಟೈಪ್ಸ್ಕ್ರಿಪ್ಟ್‌ನಲ್ಲಿ ವಿವರವಾದ ಹೋಲಿಕೆ ಮತ್ತು ಕೋಡ್ ಉದಾಹರಣೆಗಳು.
  - **ಹ್ಯಾಂಡ್ಲರ್-ಆಧಾರಿತ ವಿನ್ಯಾಸ**: ತೂಕಸಾರುವ, ಸುಲಭವಾಗಿ ಬದಲು ಮಾಡಬಹುದಾದ సರ್ವర్ ನೈರ್ಮಾಣಗಳಿಗೆ ಹ್ಯಾಂಡ್ಲರ್-ಆಧಾರಿತ ಟೂಲ್/ಸಂಪನ್ಮೂಲ/ಪೂರ್ಣಾಂಶ ನಿರ್ವಹಣೆಯ ವಿವರಣೆ.
  - **ಪ್ರಾಯೋಗಿಕ ಮಾದರಿಗಳು**: ಕಡಿಮೆ-ಮಟ್ಟದ ಸರ್ವರ್ ಮಾದರಿಗಳು ಸುಧಾರಿತ ವೈಶಿಷ್ಟ್ಯಗಳು ಮತ್ತು ವಾಸ್ತುಶಿಲ್ಪಕ್ಕೆ ಹೇಗೆ ಪ್ರಯೋಜನಕರವಾಗುತ್ತವೆ ಎಂಬ ನೈಜ-ಜೀವನ ಘಟನೆಗಳು.

#### ಸರಳ ಪ್ರಾಮಾಣೀಕರಣ (03-GettingStarted/11-simple-auth)
- **ಹೊಸ ಅಧ್ಯಾಯ ಸೇರಿಸಲಾಗಿದೆ**: MCP ಸರ್ವರ್‌ಗಳಲ್ಲಿ ಸರಳ ಪ್ರಾಮಾಣೀಕರಣವನ್ನು ಅಳವಡಿಸುವ ಹಂತಗಳಿಂದ ಹಂತದ ಮಾರ್ಗದರ್ಶನ.
  - **ಆಥ್ ಸಂಪ್ರದಾಯಗಳು**: ಪ್ರಾಮಾಣೀಕರಣ ಮತ್ತು ಪ್ರಾಧಿಕಾರೀಕರಣದ ಸ್ಪಷ್ಟ ವಿವರಣೆ ಮತ್ತು ಪ್ರಮಾಣೀಕ ವಹಿವಾಟು.
  - **ಮೂಲ ಆಧಾರಿತ ಆಥ್ ಅನ್ವಯಿಕೆ**: ಪೈಥಾನ್ (ಸ್ಟಾರ್ಲೆಟ್) ಮತ್ತು ಟೈಪ್ಸ್ಕ್ರಿಪ್ಟ್ (ಎಕ್ಸ್‌ಪ್ರೆಸ್)ನಲ್ಲಿ ಮಿಡಲ್ವೇರ್-ಆಧಾರಿತ ಪ್ರಾಮಾಣೀಕರಣ ಮಾದರಿಗಳು, ಕೋಡ್ ಉದಾಹರಣೆಗಳೊಂದಿಗೆ.
  - **ಸುಧಾರಿತ ಭದ್ರತೆದ ಕಡೆ ಪ್ರಗತಿ**: ಸರಳ ಆಥ್‌ನಿಂದ ಪ್ರಾರಂಭಿಸಿ OAuth 2.1 ಮತ್ತು RBACಕ್ಕೆ ಸಾಗುವ ಮಾರ್ಗದರ್ಶನ, ಸುಧಾರಿತ ಭದ್ರತೆ ಮಾಡ್ಯೂಲ್‌ಗಳಿಗೆ ಸುದೀರ್ಘ ಸೂಚನೆಗಳೊಂದಿಗೆ.

ಈ ಸೇರಿಕೆಗಳು ಗಟ್ಟಿಯಾದ, ಸುರಕ್ಷಿತ ಮತ್ತು ಬದಲಾಯಿಸಬಹುದಾದ MCP ಸರ್ವರ್ ಅನ್ವಯಿಕೆಗಳನ್ನು ನಿರ್ಮಿಸಲು ಪ್ರಾಯೋಗಿಕ, ಕೈಗೆಲಸ ಮಾರ್ಗದರ್ಶನವನ್ನು ಒದಗಿಸುತ್ತವೆ, ಮೂಲ ಸಂಪ್ರದಾಯಗಳನ್ನು ಸುಧಾರಿತ ಉತ್ಪಾದನಾ ಮಾದರಿಗಳೊಂದಿಗೆ ಸೇರುತ್ತವೆ.

    - **ವ್ಯಾಪಾರ ಪ್ರಭಾವ**: ಅಭಿವೃದ್ಧಿಪಡಿಸುವವರಿಗೆ ವೇಗದ ಅಡ್ಡ ಸೇವೆ ಮತ್ತು ಉತ್ಪಾದಕತೆ ಸುಧಾರಣೆ
    - **ವ್ಯುಹಾತ್ಮಕ ಮೌಲ್ಯ**: ಮಾದರಿಯ ಘಟಕ ನಿಯೋಜನೆ ಮತ್ತು ಸಾಧನಗಳ ನಡುವಣ ಸಹಕಾರದ ಮೇಲೆ ಒತ್ತು
    - **ಪರಿಸರ ವ್ಯವಸ್ಥೆ ಅಭಿವೃದ್ಧಿ**: ಏಜೆಂಟ್ ಸಂಯೋಜನೆಗೆ ಮೂಲ ವೇದಿಕೆ
  - **ಸುಧಾರಿತ ಪ್ರಕರಣ ಅಧ್ಯಯನ ರಚನೆ**: ಎಲ್ಲ ಏಳು ಪ್ರಕರಣ ಅಧ್ಯಯನಗಳನ್ನು ಅನುಕೂಲವಾದ ವಿನ್ಯಾಸ ಮತ್ತು ಸಮಗ್ರ ವಿವರಣೆಗಳಿಂದ ನವೀಕರಿಸಲಾಗಿದೆ
    - ಅಜ್ಯೂರ್ AI ಪ್ರಯಾಣ ಏಜೆಂಟ್‌ಗಳು: ಬಹು-ಏಜೆಂಟ್ ஒர்க್‌ಷ್ಟ್ರೇಶನ್ ಒತ್ತು
    - ಅಜ್ಯೂರ್ ಡೆವ್ಓಪ್ಸ್ ಸಂಯೋಜನೆ: ಕಾರ್ಯಪ್ರवाह ಸ್ವಯಂಚಾಲನೆ ಆದ್ಯತೆ
    - ನಿಜ-ಸಮಯ ದಾಖಲಾತಿ ಮರುಪಡೆಯುವಿಕೆ: ಪೈಥಾನ್ ಕಾನ್ಸೋಲ್ ಕ್ಲೈಯಂಟ್ ಅನುಷ್ಠಾನ
    - ಸಂವಾದಾತ್ಮಕ ಕಲಿಕೆ ಯೋಜಕ: ಚೈನ್ಲಿಟ್ ಸಂವಾದ ಜಾಲ ಅಪ್ಲಿಕೇಶನ್
    - ಸಂಪಾದಕ ಒಳಗಿನ ದಾಖಲೆ : VS ಕೋಡ್ ಮತ್ತು GitHub ಕೋಪೈಲಟ್ ಸಂಯೋಜನೆ
    - ಅಜ್ಯೂರ್ API ನಿರ್ವಹಣೆ: ಎಂಟರ್‌ಪ್ರೈಸ್ API ಸಂಯೋಜನೆ ಮಾದರಿಗಳು
    - GitHub MCP ರೆಜಿಸ್ಟ್ರಿ: ಪರಿಸರ ವ್ಯವಸ್ಥೆ ಅಭಿವೃದ್ಧಿ ಮತ್ತು ಸಮುದಾಯ ವೇದಿಕೆ
  - **ಸಮಗ್ರ ನಿರ್ಣಯ**: ಏಳು ಪ್ರಕರಣ ಅಧ್ಯಯನಗಳನ್ನು ವಿಸ್ತೃತವಾಗಿ ಒಳಗೊಂಡು ಪುನರ್ ಬರೆಯಲ್ಪಟ್ಟ ನಿರ್ಣಯ ವಿಭಾಗ
    - ಎಂಟರ್‌ಪ್ರೈಸ್ ಸಂಯೋಜನೆ, ಬಹು-ಏಜೆಂಟ್ ஒர್ಕ್‌ಷ್ಟ್ರೇಶನ್, ಅಭಿವೃದ್ಧಿಪಡಿಸುವವರ ಉತ್ಪಾದಕತೆ
    - ಪರಿಸರ ವ್ಯವಸ್ಥೆ ಅಭಿವೃದ್ಧಿ, ಶಿಕ್ಷಣಾತ್ಮಕ ಅನ್ವಯಿಕೆ ವರ್ಗೀಕರಣ
    - ವಾಸ್ತುಶಿಲ್ಪ ಮಾದರಿಗಳು, ಅನುಷ್ಠಾನ ತಂತ್ರಗಳು ಮತ್ತು ಉತ್ತಮ ಅಭ್ಯಾಸಗಳ ಮೇಲೆ ವಿಶಿಷ್ಟ洞察ಗಳು
    - MCP ಅನ್ನು ಪರಿಪಕ್ವ, ಉತ್ಪಾದನಾ-ತಯಾರಿನ ಪ್ರೊಟೋಕಾಲ್ ಆಗಿ ಒತ್ತಡ ನೀಡಿ

#### ಅಧ್ಯಯನ ಮಾರ್ಗದರ್ಶಿ ನವೀಕರಣಗಳು (study_guide.md)

- **ಕೇಸ್ ಸ್ಟಡಿಸ್ ವಿವರಣೆ**: ಸಾಮಾನ್ಯ ವಿವರಣೆಯಿಂದ ಏಳು ಸಮಗ್ರ ಪ್ರಕರಣ ಅಧ್ಯಯನಗಳ ವಿವರವಾದ ವಿಭಾಗಗಳಿಗೆ ಸುಧಾರಣೆ
- **ಗ್ರಂಥಾಲಯ ರಚನೆ**: ವಿಭಾಗ 10 ಆಗಿ ನವೀಕರಿಸಿ, ನಿರ್ದಿಷ್ಟ ಅನುಷ್ಠಾನ ವಿವರಗಳೊಂದಿಗೆ ಸಮಗ್ರ ಪ್ರಕರಣ ಅಧ್ಯಯನ ಒಳಗೊಂಡಿದೆ
- **ಚೇಂಜ್‌ಲಾಗ್ ಸಂಯೋಜನೆ**: 2025 ಸೆಪ್ಟೆಂಬರ್ 26 ದಿನಾಂಕದ ದಾಖಲೆಯನ್ನು ಸೇರಿಸಿ GitHub MCP ರೆಜಿಸ್ಟ್ರಿ ಸೇರ್ಪಡೆ ಮತ್ತು ಪ್ರಕರಣ ಅಧ್ಯಯನ ಸುಧಾರಣೆಗಳನ್ನು ದಾಖಲಿಸಲಾಗಿದೆ
- **ದಿನಾಂಕ ನವೀಕರಣಗಳು**: ಕೊನೆಯ ಸಂಪಾದನೆಗೆ ಅನುಗುಣವಾಗಿ ಪಾದಲೆಖನ ಸಮಯStamp ನವೀಕರಿಸಲಾಗಿದೆ (ಸೆಪ್ಟೆಂಬರ್ 26, 2025)

### ದಾಖಲೆ ಗುಣಮಟ್ಟ ಸುಧಾರಣೆಗಳು
- **ಸನಿಹತಾ ಸುಧಾರಣೆ**: ಎಲ್ಲಾ ಏಳು ಉದಾಹರಣೆಗಳಲ್ಲಿ ಪ್ರಕರಣ ಅಧ್ಯಯನ ಮಾದರಿ ಮತ್ತು ರಚನೆ ಮಾನ್ಯತೆಯಾಗಿದ್ದು ಸರಳೀಕರಿಸಲಾಗಿದೆ
- **ಸಮಗ್ರ ವ್ಯಾಪ್ತಿ**: ಪ್ರಕರಣ ಅಧ್ಯಯನಗಳು ಈಗ ಎಂಟರ್‌ಪ್ರೈಸ್, ಅಭಿವೃದ್ಧಿಪಡಿಸುವವರ ಉತ್ಪಾದಕತೆ ಮತ್ತು ಪರಿಸರ ವ್ಯವಸ್ಥೆ ಅಭಿವೃದ್ಧಿ ಘಟನೆಗಳನ್ನು ಒಳಗೊಂಡಿವೆ
- **ವ್ಯೂಹಾತ್ಮಕ ಸ್ಥಾನಮಾನ**: MCP ಅನ್ನು ಏಜೆಂಟಿಕ್ ಸಿಸ್ಟಮ್ ನಿಯೋಜನೆಯ ಮೂಲ ವೇದಿಕೆ ಎಂದು ಹೆಚ್ಚುವರಿ ಒತ್ತಡವಿದೆ
- **ಸಂಪನ್ಮೂಲ ಸಂಯೋಜನೆ**: GitHub MCP ರೆಜಿಸ್ಟ್ರಿ ಲಿಂಕ್ ಸೇರಿ ಇನ್ನತಃ ಸಂಪನ್ಮೂಲಗಳನ್ನು ನವೀಕರಿಸಲಾಗಿದೆ

#### MCP ಕಸ್ಟಮ್ ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್‌ಗಳು (05-AdvancedTopics/mcp-transport/) - ಹೊಸ ಸುಧಾರಿತ ಅನುಷ್ಠಾನ માર્ગದರ್ಶಿ

  - **ಅಜ್ಯೂರ್ ಘಟನೆ ಗ್ರಿಡ್ ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್**: ಸಮಗ್ರ ಸರ್ವರ್‌ಲೇಸ್ ಘಟನೆ ಚಲಿತ ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಅನುಷ್ಠಾನ
    - C#, ಟೈಪ್ಸ್ಕ್ರಿಪ್ಟ್ ಮತ್ತು ಪೈಥಾನ್ ಉದಾಹರಣೆಗಳು ಅಜ್ಯೂರ್ ಫಂಕ್ಷನ್‌ಗಳ ಸಂಯೋಜನೆಯೊಂದಿಗೆ
    - ತೂಕಸಾರುವ MCP ಪರಿಹಾರಗಳಿಗೆ ಘಟನೆ ಚಾಲಿತ ವಾಸ್ತುಶಿಲ್ಪ ಮಾದರಿಗಳು
    - ವೆಬ್‌ಹುಕ್ ರಿಸೀವರ್ ಮತ್ತು ಪುಷ್ ಆಧಾರಿತ ಸಂದೇಶ ನಿರ್ವಹಣೆ
  - **ಅಜ್ಯೂರ್ ಘಟನೆ ಆರಂಭವೃಂದ ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್**: ಹೆಚ್ಚಿನ-ಪ್ರವಾಹ ಹರಿವು ಹೊಂದಿರುವ ಸ್ಟ್ರೀಮಿಂಗ್ ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಅನುಷ್ಠಾನ
    - ಕಡಿಮೆ ವಿಳಂಬದ ಸಂದರ್ಭಗಳಿಗೆ ನಿಜ-ಸಮಯ ಸ್ಟ್ರೀಮಿಂಗ್ ಸಾಮರ್ಥ್ಯಗಳು
    - ವಿಭಾಗರೇಖಣ ತಂತ್ರಗಳನ್ನು ಮತ್ತು ತಪಾಸಣಾ ನಿರ್ವಹಣೆ
    - ಸಂದೇಶ ಬ್ಯಾಚಿಂಗ್ ಮತ್ತು ಕಾರ್ಯಕ್ಷಮತೆ ಸುಧಾರಣೆ
  - **ಎಂಟರ್‌ಪ್ರೈಸ್ ಸಂಯೋಜನೆ ಮಾದರಿಗಳು**: ಉತ್ಪಾದನಾ ತಯಾರಿನ ವಾಸ್ತುಶಿಲ್ಪ ಉದಾಹರಣೆಗಳು
    - ಬಹು ಅಜ್ಯೂರ್ ಫಂಕ್ಷನ್ಸ್ ನಡುವಣ ವಿತರಿತ MCP ಪ್ರೊಸೆಸ್
    - ಹೆಚ್ಚು ಟ್ರಾನ್ಸ್‌ಪೋರ್‌ಟ್ಸ್ ಸಂಯೋಜಿಸುವ ಸಂಯುಕ್ತ ವಾಸ್ತುಶಿಲ್ಪ
    - ಸಂದೇಶ ಸ್ಥಾಯಿತ್ವ, ವಿಶ್ವಾಸಾರ್ಹತೆ ಮತ್ತು ದೋಷ ನಿರ್ವಹಣೆ ತಂತ್ರಗಳು
  - **ಭದ್ರತೆ ಮತ್ತು ಮೇಲ್ವಿಚಾರಣೆ**: ಅಜ್ಯೂರ್ ಕೀ ವಾಲ್ಟ್ ಸಂಯೋಜನೆ ಮತ್ತು ನೋಟ ಮಾದರಿಗಳು
    - ನಿರ್ವಹಿತ ಗುರುತು ಪ್ರಾಮಾಣೀಕರಣ ಮತ್ತು ಕನಿಷ್ಟ привಿಲೇಜ್ ಪ್ರವೇಶ
    - ಅಪ್ಲಿಕೇಶನ್ ಇನ್‌ಸೈಟ್ಸ್ ದೂರವಾಣಿ ಮತ್ತು ಕಾರ್ಯಕ್ಷಮತೆ ಮೇಲ್ವಿಚಾರಣೆ
    - ಸರ್ಕ್ಯೂಟ್ ಬ್ರೇಕರ್ಸ್ ಮತ್ತು ದೋಷ ತಾಳುವಿಕೆ ಮಾದರಿಗಳು
  - **ಪರೀಕ್ಷಾ ಫ್ರೇಮ್ವರ್ಕ್‌ಗಳು**: ಕಸ್ಟಮ್ ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್‌ಗಳ ಸಮಗ್ರ ಪರೀಕ್ಷಾ ತಂತ್ರಗಳು
    - ಟೆಸ್ಟ್ ಡಬಲ್ಸ್ ಮತ್ತು ಮಾಕಿಂಗ್ ಫ್ರೇಮ್ವರ್ಕ್‌ಗಳೊಂದಿಗೆ ಯೂನಿಟ್ ಪರೀಕ್ಷೆ
    - ಅಜ್ಯೂರ್ ಟೆಸ್ಟ್ ಕಂಟೈನರ್‌ಗಳೊಂದಿಗೆ ಸಂಯೋಜನೆ ಪರೀಕ್ಷೆ
    - ಕಾರ್ಯಕ್ಷಮತೆ ಮತ್ತು ಭಾರ ಪರೀಕ್ಷೆ ಪರಿಗಣನೆಗಳು

    - ಕ್ರಮಬದ್ಧ ಸಂಧರ್ಭ ಲೋಡಿಂಗ್ ಮತ್ತು ಸಂಕುಚಿತ ತಂತ್ರಗಳು
    - ಪದರಬದ್ಧ ಸಂಧರ್ಭ ವಿಧಾನಗಳು ಮತ್ತು ಮರುಪಡೆಯುವಿಕೆ ಗುಣಮಟ್ಟ
  - **ಮಾಪನ ಚಟುವಟಿಕೆ**: ಸಂಧರ್ಭ ಪರಿಣಾಮಕಾರಿತ್ವ ಮೌಲ್ಯಮಾಪನಕ್ಕಾಗಿ ಉದಯವಾಗುತ್ತಿರುವ ಮಾನಕಗಳು

- **ಸಂಕಲನ ಸುಧಾರಣೆ**: ಹೋಸ್ಟ್‌ಗಳು, ಕ್ಲೈಂಟ್‌ಗಳು ಮತ್ತು ಸರ್ವರ್‌ಗಳ ವಿವರಣೆಗಳನ್ನು MCP ಆರ್ಕಿಟೆಕ್ಚರ್ ಮಾದರಿಗಳ ಪ್ರಕಾರ ಹೆಚ್ಚು ವಿವರವಾಗಿ ಮಾಡಲಾಗಿದೆ  
  - ಹೋಸ್ಟ್‌ಗಳನ್ನು ಈಗ ಬಹು MCP ಕ್ಲೈಂಟ್ ಸಂಪರ್ಕಗಳನ್ನು ಸಂಯೋಜಿಸುವ AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳಾಗಿ ಸ್ಪಷ್ಟವಾಗಿ ವ್ಯಾಖ್ಯಾನಿಸಲಾಗಿದೆ  
  - ಕ್ಲೈಂಟ್‌ಗಳನ್ನು ಪ್ರೊಟೋಕಾಲ್ ಕನೆಕ್ಟರ್‌ಗಳಾಗಿ ವರ್ಣಿಸಲಾಗಿದೆ, ಒಂದು ಸರ್ವರ್ೊಂದಿಗೆ ಒಂದು-ಒಂದು ಸಂಬಂಧವನ್ನು ಉಳಿಸಿಕೊಳ್ಳುತ್ತದೆ  
  - ಸರ್ವರ್‌ಗಳನ್ನು ಸ್ಥಳೀಯ ಮತ್ತು ದೂರದ ಸ್ಥಳದಲ್ಲಿ ನಿಯೋಜಿಸುವ ದೃಶ್ಯದೊಂದಿಗೆ ಹೆಚ್ಚಿಸಲಾಗಿದೆ  
- **ಪ್ರಿಮಿಥಿವ್ ಪುನರ್‌ರೂಪರಚನೆ**: ಸರ್ವರ್ ಮತ್ತು ಕ್ಲೈಂಟ್ ಪ್ರಿಮಿಥಿವ್‌ಗಳ ಸಂಪೂರ್ಣ ಪುನರ್‌ರೂಪರಚನೆ  
  - ಸರ್ವರ್ ಪ್ರಿಮಿಥಿವ್‌ಗಳು: ಸಂಪನ್ಮೂಲಗಳು (ಡೇಟಾ ಮೂಲಗಳು), ಪ್ರಾಂಪ್ಟ್‌ಗಳು (ಟೆಂಪ್ಲೇಟ್ಗಳು), ಉಪಕರಣಗಳು (ಕಾರ್ಯನಿರ್ವಹಣಾ ಕಾರ್ಯಗಳು) ವಿವರ ಮತ್ತು ಉದಾಹರಣೆಗಳೊಂದಿಗೆ  
  - ಕ್ಲೈಂಟ್ ಪ್ರಿಮಿಥಿವ್‌ಗಳು: ಸ್ಯಾಂಪ್ಲಿಂಗ್ (LLM ಪೂರ್ಣಗೊಳಣೆಗಳು), ಎಲಿಸಿಟೇಷನ್ (ಬಳಕೆದಾರ ಇನ್‌ಪುಟ್), ಲಾಗಿಂಗ್ (ದೋಷಸೃಷ್ಟಿ/ಮಾನಿಟರಿಂಗ್)  
  - ಇತ್ತೀಚಿನ ಪತ್ತೆಹಚ್ಚುವಿಕೆ (`*/list`), ಪಡೆದಲೆ (`*/get`), ಮತ್ತು ಕಾರ್ಯನಿರ್ವಹಣೆ (`*/call`) ವಿಧಾನ ಮಾದರಿಗಳೊಂದಿಗೆ ನವೀಕರಿಸಲಾಗಿದೆ  
- **ಪ್ರೊಟೋಕಾಲ್ ಆರ್ಕಿಟೆಕ್ಚರ್**: ಎರಡು-ಮಟ್ಟದ ಆರ್ಕಿಟೆಕ್ಚರ್ ಮಾದರಿಯನ್ನು ಪರಿಚಯಿಸಲಾಗಿದೆ  
  - ಡೇಟಾ ಲೇಯರ್: JSON-RPC 2.0 ನೆಲೆ, ಜೀವನಚಕ್ರ ನಿರ್ವಹಣೆ ಮತ್ತು ಪ್ರಿಮಿಥಿವ್‌ಗಳೊಂದಿಗೆ  
  - ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಲೇಯರ್: STDIO (ಸ್ಥಳೀಯ) ಮತ್ತು ಸ್ಟ್ರೀಮಬಲ್ HTTP ಮತ್ತು SSE (ದೂರದ ಪ್ರವೇಶ) ಸಾರಿಗೆ ಕ್ರಮಗಳು  
- **ಭದ್ರತಾ ಚಟುವಟಿಕೆ**: ಸ್ಪಷ್ಟ ಬಳಕೆದಾರ ಅಮತ, ಡೇಟಾ ಗೌಪ್ಯತೆ, ಉಪಕರಣ ಕಾರ್ಯನಿರ್ವಹಣೆ ಸುರಕ್ಷತೆ ಮತ್ತು ಸಾರಿಗೆ ಲೇಯರ್ ಸುರಕ್ಷತೆಯನ್ನು ಒಳಗೊಂಡ ಸಮಗ್ರ ಭದ್ರತಾ ಸಿದ್ಧಾಂತಗಳು  
- **ಸಂವಹನ ಮಾದರಿಗಳು**: ಪ್ರೊಟೋಕಾಲ್ ಸಂದೇಶಗಳನ್ನು ಪ್ರಾರಂಭ, ಪತ್ತೆಹಚ್ಚುವಿಕೆ, ಕಾರ್ಯನಿರ್ವಹಣೆ ಮತ್ತು ಸೂಚನೆ ಹರಿವನ್ನು ತೋರಿಸಲು ನವೀಕರಿಸಲಾಗಿದೆ  
- **ಕೋಡ್ ಉದಾಹರಣೆಗಳು**: ಇತ್ತೀಚಿನ MCP SDK ಮಾದರಿಗಳೊಂದಿಗೆ ಬಹುವಿಧ ಭಾಷೆಯ ಉದಾಹರಣೆಗಳು (.NET, ಜಾವಾ, ಪೈಥಾನ್, ಜಾವಾಸ್ಕ್ರಿಪ್ಟ್) ನವೀಕರಿಸಲಾಗಿದೆ  

#### ಭದ್ರತೆ (02-Security/) - ಸಮಗ್ರ ಭದ್ರತೆ ಪೂರಕ ಸುಧಾರಣೆ  
- **ಮಾನದಂಡಗಳ ಹೊಂದಾಣಿಕೆ**: MCP ವಿಶಿಷ್ಟ 2025-06-18 ಭದ್ರತಾ ಅಗತ್ಯತೆಗಳಿಗೆ ಪೂರ್ಣ ಹೊಂದಾಣಿಕೆ  
- **ಮಾನ್ಯತೆ ಪ್ರಗತಿ**: ಕಸ್ಟಮ್ OAuth ಸರ್ವರ್‌ಗಳಿಂದ ಹೊರಗಿನ ಐಡೆಂಟಿಟಿ ಪ್ರೊವೈಡರ್ ನಿಯೋಜನೆಗೆ (ಮೈಕ್ರೋಸಾಫ್ಟ್ ಎಂಟ್ರಾ ID) ವೃತ್ತಾಂತ ದಾಖಲಿಸಲಾಗಿದೆ  
- **AI-ನಿರ್ದಿಷ್ಟ ಬೆದರಿಕೆ ವಿಶ್ಲೇಷಣೆ**: ಆಧುನಿಕ AI ದಾಳಿ ಮಾರ್ಗಗಳ ವಿಶೇಷ ವಿವರಣೆ  
  - ವಾಸ್ತವಿಕ ಉದಾಹರಣೆಗಳೊಂದಿಗೆ ವಿವರವಾದ ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್ ದಾಳಿ ದೃಶ್ಯಗಳು  
  - ಉಪಕರಣ ವಿಷಕಾರಕತೆಯ ಯಂತ್ರಗಳು ಮತ್ತು "ರಗ್ ಪುಲ್" ದಾಳಿ ಮಾದರಿಗಳು  
  - ಸಂಧರ್ಭ ವಿಂಡೋ ವಿಷಕಾರಕತೆ ಮತ್ತು ಮಾದರಿ ಗೊಂದಲ ದಾಳಿ  
- **ಮೈಕ್ರೋಸಾಫ್ಟ್ AI ಭದ್ರತಾ ಪರಿಹಾರಗಳು**: ಮೈಕ್ರೋಸಾಫ್ಟ್ ಭದ್ರತಾ ಪರಿಸರದ ಸಂಪೂರ್ಣ ಸಮೀಕ್ಷೆ  
  - AI ಪ್ರಾಂಪ್ಟ್ ಶೀಲ್ಡ್‌ಗಳು ಅತ್ಯಾಧುನಿಕ ಪತ್ತೆಹಚ್ಚುವಿಕೆ, ಸ್ಪಾಟ್‌ಲೈಟಿಂಗ್ ಮತ್ತು ಡೆಲಿಮಿಟರ್ ತಂತ್ರಗಳು  
  - ಅಜೂರ್ ವಿಷಯ ಸುರಕ್ಷತೆ ಸಂಯೋಜನೆ ಮಾದರಿಗಳು  
  - ಸರಬರಾಜು ಸರಪಳಿ ರಕ್ಷಣೆಗಾಗಿ GitHub ಅಡ್ವಾನ್ಸ್‌ಡ್ ಸیکیುರಿಟಿ  
- **ಅತ್ಯಾಧುನಿಕ ಬೆದರಿಕೆ ತಗ್ಗಿಸುವಿಕೆ**: ವಿವರವಾದ ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳು  
  - MCP-ನಿರ್ದಿಷ್ಟ ದಾಳಿ ದೃಶ್ಯಗಳೊಂದಿಗೆ ಸೆಷನ್ ಹೈಜಾಕಿಂಗ್ ಮತ್ತು ಕ್ರಿಪ್ಟೋಗ್ರಾಫಿಕ್ ಸೆಷನ್ ID ಬೇಕಾಗಿರುವುದು  
  - MCP ಪ್ರೋಕ್ಸಿ ದೃಶ್ಯಗಳಲ್ಲಿ ಗೊಂದಲದಿಂದ ಉಂಟಾಗುವ ಪ್ರತಿನಿಧಿ ಸಮಸ್ಯೆಗಳು ಸ್ಪಷ್ಟ ಮಂಜೂರಾತಿ ಅಗತ್ಯಗಳೊಂದಿಗೆ  
  - ಟೋಕನ್ ಪಾಸ್‌ಥ್ರೂ ದುರ್ಬಲತೆಗಳು ಕಡ್ಡಾಯ ಪರಿಶೀಲನೆ ನಿಯಂತ್ರಣಗಳೊಂದಿಗೆ  
- **ಸರಬರಾಜು ಸರಪಳಿ ಭದ್ರತೆ**: ಸ್ಥಾಪಿತ ಮಾದರಿಗಳು, ಎಂಬೆಡೆಯಿಂಗ್ ಸೇವೆಗಳು, ಸಂಧರ್ಭ ಪ್ರೊವೈಡರ್‌ಗಳು ಮತ್ತು ತೃತೀಯ-ಪಕ್ಷ APIs-ಗಳನ್ನು ಒಳಗೊಂಡ AI ಸರಬರಾಜು ಸರಪಳಿಯ ವ್ಯಾಪಕ ಒಳಗೊಂಡಿಕೆ  
- **ಸ್ಥಾಪನಾ ಭದ್ರತೆ**: ಶೂನ್ಯ ವಿಶ್ವಾಸ ಆರ್ಕಿಟೆಕ್ಚರ್ ಮತ್ತು ಮೈಕ್ರೋಸಾಫ್ಟ್ ಭದ್ರತಾ ಪರಿಸರದೊಂದಿಗೆ ಬಲಿಷ್ಠ ಸಂಯೋಜನೆ  
- **ಸ Ressourcen ವ್ಯವಸ್ಥೆ**: ಪ್ರಕಾರವಾಗಿ ವರ್ಗೀಕರಿಸಿದ ಸಂಪನ್ಮೂಲ ಸಂಪರ್ಕಗಳು (ಅಧಿಕೃತ ಡಾಕ್ಸ್, ಮಾನದಂಡಗಳು, ಸಂಶೋಧನೆ, ಮೈಕ್ರೋಸಾಫ್ಟ್ ಪರಿಹಾರಗಳು, ಅನುಷ್ಠಾನ ಮಾರ್ಗದರ್ಶಿಗಳು)  

### ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಗುಣಮಟ್ಟ ಸುಧಾರಣೆಗಳು  
- **ಸಂರಚಿತ ಅಧ್ಯಯನ ಗುರಿಗಳು**: ನಿರ್ದಿಷ್ಟ ಕಾರ್ಯಗತ પરિણામಗಳೊಂದಿಗೆ ಸುಧಾರಿತ ಅಧ್ಯಯನ ಗುರಿಗಳು  
- **ಅಂತರಸಂಬಂಧ ಉಲ್ಲೇಖಗಳು**: ಸಂಬಂಧಿತ ಭದ್ರತಾ ಮತ್ತು ಕೋರ್ ಸಂಯೋಜನೆ ವಿಷಯಗಳ ನಡುವಣ ಲಿಂಕ್‌ಗಳನ್ನು ಸೇರಿಸಲಾಗಿದೆ  
- **ಪ್ರಸ್ತುತ ಮಾಹಿತಿ**: ಎಲ್ಲಾ ದಿನಾಂಕ ಉಲ್ಲೇಖಗಳು ಮತ್ತು ವಿಶಿಷ್ಟ ಸಂಪರ್ಕಗಳನ್ನು ನವೀಕರಿಸಲಾಗಿದೆ  
- **ಅನುಷ್ಠಾನ ಮಾರ್ಗದರ್ಶನ**: ಎರಡು ವಿಭಾಗಗಳಲ್ಲಿಯೂ ನಿರ್ದಿಷ್ಟ ಹಾಗೂ ಕಾರ್ಯಗತ ಅನುಷ್ಠಾನ ಮಾರ್ಗದರ್ಶನವನ್ನು ಸೇರಿಸಲಾಗಿದೆ
