# Генератор на учебен план с Chainlit & Microsoft Learn Docs MCP

## Предварителни изисквания

- Python 3.8 или по-нова версия
- pip (мениджър на пакети за Python)
- Интернет достъп за свързване със сървъра Microsoft Learn Docs MCP

## Инсталация

1. Клонирайте това хранилище или изтеглете файловете на проекта.
2. Инсталирайте необходимите зависимости:

   ```bash
   pip install -r requirements.txt
   ```

## Използване

### Сценарий 1: Проста заявка към Docs MCP
Команден клиент, който се свързва със сървъра Docs MCP, изпраща заявка и отпечатва резултата.

1. Стартирайте скрипта:
   ```bash
   python scenario1.py
   ```
2. Въведете въпроса си за документацията в подканата.

### Сценарий 2: Генератор на учебен план (уеб приложение с Chainlit)
Уеб-базиран интерфейс (използващ Chainlit), който позволява на потребителите да създават персонализиран учебен план седмица по седмица за всяка техническа тема.

1. Стартирайте Chainlit приложението:
   ```bash
   chainlit run scenario2.py
   ```
2. Отворете локалния URL адрес, посочен в терминала ви (например http://localhost:8000) в браузъра си.
3. В чата въведете темата за учене и броя на седмиците, които искате да учите (например „AI-900 certification, 8 weeks“).
4. Приложението ще отговори с учебен план седмица по седмица, включително връзки към съответната Microsoft Learn документация.

**Необходими променливи на средата:**

За да използвате Сценарий 2 (уеб приложението Chainlit с Azure OpenAI), трябва да зададете следните променливи на средата в `.env` файл в директорията `python`:

```
AZURE_OPENAI_CHAT_DEPLOYMENT_NAME=
AZURE_OPENAI_API_KEY=
AZURE_OPENAI_ENDPOINT=
AZURE_OPENAI_API_VERSION=
```

Попълнете тези стойности с детайлите на вашия Azure OpenAI ресурс преди да стартирате приложението.

> [!TIP]
> Лесно можете да разположите свои модели с помощта на [Microsoft Foundry](https://ai.azure.com/).

### Сценарий 3: Документация в редактора с MCP сървър във VS Code

Вместо да превключвате раздели в браузъра за търсене на документация, можете да вградите Microsoft Learn Docs директно във VS Code, използвайки MCP сървъра. Това ви позволява:
- Да търсите и четете документация вътре в VS Code без да напускате средата си за кодиране.
- Да цитирате документация и да вмъквате връзки директно в README или учебни файлове.
- Да използвате заедно GitHub Copilot и MCP за безпроблемен, базиран на ИИ работен процес с документацията.

**Примери за използване:**
- Бързо добавяне на справочни връзки в README докато пишете курс или проектна документация.
- Използване на Copilot за генериране на код и MCP за незабавно намиране и цитиране на релевантна документация.
- Запазване на фокуса в редактора и повишаване на продуктивността.

> [!IMPORTANT]
> Уверете се, че имате валидна конфигурация [`mcp.json`](../../../../../../09-CaseStudy/docs-mcp/solution/scenario3/mcp.json) в работното си пространство (намира се в `.vscode/mcp.json`).

## Защо Chainlit за Сценарий 2?

Chainlit е модерен, с отворен код фреймуърк за изграждане на конверсационни уеб приложения. Той прави лесно създаването на чат-базирани потребителски интерфейси, които се свързват със сървисите на заден план като Microsoft Learn Docs MCP сървъра. Този проект използва Chainlit, за да предостави прост и интерактивен начин за генериране на персонализирани учебни планове в реално време. С помощта на Chainlit можете бързо да изграждате и разгръщате чат-базирани инструменти, които повишават продуктивността и ученето.

## Какво прави това

Това приложение позволява на потребителите да създадат персонализиран учебен план, като просто въведат тема и продължителност. Приложението обработва вашия вход, прави заявка към сървъра Microsoft Learn Docs MCP за релевантно съдържание и организира резултатите в структуриран учебен план седмица по седмица. Препоръките за всяка седмица се показват в чата, което улеснява следенето и напредъка ви. Интеграцията гарантира, че винаги получавате най-новите и най-подходящи ресурси за обучение.
