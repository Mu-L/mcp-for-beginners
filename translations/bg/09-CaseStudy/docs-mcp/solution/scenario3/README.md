# Сценарий 3: Документация в редактора с MCP сървър във VS Code

## Преглед

В този сценарий ще научите как да интегрирате Microsoft Learn Docs директно във вашата Visual Studio Code среда, използвайки MCP сървър. Вместо постоянно да превключвате между браузърни табове, за да търсите документация, можете да имате достъп, да търсите и да позовавате официалната документация директно във вашия редактор. Този подход оптимизира вашия работен процес, запазва фокуса ви и позволява безпроблемна интеграция с инструменти като GitHub Copilot.

- Търсете и четете документация вътре в VS Code, без да напускате средата за писане на код.
- Позовавайте се на документация и вмъквайте връзки директно във вашите README или учебни файлове.
- Използвайте GitHub Copilot и MCP заедно за безпроблемен работен процес с документи, подсилен от изкуствен интелект.

## Учебни цели

Към края на тази глава ще разберете как да настроите и използвате MCP сървъра във VS Code, за да подобрите вашия работен процес с документация и разработка. Ще можете да:

- Конфигурирате вашето работно пространство да използва MCP сървъра за търсене на документация.
- Търсите и вмъквате документация директно от VS Code.
- Комбинирате възможностите на GitHub Copilot и MCP за по-продуктивен, подсилен с ИИ работен процес.

Тези умения ще ви помогнат да останете фокусирани, да подобрите качеството на документацията и да повишите продуктивността си като разработчик или технически писател.

## Решение

За да постигнете достъп до документация директно в редактора, ще следвате серия от стъпки, които интегрират MCP сървъра с VS Code и GitHub Copilot. Това решение е идеално за автори на курсове, писатели на документация и разработчици, които искат да останат съсредоточени в редактора, докато работят с документи и Copilot.

- Бързо добавяйте референтни връзки към README докато пишете курс или проектна документация.
- Използвайте Copilot за генериране на код и MCP за незабавно намиране и цитиране на релевантна документация.
- Останете фокусирани в редактора и увеличете продуктивността си.

### Стъпка по стъпка

За да започнете, следвайте тези стъпки. За всяка стъпка можете да добавите скрийншот от папката assets, за да илюстрирате процеса визуално.

1. **Добавете конфигурация за MCP:**
   В корена на вашия проект създайте файл `.vscode/mcp.json` и добавете следната конфигурация:
   ```json
   {
     "servers": {
       "LearnDocsMCP": {
         "url": "https://learn.microsoft.com/api/mcp"
       }
     }
   }
   ```
   Тази конфигурация указва на VS Code как да се свърже със [`Microsoft Learn Docs MCP сървъра`](https://github.com/MicrosoftDocs/mcp).
   
   ![Стъпка 1: Добавете mcp.json в папката .vscode](../../../../../../translated_images/bg/step1-mcp-json.c06a007fccc3edfa.webp)
    
2. **Отворете панела GitHub Copilot Chat:**
   Ако все още нямате инсталирано разширението GitHub Copilot, отидете в изгледа Extensions във VS Code и го инсталирайте. Можете да го изтеглите директно от [Visual Studio Code Marketplace](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot-chat). След това отворете панела Copilot Chat от страничната лента.

   ![Стъпка 2: Отворете панела Copilot Chat](../../../../../../translated_images/bg/step2-copilot-panel.f1cc86e9b9b8cd1a.webp)

3. **Активирайте агентски режим и проверете инструментите:**
   В панела Copilot Chat активирайте агентски режим.

   ![Стъпка 3: Активирайте агентски режим и проверете инструментите](../../../../../../translated_images/bg/step3-agent-mode.cdc32520fd7dd1d1.webp)

   След включване на агентския режим, проверете дали MCP сървърът е изброен като един от наличните инструменти. Това гарантира, че агентът Copilot може да достъпва сървъра на документацията, за да получава релевантна информация.
   
   ![Стъпка 3: Проверете инструмента MCP сървър](../../../../../../translated_images/bg/step3-verify-mcp-tool.76096a6329cbfecd.webp)
4. **Започнете нов чат и подайте заявка на агента:**
   Отворете нов чат в панела Copilot Chat. Сега можете да зададете въпроси на агента свързани с документацията. Агентът ще използва MCP сървъра, за да получи и покаже релевантна документация от Microsoft Learn директно в редактора ви.

   - *"Опитвам се да напиша учебен план за тема X. Ще я изучавам 8 седмици, за всяка седмица предложи съдържание, което трябва да покрия."*

   ![Стъпка 4: Подайте заявка на агента в чата](../../../../../../translated_images/bg/step4-prompt-chat.12187bb001605efc.webp)

5. **Живо запитване:**

   > Нека вземем живо запитване от секцията [#get-help](https://discord.gg/D6cRhjHWSC) в Microsoft Foundry Discord ([виж оригиналното съобщение](https://discord.com/channels/1113626258182504448/1385498306720829572)):
   
   *"Търся отговори как да разположа мултиагентно решение с AI агенти разработени на Azure AI Foundry. Виждам, че няма директен метод за разполагане, като Copilot Studio канали. Така че, какви са различните начини да направим това разполагане, за да могат корпоративни потребители да взаимодействат и да свършат работата?
Има много статии/блогове, които твърдят, че можем да използваме Azure Bot service за тази задача, който може да действа като мост между MS Teams и Azure AI Foundry Agents, ще проработи ли това ако настроя Azure bot, който се свързва с Orchestrator агента в Azure AI Foundry чрез Azure функция за оркестрация, или трябва да създам Azure функция за всеки AI агент в мултиагентното решение, за да направя оркестрация на Bot framework? Други предложения са повече от добре дошли.
"*

   ![Стъпка 5: Живи запитвания](../../../../../../translated_images/bg/step5-live-queries.49db3e4a50bea273.webp)

   Агентът ще отговори с релевантни връзки към документация и обобщения, които можете да вмъкнете директно във вашите markdown файлове или да използвате като препратки в кода си.
   
### Примерни Запитвания

Ето няколко примерни запитвания, които може да пробвате. Тези запитвания ще демонстрират как MCP сървърът и Copilot работят заедно, за да предоставят мигновена, контекстуално осведомена документация и препратки без да напускате VS Code:

- "Намери бърз старт за Azure AI услуги."

Тези запитвания ще демонстрират как MCP сървърът и Copilot работят заедно, за да предоставят мигновена, контекстуално осведомена документация и препратки без да напускате VS Code.
