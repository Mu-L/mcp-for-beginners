## 🔧 Разбиране на MCP сървърите: Ръководство за начинаещи

### Какво са MCP сървъри?

Като начинаещ в Model Context Protocol (MCP) може да се питате: „Какво точно е MCP сървър и защо трябва да ме интересува?“ Нека започнем с проста аналогия.

Представете си MCP сървърите като специализирани асистенти, които помагат на вашия AI спътник при писане на код (като GitHub Copilot) да се свързва с външни инструменти и услуги. Както използвате различни приложения на телефона си за различни задачи – едно за прогноза за времето, друго за навигация, трето за банкови операции – така MCP сървърите дават на AI асистента ви възможността да взаимодейства с различни инструменти и услуги за разработка.

### Проблемът, който MCP сървърите решават

Преди MCP сървърите, ако искахте да:
- Проверите Azure ресурсите си
- Създадете GitHub задача
- Изпълните заявка към базата данни
- Претърсите документация

трябваше да спрете да пишете код, да отворите браузър, да навигирате до съответния уебсайт и ръчно да извършите тези задачи. Това постоянно превключване на контекста разрушава вашия поток и намалява продуктивността.

### Как MCP сървърите трансформират вашето преживяване при разработка

С MCP сървъри можете да останете в средата си за разработка (VS Code, Visual Studio и др.) и просто да помолите AI асистента да се справи с тези задачи. Например:

**Вместо този традиционен работен поток:**
1. Спиране на писане на код
2. Отваряне на браузър
3. Навигация до Azure портала
4. Проверка на детайли за storage акаунт
5. Връщане към VS Code
6. Продължаване на писане на код

**Сега можете да направите това:**
1. Попитайте AI: „Какъв е статусът на моите Azure storage акаунти?“
2. Продължете да пишете код с предоставената информация

## Започване с MCP във VS Code и Visual Studio 🚀

Настройването на тези MCP сървъри е лесно, ако използвате Visual Studio Code или Visual Studio 2022 с GitHub Copilot.

### Настройка във VS Code

Ето основния процес за VS Code:

1. **Активирайте режим Agent**: Във VS Code превключете към Agent mode в прозореца на Copilot Chat
2. **Конфигурирайте MCP сървъри**: Добавете конфигурации на сървъри във файла settings.json на VS Code
3. **Стартирайте сървърите**: Натиснете бутона „Start“ за всеки сървър, който искате да използвате
4. **Изберете инструменти**: Изберете кои MCP сървъри да активирате за текущата сесия

За подробни инструкции вижте [документацията за MCP на VS Code](https://code.visualstudio.com/docs/copilot/copilot-mcp).

> **💡 Професионален съвет: Управлявайте MCP сървърите като експерт!**
> 
> Във VS Code Extensions изгледа вече има [удобен нов интерфейс за управление на инсталираните MCP сървъри](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode)! Имаш бърз достъп до стартиране, спиране и управление на всеки инсталиран MCP сървър с ясен и прост интерфейс. Опитайте го!

### Настройка във Visual Studio 2022

За Visual Studio 2022 (версия 17.14 или по-нова):

1. **Активирайте режим Agent**: Кликнете на падащото меню „Ask“ в прозореца на GitHub Copilot Chat и изберете „Agent“
2. **Създайте конфигурационен файл**: Създайте файл `.mcp.json` в директорията на решението си (препоръчително местоположение: `<SOLUTIONDIR>\.mcp.json`)
3. **Конфигурирайте сървърите**: Добавете конфигурациите за MCP сървърите си, използвайки стандартния MCP формат
4. **Одобрение на инструменти**: При поискване одобрете инструментите, които искате да използвате с необходимите разрешения за достъп

За подробни инструкции за настройка във Visual Studio вижте [документацията за MCP на Visual Studio](https://learn.microsoft.com/visualstudio/ide/mcp-servers).

Всеки MCP сървър има свои изисквания за конфигурация (връзки, удостоверяване и др.), но моделът за настройка е последователен и за двете IDE.

**Какво прави**: MCP сървърът на Microsoft Learn Docs е облачна услуга, която предоставя на AI помощниците достъп в реално време до официалната документация на Microsoft чрез Model Context Protocol. Свързва се към `https://learn.microsoft.com/api/mcp` и позволява семантично търсене в Microsoft Learn, документацията на Azure, документацията на Microsoft 365 и други официални източници на Microsoft.

**Защо е полезен**: Въпреки че може да изглежда като „само документация“, този сървър е от ключово значение за всеки разработчик, който използва технологии на Microsoft. Един от най-честите упреци на .NET разработчиците към AI помощници за кодиране е, че те не са актуални с последните версии на .NET и C#. MCP сървърът на Microsoft Learn Docs решава това като предоставя достъп в реално време до най-актуалната документация, API справочници и добри практики. Независимо дали работите с най-новите Azure SDK-та, разглеждате нови функции в C# 13 или имплементирате модерни .NET Aspire шаблони, този сървър гарантира, че AI асистентът ви има достъп до авторитетна, актуална информация, необходима за генериране на точен, модерен код.

**Реална употреба**: „Кои са командите на az cli за създаване на Azure контейнерно приложение според официалната Microsoft Learn документация?“ или „Как да конфигурирам Entity Framework с dependency injection в ASP.NET Core?“ Или пък: „Прегледай този код, за да се увериш, че отговаря на препоръките за производителност в документацията на Microsoft Learn.“ Сървърът предоставя обхватно покритие на Microsoft Learn, Azure docs и документацията на Microsoft 365, използвайки усъвършенствано семантично търсене за намиране на най-релевантната информация в контекст. Връща до 10 висококачествени части от съдържание с заглавия на статии и URL адреси, винаги осигурявайки достъп до последната официална документация на Microsoft веднага след публикуването ѝ.

**Пример**: Сървърът експонира инструмента `microsoft_docs_search`, който извършва семантично търсене в официалната техническа документация на Microsoft. След конфигуриране, можете да задавате въпроси като „Как да имплементирам JWT удостоверяване в ASP.NET Core?“ и да получавате подробни, официални отговори с линкове към източници. Качеството на търсенето е изключително, тъй като разбира контекста – например дума като „контейнери“ в Azure контекст връща документация за Azure Container Instances, докато същият термин в .NET контекст връща информация за колекции в C#.

Това е особено полезно при бързо променящи се или наскоро обновени библиотеки и случаи на употреба. Например, в няколко скорошни проекта исках да използвам функции от последните версии на .NET Aspire и Microsoft.Extensions.AI. Включвайки MCP сървъра на Microsoft Learn Docs, успях да използвам не само API документация, но и ръководства и насоки, които току-що бяха публикувани.

### 2. ☁️ Azure MCP сървър
[![Инсталирай в VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![Инсталирай в VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

**Какво прави**: Azure MCP сървърът е пълен комплект от над 15 специализирани конектора за услуги на Azure, който интегрира цялата екосистема на Azure във вашия AI работен процес. Това не е просто един сървър – това е мощна колекция, включваща управление на ресурси, свързаност с бази данни (PostgreSQL, SQL Server), анализ на логове с KQL чрез Azure Monitor, интеграция с Cosmos DB и още много.

**Защо е полезен**: Освен че управлява Azure ресурсите, този сървър значително подобрява качеството на кода при работа с Azure SDK. Когато използвате Azure MCP в режим Agent, той не просто ви помага да пишете код – той ви помага да пишете *по-добър* Azure код, който следва актуалните модели за удостоверяване, най-добрите практики за обработка на грешки и използва последните функционалности на SDK. Вместо да получите общ код, който може и да работи, получавате код, който следва препоръчаните от Azure модели за продукционни натоварвания.

**Основни модули включват**:
- **🗄️ Конектори за бази данни**: Директен достъп с естествен език до Azure Database за PostgreSQL и SQL Server
- **📊 Azure Monitor**: Анализ на логове с KQL и оперативни прозрения
- **🌐 Управление на ресурси**: Пълно управление на жизнения цикъл на Azure ресурсите
- **🔐 Удостоверяване**: DefaultAzureCredential и модели с управлявана идентичност
- **📦 Услуги за съхранение**: Операции с Blob Storage, Queue Storage и Table Storage
- **🚀 Услуги за контейнери**: Управление на Azure Container Apps, Container Instances и AKS
- **И много други специализирани конектори**

**Примери от практиката**: "Изброи моите Azure storage акаунти", "Направи заявка в моя Log Analytics workspace за грешки през последния час" или "Помогни ми да създам Azure приложение с Node.js с правилно удостоверяване"

**Пълен демонстрационен сценарий**: Ето пълен пример, който показва силата на комбинирането на Azure MCP с разширението GitHub Copilot for Azure в VS Code. Когато имате инсталирани и двете и въведете:

> "Създай Python скрипт, който качва файл в Azure Blob Storage, използвайки удостоверяване с DefaultAzureCredential. Скриптът трябва да се свърже с моя Azure storage акаунт на име 'mycompanystorage', да качи файла в контейнер с име 'documents', да създаде тестов файл с текущия времеви печат за качване, да обработва грешки внимателно и да предоставя информативен изход, да следва най-добрите практики на Azure за удостоверяване и обработка на грешки, да включва коментари, обясняващи как работи удостоверяването с DefaultAzureCredential, и да направи скрипта добре структуриран с подходящи функции и документация."

Azure MCP сървърът ще генерира пълен, готов за продукция Python скрипт, който:
- Използва най-новия Azure Blob Storage SDK с правилни асинхронни модели
- Прилага DefaultAzureCredential с изчерпателно обяснение на fallback веригата
- Включва надеждна обработка на грешки със специфични типове изключения на Azure
- Следва най-добрите практики на Azure SDK за управление на ресурси и връзки
- Осигурява детайлно логване и информативен изход в конзолата
- Създава правилно структуриран скрипт с функции, документация и указания за типове

Изключителното тук е, че без Azure MCP бихте получили общ код за blob storage, който работи, но не следва актуалните модели на Azure. С Azure MCP получавате код, който използва най-новите методи за удостоверяване, обработва специфични сценарии за грешки в Azure и следва препоръчаните от Microsoft практики за продукционни приложения.

**Показан пример**: Аз изпитвах затруднения да запомня конкретните команди за CLI инструментите `az` и `azd` за спонтанна употреба. Винаги за мен е двукрачен процес: първо търся синтаксиса, после изпълнявам командата. Често влизам в портала и кликвам, за да свърша работа, защото не искам да призная, че не помня CLI синтаксиса. Възможността просто да опиша какво искам е невероятна, а още по-добре е, че мога да го направя без да напускам своя IDE!

В [Azure MCP хранилището](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) има страхотен списък с употреби, с които да започнете. За пълни ръководства за настройка и разширени опции за конфигурация, разгледайте [официалната документация на Azure MCP](https://learn.microsoft.com/azure/developer/azure-mcp-server/).

**Опции за удостоверяване**: Сървърът поддържа както OAuth (безпроблемно във VS Code), така и персонални токени за достъп, с възможност за конфигуриране на набори инструменти, за да се активира само необходимата GitHub функционалност. > **💡 Професионален съвет**

Например добавете `"--toolsets", "repos,issues,pull_requests,actions"` за основни работни потоци за разработка, или използвайте `"--toolsets", "notifications, security"`, ако основно искате възможности за GitHub наблюдение.

За разлика от обикновените инструменти за извличане на текст, той се фокусира върху запазване на семантичното значение и форматирането, което е ценно както за AI обработка, така и за четимост от човека.

- **Офис документи**: PDF, PowerPoint (PPTX), Word (DOCX), Excel (XLSX/XLS)

- **Уеб съдържание**: HTML, RSS емисии, YouTube URL адреси, страници от Wikipedia
- **Данни формати**: CSV, JSON, XML, ZIP файлове (рекурсивна обработка на съдържанието)
- **Формати за публикуване**: EPub, Jupyter notebooks (.ipynb)

**Разширени възможности**: MarkItDown поддържа описания на изображения с LLM (когато е предоставен OpenAI клиент), Azure Document Intelligence за по-добра обработка на PDF, аудио транскрипция на речево съдържание и система за плъгини за разширяване към допълнителни формати на файлове.

**В практиката**: "Конвертирай тази PowerPoint презентация в Markdown за нашия сайт за документация", "Извлечи текст от този PDF с правилна структура на заглавията" или "Преобразувай тази Excel таблица в четим табличен формат"

**Пример от документацията**: За да цитираме [MarkItDown docs](https://github.com/microsoft/markitdown#why-markdown):

> Markdown е изключително близък до обикновения текст, с минимална маркировка или форматиране, но въпреки това предоставя начин за представяне на важна структура на документа. Основни LLM, като GPT-4o на OpenAI, естествено „говорят“ Markdown и често включват Markdown в отговорите си без подсещане. Това предполага, че те са били обучени върху огромни количества текст във формат Markdown и го разбират добре. Като допълнителна полза, конвенциите на Markdown са също много икономични по отношение на токените.

MarkItDown е много добър в запазване на структурата на документа, което е важно за AI работните потоци. Например, при конвертиране на PowerPoint презентация, той запазва организацията на слайдовете с правилните заглавия, извлича таблиците като Markdown таблици, включва алтернативен текст за изображения и дори обработва бележките на говорещия. Това го прави перфектен за подаване на съдържание от презентации към AI системи или създаване на документация от съществуващи слайдове.

MCP стартира браузър сесия, изследва структурата на страницата с помощта на DOM снимки, откри подходящите селектори и генерира напълно работещ TypeScript тест, който премина от първия опит.

Това, което прави тази възможност наистина мощна, е че тя преодолява разликата между инструкции на естествен език и изпълним код за тестове. Традиционните подходи изискват или ръчно писане на тестове, или достъп до кода за контекст. С Playwright MCP може да се тестват външни сайтове, клиентски приложения или в черни кутии, където достъпът до кода не е възможен.

**Какво прави**: Microsoft Foundry MCP Server предоставя на разработчиците пълен достъп до AI екосистемата на Azure, включително каталози с модели, управление на внедрявания, индексиране на знания с Azure AI Search и инструменти за оценка. Този експериментален сървър премахва пропастта между разработката на AI и мощната AI инфраструктура на Azure, улеснявайки изграждането, внедряването и оценяването на AI приложения.

**Защо е полезен**: Този сървър променя начина, по който работите с Azure AI услуги, като внася корпоративни AI възможности директно в работния ви процес за разработка. Вместо да превключвате между Azure портала, документацията и IDE-то си, можете да откривате модели, внедрявате услуги, управлявате бази знания и оценявате AI производителността чрез команди на естествен език. Особено е мощен за разработчици, изграждащи RAG (Retrieval-Augmented Generation) приложения, управляващи многомоделни внедрявания или прилагайки комплексни AI оценъчни потоци.

**Основни възможности за разработчици**:
- **🔍 Откриване и внедряване на модели**: Разглеждайте каталога с модели на Microsoft Foundry, получавайте подробна информация за модели с примерен код и ги внедрявайте в Azure AI Services
- **📚 Управление на знания**: Създавайте и управлявайте индекси в Azure AI Search, добавяйте документи, конфигурирайте индексери и изграждайте сложни RAG системи
- **⚡ Интеграция с AI агенти**: Свързвайте се с Azure AI агенти, правете запитвания към съществуващи агенти и оценявайте представянето на агенти в производствени сценарии
- **📊 Оценяваща рамка**: Провеждайте цялостни текстови и агентски оценки, генерирайте markdown отчети и прилагайте контрол на качеството за AI приложения
- **🚀 Инструменти за прототипиране**: Получавайте инструкции за настройка на прототипиране базирано на GitHub и достъп до Microsoft Foundry Labs за най-съвременни изследователски модели

**Примерна употреба в реалния свят**: "Внедри модел Phi-4 в Azure AI Services за моето приложение", "Създай нов индекс за търсене за моята RAG система на документацията", "Оцени отговорите на моя агент спрямо показатели за качество" или "Намери най-добрия модел за разсъждения за моите сложни аналитични задачи"

**Пълен демонстрационен сценарий**: Ето един мощен AI работен поток за разработка:

> "Изграждам агент за обслужване на клиенти. Помогни ми да намеря добър модел за разсъждения от каталога, да го внедря в Azure AI Services, да създам база знания от нашата документация, да настроя рамка за оценка, която да тества качеството на отговорите, и след това помогни да прототипирам интеграцията с GitHub токен за тестване."

Microsoft Foundry MCP Server ще:
- Направи запитване към каталога с модели, за да препоръча оптимални модели за разсъждения според твоите изисквания
- Предостави команди за внедряване и информация за квоти за предпочитания от теб Azure регион
- Настрои индекси на Azure AI Search със съответната схема за твоята документация
- Конфигурира оценяващи потоци с показатели за качество и проверки за безопасност
- Генерира прототипен код с GitHub автентикация за незабавно тестване
- Предостави цялостни ръководства за настройка, съобразени с твоя технологичен стек

**Отличен пример**: Като разработчик изпитвах затруднения да следя различните налични LLM модели. Знам няколко основни, но се чувствах, че изпускам някакви ползи за продуктивност и ефективност. Токените и квотите са стресиращи и трудни за управление – никога не съм сигурен дали избирам правилния модел за конкретната задача или изразходвам бюджета неефективно. Току-що чух за този MCP Server от James Montemagno, докато търсех препоръки за MCP Server-и от екипа си, и съм развълнуван да го използвам! Възможностите за откриване на модели изглеждат особено впечатляващи за човек като мен, който иска да изследва извън обичайните модели и да намери такива, оптимизирани за конкретни задачи. Оценяващата рамка ще ми помогне да валидирам, че наистина постигам по-добри резултати, а не просто да опитвам нещо ново заради самото ново.

## 📋 Заключение

Model Context Protocol (MCP) променя начина, по който разработчиците взаимодействат с AI асистенти и външни инструменти. Тези 10 Microsoft MCP сървъра демонстрират силата на стандартизирана AI интеграция, позволяваща безпроблемни работни потоци, които поддържат разработчиците в тяхното flow състояние, като същевременно осигуряват достъп до мощни външни възможности.

От цялостната интеграция на Azure екосистемата до специализирани инструменти като Playwright за браузърна автоматизация и MarkItDown за обработка на документи, тези сървъри показват как MCP може да подобри продуктивността в различни сценарии на разработка. Стандартизираният протокол гарантира, че тези инструменти работят заедно безпроблемно, създавайки кохезивно преживяване за разработчиците.

Докато MCP екосистемата продължава да се развива, активното участие в общността, изследването на нови сървъри и изграждането на персонализирани решения ще бъдат ключови за максимизиране на продуктивността ви при разработка. Отвореният стандарт на MCP позволява да смесвате и съчетавате инструменти от различни доставчици, за да създадете перфектния работен поток за вашите специфични нужди.
