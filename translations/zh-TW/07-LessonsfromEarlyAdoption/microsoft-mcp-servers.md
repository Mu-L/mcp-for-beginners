## 🔧 認識 MCP 伺服器：新手指南

### 什麼是 MCP 伺服器？

作為 Model Context Protocol (MCP) 的新手，您可能會想：「到底什麼是 MCP 伺服器？我為什麼要在意？」那我們先用個簡單比喻開始。

把 MCP 伺服器想像成專門的助手，它幫助您的 AI 編碼夥伴（例如 GitHub Copilot）連結到外部工具和服務。就像您手機會針對不同任務使用不同的應用程式—天氣、導航、銀行等—MCP 伺服器讓您的 AI 助手能與多種開發工具和服務互動。

### MCP 伺服器解決了什麼問題

在有 MCP 伺服器之前，如果您想：
- 查看 Azure 資源
- 創建 GitHub 問題
- 查詢資料庫
- 搜索文件

您必須暫停編碼，打開瀏覽器，前往對應網站並手動執行這些操作。不斷切換情境會打斷專注並降低生產力。

### MCP 伺服器如何改變您的開發體驗

有了 MCP 伺服器，您可以留在開發環境裡（VS Code、Visual Studio 等），直接請 AI 助手幫忙處理例如這樣的任務。例如：

**傳統流程是這樣：**
1. 停止編碼
2. 開啟瀏覽器
3. 進入 Azure 入口網站
4. 查看儲存帳戶詳細資料
5. 回到 VS Code
6. 繼續編碼

**現在可以這樣做：**
1. 問 AI：「我的 Azure 儲存帳戶狀態如何？」
2. 根據回覆繼續編碼

### 新手的主要好處

#### 1. 🔄 <strong>保持專注狀態</strong>
- 不必切換多個應用程式
- 持續專注於書寫的程式碼
- 降低管理多種工具的心理負擔

#### 2. 🤖 <strong>使用自然語言代替複雜指令</strong>
- 不必背 SQL 語法，直接描述需要的資料
- 不用記 Azure CLI 指令，說明目標即可
- 由 AI 處理技術細節，您專注邏輯

#### 3. 🔗 <strong>連結多種工具</strong>
- 透過結合不同服務建立強大工作流程
- 例如：「取得最近 GitHub 問題並創建相對應的 Azure DevOps 任務」
- 無需撰寫複雜腳本即可實現自動化

#### 4. 🌐 <strong>存取日益壯大的生態系</strong>
- 享受由 Microsoft、GitHub 及其他公司建置的伺服器
- 輕鬆混合搭配來自不同廠商的工具
- 加入跨多款 AI 助手通用的標準化生態系

#### 5. 🛠️ <strong>實做中學習</strong>
- 從預建伺服器開始了解概念
- 漸進式建立自己的伺服器
- 利用可用的 SDK 與文件指導學習

### 新手實例

假設您剛開始學習網頁開發、進行第一個專案，MCP 伺服器如何幫助您：

```
5. Continue coding with all the information you need
```

## 在 VS Code 和 Visual Studio 裡開始使用 MCP 🚀

在 Visual Studio Code 或 Visual Studio 2022 搭配 GitHub Copilot 時，設定這些 MCP 伺服器相當簡單。

### VS Code 設定

VS Code 的基本流程如下：

1. <strong>啟用代理模式</strong>：在 Copilot Chat 視窗切換到代理模式
2. **設定 MCP 伺服器**：將伺服器配置加入 VS Code 的 settings.json
3. <strong>啟動伺服器</strong>：點選要使用的伺服器旁的「啟動」按鈕
4. <strong>選擇工具</strong>：選擇本次會話要啟用的 MCP 伺服器

詳細設定說明，請參考[VS Code MCP 文件](https://code.visualstudio.com/docs/copilot/copilot-mcp)。

> **💡 專業小技巧：像專家一樣管理 MCP 伺服器！**
> 
> VS Code 擴充視窗現新增了[好用的新介面管理已安裝的 MCP 伺服器](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode)！您可以輕鬆快速地啟動、停止和管理任意已安裝的 MCP 伺服器，介面清楚簡單。快試試看！

### Visual Studio 2022 設定

Visual Studio 2022（版本 17.14 或更新）設定步驟：

1. <strong>啟用代理模式</strong>：在 GitHub Copilot Chat 視窗點選「問」下拉選單，選擇「代理」
2. <strong>建立設定檔</strong>：於解決方案目錄建立 `.mcp.json` 檔案（建議位置：`<SOLUTIONDIR>\.mcp.json`）
3. <strong>設定伺服器</strong>：使用標準 MCP 格式加入您的 MCP 伺服器設定
4. <strong>工具批准</strong>：提示時核准欲使用的工具及範圍權限

Visual Studio 詳細設定請參考[Visual Studio MCP 文件](https://learn.microsoft.com/visualstudio/ide/mcp-servers)。

每個 MCP 伺服器都會有自己的配置需求（連線字串、認證等），但兩種 IDE 的設定模式一致。

### 1. 📚 Microsoft Learn Docs MCP 伺服器

[![在 VS Code 安裝](https://img.shields.io/badge/VS_Code-Install_Microsoft_Docs_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D) [![在 VS Code Insiders 安裝](https://img.shields.io/badge/VS_Code_Insiders-Install_Microsoft_Docs_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/mcp)

<strong>功能說明</strong>：Microsoft Learn Docs MCP 伺服器是一個雲端服務，透過 Model Context Protocol 提供 AI 助手即時存取官方 Microsoft 文件的能力。它連接至 `https://learn.microsoft.com/api/mcp`，可對 Microsoft Learn、Azure 文件、Microsoft 365 文件及其他官方來源進行語意搜尋。

<strong>為何有用</strong>：雖然看似只是「文件」，但此伺服器對每位使用 Microsoft 技術的開發者非常關鍵。許多 .NET 開發者對 AI 編碼助手最大抱怨是版本不夠新。Microsoft Learn Docs MCP 伺服器透過提供最新的官方文件、API 參考及最佳實務，解決這個問題。無論您正在使用最新 Azure SDK、探索 C# 13 新功能，或實作尖端的 .NET Aspire 範式，此伺服器保證您的 AI 助手能存取權威且即時的資訊，產生準確、現代的程式碼。

<strong>實際應用</strong>：「根據官方 Microsoft Learn 文件，建立 Azure container app 的 az cli 指令是什麼？」或「如何在 ASP.NET Core 使用依賴注入設定 Entity Framework？」又或者「審查這段程式碼，確保符合 Microsoft Learn 文件中的效能建議。」此伺服器使用進階語意搜尋，在 Microsoft Learn、Azure 文件和 Microsoft 365 文件提供全面覆蓋，找出最貼近上下文的資訊。它會返回最多 10 個高品質內容區塊，包含文章標題與 URL，始終存取最新發佈的 Microsoft 文件。

<strong>特色範例</strong>：伺服器公開執行 `microsoft_docs_search` 工具對官方技術文件執行語意搜尋。設定好後，您可以問：「如何在 ASP.NET Core 實作 JWT 認證？」 等問題，並獲得詳細且帶來源連結的官方回答。搜尋品質優異，能理解上下文——譬如在 Azure 相關的「容器」會返回 Azure Container Instances 文件，在 .NET 相關情境則返回 C# 集合相關資訊。

此功能對快速變化或近期更新的函式庫和使用案例尤其有用。例如近期某些專案中，我想利用 .NET Aspire 與 Microsoft.Extensions.AI 的最新發佈特性。加入 Microsoft Learn Docs MCP 伺服器後，我不僅能使用 API 文件，還能得到剛發佈的教學與指引。

> **💡 專業小技巧**
> 
> 即使是有支援工具的模型也需要激勵去使用 MCP 工具！可考慮加入系統提示或 [copilot-instructions.md](https://docs.github.com/copilot/how-tos/custom-instructions/adding-repository-custom-instructions-for-github-copilot) 來說：「您可以使用 `microsoft.docs.mcp` — 當處理關於 Microsoft 技術（如 C#、Azure、ASP.NET Core 或 Entity Framework）的問題時，請用此工具搜尋 Microsoft 最新官方文件。」
>
> 想看實例，請參考 Awesome GitHub Copilot 倉庫的 [C# .NET Janitor chat mode](https://github.com/awesome-copilot/chatmodes/blob/main/csharp-dotnet-janitor.chatmode.md)。此模式特別利用 Microsoft Learn Docs MCP 伺服器幫助清理與現代化 C# 程式碼，使用最新的範式與最佳實務。

### 2. ☁️ Azure MCP 伺服器
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

<strong>它的功能</strong>：Azure MCP Server 是一套包含 15+ 個專用 Azure 服務連接器的綜合套件，將整個 Azure 生態系統帶入您的 AI 工作流程。這不僅僅是一個單一伺服器，它是一個強大的集合，包括資源管理、資料庫連接（PostgreSQL、SQL Server）、使用 KQL 的 Azure Monitor 日誌分析、Cosmos DB 整合，以及更多功能。

<strong>它的用處</strong>：除了管理 Azure 資源之外，這個伺服器在使用 Azure SDK 時能大幅提升程式碼品質。當您以 Agent 模式使用 Azure MCP 時，它不僅幫助您撰寫程式碼，更幫助您撰寫<em>更好的</em> Azure 程式碼，遵循當前的身份驗證模式、最佳錯誤處理實踐，並利用最新的 SDK 功能。您不會拿到可能能用的通用程式碼，而是會得到符合 Azure 生產工作負載推薦模式的程式碼。

<strong>主要模組包括</strong>：
- **🗄️ 資料庫連接器**：透過自然語言直接存取 Azure Database for PostgreSQL 和 SQL Server
- **📊 Azure Monitor**：基於 KQL 的日誌分析與營運洞察
- **🌐 資源管理**：完整的 Azure 資源生命周期管理
- **🔐 身份驗證**：DefaultAzureCredential 及管理身份模式
- **📦 儲存服務**：Blob 儲存、佇列儲存及表格儲存操作
- **🚀 容器服務**：Azure 容器應用、容器實例及 AKS 管理
- <strong>以及更多專用連接器</strong>

<strong>實際使用範例</strong>：「列出我的 Azure 儲存帳戶」、「查詢我 Log Analytics 工作區過去一小時內的錯誤」、「協助我用 Node.js 建立一個具備適當身份驗證的 Azure 應用程式」

<strong>完整示範情境</strong>：這裡有一個完整的示範，展示如何將 Azure MCP 與 VS Code 中的 GitHub Copilot for Azure 擴充功能結合使用。當您兩者皆安裝並輸入：

> 「建立一個 Python 腳本，使用 DefaultAzureCredential 認證上傳檔案到 Azure Blob Storage。腳本應該連接到名為 'mycompanystorage' 的 Azure 儲存帳戶，並上傳到名為 'documents' 的容器，建立一個包含當前時間戳記的測試檔案來上傳，優雅處理錯誤並提供資訊輸出，遵循 Azure 身份驗證與錯誤處理最佳實踐，包含說明 DefaultAzureCredential 認證運作的註解，且讓腳本結構良好，具備適當的函式與文件說明。」

Azure MCP Server 將生成一個完整的、可生產使用的 Python 腳本：
- 使用最新的 Azure Blob Storage SDK 及適當的非同步模式
- 實作 DefaultAzureCredential 並說明完整的後備鏈
- 包含強健的錯誤處理與具體的 Azure 例外類型
- 遵守 Azure SDK 在資源管理與連接處理上的最佳實務
- 提供詳細的日誌與資訊性主控台輸出
- 撰寫結構完善的腳本，包含函式、文件說明與型別提示

此處的卓越之處在於，沒有 Azure MCP，您可能會拿到一段通用但不符合目前 Azure 作法的 blob 儲存程式碼。有了 Azure MCP，您會得到使用最新身份驗證方法、處理 Azure 特有錯誤情境，且遵循微軟推薦生產應用實踐的程式碼。

<strong>特色範例</strong>：我常常記不住 `az` 和 `azd` CLI 的特定指令，臨時需要時總是先查語法再執行指令。經常跳去門戶點來點去才能完成工作，因為不想承認自己記不起 CLI 語法。能夠僅用描述方式表達需求真的很棒，更棒的是不離開 IDE 就能做到！

您可以在 [Azure MCP repository](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) 找到一份很棒的用例列表供您起步。更完整的安裝指南和進階設定選項可參考 [官方 Azure MCP 文件](https://learn.microsoft.com/azure/developer/azure-mcp-server/)。

### 3. 🐙 GitHub MCP Server

[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Server-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=github&config=%7B%22type%22%3A%20%22http%22%2C%22url%22%3A%20%22https%3A%2F%2Fapi.githubcopilot.com%2Fmcp%2F%22%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Server-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=github&config=%7B%22type%22%3A%20%22http%22%2C%22url%22%3A%20%22https%3A%2F%2Fapi.githubcopilot.com%2Fmcp%2F%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/github/github-mcp-server)

<strong>它的功能</strong>：官方 GitHub MCP Server 提供與 GitHub 全生態系的無縫整合，提供託管遠端存取及本機 Docker 部署選項。它不只是基礎的儲存庫操作，而是一套完整工具組，涵蓋 GitHub Actions 管理、Pull Request 流程、議題追蹤、安全掃描、通知與進階自動化能力。

<strong>它的用處</strong>：該伺服器改變了您與 GitHub 的互動方式，將完整平台體驗直接帶入開發環境。您不須在 VS Code 與 GitHub.com 間頻繁切換，以管理專案、碼審與 CI/CD 監控，而能用自然語言指令執行所有操作，一心專注於程式碼。

> **ℹ️ 注意：不同類型的「Agent」**
> 
> 請勿將此 GitHub MCP Server 與 GitHub 的 Coding Agent 混淆（Coding Agent 是可被指派議題以執行自動化編碼任務的 AI 代理）。GitHub MCP Server 在 VS Code 的 Agent 模式中提供 GitHub API 整合，而 Coding Agent 則是獨立功能，會在被指派到 GitHub 議題時建立 Pull Request。

<strong>主要功能包括</strong>：
- **⚙️ GitHub Actions**：完整 CI/CD 管線管理，工作流程監控與工件處理
- **🔀 Pull Requests**：建立、審查、合併及管理 PR，詳盡追蹤狀態
- **🐛 議題**：完整議題生命週期管理、留言、標籤與指派
- **🔒 安全**：程式碼掃描警示、機密偵測及 Dependabot 整合

- <strong>資料格式</strong>：CSV、JSON、XML、ZIP 檔（遞迴處理內容）
- <strong>出版格式</strong>：EPub、Jupyter 筆記本 (.ipynb)
- <strong>電子郵件</strong>：Outlook 郵件 (.msg)

<strong>進階功能</strong>：MarkItDown 支援大型語言模型驅動的影像描述（當提供 OpenAI 用戶端時）、Azure 文件智能以強化 PDF 處理、語音內容轉錄，以及擴充額外文件格式的外掛系統。

<strong>真實案例應用</strong>：如「將這個 PowerPoint 簡報轉換成文件站的 Markdown」、「擷取此 PDF 的帶有正確標題結構的文字」、或「把這個 Excel 試算表變成可讀表格格式」。

<strong>特色範例</strong>：引用[MarkItDown 文件](https://github.com/microsoft/markitdown#why-markdown)：

> Markdown 與純文字極為接近，僅有極少的標記或格式，但仍提供呈現重要文件結構的方式。主流大型語言模型，如 OpenAI 的 GPT-4o，本身「說」Markdown，且經常在未被提示下直接用 Markdown 回應。這暗示它們是基於大量 Markdown 格式文本訓練，並且相當熟悉它。此外，Markdown 規範同時極為節省 token。

MarkItDown 非常擅長保留文件結構，這對於 AI 工作流程至關重要。例如，轉換 PowerPoint 簡報時，它會保持幻燈片組織及正確的標題階層，擷取表格作為 Markdown 表格，附帶圖片替代文字，甚至處理演講者備註。圖表會轉換為可讀的數據表，而輸出的 Markdown 保持原始簡報的邏輯流程。這使它適合將簡報內容餵給 AI 系統或從現有投影片創建文檔。

## 📋 結論

模型上下文協定（MCP）正在改變開發者與 AI 輔助工具及外部工具的互動方式。這 10 個 Microsoft MCP 伺服器展現了標準化 AI 整合的強大力量，實現無縫工作流程，讓開發者在維持專注狀態的同時，存取強大外部功能。

從 Azure 生態系統的全面整合，到 Playwright 用於瀏覽器自動化、MarkItDown 用於文檔處理等專門化工具，這些伺服器示範了 MCP 如何提升不同開發場景的生產力。標準化協定確保這些工具協同運作，打造連貫的開發體驗。

隨著 MCP 生態系統持續演進，積極參與社群、探索新伺服器與打造自訂解決方案，將是最大化開發生產力的關鍵。MCP 的開放標準特性讓您能混合不同廠商的工具，打造符合您需求的完美工作流程。
