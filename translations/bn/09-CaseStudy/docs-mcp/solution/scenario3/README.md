# দৃশ্যপট ৩: VS Code-এ MCP সার্ভারের সাথে ইন-এডিটর ডকস

## ওভারভিউ

এই দৃশ্যপটে, আপনি শিখবেন কিভাবে MCP সার্ভার ব্যবহার করে মাইক্রোসফট লার্ন ডকস সরাসরি আপনার ভিজ্যুয়াল স্টুডিও কোড পরিবেশে নিয়ে আসতে হয়। ডকুমেন্টেশন খুঁজতে ক্রমাগত ব্রাউজার ট্যাব পরিবর্তন করার পরিবর্তে, আপনি সম্পাদকের ভিতরেই অফিসিয়াল ডকস অ্যাক্সেস, সার্চ এবং রেফারেন্স করতে পারবেন। এই পদ্ধতি আপনার কাজের ধারা সরল করে, আপনাকে মনোযোগ কেন্দ্রীভূত রাখতে সাহায্য করে এবং GitHub Copilot এর মতো টুলসের সঙ্গে নির্বিঘ্ন সংমিশ্রণ সক্ষম করে।

- VS Code-এর ভিতরেই আপনার কোডিং পরিবেশ ত্যাগ না করে ডকস সার্চ এবং পড়ুন।
- ডকুমেন্টেশন রেফারেন্স করুন এবং লিঙ্ক সরাসরি আপনার README বা কোর্স ফাইলগুলোতে ইনসার্ট করুন।
- GitHub Copilot এবং MCP একসাথে ব্যবহার করুন একটি নির্বিঘ্ন, AI-সক্ষম ডকুমেন্টেশন ওয়ার্কফ্লোর জন্য।

## শেখার লক্ষ্যসমূহ

এই অধ্যায়ের শেষে, আপনি VS Code-এ MCP সার্ভার সেটআপ এবং ব্যবহার করে ডকুমেন্টেশন ও ডেভেলপমেন্ট কাজের ধারা উন্নত করতে সক্ষম হবেন। আপনি পারবেন:

- MCP সার্ভার ব্যবহার করার জন্য আপনার ওয়ার্কস্পেস কনফিগার করা।
- VS Code-এর ভিতর থেকেই ডকুমেন্টেশন সার্চ এবং ইনসার্ট করা।
- আরও উত্পাদনশীল, AI-সহায়িত কাজের ধারা জন্য GitHub Copilot এবং MCP এর ক্ষমতা একত্র করা।

এই দক্ষতাগুলো আপনাকে ফোকাসে থাকতে সাহায্য করবে, ডকুমেন্টেশনের গুণগত মান বাড়াবে এবং একজন ডেভেলপার বা টেকনিক্যাল রাইটার হিসেবে আপনার উৎপাদনশীলতা বাড়াবে।

## সমাধান

ইন-এডিটর ডকুমেন্টেশন অ্যাক্সেস অর্জন করার জন্য, আপনি একটি সিরিজ ধাপ অনুসরণ করবেন যা MCP সার্ভারকে VS Code এবং GitHub Copilot এর সাথে সংযুক্ত করে। এই সমাধানটি কোর্স লেখক, ডকুমেন্টেশন লেখক এবং ডেভেলপারদের জন্য আদর্শ যারা ডকস এবং Copilot নিয়ে কাজ করার সময় সম্পাদকেই ফোকাস রাখতে চান।

- কোর্স বা প্রজেক্ট ডকুমেন্টেশন লিখার সময় দ্রুত রেফারেন্স লিঙ্ক README-তে যোগ করুন।
- কোড জেনারেট করার জন্য Copilot এবং প্রাসঙ্গিক ডকস খুঁজে পেতে MCP ব্যবহার করুন।
- সম্পাদকেই ফোকাস রাখুন এবং উত্পাদনশীলতা বাড়ান।

### ধাপে ধাপে গাইড

শুরু করতে, এই ধাপগুলো অনুসরণ করুন। প্রতিটি ধাপের জন্য, প্রক্রিয়াটি দৃশ্যায়িত করতে আপনি অ্যাসেটস ফোল্ডার থেকে স্ক্রিনশট যোগ করতে পারেন।

1. **MCP কনফিগারেশন যোগ করুন:**
   আপনার প্রজেক্ট রুটে `.vscode/mcp.json` ফাইল তৈরি করুন এবং নিম্নলিখিত কনফিগারেশন যোগ করুন:
   ```json
   {
     "servers": {
       "LearnDocsMCP": {
         "url": "https://learn.microsoft.com/api/mcp"
       }
     }
   }
   ```
   এই কনফিগারেশনটি VS Code কে জানায় কিভাবে [`Microsoft Learn Docs MCP সার্ভার`](https://github.com/MicrosoftDocs/mcp)-এর সাথে সংযোগ স্থাপন করতে হয়।
   
   ![Step 1: Add mcp.json to .vscode folder](../../../../../../translated_images/bn/step1-mcp-json.c06a007fccc3edfa.webp)
    
2. **GitHub Copilot Chat প্যানেল খুলুন:**
   যদি আপনার কাছে GitHub Copilot এক্সটেনশন ইতিমধ্যেই ইনস্টল না থাকে, তবে VS Code এর এক্সটেনশন ভিউতে যান এবং এটি ইনস্টল করুন। আপনি এটি সরাসরি [Visual Studio Code Marketplace](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot-chat)-থেকে ডাউনলোড করতে পারেন। এরপর, সাইডবার থেকে Copilot Chat প্যানেল খুলুন।

   ![Step 2: Open Copilot Chat panel](../../../../../../translated_images/bn/step2-copilot-panel.f1cc86e9b9b8cd1a.webp)

3. **এজেন্ট মোড সক্ষম করুন এবং টুলস যাচাই করুন:**
   Copilot Chat প্যানেলে এজেন্ট মোড চালু করুন।

   ![Step 3: Enable agent mode and verify tools](../../../../../../translated_images/bn/step3-agent-mode.cdc32520fd7dd1d1.webp)

   এজেন্ট মোড চালুর পর, যাচাই করুন MCP সার্ভার একটি উপলব্ধ টুল হিসেবে তালিকাভুক্ত আছে কিনা। এর মাধ্যমে Copilot এজেন্ট ডকুমেন্টেশন সার্ভার থেকে প্রাসঙ্গিক তথ্য আনতে পারবে।
   
   ![Step 3: Verify MCP server tool](../../../../../../translated_images/bn/step3-verify-mcp-tool.76096a6329cbfecd.webp)
4. **নতুন চ্যাট শুরু করুন এবং এজেন্টকে প্রম্পট দিন:**
   Copilot Chat প্যানেলে একটি নতুন চ্যাট খুলুন। এখন আপনি আপনার ডকুমেন্টেশন প্রশ্ন নিয়ে এজেন্টকে প্রম্পট করতে পারবেন। এজেন্ট MCP সার্ভার ব্যবহার করে প্রাসঙ্গিক Microsoft Learn ডকুমেন্টেশন সরাসরি আপনার সম্পাদকেই নিয়ে আসবে।

   - *"আমি একটি বিষয় X এর জন্য ৮ সপ্তাহের একটি স্টাডি প্ল্যান লিখতে যাচ্ছি। প্রতি সপ্তাহে আমাকে কোন কোন বিষয়বস্তু নিতে হবে তা সুপারিশ করুন।"*

   ![Step 4: Prompt the agent in chat](../../../../../../translated_images/bn/step4-prompt-chat.12187bb001605efc.webp)

5. **সরাসরি কোয়েরি:**

   > আসুন Microsoft Foundry Discord-এর [#get-help](https://discord.gg/D6cRhjHWSC) সেকশন থেকে সরাসরি একটি কোয়েরি নিই ([মূল মেসেজ দেখুন](https://discord.com/channels/1113626258182504448/1385498306720829572)):
   
   *"আমি জানতে চাই কিভাবে Azure AI Foundry তে তৈরি AI এজেন্ট দিয়ে একটি মাল্টি-এজেন্ট সলিউশন ডিপ্লয় করতে হয়। আমি দেখেছি সরাসরি ডিপ্লয়মেন্ট পদ্ধতি যেমন Copilot Studio চ্যানেল নেই। তাহলে এন্টারপ্রাইজ ব্যবহারকারীরা ইন্টারঅ্যাক্ট করতে এবং কাজ সম্পন্ন করতে বিভিন্ন উপায় কি কি? অনেক আর্টিকেল/ব্লগ বলছে Azure Bot সার্ভিস ব্যবহার করা যেতে পারে যা MS Teams এবং Azure AI Foundry এজেন্টের মধ্যে একটি সেতুবন্ধন হিসেবে কাজ করবে, তাহলে কি Azure Bot সেটআপ করলে যা Azure AI Foundry এর Orchestrator Agent এর সাথে Azure ফাংশনের মাধ্যমে সংযুক্ত হবে, কাজ করবে? নাকি প্রতিটি AI এজেন্ট অংশের জন্য আলাদা Azure ফাংশন তৈরি করতে হবে যাতে বট ফ্রেমওয়ার্কে অর্কেস্ট্রেশন করা যায়? অন্য কোন পরামর্শ থাকলেও স্বাগত।"*

   ![Step 5: Live queries](../../../../../../translated_images/bn/step5-live-queries.49db3e4a50bea273.webp)

   এজেন্ট প্রাসঙ্গিক ডকুমেন্টেশন লিঙ্ক এবং সারাংশ দিয়ে প্রতিক্রিয়া জানাবে, যা আপনি সরাসরি আপনার মার্কডাউন ফাইলে ইনসার্ট করতে পারবেন বা কোডে রেফারেন্স হিসেবে ব্যবহার করতে পারবেন।

### নমুনা কোয়েরি

এখানে কিছু উদাহরণ কোয়েরি দেওয়া হলো যা আপনি চেষ্টা করতে পারেন। এগুলো দেখাবে কিভাবে MCP সার্ভার এবং Copilot একসাথে কাজ করে দ্রুত, প্রসঙ্গ সচেতন ডকুমেন্টেশন এবং রেফারেন্স সরবরাহ করে VS Code ত্যাগ না করেই:

- "আমাকে দেখাও কিভাবে Azure Functions ট্রিগার ব্যবহার করতে হয়।"
- "Azure Key Vault-এর অফিসিয়াল ডকুমেন্টেশনের লিঙ্ক ইনসার্ট করো।"
- "Azure রিসোর্স সুরক্ষার সেরা অনুশীলনগুলি কি কি?"
- "Azure AI সার্ভিসের জন্য একটি Quickstart খুঁজে দাও।"

এই কোয়েরিগুলো MCP সার্ভার এবং Copilot কিভাবে একসাথে কাজ করে VS Code ত্যাগ না করেই তাৎক্ষণিক, প্রসঙ্গ সচেতন ডকুমেন্টেশন ও রেফারেন্স প্রদান করতে পারে তা প্রমাণ করবে।
