# MCP নিরাপত্তা: AI সিস্টেমের জন্য ব্যাপক নিরাপত্তা

[![MCP Security Best Practices](../../../translated_images/bn/03.175aed6dedae133f.webp)](https://youtu.be/88No8pw706o)

_(এই পাঠের ভিডিও দেখতে উপরে ছবিটিতে ক্লিক করুন)_

নিরাপত্তা AI সিস্টেম ডিজাইনের মৌলিক, এ কারণেই আমরা এটিকে আমাদের দ্বিতীয় বিভাগ হিসেবে অগ্রাধিকার দিই। এটি Microsoft-এর [Secure Future Initiative](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) থেকে **Secure by Design** নীতির সঙ্গে সামঞ্জস্যপূর্ণ।

মডেল কনটেক্সট প্রোটোকল (MCP) AI-চালিত অ্যাপ্লিকেশনগুলিতে শক্তিশালী নতুন ক্ষমতা নিয়ে আসে তবে এটি ঐতিহ্যবাহী সফটওয়্যার ঝুঁকির বাইরে অনন্য নিরাপত্তা চ্যালেঞ্জ উপস্থাপন করে। MCP সিস্টেমগুলি প্রতিষ্ঠিত নিরাপত্তা উদ্বেগ যেমন সুরক্ষিত কোডিং, সর্বনিম্ন বিশেষাধিকার, সরবরাহ চেইন নিরাপত্তা এবং নতুন AI-নির্দিষ্ট হুমকি যেমন প্রম্পট ইনজেকশন, টুল পয়জনিং, সেশন হাইজ্যাকিং, কনফিউজড ডেপুটি আক্রমণ, টোকেন পাসথ্রু দুর্বলতা, এবং গতিশীল ক্ষমতা পরিবর্তন সহ মোকাবিলা করে।

এই পাঠে MCP বাস্তবায়নে সবচেয়ে গুরুত্বপূর্ণ নিরাপত্তা ঝুঁকিগুলো অন্বেষণ করা হয়েছে—যা প্রমাণীকরণ, অনুমোদন, অতিরিক্ত অনুমতিসমূহ, পরোক্ষ প্রম্পট ইনজেকশন, সেশন নিরাপত্তা, কনফিউজড ডেপুটি সমস্যা, টোকেন ব্যবস্থাপনা, এবং সরবরাহ চেইন দুর্বলতা অন্তর্ভুক্ত। আপনি ঝুঁকিগুলো কমানোর জন্য কার্যকর নিয়ন্ত্রণ এবং সেরা অনুশীলন শিখবেন Microsoft-এর সমাধান যেমন Prompt Shields, Azure Content Safety, এবং GitHub Advanced Security কে ব্যবহার করে আপনার MCP মোতায়েনের শক্তি বাড়ানো।

## শেখার উদ্দেশ্যসমূহ

এই পাঠের শেষে আপনি সক্ষম হবেন:

- **MCP-নির্দিষ্ট হুমকি চিন্হিত করা**: MCP সিস্টেমের অনন্য নিরাপত্তা ঝুঁকি চিন্তা করা যেমন প্রম্পট ইনজেকশন, টুল পয়জনিং, অতিরিক্ত অনুমতি, সেশন হাইজ্যাকিং, কনফিউজড ডেপুটি সমস্যা, টোকেন পাসথ্রু দুর্বলতা, এবং সরবরাহ চেইন ঝুঁকি
- **নিরাপত্তা নিয়ন্ত্রণ প্রয়োগ করা**: শক্তিশালী প্রমাণীকরণ, সর্বনিম্ন বিশেষাধিকার প্রবেশাধিকার, সুরক্ষিত টোকেন ব্যবস্থাপনা, সেশন নিরাপত্তা নিয়ন্ত্রণ এবং সরবরাহ চেইন যাচাই সহ কার্যকর প্রতিকারসমূহ বাস্তবায়ন করা
- **Microsoft নিরাপত্তা সমাধান ব্যবহার করা**: MCP ওয়ার্কলোড সুরক্ষার জন্য Microsoft Prompt Shields, Azure Content Safety, এবং GitHub Advanced Security বুঝে ও স্থাপন করা
- **টুল নিরাপত্তা যাচাই করা**: টুল মেটাডেটা যাচাই, গতিশীল পরিবর্তন নজরদারি এবং পরোক্ষ প্রম্পট ইনজেকশন আক্রমণের বিরুদ্ধে সুরক্ষা গুরুত্বপূর্ণতা চিন্তা করা
- **সেরা অনুশীলন একীভূত করা**: প্রতিষ্ঠিত নিরাপত্তার মৌলিক বিষয়াবলী (সুরক্ষিত কোডিং, সার্ভার শক্তিশালীকরণ, শূন্য বিশ্বাস) MCP-নির্দিষ্ট নিয়ন্ত্রণের সঙ্গে মিলিয়ে ব্যাপক সুরক্ষা নিশ্চিত করা

# MCP নিরাপত্তা স্থাপত্য ও নিয়ন্ত্রণসমূহ

আধুনিক MCP বাস্তবায়নগুলি স্তরবদ্ধ নিরাপত্তা পন্থা দাবি করে যা ঐতিহ্যবাহী সফটওয়্যার নিরাপত্তা এবং AI-নির্দিষ্ট হুমকিগুলো উভয়ই মোকাবিলা করে। দ্রুত পরিবর্তিত MCP স্পেসিফিকেশন তার নিরাপত্তা নিয়ন্ত্রণগুলি পরিণত করছে, যা এন্টারপ্রাইজ নিরাপত্তা স্থাপত্য এবং প্রতিষ্ঠিত সেরা অনুশীলনের সাথে ভালো ইন্টিগ্রেশনকে সক্ষম করে।

[Microsoft Digital Defense Report](https://aka.ms/mddr) থেকে গবেষণা প্রমাণ করে যে **রিপোর্টকৃত ৯৮% বিচ্ছেদ শক্তিশালী নিরাপত্তা ব্যবস্থা দ্বারা প্রতিরোধযোগ্য**। সবচেয়ে কার্যকর সুরক্ষা কৌশল হল ভিত্তিমূলক নিরাপত্তা অনুশীলনকে MCP-নির্দিষ্ট নিয়ন্ত্রণের সঙ্গে মিলিয়ে নেওয়া—প্রমাণিত মূল নিরাপত্তা ব্যবস্থা সামগ্রিক ঝুঁকি কমাতে সবচেয়ে প্রভাবশালী।

## বর্তমান নিরাপত্তা দৃশ্যপট

> **দ্রষ্টব্য:** এই তথ্য **৫ ফেব্রুয়ারি, ২০২৬** তারিখে MCP নিরাপত্তা মানটি প্রতিফলিত করে, যা **MCP স্পেসিফিকেশন ২০২৫-১১-২৫** এর সাথে সামঞ্জস্যপূর্ণ। MCP প্রোটোকল দ্রুত বিবর্তিত হচ্ছে এবং ভবিষ্যতের বাস্তবায়নে নতুন প্রমাণীকরণ প্যাটার্ন ও উন্নত নিয়ন্ত্রণ আসতে পারে। সর্বশেষ নির্দেশিকার জন্য বর্তমান [MCP স্পেসিফিকেশন](https://spec.modelcontextprotocol.io/), [MCP GitHub রেপোজিটরি](https://github.com/modelcontextprotocol), এবং [সিকিউরিটি বেস্ট প্র্যাকটিস ডকুমেন্টেশন](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) দেখুন।

## 🏔️ MCP সিকিউরিটি সামিট ওয়ার্কশপ (শের্পা)

**হ্যান্ডস-অন সিকিউরিটি প্রশিক্ষণের জন্য**, আমরা অত্যন্ত পরামর্শ দিই **MCP সিকিউরিটি সামিট ওয়ার্কশপ** (শের্পা)—Microsoft Azure-এ MCP সার্ভার সুরক্ষায় একটি ব্যাপক নির্দেশিত অভিযান।

### ওয়ার্কশপ ওভারভিউ

[MCP সিকিউরিটি সামিট ওয়ার্কশপ](https://azure-samples.github.io/sherpa/) কার্যকর, ব্যবহারিক নিরাপত্তা প্রশিক্ষণ প্রদান করে "ভালনারেবল → এক্সপ্লয়েট → ফিক্স → ভ্যালিডেট" পদ্ধতির মাধ্যমে। আপনি:

- **ভঙ্গ করে শিখুন**: ইচ্ছাকৃতভাবে অসুরক্ষিত সার্ভার ব্যবহার করে দুর্বলতাগুলো নিজে থেকে অনুভব করবেন
- **Azure-নেটিভ নিরাপত্তা ব্যবহার করুন**: Azure Entra ID, Key Vault, API Management, এবং AI Content Safety ব্যবহার করুন
- **ডিফেন্স-ইন-ডেপথ অনুসরণ করুন**: প্রচুর নিরাপত্তা স্তর তৈরি করে অগ্রসর হোন
- **OWASP মান অনুসরণ করুন**: প্রতিটি প্রযুক্তি [OWASP MCP Azure Security Guide](https://microsoft.github.io/mcp-azure-security-guide/) অনুযায়ী
- **প্রডাকশন কোড পান**: কাজের, পরীক্ষিত বাস্তবায়ন নিয়ে যান

## OWASP MCP শীর্ষ ১০ নিরাপত্তা ঝুঁকি

| ঝুঁকি | বর্ণনা | Azure প্রতিকার |
|--------|---------|-----------------|
| **MCP01** | টোকেন ব্যবস্থাপনা ও গোপনীয়তা উৎঘাটন | Azure Key Vault, Managed Identity |
| **MCP02** | স্কোপ ক্রিপের মাধ্যমে বিশেষাধিকার উর্ধ্বগতি | RBAC, Conditional Access |
| **MCP03** | টুল পয়জনিং | টুল যাচাইকরণ, অখণ্ডতা যাচাই |
| **MCP04** | সফটওয়্যার সরবরাহ চেইন আক্রমণ ও নির্ভরশীলতা পরিবর্তন | GitHub Advanced Security, নির্ভরশীলতা স্ক্যানিং |
| **MCP05** | কমান্ড ইনজেকশন ও এক্সিকিউশন | ইনপুট যাচাই, স্যান্ডবক্সিং |
| **MCP06** | ইন্টেনট ফ্লো সাবভার্শন | Azure AI Content Safety, Prompt Shields |
| **MCP07** | অপর্যাপ্ত প্রমাণীকরণ ও অনুমোদন | Azure Entra ID, OAuth 2.1 এবং PKCE |
| **MCP08** | নিরীক্ষা এবং টেলিমেট্রি অভাব | Azure Monitor, Application Insights |
| **MCP09** | ছায়া MCP সার্ভার | API সেন্টার গভর্নেন্স, নেটওয়ার্ক আইসোলেশন |
| **MCP10** | কনটেক্সট ইনজেকশন ও অতিরিক্ত শেয়ারিং | ডেটা শ্রেণিবিভাগ, সর্বনিম্ন প্রকাশ |

### MCP প্রমাণীকরণের বিবর্তন

MCP স্পেসিফিকেশন প্রমাণীকরণ এবং অনুমোদনের দৃষ্টিভঙ্গিতে উল্লেখযোগ্য পরিবর্তন এনেছে:

- **প্রাথমিক পন্থা**: প্রাথমিক স্পেসিফিকেশনগুলিতে ডেভেলপারদের কাস্টম প্রমাণীকরণ সার্ভার তৈরি করতে হতো, MCP সার্ভার OAuth 2.0 Authorization Server হিসেবে ব্যবহারকারীর প্রমাণীকরণ সরাসরি পরিচালনা করতো
- **বর্তমান মান (২০২৫-১১-২৫)**: আপডেটকৃত স্পেসিফিকেশন MCP সার্ভারকে বহিরাগত আইডেন্টিটি প্রদানকারীর কাছে প্রমাণীকরণ জমা দিতে দেয় (যেমন Microsoft Entra ID), যা নিরাপত্তা বৃদ্ধি এবং বাস্তবায়ন জটিলতা হ্রাস করে
- **ট্রান্সপোর্ট লেয়ার সিকিউরিটি**: লোকাল (STDIO) এবং রিমোট (Streamable HTTP) সংযোগের জন্য সুশৃঙ্খল প্রমাণীকরণ প্যাটার্ন সহ নিরাপদ পরিবহন ব্যবস্থার উন্নত সমর্থন

## প্রমাণীকরণ ও অনুমোদন নিরাপত্তা

### বর্তমান নিরাপত্তা চ্যালেঞ্জসমূহ

আধুনিক MCP বাস্তবায়নগুলো বেশ কিছু প্রমাণীকরণ এবং অনুমোদন সমস্যার মুখোমুখি:

### ঝুঁকি ও হুমকির উৎস

- **ভুলভাবে কনফিগার করা অনুমোদন লজিক**: MCP সার্ভারে অনুমোদন ত্রুটি সংবেদনশীল ডেটা উন্মোচন এবং ভুলভাবে প্রবেশাধিকার নিয়ন্ত্রণ প্রয়োগ করতে পারে
- **OAuth টোকেন চুরি**: স্থানীয় MCP সার্ভারের টোকেন চুরি করলে আক্রমণকারীরা সার্ভারদের ছদ্মবেশে প্রবেশ করে ডাউনস্ট্রীম সেবাগুলোতে অ্যাক্সেস পেতে পারে
- **টোকেন পাসথ্রু দুর্বলতা**: অনিয়মিত টোকেন পরিচালনা নিরাপত্তা নিয়ন্ত্রণ ফাঁকি এবং জবাবদিহিতা ফাঁকা সৃষ্টি করে
- **অতিরিক্ত অনুমতি**: MCP সার্ভারগুলোর অতিরিক্ত অধিকৃত প্রবেশাধিকার সর্বনিম্ন বিশেষাধিকার নীতি লঙ্ঘন করে আক্রমণের ক্ষেত্র বাড়ায়

#### টোকেন পাসথ্রু: একটি গুরুত্বপূর্ণ বিরুপ প্যাটার্ন

বর্তমান MCP অনুমোদন স্পেসিফিকেশন অনুসারে **টোকেন পাসথ্রু কঠোরভাবে নিষিদ্ধ**, কারণ এর গুরুতর নিরাপত্তা প্রভাব রয়েছে:

##### নিরাপত্তা নিয়ন্ত্রণ এড়ানো
- MCP সার্ভার এবং ডাউনস্ট্রীম API গুলো গুরুত্বপূর্ণ নিরাপত্তা নিয়ন্ত্রণ (রেট লিমিটিং, রিকোয়েস্ট যাচাই, ট্রাফিক মনিটরিং) প্রয়োগ করে যা সঠিক টোকেন যাচাইয়ের ওপর নির্ভরশীল
- সরাসরি ক্লায়েন্ট-টু-API টোকেন ব্যবহার এই অপরিহার্য সুরক্ষাগুলো বাইপাস করে নিরাপত্তা স্থাপত্য দুর্বল করে

##### জবাবদিহিতা ও নিরীক্ষা চ্যালেঞ্জ  
- MCP সার্ভার ক্লায়েন্টদের পার্থক্য করতে পারে না যারা আপস্ট্রিম-ইস্যু করা টোকেন ব্যবহার করছে, ফলে নিরীক্ষার রেকর্ড ভেঙে যায়
- ডাউনস্ট্রীম রিসোর্স সার্ভার লগে প্রকৃত MCP সার্ভার মধ্যস্থতা নয়, ভুল রিকোয়েস্ট উৎস প্রদর্শিত হয়
- ঘটনা তদন্ত এবং সামঞ্জস্য নিরীক্ষা কঠিন হয়

##### ডেটা নির্বাহী ঝুঁকি
- অবৈধ টোকেন দাবিগুলো খারাপ উদ্দেশ্যের কারীদের MCP সার্ভারকে ডেটা নির্বাহের প্রক্সি হিসেবে ব্যবহার করতে দেয়
- বিশ্বাস সীমানা লঙ্ঘন করে অনুমোদিত প্রবেশাধিকারে বাইপাস সম্ভাবনা তৈরি করে

##### বহু-সেবা আক্রমণ রুট
- চুরি করা টোকেন অনেক সেবায় গ্রহণযোগ্য হলে সিস্টেমগুলোর মধ্যে পাশাপাশির গতিশীল আক্রমণ সম্ভব হয়
- টোকেন উৎস যাচাইয়ের অভাব সেবাগুলোর মধ্যে বিশ্বাসের ভিত্তি নষ্ট করে

### নিরাপত্তা নিয়ন্ত্রণ ও প্রশমন

**গুরুত্বপূর্ণ নিরাপত্তা শর্তাবলী:**

> **আবশ্যক**: MCP সার্ভার **একেবারেই** এমন কোনো টোকেন গ্রহণ করবে না যা স্পষ্টভাবে MCP সার্ভারের জন্য ইস্যু করা হয়নি

#### প্রমাণীকরণ ও অনুমোদন নিয়ন্ত্রণ

- **কঠোর অনুমোদন পর্যালোচনা**: MCP সার্ভার অনুমোদন লজিকের পূর্ণাঙ্গ নিরীক্ষা পরিচালনা করুন যাতে শুধুমাত্র উদ্দেশ্যপ্রণীত ব্যবহারকারীরা সংবেদনশীল সম্পদে প্রবেশ করতে পারে
  - **বাস্তুতায়ন গাইড**: [MCP সার্ভারের জন্য Azure API Management কে Authentication Gateway হিসাবে ব্যবহার](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **আইডেন্টিটি ইন্টিগ্রেশন**: [MCP সার্ভার প্রমাণীকরণের জন্য Microsoft Entra ID ব্যবহার](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **নিরাপদ টোকেন ব্যবস্থাপনা**: [Microsoft-এর টোকেন যাচাই এবং লাইফসাইকেল সেরা অনুশীলন](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens) প্রয়োগ করুন
  - টোকেন দর্শক দাবিগুলো MCP সার্ভারের পরিচয়ের সাথে মিল আছে কিনা যাচাই করুন
  - সঠিক টোকেন ঘূর্ণন এবং মেয়াদ উত্তীর্ণ নীতি প্রয়োগ করুন
  - টোকেন পুনঃপ্রচলন আক্রমণ এবং অননুমোদিত ব্যবহার প্রতিরোধ করুন

- **রক্ষিত টোকেন সংরক্ষণ**: বিশ্রাম এবং ট্রানজিটে এনক্রিপশনের মাধ্যমে টোকেন সংরক্ষণ সুরক্ষিত করুন
  - **সেরা অনুশীলন**: [সিকিউর টোকেন স্টোরেজ ও এনক্রিপশন নির্দেশিকা](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### প্রবেশাধিকার নিয়ন্ত্রণ বাস্তবায়ন

- **সর্বনিম্ন বিশেষাধিকার নীতি**: MCP সার্ভারকে শুধুমাত্র প্রয়োজনীয় সর্বনিম্ন অনুমতি প্রদান করুন
  - নিয়মিত অনুমতি পর্যালোচনা ও আপডেট করে বিশেষাধিকার বৃদ্ধি রোধ করুন
  - **Microsoft ডকুমেন্টেশন**: [নিরাপদ সর্বনিম্ন বিশেষাধিকার প্রবেশাধিকার](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **রোল-ভিত্তিক প্রবেশাধিকার নিয়ন্ত্রণ (RBAC)**: সূক্ষ্ম-গ্র্যানুলার রোল নিয়োগ করুন
  - রোলগুলো নির্দিষ্ট সম্পদ ও ক্রিয়াগুলোর জন্য কসরত করে নির্দিষ্ট করুন
  - বিস্তৃত বা অপ্রয়োজনীয় অনুমতি যা আক্রমণ ক্ষেত্র বৃদ্ধি করে তা এড়িয়ে চলুন

- **সতত প্রবেশাধিকার মনিটরিং**: অব্যাহত প্রবেশাধিকার নিরীক্ষা এবং মনিটরিং প্রয়োগ করুন
  - অনুমতি ব্যবহারের নিয়মিত প্যাটার্ন বিশ্লেষণ করুন অসঙ্গতি সনাক্ত করতে
  - অতিরিক্ত বা অনাবশ্যক অনুমতি দ্রুত সংশোধন করুন

## AI-নির্দিষ্ট নিরাপত্তা হুমকি

### প্রম্পট ইনজেকশন ও টুল ম্যানিপুলেশন আক্রমণ

আধুনিক MCP বাস্তবায়নগুলো উন্নত AI-নির্দিষ্ট আক্রমণ পথের সম্মুখীন, যা ঐতিহ্যবাহী নিরাপত্তা ব্যবস্থা পুরোপুরি মোকাবিলা করতে পারে না:

#### **পরোক্ষ প্রম্পট ইনজেকশন (ক্রস-ডোমেইন প্রম্পট ইনজেকশন)**

**পরোক্ষ প্রম্পট ইনজেকশন** MCP সক্রিয় AI সিস্টেমের সবচেয়ে গুরুতর দুর্বলতাগুলোর একটি। আক্রমণকারীরা বাইরের কনটেন্ট—ডকুমেন্ট, ওয়েব পেজ, ইমেল অথবা ডেটা উৎসের মধ্যে ক্ষতিকর নির্দেশাবলী ছড়িয়ে দিয়ে AI সিস্টেমকে বৈধ কমান্ড হিসেবে সেটি প্রক্রিয়া করতে বাধ্য করে।

**আক্রমণ পরিস্থিতি:**
- **ডকুমেন্ট-ভিত্তিক ইনজেকশন**: প্রক্রিয়াজাত ডকুমেন্টে লুকানো ক্ষতিকর নির্দেশাবলী যা AI তেই ভুল আচরণ চালায়
- **ওয়েব কনটেন্ট শোষণ**: ক্ষতিগ্রস্ত ওয়েব পেজে এমবেডেড প্রম্পট যা স্ক্র্যাপ করার সময় AI আচরণ নিয়ন্ত্রণ করে
- **ইমেল-ভিত্তিক আক্রমণ**: ইমেলে ক্ষতিকর প্রম্পট যা AI অ্যাসিস্ট্যান্টকে তথ্য ফাঁস বা অননুমোদিত কাজ করতে বাধ্য করে
- **ডেটা উৎস দূষণ**: দূষিত ডেটাবেস বা API যা AI সিস্টেমকে দূষিত কনটেন্ট সরবরাহ করে

**বাস্তব প্রভাব**: এই আক্রমণগুলো ডেটা নির্বাহ, গোপনীয়তা লঙ্ঘন, ক্ষতিকর কনটেন্ট উত্পাদন, এবং ব্যবহারকারী ইন্টারঅ্যাকশনের ম্যানিপুলেশন ঘটাতে পারে। বিস্তারিত বিশ্লেষণের জন্য দেখুন [MCP তে প্রম্পট ইনজেকশন (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/)।

![Prompt Injection Attack Diagram](../../../translated_images/bn/prompt-injection.ed9fbfde297ca877.webp)

#### **টুল পয়জনিং আক্রমণ**

**টুল পয়জনিং** MCP টুল সংজ্ঞায়িত মেটাডেটাকে লক্ষ্য করে, যেখানে LLM গুলো টুল বর্ণনা এবং প্যারামিটার বিশ্লেষণ করে নির্বাহ সিদ্ধান্ত নেয়।

**আক্রমণ পদ্ধতি:**
- **মেটাডেটা ম্যানিপুলেশন**: আক্রমণকারীরা টুল বর্ণনা, প্যারামিটার সংজ্ঞা, বা ব্যবহার উদাহরণে ক্ষতিকর নির্দেশাবলী ইনজেক্ট করে
- **অদৃশ্য নির্দেশাবলী**: টুল মেটাডেটায় লুকানো প্রম্পট যা AI মডেল প্রসেস করে কিন্তু মানুষের কাছে অদৃশ্য
- **গতিশীল টুল পরিবর্তন ("রাগ পুল")**: ব্যবহারকারীরা অনুমোদিত টুল পরে ক্ষতিকর কাজ করতে পরিবর্তিত হয়, ব্যবহারকারী অজানা থাকে
- **প্যারামিটার ইনজেকশন**: মডেল আচরণ প্রভাবিত করতে টুল প্যারামিটার স্কিমায় ক্ষতিকর কনটেন্ট এমবেড করা

**হোস্টেড সার্ভার ঝুঁকি**: দূরবর্তী MCP সার্ভারগুলিতে টুল সংজ্ঞাগুলো ব্যবহারকারী অনুমোদনের পরে আপডেট হতে পারে, ভবিষ্যতে নিরাপদ টুলগুলো ক্ষতিকর হয়ে উঠতে পারে। বিস্তৃত বিশ্লেষণের জন্য দেখুন [টুল পয়জনিং আক্রমণ (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks)।

#### **অতিরিক্ত AI আক্রমণ পথ**

- **ক্রস-ডোমেইন প্রম্পট ইনজেকশন (XPIA)**: একাধিক ডোমেইনের কনটেন্ট ব্যবহার করে জটিল আক্রমণ যা নিরাপত্তা নিয়ন্ত্রণ পাস করে
- **ডায়নামিক সক্ষমতা পরিবর্তন**: সরঞ্জাম সক্ষমতাগুলির রিয়েল-টাইম পরিবর্তন যা প্রাথমিক নিরাপত্তা মূল্যায়ন এড়িয়ে যায়  
- **সন্দর্ভ জানালা বিষক্রিয়া**: বড় বড় প্রসঙ্গ জানালাগুলিকে শত্রুতাপূর্ণ নির্দেশাবলী লুকাতে পরিচালনা করার আক্রমণ  
- **মডেল বিভ্রান্তিকর আক্রমণ**: মডেলের সীমাবদ্ধতাকে কাজে লাগিয়ে অপ্রত্যাশিত বা অসুরক্ষিত আচরণ তৈরি করা  

### AI নিরাপত্তা ঝুঁকি প্রভাব

**উচ্চ-প্রভাব পরিণতি:**  
- **ডেটা চুরি**: অবৈধ প্রবেশাধিকার এবং সংবেদনশীল এন্টারপ্রাইজ বা ব্যক্তিগত ডেটার চুরি  
- **গোপনীয়তা লঙ্ঘন**: ব্যক্তিগত সনাক্তকরণযোগ্য তথ্য (PII) এবং গোপনীয় ব্যবসায়িক ডেটার প্রকাশ  
- **সিস্টেম নিয়ন্ত্রণ অপব্যবহার**: গুরুত্বপূর্ণ সিস্টেম ও কর্মপ্রবাহের অনিচ্ছাকৃত পরিবর্তন  
- **প্রমাণপত্র চুরি**: প্রমাণীকরণ টোকেন এবং সার্ভিস প্রমাণপত্রের অবনতি  
- **পাশের গতি**: বিস্তৃত নেটওয়ার্ক আক্রমণের জন্য দোষগ্রস্ত AI সিস্টেমগুলিকে কেন্দ্র হিসেবে ব্যবহার করা  

### Microsoft AI নিরাপত্তা সমাধান

#### **AI প্রম্পট শিল্ডস: ইনজেকশন আক্রমণের বিরুদ্ধে উন্নত সুরক্ষা**

Microsoft **AI প্রম্পট শিল্ডস** সরাসরি ও পরোক্ষ উভয় প্রম্পট ইনজেকশন আক্রমণের বিরুদ্ধে বহুস্তরের নিরাপত্তার মাধ্যমে ব্যাপক সুরক্ষা প্রদান করে:

##### **মূল সুরক্ষা পদ্ধতিসমূহ:**

1. **উন্নত সনাক্তকরণ ও ফিল্টারিং**  
   - বাহ্যিক বিষয়বস্ত্রে শত্রুতাপূর্ণ নির্দেশাবলী সনাক্ত করতে মেশিন লার্নিং অ্যালগরিদম এবং NLP কৌশল  
   - নথিপত্র, ওয়েব পেজ, ইমেল এবং তথ্য উৎসগুলির বাস্তব-সময়ের বিশ্লেষণ দ্বারা সন্নিবেশিত ঝুঁকি নির্ণয়  
   - বৈধ বনাম শত্রুতাপূর্ণ প্রম্পট প্যাটার্নের প্রসঙ্গভিত্তিক বোঝাপড়া  

2. **স্পটলাইটিং কৌশল**  
   - বিশ্বাসযোগ্য সিস্টেম নির্দেশাবলী এবং সম্ভাব্য দোষগ্রস্ত বাহ্যিক ইনপুটের পার্থক্য নির্ধারণ  
   - টেক্সট রূপান্তর পদ্ধতি যা মডেলের প্রাসঙ্গিকতা বৃদ্ধি করে এবং শত্রুতাপূর্ণ বিষয়বস্তু আলাদা করে  
   - AI সিস্টেমকে সঠিক নির্দেশনা স্তর বজায় রাখতে সহায়তা এবং ইনজেক্টেড কমান্ড উপেক্ষা করা  

3. **ডেলিমিটার ও ডেটামার্কিং সিস্টেম**  
   - বিশ্বাসযোগ্য সিস্টেম মেসেজ এবং বাহ্যিক ইনপুট টেক্সটের মধ্যে স্পষ্ট সীমানা নির্ধারণ  
   - বিশেষ মার্কার যা বিশ্বাসযোগ্য এবং অবিশ্বস্ত ডেটা উৎসের সীমানা হাইলাইট করে  
   - পরিষ্কার পৃথকীকরণ নির্দেশনা বিভ্রান্তি ও অননুমোদিত কমান্ড কার্যকর হওয়া রোধ করে  

4. **নিরবিচ্ছিন্ন হুমকি বুদ্ধিমত্তা**  
   - Microsoft ক্রমাগত নতুন আক্রমণ প্যাটার্ন পর্যবেক্ষণ ও প্রতিরক্ষা আপডেট করে  
   - নতুন ইনজেকশন কৌশল এবং আক্রমণ ভেক্টরগুলির জন্য প্রোঅ্যাকটিভ হান্টিং  
   - বিকাশমান হুমকির বিরুদ্ধে কার্যকারিতা বজায় রাখতে নিয়মিত নিরাপত্তা মডেল আপডেট  

5. **Azure কন্টেন্ট সেফটি ইন্টিগ্রেশন**

## উন্নত MCP নিরাপত্তা হুমকি

### সেশন ক্যাপচার দুর্বলতা

**সেশন ক্যাপচার** একটি গুরুত্বপূর্ণ আক্রমণ ভেক্টর, যেখানে অবৈধ পক্ষসমূহ বৈধ সেশন সনাক্তকারী অর্জন ও অপব্যবহার করে ক্লায়েন্টের ছদ্মবেশ গ্রহণ এবং অননুমোদিত কার্য সম্পাদন করে।

#### **আক্রমণ দৃশ্যপট ও ঝুঁকি**

- **সেশন ক্যাপচার প্রম্পট ইনজেকশন**: চুরিকৃত সেশন আইডি ব্যবহার করে আক্রমণকারী শেয়ার করা সেশন স্টেট সহ সার্ভারে শত্রুতাপূর্ণ ইভেন্ট ইনজেক্ট করতে পারে, যা ক্ষতিকর কার্যাবলী বা সংবেদনশীল ডেটা অ্যাক্সেস করতে পারে  
- **সরাসরি ছদ্মবেশ**: চুরিকৃত সেশন আইডি দিয়ে সরাসরি MCP সার্ভার কল করা যায় যা প্রমাণীকরণ এড়িয়ে যায়, আক্রমণকারীকে বৈধ ব্যবহারকারীর মতো আচরণ করায়  
- **দোষগ্রস্ত পুনরাবৃত্ত স্ট্রিমস**: আক্রমণকারী অমান্য অনুরোধ বাতিল করতে পারে, বৈধ ক্লায়েন্টদের সম্ভাব্য শত্রুতাপূর্ণ বিষয়বস্তু নিয়ে পুনরায় শুরু করার কারণ সৃষ্টি করে  

#### **সেশন ব্যবস্থাপনার জন্য নিরাপত্তা নিয়ন্ত্রণ**

**গুরুত্বপূর্ণ চাহিদা:**  
- **অনুমোদন যাচাইকরণ**: MCP সার্ভারসমূহে প্রত্যেক ইনবাউন্ড অনুরোধ যাচাই করা অবশ্যক এবং প্রমাণীকরণের জন্য সেশন নির্ভর করা যাবে না  
- **নিরাপদ সেশন তৈরি**: ক্রিপ্টোগ্রাফিক্যালি সুরক্ষিত, অপ্রত্যাশিত সেশন আইডি ব্যবহার করুন যা নিরাপদ র্যান্ডম নম্বর জেনারেটর দ্বারা তৈরি  
- **ব্যবহারকারী-নির্দিষ্ট সংযোজন**: সেশন আইডি ব্যবহারকারী নির্দিষ্ট তথ্যের সাথে বেঁধে দিন (যেমন `<user_id>:<session_id>`) যাতে পারস্পরিক সেশন অপব্যবহার বাধা দেয়  
- **সেশন জীবনচক্র পরিচালনা**: সঠিক মেয়াদ উত্তীর্ণকরণ, ঘোরতর এবং অবৈধকরণের মাধ্যমে ঝুঁকি সীমিত করুন  
- **পরিবহন নিরাপত্তা**: সকল যোগাযোগে HTTPS বাধ্যতামূলক করা যাতে সেশন আইডি ইন্টারসেপ্ট না হয়  

### বিভ্রান্ত দপ্তর সমস্যা

**বিভ্রান্ত দপ্তর সমস্যা** ঘটে যখন MCP সার্ভার ক্লায়েন্ট এবং তৃতীয় পক্ষের পরিষেবার মধ্যে প্রমাণীকরণ প্রক্সি হিসেবে কাজ করে, স্থিতিশীল ক্লায়েন্ট আইডি ব্যবহার করে অনুমোদন এড়ানোর সুযোগ তৈরি হয়।

#### **আক্রমণ প্রক্রিয়া ও ঝুঁকি**

- **কুকি-ভিত্তিক সম্মতি এড়ানো**: পূর্বের ব্যবহারকারী প্রমাণীকরণ সৃষ্ট সম্মতি কুকি আক্রমণকারী দ্বারা তৈরি কৃত্রিম রিডাইরেক্ট URI সহ শত্রুতাপূর্ণ অনুমোদন অনুরোধে ব্যবহার হয়  
- **অনুমোদন কোড চুরি**: বিদ্যমান সম্মতি কুকিগুলি অনুমোদন সার্ভারকে সম্মতি স্ক্রিন এড়িয়ে যাওয়ার জন্য বাধ্য করতে পারে এবং কোড আক্রমণকারী নিয়ন্ত্রিত এন্ডপয়েন্টে পাঠায়  
- **অননুমোদিত API অ্যাক্সেস**: চুরি করা অনুমোদন কোড ব্যবহার করে টোকেন পরিবর্তন এবং ব্যবহারকারী ছদ্মবেশ অনুমোদন ব্যতীত সম্ভব  

#### **উপশম কৌশল**

**আবশ্যিক নিয়ন্ত্রণসমূহ:**  
- **স্পষ্ট সম্মতি প্রয়োজন**: স্থির ক্লায়েন্ট আইডি সহ MCP প্রক্সি সার্ভারগুলোকে প্রত্যেক গতিশীলভাবে নিবন্ধিত ক্লায়েন্টের জন্য ব্যবহারকারীর সম্মতি গ্রহণ করতে হবে  
- **OAuth 2.1 নিরাপত্তা বাস্তবায়ন**: সমস্ত অনুমোদন অনুরোধের জন্য PKCE (প্রুফ কি ফর কোড এক্সচেঞ্জ) সহ আধুনিক OAuth নিরাপত্তা অনুশীলন অনুসরণ  
- **ক্লায়েন্ট কঠোর যাচাইকরণ**: রিডাইরেক্ট URI এবং ক্লায়েন্ট শনাক্তকরণের কঠোর যাচাইকরণ প্রয়োগ করা যাতে অপব্যবহার প্রতিরোধ হয়  

### টোকেন পাসথ্রু দুর্বলতা  

**টোকেন পাসথ্রু** একটি স্পষ্ট বিরোধী-প্যাটার্ন যেখানে MCP সার্ভার ক্লায়েন্ট টোকেন যথাযথ যাচাইকরণ ছাড়াই গ্রহণ করে এবং তা নীচের API-তে ফরওয়ার্ড করে, যা MCP অনুমোদন স্পেসিফিকেশন লঙ্ঘন করে।

#### **নিরাপত্তা প্রভাব**

- **নিয়ন্ত্রণ এড়ানো**: ক্লায়েন্ট-থেকে-API সরাসরি টোকেন ব্যবহারে গুরুত্বপূর্ণ রেট লিমিটিং, যাচাই এবং পর্যবেক্ষণ নিয়ন্ত্রণ এড়ানো হয়  
- **অডিট ট্রেইল নষ্ট**: আপস্ট্রিম ইস্যুকৃত টোকেনগুলি ক্লায়েন্ট সনাক্তকরণ অক্ষম করে, ফলে ঘটনা তদন্ত ব্যাহত হয়  
- **প্রক্সি দ্বারা ডেটা চুরি**: অবৈধ টোকেন দ্বারা দুর্বৃত্তরা সার্ভারকে অননুমোদিত ডেটা অ্যাক্সেসের জন্য প্রক্সি হিসেবে ব্যবহার করতে পারে  
- **ট্রাস্ট বাউন্ডারি লঙ্ঘন**: টোকেন উৎস যাচাই না পেলে নীচের পরিষেবা গুলোর বিশ্বাসযোগ্যতা লঙ্ঘিত হতে পারে  
- **বহু-সার্ভিস আক্রমণ সম্প্রসারণ**: একাধিক পরিষেবা জুড়ে দোষগ্রস্ত টোকেন গ্রহণের মাধ্যমে পাশের গতি অর্জন সম্ভব  

#### **দরকারি নিরাপত্তা নিয়ন্ত্রণ**

**অমীমাংসিত বাধ্যবাধকতা:**  
- **টোকেন যাচাইকরণ**: MCP সার্ভার কখনোই এমন টোকেন গ্রহণ করবে না যা স্পষ্টভাবে MCP সার্ভারের জন্য ইস্যু করা হয়নি  
- **অডিয়েন্স যাচাইকরণ**: সর্বদা টোকেন অডিয়েন্স দাবি MCP সার্ভারের পরিচয়ের সাথে মেলে কিনা যাচাই করা  
- **নিয়মিত টোকেন জীবনচক্র ব্যবস্থাপনা**: সুরক্ষিত ঘোরতর প্রক্রিয়া সহ স্বল্পমেয়াদী অ্যাক্সেস টোকেন প্রয়োগ  

## AI সিস্টেমের জন্য সাপ্লাই চেইন নিরাপত্তা

সাপ্লাই চেইন নিরাপত্তা ঐতিহ্যগত সফটওয়্যার নির্ভরশীলতার বাইরে গিয়ে সম্পূর্ণ AI ইকোসিস্টেমকে অন্তর্ভুক্ত করেছে। আধুনিক MCP বাস্তবায়নগুলোকে প্রতিটি AI-সম্পর্কিত উপাদান কঠোরভাবে যাচাই ও পর্যবেক্ষণ করতে হবে, কারণ প্রত্যেকটি উপাদান সম্ভাব্য দুর্বলতা সৃষ্টি করতে পারে যা সিস্টেমের অখণ্ডতা ক্ষুন্ন করতে পারে।

### বিস্তৃত AI সাপ্লাই চেইন উপাদানসমূহ

**ঐতিহ্যগত সফটওয়্যার নির্ভরশীলতা:**  
- ওপেন-সোর্স লাইব্রেরি ও ফ্রেমওয়ার্ক  
- কনটেইনার ইমেজ ও বেস সিস্টেম  
- ডেভেলপমেন্ট টুলস ও বিল্ড পাইপলাইন  
- অবকাঠামো উপাদান ও সার্ভিস  

**AI-নির্দিষ্ট সাপ্লাই চেইন উপাদান:**  
- **ফাউন্ডেশন মডেল**: বিভিন্ন প্রদানকারীর প্রিপ্রেইনড মডেল যার উৎস যাচাই প্রয়োজন  
- **এম্বেডিং সার্ভিসেস**: বাহ্যিক ভেক্টরাইজেশন ও সিমান্তিক অনুসন্ধান সার্ভিস  
- **সন্দর্ভ প্রদানকারীরা**: ডেটা উৎস, জ্ঞানের ভাণ্ডার, এবং ডকুমেন্ট রিপোজিটরি  
- **তৃতীয় পক্ষের API**: বাহ্যিক AI সার্ভিস, ML পাইপলাইন, এবং ডেটা প্রক্রিয়াকরণ এন্ডপয়েন্ট  
- **মডেল আর্টিফ্যাক্টস**: ওজন, কনফিগারেশন এবং ফাইন-টিউনড মডেল বৈচিত্র্য  
- **ট্রেনিং ডেটা উৎস**: মডেল প্রশিক্ষণ ও ফাইন-টিউনিংয়ের জন্য ব্যবহৃত ডেটাসেট  

### ব্যাপক সাপ্লাই চেইন নিরাপত্তা কৌশল

#### **উপাদান যাচাইকরণ ও বিশ্বাসযোগ্যতা**  
- **উৎস যাচাই**: সব AI উপাদানের উৎস, লাইসেন্স ও অখণ্ডতা একীভূত করার আগে যাচাই করা  
- **নিরাপত্তা মূল্যায়ন**: মডেল, ডেটা উৎস এবং AI সার্ভিসের জন্য দুর্বলতা স্ক্যান এবং নিরাপত্তা পর্যালোচনা করা  
- **নামজাদা বিশ্লেষণ**: AI সার্ভিস প্রদানকারীর নিরাপত্তা রেকর্ড ও অনুশীলন মূল্যায়ন  
- **অনুমোদনের নিশ্চয়তা**: সব উপাদান সংস্থার নিরাপত্তা ও নিয়ন্ত্রণবিধির সঙ্গে সামঞ্জস্যপূর্ণ কিনা নিশ্চিত করা  

#### **নিরাপদ ডিপ্লয়মেন্ট পাইপলাইন**  
- **স্বয়ংক্রিয় CI/CD নিরাপত্তা**: স্বয়ংক্রিয় ডিপ্লয়মেন্ট পাইপলাইনে নিরাপত্তা স্ক্যানিং একত্রিতকরণ  
- **আর্টিফ্যাক্ট অখণ্ডতা**: সমস্ত ডিপ্লয়ড আর্টিফ্যাক্ট (কোড, মডেল, কনফিগারেশন) ক্রিপ্টোগ্রাফিক যাচাইকরণ  
- **ধাপ-ধাপে ডিপ্লয়মেন্ট**: প্রতিটি ধাপে নিরাপত্তা যাচাই সহ প্রগতিশীল ডিপ্লয়মেন্ট কৌশল ব্যবহার  
- **বিশ্বাসযোগ্য আর্টিফ্যাক্ট রিপোজিটরি**: যাচাইকৃত, নিরাপদ আর্টিফ্যাক্ট রিপোজিটরি থেকে শুধুমাত্র ডিপ্লয়মেন্ট  

#### **নিরবিচ্ছিন্ন পর্যবেক্ষণ ও প্রতিক্রিয়া**  
- **নির্ভরতামূলক স্ক্যানিং**: সব সফটওয়্যার এবং AI উপাদান নির্ভরতায় ক্রমাগত দুর্বলতা নজরদারি  
- **মডেল পর্যবেক্ষণ**: মডেল আচরণ, পারফরম্যান্স বিচ্যুতি এবং নিরাপত্তা অস্বাভাবিকতা ক্রমাগত মূল্যায়ন  
- **সার্ভিস সুস্থতা ট্র্যাকিং**: বাহ্যিক AI সার্ভিসের উপলব্ধতা, নিরাপত্তা ঘটনা ও নীতি পরিবর্তন মনিটরিং  
- **হুমকি বুদ্ধিমত্তা সংযোজন**: AI এবং ML নিরাপত্তা ঝুঁকি সংশ্লিষ্ট হুমকি ফিড অন্তর্ভুক্তি  

#### **অ্যাক্সেস নিয়ন্ত্রণ ও ন্যূনতম প্রয়োজনীয়তা**  
- **উপাদান-স্তর অনুমতি**: ব্যবসায়িক প্রয়োজন অনুযায়ী মডেল, ডেটা এবং সার্ভিসের অ্যাক্সেস সীমাবদ্ধ করা  
- **সার্ভিস অ্যাকাউন্ট ব্যবস্থাপনা**: সরিবদ্ধ হ্রাসকৃত অনুমতি সহ ডেডিকেটেড সার্ভিস অ্যাকাউন্ট প্রয়োগ  
- **নেটওয়ার্ক বিভাজন**: AI উপাদান পৃথক করার ও সার্ভিসের মধ্যে নেটওয়ার্ক অ্যাক্সেস সীমাবদ্ধ করা  
- **API গেটওয়ে নিয়ন্ত্রণ**: বাহ্যিক AI সার্ভিসে অ্যাক্সেস নিয়ন্ত্রণ এবং মনিটর করার কেন্দ্রীভূত API গেটওয়ে ব্যবহার  

#### **ঘটনা প্রতিক্রিয়া ও পুনরুদ্ধার**  
- **দ্রুত প্রতিক্রিয়া পদ্ধতি**: দোষগ্রস্ত AI উপাদান প্যাচ বা প্রতিস্থাপন করার প্রতিষ্ঠিত প্রক্রিয়া  
- **প্রমাণপত্র ঘোরতর**: সিক্রেট, API কী এবং সার্ভিস প্রমাণপত্র স্বয়ংক্রিয় ঘোরতর সিস্টেম  
- **পূর্ববর্তী স্থিতিশীল সংস্করণে রোলব্যাক**: দ্রুত সনাক্ত করা ভাল সংস্করণে AI উপাদান পুনরায় ফিরিয়ে আনা সক্ষমতা  
- **সাপ্লাই চেইন লঙ্ঘন পুনরুদ্ধার**: আপস্ট্রিম AI সার্ভিস লঙ্ঘনের প্রতিক্রিয়া জন্য নির্দিষ্ট প্রক্রিয়া

## ফাউন্ডেশন নিরাপত্তা শ্রেষ্ঠ অনুশীলন

MCP বাস্তবায়নগুলি আপনার সংস্থার বিদ্যমান নিরাপত্তা দৃষ্টিভঙ্গি উত্তরাধিকারসূত্রে গ্রহণ ও ভিত্তি করে গড়ে তোলে। মৌলিক নিরাপত্তা অনুশীলন শক্তিশালীকরণ AI সিস্টেম ও MCP ডিপ্লয়মেন্টের সামগ্রিক নিরাপত্তা উল্লেখযোগ্যভাবে বৃদ্ধি করে।

### মূল নিরাপত্তা মৌলিক মূলনীতি

#### **নিরাপদ ডেভেলপমেন্ট অনুশীলন**  
- **OWASP অনুসরণ**: [OWASP টপ ১০](https://owasp.org/www-project-top-ten/) ওয়েব অ্যাপ্লিকেশন দুর্বলতার বিরুদ্ধে সুরক্ষা  
- **AI-নির্দিষ্ট সুরক্ষা**: [OWASP Top 10 for LLMs](https://genai.owasp.org/download/43299/?tmstv=1731900559) এর জন্য নিয়ন্ত্রণ প্রয়োগ  
- **নিরাপদ সিক্রেট ব্যবস্থাপনা**: টোকেন, API কী এবং সংবেদনশীল কনফিগারেশন ডেটার জন্য নির্দিষ্ট ভল্ট ব্যবহার  
- **এন্ড-টু-এন্ড এনক্রিপশন**: সব অ্যাপ্লিকেশন উপাদান ও ডেটা প্রবাহে নিরাপদ যোগাযোগ বাস্তবায়ন  
- **ইনপুট যাচাইকরণ**: সব ব্যবহারকারী ইনপুট, API প্যারামিটার এবং ডেটা উৎসের কঠোর যাচাইকরণ  

#### **অবকাঠামো কঠোরতা**  
- **মাল্টি-ফ্যাক্টর প্রমাণীকরণ**: সব প্রশাসনিক ও সার্ভিস অ্যাকাউন্টের জন্য বাধ্যতামূলক MFA  
- **প্যাচ ব্যবস্থাপনা**: অপারেটিং সিস্টেম, ফ্রেমওয়ার্ক ও নির্ভরশীলতার জন্য স্বয়ংক্রিয় এবং সঠিক সময়মত প্যাচিং  
- **পরিচয় প্রদানকারী সংযুক্তকরণ**: এন্টারপ্রাইজ পরিচয় প্রদানকারীর মাধ্যমে কেন্দ্রীভূত পরিচয় ব্যবস্থাপনা (Microsoft Entra ID, Active Directory)  
- **নেটওয়ার্ক বিভাজন**: পাশের গতি সীমাবদ্ধ করতে MCP উপাদানগুলোর যৌক্তিক পৃথকীকরণ  
- **ন্যূনতম প্রিভিলেজ নীতি**: সব সিস্টেম উপাদান ও অ্যাকাউন্টের জন্য সর্বনিম্ন প্রয়োজনীয় অনুমতি  

#### **নিরাপত্তা পর্যবেক্ষণ ও সনাক্তকরণ**  
- **পূর্ণাঙ্গ লগিং**: AI অ্যাপ্লিকেশন কার্যকলাপের বিস্তারিত লগিং, MCP ক্লায়েন্ট-সার্ভার যোগাযোগসহ  
- **SIEM সংযোগ**: কেন্দ্রিভূত নিরাপত্তা তথ্য ও ঘটনা ব্যবস্থাপনা অস্বাভাবিকতা সনাক্তকরণের জন্য  
- **আচরণ বিশ্লেষণ**: ব্যবস্থায় অস্বাভাবিক প্যাটার্ন সনাক্ত করতে AI চালিত পর্যবেক্ষণ  
- **হুমকি বুদ্ধিমত্তা**: বাহ্যিক হুমকি ফিড এবং দায়িত্বজ্ঞাপন সূচকের সংযোজন  
- **ঘটনা প্রতিক্রিয়া**: নিরাপত্তা ঘটনা সনাক্তকরণ, প্রতিক্রিয়া এবং পুনরুদ্ধারের জন্য সুপরিকল্পিত প্রক্রিয়া  

#### **জিরো ট্রাস্ট আর্কিটেকচার**  
- **কখনই বিশ্বাস করবেন না, সর্বদা যাচাই করুন**: ব্যবহারকারী, ডিভাইস ও নেটওয়ার্ক সংযোগের অবিরাম যাচাইকরণ  
- **মাইক্রো-বিভাজন**: স্বতন্ত্র ওয়ার্কলোড ও সার্ভিস আলাদা করার সূক্ষ্ম নেটওয়ার্ক নিয়ন্ত্রণ  
- **পরিচয়-কেন্দ্রিক নিরাপত্তা**: নেটওয়ার্ক অবস্থানের বদলে যাচাইকৃত পরিচয়ের উপর ভিত্তি করে নিরাপত্তা নীতি  
- **নিরবিচ্ছিন্ন ঝুঁকি মূল্যায়ন**: বর্তমান প্রসঙ্গ এবং আচরণ অনুযায়ী গতিশীল নিরাপত্তা অবস্থা মূল্যায়ন  
- **শর্তসাপেক্ষ অ্যাক্সেস**: ঝুঁকি, অবস্থান এবং ডিভাইস বিশ্বাসের উপর ভিত্তি করে অভিযোজিত প্রবেশ নিয়ন্ত্রণ  

### এন্টারপ্রাইজ সংযোগ প্যাটার্ন

#### **Microsoft নিরাপত্তা ইকোসিস্টেম সংযোগ**  
- **Microsoft Defender for Cloud**: ব্যাপক ক্লাউড নিরাপত্তা অবস্থা ব্যবস্থাপনা  
- **Azure Sentinel**: AI ওয়ার্কলোড সুরক্ষার জন্য ক্লাউড-নেটিভ SIEM এবং SOAR দক্ষতা  
- **Microsoft Entra ID**: শর্তসাপেক্ষ প্রবেশ নীতিসহ এন্টারপ্রাইজ পরিচয় ও অ্যাক্সেস ব্যবস্থাপনা  
- **Azure Key Vault**: হার্ডওয়্যার নিরাপত্তা মডিউল (HSM) সমর্থিত কেন্দ্রীভূত সিক্রেট ব্যবস্থাপনা  
- **Microsoft Purview**: AI ডেটা উৎস ও কর্মপ্রবাহের জন্য ডেটা শাসন ও নিয়ন্ত্রণ  

#### **নিয়মাবলী ও শাসন**  
- **নিয়ন্ত্রক সঙ্গতি**: MCP বাস্তবায়নগুলি শিল্প-নির্দিষ্ট নিয়ম অনুসরণ নিশ্চিত করা (GDPR, HIPAA, SOC 2)  
- **ডেটা শ্রেণীবিভাগ**: AI সিস্টেম দ্বারা প্রক্রিয়াজাত সংবেদনশীল ডেটার যথাযথ শ্রেণীবিভাগ এবং পরিচালনা  
- **অডিট ট্রেইল**: নিয়ন্ত্রক সঙ্গতি ও ফৌজদারী তদন্তের জন্য ব্যাপক লগিং  
- **গোপনীয়তা নিয়ন্ত্রণ**: AI সিস্টেম স্থাপত্যে প্রাইভেসি-বাই-ডিজাইন নীতি প্রয়োগ  
- **পরিবর্তন ব্যবস্থাপনা**: AI সিস্টেম পরিবর্তনের নিরাপত্তা পর্যালোচনার জন্য আনুষ্ঠানিক প্রক্রিয়া  

এই মৌলিক অনুশীলনগুলো একটি শক্তিশালী নিরাপত্তা ভিত্তি গড়ে তোলে যা MCP-নির্দিষ্ট নিরাপত্তা নিয়ন্ত্রণের কার্যকারিতা বাড়ায় এবং AI-চালিত অ্যাপ্লিকেশনগুলোর জন্য ব্যাপক সুরক্ষা প্রদান করে।

## প্রধান নিরাপত্তা শিক্ষণীয় বিষয়সমূহ  

- **স্তরভিত্তিক নিরাপত্তা পন্থা**: মৌলিক নিরাপত্তা অনুশীলনগুলি (নিরাপদ কোডিং, সর্বনিম্ন বিশেষাধিকার, সরবরাহ চেইন যাচাইকরণ, অবিচ্ছিন্ন পর্যবেক্ষণ) এআই-নির্দিষ্ট নিয়ন্ত্রণগুলির সাথে মিলিয়ে ব্যাপক সুরক্ষা প্রদান করুন

- **এআই-নির্দিষ্ট হুমকির পরিবেশ**: MCP সিস্টেমগুলি অনন্য ঝুঁকির সম্মুখীন হয় যেমন প্রম্পট ইঞ্জেকশন, টুল পয়জনিং, সেশন হাইজ্যাকিং, বিভ্রান্ত ডেপুটি সমস্যা, টোকেন পাসথ্রু দুর্বলতা, এবং অতিরিক্ত অনুমতিগুলি যা বিশেষায়িত প্রতিকার প্রয়োজন

- **প্রমাণীকরণ ও অনুমোদনে উৎকৃষ্টতা**: বাহ্যিক পরিচয় প্রদানকারী (Microsoft Entra ID) ব্যবহার করে দৃঢ় প্রমাণীকরণ বাস্তবায়ন করুন, সঠিক টোকেন যাচাইকরণ নিশ্চিত করুন, এবং কখনই আপনার MCP সার্ভারের জন্য স্পষ্টভাবে ইস্যুকৃত নয় এমন টোকেন গ্রহণ করবেন না

- **এআই আক্রমণ প্রতিরোধ**: Microsoft Prompt Shields এবং Azure Content Safety ব্যবহার করে পরোক্ষ প্রম্পট ইঞ্জেকশন এবং টুল পয়জনিং আক্রমণের বিরুদ্ধে রক্ষামূলক ব্যবস্থা গ্রহণ করুন, টুল মেটাডেটা যাচাই করুন এবং গতিশীল পরিবর্তনের জন্য নজরদারি করুন

- **সেশন ও ট্রান্সপোর্ট নিরাপত্তা**: ক্রিপ্টোগ্রাফিক্যালি সুরক্ষিত, অপ্রাকৃততাসংক্রান্ত সেশন আইডি ব্যবহার করুন যা ব্যবহারকারীর পরিচয়ের সাথে যুক্ত, সঠিক সেশন জীবনচক্র ব্যবস্থাপনা প্রয়োগ করুন, এবং কখনও প্রমাণীকরণের জন্য সেশন ব্যবহার করবেন না

- **OAuth নিরাপত্তার সেরা অনুশীলন**: গতিশীলভাবে নিবন্ধিত ক্লায়েন্টদের জন্য স্পষ্ট ব্যবহারকারীর সম্মতি মাধ্যমে বিভ্রান্ত ডেপুটি আক্রমণ প্রতিরোধ করুন, PKCE সহ সঠিক OAuth 2.1 বাস্তবায়ন করুন, এবং কঠোর রিডাইরেক্ট URI যাচাইকরণ নিশ্চিত করুন

- **টোকেন নিরাপত্তা নীতিমালা**: টোকেন পাসথ্রু বিরোধী প্যাটার্নগুলি এড়িয়ে চলুন, টোকেন দর্শকদের দাবি যাচাই করুন, সুরক্ষিত রোটেশন সহ স্বল্পমেয়াদী টোকেন বাস্তবায়ন করুন, এবং স্পষ্ট বিশ্বাস সীমানা বজায় রাখুন

- **ব্যापक সরবরাহ চেইন নিরাপত্তা**: সমস্ত এআই পরিবেশগত উপাদানগুলি (মডেল, এমবেডিংস, প্রসঙ্গ প্রদানকারী, বাহ্যিক API) ঐতিহ্যগত সফ্টওয়্যার নির্ভরতার মতো একই সুরক্ষা কঠোরতার সাথে বিবেচনা করুন

- **অবিচ্ছিন্ন পরিবর্তনশীলতা**: দ্রুত বিকাশমান MCP স্পেসিফিকেশনগুলির সাথে আপডেট থাকুন, নিরাপত্তা সম্প্রদায়ের মানদণ্ডে অবদান রাখুন, এবং প্রোটোকল প্রাপ্তির সাথে নিরাপত্তা অবস্থানগুলি অভিযোজিত রাখুন

- **মাইক্রোসফট নিরাপত্তা সংহতকরণ**: Microsoft-এর ব্যাপক নিরাপত্তা পরিবেশ (Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID) ব্যবহার করে উন্নত MCP স্থাপনার সুরক্ষা নিশ্চিত করুন
