# AGENTS.md

## প্রকল্পের ওভারভিউ

**শুরু করার জন্য MCP** হল একটি ওপেন সোর্স শিক্ষামূলক পাঠ্যক্রম যা মডেল কনটেক্সট প্রটোকল (MCP) শেখার জন্য তৈরি - যা AI মডেল এবং ক্লায়েন্ট অ্যাপ্লিকেশনের মধ্যে ইন্টারঅ্যাকশনের জন্য একটি মানককৃত ফ্রেমওয়ার্ক। এই রেপোজিটরিটি বিভিন্ন প্রোগ্রামিং ভাষায় হাতে কলমে কোড উদাহরণের মাধ্যমে ব্যাপক শেখার উপকরণ সরবরাহ করে।

### প্রধান প্রযুক্তি

- **প্রোগ্রামিং ভাষাসমূহ**: C#, Java, JavaScript, TypeScript, Python, Rust
- **ফ্রেমওয়ার্ক ও SDK**: 
  - MCP SDK (`@modelcontextprotocol/sdk`)
  - Spring Boot (Java)
  - FastMCP (Python)
  - LangChain4j (Java)
- **ডাটাবেস**: PostgreSQL সহ pgvector এক্সটেনশন
- **ক্লাউড প্ল্যাটফর্মস**: Azure (Container Apps, OpenAI, Content Safety, Application Insights)
- **বিল্ড টুলস**: npm, Maven, pip, Cargo
- **ডকুমেন্টেশন**: স্বয়ংক্রিয় বহু-ভাষা অনুবাদের সাথে Markdown (৪৮+ ভাষা)

### আর্কিটেকচার

- **১১টি মূল মডিউল (০০-১১)**: মূল বিষয় থেকে উন্নত বিষয় পর্যন্ত ধারাবাহিক শেখার পথ
- **হাতে কলমে ল্যাব**: একাধিক ভাষায় সম্পূর্ণ সমাধান সহ প্রায়োগিক অনুশীলন
- **নমুনা প্রকল্পসমূহ**: কার্যকর MCP সার্ভার এবং ক্লায়েন্ট বাস্তবায়ন
- **অনুবাদ সিস্টেম**: স্বয়ংক্রিয় GitHub Actions ওয়ার্কফ্লো বহুভাষী সহায়তার জন্য
- **ছবি অ্যাসেট**: অনূদিত সংস্করণের সঙ্গে কেন্দ্রীভূত ছবি ডিরেক্টরি

## সেটআপ কমান্ড

এটি ডকুমেন্টেশন-কেন্দ্রিক রেপোজিটরি। বেশিরভাগ সেটআপ পৃথক নমুনা প্রকল্প এবং ল্যাবগুলির মধ্যে ঘটে।

### রেপোজিটরি সেটআপ

```bash
# রিপোজিটরি ক্লোন করুন
git clone https://github.com/microsoft/mcp-for-beginners.git
cd mcp-for-beginners
```

### নমুনা প্রকল্পে কাজ করা

নমুনা প্রকল্প অবস্থিত:
- `03-GettingStarted/samples/` - ভাষা নির্দিষ্ট উদাহরণ
- `03-GettingStarted/01-first-server/solution/` - প্রথম সার্ভার বাস্তবায়ন
- `03-GettingStarted/02-client/solution/` - ক্লায়েন্ট বাস্তবায়ন
- `11-MCPServerHandsOnLabs/` - ব্যাপক ডাটাবেস ইন্টিগ্রেশন ল্যাব

প্রতিটি নমুনা প্রকল্পের নিজস্ব সেটআপ নির্দেশনা রয়েছে:

#### TypeScript/JavaScript প্রকল্পসমূহ
```bash
cd <project-directory>
npm install
npm start
```

#### Python প্রকল্পসমূহ
```bash
cd <project-directory>
pip install -r requirements.txt
# অথবা
pip install -e .
python main.py
```

#### Java প্রকল্পসমূহ
```bash
cd <project-directory>
mvn clean install
mvn spring-boot:run
```

## উন্নয়ন কর্মপ্রবাহ

### ডকুমেন্টেশন কাঠামো

- **মডিউল ০০-১১**: ক্রমাগত মূল পাঠ্যক্রমের বিষয়বস্তু
- **translations/**: ভাষা-নির্দিষ্ট সংস্করণ (স্বয়ংক্রিয়ভাবে তৈরি, সরাসরি সম্পাদনা করবেন না)
- **translated_images/**: স্থানীয়কৃত ছবি সংস্করণ (স্বয়ংক্রিয়ভাবে তৈরি)
- **images/**: উত্স ছবি এবং চিত্র

### ডকুমেন্টেশন পরিবর্তন করা

১. শুধুমাত্র মূল মডিউল ডিরেক্টরির (০০-১১) ইংরেজি মার্কডাউন ফাইল সম্পাদনা করুন  
২. প্রয়োজনে `images/` ডিরেক্টরির ছবি আপডেট করুন  
৩. co-op-translator GitHub Action স্বয়ংক্রিয়ভাবে অনুবাদ তৈরি করবে  
৪. মূল ব্রাঞ্চে পুশ করার সময় অনুবাদ আবার তৈরি হবে

### অনুবাদের সাথে কাজ করা

- **স্বয়ংক্রিয় অনুবাদ**: GitHub Actions ওয়ার্কফ্লো সব অনুবাদ পরিচালনা করে  
- **directly edit করবেন না** `translations/` ডিরেক্টরির ফাইলগুলি  
- প্রতিটি অনূদিত ফাইলে অনুবাদ মেটাডেটা এমবেড করা আছে  
- সহায়তাপ্রাপ্ত ভাষাসমূহ: ৪৮+ ভাষা যেমন আরবি, চীনি, ফরাসি, জার্মান, হিন্দি, জাপানি, কোরিয়ান, পর্তুগীজ, রাশিয়ান, স্প্যানিশ এবং আরও অনেক

## পরীক্ষা প্রদানের নির্দেশনা

### ডকুমেন্টেশন মান যাচাই

যেহেতু এটি মূলত ডকুমেন্টেশন রেপোজিটরি, পরীক্ষাটি নিম্নলিখিত বিষয়গুলিতে ফোকাস করে:

১. **লিঙ্ক যাচাই**: সমস্ত অভ্যন্তরীণ লিঙ্ক কাজ করছে কিনা নিশ্চিত করুন  
```bash
# ভাঙা মার্কডাউন লিঙ্কগুলি পরীক্ষা করুন
find . -name "*.md" -type f | xargs grep -n "\[.*\](../../.*)"
```
  
২. **কোড নমুনা যাচাই**: কোড উদাহরণগুলি সঠিকভাবে কম্পাইল/চালানো হচ্ছে কিনা পরীক্ষা করুন  
```bash
# নির্দিষ্ট নমুনায় যান এবং তার পরীক্ষা চালান
cd 03-GettingStarted/samples/typescript
npm install && npm test
```
  
৩. **মার্কডাউন লিন্টিং**: ফরম্যাটিংয়ের সামঞ্জস্য পরীক্ষা করুন  
```bash
# প্রয়োজনে markdownlint ব্যবহার করুন
npx markdownlint-cli2 "**/*.md" "#node_modules"
```

#### Python  
```bash
pytest
```

#### Java  


## বিল্ড ও ডিপ্লয়মেন্ট

### ডকুমেন্টেশন ডিপ্লয়মেন্ট

রেপোজিটরিটি ডকুমেন্টেশন হোস্টিং এর জন্য GitHub Pages বা অনুরূপ ব্যবহার করে (যদি প্রযোজ্য হয়)। মূল ব্রাঞ্চের পরিবর্তনগুলি চালিত করে:

১. অনুবাদ ওয়ার্কফ্লো (`.github/workflows/co-op-translator.yml`)  
২. সব ইংরেজি মার্কডাউন ফাইলের স্বয়ংক্রিয় অনুবাদ  
৩. প্রয়োজনে ছবি স্থানীয়করণ

### কোন বিল্ড প্রক্রিয়া প্রয়োজন নেই

এই রেপোজিটরি মূলত মার্কডাউন ডকুমেন্টেশন ধারণ করে। মূল পাঠ্যক্রম সামগ্রীর জন্য কোন কম্পাইলেশন বা বিল্ড পদক্ষেপ প্রয়োজন নেই।

### নমুনা প্রকল্প ডিপ্লয়মেন্ট

পৃথক নমুনা প্রকল্পের ডিপ্লয়মেন্ট নির্দেশনা থাকতে পারে:  
- MCP সার্ভার ডিপ্লয়মেন্ট নির্দেশনার জন্য `03-GettingStarted/09-deployment/` দেখুন  
- Azure Container Apps ডিপ্লয়মেন্টের উদাহরণ `11-MCPServerHandsOnLabs/` তে রয়েছে

## অবদান Guidelines

### পুল রিকোয়েস্ট প্রক্রিয়া

১. **ফর্ক এবং ক্লোন করুন**: রেপোজিটরি ফর্ক করে আপনার লোকালেও ক্লোন করুন  
২. **একটি ব্রাঞ্চ তৈরি করুন**: বর্ণনামূলক ব্রাঞ্চ নাম ব্যবহার করুন (যেমন, `fix/typo-module-3`, `add/python-example`)  
৩. **পরিবর্তন করুন**: শুধুমাত্র ইংরেজি মার্কডাউন ফাইল সম্পাদনা করুন (অনুবাদ নয়)  
৪. **লোকালি পরীক্ষা করুন**: মার্কডাউন সঠিক প্রদর্শিত হচ্ছে নিশ্চিত করুন  
৫. **PR সাবমিট করুন**: পরিষ্কার PR শিরোনাম ও বর্ণনা ব্যবহার করুন  
৬. **CLA সই করুন**: প্রয়োজন হলে Microsoft Contributor License Agreement সই করুন

### PR শিরোনাম ফরম্যাট

স্পষ্ট, বর্ণনামূলক শিরোনাম ব্যবহার করুন:  
- `[Module XX] সংক্ষিপ্ত বর্ণনা` মডিউল-নির্দিষ্ট পরিবর্তনের জন্য  
- `[Samples] বর্ণনা` নমুনা কোড পরিবর্তনের জন্য  
- `[Docs] বর্ণনা` সাধারণ ডকুমেন্টেশন আপডেটের জন্য

### সাধারণ সমস্যা সমাধান

**Q: আমার PR অনুবাদ যাচাইতে ব্যর্থ হচ্ছে**  
উ: নিশ্চিত করুন যে আপনি শুধুমাত্র মূল মডিউল ডিরেক্টরির ইংরেজি মার্কডাউন ফাইল সম্পাদনা করেছেন, অনূদিত সংস্করণ নয়।

**Q: আমি কীভাবে নতুন ভাষা যোগ করব?**  
উ: ভাষা সমর্থন co-op-translator ওয়ার্কফ্লোর মাধ্যমে পরিচালিত হয়। নতুন ভাষা যোগ করার জন্য একটি ইস্যু খুলুন।

**Q: কোড নমুনা কাজ করছে না**  
উ: নিশ্চিত করুন আপনি নির্দিষ্ট নমুনার README তে দেওয়া সেটআপ নির্দেশনা অনুসরণ করেছেন। ডিপেন্ডেন্সি ভার্সন সঠিক আছে কিনা চেক করুন।

**Q: ছবি প্রদর্শিত হচ্ছে না**  
উ: ছবি পথ আপেক্ষিক এবং ফরোয়ার্ড স্ল্যাশ ব্যবহার করে আছে কিনা যাচাই করুন। ছবি অবশ্যই `images/` ডিরেক্টরি বা স্থানীয়কৃত সংস্করণের জন্য `translated_images/` তে থাকা উচিত।
