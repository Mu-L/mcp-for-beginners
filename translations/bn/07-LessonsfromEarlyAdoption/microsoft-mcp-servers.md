# 🚀 ১০টি Microsoft MCP সার্ভার যা ডেভেলপারদের উৎপাদনশীলতা বদলাচ্ছে

## 🎯 এই গাইডে আপনি কী শিখবেন

এই ব্যবহারিক গাইডটি দশটি Microsoft MCP সার্ভার প্রদর্শন করে যা সক্রিয়ভাবে AI সহকারী নিয়ে ডেভেলপারদের কাজ করার পদ্ধতি বদলাচ্ছে। শুধু MCP সার্ভারগুলি *কি* করতে পারে তা ব্যাখ্যা করার পরিবর্তে, আমরা আপনাকে এমন সার্ভার দেখাব যা Microsoft এবং তার বাইরের প্রতিদিনের ডেভেলপমেন্ট ওয়ার্কফ্লোতে ইতিমধ্যেই একটি বাস্তব পার্থক্য তৈরি করছে।

এই গাইডের প্রতিটি সার্ভার বাস্তব বিশ্বের ব্যবহার এবং ডেভেলপার প্রতিক্রিয়ার ভিত্তিতে নির্বাচিত হয়েছে। আপনি কেবল প্রতিটি সার্ভার কী করে তা জানতে পারবেন না, বরং কেন তা গুরুত্বপূর্ণ এবং কীভাবে আপনার নিজের প্রকল্পে এর সর্বোত্তম ব্যবহার করবেন তা আবিষ্কার করবেন। আপনি যদি MCP-তে একেবারেই নতুন হন বা আপনার বিদ্যমান সেটআপ সম্প্রসারণ করতে চান, এই সার্ভারগুলি Microsoft ইকোসিস্টেমে পাওয়া সবচেয়ে ব্যবহারিক এবং প্রভাবশালী সরঞ্জামগুলির কিছু প্রতিনিধিত্ব করে।

> **💡 দ্রুত শুরু করার টিপ**
> 
> MCP-তে নতুন? চিন্তা করবেন না! এই গাইডটি শুরু-সোহার্দ্র। আমরা চলতে চলতেই ধারণাগুলো ব্যাখ্যা করব, এবং আপনি সর্বদা আমাদের [MCP পরিচিতি](../00-Introduction/README.md) এবং [মুল ধারণা](../01-CoreConcepts/README.md) মডিউলগুলিতে ফিরে যেতে পারেন গভীর পটভূমির জন্য।

## পর্যালোচনা

এই বিস্তৃত গাইডটি দশটি Microsoft MCP সার্ভার অন্বেষণ করে যা ডেভেলপাররা কীভাবে AI সহকারী এবং বাহ্যিক সরঞ্জামগুলোর সাথে যোগাযোগ করে সে পদ্ধতিতে বিপ্লব ঘটাচ্ছে। Azure রিসোর্স ব্যবস্থাপনা থেকে ডকুমেন্ট প্রক্রিয়াকরণ পর্যন্ত, এই সার্ভারগুলি মডেল কনটেক্সট প্রোটোকল-এর শক্তি প্রদর্শন করে যাতে ধারাবাহিক, উৎপাদনশীল ডেভেলপমেন্ট ওয়ার্কফ্লো তৈরি করা যায়।

## শেখার লক্ষ্যসমূহ

এই গাইডের শেষে আপনি:
- বুঝতে পারবেন কীভাবে MCP সার্ভারগুলি ডেভেলপারদের উৎপাদনশীলতা বাড়ায়
- Microsoft-এর সবচেয়ে প্রভাবশালী MCP সার্ভার বাস্তবায়ন সম্পর্কে জানবেন
- প্রতিটি সার্ভারের ব্যবহারিক ব্যবহার আবিষ্কার করবেন
- বুঝতে পারবেন কীভাবে এই সার্ভারগুলি VS Code এবং Visual Studio তে সেটআপ এবং কনফিগার করবেন
- বৃহত্তর MCP ইকোসিস্টেম এবং ভবিষ্যৎ দিকনির্দেশনা অন্বেষণ করবেন

## 🔧 MCP সার্ভার বুঝতে: একজন শিক্ষানবীশের জন্য গাইড

### MCP সার্ভার কি?

Model Context Protocol (MCP)-তে নতুন হলে আপনি হয়তো ভাবছেন: "একটি MCP সার্ভার ঠিক কী, এবং কেন আমি তা নিয়ে ভাবব?" চলুন একটি সাধারণ তুলনা দিয়ে শুরু করি।

MCP সার্ভারকে অসাধারণ সহকারী হিসেবে ভাবুন, যারা আপনার AI কোডিং সহকারী (যেমন GitHub Copilot)-কে বাহ্যিক সরঞ্জাম এবং সার্ভিসের সাথে সংযুক্ত করতে সাহায্য করে। যেমন আপনি আপনার ফোনে বিভিন্ন কাজের জন্য আলাদা-আলাদা অ্যাপ ব্যবহার করেন—আবহাওয়ার জন্য একটি, নেভিগেশন জন্য একটি, ব্যাংকিং জন্য একটি—MCP সার্ভার আপনার AI সহকারীকে বিভিন্ন ডেভেলপমেন্ট সরঞ্জাম ও সার্ভিসের সঙ্গে কাজ করার ক্ষমতা দেয়।

### MCP সার্ভারগুলোর সমাধান করা সমস্যা

MCP সার্ভার আসার আগে, আপনি যদি চান:
- Azure রিসোর্স চেক করতে
- GitHub ইস্যু তৈরি করতে
- আপনার ডাটাবেস থেকে তথ্য অনুসন্ধান করতে
- ডকুমেন্টেশন সার্চ করতে

তাহলে আপনাকে কোড লেখা থামাতে হতো, ব্রাউজার খুলতে হতো, সংশ্লিষ্ট ওয়েবসাইটে যেতে হতো এবং ম্যানুয়ালি কাজগুলো করতে হতো। এই ধারাবাহিক প্রসঙ্গ বদল আপনার কাজের ধারাকে ভাঙে এবং উৎপাদনশীলতা কমায়।

### MCP সার্ভার কীভাবে আপনার ডেভেলপমেন্ট অভিজ্ঞতা পরিবর্তন করে

MCP সার্ভার থাকলে, আপনি আপনার ডেভেলপমেন্ট পরিবেশ (VS Code, Visual Studio, ইত্যাদি) তেই থাকবেন এবং সহজেই আপনার AI সহকারীকে এই কাজগুলো পরিচালনা করতে বলতে পারেন। উদাহরণস্বরূপ:

**এই প্রচলিত ওয়ার্কফ্লোটির পরিবর্তে:**
1. কোড লেখা বন্ধ করুন
2. ব্রাউজার খুলুন
3. Azure পোর্টালে যান
4. স্টোরেজ অ্যাকাউন্ট বিস্তারিত দেখুন
5. VS Code-এ ফিরে আসুন
6. কোড লেখা আবার শুরু করুন

**এখন আপনি করতে পারেন:**
1. AI-কে জিজ্ঞাসা করুন: "আমার Azure স্টোরেজ অ্যাকাউন্টগুলোর অবস্থা কী?"
2. তথ্য পেয়ে কোড লেখা চালিয়ে যান

### শিক্ষানবীশদের জন্য মূল সুবিধাসমূহ

#### ১. 🔄 **আপনার ফ্লো স্টেটে থাকুন**
- একাধিক অ্যাপ্লিকেশনের মধ্যে আরও ট্রানজিশন নেই
- আপনি যে কোড লিখছেন তাতে ফোকাস বজায় রাখুন
- বিভিন্ন সরঞ্জাম পরিচালনার মানসিক বোঝা কমান

#### ২. 🤖 **জটিল কমান্ডের পরিবর্তে প্রাকৃতিক ভাষা ব্যবহার করুন**
- SQL সিনট্যাক্স মনে রাখার পরিবর্তে প্রয়োজনীয় ডেটা বর্ণনা করুন
- Azure CLI কমান্ড মনে রাখার পরিবর্তে কি করতে চান তা ব্যাখ্যা করুন
- AI-কে প্রযুক্তিগত বিশদ সামলাতে দিন, আপনি লজিকের উপর ফোকাস করুন

#### ৩. 🔗 **বিভিন্ন সরঞ্জাম একসঙ্গে সংযুক্ত করুন**
- বিভিন্ন সার্ভিস মিলিয়ে শক্তিশালী ওয়ার্কফ্লো তৈরি করুন
- উদাহরণ: "সাম্প্রতিক সব GitHub ইস্যু নিয়ে Azure DevOps-এ সংশ্লিষ্ট কাজের আইটেম তৈরি করুন"
- জটিল স্ক্রিপ্ট না লিখেই অটোমেশন তৈরি করুন

#### ৪. 🌐 **বর্ধমান ইকোসিস্টেমে প্রবেশাধিকার পান**
- Microsoft, GitHub এবং অন্যান্য কোম্পানির তৈরি সার্ভার থেকে সুবিধা নিন
- বিভিন্ন বিক্রেতার সরঞ্জাম সহজেই মিশিয়ে ব্যবহার করুন
- এমন একটি মানসম্মত ইকোসিস্টেমে যোগ দিন যা বিভিন্ন AI সহকারীর সঙ্গে কাজ করে

#### ৫. 🛠️ **করতে-করতে শিখুন**
- ধারণা বোঝার জন্য প্রি-বিল্ট সার্ভার দিয়ে শুরু করুন
- আস্তে আস্তে নিজের সার্ভার তৈরি করুন যতক্ষণ আপনি স্বাচ্ছন্দ্য বোধ করেন
- উপলব্ধ SDK এবং ডকুমেন্টেশন দিয়ে আপনার শেখার পথে গাইড পান

### শিক্ষানবীশদের জন্য বাস্তব উদাহরণ

ধরুন আপনি ওয়েব ডেভেলপমেন্টে নতুন এবং আপনার প্রথম প্রজেক্টে কাজ করছেন। MCP সার্ভারগুলি কিভাবে সাহায্য করতে পারে:

**প্রচলিত পদ্ধতি:**
```
1. Code a feature
2. Open browser → Navigate to GitHub
3. Create an issue for testing
4. Open another tab → Check Azure docs for deployment
5. Open third tab → Look up database connection examples
6. Return to VS Code
7. Try to remember what you were doing
```

**MCP সার্ভার সহ:**
```
1. Code a feature
2. Ask AI: "Create a GitHub issue for testing this login feature"
3. Ask AI: "How do I deploy this to Azure according to the docs?"
4. Ask AI: "Show me the best way to connect to my database"
5. Continue coding with all the information you need
```

### এন্টারপ্রাইজ স্ট্যান্ডার্ড সুবিধা

MCP এক ধরনের শিল্প মান হয়ে উঠছে, যা মানে:
- **সামঞ্জস্যতা**: বিভিন্ন সরঞ্জাম ও কোম্পানির মধ্যে সমান অভিজ্ঞতা
- **ইন্টারঅপারেবিলিটি**: বিভিন্ন বিক্রেতার সার্ভার একসঙ্গে কাজ করে
- **ভবিষ্যৎ প্রমাণ**: দক্ষতা এবং সেটআপ বিভিন্ন AI সহকারীর মধ্যে স্থানান্তরযোগ্য
- **কমিউনিটি**: বড় ইকোসিস্টেম ভাগ করা জ্ঞান এবং সম্পদ সহ

### শুরু করা: আপনি কী শিখবেন

- সাধারণ ডেভেলপমেন্ট চ্যালেঞ্জ সমাধান করা
- পুনরাবৃত্ত কাজ কমানো
- কোডের গুণগত মান উন্নত করা

## VS Code এবং Visual Studio তে MCP সেটআপ 🚀

যদি আপনি Visual Studio Code বা Visual Studio 2022 ব্যবহার করেন GitHub Copilot সহ, তাহলে এই MCP সার্ভারগুলি সেটআপ করা সহজ।

### VS Code সেটআপ

VS Code-এর জন্য মৌলিক প্রক্রিয়া:

১. **Agent Mode চালু করুন**: VS Code-তে Copilot Chat উইন্ডোতে Agent মোড সিলেক্ট করুন  
২. **MCP সার্ভার কনফিগার করুন**: VS Code settings.json ফাইলে সার্ভার কনফিগারেশন যোগ করুন  
৩. **সার্ভার শুরু করুন**: আপনি যেসব সার্ভার ব্যবহার করতে চান সেগুলোর জন্য "Start" বাটনে ক্লিক করুন  
৪. **টুল সিলেক্ট করুন**: আপনার বর্তমান সেশনের জন্য কোন কোন MCP সার্ভার সক্রিয় করবেন তা বেছে নিন

বিস্তারিত সেটআপ নির্দেশনার জন্য [VS Code MCP ডকুমেন্টেশন](https://code.visualstudio.com/docs/copilot/copilot-mcp) দেখুন।

**বাস্তব জীবনের ব্যবহার**: "আমার Azure স্টোরেজ অ্যাকাউন্টের তালিকা দেখাও", "গত এক ঘণ্টায় আমার Log Analytics ওয়ার্কস্পেসে ত্রুটির জন্য অনুসন্ধান করো", অথবা "সঠিক প্রমাণীকরণ সহ Node.js ব্যবহার করে Azure অ্যাপ্লিকেশন তৈরি করতে সাহায্য করো"

**সম্পূর্ণ ডেমো দৃশ্য**: এখানে একটি সম্পূর্ণ ওয়াকথ্রু রয়েছে যা Azure MCP এবং GitHub Copilot for Azure এক্সটেনশনকে VS Code-এ একসাথে ব্যবহার করার ক্ষমতা দেখায়। যখন আপনার কাছে উভয়ই ইনস্টল থাকে এবং আপনি প্রম্পট দেন:

> "একটি পায়থন স্ক্রিপ্ট তৈরি করো যা DefaultAzureCredential প্রমাণীকরণ ব্যবহার করে Azure Blob Storage-এ ফাইল আপলোড করে। স্ক্রিপ্টটি আমার Azure স্টোরেজ অ্যাকাউন্ট 'mycompanystorage'-এ সংযোগ করবে, 'documents' নামক কনটেইনারে আপলোড করবে, আপলোড করার জন্য বর্তমান টাইমস্ট্যাম্প সহ একটি টেস্ট ফাইল তৈরি করবে, ত্রুটি সুন্দরভাবে পরিচালনা করবে এবং তথ্যবহুল আউটপুট দেবে, প্রমাণীকরণ এবং ত্রুটি পরিচালনার জন্য Azure সেরা অনুশীলন অনুসরণ করবে, DefaultAzureCredential প্রমাণীকরণের কাজ ব্যাখ্যা করতে মন্তব্য থাকবে, এবং উপযুক্ত ফাংশন ও ডকুমেন্টেশনের মাধ্যমে স্ক্রিপ্টটি সুগঠিত করবে।"

Azure MCP সার্ভার একটি পূর্ণাঙ্গ, প্রোডাকশন-রেডি পায়থন স্ক্রিপ্ট তৈরি করবে যা:
- সর্বশেষ Azure Blob Storage SDK ব্যবহার করে যথাযথ async প্যাটার্নসহ
- DefaultAzureCredential প্রয়োগ করে বিস্তৃত ফালব্যাক চেইন ব্যাখ্যা সহ
- নির্দিষ্ট Azure এক্সসেপশন টাইপ সহ দৃঢ় ত্রুটি পরিচালনা অন্তর্ভুক্ত করে
- Azure SDK সেরা অনুশীলন অনুসরণ করে রিসোর্স ম্যানেজমেন্ট এবং সংযোগ পরিচালনা
- বিস্তারিত লগিং এবং তথ্যবহুল কনসোল আউটপুট দেয়
- ফাংশন, ডকুমেন্টেশন, এবং টাইপ হিন্টসহ সুগঠিত স্ক্রিপ্ট তৈরি করে

দারুণ ব্যাপার হলো, Azure MCP ছাড়া আপনি হয়তো এমন একটি সাধারণ ব্লব স্টোরেজ কোড পাবেন যা কাজ করে কিন্তু বর্তমান Azure প্যাটার্ন অনুসরণ করে না। Azure MCP সহ, আপনি এমন কোড পাবেন যা সর্বশেষ প্রমাণীকরণ পদ্ধতি ব্যবহার করে, Azure-সম্পর্কিত ত্রুটি পরিস্থিতি পরিচালনা করে এবং মাইক্রোসফটের প্রোডাকশন অনুশীলন অনুসরণ করে।

**বৈশিষ্ট্যমণ্ডিত উদাহরণ**: আমি `az` এবং `azd` CLI গুলোর সঠিক কমান্ড মনে রাখার জন্য সংগ্রাম করেছি। আমার জন্য এটা সবসময় দুই ধাপের প্রক্রিয়া: প্রথমে সিনট্যাক্স দেখছি, তারপর কমান্ড চালাচ্ছি। অনেক সময় আমি শুধু পোর্টালে ঢুকে ক্লিক করে কাজ করে ফেলি কারণ আমি স্বীকার করতে চাই না যে CLI সিনট্যাক্স মনে রাখতে পারি না। যা আমি চাই তা শুধু বর্ণনা করতে পারা দারুণ এবং আরও ভালো হলো IDE ত্যাগ না করেই সেটা করতে পারা!

শুরু করার জন্য [Azure MCP repository](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) এ উজ্জ্বল ব্যবহারের একটি দুর্দান্ত তালিকা রয়েছে। বিস্তৃত সেটআপ গাইড এবং উন্নত কনফিগারেশন অপশনসমূহের জন্য [সরকারি Azure MCP ডকুমেন্টেশন](https://learn.microsoft.com/azure/developer/azure-mcp-server/) দেখুন।

### ৩. 🐙 GitHub MCP Server

[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_Server-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=github&config=%7B%22type%22%3A%20%22http%22%2C%22url%22%3A%20%22https%3A%2F%2Fapi.githubcopilot.com%2Fmcp%2F%22%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Server-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=github&config=%7B%22type%22%3A%20%22http%22%2C%22url%22%3A%20%22https%3A%2F%2Fapi.githubcopilot.com%2Fmcp%2F%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/github/github-mcp-server)

**এটি কি করে**: অফিসিয়াল GitHub MCP সার্ভার GitHub-এর পুরো ইকোসিস্টেমের সাথে নিরবিচ্ছিন্ন ইন্টিগ্রেশন প্রদান করে, যেখানে উভয় হোস্টেড রিমোট অ্যাক্সেস এবং লোকাল Docker ডিপ্লয়মেন্ট বিকল্প রয়েছে। এটি শুধু সাধারণ রিপোজিটরি অপারেশন নয় – এটি একটি বিস্তৃত টুলকিট যা GitHub Actions ম্যানেজমেন্ট, পুল রিকোয়েস্ট ওয়ার্কফ্লো, ইস্যু ট্র্যাকিং, সিকিউরিটি স্ক্যানিং, নোটিফিকেশন এবং উন্নত অটোমেশন ক্ষমতা অন্তর্ভুক্ত করে।

**এটি কেন উপকারী**: এই সার্ভার GitHub-এর সাথে আপনার ইন্টারঅ্যাকশন পরিবর্তিত করে দেয়, পুরো প্ল্যাটফর্ম অভিজ্ঞতাকে সরাসরি আপনার ডেভেলপমেন্ট এনভায়রনমেন্টে নিয়ে আসে। প্রকল্প ব্যবস্থাপনা, কোড রিভিউ এবং CI/CD মনিটরিং-এর জন্য বারবার VS Code থেকে GitHub.com-এ স্যুইচ করার পরিবর্তে, আপনি প্রাকৃতিক ভাষার কমান্ডের মাধ্যমে সবকিছু পরিচালনা করতে পারেন এবং আপনার কোডে মনোযোগ রাখতে পারেন।

> **ℹ️ নোট: 'এজেন্ট' এর বিভিন্ন ধরনের পার্থক্য**
> 
> এই GitHub MCP সার্ভারটিকে GitHub-এর Coding Agent (যে AI এজেন্টটি স্বয়ংক্রিয় কোডিং কাজের জন্য ইস্যুতে নিয়োগ করা যায়) এর সঙ্গে বিভ্রান্ত করবেন না। GitHub MCP সার্ভার VS Code-এর Agent মোডে GitHub API ইন্টিগ্রেশন প্রদান করে, যেখানে GitHub-এর Coding Agent একটি আলাদা ফিচার যা GitHub ইস্যুতে নিয়োগ পেলে পুল রিকোয়েস্ট তৈরি করে।

**মূল ক্ষমতাসমূহ অন্তর্ভুক্ত**:
- **⚙️ GitHub Actions**: সম্পূর্ণ CI/CD পাইপলাইন ম্যানেজমেন্ট, ওয়ার্কফ্লো মনিটরিং, এবং আর্টিফ্যাক্ট হ্যান্ডলিং
- **🔀 পুল রিকোয়েস্ট**: PR তৈরি, পর্যালোচনা, মার্জ এবং বিস্তৃত স্ট্যাটাস ট্র্যাকিং সহ ব্যবস্থাপনা
- **🐛 ইস্যুগুলো**: পূর্ণ ইস্যু লাইফসাইকেল ম্যানেজমেন্ট, মন্তব্য, লেবেল এবং নিয়োগ
- **🔒 সিকিউরিটি**: কোড স্ক্যানিং সতর্কতা, সিক্রেট সনাক্তকরণ, এবং Dependabot ইন্টিগ্রেশন
- **🔔 নোটিফিকেশন**: স্মার্ট নোটিফিকেশন ম্যানেজমেন্ট ও রিপোজিটরি সাবস্ক্রিপশন নিয়ন্ত্রণ
- **📁 রিপোজিটরি ম্যানেজমেন্ট**: ফাইল অপারেশন, ব্রাঞ্চ ম্যানেজমেন্ট, এবং রিপোজিটরি প্রশাসন
- **👥 সহযোগিতা**: ব্যবহারকারী ও সংগঠন অনুসন্ধান, টিম ম্যানেজমেন্ট, ও অ্যাক্সেস নিয়ন্ত্রণ

**বাস্তব জীবনের ব্যবহার**: "আমার ফিচার ব্রাঞ্চ থেকে একটি পুল রিকোয়েস্ট তৈরি করো", "আমার এই সপ্তাহের সব ব্যর্থ CI রানগুলো দেখাও", "আমার রিপোজিটরি গুলোর জন্য সব ওপেন সিকিউরিটি এলার্টগুলো তালিকা করো", অথবা "আমার সংগঠনগুলোতে আমার কাছে নিয়োগকৃত সব ইস্যু খুঁজে বের করো"

**সম্পূর্ণ ডেমো দৃশ্য**: এখানে একটি শক্তিশালী ওয়ার্কফ্লো রয়েছে যা GitHub MCP সার্ভারের ক্ষমতা প্রদর্শন করে:

> "আমাদের স্প্রিন্ট রিভিউ এর জন্য আমাকে প্রস্তুত করতে হবে। এই সপ্তাহে আমি যেসব পুল রিকোয়েস্ট তৈরি করেছি তা দেখাও, আমাদের CI/CD পাইপলাইনের স্ট্যাটাস চেক করো, যে কোন সিকিউরিটি এলার্টের সারাংশ তৈরি করো যা আমাদের সমাধান করতে হবে, এবং 'feature' লেবেল সহ মার্জ করা PR গুলোর ভিত্তিতে রিলিজ নোটস খসড়া করতে সাহায্য করো।"

GitHub MCP সার্ভার:
- বিস্তারিত স্ট্যাটাস তথ্য সহ আপনার সাম্প্রতিক পুল রিকোয়েস্টগুলি প্রশ্ন করবে
- ওয়ার্কফ্লো রান বিশ্লেষণ করবে এবং যেকোনো ব্যর্থতা বা পারফরম্যান্স সমস্যা হাইলাইট করবে
- সিকিউরিটি স্ক্যানিং ফলাফল সংকলন করবে এবং গুরুত্বপূর্ণ এলার্ট অগ্রাধিকার দেবে
- মার্জ করা PR থেকে তথ্য বের করে বিস্তৃত রিলিজ নোট তৈরি করবে
- স্প্রিন্ট পরিকল্পনা ও রিলিজ প্রস্তুতির জন্য কার্যকর পরবর্তী ধাপ সরবরাহ করবে

**বৈশিষ্ট্যমণ্ডিত উদাহরণ**: আমি কোড রিভিউ ওয়ার্কফ্লোতে এটি ব্যবহার করতে পছন্দ করি। VS Code, GitHub নোটিফিকেশন, এবং পুল রিকোয়েস্ট পৃষ্ঠার মধ্যে লাফানোর পরিবর্তে, আমি বলতে পারি "আমার পর্যালোচনার জন্য অপেক্ষমাণ সব PR দেখাও" এবং তারপর "PR #123-এ কমেন্ট যোগ করো যা প্রমাণীকরণ পদ্ধতির ত্রুটি পরিচালনা সম্পর্কে প্রশ্ন করে।" সার্ভার GitHub API কলগুলি পরিচালনা করে, আলোচনাটির প্রসঙ্গ বজায় রাখে, এবং এমনকি আমাকে আরও গঠনমূলক পর্যালোচনা মন্তব্য তৈরি করতে সাহায্য করে।

**প্রমাণীকরণ অপশন**: সার্ভার OAuth (VS Code-এ নিরবিচ্ছিন্ন) এবং Personal Access Tokens উভয়ই সমর্থন করে, কনফিগারযোগ্য টুলসেট সরবরাহ করে যা শুধু আপনার প্রয়োজনীয় GitHub ফাংশনালিটি সক্রিয় করে। আপনি এটি রিমোট হোস্টেড সার্ভিস হিসেবে তাত্ক্ষণিক সেটআপের জন্য চালাতে পারেন বা সম্পূর্ণ নিয়ন্ত্রণের জন্য লোকাল Docker হিসাবে চালাতে পারেন।

### ৫. 📝 MarkItDown MCP Server
[![Install in VS Code](https://img.shields.io/badge/VS_Code-Install_MarkItDown_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D) [![Install in VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_MarkItDown_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/markitdown)

**এটি কী করে**: MarkItDown একটি ব্যাপক ডকুমেন্ট রূপান্তর সার্ভার যা বিভিন্ন ফাইল ফরম্যাটকে উচ্চ-গুণমানের Markdown-এ রূপান্তর করে, যা LLM ব্যবহারের জন্য এবং টেক্সট বিশ্লেষণ ওয়ার্কফ্লোর জন্য অপ্টিমাইজ করা হয়েছে।

**কেন এটি উপকারী**: আধুনিক ডকুমেন্টেশান ওয়ার্কফ্লোর জন্য অপরিহার্য! MarkItDown একটি বিস্ময়কর পরিসরের ফাইল ফরম্যাট পরিচালনা করে, একই সাথে গুরুত্বপূর্ণ ডকুমেন্ট গঠন যেমন শিরোনাম, তালিকা, টেবিল, এবং লিঙ্ক সংরক্ষণ করে। সাধারণ টেক্সট এক্সট্র্যাকশন টুলগুলোর থেকে ভিন্ন, এটি অর্থবোধকতা এবং বিন্যাস বজায় রাখার উপর গুরুত্ব দেয় যা AI প্রক্রিয়াকরণ এবং মানুষের পাঠযোগ্যতার জন্য গুরুত্বপূর্ণ।

**সমর্থিত ফাইল ফরম্যাটসমূহ**:
- **অফিস ডকুমেন্টস**: PDF, PowerPoint (PPTX), Word (DOCX), Excel (XLSX/XLS)
- **মিডিয়া ফাইলসমূহ**: চিত্র (EXIF মেটাডেটা ও OCR সহ), অডিও (EXIF মেটাডেটা এবং স্পিচ ট্রান্সক্রিপশন সহ)
- **ওয়েব কনটেন্ট**: HTML, RSS ফিড, YouTube URL, উইকিপিডিয়া পৃষ্ঠা
- **ডাটা ফরম্যাট**: CSV, JSON, XML, ZIP ফাইল (পুনরায় কনটেন্ট প্রসেসিং)
- **প্রকাশনার ফরম্যাট**: EPub, Jupyter নোটবুকস (.ipynb)
- **ইমেইল**: Outlook মেসেজ (.msg)
- **অগ্রসর**: উন্নত PDF প্রক্রিয়াকরণের জন্য Azure Document Intelligence ইন্টিগ্রেশন

**অগ্রসর ক্ষমতাসমূহ**: MarkItDown LLM-চালিত ছবি বিবরণ (OpenAI ক্লায়েন্ট প্রদান করা হলে), উন্নত PDF প্রক্রিয়াকরণের জন্য Azure Document Intelligence, স্পিচ কনটেন্টের জন্য অডিও ট্রান্সক্রিপশন, এবং অতিরিক্ত ফাইল ফরম্যাটের জন্য প্লাগইন সিস্টেম সমর্থন করে।

**বাস্তব জীবনের ব্যবহার**: "এই PowerPoint উপস্থাপনাটি Markdown-এ রূপান্তর কর যাতে আমাদের ডকুমেন্টেশন সাইটে ব্যবহার করা যায়", "এই PDF থেকে সঠিক হেডিং স্ট্রাকচারসহ টেক্সট এক্সট্র্যাক্ট করো", অথবা "এই Excel স্প্রেডশীটকে একটি পাঠযোগ্য টেবিল ফরম্যাটে রূপান্তর করো"

**বৈশিষ্ট্যপূর্ণ উদাহরণ**: [MarkItDown ডকস](https://github.com/microsoft/markitdown#why-markdown) থেকে উদ্ধৃতি:

> Markdown প্রায় সাধারণ টেক্সটের খুব কাছাকাছি, খুব কম মার্কআপ বা ফরম্যাটিং সহ, কিন্তু তবুও গুরুত্বপূর্ণ ডকুমেন্ট গঠন উপস্থাপনের একটি উপায় দেয়। প্রধান LLM, যেমন OpenAI-এর GPT-4o, স্বতঃস্ফূর্তভাবে "Markdown" ভাষায় উত্তর দেয় এবং প্রায়ই তাদের প্রতিক্রিয়ায় Markdown অন্তর্ভুক্ত করে। এতে বোঝা যায় যে তারা প্রচুর Markdown-ফরম্যাটকৃত টেক্সটে প্রশিক্ষিত হয়েছে এবং এটি ভালভাবে বোঝে। একটি পার্শ্বফল হিসেবে, Markdown প্রচলন অত্যন্ত টোকেন-ব্যবহার দক্ষ।

MarkItDown ডকুমেন্ট গঠন সংরক্ষণে খুব দক্ষ, যা AI ওয়ার্কফ্লোর জন্য গুরুত্বপূর্ণ। উদাহরণস্বরূপ, যখন একটা PowerPoint উপস্থাপনাকে রূপান্তর করা হয়, তখন এটি স্লাইডের সংগঠন যথাযথ শিরোনাম দিয়ে রাখে, টেবিলগুলো Markdown টেবিল আকারে এক্সট্র্যাক্ট করে, ছবিগুলোর জন্য বিকল্প টেক্সট যোগ করে, এবং বক্তার নোট প্রসেস করে। চার্টগুলো পাঠযোগ্য ডাটা টেবিলে রূপান্তরিত হয় এবং প্রাপ্ত Markdown মূল উপস্থাপনার যুক্তিসঙ্গত প্রবাহ বজায় রাখে। এই কারণে এটি উপস্থাপনার বিষয়বস্তু AI সিস্টেমে খাওয়ানোর বা বিদ্যমান স্লাইড থেকে ডকুমেন্টেশন তৈরি করার জন্য আদর্শ।

**বৈশিষ্ট্যপূর্ণ উদাহরণ**: একবার ডাটাবেস সংযোগ স্থাপন করলে, আপনি তৎক্ষণাৎ আপনার ডাটার সাথে কথোপকথন শুরু করতে পারেন। ব্লগ পোস্টটি একটি সহজ প্রশ্ন দিয়ে এটি দেখায়: "আপনি কোন ডাটাবেসের সাথে সংযুক্ত?" MCP সার্ভার প্রাসঙ্গিক ডাটাবেস টুল অ্যাক্সেস করে, SQL সার্ভার ইনস্ট্যান্সের সাথে সংযোগ করে, এবং কৌণিক SQL কোড ছাড়াই আপনার বর্তমান ডাটাবেস সংযোগের বিস্তারিত তথ্য প্রদান করে। সার্ভার স্কিমা ব্যবস্থাপনা থেকে ডাটা ম্যানিপুলেশন পর্যন্ত ব্যাপক ডাটাবেস অপারেশনগুলো প্রাকৃতিক ভাষার মাধ্যমে পরিচালিত করে। সম্পূর্ণ সেটআপ নির্দেশিকা এবং VS Code ও Claude Desktop এর কনফিগারেশন উদাহরণের জন্য দেখুন: [Introducing MSSQL MCP Server (Preview)](https://devblogs.microsoft.com/azure-sql/introducing-mssql-mcp-server/)।

> "আমি একজন কাস্টমার সাপোর্ট এজেন্ট তৈরি করছি। আমাকে ক্যাটালগ থেকে একটি ভাল রিজনিং মডেল খুঁজে দিতে সাহায্য করো, Azure AI সার্ভিসে সেটি ডিপ্লয় করো, আমাদের ডকুমেন্টেশন থেকে একটি জ্ঞানভিত্তিক বেস তৈরি করো, প্রতিক্রিয়ার গুণগত মান পরীক্ষা করার জন্য একটি মূল্যায়ন ফ্রেমওয়ার্ক সেট আপ করো, এবং তারপর GitHub টোকেন দিয়ে ইন্টিগ্রেশন প্রোটোটাইপ করতে সাহায্য করো।"

Microsoft Foundry MCP Server নিম্নলিখিত কাজ করবে:
- আপনার চাহিদার ভিত্তিতে সর্বোত্তম রিজনিং মডেল প্রস্তাব করার জন্য মডেল ক্যাটালগ থেকে প্রশ্ন করবে
- আপনার পছন্দের Azure অঞ্চলের জন্য ডিপ্লয়মেন্ট কমান্ড এবং কোটা তথ্য সরবরাহ করবে
- আপনার ডকুমেন্টেশনের জন্য সঠিক স্কিমা সহ Azure AI Search ইনডেক্স সেট আপ করবে
- গুণগত মানের মেট্রিক এবং সুরক্ষা যাচাইসহ মূল্যায়ন পাইপলাইন কনফিগার করবে
- তাৎক্ষণিক পরীক্ষার জন্য GitHub প্রমাণীকরণের সাথে প্রোটোটাইপিং কোড তৈরি করবে
- আপনার নির্দিষ্ট প্রযুক্তি স্ট্যাকের জন্য বিস্তৃত সেটআপ নির্দেশিকা প্রদান করবে

**বৈশিষ্ট্যপূর্ণ উদাহরণ**: একজন ডেভেলপার হিসেবে, আমি বিভিন্ন LLM মডেলের সাথে মিল রেখে যেতে সংগ্রাম করছি। আমি কিছু প্রধান মডেল জানি, কিন্তু মনে হচ্ছে কিছু প্রোডাক্টিভিটি এবং দক্ষতা সুযোগ হারাচ্ছি। টোকেন এবং কোটা পরিচালনা করা চাপের এবং কঠিন – কখন সঠিক মডেল নির্বাচন করছি অথবা বাজেট অপচয় হচ্ছে বলেই বুঝতে পারি না। সম্প্রতি James Montemagno থেকে এই MCP Server সম্পর্কে শুনেছি যখন সহকর্মীদের থেকে MCP সার্ভারের সুপারিশ নিচ্ছিলাম, এবং এটি ব্যবহার করতে আমি উত্তেজিত। মডেল আবিষ্কার ক্ষমতাগুলো বিশেষত আমার মতো কারো জন্য যথেষ্ট প্রভাববিস্তারকারী যারা প্রচলিত মডেলের বাইরে অনুসন্ধান করতে চায় এবং কাজ-কর্মে যথাযথ মডেল খুঁজে পেতে চায়। মূল্যায়ন কাঠামো আমাকে নিশ্চিত করবে আমি সত্যিই ভাল ফল পাচ্ছি, শুধু নতুন কিছু চেষ্টা করছি না।

> **ℹ️ পরীক্ষামূলক স্থিতি**
> 
> এই MCP সার্ভার পরীক্ষামূলক এবং সক্রিয় উন্নয়নাধীন। ফিচার এবং API পরিবর্তিত হতে পারে। Azure AI ক্ষমতা অনুসন্ধান এবং প্রোটোটাইপ বানানোর জন্য আদর্শ, তবে প্রোডাকশন ব্যবহারের জন্য স্থিতিশীলতা যাচাই করুন।

**বৈশিষ্ট্যপূর্ণ উদাহরণ:** Build-এর সময় আমি আমার বন্ধু John Miller-এর সাথে M365 Agents নিয়ে কথা বললাম, তিনি আমাকে এই MCP সুপারিশ করলেন। যারা M365 Agents-এ নতুন, তাদের জন্য এটি চমৎকার কারণ এটি টেমপ্লেট, নমুনা কোড এবং স্ক্যাফোল্ডিং সরবরাহ করে যা ডকুমেন্টেশনের পাহাড়ে ডুবে না গিয়ে শুরু করতে সাহায্য করে। স্কিমা যাচাই ফিচারগুলি বিশেষভাবে কার্যকর যা মেনিফেস্ট স্ট্রাকচারের ত্রুটি থেকে উদ্ভূত ঘন্টাব্যাপী ডিবাগিং এড়াতে সাহায্য করে।

## 📋 উপসংহার

Model Context Protocol (MCP) ডেভেলপারদের AI সহকারী এবং বাইরের সরঞ্জামের সাথে যোগাযোগের পদ্ধতি পরিবর্তন করছে। এই ১০টি Microsoft MCP সার্ভার স্ট্যান্ডার্ডাইজড AI ইন্টিগ্রেশনের শক্তি প্রদর্শন করে, যা শক্তিশালী বাইরের ক্ষমতা অ্যাক্সেস করতে গিয়ে ডেভেলপারদের ফ্লো স্টেট বজায় রাখার জন্য নির্বিঘ্ন কর্মপ্রবাহ নিশ্চিত করে।

সমগ্র Azure ইকোসিস্টেম ইন্টিগ্রেশন থেকে শুরু করে Playwright এর মত স্পেশালাইজড সরঞ্জাম এবং MarkItDown যেমন ডকুমেন্ট প্রসেসিংয়ের জন্য, এই সার্ভারগুলো দেখায় MCP কীভাবে বিভিন্ন ডেভেলপমেন্ট পরিস্থিতিতে উৎপাদনশীলতা বৃদ্ধি করতে পারে। স্ট্যান্ডার্ডাইজড প্রোটোকল নিশ্চিত করে যে এই সরঞ্জামগুলো নির্বিঘ্নে একসাথে কাজ করে, একটি সান্নিধ্যপূর্ণ ডেভেলপমেন্ট অভিজ্ঞতা সৃষ্টি করে।

MCP ইকোসিস্টেম আরও বিকাশ পাওয়ার সঙ্গে কমিউনিটির সাথে যুক্ত থাকা, নতুন সার্ভার আবিষ্কার করা, এবং কাস্টম সমাধান নির্মাণ করা হবে আপনার ডেভেলপমেন্ট উৎপাদনশীলতা সর্বাধিক করার চাবিকাঠি। MCP-এর ওপেন স্ট্যান্ডার্ড প্রকৃতি আপনাকে বিভিন্ন বিক্রেতার সরঞ্জাম মিশ্রিত ও মিলিয়ে আপনার নির্দিষ্ট প্রয়োজনের জন্য আদর্শ ওয়ার্কফ্লো তৈরি করার সুযোগ দেয়।
