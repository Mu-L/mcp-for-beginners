# تنفيذ أمان محتوى أزور مع MCP

> **مخاطر OWASP MCP المعالجة**: [MCP06 - الالتفاف على تدفق النوايا](https://microsoft.github.io/mcp-azure-security-guide/mcp/mcp06-prompt-injection/)

لتعزيز أمان MCP ضد حقن الأوامر، وتسمم الأدوات، وغير ذلك من الثغرات الخاصة بالذكاء الاصطناعي، يوصى بشدة بدمج أمان محتوى أزور. يتماشى هذا الدليل التنفيذي مع [ورشة عمل قمة أمان MCP (شيربا)](https://azure-samples.github.io/sherpa/) المخيم 3: أمان الإدخال/الإخراج.

## التكامل مع خادم MCP

للتكامل مع أمان محتوى أزور في خادم MCP الخاص بك، أضف مرشح أمان المحتوى كوسيط في خط معالجة الطلبات لديك:

1. تهيئة المرشح أثناء بدء تشغيل الخادم  
2. التحقق من جميع طلبات الأدوات الواردة قبل المعالجة  
3. فحص جميع الاستجابات الصادرة قبل إرجاعها إلى العملاء  
4. تسجيل التنبيهات عن انتهاكات الأمان والتنبيه بها  
5. تنفيذ معالجة الأخطاء المناسبة لفشل فحوصات أمان المحتوى  

يوفر هذا دفاعاً قوياً ضد:  
- هجمات حقن الأوامر  
- محاولات تسمم الأدوات  
- استخراج البيانات عبر مدخلات خبيثة  
- توليد محتوى ضار  

## أفضل الممارسات لتكامل أمان محتوى أزور

1. **قوائم الحظر المخصصة**: إنشاء قوائم حظر مخصصة لأنماط حقن MCP  
2. **ضبط شدة المخاطر**: تعديل عتبات الشدة حسب حالة الاستخدام الخاصة بك وتحملك للمخاطر  
3. **تغطية شاملة**: تطبيق فحوصات أمان المحتوى على جميع المدخلات والمخرجات  
4. **تحسين الأداء**: التفكير في تنفيذ التخزين المؤقت لفحوصات أمان المحتوى المتكررة  
5. **آليات التراجع**: تحديد سلوكيات واضحة عند عدم توفر خدمات أمان المحتوى  
6. **تغذية راجعة للمستخدم**: توفير تغذية راجعة واضحة للمستخدم عند حظر المحتوى بسبب مخاوف الأمان  
7. **التحسين المستمر**: تحديث قوائم الحظر والأنماط بانتظام بناءً على التهديدات الناشئة
