# أمان MCP: حماية شاملة لأنظمة الذكاء الاصطناعي

[![أفضل ممارسات أمان MCP](../../../translated_images/ar/03.175aed6dedae133f.webp)](https://youtu.be/88No8pw706o)

_(انقر على الصورة أعلاه لمشاهدة فيديو هذا الدرس)_

الأمان هو أحد الأسس الأساسية في تصميم نظام الذكاء الاصطناعي، ولهذا نمنحه الأولوية في القسم الثاني لنا. يتماشى هذا مع مبدأ مايكروسوفت **المؤمن عبر التصميم** من [مبادرة المستقبل الآمن](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/).

يجلب بروتوكول سياق النموذج (MCP) قدرات جديدة قوية للتطبيقات المدفوعة بالذكاء الاصطناعي، في حين يقدم تحديات أمنية فريدة تتجاوز مخاطر البرمجيات التقليدية. تواجه أنظمة MCP اهتمامًا متزايدًا لمخاوف الأمان المعروفة (البرمجة الآمنة، أقل صلاحيات، أمان سلسلة التوريد) بالإضافة إلى تهديدات جديدة خاصة بالذكاء الاصطناعي تشمل حقن الأوامر، تسميم الأدوات، اختطاف الجلسات، هجمات الوكيل المرتبك، ثغرات تمرير الرموز، وتعديل القدرات الديناميكي.

يستكشف هذا الدرس أبرز مخاطر الأمان في تطبيقات MCP — التي تغطي المصادقة، التفويض، الصلاحيات المفرطة، حقن الأوامر غير المباشر، أمان الجلسة، مشاكل الوكيل المرتبك، إدارة الرموز، وضعف سلسلة التوريد. ستتعلم الضوابط العملية وأفضل الممارسات لتخفيف هذه المخاطر مع الاستفادة من حلول مايكروسوفت مثل دروع الأوامر، أمان المحتوى على أزور، وأمان جِت هَب المتقدم لتعزيز نشر MCP الخاص بك.

## أهداف التعلم

بحلول نهاية هذا الدرس، ستكون قادرًا على:

- **تحديد التهديدات الخاصة بـ MCP**: التعرف على المخاطر الأمنية الفريدة في أنظمة MCP بما في ذلك حقن الأوامر، تسميم الأدوات، الصلاحيات المفرطة، اختطاف الجلسات، مشاكل الوكيل المرتبك، ثغرات تمرير الرموز، ومخاطر سلسلة التوريد
- **تطبيق الضوابط الأمنية**: تنفيذ تدابير تخفيف فعالة تشمل مصادقة قوية، وصول بأقل صلاحيات، إدارة آمنة للرموز، ضوابط أمان الجلسة، والتحقق من سلسلة التوريد
- **الاستفادة من حلول مايكروسوفت الأمنية**: فهم ونشر دروع الأوامر من مايكروسوفت، أمان المحتوى على أزور، وأمان جِت هَب المتقدم لحماية حمل عمل MCP
- **التحقق من أمان الأدوات**: التعرف على أهمية التحقق من بيانات الأدوات الوصفية، المراقبة للتغييرات الديناميكية، والدفاع ضد هجمات الحقن غير المباشر
- **دمج أفضل الممارسات**: الجمع بين أساسيات الأمان المعروفة (البرمجة الآمنة، تقوية الخوادم، الثقة الصفرية) مع ضوابط MCP المخصصة لحماية شاملة

# هندسة أمان MCP والضوابط

تتطلب تطبيقات MCP الحديثة نهجًا أمنيًا متعدد الطبقات يعالج كل من أمان البرمجيات التقليدية وتهديدات الذكاء الاصطناعي الخاصة. يستمر معيار MCP سريع التطور في نضج ضوابط الأمان، مما يمكن من تكامل أفضل مع هندسة أمان المؤسسات وأفضل الممارسات المعمول بها.

تُظهر بحوث [تقرير الدفاع الرقمي من مايكروسوفت](https://aka.ms/mddr) أن **98٪ من الاختراقات المبلغ عنها يمكن منعها من خلال جودة أمان محسنة**. تجمع أكثر استراتيجيات الحماية فعالية بين ممارسات الأمان الأساسية مع ضوابط مخصصة لـ MCP — حيث تبقى الإجراءات الأمنية الأساسية المثبتة أكثر تأثيرًا في تقليل مخاطر الأمان الإجمالية.

## المشهد الأمني الحالي

> **ملاحظة:** تعكس هذه المعلومات معايير أمان MCP اعتبارًا من **5 فبراير 2026**، متماشية مع **مواصفة MCP بتاريخ 2025-11-25**. يستمر بروتوكول MCP في التطور بسرعة، وقد تقدم التطبيقات المستقبلية أنماط مصادقة جديدة وضوابط محسنة. يُنصح دائمًا بالرجوع إلى [مواصفة MCP الحالية](https://spec.modelcontextprotocol.io/)، ومستودع [MCP على جِت هَب](https://github.com/modelcontextprotocol)، و[وثائق أفضل ممارسات الأمان](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) للحصول على الإرشادات الأحدث.

- **استخدام ميزات أمان أزور الأصلية**: الاستفادة من Azure Entra ID، Key Vault، إدارة API، وأمان محتوى الذكاء الاصطناعي

- **الحصول على كود إنتاجي**: الخروج بتنفيذات عملية ومختبرة

## أفضل 10 مخاطر أمنية لـ OWASP MCP

يستعرض [دليل أمان MCP على أزور من OWASP](https://microsoft.github.io/mcp-azure-security-guide/) أهم عشرة مخاطر أمنية في تطبيقات MCP:

| الخطر | الوصف | التخفيف عبر أزور |
|------|-------------|------------------|
| **MCP01** | سوء إدارة الرموز وكشف الأسرار | Azure Key Vault، الهوية المُدارة |
| **MCP02** | تصعيد الصلاحيات عبر زيادة النطاقات | RBAC، الوصول الشرطي |
| **MCP03** | تسميم الأدوات | التحقق من الأدوات، مراجعة السلامة |
| **MCP04** | هجمات سلسلة توريد البرمجيات والتلاعب بالاعتمادات | أمان جِت هَب المتقدم، فحص الاعتمادات |
| **MCP05** | حقن الأوامر والتنفيذ | التحقق من المدخلات، الحماية بالحاوية |
| **MCP06** | تحويل تدفق النوايا | أمان محتوى الذكاء الاصطناعي في أزور، دروع الأوامر |
| **MCP07** | ضعف في المصادقة والتفويض | Azure Entra ID، OAuth 2.1 مع PKCE |
| **MCP08** | غياب السجلات والمراقبة | Azure Monitor، Application Insights |
| **MCP09** | خوادم MCP الظلية | حوكمة مركز API، عزل الشبكة |
| **MCP10** | حقن السياق والمشاركة المفرطة | تصنيف البيانات، تقليل التعرض |

### تطور مصادقة MCP

شهدت مواصفة MCP تطورًا كبيرًا في نهج المصادقة والتفويض:

- **النهج الأصلي**: تطلبت المواصفات المبكرة من المطورين إنشاء خوادم مصادقة مخصصة، وكانت خوادم MCP تعمل كخوادم تفويض OAuth 2.0 تدير مصادقة المستخدم مباشرة
- **المعيار الحالي (2025-11-25)**: تسمح المواصفة المحدثة لخوادم MCP بتفويض المصادقة لمزودي هوية خارجيين (مثل Microsoft Entra ID)، مما يحسن وضع الأمان ويقلل تعقيدات التنفيذ
- **أمان طبقة النقل**: دعم معزز لآليات النقل الآمن مع أنماط مصادقة مناسبة للاتصال المحلي (STDIO) والبعيد (HTTP القابل للبث)

## أمان المصادقة والتفويض

### تحديات الأمان الحالية

تواجه تطبيقات MCP الحديثة عدة تحديات في المصادقة والتفويض:

### المخاطر ومسارات الهجوم

- **سوء تكوين منطق التفويض**: التنفيذ الخاطئ لتفويض الوصول في خوادم MCP يمكن أن يكشف عن بيانات حساسة ويطبق ضوابط وصول بشكل خاطئ
- **اختراق رمز OAuth**: سرقة رموز خادم MCP المحلي تمكن المهاجمين من انتحال الخوادم والوصول إلى خدمات أخرى
- **ثغرات تمرير الرموز**: التعامل غير السليم مع الرموز يخلق تجاوزات لضوابط الأمان وفجوات في المساءلة
- **الصلاحيات المفرطة**: خوادم MCP ذات الصلاحيات الزائدة تنتهك مبدأ أقل صلاحية وتوسع سطح الهجوم

#### تمرير الرموز: نمط معاد للأمان حرج

**تمرير الرموز ممنوع صراحةً** في مواصفة MCP الحالية للتفويض بسبب تبعات أمان شديدة:

##### تجاوز ضوابط الأمان  
- تنفذ خوادم MCP وواجهات برمجة التطبيقات التي تعمل لاحقًا ضوابط أمنية حرجة (حدود المعدل، التحقق من الطلبات، مراقبة المرور) تعتمد على التحقق الصحيح من الرموز
- يسمح الاستخدام المباشر للرمز من العميل إلى API بتجاوز هذه الحمايات الأساسية وتهديد هندسة الأمان

##### تحديات المساءلة والتدقيق  
- لا يمكن لخوادم MCP تمييز العملاء الذين يستخدمون رموز صادرة من مصادر أعلى، مما يكسر سجلات التدقيق  
- تسجل خوادم الموارد اللاحقة طلبات بأصول مضللة بدلاً من المنوسط الحقيقي لخادم MCP  
- يصعب التحقيق في الحوادث والتدقيق للامتثال بشكل كبير

##### مخاطر تسرب البيانات  
- تتيح المطالبات غير الموثقة للرموز للمهاجمين باستخدام رموز مسروقة استغلال خوادم MCP كوسطاء لتسريب البيانات  
- يسمح انتهاك حدود الثقة بأنماط وصول غير مصرح لها تتجاوز ضوابط الأمان المصممة

##### مسارات هجوم متعددة الخدمات  
- الرموز المخترقة التي يتم قبولها بواسطة خدمات متعددة تمكن التنقل الجانبي عبر الأنظمة المتصلة  
- قد تُنتهك افتراضات الثقة بين الخدمات عندما لا يمكن التحقق من مصدر الرموز

### ضوابط الأمان والتخفيفات

**متطلبات الأمان الحرجة:**

> **إلزامي**: يجب على خوادم MCP **عدم قبول أي رمز لم يُصدر صراحةً للخادم نفسه**

#### ضوابط المصادقة والتفويض

- **مراجعة تفويض صارمة**: إجراء تدقيق شامل لمنطق تفويض الوصول في خوادم MCP لضمان وصول المستخدمين والعملاء المقصودين فقط إلى الموارد الحساسة  
  - **دليل التنفيذ**: [إدارة API على أزور كبوابة مصادقة لخوادم MCP](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)  
  - **تكامل الهوية**: [استخدام Microsoft Entra ID لمصادقة خادم MCP](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **إدارة الرموز الآمنة**: تنفيذ [أفضل ممارسات مايكروسوفت للتحقق وإدارة دورة حياة الرموز](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens)  
  - التحقق من أن مطالبات الجمهور في الرموز تطابق هوية خادم MCP  
  - تنفيذ سياسات تدوير وانتهاء صلاحية مناسبة للرموز  
  - منع هجمات إعادة تشغيل الرموز والاستخدام غير المصرح به

- **تخزين الرموز المحمي**: تأمين تخزين الرموز بالتشفير أثناء الراحة والنقل  
  - **أفضل الممارسات**: [إرشادات تخزين وتشفير الرموز الآمن](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### تنفيذ ضوابط الوصول

- **مبدأ أقل صلاحية**: منح خوادم MCP الحد الأدنى من الصلاحيات الضرورية للوظائف المقصودة  
  - مراجعات دورية وتصحيحات لمنع زيادة الصلاحيات  
  - **وثائق مايكروسوفت**: [الوصول الآمن بأقل صلاحيات](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **التحكم في الوصول القائم على الأدوار (RBAC)**: تنفيذ تعيينات دور دقيقة  
  - تضييق نطاق الأدوار على الموارد والإجراءات المحددة  
  - تجنب الصلاحيات الواسعة أو غير الضرورية التي توسع سطح الهجوم

- **مراقبة الصلاحيات المستمرة**: تنفيذ تدقيق ومراقبة مستمرة للوصول  
  - مراقبة أنماط استخدام الصلاحيات للكشف عن الشذوذ  
  - معالجة الصلاحيات المفرطة أو غير المستخدمة بسرعة

## تهديدات الأمان الخاصة بالذكاء الاصطناعي

### هجمات حقن الأوامر وتلاعب الأدوات

تواجه تطبيقات MCP الحديثة مسارات هجوم متطورة خاصة بالذكاء الاصطناعي لا تستطيع التدابير الأمنية التقليدية التعامل معها بالكامل:

#### **حقن الأمر غير المباشر (حقن أوامر عبر المجالات)**

يمثل **حقن الأمر غير المباشر** واحدة من أكثر الثغرات الأمنية حرجًا في أنظمة MCP المدعومة بالذكاء الاصطناعي. حيث يُدرج المهاجمون تعليمات خبيثة ضمن محتويات خارجية — مثل الوثائق، صفحات الويب، البريد الإلكتروني، أو مصادر البيانات — التي تعالجها أنظمة الذكاء الاصطناعي لاحقًا كأوامر شرعية.

**سيناريوهات الهجوم:**
- **حقن مستندات**: تعليمات خبيثة مخفية ضمن مستندات تتم معالجتها مما يؤدي إلى تفعيل إجراءات غير مرغوبة من الذكاء الاصطناعي
- **استغلال محتوى الويب**: صفحات ويب مخترقة تحتوي على أوامر مضمنة تؤثر على سلوك الذكاء الاصطناعي عند سحب البيانات منها
- **هجمات البريد الإلكتروني**: أوامر خبيثة في الرسائل الإلكترونية تدفع مساعدين ذكاء اصطناعي لتسريب معلومات أو تنفيذ إجراءات غير مصرح بها
- **تلويث مصادر البيانات**: قواعد بيانات أو واجهات API مخترقة تقدم محتوى ملوث لأنظمة الذكاء الاصطناعي

**الأثر الواقعي**: يمكن لهذه الهجمات أن تؤدي إلى تسريب بيانات، انتهاك الخصوصية، توليد محتوى ضار، وتلاعب بتفاعلات المستخدم. لمزيد من التحليل، راجع [حقن الأوامر في MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/).

![مخطط هجوم حقن الأوامر](../../../translated_images/ar/prompt-injection.ed9fbfde297ca877.webp)

#### **هجمات تسميم الأدوات**

تستهدف **تسميم الأدوات** البيانات الوصفية التي تحدد أدوات MCP، مستغلة كيفية تفسير نماذج اللغة الكبيرة (LLMs) لوصف الأدوات ومعلماتها لاتخاذ قرارات التنفيذ.

**آليات الهجوم:**
- **تحريف البيانات الوصفية**: يدرج المهاجمون تعليمات خبيثة في وصف الأدوات، تعريفات المعلمات، أو أمثلة الاستخدام  
- **تعليمات غير مرئية**: أوامر مخفية في بيانات الأدوات الوصفية تعالجها نماذج الذكاء الاصطناعي لكنها لا تظهر للمستخدمين البشر  
- **تعديل الأدوات ديناميكيًا ("سحب السجاد")**: أدوات تمت الموافقة عليها من قبل المستخدم يتم تعديلها لاحقًا لأداء إجراءات خبيثة دون علم المستخدم  
- **حقن المعلمات**: محتوى خبيث مدمج في مخططات معلمات الأدوات يؤثر على سلوك النموذج

**مخاطر الخوادم المستضافة**: تزيد خوادم MCP البعيدة من المخاطر حيث يمكن تحديث تعريف الأدوات بعد موافقة المستخدم، مما يخلق سيناريوهات تتحول فيها الأدوات الآمنة سابقًا إلى أدوات خبيثة. لمزيد من التحليل الشامل، راجع [هجمات تسميم الأدوات (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks).

![مخطط هجوم تسميم الأدوات](../../../translated_images/ar/tool-injection.3b0b4a6b24de6bef.webp)

#### **مسارات هجوم إضافية للذكاء الاصطناعي**

- **حقن الأوامر عبر المجالات (XPIA)**: هجمات متطورة تستخدم محتوى من مجالات متعددة لتجاوز ضوابط الأمان
- **تعديل القدرات الديناميكي**: تغييرات في الوقت الفعلي لقدرات الأدوات التي تهرب من تقييمات الأمان الأولية  
- **تسميم نافذة السياق**: هجمات تستخدم نوافذ سياق كبيرة لإخفاء تعليمات خبيثة  
- **هجمات إرباك النموذج**: استغلال القيود النموذجية لإحداث سلوكيات غير متوقعة أو غير آمنة  


### تأثير مخاطر أمان الذكاء الاصطناعي

**عواقب ذات تأثير عالي:**
- **استخلاص البيانات**: الوصول غير المصرح به وسرقة بيانات حساسة للمؤسسات أو الأفراد  
- **اختراق الخصوصية**: كشف المعلومات الشخصية المعروفة (PII) وبيانات العمل السرية  
- **تلاعب النظام**: تعديلات غير مقصودة على أنظمة حيوية وسير العمل  
- **سرقة بيانات الاعتماد**: اختراق رموز المصادقة وبيانات اعتماد الخدمات  
- **الحركة الجانبية**: استخدام أنظمة الذكاء الاصطناعي المخترقة كنقاط انطلاق لهجمات أوسع على الشبكة  


### حلول أمان الذكاء الاصطناعي من مايكروسوفت

#### **دروع مطالبات الذكاء الاصطناعي: حماية متقدمة ضد هجمات حقن الطلبات**

تقدم مايكروسوفت **دروع مطالبات الذكاء الاصطناعي** دفاعًا شاملًا ضد هجمات حقن الطلبات المباشرة وغير المباشرة عبر طبقات أمنية متعددة:

##### **آليات الحماية الأساسية:**

1. **الكشف المتقدم والتصفية**  
   - تستخدم خوارزميات التعلم الآلي وتقنيات معالجة اللغة الطبيعية للكشف عن التعليمات الخبيثة في المحتوى الخارجي  
   - تحليل في الوقت الحقيقي للمستندات وصفحات الويب والبريد الإلكتروني ومصادر البيانات للكشف عن التهديدات المدمجة  
   - فهم سياقي لأنماط المطالبات الشرعية مقابل الخبيثة  

2. **تقنيات الإضاءة**  
   - تمييز بين تعليمات النظام الموثوقة والمدخلات الخارجية المحتمل أن تكون مخترقة  
   - أساليب تحويل النصوص لتعزيز ملاءمة النموذج مع عزل المحتوى الخبيث  
   - تساعد أنظمة الذكاء الاصطناعي على الحفاظ على التسلسل الصحيح للتعليمات وتجاهل الأوامر المحقونة  

3. **أنظمة الفواصل وعلامات البيانات**  
   - تحديد حدود صريحة بين رسائل النظام الموثوقة ونص المدخلات الخارجية  
   - علامات خاصة تسلط الضوء على الحدود بين مصادر البيانات الموثوقة وغير الموثوقة  
   - الفصل الواضح يمنع ارتباك التعليمات وتنفيذ الأوامر غير المصرح بها  

4. **معلومات استخبارات التهديد المستمرة**  
   - تراقب مايكروسوفت باستمرار أنماط الهجوم الناشئة وتحدث آليات الدفاع  
   - بحث نشط ومبكر عن تقنيات حقن جديدة ومتجهات هجوم  
   - تحديثات منتظمة لنماذج الأمان للحفاظ على الفعالية ضد التهديدات المتطورة  

5. **تكامل Azure Content Safety**  
   - جزء من مجموعة أمان محتوى Azure الشاملة للذكاء الاصطناعي  
   - كشف إضافي لمحاولات كسر الحماية، والمحتوى الضار، وانتهاكات سياسات الأمان  
   - ضوابط أمان موحدة عبر مكونات تطبيقات الذكاء الاصطناعي

## تهديدات أمان MCP المتقدمة

### ثغرات اختطاف الجلسات

يمثل **اختطاف الجلسة** نقطة هجوم حرجة في تنفيذات MCP التي تعتمد على الحالة، حيث يحصل أطراف غير مصرح لهم على معرفات جلسات شرعية ويستخدمونها للتنكر كعملاء وأداء إجراءات غير مصرح بها.

#### **سيناريوهات الهجوم والمخاطر**

- **حقن مطالبات اختطاف الجلسة**: المهاجمون الذين يمتلكون معرفات جالسة مسروقة يحقنون أحداثًا خبيثة في الخوادم التي تشارك حالة الجلسة، مما قد يؤدي لنشاطات ضارة أو الوصول إلى بيانات حساسة  
- **التنكر المباشر**: معرفات الجلسة المسروقة تتيح مكالمات مباشرة لخادم MCP متجاوزة المصادقة، معاملة المهاجمين كمستخدمين شرعيين  
- **انقطاع التدفقات القابلة للاستئناف المخترقة**: يستطيع المهاجمون إنهاء الطلبات مبكرًا، مما يجعل العملاء الشرعيين يستأنفون بمحتوى قد يكون خبيثًا  

#### **ضوابط الأمان لإدارة الجلسات**

**متطلبات حاسمة:**
- **التحقق من التفويض**: يجب على خوادم MCP التي تنفذ التفويض التحقق من جميع الطلبات الواردة وعدم الاعتماد على الجلسات للمصادقة  
- **توليد جلسات آمنة**: استخدام معرفات جلسات عشوائية غير محددة مسبقًا وآمنة تشفيرًا  
- **ربط معرفات الجلسات بالمستخدم**: ربط معرف الجلسة بمعلومات المستخدم باستخدام تنسيق مثل <user_id>:<session_id> لمنع سوء استخدام الجلسات عبر المستخدمين  
- **إدارة دورة حياة الجلسة**: تنفيذ نظام انتهاء صلاحية وتدوير وإبطال صحيح لتقليل نوافذ الضعف  
- **أمان النقل**: استخدام HTTPS إلزاميًا لكل الاتصالات لمنع اعتراض معرفات الجلسة  


### مشكلة الوكيل المشوش

تحدث مشكلة **الوكيل المشوش** عندما تتصرف خوادم MCP كوسطاء مصادقة بين العملاء وخدمات الطرف الثالث، مما يخلق فرصًا لتجاوز التفويض عبر استغلال معرفات العملاء الثابتة.

#### **آليات الهجوم والمخاطر**

- **تجاوز الموافقة اعتمادًا على الكوكيز**: مصادقة المستخدم السابقة تخلق كوكيز موافقة يستغلها المهاجمون عبر طلبات تفويض خبيثة تحتوي على URI إعادة توجيه مصممة  
- **سرقة رمز التفويض**: كوكيز الموافقة القائمة قد تجعل خوادم التفويض تتخطى شاشات الموافقة، معيدة التوجيه إلى نقاط نهاية تسيطر عليها الجهات المهاجمة  
- **الوصول غير المصرح به لواجهة برمجة التطبيقات**: رموز التفويض المسروقة تمكن تبادل الرموز وانتحال المستخدم بدون موافقة صريحة  

#### **استراتيجيات التخفيف**

**ضوابط إلزامية:**
- **متطلبات الموافقة الصريحة**: يجب على خوادم الوكيل في MCP التي تستخدم معرفات عملاء ثابتة طلب موافقة المستخدم لكل عميل مُسجل ديناميكيًا  
- **تنفيذ أمان OAuth 2.1**: اتباع أفضل ممارسات أمان OAuth الحالية بما في ذلك PKCE لكل طلبات التفويض  
- **التحقق الصارم من العميل**: تنفيذ تحقق صارم من URIs الإعادة ومعرفات العملاء لمنع الاستغلال  


### ثغرات تمرير الرموز  

تمثل ثغرات **تمرير الرموز** نمطاً مضادًا صريحًا حيث تقبل خوادم MCP رموز العملاء دون تحقق صحيح وتعيد توجيهها إلى واجهات برمجة التطبيقات التالية، ما ينتهك مواصفات تفويض MCP.

#### **تداعيات الأمان**

- **التجاوز في التحكم**: الاستخدام المباشر لرموز العميل إلى واجهة API يتجاوز ضوابط تحديد المعدل والتحقق والمراقبة  
- **فساد أثر التدقيق**: الرموز الصادرة من الأعلى تعيق التعريف الدقيق للعميل، مما يحطم قدرات التحقيق في الحوادث  
- **استخلاص البيانات عبر الوكيل**: الرموز غير الموثوقة تمكن الجهات الخبيثة من استخدام الخادم كوسيط للوصول غير المصرح به للبيانات  
- **انتهاكات حدود الثقة**: تنتهك فرضيات الثقة الخاصة بالخدمات المستقبلة عندما لا يمكن التحقق من مصدر الرموز  
- **توسع الهجمات عبر خدمات متعددة**: قبول الرموز المخترقة عبر خدمات متعددة يمكّن الحركة الجانبية  

#### **الضوابط الأمنية المطلوبة**

**متطلبات غير قابلة للتفاوض:**
- **التحقق من الرموز**: يجب ألا تقبل خوادم MCP رموزًا لم تصدر صراحةً لخادم MCP  
- **التحقق من الجمهور**: دائماً تحقق من أن مطالبات جمهور الرمز تطابق هوية خادم MCP  
- **إدارة دورة حياة الرمز**: تنفيذ رموز وصول قصيرة الأمد مع ممارسات تدوير آمنة  


## أمان سلسلة التوريد لأنظمة الذكاء الاصطناعي

تطور أمان سلسلة التوريد ليشمل أكثر من الاعتماديات التقليدية للبرمجيات ليشتمل النظام البيئي الكامل للذكاء الاصطناعي. يجب على تنفيذات MCP الحديثة التحقق بدقة من جميع مكونات الذكاء الاصطناعي ومراقبتها، حيث يضيف كل منها نقاط ضعف محتملة قد تهدد سلامة النظام.

### مكونات موسعة لسلسلة توريد الذكاء الاصطناعي

**الاعتماديات البرمجية التقليدية:**
- المكتبات والأطر مفتوحة المصدر  
- صور الحاويات والأنظمة الأساسية  
- أدوات التطوير وأنابيب الإنشاء  
- مكونات وخدمات البنية التحتية  

**عناصر خاصة بسلسلة توريد الذكاء الاصطناعي:**
- **نماذج أساسية**: النماذج المدربة مسبقًا من مزودين مختلفين وتتطلب التحقق من مصدرها  
- **خدمات التضمين**: خدمات التمثيل الشعاعي والبحث الدلالي الخارجية  
- **مزودو السياق**: مصادر البيانات، قواعد المعرفة، ومستودعات الوثائق  
- **واجهات برمجة التطبيقات الخارجية**: خدمات الذكاء الاصطناعي الخارجية، أنابيب التعلم الآلي، ونقاط النهاية لمعالجة البيانات  
- **قطع نموذجية**: الأوزان، التكوينات، والنماذج الدقيقة المنفذة  
- **مصادر بيانات التدريب**: مجموعات البيانات المستخدمة في التدريب والتخصيص  

### استراتيجية شاملة لأمان سلسلة التوريد

#### **التحقق والثقة في المكونات**
- **التحقق من المصدر**: التحقق من الأصل، الترخيص، وسلامة جميع مكونات الذكاء الاصطناعي قبل الدمج  
- **تقييم الأمان**: إجراء فحوصات الثغرات والاستعراضات الأمنية للنماذج، مصادر البيانات، وخدمات الذكاء الاصطناعي  
- **تحليل السمعة**: تقييم سجل الأمان وممارسات مزودي خدمات الذكاء الاصطناعي  
- **التحقق من الامتثال**: التأكد من مطابقة كل المكونات لمتطلبات الأمان والتنظيم المؤسسية  

#### **أنابيب النشر الآمنة**  
- **أمان CI/CD آلي**: دمج فحوصات الأمان عبر أنابيب النشر الآلية  
- **سلامة القطع**: تنفيذ تحقق تشفير لجميع القطع المنشورة (الكود، النماذج، التكوينات)  
- **نشر مرحلي**: استخدام استراتيجيات نشر تدريجية مع تحقق أمني في كل مرحلة  
- **مستودعات القطع الموثوقة**: النشر فقط من سجلات ومستودعات معتمدة وآمنة  

#### **المراقبة والاستجابة المستمرة**
- **مسح الاعتماديات**: مراقبة مستمرة للثغرات في كل اعتماديات البرمجيات ومكونات الذكاء الاصطناعي  
- **مراقبة النماذج**: تقييم مستمر لسلوك النموذج، انحراف الأداء، وشذوذات الأمان  
- **تتبع صحة الخدمات**: مراقبة خدمات الذكاء الاصطناعي الخارجية للتوفر، الحوادث الأمنية، وتغييرات السياسات  
- **دمج استخبارات التهديدات**: إدماج مصادر معلومات التهديدات الخاصة بأمان الذكاء الاصطناعي والتعلم الآلي  

#### **ضوابط الوصول وأقل الامتيازات**
- **أذونات مكونات على مستوى دقيق**: تقييد الوصول للنماذج والبيانات والخدمات بحسب الضرورة العملية  
- **إدارة حسابات الخدمات**: تنفيذ حسابات خدمات مخصصة بأدنى الصلاحيات المطلوبة  
- **تقسيم الشبكة**: عزل مكونات الذكاء الاصطناعي وتقليل وصول الشبكة بين الخدمات  
- **ضوابط بوابات API**: استخدام بوابات API مركزية للتحكم والمراقبة على وصول خدمات الذكاء الاصطناعي الخارجية  

#### **الاستجابة للحوادث والتعافي**
- **إجراءات الاستجابة السريعة**: عمليات مثبتة لترقيع أو استبدال مكونات الذكاء الاصطناعي المخترقة  
- **تدوير بيانات الاعتماد**: أنظمة آلية لتدوير الأسرار ومفاتيح API وبيانات الاعتماد  
- **قدرات التراجع**: القدرة على الرجوع السريع إلى نسخ سابقة معروفة وصالحة من مكونات الذكاء الاصطناعي  
- **التعافي بعد اختراق سلسلة التوريد**: إجراءات محددة للاستجابة لمخاطر الاختراق في خدمات الذكاء الاصطناعي من الأعلى  

### أدوات وتكامل الأمان من مايكروسوفت

يقدم **GitHub Advanced Security** حماية شاملة لسلسلة التوريد تشمل:  
- **مسح الأسرار**: الكشف التلقائي عن بيانات الاعتماد، مفاتيح API، والرموز في المستودعات  
- **مسح الاعتماديات**: تقييم الثغرات في الاعتماديات والمكتبات مفتوحة المصدر  
- **تحليل CodeQL**: تحليل كود ثابت لاكتشاف ثغرات الأمان ومشاكل الترميز  
- **رؤى سلسلة التوريد**: رؤية شاملة لحالة الاعتماديات والأمان  

**تكامل Azure DevOps و Azure Repos:**  
- تكامل سلس لفحوصات الأمان عبر منصات تطوير مايكروسوفت  
- فحوصات أمان آلية في Azure Pipelines لأحمال الذكاء الاصطناعي  
- تطبيق السياسات لنشر مكونات الذكاء الاصطناعي الآمن  

**ممارسات مايكروسوفت الداخلية:**  
تطبق مايكروسوفت ممارسات أمان سلسلة التوريد الشاملة عبر كل المنتجات. تعرف على الأساليب المثبتة في [رحلة تأمين سلسلة توريد البرمجيات لدى مايكروسوفت](https://devblogs.microsoft.com/engineering-at-microsoft/the-journey-to-secure-the-software-supply-chain-at-microsoft/).


## أفضل ممارسات الأمان الأساسية للبنية التحتية

ترث تنفيذات MCP وتعزز على مواقف الأمان الحالية لمؤسستك. تعزيز الممارسات الأمنية الأساسية يحسن بدرجة كبيرة أمن أنظمة الذكاء الاصطناعي ونشر MCP.

### أساسيات الأمان الجوهرية

#### **ممارسات التطوير الآمن**
- **الامتثال لـ OWASP**: الحماية من [أفضل 10 ثغرات تطبيقات ويب من OWASP](https://owasp.org/www-project-top-ten/)  
- **حمايات خاصة بالذكاء الاصطناعي**: تنفيذ ضوابط لـ [أفضل 10 ثغرات للنماذج اللغوية الكبيرة](https://genai.owasp.org/download/43299/?tmstv=1731900559)  
- **إدارة الأسرار الآمنة**: استخدام خزائن مخصصة للرموز، مفاتيح API، وبيانات التكوين الحساسة  
- **التشفير من الطرف إلى الطرف**: تنفيذ اتصالات آمنة عبر كل مكونات التطبيق وتدفقات البيانات  
- **التحقق من المدخلات**: تحقق صارم من كل مدخلات المستخدم، معاملات API، ومصادر البيانات  

#### **تقوية البنية التحتية**
- **المصادقة متعددة العوامل**: إلزامية للمستخدمين الإداريين وحسابات الخدمات  
- **إدارة الترقيعات**: تطبيق ترقيعات منتظمة وآلية للنظم التشغيلية، الأُطر، والاعتماديات  
- **تكامل مزود الهوية**: إدارة مركزية للهويات عبر موفري الهوية المؤسسيين (Microsoft Entra ID، Active Directory)  
- **تقسيم الشبكة**: العزل المنطقي لمكونات MCP لتقليل فرص الحركة الجانبية  
- **مبدأ أقل امتياز**: منح أدنى الصلاحيات المطلوبة لكل مكونات النظام والحسابات  

#### **المراقبة والكشف الأمني**
- **تسجيل شامل**: تسجيل تفصيلي لأنشطة تطبيقات الذكاء الاصطناعي، بما في ذلك تفاعلات عملاء وخوادم MCP  
- **تكامل SIEM**: إدارة مركزية لمعلومات وأحداث الأمان للكشف عن الشذوذ  
- **تحليلات سلوكية**: مراقبة مدعومة بالذكاء الاصطناعي لاكتشاف الأنماط غير المعتادة في سلوك النظام والمستخدم  
- **معلومات التهديدات**: دمج مصادر خارجية لمعلومات التهديدات ومؤشرات الاختراق  
- **الاستجابة للحوادث**: إجراءات معرفّة جيدًا لاكتشاف الحوادث الأمنية والاستجابة والتعافي  

#### **هندسة الثقة الصفرية**
- **لا تثق أبدًا، تحقق دائمًا**: التحقق المستمر من المستخدمين، الأجهزة، واتصالات الشبكة  
- **التقسيم الصغير (Micro-Segmentation)**: ضوابط شبكية دقيقة تعزل حمولات العمل والخدمات الفردية  
- **أمان مركزي الهوية**: سياسات أمان مبنية على الهويات الموثقة بدلاً من الموقع الشبكي  
- **تقييم المخاطر المستمر**: تقويم ديناميكي لموقف الأمان اعتمادًا على السياق والسلوك الحالي  
- **الوصول الشرطي**: ضوابط وصول تتكيف بناءً على عوامل المخاطر، الموقع، وثقة الجهاز  

### أنماط التكامل المؤسسي

#### **تكامل منظومة أمان مايكروسوفت**
- **Microsoft Defender for Cloud**: إدارة شاملة لوضع أمان السحابة  
- **Azure Sentinel**: منصة SIEM و SOAR أصلية للسحابة لحماية أحمال الذكاء الاصطناعي  
- **Microsoft Entra ID**: إدارة هوية المؤسسات والوصول بسياسات وصول شرطي  
- **Azure Key Vault**: إدارة الأسرار المركزية مدعومة بوحدة أمان الأجهزة (HSM)  
- **Microsoft Purview**: حوكمة البيانات والامتثال لمصادر بيانات سير العمل والذكاء الاصطناعي  

#### **الامتثال والحوكمة**
- **الامتثال التنظيمي**: ضمان تلبية تطبيقات MCP للمتطلبات التنظيمية الخاصة بالصناعة (GDPR, HIPAA, SOC 2)  
- **تصنيف البيانات**: تصنيف ومعالجة مناسبة للبيانات الحساسة المعالجة بواسطة أنظمة الذكاء الاصطناعي  
- **مسارات التدقيق**: تسجيل شامل للامتثال التنظيمي والتحقيقات الجنائية  
- **ضوابط الخصوصية**: تنفيذ مبادئ الخصوصية من التصميم في هندسة نظم الذكاء الاصطناعي  
- **إدارة التغيير**: عمليات رسمية لمراجعة الأمان على تعديلات أنظمة الذكاء الاصطناعي  

تُنشئ هذه الممارسات الأساسية قاعدة أمان متينة تعزز فعالية ضوابط أمان MCP المحددة وتوفر حماية شاملة لتطبيقات الذكاء الاصطناعي.

## خلاصة أمنية رئيسية
- **نهج الأمان متعدد الطبقات**: دمج الممارسات الأمنية الأساسية (البرمجة الآمنة، أقل الامتيازات، التحقق من سلسلة التوريد، المراقبة المستمرة) مع الضوابط الخاصة بالذكاء الاصطناعي للحماية الشاملة

- **مشهد التهديدات الخاص بالذكاء الاصطناعي**: تواجه أنظمة MCP مخاطر فريدة تشمل حقن الأوامر، تسميم الأدوات، اختطاف الجلسات، مشكلات الوكيل المحتار، ثغرات تمرير الرموز، والأذونات المفرطة التي تتطلب تدابير متخصصة

- **تفوق المصادقة والتفويض**: تنفيذ مصادقة قوية باستخدام موفري الهوية الخارجيين (Microsoft Entra ID)، فرض التحقق الصحيح من الرموز، وعدم قبول الرموز غير الصادرة صراحةً لخادم MCP الخاص بك

- **منع هجمات الذكاء الاصطناعي**: نشر Microsoft Prompt Shields وAzure Content Safety للدفاع ضد هجمات حقن الأوامر غير المباشرة وتسميم الأدوات، مع التحقق من بيانات تعريف الأدوات والمراقبة للتغييرات الديناميكية

- **أمان الجلسة والنقل**: استخدام معرفات جلسة آمنة تشفيرياً وغير حتمية مرتبطة بهويات المستخدم، تنفيذ إدارة دورة حياة الجلسة بشكل صحيح، وعدم استخدام الجلسات للمصادقة أبداً

- **أفضل ممارسات أمان OAuth**: منع هجمات الوكيل المحتار من خلال موافقة المستخدم الصريحة للعملاء المسجلين ديناميكياً، تنفيذ OAuth 2.1 بشكل صحيح مع PKCE، والتحقق الصارم من عناوين URI لإعادة التوجيه

- **مبادئ أمان الرموز**: تجنب أنماط المرور السيئة للرموز، التحقق من مطالبات جمهور الرمز، تنفيذ رموز قصيرة العمر مع تدوير آمن، والحفاظ على حدود ثقة واضحة

- **أمان شامل لسلسلة التوريد**: التعامل مع جميع مكونات نظام الذكاء الاصطناعي (النماذج، التجسيدات، موفري السياق، واجهات برمجة التطبيقات الخارجية) بنفس الصرامة الأمنية المطبقة على تبعيات البرمجيات التقليدية

- **التطور المستمر**: البقاء على اطلاع بالمواصفات المتطورة بسرعة لـ MCP، المساهمة في معايير مجتمع الأمن، والحفاظ على مواقف أمان تكيفية مع نضوج البروتوكول

- **تكامل أمان مايكروسوفت**: الاستفادة من نظام أمان مايكروسوفت الشامل (Prompt Shields، Azure Content Safety، GitHub Advanced Security، Entra ID) لتعزيز حماية نشر MCP
