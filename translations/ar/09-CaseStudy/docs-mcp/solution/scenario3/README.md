# السيناريو 3: التوثيق داخل المحرر باستخدام خادم MCP في VS Code

## نظرة عامة

في هذا السيناريو، ستتعلم كيفية جلب مستندات Microsoft Learn مباشرة إلى بيئة Visual Studio Code الخاصة بك باستخدام خادم MCP. بدلاً من التنقل باستمرار بين علامات تبويب المتصفح للبحث عن التوثيق، يمكنك الوصول إلى المستندات الرسمية والبحث فيها والرجوع إليها مباشرة داخل المحرر الخاص بك. هذه الطريقة تبسط سير عملك، وتحافظ على تركيزك، وتتيح التكامل السلس مع أدوات مثل GitHub Copilot.

- البحث وقراءة المستندات داخل VS Code دون مغادرة بيئة البرمجة الخاصة بك.
- الرجوع إلى التوثيق وإدراج الروابط مباشرة في ملفات README أو ملفات الدورة التدريبية.
- استخدام GitHub Copilot وMCP معًا لسير عمل سلس مدعوم بالذكاء الاصطناعي.

## أهداف التعلم

بحلول نهاية هذا الفصل، ستفهم كيفية إعداد واستخدام خادم MCP داخل VS Code لتعزيز عمل التوثيق والتطوير الخاص بك. ستكون قادرًا على:

- تكوين مساحة العمل لاستخدام خادم MCP للبحث في التوثيق.
- البحث وإدراج المستندات مباشرة من داخل VS Code.
- دمج قوة GitHub Copilot وMCP للحصول على سير عمل أكثر إنتاجية مدعوم بالذكاء الاصطناعي.

هذه المهارات ستساعدك على البقاء مركزًا، تحسين جودة التوثيق، وزيادة إنتاجيتك كمطور أو كاتب تقني.

## الحل

لتحقيق وصول التوثيق داخل المحرر، ستتبع سلسلة من الخطوات التي تدمج خادم MCP مع VS Code وGitHub Copilot. هذا الحل مثالي لمؤلفي الدورات، وكتاب التوثيق، والمطورين الذين يرغبون في الحفاظ على تركيزهم داخل المحرر أثناء العمل مع المستندات وCopilot.

- إضافة روابط مرجعية بسرعة إلى ملف README أثناء كتابة الدورة أو وثائق المشروع.
- استخدام Copilot لتوليد الأكواد وMCP للعثور الفوري على المستندات ذات الصلة والاستشهاد بها.
- البقاء مركزًا في المحرر وزيادة الإنتاجية.

### الدليل خطوة بخطوة

للبدء، اتبع هذه الخطوات. لكل خطوة، يمكنك إضافة لقطة شاشة من مجلد الأصول لتوضيح العملية بصريًا.

1. **أضف تكوين MCP:**
   في جذر مشروعك، أنشئ ملف `.vscode/mcp.json` وأضف التكوين التالي:
   ```json
   {
     "servers": {
       "LearnDocsMCP": {
         "url": "https://learn.microsoft.com/api/mcp"
       }
     }
   }
   ```
   هذا التكوين يخبر VS Code كيفية الاتصال بـ [`خادم مستندات Microsoft Learn MCP`](https://github.com/MicrosoftDocs/mcp).
   
   ![الخطوة 1: إضافة mcp.json إلى مجلد .vscode](../../../../../../translated_images/ar/step1-mcp-json.c06a007fccc3edfa.webp)
    
2. **افتح لوحة دردشة GitHub Copilot:**
   إذا لم تكن قد ثبتت امتداد GitHub Copilot بعد، انتقل إلى عرض الامتدادات في VS Code وقم بتثبيته. يمكنك تنزيله مباشرة من [سوق Visual Studio Code](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot-chat). ثم افتح لوحة دردشة Copilot من الشريط الجانبي.

   ![الخطوة 2: فتح لوحة دردشة Copilot](../../../../../../translated_images/ar/step2-copilot-panel.f1cc86e9b9b8cd1a.webp)

3. **تمكين وضع العميل والتحقق من الأدوات:**
   في لوحة دردشة Copilot، قم بتمكين وضع العميل.

   ![الخطوة 3: تمكين وضع العميل والتحقق من الأدوات](../../../../../../translated_images/ar/step3-agent-mode.cdc32520fd7dd1d1.webp)

   بعد تمكين وضع العميل، تحقق من إدراج خادم MCP كواحدة من الأدوات المتاحة. هذا يضمن أن وكيل Copilot يمكنه الوصول إلى خادم التوثيق لجلب المعلومات ذات الصلة.

   ![الخطوة 3: التحقق من أداة خادم MCP](../../../../../../translated_images/ar/step3-verify-mcp-tool.76096a6329cbfecd.webp)

4. **ابدأ محادثة جديدة واطلب من الوكيل:**
   افتح محادثة جديدة في لوحة دردشة Copilot. يمكنك الآن طرح استفسارات التوثيق الخاصة بك على الوكيل. سيستخدم الوكيل خادم MCP لجلب وعرض توثيق Microsoft Learn ذي الصلة مباشرة في المحرر الخاص بك.

   - *"أحاول كتابة خطة دراسة الموضوع X. سأدرسه لمدة 8 أسابيع، لكل أسبوع اقترح المحتوى الذي يجب علي أخذه."*

   ![الخطوة 4: اطلب من الوكيل في الدردشة](../../../../../../translated_images/ar/step4-prompt-chat.12187bb001605efc.webp)

5. **استعلام حي:**

   > لنأخذ استعلامًا حيًا من قسم [#get-help](https://discord.gg/D6cRhjHWSC) في Discord الخاص بـ Microsoft Foundry ([عرض الرسالة الأصلية](https://discord.com/channels/1113626258182504448/1385498306720829572)):
   
   *"أنا أبحث عن إجابات حول كيفية نشر حل متعدد العملاء مع وكلاء ذكاء اصطناعي تم تطويرهم على Azure AI Foundry. أرى أنه لا توجد طريقة نشر مباشرة، مثل قنوات Copilot Studio. فكيف هي الطرق المختلفة لتنفيذ هذا النشر لمستخدمي المؤسسات للتفاعل وإنجاز العمل؟ هناك العديد من المقالات / المدونات التي تقول إنه يمكننا استخدام خدمة Azure Bot للقيام بهذه المهمة والتي يمكن أن تعمل كجسر بين MS Teams ووكلاء Azure AI Foundry، فهل هذا سينجح إذا قمت بإعداد بوت Azure يتصل بوكيل المنسق (Orchestrator Agent) على Azure AI Foundry عبر وظيفة Azure لأداء التنسيق أم أحتاج إلى إنشاء وظيفة Azure لكل واحد من وكلاء الذكاء الاصطناعي كجزء من الحل متعدد العملاء لأداء التنسيق ضمن إطار بوت؟ أي اقتراحات أخرى مرحب بها."*

   ![الخطوة 5: الاستعلامات الحية](../../../../../../translated_images/ar/step5-live-queries.49db3e4a50bea273.webp)

   سيرد الوكيل بروابط ملائمة للمستندات وملخصات يمكنك بعدها إدراجها مباشرة في ملفات العلامات الخاصة بك أو استخدامها كمرجع في شفراتك.
   
### استعلامات نموذجية

إليك بعض الاستعلامات النموذجية التي يمكنك تجربتها. هذه الاستعلامات ستوضح كيف يمكن لخادم MCP وCopilot العمل معًا لتوفير توثيق فوري وذكي ومرتبط بالسياق دون مغادرة VS Code:

- "أرني كيفية استخدام مُشغلات Azure Functions."
- "أدخل رابطًا إلى التوثيق الرسمي لـ Azure Key Vault."
- "ما هي الممارسات الأفضل لتأمين موارد Azure؟"
- "ابحث عن دليل بدء سريع لخدمات Azure AI."

هذه الاستعلامات ستوضح كيف يمكن لخادم MCP وCopilot العمل معًا لتوفير توثيق فوري وذكي ومرتبط بالسياق دون مغادرة VS Code.
