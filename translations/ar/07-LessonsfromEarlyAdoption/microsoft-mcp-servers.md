تم اختيار كل خادم في هذا الدليل بناءً على الاستخدام الفعلي وردود فعل المطورين. ستكتشف ليس فقط ماذا يفعل كل خادم، بل لماذا هو مهم وكيفية تحقيق أقصى استفادة منه في مشاريعك الخاصة. سواء كنت جديدًا تمامًا على MCP أو تسعى لتوسيع إعداداتك الحالية، تمثل هذه الخوادم بعضًا من الأدوات الأكثر عملية وتأثيرًا المتاحة في نظام Microsoft البيئي.

## 🔧 فهم خوادم MCP: دليل للمبتدئين

### ما هي خوادم MCP؟

كمبتدئ في بروتوكول نموذج السياق (MCP)، قد تتساءل: "ما هو بالضبط خادم MCP، ولماذا يجب أن أهتم؟" لنبدأ بتشبيه بسيط.

تخيل خوادم MCP كمساعدين متخصصين يساعدون رفيقك البرمجي الذكي (مثل GitHub Copilot) على الاتصال بالأدوات والخدمات الخارجية. تمامًا كما تستخدم تطبيقات مختلفة على هاتفك لمهام مختلفة—واحد للطقس، وآخر للملاحة، وآخر للبنك—تعطي خوادم MCP لمساعد الذكاء الاصطناعي الخاص بك القدرة على التفاعل مع أدوات وخدمات تطوير مختلفة.

### المشكلة التي تحلها خوادم MCP

قبل خوادم MCP، إذا أردت:
- التحقق من موارد Azure الخاصة بك
- إنشاء قضية في GitHub
- استعلام قاعدة بياناتك
- البحث في الوثائق

كان عليك التوقف عن البرمجة، وفتح متصفح، والانتقال إلى الموقع المناسب، وأداء هذه المهام يدويًا. هذا التبديل المستمر في السياق يكسر تدفق العمل ويقلل الإنتاجية.

### كيف تحوّل خوادم MCP تجربة التطوير الخاصة بك

مع خوادم MCP، يمكنك البقاء في بيئة التطوير الخاصة بك (VS Code، Visual Studio، إلخ) وطلب من مساعد الذكاء الاصطناعي التعامل مع هذه المهام ببساطة. على سبيل المثال:

**بدلاً من هذا سير العمل التقليدي:**
1. التوقف عن البرمجة
2. فتح المتصفح
3. الانتقال إلى بوابة Azure
4. البحث عن تفاصيل حساب التخزين
5. العودة إلى VS Code
6. متابعة البرمجة

**يمكنك الآن القيام بهذا:**
1. اسأل الذكاء الاصطناعي: "ما حالة حسابات التخزين الخاصة بي في Azure؟"
2. استمر في البرمجة مع المعلومات المقدمة

### الفوائد الرئيسية للمبتدئين

#### 1. 🔄 **ابق في حالة تدفق العمل الخاصة بك**
- لا مزيد من التبديل بين تطبيقات متعددة
- حافظ على تركيزك على الكود الذي تكتبه
- قلل العبء الذهني لإدارة أدوات مختلفة

#### 2. 🤖 **استخدم اللغة الطبيعية بدلاً من الأوامر المعقدة**
- بدلًا من حفظ بناء جملة SQL، وصف المعلومات التي تحتاجها
- بدلًا من تذكر أوامر Azure CLI، اشرح ما تريد إنجازه
- دع الذكاء الاصطناعي يتعامل مع التفاصيل التقنية بينما تركز على المنطق

#### 3. 🔗 **ربط أدوات متعددة معًا**
- إنشاء تدفقات عمل قوية عن طريق دمج خدمات مختلفة
- مثال: "احصل على كل قضايا GitHub الحديثة وأنشئ عناصر عمل في Azure DevOps المقابلة"
- بناء أتمتة بدون كتابة نصوص معقدة

#### 4. 🌐 **الوصول إلى نظام بيئي متنامٍ**
- الاستفادة من الخوادم التي تبنيها Microsoft وGitHub وشركات أخرى
- المزج والمطابقة بين أدوات من بائعين مختلفين بسلاسة
- الانضمام إلى نظام بيئي معياري يعمل عبر مختلف مساعدي الذكاء الاصطناعي

#### 5. 🛠️ **التعلم من خلال التطبيق**
- ابدأ بخوادم مُعدة مسبقًا لفهم المفاهيم
- بناء خوادمك الخاصة تدريجيًا مع اكتساب خبرة أكبر
- استخدم SDKs والوثائق المتاحة لتوجيه تعلمك

### مثال عملي للمبتدئين

لنفترض أنك جديد في تطوير الويب وتعمل على مشروعك الأول. إليك كيف يمكن أن تساعدك خوادم MCP:

**النهج التقليدي:**
```
1. Code a feature
2. Open browser → Navigate to GitHub
3. Create an issue for testing
4. Open another tab → Check Azure docs for deployment
5. Open third tab → Look up database connection examples
6. Return to VS Code
7. Try to remember what you were doing
```

**مع خوادم MCP:**
```
1. Code a feature
2. Ask AI: "Create a GitHub issue for testing this login feature"
3. Ask AI: "How do I deploy this to Azure according to the docs?"
4. Ask AI: "Show me the best way to connect to my database"
5. Continue coding with all the information you need
```

## البدء مع MCP في VS Code و Visual Studio 🚀

إعداد هذه الخوادم MCP سهل إذا كنت تستخدم Visual Studio Code أو Visual Studio 2022 مع GitHub Copilot.

### إعداد VS Code

فيما يلي العملية الأساسية لـ VS Code:

1. **تمكين وضع الوكيل**: في VS Code، قم بالتبديل إلى وضع الوكيل في نافذة محادثة Copilot
2. **تكوين خوادم MCP**: أضف تكوينات الخادم إلى ملف settings.json الخاص بـ VS Code
3. **تشغيل الخوادم**: انقر على زر "ابدأ" لكل خادم تريد استخدامه
4. **اختيار الأدوات**: اختر خوادم MCP التي تريد تمكينها للجلسة الحالية

للحصول على تعليمات إعداد مفصلة، انظر [توثيق MCP لـ VS Code](https://code.visualstudio.com/docs/copilot/copilot-mcp).

> **💡 نصيحة احترافية: إدارة خوادم MCP كالمحترفين!**
> 
> تضمين VS Code الآن [واجهة جديدة سهلة لإدارة خوادم MCP المثبتة](https://code.visualstudio.com/docs/copilot/chat/mcp-servers#_use-mcp-tools-in-agent-mode)! لديك وصول سريع لبدء، إيقاف، وإدارة أي خادم MCP مثبت باستخدام واجهة واضحة وبسيطة. جربها!

### إعداد Visual Studio 2022

لـ Visual Studio 2022 (الإصدار 17.14 أو الأحدث):

1. **تمكين وضع الوكيل**: انقر على قائمة "اسأل" المنسدلة في نافذة محادثة GitHub Copilot واختر "وكيل"
2. **إنشاء ملف التكوين**: أنشئ ملف `.mcp.json` في دليل الحل (الموقع الموصى به: `<SOLUTIONDIR>\.mcp.json`)
3. **تكوين الخوادم**: أضف تكوينات خوادم MCP باستخدام تنسيق MCP القياسي
4. **موافقة الأدوات**: عندما يُطلب منك، وافق على الأدوات التي تريد استخدامها مع أذونات المجال المناسبة

للحصول على تعليمات إعداد Visual Studio مفصلة، راجع [توثيق MCP لـ Visual Studio](https://learn.microsoft.com/visualstudio/ide/mcp-servers).

كل خادم MCP يأتي مع متطلبات تكوين خاصة به (سلاسل الاتصال، المصادقة، الخ)، لكن نمط الإعداد متسق عبر بيئتي التطوير.

### 1. 📚 خادم Microsoft Learn Docs MCP

[![التثبيت في VS Code](https://img.shields.io/badge/VS_Code-Install_Microsoft_Docs_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D) [![التثبيت في VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Microsoft_Docs_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=microsoft.docs.mcp&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Flearn.microsoft.com%2Fapi%2Fmcp%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/mcp)

**ما يفعله**: خادم Microsoft Learn Docs MCP هو خدمة مستضافة سحابياً توفر لمساعدي الذكاء الاصطناعي وصولًا في الوقت الحقيقي إلى الوثائق الرسمية لمايكروسوفت من خلال بروتوكول نموذج السياق. يتصل بـ `https://learn.microsoft.com/api/mcp` ويُمكّن البحث الدلالي عبر Microsoft Learn، وثائق Azure، وثائق Microsoft 365، ومصادر رسمية أخرى لمايكروسوفت.

**لماذا هو مفيد**: بينما قد يبدو كـ"مجرد وثائق"، فإن هذا الخادم ضروري لكل مطور يستخدم تقنيات مايكروسوفت. إحدى أكبر شكاوى مطوري .NET حول مساعدين البرمجة بالذكاء الاصطناعي هي أنهم ليسوا محدثين بأحدث إصدارات .NET و C#. يحل خادم Microsoft Learn Docs MCP هذه المشكلة من خلال توفير وصول في الوقت الحقيقي لأحدث الوثائق، مراجع API، وأفضل الممارسات. سواء كنت تعمل مع أحدث SDKs لـ Azure، تستكشف ميزات C# 13 الجديدة، أو تنفذ أنماط Aspire المتطورة في .NET، يضمن هذا الخادم أن يكون لدى مساعد الذكاء الاصطناعي الخاص بك وصول للمعلومات الرسمية والمحدثة التي يحتاجها لتوليد كود دقيق وحديث.

**الاستخدام في العالم الحقيقي**: "ما هي أوامر az cli لإنشاء تطبيق حاوية Azure وفقًا للوثائق الرسمية لمايكروسوفت؟" أو "كيف أُركّب Entity Framework مع حقن التبعية في ASP.NET Core؟" أو ماذا عن "راجع هذا الكود للتأكد من مطابقته لتوصيات الأداء في وثائق Microsoft Learn." يوفر الخادم تغطية شاملة عبر Microsoft Learn، وثائق Azure، ووثائق Microsoft 365 باستخدام بحث دلالي متقدم للعثور على المعلومات الأكثر ملاءمة للسياق. يعيد ما يصل إلى 10 قطع محتوى عالية الجودة مع عناوين المقالات وروابطها، دائمًا بالوصول إلى أحدث الوثائق الخاصة بمايكروسوفت بمجرد نشرها.

**مثال مميز**: يعرض الخادم أداة `microsoft_docs_search` التي تقوم ببحث دلالي ضد الوثائق التقنية الرسمية لمايكروسوفت. بمجرد التكوين، يمكنك طرح أسئلة مثل "كيف أُنفذ مصادقة JWT في ASP.NET Core؟" والحصول على ردود رسمية مفصلة مع روابط المصادر. جودة البحث استثنائية لأنه يفهم السياق – الاستعلام عن "الحاويات" في سياق Azure سيُرجع وثائق Azure Container Instances، بينما نفس المصطلح في سياق .NET يُرجع معلومات مجمعة لـ C# ذات صلة.

هذا مفيد بشكل خاص للمكتبات وحالات الاستخدام المتغيرة بسرعة أو التي تم تحديثها مؤخرًا. على سبيل المثال، في بعض مشاريع البرمجة الحديثة أردت الاستفادة من ميزات إصدارات .NET Aspire و Microsoft.Extensions.AI الأخيرة. من خلال تضمين خادم Microsoft Learn Docs MCP، تمكنت من الاستفادة ليس فقط من وثائق API، بل من الشروحات والإرشادات التي تم نشرها للتو.

> **💡 نصيحة احترافية**
> 
> حتى النماذج المُصممة لاستخدام الأدوات تحتاج إلى تحفيز لاستخدام أدوات MCP! فكر في إضافة مطالبة نظام أو [copilot-instructions.md](https://docs.github.com/copilot/how-tos/custom-instructions/adding-repository-custom-instructions-for-github-copilot) مثل: "لديك وصول إلى `microsoft.docs.mcp` – استخدم هذه الأداة للبحث في أحدث الوثائق الرسمية لمايكروسوفت عند التعامل مع أسئلة عن تقنيات مايكروسوفت مثل C#، Azure، ASP.NET Core، أو Entity Framework."
>
> لمثال رائع على ذلك، تحقق من وضع محادثة [مدبر C# .NET Janitor](https://github.com/awesome-copilot/chatmodes/blob/main/csharp-dotnet-janitor.chatmode.md) من مستودع Awesome GitHub Copilot. يستفيد هذا الوضع تحديدًا من خادم Microsoft Learn Docs MCP للمساعدة في تنظيف وتحديث كود C# باستخدام أحدث الأنماط وأفضل الممارسات.

### 2. ☁️ خادم Azure MCP
[![التثبيت في VS Code](https://img.shields.io/badge/VS_Code-Install_Azure_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D) [![التثبيت في VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Azure_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=Azure%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40azure%2Fazure-mcp%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/Azure/azure-mcp)

**ما يقوم به**: خادم Azure MCP هو مجموعة شاملة من أكثر من 15 موصل خدمة Azure متخصصة تجلب النظام البيئي الكامل لـ Azure إلى سير عمل الذكاء الاصطناعي الخاص بك. هذا ليس مجرد خادم واحد – بل هو مجموعة قوية تشمل إدارة الموارد، الاتصال بقاعدة البيانات (PostgreSQL, SQL Server)، تحليل سجلات Azure Monitor باستخدام KQL، تكامل Cosmos DB، وأكثر من ذلك بكثير.

**لماذا هو مفيد**: إلى جانب إدارة موارد Azure، يحسن هذا الخادم بشكل كبير جودة الشيفرة عند العمل مع Azure SDKs. عندما تستخدم Azure MCP في وضع الوكيل، فإنه لا يساعدك فقط على كتابة الشيفرة – بل يساعدك على كتابة شيفرة Azure *أفضل* تتبع أنماط المصادقة الحالية وأفضل ممارسات التعامل مع الأخطاء، وتستفيد من أحدث ميزات SDK. بدلاً من الحصول على شيفرة عامة قد تعمل، تحصل على شيفرة تتبع أنماط Azure الموصى بها للأحمال الإنتاجية.

**الوحدات الرئيسية تشمل**:
- **🗄️ موصلات قواعد البيانات**: وصول مباشر بلغة طبيعية إلى Azure Database لـ PostgreSQL وSQL Server
- **📊 Azure Monitor**: تحليل السجلات مطور بواسطة KQL ورؤى تشغيلية
- **🌐 إدارة الموارد**: إدارة كاملة لدورة حياة موارد Azure
- **🔐 المصادقة**: نماذج DefaultAzureCredential والهوية المُدارة
- **📦 خدمات التخزين**: عمليات Blob Storage وQueue Storage وTable Storage
- **🚀 خدمات الحاويات**: إدارة Azure Container Apps وContainer Instances وAKS
- **وعديد من الموصلات المتخصصة الأخرى**

**الاستخدام العملي**: "اعرض لي حسابات التخزين الخاصة بي في Azure"، "استعلم من مساحة عمل Log Analytics عن الأخطاء في الساعة الماضية"، أو "ساعدني في بناء تطبيق Azure باستخدام Node.js مع مصادقة صحيحة"

**سيناريو العرض الكامل**: إليك شرح كامل يوضح قوة الجمع بين Azure MCP مع امتداد GitHub Copilot for Azure في VS Code. عندما يكون لديك كلاهما مثبتًا وتُعطي الطلب:

> "أنشئ سكريبت Python يرفع ملفًا إلى Azure Blob Storage باستخدام مصادقة DefaultAzureCredential. يجب أن يتصل السكريبت بحسابي التخزيني في Azure المسمى 'mycompanystorage'، ويرفع إلى حاوية باسم 'documents'، ينشئ ملف اختبار مع طابع زمني حالي للرفع، يتعامل مع الأخطاء بسلاسة ويوفر مخرجات إعلامية، يتبع أفضل ممارسات Azure للمصادقة والتعامل مع الأخطاء، يتضمن تعليقات تشرح كيفية عمل مصادقة DefaultAzureCredential، ويرتب السكريبت بشكل جيد مع دوال ووثائق مناسبة."

سيولد خادم Azure MCP سكريبت Python كامل وجاهز للإنتاج يقوم بـ:
- استخدام أحدث SDK لـ Azure Blob Storage مع أنماط async الصحيحة
- تطبيق DefaultAzureCredential مع شرح شامل لسلسلة التراجع
- تضمين معالجة أخطاء قوية مع أنواع استثناءات Azure المحددة
- اتباع أفضل ممارسات SDK الخاصة بـ Azure لإدارة الموارد والتعامل مع الاتصالات
- توفير تسجيل مفصل ومخرجات إعلامية في الكونسول
- إنشاء سكريبت منظم جيدًا مع دوال، توثيق، وتلميحات نوع

ما يجعل هذا مميزًا هو أنه بدون Azure MCP قد تحصل على شيفرة عامة لتخزين الـ blob تعمل لكنها لا تتبع أنماط Azure الحالية. مع Azure MCP، تحصل على شيفرة تستفيد من أحدث طرق المصادقة، تتعامل مع سيناريوهات أخطاء Azure الخاصة، وتتبع ممارسات Microsoft الموصى بها لتطبيقات الإنتاج.

**مثال مميز**: كنت أواجه صعوبة في تذكر الأوامر الخاصة بـ `az` و`azd` لاستخدامها العشوائي. دائمًا ما تكون عملية من خطوتين بالنسبة لي: أولًا البحث عن الصياغة، ثم تنفيذ الأمر. غالبًا ما أدخل إلى البوابة وانقر لإنجاز العمل لأنني لا أريد الاعتراف بأنني لا أتذكر صياغة CLI. القدرة على وصف ما أريده فقط شيء مذهل، والأفضل هو أن أتمكن من فعل ذلك دون مغادرة بيئة تطويري!

هناك قائمة رائعة بحالات الاستخدام في [مستودع Azure MCP](https://github.com/Azure/azure-mcp?tab=readme-ov-file#-what-can-you-do-with-the-azure-mcp-server) لتبدأ منها. للإرشادات الشاملة للإعداد وخيارات التكوين المتقدمة، راجع [توثيق Azure MCP الرسمي](https://learn.microsoft.com/azure/developer/azure-mcp-server/).

### 3. 🐙 خادم GitHub MCP

[![التثبيت في VS Code](https://img.shields.io/badge/VS_Code-Install_Server-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=github&config=%7B%22type%22%3A%20%22http%22%2C%22url%22%3A%20%22https%3A%2F%2Fapi.githubcopilot.com%2Fmcp%2F%22%7D) [![التثبيت في VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_Server-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=github&config=%7B%22type%22%3A%20%22http%22%2C%22url%22%3A%20%22https%3A%2F%2Fapi.githubcopilot.com%2Fmcp%2F%22%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/github/github-mcp-server)

**ما يقوم به**: يوفر خادم GitHub MCP الرسمي تكاملًا سلسًا مع النظام البيئي الكامل لـ GitHub، ويقدم خيارات وصول عن بُعد مستضافة وخيارات نشر محلي باستخدام Docker. هذا ليس فقط عن العمليات الأساسية للمستودعات – بل مجموعة أدوات شاملة تشمل إدارة GitHub Actions، سير العمل لطلبات السحب، تتبع القضايا، فحص الأمان، الإشعارات، وقدرات الأتمتة المتقدمة.

**لماذا هو مفيد**: يحول هذا الخادم طريقة تفاعلك مع GitHub من خلال جلب تجربة المنصة الكاملة مباشرة إلى بيئة تطويرك. بدلاً من التبديل المستمر بين VS Code وGitHub.com لإدارة المشاريع، مراجعات الشيفرة، ومراقبة CI/CD، يمكنك التعامل مع كل شيء عبر أوامر اللغة الطبيعية مع البقاء مركزًا على الشيفرة الخاصة بك.

> **ℹ️ ملاحظة: أنواع مختلفة من 'الوكلاء'**
> 
> لا تخلط بين هذا الخادم GitHub MCP ووكيل الترميز الخاص بـ GitHub (وهو الوكيل AI الذي يمكن تعيين القضايا إليه لتنفيذ مهام الترميز تلقائيًا). يعمل خادم GitHub MCP داخل وضع الوكيل في VS Code لتوفير تكامل API الخاص بـ GitHub، بينما وكيل الترميز الخاص بـ GitHub هو ميزة منفصلة تنشئ طلبات سحب عند تعيينها لقضايا GitHub.

**القدرات الرئيسية تشمل**:
- **⚙️ GitHub Actions**: إدارة كاملة لأنابيب CI/CD، مراقبة سير العمل، معالجة التحف الفنية
- **🔀 طلبات السحب**: إنشاء، مراجعة، دمج، وإدارة طلبات السحب مع تتبع الحالة بشكل شامل
- **🐛 القضايا**: إدارة دورة حياة القضايا بالكامل، التعليق، الوسم، والتعيين
- **🔒 الأمان**: تنبيهات فحص الشيفرة، كشف الأسرار، وتكامل Dependabot
- **🔔 الإشعارات**: إدارة ذكية للإشعارات والتحكم في الاشتراك بالمستودعات
- **📁 إدارة المستودعات**: عمليات الملفات، إدارة الفروع، وإدارة المستودع
- **👥 التعاون**: البحث عن المستخدمين والمنظمات، إدارة الفرق، والتحكم في الوصول

**الاستخدام العملي**: "أنشئ طلب سحب من فرع الميزات الخاص بي"، "اعرض لي كل عمليات CI الفاشلة هذا الأسبوع"، "سرد التنبيهات الأمنية المفتوحة لمستودعاتي"، أو "ابحث عن كل القضايا المعينة لي عبر منظماتي"

**سيناريو العرض الكامل**: إليك سير عمل قوي يبرز قدرات خادم GitHub MCP:

> "أحتاج للتحضير لمراجعة السبرينت لدينا. اعرض لي كل طلبات السحب التي أنشأتها هذا الأسبوع، تحقق من حالة أنابيب CI/CD الخاصة بنا، أنشئ ملخصًا لأي تنبيهات أمان يجب معالجتها، وساعدني في صياغة ملاحظات الإصدار بناءً على طلبات السحب المدمجة الموسومة بـ 'feature'."

سيقوم خادم GitHub MCP بـ:
- استعلام طلبات السحب الأخيرة مع معلومات حالة مفصلة
- تحليل سير العمل وإبراز أي إخفاقات أو مشكلات في الأداء
- تجميع نتائج فحص الأمان وترتيب التنبيهات الحرجة
- إنشاء ملاحظات إصدار شاملة باستخلاص المعلومات من طلبات السحب المدمجة
- تقديم خطوات قابلة للتنفيذ للتخطيط للسبرينت والتحضير للإصدار

**مثال مميز**: أحب استخدام هذا لسير عمل مراجعة الشيفرة البرمجية. بدلاً من التنقل بين VS Code، إشعارات GitHub، وصفحات طلبات السحب، يمكنني القول "اعرض لي كل طلبات السحب التي تنتظر مراجعتي" ثم "أضف تعليقًا على PR #123 يسأل عن معالجة الخطأ في طريقة المصادقة." يتولى الخادم مكالمات API الخاصة بـ GitHub، يحتفظ بسياق المناقشة، ويساعدني حتى في صياغة تعليقات مراجعة أكثر بناءً.

**خيارات المصادقة**: يدعم الخادم كل من OAuth (بشكل سلس في VS Code) ورموز الوصول الشخصية (PAT)، مع مجموعات أدوات قابلة للتكوين لتمكين فقط وظائف GitHub التي تحتاجها. يمكنك تشغيله كخدمة مستضافة عن بُعد للإعداد الفوري أو محليًا عبر Docker للتحكم الكامل.

> **💡 نصيحة احترافية**
> 
> فعّل فقط مجموعات الأدوات التي تحتاجها من خلال تكوين معامل `--toolsets` في إعدادات خادم MCP الخاص بك لتقليل حجم السياق وتحسين اختيار أدوات AI. على سبيل المثال، أضف `"--toolsets", "repos,issues,pull_requests,actions"` إلى معاملات إعداد MCP الخاصة بك لسير العمل الأساسي للتطوير، أو استخدم `"--toolsets", "notifications, security"` إذا كنت تركز أساسًا على قدرات مراقبة GitHub.

### 5. 📝 خادم MarkItDown MCP
[![التثبيت في VS Code](https://img.shields.io/badge/VS_Code-Install_MarkItDown_MCP-0098FF?style=flat-square&logo=visualstudiocode&logoColor=white)](https://vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D) [![التثبيت في VS Code Insiders](https://img.shields.io/badge/VS_Code_Insiders-Install_MarkItDown_MCP-24bfa5?style=flat-square&logo=visualstudiocode&logoColor=white)](https://insiders.vscode.dev/redirect/mcp/install?name=MarkItDown%20MCP&config=%7B%22command%22%3A%22npx%22%2C%22args%22%3A%5B%22-y%22%2C%22%40microsoft%2Fmcp-markitdown%40latest%22%5D%7D&quality=insiders) [![GitHub](https://img.shields.io/badge/GitHub-View_Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/microsoft/markitdown)

**ما الذي يفعله**: MarkItDown هو خادم تحويل مستندات شامل يحول تنسيقات الملفات المختلفة إلى لغة Markdown عالية الجودة، محسّنة للاستهلاك من قِبل نماذج اللغة الكبيرة (LLM) وسير عمل تحليل النصوص.

**لماذا هو مفيد**: ضروري لسير عمل التوثيق الحديث! يتعامل MarkItDown مع مجموعة مثيرة للإعجاب من تنسيقات الملفات مع الحفاظ على الهيكل الحاسم للوثيقة مثل العناوين والقوائم والجداول والروابط. وعلى عكس أدوات استخراج النصوص البسيطة، يركز على الحفاظ على المعنى الدلالي والتنسيق الذي يمثل قيمة لكل من المعالجة بالذكاء الاصطناعي وقابلية القراءة البشرية.

**تنسيقات الملفات المدعومة**:
- **مستندات Office**: PDF، PowerPoint (PPTX)، Word (DOCX)، Excel (XLSX/XLS)
- **ملفات الوسائط**: الصور (مع بيانات EXIF وOCR)، الصوت (مع بيانات EXIF ونصوص الكلام)
- **محتوى الويب**: HTML، خلاصة RSS، عناوين YouTube، صفحات ويكيبيديا
- **تنسيقات البيانات**: CSV، JSON، XML، ملفات ZIP (تعالج المحتويات بشكل متكرر)
- **تنسيقات النشر**: EPub، دفاتر Jupyter (.ipynb)
- **البريد الإلكتروني**: رسائل Outlook (.msg)
- **متقدم**: تكامل مع Azure Document Intelligence لمعالجة PDF المعززة

**القدرات المتقدمة**: يدعم MarkItDown وصف الصور المدعوم بنماذج اللغة الكبيرة (عند توفير عميل OpenAI)، وAzure Document Intelligence لتحسين معالجة PDF، وترجمة نصوص الصوت للمحتوى الصوتي، ونظام الإضافات لتمديد الدعم إلى المزيد من تنسيقات الملفات.

**الاستخدام العملي**: "حوّل هذا العرض التقديمي في PowerPoint إلى Markdown لموقع التوثيق الخاص بنا"، "استخراج النص من هذا PDF مع الحفاظ على هيكل العناوين الصحيح"، أو "تحويل جدول Excel إلى شكل جدول قابل للقراءة"

**مثال مميز**: للاقتباس من [وثائق MarkItDown](https://github.com/microsoft/markitdown#why-markdown):

> Markdown قريب جدًا من النص العادي مع أقل قدر من العلامات أو التنسيق، لكنه ما زال يوفر طريقة لتمثيل هيكل الوثيقة المهم. النماذج الرئيسية للغة الكبيرة، مثل GPT-4o من OpenAI، "تتحدث" Markdown بشكل طبيعي، وغالبًا ما تدمج Markdown في ردودها بدون طلب. وهذا يشير إلى أنها تم تدريبها على كميات هائلة من النصوص المنسقة بـ Markdown، وتفهمها جيدًا. كفائدة جانبية، فإن قواعد Markdown فعالة أيضًا من حيث عدد الرموز المستخدمة.

MarkItDown ماهر جدًا في الحفاظ على هيكل الوثيقة، وهو أمر مهم لسير عمل الذكاء الاصطناعي. على سبيل المثال، عند تحويل عرض تقديمي في PowerPoint، يحتفظ بتنظيم الشرائح مع العناوين الصحيحة، يستخرج الجداول كجداول Markdown، يتضمن نصًا بديلًا للصور، ويعالج حتى ملاحظات المتحدث. تتحول الرسوم البيانية إلى جداول بيانات قابلة للقراءة، ويحافظ Markdown الناتج على التدفق المنطقي للعرض التقديمي الأصلي. هذا يجعله مثاليًا لإدخال محتوى العرض التقديمي في أنظمة الذكاء الاصطناعي أو إنشاء توثيق من الشرائح الموجودة.

**ما الذي يفعله**: يوفر خادم Microsoft Foundry MCP للمطورين وصولاً شاملاً إلى نظام Azure للذكاء الاصطناعي، بما في ذلك فهارس النماذج، وإدارة النشر، وفهرسة المعرفة باستخدام Azure AI Search، وأدوات التقييم. يجسر هذا الخادم التجريبي الفجوة بين تطوير الذكاء الاصطناعي والبنية التحتية القوية لـ Azure AI، مما يسهل بناء ونشر وتقييم تطبيقات الذكاء الاصطناعي.

**لماذا هو مفيد**: يحول هذا الخادم طريقة عملك مع خدمات Azure AI من خلال جلب قدرات الذكاء الاصطناعي على مستوى المؤسسات مباشرة إلى سير عمل التطوير الخاص بك. بدلاً من التنقل بين بوابة Azure والتوثيق وبيئة التطوير المتكاملة، يمكنك اكتشاف النماذج، ونشر الخدمات، وإدارة قواعد المعرفة، وتقييم أداء الذكاء الاصطناعي عبر أوامر اللغة الطبيعية. إنه قوي بشكل خاص للمطورين الذين يبنون تطبيقات RAG (التوليد المعزز بالاسترجاع)، أو يديرون نشرات متعددة للنماذج، أو ينفذون خطوط تقييم شاملة للذكاء الاصطناعي.

**القدرات الرئيسية للمطورين**:
- **🔍 اكتشاف النماذج والنشر**: استكشف فهرس نماذج Microsoft Foundry، احصل على معلومات تفصيلية عن النماذج مع عينات الشيفرات، وانشر النماذج إلى خدمات Azure AI
- **📚 إدارة المعرفة**: أنشئ ونم إدارة فهارس Azure AI Search، أضف مستندات، هيئ الفهارس، وابنِ أنظمة RAG متقدمة
- **⚡ دمج وكلاء الذكاء الاصطناعي**: اتصل بوكلاء Azure AI، استعلم عن الوكلاء الحاليين، وقيم أداء الوكلاء في سيناريوهات الإنتاج
- **📊 إطار التقييم**: نفذ تقييمات شاملة للنصوص والوكلاء، أنشئ تقارير بتنسيق Markdown، وطبق ضمان الجودة لتطبيقات الذكاء الاصطناعي
- **🚀 أدوات النماذج الأولية**: احصل على تعليمات الإعداد للنماذج الأولية القائمة على GitHub وادخل إلى Microsoft Foundry Labs لنماذج البحث المتقدمة

**الاستخدام العملي للمطورين**: "انشر نموذج Phi-4 إلى خدمات Azure AI لتطبيقي"، "أنشئ فهرس بحث جديد لنظام RAG الخاص بوثائقاتي"، "قيم استجابات الوكيل الخاص بي مقابل مقاييس الجودة"، أو "ابحث عن أفضل نموذج استدلال لمهام التحليل المعقدة الخاصة بي"

سيقوم خادم Microsoft Foundry MCP بـ:

- توفير أوامر النشر ومعلومات الحصة لمنطقتك المفضلة في Azure
- إعداد فهارس Azure AI Search مع مخطط مناسب لوثائقك
- تكوين خطوط تقييم باستخدام مقاييس الجودة وفحوصات السلامة
- إنشاء رمز النمذجة الأولية مع مصادقة GitHub للاختبار الفوري
- توفير أدلة إعداد شاملة مصممة خصيصًا لمكدس التكنولوجيا الخاص بك

**مثال مميز**: كمطور، واجهت صعوبة في متابعة نماذج LLM المختلفة المتاحة. أعرف بعض النماذج الرئيسية، لكن كنت أشعر أنني أفتقد بعض مكاسب الإنتاجية والكفاءة. والتوكنات والحصص مصدر ضغط وصعب الإدراة – لا أعلم أبدًا إذا كنت أختار النموذج المناسب للمهمة المناسبة أو أحرق الميزانية بطريقة غير فعالة. سمعت للتو عن هذا الخادم MCP من جيمس مونتماغنو عندما كنت أتحقق مع زملائي من توصيات خادم MCP لهذا المنشور، وأنا متحمس لاستخدامه! قدرات اكتشاف النماذج تبدو مثيرة للإعجاب لشخص مثلي يريد استكشاف ما وراء النماذج المعتادة والعثور على نماذج محسنة لمهام محددة. يجب أن يساعدني إطار التقييم في التحقق من أنني فعلاً أحصل على نتائج أفضل، وليس فقط أجرب شيئًا جديدًا لمجرد التجربة.

## 📋 الخاتمة

يغير بروتوكول نموذج السياق (MCP) طريقة تفاعل المطورين مع مساعدين الذكاء الاصطناعي والأدوات الخارجية. تعرض هذه الخوادم العشرة لـ Microsoft MCP قوة التكامل الموحد للذكاء الاصطناعي، مما يمكن من سير عمل سلس يحافظ على تركيز المطورين أثناء الوصول إلى قدرات خارجية قوية.

من التكامل الشامل لنظام Azure البيئي إلى الأدوات المتخصصة مثل Playwright لأتمتة المتصفح و MarkItDown لمعالجة الوثائق، تعرض هذه الخوادم كيف يمكن لـ MCP تعزيز الإنتاجية عبر سيناريوهات تطوير متنوعة. يضمن البروتوكول الموحد عمل هذه الأدوات معًا بسلاسة، مما يخلق تجربة تطوير متماسكة.

مع استمرار تطور نظام MCP البيئي، سيكون البقاء منخرطًا مع المجتمع، واستكشاف الخوادم الجديدة، وبناء حلول مخصصة مفتاحًا لتعظيم إنتاجيتك في التطوير. الطبيعة المفتوحة المعيارية لـ MCP تعني أنه يمكنك مزج الأدوات من بائعين مختلفين لإنشاء سير العمل المثالي لاحتياجاتك الخاصة.

2. **تكامل سير العمل**: صمم سير عمل تطوير يجمع بين ثلاثة خوادم MCP مختلفة على الأقل.
3. **تخطيط خادم مخصص**: حدد مهمة في روتين تطويرك اليومي يمكن أن تستفيد من خادم MCP مخصص وأنشئ مواصفات لها.
4. **تحليل الأداء**: قارن كفاءة استخدام خوادم MCP مقابل الأساليب التقليدية للمهام التطويرية الشائعة.
5. **تقييم الأمان**: قيّم التداعيات الأمنية لاستخدام خوادم MCP في بيئة تطويرك واقترح أفضل الممارسات.


التالي: [أفضل الممارسات](../08-BestPractices/README.md)
