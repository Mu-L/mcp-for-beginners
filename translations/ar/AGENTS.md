# AGENTS.md

## نظرة عامة على المشروع

**MCP للمبتدئين** هو منهج تعليمي مفتوح المصدر لتعلم بروتوكول سياق النموذج (MCP) - إطار عمل موحد للتفاعل بين نماذج الذكاء الاصطناعي وتطبيقات العميل. يوفر هذا المستودع مواد تعليمية شاملة مع أمثلة عملية للكود عبر لغات برمجة متعددة.

### التقنيات الرئيسية

- **لغات البرمجة**: C#، Java، JavaScript، TypeScript، Python، Rust  
- **الأُطُر ومجموعات تطوير البرمجيات**:  
  - MCP SDK (`@modelcontextprotocol/sdk`)  
  - Spring Boot (جافا)  
  - FastMCP (بايثون)  
  - LangChain4j (جافا)  
- **قواعد البيانات**: PostgreSQL مع امتداد pgvector  
- **منصات السحابة**: أزور (تطبيقات الحاويات، OpenAI، أمان المحتوى، Application Insights)  
- **أدوات البناء**: npm، Maven، pip، Cargo  
- **التوثيق**: ماركداون مع ترجمة تلقائية متعددة اللغات (أكثر من 48 لغة)

### البنية المعمارية

- **11 وحدة أساسية (00-11)**: مسار تعلم متسلسل من الأساسيات إلى الموضوعات المتقدمة  
- **مختبرات عملية**: تمارين عملية مع كود حلول كاملة بلغات متعددة  
- **مشاريع نموذجية**: تطبيقات MCP لخادم وعميل تعمل بشكل فعلي  
- **نظام الترجمة**: سير عمل تلقائي عبر GitHub Actions لدعم متعدد اللغات  
- **أصول الصور**: مجلد مركزي للصور مع نسخ مترجمة

## أوامر الإعداد

هذا المستودع يركز على التوثيق. معظم الإعداد يتم داخل مشاريع ومختبرات العينة الفردية.

### إعداد المستودع

```bash
# استنساخ المستودع
git clone https://github.com/microsoft/mcp-for-beginners.git
cd mcp-for-beginners
```

### العمل مع المشاريع النموذجية

تقع المشاريع النموذجية في:  
- `03-GettingStarted/samples/` - أمثلة خاصة بكل لغة  
- `03-GettingStarted/01-first-server/solution/` - تطبيقات الخادم الأولى  
- `03-GettingStarted/02-client/solution/` - تطبيقات العميل  
- `11-MCPServerHandsOnLabs/` - مختبرات تكامل قاعدة البيانات الشاملة  

كل مشروع نموذجي يحتوي على تعليمات إعداد خاصة به:

#### مشاريع TypeScript/JavaScript
```bash
cd <project-directory>
npm install
npm start
```

#### مشاريع Python
```bash
cd <project-directory>
pip install -r requirements.txt
# أو
pip install -e .
python main.py
```

#### مشاريع Java
```bash
cd <project-directory>
mvn clean install
mvn spring-boot:run
```

## سير عمل التطوير

### هيكل التوثيق

- **الوحدات 00-11**: محتوى المنهج الأساسي بالترتيب التسلسلي  
- **translations/**: نسخ بلغات مختلفة (توليد تلقائي، لا تعدل مباشرة)  
- **translated_images/**: نسخ الصور المحلية (توليد تلقائي)  
- **images/**: الصور الأصلية والرسومات

### إجراء تغييرات على التوثيق

1. حرر فقط ملفات الماركداون الإنجليزية في مجلدات الوحدات الرئيسية (00-11)  
2. حدث الصور في مجلد `images/` عند الحاجة  
3. سيقوم سير العمل co-op-translator بـ GitHub Actions بتوليد الترجمات تلقائياً  
4. تتم إعادة توليد الترجمات مع كل دفع على فرع main

### العمل مع الترجمات

- **الترجمة التلقائية**: سير عمل GitHub Actions يدير جميع الترجمات  
- **لا تقم بتعديل** الملفات في مجلد `translations/` يدوياً  
- بيانات وصفية للترجمة مضمنة في كل ملف مترجم  
- اللغات المدعومة: أكثر من 48 لغة تشمل العربية، الصينية، الفرنسية، الألمانية، الهندية، اليابانية، الكورية، البرتغالية، الروسية، الإسبانية، والكثير غيرها

## إرشادات نمط الكتابة

### نمط التوثيق

- استخدم لغة واضحة ومناسبة للمبتدئين  
- ضمن أمثلة الكود في لغات متعددة حيثما أمكن  
- اتبع أفضل ممارسات الماركداون:  
  - استخدم رؤوس ATX (`#`)  
  - استخدم كتل الكود المحاطة مع معرفات للغة  
  - أدرج نصًا بديلاً وصفيًا للصور  
  - اجعل طول الأسطر معقولًا (لا حد صارم، لكن تعقل)

### نمط أمثلة الكود

#### TypeScript/JavaScript
- استخدم وحدات ES (`import`/`export`)  
- اتبع قواعد الوضع الصارم لـ TypeScript  
- أدرج توضيحات النوع  
- استهدف ES2022  

#### Python
- اتبع معايير أسلوب PEP 8  
- استخدم تلميحات النوع عند الاقتضاء  
- ضمّن docstrings للدوال والفئات  
- استخدم ميزات بايثون الحديثة (3.8+)  

#### Java
- اتبع قواعد Spring Boot  
- استخدم ميزات Java 21  
- اتبع هيكل مشروع Maven القياسي  
- أدرج تعليقات Javadoc  

### تنظيم الملفات

```
<module-number>-<ModuleName>/
├── README.md              # Main module content
├── samples/               # Code examples (if applicable)
│   ├── typescript/
│   ├── python/
│   ├── java/
│   └── ...
└── solution/              # Complete working solutions
    └── <language>/
```

## البناء والنشر

### نشر التوثيق

يستخدم المستودع خدمات مثل GitHub Pages أو ما شابه لاستضافة التوثيق (إن وجدت). يؤدي التغيير على الفرع الرئيسي إلى:

1. تشغيل سير عمل الترجمة (`.github/workflows/co-op-translator.yml`)  
2. الترجمة التلقائية لكل ملفات الماركداون الإنجليزية  
3. توطين الصور عند الحاجة  

### لا حاجة لعملية بناء

يحتوي المستودع بشكل أساسي على توثيق ماركداون. لا يتطلب تجميع أو خطوة بناء لمحتوى المنهج الأساسي.

### نشر المشاريع النموذجية

قد تحتوي المشاريع النموذجية على تعليمات نشر خاصة بها:  
- راجع `03-GettingStarted/09-deployment/` لإرشادات نشر خادم MCP  
- أمثلة نشر تطبيقات الحاويات على Azure في `11-MCPServerHandsOnLabs/`

## إرشادات المساهمة

### عملية طلب السحب (Pull Request)

1. **استنساخ الشجرة**: قم بعمل فورك للمستودع واستنساخ نسخة المحلّي  
2. **إنشاء فرع**: استخدم أسماء فرعية وصفية (مثل `fix/typo-module-3`، `add/python-example`)  
3. **قم بالتعديلات**: حرر فقط ملفات الماركداون الإنجليزية (لا تعدل الترجمات)  
4. **اختبر محليًا**: تحقق من عرض الماركداون بشكل صحيح  
5. **قدم طلب السحب**: استخدم عناوين ووصفاً واضحاً  
6. **اتفاقية المساهمة**: وقع اتفاقية مساهمة مايكروسوفت عندما يُطلب منك ذلك  

### تنسيق عنوان طلب السحب

استخدم عناوين واضحة ووصفية:  
- `[Module XX] وصف موجز` لتغييرات الوحدة المحددة  
- `[Samples] وصف` لتغييرات أمثلة الكود  
- `[Docs] وصف` لتحديثات التوثيق العامة  

### ماذا تساهم

- إصلاحات الأخطاء في التوثيق أو أمثلة الكود  
- إضافة أمثلة كود جديدة بلغات إضافية  
- توضيحات وتحسينات للمحتوى الحالي  
- دراسات حالة جديدة أو أمثلة عملية  
- تقارير عن مشكلات محتوى غير واضحة أو غير صحيحة  

### ما لا تفعله

- لا تعدل ملفات داخل مجلد `translations/` مباشرة  
- لا تعدل مجلد `translated_images/`  
- لا تضف ملفات ثنائية كبيرة بدون مناقشة  
- لا تغير ملفات سير عمل الترجمة بدون تنسيق
